想成为牛逼程序员？先数数编程语言之父们的头发

事实是否如上所说的那样呢?那就不得而知了，不过深入了解了解那些编程语言之父们头顶的雨林是否茂盛，以及他们的丰功伟绩，没准能得出什么参考标准呢。

01.Java 之父 James Gosling

头顶茂密指数：★★

第一位出场的就是我们大名鼎鼎的 Java 之父，James Gosling，看其光滑的头顶，一眼就看出平常肯定没少用脑。毕竟人家12岁的时候就已经能设计电子游戏机，说他是一位计算机编程天才一点也不为过。但是看到他的头顶莫名的心疼他 3 秒钟。

还是把重心放在他的成就上，首先看看他发明的语言 Java ，全球所有顶级的、核心的项目，绝大部分都是基于Java 做的，即使不是完全基于 Java，也和 Java 有千丝万缕的关系。

再来看看高斯林的事业发展，前不久宣布，他将加入亚马逊云计算部门。他在 Liquid Robotics 公司负责水下机器人的软件设计工作。与高斯林共事的还有很多技术大牛，包括：微服务架构专家阿德里安-考克罗夫(Adrian Cockroft)、人工智能副总裁斯瓦米-西瓦苏布拉马尼娜(Swami Sivasubramanian)和首席技术官沃纳-威格尔(Verner Vogels)。

02.C语言之父 丹尼斯·里奇

头顶茂密指数：★★★

从这张照片的发型可以看出咱 C 语言之父很心机的把有些“空地”盖住了，不过整体看起来还算过得去。

C 语言之父丹尼斯·里奇对 C 语言和其他编程语言、 Multics 和 Unix 等操作系统的发展做出了巨大贡献。2011年10月与乔布斯相继离世。

麻省理工大学计算机系的马丁教授评价这样他的：如果说，乔布斯是可视化产品中的国王，那么里奇就是不可见王国中的君主。乔布斯的贡献在于，他如此了解用户的需求和渴求，以至于创造出了让当代人乐不思蜀的科技产品。然而，却是里奇先生为这些产品提供了最核心的部件，人们看不到这些部件，却每天都在使用着。

03.C++之父 Bjarne Stroustrup

头顶茂密指数：★★

Bjarne Stroustrup(本贾尼·斯特劳斯特卢普)博士的发量和Java之父可以相似度99%，先后毕业于丹麦阿鲁斯大学和英国剑桥大学的Bjarne Stroustrup ，在1979年，开始开发一种语言，当时称为“C with Class”，后来为了表达该语言与 C语言的渊源关系，它被命名为C++

此前人们曾猜想 C++ 最终将一统天下，然而随着 Internet 的爆炸性增长没能实现，但是现在的 C++ 也是稳居编程语言排行榜前五的位置，实力雄厚。

04.Python之父 Guido van Rossum

头顶茂密指数：★★★★★

谁说程序员光头的，我就问你打脸不，瞧瞧咱 Python 之父浓密的秀发。

Guido van Rossum(吉多·范罗苏姆)1982 年获得阿姆斯特丹大学的数学和计算机科学的硕士学位，并于同年加入一个多媒体组织 CWI，做调研员。1989年，他创立了 Python 语言。那时，他还在荷兰的 CWI(Centrum voor Wiskunde en Informatica，国家数学和计算机科学研究院)。

1991 年初，Python发布了第一个公开发行版。Guido 原居荷兰，1995 移居到美国。从 2005 年开始就职于 Google 公司，其中有一半时间是花在Python上，2012 年加入 Dropbox。Dropbox 就是用 Python 写的

05.Delphi、C#和TypeScript之父

头顶茂密指数：★★★★

Delphi、C# 和 TypeScript 都是出自安德斯·海尔斯伯格之手 ，据说当年为了挖他，可是出动了比尔盖茨，年薪三百万美金， 进入微软公司后，首先主持了Visual J++的开发工作，后来由于在Java开发工具授权问题上和 Sun 公司的纠纷，微软停止了 Visual J++ 的后续开发。 这之后，作为 .Ne t概念的发起人之一，安德斯·海尔斯伯格被任命为微软 .Net 的首席架构师，主持 .Net 的开发工作。真的是集一身成就于一身啊。

06.PHP 之父 Rasmus Lerdorf

头顶茂密指数：★★★

估摸着头顶图案发型的鼻祖就是 Rasmus Lerdorf ，当然人家这叫原生态，别人的都是人工雕琢的。

PHP 创始人，Rasmus Lerdorf，PHP 是世界上最好的编程语言之父，无论你多不认可这句话，你都不能忽略有一半的互联网都运行着 PHP 的事实。Lerdorf 在1995年利用一组 C 语言的宏指令开发了 PHP，并将其用于网络应用的开发。后来 PHP 的开发和商业化由 Zend 公司负责。

Rasmus Lerdorf 在 2002 年 9 月至 2009 年 11 月 6 日间，他在 Yahoo! 公司担任基础设施架构师。2010 年，他加入 WePay 公司帮助开发其 API。2011 年起，他担任创业顾问。

07.JavaScript 之父

头顶茂密指数：★★★

布兰登·艾奇于 1964 年生于美国加州的森尼维尔市，毕业于伊利诺伊大学香槟分校。1995 年任职于网景期间开发出了 JavaScript，之后成为了浏览器上应用最广泛的脚本语言之一。

不过作为设计者，他一点也不喜欢自己的这个作品： "与其说我爱Javascript，不如说我恨它。它是 C 语言和 Self 语言***的产物。英国文学家约翰逊博士这样评价 JavaScript 的： '它的优秀之处并非原创，它的原创并不优秀。

08.Perl之父 Larry Wall

头顶茂密指数：★★★

别看 Larry Wall 雨林称不上茂盛，不过他有一妙招，那就是戴上帽子，需要的程序猿们赶紧get这项新技能。

你以为他只是一个牛逼的程序员?那你就大错特错了。Larry Wall，他的身份很多，是程序员、黑客、系统管理者、语言学家同时也是作家，当然，他最为人知的是他设计了Perl程序语言

Larry Wall 喜欢把 Perl 叫做一门"粗陋的"语言，他是这样描述 Perl：他是一个顺从，温顺的仆人，只是为了"让他服从于你的需要"而存在。

09.Ruby之父松本行弘

头顶茂密指数：★★★★★

松本行弘，1965年4月14日出生于日本鸟取县，上了高中之后，就迷上计算机了。成天看计算机杂志。能让计算机按照自己的指示来运行，这比什么都有意思。特别是对编程语言非常感兴趣。当时对汇编语言、Basic都不喜欢，不想用它们来编程。于是就开始自己发明语言，不过，不是Ruby~，具体是什么，松本行弘本人也不曾透露

1990年，由于经济不景气，公司没什么业务，松本行弘有着大把的空闲时间，于是开始开发 Ruby，直到 1995 年，松本行弘将 Ruby 公开于众。反响极其强烈，如果你学计算机的人数学都很厉害的话，那么，松本行弘就是个例外，他的数学从来就是1分(满分10分)

10.Swift之父Chris Lattner

头顶茂密指数：★★★★

Chris Lattner(1978年出生)是 LLVM 项目的主要发起人与作者之一，Clang 编译器的作者。2005年加入苹果，在2010年开始 Swift 语言的开发，从开始研发到最终发布不到 4 年时间，前期架构的开发都是由他自己一人独立完成，最终在 Swift 在2014年 WWDC 大会上发布，并且成为 Linux 平台增长最快的语言。

不过今年 Lattner 在工作上变动相当大，在特斯拉任职半年后宣布离职，时隔两个月后，Chris Lattner 前不久宣布决定加入 Google Brain 继续深入人工智能方面的研究。
