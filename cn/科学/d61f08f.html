编程与法律契合探索（1）——侵权行为因果关系的数据结构。

2020-10-29
| 科学



一、法律规定

我国《民法总则》第一百二十条规定：“民事权益受到侵害的，被侵权人有权请求侵权人承担侵权责任。”

第一百八十六条规定：“因当事人一方的违约行为，损害对方人身权益、财产权益的，受损害方有权选择请求其承担违约责任或者侵权责任。”

二、法律概念：

在民法理论上，根据《民法总则》的有关规定并结合侵权行为的归责原则理论，可将侵权行为分为一般侵权行为和特殊侵权行为两大基本类型。一般侵权行为是指行为人基于过错致人损害而应承担民事责任的行为。

一般侵权行为的构成要件，是指在一般情况下，构成侵权行为所必须具备的因素。只有同时具备这些因素，侵权行为才能成立。一般侵权行为的构成要件包括：有加害行为、有损害事实的存在、加害行为与损害事实之间有因果关系、行为人主观上有过错四个方面。

三、一般侵权行为的数理逻辑表示：

A == 有加害行为

B == 有损害事实存在

C == 加害行为与损害事实之间有因果关系

D == 行为人主观上有过错

一般侵权行为的成立 == A ∧ B ∧ C ∧ D

四、契合探索思路：

人类活动是高度复杂的社会化活动，而法律规范是在人类政治、经济、生活、社会中形成的高度科学、理性，符合社会发展趋势和发展规律而形成的统一规范，法律规范必须要符合发展规律，也要符合伦理道德、公序良俗等方面。王泽鉴先生也认为：侵权行为责任法律规范，是在“行动自由”与“权益保护”之间如何平衡。立法、司法的目的也就在于此。因此，王老先生坚持“相当因果关系说”，因果关系的认定倾向于保护被害人。但是什么是“相当”，怎么量化“相当”？实践中是个难题。因此，侵权责任特别是一般侵权责任的认定，不能单纯割裂四种构成要件来判断，而应该你中有我，我中有你，综合衡量。

例如： A加害行为，不能单纯从加害行为范畴考虑。而是应该从A、B、C、D四个范畴综合考虑加害行为。同理，B有损害事实存在，C加害行为与损害事实之间有因果关系，D行为人主观有过错。每个问题的判断，也应该从A、B、C、D四个范畴去考虑。这样构成要件的判断，就变成了三个子问题的判断（判断最终因果关系是目的）。当然中间可能有较多重复和假设，不同的人持不同观点立场不同，可能得出的数据结论也有异同。这并不关键，因为即使不拆分为子问题，一样是不同的人持不同观点。通过算法来拆分的问题，并形成多种因果关系，目的是把法律问题的事实和因果关系契合的颗粒度更加细化，建立一种建筑结构，来罗列所有的因果关系（包括假设和重复），通过论证和分辨，以取得最符合保护被害人利益的“相当因果关系”（其实这就是律师庭审前要做的准备功夫）。

四、实例探索

《施工造成伤害引发败血症致人死亡案》，是最高人民法院颁布的一宗典型民事侵权案例。案例如下：

１９８６年１０月，某建筑工程公司将其承包的一项拆除工程转包给个体工商户张某经营的工人新村青年合作服务站，并签订了承包合同。１９８６年１１月１７日，张某委托丈夫徐某负责施工，指挥临时工张某等人拆除混凝土大梁。在拆除过程中，张某因第６根大梁折断而滑落坠地受伤，急送医院检查，诊断为左下踝关节内侧血肿压痛，活动障碍。１２月７日，张某因伤口感染化脓治疗无效死亡。法医鉴定结论为：张某系左内踝外伤后，引起局部组织感染、坏死，致脓毒败血症死亡。后又经区医疗事故鉴定委员会鉴定认为，张某系因外伤所致脓毒败血症死亡，与其他因素无关，医院治疗无误。张某死亡后，其父亲和姐弟向法院提起诉讼，请求人身损害赔偿。被告以死者生前与其签订了“工伤概不负责任”的合同条款而拒绝承担责任。
