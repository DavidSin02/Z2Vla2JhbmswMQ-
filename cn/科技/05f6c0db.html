Windows系统不打补丁！看黑客是如何攻击你电脑的

2020-10-29
| 科技

我们的台式机和笔记本常用系统无非就是微软的Windows 7 和Windows 10 两个版本。服务器版本常用的是Windows server 2008 R2 ，Windows server 2012 R2 ， Windows server 2016 。在工作生活中我们的电脑系统、服务器系统经常提示升级、安装补丁包，然而我们并没有去安装微软推荐的补丁包，甚至将微软更新功能想尽一切办法关闭。这是一种非常危险的行为，尤其是服务器系统，这无异于服务器在互联网上裸奔。

下面我将以非常著名的漏洞--永恒之蓝漏洞为大家简单演示黑客是如何一步一步攻击你的服务器或者PC的。各位请遵守《中华人民共和国网络安全法》。

网络安全法

本次演示环境：

靶机：

系统：windows server 2008R2，

IP地址：192.168.88.105

攻击机：

系统：kali

IP地址：192.168.88.104

使用漏洞扫描工具扫描对应服务器，确认存在的漏洞。根据扫描结果，我们发现该服务器存在MS17-010漏洞未修复。因此我们利用该漏洞发起攻击。

漏洞扫描

查找攻击工具集中是否有该漏洞的攻击模块并确定该模块的位置，发现MS17-010模块而且有多个，我们使用第四个，位置为：exploit/windows/smb/ms17_010_eternalblue。

MS17-010模块

找到该模块后使用use命令加载使用该模块

加载MS模块

完成模块加载后需要对该模块进行相对应的参数设置，设置rhosts地址，也就要攻击的靶机地址，本演示中地址为192.168.88.105

设置靶机地址

发起攻击后要在对应的地方接收会话连接返回，因此需要设置对应的返回会话的监听，使用set命令设置监听模块。

加载攻击模块

使用set命令设置lhost地址，也就是攻击机的地址，这样我们才能正确接收靶机返回的会话连接。

设置攻击机地址

所有参数设置完成后，使用options命令检查参数设置是否正确，

options选项

确认上面的参数设置正确后我们就可以发起攻击了，等待最后的胜利。

发起攻击

我们发起攻击后在第一轮攻击中就已经把服务器攻陷了。先查看下系统版本是不是我们的靶机。返回结果是Windows 2008 R2 的系统版本，正是我们准备的靶机系统版本。

靶机系统版本查看

进入靶机后我们执行shell命令，相当于在Windows服务器上开启cmd命令行。然后进行创建用户等一系列操作。，由于Windows系统和Linux系统之间编码方式不同，因此在Linux系统中连接Windows系统的shell就存在字符乱码问题。这个问题可以在Linux系统终端中进行设置，我们这里只是演示攻击过程，就不进行编码设置了。

登录shell

接下来就可以使用cmd命令行中的命令对服务器进行操作了，我们使用net user 命令在靶机系统中添加一个用户。

添加用户

将新增加的密码修改掉，如果不修改我们在后面没有办法进行远程桌面登录操作

修改新用户密码

我们将新增加的用户添加到Administrators用户组中，这样我们新增加的用户也就拥有管理员权限了。

提升新用户权限

然后我们通过修改Windows系统的注册表的方式开始远程桌面进行下一步操作。

开启远程桌面

由于Windows系统默认开启防火墙并且拒绝所有的外部用户主动访问服务器，因此我们需要修改防火墙策略允许我们进行远程桌面登录。

修改防火墙策略

此时我们已经完成攻击，创建用户并提升用户权限到管理员级别。

下面我们用其他Windows主机使用mstsc远程桌面登录靶机。

使用远程桌面登录靶机

选择信任证书后进入Windows 2008 R2操作系统。

进入系统

总结：

我们要针对微软提出的高位及以上漏洞进行及时修复，尤其是服务器系统。或者通过安装专业的安全防护软件。
