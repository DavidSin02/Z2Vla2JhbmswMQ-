安装操作系统必须的硬盘分区知识，你了解多少

2020-10-29
| 科技

分区是写入构成分区表的硬盘扇区的过程，它包含有关分区的信息，包括扇区大小、相对于主分区的位置、存在的分区类型、安装的操作系统等。

创建分区时，会给它一个卷名，这样就可以很容易地识别它。

一、什么是分区？

硬盘驱动器的分区发生在对其进行物理格式化之后，但在对其进行逻辑格式化之前。它包括在磁盘上创建不混合数据的区域。

例如，可以使用它来安装使用不同文件系统的不同操作系统，因此，至少会有与使用不同文件系统的操作系统一样多的分区。如果只使用一个操作系统，一个磁盘分区就足够了。


有三种类型的分区：主分区、扩展分区和逻辑分区。一个磁盘最多可以包含四个主分区（其中只有一个可用），或者三个主分区和一个扩展分区。在扩展分区中，用户可以创建逻辑分区（即“模拟”几个较小的硬盘）。主分区可以用来安装操作系统，其中只有一个主分区还可以被设置为活动分区。如果硬盘上的4个主分区都安装了操作系统，那么活动分区的设置决定了是从哪个分区的系统启动。被设置为活动分区的主分区具有启动引导程序（例如，NTLDR，bootmgr）用来从硬盘加载操作系统。

二、主引导记录

引导扇区（称为主引导记录或MBR）是硬盘的第一个扇区（柱面0、磁头0、扇区1），它包含主分区表和称为引导加载程序的代码，将其加载到内存中后将允许系统引导。

在它被加载到内存中之后，这个程序将决定从哪个系统分区启动，并启动这个程序（称为bootstrap），该程序将启动该分区上存在的操作系统。

该磁盘扇区还包含有关硬盘的所有信息（即制造商、序列号、每个扇区的字节数、每个簇的扇区数、扇区数等），因此，这个扇区是硬盘上最重要的扇区，BIOS设置程序也使用它来识别硬盘。换句话说，没有它，你的硬盘是无用的，这使它成为病毒最喜欢的目标。

三、文件系统

FAT是DOS操作系统（DOS和Windows 95以及支持它的Windows NT和OS/2）使用的文件系统的名称。

下面是各种操作系统使用的文件系统：

DOS：FAT16

Windows 10：NTFS

Windows 7：NTFS

Windows XP：NTFS

Windows 98：FAT32

Windows 95：FAT16-FAT32（用于OSR2版本）

Windows NT：NTFS

OS/2：HPFS

Linux：Linux Ext2，Linux Ext3

FAT文件系统的特点是使用文件分配表和簇。

簇是FAT文件系统中最小的存储单元。簇实际上代表固定数量的磁盘扇区。

FAT（文件分配表）是文件系统的核心，它位于0柱面、1磁头、2扇区中（为了预防事故，在另一个扇区中复制）。该表记录了所使用的簇的数量以及文件在簇中的位置。

FAT文件系统支持最大2GB的磁盘或分区，最多允许65536个簇，因此，无论分区或磁盘的大小如何，每个簇必须有足够的扇区，以便可以在这65525个簇中包含整个磁盘空间，结果，磁盘（或分区）越大，每个簇的扇区数越多。

FAT文件系统使用根目录（在使用此类文件系统的操作系统上用符号C:\表示），该文件必须位于硬盘上的特定位置。此根目录存储有关它包含的子目录和文件的信息。对于文件，它将存储文件名、文件大小、上次修改文件的日期和时间、文件属性以及文件开始的簇号。

四、使用多个分区

如前所述，共有三种类型的分区：主分区、扩展分区和逻辑分区。一个磁盘最多可以包含四个主分区（其中只有一个是活动的），或者三个主分区和一个扩展分区。在扩展分区中，用户可以创建逻辑分区（给人的印象是有几个较小的硬盘）。

主分区

主分区必须经过逻辑格式化，并具有适合其上安装的操作系统的文件系统。

如果磁盘上有几个主分区，一次只能激活一个主分区，并且该分区可见，具体取决于启动计算机所使用的操作系统。通过选择要在启动时加载的操作，可以确定哪个分区是可见的。活动分区是计算机启动时加载操作系统的分区。除开始时使用的活动分区以外的其他分区将被隐藏，隐藏的分区就不能被访问，因此，只能从安装操作系统的主分区访问数据。

扩展分区

开发扩展分区是为了克服四个主分区的限制，因为你可以在其中创建任意数量的逻辑分区。扩展分区中至少需要一个逻辑分区。用户可以在主分区和逻辑分区中存储数据，但是扩展分区不能直接存储数据，因为扩展分区的作用是保存逻辑分区。

许多人在使用计算机时只把整个硬盘分为一个区，从而占用了硬盘上的所有可用空间，但是，就性能和容量而言，这不是最有利的解决方案。一个好的解决方案是创建多个分区，这样就可以在磁盘上安装多个操作系统，节省磁盘空间，提高文件安全性以及更轻松地组织数据。
