IoC部分将会讲解有关Bean装配的相关知识，比较零散，主要分三大块：

  • Bean装配：对象的创建，不涉及过程也就是属性初始化。

1.静态工厂

上一章节我们讲解了动态工厂的简单配置，本节将简单介绍静态工厂的配置。那什么是静态工厂呢？

静态工厂其实也就是类的创建模式，又叫做简单方法模式。不考虑其它，只考虑静态工厂类的定义，即使用静态方法来对外提供目标对象的实例，也就是在动态工厂类中的方法上加上static关键字。

2.静态工厂的配置

首先我们要知道，静态工厂获取Bean的实例是直接调用静态方法返回的，也就是说静态工厂不需要实例化。在Spring中可以通过<bean/>标签属性class(值为静态工厂类的全限定性类名)和factory-method(值为静态工厂中获取指定Bean实例的方法名)进行静态工厂的配置。

  • 静态工厂：其实就是获取目标Bean的方法是静态的
/** * 静态工厂类 * 其实就是获取/** * 静态工厂类 * 其实就是获取实例的方法是静态的 */public class MyBeanFactory{    /**     * 获取String对象     * @return     */    public static String getStringObj(){        return "String 对象";    }    /**     * 获取Integer对象     * @return     */    public static Integer getIntegerObj(){        return Integer.MAX_VALUE;    }}的方法是静态的 */public class MyBeanFactory{    /**     * 获取String对象     * @return     */    public static String getStringObj(){        return "String 对象";    }    /**     * 获取Integer对象     * @return     */    public static Integer getIntegerObj(){        return Integer.MAX_VALUE;    }}
  • xml配置
<!-- 1.静态工厂配置 --><bean id="stringObj" class="com.spring.bean.MyBeanFactory" factory-method="getStringObj"/><bean id="integerObj" class="com.spring.bean.MyBeanFactory" factory-method="getIntegerObj"/>

4.静态和动态工厂Bean的配置总结

静态工厂中的方法是static方法，动态工厂中的是实例方法。

  • 动态工厂Bean，xml配置的两种方法(建议第二种)
<!-- 1.将动态工厂作为普通Bean配置 --><bean id="factory1" class="com.spring.bean.MyBeanFactory"/><!-- 2.使用Spring提供的动态工厂配置 --><bean id="factory2" class="com.spring.bean.MyBeanFactory"/><bean id="stringObj" factory-bean="factory2" factory-method="getStringObj"/><bean id="integerObj" factory-bean="factory2" factory-method="getIntegerObj"/>
  • 静态工厂Bean的xml配置
<!-- 1.静态工厂配置 --><bean id="stringObj" class="com.spring.bean.MyBeanFactory" factory-method="getStringObj"/><bean id="integerObj" class="com.spring.bean.MyBeanFactory" factory-method="getIntegerObj"/>
