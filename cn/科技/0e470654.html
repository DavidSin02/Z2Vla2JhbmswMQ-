企业实施整合管理体系的现状与建议

2020-10-29
| 科技

近10年以来，随着ISO9000质量管理标准宣贯工作的深入开展，越来越多的企业逐渐意识到管理体系的重要性，随之也陆续建立和运行ISO14000环境管理体系，以及OHSAS18000职业安全健康管理体系（现已替代为ISO45001），这些标准管理体系的实施，在很大程度上，帮助企业建立了一套结构化，科学化的系统管方法，在企业日常经营管理的各项活动中，起到了非常积极的作用。


然而，随着管理体系推行的逐步深入，以及越来越多的政府监管（比如企业标准化建设）或者客户特定的业务要求，比如企业供应链上游客户的一些特定要求，很多企业实施的管理体系越来越多，内部负责推进各管理体系的团队也越来越大，慢慢就出现了一个不得不重视的问题：资源的浪费以及效率的低下。笔者在2006年在珠三角加入一家美资电子代工巨头，曾亲身经历过公司内推行多管理体系的纷乱局面，质量管理体系ISO9000，环境管理体系ISO4000，职业健康安全管理体系（当时还是OHSAS18000），后来因为几个大客户如戴尔，微软，惠普等组成了一个联盟，制定了电子行业行为准则 （EICC，Electronic Industry Code of Conduct）,要求其供应链上的下游厂商必须遵守，再后来，戴尔又进一步推出Dell Global Citizenship（一个关注可持续发展的全球性项目），由于这些都是围绕国际劳工组织的 SA8000 体系要求，或者是基于该体系衍生出的企业规定，为了业务的可持续性，我们又开始建立实施 SA8000管理体系，每年企业要接待不同的外部审核机构开展不同层次的审核活动，加上内部又分别由不同的团队人员负责不同体系的实施工作，这里面的沟通成本，实在是一笔巨大的负担。


因其种种，推进“整合管理体系”的呼声越来越高，尤其是最近几年，在一些已经实施了几个不同管理体系的企业里，几乎都开始管理体系的整合工作。以ISO9000，ISO4000和OHSAS18000 为例，三个体系的整合，固然有其先天的基础，这是因为：

1. 三个体系的原生内核是相同的，都是基于PDCA的戴明循环，采用了相同的管理体系框架，其核心内容都是根据管理学原理，为组织建立了一个动态循环的管理模型，以持续改进的思想指导组织系统地实现其既定的目标；

2. 从体系的输入/输出两端，三者也高度类似。比如：ISO9000质量管理体系的输入端是客户需求，输出端是有形的产品或无形的客户满意度，而ISO4001和OHSAS8001，输入端是相关方要求或关注，输出端是良好的环境或职业健康安全绩效（实施上也是满足相关方要求，与质量管理体系的客户满意度异曲同工。

3. 三个标准体现相同的管理原则。三个标准的内容都体现了“领导作用”、“全员参与”、“过程方法”、“管理的系统方法”、“持续改进”、“基于事实的决策方法”、“互利的供方关系”等管理原则。

4. 三个标准的管理性要求有很多相似之处。三个标准的管理性要求有很多相似的部分。如“文件控制”、“记录控制”、“方针目标”、“组织结构和职责”、“培训、意识和能力”、“资源管理”（如设备管理）、“法律法规要求”、“交流与沟通”、“监视和测量装置的控制”、“纠正措施”、“预防措施”、“内部审核”、“管理评审”等方面。

5. 三个标准要求的管理手段和方法一致。都采取系统的方法，建立一个完整的、有效的、文件化的管理体系；都按PDCA循环的思想，通过识别影响质量、环境、职业安全健康的因素，有针对性地制定计划和管理方案，实施运行控制，并采取必要的监视和测量，发现问题，实施改进，实现管理体系的持续有效运行；都要求组织配备适当的资源；都要求通过日常管理的监视和测量、内审、管理评审等管理手段来评价体系的运行状况。

尽管如此，我们仍然注意到，还有为数众多的企业，在实施整合管理体系的过程中，出现了很多的问题

，尤其是一些中小企业，与一些大公司相比，因其企业规模相对较小，资源配置（包括对管理体系有深刻理解的人员）也不是完全充分，在体系实施过程中的痛点尤为突出。具体表现为：

1. 整合效果不明显，原来是3个管理体系，实施整合后并没有降低工作量或精简了管理流程。相反，因为要推进体系整合，有些部门感觉工作量加大了。这就没有起到整合体系，提高效率的目的。

2. 体系文件整合了，但实施起来更不能落地了。为了整合而整合，体系文件成了抽屉里的一沓纸，只有在审核的时候才“重见天日”。

3. 整合后体系管理“两张皮”现象严重，体系文件描述的是一回事，实际运行情况是另一回事。这在一些同时推进标准化建设的企业里尤其突出。一套体系文件用于应付检查，一套管理程序用于指导日常工作。试想，这到底是“减负”还是“增负”？

4. 错把三个管理体系的同时审核/认证，就认为是已经实施了“整合”的管理体系。事实上这其实只是“整合”了体系的认证机构，某种程度上可能只是为企业节约了一些认证成本而已。

事实上，整合管理体系的实施是公认的优化企业内部资源，建立良好运行机制，通过PDCA循环来持续改善企业管理水平的最佳工具之一。国际上很多著名的企业，如杜邦，壳牌，国内的中石油，中石化等都在探索建立企业自己的整合管理体系。笔者结合自身多年体系管理实践经验，总结得失，针对上述体系整合的痛点难点，提出下述改善建议：

1. 立足企业自身实际，打造“量体裁衣”的整合管理体系。当我们谈“整合管理体系”时，并不是要企业去创建一个“独一无二”的新的管理体系，而是要立足企业自身实际状况，依托现有资源，根据已经实施的管理体系要求，识别不同的体系要求中重复的管理流程，比如文件控制，内审，管理评审等等，从流程优化的角度去做精益，达到精简流程，提高效率的目的，这也是企业实施管理体系的初心。

2. 警惕“整合管理体系”等同于“整合程序文件”。诚然，程序文件是管理体系的重要组成部分，管理体系的要求是通过程序文件这一载体来传达和实施，但一定要注意，整合管理体系并不等同于整合程序文件。我们建立实施管理体系的目的是为了帮助企业优化资源，规范管理，达成业务目标。任何的管理措施都要跟这个目的相一致。千万不要因为要“整合管理体系”而再多付出其他的管理资源（人力/物力），这就跟建立实施管理体系的目的相违背了。

3. 重点关注整合管理体系中具有共性的，普适性的要求，从“不同中找相同”。前文提到，例举的三个管理体系，都是采用过程管理/控制的方法，通过建立目标，制定流程，监测进度，实施改进来持续提高管理绩效。那么，三个体系中与这一过程相对应的管理要素，比如目标指标的设定，监测测量，检查改进（包括内审和管理评审）等程序就可以进行合理的整合。

4. 做减法，不做加法！Less is More，尽量保持整合管理体系简单化，简单意味着容易被理解，容易被实施。企业在实施整合管理体系的时候，一定要避免走入一个误区，追求一个大而全，包罗万象的庞然大物。实施上，从体系要求落地的角度，多来来的实践早已证明，大则虚空，小而弥坚。

随着我国进入到“十四五“以及”中国智造2025“的战略实施，各行各业的发展已经日新月异，行业之间的良性竞争也愈发激烈。任何一家企业，都必须切实提高自身管理水平，“向管理要红利“才能在竞争的浪潮中抢得一席之地。相信”整合管理体系“在未来的很长一段时间里也还会是热点词汇，而管理体系整合的理念也必将随着实践发展到另一个高度。我们也希望所有的企业能利用好这套工具，勤练内功，科学发展。期待中国企业管理百花齐放，百家争鸣的那一天早日到来！
