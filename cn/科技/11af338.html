Windows 10 Build 20241发布：应用启动屏幕可跟随系统主题

2020-10-29
| 科技

本周三面向 Dev 频道的 Windows Insider 成员，微软发布了 Windows 10 Build 20241 预览版更新。由于 Dev 频道版本的开发进度不再跟随功能更新，因此某些新功能可能不会装备在下个功能更新中。但目前下个功能更新存在诸多不确定因素，有消息称 21H1 可能会被 Windows 10X 系统取代。

Windows 10 Build 20241 预览版更新主要引入以下新功能

为应用程序引入主题感知的启动屏幕

继开始菜单中创建跟随系统主题的动态磁贴之后，微软为 UWP 应用创建了跟随系统主题的初始屏幕。现在，符合条件的 UWP 应用在打开的时候，初始屏幕颜色将会跟随系统主题显示。例如当前系统中使用的是深色模式，那么你会看到深色模式的启动界面。

浅色模式

深色模式

目前已经支持主题感知的应用程序包括

● Settings

● Store

● Windows Security

● Alarms & Clock

● Calculator

● Maps

● Voice Recorder

● Groove

● Movies & TV

● Snip & Sketch

● Microsoft ToDo

● Office

● Feedback Hub

● Microsoft Solitaire Collection

微软表示通过 Microsoft Store 更新，会有更多的应用程序获得主题感知功能。这项功能目前仅向部分 Windows Insider 成员开放，帮助微软快速识别问题，性能影响和可靠性等。

增强碎片整理体验

微软对“优化驱动器”页面（设置>系统>存储>优化驱动器）进行了一些更改，包括：

● 增加全新的“高级视图”复选框，列出包括隐藏卷在内的所有卷

请注意，微软仍在推进和优化该程序，因此您会在此版本中看到该复选框，但是单击该复选框可能不会发现任何差异。

● 在“当前状态”列中列出更多详细信息。

当卷不可用于碎片整理时（例如，“不支持分区类型”和“不支持文件系统类型”），在“当前状态”列中列出更多详细信息。

● 添加了对 F5 刷新的支持。
