前端数据类型的动态与静态检查

2020-10-29
| 科技

前端数据类型检查工具，目前至少有TypeScript、flow.js等常用工具，另外在此推荐另一款类型检测库：helloType。三者做一个简单对比如下：

ts、flow和helltype对比图

一、Typescript

  • 首先是JavaScript开发者需要再进一步学习，内容不少，但很强大
  • 有一定陡峭的学习曲线
  • 已经在使用的应用代码，需要整个改用TypeScript代码语法，才能发挥完整的功用。这对很多已经有内部代码库的大型应用开发团队而言，将会是个重大的决定，因为如果不往全面重构的路走，将无法发挥强(静态)类型语言的最大效用

二、flowJS

1.类型推断：

/*@flow*/

function foo(x) {

return x.split(' ');

}

foo(34);

报错提示大概如下：

index.js:4

4: return x.split(' ');

^^^^^ property `split`. Property not found in

4: return x.split(' ');

^ Number

2.注解方式

/*@flow*/

function foo(x : number, y : number) : number {

return x + y;

}

foo('Hello', 42);

错误信息如下所示:

index.js:7

7: foo('Hello', 42);

^^^^^^^ string. This type is incompatible with the expected param type of

3: function foo(x : number, y : number) : number{

^^^^^^ number

三、HelloType

import { Dict, IfExists } from 'hello-type'

const PersonType =

Dict({

name: String,

age: IfExists(Number),

}) // 事先定义数据类型

PersonType.test({ name: 'tomy' }) // true

PersonType.test({ name: 'tomy', age: 10 }) // true

PersonType.test({ name: 'tomy', age: null }) // false
