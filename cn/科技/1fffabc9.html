springboot 官方例子--使用Thymeleaf获得参数值

2020-11-14
| 科技

本指南将指导使用Thymeleaf获得参数值获得GET参数及表格POST参数。原官网上是写了两篇的文章，这里把它们合在一起了.Thymeleaf是一种用于Web和独立环境的现代服务器端的Java模板引擎,主要目标是将优雅的自然模板带到开发工作流程中，并将HTML在浏览器中正确显示，并且可以作为静态原型，让开发团队能更容易地协作,Thymeleaf能够处理HTML，XML，JavaScript，CSS甚至纯文本。springboot 推荐使用thymeleaf，支持最好。

我利用业余时间，翻译了Spring官网的例子，方便中文不好的同学，将陆续发到头条上，欢迎大家关注，也可以上我个人BLOG：itmanclub.com，上面有已经翻译过的。

1、通过GET方式获得参数

请求http://localhost:8080/greeting

会返回HTML的如下信息:

"Hello, World!"

如果加一个name参数，如下

http://localhost:8080/greeting?name=User

这个name参数，将代替里面的"World" ，返回结果如下:

"Hello, User!"

2、通过POST方式获得参数

本例访问http://localhost:8080/greeting，会看到表单，然后填入内容，提交，将看到显示结果页

1、 通过GET方式获得参数

创建一个Controller

在Spring构建网站的方法中，HTTP请求由控制器处理。您可以通过@Controller注释够识别这些请求。在下面的示例中，GreetingController通过返回视图的名称来处理get请求/greeting ，在本例中是“greeting”视图负责呈现HTML内容：

src/main/java/hello/GreetingController.java

package hello;import org.springframework.stereotype.Controller;import org.springframework.ui.Model;import org.springframework.web.bind.annotation.GetMapping;import org.springframework.web.bind.annotation.RequestParam;@Controllerpublic class GreetingController { @GetMapping("/greeting") public String greeting(@RequestParam(name="name", required=false, defaultValue="World") String name, Model model) { model.addAttribute("name", name); return "greeting"; }}

这个控制器简洁明了，但有很多事情要做。让我们一步一步地把它分解。

@GetMapping注释确保HTTP GET请求到/greeting被映射到greeting()方法。

@RequestParam将查询字符串参数name的值绑定到 greeting()方法的name参数中。此查询字符串参数不是必须的，如果请求中没有此参数，则使用默认值“World”。名称参数的值被添加到模型对象中，最终使其可被视图模板访问。

方法体的实现依赖于视图模版引擎技术（在本例中是Thymeleaf）来执行HTML的服务器端呈现。Thymeleaf解析下面的greeting.html模板并计算th:text表达式以呈现在控制器中设置的$name参数的值。

src/main/resources/templates/greeting.html

<!DOCTYPE HTML><html xmlns:th="http://www.thymeleaf.org"><head> <title>Getting Started: Serving Web Content</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <p th:text="'Hello, ' + ${name} + '!'" /></body></html>

部署程序

开发Web应用程序的一个常见功能是代码更改、重新启动应用程序以及刷新浏览器以查看更改。整个过程会占用很多时间。为了加速事情的循环，SpringBoot附带了一个称为SpringBootDevTools的方便模块。

  • 启用热插拔
  • 切换模板引擎以禁用缓存
  • 允许Livereload自动刷新浏览器
  • 其他基于开发需要的功能

创建执行的Appcation

src/main/java/hello/Application.java

package hello;import org.springframework.boot.SpringApplication;import org.springframework.boot.autoconfigure.SpringBootApplication;@SpringBootApplicationpublic class Application { public static void main(String[] args) { SpringApplication.run(Application.class, args); }}

@SpringBootApplication包含如下注解:

@Configuration 将类标记为应用程序上下文的bean定义源。

@EnableAutoConfiguration 告诉SpringBoot根据类路径设置、其他bean和各种属性设置开始添加bean。

@ComponentScan 告诉Spring在hello包中查找其他组件、配置和服务。

您注意到没有一行XML吗？也没有web.xml文件。这个Web应用程序是100%纯Java，您不必麻烦的基础配置

运行你的程序

STS下(Maven可参考前面文章)：右键-选择Run as-Spring Boot App ，

测试应用程序

在浏览器中访问http://localhost:8080/greeting，输出如下:

"Hello, World!"

增加name参数 http://localhost:8080/greeting?name=User ，输出如下:

Hello, User!"

此更改表明GreetingController中的@RequestParam安排按预期工作。名称参数已被赋予默认值“world”，但始终可以通过查询字符串显式重写。

增加一个主页

静态资源，如HTML、JavaScript或CSS，只要将它们放在源代码中的正确位置，就可以从Springboot就可以访问他们。默认情况下，Springboot的静态内容位于“/static”（或“/public”）。index.html资源是一般网站的主页，这意味着它将被用作/资源，即在我们在访问http://localhost:8080/时，就访问到这个页面。所以创建这个文件：

src/main/resources/static/index.html

<!DOCTYPE HTML><html><head> <title>Getting Started: Serving Web Content</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <p>Get your greeting <a href="../../greeting">here</a></p></body></html>

重启后你就可以访问 http://localhost:8080/.

2、 通过POST方式获得参数

这里列的都和前面不一样，其余都一样：

创建一个Controller

src/main/java/hello/GreetingController.java

package hello;import org.springframework.stereotype.Controller;import org.springframework.ui.Model;import org.springframework.web.bind.annotation.GetMapping;import org.springframework.web.bind.annotation.ModelAttribute;import org.springframework.web.bind.annotation.PostMapping;@Controllerpublic class GreetingController { @GetMapping("/greeting") public String greetingForm(Model model) { model.addAttribute("greeting", new Greeting()); return "greeting"; } @PostMapping("/greeting") public String greetingSubmit(@ModelAttribute Greeting greeting) { return "result"; }}

这个控制器简洁明了，但有很多事情要做。让我们一步一步地分析它。

此控制器中的两个方法都映射到/greeting。您可以使用@RequestMapping，它默认映射所有HTTP操作，如get、post等。但在这种情况下，greetingForm()方法被专门映射为使用@GetMapping获取，而 greetingSubmit()则被映射为使用@PostMapping发布。

greetingForm()方法使用模型对象向视图模板公开新的Greeting对象。以下代码中的Greeting对象包含与greeting视图中的表单字段相对应的id和content等字段，并将用于从表单中捕获信息。

src/main/java/hello/Greeting.java

package hello;public class Greeting { private long id; private String content; public long getId() { return id; } public void setId(long id) { this.id = id; } public String getContent() { return content; } public void setContent(String content) { this.content = content; }}

方法体的实现依赖于视图技术，通过将视图名“greeting”转换为要显示的模板。在这种情况下，我们使用Thymeleaf，它解析下面的greeting.html模板，并呈现表单。

src/main/resources/templates/greeting.html

<!DOCTYPE HTML><html xmlns:th="http://www.thymeleaf.org"><head> <title>Getting Started: Handling Form Submission</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <h1>Form</h1> <form action="#" th:action="@{/greeting}" th:object="${greeting}" method="post"> <p>Id: <input type="text" th:field="*{id}" /></p> <p>Message: <input type="text" th:field="*{content}" /></p> <p><input type="submit" value="Submit" /> <input type="reset" value="Reset" /></p> </form></body></html>

th:action="@{/greeting}"表达式指示表单内容提交到/greeting，而 th:object="${greeting}"表达式声明用于收集表单数据的模型对象。用th:field="{id}"和th:field="{content}"表示的两个表单字段对应于上面Greeting对象中的字段。

现在让我们回顾一下提交表单的过程。如上所述，表单使用一个POST提交到/greeting。greetingSubmit()方法接收由form填充的Greeting对象。Greeting对象是@ModelAttribute，因此它绑定到传入表单内容，并且可以通过名称（默认为方法参数的名称，在本例中为“greeting”）在结果视图中呈现提交的数据。id 呈现在 <p th:text="'id: ' + ${greeting.id}" /> 表达式中。同样，content 也呈现在<p th:text="'content: ' + ${greeting.content}" /> 表达式中。
