干货｜你真的了解有限状态机（FSM）算法的使用方法吗？

2020-10-29
| 科技

由于我以前是从事路由协议开发，所以对FSM比较熟悉，因为网络协议RFC文档里面有大量的关于状态机的描述。偶然一次在知乎上看到一个算法题，不少回答说可以通过DFA来解决，最初以为是一个更高级的解法，当看到别人画的转换图后，觉得跟FSM没有区别，然后就百度了一下DFA和FSM的关系。wikipidia是这样描述的：

Finite state machines are of two types – deterministic finite state machines and non-deterministic finite state machines.

In the theory of computation, a branch of theoretical computer science, a deterministic finite automaton (DFA)—also known as deterministic finite acceptor (DFA), deterministic finite state machine (DFSM), or deterministic finite state automaton (DFSA)—is a finite-state machine that accepts or rejects strings of symbols and only produces a unique computation (or run) of the automaton for each input string.

从以上描述应该明白了FSM和DFA的关系，也就是全集与子集的关系。

如果简单一点描述FSM，可以这样理解：有限状态机的工作原理是，当事件(event)发生后，根据当前状态(current_state) ，决定执行的动作(action)，并设置下一个状态(next_state)。

一般用二维表来描述其逻辑关系，见下图：

状态机一般有两种写法：横着写和竖着写。横着写是在事件中判断状态。竖着写是在状态中判断事件。这两种写法是等效的，但在实际操作中，效果有点诧异，但各有优缺点。

我们来分析一个Leetcode上通过率最低的算法题，第65题---Valid Number，来感受一下FSM的强大威力。

题目如下：

Validate if a given string can be interpreted as a decimal number.

Some examples:

"0" => true

" 0.1 " => true

"abc" => false

"1 a" => false

"2e10" => true

" -90e3 " => true

" 1e" => false

"e3" => false

" 6e-1" => true

" 99e2.5 " => false

"53.5e93" => true

" --6 " => false

"-+3" => false

"95a54e53" => false

Note: It is intended for the problem statement to be ambiguous. You should gather all requirements up front before implementing one. However, here is a list of characters that can be in a valid decimal number:

Numbers 0-9

Exponent - "e"

Positive/negative sign - "+"/"-"

Decimal point - "."

Of course, the context of these characters also matters in the input.

只所以通过率低，是因为很多人不清楚为Valid Number的条件，只能一次一次的去试验。

方案一：状态机

这道题比较好的一种解法是用DFA，通过题意，可以得出如下的状态转换图：

根据上面的转换图，可以得出如下的二维表，其中-1表示非法状态。

其中横座标表示状态，纵座标表示输入，当状态处于3,5,8,9时，表示合法的状态。对于在状态q1输入空格，仍然处于状态1.

此时肯定会有人觉得，当处于q3状态时，输入“.”符号，应该是从q3到q4，而不应是从q3到q5。看“3.”这个测试用例，人眼观察是应该输出true。如果是到q4，就是输出false了。

根据以上规则写出来的处理代码如下：

bool isNumber(char* s) { int FSM[5][9]={ { 1,-1, 9,-1, 9,-1,-1, 9, 9}, { 2,-1,-1,-1,-1, 7,-1,-1,-1}, { 3, 3, 3, 5, 5, 8, 8, 8,-1}, { 4, 4, 5,-1,-1,-1,-1,-1,-1}, {-1,-1, 6,-1, 6,-1,-1,-1,-1} }; if (NULL == s) { return false; } int curState = 1; while ('\0' != *s) { int input = 0; char c = *(s++); if ('0' <= c && '9' >= c) { input = 2; } else if (' ' == c) { input = 0; } else if ('.' == c) { input = 3; } else if ('+'== c || '-' == c) { input = 1; } else if ('e' == c) { input = 4; } else { input = -1; } if (!(0 <= input && 5 >= input)) { return false; } curState = FSM[input][curState-1]; if (-1 == curState) { return false; } } if (curState == 3 || curState == 5 || curState == 8 || curState == 9) { return true; } else { return false; }}

状态机其他使用场景：

A. 验证一串二进制bit，是否包含偶数个 0 和奇数个 1。这个题目也很适合用状态机来解决。组合状态有：

  1. 偶数个 0 和偶数个 1（记作 EE）
  2. 偶数个 0 和奇数个 1（记作 EO）
  3. 奇数个 0 和偶数个 1（记作 OE）
  4. 奇数个 0 和奇数个 1（记作 OO）

状态转换图为：

FSM 初始化的状态是 EE，一个 bit 都没处理，0 和 1 都是偶数个。FSM 的接受状态是 EO。如果最终到达这个状态，那么处理成功。

B. 判断一个 binary string 是否能被 3 整除

一个数能不能被整除，关键看余数是否为 0。

如果第一个输入是 1，那么它不能被 3 整除，商 0 余 1。除了 0 之外，这里余数可能的取值还有 1 和 2。我们试试把状态应该设置为 0，1，2，看看是否有解。这里 0 是接受状态，也是初始状态。OK，从初始状态 0 起，输入是 1，那么状态迁移到 1。

如果第二个输入是 0，也就是说现在看到的串是 10，左移了一位，相当于被除数乘以2。余数为 (1 * 2) % 3 = 2，状态从 1 迁移到 2。

如果第二个输入是 1，那么就乘 2 加 1 再取模，余数是 (1 * 2 + 1) % 3 = 0，状态从 1 迁移到 0。
