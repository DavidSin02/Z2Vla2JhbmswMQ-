Excel Power Query中批量处理列

2020-10-29
| 科技

今天接着给大家讲解有关Excel Power Query的案例。

批量处理列，在Power Query中使用的是Table.TransformColumns函数。


函数的语法如下：


function (table as table,transformOperation as list,optional defaultTransformation as nullable fuction,optional missingFiled as nullable MissingFiled.Type) as table


从上面的函数的语法来看，很多的小伙伴们可能有点崩溃，一个也没有看懂吧。没有关系，我们下面通过一些实例来具体地看看。


案例1


如下图所示，想给技能1与技能2的得分，分别加上5分与10分。

具体的公式可以写成：


= Table.TransformColumns(源,{{"技能1",each _+5},{"技能2",each _+10}})



这个公式跟我们之前讲过的公式是一样的，第一参数是源，类型是table，第二个参数是一个list，对几个列操作就写几个list。如果只有一个列的时候，最外面的大括号也可以不写。如：


= Table.TransformColumns(源,{"技能1",each _+5})


案例2


Table.TransformColumns函数还有一种比较特殊的用法。

如下图所示，将除“员工编号”与“员工姓名”两个列以后，其他的最后的列的格式转化成文本型。在前面连接“成绩”两个字。


let    源 = Excel.CurrentWorkbook(){[Name="表4"]}[Content],    自定义1 = Table.TransformColumns(源,                                        {                                            {"员工编号",each _},                                            {"员工姓名",each _}                                        }                                        ,each "成绩"&Text.From(_)                                    )in    自定义1



如果只是将数字转化成文本的话，可以有下面两种写法：

= Table.TransformColumns(源,{{"员工编号",each _},{"员工姓名",each _}},Text.From)


或者

= Table.TransformColumns(源,{{"员工编号",each _},{"员工姓名",each _}},each Text.From(_))

上面的这两种写法都是等价的。


案例3


还有另外一种情形是，可以给当前的表，增加一列。

如下图所示，增加一列空列为“备注”。


= Table.TransformColumns(源,{"备注",each _},each _,2)


这个函数的第4个函数是指当第2个参数里面的列不存在时，可以设置相关的条件。一共有下面的几种，此处使用的最后一个。MissingField.UseNull.


该函数的第三个参数是一个function，此处可以可以有很多的扩展的用法。
