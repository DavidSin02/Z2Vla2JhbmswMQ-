概念辨析：同步/异步，阻塞/非阻塞

2020-10-29
| 科技

同步与异步

同步与异步要从服务调用方的视角区分.

以下发 Flink 计算任务为例:

我们调用 Flink 的 API 接口提交计算任务,

  • 如果需要等到 Flink 计算完成后, API 再响应, 那么对于调用方来说, 这个 API 是同步的;
  • 如果不需要等到 Flink 计算完成, 调用 API 后立即响应一个类似于 JobId 之类的中间结果, 调用方需要通过中间结果去查询计算任务的进度, 那么对于调用方来说, 这个 API 就是异步的.

阻塞与非阻塞

阻塞与非阻塞是针对 CPU 而言的.

以上面的例子继续谈, 当调用方调用一个同步 API 时, 在等待服务方返回结果期间,

  • 如果调用方的 CPU 也陷入等待, 那么这就是阻塞的;
  • 如果调用的 CPU 挂起, 趁机去执行其他的操作, 那么这就是非阻塞的.

举例

人们常说, JavaScript 是一门异步编程语言, 就是因为它的很多接口都是异步的, 调用一个接口时返回的是一个中间结果, 而不是最终结果. 为了解决异步编程的困扰, JS 先后有 callback, Promise, Async/Await 等编程方案.

为什么 JavaScript 会设计成异步的呢? 同步明明更省事啊. 因为 JavaScript 在浏览器上最初设计时是单线程的, 如果设计成同步的, 调用一些耗时接口时, 该进程就会阻塞, 后续的操作就无法执行了.
