LTE协议学习总结之第二章：LTE无线通信技术主体介绍

2020-11-14
| 科技

LTE(Long Term Evolution,长期演进技术)是3GPP主导制定的无线通信技术，其实准确来说，LTE不等于4G，我们更愿意称它为“准4G”或“3.9G”。它是一个3G到4G之间的过渡技术。

在了解了LTE技术大概是什么之后，我们需要明确它的目标是什么？通过查阅相关资料可知，这种以OFDM/FDMA为核心的技术的主要性能目标包括：能灵活配置1.4MHz、3MHz、5MHz、10MHz、15MHz、20MHz的频谱带宽；并且保证在频谱带宽在20MHz的条件下能提供下行100Mbps、上行50Mbps的峰值速率；在系统延迟方面表现为用户平面单向传输时时延应该低于5ms，而在控制面由睡眠到激活状态时的时延应低于50ms，从驻留到激活状态的时延小于100ms；提高小区容量，改善小区边缘用户性能；能够为350Km/h高速移动用户提供最少100kbps的接入服务；取消电路(CS)域和无线网络控制(RNC)节点。LTE允许高速数据通信 (接收: 100Mbps或更高，传送: 50Mbps或更高) 。因为频段可扩展到20MHz，所以最大理论速度为接收326.4Mbps，传送86.4Mbps。 数据通信速度高于相同频段的3G/3.5G，可让许多用户使用。 另外，LTE可实现低延迟传送，连接延迟为100ms或更短，无线段传送延迟为5ms或更短。因此LTE适用于音频通信、动态图像分发和在线游戏。


LTE目标

接着我们介绍LTE技术的主要技术：

LTE采用了宽广的频率范围：700MHz-2.7GHz，即对应特高频（UHF），对应电磁波的波长为分米波：1m~10cm。为了增强此频段下的效率，采用了多个天线进行发送/接收的 Multi Input Multi Output（多入多出技术 MIMO）以及云讯基站间多重访问的Space Dvision Multiple Access（空分多址 SDMA）等天线技术。另外采用了Orthogonal Frequency Division Multiple Access （正交频分多址 OFDMA）和Single Carrier Frequency Division Multiple Access（单载波频分多址 SC-FDMA）作为多路传输系统。

为了明白LTE中网元彼此之间如何联系，最后我们介绍一下LTE的基本架构-协议架构：

接口协议的通用模型

LTE无线侧的主要接口分为空中接口和地面接口。LTE空中接口是UE和eNodeB的LTE-Uu接口，LTE无线侧的地面接口主要是eNodeB之间的X2接口及eNodeB和EPC之间的S1接口。

接口协议主要分为三层两面，三层即：物理层(Physical Layer)、数据链路层(Transport Network Layer)、网络层(Radio Network Layer)。两面为：用户面和控制面。物理层的作用是提供两个物理实体间的可靠比特流的传送，适配传输媒介。无线的空中接口中，适配的是无线环境，而在地面接口中，适配的则是EI、网线以及光纤等传输媒介。数据链路层的作用有信道复用和解复用、数据格式的包装、数据包调度等，其主要的作用是具有个性的业务数据向没有个性的通用数据帧的转换。网络层的作用则是寻址、路由选择、连接的建立和控制、资源的配置策略等。如何理解两面呢？两面相当于企业里的业务线和管理线，业务线负责实际干活，而管理则负责人力、物力和财力上的协调控制工作，为了使两个线彻底分开，企业中确立了职业经理人的制度，也就是尽量将业务线和管理线能独立工作使其彻底分开。用户面的主要作用是负责传输处理业务数据，而控制面则负责协调和控制指令的传送和处理。由于它们都属于逻辑层面上的概念，因此在物理层不区分两面，而在数据链路层完成数据处理功能，其实体完成工作则位于网络层。

LTE空中接口用户面协议

LTE空中接口控制面协议

用户面只有物理层和数据链路层两个层。用户面协议模块的数据链路层包括MAC(Medium Access Control,媒质接入控制)、RLC(Radio Link Control，无线链路控制)、PDCP(Packet Data Convergence Protocal,包数据汇聚协议)三个功能模块。在用户面，它的主要功能是处理业务数据流。发送端将承载高层业务应用的IP数据流经过头压缩(PDCP)、加密(PDCP)、分段(RLC)、复用(MAC)、调度等过程变成物理层可处理的传输块；接收端，将物理层接收到的比特数据流按调度要求解复用(MAC)、级联(RLC)、解密(PDCP)、解压缩(PDCP)成为高层应用可以识别的数据流。

控制面则包括物理层、数据链路层和网络层三个层。控制面协议模块的数据链路层包括MAC(Medium Access Control,媒质接入控制)、RLC(Radio Link Control，无线链路控制)、PDCP(Packet Data Convergence Protocal,包数据汇聚协议)三个功能模块。其功能模块的作用与用户面一致，但是PDCP层功能与用户层稍有区别，即除了对控制信令进行加密解密外海要对控制信令数据进行完整性保护和完整性验证。在控制面中还有网络层的内容，它包括两个功能模块：RRC(Radio Resource Control,无限资源控制)和NAS(Non Access Stratum,非接入层)。UE(用户)和eNodeB(基站)之间的控制信令主要是无限资源控制RRC消息。它相当于地方军区的一个司令部，其作用是负责军区资源管理和给各个下属单位下达任务和命令。另一方面还得接收来自中央军委(高层)的命令。RRC相当于eNodeB内部的一个司令部，RRC消息携带建立、修改和释放数据链路层和物理层协议实体所需的全部参数。另一方面，它还要给UE用户端透明传达来自核心网的指示。RRC模块的主要功能有系统信息的广播、寻呼、RRC连接管理、无线资源控制和移动性管理等。
