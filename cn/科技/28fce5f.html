Spring Statemachine 状态机入门

2020-10-29
| 科技

什么是有限状态机

有限个状态，并且可以在这些状态之间进行转移并执行相应的动作的行为模型。

状态机作用

举例：十字旋转门多用于旅游景地。而且是状态机的典型案例。

进：刷卡->推动->进->锁定

我们以前代码：If(投币) 解锁 else if(推) 进入，锁定。

用了状态机：我们把状态和事件进行分离（这是核心）。

State {解锁，锁定 }

Event {投币，推}

状态机使用场景：可以用枚举业务场景，用大量if判断业务状态并进行下一步业务状态流转。

Spring Statemachine 状态机

系统要求：

Spring Statemachine 是使用JDK 8（所有工件都具有JDK 7兼容性）和Spring Framework构建和测试的。它不需要在其核心系统内的Spring Framework之外的任何其他依赖项。

配置类

Action：事件被相应后相执行转换之间的动作

Guard：哨兵 用于是否可以从source到target的转换，如果true,就执行Action，并且把当前状态转到target，如果false,当前状态还停留在source上。

statecontext包含了上下文信息，你可以传自定义对象，还可以从中获取源状态，目标状态，当前事件，等

source:源状态

target:目标状态

event:相应的事件

withExternal: 外部转换，有状态转移.

运行类

相关名词解释

Transition: 转换是源状态和目标状态之间的关系。它可能是复合转换的一部分，它将状态机从一个状态配置带到另一个状态配置，表示状态机对特定类型事件发生的完整响应。

Initial State:初始状态

End State：业务结束状态

Fork State: 可以从一个状态进入两个以上的目标状态

Join State：从多个源状态到一个目标状态。

状态机事件

OnStateMachineStart :Statemachine启动时相应的事件

OnStateEntry：从一个状态进入时相应的事件

OnStateExit：从一个状态退出时相应的事件

OnTransition：从一个状态转换到另一个状态的事件

OnStateMachineError：状态机出错时事件
