Java基础

一：java的基本类型，

java有8种基本类型，分别是int,double ,char,byte，short，long，float,boolean

按照分类来说，其基本数据类型分为：整数类型、浮点类型、字符类型、布尔类型这四个类型。

整数类型：字节型（byte）、短整型（short）、整型（int）、长整型（long）这四种；

浮点类型：分为float（单精度型）和double（双精度类型）这两种。float（单精度型）一般在精度要求不高的时候使用。double（双精度类型）一般在数值精度高的时候使用。

布尔类型：一般用true表示“真”、false表示“假”。用关键字boolean来定义；

java里面采用了一个BigDecimal类来控制精度，具体如何使用大家可以自行查询，

常见的操作有这样：

加减乘除如下：

add(BigDecimal augend)

subtract(BigDecimal subtrahend)

multiply(BigDecimal multiplicand)

divide(BigDecimal divisor)

这个问题也是也是一个常见的面试题目，在金融行业经常会基于BigDecimal类来封装常见的金额操作，比如精度控制，比如分，元的转换等。

如果你用double,float等来做金额计算，会发现得不到你想要的结果，甚至可能在不同的场景下等到的结果也会不一样。

其实这个是阿里巴巴java开发规约中提到的：

【强制】小数类型为decimal，禁止使用float和double。

说明：float和double在存储的时候，存在精度损失的问题，很可能在值的比较时，得到不正确的结果。如果存储的数据范围超过decimal的范围，建议将数据拆成整数和小数分开存储

二：参数传递、

Java的参数到底倒是值传递还是引用地址传递？分两种情况：

1.当基本类型（8种基本类型）作为参数传递时，是传递值的拷贝，无论你怎么改变这个拷贝，原值是不会改变的

2、对象作为参数传递时，是把对象在内存中的地址拷贝了一份传给了参数。

例如1，基本类型传递；

public class Example1{

public static void main(String[] args) {

int i = 4;

System.out.println("改变之前, n = " + n);

changeData(n);

System.out.println("调用changeData(n)之后, n = " + n);

}

public static void changeData(int number) {

n = 14;

}

}

执行下看打印结果：

改变之前, n = 4

调用changeData(n)之后, n = 3

例如2，Java里面的String 比较特殊，如果是这样定义的

String str = “Java按照值传递”;

那么它的传递方式和java的基本类型传递方式是一样的，

因为String的API中有这么一句话：“their values cannot be changed after they are created”， 表示string对象一旦被创建是不能被改变的；

如果是 这样定义：

String str = new String(“Java按照值传递”);

这种方式创建的字符串不放入常量池，是在堆中复制改变量的副本，会将堆中的对象的地址复制给str;；

例如3，当对象作为参数传递的时候，会出现什么情况呢？

public class Example1{

public static void main(String[] args) {

int n = 4;

System.out.println("改变之前, n = " + n);

Example1 example1 = new Example1();

example1.changeData(n);

System.out.println("调用changeData(n)之后, n = " + n);

}

public void changeData(int number) {

number = 14;

}

}

先看输出结果：

改变之前, sb = Hello

A调用changeData(n),之后 sb = Hello World!

对象作为参数传递时，是把对象的引用传递过去，如果引用在方法内被改变了，那么原对象也跟着改变；

所以：java中没有引用传递，只有值传递，引用类型属于特殊值传递（是将它的地址副本给了参数，但是它与基本数据类型不同，如果地址指向的对象发生了变化，因为共享原因，原始对象也会改变）。
