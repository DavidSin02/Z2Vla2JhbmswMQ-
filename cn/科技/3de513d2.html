GitHub怎么了？连续3天出现严重宕机情况 微软未回应

2020-11-14
| 科技

最近几周，GitHub连续多次出现了宕机事件，不过微软官方始终没有透露原因。特别在过去的三天时间内，有开发者反馈GitHub中断的情况非常严重，甚至于基本的编程任务都无法执行。

正如外媒ZDNet在一篇文章中所指出的那样，Mary Jo Foley详细介绍了过去几天GitHub服务受到的影响：

4月21日，由于某些问题导致部分GitHub服务中止服务，持续了一个半小时。 4月22日，GitHub又出现两次宕机情况，影响了至少2个小时的时间。而今天，也就是4月23日，GitHub多项服务出现了各种问题，开始时间大约是美国东部时间上午9点20分，持续时间一个半小时。

今天的宕机对Git Operations; API Requests; Webhooks; Issues, pull requests and projects; GitHub Actions; GitHub Packages; GitHub Pages等产品产生影响。在GitHub的状态网站上，只有Git Operations被列为“ incident”，其他服务被标注为“degraded”。"

在Twitter上，GitHub Education承认他们的服务出现了中断，尤其是GitHub Classroom和education.github.com。
