聪明人的带宽｜42章经

2020-11-14
| 科技

｜本文来自于连续创业者 李天放

我们之前发过两篇绿洲资本张津剑的文章，分别是

投资中的信号与噪声丨42章经

投资决策中的信噪比｜42章经

这两篇都是从信号与系统的角度来解释创投市场，今天的文章可以看做是对其的一个补充，讲的是“带宽”对于沟通和创业者的意义。

我相信从某个层面来说，团队的带宽会间接甚至直接决定创业的成败，而个体的带宽则是我们对自我智力的永恒挑战与追求。

带宽很重要。

假如网速带宽太低，游戏玩不好、视频看不了、再好的网站也用不了。

带宽的单位是 bit per second 或每秒钟的信息传达效率。人与人之间的沟通也是一种信息传达，也可以用带宽来衡量。沟通带宽衡量的不是说话速度或内容长度，而是效率。有些话特别多的人说了半天其实什么都没说。反过来，有些人可以用一句话甚至一个词抓住一件事的重点。

就算你没想过带宽这个概念，在每天生活中也能感受到它的存在。

在工作中与带宽低的人讨论事情会"说不清楚"，在生活中与带宽低的人聊天会"没有共同话题"。带宽直接影响到我们的事业、社交、幸福指标。

在其它同等条件下，思维清晰的人带宽会更高，因为想高效传输一个概念，首先自己要想清楚。知识广泛的人与词汇量高的人带宽会更高，因为他们更容易运用恰当的词与精准的类比。

高带宽是一种技巧。有时我们需要讲故事、有时我们需要找到一个好的比喻、有时我们需要把一个概念抽象化、有时画一张图是最有效的（A picture is worth a thousand words)。

带宽不仅是每个人身上的属性，同时也是两个人之间的属性。

产品经理的面试中有一类题如下："你怎么跟你的父母介绍这个产品？"或"你怎么跟 18 世纪的人解释这个产品？"。这类问题考的是怎么在低带宽的情况下讲清楚一个产品。你的父母和 18 世纪的人都不笨，难的是你与他们之间缺乏一个共同的 Context (共同语言与知识重合度)。

Context 是人与人之间提高带宽的关键，有共同 Context 就可以用熟悉的类比来简化复杂的概念。

比如，我在讨论狭窄领域的问题时喜欢用比喻的方式来说明，类似用生活中常见的约会的概念来解释对冲基金等。

同理，影友间可以用一部电影形容另一部电影，比如"太空版的泰坦尼克"或"变形金刚版的奥特曼"。创业者之间经常用一个产品来描述另一个产品, 比如"给企业用的微博"或"给艺术家用的 GitHub"。这就是 Context 对沟通宽带的提高作用。

对于网络带宽我们可以通过压缩来提高效率。

一种压缩方式就是用更短更紧凑的方式去形容数据。比如你要传输"00000000000000000000", 可以只传输"20 x 0"。同样，我们也可以压缩沟通内容。一个小时的会也许可以压到半个小时或者干脆不开会，用发邮件代替。一封长邮件很多时候可以简化成一封短邮件。用一张图有时可以省略几百个字。

另外一种压缩方式是利用差别。

视频中每一张图与上一帧图之间的变动不大，只存两张图之间的差别就可以让文件变得小几十倍。有共同 Context 的人很多时候只描述差别就够了。假如你跟一个非程序员讲 Ruby，需要从零开始讲，但跟一个 Java 程序员讲 Ruby 只需要提到 Ruby 和 Java 之间的差别就差不多了。

带宽效应在团队中会放大。

假如两个人之间的沟通成本是 n，团队之间的沟通成本就是 n^2。理想情况下团队里面每个人都应该有很强的沟通能力，现实情况中大部分团队会自然产生一些沟通节点，大多数团员会通过少数中间人来传递信息。这些人"人缘好"，"善解人意"，其实就是带宽高。

有些职位比如产品经理，是天生的沟通节点，这些职位更需要带宽高的人。

最终，带宽不仅存在于人与人之间的信息传达。编程其实就是人与电脑之间的指令传达。设计中的 「 Human Computer Interaction」就是人与软件之间的对接论。

对于创业者与创造者，提高带宽的目的并不是把自己变成一个能言善辩的销售能手，而是让自己能够更便捷的把脑子里的零碎想法变成现实。

｜n^2 沟通成本可参考《人月神话》一书
