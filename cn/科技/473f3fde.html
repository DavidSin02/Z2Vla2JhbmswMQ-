Linux文件系统简介

在Linux操作系统中，所有被操作系统管理的资源，例如网络接口卡、磁盘驱动器、打印机、输入输出设备、普通文件或者是目录都是文件

在LINUX系统中有一个重要的概念：一切都是文件。 其实这是UNIX哲学的一个体现，而Linux是重写UNIX而来，所以这个概念也就传承了下来。在UNIX系统中，把一切资源都看作是文件，包括硬件设备。UNIX系统把每个硬件都看成是一个文件，通常称为设备文件，这样用户就可以用读写文件的方式实现对硬件的访问。

目前在Linux操作系统中支持很多种文件系统，包括Ext2、Ext4、Btrfs和XFS等，多达几十个文件系统。虽然支持的文件系统种类很多，但从用户层面使用方式无任何差别，用户并不感知其中的差异。对于普通用来说，数据都是组织成上文所述的树状结构。

Linux操作系统对各种文件系统的支持是通过名为VFS的组件实现的，也就是虚拟文件系统（Virtual File System）。如下图所示，VFS作为一个抽象层，为用户提供统一的接口，屏蔽了其它具体文件系统（例如Ext4和Btrfs等）的实现。VFS为用户提供了open、close、read和write等接口。

文件类型与目录结构

Linux支持5中文件类型：

  1. 普通文件：用来在辅助存储设备（如磁盘）上存储信息和数据
  2. 目录文件：用于表示和管理系统中的文件，目录文件中包含一些文件名和子目录名
  3. 链接文件：用于不同目录下的文件共享
  4. 设备文件：用来访问硬件设备
  5. 命名管道FIFO：一种特殊类型文件，Linux系统下，进程间通信可以通过该文件完成

常见目录说明：

  1. /bin 目录是包含一些二进制文件的目录，即可以运行的一些应用程序。 你会在这个目录中用于新建和删除文件和目录、移动它们基本工具等
  2. /boot 目录包含启动系统所需的文件。如果你在这里弄乱了其中一个文件，你可能无法运行你的 Linux，必须拥有超级用户权限才能执行此操作
  3. /dev 目录包含设备文件。 其中许多是在启动时或甚至在运行时生成的。 例如，如果你将新的网络摄像头或 USB 随身碟连接到你的机器中，则会自动弹出一个新的设备条目
  4. /etc 是要配置的所有内容，它包含大部分系统配置文件。 例如，包含系统名称、用户及其密码、网络上计算机名称以及硬盘上分区的安装位置和时间的文件都在这里
  5. /home 是可以找到用户个人目录的地方
  6. /lib 是库文件所在的地方。库是包含应用程序可以使用的代码文件。它们包含应用程序用于在桌面上绘制窗口、控制外围设备或将文件发送到硬盘的代码片段。在文件系统周围散布着更多的 lib 目录，但是这个直接挂载在 / 的 /lib 目录是特殊的，除此之外，它包含了所有重要的内核模块。 内核模块是使你的显卡、声卡、WiFi、打印机等工作的驱动程序
  7. /root 是系统的超级用户（也称为“管理员”）的主目录。 它与其他用户的主目录是分开的
  8. /sbin 与 /bin 类似，但它包含的应用程序只有超级用户（即首字母的 s ）才需要。你可以使用 sudo 命令使用这些应用程序，该命令暂时允许你在许多 Linux 发行版上拥有超级用户权限。/sbin 目录通常包含可以安装、删除和格式化各种东西的工具
  9. /usr 目录是在 UNIX 早期用户的主目录所处的地方
  10. /var用于存放运行时需要改变数据的文件，也是某些大文件的溢出区，比如各种服务的日志文件等
