设计模式-适配器模式

2020-10-29
| 科技

一、适配器模式的定义

适配器模式解决类的不兼容问题，在一些场景下，接口定义的与接口使用方可能存在不一致的地方，但是又需要复用这些接口，就需要就行接口的转换，自己定义一个相同功能的接口作为转化。让实现类同时拥有多个相同方法的接口，这样就可以达到兼容的目的。

适配模式分为三种形式的适配：类适配器、对象适配器、接口适配器

适配器模式

二、3种适配器示例

1.类适配器

类适配器顾名思义就是适配一个类，我们示例分为以下流程

1） 创建待适配的对象，我此我们创建一个Peple对象

public class People {
public void drive(){
System.out.println("这个人在开车~");
}
}

对象有一个行为，驾驶

2） 创建创建需要适配的接口Action

public interface Action { /** * 会开火车 */ void drive(); /** * 可以走路 */ void walk(); /** * 还可以乱跑 */ void run();}

drive()方法是需要与people适配的

3） 创建适配器

/*** * @description :  * 类适配器， people是待适配的对象， action是适配的目标， * 这样就将待适配的对象扩展了带有目标的功能 * 目标也有了待适配对象的功能 * 通常在源于目标有重叠功能时 */public class ClassAdapter extends People implements Action { @Override public void walk() { System.out.println("这个人在步行"); } @Override public void run() { System.out.println("这个人在跑步~"); }}

通过以上适配的过程，Action拥有了People的dive ()行为，很多时候还有不同的表示方式，在此没有在ClassAdapter的dive（）做任何的修饰，当然也可以重写，改变其行为模式。

2.对象适配器模式

类适配采用继承的方式实现，而对象适配采用成员变量的方式实现，如果不加任何修饰，有点像静态代理模式，如果加以修饰，就变成了装饰模式，我们来看看具体的代码，我们仍然用类适配器中的People对象与Action接口

public class ObjectAdapter implements Action { private People people; public ObjectAdapter(People people){ this.people = people; } @Override public void drive() { people.drive(); } @Override public void walk() { System.out.println("具有了行走了能力"); } @Override public void run() { System.out.println("具有跑的能力"); } public static void main(String[] args) { Action action = new ObjectAdapter(new People()); action.walk(); action.run(); action.drive(); }}

我们把Action的drive()行为直接代理到People成为去实现，达到了转换的目的

3. 接口适配

适配接口，我们在定义一个HighAction作为高级行为，和Action中有共同的run()行为一致

public interface HighAction { /** * 可以飞，能上天 */ void fly(); /** * 可以跑，能下海 */ void run();}

我们在整合两种行为在一起，对其进行适配

public class AbstractAction implements Action, HighAction { @Override public void drive() { System.out.println("驾驶"); } @Override public void walk() { System.out.println("走路"); } @Override public void run() { System.out.println("奔跑大家都会，蜗牛都可以狂奔"); } @Override public void fly() { System.out.println("可能会飞了~"); }}

那么他们就都有了run()的行为了，行为一致

适配器模式的目的就是要将不同对象的行为转化成一致，不同的场景下用不同的方式去实现适配。
