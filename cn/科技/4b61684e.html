「Spring Boot专题」CORS跨域

2020-11-14
| 科技

什么是CORS

CORS是一个W3C标准，全称"跨域资源共享" (Cross-origin resource sharing)。

它允许浏览器向跨域服务器进行XMLHttpRequest请求，从而解决AJAX只能进行同源请求的限制。

跨域需要浏览器和服务器同时支持，现在的浏览器都支持跨域，老版本的IE不在此范围内。

@CrossOrigin

浏览器跨域的问题不需要开发人员去考虑。我们只需要关注后端怎么支持跨域。

在Spring中提供了 @CrossOrigin 注解，可以让我们快速的实现后端Controller请求跨域支持。@CrossOrigin可以在两个地方去使用：

  1. 类
  2. 方法

在Controller类上去使用@CrossOrigin，则该Controller内部的所有Http请求都支持跨域。

@CrossOrigin(origins = "http://external-system.com", maxAge = 3600)@RestController@RequestMapping("/cors")public class CorsController {    // ...}


在方法上使用，则被标记的方法支持跨域。

@CrossOrigin(origins = "http://external-system.com", maxAge = 3600)@GetMapping("{code}")public Product getProductByCode(@PathVariable String code) {    // ...}


CorsRegistry跨域

使用@CrossOrigin方式解决跨域时，只有被标记了的Controller或Method才能支持跨域。

Spring提供了CorsRegistry，可以用来注册跨域规则，实现全局跨域。

    @Bean    public WebMvcConfigurer corsConfigurer() {        return new WebMvcConfigurer() {            @Override            public void addCorsMappings(CorsRegistry registry) {                registry.addMapping("/rest/v2//**").allowedMethods("PUT", "DELETE");            }        };    }

Filter跨域

除了使用CorsRegistry，我们还可以是用Filter的方式，实现全局跨域。

@Beanpublic FilterRegistrationBean corsFilter() {    final UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();    final CorsConfiguration config = new CorsConfiguration();    config.setAllowCredentials(true);    config.setAllowedOrigins(Collections.singletonList("http://sample-domain.com"));    config.setAllowedHeaders(Arrays.asList("Origin", "Content-Type", "Accept"));    config.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "OPTIONS", "DELETE", "PATCH"));    source.registerCorsConfiguration("/**", config);    FilterRegistrationBean registration = new FilterRegistrationBean(new CorsFilter(source));    return registration;}

总结

CORS标准的存在可以使我们的应用安全性更高一些，避免我们的网站服务被其他人乱用。
