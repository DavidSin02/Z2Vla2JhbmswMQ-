选择器快速短路

选择器是CSS的一部分，它告诉浏览器目标代码的哪一部分。 基本选择器的目标是HTML标签，例如h1，p和img标签。 定位根标记后，您应用的所有样式都会成为"全局"样式的一部分。

这是通过CSS选择元素的最简单方法。

许多刚接触CSS的开发人员往往会做的事情，尤其是当事情进展不顺利时，是在CSS文件中进一步添加了另一个选择器规则块。

由于级联，所写的内容从上到下依次应用。 这会产生覆盖效果，因此不建议使用，因为它会创建垃圾代码。 这也使跟踪问题的根本原因变得困难。

编写CSS规则时，仅在一个地方的特定情况下为特定选择器提供一个属性。

这意味着，如果您已经声明了带有color属性的p选择器，则不应稍后在文件中更改它，除非它处于其他状态，例如：hover

随着CSS样式的增长，这将省去跟踪选择器及其属性的麻烦。

您还可以通过类和id选择器来定位HTML的特定块。 如今，ID只能使用一次，并且通常保留给JavaScript挂钩。 类通常是HTML块的一种更标准的方法，并且本质上比id更可重用。

类通过HTML中的class属性声明，并由CSS使用调用。 作为前面的符号。 选择特定的id是相同的概念，其中id是属性，而＃是前面的符号。

在DOM中，彼此直接相邻的标记称为同级。 这意味着h1和p是同级，而h1和ul不是同级。 兄弟姐妹必须彼此相邻，否则放下h1 + ul将会打击存在的每个可能的ul，无论其接近程度如何。

使用+号时，可以创建选择条件。 它告诉您的浏览器选择相邻的兄弟姐妹，即直接位于另一个指定元素旁边的元素组合对。

伪元素

伪元素将元素之前和之后的"空格"作为目标。 HTML中并不存在实际的空间，但是CSS插入了一个空容器，为您提供了一个可操作的空间。

让我们看一下CSS以及将内容插入到哪里。

.item:before { content: "icon.jpg"; }.item:after { content: "anotherIcon.jpg"; }

与：target和#someLinkHere有所不同

在HTML中，您可以链接页面内的内容而无需刷新它。 您可以通过和href属性执行此操作。 当在假定的链接前放置＃时，它将在DOM中查找相关的ID，并将页面向下滚动到适当的位置。

代码看起来像这样：

<a href="#sectionOne">Section One</a><a href="#sectionTwo">Section Two</a>… some more code here …<div id="sectionOne">some content here</div>………<div id="sectionTwo">some content here</div>

当您单击第一部分时，浏览器将查找href部分中设置的相应ID，并将您带到页面上的该特定位。 在样式方面，：target可让您配置选择并导航至页面上目标链接后的视觉效果。

因此，您的CSS可能看起来像这样：

:target { border:solid 2px black; }

这将导致目标宿主元素（又称div，基于上面的示例）周围的边框。 当您单击其他目标元素时，样式将从原来的样式中删除，并应用于新的目标。
