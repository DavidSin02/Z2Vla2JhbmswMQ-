Git，是目前世界上最先进的分布式版本控制系统。

版本控制系统（version control system），是一种记录一个或若干文件内容变化，以便便将来查阅。

特定版本修订情况的系统。版本控制系统不不仅可以应⽤用于软件源代码的⽂文本文件，而且可以对任何类型的文件进行版本控制。


就像哆啦A梦的时光机，它记录了了我们每一个时间段做过的每一件事情，因此我们可以随时通过时光机回到过去。

版本控制系统除了了 Git 这种分布式以外，还有一种集中式。

集中式

最早期的时候，为了了让不同系统上的开发者能够协同工作，诞生了集中式版本控制系统（CVCS）。

CVCS 都有一个单一的集中管理理的服务器，用来保存所有文件的修订版本。而协同工作的人们都通过客户端连接到这台服务器，获取最新的文件或者提交更新。


集中式的缺点

CVCS 有一个最显而易见的缺点，就是中央服务器的单点故障问题。如果负责集中管理的服务器死翘翘了，那么就会导致谁都无法提交更新的，也就⽆无法协同工作，那我们只能集体去喝茶聊人生了；

如果磁盘发生故障，而备份又不够即时，那么就有丢失数据的风险，最坏的情况是丢失整个项目的历史更改记录，那我们只能以身相…不，不是，是以死谢罪了了！

分布式

正是由于集中式版本控制系统的缺点，分布式版本控制系统问世了（DVCS）。

DVCS 没有“中央服务器”，每个人的电脑上都是一个完整的版本库，这样，我们在工作的时候，就不需要联网了，因为版本库就在自己的电脑上。

但是既然每个人电脑上都有一个完整的版本库，那多人之间如何协作呢？例如你在自己电脑上改了文件A，你的同事也在他的电脑上改了文件A，这时，你们俩之间只需把各自的修改推送给对方，就可以互相看到对方的修改了。


关于 Git

Linus 在 1991 年年创建了开源的 Linux，从此，Linux 系统不断发展，成为最大的服务器系统软件了。


Linus 虽然创建了 Linux，但 Linux 的壮大是靠全世界热心的志愿者参与的，世界各地的程序员都在为Linux 编写代码，那么这么多的代码怎么管理？那个时期集中式版本控制系统 CVS、SVN 等已经诞生了，但是 Linus 义正言辞斩钉截铁铿锵有力……的拒绝了！

因为集中式版本控制系统不但速度慢，而且必须联网才能使用，Linus 表示：不！喜！欢！

到 2002 年，BitMover 公司出于人道主义精神，授权 Linux 社区免费使用他们公司本来要收费的BitKeeper 版本控制系统。Linus 表示：很喜欢嘻嘻嘻….

但是安定团结的大好局面在 2005 年被打破了。

因为 Linux 社区中开发 Samba 的 Andrew 试图破解BitKeeper 的协议，结果被 BitMover 公司发现了，于是 BitMover 公司怒了，收回了了 Linux 社区对于

BitKeeper 的免费使用权。

正常剧情接下来应该是 Linus 向 BitMover 公司道个歉，然后大家握手言和执手相看泪眼……

但是，Linus 表示：呸！老子就不！

于是 Linus 花了两周时间自己用 C 语言写了一个分布式版本控制系统，Git 就此诞生了！一个月之内，Linux 系统的源码就已经由 Git 管理了。

Git 工作流


Workspace：工作区

Index（Stage）：暂存区

Repository：仓库区（本地仓库）

Remote：远程仓库

当然，如果需要，你随时可以用相同的命令修改已有的配置。

Git 本地仓库的使用，添加文件到仓库


要将一个本地文件添加到本地仓库，主要分为四步：

1. 创建本地仓库（版本库）

2. 将本地文件添加至工作区

3. 将⼯工作区文件添加至暂存区

4. 将暂存区文件提交至仓库

创建版本库

版本库，中⽂文名又叫"仓库"，英文名叫"repository"。它其实就是⼀一个本地的目录文件，用来保存我们项目中的所有的文件，然后统一交给 Git 管理。

创建一个版本库非常简单，主要分为两步：

第一步，选择一个合适的地方，创建一个空目录（例如我在桌面创建了一个 testgit ⽂文件）。

第二步，在命令执行工具中进入目录路路径，执行 git init 命令。


执行完成后，就可以看到在刚刚的空目录文件中生成了一个 .git 文件，这个 .git 就是我们的本地仓库。


注意： .git 是一个隐藏文件，有些电脑默认不显示隐藏文件，需要去更改一下设置。

工作区

这个包含了 .git 文件的目录（就是我们一开始创建的空目录，我创建的 testgit 目录），就是我们的工作区目录。

如果后续我们需要向本地仓库提交文件的话，这个文件就必须在工作区目录中。

例如，我们想要提交一个 readme.md 的文件至本地仓库。

那么首先需要在工作区中添加该文件。


添加文件至暂存区

当我们想要把工作区的文件交给 Git 来管理时，就需要将文件提交到 Git 仓库，但是在提交到仓库前还有一个步骤，就是先将文件添加到暂存区。

执行命令 git add 文件名 （ 文件名 输入我们要添加的文件名，例如： readme.txt ），将文件添加至暂存区。


没有任何提示则说明添加成功。

版本回退

当我们对文件做了多次修改后，又想要回退到之前的某一个版本。




例如这里我对 readme.md 文件进行了三次修改。

我们可以通过命令 git log 查看修改记录。

如果我们现在想要回退到上一个版本，可以通过 git reset --hard HEAD~1 来实现。




想要回退到上面几个版本，命令最后的数字就写几。例如我们想要回退到前 100 个版本，就执行命令git reset --hard HEAD~100 。

但是，实际操作中，我们不可能挨着挨着去数要回退的是第几个版本。

因此，我们还可以通过 commitid 来进行指定版本的回退。




例如我现在要回退到第一次修改。可以通过命令 git reset --hard b83bd 来实现。




命令最后跟的就是 commit id ，不需要写全，写前几位 Git 就会自动去查找。
