对大数据与商务智能的粗浅认识

2020-11-14
| 科技

进入21世纪，全球信息革命的一个重要标志就是大数据时代的到来。究竟什么是大数据？为什么在21世纪兴起？大数据对人类社会的发展有什么影响？我们将如何应对？本文将带着大家一起沿着信息革命的历程来探究这些问题的答案。

什么是大数据？

大数据（Big Data）实际上是一种“俗称”，就如“云计算”的命名一样，都是为了让社会民众更好的了解这些专业领域的新生事物而起的一种通俗的名称。

大数据实际上指的就是信息系统产生的海量数据，因为此类数据具有规模庞大、组成复杂、来源多样、格式各异、持续涌现的特性，在如何处理和发挥其价值方面存在诸多困难、面临很多挑战，我们才用“大”这个词对其进行修饰。大数据的“大”既体现了其规模大（Big Volume），也体现了其价值“大”(Great Value)，更说明了它是个不容易对付的“大”问题(Big problem)。

目前大数据主要产生和应用的领域包括：

  • IT、通信领域及物联网领域
  • 电子商务及互联网领域
  • 交通与物流领域
  • 商业服务领域
  • 金融领域
  • 医疗健康领域
  • 气象服务领域
  • 以上领域都是和民众日常工作、生活息息相关的业务领域，还有很多专业领域如天文、宇航领域、工业控制领域等等，也是大数据应用的主战场。

大数据为什么在21世纪兴起？

大数据出现的根本原因，也是其发展的基本动力，就是信息技术的进步及普及应用，也可以说是信息化发展的结果。

随着信息技术的普及应用以及各行各业信息化的发展，信息系统所产生的数据越来越多，产生数据的速度越来越快，不断积累的海量数据逐渐成为组织运营的负担，同时海量数据中蕴含的“规律”、“经验”等知识也逐步引起大家的兴趣和重视，“大数据”随之应运而生。可以说，我们对大数据的爱与恨、悲与欣是交织在一起的。

“大数据”并非在21世纪出现，早在20世纪90年代就初现端倪。数据仓库之父比尔·恩门（Bill Inmon）在1991年出版的“Building the Data Warehouse”（《建立数据仓库》）一书中就已经提醒社会对海量数据的重视。从那时起，对大数据的研究就进入了一个小高潮，突出表现在数据仓库和商务智能技术的诞生。当时社会对大数据的认识主要集中在专业领域，没有达到现在“全民化、全球化”的“现象级”水平。

但是随之几年后进入低潮期，主要原因是受到当时IT资产性能水平和性价比的限制，很多技术、功能“看上去很美”但无法实现，无法普及应用。直至21世纪IT资产的性能水平大幅提升、建设成本显著下降，大数据技术的应用普及逐步走入寻常百姓之家。

大数据会对人类社会产生哪些影响？

任何事物对人类社会的影响都分为正反两个方面，而且可以相互转化，大数据也不例外。
  • 大数据首先给我们带来的就是沉重的管理负担。

这一点从数据容量单位上就可以看出来。20世纪90年代以Mb计量数据就已经是天量的感觉了，后来短短几年就上升到Gb、Tb、Pb、Eb、Zb，数据量级呈几何级数增长，社会组织在数据存储和管理方面付出了高额的代价。以银行业为例，每TB单位就要花费10万元的运维成本，设想一下，在电子银行成为主流服务形式的时代，银行业每天产生的海量数据将会给银行增加多少费用支出？有机构统计，银行业每创收100万，平均就会产生约1TGB的数据，仅数据运维成本就占到总收入的10%！

  • 大数据的泛滥会给我们带来极大的社会风险

大数据来源广泛，内容各异，良莠不齐，会制造很多社会动乱。目前互联网谣言四起、小道消息满天飞，不但影响了社会组织的正常运营，也给民众生活带来很多困扰，小则误导人们做出错误判断，大至影响国家安定团结和世界格局的稳定。以当年日本福岛核电危机为例，“食用海盐防辐射”的谣言就让“粗粒海盐”抢购一空，各家各户囤积海盐，不但影响了民众的日常生活，而且成了全球的笑柄。在金融领域，“虚假消息”的快速蔓延对股市、汇市、期货市场的影响更加严重，引发全球经济波动，“蝴蝶效应”凸显，全球经济危机一触即发。

  • 大数据是管理创新、技术创新的重要驱动力

大数据的价值之一就是决策支持。在大数据的支持下，组织管理决策将获得更多的信息支持，保证管理者更加全面地认识问题、分析问题，做出更加准确的判断和决策，从而大幅提高组织的经营管理水平和运营效率，提升组织的竞争力。

大数据的价值之二就是智能化。所谓智能化就是应用人工智能技术，让组织的业务流程可以自行运转，提高业务处理的效率和质量。无人办公、无人商场、自动驾驶、智能交通、智能医疗就是非常典型的应用。而人工智能的核心是机器学习，机器学习脱离了大数据的支持，就成了无源之水、无本之木。近几年人工智能脍炙人口，与大数据的发展与支持是息息相关的。

  • 大数据能够提供更好、更多的民生服务¶

民众的日常生活离不开信息服务，更高质量、更加丰富的信息服务使我们的日常生活变得更加便捷。在交通、旅游、购物、餐饮、教育、医疗等关键民生领域，我们目前无时无刻不在享受大数据的服务。就以旅游服务为例，以前我们采取的基本模式就是通过旅行社获得服务，既不灵活也不自由成本也高，原因在于我们获得信息的渠道太少。如今自驾游、自由行基本成为民众旅游的主要形式，这全依赖于丰富的旅游服务信息和交通服务信息，从选择景点到购买门票、预订酒店、预订车票都可以在出行前一气呵成，回退到十几年前，这是不可想象的。

如何应对大数据？

将大数据比作垃圾不太合适，但是如果没有正确的处理，大数据就会变成信息世界的垃圾，引发严重的环保问题。

幸运的是，在大家的共同努力下，目前我们处理大数据的思想、方法、技术和工具不断涌现，不断进步，为我们处理这个“大麻烦”提供了极大的帮助，我们正经历“变废为宝”的革命历程。以下我们将从平台级、工具级、应用级分别介绍应对大数据的各种优秀措施。

  • 大数据平台级处理

大数据平台级处理主要完成对大数据的存储管理、检索支持。这一方面与我们熟悉的数据库平台极为相似。

在平台级应用中，首当其冲的就是Hadoop平台。Hadoop是大数据领域主流的平台级应用。Hadoop实现了一个分布式文件系统（Hadoop Distributed File System），简称HDFS。HDFS有高容错性的特点，并且设计用来部署在低廉的（low-cost）硬件上；而且它提供高吞吐量（high throughput）来访问应用程序的数据，适合那些有着超大数据集（large data set）的应用程序。目前全球知名的互联网信息服务基本采用了这一技术。Hadoop本身还是一个大数据处理工具的大家庭，内置了很多优秀的数据处理和数据分析的工具。

其次是文档数据库和内存数据库平台。这两个领域代表性的产品为MongoDB文档数据库和Redis内存数据库。

MongoDB是一个介于关系数据库和非关系数据库之间的分布式非结构化数据库产品，是非关系数据库当中功能最丰富，最像关系数据库的。它支持的数据结构非常松散，是类似json的bson格式，因此可以存储比较复杂的数据类型。Mongo最大的特点是它支持的查询语言非常强大，其语法有点类似于面向对象的查询语言，几乎可以实现类似关系数据库单表查询的绝大部分功能，而且还支持对数据建立索引。目前深为中小型大数据应用所青睐。

Redis 是一个高性能的key-value内存数据库，支持对高频访问数据的高速访问。它支持string(字符串)、list(链表)、set(集合)、zset(sorted set --有序集合)和hash（哈希类型）数据类型。为了保证效率，数据都是缓存在内存中，同时redis会周期性的把更新的数据写入磁盘或者把修改操作写入追加的记录文件，并且在此基础上实现了master-slave(主从)同步，从而保证数据在服务器断电后的恢复。

  • 大数据工具级处理

大数据工具级处理主要完成对数据的采集、清洗、变换和加载，也就是平时所说的ETL操作。采集操作主要从各类数据源采集所需要的数据，这些数据源包括：文本文件、数据库和在线实时数据；清洗操作主要用于清除数据中的“垃圾数据”，这些数据包括：非法数据（数值超出合法范围）、噪声数据（因采集方法或数据源本身的原因而产生的干扰数据，比如数据在传输过程中出现的畸变、测量误差等）和严重残缺数据（信息内容不完整，无法推测补全的数据）；变换则根据数据格式和取值类型的标准，对多种表示的同类数据进行重新表达，以便后续处理；加载则完成将数据输送到应用处理环节，或者补充到数据库中。

目前ETL方面的工具非常多，有产品级的（如Pentaho），也有构架级的(如Storm和flink，见hadoop体系结构图)等，可以满足不同场景的数据ETL处理需要。

大数据应用级处理

大数据应用级处理分类比较多，我们在此将其分为基本信息服务和商务智能两个领域。（不算完整、严谨的一种狭义分类）

基本信息服务专指将数据汇集在一起，提供基本的信息检索服务，比如天气预报、交通信息服务等等，在此不再赘述。

商务智能(Business Intelligence),也称商业智能。业界普遍认为商务智能的概念在1996年最早由加特纳集团（Gartner Group）提出，加特纳集团将商业智能定义为：商业智能描述了一系列的概念和方法，通过应用基于事实的支持系统来辅助商业决策的制定。商业智能提供使企业迅速分析数据的技术和方法，包括收集、管理和分析数据，将这些数据转化为有用的信息，然后分发到企业各处。

商务智能目前分为联机分析处理和数据挖掘两个专业领域。其中：

  • 联机分析处理（OLAP）：支持复杂的分析操作，侧重决策支持，并且提供直观易懂的查询结果。OLAP使分析人员、管理人员或执行人员能够从多角度对信息进行快速、一致、交互地存取，从而获得对数据的更深入了解。OLAP的目标是满足决策支持或者满足在多维环境下特定的查询和报表需求。OLAP的基本多维分析操作有钻取（roll up和drill down）、切片（slice）和切块（dice）、以及旋转（pivot）等，并将结果以图表的形式进行直观展现（数据可视化）。目前市面上的商务智能产品多指该类应用。我们此次开设的《商务智能分析师初级进阶课程》就面向联机分析处理。

  • 数据挖掘（Data Mining）：是数据库知识发现（Knowledge-Discovery in Databases，简称：KDD)中的核心环节。一般特指从大量的数据中通过算法搜索隐藏于其中的规则、模式等高级规律性知识的过程。这些挖掘成果被模型化、软件化后，可以内置到信息系统中，实现业务流程的智能化和自动化。目前数据挖掘技术在风险控制、医疗服务、交通物流等领域广泛应用，是大数据应用的热点。数据挖掘通常与计算机科学有关，并通过数理统计、机器学习、专家系统（依靠经验法则）和模式识别等诸多方法来实现上述目标。我们此次开设的《商务智能分析师高级进阶课程》专门面向数据挖掘领域。
