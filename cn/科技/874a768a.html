可重入锁ReentrantLock

2020-11-14
| 科技

0. 说明

在多线程变成的时候，当多个线程需要对临界资源进行操作的时候，这个时候就需要通过加锁的机制，保证在同一时间只能有一个线程操作临界资源，从而保证数据一致性。

这里我们先介绍下什么叫做可重入，在Java中可重入的意思就是当前线程已经获得了锁，但是还可以再次获得（当然，其他线程不能再次获得，至于获得什么，本文当时是获得锁了，锁也算是一种临界资源）。

  • 不可重入锁：当前线程已经获得锁了，当再次获得锁的时候，将获取不到，容易出现死锁。
  • 可重入锁：当前线程已经获得锁了，当再次获得锁的时候，能够获取的到，但是在释放的时候，需要释放多次。


锁分类


1. 不可重入锁

话不多说，直接上代码。

package cn.ideacs.learn.lock;class NoReentrantLock { private boolean lock; public Boolean lock() { if (!lock) { lock = true; return lock; } else { return false; } } public Boolean unLock() { if (lock) { lock = false; return true; } else { return false; } } @Override public String toString() { return "NoReentrantLock{" + "lock=" + lock + '}'; }}public class NoReentrantLockTest { private NoReentrantLock lock; public NoReentrantLockTest(NoReentrantLock lock) { this.lock = lock; } public void methodA() { while (!lock.lock()) { System.out.println("A正在要锁"); } methodB(); System.out.println("A假装进行了一波骚操作"); lock.unLock(); } public void methodB() { while (!lock.lock()) { System.out.println("B正在要锁"); } System.out.println("B假装进行了一波骚操作"); lock.unLock(); } public static void main(String[] args) { NoReentrantLock noReentrantLock = new NoReentrantLock(); NoReentrantLockTest test = new NoReentrantLockTest(noReentrantLock); test.methodB(); System.out.println(noReentrantLock); test.methodA(); }}


运行结果：

B假装进行了一波骚操作NoReentrantLock{lock=false}B正在要锁B正在要锁B正在要锁B正在要锁B正在要锁...


可见B一直在试图获得锁，但是却因为是不可重入锁，所以一直不能获取的到。


2. 可重入锁

一样，话不多说，直接上代码

package cn.ideacs.learn.lock;class MyReentrantLock { private int lock = 0; private Thread lockThread; public boolean lock() { if (lockThread == null) { lockThread = Thread.currentThread(); lock++; return true; } else if (lockThread == Thread.currentThread()) { lock++; return true; } else { return false; } } public boolean unLock() { if (lockThread == null || lock <= 0) { return false; } lock--; if (lock <= 0) { lockThread = null; } return true; } @Override public String toString() { return "MyReentrantLock{" + "lock=" + lock + ", lockThread=" + lockThread + '}'; }}public class MyReentrantLockTest { private MyReentrantLock lock; public MyReentrantLockTest(MyReentrantLock lock) { this.lock = lock; } public void methodA() { while (!lock.lock()) { System.out.println("A正在要锁"); } System.out.println("methodA:" + lock); methodB(); System.out.println("A假装进行了一波骚操作"); lock.unLock(); System.out.println("methodA unLock:" + lock); } public void methodB() { while (!lock.lock()) { System.out.println("B正在要锁"); } System.out.println("methodB:" + lock); System.out.println("B假装进行了一波骚操作"); lock.unLock(); System.out.println("methodB unLock:" + lock); }  public static void main(String[] args) { MyReentrantLock lock = new MyReentrantLock(); MyReentrantLockTest test = new MyReentrantLockTest(lock); test.methodB(); System.out.println(lock); test.methodA(); }}


运行结果：


methodB:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}B假装进行了一波骚操作methodB unLock:MyReentrantLock{lock=0, lockThread=null}MyReentrantLock{lock=0, lockThread=null}methodA:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}methodB:MyReentrantLock{lock=2, lockThread=Thread[main,5,main]}B假装进行了一波骚操作methodB unLock:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}A假装进行了一波骚操作methodA unLock:MyReentrantLock{lock=0, lockThread=null}


可见，没有产生死锁。


3. 总结

本文只是介绍可重入锁的概念，以及简单的实现，可重入锁，主要有可能会又在循环调用或者递归的时候，这里面还是有很多问题的，其实都是线程安全问题，在JDK的ReentrantLock中会有更全面的考虑，以后我们在做分析。希望本文对你理解可重入锁有帮助。
