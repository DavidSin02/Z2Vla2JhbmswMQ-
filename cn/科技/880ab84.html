Java笔记之——多态详解（重要）

2020-10-29
| 科技

1---多态概述

某一个事物，在不同时刻表现出来的不同状态。

举例：

猫可以是猫的类型。猫 m = new 猫();

同时猫也是动物的一种，也可以把猫称为动物。

动物 d = new 猫();

在举一个例子：水在不同时刻的状态

多态前提和体现

有继承关系

有方法重写

有父类引用指向子类对象

父 f =new 子；

案例-1

  1. /*
  2. 多态基础案例
  3. */
  4. class Fa{
  5. public void show(){
  6. System.out.println("show Fa");
  7. }
  8. }
  9. class Zi extends Fa{
  10. public void show(){
  11. System.out.println("show Zi");
  12. }
  13. }
  14. class DuotaiDemo{
  15. public static void main(String[] args){
  16. Fa f = new Zi();
  17. f.show();
  18. }
  19. }

这就是多态。

3---多态的好处和弊端

多态的好处：

A:提高代码的维护性(继承体现)

B:提高代码的扩展性(多态体现)

案例-5

  1. /*
  2. 多态的好处：
  3. 提高代码的扩展性
  4. */
  5. class Animal{
  6. public void eat(){
  7. System.out.println("吃饭");
  8. }
  9. public void sleep(){
  10. System.out.println("睡觉");
  11. }
  12. }
  13. class Cat extends Animal{
  14. public void eat(){
  15. System.out.println("猫吃饭");
  16. }
  17. public void sleep(){
  18. System.out.println("猫睡觉");
  19. }
  20. }
  21. class DuotaiDemo4{
  22. public static void main(String[] args){
  23. Cat c = new Cat();
  24. c.eat();
  25. c.sleep();
  26. System.out.println("-------------");
  27. Cat c1 = new Cat();
  28. c1.eat();
  29. c1.sleep();
  30. System.out.println("改进后");
  31. System.out.println("-------------");
  32. //这就是扩展性
  33. useCat(c);
  34. useCat(c1);
  35. }
  36. public static void useCat(Cat c){
  37. c.eat();
  38. c.sleep();
  39. }
  41. }

多态的弊端：

父不能使用子的特有功能。

案例-2 24行已演示过 ，这里提供解决办法

案例-6

  1. /*
  2. 解决不能访问子类特有功能
  3. 那么我们如何才能访问子类的特有功能呢?
  4. */
  5. class Fu{
  6. public void show(){
  7. System.out.println("show fu");
  8. }
  9. }
  10. class Zi extends Fu{
  11. public void show(){
  12. System.out.println("show zi");
  13. }
  14. public void sleep(){
  15. System.out.println("sleep zi");
  16. }
  17. }
  18. class DuotaiDemo5{
  19. public static void main(String[] args){
  20. Fu f = new Zi();
  21. f.show(); //打印的为子类的方法 所以运行看右
  22. System.out.println("--------------");
  23. //第一种解决办法 创建子类对象（这种方法太占用内存空间）
  24. Zi z = new Zi();
  25. z.sleep();
  26. //第二种解决办法 将父类引用赋值给子类引用 （向下转型）
  27. Zi z1 = (Zi)f;
  28. z1.sleep();
  29. }
  30. }

B:向下转型

从父到子 Zi z = (Zi)f;
