C#中的多态

2020-10-29
| 科技


(一)里氏替换原则(LSP)

1、在一个软件系统中，子类对象可以替换所有使用的父类对象，且程序行为没有变化


(二)is与as操作符

1、is 检查对象是否与指定类型兼容


2、as 用于在兼容的类型之间执行转换


3、一种类型转换方式：将empls集合的元素转换为SE对象；转换失败返回null，

用as操作符进行类型转换不会产生异常，但这并不代表不需要异常处理。

(二)总结

1、用虚方法实现多态的步骤

2、子类重写父类的虚方法

3、两种方式

4、创建父类变量，实例化子类对象

5、把父类类型作为参数类型，该子类及子类对象作为参数传入

6、运行时，根据实际创建的对象决定执行哪个方法

(三)为什么需要抽象方法

1、交通工具类是抽象的概念，根本不可能实例化，Run()方法不具有实际意义！

(四)抽象类和抽象方法

1、抽象类用来列举一个类所需要的行为

2、抽象类不明确提供具体实现方法

3、抽象类必须由其子类实现它的抽象方法（除非子类也具有抽象性）

4、由abstract声明的方法称为抽象方法。抽象方法只有方法签名但没有方法体，定义格式如下：

访问修饰符 abstract 返回类型 方法名(参数列表);

示例：

public abstract void doing();

5、由abstract关键字声明的类称为抽象类，定义格式如下：

访问修饰符 abstract class 类名

{

类体

}

6、说明：

(1)当类中出现一个抽象方法时，该类就必须定义为抽象类。

(2)抽象类也可以没有抽象方法。

(3)若子类的父类是抽象类，则子类必须实现父类中的抽象方法，若不实现则该子类必须也定义为抽象类。

7、抽象类不能被实例化


8、抽象类不能是密封或者静态的


9、说明：

(1)静态方法不能声明为抽象方法。

(2)private修饰的方法不能声明为抽象方法。


10、将光标放在“：”后的父类名上，按快捷键Shift+Alt+F10+Enter，系统会给出抽象方法的默认实现


(五)抽象类的应用场合

1、父类提供一系列规定，约束子类的行为


2、抽象类和抽象方法实现多态性


(六)对比虚方法

1、抽象类和抽象方法

抽象类通常代表一个抽象的概念

抽象方法约束子类对象的行为

比如：抽象的动物类，具有抽象方法吃()、喝()


2、虚方法

具体类的默认实现，提供其子类扩展实现方式

比如：数据库连接类，Connect()方法实现与SQL Server的连接，数据库连接子类继承并重写它，实现与其它数据库的连接

(七)抽象方法与虚方法

抽象方法 虚方法

用 abstract 修饰 用 virtual 修饰

不允许有方法体 要有方法体，哪怕是一个分号

必须被子类 override 可以被子类 override

只能在抽象类中 除了密封类都可以写

(八)

抽象类中可以包括抽象方法和非抽象方法

(九)面向对象的三大特性

1、封装：隐藏内部实现，稳定外部接口

2、继承:子类继承父类成员，实现代码复用


3、多态:不同子类对同一个消息作出不同的反应
