Android矢量图Vector Drawable使用及适配

2020-10-29
| 科技

在开发Android应用时，开发者免不了会使用各种图标资源，而对于那些想自己写点东西而又不会切图的开发者来说就很困难了。今天我要说的Vector Drawable就可以解决这个问题，即使你不会切图，一样可以使用漂亮的图标。

Vector Drawable矢量图是Android 5.X中增加的对SVG矢量图形支持的API，有了这些API的支持，我们就可以在Android轻松的使用SVG图形。

Vector Drawable优点

可能有人觉得在工作中反正有UX给我们切图，我们干嘛还要去用矢量图呢，其实矢量图相对普通的位图来说还是有很多优点的：

  1. 矢量图是以点和和线来描述图像，适合直接打印输出，因此它的图像文件比较小。而我们平时使用的png，jpg等位图则是由每一点上存储的像素值组成，因此位图图像文件相对来说比较大。所以在APP中使用矢量图图标代替位图图标可以减小APP的体积。（其实细心的朋友应该可以发现在AS中新建一个内置的Activity模板时应该可以看到google使用的就是svg矢量图标）

  2. 在第一点中我已经说过了矢量图是点和线来描述图像，也就是说是根据图形座标位置实时画出来的，所以它可以自行适配各种分辨率的手机而不会失真。

  3. 即使是让UX直接给我们切SVG矢量图对他们来说也没有增加任何难度和工作量，因为现在的设计功能都支持直接导出为SVG矢量图，所以对他们来说还是很方便的。

  4. Android中还支持矢量图动画，我们不用写太多的代码就可以实现很多复杂的动画效果

Vector Drawable创建

通常创建一个Vector Drawable有两种方式：

  1. 通过AS右键New一个Vector Asset(AS默认提供了一些矢量图)，当然我们也可以选择本地的SVG图进行转换得到Vector Drawable，这里给大家推荐一个图标网址，里面可以搜索到各种图标，既可以生成PNG格式的也可以生成SVG格式http://www.iconfont.cn/

  2. 自己通过数学方式计算自己想要的图标各个关键点的座标位置，然后自己构造Vector Drawable的pathData数据从而创建一个Vector Drawable

我们来看看创建的Vector Drawable：

<vector xmlns:android="http://schemas.android.com/apk/res/android"android:width="24dp"android:height="24dp"android:viewportHeight="1024.0"android:viewportWidth="1024.0"> <pathandroid:fillColor="#ffffff"android:pathData="M926.86 484.66H176.19l292.29-270.87a30 30 0 1 0-40.78-44L81.93 490.21a30.17 30.17 0 0 0-5.83 5.43 30 30 0 0 0 1 43.77l351 314.65a30 30 0 0 0 40-44.68L172.81 544.66h754.05a30 30 0 1 0 0-60z"/></vector>

上面是一个返回效果的图标，如上面代码所示，创建的Vector Drawable都是以<Vector>标签开头的xml文件，下面我说下这里面各个属性代表什么意思：

  1. width和height就代表了这个图形的大小

  2. viewportWidth和viewportHeight表示将这个图形横纵方向均等分为1024份，每份代表一个座标点位，下面的pathData中的轨迹数据座标就是以这个来进行定位

  3. <path>元素代表了这个图形中的一条点线轨迹

  4. fillColor表示这个图形显示出来的颜色

  5. pathData就是表示图形的点线轨迹

pathData中的数据格式基本与SVG图形中的数据轨迹格式是一致的，下面再说下SVG-Path的指令所代表的意思：

  • M = moveto(M X,Y) ：将画笔移动到指定的座标位置(还没开始绘制)

  • L = lineto(L X,Y) ：画直线到指定的座标位置

  • H = horizontal lineto(H X)：画水平线到指定的X座标位置

  • V = vertical lineto(V Y)：画垂直线到指定的Y座标位置

  • C = curveto(C X1,Y1,X2,Y2,ENDX,ENDY)：三次贝赛曲线

  • S = smooth curveto(S X2,Y2,ENDX,ENDY)：三次贝赛曲线

  • Q = quadratic Belzier curve(Q X,Y,ENDX,ENDY)：二次贝赛曲线

  • T = smooth quadratic Belzier curveto(T ENDX,ENDY)：映射

  • A = elliptical Arc(A RX,RY,XROTATION,FLAG1,FLAG2,X,Y)：弧线

  • Z = closepath()：关闭路径

Android中Vector Drawable的使用

前面说了这么多，那么在Android中Vector到底该怎么用，5.0系统以下又该怎么办呢？

前面说过Android5.X才开始支持Vector Drawable，但是Android同时也为我们提供了兼容包，使用起来也是很方便的，但是也有些问题需要注意：

  1. 必须使用appcompat-v7:23.2.0及以上版本

  2. gradle:2.0.0

  3. 在主Moudul的的build.gradle文件的defaultConfig中添加vectorDrawables.useSupportLibrary = true配置

  4. 所使用的Activity必须继承AppCompatActivity

  5. 在ImageView中使用app:srcCompat属性代替src属性设置Vector资源

<ImageView

android:layout_width="100dp“

android:layout_height="100dp"

app:srcCompat="@drawable/five_star"/>

另外还有一个特别需要注意的问题就是在Button，RadiaoGroup或compound drawable之类的View设置background为Vector Drawable时必须将Vector Drawable进行selector包装后再进行设置，另外还要在这些View所在的Activity里设置如下代码才能生效，否则会出现crash

AppCompatDelegate.setCompatVectorFromResourcesEnabled(true)

Android中Vector Animation的使用

上面说完了Vector的使用，下面接下来说说Vector Animation的使用。

其实使用Vector Animation也比较简单，首先在res/drawable目录下创建一个<animated-vector>文件，然后使用AnimatedVectorDrawableCompat加载上述drawable即可，示例代码如下：

//R.drawable.five_star_animate文件代码

<animated-vector

xmlns:android="http://schemas.android.com/apk/res/android"

android:drawable="@drawable/five_star">

<target android:animation="@animator/animator_five_star"

android:name="five_star"/>

</animated-vector>

//Java代码

AnimatedVectorDrawableCompat starDrawable =

AnimatedVectorDrawableCompat.create(this, R.drawable.five_star_animate);

starImageView.setImageDrawable(starDrawable);

starDrawable.start();

//动画文件animator_five_star，下面这个文件大家比较熟悉了

<objectAnimator

xmlns:android="http://schemas.android.com/apk/res/android"

android:propertyName="trimPathEnd"

android:interpolator="@android:interpolator/linear"

android:repeatCount="infinite"

android:repeatMode="restart"

android:valueType="floatType"

android:valueFrom="0"

android:valueTo="1"

android:duration="3000">

</objectAnimator>

上面的动画是针对Vector的trimPathEnd属性来操作的，Vector具有哪些属性代价可以去Android API里面查查，这里就不列举了。
