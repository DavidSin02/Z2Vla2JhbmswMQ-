Spring Cloud Feign/Ribbon 超时时间设置和Istio故障注入测试

2020-10-29
| 科技

很多小伙伴在引入Spring Cloud Feign后，都会碰到如下故障：

feign.RetryableException: Read timed out executing GET http://test-order/orderInfo?id=1] with root cause java.net.SocketTimeoutException: Read timed out

由于SpringCloudFeign的客户端负载均衡是通过SpringCloudRibbon实现，而SpringCloudRibbon的默认连接超时时间为2秒，默认读取超时时间为5秒，所以，当调用超过2秒的时候，就会报错，如图：

Ribbon默认超时时间

超时例子

图中所示的例子，是有微服务"test-user"发起，调用微服务"test-order"，我们利用istio的故障注入，使得调用"test-order"微服务的http请求都会延时7秒返回，如下：

apiVersion: networking.istio.io/v1alpha3kind: VirtualServicemetadata:  name: test-order  namespace: econspec:  hosts:    - test-order  http:    - route:        - destination:            host: test-order            subset: v1            port:              number: 8259      fault:        delay:          percentage:            value: 100.0          fixedDelay: 7s

针对这种情况，我们可以手动调长ribbon的延时时间，在服务调用发起方的application.yaml里加入，注意超时时间必须大于istio的故障注入延迟时间：

#--------------Feign负载均衡配置 配置全局超时时间 毫秒单位   根据业务酌情配置ribbon:  ConnectTimeout: 10000 #请求连接的超时时间    ReadTimeout: 10000 #请求处理的超时时间

或者是在kubernetes deploy的env变量里加入：

- env:

- name: ribbon_ConnectTimeout

value: "10000"

- name: ribbon_ReadTimeout

value: "10000"

加入之后，效果变成，7秒钟之后，请求结果就顺利返回：


花费时间7秒钟，结果顺利返回
