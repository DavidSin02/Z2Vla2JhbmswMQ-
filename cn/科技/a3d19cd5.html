GitHub连续3天出现严重宕机情况 微软未回应

2020-11-14
| 科技

【点击右上角加'关注'，全国产经信息不错过】

4月24日消息 在COVID-19大流行期间，更多的人在远程工作。但在过去的三天里，全球范围内的GitHub开发者都遭遇到了服务宕机的问题，导致他们无法执行基本的编程任务。

4月21日，多个GitHub服务被问题困扰了至少一个半小时。4月22日，再次出现的中断影响了众多开发者至少两个小时的时间。而4月23日，多个GitHub服务也遇到了各种问题的影响，开始时间大约是美国东部时间上午9点20分，持续了近三个小时。和最近的其他中断事件一样，官方没有提供任何原因，也没有公布恢复过程的信息。

今天的故障影响到了Git运营；API请求；Webhooks；问题、拉取请求和项目；GitHub行动；GitHub包；GitHub页面等。在 GitHub 状态网站上，只有 Git Operations 被列为“故障”，其他服务被标注为 "已降级"。

虽然GitHub状态网站上确实定期更新了微软试图修复各种故障的情况，但并没有提供任何关于问题的细节。在Twitter上，一些开发者一直在要求微软提供更多关于GitHub问题的详细事后说明。

IT之家了解到，外媒ZDNET已经向微软发去置评请求，询问为什么本周GitHub出现这么多的宕机问题，目前微软还没有回应。

今天早上，GitHub教育账号在Twitter上承认GitHub Classroom和education.github.com上出现了宕机。

"啊，每天的GitHub中断，"4月23日，Mijndert在推特上写道。

"Github这个月中断时间高达159分钟.....但谁在计算呢？"4月23日，Alex Atkinson在推特上写道。

"对那些今天早上抱怨@github今天早上又发生了一次#outage（继本周早些时候的一次中断之后）的开发者们来说...... 就当这是又一次提醒大家，#云并不神奇...........它只是别人的电脑，"Kellen Murphy在4月23日的推特上写道。
