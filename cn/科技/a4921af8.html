方法引用和Lambda使用建议

2020-11-14
| 科技

在Java8中，增加了函数接口Functional Interface、 Lambda和方法引用Method Reference，使得创建函数对象变得很容易。

建议1：Lambda优先于匿名类

下面是使用匿名类进行排序的写法：

    //匿名函数写法    List<String> list = new ArrayList<>();    Collections.sort (list, new Comparator<String>() {        @Override        public int compare (String o1,  String o2) {            return Integer.compare (o1.length(), o2.length());        }    });

Lambda 类似于匿名类的函数，但是比它简洁得多

    //Lambda表达式写法    Collections.sort(list,(o1,o2)->Integer.compare(o1.length(),o2.length()));

Lambda表达式写法更加简洁，参数和返回值类型都没有进行指定，编译器将自动进行类型推导。

建议2：方法引用优先于Lambda

方法引用常常比Lambda 表达式更加简洁明了 。只要方法引用更加简洁、清晰，就用方法引用；如果方法引用并不简洁，就坚持使用 Lambda。

下面以map的merge(）来举例说明，merge方法表示，如果指定的键不存在，插入指定值，如果存在该键，在对应原值上增加指定值并覆盖原值。运行下面的代码将输出 a=3

     Map<String,Integer> map = new HashMap<>();     map.merge("a",  1,  Integer::sum) ;     map.merge("a",  1,  Integer::sum) ;     map.merge("a",  1,  (old, inc) ->old + inc);     System.out.println("a=" + map.get("a"));

从该例子可以看出，方法引用比Lambda更加简洁和容易理解。如果方法参数越多，效果就越明显。如果Lambda 表达式太长，或者过于复杂，一种常用的优化方式是从 Lambda 中提取代码，放到一个新方法中，并用该方法的一个引用代替Lambda。

建议3：坚持使用标准的函数接口

java.util.function 包已经为此提供了大量标准的函数接口 。只要标准的函数接口能够满足需求，通常应该优先考虑，而不是专门再构建一个新的函数接口。

常用函数接口如下：

常用函数接口

Supplier<T> T get() 无输入参数，对外返回T类型对象

Consumer<T> void accept(T t) 输入参数类型为T，无返回值

Predicate<T> boolean test(T t) 输入参数类型T，进行断言

Function<T,R> R apply(T t) 输入参数类型为T，方法返回值类型为R

支持2个参数的升级版:

BiConsumer<T，U> void accept(T t，U u) 输入2个参数，无返回值

BiPredicate<T，U> boolean test(T t，U u) 对2个输入参数，进行断言

BiFunction<T，U，R> R apply(T t，U u) 输入2参数，方法返回值类型为R
