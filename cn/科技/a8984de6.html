面向对象程序设计三大重要机制之多态

2020-11-14
| 科技

多态性是面向对象程序设计的三大重要机制之一。其与类（数据封装），类的继承（继承性）构成了面向对象程序设计的核心。

这三大机制相互关联，类是基础，继承是核心，而多态便是补充。而且，多态性必须存在继承的环境之中。

所谓的多态是指对于同一种消息，多种对象成员有完全不同的行为表现。举个例子：同时送礼物给不同的人，每个人的表现不会完全相同，有人开心，有人疑惑，也有人无所谓。在C++中消息主要是函数成员的调用（也就是上例中的送礼物）而不同的行为表现则是指成员函数的调用，而不同的行为表现则是指成员函数不同的实现（也就是上例中的人的表现）。这样，也与现实中的处理方式符合。

在C++中多态的实现又以下几种：

  • 函数的重载
  • 运算符的重载
  • 虚函数

其中，函数和运算符的重载，这两种都是类的简单多态性。而虚函数是多态性的主要表现。

虚函数

C++中采用虚函数来实现动态绑定，虚函数是动态绑定的基础。如果一个函数被声明为虚函数，那么其成员函数在子类中可以有不同的实现。

声明格式：

virtual 数据类型 函数名（参数表）

其中，virtual是关键字。用其声明虚函数。

实例分析：

#include <iostream>

using namespace std;

class Base1{

public:

virtual void display() const; //虚函数

};

void Base1::display() const{

cout<<"Base1::display()"<<endl;

}

class Base2:public Base1{

public:

void display() const; //覆盖基类的虚函数

};

void Base2::display() const{

cout<<"Base2::display()"<<endl;

}

class Derived:public Base2{

public:

void display() const; //覆盖基类的虚函数

};

void Derived::display() const{

cout<<"Derived::display()"<<endl;

}

void fun(Base1 *ptr){ //参数为指向基类对象的指针

ptr->display(); //“对象指针->成员名”

}

int main() {

Base1 base1;

Base2 base2;

Derived derived;

fun(&base1);

fun(&base2);

fun(&derived);

return 0;

}

注意：

  1. 类的构造函数也可以调用虚函数（注意是调用不是声明），不过其采用静态绑定的方式（当基类构造函数调用虚函数时，不会调用派生类的虚函数）。
  2. 类的析构函数可以是虚函数（注：父类是虚析构函数，子类的析构函数也为虚不管是否用 virtual 声明）
  3. 只要类的成员函数才可以被声明为虚函数。因为虚函数只适用于有继承关系的类对象So 普通函数不能被说明为虚函数
  4. 静态成员不能是虚函数因为其不受限于某个对象
  5. 内联函数不能是虚函数因为内联函数是不能在运行过程中动态确定其位置的，即使虚函数在类体内实现，编译时，依然将其看作非内联的。
