本节主要学习Spring的核心之一AOP面向切面编程。

1.AOP概述

AOP，全称Aspect Oriented Programming，即面向切面编程，也叫面向方面编程。是一种通过预编译方式和运行期间动态代理实现程序功能的统一维护的一种技术。AOP是OOP的延续，是软件开发中的一个热点，也是Spring框架中的一个重要内容，是函数式编程的一种衍生范型。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

AOP是面向对象OOP编程的有益补充，运用动态代理技术，在不改变源代码的情况下对程序进行功能增强(额外增加功能)。AOP把软件系统分为两个部分，核心关注点和横切关注点。

核心关注点：主要关注系统核心业务，比如用户，产品，订单，支付，对账，统计等

横切关注点：主要关注系统非核心业务，比如事务，日志，性能，安全，异常等

AOP的目标是把散落在各业务代码中的非核心代码抽离出来，集中管理，降低耦合，提高代码可复用性，可扩展性和可维护性。

AOP-OOP

2.AOP核心概念

AOP技术的核心概念主要包括横切关注点，切面，连接点，切入点，通知，目标对象，注入，引入。

AOP核心概念

2.1 横切关注点

对哪些方法进行拦截，拦截后怎么处理，这些关注点称之为横切关注点。

2.2 切面（Aspect）

类是对物体特征的抽象，切面就是对横切关注点的抽象。

2.3 连接点（Join point）

被拦截到的点，因为Spring只支持方法类型的连接点，所以在Spring中连接点指的就是被拦截到的方法，实际上连接点还可以是字段或者构造器。

2.4 切入点（Pointcut）

对连接点进行拦截的定义，就是定义对哪些方法进行拦截，可以对一个类的所有方法进行拦截，也可以只拦截一个类并以特定字符开头的方法.

2.5 通知（Advice）

所谓通知指的就是指拦截到连接点之后要执行的代码，通知分为前置、后置、异常、最终、环绕通知五类，就是拦截到方法之后，在什么时机执行切面的代码。

2.6 目标对象（Target Object）

代理的目标对象，就是源对象

2.7 织入（Weaving）

将切面应用到目标对象并导致代理对象创建的过程。

2.8 引入（introduction）

在不修改代码的前提下，引入可以在运行期为类动态地添加一些方法或字段。

2.9 代理（Proxy）

AOP框架创建的代理对象，包含通知。在Spring中，AOP代理可以是JDK动态代理或CGLIB代理。

3.5编写增强代码

AspectjXmlLogger.java

package raky.train.aop;import java.util.Arrays;import lombok.extern.slf4j.Slf4j;import org.aspectj.lang.JoinPoint;import org.aspectj.lang.ProceedingJoinPoint;import raky.train.entity.User;/** *  五种增强 *  @Author raky */@Slf4jpublic class AspectjXmlLogger {	    //前置通知[增强] :	public void before(JoinPoint jp) {		log.info("--------1.----前置通知[增强]before------------------");//		log.info("调用 " + jp.getTarget() + " 的 " + jp.getSignature().getName()//				+ " 方法。方法入参：" + Arrays.toString(jp.getArgs()));	}		//环绕通知[增强]	public Object around(ProceedingJoinPoint jp) throws Throwable{		log.info("--------2.----环绕增强之前around--------------------");        Object object = jp.proceed(); //--显式的调用目标方法		log.info("--------3.----环绕增强之后around--------------------");        return object;    }		//后置通知[增强]	public void afterReturning(JoinPoint jp, Object result) {		log.info("--------4.----后置通知[增强]afterReturning----------");//		logger.info("调用 " + jp.getTarget() + " 的 " + jp.getSignature().getName()//				+ " 方法。方法返回值：" + result);	}		//最终通知[增强]	public void after(JoinPoint jp){		log.info("--------5.----最终通知[增强]after-------------------");//		log.info("调用 " + jp.getTarget() + " 的 " + jp.getSignature().getName()//				+ " 方法。");    }		//异常通知[增强]	public void afterThrowing(JoinPoint jp,Throwable e){		log.info("------------异常通知[增强]afterThrowing----------");//		log.info("调用 " + jp.getTarget() + " 的 " + jp.getSignature().getName()//				+ " 方法。" + e.getMessage());    }}

3.6编写配置文件

spring-aop-aspectj.xml

<?xml version="1.0" encoding="UTF-8"?><beans xmlns="http://www.springframework.org/schema/beans"    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"    xmlns:aop="http://www.springframework.org/schema/aop"    xsi:schemaLocation="http://www.springframework.org/schema/beans    	        https://www.springframework.org/schema/beans/spring-beans.xsd        http://www.springframework.org/schema/aop        http://www.springframework.org/schema/aop/spring-aop.xsd">	<!-- 1.核心业务组件 -->	<bean id="userDao" class="raky.train.dao.impl.UserDaoImpl" />	<bean id="userService" class="raky.train.service.impl.UserServiceImpl">		<property name="userDao" ref="userDao" /><!-- 把dao对象注入到service层中 -->	</bean>		<!-- 2.非核心业务组件，声明增强方法所在的Bean -->	<!-- 前置/后置/环绕/最终/异常 -->	<bean id="aspectjXmlLogger" class="raky.train.aop.AspectjXmlLogger" />	<!-- 3.配置切面 -->	<aop:config>		<!-- 定义切入点 -->		<aop:pointcut id="pointcut" expression="execution(* raky.train.service.UserService.*(..))" />		<!-- 引用包含增强方法的Bean [增强处理对象] -->		<aop:aspect ref="aspectjXmlLogger"><!-- 定义切面 -->			<!-- 增强的配置顺序，决定了增强的执行顺序 -->			<!-- 前置增强 ，在切入点织入方法-->			<aop:before method="before" pointcut-ref="pointcut" />			<!-- 环绕增强 -->			<aop:around method="around" pointcut-ref="pointcut" />			<!-- 后置增强 -->			<aop:after-returning method="afterReturning" pointcut-ref="pointcut" returning="result"/>			<!-- 异常增强 -->			<aop:after-throwing method="afterThrowing" pointcut-ref="pointcut" throwing="e" />			<!-- 最终增强 -->			<aop:after method="after" pointcut-ref="pointcut" />		</aop:aspect>	</aop:config></beans>
