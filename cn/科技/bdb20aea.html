C｜实例说明变量存储类别的时间存续期和空间作用域

2020-11-14
| 科技

变量的值可能在整个程序运行期间一直存在，也可能是临时分配存储空间，使用结束，变量就不存在了（再次使用时，再重新分配释放）。变量值的存在时间是由变量的存储方式来决定的。

变量的存储方式分为静态存储方式和动态存储方式。

采用静态存储方式的变量，在程序运行期间，变量由系统在静态存储区为其分配存储空间，且存储空间在程序运行期间是不释放的，即变量可以一直使用为其分配的存储空间。

采用动态存储方式的变量，只有在函数被调用时，系统才为其分配动态存储空间，函数调用结束则系统释放该变量的存储空间，变量的值也就不复存在了。

在C语言中，变量的存储类别可以通过关键字auto、static、extern、register来指定。用register声明的变量其值存入在寄存器中（一种比内存速度更快的存储器），使用寄存器变量只为提高数据的存取速度，但由于机器性能越来越高，目前已不常用。

auto

#include <stdio.h>

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

}

system("pause");

return 0;

}

void func()

{

auto int j = 0;

++j;

printf("\t%d",j);

}

//输出结果：1 1 1 1 1

auto声明的变量j是自动变量，为局部变量，定义在代码块内部（ auto关键字可以省略)。函数调用时对其赋值（只有在函数被调用时，系统才为其分配动态存储空间，函数调用结束时则系统释放该变量的存储空间），函数调用一次重新赋值一次。

需要补充说明的是，函数的形式参数也同样被看作局部变量。另外，主函数main语句块的变量的时效性与空间性与其他函数处于同等地位。

static

#include <stdio.h>

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

}

system("pause");

return 0;

}

void func()

{

static int j = 0;

++j;

printf("\t%d",j);

}

//输出结果：1 2 3 4 5

static定义的变量j为静态变量，因为位于代码块内，所以也是局部的，所以属于局部静态变量。编译时赋初期，无论函数调用多少次只赋值一次。虽然是局部变量，因为static的静态声明，函数调用结束后与自动变量不同的是，空间不释放，值不消失，变量值保留到下次函数调用，调用时的值为下次函数调用结束时的值。当然，因为变量定义在块内，虽然函数调用结束后其值仍存在，但其他函数不能使用其值，即变量的作用域只能是当前函数。

（所谓静态就是一直存在，动态就是调用时存在，调用完消失。静态如果是对全局的修饰，其实是对全局的限制，在多文件源程序中，通过静态限制到定义该变量的文件自身，而不是全部文件。）

extern

#include <stdio.h>

extern j;

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

j++;

}

system("pause");

return 0;

}

int j = 0;

void func()

{

++j;

printf("\t%d",j);

}

//输出结果：1 3 5 7 9

上例声明和定义的j处于语句块外，属于外部变量，也叫全局变量，属于静态存储方式。外部变量只能定义一次，可以在文件的开头，也可以在任意两个函数定义之间，但作用域只从定义点开始到文件结束。全局变量若不在文件开头定义，可在当前文件任意位置用extern对全局变量进行声明（可以多次声明），这样也就扩大了变量的作用域，变量的有效区域从定义处扩充到了extern声明处。也就是说，extern只是用来说明该变量已经在其他地方定义了。当然，如果全局变量定义在其它文件中，则本文件需要加extern进行声明。

(extern也是隐含在一般的函数定义前面的，表面函数的全局性。)

全局变量（外部变量）的说明之前再冠以static就构成了静态的全局变量。未冠static的全局变量和静态的全局变量二者都是静态存储方式，在单文件程序中二者无任何区别。区别在于当一个源程序由多个源文件组成时，非静态的全局变量的作用域是整个源程序，当一个源程序由多个源文件组成时，非静态的全局变量在各个源文件中都是有效的，而静态的全局变量则限制了作用域，即只在定义该变量的源文件内有效，在同一源程序的其他源文件中不能使用它。由于静态全局变量的作用域局限于一个源文件内，只能为该源文件内的函数公用，因此可以避免在其他源文件中引起错误。从以上分析可以看出，把局部变量增加静态声明后是改变了它的存储生存期。把全局变量增加静态声明后是限制了它的使用范围（作用域）。因此static这个说明符在不同的的地方所起的作用是不同，应该是语言开发者为避免使用更多的关键字而增加一些关键字在不同的上下文环境中的功能。

如是一个全局变量被局部变量所屏蔽，可以通过一元作用域运算符::（两个半角冒号）来访问这个全局变量。
