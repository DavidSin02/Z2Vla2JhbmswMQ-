一.AOP概述

AOP(Aspect Oriented Programming)意为:面向切面编程，通过预编译方式和运行期间动态代理实现程序功能的统一维护的一种技术。AOP是OOP(面向对象编程)的延续，是软件开发中的一个热点，也是Spring框架中的一个重要内容，是函数式编程的一种衍生范型。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

AOP底层是采用动态代理模式实现的，主要是JDK的动态代理与CGLIB动态代理。

二.AOP举例描述

比如，我们在开发中要打印一个方法执行的日志：某某方法开始执行，某某方法执行结束。

一般我们会在每个方法中都加入，这样方法一多，日志就多，如果说某天，这样的日志不要了，处理起来将会是很糟糕的。

此时我们就可以使用AOP面向切面编程来实现。

首先温习一下动态代理的实现

  • (1)DAO接口及其实现
/** * 目标接口 * @author Administrator */public interface TargetDAO {  /**目标方法*/  String print();}public class TargetDAOImpl implements TargetDAO{  @Override  public String print() {     return "TargetDAOImpl";  }}
  • (2)创建调用处理器
//动态代理处理器public class OwnInvocationHandler implements InvocationHandler{   private TargetDAO targetDao;//目标类   //无参构造器   public OwnInvocationHandler() {     super();   }   //有参构造器   public OwnInvocationHandler(TargetDAO targetDao) {     super();     this.targetDao = targetDao;   }   /**    * @param proxy 动态生成的代理对象    * @param method 目标方法    * @param args 方法参数    * @return 目标方法返回值    */   @Override   public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {     System.out.println("某某方法开始执行");     //1.调用反射类Method的invoke,就是调用方法     Object returnObj = method.invoke(targetDao, args);     if(returnObj!= null){       return (String)returnObj+"--invoke";     }     System.out.println("某某方法执行完毕");     return returnObj;  }}
  • (3)测试动态代理
/** * 测试 * @author Administrator */public class Test {  public static void main(String[] args) {   //创建目标方法   TargetDAO target = new TargetDAOImpl();   //创建处理器   OwnInvocationHandler handler = new OwnInvocationHandler(target);   //生成动态代理类   TargetDAO TargetDAOProxy = (TargetDAO)Proxy.newProxyInstance(target.getClass().getClassLoader(),           target.getClass().getInterfaces(), handler);   //执行方法   String printStr = TargetDAOProxy.print();   System.out.println(printStr); }}

三、AOP编程术语

  • 1.Aspect（切面）：泛指交叉业务逻辑，比如上面举例中的日志打印，再比如事务处理、登录验证等，都可以理解为切面。常用切面通知顾问 实际上，切面的存在是对业务逻辑处理的一种增强。
  • 2.Target（目标对象）：目标对象就是要被AOP作用的对象，也就是要被增强处理的对象。
  • 3.Weaving（织入）：织入是一个过程，就是将切面插入到目标对象的一个过程。
  • 4.Joint point（连接点）：连接点指可以被切面织入的方法，也泛指业务接口中的方法，也就是所有的方法都可以看做是连接点。
  • 5.Pointcut（切入点）：指切面具体织入的方法。比如一个A类中有a(),b()两个方法，我们要对a()方法使用切面，那么a()方法就是切入点(也是连接点)。b()仅为连接点。
  • 6.Advice（通知）：切面的一种简单的具体实现。其中定义了切面织入到目标切入点的时间，也就是在目标方法之前还是之后执行。可以认为切入点定义织入的位置；通知定义织入的时间。
  • 7.Advisor(顾问)切面的一种复杂的具体实现，能够将Advice（通知）包装为更复杂的切面。
