Elasticsearch集群配置文件elasticsearch.yml详解

2020-10-29
| 科技

下面分享Elasticsearch v7.x 集群搭建过程中，节点配置参数详细说明。在实际工作中，要根据不同的业务场景和现实条件，做出切合实际的方案。

Elasticsearch的配置文件有elasticsearch.yml、log4j2.properties两个配置文件，其中elasticsearch.yml是用来配置Elasticsearch服务的，log4j2.properties是用来配置日志文件的。

下面是elasticsearch.yml配置文件的详细说明。

Elasticsearch 7.x 订单中心技术专栏，从真实项目实践、项目源码和生产故障分析等各个方面介绍了Elasticsearch相关的知识。


一、基本配置参数

# 集群名称

cluster.name: "es_ordercluster"

# 节点名称，这儿我直接取名为order-node1

node.name:order-node1

# 是否可以成为master节点

node.master: true

# 是否允许该节点存储数据，默认开启

node.data: true

# 网络绑定，这里我绑定 10.19.156.194

network.host: 10.19.156.194



二、网络端口号

Elasticsearch 启动监听两个端口，9300和9200（默认）。

9300端口是使用tcp客户端连接使用的端口；

9200端口是通过http协议连接Elasticsearch使用的端口；


可以根据需要修改默认的端口号，比如和其他应用使用的端口号存在冲突：

#

# Set a custom port for HTTP:

#

transport.port: 8300

http.port: 8200


Elasticsearch支持开箱即用，可以没有任何网络配置。Elasticsearch将绑定到可用的环回地址，并将扫描本地端口9300到9305以尝试连接到在同一服务器上运行的其他节点。 这提供了自动集群体验，无需进行任何配置。


三、主节点的节点地址配置

discovery.seed_hosts 配置

此设置通常应包含集群中所有符合主节点的节点地址（ master-eligible nodes ）。 此设置包含主机数组或逗号分隔的字符串。

每个值应采用host:port或host的形式（其中port默认为设置transport.profiles.default.port，如果未设置则返回transport.port）。 请注意，必须将IPv6主机置于括号内。 此设置的默认值为127.0.0.1，[:: 1]。

该设置以前称为discovery.zen.ping.unicast.hosts。它的旧名称已经不赞成使用了，但为了保持向后兼容性，它将继续工作。在将来的版本中，对旧名称的支持将被删除。

因此，在配置文件中，建议使用discovery.seed_hosts。

discovery.seed_hosts: ["10.131.113.95:8300", "10.131.213.95:8300", "10.131.213.97:8300"]


四、主节点选举列表

cluster.initial_master_nodes 配置

在第一次启动全新的Elasticsearch集群时，会出现一个集群引导步骤，该步骤确定在第一次选举中计票的主合格节点集（master-eligible）。

在开发模式下，如果未配置，则此步骤由节点本身自动执行。

由于此自动引导本质上是不安全的，因此当我们在生产模式下启动全新集群时，必须明确列出符合条件的节点的名称或IP地址，这些节点的投票应在第一次选举中计数。

使用cluster.initial_master_nodes设置此列表。

cluster.initial_master_nodes: ["order-node1", "order-node2","order-node3"]


五、设置最少数量主节点

discovery.zen.minimum_master_nodes 参数

为了防止数据丢失， discovery.zen.minimum_master_nodes 配置至关重要， 以便每个候选主节点知道为了形成集群而必须可见的最少数量的候选主节点。这个配置就是告诉 Elasticsearch 当没有足够 master 候选节点的时候，就不要进行 master 节点选举，等 master 候选节点足够了才进行选举。

没有这种设置，遇到网络故障的群集有可能将群集分成两个独立的群集（脑裂），这将导致数据丢失。

为了避免脑裂，候选主节点的数量应该设置为：

(master_eligible_nodes / 2) + 1

换句话说，如果现在有3个节点，最小候选主节点数应该是（3/2)+1=2。

discovery.zen.minimum_master_nodes: 2

如果你有两个节点，你遇到难题了。法定数当然是 2 ，但是这意味着如果有一个节点挂掉，你整个集群就不可用了。 设置成 1 可以保证集群的功能，但是就无法保证集群脑裂了，像这样的情况，你最好至少保证有 3 个节点。

discovery.zen.minimum_master_nodes: 2


注意：在Elasticsearch 7.0中，已经完全重新考虑并重建了集群协调层（cluster coordination layer）。新的实现提供了安全的、秒级的master节点选举时间，其中可能需要几秒钟选择一个新的master，针对关键任务的部署而言，这个时间是值得的。这样，discovery.zen.minimum_master_nodes 参数就取消了，集群的增长和收缩将变得更加安全和方便，从而也大大减少了错误配置系统参数的可能性。

在Elasticsearch 7.x节点上，可以配置discovery.zen.minimum_master_nodes 参数，但是会被忽略。

参考官方文档链接：https://www.elastic.co/guide/en/elasticsearch/reference/current/breaking-changes-7.0.html


六、设置存储路径

# ----------------------------------- Paths ------------------------------------

# 设置配置文件的存储路径，默认是es根目录下的config文件夹

# path.conf: /path/to/conf

#

# 设置索引数据的存储路径，默认是es根目录下的data文件夹，可以设置多个存储路径，用逗号隔开，例：

# path.data: /path/to/data1,/path/to/data2

# path.data: /path/to/data

#

# 设置日志文件的存储路径，默认是es根目录下的logs文件夹

# path.logs: /path/to/logs

一般情况下，可以保持默认值，不需要调整。


七、锁定物理内存地址

bootstrap.memory_lock: true

这个配置的意义：锁定物理内存地址，防止Elasticsearch内存被交换出去，也就是避免Elasticsearch使用swap交换分区，频繁的交换，会导致IOPS变高。

# ----------------------------------- Memory -----------------------------------

#

# Lock the memory on startup:

#

bootstrap.memory_lock: true

#

# Make sure that the heap size is set to about half the memory available

# on the system and that the owner of the process is allowed to use this

# limit.

#

# Elasticsearch performs poorly when the system is swapping the memory.

#

设置好后，启动Elasticsearch会报错：

[1] bootstrap checks failed

[1]: memory locking requested for elasticsearch process but memory is not locked


这里需要额外设置下系统配置文件 需要切换到root用户：

（1）修改/etc/security/limits.conf

最后添加：

* soft nofile 65536

* hard nofile 65536

* soft nproc 131072

* hard nproc 131072

* soft memlock unlimited

* hard memlock unlimited

（2）修改/etc/systemd/system.conf

修改下面几项：

DefaultLimitNOFILE=65536

DefaultLimitNPROC=32000

DefaultLimitMEMLOCK=infinity


（3）最后执行重新载入

systemctl daemon-reload

有关如何正确设置内存锁定的更多详细信息，请参阅启用bootstrap.memory_lock。

https://www.elastic.co/guide/en/elasticsearch/reference/current/setup-configuration-memory.html#bootstrap-memory_lock
