GitHub宕机两小时，众多软件开发人员受影响

2020-11-14
| 科技

IT之家 6 月 29 日消息 今天晚上，GitHub 再次出现了宕机事故，刚刚已经恢复正常，宕机时间持续了 2 小时左右。

这家微软旗下的服务通过 Git 提供版本控制，并为软件开发提供托管服务，由于软件开发与该服务紧密相连，GitHub 用户今晚在登录该服务时遇到错误，一些用户甚至无法使用其集成开发环境（IDE）。

GitHub 官方在出现问题两个多小时后才确定错误并进行修复，并表示正在调查此次故障。

GitHub 是一个大型代码存储库，很多开发者和公司将整个项目和代码托管在该服务上，非常受欢迎。苹果、亚马逊、谷歌、Facebook 和许多其他大型科技公司都在使用 GitHub。在 GitHub 上托管的仓库超过 1 亿个，有 4000 万开发者为其做出贡献。

IT之家了解到，微软早在 2018 年就以 75 亿美元的价格收购了 GitHub，并一直在逐步改善对开发者的服务。
