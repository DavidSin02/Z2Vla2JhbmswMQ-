巧妙结合spring与简单模板方法实现代码简化

2020-11-14
| 科技

在之前的一个项目中，有遇到过分步处理的问题，简单的来说就是极简的工作流，一般是指处理步骤在三步以内。

需求场景是这样的：公司订单线的系统需要升级重构，原本两个代码比较乱的服务（其中一个是用php写的）需要拆分成三个新服务（全部用java重构）

服务A作为前置服务，直接和前端做交互；服务B作为中间调度服务，承载所有的订单调度业务；服务C作为交易服务，负责所有订单的支付业务。

我负责的是服务B，其中有一个环节是：服务A将用户的的下单信息推送到服务B，服务B接收后会生成订单信息，然后对订单做业务调度（正向成单，逆向退单等等），最后会将该订单的信息推送到各个关联系统（大概有三十多个），但是这些关联系统不一定要全量的订单信息（如果所有关联系统都全量推，消息队列也扛不住），所以需要对这些信息做个性化处理。

我是使用了简单的模板方法+spring事件来实现的

首先，对于上述的场景，我们可以拆分为两个步骤

  1. 数据组装。由于各个关联系统比较杂乱，有的是go写的，有的是php，还有不同的报文格式要求，所以数据组装这一步需要自定义，可以是简单的json字符串报文，可以是xml，也可以是自定义格式的报文
  2. 数据推送。虽然公司有统一的rpc框架，但是对于go，php编写的系统不够友好，所以这类的系统就统一使用http推送，但是一部分用java编写的系统，经讨论后还是考虑使用rpc远程调用，最后测试下来也确实比用http接口调用要更有效率

ok，到这里基本上就算是做好了分析了，接下来就是针对分析结果做设计，很自然的我想到了使用模板方法来解决这个问题，主要是基于以下两点考虑：

  1. 步骤简单，虽然有现成的工作流框架可以使用，但是自己实现的话应该会更轻量
  2. 只需要定义两个模板，并且借助spring可以很好的管理

接下来就上代码

AbstractDataGroupFunc这个类可以看作是数据组装的基类，通过模板方法实现了注册的功能实现了spring的增强接口InitializingBean，在spring启动的时候会自动注册所有继承该基类的实现类
package cn.magicnian.demo1022.group;import org.slf4j.Logger;import org.slf4j.LoggerFactory;import org.springframework.beans.factory.InitializingBean;import org.springframework.context.ApplicationEvent;import org.springframework.util.Assert;import java.util.Map;import java.util.concurrent.ConcurrentHashMap;public abstract class AbstractDataGroupFunc implements InitializingBean {    private static Logger logger = LoggerFactory.getLogger(AbstractDataGroupFunc.class);    private static final Map<Class<? extends ApplicationEvent>, AbstractDataGroupFunc> DATA_GROUP_MAP = new ConcurrentHashMap<>();    //初始化    protected void register(Class<? extends ApplicationEvent> eventClass, AbstractDataGroupFunc dataGroup) {        DATA_GROUP_MAP.put(eventClass, dataGroup);    }    public static AbstractDataGroupFunc getDataGroupFunc(Class<? extends ApplicationEvent> eventClass) {        Assert.isTrue(eventClass != null, "事件class不能为null!");        return DATA_GROUP_MAP.get(eventClass);    }    /**     * 数据组装-按需自由处理     * 如可以组装成json字符串或者是自定义报文格式     * @param event     * @return     */    public abstract String doDataGroup(ApplicationEvent event);}

AbstractDataPushFunc数据推送的基类，设计原理和数据组装基类一致
package cn.magicnian.demo1022.push;import org.slf4j.Logger;import org.slf4j.LoggerFactory;import org.springframework.beans.factory.InitializingBean;import org.springframework.context.ApplicationEvent;import org.springframework.util.Assert;import java.util.Map;import java.util.concurrent.ConcurrentHashMap;public abstract class AbstractDataPushFunc implements InitializingBean {    private static Logger logger = LoggerFactory.getLogger(AbstractDataPushFunc.class);    private static final Map<Class<? extends ApplicationEvent>, AbstractDataPushFunc> DATA_PUSH_FUNC_MAP = new ConcurrentHashMap<>();    protected void register(Class<? extends ApplicationEvent> eventClass, AbstractDataPushFunc dataPushFunc) {        DATA_PUSH_FUNC_MAP.put(eventClass, dataPushFunc);    }    public static AbstractDataPushFunc getDataPushFunc(Class<? extends ApplicationEvent> eventClass) {        Assert.isTrue(eventClass != null, "事件class不能为null!");        return DATA_PUSH_FUNC_MAP.get(eventClass);    }    /**     * 数据推送-自由处理     * 可以是简单的http接口，或者是rpc调用，自定义报文协议通过netty发送等     * @param param     */    public abstract void doDataPush(String... param);}
