线程与进程概念

在开始前，有必要重复下操作系统中的进程与线程的概念:

进程是指一个操作系统内存中独立运行的应用程序，每个进程都有自己独立的一块内存空间，进程包含一个或多个线程。

线程是指进程中的一个执行流程，是cpu的最小调度单元。

运行java程序，整个jvm就是一个进程，而你写的main就是启动一个线程，而GC是另一个线程。也就是说，一个jvm进程至少包含了两个线程。

为何要用多线程？

可以说，多线程的出现，大半是搞硬件的甩锅。

比如说:

为了cpu更好的与比较慢的内存协同，搞出来高速缓存，导致数据可见性问题。。。

为了充分利用cpu的处理速度，平衡和速度慢的设备如io，硬盘等，操作系统搞出了分时复用，多线程争抢资源得阻塞或等待。。。

为了让cpu更快，在主频很难提升的情况下，搞出了多核，你塞两颗，我就塞四颗。。。

java中的进程和线程实现

在 java 中，创建进程有直接关系的类为 Java.lang.Process，和实现该抽象类的 ProcessImpl 类，如果用户创建了一个进程，那么肯定会伴随着一个新的 ProcessImpl 实例。

在 JDK 的代码中，只提供了 ProcessImpl 类来实现 Process 抽象类。而这个类引用了 native 的 create, close, waitfor, destory 和 exitValue 方法来实现进程相关操作。

众所周知，java中的native方法是c/c++实现的与平台相关的实现，如在windows中，native的create方法会调用windows的api接口的CreateProcess和CreatePipe，由操作系统来创建一个真正的进程。正是由于众多native方法，才能让java有平台无关的特性。

总结，java中创建进程，会通过native方法调用操作系统级的命令，创建一个真正的进程。

线程实现 Runnable 接口或者继承 Thread类，一个 Java 线程的创建根本上就对应了一个本地线程（native thread）的创建，两者是一一对应的。

最终调用的本地方法

JVM_ENTRY(void, JVM_StartThread(JNIEnv* env, jobject jthread))

…

native_thread = new JavaThread(&thread_entry, sz);

总结，java中创建线程程，会通过native方法调用操作系统创建一个真正的线程的命令，创建一个真正的线程。
