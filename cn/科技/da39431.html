读书笔记-java高级编程之面向对象程序设计

2020-10-29
| 科技

图文无关


Java特性：一次编译，到处运行。

封装是一个重要的面向对象程序设计特征，是设计类的一个基本思想。

面向过程（Process-Oriented，PO）的程序设计强调的是模块化、结构化的程序设计方法——通过函数实现。使用面向过程的程序设计语言设计程序时，不能很好地体现现实世界中事物的本来面目，丢失了数据本来应该携带的一些逻辑信息，增加了程序维护和重用的难度。

面向对象（Object-Oriented，PO）是对现实世界模型的自然体现和延伸，它认为现实世界的各种实体（甚至是虚构的事物）都可以看成对象，对象之间通过消息相互作用。

一个对象需要有两部分描述：第一部分是它的“数据”，包括该对象的特性、状态等静态信息；第二部分是它的“行为”，包括对该对象的操作、对象的功能等能动信息。数据和行为是对象的两个基本特征。对象是可存储数据的变量和可提供操作的方法的集合，对象的核心是变量和方法。

对于现实世界中存在的类似的多个实体，我们可以将其抽象概括为某一类事物；对于在数据和行为上类似的多个对象（Object），可以将其抽象为一个类（class）来表示。类是对象的抽象。类（抽象的）是对象（具体的）的模板，而对象是类的实例。类具有对象的两个要素：一是描述对象的数据元素，称为类的属性（attribute）；另一个是描述对象的行为和功能，称为类的方法（method），它用于操作对象或执行相关对象之间的交互。属性和方法一起称为类的成员（member）。

类的声明/定义：

[<修饰符>] class <类名>{

[<属性声明>]

[<构造器声明>]

[<方法声明>]

}

类声明中所包含的数据称为属性，语法为：

[<修饰符>] <类型> <属性名称>

方法的声明：

<修饰符> <返回值类型> <方法名称> ([<参数列表>]){

[<语句>]

}

构造器是Java类中一个特殊的方法，它用于创建该类的一个新对象。换句话说，对象是构造的，不调用构造器，就不能创建一个新对象。构造器是一段无论何时使用new关键字都要运行的代码。一个类可以声明一个或者多个构造器。通常在构造器中进行一些初始化工作，如初始化类的实例变量的状态。

<修饰符> <类名> ([<参数列表>]){

[<语句>]

}

类名应做到“见名知意”，通常类名应该是名词，第一个字母和名字内其他所有单词的第一个字母应该大写，其他字母小写。属性名通常为名词，方法名通常为动词，第一个单词全部小写，后面每一个单词的第一个字母大写，其他字母小写。

构造器和方法的区别：

构造器没有返回值，而一般的方法必须指明返回值，即使是void；构造器的名称和类名相同，而一般的方法名称则没有这个限制。

在声明类时如果没有声明构造器，Java编译器将自动生成默认的构造器。默认的构造器没有参数，并且构造器代码为空。如果在声明类时显式声明了其他的构造器，那么编译器将不再提供没有参数的默认构造器。建议显式的声明一个无参数的构造器，否则使用默认构造器创建对象会引起编译错误。

信息的隐藏和封装

使用private关键字，只允许在同一个类中对属性值进行访问，而不允许在其它类中直接对其操作，这样就实现了信息隐藏。这种方法常用于需要对外隐藏敏感信息的场合。

封装（Encapsulation），即为客户端提供访问这些元素的公有接口——声明为public的方法。封装可以使我们在设计代码中做修改而不中断使用该代码的其他人的代码，这是封装的主要优点。

封装的一般做法：1.保护实例变量（用访问修饰符，通常是private）2.建立公有的读取器和设置器方法，强调调用代码使用这些方法而不是直接访问被保护的类成员3.对于读取器和设置器方法，分别使用JavaBeans命名规则get<someProperty>和set<someProperty>。

Java源文件结构：

[<包声明>]

<导入声明>*

<类声明>+

说明：<>里面是描述条款；[]代表可选项；*代表有0或多个；+代表有1或多个。一个源文件中可以声明一个或多个类，但只能有一个声明为公有的。源文件的名称必须和声明为公有的类的名称相同。

为了创建一个包，需要在声明了类和接口的Java源文件的开头写上package语句。点号“.”指明了包的层次关系。如果在Java源文件中不使用package语句，则表明所声明的类和接口位于一个默认包中。源程序中的package语句实际上是告诉编译器属于哪个包。包的命名通常采用全部小写的字母表示。Java语言要求包声明的层次和实际存放类文件（class文件）的目录结构存在对应关系，以便将来使用该类时能够通过包名（也就是目录名）查找到所需要的类文件。

import语句告诉编译器用到的类位于哪个包中。

import <包名> [.<子包名>].<类名>

import <包名> [.<子包名>].*

import语句的作用只是使当前类能访问其他包中的类（有了import语句以后就可以使用短类名，即使用类时不用加上包名，在有些条件下只能使用长类名），它不会导致编译器在内存中加载任何附加的内容，因此上面两种导入包的方式是一样的，不会对程序的编译和运行速度有影响。它和C及C++中的#include语句是不同的。

这里的package语句和import语句都只是针对class文件而言的，它指的是类文件的层次结构安排，而不是针对Java文件。package语句和import语句可以应用于源文件中所有的类和接口。

Java编译器默认为所有Java源程序都引入java.lang中所有的类（import.java.lang.*），无需显示引入。

编译文件：javac -d 目录 文件名.java
