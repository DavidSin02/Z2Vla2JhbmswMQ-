一、简介

docker镜像是由多个只读层叠加而成，启动容器时，Ddokcer会加载只读镜像层并在镜像栈顶部添加一个读写层。

如果运行中的容器修改了现有的一个已经存在的文件，那该文件将会从读写层下面的只读层复制到读写层，该文件的只读版本依然存在，只是已经被读写层中该文件的副本锁隐藏，这个就叫"写时复制（cow）"机制。

通俗易懂的就是说用户在使用Docker的过程中，往往需要能查看容器内产生的数据，或者备份容器内的数据，甚至需要多个容器之间数据共享。这就需要持久化。

实现方式：数据持久实现-->使用或者关联到宿主主机存

容器中管理数据数据主要有两种类型：数据卷、数据卷容器。

二、数据卷

数据卷是一个给容器使用的目录，他绕过容器的系统，有很多用处，如下：

1、数据可以在容器之间共享使用

2、对数据卷的修改会立马生效

3、对数据卷的更新，不会影响镜像

4、卷会一直存在的，知道没有容器使用它。

数据卷的使用和linux系统monut一样。


在使用docker run的时候使用-v参数即可在容器内创建一个数据卷，可以多次使用（创建多个卷）


使用方法如下：

# docker run --name d2 -it -v /data busybox

补充：数据卷可以挂载目录，当然也可以直接挂载文件。

# docker run --name d3 -it -v /root/.bash_history:/root/.bash_history busybox

这样就可以记录在容器里输入过的命令了。一般情况不用这种方式，容易导致inode的改变。官方推荐还是使用目录。

三、数据卷容器

如果要在容器之间共享一些持续更新的数据，最简单的方式就是使用数据卷容器，数据卷容器其实就是一个简单的容器。用他来提供数据卷给其他容器挂载使用。

方法一：

先使用数据卷挂载方法，挂载

# docker run --name vol01 -it --rm -v /data/vol01:/data busybox

/data/vol01这个目录一般会自动创建，当然用创建好的也行

# docker inspect vol01

# docker inspect -f {{.Mounts}} vol01

[{bind /data/vol01 /data true rprivate}]

依然创建文件

下面启动两个容器分别使用

docker run --name vol02 -it --rm -v /data/vol01:/data busybox

docker run --name vol03 -it --rm -v /data/vol01:/data busybox

方法二：

也可以使用下面的命令--volumes-from，顺带做一个共享网络的测试--network

joined containers，在nginx对外，tomcat，mysql不需要对外的情况

这样共享网络资源，减少开销

docker run --name joined -it -v /data/joined/vol01:/data/web/html busybox

其他容器使用

docker run --name nginx --network container:joined --volumes-from joined -it --rm busybox

docker run --name tomcat --network container:joined --volumes-from joined -it --rm busybox

以上nginx-tomcat-mysql等就是属于容器编排--docker-compose

--volumes-from可以多次使用，如果删除容器，卷并不会被删除。如果要删除一个卷，必须删除关联容器。

使用数据卷容器，可以自由升级避免数据丢失。
