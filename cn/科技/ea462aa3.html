分支

到现在为止，我们的小型汇编程序在一条指令之后执行另一条指令。如果我们的ARM处理器只能以这种方式运行，那将是有限的用途。它无法对可能需要不同指令顺序的现有条件做出反应。这是分支指令的目的。

特殊寄存器

在第2章中，我们了解到Raspberry Pi ARM处理器具有16个整数通用寄存器，并且当时指出其中有些寄存器有特殊的作用。 因为那时还用不到，我故意忽略了。

现在我们将用到其中一个特殊的寄存器R15,听到这个名字可能你对这个寄存器的功能会很困惑，但是听到他的别名PC寄存器你可能会有更深刻的认知，当然前提是你对计算机原理有过了解。

PC代表什么？ pc表示程序计数器。 这个名字的起源是在计算的蛮荒时期，如今几乎没有任何意义。 通常，pc寄存器（在其他体系结构（如386或x86_64）中也称为ip，指令指针）包含要执行的下一条指令的地址。

当ARM处理器执行一条指令时，在其执行结束时可能会发生两件事。 如果指令没有修改pc（大多数指令没有修改），则pc只会增加4（就像我们确实添加了pc，pc，＃4一样）。 为什么是4？ 因为在ARM中，指令为32位宽，所以每条指令之间有4个字节。 如果指令修改了pc，则使用pc的新值。

一旦处理器完全执行了一条指令，它就将pc中的值用作下一条要执行的指令的地址。 这样，一条不会修改pc的指令将紧随内存中的下一条连续指令（因为它已自动增加4）。 这称为指令的隐式排序：一条指令运行后，通常会运行内存中的下一条指令。 但是，如果一条指令确实将pc修改为例如pc + 4以外的其他值，那么我们可以运行该程序的另一条指令。 更改pc值的过程称为分支。 在ARM中，这是使用分支指令完成的。

无条件分支

您可以使用指令b（用于转移）和标签来告诉处理器无条件转移。 考虑下面的程序。

/* -- branch01.s */

.text

.global main

main:

mov r0, #2 /* r0 ← 2 */

b end /* branch to 'end' */

mov r0, #3 /* r0 ← 3 */

end:

bx lr

如果执行此程序，您将看到它返回错误代码2。（如果不知道下面输出的意思可以参考第一章的说明）

$ ./branch01 ; echo $?

2

发生的是，指令b end分支（修改了pc）到标签端的指令bx lr，即我们在程序末尾运行的指令。 这样，指令mov r0，＃3实际上根本没有运行（处理器从未到达该指令）。

在这一点上，无条件分支指令b可能看起来毫无用处。 事实并非如此。 实际上，该指令在某些情况下是必不可少的，特别是在与条件分支一块使用时。

条件分支

如果我们的处理器仅能够顺序执行，那将不会很有用。 满足某些条件时分支将更加有用。 因此，处理器应该能够根据某些条件进行分别处理。

在继续之前，我们需要公开另一个名为cpsr的寄存器（用于“当前程序状态寄存器”）。 该寄存器有点特殊，直接修改它超出了本章的范围。 也就是说，它保留一些在执行指令时可以读取和更新的值。 该寄存器的值包括四个条件代码标志，分别称为N（负），Z（零），C（进位）和V（溢出）。 这四个条件代码标志通常由分支指令读取。 如果需要，算术指令以及特殊的测试和比较指令也可以更新这个寄存器的值。

在更新cpsr的指令中，这四个条件代码的语义大致如下:

  • N: 如果指令结果为负数，则启用N。 否则禁用。
  • Z: 如果指令的结果为零，则将启用Z。 如果非零，则禁用。
  • C: 如果指令的结果产生的值需要完全表示第33位，则将启用C。 例如，加法溢出了32位整数范围。
  • V: 如果指令结果产生的值不能用32位二进制补码表示，则将启用V。

因此，我们具有所有必要的条件来有条件地执行分支。 但首先，让我们开始比较两个值。 为此，我们使用指令cmp。

cmp r1, r2 /* updates cpsr doing "r1 - r2", but r1 and r2 are not modified */

该指令将第二个寄存器中的值减去第一个寄存器中的值。 上面的代码片段可能发生什么事？

  • 如果r2的值（严格地）大于r1，则将启用N，因为r1-r2将产生负结果。
  • 如果r1和r2具有相同的值，则将启用Z，因为r1-r2将为零。
  • 如果r1为1且r2为0，则r1-r2将不会借用，因此在这种情况下将启用C。 如果交换了值（r1为0，r2为1），则C将被禁用，因为减法确实会借用。
  • 如果r1为2147483647（32位二进制补码中的最大正整数）且r2为-1，则r1-r2将为2147483648，但该数字不能用32位二进制补码表示，因此将启用V来表示这一点。

我们如何使用这些标志来表示程序的有用条件？

  • EQ（相等）启用Z时（Z为1）
  • NE（不相等）。 Z禁用时。 （Z为0）
  • GE（大于或等于，用两个补码表示）。 当V和N都启用或禁用时（V为N）
  • LT（低于两位数）。 这与GE相反，因此当V和N均未启用或禁用时（V不是N）
  • GT（大于2）。 当禁用Z且同时启用或禁用N和V时（Z为0，N为V）
  • LE（小于或等于，用两个补码表示）。 启用Z或启用Z时，N和V均启用或禁用（Z为1。如果Z不为1，则N为V）
  • MI（负/负）启用N时（N为1）
  • PL（正/正或零）当禁用N时（N为0）
  • VS（溢出设置）启用V时（V为1）
  • VC（溢出清除）禁用V时（V为0）
  • HI（较高）当启用C而禁用Z时（C为1并且Z为0）
  • LS（较低或相同）当禁用C或启用Z（C为0或Z为1）时
  • CS / HS（进位设置/更高或相同）当启用C（C为1）时
  • CC / LO（进位清除/降低）当C被禁用（C为0）时

这些条件可以与我们的b指令组合以生成新指令。 这样，仅当Z为1时，beq才会分支。如果不满足条件分支的条件，则忽略该分支，并运行下一条指令。 确保在条件分支之前正确设置条件代码是程序员的任务。

/* -- compare01.s */

.text

.global main

main:

mov r1, #2 /* r1 ← 2 */

mov r2, #2 /* r2 ← 2 */

cmp r1, r2 /* update cpsr condition codes with the value of r1-r2 */

beq case_equal /* branch to case_equal only if Z = 1 */

case_different :

mov r0, #2 /* r0 ← 2 */

b end /* branch to end */

case_equal:

mov r0, #1 /* r0 ← 1 */

end:

bx lr

如果运行该程序，则由于r1和r2的值相同，因此将返回错误代码1。 现在，将第5行中的mov r1，＃2更改为mov r1，＃3，返回的错误代码应为2。请注意，case_different我们不想运行case_equal指令，因此我们必须跳转到结束（否则错误 代码始终为1）。
