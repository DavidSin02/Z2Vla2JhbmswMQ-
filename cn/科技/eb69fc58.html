git 解决冲突的命令

2020-11-14
| 科技

几乎每个程序员都有一个GitHub账号，成为程序员自后你会发现使用GitHub搜索问题会比百度或者谷歌等搜索引擎，还没有申请GitHub的你赶快去申请吧！GitHub的上面开源的语言和各种程序库，上面的文档是跟mk一样的阅读代码特别的舒服，使用GitHub离不开git的使用，现在也有很多的工具免命令比如乌龟仔，但是我还是喜欢使用命令行来操作。

如何申请GitHub和安装git我就不详细介绍了，不是本篇文章的重点，自己自行谷歌或者百度去吧！

git 分三个区域：

  1. 工作区（R）
  2. 暂存区 (B)
  3. 版本区 (V）

理解git的这3个区的区别基本就学会一半的git了，这里推荐廖雪峰老师的git教程那里面写的很详细！本篇主要是讲git冲突的解决。

git pull

命令用于从另一个存储库或本地分支获取并集成(整合)。

作用是：取回远程主机某个分支的更新，再与本地的指定分支合并，它的完整格式稍稍有点复杂。

相当于：

git fetch origin 获取远程仓库的代码

git merge origin/master 本地代码跟远程代码合并

git pull 是上面两个命令的组合。

注意：在执行代码 git merge origin/master 的时候有可能会报错，

执行 git branch origin/master md 也就是将远程仓库分支更改名字，在执行合并代码的命令就行。

介绍一个命令：

git branch --set-upstream master origin/next

git 追踪关系：origin 追踪远程的 origin、next
