异步(async和await)

2020-10-29
| 科技

一.为什么要使用异步编程？

现在的大多数的应用都广泛使用文件和网络I/O，而这些操作大多都是阻塞API的，为了充分利用CPU资源，提高效率。

注意：异步(async和await)可以提高响应能力也就是吞吐量，但是不能提高响应速度。也就是说处理一个请求的时间是保持不变的，但是可以释放当前线程，增加处理能力

二.基于任务的异步编程模型TAP

计算密集型任务

  • .net frameowrk 4 可以使用TaskFactory.StartNew或者Task.Factory.StartNew的方式创建异步任务
  • .net 4.5以上以及.net core使用Task.Run作为TaskFactory.StartNew的快捷方式

其中Task.ContinueWith、TaskFacotry.ContinueWhenAll、TaskFactory.ContinueWhenAny方法可以实现创建一下再另一个任务完成后需要执行的新任务.

如果在任务执行前，可以通过取消标记Cancellation对象，可以停止任务

  class Program    {        //声明CancellationTokenSource对象        static CancellationTokenSource cancelTokenSource = new CancellationTokenSource();        //程序入口        static void Main(string[] args)        {            Task.Factory.StartNew(Test, cancelTokenSource.Token);            Console.ReadLine();            cancelTokenSource.Cancel();            Console.WriteLine("已停止");            Console.ReadLine();        }        //测试方法        static void Test()        {            //判断是否取消任务            while (!cancelTokenSource.IsCancellationRequested)            {                Console.WriteLine(DateTime.Now);                Thread.Sleep(1000);            }        }    }

I/O密集型任务

I/O密集型任务是以Windows事件为基础的，因此不需要新建一个线程或使用线程池里面的线程来执行具体工作。不要使用Task.Run,反而浪费资源。 IO场景中，文件IO由win32用户模式的API到windows内核模式，在内核模式中操作磁盘驱动程序。可能会调入系统 API 调用（例如 Linux 上套接字的 write()）。可能会使用 TaskCompletionSource 在本机/托管边界创建一个任务对象。

三.async 和 await

什么是async和await

这是是一种语法糖，是对异步编程模型(TPL基于任务的异步编程模型)的封装

async和await的本质
  • async会在编译层生成状态机，每一个await就是一个状态，如果存在await嵌套，那就是一个状态一个状态的转换。如果只有async而没有await，此时状态机将不会生成getawater，此时将不会等待处理结果。相当于多线程编程
  • async和await本身并不会在生成的代码中新建一个线程，新开的线程都是在await后面的方法中自己创建的。

四.异步和同步流程比较

当主线程读取一个应用程序之外的资源，比如处理文件资源或者外部服务的时候。主线程会从用户模式进入到内核模式。在内核模式中，windows会将请求数据移交给对应的驱动程序去处理。

  • 如果是同步的话，那么当前主线程会进入休眠状态。当驱动程序和外部资源进行交互后，网络驱动程序拿到结果，然后移交给当初睡眠的线程，windows会唤醒线程，继续执行后面的C#代码

  • 异步的话，此时主线程不会休眠，而是直接返回线程池。当网络驱动程序拿到结果后，会将数据交给CLR线程池中的IO线程，然后由这个线程触发回调函数。

 class Program    {        static void Main(string[] args)        {            CallerWithAsync("jack");            Console.ReadKey();        }                //重点调用方法        async static void CallerWithAsync(string name)        {            Console.WriteLine($"异步调用头部执行，当前线程Id为：{Thread.CurrentThread.ManagedThreadId}");            string result =  SayHiAsync(name).Result;            Console.WriteLine($"异步调用尾部执行，当前线程Id为：{Thread.CurrentThread.ManagedThreadId}");            Console.WriteLine(result);        }                 async static void CallerWithAsyn1c(string name)        {            Console.WriteLine($"异步调用头部执行，当前线程Id为：{Thread.CurrentThread.ManagedThreadId}");            string result =await  SayHiAsync(name);            Console.WriteLine($"异步调用尾部执行，当前线程Id为：{Thread.CurrentThread.ManagedThreadId}");            Console.WriteLine(result);        }        static Task<string> SayHiAsync(string name)        {            return Task.Run<string>(() => { return SayHi(name); });        }        static string SayHi(string name)        {            Task.Delay(2000).Wait();//异步等待2s            Console.WriteLine($"SayHi执行，当前线程Id为：{Thread.CurrentThread.ManagedThreadId}");            return $"Hello,{name}";        }             }
  • CallerWithAsync采用的是Result的方式获取方法的值，这种方式是阻塞当前线程的。也就是当前线程是在等待。此时会造成当前线程资源的浪费。
  • CallerWithAsync1采用的是await的方式获取返回值。这是非阻塞的方式获取，此时会将当前线程直接释放回线程池。当Task.Run中的代码执行完毕后，会重新获取一个线程，然后执行主线程后面的代码。所以重新获取的这个线程可能和原来的主线程一样。但是基本上是另外一个线程。
