WinCC 的输入确认对话框制作方法

2020-11-14
| 科技

有时候客户会要求在做wincc重要的数值输入时需要进行弹窗确认，这样会比较安全。

思路大概就是给输入输出域的输入和输出属性分别连接一个变量，判断输入值变化时弹出对话框，让操作员确认。如果操作员确认，则将输入值赋值给输出值，反之则什么都不做。

这样做法的好处是：1、操作员界面直观，2、输出值是真正的过程参数，如果操作员不确认，这个变量不会受到任何干扰。

下面就来讲解制作方法。

1.首先在变量的内部变量中建立一个临时变量和一个plc的寄存器变量。然后画面中制作输入输出域，在属性中的输入属性中选择新建的临时变量。在输出属性中选择PLC的寄存器变量。




由于我的软件的输入输出属性是相反的所以我选的变量是相反的。正常应该是输入选临时变量，输出选寄存器变量。


2.在事件对话框中选择键盘中的释放功能，右键选择C动作写入下面的脚本。编译没有错误后就可以确定运行了。

{int TempValue=GetTagWord("TempValue"); //这段意识是获取你写入临时变量的数值。

char szBuffer[20]; 设置数据长度

sprintf ( szBuffer,"是否输入数值:%d",TempValue); //这段就是对话框显示的询问语言

if ( nChar==13 )//if press enter 这段是获取按回车键状态

{

int xRet = MessageBox(NULL,szBuffer,"确认窗口",MB_YESNO|MB_ICONQUESTION|MB_SYSTEMMODAL);//message 这段是调用弹窗语句

if ( xRet == IDYES )//confirm operate这段是确认是否操作

{

SetTagWord("d10",TempValue);// set data 这段是按是后将临时变量的数值写入寄存器变量

}

}





上图是运行后的效果点击是数值写入，点击否数值不写入。这样就可以有效的提示操作人员确定数值正确后再输入了。
