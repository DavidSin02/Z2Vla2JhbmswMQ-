3.进入SVC世界

使用svc的代码与朴素贝叶斯的代码相似。 我们首先从库中导入svc。 接下来，我们提取训练功能和标签。 最后，我们要求模型预测测试集的标签。 基本的代码块片段如下所示：

嗯...我们如何减少训练时间？

一种方法是减少训练集的大小。 我们将其减小为原始大小的1/10，然后检查准确性。 当然它会减少。

在这里，我们有702封电子邮件用于培训。 1/10意味着70封电子邮件用于培训，这是非常少的。 （尽管结帐使我们感到惊奇）。

在训练模型之前，添加以下行。 （它将feature_matrix和标签减少1/10）。

features_matrix = features_matrix[:len(features_matrix)/10]labels = labels[:len(labels)/10]


现在什么是训练时间和准确性？

参数调整

我想您会收到大约56％的准确性。 太低了

现在将训练集保持为1/10，让我们尝试调整三个参数：kernel，C和gamma。

1.内核

将内核更改为rbf。 即在模型= SVC（）中添加内核参数

model = svm.SVC(kernel="rbf", C = 1)


2. C

接下来，将C（正则化参数）更改为10、100、1000、10000。确定精度是增加还是减少？

您会注意到，在C = 100时，准确度分数提高到85.38％，并且在此之上几乎保持相同。

3.伽玛

最后，让我们玩伽玛。 再增加一个参数gamma = 1.0

model = svm.SVC(kernel="rbf", C=100, gamma=1)

糟糕！ 准确性得分下降。 对？ 尝试将gamma = 10设为更高的值。 尝试减少。 使用值0.1、0.01、0.001。 现在的精度是多少？ 它在增加吗？

您会注意到，在这种运动情况下，较低的伽玛值使我们具有很高的准确性。 （直觉：这意味着数据点稀疏，离图形图中的决策边界足够远）。

在这种情况下，我们注意到减小训练集大小可以达到85.4的最佳效果。 （PS：朴素贝叶斯的准确度得分是多少？）

最后的想法

通常，SVC比朴素贝叶斯需要更多的训练时间，但预测速度更快。 在上面的编码练习中，朴素贝叶斯的表现优于SVC。 但是，这完全取决于性能最佳的方案和数据集。

即使将训练数据集减少到1/10，也可以获得更高的准确性得分。

但是，为什么我们需要减少训练量呢？

训练时间更长，SVC的训练时间通常是朴素贝叶斯的3倍。 在某些应用中，与准确度相比，我们需要更快的预测。

· 想想信用卡交易。 快速响应交易欺诈标记比99％的准确性更为重要。 在这里可以容忍90％的精度。

· 另一方面，只有将电子邮件标记为垃圾邮件可以忍受延迟，我们将努力提高准确性。

我们是否需要始终调整参数？

并不是的。 sklearn工具套件中有内置功能可以为我们完成任务。 我们将在以后的文章中探讨它们。

希望本教程为您提供有关SVC编码的基本概念。 即使对于较小的数据集，我们如何调整参数并实现合理的精度。 （我们在培训集中只有70封电子邮件，在针对350封电子邮件的测试中达到了85％的准确性）。
