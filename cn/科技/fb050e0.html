访问控制列表－ACL

2020-10-29
| 科技

在企业网络通信时，要保证网络的安全可靠性和稳定性。访问控制列表ACL（Access Control List）可以定义一系列不同的规则，设备根据这些规则对数据包进行分类，并针对不同类型的报文进行不同的处理，从而可以实现对网络访问行为的控制、限制网络流量、提高网络性能、防止网络攻击等等。

如上图所示：ACL可以控制流量可访问的区域。如图：主机A B 可以访问Internet网但是不能访问服务器，主机C D 则相反。这都是通过ACL控制的。

如上图所示：也可以通过ACL匹配网段进行加密处理。

ACL分三类：

ACL 2000-2999：可以匹配源IP地址。

ACL 3000-3999：可以匹配源IP 目的IP 源端口 目的端口等

ACL 4000-4999：可以匹配源Mac 目Mac和以太帧协议类型等。

ACL后面数字越高就越高级，可以匹配的条件就越多。

一条ACL可以匹配创建多个规则，数据会根据规则序号从上往下匹配，直到匹配到为止。若无匹配则丢弃数据。

配置：如图所示，拒绝主机A访问Internet网。

命令行：

ACL 2000（创建acl）

rule 5 deny source 192.168.1.0 0.0.0.255（创建第五条规则 拒绝源地址为 192.168.1.0掩码为24位IP网段 rule 规则条目 不输入默认第五条开始 再创建不输入就从第十条开始目的是方便在前面添加规则 deny拒绝的意思 source源地址的意思 permit允许的意思）

int g0/0/0 （进入接口0）

traffic-filter outbound acl 2000 （调用acl 2000）

这样配置192.168.1.0网段就不能访问Internet网了

高级acl配置：

acl 3000

rule deny tcp soure 192.168.1.0 0.0.0.255 destination 172.16.10.0 0.0.0.0 destination-port ep 21（拒绝源地址为192.168.1.0IP网段的主机访问目的IP为172.16.10.1端口号为21的服务器。）（tcp是协议 匹配所有tcp报文 destination是目的）

rule deny tcp soure 192.168.2.0 0.0.0.255 destination 172.16.10.2 0.0.0.0 （上面注释那么多这个应该懂了吧）

rule permit ip （匹配所有IP报文）

int g0/0/0

traffic-filter outbound acl 3000（调用acl 3000）
