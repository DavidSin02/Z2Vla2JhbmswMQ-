導論

bibliometrix軟件包為文獻計量學和科學計量學的定量研究提供了一套工具。

文獻計量學本身就是科學的主要工具--定量分析。從本質上講，文獻計量學是對期刊文章及其附帶的引文計數等出版物進行定量分析和統計的應用。現在幾乎所有的科學領域都使用對出版和引用數據的定量評估來評估一個科學界的成長、成熟度、主要作者、概念圖和智力圖以及趨勢。文獻計量學也被用於研究績效評估，特別是在大學和政府實驗室，也被政策制定者、研究主管和管理者、信息專家和圖書館員以及學者本身使用。

bibliometrix在三個關鍵的分析階段為學者提供支持：

  • 數據導入和轉換為R格式；
  • 出版物數據集的文獻計量學分析；
  • 構建共引、耦合、協作和共詞分析矩陣。矩陣是用於執行網絡分析、多重對應分析和任何其他數據簡化技術的輸入數據。

書目數據庫

Bibliometrix使用從四個主要書目數據庫提取的數據：Scopus、Clarivate Analytics Web of Science、Cochrane Database of Systemical Reviews(CDSR)和RISmed PubMed/Medline。

SCOPUS(http://www.scopus.com)，成立於2004年，為文獻計量用戶提供了極大的靈活性。它允許查詢不同的字段，如標題、摘要、關鍵字、參考文獻等。Scopus允許相對容易地下載數據查詢，儘管對於超過2,000個項目的非常大的結果集有一些限制。

Clarivate Analytics Web of Science(http://www.webofknowledge.com)是由文獻計量學的先驅尤金·加菲爾德創立的。這個平臺包括許多不同的集合。科克倫系統評價數據庫(http://www.cochranelibrary.com/cochrane-database-of-systematicreviews/index.html)是醫療保健系統評價的領先資源。

CDSR包括Cochrane評價(系統評價)和Cochrane評價方案以及社論。CDSR也有偶爾的補充劑。當Cochrane評論“準備就緒”並形成月刊時，CDSR會定期更新；參見出版時間表。

PubMed包括MEDLINE、生命科學期刊和在線圖書對生物醫學文獻的2800萬多條引文。引文可能包括指向PubMed Central和出版商網站全文內容的鏈接。

bibliometrix軟件包安裝

下載並安裝最新版本的R及R Studio，在rstudio控制檯窗口中輸入：

install.packages("bibliometrix",dependencies=TRUE)library(bibliometrix)   ### load bibliometrix package

數據加載和轉換

可以使用函數convert2df通過R讀取和轉換導出文件：

convert2df(file, dbsource, format)

參數文件是一個字符矢量，包含從Scopus、Clarivate Analytics WOS、Digital Science Dimenion、PubMed或Cochrane CDSR網站下載的導出文件的名稱。file還可以包含使用Digital Science Dimenion或PubMed API下載的json/xlm對象的名稱(通過包dimensionsR和pubmedR)。

es. file <- c(“file1.txt”,“file2.txt”, …)

data(biblio)M<- convert2df(file = biblio, dbsource = "isi", format = "bibtex")

convert2df創建書目數據框，其中案例對應於原稿，變量對應於原始導出文件中的Field Tag。

convert2df接受兩個附加參數：dbsource和format。

它可以是：

  • “isi”或“wos”(用於Clarivate Analytics Web of Science數據庫)，
  • “scopus”(用於Scopus數據庫)，
  • “dimension”(用於DS Dimensions數據庫)，
  • “pubmed”(用於PubMed/Medline數據庫)，
  • “cochrane”(用於Cochrane圖書館系統評價數據庫)。

參數format指示導入集合的文件格式。對於WOS集合，它可以是“plaintext”或“bibtex”，對於SCOPUS集合，它可以是強制的“bibtext”。如果集合來自Pubmed或Cochrane，則忽略該參數。

函數摘要和繪圖

要彙總文獻計量分析的主要結果，請使用泛型函數summary。它顯示了關於書目數據框架和幾個表格的主要信息，如年度科學產量、每引文數量最高的手稿、最多產的作者、最多產的國家、每個國家的總引文量、最相關的來源(期刊)和最相關的關鍵字。

主要信息表根據文獻數、作者數、來源數、關鍵詞數、時間跨度和平均引文數來描述集合大小。

此外，還顯示了許多不同的合著索引。特別地，每篇文章的作者指數（Authors per Article index）被計算為作者總數與文章總數之間的比率。每篇文章的合著者指數（Co-Authors per Articles index ）是根據每篇文章的平均合著者數計算的。在這種情況下，指數考慮了作者出現的情況，而對於“每篇文章的作者”，即使作者發表了一篇以上的文章，也只計算一次。因此，每篇文章的作者指數≤每篇文章的合著者指數。

協作指數(Collaboration Index ,CI)的計算方式為所有作者的多作者文章總數/多作者文章總數(Elango和Rajendran，2012；Koseoglu，2016)。換言之，協作指數是僅使用多作者文章集計算的每篇文章的合著者指數。

Elango, B., & Rajendran, P. (2012). Authorship trends and collaboration pattern in the marine sciences literature: a scientometric study. International Journal of Information Dissemination and Technology, 2(3), 166.

Koseoglu, M. A. (2016). Mapping the institutional collaboration network of strategic management research: 1980–2014. Scientometrics, 109(1), 203-226.

被引文獻分析

函數citations生成最常被引用的參考文獻或最常被引用的(參考文獻的)第一作者的頻率表。

對於每份手稿，引用的參考文獻都存儲在數據框的列“CR”中的單個字符串中。

為了正確提取，您需要在ISI或SCOPUS數據庫使用的不同引用中標識分隔符字段。通常，默認分隔符是“;”或“. ”(帶雙空格的點)。

M$CR[1]

要獲得最常被引用的手稿：

CR <- citations(M, field = "article", sep = ";")cbind(CR$Cited[1:10])

要獲得最常被引用的第一作者，請執行以下操作：

CR <- citations(M, field = "author", sep = ";")cbind(CR$Cited[1:10])

函數localCitations生成被引用最多的本地作者的頻率表。本地引文量衡量此集合中包含的作者(或文檔)被集合中的其他作者引用的次數。要獲取當地最常被引用的作者，請執行以下操作：

CR <- localCitations(M, sep = ";")CR$Authors[1:10,]

CR$Papers[1:10,]

作者優勢排名

Kumar&Kumar，2008提出的函數 dominance計算作者的優勢度排名。

Kumar, S., & Kumar, S. (2008). Collaboration in research productivity in oil seed research institutes of India. In Proceedings of Fourth International Conference on Webometrics, Informetrics and Scientometrics.

函數參數是：由biblioAnalysis獲得的results(bibliometrix類的對象)；以及k(分析中要考慮的作者數量)。

DF <- dominance(results, k = 10)DF

主導因子是一個比率，表示學者作為第一作者出現在多作者論文中的比例。

在這個例子中，科斯托夫和霍爾登主導了他們的研究團隊，因為他們在所有論文中都是第一作者(科斯托夫8篇，霍爾登3篇)。

作者h指數

h指數是一種作者級別的衡量標準，它試圖衡量科學家或學者發表的論文的生產率和引文影響。

該指數是基於這位科學家被引用最多的論文集和他們在其他出版物上收到的被引用數量。

函數Hindex計算書目集合中作者的H指數或來源的H指數及其變體(g-index and m-index)。

函數參數包括：M書目數據框；field是根據作者(field=“author”)或來源(field=“source”)定義分析單位的字符元素；elements包含要計算其H指數的作者名稱(或來源名稱)的字符向量。參數的形式為c(“SURNAME1 N”，“SURNAME2 N”，…)。

換句話說，對於每個作者：姓氏和首字母之間用一個空格隔開。也就是說，對於作者Aria Massimo和Cuccurullo Corrado，elements參數是elements=c(“Aria M”，“Cuccurullo C”)。

要計算此集合中Lutz Bornmann的h指數，請執行以下操作：

indices <- Hindex(M, field = "author", elements="BORNMANN L", sep = ";", years = 10)# Bornmann's impact indices:indices$H

# Bornmann's citationsindices$CitationList

要計算前10位生產率最高的作者(在此集合中)的h指數，請執行以下操作：

authors=gsub(","," ",names(results$Authors)[1:10])indices <- Hindex(M, field = "author", elements=authors, sep = ";", years = 50)indices$H

洛特卡定律係數估計

lotka函數估計科學生產力的洛特卡定律係數(Lotka A.J.，1926)。

洛特卡定律將作者在任何給定領域的發表頻率描述為反平方律，其中發表一定數量的文章的作者數量與發表一篇文章的作者數量的比例是固定的。這一假設意味著洛特卡定律的理論貝塔係數等於2。

利用lotka函數可以估計我們的書目館藏的貝塔係數，並通過統計檢驗來評估這一經驗分佈與理論分佈的相似性。

L <- lotka(results)# Author Productivity. Empirical DistributionL$AuthorProd

# Beta coefficient estimateL$Beta# ConstantL$C# Goodness of fitL$R2# P-value of K-S two sample testL$p.value

表L$AuthorProd顯示了我們示例中觀察到的科學生產力分佈。

估計的貝塔係數為2.78，擬合優度為0.97。Kolmogorov-Smirnov兩樣本檢驗提供的p值為0.699，這意味著觀測到的洛特卡分佈與理論洛特卡分佈之間沒有顯著差異。

您可以使用plot函數比較這兩個分佈：

# Observed distributionObserved=L$AuthorProd[,3]# Theoretical distribution with Beta = 2Theoretical=10^(log10(L$C)-2*log10(L$AuthorProd[,1]))plot(L$AuthorProd[,1],Theoretical,type="l",col="red",ylim=c(0, 1), xlab="Articles",ylab="Freq. of Authors",main="Scientific Productivity")lines(L$AuthorProd[,1],Observed,col="blue")legend(x="topright",c("Theoretical (B=2)","Observed"),col=c("red","blue"),lty = c(1,1,1),cex=0.6,bty="n")

書目網絡矩陣

手稿的屬性通過手稿本身相互聯繫：作者與期刊、關鍵詞與出版日期等。

這些不同屬性的聯繫形成了可用矩形矩陣(手稿×屬性)表示的二元網絡。

此外，科學出版物經常引用其他科學著作。這產生了另一個網絡，即共引或耦合網絡。

分析這些網絡是為了捕捉底層研究系統的有意義的屬性，特別是確定諸如學者和期刊等文獻計量單位的影響。

二部網絡

cocMatrix是用於計算選擇元數據屬性之一的二部網絡的通用函數。例如，要創建網絡手稿x出版源，您必須使用字段標記“SO”：

A <- cocMatrix(M, Field = "SO", sep = ";")

A是矩形二進制矩陣，表示二部網絡，其中行和列分別是手稿和來源。

通用元素aij，如果手稿已在源代碼中發佈，則為1，否則為0。

第j列的和aj是在來源j中發表的稿件數量。

按降序對A的列和進行排序，您可以看到最相關的出版物來源：

A <- cocMatrix(M, Field = "SO", sep = ";")sort(Matrix::colSums(A), decreasing = TRUE)[1:5]

按照此方法，您可以計算多個二部網絡：

引文網絡

A <- cocMatrix(M, Field = "CR", sep = ". ")

作者網絡

A <- cocMatrix(M, Field = "AU", sep = ";")

國家網絡

作者國家不是書目數據框架的標準屬性。您需要使用函數metaTagExtraction從從屬關係屬性中提取此信息。

M <- metaTagExtraction(M, Field = "AU_CO", sep = ";")# A <- cocMatrix(M, Field = "AU_CO", sep = ";")

metaTagExtraction允許提取以下附加字段標籤：作者的國家(Field = "AU_CO")；第一作者的國家(Field = "AU_CO")；每個引用文獻的第一作者(Field = "CR_AU")；每個引用文獻的出版物來源(Field = "CR_SO")；以及作者的從屬關係(Field = "AU_UN")。

作者關鍵詞網絡

A <- cocMatrix(M, Field = "DE", sep = ";")

Keyword Plus網絡

A <- cocMatrix(M, Field = "ID", sep = ";")

等等。

書目耦合

如果兩篇文章的參考書目或參考文獻列表中至少出現一個被引用的來源，那麼這兩篇文章就被認為是書目耦合的(Kessler，1963)。

可以使用通式獲得耦合網絡：

B = A × AT

其中A是二部網絡。元素bij指示在文稿i和文稿j之間存在多少書目耦合換句話說，bij給出了長度為2的路徑的數量，通過這些路徑，人們可以沿著箭頭從j移動，然後向相反的方向j移動。

B是一個對稱矩陣 B=BT。

兩篇文章的耦合強度，簡單地由文章共有的引用次數來定義，如矩陣B所給出的元素bij。

函數biblioNetwork從書目數據框架開始計算最常用的耦合網絡：作者、來源和國家。

biblioNetwork使用兩個參數來定義要計算的網絡：

analysis參數可以是“co-citation”, “coupling”, “collaboration”, 或 “co-occurrences”。

network參數可以是 “authors”, “references”, “sources”, “countries”, “universities”, “keywords”,

“author_keywords”, “titles” 和 “abstracts”。

以下代碼計算經典文章耦合網絡：

NetMatrix <- biblioNetwork(M, analysis = "coupling", network = "references", sep = ".  ")

因此，如果僅僅根據文章包含的共同參考文獻數量來衡量耦合強度，那麼只有幾篇參考文獻的文章往往具有更弱的書目耦合。這表明，切換到書目耦合的相對度量可能更實際。

normalizeSimilarity函數計算網絡頂點之間的Association strength、Inclusion、Jaccard或Salton相似度。可以使用參數normalize直接從networkPlot()函數調用normalizeSimilarity。

NetMatrix <- biblioNetwork(M, analysis = "coupling", network = "authors", sep = ";")net=networkPlot(NetMatrix,  normalize = "salton", weighted=NULL, n = 100,                 Title = "Authors' Coupling", type = "fruchterman", size=5,size.cex=T,                remove.multiple=TRUE,labelsize=0.8,label.n=10,label.cex=F)

書目共引

我們談到當兩篇文章在第三篇文章中都被引用時，我們會討論兩篇文章的共同引文。因此，共引可視為書目耦合的對應物。使用一般公式可以獲得共引網絡：

C = AT × A

其中A是二部網絡。和矩陣B一樣，矩陣C也是對稱的。C的主對角線包含在我們的數據框中引用的案例數。

換言之，對角線元素ci是參考文獻的局部引用次數。使用函數biblioNetwork，您可以計算經典的參考共引網絡：

# NetMatrix <- biblioNetwork(M, analysis = "co-citation", network = "references", sep = ".  ")

書目協作

科學協作網絡是一個網絡，其中節點是作者，鏈接是合著者，因為後者是記錄最充分的科學協作形式之一(Glanzel，2004)。使用通式可以獲得作者協作網絡：

AC = AT × A

其中A是一個二部網絡，手稿x作者。對角線元素aci是研究人員i撰寫或合著的稿件數量。

使用函數biblioNetwork，您可以計算出作者的協作網絡：

NetMatrix <- biblioNetwork(M, analysis = "collaboration", network = "authors", sep = ";")

或國家/地區協作網絡：

NetMatrix <- biblioNetwork(M, analysis = "collaboration", network = "countries", sep = ";")

網絡圖特徵的描述性分析

函數networkStat計算幾個彙總統計信息。

特別地，從書目矩陣(或igraph對象)開始，計算兩組描述性度量：

網絡的彙總統計；

頂點的中心度和威望的主要指數。

# An example of a classical keyword co-occurrences networkNetMatrix <- biblioNetwork(M, analysis = "co-occurrences", network = "keywords", sep = ";")netstat <- networkStat(NetMatrix)

網絡的彙總統計

這組統計數據可用於描述網絡的結構屬性：

  • Size是組成網絡的折點數量；
  • Density是網絡中所有可能邊的當前邊的比例；
  • Transitivity是三角形與相連三角形的比率；
  • Diameter是網絡中最長的測地距離(兩個結點之間最短路徑的長度)；
  • Degree distribution是折點度的累積分佈；
  • Degree centralization是整個網絡的歸一化程度；
  • Closeness centralization是折點的歸一化逆。
  • Eigenvector centralization是圖矩陣的第一個特徵向量；
  • Betweenness centralization是經過頂點的測地線的歸一化數量；
  • Average path length是網絡中每對頂點之間最短距離的平均值。
names(netstat$network)

頂點中心性和威望的主要指標

這些測量有助於識別網絡中最重要的頂點以及連接到第三個頂點的兩個頂點的傾向。

networkStat在頂點級別返回的統計數據是：

  • Degree centrality
  • Closeness centrality度量從給定頂點訪問每個其他頂點需要多少步驟；
  • Eigenvector centrality度量良好連接到連接良好的頂點；
  • Betweenness centrality度量經紀業務或把關潛力。它(大約)是通過特定頂點的頂點之間的最短路徑數；
  • PageRank score近似表示任何消息將到達特定頂點的概率。這個算法是由Google創始人開發的，最初應用於網站鏈接；
  • Hub Score估計從頂點傳出的鏈接的值。它最初應用於網頁；
  • Authority Score是最初應用於Web的另一種中心性度量。當一個頂點被鏈接許多其他頂點的許多其他頂點鏈接時，它具有較高的權威性；
  • Vertex Ranking是作為中心性和聲望頂點度量的線性加權組合獲得的整體頂點排名。權重與主成分分析的第一個分量的載荷成比例。

要總結networkStat函數的主要結果，請使用泛型函數summary。它通過多個表格顯示有關網絡和頂點描述的主要信息。

summary接受一個額外的參數。k是表示每個表的行數的格式值。選擇k=10，您決定查看前10個頂點。

summary(netstat, k=10)

共詞分析：一個領域的概念結構

共詞分析的目的是使用書目集合中的詞共現來映射框架的概念結構。

可以通過諸如多維縮放(MDS)、對應分析(CA)或多重對應分析(MCA)的降維技術來執行分析。

在這裡，我們展示了一個示例，它使用函數conceptualStructure執行CA或MCA來繪製字段的概念結構，並使用K-Means聚類來識別表達共同概念的文檔聚類。結果繪製在二維地圖上。

conceptualStructure包括從標題和摘要中提取術語的自然語言處理(NLP)例程(請參閱函數termExtraction)。此外，它還實現了Porter的詞幹劃分算法，以將詞形變化(有時是派生的)詞減少到詞幹、詞基或詞根形式。

# Conceptual Structure using keywords (method="CA")CS <- conceptualStructure(M,field="ID", method="CA", minDegree=4, clust=5, stemming=FALSE, labelsize=10, documents=10)
