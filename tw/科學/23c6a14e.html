5個方面詳解：AI產品運營必知的軟硬件技術

2020-11-14
| 科學

比算法更難得是算法的思想，比編程工具更難的是編程的思維，比做產品更難的是產品的夢想。本文主要從5個方面，詳細闡述AI產品運營必知的軟硬件技術。

一、AI產品運營對基礎關係的安排

1. 智能軟硬件與軟件和硬件

在AI產品裡沒有純粹單獨的軟件和硬件，尤其是產品經理更應該系統來看，把軟件和硬件看成是AI賦能的智能軟硬件。

例如：單片機一般意義上被看做硬件，但是我們以一種單片機Arduino來看，Arduino板子上的微控制器可以通過Arduino的編程語言來編寫程序，編譯成二進制文件，燒錄進微控制器，而程序本身又是軟件部分。其他AI產品部件也類似像STM32，瑞芯微3288等等。

單片機只是舉例，現在這麼大的數據量單片機是處理不了的，AI也不只是跑在單片機上，X86，服務端也有。

AI裡軟件相當於人的大腦，硬件相當於人的身體！所以離開軟件硬件沒有靈魂，離開硬件軟件沒有肉身！當下AI硬件主要被用來採集數據和作為算力！

2. 算法和數據

有人說算法重要，因為它體現了技術水平的高低，駕馭數學知識的能力；有人說數據重要沒有數據算法如何實現效果，實際上LineLian和工程院院士楊善林的學生討論後發現，算法和數據是魚和水的關係。AI時代裡算法跟傳統算法的區別在於AI的算法是對人腦的模擬，是一種智能。

AI是在大量的樣本數據基礎上，通過神經網絡算法訓練數據，建立了輸入數據和輸出數據之間的映射關係，其最直接的應用是在分類識別方面。例如：訓練樣本的輸入是語音數據，訓練後的神經網絡實現的功能就是語音識別，如果訓練樣本輸入是人臉圖像數據，訓練後實現的功能就是人臉識別。

AI算法使得解決問題的步驟智能，數據使得算法得到訓練從而實現算法所構建的商業模式！

3. 服務端和應用端

服務端和應用端是相對的。

開發應程序調操作系統的API， 操作系統的API有（創建線程、 讀寫文件【讀、寫、偏移到指定地址】、 網絡通信、 圖形渲染），那麼操作系統就是應用程序的服務端。

而寫一個常規的小程序或者APP，前端用戶界面上需要的數據就是分別通過WEB程序調用瀏覽器功能接口然後OS向後臺服務端發請求傳數據。

另外web程序員，和底層嵌入式程序員理解的服務端和應用端還有差別，這裡產品經理明瞭這種關係即可。服務端即底層就是功能的實現者， 應用端上層就是功能的使用者，這一關係利於產品開發過程中需求時間安排規劃。

AI產品經理明瞭基礎關係，能更好的協調資源，補充Team短板，提升產品生命期效率！

二、站在硬件肩旁上賦能硬件

智能軟硬件是指通過將硬件和軟件相結合，對設備進行智能化創造或者改造。而智能軟硬件移動應用端則是軟件，通過應用連接智能硬件，操作簡單、開發簡便，各式應用層出不窮，也是企業獲取用戶的重要入口。例如：新零售的店鋪，智能販賣機等！

創造和改造對象可能是電子設備，例如：手錶、電視和其他電器；也可能是以前沒有電子化的設備，例如：門鎖、茶杯、汽車甚至房子。

智能軟硬件已經從可穿戴設備延伸到智能電視、智能家居、智能汽車、醫療健康、智能玩具、機器人等領域。比較典型的智能軟硬件包括Google Glass、三星Gear、FitBit、麥開水杯、咕咚手環、Tesla、無屏電視等。

2. 智能硬件的組成

（1）傳感器

傳感器是一種檢測裝置，能感受到被測量的信息，並能將感受到的信息，按一定規律變換成為電信號或其他所需形式的信息輸出，以滿足信息的傳輸、處理、存儲、顯示、記錄和控制等要求。

例如：檢測距離的聲波傳感器，尋找軌跡的紅外傳感器，通信的藍牙、NB-IoT傳感器等等！

（2）控制器

控制器是指按照預定順序改變主電路或控制電路的接線和改變電路中電阻值，來控制電動機的啟動、調速、制動和反向的主令裝置。

由程序計數器、指令寄存器、指令譯碼器、時序產生器和操作控制器組成，它是發佈命令的“決策機構”，即完成協調和指揮整個計算機系統的操作。例如：一般功能的叫MCU單片機，複合功能的叫操作系統OS！

首先，控制器在智能硬件中一般叫做芯片，AI與CPU比較在架構和功能特點上有著非常大的區別。

傳統的CPU運行的所有的軟件是由程序員編寫，完成的固化的功能操作，其計算過程主要體現在執行指令這個環節。但與傳統的計算模式不同，人工智能要模仿的是人腦的神經網絡，從最基本的單元上模擬了人類大腦的運行機制。它不需要人為的提取所需解決問題的特徵，或者總結規律來進行編程。

AI一般包含機器學習和深度學習，但不管是機器學習還是深度學習都需要構建算法和模式，以實現對數據樣本的反覆運算和訓練，降低對人工理解功能原理的要求。

因此，AI芯片需要具備高性能的並行計算能力，同時要能支持當前的各種人工神經網絡算法。傳統CPU由於計算能力弱，支撐深度學習的海量數據並行運算，且串行的內部結構設計架構為的是以軟件編程的方式實現設定的功能，並不適合應用於人工神經網絡算法的自主迭代運算。

傳統CPU架構往往需要數百甚至上千條指令才能完成一個神經元的處理，在AI芯片上可能只需要一條指令就能完成。

其次，解讀主流的人工智能芯片。

AI的高級階段是深度學習，而對於深度學習過程則可分為：訓練和推斷兩個環節。

訓練環節通常需要通過大量的數據輸入或採取增強學習等非監督學習方法，訓練出一個複雜的深度神經網絡模型。訓練過程由於涉及海量的訓練數據和複雜的深度神經網絡結構，需要的計算規模非常龐大，通常需要GPU集群訓練幾天甚至數週的時間，在訓練環節GPU目前暫時扮演著難以輕易替代的角色。

推斷環節指利用訓練好的模型，使用新的數據去“推斷”出各種結論，如視頻監控設備通過後臺的深度神經網絡模型，判斷一張抓拍到的人臉是否屬於黑名單。雖然推斷環節的計算量相比訓練環節少，但仍然涉及大量的矩陣運算。

在推斷環節，除了使用CPU或GPU進行運算外，FPGA以及ASIC均能發揮重大作用。目前，主流的人工智能芯片基本都是以GPU、FPGA、ASIC以及類腦芯片為主。

FPGA：

即專用集成電路，一種集成大量基本門電路及存儲器的芯片，可通過燒入FPGA配置文件來來定義這些門電路及存儲器間的連線，從而實現特定的功能。而且燒入的內容是可配置的，通過配置特定的文件可將FPGA轉變為不同的處理器，就如一塊可重複刷寫的白板一樣。

FPGA有低延遲的特點，非常適合在推斷環節支撐海量的用戶實時計算請求，如語音識別如LineLian2017年打造的裸眼3D試衣鏡所採用的芯片。

由於FPGA適合用於低延遲的流式計算密集型任務處理，意味著FPGA芯片做面向與海量用戶高併發的雲端推斷，相比GPU具備更低計算延遲的優勢，能夠提供更佳的消費者體驗。在這個領域，主流的廠商包括Intel、亞馬遜、百度、微軟和阿里雲。

ASIC：

即專用集成電路，不可配置的高度定製專用芯片。特點是需要大量的研發投入，如果不能保證出貨量其單顆成本難以下降，而且芯片的功能一旦流片後則無更改餘地，若市場深度學習方向一旦改變，ASIC前期投入將無法回收，意味著ASIC具有較大的市場風險。但ASIC作為專用芯片性能高於FPGA，如能實現高出貨量，其單顆成本可做到遠低於FPGA。

谷歌推出的TPU就是一款針對深度學習加速的ASIC芯片，而且TPU被安裝到AlphaGo系統中。但谷歌推出的第一代TPU僅能用於推斷，不可用於訓練模型，但隨著TPU2.0的發佈，新一代TPU除了可以支持推斷以外，還能高效支持訓練環節的深度網絡加速。

根據谷歌披露的測試數據，谷歌在自身的深度學習翻譯模型的實踐中，如果在32塊頂級GPU上並行訓練，需要一整天的訓練時間，而在TPU2.0上，八分之一個TPUPod(TPU集群，每64個TPU組成一個Pod)就能在6個小時內完成同樣的訓練任務。

GPU：

即圖形處理器。最初是用在個人電腦、工作站、遊戲機和一些移動設備上運行繪圖運算工作的微處理器，可以快速地處理圖像上的每一個像素點。後來科學家發現，其海量數據並行運算的能力與深度學習需求不謀而合，因此，被最先引入深度學習。

GPU之所以會被選為超算的硬件，是因為目前要求最高的計算問題正好非常適合並行執行。一個主要的例子就是深度學習，這是人工智能（AI）最先進的領域。

深度學習以神經網絡為基礎，神經網絡是巨大的網狀結構，其中的節點連接非常複雜。訓練一個神經網絡學習，很像我們大腦在學習時，建立和增強神經元之間的聯繫。

從計算的角度說，這個學習過程可以是並行的，因此它可以用GPU硬件來加速。這種機器學習需要的例子數量很多，同樣也可以用並行計算來加速，在GPU上進行的神經網絡訓練能比CPU系統快許多倍。

目前，全球70%的GPU芯片市場都被NVIDIA佔據，包括谷歌、微軟、亞馬遜等巨頭也通過購買NVIDIA的GPU產品擴大自己數據中心的AI計算能力。

三、站在軟件肩旁上賦能軟件

上文第二節裡講站在硬件的肩旁上賦能硬件裡，硬件本身亦不是純硬件，其中有軟件部分且是必不可少的部分。

軟件包含系統軟件和應用軟件，系統軟件是軟件的軟件，是軟件運行的平臺，應用軟件是為了完成特定產品業務運營而編寫的軟件。

常用的系統軟件有操作系統（DOS、WINDOWS、UNIX、OS/2）、數據庫管理系統（FoxPro、DB-2、Access、SQL-server）、編譯軟件（VB、C++、JAVA）。應用軟件有文字處理軟件（WORD、WPS）、信息管理軟件、輔助設計軟件（CAD）、實時控制軟件(CAM)。再例如：APP、小程序等屬於應用軟件，Android/IOS等屬於系統軟件。

其次芯片選型。

選用M451LG6AE作為主控芯片M451LG6AE為 32位單片機，基於M4內核，具有足夠的RAM和 Flash能夠滿足運行程序的要求。其具有多個串口，支持硬件的 SPI 及 I2C，能夠方便實現的通信，具有 AD能夠實現一些狀態量的測量。

使用 DTH11進行溫溼度的測量，其集成了溫度及溼度檢測功能,接口為單線制串口。節約 IO，使用 DS18B20進行範圍較大的溫度測量，其測溫範圍較廣，具有密封的封裝，能夠測量液體溫度，能多個並聯。對於土壤溼度的測量，使用電阻式溼度傳感器進行測量。

限位檢測主要用於檢測物體是否移動到了指定位置，或者在餵食裝置中電機旋轉的圈數（決定餵食的量），只需要輸出開關量，因此使用光電門進行檢測，使用非接觸測量能夠有效地延長傳感器的使用時間，同時能夠提供足夠的精度。

為了實現高精度的計時，從而減小離線時出現的定時誤差，使用外置的RTC DS3231 作為計時基準。DS3231 內置了帶溫補的晶振,能夠實現較高精度的計時，滿足運用的需求。

選擇擦寫次數多和讀取速度塊的FRAM FM24V系列，來進行設置數據的存儲%使掉電覆位後，仍然能夠進行中斷的任務。

使用 HLK-PM01 作為 AC/DC芯片，HLK-PM01 用於實現220VAC到 5VDC的轉換，能夠提供 3W的功率,足夠系統的使用。使用 HLK_M30模塊作為WIFI 模塊，其體積很小，功耗較低 ，支持TCP Server/TCP Client /UDP Server/UDP Client工作方式，能夠方便地實現互聯網連接，串行接口能夠方便地與主控器連接。

整體軟件功能模塊如下圖：

其中1：軟件部分與硬件之間交互通信模塊， APP通過HTTP 協議與服務器進行通信 ，採用 OkHttp 進行網絡請求，請求的方式以 Post為主。

Android系統提供了兩種 HTTP 交互的方式HTTPURL-CONNECTION和 APACHE HTTP CLIENT，雖然兩者都支持 HTTPS流的上傳和下載，配置超時IPV6和連接池，已足夠滿足各種HTTP請求的需求，但原生的HTTP 接口存在著 API 數量過多，擴展困難等不足。

因此產生了許多出色的網絡通信庫來替代原生的 HTTPURLConnection，其中OkHttp是最出色的一個。

使用 OkHttp 庫進行更高效 HTTP請求可以讓應用運行更快、更節省流量，因此，本項目中使用 OkHttp庫依賴開發。

APP與 服務器通信時採用 JSON格式進行數據傳輸 ，JSON 作為一種輕量級的數據交換格式，易於閱讀和編寫，並且多種編程語言均提供了對 JSON 格式數據解析的支持，易於機器解析和生成。

並且相比XML格式，JSON數據量更小，具有較高的有效數據率，有助於節省流量，提升網絡傳輸速率。APP採用 Android 自帶的 JSONObject 類進行 JSON數據的解析。

其中2：登錄模塊，由於智能家居控制系統是一個面向多種需求多個用戶的控制系統，需要建立用戶系統，使不同的用戶可以有序地管理自己的設備。

APP在啟動時會首先檢查本地是否存有用戶的 token，如果不存在，跳轉到登錄界面，密碼採用 MD5進行加密。用戶登錄成功後， 將服務器發來的 token通過 SharedPreferences 保存在本地，以後每次網絡請求通過這個token進行用戶驗證，而不需要重新進行登錄。

其中3：設備模塊設備列表中選擇一個設備進入設備控制頁面，在設備控制頁面可以進行查看設備信息，查看歷史記錄，對設備進行控制等操作。

進行設備控制時，智能照明控制開、關兩種狀態，智能澆水對澆水量進行控制，智能餵食對餵食量進行控制，每種設備的控制均提供手動和定時兩種工作方式。 在啟動定時任務時，可以設置每週某幾天在某個時間重複執行設備控制的任務。

其中4：場景頁面中可以添加的任務分為兩種類型：

  • 一種是對設備的控制任務，包含控制照明設備的開關，澆水設備的澆水量、餵食設備的餵食量等。
  • 另一種是延時任務，用於設定不同設備控制任務之間的延時間隔，場景可以設置為手動或定時啟動兩種方式。為了減少控制的複雜性，場景中的單項任務不能進行定時的設置，但可以通過設置場景整體的定時任務來實現場景中每個任務的定時執行。
