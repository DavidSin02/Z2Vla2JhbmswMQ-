如何評估聚類模型？蘭德指數、輪廓係數、Calinski Harabaz指數

2020-10-29
| 科學

我們可以通過對一系列曲目進行聚類來創建歌曲的自動播放列表，我們可以展示如何自動創建相似歌曲的子組。通過我們現有的歌曲知識，我們能夠驗證該聚類練習的結果。

但是，如果我們對數據沒有這種先驗知識怎麼辦？如果數據甚至都沒有被標記怎麼辦（在許多實際的聚類案例中就是這種情況）？即使是這樣，如果這些標籤最初對我們沒有意義，該怎麼辦？有很多我從未聽說過的藝術家，如果我們要對數千首曲目進行分組，那麼手動驗證每個集群顯然是不切實際的。在這些情況下，我們需要某種數學方法來衡量聚類的"成功"程度。

為了探索如何做到這一點，我們再次轉向Spotify的API。假設我們從四個完全不同的播放列表中選取歌曲：

  • Rap UK
  • Smooth Jazz
  • Classical Essentials
  • Essential K-Pop

如果我們將它們合併到一個數據集中，則無監督的機器學習算法應該能夠將其歌曲分組為四個群集，這些群集在某種程度上類似於原始的四個播放列表。

在開始之前，我們確實可能需要檢查一下我們的假設，即來自這些不同播放列表的歌曲確實"與眾不同"。當然，在單個圖表上同時可視化兩個以上的特徵是一項挑戰。但是，如果我們繪製所有功能的散佈矩陣，並按播放列表進行顏色編碼，則可以看到有很多度量組合可以證明每種流派的獨特特徵。



我們在上中看到，在Scikit-Learn中運行聚類算法非常簡單：

#We scale the data to ensure that#feature units don't impact distancesfrom sklearn.preprocessing import StandardScalersclaer = StandardScaler()X_scaled = scaler.fit_transform(X)#This scaled data can then be fed into the HAC algorithmfrom sklearn.cluster import AgglomerativeClustering#We can tell it how many clusters we're aiming foragg_clust = AgglomerativeClustering(n_clusters=4)assigned_clusters = agg_clust.fit_predict(X_scaled)

我們還看到，HAC具有三個不同的"鏈接標準"，即該算法將群集逐漸連接在一起的方法：

  • ward（默認）：選擇兩個聚類，以所有聚類中的方差增加最小的方式合併。通常，這會導致大小相當相等的群集。
  • complete（或最大鏈接）：合併兩個點之間最大距離最小的兩個聚類。
  • ·average：合併所有點之間平均距離最小的兩個聚類。

讓我們看看這三個鏈接標準如何處理類型播放列表數據集。我們用矩陣表示結果，顯示每個播放列表的歌曲在每個群集中所佔的百分比（群集被任意命名為" A"，" B"，" C"和" D"）。

當然，如果聚類是完美的，我們希望矩陣的每一行和每一列都恰好包含一個100％的條目（當然，它不需要成對角線，因為聚類名稱的分配是任意的）。

默認的"向後"鏈接試圖最小化群集中的方差，儘管這四個群集都有一些洩漏，但在所有四種類型中都表現出色。

"完全"鏈接顯然效果不佳。它已將許多數據集放入群集A。群集C由一首說唱歌曲組成。

"平均"鏈接與"完全"鏈接具有類似的問題。許多數據點已放置在一個群集中，其中兩個群集由一首歌曲組成。

值得一提的是，還有另一種常見的聚類類型，即K-Means，其工作原理略有不同。HAC通過將它們合併在一起來迭代地減少聚類的數量，而K-Means聚類保持固定數量的聚類（名義上的k），但是迭代地更改每個聚類的成員。



每個群集的"中心"由帶有黑色" +"號的"大"標記表示。所有點都分配給它們最接近中心的聚類。在此分配步驟之後，將重新計算群集中心以包括新成員，然後發生另一個重新分配步驟。如果在重新分配步驟中沒有點更改聚類，則算法結束（動畫：）。

K-Means很容易在Python中實現：

#We scale the data to ensure that#feature units don't impact distancesfrom sklearn.preprocessing import StandardScalersclaer = StandardScaler()X_scaled = scaler.fit_transform(X)#This scaled data can then be fed into the K-Means alorithmfrom sklearn.cluster import KMeans#We can tell it how many clusters we're aiming forkm_clust = KMeans(n_clusters=4)assigned_clusters = km_clust.fit_predict(X_scaled)

儘管這些矩陣可以很好地"盯住"我們的結果，但它們在數學上還不夠嚴格。讓我們考慮一些指標，這些指標實際上可以幫助我們為集群質量分配一個數字。



調整後的Rand Index(蘭德指數)

該是對經典蘭德指數的變化，並試圖表達什麼群集分配比例是"正確"。通過考慮所有樣本對，並根據預測的真實聚類對分配在預測的相同或不同聚類中的對進行計數，並針對隨機機會進行調整，計算出兩個不同聚類之間的相似性度量。

可以使用Scikit-Learn評估此（以及我們將考慮的其他指標）。

from sklearn import metricsmetrics.adjusted_rand_score(predicted_labels, actual)

調整後的蘭德指數限制在-1和1之間。接近1表示好，而接近-1表示差。

我們看到K均值和Ward Linkage得分很高。基於我們先前觀察到的矩陣，我們期望得到這一結果。

Fowlkes Mallows Score

該是相似的，在儘可能多的，它會告訴你哪個集群分配是"正確"的程度。特別是，它計算精度和召回率之間的幾何平均值。它的範圍是0到1，值越高越好。

metrics.fowlkes_mallows_score（predicted_labels，actual）

我們的排名與調整後的蘭德指數相似，這是我們期望的，因為它們是嘗試回答同一問題的兩種方法。

值得注意的是，我們需要了解原始標籤才能計算這些指標。鑑於處理無標籤數據是無監督學習的主要用例之一，因此我們需要一些其他指標來評估聚類結果，而無需引用"真實"標籤。

輪廓分數

該試圖描述一個數據點的相似程度在其集群的其他數據點，相對於數據點並不在其集群（這是匯聚了所有數據點來獲得總體集群分數）。換句話說，它考慮了集群在空間中的"區別度"-實際上人們可以使用任何"距離"度量來計算分數。

它的範圍是-1和1。接近-1表示群集不正確，而接近+1則表示每個群集都非常密集。

metrics.silhouette_score（scaled_feature_data，cluster_labels）

我們看到，所有聚類都沒有超高的輪廓分數。有趣的是，我們看到平均鏈接集群的得分最高。但是請記住，該算法產生了兩個群集，每個群集僅包含一個數據點，這在現實情況下不太可能是理想的結果（這是一個教訓，您通常不能依靠單個指標來做出決策算法的質量！）

Calinski Harabaz指數

所述Calinski Harabaz指數是相對於點的數據點的方差相比，在其它簇的點的比率，對所述方差中的集群。由於我們希望第一部分較高，而第二部分較低，因此需要較高的CH指數。與我們看到的其他指標不同，此分數沒有界限。

metrics.calinski_harabasz_score（scaled_feature_data，cluster_labels）

在這裡，我們看到我們的K均值和Ward Linkage算法得分很高。完全和平均鏈接算法因具有一個或兩個較大的聚類而受到懲罰，這將具有較高水平的內部方差。
