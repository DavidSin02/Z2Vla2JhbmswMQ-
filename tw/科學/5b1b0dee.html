JTAG接口學習筆記

接口名稱：JTAG

英文全稱：Joint Test Action Group

中文：聯合測試工作組

標準號： IEEE1149.1-1990/IEEE1149.1a-1993

標準維護：IEEE

當前規範：www.IEEE.org

最新版本：IEEE1149.1-2011

生效時間：

下一版本：

說明：是一種國際標準測試協議（IEEE 1149.1兼容），主要用於芯片內部測試。標準的JTAG接口是4線：TMS、TCK、TDI、TDO，分別為模式選擇、時鐘、數據輸入和數據輸出線。是一種芯片測試技術接口。

JTAG是IEEE下屬的一個工作組，該組織研究標準測試訪問接口和邊界掃描結構Test Access Port and Boundary-Scan Architecture。JTAG的前身是JETAG(Joint European Test Action Group歐洲聯合測試行動小組) 。

一、接口結構

支持JTAG接口的芯片，其內部結構如下圖所示。TAP（Test Access Port)TAP是一個通用的端口，通過TAP可以訪問芯片提供的所有數據寄存器（DR）和指令寄存器（IR）。當芯片內部存在多條BSC時，需要有相應的機制來控制訪問。對整個TAP的控制是通過TAP Controller來完成的。

TMS測試模式選擇（Test Mode Select），通過TMS信號控制JTAG狀態機的狀態；

TCK為JTAG的時鐘信號，TDI為數據輸入信號，TDO為數據輸出信號，nTRST為JTAG復位信號，復位JTAG的狀態機和內部的宏單元（Macro cell）。



JTAG引腳定義

TCK——測試時鐘輸入；

TDI——測試數據輸入，數據通過TDI輸入JTAG口；

TDO——測試數據輸出，數據通過TDO從JTAG口輸出；

TMS——測試模式選擇，TMS用來設置JTAG口處於某種特定的測試模式。

可選引腳TRST——測試復位，輸入引腳，低電平有效。

二、工作原理

JTAG的基本原理是在器件內部定義一個TAP（Test Access Port測試訪問口），通

過專用的JTAG測試工具對內部節點進行測試。

² 方便芯片的故障定位,迅速準確地測試兩個芯片管腳的連接是否可靠,提高測試檢驗效率。

² 具有J TAG接口的芯片,內置一些預先定義好的功能模式,通過邊界掃描通道來使芯片處於某個特定的功能模式,以提高系統控制的靈活性和方便系統設計。

JTAG測試允許多個器件通過JTAG接口串聯在一起，形成一個JTAG鏈，能實現對各個器件分別測試。如今，JTAG接口還常用於實現ISP（In-System Programmer，在系統編程），對FLASH等器件進行編程。

JTAG編程方式是在線編程，傳統生產流程中先對芯片進行預編程然後再裝到板上，簡化的流程為先固定器件到電路板上，再用JTAG編程，從而大大加快工程進度。JTAG接口可對DSP芯片內部的所有部件進行編程。

JTAG內部有一個狀態機，稱為TAP控制器。TAP控制器的狀態機通過TCK和TMS進行狀態的改變，實現數據和指令的輸入。

邊界掃描

邊界掃描技術的基本思想是在靠近芯片的輸入輸出管腳上增加一個移位寄存器單元。因為這些移位寄存器單元都分佈在芯片的邊界上（周圍），所以被稱為邊界掃描寄存器（Boundary-Scan Register Cell）.邊界掃描測試是通過邊界掃描單元以及一些附加的測試控制邏輯實現的。 當芯片處於調試狀態的時候，這些邊界掃描寄存器可以將芯片和外圍的輸入輸出隔離開來。通過這些邊界掃描寄存器單元，可以實現對芯片輸入輸出信號的觀察和控制。

在正常的運行狀態下，這些邊界掃描寄存器對芯片來說是透明的，所以正常的運行不會受到任何影響。

JTAG標準定義了一個串行的移位寄存器。寄存器的每一個單元分配給IC芯片的相應引腳，每一個獨立的單元稱為BSC（Boundary-Scan Cell）邊界掃描單元。這個串聯的BSC在IC內部構成JTAG迴路，所有的BSR（Boundary-Scan Register）邊界掃描寄存器通過JTAG測試激活，平時這些引腳保持正常的IC功能。

每個I/ O 管腳都有一個BSC ,每個BSC 有兩個數據通道:一個是測試數據通道,測試數據輸入TDI ( test data input ) 、測試數據輸出TDO (test data output) ;另一個是正常數據通道,正常數據輸入NDI ( normal data input ) 、正常數據出NDO　(normal data output) 。如圖1 所示。

在正常工作狀態,輸入和輸出數據可以自由通過每個BSC ,正常工作數據從NDI 進,從NDO 出。在測試狀態,可以選擇數據流動的通道:對於輸入的IC管腳,可以選擇從NDI 或從TDI 輸入數據;對於輸出的IC 管腳,可以選擇從BSC 輸出數據至NDO ,也可以選擇從BSC輸出數據至TDO。

另外，芯片輸入輸出管腳上的邊界掃描寄存器單元可以相互連接起來，在芯片的周圍形成一個邊界掃描鏈（Boundary-Scan Chain）。一般的芯片都會提供幾條獨立的邊界掃描鏈，用來實現完整的測試功能。邊界掃描鏈可以串行的輸入和輸出，通過相應的時鐘信號和控制信號，就可以方便的觀察和控制處在調試狀態下的芯片。

圖2 所示是邊界掃描測試應用的示意圖。為了測試兩個JTAG 設備的連接,首先將JTAG 設備1某個輸出測試腳的BSC 置為高或低電平,輸出至NDO ,然後,讓JTAG設備2 的輸入測試腳來捕獲（capture)從管腳輸入的NDI 值,再通過測試數據通道將捕獲到的數據輸出至TDO ,對比測試結果,即可快速準確的判斷這兩腳是否連接可靠。

JTAG控制器

JTAG控制器主要由三個部分組成:

l 測試端口( TAP ,test access port) 控制器

l 指令寄存器(包括指令譯碼器)

l 數據寄存器

TAP 控制器是邊界掃描測試核心控制器。TAP 有以下5個控制信號:

n TCK: TCK 為TAP 的操作提供了一個獨立的、基本的時鐘信號，TAP 的所有操作都是通過這個時鐘信號來驅動的。TCK 在IEEE 1149.1 標準裡是強制要求的。

n TMS : TMS 信號用來控制TAP 狀態機的轉換。通過TMS 信號，可以控制TAP 在不同的狀態間相互轉換。TMS 信號在TCK 的上升沿有效。TMS 在IEEE 1149.1 標準裡是強制要求的 。

n TDI : TDI 是數據輸入的接口。所有要輸入到特定寄存器的數據都是通過TDI 接口一位一位串行輸入的（由TCK 驅動）。TDI 在IEEE 1149.1 標準裡是強制要求的。

n TDO 是數據輸出的接口。所有要從特定的寄存器中輸出的數據都是通過TDO 接口一位一位串行輸出的（由TCK 驅動）。TDO 在IEEE 1149.1 標準裡是強制要求的。

n TRST: TRST可以用來對TAP Controller 進行復位（初始化）。不過這個信號接口在IEEE 1149.1標準裡是可選的，並不是強制要求的。因為通過TMS 也可以對TAP 進行復位（初始化）。

TAP控制器內部有多個寄存器：JTAG控制指令寄存器、測試數據寄存器、旁路寄存器(BYPASS)、器件識別碼(IDCODE)寄存器和掃描路徑選擇寄存器等。

指令寄存器：若執行數據寄存器邊界掃描測試,則指令寄存器負責提供地址和控制信號去選擇某個特定的數據寄存器;也可以通過指令寄存器執行邊界掃描測試,這時,TAP 輸出的SEL ECT 信號選擇指令寄存器的輸出去驅動TDO。

數據寄存器：邊界掃描鏈屬於數據寄存器中很重要一種。IEEE1149.1規定必須具有的兩個數據寄存器是邊界掃描寄存器(boundary scan register) 和旁通(bypass) 寄存器。其它的寄存器是任選的。由指令寄存器選擇某個特定的數據寄存器作為邊界掃描測試寄存器,當一個掃描路徑選定後,其它的路徑處於高阻態。邊界掃描寄存器是由圍繞IC 管腳的一系列的BSC 組成的,正是由它來實現測試管腳信號的輸入、輸出;旁通寄存器只由一個掃描寄存器位組成,當選擇了旁通寄存器, TDI 和TDO 之間只有一位寄存器,實際上沒有執行邊界掃描測試。旁通寄存器的作用是為了縮短掃描路徑而對不需要進行測試的IC 進行旁通。

如上所述,TAP 控制器是邊界掃描測試在TCK和TMS 的控制下,可以選擇使用指令寄存器掃描或數據寄存器掃描,以及控制邊界掃描測試的各個狀態。TMS 和TDI在TCK 上升沿被採樣, TDO 在TCK 下降沿變化。

TAP控制狀態機

TAP 控制器的狀態機共有16種狀態，但只有6 個穩定狀態:測試邏輯復位( test-logic-reset ) 、測試/ 等待(run-test/idle) 、數據寄存器移位( shift-DR) 、數據寄存器移位暫停(pause-DR) 、指令寄存器移位( shift-IR) 、指令寄存器移位暫停(pause-IR) 。其它狀態都不是穩態,而只是暫態.

在上電或IC 的正常運行時,必須使TMS 在最少持續5個TCK 保持為高電平,則TAP 進入測試邏輯復位態。這時,TAP 發出復位信號使所有的測試邏輯不影響元件的正常運行。若需要進行邊界掃描測試,可以在TMS 與TCK 的配合控制下,退出復位,進入邊界掃描測試需要的各個狀態。

指令寄存器訪問流程：

u 系統上電，TAP Controller進入Test-Logic Reset狀態。

u Run-Test/Idle->Select-DR-Scan->Select-IR-Scan->Capture-IR->Shift-IR->Exit1-IR ->Update-IR，最後回到Run-Test/Idle狀態。

u 在Capture-IR狀態，特定的邏輯序列被加載到指令寄存器中；然後進入到Shift-IR狀態。

數據寄存器訪問流程：

u 當前可以訪問的數據寄存器由指令寄存器中的當前指令決定。

u 以Run-Test/Idle為起點，依次進入Select-DR-Scan->Capture-DR->Shift-DR->Exit1-DR->Update-DR，最後回到Run-Test/Idle狀態。

n JTAG掃描鏈工作原理：

每一個掃描單元的內部由兩個D觸發器和兩個多路選擇器組成。


在正常狀態下即掛起，BSR對芯片是透明的，不影響芯片的正常運行。

當芯片處於調試狀態時，BSR將芯片與外圍隔離。通過BSR可實現對芯片管腳的觀察與控制。

對於輸入管腳，可通過對BSR的操作，將信號（數據）加載至管腳。

對於輸出管腳，可通過BSR將管腳上的輸出信號“捕獲”（Capture)。

BSR可以相互連接起來，在芯片周圍形成一個邊界掃描鏈。（Boundary-Scan Chain）

一般芯片會提供多條獨立的BSC，用於實現完整的測試功能。BSC可以串行的輸入輸出，再配合時鐘及控制信號，可觀察與控制調試狀態下的芯片。JTAG掃描鏈一共有四種操作：掛起、捕獲、移位和更新。

BSDL：Boundary Scan Description Language邊界掃描描述語言
