3分鐘閱讀 | 最難用的 Bash，最常用的輸入輸出重定向

2020-10-29
| 科學

引言

將命令的輸出重定向到文件，或將其通過管道傳遞到另一個命令時，你可能會注意到錯誤消息會被打印在屏幕上。

在Bash和其他Linux Shell中，執行程序時，它使用三個標準 I/O 流。每個流都由一個數字文件描述符表示：

  • 0- stdin，標準輸入流。
  • 1- stdout，標準輸出流。
  • 2- stderr，標準錯誤流。

文件描述符只是代表打開文件的數字。

輸入流通常通過鍵盤輸入。程序輸出進入標準輸入流，錯誤消息進入標準錯誤流。默認情況下，輸入流和錯誤流都打印在屏幕上。

重定向輸出

重定向是一種捕獲程序輸出並將其作為輸入發送到另一個程序或文件的方法。

可以使用n>運算符重定向流，其中n為文件描述符號。

如果n省略，則默認為1標準輸出流。例如，以下兩個命令是相同的；兩者都會將命令輸出（stdout）重定向到文件。

command > filecommand 1> file

要重定向標準錯誤（stderr），請使用2>運算符：

command 2> file

你可以同時寫入stderr和寫入stdout兩個單獨的文件：

command 2> error.txt 1> output.txt

要阻止錯誤消息顯示在屏幕上，請重定向stderr至/dev/null：

command 2> /dev/null

重定向stderr到stdout

將程序的輸出保存到文件中時，重定向stderr到 stdout很常見，因此你可以將所有內容都保存在一個文件中。

要將重定向stderr到stdout標準輸出並將錯誤消息發送到同一文件，請使用以下命令：

command > file 2>&1

> file將重定向stdout到file，並將2>&1重定向stderr到的當前位置stdout。

重定向的順序很重要。例如，以下示例僅重定向stdout到file。發生這種情況是因為將stderr重定向到stdout之前，stdout已經重定向到file了。

command 2>&1 > file

重定向stderr到的另一種方法stdout是使用&>構造。在Bash中&>具有以下含義2>&1：

command &> file



寫在最後

在命令行上工作時，瞭解重定向和文件描述符的概念非常重要。

要重定向stderr和stdout，請使用2>&1或&>構造。

Happy coding :_)
