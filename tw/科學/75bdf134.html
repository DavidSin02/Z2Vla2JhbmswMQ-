OSI分層模型與TCP/IP協議棧

2020-11-14
| 科學

標準化組織ISO

ISO:國際標準化組織 ( International Organization for Standardization )

OSI:開放系統互聯 ( open system interconnection )

20世紀70年代後期，ISO創建OSI參考模型，希望不同供應商的網絡能夠相互協同工作，但迄今為止，這仍然是一個偉大的目標

網絡分層的優點

層 layer：描述了所有需求有效的通訊過程，並把這些過程邏輯上的組叫做層。

分層的優點：

1.促進標準化工作,允許各個供應商進行開發.

2.各層間相互獨立,把網絡操作分成低複雜性單元.

3.靈活性好,某一層變化不會影響到別層，設計者可專心設計和開發模塊功能.

4.各層間通過一個接口在相鄰層上下通信

TCP/IP協議棧道和OSI的區別

TCP/IP協議簇

本質上是一樣的，主要區別：

1、層數不同

2、TCP/IP在網絡層僅支持ip協議 OSI支持多種網絡層協議

3、TCP/IP支持跨層封裝

各層之間的作用及對應的協議

PDU

PDU（protocol data unit):每一層使用自己層的協議和別的系統對應層相互通信，協議層的協議在對等層之間交換的信息叫協議數據單元。

上層 : message

transport layer : segment

Network layer: packet

Data-link layer: Frame

Physical layer: bit

封裝與解封裝

封裝 （ encapsulate/encapsulation）：數據要通過網絡進行傳輸，要從高層一層一層的向下傳送，如果一個主機要傳送數據到別的主機，先把數據裝到一個特殊協議報頭中，這個過程叫-----封裝

封裝分為：切片和加控制信息

解封裝：上述的逆向過程
