coord簡介

coord是指圖表中的座標系，通過對這個屬性的配置可以改變座標系類型及展現方式。

目前G2中的座標系可以分為直角座標系和極座標系兩大類，具體分類見下表：

Coord配置

之前也提到過，像scale一樣，coord也有屬性式和函數式兩種配置方式：

  • 屬性式

coord:{ type: {string}, // 座標系類型聲明，默認為rect，可取值: rect polar theta map helix gauge clock cfg: {object}, // 對應各個 type 座標系屬性配置，同 `chart.coord(type, cfg)` 中的 cfg actions: {array} // 聲明座標系需要進行的變換操作,可取值:rotate scale reflect transpose }

從以上代碼中可以看出，coord一共包含3個可配置屬性

其中重點講解actions參數，其支持的變換操作如下：

  • rotate(angle): 座標系旋轉，angle 表示旋轉的度數，單位為角度。
  • scale(sx, sy): 座標系縮放，sx 代表 x 方向縮放比例，sy 代表 y 方向縮放比例，單位為數值。
  • reflect('' | 'x' | 'y'): 座標系轉置，將 x 或者 y 的起始、結束值倒置。
  • transpose(): 將座標系 x 軸和 y 軸轉置。

actions屬性的配置方式如下：

actions: [ [ 'transpose' ], [ 'rotate', 90 ], [ 'scale', 1, -1 ], [ 'reflect', 'x' ]]

各種變換的示例將在下面“直角座標系配置”部分中進行講解。

  • 函數式
chart.coord(type:{string},cfg:{object}).transpose().rotate(angle);

這兩種配置中的type和cfg屬性配置是一一對應的，但是需要注意的是：屬性設置中的actions配置可以通過鏈式設置的方式來調用。

說完coord的類型和配置形式，下面說一下每種coord具體類型的配置，假設基初代碼如下:

html代碼：

<div id="root"></div>

JS代碼：

 const data=[ {name:"apple",value:2,time:"1996-05-01"}, {name:"orange",value:3,time:"1996-05-02"}, {name:"banana",value:4,time:"1996-05-03"}, ]; const chart=new G2.Chart({ container:"root", width:400, height:400, options:{ scales:{ name:{ type: "cat" } }， coord:{ type:"rect", cfg:{}, actions:[] } } }); console.log(chart); chart.source(data); chart.interval().position("name*value").color("name",["red","black","pink","blue"]); chart.render();

rect配置

注意：rect類型中時一般不需要配置cfg屬性，actions可配置。

  • transpose變換coord配置如下：
coord：{ type:"rect", actions:[ ["transpose"] ]}

tranapose變換(將x軸與y軸轉置)

  • rotate變換coord配置如下：

注意：默認以座標系中心沿順時針旋轉

coord：{ type:"rect", actions:[ ["rotate"，90] //繞座標系中心繞順時針旋轉90度 ]}

rotate(旋轉變換)

  • reflect變換coord配置如下：
coord：{ type:"rect", actions:[ ["reflect"，"x"]， //將x軸方向換成相反的方向 //["reflect"，"y"] //將x軸y軸方向都換成相反的方向 ]}

coord：{ type:"rect", actions:[ ["scale",0.5,1]， //將x軸縮小0.5倍，y軸不變 ]}

極座標系配置

極座標均可以進行如下屬性配置：

chart.coord('polar' || 'theta' || 'helix', { startAngle: 弧度, // 起始弧度,用Math.PI表示，如：-Math.PI/2 endAngle: 弧度, // 結束弧度，用Math.PI表示，如：Math.PI/2 innerRadius: 0 至 1 範圍的數值, // 用於空心部分的半徑設置 radius: 0 至 1 範圍的數值 // 實心圓的半徑大小設置});

這裡需要說明的是，G2 極座標默認的startAngle起始角度和endAngle結束角度如下圖所示：

polar配置

因為polar是通過 半徑來反映數據 的，所以常 與interval形狀配合 繪製玫瑰圖。

示例：

coord:{ type:"polar", cfg:{ startAngle:-Math.PI/2, //起始角度為-π/2 endAngle:Math.PI, //結束角度為π/2，也就是說可用於畫圖的角度為-π/2~π innerRadius:0.5， //空心部分是0.5 radius:0.5 //實心部分是0.5，因此兩者半徑一樣大 } }, actions:[] //actions配置參照直角座標系 }
chart.interval().position("name*value").color("name",["red","black","pink","blue"]);

polar玫瑰圖

theta配置

因為theta是通過 角度大小來反映數據 的，所以常 與intervalStack層疊形狀配合 繪製餅圖或者環圖。

示例：

coord:{ type:"theta", cfg:{ startAngle:-Math.PI/2, //起始角度為-π/2 endAngle:Math.PI/2, //結束角度為π/2，也就是說可用於畫圖的角度為-π/2~π innerRadius:0.5， //空心部分是0.5 radius:0.5 //實心部分是0.5，因此兩者半徑一樣大 } }, actions:[] //actions配置參照直角座標系 }
chart.intervalStack().position("value").color("name",["red","black","pink","blue"]);

注意：因為intervalStack是層疊圖（即一個座標軸的點是定的，只有一個變量），因此position只能接受一個變量作為可變參數。

theta環圖

helix

螺旋座標系，基於阿基米德螺旋線，常用於週期性數據的表示，是一種不常用的一種座標系。

示例：

coord:{ type:"helix", cfg:{ startAngle:-Math.PI/2, //起始角度為-π/2 endAngle:5*Math.PI, //結束角度為π/2，也就是說可用於畫圖的角度為-π/2~5π innerRadius:0.5， //空心部分是0.5 radius:0.5 //實心部分是0.5，因此兩者半徑一樣大 } }, actions:[] //actions配置參照直角座標系 }
