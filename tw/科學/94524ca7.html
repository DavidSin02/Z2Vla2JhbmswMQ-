網絡菜鳥筆記七 -第2層交換

2020-11-14
| 科學

小編今天和大家一起討論一下第2層交換：

交換式服務：根網橋使用軟件來創建和管理過濾表不一樣，交換機使用專用繼承電路來創建並維護其過濾表。

第二層交換提供的性能：

1． 基於硬件的橋接 2． 線速率 3． 低延遲 4． 低成本

第二層交換的侷限性

要用網橋設計出一個好的網絡，就必須考慮到下面兩個最重要的方面：

1． 必須絕對正確的分隔出衝突域。

2． 創建一個功能強大的僑界網絡的正確方法是，確信其用戶在本地網段上花費了80%的時間。

橋接的網絡能夠分隔衝突域，但是要記住，網絡仍然是一個大的廣播域。默認時，第二層交換機和網橋都不能分隔廣播域-這不僅限制了網絡的規模和增長潛力，還降低了他的整體性能。

橋接與LAN交換的比較

他們之間的區別：

1． 網橋是基於軟件的，而交換機是基於硬件的，因為交換機使用ASIC芯片來幫助做出過濾得決定。

2． 交換機可以看成是多端口的網橋

3． 每個網橋只有一個省成熟實例，而交換機可以有許多生成樹實例。

4． 交換機的端口數量比大多數的網橋都多。

5． 網橋和交換機都轉發第二層廣播。

6． 通過檢查所接收的每個數據幀的源地址，網橋和交換機就學到了MAC地址。

7． 網橋和交換機都基於第二層地址做出轉發的決定。

第二層的三種交換功能

1． 地址學習 通過查看幀的源MAC地址來加進1個叫做轉發/過濾表的MAC地址數據庫裡

2． 轉發/過濾決定 當1個接口收到1個幀的時候,switch在MAC地址數據庫裡查看目標MAC地址和出口接口,然後轉發到符合條件的那個目標端口去

3． 避免環路 假如有冗餘的連接,可能會造成循環的產生,STP就用來破壞這些循環

生成樹協議

生成樹術語

1． STP 2． 根橋（Root bridge） 3． BPDU 4． 橋ID 5． 非根橋 6． 根端口 7． 指定端口 8． 端口開銷 9． 非指定端口 10． 轉發端口 11． 阻塞端口

生成樹的操作

STP的任務是找到網絡中的所有鏈路，並關閉任何冗餘的鏈路，這樣就可以防止網絡環路的產生。STP首先選舉一個根橋，由根橋來負責決定網絡拓撲。一旦所有的交換機都同意將某臺交換機選舉為根橋，其餘的每臺交換機就必須找到其根端口。如果在交換機之間有多條鏈路，就只能由唯一的一個指定端口。

選舉根橋

在網絡中，橋ID用來選舉根橋，並決定根端口。

只需要記住，在選舉根橋時值越低越好。

選擇指定端口

如果有多條鏈路連接到根端口，那麼，斷口開銷就變成了決定哪一個端口將成為根端口的依據。

各種Ethernet網絡的典型開銷

生成樹端口狀態

對於運行STP的網橋或交換機，其端口狀態會有下列5種狀態之間轉變：

1． 阻塞 2． 監聽 3． 學習 4． 轉發 5． 禁用

收斂

當網橋或交換機轉變到轉發或阻塞狀態時，就會產生收斂，再此期間，不會轉發任何數據。

LAN交換機的工作方式

1． 直通轉發 在這種工作方式下，在交換機查看MAC地址過濾表中的目的地址之間，交換機等待接收目的硬件地址。

2． 碎片丟棄 在這種方式下，交換機在轉發幀之前，先檢查幀的前64個字節，看它是不是碎片，由此保證不會轉發可能產生衝突的幀。

3． 存儲轉發 在這種工作方式下，交換機接收整個數據幀，放在其緩衝區內，並運行CRC，然後查看在MAC過濾表中的目的地址。

配置交換機

當1900啟動後,先運行POST,此時每個端口的發光二級管(LEDs)是綠色的,當POST完成後,LEDs開始閃爍然後熄滅;假如POST發現某個端口出問題了的話,系統LED和那個出問題的端口的LED開始變黃.假如你的console線纜接好了的話,POST開始顯示菜單,如下:

Setting the User Mode and Enable Mode Passwords

1900下,輸入K進入CLI,輸入enable進入特權模式,再輸入config t進入全局配置模式,如下:

>en

#config t

(config)#

當進入全局配置模式後,使用enable password命令配置登陸密碼個啟用密碼,如下:

(config)#enable password ?

level Set exec level password

(config)#enable password level ?

<1-15> Level Number

level1為登陸密碼,level15為啟用密碼,密碼長度範圍是4到8字符之間

2950下的配置和配置router有點類似,如下:

Switch>en

vty Virtual terminal

<0-15> First Line number

Switch(config)#line vty 0 15

Switch(config-line)#login

設置主機名

給1900配置主機名,使用hostname命令,如下:

(config)#hostname SZ_Switch

SZ_Switch(config)#

給2950配置主機名,使用hostname命令,如下:

Switch(config)#hostname SZ_Switch

SZ_Switch(config)#

設置IP信息

你可以不配置IP信息,直接把線纜插進端口,一樣可以工作.配置IP地址信息有2點原因:

1.通過telnet或其他軟件方式來管理switch

2.配置VLANs和其他等網絡功能

默認下,沒有IP地址和默認網關信息配置,在1900下,使用show ip命令查看默認IP配置,如下:

1900#sh ip

IP Address: 0.0.0.0

Subnet Mask: 0.0.0.0

Default Gateway: 0.0.0.0

Management VLAN: 1

Domain name:

Name server 1: 0.0.0.0

Name server 2: 0.0.0.0

HTTP server: Enable

HTTP port: 80

RIP: Enable

在1900下使用ip address和ip default-gateway命令來配置IP地址信息和默認網關信息,如下:

1900(config)#ip address 172.16.10.16 255.255.255.0

1900(config)#ip default-gateway 172.16.10.1

1900(config)#

2950下的配置是在VLAN1接口下配置,VLAN1是管理VLAN,默認所有接口均是VLAN1的成員,配置如下:

2950(config)#int vlan1

2950(config-if)#ip address 172.16.10.17 255.255.255.0

2950(config-if)#no shut

2950(config-if)#exit

2950(config)#ip default-gateway 172.16.10.1

2950(config)#

注意2950的IP地址配置是在VLAN1接口下,另外要注意打開接口

配置接口描述

配置描述信息,在接口配置模式下使用description命令,1900下的描述信息不能使用空格鍵,如下:

1900(config)#int e0/1

1900(config-if)#description Cisco_VLAN

1900(config-if)#int f0/26

1900(config-if)#description trunk_to_building_4

1900(config-if)#

2950下的描述就可以使用空格鍵,如下:

2950(config)#int fa 0/1

2950(config-if)#description Sales Printer

2950(config-if)#^Z

可以使用show interface和show running-config命令來查看這些描述信息

擦除交換機的配置

1900和2950的配置文件是存儲在NVRAM裡的,但是1900裡你不能查看NVRAM或startup-config的內容,只能查看running-config的內容,在1900裡,你對配置所進行的修改自動被複制到NVRAM裡,所以沒有copy run start這樣的命令;但是2950就有startup-config和running-config,使用copy run start來保存配置到NVRAM裡,擦除2950裡startup-config文件使用erase startup-config命令;擦除1900裡的使用delete命令.如下:

1900#delete ?

nvram NVRAM configuration

vtp Reset VTP configuration to defaults

1900#delete nvram

2950如下:

2950#erase startup-config
