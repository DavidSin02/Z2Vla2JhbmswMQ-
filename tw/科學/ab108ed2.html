使用fswatch命令監控Linux中的文件和目錄的變化工具

1. 前言

fswatch是一個跨平臺的文件修改監視工具，當指定文件或目錄的內容被更改時，它會收到告警通知。

它在不同的操作系統上執行四種類型的監視，例如:

  • 一個建立在蘋果OS X的File System Events API 。
  • 基於kqueue的監視器，FreeBSD 4.1中出現的通知接口也支持許多類BSD系統，包括OS X。
  • 一個基於Solaris內核的File Events Notification API 及其派生的監視器。
  • 一個基於inotify的監視器，inotify是一個內核子系統，用於顯示應用程序對文件系統的修改。
  • 一個基於ReadDirectoryChangesW的監視器，這是一種Windows API，用於記錄對目錄的更改。
  • 一個定期檢查文件系統狀態的監視器，將文件修改時間保存在內存中，並手動檢測文件系統的變化(在任何地方都可以使用stat)。

fswatch的特點

  • 支持幾個特定操作系統的api
  • 允許遞歸目錄監視
  • 使用包含和排除正則表達式執行路徑篩選
  • 支持自定義記錄格式
    此外，它還支持週期性的空閒事件

2. 如何在Linux系統中安裝`fswatch`

值得注意的是，在任何Linux發行版中的默認系統倉庫中都無法安裝fswatch包。安裝最新版本的fswatch的唯一方法是從源代碼開始構建，如下面的安裝說明所示。

首先獲取最新的fswatch源代碼，使用以下wget命令，並安裝如下:

wget https://github.com/emcrisostomo/fswatch/releases/download/1.9.3/fswatch-1.9.3.tar.gztar -xvzf fswatch-1.9.3.tar.gzcd fswatch-1.9.3./configuremakesudo make install 

如何在Linux系統中安裝fswatch


重要提示:在從源代碼編譯fswatch之前，請確保在系統上安裝了GNU GCC (C和c++編譯器)和開發工具(在Debian/Ubuntu上是必需的)。如果沒有，使用以下命令在各自的Linux發行版上安裝它。基於紅帽CentOS/RHEL系統

yum group install 'Development Tools'

基於紅帽Fedora 22+系統

dnf group install 'Development Tools'

基於Debian/Ubuntu系統

sudo apt-get install build-essential

在Debian/Ubuntu發行版上，執行fswatch命令時可能會出現以下錯誤。

fswatch: error while loading shared libraries: libfswatch.so.6: cannot open shared object file: No such file or directory

要修復它，您需要執行下面的命令，這將有助於在開始使用fswatch之前刷新到動態庫的鏈接和緩存。

sudo ldconfig

3. 如何在Linux上使用fswatch ?

運行fswatch的一般語法是:

fswatch [option] [path]

在Linux上，建議您使用默認的inotify監視器，您可以使用-M或- list-monitors選項來列出可用的監視器:

fswatch -Mfswatch --list-monitors

如何在Linux上使用fswatch


下面的命令允許您監控當前目錄(/home/zcwyou)中的變化，每4秒做一次標準輸出。

-l或--latency選項允許您以秒為單位設置延遲，默認為1秒。

fswatch -l 4 . 
fswatch --timestamp /var/log/auth.log

接下來，-x或--event-flags告訴fswatch沿事件路徑打印事件標誌。您可以使用–event-field-seperator選項來使用特定的分隔符打印事件。

fswatch --events-flags ~ /var/log/auth.log

要打印主目錄和/var/log/auth.log文件中發生變化事件數值，請使用-n或--numeric選項，如下所示:

fswatch --numeric ~ /var/log/auth.log 

也許你可以通過fswatch手冊頁查看詳細的使用選項和信息:

man fswatch
