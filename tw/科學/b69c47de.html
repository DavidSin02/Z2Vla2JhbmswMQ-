讀《演進式架構》有感

2020-11-14
| 科學

最近讀了一本很有意思的書《演進式架構》，從一個不太一樣的角度來描述架構這件事。也讓我痛定思痛的反省了一下過去犯的錯誤，以及由於知識欠缺，所留下的遺憾。

這不是一篇介紹書目的軟文，所以不會通篇介紹書中內容，我也沒有電子版的鏈接，單純是結合個人經歷抒發一些感慨......

Evolutionary Architecture

它的定義原文如下：

An evolutionary architecture supports incremental, guided change as a first principle across multiple dimensions.

翻譯過來大致是演進式架構是一種支持將增量式、指導式的變更作為跨多個維度中的第一原則的架構。

這裡涉及到幾個要素：

  • Incremental change
  • Guided change with fitness functions
  • Appropriate coupling

獨輪車理論

做架構，就像表演雜技獨輪車，你要保持車身的平衡，同時還要接住一個個球，並且讓他們也加入這個平衡裡來。

一個個球，就是人力資源、時間節點、功能需求、性能需求等等，在不摔下來的前提下，你不能接住所有的球，那麼如何選擇你的策略，就是亟待解決的問題，在時間節點固定的前提下是加大人力投入？還是砍需求？或是，客戶要求的功能一定要實現的前提下，去遊說到更多的時間和人力資源...

長袖善舞...也是架構師必要的技能

康威定律

Conway’s law: Organizations which design systems are constrained to produce designs which are copies of the communication structures of these organizations. - Melvin Conway(1967)

設計系統的組織其產生的設計等價於組織間的溝通結構。

反向理解起來也是成立的。

Conway’s law reversed：You won’t be able to successfully establish an efficient organizational structure that is not supported by your system architecture design.

如果系統架構不支持，你無法建立一個高效的組織。

對於這一點，我的體會更深一點，在做BI產品的時候，起初我們將產品分成了若干功能模塊，然後按照模塊來構建團隊，起初還是很順利的各個模塊表現良好，每一個模塊其實都可以作為一款單獨的產品來賣了。但是攢成一個BI產品，整體上，總是欠缺一點東西，融合不到一起的感覺，這也是困擾V3-V5的一個通病了。現在回想起來，當初一直要推行杜絕“各掃門前雪”的行為，也真的挺無奈的，是不是當時做一些組織結構的調整能好一些呢？

增量變更

Incremental change describes two aspects of software architecture: how teams build software incrementally and how they deploy it.

其中增量構建，比如向前兼容，多版本支持；部署的話，比如藍綠部署、金絲雀部署、feature toggles等。

它要求這些變更是可逆的，即可以回滾。當然有些架構是用來拋棄/犧牲的。比如可犧牲的架構中提到的：

支持1996年eBay的合適架構，對於2006年eBay來說，就不是合適的了。1996年的架構無法處理2006年的負載，但是2006年的版本太過複雜而難以建立、維護，它是根據1996年的需求演化而來的。的確，這個原則可以引出工作的一種組織方式。在Google，大家熟知的要求就是設計一個滿足當前10倍需求的系統，這暗示著如果需求超過了一個數量級，那麼扔掉並從頭做起是更好的。每隔幾年就被重新設計與拋棄的子系統而言，這是非常普遍的。

適應度函數

這個是書中最晦澀的部分，我特意查了下原文，“fitness functions”翻譯上似乎沒有什麼問題，但是總覺得怪怪的。

A particular type of objective function that is used to summarize…how close a given design solution is to achieving the set aims.

意思大概就是，在構建fitness functions的時候，要找出系統的關鍵能力，那些能力是可以被弱化或是移除的......

反正這塊是看的雲裡霧裡的.....

總體上，這本書還是推薦一讀的，我也還在研習中，先發一點感慨上來，也許通讀以後，還會再寫點啥，也歡迎有興趣的同學與我交流....
