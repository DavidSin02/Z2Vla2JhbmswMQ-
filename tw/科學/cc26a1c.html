Day9 G2/Axis座標軸詳解

| 科學

在學會怎麼配置座標軸之前，我們需要了解G2中座標軸的組成。

座標軸的配置

同其他屬性一樣，座標軸的配置也有兩種方式：

  • 屬性式配置
options:{ axes: { field: { title:{}, line:{}, label:{}, tickLine:{}, grid:{} } }}
  • 函數式配置
 chart.axis("field",{ title:{}, line:{}, label:{}, tickLine:{}, grid:{} });

說完座標軸的總體配置方式，下面給大家介紹座標軸各部分的具體配置。

標題 title

title的可配置參數如下：

title: { autoRotate: {boolean}, // 是否需要自動旋轉，默認為 true offset: {number}, // 數值，設置座標軸標題距離座標軸線的距離 // 設置標題的文本樣式 textStyle: { textAlign: 'center', // 文本對齊方向，可取值為： start middle end fill: '#404040', // 文本的顏色 fontSize: '12', // 文本大小 fontWeight: 'bold', // 文本粗細 rotate: 30, // 文本旋轉角度，以角度為單位，僅當 autoRotate 為 false 時生效 textBaseline: 'top' // 文本基準線，可取 top middle bottom，默認為middle }, position: 'start' | 'center' | 'end' // 標題的顯示位置（相對於座標軸線），可取值為 start center end}

注意：在G2默認主題中，座標軸標題是默認不顯示的。

//name代表下圖中x軸的變量，value表示y軸的變量 axes: { name: { }}

我們發現當未設置name即X軸中的title時，默認是不顯示的，如果想要顯示座標軸標題，則必須將title屬性設置為真值（備註：在JS中除了null、undefined、0、false、""這5個假值，其他都為真值），如果需要修改title的樣式位置等屬性時，只要對title內部進行屬性設置就可以。

axes: { name: { title:{ textStyle: { //標題樣式設置 fontSize:20,  fill:"red" } }  }}

從上圖可以看出，當我們將name中的title設置為真值時，座標軸默認的標題是該數字字段對應的屬性名（這裡X軸對應的屬性名為name），如果要更改座標軸的標題，需要到度量scales中的alias屬性修改，修改方式如下：

scales: { name: { type: "cat", alias: "水果名" //座標軸標題名修改 } } axes: { name: { title:{ textStyle: { //標題樣式設置 fontSize:20,  fill:"red" } }  } }

總結：座標軸的title屬性主要是對標題樣式的一些配置，而如果要修改座標軸標題的名字，則需要到度量scales中的alias屬性中設置。

座標軸線 line

line可配置參數如下：

line: { stroke: {string}, // 座標軸線的顏色 strokeOpacity: {number}, // 座標軸線的透明度，數值範圍為 0 - 1 lineDash: {array}, // 設置虛線的樣式，如 [2, 3]第一個用來表示實線的像素，第二個用來表示空白的像素。如果提供了奇數個值，則這個值的數列重複一次，從而變成偶數個值 lineWidth: {number} // 設置座標軸線的粗細}

這一塊參數設置比較簡單，自行實驗調參即可，這裡不再詳述，放一個虛線座標軸的配置和效果圖：

 line:{ stroke:"red", lineDash:[2,3], lineWidth:2 }

注意：G2默認主題中，y軸座標線默認不顯示，如果想讓它顯示，可將line屬性設置為真值，如想進一步設置y軸樣式，可對y軸對應的字段名的line參數進行配置。

文本 label

label可配置參數如下：

label: { offset: {number}, // 數值，設置座標軸文本 label 距離座標軸線的距離，圖例legend也會移動 offsetX: {number}, // 在 offset 的基礎上 x 方向的偏移量 offsetY: {number}, // 在 offset 的基礎上 y 方向的偏移量 // 設置文本的顯示樣式，還可以是個回調函數，回調函數的參數為該座標軸對應字段的數值 rotate: 30, // 注意，旋轉角度的配置不再在 textStyle 裡配置 textStyle: { rotate:30,//label沿順時針方向的旋轉度數 textAlign: 'center', // 文本對齊方向，可取值為： start center end fill: '#404040', // 文本的顏色 fontSize: '12', // 文本大小 fontWeight: 'bold', // 文本粗細 textBaseline: 'top' // 文本基準線，可取 top middle bottom，默認為middle } | (text) => { // text: 座標軸對應字段的數值 }, autoRotate: {boolean}, // 文本是否需要自動旋轉，默認為 true
  • label更詳細參數配置項請看chart基本屬性配置

注意：目前的3.4版本的G2對於label還存在一個bug，就是在label的textStyle中設置rotate只有首個label生效，但是在textStyle外部設置label全部有效。

刻度線 tickLine

tickLine可配置參數如下：

tickLine: { lineWidth: 1, // 刻度線寬 stroke: '#ccc', // 刻度線的顏色 strokeOpacity: 0.5, // 刻度線顏色的透明度 length: 5, // 刻度線的長度，可以為負值（表示反方向渲染） alignWithLabel:true//設為負值則顯示為category數據類型特有的樣式}

示例：

 tickLine:{ stroke:'black', length:5, //注意正方向是x軸反方向 lineWidth:2 }

注意：X軸的tickLine是默認存在的，Y軸的axis配置信息都是默認不存在的，包括tickLine。

網格 grid

網格可配置參數如下：

grid: { align: 'center', // 聲明網格頂點從兩個刻度中間開始，默認從刻度點開始 type: 'line' | 'polygon', // 聲明網格的類型，line 表示線，polygon 表示矩形框 // 當網格類型 type 為 line 時，使用 lineStyle 設置樣式 lineStyle: { stroke: '#d9d9d9', // 網格線的顏色 lineWidth: 1, // 網格線的粗細 lineDash: [4, 4 ] // 網格線的虛線配置，第一個參數描述虛線的實部佔多少像素，第二個參數描述虛線的虛部佔多少像素 }, hightLightZero: true, // 默認不高亮0軸 zeroLineStyle: { // 當且僅當 highLightZero 為 true 時生效 stroke: '#595959', lineDash: [ 0, 0 ] }, alternateColor: '#ccc' | [ '#f80', '#ccc' ], // 當網格類型 type 為 polygon 時，使用 alternateColor 為網格設置交替的顏色，指定一個值則先渲染奇數層，兩個值則交替渲染 hideFirstLine: true | false, // 是否隱藏第一條網格線，默認為 false hideLastLine: true | false // 是否隱藏最後一條網格線，默認為 false}

常用的grid屬性配置如下：

grid:{ type: 'line', lineStyle: { stroke: 'blue', lineWidth: 1, lineDash: [ 4, 4 ] }, }

注意:G2默認主題下，Y軸默認會生成虛線網格線，X軸默認不會生成網格線。

Scale度量和Axis的區別

我們知道Scale和Axis都能對座標系進行設置，所以在這裡比較一下兩者的區別：

Scale：度量可以設置座標軸的刻度點個數，間距等，更多的是對刻度的設置。

Axis：座標軸更多的是座標軸刻度和座標軸的樣式設置，但是無法設置刻度。
