SQLite中的數據庫Btree索引

2020-10-29
| 科學


當我們考慮數據庫的性能時，首先想到的是索引。 在這裡，我們將研究數據庫索引如何在數據庫上工作。 請注意，此處基於SQLite 2.x數據庫架構描述了架構細節。 您可以在https://github.com/madushadhanushka/simple-sqlite中找到與該博客相關的SQLite 2.5.0後端實現以及示例測試。

什麼是Btree？

Btree是一種數據結構，它按排序順序將數據存儲在其節點中。 我們可以將樣本Btree表示如下。

Btree存儲數據，以便每個節點包含升序的鍵。 這些鍵中的每個鍵都具有對另外兩個子節點的兩個引用。 左側子節點密鑰小於當前密鑰，右側子節點密鑰大於當前密鑰。 如果單個節點具有" n"個鍵，則它最多可以具有" n + 1"個子節點。

為什麼在數據庫中使用索引？

認為您需要在文件中存儲數字列表，然後在該列表中搜索給定的數字。 最簡單的解決方案是將數據存儲在數組中，並在出現新值時附加值。 但是，如果需要檢查數組中是否存在給定值，則需要逐一搜索所有數組元素，並檢查給定值是否存在。 如果足夠幸運，您可以在第一個元素中找到給定的值。 在最壞的情況下，該值可以是數組中的最後一個元素。 我們可以將這種最壞情況表示為O（n）的漸近符號。 這意味著，如果您的數組大小最多為" n"，則需要執行" n"次搜索才能在數組中找到給定值。

您如何減少此搜索時間？ 最簡單的解決方案是對數組進行排序，然後使用二進制搜索來找到值。 每當您將值插入數組時，它都應保持順序。 從數組中間選擇一個值開始搜索。 然後將所選值與搜索值進行比較。 如果選擇的值大於搜索值，則忽略數組的左側，而在右側搜索值，反之亦然。

> Binary search

在這裡，我們嘗試從已排序的數組3、6、8、11、15、18和18中搜索鍵15。 如果您進行常規搜索，則自該元素位於第五位置以來，將需要五個單位的時間進行搜索。 但是，在二進制搜索中，只需要進行三個搜索。

> Binary search to all element

看起來很熟悉？ 它是一棵二叉樹。 這是Btree的最簡單形式。 對於二叉樹，我們可以使用指針而不是將數據保留在已排序的數組中。 從數學上講，我們可以證明二叉樹的最壞情況搜索時間為O（log（n））。 二叉樹的概念可以擴展為更廣義的形式，即Btree。 Btree不是為單個節點提供單個條目，而是為單個節點使用條目數組，併為每個條目引用子節點。 下面是每個上述方法的時間複雜度比較。

B + tree是另一個用於存儲看起來與Btree幾乎相同的數據的數據結構。 B + tree的唯一區別是它在葉節點上存儲數據。 這意味著所有非葉節點的值將再次在葉節點中重複。 下面是示例B +樹。

是的，葉節點包含所有值，並且所有記錄均按排序順序。 在B + tree的特殊情況下，您可以進行與Btree相同的搜索，此外，如果我們按如下所示將指針指向每個葉節點，則可以遍歷葉節點中的所有值。

如何在數據庫中使用索引？

當Btree進行數據庫索引時，由於不僅具有鍵，而且具有與該鍵關聯的值，因此該數據結構幾乎沒有什麼複雜的。 該值是對實際數據記錄的引用。 此鍵和值一起稱為有效負載。

假設下面的三列表需要存儲在數據庫中。

┌───────┬───────┬─────┐│ Name  │ Marks │ Age │├───────┼───────┼─────┤│ Jone  │     5 │  28 ││ Alex  │    32 │  45 ││ Tom   │    37 │  23 ││ Ron   │    87 │  13 ││ Mark  │    20 │  48 ││ Bob   │    89 │  32 │└───────┴───────┴─────┘

首先，數據庫為每個給定記錄創建唯一的隨機索引（或主鍵），並將相關行轉換為字節流。 然後，它將每個密鑰存儲在B +樹上並記錄字節流。 這裡，隨機索引用作索引的鍵。 密鑰和記錄字節流共稱為有效負載。 生成的B + tree可以表示如下。

> B+tree on database pages

在這裡您可以看到，所有記錄都存儲在B +樹的葉節點中，索引用作創建B +樹的關鍵。 沒有記錄存儲在非葉子節點上。 每個葉節點都引用樹中的下一個記錄。 數據庫可以使用索引執行二進制搜索，也可以通過僅遍歷葉節點來搜索每個元素來進行順序搜索。

如果未使用索引，則數據庫將讀取每個記錄以查找給定記錄。 啟用索引後，數據庫將為表中的每一列創建三個Btree，如下所示。 這裡的鍵是用於索引的Btree鍵。 索引是對實際數據記錄的引用。

使用索引時首先，數據庫在對應的Btree中搜索給定鍵，並以O（log（n））時間獲得索引。 然後，它使用O（log（n））時間中已找到的索引在B + tree中執行另一個搜索並獲取記錄。

數據庫頁面實施

Btree和B + tree中的每個節點都存儲在頁面內。 頁面大小固定。 頁面的唯一編號從1開始。 一個頁面可以通過使用頁碼來引用另一個頁面。 在頁面的開頭，存儲頁面元詳細信息，例如最右邊的子頁面編號，第一個空閒單元格偏移量和第一個單元格偏移量。 數據庫中可以有兩種類型的頁面。

· 用於索引的頁面：這些頁面僅存儲索引和對另一個頁面的引用。

· 存儲記錄的頁面：這些頁面存儲實際數據，並且頁面應該是葉子頁面。

結論

數據庫應具有一種有效的方法來存儲，讀取和修改數據。 Btree提供了一種有效的方式來插入和讀取數據。 在實際的數據庫實現中，數據庫同時使用Btree和B + tree來存儲數據。 Btree用於索引，B + tree用於存儲實際記錄。 除了二進制搜索，B + tree還提供順序搜索功能，該功能使數據庫可以更好地控制搜索數據庫中的非索引值。
