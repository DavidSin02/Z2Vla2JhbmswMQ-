單片機很好玩7，製作一個智能溫度計和溼度計，點擊鼠標就能查詢

2020-10-29
| 科學

軟件總是給人一種“無所不能”的感覺，衣有購物軟件，食有外賣軟件，住有訂房軟件，行有打車軟件。甚至，想知道電腦的主板溫度，還可以下載監控軟件查看。

藉助外界傳感器

看了上一節文章的朋友應該明白，我們之所以能夠使用監控軟件查看主板溫度，是因為電腦主板集成了溫度傳感器。軟件本身是無法主動獲取外界環境狀態的，例如，如果我想知道現在室內的溫度和溼度，該下載哪個軟件呢？除非電腦本身延伸出了溫度傳感器，否則無論哪個軟件也不能獲得室內的溫度和溼度。

我的電腦沒有額外的傳感器，是不是就沒有辦法查看室內的溫度和溼度了呢？只依靠軟件的確不行，但是配合單片機和傳感器就沒問題了。本節將介紹這種方法。最終的效果如下，可以在電腦上打開軟件查看我室內的溫度和溼度信息：

RH 表示溼度信息，88% 左右，TM 表示溫度，7度左右。

計劃使用的還是 51 單片機，傳感器是 DHT11。DHT11 數字溫溼度傳感器是一款含有已校準數字信號輸出的溫溼度複合傳感器，包括一個電阻式感溼元件和一個 NTC 測溫元件，並與一個高性能的 8 位單片機連接。DHT11 只有一根數據線，可如下與單片機相連：

編寫 C語言代碼，啟動 DHT11 溫溼度傳感器

那麼怎樣使用 51 單片機採集 DHT11 採集的溫溼度信息呢？這就需要看它的通信協議了

這就明白了，51 單片機先將總線拉低 18 ms 以上，然後再拉高 20~40 us。接著就可以等待 DHT11 的響應了，響應信號也很簡單，就是先低後高，之後再傳來的就是數據了。所以，初始化動作的 C語言代碼可以如下寫：

void dht11_start(){ P20 = 1; delay_10us(2000); P20 = 0; delay_10us(2000); P20 = 1; while(P20); while(!P20);}

delay_10us() 函數是在上一節的內容基礎上實現的函數，它的C語言代碼如下：

set_timer0(10);void delay_10us(unsigned int n){ while(n--){ start_timer0(); wait_timer0(); }}

dht11_start() 函數的邏輯非常簡單，先將與 DHT11 數據線連接的 P20 口拉低 20 ms 再拉高，接著的 while(P20) 和 while(!P20) 是等待 DHT11 的響應信號的。

編寫C語言代碼，讀取DHT11溫溼度信息

DHT11 啟動後，會一次性發送 40 bit 的數據，高位先出，它的數據格式為：

1Byte溼度整數部分 + 1Byte溼度小數部分 + 1Byte溫度整數部分 + 1Byte溫度小數部分 + 1Byte校驗和

那麼， 什麼樣的信號表示 bit 1，什麼樣的信號表示 bit 0 呢？請看下圖：

每一 bit 的數據都是跟在 50us 的低電平之後的，究竟是 0 還是 1 則取決於高電平的持續時間，26us~28us 的高電平表示 0，70us 的高電平表示 1。

所以讀取一次溫溼度信息的 C語言代碼可以如下寫：

void dht11_read_once(char* dat){ unsigned char i=0, j; char tmp = 0; while(P20); for(i=0; i<5; i++){ tmp = 0; for(j=0;j<8;j++){ while(!P20);  delay_10us(4); tmp <<= 1; if(P20) tmp |= 1; while(P20); } dat[i] = tmp; }}

dht11_read_once()函數也是非常的簡單，核心就是判斷高電平的持續時間。while(!P20) 執行之後，數據總線剛好到達高電平，接下來延時 40us，如果數據總線還是高電平，則可以認為該 bit 數據是 1，否則就是 0。接下來的 while(P20) 可以等待高電平結束，到達下一 bit 數據開始前的低電平。

編寫C語言代碼，將讀取到的數據發送到電腦

dht11_read_once() 函數返回的是 5 字節數據，怎樣將其組合，發送到電腦呢？其實使用 printf 函數是非常簡單，但是經過我的測試，發現該函數消耗的資源太多了，資源匱乏的 51 單片機扛不住，直接罷工，無法正在工作。

所以還是要自己封裝函數，上一節封裝的 prints 函數只能傳輸字符串，那麼我們就需要再定義一個可以傳送數字的函數，請看如下C語言代碼：

void printn(unsigned char num){ char buf[3] = {0}; buf[0] = num/100 + '0'; buf[1] = (num/10)%10 + '0'; buf[2] = num%10 + '0'; prints(buf);}

printn() 函數可以把一個 unsigned char 型整數轉換為字符發送到串口。所以DHT11的溫溼度信息可以按照如下邏輯傳輸，請看C語言代碼：

 cks = 0; cks += dat[0]; cks += dat[1]; cks += dat[2]; cks += dat[3]; if((char)cks == dat[4]){ prints("RH: ");printn(dat[0]);prints(".");printn(dat[1]);prints(" "); prints("TM: ");printn(dat[2]);prints(".");printn(dat[3]);prints("\r\n"); }else{ prints("capture failed\r\n"); }
