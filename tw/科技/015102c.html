Java面試基礎回顧17-多態

2020-10-29
| 科技

面向對象的三大特徵：封裝、繼承、多態。

規則：父類類型 對象名稱 = new 子類構造器 或者 接口 對象名稱 = new 實現類構造器

概念：同一個類型的對象，執行同一個行為，在不同的狀態下會表現出不同的行為特徵。

/** * 多態測試 */public class Demo {    public static void main(String[] args) {        //都是屬於Animal對象        Animal dog = new Dog();        //都執行eat行為表現出不同的行為特徵        dog.eat();        Animal cat = new Cat();        cat.eat();    }}/** * Cat子類 */class Cat extends Animal{    @Override    public void eat() {        System.out.println("貓吃魚");    }}/** * Dog子類 */class Dog extends Animal{    @Override    public void eat() {        System.out.println("狗吃骨頭");    }}/** * 父類 */class Animal{    public void eat(){        System.out.println("動物吃東西");    }}

運行結果：

多態表現出不同的行為特徵

條件：

1.必須存在繼承或者實現關係。

2.必須是父類類型變量引用子類類型的對象。

3.需要存在方法重寫。

優點：子類對象的改變，業務功能也隨之改變，便於功能的擴展和維護，可以實現類與類之間的解耦。

/** * 多態測試 */public class Demo {    public static void main(String[] args) {        //都是屬於Animal對象        Animal dog = new Dog();        //都執行eat行為表現出不同的行為特徵        dog.eat();        Animal cat = new Cat();        cat.eat();        go(dog);        go(cat);    }    /**     * 使用父類類型對象作為形參，可以靈活的傳入子類類型的對象     * 從而實現不同的行為特徵     * @param animal     */    public static void go(Animal animal){        System.out.println("begin");        animal.eat();        System.out.println("done");    }}/** * Cat子類 */class Cat extends Animal{    @Override    public void eat() {        System.out.println("貓吃魚");    }}/** * Dog子類 */class Dog extends Animal{    @Override    public void eat() {        System.out.println("狗吃骨頭");    }}/** * 父類 */class Animal{    public void eat(){        System.out.println("動物吃東西");    }}

缺點：多態情況下，不能直接調用子類特有的功能，因為父類沒有。 但是也有解決辦法，就是父類對象強制轉換為子類對象然後調用子類特有的方法。
