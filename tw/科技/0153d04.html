面向過程程序設計思路：函數封裝、泛化、重構

2020-10-29
| 科技

1 面向過程開發計劃

A development plan(開發計劃) is a process for writing programs. The process we used in this case study is “encapsulation（封裝，這裡是指函數封裝） and generalization（泛化）”. The steps of this process are:

1.1 Start by writing a small program with no function definitions.

1.2 Once you get the program working, identify a coherent piece of it, encapsulate（封裝） the piece in a function and give it a name.

1.3 Generalize the function by adding appropriate parameters.（函數添加合適的參數實現泛化）

1.4 Repeat steps 1–3 until you have a set of working functions. Copy and paste working code to avoid retyping (and re-debugging).

1.5 Look for opportunities to improve the program by refactoring（重構）. For example, if you have similar code in several places, consider factoring it into an appropriately general function.

This process has some drawbacks—we will see alternatives later—but it can be useful if you don’t know ahead of time how to divide the program into functions. This approach lets you design as you go along.

2 面向過程開發中的函數

In the context of programming, a function is a named sequence of statements that performs a computation. When you define a function, you specify the name and the sequence of statements. Later, you can “call” the function by name.

3 Parameters and arguments（形參和實參）

Some of the functions we have seen require arguments. For example, when you call math.sin you pass a number as an argument. Some functions take more than one argument: math.pow takes two, the base and the exponent.

Inside the function, the arguments are assigned（賦值） to variables called parameters.

When you create a variable inside a function, it is local（局部的）, which means that it only exists inside the function.

Parameters are also local.

（當實參是一個已賦值的變量時，當這個變量是可變對象還是不可對象，以及可變對象在函數內部是賦值操作還是修改操作，是否修改實參變量會有所區別）

4 interface of a function函數接口

The interface of a function is a summary of how it is used: what are the parameters? What does the function do? And what is the return value? An interface is “clean” if it allows the caller to do what they want without dealing with unnecessary details.

5 Flow of execution執行流程

To ensure that a function is defined before its first use, you have to know the order statements run in, which is called the flow of execution.

Execution always begins at the first statement of the program. Statements are run one at a time, in order from top to bottom.

Function definitions do not alter the flow of execution of the program, but remember that statements inside the function don’t run until the function is called.

A function call is like a detour（迂迴路徑） in the flow of execution. Instead of going to the next statement, the flow jumps to the body of the function, runs the statements there, and then comes back to pick up where it left off.

That sounds simple enough, until you remember that one function can call another. While in the middle of one function, the program might have to run the statements in another function. Then, while running that new function, the program might have to run yet another function!

Fortunately, Python is good at keeping track of where it is, so each time a function completes, the program picks up where it left off in the function that called it. When it gets to the end of the program, it terminates.

In summary, when you read a program, you don’t always want to read from top to bottom. Sometimes it makes more sense if you follow the flow of execution.

6 函數定義的前置條件與調用的後置條件

An interface is like a contract between a function and a caller. The caller agrees to provide certain parameters and the function agrees to do certain work.

These requirements are called preconditions（前置條件） because they are supposed to be true before the function starts executing. Conversely, conditions at the end of the function are postconditions（後置條件）. Postconditions include the intended effect of the function (like drawing line segments) and any side effects (like moving the Turtle or making other changes).

Preconditions are the responsibility of the caller. If the caller violates a (properly documented!) precondition and the function doesn’t work correctly, the bug is in the caller, not the function.

If the preconditions are satisfied and the postconditions are not, the bug is in the function. If your pre- and postconditions are clear, they can help with debugging.

7 Why functions?為什麼要有函數？

It may not be clear why it is worth the trouble to divide a program into functions. There are several reasons:

  • Creating a new function gives you an opportunity to name a group of statements, which makes your program easier to read and debug（調試）.
  • Functions can make a program smaller by eliminating repetitive code. Later, if you make a change, you only have to make it in one place.
  • Dividing a long program into functions allows you to debug the parts one at a time and then assemble（組裝） them into a working whole.
  • Well-designed functions are often useful for many programs. Once you write and debug one, you can reuse（複用） it.

8 Glossary術語

function:

A named sequence of statements that performs some useful operation. Functions may or may not take arguments and may or may not produce a result.

function definition:

A statement that creates a new function, specifying its name, parameters, and the statements it contains.

function object:

A value created by a function definition. The name of the function is a variable that refers to a function object.

function header:

The first line of a function definition.

function body:

The sequence of statements inside a function definition.

parameter（形參）:

A name used inside a function to refer to the value passed as an argument.

function call（函數調用）:

A statement that runs a function. It consists of the function name followed by an argument list in parentheses.

argument（實參）:

A value provided to a function when the function is called. This value is assigned to the corresponding parameter in the function.

local variable（局部變量）:

A variable defined inside a function. A local variable can only be used inside its function.

return value（返回值）:

The result of a function. If a function call is used as an expression, the return value is the value of the expression.

fruitful function（有返回值函數）:

A function that returns a value.

void function（無返回值函數）:

A function that always returns None. None is a special value returned by void functions.

module（模塊）:

A file that contains a collection of related functions and other definitions.

import statement（import語句）:

A statement that reads a module file and creates a module object.

module object（模塊對象）:

A value created by an import statement that provides access to the values defined in a module.

dot notation（句點表示法）:

The syntax for calling a function in another module by specifying the module name followed by a dot (period) and the function name.

composition（組合）:

Using an expression as part of a larger expression, or a statement as part of a larger statement.

flow of execution:

The order statements run in.

stack diagram（棧圖）:

A graphical representation of a stack of functions, their variables, and the values they refer to.

encapsulation（封裝）:

The process of transforming a sequence of statements into a function definition.

generalization（泛化）:

The process of replacing something unnecessarily specific (like a number) with something appropriately general (like a variable or parameter).

interface（接口）:

A description of how to use a function, including the name and descriptions of the arguments and return value.

refactoring（重構）:

The process of modifying a working program to improve function interfaces and other qualities of the code.

development plan（開發計劃）:

A process for writing programs.

docstring（文檔字符串）:

A string that appears at the top of a function definition to document the function’s interface.

precondition（先置條件）:

A requirement that should be satisfied by the caller before a function starts.

postcondition（後置條件）:

A requirement that should be satisfied by the function before it ends.
