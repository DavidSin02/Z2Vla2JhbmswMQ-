Java子類創建實例時是如何加載父類的？會創建一個父類對象嗎？

2020-10-29
| 科技

首先，我們寫一個父類：

package com.wangyj.demo;public class Widget {public Widget(){ System.out.println("父類構造器");}public synchronized void doSomething() {}public String test() { System.out.println("父類test 方法"); return "";}}

父類

然後定義一個子類：

package com.wangyj.demo;public class LoggingWidget extends Widget {public LoggingWidget(){System.out.println("子類構造方法");}@Overridepublic synchronized void doSomething() {System.out.println(toString()+ " call doSomething"); super.doSomething();}public static void main(String[] args) { LoggingWidget widget = new LoggingWidget(); widget.doSomething();}}

子類

運行結果如下：

運行結果

分析：

我們使用jdk 自帶的工具javap進行反編譯。進入到.class文件所在的目錄執行命令 javap -c -l -v LoggingWidget;

輸出結果如下：

javap返回結果1-1

javap 返回結果圖1-2

javap 返回結果圖1-3

從javap 返回結果圖1-3中可以看出，main方法的執行順序 先執行的 #12 的LoggingWidget的構造器，再觀察javap 返回結果圖1-2的LoggingWidget的構造器中執行了#1方法，#1便是父類的init 方法；然後main方法中又執行了子類的init方法

最後 執行doSomething()方法。從而可知，創建子類的時候先調用了父類的init方法，然後調用了自身的init 方法。

結論：

子類創建時，先調用的是父類的構造器方法，然後再次調用的子類的構造器方法。只會創建一個子類對象，不會創建父類對象，但是通過調用父類的init 方法，可以加載父類的方法。
