異常Throwable的概述以及子類Exception的說明

異常Throwable

異常Throwable是整個java異常體系中的總根

下面 分為 兩大類 一個是 Error 一個是Exception

其中Error是錯誤 如服務器崩潰 數據庫崩潰等 與代碼關聯不大

Excpetion中又分為兩大類 一個是RuntimeException 另一個則是 非RuntimeException

其中RuntimeException指的則是平常程序員敲代碼所犯的錯誤 造成運行時異常

而非RuntimeException 為編譯時異常

處理異常方法1

try{} catch{Exception}

try catch finally

這種處理異常的方法不會終止程序

注意點:捕獲的異常中 父類異常要向後放 不然如果父類異常為Exception RuntimeException時 後面子類異常的捕獲將毫無意義

處理異常方法2

throw 一個異常對象（類的具體實例）

同時在方法上 throws 一個異常類

這種處理異常的方法會終止程序

throws 和 throw 的區別

注意點 :當拋出的對象為RuntimeException 也就是說拋出的異常對象為運行時異常時，此時就算不在方法上寫 throws 異常類，編譯也不會報錯

finally

除了Sytstem.exit（0）；以外 ，任何語句都無法跳過finally中的語句！

其功能是為了關閉各種IO流 或者 數據庫

面試題

主要解釋一下面試題2吧

public class Test1 {	/**	 * * A:面試題1			* final,finally和finalize的區別			* final可以修飾類,不能被繼承			* 修飾方法,不能被重寫			* 修飾變量,只能賦值一次			* 			* finally是try語句中的一個語句體,不能單獨使用,用來釋放資源			* 			* finalize是一個方法,當垃圾回收器確定不存在對該對象的更多引用時，由對象的垃圾回收器調用此方法。		* B:面試題2			* 如果catch裡面有return語句，請問finally的代碼還會執行嗎?如果會，請問是在return前還是return後。	 */	public static void main(String[] args) {		Demo d = new Demo();		System.out.println(d.method());	}}class Demo {	public int method() {		int x = 10;		try {			x = 20;			System.out.println(1/0);			return x;		} catch (Exception e) {			x = 30;							//此時x為30已經包裝好給return 接著去看finally了													return x;						//當finally結束後 按照原來的包裝路徑返回 所以返回值為30		} finally {			x = 40;			//return x;					千萬不要在finally裡面寫返回語句,因為finally的作用是為了釋放資源,是肯定會執行的										//如果在這裡面寫返回語句,那麼try和catch的結果都會被改變,所以這麼寫就是犯罪					//同時要注意finally中的賦值語句毫無意義！！finally存在的意義就是為了關閉各種流 以及對數據庫的操作												}	}}

自定義異常

自定義異常有何意義：只是為了區分異常的名字而已 功能都是由總根父類Throwable來提供
