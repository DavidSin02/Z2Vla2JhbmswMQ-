隔牆取物！100個Excel不打開也能取到工作表名稱

2020-11-14
| 科技



在用vba處理多個excel文件時，經常會遇到需要遍歷不同的excel文件。


如果是需要對每個excel文件都執行某些動作，那可以用Workbooks.Open方法逐個打開excel文件，然後編寫要執行的動作代碼，最後保存後關閉。


但是如果只是要過濾每個Excel文件內的工作表名稱，滿足指定條件的工作表名稱才需要打開處理時，如果還是逐個打開來判斷，效率就低了，如果再遇到每個excel文件都比較龐大，逐個打開那就更加悲劇了，加載數據都需要消耗較長的時間。


比如現在手上有100多個Excel工作簿，需要用vba判斷是否有工作表名稱含"統計"兩個字的工作簿，然後打開對其執行一系列操作，應該如何操作才比較高效呢？


今天介紹一種用ado技術實現的方法，可以在不打開excel工作簿的情況下就可以獲取到其內部的excel工作表名稱，代碼如下：


Sub exceloffice()    Dim sFN As String    '獲取要讀取工作表名稱的excel工作簿    sFN = Excel.Application.GetOpenFilename()    If Len(sFN) Then        Dim arrName()        Dim objCatalog        Set objCatalog = VBA.CreateObject("ADOX.Catalog")        Dim sVersion As String        sVersion = Excel.Application.Version        Dim sConStr As String        '創建連接字符串，不同的excel版本使用不同的連接字符串        If sVersion <= 12 Then            sConStr = "Provider='Microsoft.Jet.OLEDB.4.0';Data Source=" & sFN & ";Extended Properties='Excel 8.0;HDR=YES'"        Else            sConStr = "Provider='Microsoft.ACE.OLEDB.12.0';Data Source=" & sFN & ";Extended Properties='Excel 12.0;HDR=YES'"        End If        Dim oConStr        Set oConStr = VBA.CreateObject("ADODB.Connection")        '使用Connection連接數據源        oConStr.Open sConStr        With objCatalog            '關聯Connection對象            Set .ActiveConnection = oConStr            Dim oTable            For Each oTable In .Tables                Dim sName As String                sName = oTable.Name                '提取工作表名稱                If Right(sName, 1) = "$" Then                    Debug.Print sName                    ReDim Preserve arrName(k)                    '將工作表名稱存在數組中                    arrName(k) = Left(sName, Len(sName) - 1)                    k = k + 1                End If            Next        End With        Set oConStr = Nothing        If UBound(VBA.Filter(arrName, "統計")) >= 0 Then            MsgBox "存在名稱為【統計】的excel工作表"        Else            MsgBox "不存在名稱為【統計】的excel工作表"        End If    End IfEnd Sub


限於篇幅，上述代碼僅舉例瞭如何對任意一個選中的excel工作簿，不打開提取其內部的excel工作表名稱，然後將工作表名稱保存在數組arrName中。


如果是多個excel工作簿，只需要添加循環遍歷的代碼即可。


這裡再額外介紹下ado技術，ado是一種專門用於訪問不同數據源內數據的技術，通過ado我們可以在同一個軟件內訪問各種不同的外部數據源。


但是實現的前提需要提供訪問數據源的鑰匙，通過鑰匙搭建與外部數據源之間連接的橋樑，這個鑰匙就是連接字符串。


不同的外部數據源有不同的連接字符串，比如針對Excel 2003版本的外部數據源，可以使用如下的連接字符串


Provider=Microsoft.Jet.OLEDB.4.0;Data Source=C:\MyExcel.xls;Extended Properties="Excel 8.0;HDR=Yes";


對於Excel 2007(含 2007) 以上的外部數據源，可以使用如下的連接字符串：


Provider=Microsoft.ACE.OLEDB.12.0;Data Source=c:\myFolder\myExcel2007file.xlsx;Extended Properties=Excel 12.0;HDR=YES";


對於各種不同的外部數據源對應的連接字符串可以在


https://www.connectionstrings.com/


這個網站都找到各種不同的數據源對應的連接字符串。
