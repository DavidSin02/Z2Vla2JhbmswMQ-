C語言：數據結構-歸併排序

2020-11-14
| 科技

歸併排序（Merging Sort）是基於兩個有序序列歸併的一種排序算法，設待排序的序列有n 個記錄，這n個記錄可以看成是n個有序序列，可以每兩個一組，分別進行兩個長度為1的有序序列的歸併，稱它為（1，1）歸併，這樣原序列就成為若干個長度為2的有序序列（當元素個數為奇數時也可能有長度為1的有序序列，程序將另行處理）。再把長度為2的相鄰的子序列每兩個一組，進行（2，2）歸併，得到若干個長度為4的有序序列。（也可能有長度小於4的有序序列，程序另行處理），接著進行（4，4）歸併•••最後得到排好序的長度為n的序列。稱上述過程中的一次歸併為一趟歸併。

例如有一組記錄的關鍵子序列（30，25，45，15，60，90，20，60），採用歸併排序算法對其排序，排序過程如圖9-7所示：

歸併排序示例

一趟歸併排序算法

	/*把a[0],a[1],....,a[n-1]中的記錄序列按關鍵字進行一趟歸併，其中每個子序列長度為s的（s,s）歸併，結果存入數組order中*/	mergepass(NODE a[],NODE order[],int s,int n)	{	 int i=0;	 while(i+2*s-1<=n-1)	 /*s為一趟歸併中子序列的長度，在劃分中只要夠兩個子序列就繼續歸併*/	 {	 merge(a,i,i+s-1,i+2*s-1,order);	 /*i和i+s-1分別為第一個序列開始和終止下標，i+2*s-1為第二個序列的終止下標，進行兩個	 序列的歸併*/	 i=i+2*s; /*下一段兩個子序列的起始下標*/	 }	 if(i+s<n-1)	 merge(a,i,i+s-1,n-1,order);/*最後兩個子序列中有一個長度不足s*/	 else	 while(i<n)	 order[i++]=a[i++]; /*最後只剩下一個長度〈=s的子序列，直接拷貝到結果數組中*/	 }

在一趟歸併排序算法的基礎上，就能完成對長度為n的整個序列的歸併排序，具體做法是逐次進行（1，1）、（2，2）、（4，4）等歸併，最終得到長度為n的有序序列。值得注意的是在一趟歸併排序算法中作為形式參數的兩個數組，一個是原始數組a，另一個是一趟歸併後的結果order，在調用時，如果相應於它們的實參分別是數組a 1 和a 2 ，則下一趟歸併要把a 2 作為原始數組，a 1 作為結果數組，這樣交替使用。

歸併排序算法

	/*對存儲在數組c1中，長度為n的記錄序列按關鍵字進行歸併排序，結果存放於數組c1中*/	mergesort(NODE c1[],int n)	{	 int i,s=1; /*s中存放每一趟兩兩歸併時，每個子序列的長度*/	 NODE c2[MAX]; /*MAX表示數組c2有足夠大的長度，數組c1，c2輪換著作為原始	 序列和一趟歸併後的結果序列*/	 while(s<n) /*當子序列長度小於n時繼續進行一趟歸併*/	 {	 mergepass(c1,c2,s,n); /*奇數次一趟歸併後，結果存放於c2中*/	 s=s*2; /*每進行一趟歸併子序列長度加倍*/	 if(s<n)	 /*子序列長度不超過n，進行偶數次一趟歸併，c2作為原始序列，結果存放於c1中*/	 {	 mergepass(c2,c1,s,n);	 s=s*2;	 }	 else /*如果子序列長度已等於n，證明已排序結束，且沒有進行偶數次一趟	 歸併，此時結果在C2中，從c2中把數據拷貝到c1中*/	 {	 for(i=1;i<=n;i++)	 c1[i]=c2[i];	 }	 }	}

歸併排序的時間複雜度是O(nlog2n)，是一種穩定的排序方法。

例：已知序列{10，18，4，3，6，12，1，9，15，8}，給出用歸併排序法對序列作升序排列時的每一趟結果（n為偶數）。

初始：10，18，4，3，6，12，1，9，15，8

[10，18] [3，4 ] [6，12] [1，9 ] [8，15]

[3，4，10，18] [1，6，9，12] [8，15]

最後只剩下長度<= s的一個子序列

[1，3，4，6，9，10，12，18] [8，15]

兩個子序列中只有一個長度不足

[1，3，4，6，8，9，12，15，18]
