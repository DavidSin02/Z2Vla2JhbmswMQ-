Windows系統不打補丁！看黑客是如何攻擊你電腦的

2020-10-29
| 科技

我們的臺式機和筆記本常用系統無非就是微軟的Windows 7 和Windows 10 兩個版本。服務器版本常用的是Windows server 2008 R2 ，Windows server 2012 R2 ， Windows server 2016 。在工作生活中我們的電腦系統、服務器系統經常提示升級、安裝補丁包，然而我們並沒有去安裝微軟推薦的補丁包，甚至將微軟更新功能想盡一切辦法關閉。這是一種非常危險的行為，尤其是服務器系統，這無異於服務器在互聯網上裸奔。

下面我將以非常著名的漏洞--永恆之藍漏洞為大家簡單演示黑客是如何一步一步攻擊你的服務器或者PC的。各位請遵守《中華人民共和國網絡安全法》。

網絡安全法

本次演示環境：

靶機：

系統：windows server 2008R2，

IP地址：192.168.88.105

攻擊機：

系統：kali

IP地址：192.168.88.104

使用漏洞掃描工具掃描對應服務器，確認存在的漏洞。根據掃描結果，我們發現該服務器存在MS17-010漏洞未修復。因此我們利用該漏洞發起攻擊。

漏洞掃描

查找攻擊工具集中是否有該漏洞的攻擊模塊並確定該模塊的位置，發現MS17-010模塊而且有多個，我們使用第四個，位置為：exploit/windows/smb/ms17_010_eternalblue。

MS17-010模塊

找到該模塊後使用use命令加載使用該模塊

加載MS模塊

完成模塊加載後需要對該模塊進行相對應的參數設置，設置rhosts地址，也就要攻擊的靶機地址，本演示中地址為192.168.88.105

設置靶機地址

發起攻擊後要在對應的地方接收會話連接返回，因此需要設置對應的返回會話的監聽，使用set命令設置監聽模塊。

加載攻擊模塊

使用set命令設置lhost地址，也就是攻擊機的地址，這樣我們才能正確接收靶機返回的會話連接。

設置攻擊機地址

所有參數設置完成後，使用options命令檢查參數設置是否正確，

options選項

確認上面的參數設置正確後我們就可以發起攻擊了，等待最後的勝利。

發起攻擊

我們發起攻擊後在第一輪攻擊中就已經把服務器攻陷了。先查看下系統版本是不是我們的靶機。返回結果是Windows 2008 R2 的系統版本，正是我們準備的靶機系統版本。

靶機系統版本查看

進入靶機後我們執行shell命令，相當於在Windows服務器上開啟cmd命令行。然後進行創建用戶等一系列操作。，由於Windows系統和Linux系統之間編碼方式不同，因此在Linux系統中連接Windows系統的shell就存在字符亂碼問題。這個問題可以在Linux系統終端中進行設置，我們這裡只是演示攻擊過程，就不進行編碼設置了。

登錄shell

接下來就可以使用cmd命令行中的命令對服務器進行操作了，我們使用net user 命令在靶機系統中添加一個用戶。

添加用戶

將新增加的密碼修改掉，如果不修改我們在後面沒有辦法進行遠程桌面登錄操作

修改新用戶密碼

我們將新增加的用戶添加到Administrators用戶組中，這樣我們新增加的用戶也就擁有管理員權限了。

提升新用戶權限

然後我們通過修改Windows系統的註冊表的方式開始遠程桌面進行下一步操作。

開啟遠程桌面

由於Windows系統默認開啟防火牆並且拒絕所有的外部用戶主動訪問服務器，因此我們需要修改防火牆策略允許我們進行遠程桌面登錄。

修改防火牆策略

此時我們已經完成攻擊，創建用戶並提升用戶權限到管理員級別。

下面我們用其他Windows主機使用mstsc遠程桌面登錄靶機。

使用遠程桌面登錄靶機

選擇信任證書後進入Windows 2008 R2操作系統。

進入系統

總結：

我們要針對微軟提出的高位及以上漏洞進行及時修復，尤其是服務器系統。或者通過安裝專業的安全防護軟件。
