iptables和nftables有什麼區別?

1. 前言

在本文中，我們將討論nftables和iptables之間的區別，並展示在新的nftables語法中配置防火牆規則的示例。

每個Linux管理員肯定都使用過iptables，這是一個長期存在的Linux防火牆，多年來為我們提供了良好的服務。但是您可能還不熟悉nftables，它是一個Linux系統的新工具，旨在為我們提供一些非常需要的升級，並最終取代年紀大的iptables。

iptables和nftables有什麼區別?


2. 為什麼要使用nftables而不是iptables?

nftables是由Netfilter開發的，該組織與當前維護iptables的組織相同。它的創建是為了解決iptables的問題，即可伸縮性和性能。

除了新的語法和一些升級之外，您還會發現它的功能與以前的版本非常相似。

新推出的nftables的另一個理由是，iptables框架已經變得有點複雜，iptables、ip6tables、arptables和ebtables都提供了不同但類似的功能。

例如，在iptables中創建IPv4規則和在ip6tables中創建IPv6規則並保持二者同步是非常低效的。nftables的目標是取代所有這些，成為一個集中的解決方案。

儘管nftables從2014年開始就被包含在Linux內核中，但是隨著採用範圍的擴大，它最近獲得了更多的關注。在Linux世界中，變化是緩慢的，過時的實用程序通常需要幾年甚至更長時間才能被淘汰，取而代之的是升級版。

nftables正在成為推薦的防火牆，Linux管理員有必要更新它們的工具列表。現在是學習nftables和更新現有iptables配置的好時機。

如果您已經使用iptables很多年了，並且對於必須學習一種全新的實用程序這個想法不是很興奮，不要擔心，我們已經在本指南中介紹了。

為什麼要使用nftables而不是iptables?


3. nftable中的鏈(chains)和規則(rules)

在iptables中，有三個默認的鏈:輸入(input)、輸出(output)和轉發(forward)。這三個“鏈”(以及其他鏈，如果您配置了任何鏈)包含“rules/規則”，iptables通過將網絡流量與鏈中的規則列表匹配來工作。如果所檢查的流量與任何規則都不匹配，則鏈的默認策略將用於流量，即接受(ACCEPT), DROP(丟棄)。

nftables的工作原理與此類似，也有“鏈”和“規則”。但是，它沒有從任何基礎鏈開始，這使得配置更加靈活。

iptables的一個低效之處是，所有網絡數據都必須通過上述一個或多個鏈，即使流量與任何規則都不匹配。無論是否配置了鏈，iptables仍然會根據這些鏈檢查網絡數據。

4. 在Linux上安裝`nftables`

nftables在所有主要的Linux發行版中都可用，您可以使用發行版的包管理器輕鬆安裝它。

在Ubuntu或基於debian的發行版上，你可以使用以下命令:

sudo apt install nftables

確保nftables在系統重新啟動時自動啟動:

sudo systemctl enable nftables.service

5. `iptables`和`nftable`之間的語法差異

nftables的語法與iptables不同，也簡單得多。說實話，iptables語法總是不清楚，需要額外的學習。幸運的是，對於那些從iptables遷移過來的人來說，nftables仍然接受舊的語法。

Linux iptables防火牆


您還可以使用iptables-translate實用程序，它將接受iptables命令並將它們轉換為nftables。這是查看這兩種語法差異的簡單方法。

使用以下命令在Ubuntu和基於debian的發行版上安裝iptables-translate:

sudo apt install iptables-nftables-compat

安裝之後，可以將iptables語法傳遞給iptables-translate命令，它將返回nftables等效的命令。

讓我們來看一些示例，以便您可以瞭解這些命令之間的不同之處。

5.1 阻塞入方向的流量

這個命令會阻止來自IP地址192.168.2.1的連接:

iptables-translate -A INPUT -s 192.168.2.1 -j DROP
nft add rule ip filter INPUT ip saddr 192.168.2.1 counter drop

5.2 允許入口方向SSH連接

讓我們來看看更多的例子—在加固Linux服務器時，您通常會發現自己在iptables中鍵入的一些常見內容。

iptables-translate -A INPUT -p tcp --dport 22 -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT
nft add rule ip filter INPUT tcp dport 22 ct state new,established counter accept

5.3 允許來自特定IP範圍的SSH連接

如果你想允許192.168.1.0/24的SSH連接:

iptables-translate -A INPUT -p tcp -s 192.168.1.0/24 --dport 22 -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT
nft add rule ip filter INPUT ip saddr 192.168.1.0/24 tcp dport 22 ct state new,established counter accept

5.4 允許MySQL連接到eth0網絡接口

以下是iptables和nftables的語法:

iptables-translate -A INPUT -i eth0 -p tcp --dport 3306 -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT
nft add rule ip filter INPUT iifname eth0 tcp dport 3306 ct state new,established counter accept

5.5 允許入口方向的HTTP和HTTPS流量

為了允許特定類型的流量，下面是這兩個命令的語法:

iptables-translate -A INPUT -p tcp -m multiport --dports 80,443 -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT
nft add rule ip filter INPUT ip protocol tcp tcp dport { 80,443} ct state new,established counter accept

從這些示例中可以看到，語法仍然與iptables非常相似，但是命令更直觀一些。

使用nftables作日誌記錄上面的nft命令示例中的counter選項告訴nftables計算一個規則被觸發的次數，就像iptables在默認情況下所做的那樣。

在nftables中，它們是可選的，必須指定它們。

nft add rule ip filter INPUT ip saddr 192.168.2.1 counter accept

nftables內置了用於導出配置的選項。它目前支持XML和JSON。

nft export xml

6. 結論

在本文中，我解釋了為什麼推薦nftables是Linux防火牆的新選擇。我還列出了舊的iptables和新nftables之間的許多不同之處，包括它們的功能和語法。

本指南向您展示了為什麼要考慮升級到nftables，以及如何開始使用您需要熟悉的新語法，以便成功升級舊的iptables規則。
