關於位圖畫刷的其他兩種用法

2020-10-29
| 科技

位圖畫刷(是的，我沒說錯)

位圖畫刷通常是那些 8像素 * 8像素的單色圖案，你可以在這些圖案上進行重新創作，又或者使用一些你認為比較複雜的圖案，例如犬牙花紋(houndstooth)。實際上，除了這些，還有一些其他的妙用值得嘗試。

CreatePatternBrush這個API函數會接受一箇舊的位圖並基於這個位圖創建一個畫刷，即使這個位圖的尺寸十分巨大，也同樣如此。

這個位圖將會自動平鋪，因此，這是一個位圖平鋪的的快速方法。這意味著：GDI將會為你做底層的數學計算。

當你正苦於嘗試將一個HBITMAP句柄當做HBRUSH來傳遞的時候，上面所提到的就是一個絕佳的解決方法：根據位圖創建畫刷並返回畫刷的句柄。

舉個例子

在這個例子中，我們將從頭開始編寫一個示例程序，它能通過位圖畫刷實現一個自定義的位圖平鋪背景。

在程序的入口點WinMain函數中，我們使用如下的替代調用：if (!InitApp(lpCmdLine)) return 0;

在這裡，我們在命令行上傳遞了一個指向一個位圖(BMP)文件的路徑，在此之後，奇蹟就出現了：窗口的背景會顯示出平鋪的位圖。請注意了，在這個過程中，我們僅僅是修改了窗口類的註冊部分的代碼，其他的地方我們可一點兒也沒動。

一切就是這樣的絲般柔順，不是嗎？

還有另一個用法

下面的程序是在上面的基礎程序上面修改而成，在這個程序中，我們可以通過使用位圖畫刷來簡化相當一大部分的工作。

在WinMain入口點中，我們在CreateWindowEx函數調用之前添加下面的代碼：g_hbr = CreatePatternBrushFromFile(lpCmdLine);if (!g_hbr) return 0;

這一次，因為我們自己負責管理(而不是窗口類)畫刷，所以別忘在WinMain返回之前釋放畫刷資源：DeleteObject(g_hbr);

這個程序將會繪製一個橢圓，並使用我們創建的位圖來進行填充。FillPath函數將使用當前選入到設備上下文中的畫刷，所以在FillPath之前，我們需要將我們的位圖畫刷選入到設備上下文中，然後才進行繪製。繪製的結果就是：你得到了一個使用位圖填充的橢圓。

換句話說，如果你不使用這個技巧，你將會需要自己實現一大堆的細節，例如位圖裁切，平鋪等。那樣的話，就只能祝你好運了。

總結

今天我們兩種不同的場景下使用了位圖畫刷，你預想中的髒活累活，那是一點都沒有接觸到（Windows為你做了）：

1) 使用指定的位圖來繪製窗口的背景色並自動平鋪。

2) 使用位圖來填充GDI繪製的各種圖形。

是時候GET這個技巧了，老夥計。
