內存尋址

1.邏輯地址--->線性地址--->物理地址，線性地址是指一段連續的、不分段的、範圍從0-4G的地址空間

16位段寄存器中存放的不是某個段的基地址，而是某個段的選擇符（選擇符=索引號+TI+RPL），所以段寄存器=端的選擇符；；16位的段寄存器無法存放32的段基地址，所以段基地址存放到段描述符表中

  • 邏輯地址（虛地址）=段標識符：指定段內相對地址的偏移量

  • MMU把邏輯地址轉換成物理地址

  • 分段基址（邏輯地址轉換成線性地址）

    段：虛擬地址空間的基本單位

    總結：邏輯地址=段選擇符：偏移量----->16位段寄存器=段選擇符（索引號：TI：RPL）------->段描述符表------->段描述符------>32位的段基址、段的界限、------->基址和ESI，EDI等中的32有效地址相加形成32位線性地址

  • 分頁機制（線性地址轉換成物理地址）

    頁表：線性地址映射到物理地址的一中數據結構

    頁（page，線性）：將線性地址劃分成若干大小相等的片，叫做頁，大小為4K

    塊或頁面（page frame 物理）：把物理地址空間分成與頁大小相等的若干存儲塊

    4G的線性地址=1M個*4K大小的頁，每個頁表項佔4個字節；1M個頁表需要4M空間（1M*4個字節）；4M的頁表再次分頁（4M/4K）可以分成1K個頁，同樣每個頁表項佔4個字節，可以推算出頁目錄佔1K*4個字節=4K，正好是一個頁的也目錄。這就是二級分頁。

進程

Linux為每個進程都創建一個task_struct結構

  • init進程：內核啟動的第一個進程，稱為0號進程。init進程還負責“收養”孤兒進程。

  • 進程描述符：是一個task_struct結構，用於存放進程的屬性和信息，與進程所有相關的內核信息都存儲在這個結構體中。進程描述符存放在動態內存中，而不是存放在永久分配給內核的內存區。

  • fork後的子進程和父進程有什麼區別：

    在fork之後exec之前兩個進程用的是相同的物理空間（內存區）， 子進程的代碼段、數據段、堆棧都是指向父進程的物理空間，也就是說，兩者的虛擬空間不同，但其對應的物理空間是同一個。當父子進程中有更改相應段的行為發生時，再為子進程相應的段分配物理空間，如果不是因為exec，內核會給子進程的數據段、堆棧段分配相應的物理空間（至此父進程子進程有各自的進程空間，互不影響），而代碼段繼續共享父進程的物理空間（兩者的代碼完全相同）。而如果是因為exec，由於兩者執行的代碼不同，子進程的代碼段也會分配單獨的物理空間。

父進程和子進程區別

進程和線程區別：

  • 進程是操作系統分配資源的最小單位；線程是操作系統調度執行的最小單位，是幹活的。

    進程有自己的地址空間，線程使用進程的地址空間，進程的資源（堆，棧，靜態存儲區等）線程是有權訪問的，但每個線程還擁有自己的棧，這個棧仍然使用進程的地址空間，這個棧是不可以被其他的線程所訪問的。（第一個是進程的環境示意圖，第二個是線程的環境示意圖

進程和線程區別

內存四區

  • 棧區（stack）：存放函數的參數值，局部變量的值，

  • 堆區（heap）：動態分配的值

  • 全局區/靜態區（static）/常量區：初始化的全局變量，靜態變量。包含常量區（常量字符串就放在這裡，結束後由系統釋放）

  • 代碼區（code）：存放二級制代碼

內存四區

char*fa()

{

char*pa = "123456"; //pa指針在棧區，“123456”在常量區，該函數調用完後指針變量pa就被釋放了

char*p = NULL; //指針變量p在棧中分配4字節

p=(char*)malloc(100); //本函數在這裡開闢了一塊堆區的內存空間，並把地址賦值給p

strcpy(p, "hello world 1234566"); //把常量區的字符串拷貝到堆區

return p; //返回給主調函數fb()，相對fa來說fb是主調函數，相對main來說，fa(),fb()都是被調用函數

}

char*fb()

{

char*pstr = NULL;

pstr = fa();

return pstr; //指針變量pstr在這就結束

}

void main()

{

char*str = NULL;

str = fb();

printf("str = %s\n",str);

free(str); //防止內存洩露，被調函數fa()分配的內存存的值通過返回值傳給主調函數，然後主調函數釋放內存

str = NULL; //防止產生野指針

system("pause");

}
