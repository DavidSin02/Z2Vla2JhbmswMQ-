IoC部分將會講解有關Bean裝配的相關知識，比較零散，主要分三大塊：

  • Bean裝配：對象的創建，不涉及過程也就是屬性初始化。
  • 基於XML的DI：主要講解Bean的初始化配置，也就是屬性的初始化配置。
  • 基於註解的DI：將XML配置轉為註解配置。

在IoC部分結束後，會做相關總結，大家在學習的過程中，也可以試著做些總結。

1.靜態工廠

上一章節我們講解了動態工廠的簡單配置，本節將簡單介紹靜態工廠的配置。那什麼是靜態工廠呢？

靜態工廠其實也就是類的創建模式，又叫做簡單方法模式。不考慮其它，只考慮靜態工廠類的定義，即使用靜態方法來對外提供目標對象的實例，也就是在動態工廠類中的方法上加上static關鍵字。

2.靜態工廠的配置

首先我們要知道，靜態工廠獲取Bean的實例是直接調用靜態方法返回的，也就是說靜態工廠不需要實例化。在Spring中可以通過<bean/>標籤屬性class(值為靜態工廠類的全限定性類名)和factory-method(值為靜態工廠中獲取指定Bean實例的方法名)進行靜態工廠的配置。

  • 靜態工廠：其實就是獲取目標Bean的方法是靜態的
/** * 靜態工廠類 * 其實就是獲取/** * 靜態工廠類 * 其實就是獲取實例的方法是靜態的 */public class MyBeanFactory{    /**     * 獲取String對象     * @return     */    public static String getStringObj(){        return "String 對象";    }    /**     * 獲取Integer對象     * @return     */    public static Integer getIntegerObj(){        return Integer.MAX_VALUE;    }}的方法是靜態的 */public class MyBeanFactory{    /**     * 獲取String對象     * @return     */    public static String getStringObj(){        return "String 對象";    }    /**     * 獲取Integer對象     * @return     */    public static Integer getIntegerObj(){        return Integer.MAX_VALUE;    }}
  • xml配置
<!-- 1.靜態工廠配置 --><bean id="stringObj" class="com.spring.bean.MyBeanFactory" factory-method="getStringObj"/><bean id="integerObj" class="com.spring.bean.MyBeanFactory" factory-method="getIntegerObj"/>
  • 測試及結果
@Testpublic void test1(){    ClassPathXmlApplicationContext application = new ClassPathXmlApplicationContext("applicationContext.xml");    //2.使用Spring提供的動態工廠配置    String stringObj = (String)application.getBean("stringObj");    System.out.println(stringObj);    Integer integerObj = (Integer)application.getBean("integerObj");    System.out.println(integerObj);    application.close();}--測試結果String 對象2147483647

4.靜態和動態工廠Bean的配置總結

靜態工廠中的方法是static方法，動態工廠中的是實例方法。

  • 動態工廠Bean，xml配置的兩種方法(建議第二種)
<!-- 1.將動態工廠作為普通Bean配置 --><bean id="factory1" class="com.spring.bean.MyBeanFactory"/><!-- 2.使用Spring提供的動態工廠配置 --><bean id="factory2" class="com.spring.bean.MyBeanFactory"/><bean id="stringObj" factory-bean="factory2" factory-method="getStringObj"/><bean id="integerObj" factory-bean="factory2" factory-method="getIntegerObj"/>
  • 靜態工廠Bean的xml配置
