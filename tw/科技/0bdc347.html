走入計算機世界-計算機系統的組成

大家熟悉的計算機系統是由軟件和硬件組合而成，可以按照功能換分為多級層次結構。

硬件通常指一切可以看得見，摸的著的設備實體。原始的馮諾依曼（VonNeumann）計算機在結構上是以運算器為中心，現今轉向以存儲為中心了。如下圖最基本的計算機結構圖：

計算機各個部件之間的合作關係

  • 控制器：是分析和執行指令的部件，也是統一指揮並控制計算機各個部件協調工作的中心部件，所依據的是機器指令。控制器組成包含如下：

    程序計數器PC：存儲下一條要執行指令的地址；

    指令寄存器IR：存儲即將執行的指令；

    指令譯碼器ID：對指令中的操作碼字段進行分析解釋；

    時序部件：提供時序控制信號。

  • 運算器：也稱為算數邏輯單元，主要功能是在控制器的控制下完成各種算數運算和邏輯運算。運算器的組成如下：

    算數邏輯單元ALU：數據的算數運算和邏輯運算；

    累加寄存器AC：通用寄存器，為ALU提供一個工作區，用於暫存數據；

    數據緩衝寄存器DR：寫內存時，暫存指令或數據；

    狀態條件寄存器PSW：存狀態標誌和控制標誌。

  • 主存儲器：也稱內存存儲，用於存儲現場操作的信息與中間結果，包括機器指令和數據。

  • 輔助存儲器：也稱為外存儲器。用於存儲需要長時間保存的各種信息。

  • 輸入設備：把人們編好的程序和原始數據送到計算機中去，並且將它們轉換為計算機內部能夠識別和接受的信息方式。按輸入信息的形態可以分為字符輸入、圖形輸入、圖像輸入和語音輸入等。目前，常見的輸入設備有鼠標、鍵盤、掃描儀等。

  • 輸出設備：將計算機的處理結果以人或者其他設備所能接受的形式送出計算機。目前，最常見的輸出設備有顯示器和打印機。有些設備既可以是輸入設備，也可以是輸出設備，如自動控制、輔助存儲器和檢測系統中使用的數模轉換裝置等。

複雜指令集系統CISC和精簡指令集系統RISC

在計算機系統結構發展過程中，指令系統的優化設計有著兩個截然相反的反向，一個是（Complex Instruction Set Computer,CISC）增強指令功能，設置一些功能複雜的指令，把一些由軟件實現的、常用的功能改用硬件的指令系統來實現；另一個是（Reduced Instruction Set Computer,RISS）儘量簡化指令功能，只保留那些功能簡單，能在一個節拍內完成的指令，較複雜的功能用一段子程序來實現。

CISC指令系統特點：

  1. 指令數量眾多，指令系統擁有大量指令，通常100~250條；

  2. 指令的使用頻率相差懸殊。最常使用的指令是一些比較簡單的指令，僅佔指令總數的20%，但在程序中使用的頻率是80%，而大部分複雜的指令卻很少使用；

  3. 支持多種尋址方式，支持的尋址方式通常有5~20種；

  4. 變長的指令，指令的長度不固定，變長的指令增加指令譯碼電路的複雜性；

  5. 指令可以對主存單元中的數據進行處理，其執行速度較慢；

  6. 以微程序控制為主，CISC的指令系統很複雜，難以用硬佈線邏輯（組合邏輯）電路實現控制器，通常採用微程序控制。

RISC指令系統特點：

  1. 指令數量少。優先選用使用頻率最高的一些簡單指令和一些常用指令。避免使用複雜指令。只提供了從存儲器中讀數（LOAD）和把數據寫入存儲器（STORE）兩條指令對存儲器進行操作，其餘所有操作都在CPU的寄存器之間進行；

  2. 指令的尋址方式少。通常只支持寄存器尋址、立即數尋址和相對尋址方式；、

  3. 指令長度固定，指令格式種類少；

  4. 以硬佈線邏輯控制為主。提高操作的執行速度；

  5. 單週期指令執行，採用流水線技術；

  6. CPU中的通用寄存器數量多，一般32個以上，有的上千個。

大多數RISC採用Cache方案，使用Cache來提高讀取指令的速度。

總線

是一組能為多個部件分時共享的公共信息傳送線路。共享是指總線上可以接掛多個部件，各個部件之間相互交換信息都可以通過這組公共線路傳送；分時是指同一時刻只允許有一個部件向總線發送信息，如果出現兩個或兩個以上部件同時向總線發送信息，就會導致信號衝突。當然，允許多個部件同時從總線上接受相同的信息。

按總線功能劃分：地址總線，數據總線，控制總線三類。

  • 地址總線：用來傳輸地址信息；

  • 數據總線：用來傳輸數據信息；

  • 控制總線：用來控制傳送各種控制信號。
