題目描述

在數組中的兩個數字，如果前面一個數字大於後面的數字，則這兩個數字組成一個逆序對。輸入一個數組，求出這個數組中的逆序對的總數。

0 <= 數組長度 <= 50000

題目樣例

示例

  • 輸入: [7,5,6,4]
  • 輸出: 5

題目思考

  1. 如何批量得到逆序對?

解決方案

思路

  • 一個比較容易想到的思路是雙重循環, 暴力求逆序對, 但這樣時間複雜度是 O(N^2), 根據數據規模肯定超時, 如何進行優化呢?
  • 如果我們能將問題進行分解, 先求出數組左右兩部分的逆序對數目, 最後再一次遍歷將分屬兩部分的逆序對(也即第一個數在左半部分, 第二個數在右半部分)也求出來並累加在一起, 這樣就能優化時間複雜度到 O(NlogN), 這就是分治法的思想
  • 此時關鍵在於如何一次遍歷就求得分屬兩部分的逆序對呢? 如果左右兩部分是有序的就很好辦了, 可以利用歸併排序的思路, 雙指針遍歷, 哪邊小就把哪邊加入結果數組中. 如果是右側小的話就意味著左側剩餘的數字都和當前右側數字構成逆序對, 將對應的數目加入結果中即可.
  • 而左右部分有序正好也是歸併排序的結果, 所以整個思路就是在歸併排序的基礎上加上逆序對的統計
  • 下面代碼對必要的步驟有詳細的解釋, 方便大家理解

複雜度

  • 時間複雜度 O(NlogN): 使用了歸併排序, 時間複雜度是(N+(N/2)*2+(N/4)*4+...+1*N = NlogN, 因為一共 logN 個乘積)
  • 空間複雜度 O(N): 歸併排序需要用到臨時數組, 長度為 N

代碼

class Solution:    def reversePairs(self, nums: List[int]) -> int:        self.res = 0        def mergesort(s, e):            if s >= e:                return            m = (s + e) >> 1            # 先排序並統計左右部分            mergesort(s, m)            mergesort(m + 1, e)            # 使用臨時數組存儲排序好的左右部分            # 注意這裡選擇逆序存儲, 是因為可以利用O(1)的pop操作得到最小的元素            # 當然這裡也可以使用雙端隊列, 這樣就無需逆序了            left = nums[s:m + 1][::-1]            right = nums[m + 1:e + 1][::-1]            for i in range(s, e + 1):                if not right or left and left[-1] <= right[-1]:                    # 當前左側數字更小或者右側數字已經用完, 此時不構成逆序對                    nums[i] = left.pop()                else:                    # 當前右側數字更小, 和剩餘的左側部分都構成逆序對                    self.res += len(left)                    nums[i] = right.pop()        mergesort(0, len(nums) - 1)        return self.res
