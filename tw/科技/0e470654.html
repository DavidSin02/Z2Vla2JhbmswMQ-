企業實施整合管理體系的現狀與建議

2020-10-29
| 科技

近10年以來，隨著ISO9000質量管理標準宣貫工作的深入開展，越來越多的企業逐漸意識到管理體系的重要性，隨之也陸續建立和運行ISO14000環境管理體系，以及OHSAS18000職業安全健康管理體系（現已替代為ISO45001），這些標準管理體系的實施，在很大程度上，幫助企業建立了一套結構化，科學化的系統管方法，在企業日常經營管理的各項活動中，起到了非常積極的作用。


然而，隨著管理體系推行的逐步深入，以及越來越多的政府監管（比如企業標準化建設）或者客戶特定的業務要求，比如企業供應鏈上游客戶的一些特定要求，很多企業實施的管理體系越來越多，內部負責推進各管理體系的團隊也越來越大，慢慢就出現了一個不得不重視的問題：資源的浪費以及效率的低下。筆者在2006年在珠三角加入一家美資電子代工巨頭，曾親身經歷過公司內推行多管理體系的紛亂局面，質量管理體系ISO9000，環境管理體系ISO4000，職業健康安全管理體系（當時還是OHSAS18000），後來因為幾個大客戶如戴爾，微軟，惠普等組成了一個聯盟，制定了電子行業行為準則 （EICC，Electronic Industry Code of Conduct）,要求其供應鏈上的下游廠商必須遵守，再後來，戴爾又進一步推出Dell Global Citizenship（一個關注可持續發展的全球性項目），由於這些都是圍繞國際勞工組織的 SA8000 體系要求，或者是基於該體系衍生出的企業規定，為了業務的可持續性，我們又開始建立實施 SA8000管理體系，每年企業要接待不同的外部審核機構開展不同層次的審核活動，加上內部又分別由不同的團隊人員負責不同體系的實施工作，這裡面的溝通成本，實在是一筆巨大的負擔。


因其種種，推進“整合管理體系”的呼聲越來越高，尤其是最近幾年，在一些已經實施了幾個不同管理體系的企業裡，幾乎都開始管理體系的整合工作。以ISO9000，ISO4000和OHSAS18000 為例，三個體系的整合，固然有其先天的基礎，這是因為：

1. 三個體系的原生內核是相同的，都是基於PDCA的戴明循環，採用了相同的管理體系框架，其核心內容都是根據管理學原理，為組織建立了一個動態循環的管理模型，以持續改進的思想指導組織系統地實現其既定的目標；

2. 從體系的輸入/輸出兩端，三者也高度類似。比如：ISO9000質量管理體系的輸入端是客戶需求，輸出端是有形的產品或無形的客戶滿意度，而ISO4001和OHSAS8001，輸入端是相關方要求或關注，輸出端是良好的環境或職業健康安全績效（實施上也是滿足相關方要求，與質量管理體系的客戶滿意度異曲同工。

3. 三個標準體現相同的管理原則。三個標準的內容都體現了“領導作用”、“全員參與”、“過程方法”、“管理的系統方法”、“持續改進”、“基於事實的決策方法”、“互利的供方關係”等管理原則。

4. 三個標準的管理性要求有很多相似之處。三個標準的管理性要求有很多相似的部分。如“文件控制”、“記錄控制”、“方針目標”、“組織結構和職責”、“培訓、意識和能力”、“資源管理”（如設備管理）、“法律法規要求”、“交流與溝通”、“監視和測量裝置的控制”、“糾正措施”、“預防措施”、“內部審核”、“管理評審”等方面。

5. 三個標準要求的管理手段和方法一致。都採取系統的方法，建立一個完整的、有效的、文件化的管理體系；都按PDCA循環的思想，通過識別影響質量、環境、職業安全健康的因素，有針對性地制定計劃和管理方案，實施運行控制，並採取必要的監視和測量，發現問題，實施改進，實現管理體系的持續有效運行；都要求組織配備適當的資源；都要求通過日常管理的監視和測量、內審、管理評審等管理手段來評價體系的運行狀況。

（ISO9000，ISO4000和OHSA18000三體系要素對比圖， 圖片來源網絡搜索，如有錯漏之處，敬請諒解）

儘管如此，我們仍然注意到，還有為數眾多的企業，在實施整合管理體系的過程中，出現了很多的問題

，尤其是一些中小企業，與一些大公司相比，因其企業規模相對較小，資源配置（包括對管理體系有深刻理解的人員）也不是完全充分，在體系實施過程中的痛點尤為突出。具體表現為：

1. 整合效果不明顯，原來是3個管理體系，實施整合後並沒有降低工作量或精簡了管理流程。相反，因為要推進體系整合，有些部門感覺工作量加大了。這就沒有起到整合體系，提高效率的目的。

2. 體系文件整合了，但實施起來更不能落地了。為了整合而整合，體系文件成了抽屜裡的一沓紙，只有在審核的時候才“重見天日”。

3. 整合後體系管理“兩張皮”現象嚴重，體系文件描述的是一回事，實際運行情況是另一回事。這在一些同時推進標準化建設的企業裡尤其突出。一套體系文件用於應付檢查，一套管理程序用於指導日常工作。試想，這到底是“減負”還是“增負”？

4. 錯把三個管理體系的同時審核/認證，就認為是已經實施了“整合”的管理體系。事實上這其實只是“整合”了體系的認證機構，某種程度上可能只是為企業節約了一些認證成本而已。

事實上，整合管理體系的實施是公認的優化企業內部資源，建立良好運行機制，通過PDCA循環來持續改善企業管理水平的最佳工具之一。國際上很多著名的企業，如杜邦，殼牌，國內的中石油，中石化等都在探索建立企業自己的整合管理體系。筆者結合自身多年體系管理實踐經驗，總結得失，針對上述體系整合的痛點難點，提出下述改善建議：

1. 立足企業自身實際，打造“量體裁衣”的整合管理體系。當我們談“整合管理體系”時，並不是要企業去創建一個“獨一無二”的新的管理體系，而是要立足企業自身實際狀況，依託現有資源，根據已經實施的管理體系要求，識別不同的體系要求中重複的管理流程，比如文件控制，內審，管理評審等等，從流程優化的角度去做精益，達到精簡流程，提高效率的目的，這也是企業實施管理體系的初心。

2. 警惕“整合管理體系”等同於“整合程序文件”。誠然，程序文件是管理體系的重要組成部分，管理體系的要求是通過程序文件這一載體來傳達和實施，但一定要注意，整合管理體系並不等同於整合程序文件。我們建立實施管理體系的目的是為了幫助企業優化資源，規範管理，達成業務目標。任何的管理措施都要跟這個目的相一致。千萬不要因為要“整合管理體系”而再多付出其他的管理資源（人力/物力），這就跟建立實施管理體系的目的相違背了。

3. 重點關注整合管理體系中具有共性的，普適性的要求，從“不同中找相同”。前文提到，例舉的三個管理體系，都是採用過程管理/控制的方法，通過建立目標，制定流程，監測進度，實施改進來持續提高管理績效。那麼，三個體系中與這一過程相對應的管理要素，比如目標指標的設定，監測測量，檢查改進（包括內審和管理評審）等程序就可以進行合理的整合。

4. 做減法，不做加法！Less is More，儘量保持整合管理體系簡單化，簡單意味著容易被理解，容易被實施。企業在實施整合管理體系的時候，一定要避免走入一個誤區，追求一個大而全，包羅萬象的龐然大物。實施上，從體系要求落地的角度，多來來的實踐早已證明，大則虛空，小而彌堅。

隨著我國進入到“十四五“以及”中國智造2025“的戰略實施，各行各業的發展已經日新月異，行業之間的良性競爭也愈發激烈。任何一家企業，都必須切實提高自身管理水平，“向管理要紅利“才能在競爭的浪潮中搶得一席之地。相信”整合管理體系“在未來的很長一段時間裡也還會是熱點詞彙，而管理體系整合的理念也必將隨著實踐發展到另一個高度。我們也希望所有的企業能利用好這套工具，勤練內功，科學發展。期待中國企業管理百花齊放，百家爭鳴的那一天早日到來！
