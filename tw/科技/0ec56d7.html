工作表數據與UserForm窗口交互工程的設計總結

2020-10-29
| 科技

大家好，我們今天繼續講解VBA數據庫解決方案，今日講解的是第84講:工作表數據與UserForm窗口的交互工程設計總結。這一講是講解的窗體設計的最後一講，做個總結，把需要注意的問題和大家再講一下，同時這講內容也是VBA數據庫解決方案的最後一講內容。

一 在實現窗口的交互工程的設計的時候要考慮到具體應用。

我在第77講到83講的講解中，利用了Application.Caller 來傳遞我的需求，如下截圖

在我的窗體中共用7大需求，分別對應著不同的Application.Caller值，這樣我可以在加載UserForm窗口時可以精準實現我的控制。如下面的刪除代碼傳遞的Application.Caller是8

If Right(Application.Caller, 1) = 8 Then '刪除

' UserForm1.CommandButton3.Enabled = False '開始

UserForm1.CommandButton1.Enabled = False '下一條記錄

UserForm1.CommandButton4.Enabled = False '最後一條記錄

UserForm1.CommandButton5.Enabled = False '編輯記錄

UserForm1.CommandButton7.Enabled = False '查找記錄

UserForm1.CommandButton8.Enabled = False '刪除記錄

UserForm1.CommandButton6.Enabled = False '保存記錄

UserForm1.CommandButton9.Enabled = False '錄入記錄

UserForm1.TextBox1.Enabled = False

UserForm1.TextBox2.Enabled = False

UserForm1.TextBox3.Enabled = False

End If

上述代碼就是實現了在加載窗體時，只有"開始"按鈕可以響應代碼，耳其他的十項內容是不可以響應代碼的。

二 在實現窗口的交互工程的設計的時候窗體的設計和代碼的設計要同時進行

為了能讓代碼控制窗體時達到最友好的界面，讓用戶有友好的體驗，我們要對代碼和窗體同步的設計，不要先涉及完窗體，再寫代碼，這樣往往會造成在寫代碼時發現窗體設計的不合理後修改麻煩。更不能在寫完代碼後再佈局窗體。而是應該按照一步一個腳印的辦法同步完成窗體的佈局和代碼的書寫。

1.寫一段代碼時，可能用到多個控件，那麼我就先把這些控件先佈置好；

2.在佈置一個控件時往往會用到其他的控件，那麼我們就一併佈置好。

3.佈置控件和寫代碼可以同步進行，邊寫代碼邊佈置控件

如我在寫上述工程的保存按鈕代碼時會遇到錄入保存和修改保存兩種情況，那麼我要合併來考慮，不要分開寫；

Private Sub CommandButton6_Click() '保存

If UserForm1.TextBox1.Value = "" Or UserForm1.TextBox2.Value = "" Or UserForm1.TextBox3.Value = "" Then MsgBox "信息有空值，請確認！": Exit Sub

If MsgBox("是否要保存記錄？", vbOKCancel, "提示") = vbCancel Then Exit Sub

Dim cnADO, rsADO As Object

Dim strPath, strSQL As String

Dim myData() As Variant

Set cnADO = CreateObject("ADODB.Connection")

Set rsADO = CreateObject("ADODB.Recordset")

strPath = ThisWorkbook.FullName

cnADO.Open "provider=Microsoft.ACE.OLEDB.12.0;extended properties='excel 12.0;hdr=yes;imex=0';" _

& "data source=" & strPath

strSQL = "SELECT * FROM [數據7$]"

rsADO.Open strSQL, cnADO, 1, 3

If UserForm1.TextBox1.Enabled = False Then '編輯的保存

If rsADO.RecordCount > 0 Then rsADO.MoveFirst

Do While Not rsADO.EOF

If Trim(rsADO.Fields(0)) = UserForm1.TextBox1.Value Then

rsADO.Fields(1) = UserForm1.TextBox2.Value

rsADO.Fields(2) = UserForm1.TextBox3.Value

rsADO.Update

GoTo 100

End If

rsADO.MoveNext

Loop

100:

UserForm1.TextBox1.Enabled = False

UserForm1.TextBox2.Enabled = False

UserForm1.TextBox3.Enabled = False

UserForm1.CommandButton6.Enabled = False

MsgBox ("保存OK！")

Else '錄入的保存

If rsADO.RecordCount > 0 Then

Do While Not rsADO.EOF

If Trim(rsADO.Fields(0)) = UserForm1.TextBox1.Value Then MsgBox "員工編號重複，請確認！": GoTo 110

rsADO.MoveNext

Loop

End If

rsADO.AddNew

rsADO.Fields(0) = UserForm1.TextBox1.Value

rsADO.Fields(1) = UserForm1.TextBox2.Value

rsADO.Fields(2) = UserForm1.TextBox3.Value

rsADO.Update

110:

UserForm1.TextBox1.Value = ""

UserForm1.TextBox2.Value = ""

UserForm1.TextBox3.Value = ""

UserForm1.TextBox1.SetFocus

MsgBox ("保存OK！")

End If

rsADO.Close

cnADO.Close

Set rsADO = Nothing

Set cnADO = Nothing

End Sub

上述代碼中在打開數據庫的連接設置時是一樣的，我就用同樣的連接來進行。

三 代碼要採用自己熟悉的方式寫法進行。

實現一個目的有很多方法，寫代碼也是一樣，實現一個功能，有很多代碼的寫法，有的人喜歡用這種，有的人喜歡有那種，無可厚非。大家要注意VBA的定位，我也多次講過，對於我們日常工作的OFICE內容，運行時間是可以不用作為第一要素來考慮的。往往，我們更注重的是和手工操作的比較。

所謂採用自己熟悉的方式，就是在我的書籍中經常提到的搭積木的方法，羅列你的代碼到合適的位置，然後再修正它。

四 完成自己的工程後要做細緻的調試。這種調試包括：

1 通用數據的測試。

2 首尾數據的測試。

3 異常數據的測試。

所謂通用數據就是一般正常的數據實現，首尾數據的測試為了避免由於變量設置失誤而造成數據的丟失。異常數據是指通常的"坑"，自己給自己挖個坑，然後自己跳下去，也未嘗不可，這樣可以更能檢測自己的程序運行情況。
