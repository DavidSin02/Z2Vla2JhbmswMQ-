利用樹莓派學習arm彙編 第一章

2020-11-14
| 科技


在我看來，學習一門高級編程語言比學習一門特定體系結構的彙編更有用，但是我很想學習ARM彙編程序只是為了好玩，因為我知道一些386彙編語言。這個想法不是想成為大師，而是想了解下面發生了什麼。

ARM簡介

下面的解釋不會力求面面俱到的講述arm的體系結構，我會盡量精簡講解其中實用的部分。

ARM是一種32位體系結構，具有一個簡單的目標：靈活性。儘管這對集成商非常有用（因為他們在設計硬件時有很大的自由度），但對於必須應對ARM硬件差異的系統開發人員來說卻不是那麼好。因此，在本文中，我將假設一切都在運行Raspbian的Raspberry Pi Model B上完成。

有些部分將是ARM通用的，而有些將是Raspberry Pi專用的。 我不會區分。 ARM網站上有很多文檔可查！

開始寫彙編

彙編語言只是二進制代碼之上的一個薄語法層。

二進制代碼是計算機可以運行的，它由以二進制表示形式編碼的指令組成（此類編碼已在ARM手冊中記錄）。您可以編寫二進制代碼編碼指令，但這會很麻煩（除了與Linux本身相關的其他一些技術，我們現在可以很高興地忽略它們）。

由於計算機無法運行彙編程序，因此我們必須從中獲取二進制代碼。 我們使用一種稱為彙編器的工具將彙編器代碼彙編成可以運行的二進制代碼。

在本課程中我們使用gnu assembler，這個工具是gnu project中的其中一個，有時又被稱為gas.

只需打開vim，nano或emacs之類的編輯器即可。 我們的彙編語言文件（稱為源文件）將帶有後綴.s。 我不知道為什麼是.s，但這是通常的慣例。

第一個程序

從一個簡單的程序開始，這個程序只有返回值，在其中不做任何事。

/* -- first.s *//* This is a comment */.global main /* 'main' is our entry point and must be global */main: /* This is main */mov r0, #2 /* Put a 2 inside the register r0 */bx lr /* Return from main */

創建一個文件，將上述內容保存其中，然後開始編譯，命令如下：
$ as -o first.o first.s
上述命令執行完後將會創建一個first.o的文件，下面我們將這個文件鏈接成為可執行文件
$ gcc -o first first.o
如果上述步驟執行順利，你將得到一個first的文件，這個就是你的程序，下面來執行吧。
$ ./first
它應該不會有任何反應，的確，有點小失望，但是這個程序是執行了的，讓我們看一下它的返回值

$ ./first ; echo $?2

太棒了，返回值2並非偶然，這是由於彙編代碼中的＃2引起的。
由於運行彙編器和鏈接器很快會變得很無聊，因此建議您使用以下Makefile文件或類似的文件。

# Makefileall: first first: first.o gcc -o $@ $+ first.o : first.s as -o $@ $< clean: rm -vf first *.o

OK,讓我們來解釋一下吧

我們作弊只是為了使事情變得容易一些。 我們在彙編器中編寫了一個C main函數，它僅返回2;。 這樣，我們的程序就更容易了，因為C運行時為我們處理了程序的初始化和終止。 我將一直使用這種方法。
讓我們回顧一下最小彙編文件的每一行。

/* -- first.s *//* This is a comment */

這些是評論。 註釋包含在/ 和 /中。 使用它們來記錄您的彙編器，因為它們會被忽略。 通常，不要將/ 和 /嵌套在/ *內，因為它不起作用。

.global main /* 'main' is our entry point and must be global */

這是GNU彙編程序的指令。 指令告訴GNU彙編器做一些特別的事情。 它們以點號（。）開頭，後跟指令名稱和一些參數。 在這種情況下，我們說main是一個全局名稱。 這是必需的，因為C運行時將調用main。 如果不是全局的，則C運行時將無法調用它，並且鏈接階段將失敗。

main: /* This is main */

GNU彙編程序中不是指令的每一行都將始終像label：指令。 我們可以省略label：和指令（忽略空行和空行）。 僅帶有標籤：的行將該標籤應用於下一行（您可以通過這種方式將多個標籤引用相同的內容）。 指令部分是ARM彙編語言本身。 在這種情況下，由於沒有指令，我們只是在定義main。

mov r0, #2 /* Put a 2 inside the register r0 */

在行的開頭，空格被忽略，但是縮進在視覺上暗示該指令屬於主要功能。
這是mov指令，表示移動。 我們將值2移至寄存器r0。 在下一章中，我們將瞭解有關寄存器的更多信息，現在不用擔心。 是的，語法很尷尬，因為目的地實際上在左邊。 在ARM語法中，它始終在左側，因此我們說的是將r0寄存到立即數2之類的操作。在下一章中，我們將瞭解ARM中立即數的含義，不用擔心。
總而言之，該指令將2放入寄存器r0中（這實際上會覆蓋此時的r0寄存器）。

bx lr /* Return from main */

該指令bx表示分支和交換。 在這一點上，我們實際上並不關心交換部分。 分支意味著我們將改變指令執行的流程。 ARM處理器一個接一個地依次運行指令，因此在上述動作之後，該bx將被運行（此順序執行並非特定於ARM，而是在幾乎所有架構中都會發生）。 分支指令用於更改此隱式順序執行。 在這種情況下，我們跳轉到lr寄存器說的內容。 我們現在不在乎lr包含什麼。 足以理解該指令只是離開了主要功能，從而有效地結束了我們的程序。
main的結果是程序的錯誤代碼，並且在離開函數時必須將結果存儲在寄存器r0中，因此我們main執行的mov指令實際上將錯誤代碼設置為2。
