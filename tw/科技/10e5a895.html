以下關於CPU的敘述中，錯誤的是(1) 。

(1)A.CPU產生每條指令的操作信號並將操作信號送往相應的部件進行控制

B.程序計數器PC除了存放指令地址，也可以臨時存儲算術/邏輯運算結果

C.CPU中的控制器決定計算機運行過程的自動化

D.指令譯碼器是CPU控制器中的部件

【答案】B

【解析】本題考查計算機硬件組成基礎知識。

CPU是計算機的控制中心，主要由運算器、控制器、寄存器組和內部總線等部件組成。控制器由程序計數器、指令寄存器、指令譯碼器、時序產生器和操作控制器組成，它是發佈命令的"決策機構"，即完成協調和指揮整個計算機系統的操作。它的主要功能：從內存中取出一條指令，並指出下一條指令在內存中的位置；對指令進行譯碼或測試，併產生相應的操作控制信號，以便啟動規定的動作；指揮並控制CPU、內存和輸入輸出設備之間數據的流動。

程序計數器(PC)是專用寄存器，具有寄存信息和計數兩種功能，又稱為指令計數器，在程序開始執行前，將程序的起始地址送入PC，該地址在程序加載到內存時確定，因此PC的初始內容即是程序第一條指令的地址。執行指令時，CPU將自動修改PC的內容，以便使其保持的總是將要執行的下一條指令的地址。由於大多數指令都是按順序執行的，因此修改的過程通常只是簡單地對PC加1。當遇到轉移指令時，後繼指令的地址根據當前指令的地址加上一個向前或向後轉移的位移量得到，或者根據轉移指令給出的直接轉移的地址得到。


以下關於 CISC (Complex Instruction Set Computer.複雜指令集計算機）和RISC (Reduced Instruction Set Computer,精簡指令集計算機）的敘述中，錯誤的是(2)。

(2)A.在CISC中，其複雜指令都採用硬佈線邏輯來執行

B.採用CISC技術的CPU，其芯片設計複雜度更高

C.在RISC中，更適合採用硬佈線邏輯執行指令

D.採用RISC技術，指令系統中的指令種類和尋址方式更少

【答案】A

【解析】本題考查指令系統和計算機體系結構基礎知識。

CISC (Complex Instruction Set Computer,複雜指令集計算機）的基本思想是：進一步增強原有指令的功能，用更為複雜的新指令取代原先由軟件子程序完成的功能，實現軟件功能的硬件化，導致機器的指令系統越來越龐大而複雜。CISC計算機一般所含的令數目至少300條以上，有的甚至超過500條。

RISC (Reduced Instruction Set Computer,精簡指令集計算機）的基本思想是：通過減少指令總數和簡化指令功能，降低硬件設計的複雜度，使指令能單週期執行，並通過優化編譯提高指令的執行速度，採用硬佈線控制邏輯優化編譯程序。在20世紀70年代末開始興起，導致機器的指令系統進一步精煉而簡單。


以下關於校驗碼的敘述中，正確的是(3)。

(3)A.海明碼利用多組數位的奇偶性來檢錯和糾錯

B.海明碼的碼距必須大於等於1

C.循環冗餘校驗碼具有很強的檢錯和糾錯能力

D.循環冗餘校驗碼的碼距必定為1

【答案】A

【解析】本題考查校驗碼的基礎知識。

一個編碼系統中任意兩個合法編碼（碼字）之間不同的二進數位數稱為這兩個碼字的碼距，而整個編碼系統中任意兩個碼字的最小距離就是該編碼系統的碼距。為了使一個系統能檢查和糾正一個差錯，碼間最小距離必須至少是3。

海明碼是一種可以糾正一位差錯的編碼，是利用奇偶性來檢錯和糾錯的校驗方法。海明碼的基本意思是給傳輸的數據增加r個校驗位，從而增加兩個合法消息（合法碼字) 的不同位的個數（海明距離）。假設要傳輸的信息有m位，則經海明編碼的碼字就有n=m+r位。

循環冗餘校驗碼（CRC)編碼方法是在k位信息碼後再拼接r位的校驗碼，形成長度為n位的編碼，其特點是檢錯能力極強且開銷小，易於用編碼器及檢測電路實現。

在數據通信與網絡中，通常k相當大，由一千甚至數千數據位構成一幀，而後採用CRC碼產生r位的校驗位。它只能檢測出錯誤，而不能糾正錯誤。一般取r=16,標準的16位生成多項式有 CRC-16=x16+x15+ x2+1 和CRC-CCITT=x16+ x12+ x5+1。一般情況下，r位生成多項式產生的CRC碼可檢測出所有的雙錯、奇數位錯和突發長度小於等於r的突發錯。用於糾錯目的的循環碼的譯碼算法比較複雜。

RIPv2對RIPv1協議有三方面的改進。下面的選項中，RIPv2的特點不包括(22)。

在RIPv2中，可以採用水平分割法來消除路由循環，這種方法是指(23)。

(22)A.使用組播而不是廣播來傳播路由更新報文

B.採用了觸發更新機制來加速路由收斂

C.使用經過散列的口令來限制路由信息的傳播

D.支持動態網絡地址變換來使用私網地址

(23)A.不能向自己的鄰居發送路由信息

B.不要把一條路由信息發送給該信息的來源

C.路由信息只能發送給左右兩邊的路由器

D.路由信息必須用組播而不是廣播方式發送

【答案】D B

【解析】

RIPv2是增強了的RIP協議，定義在RFC 1721和RFC 1722 (1994)中。RIPv2基本上還是一個距離矢量路由協議，但是有三方面的改進。首先，使用組播而不是廣播來傳播路由更新報文，並且採用了觸發更新（triggered update)機制加速路由收斂，即出現路由變化時立即向鄰居發送路由更新報文，而不必等待更新週期是否到達。其次，RIPv2是一個無類別的協議（classless protocol)，可以使用可變長子網掩碼CVLSM),也支持無類別域間路由（CIDR)，這些功能使得網絡的設計更具伸縮性。第三個增強是RIPv2支持認證，使用經過散列的口令字來限制路由更新信息的傳播。其他方面的特性與第一版相同，例如以跳步計數來度量路由費用，允許的最大跳步數為15等。

距離矢量算法要求相鄰路由器之間週期性地交換路由表，並通過逐步交換把路由信息擴散到網絡中所有的路由器。這種逐步交換過程如果不加以限制，將會形成路由環路 (Routing Loops),使得各個路由器無法就網絡的可到達性取得一致。

例如在上圖中，路由器A、B、C的路由表已經收斂，每個路由表的後兩項是通過交換路由信息學習到的。如果在某一時刻，網絡10.4.0.0發生故障，C檢測到故障，並通過接口 S0把故障通知B。然而，如果B在收到C的故障通知前將其路由表發送到C，C則會認為通過B可以訪問10.4.0.0，並據此將路由表中第二條記錄修改為（10.4.0.0, S0, 2)。這樣一來，路由器A、B、C都認為通過其他的路由器存在一條通往10.4.0.0的路徑，結果導致目標地址為10.4.0.0的數據包在三個路由器之間來回傳通，從而形成路由環路。

解決路由環路問題可以採用水平分割法（SplitHorizon)。這種方法規定，路由器必須有選擇地將路由表中的信息發送給鄰居，而不是發送整個路由表。具體地說，一條路由信息不會被髮送給該信息的來源。可以對上圖中B的路由表項加上一些註釋，如下圖所示，可以看出，每條路由信息都不會通過其來源接口向回發送，這樣就可以避免環路的產生。

簡單的水平分割方案是："不能把從鄰居學習到的路由發送給那個鄰居"，帶有反向毒化的水平分割方案(Split Horizon with Poisoned Reverse)是："把從鄰居學習到的路由 費用設置為無限大，並立即發送給那個鄰居"。採用反向毒化的方案更安全一些，它可以立即中斷環路。相反，簡單水平分割方案則必須等待一個更新週期才能中斷環路的形成過程。

另外，前面提到的觸發更新技術也能加快路由收斂，如果觸發更新足夠及時——路由器C在接收B的更新報文之前把網絡10.4.0.0的故障告訴B，則可以防止環路的形成。

第三層交換是指利用第二層交換的高帶寬和低延遲優勢儘快地傳送網絡層分組的技術。交換與路由不同，前者用硬件實現，速度快，而後者由軟件實現，速度慢。三層交換機的工作原理可以概括為：一次路由，多次交換。就是說，當三層交換機第一次收到一個數據包時必須通過路由功能尋找轉發端口，同時記住MAC層的目標地址和源地址，以及其他有關信息，當再次收到目標地址和源地址相同的幀時就可以直接進行交換，不再調用路由功能。所以三層交換機不但具有路由功能，而且比通常的路由器轉發得更快。

IETF 開發的多協議標記交換 MPLS (Multiprotocol Label Switching, RFC3031)把第二層的鏈路狀態信息（帶寬、延遲和利用率等）集成到第三層的協議數據單元中，從而簡化和改進了第三層分組的交換過程。理論上，MPLS支持任何第二層和第三層協議。MPLS包頭的位置界於第二層和第三層之間，可稱為第2.5層，標準格式如上圖所示。MPLS承載的報文通常是IP包，當然也可以是以太幀、AAL5包、甚至ATM信元等。可以承載MPLS的第二層協議可以是PPP、以太幀、ATM和幀中繼等，參見下圖。

當分組進入MPLS網絡時，標記邊緣路由器（Label Edge Router，LER)就為其加上一個標記，這種標記不僅包含了路由表項中的信息（目標地址、帶寬和延遲等)，而且還引用了IP頭中的源地址字段、傳輸層端口號和服務質量等。這種分類一旦建立，分組就被指定到對應的標記交換通路（Label Switch Path，LSP)中，標記交換路由器（Label Switch Router, LSR)將根據標記來處置分組，不再經過第三層轉發，從而加快了網絡的傳輸速度。

（1）非堅持型監聽算法。當一個站準備好幀，發送之前先監聽信道：

①若信道空閒，立即發送，否則轉②。

②若信道忙，則後退一個隨機時間，重複①。

由於隨機時延後退，從而減少了衝突的概率。然而，可能出現的問題是因為後退而使信道閒置一段時間，這使信道的利用率降低，而且增加了發送時延。

（2）1-堅持型監聽算法。當一個站準備好幀，發送之前先監聽信道：

①若信道空閒，立即發送，否則轉②。

②若信道忙，繼續監聽，直到信道空閒後立即發送。

①若信道空閒，以概率P發送，以概率（1-P)延遲一個時間單位。一個時間單位等於網絡傳輸時延T。

②若信道忙，繼續監聽直到信道空閒，轉①。

③如果發送延遲一個時間單位T，則重複①。

【解析】

CSMA/CA類似於802.3的CSMA/CD協議，這種訪問控制機制叫做載波監聽多路訪問/衝突避免協議。在無線網中進行衝突檢測有時是困難的，例如兩個站由於距離過大或者中間障礙物的分隔從而檢測不到衝突，但是位於它們之間的第三個站可能會檢測到衝突，這就是所謂隱蔽終端問題。採用衝突避免的辦法可以解決隱蔽終端的問題。802.11定義了一個巾貞間隔（Inter Frame Spacing, IFS)時間。另外還有一個後退計數器，它的初始值是隨機設置的，遞減計數直到'0。基本的操作過程是：

①如果一個站有數據要發送並且監聽到信道忙，則產生一個隨機數設置自己的後退計數器並堅持監聽。

②聽到信道空閒後等待IFS時間，然後開始計數。最先計數完的站可以開始發送。

③其他站在聽到有新的站開始發送後暫停計數，在新的站發送完成後再等待一個IFS時間繼續計數，直到計數完成開始發送。

分析這個算法發現，兩次IFS之間的間隔是各個站競爭發送到時間。這個算法對參與競爭的站是公平的，基本上挺按先來先服務的順序獲得發送的機會。

三層模型主要將網絡劃分為核心層、匯聚層和接入層，每一層都有著特定的作用。核心層提供不同區域或者下層的高速連接和最優傳送路徑；匯聚層將網絡業務連接到接入層，並且實施與安全、流量負載和路由相關的策略；接入層為局域網接入廣域網或者終端用戶訪問網絡提供接入。其中核心層是互連網絡的高速骨幹，由於其重要性，因此在設計中應該採用冗餘組件設計，使其具備高可靠性，能快速適應變化。

在設計核心層設備的功能時，應儘量避免使用數據包過濾、策略路由等降低數據包轉發處理的特性，以優化核心層獲得低延遲和良好的可管理性。

核心層應具有有限的和一致的範圍，如果核心層覆蓋的範圍過大，連接的設備過多，必然引起網絡的複雜度加大，導致網絡管理性降低；同時，如果核心層覆蓋的範圍不一致，必然導致大量處理不一致情況的功能都在核心層網絡設備中實現，會降低核心網絡設備的性能。

對於那些需要連接因特網和外部網絡的網絡工程來說，核心層應包括一條或多條連接到外部網絡的連接，這樣可以實現外部連接的可管理性和高效性。
