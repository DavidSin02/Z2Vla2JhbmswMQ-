兄弟，優秀程序員必會的“適配器模式”，你確定不瞭解一下？

2020-10-29
| 科技

今天給大家分享一下設計模式中的“適配器模式”。

基本介紹

適配器模式屬於“結構型模式”的一種。該模式的核心思想就是將類中原本不適合當前客戶端使用的接口轉換成適用的接口，從而大大提高程序的兼容性。

並且從使用者的角度是看不到被適配的類地，也降低了代碼之間的耦合性。

適配器模式的工作原理就是：用戶調用適配器轉換出來的接口，適配器再調用被適配類的相關接口，從而完成適配。

通過不同的實現方式，我們可以將其分成三類：類適配器模式，對象適配器模式，接口適配器模式。

講解案例

小菜鳥基於linux系統開發了一個工具類，開發完成後領導說，客戶還想在windows系統中使用。這時原來的功能肯定不再適用了，但是作為一個優秀的程序員我肯定是不會再開發一套了。這時我們小小的使用一下適配器模式，該問題就迎刃而解了。

類適配器模式

實現方式：

適配器類，通過繼承“被適配的類”，並且實現“適配器接口”，從而完成適配。

代碼：

基於linux系統開發的工具類 => 被適配的類

public class LinuxUtils {

public void linuxMethod() {

System.out.println("基於linux系統開發的方法！");

}

}

適配器接口

public interface IAdapter {

void adapterMethod();

}

轉換成windows系統可用工具 => 具體適配器類

public class WindowsAdapter extends LinuxUtils implements IAdapter {

@Override

public void adapterMethod() {

super.linuxMethod();

System.out.println("轉換成適用windows系統的工具！");

}

}

工具類的使用者

public class User {

public void useUtils(IAdapter iAdapter) {

iAdapter.adapterMethod();

}

}

測試類

public class Test {

public static void main(String[] args) {

User user = new User();

user.useUtils(new WindowsAdapter());

}

}

總結

缺點：java是單繼承語言，類適配器模式又需要繼承被適配的類，這就導致必須存在一個適配器接口。從這點就可以看出該模式是有一定侷限性。並且因繼承了被適配的類，所以類的所有方法都暴露在了適配器類中，從而增加了使用成本。

優點：由於繼承了“被適配的類”，所以可以根據需求重寫父類的方法，從而增加了代碼的靈活性。

對象適配器模式

實現方式：

適配器類，持有“被適配的類”實例，實現“適配器接口”，完成適配。

代碼實現

被適配的類、適配器接口、工具類的使用者類跟上面“類適配器模式”的示例代碼相同，就不在此贅述了。

具體適配器類

public class WindowsAdapter implements IAdapter {

// 持有被適配的類實例

private LinuxUtils linuxUtils;

public WindowsAdapter(LinuxUtils linuxUtils) {

this.linuxUtils = linuxUtils;

}

@Override

public void adapterMethod() {

this.linuxUtils.linuxMethod();

System.out.println("轉換成適用windows系統的工具！");

}

}

測試類

public class Test {

public static void main(String[] args) {

User user = new User();

user.useUtils(new WindowsAdapter(new LinuxUtils()));

}

}

總結

對象適配器模式根據合成複用原則，將原來繼承“被適配的類”改為組合，不再要求必須存在適配器接口，解決了因為繼承帶來的侷限性。降低了使用成本，增加了代碼的靈活性。

接口適配器模式

簡介

1、接口適配器模式中被適配的對象是一個接口。

2、在我們不需要全部實現被適配接口中提供的方法時，可以設計一個抽象類先實現該接口，並且空實現接口中的所有方法。這樣繼承了該抽象類的子類就可以選擇性的重寫其中的方法，以達到想要的效果。

代碼

被適配的接口：

public interface Interface {

void method1();

void method2();

void method3();

}

適配器抽象類（空實現被適配接口中的所有方法）：

public abstract class AbsAdapter implements Interface {

@Override

public void method1() {

}

@Override

public void method2() {

}

@Override

public void method3() {

}

}

客戶端測試類：

public class Client {

public static void main(String[] args) {

// 重寫被適配接口中method1方法

AbsAdapter absAdapter = new AbsAdapter() {

@Override

public void method1() {

System.out.println("使用被適配接口中method1方法！");

}

};

absAdapter.method1();

}

}

結尾

因為示例代碼足夠簡單，所以執行結果就不再展示了。以上代碼的主要目的只是為了講解適配器模式。
