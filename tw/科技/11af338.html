Windows 10 Build 20241發佈：應用啟動屏幕可跟隨系統主題

2020-10-29
| 科技

本週三面向 Dev 頻道的 Windows Insider 成員，微軟發佈了 Windows 10 Build 20241 預覽版更新。由於 Dev 頻道版本的開發進度不再跟隨功能更新，因此某些新功能可能不會裝備在下個功能更新中。但目前下個功能更新存在諸多不確定因素，有消息稱 21H1 可能會被 Windows 10X 系統取代。

Windows 10 Build 20241 預覽版更新主要引入以下新功能

為應用程序引入主題感知的啟動屏幕

繼開始菜單中創建跟隨系統主題的動態磁貼之後，微軟為 UWP 應用創建了跟隨系統主題的初始屏幕。現在，符合條件的 UWP 應用在打開的時候，初始屏幕顏色將會跟隨系統主題顯示。例如當前系統中使用的是深色模式，那麼你會看到深色模式的啟動界面。

淺色模式

深色模式

目前已經支持主題感知的應用程序包括

● Settings

● Store

● Windows Security

● Alarms & Clock

● Calculator

● Maps

● Voice Recorder

● Groove

● Movies & TV

● Snip & Sketch

● Microsoft ToDo

● Office

● Feedback Hub

● Microsoft Solitaire Collection

微軟表示通過 Microsoft Store 更新，會有更多的應用程序獲得主題感知功能。這項功能目前僅向部分 Windows Insider 成員開放，幫助微軟快速識別問題，性能影響和可靠性等。

增強碎片整理體驗

微軟對“優化驅動器”頁面（設置>系統>存儲>優化驅動器）進行了一些更改，包括：

● 增加全新的“高級視圖”複選框，列出包括隱藏卷在內的所有卷

請注意，微軟仍在推進和優化該程序，因此您會在此版本中看到該複選框，但是單擊該複選框可能不會發現任何差異。

● 在“當前狀態”列中列出更多詳細信息。

當卷不可用於碎片整理時（例如，“不支持分區類型”和“不支持文件系統類型”），在“當前狀態”列中列出更多詳細信息。

● 添加了對 F5 刷新的支持。
