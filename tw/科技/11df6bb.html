公鑰 私鑰 簽名 驗籤 說的啥？

公鑰加密，私鑰解密 私鑰簽名，公鑰驗籤

散列算法

散列算法，也叫做哈希函數，是從一個任何一種數據中創建小的數字方法，散列函數把消息或者數據壓縮成摘要，有時候也叫做摘要算法。 把數據量變小，將數據的格式固定下來。

常用的算法有：MD5 SHA1

MD5

MD5 不是一種加密算法，是一種摘要算法，無論多長的輸入， MD5 都會輸出長度為 128bits 的一個串。

public static final byte[] computeMD5(byte[] content) {    try {        MessageDigest md5 = MessageDigest.getInstance("MD5");        return md5.digest(content);    } catch (NoSuchAlgorithmException e) {        throw new RuntimeException(e);    }}

SHA1

SHA1 算法也是一個摘要算法， SHA1 比 MD5 更加安全，對於長度小於 2^64 位的消息，SHA1 會產生一個 160 位的消息摘要。

public static byte[] computeSHA1(byte[] content) {    try {        MessageDigest sha1 = MessageDigest.getInstance("SHA1");        return sha1.digest(content);    } catch (NoSuchAlgorithmException e) {        throw new RuntimeException(e);    }}

摘要算法有摘要特性和不可逆特性，一般用於檢查文件的完整性和數據簽名的場景。

加密解密

加密，解密，其實就是將原來的明文文件，使用某種算法，進行計算，得到一段不可讀的數據："密文"，是用來保護數據的，保證數據不被非法竊取。

加密算法

對稱加密

對稱加密，是指的就是：加、解密使用的是同一串祕鑰，所以叫做對稱加密，對稱加密只有一個祕鑰作為私鑰。


常用算法有：DES、3DES、Blowfish、IDEA、RC4、RC5、RC6和AES

優缺點

對稱加密比非對稱加密算法效率很高，運行速度快，但是 不是非常安全，祕鑰管理分發困難。

非對稱的加密

指的是加、解密使用不同的祕鑰，一把作為公開的公鑰，另一把作為私鑰，公鑰加密，私鑰解密，反之，私鑰加密，公鑰解密。


常用算法有：RSA、ECC（移動設備用）、Diffie-Hellman、El Gamal、DSA（數字簽名用）

優缺點

安全性更高，公鑰是公開的，私鑰自己保存， 缺點是運行時間長，速度慢，適合對少量數據進行加密。

編碼算法

Base64

Base64 算法就是編碼算法，不是加密算法。Base64 編碼的目的是把任意二進制編碼成文本。

為啥要進行base64編碼呢？

一般是因為二進制進行數據傳輸時，網絡中間的有些路由會把 ascii 碼中的不可見字符刪除，導致數據不一致，因此一般會進行 url 進行 base64編碼

測試代碼：

 String str="測試";    String b64=Base64.getEncoder().encodeToString(str.getBytes("utf-8"));    System.out.println("base64編碼之後："+b64);    String ori=new String(Base64.getDecoder().decode(b64),"utf-8");    System.out.println("base64解碼之後："+ori);

運行結果：

base64編碼之後：5rWL6K+Vbase64解碼之後：測試

base64 編碼

utf-8 -> base64(編碼) -> ASCII

base64 解碼

ASCII -> base64(解碼) -> utf-8

URLEncode

Http協議中參數的傳輸是"key=value"這種鍵值對形式。

name1=value1&name2=value2

測試代碼：

public static void main(String[] args) {    //url編碼    String str="測試";    String urlStr=URLEncoder.encode(str);    System.out.println("url編碼之後："+urlStr);    String deStr=URLDecoder.decode(urlStr);    System.out.println("url解碼之後："+deStr);}

運行結果

url編碼之後：%E6%B5%8B%E8%AF%95url解密之後：測試

簽名驗籤

數字簽名，就是通過可鑑別的數字信息驗證身份的一種方式。

數據前面有兩種運算： 簽名、驗籤。

  • 發送者使用可以代表自己身份的私鑰進行簽名。
  • 接受者使用私鑰對應的公鑰進行驗籤。這樣就實現了對消息發送者身份的驗證。


簽名驗籤的作用

簽名驗籤可以驗證發送方身份，防止 中間人攻擊 ，CSRF 跨域偽造身份攻擊。

簽名驗籤個過程

消息發送者:

  1. 報文明文（xml）轉換成字節數組， UTF-8 編碼。
  2. 使用 base64 進行編碼，生成消息內容數據： msg
  3. 使用 SHA1 對字節數組進行簽名。生成摘要數據： sign

消息接收者:

  1. 接受消息msg消息後進行 base64 解碼。
  2. 使用 SHA1 算法進行數據msg+sign 驗籤。
