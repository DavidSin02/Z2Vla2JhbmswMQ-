本次主要介紹sysfs文件系統註冊以及掛載。

一、sysfs文件系統註冊及掛載

1.文件系統的定義

sysfs是一個內存文件系統，該文件系統主要在用戶態展示系統硬件相關的設備、驅動、總線等層級關係。該文件系統將系統上的總線、設備、驅動等，按照不同的分類方式進行展示，應用層可對相應的文件屬性值進行讀寫操作。目前sysfs按照device、bus、class、kernel、drivers、power等進行分類（具體分類如下圖所示），其中devices中包含系統中所有的設備、drivers中包含所有已註冊的驅動、bus中包括所有總線相關的信息、class中系統的設備類型（包括tty、net、rtc、gpio等）

以上是針對sysfs的簡要說明，下面我們看下sysfs的文件系統的定義，sysfs的超級塊處理接口

static struct file_system_type sysfs_fs_type = {.name = "sysfs",.mount = sysfs_mount,.kill_sb = sysfs_kill_sb,.fs_flags = FS_USERNS_MOUNT,};

2.文件系統的註冊及掛載

在sysfs_init接口中實現文件系統的註冊以及掛載，該接口的實現比較簡單，如下流程所示（該流程圖忽略了錯誤處理的內容），主要就是文件系統的註冊以及文件系統的kern_mount（此時並沒有真正掛載，但是已完成超級塊、根root、根dentry、vfsmount的創建，待文件系統起來時，執行mount -t sysfs none /sys，在系統調用sys_mount時，通過調用do_add_mount實現將sysfs掛載至sys目錄下。另外在sys_mount接口中，創建或者查找超級塊是通過調用sb = sget(fs_type, sysfs_test_super, sysfs_set_super, flags, info);語句實現的，若對sysfs進行多次mount時，只創建一次超級塊，因調用sysfs_test_super時對文件系統的私有變量判斷相同，則無需再次創建超級塊）。

此處主要介紹sys_mount接口，該接口調用sget創建超級塊，並調用sysfs_fill_super對sysfs的超級塊進行設置，主要是設置超級塊的操作接口指針、根dentry的操作接口等內容（sysfs_fill_super接口的定義如下流程圖所示）。

sysfs_ops為sysfs的超級塊處理接口，包括獲取文件系統狀態的接口，刪除inode的接口，通過如下定義我們發現，其並沒有提供inode節點的創建接口。

static const struct super_operations sysfs_ops = {

.statfs = simple_statfs,

.drop_inode = generic_delete_inode,

.evict_inode = sysfs_evict_inode,

};
