Linux文件系統101

2020-11-14
| 科技


> Ancient index-card file system

Linux文件系統遵循從根開始的樹狀分層結構。 它由目錄，子目錄和數據文件組成。 此結構遵循文件系統層次結構標準（FHS）推薦的標準佈局，該文件結構是Linux Foundation維護的標準。

下面，我提供了根目錄的樹狀結構的圖片。

> Tree-like view of folders sitting on the root

在繼續之前，我想強調一點，這是類Unix系統所特有的。 這些系統旨在使事物保持簡單，並將每件事物視為字節序列。 這些字節序列被OS稱為文件。 因此，從這個意義上講，所有內容都是文件。 甚至您的硬件設備都是文件。 您的打印機是一個文件，您的內存是一個文件，基本上所有內容。 這種方法使這些系統變得非常靈活。 由於我相信Linux目錄的內部工作方式有一些需要學習的優點，因此在本教程的最後創建了一個小節來詳細解釋它。 但是，由於您可能會被忽略，因此暫時跳過此主題。

與具有多個根的Windows不同，Linux僅允許一個根。 在下面，您可以在根目錄中看到標準文件夾的圖片。 根目錄是系統上所有其他目錄和文件所在的目錄，並用正斜槓/表示。

在這裡，在本教程中，我嘗試解釋每個目錄並提供有關每個目錄的一些有用信息。 我嘗試按照重要性和相關性的順序來解釋它們。

> Standard directories sitting on the root directory


/ home目錄

主目錄（也稱為登錄目錄）是每個用戶存儲其個人文件和文檔的位置。 Linux是一個多用戶環境，因此為每個用戶分配了一個特定目錄，該目錄只能由他們和系統管理員訪問。

主目錄包含您的個人配置文件，即所謂的前導點文件。 這些文件通常是隱藏的，您需要使用帶有-a選項的ls來查看它們。 如果個人配置文件和系統範圍的配置文件之間存在衝突，則個人文件中的設置具有優先權。

當您存儲文件，下載，應用程序，視頻，圖片和聲音時，這會直接變得非常大。

請記住，該目錄是唯一可以寫入的目錄，無需root用戶的特權訪問。 對於其他目錄，儘管您可以在沒有root用戶訪問權限的情況下閱讀大多數目錄，但無法對其進行修改和寫入。

> The only user in my home directory

> This view inside my home directory. Folders with leading dot are hidden directories and they contain configuration/cache/temp files. For example .config directory has configuration files for your installed programs. The other files with leading dots are also typically for configuration purposes such as .bashrc or .profile

/ bin和/ sbin目錄

bin是二進制文件的縮寫。 這是Linux保留其基本程序和應用程序的地方。 二進制文件是包含編譯源代碼的可執行文件。 幾乎所有基本的Linux命令都可以在這裡找到，例如ls，cat，touch，pwd，rm，echo等。該目錄上的二進制文件必須可用，以便獲得用於引導和修復系統的最低功能。

> The view inside the bin directory

sbin是系統二進制文件的縮寫。 與bin相似，它也是存儲可執行程序的地方。 但是這些可執行程序對於系統配置，維護和管理任務至關重要。 Linux已決定區分普通二進制文件和這些系統二進制文件。 換句話說，此目錄保留用於引導，還原和恢復必不可少的程序。 例如，查看sbin二進制文件，fsck，文件系統檢查和修復實用程序的一些示例，或者重新引導，重新啟動系統。

由於需要特權訪問，因此該目錄不是默認$ PATH環境變量的一部分。 $ PATH包含普通用戶搜索以查找其二進制文件的路徑，並且不需要root訪問權限即可搜索這些路徑。

> The view inside the sbin directory

/ usr目錄

usr代表Unix系統資源。 它屬於用戶應用程序，與屬於系統應用程序的/ bin或/ sbin目錄相反。 此處安裝的任何應用程序均被視為對於基本系統操作而言不是必需的。 但是，這是系統中最重要的目錄之一，因為它包含所有用戶級二進制文件，它們的文檔，庫，頭文件等。此目錄是隻讀的，應用程序不應在其中寫入任何內容。

/ usr子目錄：

· / usr / bin包含系統上的絕大多數二進制文件。 此目錄中的二進制文件具有廣泛的應用程序。 例如，vi，firefox，atom，gcc，curl，dpkg等都在這裡。

· / usr / sbin包含用於管理任務的程序。 他們需要特權訪問。 與/ sbin類似，它們不屬於$ PATH。

· / usr / lib包含程序庫。 庫是常用程序例程的集合。

· / usr / local包含自編譯程序或第三方程序。 該目錄的結構與父目錄/ usr /相似，建議系統管理員在本地安裝軟件時使用。

· / usr / src包含內核源代碼，頭文件和文檔。

· / usr / include包含編譯用戶空間源代碼所需的所有頭文件。

· / usr / share包含可共享的，與體系結構無關的文件（例如docs，圖標，字體等）。 建議任何包含或不需要修改數據的程序將它們存儲在此子目錄（或/ usr / local / share，如果在本地安裝）中。

瑣事：最初，該目錄原本是與用戶相關的任何目錄，但是隨著時間的流逝，它失去了意義，現在它成為了"用戶可用程序和數據"的存放地。

> The view inside usr directory

/ etc目錄

這是您存儲所有系統範圍的配置的位置。 因此，如果您的Linux上有系統範圍的應用程序，則可以在此處找到其配置文件。

看下面的圖片，它是/ etc目錄中的視圖。 所有深藍色的名稱都是目錄，並且每個內部都是一個配置文件。 例如，python，python3，perl，ssh，apt，firefox，emacs等的配置。

/ opt目錄

opt代表可選。 這是手動安裝的軟件所在的位置。 例如，在下面的示例中，您可以看到我的計算機上安裝的zoom，BlueJeans，contained，skypeforlinux和google（Google chrome）軟件文件夾。

/ snap目錄

snap軟件包和常規Deb軟件包之間的區別如下：

· 安裝後，它們的片段不會散佈在各個目錄中。 它們將全部包含在/ snap目錄下的文件夾中。

· 它們不依賴於任何其他包，依賴項或庫。 它們捆綁了正常運行所需的一切。

· 它們與系統的其餘部分完全隔離。 因此，它們不會弄亂其他文件。

/ lib / lib32 / lib64目錄

lib代表庫。 這些是庫文件目錄。 這些庫文件是在其他二進制應用程序之間共享的程序。 bin和sbin中的二進制文件廣泛使用這些庫文件。 如果您想知道哪個命令使用哪個庫文件，則可以使用strace命令對其進行跟蹤。 例如，讓我們看看ls命令使用的庫文件。

$ strace -e open ls

答案如下圖所示。 如您所見，ls命令幾乎需要/ lib目錄中的所有庫文件才能正常運行。

> The first part shows the files inside lib directory and the second part are lib files used by ls c

/ var目錄

var代表變量。 該目錄包含可變數據，例如系統日誌文件，郵件和打印機假脫機目錄以及臨時和臨時文件。 這些通常是預期會增加大小的文件和目錄。 例如，/ var / crash保存有關進程每次崩潰的信息。 或/ var / log包含您的計算機及其應用程序的所有日誌文件，這些文件會隨著時間不斷增長。

> the view inside var directory

> A file that holds crash information of my Google Chrome. It is already 66 MB.

/ media目錄

這是操作系統自動掛載外部可移動設備（例如USB拇指驅動器）的位置。

/ mnt目錄

您可以在此處手動安裝外部設備。 它可以是軟盤，外部硬盤驅動器，網絡驅動器等。

/ media和/ mnt基本相同； 但是，建議使用/ mnt進行手動安裝，並保留操作系統的介質目錄。

/ boot目錄

這是您的引導加載程序所在的位置。 它包含啟動計算機所需的靜態引導程序，內核可執行文件和配置文件。

> The view inside the boot directory

/ sys目錄

在這裡可以與內核進行交互。 換句話說，您可以將其視為內核的接口。 該目錄是一個虛擬文件系統，這意味著文件位於內存中，並在關閉後消失。 本質上，/ sys允許您以結構化方式獲取有關係統及其組件的信息。

> The view inside the sys directory

/ dev目錄

這是您的設備（也稱為特殊文件）文件所在的位置。 這些文件允許應用程序與您的硬件設備進行交互。 請記住，這些不是設備驅動程序，而是代表計算機上每個設備並有助於訪問這些設備的文件。

這些特殊文件可以是字符文件或塊文件。 塊設備是指以塊為單位執行數據I / O的任何設備。 硬盤，而字符設備是任何以字符為單位執行數據I / O的設備，例如標準輸入/輸出（鍵盤/屏幕）。

下面，我提供了一些設備文件示例。

· 第一個硬盤：/ dev / sda（它的第一個分區是/ dev / sda1）

· 打印機：/ dev / lp0

· 電腦內存：/ dev / mem

· 終端（鍵盤或屏幕）：/ dev / tty

/ sys和/ dev都與設備有關，因此它們可能會造成混淆。 但是，它們有很大的不同。 / sys目錄包含提供有關設備信息的文件。 諸如是否打開電源，供應商名稱，型號等信息。另一方面，/ dev目錄包含允許程序訪問設備的文件（將數據寫入串行端口，讀取硬盤等）。 如您所見，它們有很大的不同。

> The view inside the boot directory

/ proc目錄

proc代表過程。 在此目錄上，您可以找到包含有關係統進程和資源信息的偽文件。 例如，您計算機上的每個進程在該目錄上都會有一個文件夾，其中包含有關該進程的信息。 該目錄是一個虛擬文件系統，在計算機關閉後會消失。

舉例來說，讓我們看看該目錄中的文件夾。 我首先使用htop命令查看計算機上的運行過程。 我根據運行時間對這些運行過程進行了排序。 讓我們選擇前3個進程，它們的PID（進程ID）分別為1916、28233和28197。讓我們看一下proc目錄中的內容。 如您所見，每個進程在proc目錄中都有一個文件夾，其中包含每個進程的必要參數。

另一個示例是有關資源的信息，例如CPU（在/ dev / cpuinfo上）或正常運行時間（在/ dev / uptime上）。

> The list of running processes on my computer. You can check them with top/htop command

> There is one folder for each running process on proc directory

/運行目錄

現代Linux發行版已將此目錄包含為臨時文件系統（tmpfs），用於存儲RAM運行時數據。 這意味著像systemd和udev這樣的守護進程在啟動過程的早期（可能在/ var / run可用之前）就已經啟動，它們具有標準化的文件系統位置，可以在其中存儲運行時信息。 由於此目錄中的文件存儲在RAM中，因此關閉後它們會消失。

/ run / udev是Linux內核的設備管理器，/ run / systemd是一組基本的構建塊，它們提供了標準的過程來控制系統啟動時運行哪些程序。

> The view inside the run directory

/ tmp目錄

應用程序在這裡存儲會話期間所需的臨時文件。 例如，當您在文字處理器中編寫文字文檔時，它會存儲一個臨時文件，以保存您編寫的所有內容。 如果由於某種原因您的系統在保存文件之前崩潰，則文字處理器可以搜索該目錄以查找最近保存的副本以恢復您的文本。 重新引導系統時，該目錄通常為空。

> The view inside the tmp directory

/根目錄

這是超級用戶（即管理員）的目錄。 您可以將其視為root用戶的主目錄。 它只能由超級用戶訪問。 在我的計算機上，它是一個空文件夾。 但是，它保留用於根帳戶的配置文件。

/ srv目錄

它是服務目錄。 如果您正在運行Web服務器，則可以在該文件夾中存儲特定於站點的數據。

/ cdrom目錄

這是掛載CD-ROM的舊目錄。 但是，今天CD-ROM會自動安裝在媒體目錄上。
