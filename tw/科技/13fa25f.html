5分鐘掌握Elasticsearch空間查詢之geo_point

2020-10-29
| 科技

我們都知道，Elasticsearch是一個全文檢索引擎，其強大的全文檢索能力可以支撐非常多的業務系統。但其實除了常規的數據類型，es還可以支持空間數據類型。目前Es可以支持兩種空間數據類型，geo_point，geo_shape。筆者今天著重介紹一下Elasticsearch對geo_point數據類型的支持。

kibana渲染geo_point數據

geo_point類型，實際上是經緯度的組合，主要用於地圖空間數據的相關查詢：

  • 以某個座標點為中心，搜索指定範圍內的點；
  • 通過空間屬性聚合數據；
  • 可以將距離作為相關性分數的一個計算因子；
  • 通過距離進行排序。

1 創建mapping

插入geo_point類型的數據之前，需要先創建mapping，可以理解為關係型數據庫中的表結構。

PUT my_index{ "mappings": { "properties": { "location": { "type": "geo_point" } } }}

2 插入geo_point類型數據的4種姿勢

2.1 作為object插入

PUT my_index/_doc/1{ "text": "Geo-point as an object", "location": {  "lat": 41.12, "lon": -71.34 }}

2.2 作為string插入

PUT my_index/_doc/2{ "text": "Geo-point as a string", "location": "41.12,-71.34" }

2.3 geohash

PUT my_index/_doc/3{ "text": "Geo-point as a geohash", "location": "drm3btev3e86" }

2.4 array

PUT my_index/_doc/4{ "text": "Geo-point as an array", "location": [ -71.34, 41.12 ] }

3 查詢實例

插入數據之後，我們來進行點的檢索，檢索出bbox範圍內的所有點。

GET my_index/_search{ "query": { "geo_bounding_box": {  "location": { "top_left": { "lat": 42, "lon": -72 }, "bottom_right": { "lat": 40, "lon": -74 } } } }}
