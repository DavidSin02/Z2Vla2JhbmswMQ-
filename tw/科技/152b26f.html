python的web服務器：django的視圖與網址的對應關係

2020-10-29
| 科技

首先新建一個項目，名稱為firstproject

sudo django-admin startproject firstproject

django-admin如果出現錯誤，就用django-admin.py，我在我的上一篇文章中也提到了。

然後用tree命令查看目錄結果如下所示：

新建一個應用，名稱叫firstapp

進入firstproject目錄，然後運行如下命令

sudo python manage.py startapp firstapp

此時回到上一級目錄，我們可以看到整個project的結構如下：

你看到的目錄結果可能與此不同，那時django版本差異造成的。差異部分暫時用不到。

把我們新定義的app加到firstproject/firstproject/settings.py中的INSTALL_APP中去。

INSTALLED_APPS = (

'django.contrib.admin',

'django.contrib.auth',

'django.contrib.contenttypes',

'django.contrib.sessions',

'django.contrib.messages',

'django.contrib.staticfiles',

'firstapp', #只有這句時新加的。

)

定義視圖函數

把firstproject/firstapp/views.py打開，修改其中的源代碼如下：

#coding:utf-8 #這是新加的，用於支持中文

from django.shortcuts import render

from django.http import HttpResponse #導入HttpResponse，類似python的print函數

# Create your views here.

def index(request) #第一個參數必須時request，其中包含get或post及其內容

return HttpResponse(u"歡迎光臨 my firstproject")

定義視圖函數對應的網址

需要打開firstproject/firstproject/urls.py這個文件。

from django.conf.urls import include, url

from django.contrib import admin

from firstapp import views as firstviews #這是新加的

urlpatterns = [

url(r'^$',firstviews.index), #這是新加的。

url(r'^admin/', include(admin.site.urls)),

]

域名訪問

如果你想通過域名訪問，還需要修改firstproject/firstproject/settings.py文件如下內容

ALLOWED_HOSTS = ['www.ddxxhome.cn']

以上修改都保存後，運行命令啟動該project

sudo python manage.py runserver 0.0.0.0:8124

最終可以通過域名和端口訪問這個project，如下圖所示：

你也可以輸入www.ddxxhome.cn:8124/admin來進入後臺管理。這就對應了urls.py中的這一行 url(r'^admin/', include(admin.site.urls)), 意思就是說，在網址中從除域名或IP地址之後的部分開始從頭匹配，一旦匹配到admin，那就執行後面的函數，這個函數是在admin模塊中定義好的。

那我們自己加的那一行是什麼意思呢？先看下面字符在正則表達式中的意思。

因此url(r'^$',firstviews.index),的意思就是如果ip或者域名之後從頭到尾匹配到空字符，也就是不帶路徑，此時在其他web框架中一般會自動執行index為前綴的文件，在這裡剛好就調用firstapp.index函數。因此如果你想調用firstviews中的其他函數，就必須在這裡增加正則匹配項。

編後語
