軟件編程也要懂“面像”？錯！是懂4個“面向”

2020-11-14
| 科技

#頭條公開課繁星計劃# #Java#

軟件設計方法主要包括面向過程編程、面向對象編程OOP、面向接口編程和麵向切面編程AOP。面向過程編程因其不利於超大規模系統設計，被逐漸淡出歷史舞臺，取而代之的是面向對象編程OOP、面向接口編程和麵向切面編程AOP。這三個方法相輔相成，不會誰代替誰。

1、面向過程編程POP(Procedure Oriented Programming)

面向過程編程以具體的一步一步實現過程為單位，關注過程的具體實現。面向過程解決問題時，會把該問題拆分為若干個步驟，然後按照一定順序執行完這些步驟，該問題就解決了。面向過程是一種自頂向下的編程。C語言就是面向過程語言的典型代表。

舉例：

用面向過程編程思想解決淘寶網購物這個實際問題，就會把該問題分解為有先後順序的四個步驟：

3.1、逛淘寶搜索Java編程

3.2、挑選合適的書

3.3、下單

3.4、付款

這四個步驟是不能顛倒順序的，必須依次執行；也不能只執行其中的某幾個步驟。

面向過程編程缺點

2、面向對象編程OOP(Object Oriented Programming)

面向對象編程以對象為單位，關注對象的屬性和方法。面向對象解決問題時，把問題抽象成若干個對象，每個對象具有自己特有的一些屬性和方法，每個對象執行自己的方法，該問題就解決了。Java、C#、C++、Delphi等都是面嚮對象語言的典型代表。也是現在最主要的軟件設計方法。

2.1、面向對象編程的三大特性

2.1.1、封裝

封裝是面向對象編程語言對現實世界的模擬，現實世界的屬性都隱藏在對象內部，外界是無法直接操作和修改的，僅對外公開訪問接口，以便控制屬性的讀寫的訪問權限。

封裝的作用是隱藏實現細節和限制不合理的操作。這樣既了保護對象的屬性，又提高了軟件系統的可維護性。

2.1.2、繼承

繼承機制允許創建分層次的類。繼承就是子類繼承父類的特徵和行為，即子類具有父類的行為。即使用現有類的所有功能，無需重新編寫這些功能。人類的繼承關係如下圖所示：

2.1.3、多態

多態同一個行為具有多個不同表現形式或形態的能力，即指一個類實例（對象）的相同方法在不同情形有不同表現形式。

3、面向接口編程IOP(Interface Oriented Programming)

面向接口編程的系統設計規範是定義（規範或約束）與該定義的實現分離。即對擴展開放，對修改關閉。

面向接口編程的主要思想是：

面向對象系統中，各種功能由不同對象協作完成。系統設計人員不太關心這些對象內部是如何實現自己的，而是重點關注對象間的協作關係。比如類間通信、模塊間交互等等，都是系統設計的主要內容。

接口的本身反映了系統設計人員對系統的抽象理解。

面向接口編程的優勢：

  • 降低程序的耦合性。

程序中緊密聯繫並不是一件好的事情，因為聯繫越緊密，更換其中一個的難度就越大，擴展功能和debug的難度也越大。

  • 易於程序的擴展。

有利於程序的維護。

4、面向切面編程AOP（Aspect Oriented Programming）

面向切面編程AOP，通過運行期動態代理實現程序功能的統一維護。AOP是OOP的補充和完善，其剖解開封裝的對象內部，並將那些影響了多個類的公共行為封裝到一個可重用模塊，並命名為切面Aspect，與具體的業務無關。其關注的是兄弟類間的橫向關係，而OOP關注的是具有父子類間的縱向關係。

使用AOP可以降低耦合度，提高程序的可重用性和開發效率。

具體實現：

接口ICustomer.java

package com.itexample.codes.aop;

public interface ICustomer {

// 定義方法/行為規範

public double BuySomethingLog(double pay);//定義購買日誌記錄

}

實現接口CustomerA.java

package com.itexample.codes.aop;

public class CustomerA implements ICustomer {

//實現接口功能

public CustomerA() {

}

//實現購買日誌記錄

@Override

public double BuySomethingLog (double pay) {

System.out.println("[購買日誌]模擬淘寶網購物過程的四個步驟:");

System.out.println("3.1、逛淘寶搜索Java編程");

System.out.println("3.2、挑選合適的書");

System.out.println("3.3、下單");

System.out.println("3.4、付款");

//返回應付款

return pay;

}

}

切面BuyAspect.java

package com.itexample.codes.aop;

@Component

@Aspect//切面註解

public class BuyAspect {

@AfterReturning(value="execution(public void com.itexample.codes.aop.ICustomer.BuySomething())",result="result")

public AfterBuying(JoinPoint joinpoint,Object result) {

String methodName = joinpoint.getSignature().getName();

List<Object> args = Arrays.asList(joinpoint.getArgs());

System.out.println("淘寶網購物應付款:" + args.get(1));

}

}

最後，總結一下。面向過程編程比面向對象編程性能高，因為類調用時實例化需要消耗資源，開銷比較大，比如單片機、嵌入式等一般用面向過程開發。但面向對象易維護、易複用、易擴展。面向接口編程和麵向切面編程都是面向對象編程的有益補充和完善。在系統設計時可以靈活運用，充分發揮各自的優點，做出更好的系統。
