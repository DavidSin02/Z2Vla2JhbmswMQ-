安裝FFmpeg多媒體庫，以及命令行程序使用介紹

2020-10-29
| 科技

FFmpeg是非常流行的多媒體框架，主要用於音視頻的解碼、編碼、轉碼、混流、過濾、播放等操作。

2000年，法國著名的程序員Fabrice Bellard創建FFmpeg項目，前兩個字母FF是Fast Forward的意思，同時他也發起MPlayer開源多媒體播放器項目。

FFmpeg圖標

圍繞FFmpeg後續將講解FFmpeg的命令行操作，使用FFmpeg的API編寫程序，深入源碼進行分析等，本篇介紹FFmpeg如何安裝，以及命令行程序的簡單使用。

使用系統為Ubuntu 18.04.4 LTS，採用Ubuntu是因為該系統對於程序員開發十分友好，沒有各種軟件彈窗，很多日常娛樂的軟件也沒有。

如果是Mac OS系統，和Ubuntu的安裝方法類似，前提需要安裝Homebrew程序。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

使用Mac OS中的brew命令代替Ubuntu中的apt命令。

如果是Windows系統，可以參看官方提供的安裝指南(https://trac.ffmpeg.org/wiki/CompilationGuide)，建議在Windows中使用虛擬機進行學習，也可以使用Docker容器。

操作系統的差異在剛開始的學習時可能比較困擾，但是熟能生巧，三個系統都可以順利地進行FFmpeg開發，沒有必要擔心繫統的不同而無法學習的情況。

舉個簡單的例子，剛開始學習java的時候，需要安裝JDK，網上有各種版本眼花繚亂。但是在理解JDK的原理之後，無論是哪種環境，都可以很容易地編寫和運行java程序。

  1. 安裝環境依賴

依賴根據系統而定，必須有最基本的編譯工具。

sudo apt install build-essential nasm git cmake
  1. 下載源碼

作為非常實用的代碼管理工具，git的使用是必備的基礎知識，後續將進行講解。

git clone https://github.com/FFmpeg/FFmpeg
  1. 配置編譯選項

除去編譯動態庫之外，都採用默認配置。

可以通過./configure --help查看具體的配置信息，在沒有掌握FFmpeg前不建議進行更多的配置。

./configure --enable-shared
  1. 編譯安裝

生成Makefile後進行編譯。

make

使用默認路徑進行安裝，install命令將庫文件安裝在/usr/local/lib/文件夾中，將頭文件安裝在/usr/local/include/文件夾中。

sudo make install
  1. 驗證安裝正確性

將FFmpeg庫相關信息打印出來。

ffmpeg -version

驗證安裝正確性

安裝之後，系統會帶有ffplay，ffmpeg，ffprobe三個程序。

ffplay用於音視頻的播放，比如需要播放視頻文件。

ffplay test.mp4

注：test.mp4是視頻測試文件，也可以任意使用MP3文件進行播放。

播放畫面

ffmpeg用於轉碼、過濾等操作，用法比較多比較複雜，比如將mp4文件轉化為avi文件。

ffmpeg -i test.mp4 out.avi

ffprobe用於查看多媒體文件流的相關信息。

ffprobe test.mp4

會顯示該文件使用H264格式進行視頻編碼，使用AAC格式進行音頻編碼。
