Collection接口的四個類及實例方法

2020-11-14
| 科技

1 ArrayList類：

ArrayList類擴展AbstractList並執行List接口。ArrayList支持可隨需要而增長的 動態數組。在 Java中，標準數組是定長的。在數組創建之後，它們不能被加長或縮短，這也就意味著開發者必須事先知道數組可以容納多少元素。但是，一般情況下，只有 在運行時才能知道需要多大的數組。為了解決這個問題，類集框架定義了ArrayList。本質上，ArrayList 是對象引用的一個變長數組。也就是說，ArrayList能夠動態地增加 或減小其大小。數組列表以一個原始大小被創建。當超過了它的大小，類集自動增大。當對象被刪除後，數組就可以縮小。注意：動態數組也被從以前版本遺留下來的類Vector所支持。關於這一點，將在後面介紹。

ArrayList有如下的構造方法： ArrayList() ArrayList(Collectionc) ArrayList(int capacity)

其中第一個構造方法建立一個空的數組列表。第二個構造方法建立一個數組列表，該數組列表由類c中的元素初始化。第三個構造函數建立一個數組列表，該數組 有指定的初始容量（capacity）。容量是用於存儲元素的基本數組的大小。當元素被追加到數組列表上時，容量會自動增加。

下面的程序是ArrayList的一個簡單應用。首先創建一個數組列表，接著添加String 類型的對象（回想一個引用字符串被轉化成一個字符串（String）對象的方法）。接著列表被顯示出來。將其中的一些元素刪除後，再一次顯示列表。

範例：ArrayListToArray.java

01 import java.util.*;

02 public class ArrayListToArray

03 {

04 public static void main(String args[])

05 {

06 // 創建一 ArrayList 對象 al

07 ArrayList al = new ArrayList();

08 // 向 ArrayList 中加入對象

09 al.add(new Integer(1));

10 al.add(new Integer(2));

11 al.add(new Integer(3));

12 al.add(new Integer(4));

13 System.out.println("ArrayList 中的內容 ： " + al);

14 // 得到對象數組

15 Object ia[] = al.toArray();

16 int sum = 0;

17 // 計算數組內容

18 for (int i = 0; i < ia.length; i++)

19 sum += ((Integer) ia[i]).intValue();

20 System.out.println("數組累加結果是： " + sum);

21 }

22 }

輸出結果：

ArrayList 中的內容 ： [1, 2, 3,4]

數組累加結果是：10

程序開始時創建一個整數的類集。由於不能將原始類型存儲在類集中，因此類型Integer 的對象被創建並被保存。接下來，toArray( )方法被調用，它獲得了一個Object對象數組。這個數組的內容被置成整型（Integer），接下來對這些值進行求和操作。

2 LinkedList 類：

LinkedList 類擴展了AbstractSequentialList類並實現List接口。它提供了一個鏈接列表的數據結構。它具有如下的兩個構造方法，說明如下：

LinkedList() LinkedList(Collection c)

第一個構造方法建立一個空的鏈接列表。第二個構造方法建立一個鏈接列表，該鏈接列表由類c中的元素初始化。

除了它繼承的方法之外，LinkedList類本身還定義了一些有用的方法，這些方法主要用於操作和訪問列表。使用addFirst()方法可以在列表頭增加元素；使用addLast() 方法可以在列表的尾部增加元素。它們的形式如下所示：

void addFirst(Objectobj) void addLast(Objectobj)

這裡，obj是被增加的項。

調用getFirst()方法可以獲得第一個元素。調用getLast()方法可以得到最後一個元 素。它們的形式如下所示：

Object getFirst() Object getLast()

為了刪除第一個元素，可以使用removeFirst()方法；為了刪除最後一個元素，可 以調用removeLast()方法。它們的形式如下所示：

Object removeFirst() Object removeLast()

下面的程序舉例是對幾個LinkedList支持的方法的說明：

範例：LinkedListDemo.java

01 import java.util.*;

02 public class LinkedListDemo

03 {

04 public static void main(String args[])

05 {

06 // 創建 LinkedList 對象

07 LinkedList ll = new LinkedList();

08 // 加入元素到 LinkedList 中

09 ll.add("F");

10 ll.add("B");

11 ll.add("D");

12 ll.add("E");

13 ll.add("C");

14 // 在鏈表的最後個位置加上數據

15 ll.addLast("Z");

16 // 在鏈表的第一個位置上加入數據

17 ll.addFirst("A");

18 // 在鏈表第二個元素的位置上加入數據

19 ll.add(1, "A2");

20 System.out.println("ll 最初的內容：" + ll);

21 // 從 linkedlist 中移除元素

22 ll.remove("F");

23 ll.remove(2);

24 System.out.println("從 ll 中移除內容之後：" + ll);

25 // 移除第一個和最後一個元素

26 ll.removeFirst();

27 ll.removeLast();

28 System.out.println("ll 移除第一個和最後一個元素之後的內容：" + ll);

29 // 取得並設置值

30 Object val = ll.get(2);

31 ll.set(2, (String) val + " Changed");

32 System.out.println("ll 被改變之後：" + ll);

33 }

34 }

輸出結果：

ll 最初的內容：[A, A2, F, B, D, E, C,Z]

從ll中移除內容之後：[A,A2,D,E,C,Z]

ll 移除第一個和最後一個元素之後的內容：[A2, D, E,C] ll 被改變之後：[A2, D, E Changed,C]

因為LinkedList實現List接口，調用add(Object)將項目追加到列表的尾部，如同 addLast()方法所做的那樣。使用add()方法的add(int,Object)形式，插入項目到指定的 位置，如例子程序中調用add（1，“A2”）的舉例。

注意如何通過調用get()和set()方法而使得ll中的第三個元素髮生了改變。為了 獲得一個元素的當前值，通過get()方法傳遞存儲該元素的下標值。為了對這個下標位 置賦一個新值，通過set()方法傳遞下標和對應的新值。

3 HashSet類：

HashSet擴展AbstractSet並且實現Set接口。它創建一個類集，該類集使用散列 表進行存儲。而散列表通過使用稱之為散列法的機制來存儲信息。

在散列（hashing）中，一個關鍵字的信息內容被用來確定唯一的一個值，稱為 散列碼（hashcode）。而散列碼被用來當做與關鍵字相連的數據的存儲下標。關鍵 字到其散列碼的轉換是自動執行的——看不到散列碼本身。程序代碼也不能直接索引散列表。散列法的優點在於即使對於大的集合，它允許一些基本操作，如：add()，contains()，remove()和size()方法的運行時間保持不變。

下面的構造方法定義為：

HashSet( )

HashSet(Collectionc)

HashSet(intcapacity)

HashSet(int capacity, floatfillRatio)

第一種形式構造一個默認的散列集合。第二種形式用c中的元素初始化散列集合。第三種形式用capacity初始化散列集合的容量。第四種形式用它的參數初始化散列集合的容量和填充比（也稱為加載容量）。填充比必須介於0.0與1.0之間，它決定在散列集合向上調整大小之前，有多少能被充滿。具體的說，就是當元素的個數大於散列集合容量乘以它的填充比時，散列集合被擴大。對於沒有獲得填充比的構造方法，默認使用0.75。

HashSet沒有定義任何超類和接口提供的其它方法。 重要的是，注意散列集合並不能確定其元素的排列順序，因為散列法的處理通常不讓自己參與創建排序集合。如果需要排序存儲，另一種類集——TreeSet將是一個 更好的選擇。

範例：HashSetDemo.java

01 import java.util.*;

02 public class HashSetDemo

03 {

04 public static void main(String args[])

05 {

06 // 創建 HashSet 對象

07 HashSet hs = new HashSet();

08 // 加入元素到 HastSet 中

09 hs.add("B");

10 hs.add("A");

11 hs.add("D");

12 hs.add("E");

13 hs.add("C");

14 hs.add("F");

15 System.out.println(hs);

16 }

17 }

輸出結果：

[D, A, F, C, B,E]

如上面解釋的那樣，元素並沒有按順序進行存儲。

4 TreeSet類：

TreeSet為使用樹來進行存儲的Set接口提供了一個工具，對象按升序存儲。訪問 和檢索是很快的。在存儲了大量的需要進行快速檢索的排序信息的情況下，TreeSet是一個很好的選擇。

下面的構造方法定義為： TreeSet( ) TreeSet(Collectionc) TreeSet(Comparatorcomp)TreeSet(SortedSetss)

第一種形式構造一個空的樹集合，該樹集合將根據其元素的自然順序按升序排 序。第二種形式構造一個包含了c的元素的樹集合。第三種形式構造一個空的樹集合，它按照由comp指定的比較方法進行排序（比較方法將在本章後面介紹）。第四種形式構造一個包含了ss的元素的樹集合。

範例：TreeSetDemo.java

01 import java.util.*;

02 public class TreeSetDemo

03 {

04 public static void main(String args[])

05 {

06 // 創建一 TreeSet 對象

07 TreeSet ts = new TreeSet();

08 // 加入元素到 TreeSet 中

09 ts.add("C");

10 ts.add("A");

11 ts.add("B");

12 ts.add("E");

13 ts.add("F");

14 ts.add("D");

15 System.out.println(ts);

16 }

17 }

輸出結果：

[A, B, C, D, E,F]

正如上面解釋的那樣，因為 TreeSet按樹存儲其元素，它們被按照排序次序自動 排序。
