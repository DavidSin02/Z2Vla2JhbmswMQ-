優化改進：傳統 Java-for 循環中複雜迭代的函數式替代方案

2020-10-29
| 科技

【注：此為Java新玩法第三篇】

儘管 for 循環包含許多可變部分，但許多開發人員仍非常熟悉它，並會不假思索地使用它。從 Java 8 開始，我們有多個強大的新方法可幫助簡化複雜迭代。在本文中，您將瞭解如何使用 IntStream 方法range、iterate 和 limit 來迭代範圍和跳過範圍中的值。您還將瞭解新的 takeWhile 和 dropWhile 方法（即將在 Java 9 中引入）。

下面來學習下3個消除JavaFor循環複雜迭代中的麻煩的新方法。

1.for 循環的麻煩

在 Java 語言的第 1 個版本中就開始引入了傳統的 for 循環，它的更簡單的變體 ForEach 是在 Java 5 中引入的。大部分開發人員更喜歡使用ForEach 執行日常迭代，但對於迭代一個範圍或跳過範圍中的值等操作，他們仍會使用 for。

for 循環非常強大，但它包含太多可變部分。甚至在打印 get/set(取值/賦值) 提示的最簡單任務中，也可以看出這一點：

7.結束語

儘管傳統 for 循環非常強大，但它有些過於複雜。Java 8 和 Java 9 中的新方法可幫助簡化迭代，甚至是簡化複雜的迭代。方法 range、iterate 和 limit 的可變部分較少，這有助於提高代碼效率。這些方法還滿足了 Java 的一個長期以來的要求，那就是局部變量必須聲明為 final，然後才能從內部類訪問它。將一個可變索引變量更換為實際的 final 參數只有很小的語義差別，但它減少了大量垃圾變量。最終您會得到更簡單、更優雅的代碼。
