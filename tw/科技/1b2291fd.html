C++拷貝構造和拷貝賦值

2020-11-14
| 科技


1 淺拷貝和深拷貝

1）如果一個類包含指針形式的成員變量，缺省拷貝構造函數只是複製了指針成員變量的本身，而沒有複製指針所指向的內容，這種拷貝稱為淺拷貝。

class STRING
{
public:
STRING( const char* s = "" ) :_str( new char[strlen(s)+1] )
{
strncpy( _str, s, strlen(s)+1 );
}

~STRING()
{
cout << "~STRING" << endl;
if (_str)
{
delete[] _str;
_str = NULL;
}
}
void show()
{
printf ("_str = %s\n", _str);
}
private:
char* _str;
};

2）淺拷貝將導致不同對象間的數據共享，同時會在析構函數中引發 "double free" 異常。因此必須提供一個支持複製內容的深拷貝構造函數。

淺拷貝原理：

int* p1 = new int(100);
int* p2 = p1;
delete p1;
delete p2; //erroe, double free

深拷貝原理：

int* p1 = new int(100);
int* p2 = new int(*p1);
delete p1;
delete p2; //ok

淺拷貝構造函數

STRING( const STRING& s )
{
_str = s._str;
}

深拷貝構造函數

STRING( const STRING& s )
{
_str = new char[strlen(s._str) + 1];
strncpy( _str, s._str, strlen(s._str) + 1 );
}

2 深拷貝賦值和淺拷貝賦值

拷貝賦值函數即對賦值運算符的重載。

類的缺省拷貝賦值和缺省的拷貝構造一樣，是淺拷貝，為了得到深拷貝效果，防止現"double free"異常，就需要自己定義一個支持深拷貝的拷貝賦值函數，其步驟如下：

類名& operator = (const 類名& that) {

1）防止自賦值

3）釋放舊資源

2）分配新資源

4）賦值新內容

5）返回自引用

*6）複用深拷貝構造和析構

}

淺拷貝賦值函數：

STRING& operator = (const STRING& s)
{
if (this != &s)
{
this->_str = s._str;
}
return *this;
}

深拷貝賦值函數：

STRING& operator = (const STRING& s)
{
if (this != &s)
{
delete[] _str;
this->_str = new char[strlen(s._str) + 1];
strncpy(this->_str, s._str, strlen(s._str) + 1);
}
return *this;
}
