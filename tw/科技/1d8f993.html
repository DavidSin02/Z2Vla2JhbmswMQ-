「實戰」安全訪問控制列表ACL，一分鐘瞭解下

2020-10-29
| 科技

一、認識ACL

A、訪問控制列表（ACL）是應用在路由器接口的指令列表。隨著網絡應用及技術的日益發展，在一些核心的路由交換機，甚至邊緣交換機上也應用了這一技術，以期在網絡的各個部分實現分佈式的有效的控制。ACL 指令列表用來告訴路由器（交換機）哪些數據報可以接收、哪一些需要拒絕。至於是接收還是被拒絕，可以由類似源地址、目的地址、端口號等的特定條件來決定。

B、ACL 通過在訪問控制列表中對目的地址進行歸類，來管理通信流量和處理待定的數據包。歸類處理每個特定接口的 ACL ，從而通過該接口的所有通信流量都要按照 ACL 所指定的條件接受檢測。

C、ACL 適用於所有的路由協議。當這些協議的數據包經過路由器（交換機）時，都可以利用 ACL 來過濾它們。可以在路由器（交換機）上配置 ACL ，用來控制對某一網絡或子網的訪問。ACL通過在路由器（交換機）接口處控制數據包是被轉發還是被阻塞來過濾網絡通信流量。路由器（交換機）根據 ACL 中指定的條件來檢測通過的每個數據包。這個條件既可以是數據包的源地址，也可以是目的地址，還可以是上層協議或其他因素。ACL 的定義是基於所有協議的。換言之，如果想要控制某種協議的通信數據流，那麼必須要對該接口處的這種協議定義單獨的 ACL（對有些協議來說，ACL 就像一個過濾器）。例如，要把路由器（交換機）接口配置成支持３種協議，那麼你至少要定義３個訪問控制列表。通過靈活的增加訪問控制列表，ACL 可以當作一種網絡控制的有力工具，用來過濾進出路由器（交換機）接口的數據包。建立 ACL 可以用來 ：

1、限制網絡流量，提高網絡性能

2、 提供對網絡流量的控制手段

3、提供網絡訪問的基本安全手段

4、在路由器接口處，決定哪種類型的通信流量被轉發，哪種被阻塞

二、ACL配置拓撲

使用標準訪問控制列表，讓 R101 可以訪問 R102，而 R103 不能訪問 R102。

三、ACL配置

1、做好基本配置，並測試連通性：

R101 上的配置：

R101#conf t

R101(config)#int e0

R101(config-if)#ip add 192.168.100.101 255.255.255.0

R101(config-if)#no shut

R102 上的配置：

R102#conf t

R102(config)#int e0

R102(config-if)#ip add 192.168.100.102 255.255.255.0

R102(config-if)#no shut

R103 上的配置：

R103#conf t

R103(config)#int e0

R103(config-if)#ip add 192.168.100.103 255.255.255.0

R103(config-if)#no shut

R102#conf t

R102(config)#access-list 1 permit host 192.168.100.101 /建立訪問控制列表

R102(config)#access-list 1 deny host 192.168.100.103

R102(config)#ing e0

R102(config-if)#ip access-group 1 in /將訪問控制列表綁在接口上，注意是 in 還是 out

2、擴展訪問控制列表

R101 上的配置：

R101#conf t

R101(config)#int s0

R101(config-if)#ip add 172.18.16.1 255.255.255.0

R101(config-if)#no shut

R101(config-if)#int e0

R101(config-if)#ip add 192.168.0.1 255.255.255.0

R101(config-if)#no shut

R101(config)#ip route 0.0.0.0 0.0.0.0 172.18.16.254

R102 上的配置：

R102#conf t

R102(config)#int s0

R102(config-if)#ip add 172.18.16.254 255.255.255.0

R102(config-if)#clock rate 64000

R102(config-if)#no shut

R102(config-if)#int s1

R102(config-if)#ip add 10.1.1.254 255.255.255.0

R102(config-if)#clock rate 64000

R102(config-if)#no shut

R102(config-if)#ip route 192.168.0.0 255.255.255.0 172.18.16.1

R102(config-if)#ip route 172.16.1.0 255.255.255.0 10.1.1.1

R103 上的配置：

R103#conf t

R103(config)#int s1

R103(config-if)#ip add 10.1.1.254.255.255.255.0

R103(config-if)#no shut

R103(config-if)#int e0

R103(config-if)#ip add 172.16.1.1 255.255.255.0

R103(config-if)#no shut

R103(config-if)#ip route 0.0.0.0 0.0.0.0 10.1.1.254

在 R102 上使用擴展訪問控制列表，使 R101,可以通過 10.1.1.1 telnet 到 r103，但無法ping 通 10.1.1.1，可以 ping 通地址 172.16.1.1，R101 通過 R102 的其他訪問全部拒絕，其他通過 R102 的訪問全部都不受影響實現命令：

R102(config)#access-list 100 permit icmp host 172.18.16.1 host 172.16.1.1 echo

R102(config)#access-list 100 permit tcp host 172.18.16.1 host 10.1.1.1 eq telnet

R102(config)#access-list 100 deny ip host 172.18.16.1 any

R102(config)#access-list 100 permit ip any any

R102(config)#int s1

R102(config-if)#ip access-group 100 out
