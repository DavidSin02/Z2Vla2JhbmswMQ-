Spring依賴注入原理分析

2020-10-29
| 科技

Spring中有幾種依賴注入的方式？


這是一個面試高頻題，但是我在面程序員的時候，聽過很多種答案。那麼標準答案是什麼？我們先不說，一步步來分析。


什麼是依賴注入

首先，我們得知道什麼是依賴注入？就是填充屬性。


一個對象通常都會有屬性，比如：


public class OrderService {    private UserService userService;        public UserService getUserService() {        return userService;    }}


OrderService中有一個屬性UserService， UserService就是OrderService的依賴。


那麼Spring的依賴注入，就是Spring框架去進行屬性的填充。那麼我們就要站在Spring的角度去思考：如果你是Spring的開發者，如果實現對一個對象的屬性進行填充?


在進行屬性填充之前，我們得先知道：在一個對象中，哪些屬性可以進行填充？


肯定是需要業務開發者去告訴程序員的，比如在屬性上加一個特定的註解，比如Spring中的@Autowired。當Spring遇到該屬性時，發現該屬性存在這個註解，Spring就會對當前這個屬性進行填充。那麼怎麼填充呢？


填充其實就是對屬性進行賦值，那麼Spring能怎麼對這個屬性進行賦值呢？賦的什麼值呢？


假設賦的值是xx。我們先考慮如果把這個xx賦值給OrderService對象。


可能我們立馬能想到的就是：

orderService.userService = xx;


這個思路沒錯，但是不能滿足所有情況，因為orderService這個對象是在Spring中實例化的，userService這個屬性的權限修飾符是private，所以在Spring中不能直接進行賦值，但是可以通過反射，比如：


// 隨便new一個表示xx，Spring中尋找要注入的值是一個比較複雜的過程UserService xx = new UserService();Class c= Class.forName("com.luban.service.OrderService");Object cInstance = c.newInstance();Field[] fields = c.getDeclaredFields();fields[0].setAccessible(true);  // fields[0]表示的就是userServce屬性fields[0].set(cInstance, xx);System.out.println(((OrderService)cInstance).getUserService());


這樣，通過反射就能對屬性進行賦值了。那麼怎麼尋找到準確的應該賦值給該屬性的值呢？


上文中的xx肯定也是一個對象，也就是說是Spring中的一個bean，那麼Spring該如何根據當前屬性去找到對應的bean呢？只有兩種方式:


  1. 根據屬性的名字
  2. 根據屬性的類型


根據屬性的名字去Spring容器中去找bean，要麼找不到，要麼就能找到一個bean，因為Spring中的beanName是唯一的。


格局屬性的類型去Spring容器中去找bean，要麼找不到，要麼可能找到一個或多個bena，因為Spring容器中的bean實際就是一個對象，而一個類型是可以有多個對象的，在Spring容器中也是如此。


那麼Spring針對這兩種方式會如何選擇呢？二選一，還是二合一。


答案很明顯，肯定是二合一，一種方式找不到就利用另外一個方式去找。那麼兩種方式中會優先利用哪種方式去找？


答案很明顯，肯定是先利用屬性的名字，因為利用名字去找更精確。
