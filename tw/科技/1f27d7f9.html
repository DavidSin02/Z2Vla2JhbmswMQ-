Dell EMC PowerEdge R7525服務器實戰

2020-11-14
| 科技

在上一篇文章（多核全能，引領風尚：基於AMD二代EPYC的戴爾易安信二路服務器首測中）介紹了Dell EMC針對第二代EPYC處理器的特點，為PowerEdge系列服務器設計了全新的主板，以更好地發揮整體性能。在本篇文章中，我們將實地測試PowerEdge系列服務器中的重要一員——R7525服務器的性能。

相比於採用EYPC處理器的其他PowerEdge服務器，R7525擔綱了更重要的角色。R7525服務器的標準2U機箱雙路處理器設計，並延續了PowerEdge服務器的一貫特點，單一服務器內支持第二代EPYC全系列處理器，哪怕是高達280W TDP的頂配型號。而其他服務器要麼侷限於空間（與散熱相關），要麼侷限於市場定位，僅靠風冷很難做到這一點。

E企研究院實驗室中的這臺R7525服務器配備了兩顆EPYC（Rome） 7302處理器和256GB DDR4內存。EPYC（中文名霄龍）7302處理器具有16核心32線程，基準頻率3.0GHz，最高頻率可達3.3GHz，並具有128MB容量的L3緩存。與上一代EPYC處理器中具有相同市場定位的7301相比，具有相同的核心與TDP，但是L3 Cache容量達到2倍，主頻也有大幅提升。

圖注：相比於上一代7301處理器，7302在相同甚至更低運行功耗的情況下，其主頻更高，從上一代的2.2GHz（最高2.7GHz）提升到了3.0GHz（最高3.3GHz），L3緩存也增大了一倍，達到了128MB；同樣支持八通道內存，但內存帶寬卻提高了約三分之一，從7301的153GB/s提升到7302的204GB/s，這都意味著能提供更高的計算能力

在EPYC 7002系列處理器中，也有一款16核心32線程的處理器——7282處理器，但定位於成本優化型市場，儘管最高可達3.2GHz，但卻只有64MB容量的L3緩存；雖然也支持3200MHz DDR4內存，但內存通道的數量僅有四個，內存帶寬更低至85GB/s，大約是7302處理器內存帶寬的四成。

關於第二代EPYC處理器的架構及7302與7282之間區別的主要成因，請參見E企研究院之前的專文分析：AMD EPYC的模塊化和NUMA之路

與眾不同：可調節的核心數量

相比於傳統x86處理器的單片式設計，EPYC家族從第一代的MCM（多芯片設計）走到現在的Chiplet（小芯片）設計，在CPU的模塊化設計上積累了豐富的經驗。從R7525中的BIOS中也能明顯感受得到，比其他服務器的BIOS有更多的可選項：

圖注：R7525服務器中“處理器”相關的BIOS選項，最上面名為“Logical Processor”的選項實際就是我們常說的“超線程”（Hyper-Threading，HT）技術，即一個物理核心提供兩個線程；圖下的紅框和綠框分別為7302處理器的NUMA選項和CCD與核心數量選項

在NUMA選項中，R7525服務器BIOS提供“0/1/2/4”四個選項，關於這一點的原因，我們在後面的測試中解釋。另外一個有關NUMA的選項是“L3 Cache as NUMA Domain”，默認為關閉（Disabled）狀態，可以選擇啟用。我們知道在EPYC處理器中，每個CCD實際上是由兩個CCX組成，每個CCX都有一塊L3緩存，供CCX內的核心共享使用。

如果選擇Disabled，意味著EPYC中所有CCX的L3緩存組成一個共享的緩存池，所有核心都可以完全訪問；如果Enabled，意味著需要操作系統調度層理解L3緩存的具體位置分佈（局部性），儘量讓不同CCX中的核心“就地訪問”L3緩存，在理論上可以降低訪問延遲。可以看作是“NUMA”技術在更微觀層面的應用。關於這個選項的作用，我們依舊在後面的測試中解答。

上面BIOS圖中綠框中的兩個選項分別對應EPYC處理器Chiplet設計中的CCD與核心數量。在上圖中，儘管在CCD只有三個選項，但可以看出7302處理器具有4個CCD，這意味著每個CCD內有4個核心每個CCX有2個核心——都是頂配的一半，所以核心總數為頂配的四分之一（7302共16核心）。而下一個選項則可選擇CCD中的核心數量，是啟用全部核心（2個）還是隻啟用一半數量（每CCX出1個，即“1+1”）的核心。

通過選擇啟用CCD及其核心數量，以及“Logical Processor”線性，R7525實際上可以“變身”為擁有不同核心數量的服務器。E企研究院結合CCD及核心數量，讓R7525分別具有不同的核心數量，並測試其計算性能：使用GeekBench工具簡單驗證7302的性能，如下圖所示：

圖注：不同核心/線程組合下的EPYC 7302處理器的整數與浮點性能。上圖中兩條平行於橫軸的藍色線與橙色線是不同核心/線程下，單個核心的整數與浮點性能，基本沒有變化，保持穩定。灰色線與黃色線則為不同核心/線程下的多核整數與浮點性能，隨著核心/線程數量的增加，整數與浮點性能幾乎成線性增加，中間具有相似性能的兩個組合實際上具有相同的核心與線程數，支持CCD與核心數量選擇不同

圖注：不同NUMA設置，與L3 Cache as NUMA Domain啟用或關閉情況下，R7525的整點與浮點性能。從測試來看，關閉NUMA（即NUMA設置為0）會導致計算性能略微下降；但打開NUMA後，不管NUMA如何設置（如1/2/4），對計算性能的影響並不明顯。同時，L3 Cache as NUMA Domain的開啟與關閉，對單核性能有略微影響，但並不影響綜合性能

不同NUMA性能揭示EPYC內存奧妙

EPYC在內存方面的顯著的特點是率先支持八通道內存，理論上可以提供更高的帶寬，但實際性能部分取決於其NUMA（Non-Uniform Memory Access，非一致內存訪問）特性。在第一代EPYC處理器中，每CCD自己提供2個內存通道，導致跨CCD的內存訪問延遲顯著增加，如果在雙路服務器，跨CPU的內存訪問延遲將更加明顯，這就是NUMA的影響。但在第二代EPYC處理器中，AMD將各CCD中將內存控制器與PCIe等I/O控制分離出來，形成單獨的芯片——IOD。所有核心都通過IOD來訪問內存和PCIe等外圍設備。

圖注：上圖中顯示了從第一代EPYC（Zen架構）到第二代EPYC（Zen 2架構）的變化。將各個CCD中的IO控制分離出來形成一個整體的IOD（I/O Die），看起來，任意CCD中的任意核心訪問任意內存都理應具有相同的延遲，也就是說，Chiplet架構能夠改善EPYC的內存訪問延遲

第一代EPYC處理器的內存訪問帶寬大多在170GB/s左右，第二代EPYC處理器普遍採用3200 MHz的DDR 4內存，再加上Zen 2架構，其內存總帶寬提升到204GB/s左右，但也有少數低端處理器只提供一半的內存通道，如7282。

E企研究院針對R7525服務器中的BIOS選項，即NUMA和L3 Cache as NUMA Domain選項的不同選擇，形成多種組合，來分別驗證R7525服務器中的內存總帶寬與內存訪問延遲。

圖注：不同NUMA選擇（0/1/2/4 NUMA，即R7525分別具有0/2/4/8個NUMA），並加上L3 Cache as NUMA Domain的啟用和關閉狀態下，R7525的內存總帶寬（測試工具：PerformanceTest 10）

從上圖來看，不同NUMA設置下，R7525的內存總帶寬還是有明顯差異的。比如在NUMA設置為4（即R7525中有8個NUMA）的情況下，內存總帶寬最高，接近了190GB/s，與AMD公佈的204GB/s相差不大。而在NUMA為0的情況下，內存帶寬幾乎降低了一半。同時從上圖也可看出，在內存帶寬的測試中，L3 Cache as NUMA Domain的啟用或關閉，對測試結果並沒有明顯的影響。

接下來是關於內存延遲方面的測試，設置不同的NUMA選項，這意味著核心在訪問內存時有兩種選擇：

1、訪問本地內存，例如NUMA內核心只訪問所在NUMA的內存

2、訪問遠程內存，例如8 NUMA設置下，NUMA 1內的核心訪問其他NUMA中的內存

同時，不同NUMA設置下，E企研究院再加上L3 Cache as NUMA Domain的啟用或關閉，來評估這一選項對內存延遲的影響。

上圖顯示了在L3 Cache as NUMA Domain啟用或關閉情況與不同NUMA設置下，核心訪問本地內存的延遲情況。從測試結果來看，如同內存帶寬的情況一樣，在NUMA設置為4（即8 NUMA）情況下，訪問本地內存的延遲最低；同時，L3 Cache as NUMA Domain在關閉狀態下，更利於訪問本地內存的延遲。

需要注意的是，這裡的“本地內存”是相對NUMA設置而言，例如當NUMA設置為0時，在任意核心看來，所有的內存都是本地內存，哪怕是跨CPU訪問內存。所以從上圖可以看到，當NUMA設置為0時，內存的訪問延遲顯著高於其他NUMA設置。而在遠程訪問方面，我們得到了一些不同的結果，如下圖所示：

圖注：上圖顯示了在L3 Cache as NUMA Domain啟用或關閉情況與不同NUMA設置下，核心訪問本地內存的延遲情況。首先從測試結果來看，L3 Cache as NUMA Domain的啟用或關閉，對於內存的遠程訪問沒有影響。當NUMA設置為0時，內存遠程訪問的平均延遲最低，明顯低於其他NUMA設置。而其他NUMA設置下，內存遠程訪問的平均延遲並沒有明顯差異從測試結果來看，EPYC處理器中的多個內存控制器仍存在一定局部性，即NUMA依然存在。因為不管是在內存總帶寬還是內存延遲方面，當NUMA設置為4時，其性能總是優於其他NUMA設置。從AMD的一些公開資料，我們找到了答案。

圖注：在IEEE 2020大會上，AMD的演講揭示了其不同內存延遲的奧祕。在第二代EPYC處理器中，依舊具有4個內存控制器，每2個CCD共享一個內存控制器，而且4個內存控制器採用環形連接，對角線上的兩個內存控制器並沒有直接連接，這就是為什麼NUMA設置為4的情況下，其本地內存訪問延遲低於NUMA為2或1時的延遲

關鍵應用場景 R7525優化到位

在瞭解了EPYC處理器的特點之後，E企研究院就可以基於R7525服務器設計一系列的應用場景測試。從我們的這款R7525的特點來看，其核多（雙路32核64線程）主頻高（基頻3.0GHz，最高可達3.3GHz），內存多（雙路16通道，32個內存插槽，配置16條16GB內存，每通道一條），容量大（共256GB），並且PCIe通道多（可支持24片NVMe SSD，且無收斂），其能支撐很多應用場景，但權衡選擇之後，我們認為以數據庫為代表的關鍵應用場景更適合評估R7525的性能特點，因此我們構建了一個單實例的Oracle數據庫場景，設置不同NUMA選項，以及分別啟用或關閉L3 Cache as NUMA Domain選項來評估R7525服務器在Oracle數據庫下的性能。E企研究院設計了一個類似電子商務網站的場景來模擬貼近真實的用戶場景，這一網站應用主要由“新用戶註冊、更新用戶詳細信息、瀏覽商品、提交訂單、處理訂單、瀏覽訂單（包括取消訂單）”等六大常見用戶行為組成，傳遞到數據庫層面（DML），即產生包括“select（查詢）、insert（插入）、update（更新）、commit（提交）、rollback（回滾）”等數據庫最常見的操作，這也是OLTP數據庫最核心的功能。六大用戶行為按一定比例混合，就會形成不同的數據庫DML操作，不同的用戶數（相當於web服務器端）的併發行為形成數據庫負載，從而測試數據庫性能。以下為不同NUMA設置下的數據庫性能：

圖注：在不同NUMA設置下的Oracle數據庫性能，從測試結果來看，當NUMA為1時（即R7525服務器具有2個NUMA），Oracle數據庫能夠有更好的性能發揮。在測試中獲得35000以上的TPS，合約210萬左右的TPM

典型如Oracle為代表的數據庫場景通常對計算、內存和存儲性能都有極苛刻的性能需求。但是通過人為的設計，可以在一定程度上減小特定因素的影響。比如在上述測試中，E企研究院通過人為設置，降低了存儲方面的因素對Oracle數據庫的影響。而結合前文中計算性能的測試——NUMA設置對計算性能影響較小，所以上述主要反映了內存性能對Oracle數據庫性能的影響。即當NUMA設置為1（R7525中有2個NUMA）最有利於Oracle數據庫性能。

結合前文中內存延遲的測試來看，當NUMA設置為4時（即R7525有8個NUMA），實際上內存性能最好。但這一測試的前提是，負載相同且恆定，如所有核心實際上都執行相同的負載。但在實際場景中，如Oracle數據庫，不同類型的數據庫操作（DML，由用戶行為產生）實際上對計算和內存都造成不同程度的負載（如不同操作需要不同的計算性能，不同的數據塊尺寸導致不同的訪問延遲），多變的壓力負載，這會導致基於應用場景的測試結果區別於純粹的實驗室性能測試。

同時，E企研究院也評估了L3 Cache as NUMA Domain的啟用或關閉，對Oracle數據庫性能影響，其結果如下：

圖注：在L3 Cache as NUMA Domain啟用或關閉情況下，Oracle數據庫性能。從測試來看，當L3 Cache as NUMA Domain關閉情況下，對負載穩定的應用環境有著較好的作用。如果應用負載隨著時間有波峰波谷的變化，那麼開啟L3 Cache as NUMA Domain則有利於承載波峰性能

從上述的測試結果來看，Dell EMC PowerEdge R7525服務器在承載實際應用場景時，其最佳設置為NUMA為1，同時關閉L3 Cache as NUMA Domain。這與R7525服務器BIOS的默認設置相吻合，這也證明了Dell EMC經過充分測試才選擇了這一默認設置。

需要注意的是，任何形式的測試，都與用戶的實際應用場景有差距。在E企研究院的測試中，我們儘可能地模擬用戶真實應用環境，但很難對負載隨著時間的變化進行模擬，我們在設計測試場景的時候，其中目的之一是儘可能達到某一子系統的瓶頸（如計算、存儲或網絡），這一目的會使得我們儘可能增大壓力負載，這會與用戶實際場景的負載有區別，進而產生不同。

結合測試結果，以及E企研究院對R7525與EPYC處理器（主要是NUMA技術）的特點，我們認為：

在負載較小情況下，例如50%以下（資源較為富裕），在R7525服務器BIOS中的NUMA設置為4，並啟用L3 Cache as NUMA Domain，這理論上有助於應用性能，典型如降低延遲；

而在負載較高情況下，尤其是計算性能接近飽和情況下（資源利用較高），將R7525服務器BIOS中的NUMA設置為1，並關閉L3 Cache as NUMA Domain，這更有利於性能。

篇幅所限，本篇測試解讀先到此。但是關於R7525服務器還有一大有別於其他服務器的特點沒有涉及——PCIe 4.0。第二代EPYC處理器率先支持了PCIe 4.0，R7525服務器也支持PCIe 4.0，但上述測試，並沒有區別PCIe 4.0與目前主流採用的PCIe 3.0的不同。所以關於PCIe 4.0的測試，我們將在下一篇文章中介紹，敬請關注。
