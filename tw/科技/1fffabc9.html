springboot 官方例子--使用Thymeleaf獲得參數值

2020-11-14
| 科技

本指南將指導使用Thymeleaf獲得參數值獲得GET參數及表格POST參數。原官網上是寫了兩篇的文章，這裡把它們合在一起了.Thymeleaf是一種用於Web和獨立環境的現代服務器端的Java模板引擎,主要目標是將優雅的自然模板帶到開發工作流程中，並將HTML在瀏覽器中正確顯示，並且可以作為靜態原型，讓開發團隊能更容易地協作,Thymeleaf能夠處理HTML，XML，JavaScript，CSS甚至純文本。springboot 推薦使用thymeleaf，支持最好。

我利用業餘時間，翻譯了Spring官網的例子，方便中文不好的同學，將陸續發到頭條上，歡迎大家關注，也可以上我個人BLOG：itmanclub.com，上面有已經翻譯過的。

1、通過GET方式獲得參數

請求http://localhost:8080/greeting

會返回HTML的如下信息:

"Hello, World!"

如果加一個name參數，如下

http://localhost:8080/greeting?name=User

這個name參數，將代替裡面的"World" ，返回結果如下:

"Hello, User!"

2、通過POST方式獲得參數

本例訪問http://localhost:8080/greeting，會看到表單，然後填入內容，提交，將看到顯示結果頁

1、 通過GET方式獲得參數

創建一個Controller

在Spring構建網站的方法中，HTTP請求由控制器處理。您可以通過@Controller註釋夠識別這些請求。在下面的示例中，GreetingController通過返回視圖的名稱來處理get請求/greeting ，在本例中是“greeting”視圖負責呈現HTML內容：

src/main/java/hello/GreetingController.java

package hello;import org.springframework.stereotype.Controller;import org.springframework.ui.Model;import org.springframework.web.bind.annotation.GetMapping;import org.springframework.web.bind.annotation.RequestParam;@Controllerpublic class GreetingController { @GetMapping("/greeting") public String greeting(@RequestParam(name="name", required=false, defaultValue="World") String name, Model model) { model.addAttribute("name", name); return "greeting"; }}

這個控制器簡潔明瞭，但有很多事情要做。讓我們一步一步地把它分解。

@GetMapping註釋確保HTTP GET請求到/greeting被映射到greeting()方法。

@RequestParam將查詢字符串參數name的值綁定到 greeting()方法的name參數中。此查詢字符串參數不是必須的，如果請求中沒有此參數，則使用默認值“World”。名稱參數的值被添加到模型對象中，最終使其可被視圖模板訪問。

方法體的實現依賴於視圖模版引擎技術（在本例中是Thymeleaf）來執行HTML的服務器端呈現。Thymeleaf解析下面的greeting.html模板並計算th:text表達式以呈現在控制器中設置的$name參數的值。

src/main/resources/templates/greeting.html

<!DOCTYPE HTML><html xmlns:th="http://www.thymeleaf.org"><head> <title>Getting Started: Serving Web Content</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <p th:text="'Hello, ' + ${name} + '!'" /></body></html>

部署程序

開發Web應用程序的一個常見功能是代碼更改、重新啟動應用程序以及刷新瀏覽器以查看更改。整個過程會佔用很多時間。為了加速事情的循環，SpringBoot附帶了一個稱為SpringBootDevTools的方便模塊。

  • 啟用熱插拔
  • 切換模板引擎以禁用緩存
  • 允許Livereload自動刷新瀏覽器
  • 其他基於開發需要的功能

創建執行的Appcation

src/main/java/hello/Application.java

package hello;import org.springframework.boot.SpringApplication;import org.springframework.boot.autoconfigure.SpringBootApplication;@SpringBootApplicationpublic class Application { public static void main(String[] args) { SpringApplication.run(Application.class, args); }}

@SpringBootApplication包含如下註解:

@Configuration 將類標記為應用程序上下文的bean定義源。

@EnableAutoConfiguration 告訴SpringBoot根據類路徑設置、其他bean和各種屬性設置開始添加bean。

@ComponentScan 告訴Spring在hello包中查找其他組件、配置和服務。

您注意到沒有一行XML嗎？也沒有web.xml文件。這個Web應用程序是100%純Java，您不必麻煩的基礎配置

此更改表明GreetingController中的@RequestParam安排按預期工作。名稱參數已被賦予默認值“world”，但始終可以通過查詢字符串顯式重寫。

增加一個主頁

靜態資源，如HTML、JavaScript或CSS，只要將它們放在源代碼中的正確位置，就可以從Springboot就可以訪問他們。默認情況下，Springboot的靜態內容位於“/static”（或“/public”）。index.html資源是一般網站的主頁，這意味著它將被用作/資源，即在我們在訪問http://localhost:8080/時，就訪問到這個頁面。所以創建這個文件：

src/main/resources/static/index.html

<!DOCTYPE HTML><html><head> <title>Getting Started: Serving Web Content</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <p>Get your greeting <a href="../../greeting">here</a></p></body></html>

重啟後你就可以訪問 http://localhost:8080/.

2、 通過POST方式獲得參數

這裡列的都和前面不一樣，其餘都一樣：

創建一個Controller

src/main/java/hello/GreetingController.java

package hello;import org.springframework.stereotype.Controller;import org.springframework.ui.Model;import org.springframework.web.bind.annotation.GetMapping;import org.springframework.web.bind.annotation.ModelAttribute;import org.springframework.web.bind.annotation.PostMapping;@Controllerpublic class GreetingController { @GetMapping("/greeting") public String greetingForm(Model model) { model.addAttribute("greeting", new Greeting()); return "greeting"; } @PostMapping("/greeting") public String greetingSubmit(@ModelAttribute Greeting greeting) { return "result"; }}

這個控制器簡潔明瞭，但有很多事情要做。讓我們一步一步地分析它。

此控制器中的兩個方法都映射到/greeting。您可以使用@RequestMapping，它默認映射所有HTTP操作，如get、post等。但在這種情況下，greetingForm()方法被專門映射為使用@GetMapping獲取，而 greetingSubmit()則被映射為使用@PostMapping發佈。

greetingForm()方法使用模型對象向視圖模板公開新的Greeting對象。以下代碼中的Greeting對象包含與greeting視圖中的表單字段相對應的id和content等字段，並將用於從表單中捕獲信息。

src/main/java/hello/Greeting.java

package hello;public class Greeting { private long id; private String content; public long getId() { return id; } public void setId(long id) { this.id = id; } public String getContent() { return content; } public void setContent(String content) { this.content = content; }}

方法體的實現依賴於視圖技術，通過將視圖名“greeting”轉換為要顯示的模板。在這種情況下，我們使用Thymeleaf，它解析下面的greeting.html模板，並呈現表單。

src/main/resources/templates/greeting.html

<!DOCTYPE HTML><html xmlns:th="http://www.thymeleaf.org"><head> <title>Getting Started: Handling Form Submission</title> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /></head><body> <h1>Form</h1> <form action="#" th:action="@{/greeting}" th:object="${greeting}" method="post"> <p>Id: <input type="text" th:field="*{id}" /></p> <p>Message: <input type="text" th:field="*{content}" /></p> <p><input type="submit" value="Submit" /> <input type="reset" value="Reset" /></p> </form></body></html>

th:action="@{/greeting}"表達式指示表單內容提交到/greeting，而 th:object="${greeting}"表達式聲明用於收集表單數據的模型對象。用th:field="{id}"和th:field="{content}"表示的兩個表單字段對應於上面Greeting對象中的字段。

現在讓我們回顧一下提交表單的過程。如上所述，表單使用一個POST提交到/greeting。greetingSubmit()方法接收由form填充的Greeting對象。Greeting對象是@ModelAttribute，因此它綁定到傳入表單內容，並且可以通過名稱（默認為方法參數的名稱，在本例中為“greeting”）在結果視圖中呈現提交的數據。id 呈現在 <p th:text="'id: ' + ${greeting.id}" /> 表達式中。同樣，content 也呈現在<p th:text="'content: ' + ${greeting.content}" /> 表達式中。
