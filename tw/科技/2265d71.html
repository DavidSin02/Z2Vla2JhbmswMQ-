乾貨｜你真的瞭解有限狀態機（FSM）算法的使用方法嗎？

2020-10-29
| 科技

由於我以前是從事路由協議開發，所以對FSM比較熟悉，因為網絡協議RFC文檔裡面有大量的關於狀態機的描述。偶然一次在知乎上看到一個算法題，不少回答說可以通過DFA來解決，最初以為是一個更高級的解法，當看到別人畫的轉換圖後，覺得跟FSM沒有區別，然後就百度了一下DFA和FSM的關係。wikipidia是這樣描述的：

Finite state machines are of two types – deterministic finite state machines and non-deterministic finite state machines.

In the theory of computation, a branch of theoretical computer science, a deterministic finite automaton (DFA)—also known as deterministic finite acceptor (DFA), deterministic finite state machine (DFSM), or deterministic finite state automaton (DFSA)—is a finite-state machine that accepts or rejects strings of symbols and only produces a unique computation (or run) of the automaton for each input string.

從以上描述應該明白了FSM和DFA的關係，也就是全集與子集的關係。

如果簡單一點描述FSM，可以這樣理解：有限狀態機的工作原理是，當事件(event)發生後，根據當前狀態(current_state) ，決定執行的動作(action)，並設置下一個狀態(next_state)。

一般用二維表來描述其邏輯關係，見下圖：

狀態機一般有兩種寫法：橫著寫和豎著寫。橫著寫是在事件中判斷狀態。豎著寫是在狀態中判斷事件。這兩種寫法是等效的，但在實際操作中，效果有點詫異，但各有優缺點。

我們來分析一個Leetcode上通過率最低的算法題，第65題---Valid Number，來感受一下FSM的強大威力。

題目如下：

Validate if a given string can be interpreted as a decimal number.

Some examples:

"0" => true

" 0.1 " => true

"abc" => false

"1 a" => false

"2e10" => true

" -90e3 " => true

" 1e" => false

"e3" => false

" 6e-1" => true

" 99e2.5 " => false

"53.5e93" => true

" --6 " => false

"-+3" => false

"95a54e53" => false

Note: It is intended for the problem statement to be ambiguous. You should gather all requirements up front before implementing one. However, here is a list of characters that can be in a valid decimal number:

Numbers 0-9

Exponent - "e"

Positive/negative sign - "+"/"-"

Decimal point - "."

Of course, the context of these characters also matters in the input.

只所以通過率低，是因為很多人不清楚為Valid Number的條件，只能一次一次的去試驗。

根據上面的轉換圖，可以得出如下的二維表，其中-1表示非法狀態。

其中橫座標表示狀態，縱座標表示輸入，當狀態處於3,5,8,9時，表示合法的狀態。對於在狀態q1輸入空格，仍然處於狀態1.

此時肯定會有人覺得，當處於q3狀態時，輸入“.”符號，應該是從q3到q4，而不應是從q3到q5。看“3.”這個測試用例，人眼觀察是應該輸出true。如果是到q4，就是輸出false了。

根據以上規則寫出來的處理代碼如下：

bool isNumber(char* s) { int FSM[5][9]={ { 1,-1, 9,-1, 9,-1,-1, 9, 9}, { 2,-1,-1,-1,-1, 7,-1,-1,-1}, { 3, 3, 3, 5, 5, 8, 8, 8,-1}, { 4, 4, 5,-1,-1,-1,-1,-1,-1}, {-1,-1, 6,-1, 6,-1,-1,-1,-1} }; if (NULL == s) { return false; } int curState = 1; while ('\0' != *s) { int input = 0; char c = *(s++); if ('0' <= c && '9' >= c) { input = 2; } else if (' ' == c) { input = 0; } else if ('.' == c) { input = 3; } else if ('+'== c || '-' == c) { input = 1; } else if ('e' == c) { input = 4; } else { input = -1; } if (!(0 <= input && 5 >= input)) { return false; } curState = FSM[input][curState-1]; if (-1 == curState) { return false; } } if (curState == 3 || curState == 5 || curState == 8 || curState == 9) { return true; } else { return false; }}

狀態機其他使用場景：

A. 驗證一串二進制bit，是否包含偶數個 0 和奇數個 1。這個題目也很適合用狀態機來解決。組合狀態有：

  1. 偶數個 0 和偶數個 1（記作 EE）
  2. 偶數個 0 和奇數個 1（記作 EO）
  3. 奇數個 0 和偶數個 1（記作 OE）
  4. 奇數個 0 和奇數個 1（記作 OO）

狀態轉換圖為：

FSM 初始化的狀態是 EE，一個 bit 都沒處理，0 和 1 都是偶數個。FSM 的接受狀態是 EO。如果最終到達這個狀態，那麼處理成功。

B. 判斷一個 binary string 是否能被 3 整除

一個數能不能被整除，關鍵看餘數是否為 0。

如果第一個輸入是 1，那麼它不能被 3 整除，商 0 餘 1。除了 0 之外，這裡餘數可能的取值還有 1 和 2。我們試試把狀態應該設置為 0，1，2，看看是否有解。這裡 0 是接受狀態，也是初始狀態。OK，從初始狀態 0 起，輸入是 1，那麼狀態遷移到 1。

如果第二個輸入是 0，也就是說現在看到的串是 10，左移了一位，相當於被除數乘以2。餘數為 (1 * 2) % 3 = 2，狀態從 1 遷移到 2。

如果第二個輸入是 1，那麼就乘 2 加 1 再取模，餘數是 (1 * 2 + 1) % 3 = 0，狀態從 1 遷移到 0。
