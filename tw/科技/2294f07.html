Selenium自動化測試中的等待時間

2020-10-29
| 科技

我們在做Web UI自動化測試過程中經常需要使用到等待時間，為什麼呢？因為對於自動化來說，它不會像人在做手工測試過程一樣，等到控件加載出來才去操作，往往自動化測試執行過程中執行時間是毫秒級的，所以如果不加等待時間的話，經常就會因為控件還未加載出來，導致自動化用例大批量的失敗。所以我們就會用到等待時間。等待時間一般有三種：1、time.sleep()第一種就是Python time模塊的等待方法，其實就是線程休眠，我們來看下源碼：

  1. def sleep(seconds): # real signature unknown; restored from __doc__
  2. """
  3. sleep(seconds)

  5. Delay execution for a given number of seconds. The argument may be
  6. a floating point number for subsecond precision.
  7. """
  8. pass

複製代碼其實通過源碼就可以看到它的使用，比如我們要去操作某個控件，可以在這個控件代碼的上一行加time.sleep(5)，這樣再執行控件操作之前就會等待5秒才會去執行它。那麼有的同學就回去問了，那麼我應該寫幾秒呢？其實就根據你在調式過程中，比如第一次調試這個控件加載用了5秒，第二次用了3秒，第三次用了8秒，那麼你就應該給一個這個控件加載需要消耗的最大時間，那麼具體怎麼實現呢？下邊通過打開論壇，等待輸入框加載出來再輸入內容給大家進行演示：

  1. from selenium import webdriver
  2. import time
  3. from selenium.webdriver.common.keys import Keys
  4. driver = webdriver.Chrome()
  5. driver.get('http://www.bcbxhome.com')

  7. driver.maximize_window()
  8. time.sleep(5)
  9. driver.find_element_by_id("scbar_txt").send_keys("selenium")

複製代碼2、隱式等待driver.implicitly_wait()當使用了隱式等待執行測試的時候，如果 WebDriver沒有在 DOM中找到元素，將繼續等待，超出設定時間後則拋出找不到元素的異常換句話說，當查找元素或元素並沒有立即出現的時候，隱式等待將等待一段時間再查找 DOM，默認的時間是0一旦設置了隱式等待，則它存在整個 WebDriver 對象實例的聲明週期中，隱式的等到會讓一個正常響應的應用的測試變慢，它將會在尋找每個元素的時候都進行等待，這樣會增加整個測試執行的時間。隱式等待時間和time.sleep的區別在於，time.sleep的等待是必須把你代碼寫的時間完全等待完後才會繼續執行，而隱式等待時間只要你控件加載出來了，它就會繼續執行往下執行，比如雖然你寫的等待10秒，但是控件要是2秒加載完畢了，那麼實際你等待的時間就只有2秒。

  1. from selenium import webdriver
  2. driver = webdriver.Chrome()
  3. driver.get('http://www.bcbxhome.com')

  5. driver.maximize_window()
  6. driver.implicitly_wait(10)
  7. driver.find_element_by_id("scbar_txt").send_keys("selenium")
