什麼是適配器模式

2020-10-29
| 科技

話說有一個培訓機構做了一個管理系統,裡面有一個教師類，教師類中有講課的方法。

實現教師類和其接口

Public interface Iteach{

Void teach ();

}

Public class Teacher implements Iteach {

void teach(){

System.out.println(“講課”);

}

}

隨著業務的發展該機構引進了一批外教，外教要用英文講課,系統如何修改？Teacher類已經寫好了，系統運行也很正常，如果貿然的修改這個類，說不定會帶來潛在問題。

怎麼解決？適配器模式登場：

一：實現新的英語外教類和其接口

Pulic interface IteachWithEnglish{

Void teachWithEnglish();

}

Public class EnglishTeacher implements IteachWithEnglish {

Void teachWithEnglish (){

System.out.println(“用英語講課”);

}

}

二：創建適配器類：

Public class AdapterTeacher extends EnglishTeacher implements Iteach{

Void teach(){

Super.teachWithEnglish();

}

}

關鍵就在於這個適配器類，這個類需要繼承英語外教類，並實現教師類的接口Iteach.

在實現接口的teach方法時實際上調用的是英語外教類的相關方法。

當要用到教師類的時候：

以前：

Public void main(String[] args){

Iteach t1 = new Teahcer();

t1.teach();

}

現在使用適配器類:

Public void main(String[] args){

Iteach t2 = new AdapterTeacher ();

t2.teach();

}

可以看到，只需要修改一個地方 由原來的 Teacher修改成AdapterTeacher,做到了最小修改。

適配器模式的定義：

來看一下適配器模式的定義：適配器模式把一個類的接口變換成客戶端所期待的另一種接口，從而使原本接口不匹配而無法在一起工作的兩個類能夠在一起工作。

這個例子中我們把EnglishTeacher 類的接口IteachWithEnglish接口變成了Teacher實現的接口Iteach，從而使需要使用Teacher類的地方都可以用適配器AdapterTeacher來代替，這就是適配器模式。

適配器模式的使用場景：

適配器模式是一種補救模式，典型的應用場景就是我們開發好的系統，由於考慮不周到，或者需求變更，需求升級，需要修改某些接口，這時候可以使用適配器模式讓我們的系統只需要最少的修改，帶來最小的影響。
