Excel Power Query中批量處理列

2020-10-29
| 科技

今天接著給大家講解有關Excel Power Query的案例。

批量處理列，在Power Query中使用的是Table.TransformColumns函數。


函數的語法如下：


function (table as table,transformOperation as list,optional defaultTransformation as nullable fuction,optional missingFiled as nullable MissingFiled.Type) as table


從上面的函數的語法來看，很多的小夥伴們可能有點崩潰，一個也沒有看懂吧。沒有關係，我們下面通過一些實例來具體地看看。


案例1


如下圖所示，想給技能1與技能2的得分，分別加上5分與10分。

具體的公式可以寫成：


= Table.TransformColumns(源,{{"技能1",each _+5},{"技能2",each _+10}})



這個公式跟我們之前講過的公式是一樣的，第一參數是源，類型是table，第二個參數是一個list，對幾個列操作就寫幾個list。如果只有一個列的時候，最外面的大括號也可以不寫。如：


= Table.TransformColumns(源,{"技能1",each _+5})


案例2


Table.TransformColumns函數還有一種比較特殊的用法。

如下圖所示，將除“員工編號”與“員工姓名”兩個列以後，其他的最後的列的格式轉化成文本型。在前面連接“成績”兩個字。


let    源 = Excel.CurrentWorkbook(){[Name="表4"]}[Content],    自定義1 = Table.TransformColumns(源,                                        {                                            {"員工編號",each _},                                            {"員工姓名",each _}                                        }                                        ,each "成績"&Text.From(_)                                    )in    自定義1



如果只是將數字轉化成文本的話，可以有下面兩種寫法：

= Table.TransformColumns(源,{{"員工編號",each _},{"員工姓名",each _}},Text.From)


或者

= Table.TransformColumns(源,{{"員工編號",each _},{"員工姓名",each _}},each Text.From(_))

上面的這兩種寫法都是等價的。


案例3


還有另外一種情形是，可以給當前的表，增加一列。

如下圖所示，增加一列空列為“備註”。


= Table.TransformColumns(源,{"備註",each _},each _,2)


這個函數的第4個函數是指當第2個參數裡面的列不存在時，可以設置相關的條件。一共有下面的幾種，此處使用的最後一個。MissingField.UseNull.


該函數的第三個參數是一個function，此處可以可以有很多的擴展的用法。
