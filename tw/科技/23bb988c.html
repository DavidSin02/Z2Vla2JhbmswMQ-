彙編語言源程序結構，初學者收藏

2020-11-14
| 科技

一，表達式

在彙編指令和偽指令中，凡是以常數、符號地址為操作數的地方，都可以換用表達式。表達式包括數值表達式和地址表達式。

1）常量

1> 數值常量 1010H

2> 符號常量

偽指令： <符號名> EQU <表達式>

功能：定義所指符號常量所代表的常量的值

說明：表達式可以是常數、指令助記符、偽指令助記符、寄存器名字或其他自定義的符號

例子： VA EQU 100 ；表達式為常數

NUM EQU VA+100 ；表達式為數值表達式

OPE EQU ADD ；表達式為指令助記符

偽指令： <符號名> = <常數表達式>

功能：定義所指符號常量所代表的常量的值

說明：“=”之後只能是常數，對於字符和字符串彙編時按照整數處理

例子： VA = 100

NUM = 3*100

NUM =VA+500

2）數值表達式

1> 算術表達式

+ - * / MOD

例如：2 MOD 5

2> 邏輯表達式

SHR、SHL、NOT、AND、OR、XOR

例如：A = 1001

A SHL 2 為0100

3> 關係表達式

EQ、NE、LT、LE（小於等於）、GT、GT

例如：NUM =50 則 NUM EQ 30 為 0

NUM GT 30 為 1

3）變量和符號

彙編中變量的屬性有：BYTE、WORD、DWORD、QWORD(4字節)、TBYTE(10字節)；Q：quad T：ten

變量定義：[<變量名>] 變量定義偽指令 <表達式>

偽指令主要為：DB、DW（1字）、DD（2個字）、DQ、DT 而在MASM 6.x以上彙編器中為BYTE、WORD、DWORD、QWORD(4字節)、TBYTE(10字節)

表達式為：常數、EQU或“=”定義的符號常量、各種類型表達式、“？”（不定義初始值只保留空間）、和DUP子句

例子：A DW 1234H ；A單元存放的是34H 而A+1單元存放的是12H

DUP子句定義：

<重複次數> DUP (<操作數1>……<操作數2>)

DW 20H DUP (0,0) ; 20H的十進制為32 定義空間大小為字 的32個單元，初始值為 （0,0）

BUFF DW 20H DUP (0,0) ; 定義BUFF為空間大小為字 的32個單元，初始值為 （0,0）

變量說明：

A DW X ; A字節存放變量X的偏移地址

A DD X ; A字節高位存放變量X的段地址，低位存放的是X的偏移地址

標號：標號也表示存儲單元，只是這種存儲單元存放的是機器指令代碼，而變量存儲數值

NEXT：MOV AL , [SI]

4）地址表達式

數值表達式的運算結果是一個數值常量，只有大小沒有屬性。

地址表達式一般為一個存儲單元的地址或數值，彙編中存儲單元有段地址、偏移地址和屬性

1> 析值操作符

也稱數值回送操作符或制止操作符，把存儲單元地址一部分作為數值返回，相當於立即數，數值在彙編時由MASM自動計算出來。

主要的析值操作符及使用格式：

SEG/OFFSET/TYPE 變量或標號名

LENGTH/SIZE 變量名

（1）SEG： 返回變量所在段的段值

MOV AX,SEG NUM ;NUM所在段的起始地址存入AX

（2）OFFSET：返回變量或標號的偏移

MOV AX,SEG NUM ;NUM所在段的偏移地址存入AX

（3）TYPE：返回變量或標號的類型，類型用數值表示，常用的類型和對應的數值：

字節（BYTE） ：1 字（WORD）：2 雙字（DWORD）：4

近（NEAR）： －1 遠（FAR）： －2

變量的類型值的對應類型的變量所佔的字節數，標號的類型值沒有實際的物理意義

（4）LENGTHOF：返回利用DUP定義的數組中元素的個數

如果沒有使用DUP，返回1

如果嵌套使用DUP，返回最外層的重複數

（5）SIZEOF：返回利用DUP定義的數組佔用的字節數，因此：

SIZEOF變量 ＝ （LENGTH 變量）* （TYPE 變量）

2> 類型操作符

為了提高訪問變量，標號和一般存儲器操作數的靈活性，彙編語言提供了類型操作符，達到按指定屬性訪問的目的。

1）PTR

如在語句：MOV [SI] 1 中，無法確定給SI指向的存儲單元中送字還是字節

MOV WORD PTR [SI] 1；指明字類型

MOV BYTE PTR [SI] 1 ；指明字節類型

PTR是最常用的合成操作符，用在地址表達式前，用於指定或臨時改變變量和標號的類型。

格式： <類型> PTR <地址表達式>

類型可以是TYPE中的類型

2）THIS

THIS <類型>

功能：建立一個新的標號、變量或地址表達式的類型屬性

該標號、變量或地址表達式的段地址和偏移地址是下一個能分配的存儲單元的段地址和偏移地址

A EQU THIS BYTE

B DW 2235H ;A和B具有相同的段地址和偏移地址

二，常用的彙編偽指令

1）段定義偽指令

<段名> SEGMENT [ <定位類型>] [<組合類型>] [<類別>]

BUF DB 10 DUP(0)

MESS DB ‘Hello’,'$',0DAH，65H

<段名> ENDS

段的定義可以嵌套不能交叉

2）假定偽指令 ASSUME

格式： ASSUME <段寄存器>:<段名>[,<段寄存器>:<段名>]

功能： 指明 段寄存器和程序段 的對應關係

例如：ASSUME CS:CODE，DS:DATA ;CS寄存器對應CODE段，DS寄存器對應DATA段

偽指令不能給寄存器和附加寄存器賦值，需呀指令語句

3）置彙編地址計數器 ORG偽指令

格式：ＯＲＧ　＜數值表達式＞

功能:告訴彙編程序,本偽指令下面的內存變量從"數值表達式"所指定的偏移地址處開始分配內存;

彙編地址計數器的值可以用偽指令ＯＲＧ設置

例子：

假設有如下定義,且變量WORD1的偏移量為0:

WORD1 DW 1234H ;offset = 0

BYTE1 DB 56H

WORD2 DW 0abcdH

ORG 1 ；設置＄的值為１　其後的變量從偏移地址為１的內存地址開始分配內存

BYTE2 DB ?

WORD3 DW ?

BYTE3 DB ?

由於偽指令"ORG 1"的作用,其後面所定義的變量要從偏移地址為"1"的內存地址處開始分配內存;

４）符號名定義偽指令LABEL

格式：＜變量名或符號＞ ＬＡＢＥＬ ＜類型＞ 其中,數據類型就是常用的數據類型:BYTE、WORD、DWORD、結構類型、記錄類型、NEAR、FAR,等等;

功能：符號名定義偽指令LABEL與存儲單元別名操作符THIS的功能類似,該偽指令定義一個指定的符號名,該符號名的段地址和段內偏移地址與下面緊跟著的一條變量定義語句中定義的變量的相應屬性完全相同;但是該符號的類型是新指定的;前面5種類型是變量的類型,後面2種類型是標號的類型;如果格式中的"數據類型"是前面5種類型之一,那麼,符號名就是變量名;如果格式中的"數據類型"是後面2中類型之一,那麼,符號名就是標號名;

例如:

WBUFFER LABEL WORD ;

WBUFFER與BUFFER具有相同的段地址和段內偏移地址,但是它們的數據類型不同;

BUFFER DB 200 DUP(?)

NEXT1 LABEL FAR ;NEXT1與NEXT具有相同的段地址和段內偏移地址,但是,NEXT1是"遠標號",NEXT是"近標號"

NEXT: MOV AX,BX

標號NEXT1和NEXT可用於不同的情況:當在同一個模塊內部轉移時,可使用近標號NEXT;當在不同的模塊之間轉移時,可使用遠標號NEXT1;

５）源程序結束偽指令 ＥＮＤ

格式：ＥＮＤ　＜標號＞　；如果遠程序定義了標號，則ＥＮＤ之後帶標號

三，彙編程序上機過程

一個源程序必須至少包含一個 代碼段和ＥＮＤ偽指令。數據段、附加數據段和堆棧段由程序具體功能而定

１）開發環境

彙編語言的源程序需要編譯和連接後才能在計算機上執行。

彙編語言源程序擴展名規定為.ASM。

用MASM編譯源程序後，將產生源程序的目標程序文件，其擴展名為.OBJ。

再經過連接程序(LINK)把目標程序文件與庫文件或其他目標文件連接在一起形成可執行文件(擴展名為.EXE)。

這時就可以運行該程序了。DOS下可直接鍵入可執行程序的文件名從而運行該程序。

２）上機過程

１＞　編輯一個彙編語言源程序，可以使用各種文本編輯軟件，如Windows記事本、MS-DOS自帶的EDIT軟件等。使用EDIT時可在DOS提示符下鍵入：

C：\MASM>EDIT 文件名.ASM(回車)

例子：下面給出一個彙編語言源程，它是求1+2+3+……+50的和並存入SUM單元中。其源程序文件為ex66.asm。

程序如下：

DATA SEGMENT

ORG 100H

SUM DW ？

DATA ENDS

CODE SEGMENT

ASSUME CS：CODE，DS：DATA

START： MOV AX，DATA

MOV DS，AX

MOV CX，32H

MOV BX，1

MOV AX，0

LP： ADD AX，BX

INC BX

LOOP LP

MOV SUM，AX

MOV AH，4CH

INT 21H

CODE ENDS

END START

用戶編寫好上面程序後，在DOS下鍵入命令：

C：\MASM> EDIT EX66.ASM

進入編輯程序中輸入上述程序並保存於磁盤（編輯與保存的方法請參見DOS操作系統中EDIT的使用）。

２＞　彙編源程序

對彙編語言源程序進行彙編時，彙編程序對．ASM文件進行掃描，若源程序中有錯，則結束彙編後，指出源程序中的錯誤。這時，可通過屏幕上提供的信息修改源程序(回到EDIT編輯狀態)，再經過彙編，直到得到無錯誤的目標程序。注意，目標程序的擴展名為．OBJ。

彙編命令的格式是：

C:＼MASM>MASM 文件名(回車) 或 C:＼MASM>MASM 文件名；(回車)

這裡，MASM是存放彙編程序MASM.EXE的子目錄。編譯、連接和調試彙編程序時，要求在計算機上要裝有MASM.EXE、LINK.EXE和DEBUG.EXE程序，可以將這些程序放入MASM子目錄下。如果子目錄不同，可以設置具體路徑。下面就以程序EX66.ASM為例，對彙編過程作—個具體介紹。

首先鍵入： C：\MASM>MASM EX66(回車)

屏幕上將出現如下提示信息：

Microsoft (R) Macro Assembler Version 5.00

Copyright (C) Microsoft Corp 1981—1985，1987. All right sreserved，

Object filename[EX66.OBJ](回車)

Source listing [NUL.LST]：(回車)

Cross-reference[NUL.CRF]：(回車)

51610+437910 Bytes symbol space free

0 Warning Errors

0 Severe Errors

屏幕各行表示的含義如下：

Object filename[EX66.OBJ]：表示要求輸入目標程序名，方括號裡的內容是程序默認的文件名，如果不改變，可直接輸入回車。

SourcelistingINUL.LST]：要求輸入列表文件名，列表文件給出源程序和彙編後的機器語言程序清單，並給出符號表，這樣便於程序的調試。方括號裡缺省的內容是不建立該文件，如果認可，可直接輸入回車。如果需要列表文件，可輸入列表文件名(擴展名可以省略)。

Cross-reference[NUL.CRF]:要求輸入索引文件名，方括號裡的內容是不建立該文件，如果認可，可直接輸入回車。如果需要索引文件，可輸入索引文件名(擴展名可以省略)。：三個提示輸入完畢，如果程序無誤，則出現：

51610+437910Bytessymbolspacefree

0 Warning Errors

0 Severe Errors

表示彙編成功。在該目錄下可以看到EX66.OBJ文件(使用DOS下的DIR命令)。

彙編時，不需要系統提示上述過程，在使用命令的：

C:＼MASM>MASM 文件名；(回車)

如果源程序有語法錯誤，則彙編程序提示源程序有錯，必須回到編輯源程序下，根據提示的錯誤信息重新修改源程錯誤，然後再進行彙編，直到彙編沒有錯誤為止。

３＞ 連接程序

經過彙編產生的目標程序並不是可執行的程序，必須經過連接才可以執行。連接程序可以將若干個目標模塊連同庫子程序連接在一起，連接程序可以使用DOS下的LINK.EXE，具體的命令格式是在DOS提示符下鍵入：

C：\MASM>LINK文件名(回車)

C：\MASM>LINK文件名；(回車)

連接後，產生擴展名為.EXE的可執行文件。下面是E66.OBJ文件連接的具體過程：

首先鍵入：

C：LMASM>LINK EX66(回車)

屏幕上將出現如下提示信息：

Microsoft (R) OverlayLinker Version 3.60

Copyright (C) MicrosoftCorp 1983—1987. Allrightsreserved.

Run File[EX66.EXE]：(回車)

List File[NUL.MAP]：(回車)

Libraries [.LIB]：(回車)

其中：

Run Filel EX66.EXE]：要求輸入可執行文件的名稱，默認是EX66.EXE，一般採用默認文件名，直接回車。

List File[NUL.MAP]：表示是否要建立連接影像文件，若要建立，則輸入文件名：若不建立，則直接按回車鍵。

Libraries[.LIB]：表示是否用到庫文件，若不需要，則直接輸入回車鍵。

連接時，不需要系統提示上述過程，使用命令：

C:＼MASM>LINK 文件名；(回車)

４＞ 程序的執行

當建立了可執行文件後，就可以在DOS下輸入該文件名，

C：LMASM>文件名(回車)

對於E21文件，可以鍵入：

C：＼MASM>EX66(回車)
