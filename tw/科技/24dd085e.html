c語言入門5，一文徹底弄懂函數的形參和實參，再也不暈了

2020-11-14
| 科技

上一節，我們介紹了函數的形參和實參，知道了有參數的函數可以根據輸入的數據，調整自己，能完成更加複雜的任務。在最後，我們舉了一個初學者容易出錯的例子，現在再把它寫一遍，請和妹子一起看如下代碼：

#include <stdio.h>void add(int a){ a=a+1;}int main(){ int p = 2; add(p); printf("p=%d", p); return 0;}

程序員小明定義了一個 add 函數，他希望 add 函數可以將 p 加一，但是卻失敗了，以上程序輸出的仍然是 p=2。


上一節，有朋友（@藍海星007）回覆說：“所以 易錯點 那個代碼怎麼寫才對呢[笑哭]”。在回答這個問題之前，我們再詳細說說小明寫的代碼為什麼沒有按照他的預期實現。

我們知道，程序運行在內存裡，請看下圖（暫時沒有必要全部看懂），小明的程序運行時，系統分配給它的內存可以劃分為好幾個段。每調用一次函數，系統就會在棧裡劃分一塊區域出來給這個函數使用。因為 main 函數是入口函數，所以棧一開始就要分配一塊區域給它使用。接著，main 函數又調用了 add 函數，所以系統又在棧裡劃分了一塊區域給 add 函數使用。

這裡說的“一塊區域”，程序員們常常稱作“棧幀”。


函數內部定義的局部變量，都在屬於自己的棧幀裡。所以小明的寫的代碼的 main 函數中的變量 p，在 main 函數的棧幀裡（上圖的最左邊淺綠色小塊裡）。add 函數裡的變量 a 在 add 函數的棧幀裡（上圖的第二塊橘黃色小塊裡），在調用 add(p) 函數時，只是把 p 的值傳給了 a，add 函數中無論如何在橘黃色小塊中對 a 操作，也不會影響到淺綠色小塊中的 p，所以，程序最終當然會輸出 p=2，而不是小明設想的 p=3。


事實上，在 add 函數執行完畢後，系統就把屬於 add 函數的棧幀收回了。這個時候，add 函數中的變量 a，也就沒了。用較專業的話說就是，函數在執行完畢後，局部變量就自動釋放了，這就是原因。

棧幀與棧幀之間是不能互相訪問的，所以 main 函數無法訪問 add 函數裡的 a，add 函數也無法訪問 main 函數裡的 p，想把 p 傳給 add 函數，只能通過參數傳遞。

那麼，怎樣寫才能實現小明的設想呢？

其實很簡單，只要在 add 函數計算後，把橘黃色小塊裡的 a 傳給淺綠色小塊裡的 p 就可以了。return 就可以實現這一的需求，我們來試一試：

#include <stdio.h>int add(int a){ a=a+1; return a;}int main(){ int p = 2; p = add(p); printf("p=%d", p); return 0;}


的確成功了。add 函數在結束自己之前，通過 return 把計算結果返回了，返回給誰了呢？從 main 函數可以看到，我們使用 p 接收了這個返回值，所以終於可以輸出 p=3 了。如果像前兩節一樣把函數比作積木，那現在我們應該把積木玩活了，哈哈。

可能你又會問，不是說 add 函數執行完，它的棧幀就被系統收回了嗎？怎麼還能把計算結果返回呢？這個問題，我之前的文章已經解答了，感興趣可以再看看：《三分鐘弄懂C語言的return，原來是這樣返回局部變量的》
