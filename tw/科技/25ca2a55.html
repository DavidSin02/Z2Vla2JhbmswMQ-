LTE協議學習總結之第二章：LTE無線通信技術主體介紹

2020-11-14
| 科技

LTE(Long Term Evolution,長期演進技術)是3GPP主導制定的無線通信技術，其實準確來說，LTE不等於4G，我們更願意稱它為“準4G”或“3.9G”。它是一個3G到4G之間的過渡技術。

在瞭解了LTE技術大概是什麼之後，我們需要明確它的目標是什麼？通過查閱相關資料可知，這種以OFDM/FDMA為核心的技術的主要性能目標包括：能靈活配置1.4MHz、3MHz、5MHz、10MHz、15MHz、20MHz的頻譜帶寬；並且保證在頻譜帶寬在20MHz的條件下能提供下行100Mbps、上行50Mbps的峰值速率；在系統延遲方面表現為用戶平面單向傳輸時時延應該低於5ms，而在控制面由睡眠到激活狀態時的時延應低於50ms，從駐留到激活狀態的時延小於100ms；提高小區容量，改善小區邊緣用戶性能；能夠為350Km/h高速移動用戶提供最少100kbps的接入服務；取消電路(CS)域和無線網絡控制(RNC)節點。LTE允許高速數據通信 (接收: 100Mbps或更高，傳送: 50Mbps或更高) 。因為頻段可擴展到20MHz，所以最大理論速度為接收326.4Mbps，傳送86.4Mbps。 數據通信速度高於相同頻段的3G/3.5G，可讓許多用戶使用。 另外，LTE可實現低延遲傳送，連接延遲為100ms或更短，無線段傳送延遲為5ms或更短。因此LTE適用於音頻通信、動態圖像分發和在線遊戲。


LTE目標

接著我們介紹LTE技術的主要技術：

LTE採用了寬廣的頻率範圍：700MHz-2.7GHz，即對應特高頻（UHF），對應電磁波的波長為分米波：1m~10cm。為了增強此頻段下的效率，採用了多個天線進行發送/接收的 Multi Input Multi Output（多入多出技術 MIMO）以及雲訊基站間多重訪問的Space Dvision Multiple Access（空分多址 SDMA）等天線技術。另外採用了Orthogonal Frequency Division Multiple Access （正交頻分多址 OFDMA）和Single Carrier Frequency Division Multiple Access（單載波頻分多址 SC-FDMA）作為多路傳輸系統。

為了明白LTE中網元彼此之間如何聯繫，最後我們介紹一下LTE的基本架構-協議架構：

接口協議的通用模型

LTE無線側的主要接口分為空中接口和地面接口。LTE空中接口是UE和eNodeB的LTE-Uu接口，LTE無線側的地面接口主要是eNodeB之間的X2接口及eNodeB和EPC之間的S1接口。

接口協議主要分為三層兩面，三層即：物理層(Physical Layer)、數據鏈路層(Transport Network Layer)、網絡層(Radio Network Layer)。兩面為：用戶面和控制面。物理層的作用是提供兩個物理實體間的可靠比特流的傳送，適配傳輸媒介。無線的空中接口中，適配的是無線環境，而在地面接口中，適配的則是EI、網線以及光纖等傳輸媒介。數據鏈路層的作用有信道複用和解複用、數據格式的包裝、數據包調度等，其主要的作用是具有個性的業務數據向沒有個性的通用數據幀的轉換。網絡層的作用則是尋址、路由選擇、連接的建立和控制、資源的配置策略等。如何理解兩面呢？兩面相當於企業裡的業務線和管理線，業務線負責實際幹活，而管理則負責人力、物力和財力上的協調控制工作，為了使兩個線徹底分開，企業中確立了職業經理人的制度，也就是儘量將業務線和管理線能獨立工作使其徹底分開。用戶面的主要作用是負責傳輸處理業務數據，而控制面則負責協調和控制指令的傳送和處理。由於它們都屬於邏輯層面上的概念，因此在物理層不區分兩面，而在數據鏈路層完成數據處理功能，其實體完成工作則位於網絡層。

LTE空中接口用戶面協議

LTE空中接口控制面協議

用戶面只有物理層和數據鏈路層兩個層。用戶面協議模塊的數據鏈路層包括MAC(Medium Access Control,媒質接入控制)、RLC(Radio Link Control，無線鏈路控制)、PDCP(Packet Data Convergence Protocal,包數據匯聚協議)三個功能模塊。在用戶面，它的主要功能是處理業務數據流。發送端將承載高層業務應用的IP數據流經過頭壓縮(PDCP)、加密(PDCP)、分段(RLC)、複用(MAC)、調度等過程變成物理層可處理的傳輸塊；接收端，將物理層接收到的比特數據流按調度要求解複用(MAC)、級聯(RLC)、解密(PDCP)、解壓縮(PDCP)成為高層應用可以識別的數據流。

控制面則包括物理層、數據鏈路層和網絡層三個層。控制面協議模塊的數據鏈路層包括MAC(Medium Access Control,媒質接入控制)、RLC(Radio Link Control，無線鏈路控制)、PDCP(Packet Data Convergence Protocal,包數據匯聚協議)三個功能模塊。其功能模塊的作用與用戶面一致，但是PDCP層功能與用戶層稍有區別，即除了對控制信令進行加密解密外海要對控制信令數據進行完整性保護和完整性驗證。在控制面中還有網絡層的內容，它包括兩個功能模塊：RRC(Radio Resource Control,無限資源控制)和NAS(Non Access Stratum,非接入層)。UE(用戶)和eNodeB(基站)之間的控制信令主要是無限資源控制RRC消息。它相當於地方軍區的一個司令部，其作用是負責軍區資源管理和給各個下屬單位下達任務和命令。另一方面還得接收來自中央軍委(高層)的命令。RRC相當於eNodeB內部的一個司令部，RRC消息攜帶建立、修改和釋放數據鏈路層和物理層協議實體所需的全部參數。另一方面，它還要給UE用戶端透明傳達來自核心網的指示。RRC模塊的主要功能有系統信息的廣播、尋呼、RRC連接管理、無線資源控制和移動性管理等。
