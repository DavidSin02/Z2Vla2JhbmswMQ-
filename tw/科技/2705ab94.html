引言

在淘系專家帶你初探端智能與MNN（上篇）裡，我們探討了端智能的特性，挑戰以及趨勢。端智能是Mobile First和AI First兩條道路的交叉點，也是許多移動開發者跨界發展的出路。


在這篇文章裡，我們會介紹一下手機淘寶中端智能的應用。然後會介紹一下手機淘寶端智能中提供核心基礎設施的推理引擎MNN。順帶說一句，MNN團隊中許多成員之前是做移動開發的。移動開發者向AI開發跨界，可以參考一下哦。


手機淘寶中端智能的應用

手機淘寶（手淘）裡面的端智能應用比比皆是。從大家最常用的商品搜索框，到利用照相機視頻流買東西的“拍立淘”，再到最近幾年興起的使用AR（增強現實）進行化妝品的試妝，到處都是端智能的應用。


個人認為，端智能在手淘中的應用方式，分為以下兩類：


1、利用AI新技術，創造全新的交互方式。這種利用AI，打開一個新賽道的方式是最難的，也是最有回報的。這方面的例子是AR試妝（口紅，染髮等）。現在在手淘裡，你可以進入雅詩蘭黛的口紅頁面，點擊“在線試妝”來玩AR技術預覽口紅的效果。見下圖。


2、利用 AI 新技術，給已有的工作的流程提供新的增長點。這個所謂的增長點，可以是更加契合用戶個人的特點，也可以是某些操作更加有效率，更加節省服務端的壓力，或者更加隱私。當然，可以是以上的幾點（個性化，時效性，省服務端算力，隱私性）結合起來。


  • 個性化：千人千模。全局模型可以被用戶的個人使用數據個性化定製。目前手淘首頁的“猜你喜歡”頁面的推薦內容，就是使用了用戶的個人數據對全局推薦模型進行了定製，取得CTR的提升 （目前此功能在灰度實驗中）。
  • 時效性+省服務端的算力：以手機淘寶首頁的信息流為例，手機端會向服務端請求展示的商品，服務端為回覆一定數量的商品以及相應的排序，但是這個排序是可以被app內的用戶行為實時改變的。這個所謂的“端上重排”就是手機端利用本地的信息，運行一個ML模型，快速做出決定，不用與服務端通信。端上重排根據用戶的瀏覽信息重排商品，可以提高轉化率。
  • 隱私性：這方面比較火的是前幾年由Google推出的Federated Learning。它可以在每個用戶數據不離開手機的情況下，訓練出一個描述所有用戶的模型，同時有安全算法保證最終的全局模型無法逆向出原始的訓練數據。Federated Learning還存在一些基本的研究挑戰沒有解決[1]，所以手機淘寶還停留在嘗試Federated Learning，並沒有完全工業落地。


MNN介紹

MNN (Mobile Neural Network) 是阿里巴巴內部自己研發的推理引擎。在它研發之初，TFLite剛剛發佈，而且由於TFLite最初版有諸多問題，為了快速響應業務的變化，阿里巴巴決定內部自研推理引擎MNN。MNN在2019年5月初在Github開源。開源至今，受益於阿里巴巴內部的用戶和開源社區用戶的反饋，MNN逐漸成長為一個優秀的、成熟穩定的推理引擎。目前，在阿里巴巴內部，MNN是端上推理引擎的事實標準。


MNN主要解決的是現在端智能的兩大痛點：1. 底層的硬件、操作系統碎片化 （CPU、GPU、NPU, Android/iOS, Mobile/IOT） 2. 機器學習的框架複雜（TFLite，PyTorch, ONNX, etc.）。為了能夠在不同的硬件、操作系統裡發揮最好的功能，MNN會在運行時根據運行環境尋找出最快的算子實現算法（我們稱這個過程為“半自動搜索”）。關於半自動搜索的技術細節，我們在第二次直播中有深度解答：http://mudu.tv/watch/4397479。


目前端上的深度學習引擎有許多家（Google有TFLite，Facebook有PyTorch，蘋果有CoreML，百度有PaddleLite，騰訊有NCNN，小米有MACE，阿里巴巴有MNN），MNN提供了什麼特殊的價值呢？為什麼用戶要選用MNN而不是別的呢？


第一：MNN 的性能是領先於業界的。2019年9月的數據：Arm CPU上MNN 的單線程推理性能大約是TFlite的2~4倍；4線程推理性能大約是 TFLite 的10~20倍。（更多具體實驗數據，我們在第二次直播給出：http://mudu.tv/watch/4397479）


第二：MNN 的大小有優勢。它的靜態庫約為 TFLite 的66.7% （MNN + OpenCL backend: 1.2M + 334K; TFLite + NNApi backend: 2.3M）。


第三：MNN 支持多種異構設備。CPU方面，MNN 支持了X86，ArmV7, V8。GPU 方面，MNN 支持了 OpenCL, OpenGL, Vulkan, Metal。NPU方面，MNN 支持了華為的 NPU ，並用在手淘的拍立淘場景中。


第四：MNN 的算子支持程度已經得到大幅的加強。MNN 在2019年12月大約增加了200個 OP 。


▐ MNN 的主要模塊


MNN 的主要模塊有：模型轉換，模型量化工具和推理引擎本身。


► 模型轉換


模型轉換器如下圖：



目前，MNN還沒有支持從頭開始訓練一個模型的能力。也就是說，用戶必須事先用其他的訓練框架比如Tensorflow, PyTorch把模型訓練好，然後再轉換為MNN的模型格式，最終部署到端上。


它主要的工作流程是，先解析“源頭”格式的模型（Caffe、TF、ONNX），進行一些與硬件無關的優化，例如圖優化、算子融合、算子替換、佈局調整等。得到初步優化的圖之後，轉換為MNN自己的格式。MNN的模型格式是一個自定義Schema的flatbuffer格式 [2]。


► 量化工具


模型轉換好之後，可以使用 MNN 的量化工具對模型進行壓縮。目前，MNN支持 post-training quantization（無訓練量化）。後續 MNN 會支持 quantization-aware training（帶訓練量化），以獲得更好的準確率和更低比特的壓縮。


模型的量化工具如下圖：


MNN的量化方案是自己實現的，它目前有ADMM和KL散度兩種方案。也就是說，“源頭”的預訓練好的模型需要是浮點的。ADMM量化方案，是MNN根據達摩院的Paper “Extremely Low Bit Neural Network: Squeeze the Last Bit Out with ADMM” [3] 實現的。它與KL散度的區別在於：ADMM是基於數學優化的方法，只需要幾十個數據點即可，但是計算較慢。而KL散度是基於概率統計的方法，需要較多的數據（500到1000個數據點），計算較快。實際操作上來說，對特徵的量化，ADMM和KL散度沒有巨大的差距；對權重的量化，推薦使用ADMM。


▐ MNN 未來展望


對於未來的發展，我們主要立足於兩點來看：第一，解決現有問題；第二，把握未來的趨勢，面向未來做技術佈局。


解決現有問題方面：


  1. MNN 的易用性還不夠。這方面，我們會有一整套的易用性提升：開箱即用的工具箱；文檔補齊；Demo 示例；預編譯包等… 這些易用性提升的改變，我們預計在今年春節之前發佈。
  2. 訓練：很多時候，要獲得一個用於某個應用場景的模型其實很簡單：拿 MobileNet 做一下遷移學習，就可以達到很好的效果。MNN 未來會加入訓練模型的能力，讓這種遷移學習的場景可以直接通過 MNN 的 API 或者工具達成。


面向未來的技術佈局方面：


  1. MNN + NPU：NPU的運行速度快了 CPU 好幾倍，而且能耗有優勢。是未來的主流。而且在可預見的未來，NPU的碎片化比較嚴重。MNN目前支持了華為的海思NPU，以後會支持更多的 NPU ，並且會提供一個統一的 API ，降低碎片化。
  2. MNN 小程序：小程序平臺降低了外部客戶把他們的軟件寫入手淘的門檻。以後，手淘裡面的端智能能力，可以通過小程序透出給外部開發者，使用簡單，功能強大。今年的雙11，modiface 與我們合作，通過小程序使用 MNN ，巨大提升了運行速度（相比於 TFJS ），降低了內存使用。未來，更多的外部客戶可以在他們的小程序裡調用 MNN ，簡單、高效地將機器學習帶到小程序裡。
  3. 模型壓縮：前文也提到了 NLP 應用是未來的一大趨勢。而 NLP 的模型普遍大於 CV 模型。在這個時候，大幅度地壓縮模型，能夠讓之前只能在服務器運行的模型放到端上運行。所以未來的 MNN ，會提供更好的模型壓縮。


結語


坊間傳言，許多人在眾多推理引擎中選擇 MNN 的原因是它快。而 MNN 並沒有那麼易用。這些用戶為了能夠使用 MNN 將他們的模型部署到 app 中所做的事情，可謂翻山越嶺。


2020年，我們希望 MNN 與用戶們一起成長，能夠真正成為好用、高效的機器學習引擎，幫助開發者把端智能玩進自己的 app 中。
