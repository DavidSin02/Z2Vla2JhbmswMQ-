「數據管理」頂級數據庫管理系統供應商

2020-11-14
| 科技

確定哪種類型的數據庫或數據庫服務最適合您的企業的最佳方法是什麼？這完全取決於您需要什麼類型的用例。在本文中瞭解更多信息。

基本上，我們每天使用的所有數字信息都在世界某處的數據庫管理系統或存儲陣列中。這些存儲設備可以小到智能手機，也可以大到基本上不受限制的雲存儲系統。

如何最好地找出哪些DBMS適合您的企業？你應該訂閱AWS、Azure、Google或其他雲服務提供商提供的服務，還是購買數據中心存儲和服務器並自己運行？這完全取決於您需要哪種類型的用例；例如，如果您是金融服務、醫療保健或國防部門的受監管行業，則可能需要同時安裝這兩種類型的用例。如果你是一個小企業，也許你需要的只是雲服務。這裡的大多數公司都提供這兩種選擇。

自20世紀80年代《個人電腦周》（PC Week）出版以來，eWEEK一直在研究和報告數據庫及其管理系統，當時IBM的DB2、微軟的SQL Server和Sybase是該行業的大腕。在這篇文章中，我們找到並評估了2019年前十大現代數據庫管理系統，包括專有和開源系統，並將它們編譯到本文中。

Oracle RDMS

加州紅木海岸。

潛在買家的價值主張：大型、功能強大且相對昂貴的條款通常附屬於Oracle的企業數據庫，但你可以得到你所付出的代價。

甲骨文在這一領域統治了30多年，該公司在這一領域已有42年的歷史。甲骨文設計了其數據庫硬件和軟件，以便在雲端和數據中心協同工作。該公司聲稱，這消除了複雜性，簡化了一般性。這通常是正確的，但是如果用例和環境發生變化，用戶通常會被鎖定在一個單一的供應商系統中，以後很難更改。

甲骨文對它的所有層都採用開放標準的方法，但它需要本地企業it人員的專業知識才能在預先配置的甲骨文系統之外更換各種組件，而且許多中小型企業都沒有這種專業知識。定價sla也會發生變化。複雜度較低，維護較少的點解決方案和一般一流的性能的權衡往往過於壓倒許多企業忽視。

作為一家早年對雲系統嗤之以鼻的公司，它現在是一家積極銷售基於雲的DBMS系統的公司，以配合其Exadata數據中心服務器。時代確實變了，甲骨文也跟著時代變了。

對於需要模塊化解決方案的客戶，Oracle的開放式體系結構和多個操作系統選項提供了來自堆棧每一層中同類最佳產品的無與倫比的好處。這允許客戶為其企業構建儘可能優化的基礎架構。

關鍵價值/差異：

  • Oracle SQL的基於UNIX的數據庫管理系統提供了在任何操作系統中選擇運行企業數據庫的靈活性。專用語言僅與同一製造商的操作系統兼容。例如，只能在基於Windows的計算機上運行Microsoft SQL Server。相比之下，您可以在Unix服務器上安裝Oracle SQL，在保持SQL標準化的同時，還可以從Unix的可靠性中獲益。
  • Unix不易受到許多常見的計算機病毒的攻擊，從而保證信息的安全。
  • Oracle SQL也是向後兼容的，因此用戶可以選擇在將來升級而不丟失任何數據。
  • 對於需要模塊化解決方案的客戶，Oracle的開放式體系結構和多種操作系統選項提供了來自堆棧每一層中同類最佳產品的好處。這允許客戶為其企業構建儘可能優化的基礎架構。

路線圖：

  • Oracle數據庫管理系統每年更新一次或兩次，並定期迭代發送。

誰使用它：中大型企業

工作原理：雲部署、物理on-prem服務

埃韋克分數：4.9/5.0

Oracle提供的MySQL

加州紅木海岸。

潛在買家的價值主張：Oracle擁有的MySQL是一個流行且廣泛使用的開源關係數據庫管理系統。

這是一個節省成本和有效的數據庫管理系統，但它需要專門知識的數據庫系統安裝和維護。它的名字是“My”和“SQL”的組合，前者是聯合創始人MichaelWidenius的女兒，後者是結構化查詢語言的縮寫。MySQL是GNU通用公共許可條款下的免費開源軟件，也可以在各種專有許可下使用。MySQL是由瑞典MySQL公司創建和贊助的，該公司被Sun Microsystems收購。2010年，當甲骨文收購Sun時，Widenius將開源MySQL項目分給了MariaDB。

關鍵價值/差異：

  • 它有一個令人印象深刻的用戶列表。MySQL被許多數據庫驅動的web應用程序使用，包括Drupal、Joomla、phpBB和WordPress。MySQL也被許多流行網站使用，包括Facebook、Twitter、Flickr和YouTube。
  • MySQL長期以來一直受到好評，評論人士稱它在一般情況下表現非常好，開發人員界面也在那裡，文檔（更不用說通過網站等在現實世界中的反饋）非常非常好。
  • 它還被測試為一個快速、穩定、真正的多用戶、多線程的SQL數據庫服務器。
  • 它的開源核心和屬性支持用例所需的任意數量的配置。
  • 甲骨文擁有一支龐大而專業的支持人員隊伍，可以與MySQL客戶合作。
  • MySQL在大多數企業服務器上工作，它們不必是Oracle服務器。

路線圖：

  • MySQL每年都會從Oracle的團隊那裡獲得幾次更新，並且會定期迭代發送。

誰使用它：中小企業到大企業

工作原理：訂閱雲服務、物理on-prem服務

埃韋克分數：4.9/5.0

Microsoft SQL服務器

雷蒙德，華盛頓。

潛在買家的價值主張：這是一個行業標準的數據庫管理系統。

Microsoft SQL Server的歷史始於第一個microsoftsqlserver產品（SQL Server 1.0，1989年用於IBM OS/2操作系統的16位服務器），一直延續到今天。微軟重新開發了Microsoft SQL Server，使其能夠與自己的Windows操作系統最佳地協同工作。與其他數據庫管理系統類似，它的主要功能是根據其他軟件應用程序的請求存儲和檢索數據，這些應用程序可以運行在同一個數據中心上，也可以運行在網絡（包括internet）上的另一臺計算機上。事實上，近一半的微軟SQL Server實例部署在微軟的Azure雲中。

關鍵價值/差異：

以下是自2019年4月起SQL Server的新功能列表。

  • 微軟已經開發了至少十幾個版本的Microsoft SQL Server，針對不同的受眾和工作負載，從小型的單機應用程序到大型的面向Internet的應用程序，這些應用程序有許多併發用戶。因此，它是迄今為止最通用的MySQL部署。
  • SQL Server企業版：這包括核心數據庫引擎和附加服務，以及一系列用於創建和管理sqlserver集群的工具。它可以管理高達524petabytes的數據庫，處理12tb的內存，並支持640個邏輯處理器（CPU核）。
  • 標準版：SQL Server標準版包括核心數據庫引擎和獨立服務。它不同於企業版，因為它支持較少的活動實例（集群中的節點數），並且不包括一些高可用性功能，如熱添加內存（允許在服務器仍在運行時添加內存）和並行索引。Web SQL Server Web版是一個低TCO的Web宿主選項。
  • 商業智能：在SQL Server 2012中引入，專注於自助服務和企業商業智能。它包括標準版功能和商業智能工具：PowerPivot、Power View、BI語義模型、主數據服務、數據質量服務和xVelocity內存分析。

路線圖：

  • SQL Server每年會獲得一到兩次主要更新，今年晚些時候將發佈一個新版本，其中包括智能查詢處理、大數據集群和更多功能。

誰使用它：中小企業、中端服務器、邊緣服務器、大型企業

工作原理：訂閱雲服務和物理on-prem服務

埃韋克分數：4.7/5.0

PostGres SQL

費城,賓夕法尼亞州。

對潛在購買者的價值定位:PostgreSQL是一個對象關係數據庫管理系統，強調可擴展性和標準遵從性。它可以處理許多併發用戶的工作負載，從單機應用程序到Web服務或數據倉庫。PostgreSQL是由PostgreSQL全球開發小組開發的，該小組由許多公司和個人貢獻者組成。它是自由和開放源代碼的軟件，在一個寬鬆的軟件許可下發布。

鍵值/差異:

  • PostgreSQL是跨平臺的，可以在許多操作系統上運行，包括Linux、FreeBSD、Solaris和Microsoft Windows。它可以處理從小型單機應用程序到具有許多併發用戶的大型面向internet的應用程序等各種工作負載。最近的版本還提供了數據庫本身的複製，以保證安全性和可伸縮性。
  • PostgreSQL支持ANSI SQL和SQL/MED等標準，但具有高度可擴展性，支持12種以上的過程語言、GIN和GIST索引、空間數據支持，以及基於文檔或鍵值的應用程序的多個類似於SQL的特性。
  • PostgreSQL是acid兼容和事務性的。它提供了對RDBMS特性的支持，如可更新和物化視圖、觸發器、外鍵;函數和存儲過程。

路線圖:

  • PostgreSQL每年都會有幾次主要的更新。

誰在使用它:中小型企業、中型企業、邊緣服務器、大型企業

它是如何工作的:訂閱雲服務，物理預發佈服務

eWEEK評分:4.8/5.0

MongoDB

紐約,紐約

對潛在購買者的價值定位:MongoDB是一個開源的、跨平臺的面向文檔的數據庫管理系統。

作為一個NoSQL數據庫管理系統，MongoDB使用類似json的文檔和模式。MongoDB由MongoDB公司開發，並在服務器端公共許可證(SSPL)下授權。作為一種開源產品，定價對於小公司來說要合理得多，這與大型、成熟的數據庫供應商提供的全服務是不同的。

鍵值/差異:

  • MongoDB可以運行在多個服務器上，平衡負載或複製數據以保持系統正常運行，以防硬件故障。
  • MongoDB提供高可用性的副本集，其中包含兩個或多個副本的數據。每個副本集成員可以在任何時候充當主副本或次副本的角色。默認情況下，所有的寫和讀都是在主副本上完成的。
  • 輔助副本使用內置的複製來維護主副本的數據副本。當主副本失敗時，副本集將自動執行一個選擇過程，以確定哪個輔助副本應該成為主副本。二級服務器可以選擇性地提供讀操作，但是默認情況下這些數據最終是一致的。
  • MongoDB使用分片進行水平擴展(分片是一個擁有一個或多個副本的主機)。

路線圖:

  • MongoDB每年都會有幾次重大更新。

誰在使用它:中型到大型企業

它是如何工作的:訂閱雲服務，物理預發佈服務

eWEEK評分:4.8/5.0

IBM DB2

紐約州阿蒙克市

潛在購買者的價值主張:IBM DB2是一種行業標準數據庫管理系統。

DB2代表了一組完整的數據管理系統，包括可在雲環境中使用的服務器，這些服務器最初是由IBM在20世紀80年代早期開發的。它們支持關係數據庫模型，但近年來，一些產品已經擴展為支持對象關係特性和非關係結構，如JSON和XML。從1983年創建到2017年，該品牌被命名為DB2。

IBM在2019年為Db2制定的目標是成為幫助增強認知應用程序的人工智能數據庫。IBM混合數據管理(HDM)是在Db2公共SQL引擎上構建的，它提供了一個平臺來跨所有源和目標管理所有數據類型。

由於該軟件的高可靠性、彈性和安全性等特點，成千上萬的組織使用IBM混合數據管理和db2(在線事務處理(OLTP)、在線分析處理(OLAP)和大數據段中的市場支柱)來運行關鍵任務應用程序。IBM打算使用Db2(混合數據管理平臺的核心組件)讓用戶能夠加速他們的AI應用程序開發，同時自動化一些數據管理。

IBM Db2現在為流行的數據科學語言和框架提供了驅動程序，包括Go、Ruby、Python、PHP、Java和Node。使開發人員和數據科學家能夠第一次使用Db2數據構建AI應用程序。這些驅動程序現在可以在GitHub上使用。

鍵值/差異:

  • IBM的商標，在幾十年的產品開發和服務中建立起來的聲譽，在所有數據中心繫統軟件和設備中意義重大。
  • DB2系統的一個重要特性是錯誤處理。SQL communications area (SQLCA)結構曾經專門用於DB2程序中，在執行每條SQL語句之後將錯誤信息返回給應用程序。主要的(但不是特別有用的)錯誤診斷位於SQLCA塊中的SQLCODE字段中。

路線圖:

  • DB2每年進行一次或兩次重大更新，並根據需要進行增量修復。

誰在使用它:中小型企業到大型企業

它是如何工作的:雲服務，物理預啟動服務

eWEEK評分:4.8/5.0

Microsoft Access

華盛頓州雷德蒙德

潛在買家的價值主張:Microsoft Access是第二代數據庫管理系統(DBMS)，它將關係型Microsoft Jet數據庫引擎與圖形用戶界面和它自己的一套軟件開發工具結合在一起。

它是Microsoft Office應用程序套件的成員，包含在專業版和高級版中，或單獨出售。Microsoft Access基於Access Jet數據庫引擎以自己的格式存儲數據。它還可以導入或直接鏈接到存儲在其他應用程序和數據庫中的數據。與其他Microsoft Office應用程序一樣，Visual Basic for applications (VBA)支持訪問，這是一種基於對象的編程語言，可以引用各種對象，包括DAO(數據訪問對象)、ActiveX數據對象和許多其他ActiveX組件。窗體和報表中使用的可視化對象在VBA編程環境中公開它們的方法和屬性，VBA代碼模塊可以聲明和調用Windows操作系統操作。

鍵值/差異:

  • 除了用作自己的數據庫存儲文件之外，Microsoft Access還可以用作程序的前端，而其他產品用作後端表，如Microsoft SQL Server和非Microsoft產品，如Oracle和Sybase。Microsoft Access Jet數據庫(ACCDB和MDB格式)可以使用多個後端源。
  • 類似地，一些應用程序如Visual Basic, ASP。NET或Visual Studio .NET將對其表和查詢使用Microsoft Access數據庫格式。Microsoft Access也可能是更復雜的解決方案的一部分，它可能與其他技術集成，如Microsoft Excel、Microsoft Outlook、Microsoft Word、Microsoft PowerPoint和ActiveX控件。
  • 訪問表支持各種標準字段類型、索引和引用完整性，包括級聯更新和刪除。訪問還包括查詢接口、用於顯示和輸入數據的表單以及用於打印的報告。包含這些對象的底層Jet數據庫是多用戶的，它處理記錄鎖定。
  • 重複的任務可以通過帶有指向和單擊選項的宏實現自動化。在網絡上放置數據庫，讓多個用戶共享和更新數據，而不覆蓋彼此的工作，這也很容易。數據被鎖定在記錄級別，這與Excel鎖定整個電子表格有很大的不同。

路線圖:

  • Microsoft give Access每年獲得一到兩次重大更新，並根據需要進行增量修復。預計今年不會有重大更新。

誰在使用它:中型到大型企業

它是如何工作的:雲服務，物理預啟動服務

eWEEK評分:4.6/5.0

加州山景城。

Apache Cassandra

門洛帕克,加利福尼亞州。

對潛在買家的價值定位:Apache Cassandra最初是在Facebook開發的，用於支持其收件箱搜索功能，現在是世界上領先的內存開源數據庫管理系統之一。它是一個免費的、開源的、分佈式的、寬列存儲的NoSQL數據庫管理系統，設計用於跨許多普通服務器處理大量數據，提供高可用性，沒有單點故障。Cassandra為跨多個數據中心的集群提供了健壯的支持，異步無主複製允許所有客戶端進行低延遲操作。

鍵值/差異:

  • 集群中的每個節點都具有相同的角色。沒有單一的失敗點。數據分佈在集群中(因此每個節點包含不同的數據)，但是沒有主節點，因為每個節點都可以為任何請求提供服務。
  • 支持複製和多數據中心複製:複製策略是可配置的。Cassandra被設計為一個分佈式系統，用於跨多個數據中心部署大量節點。Cassandra的分佈式架構的關鍵特性是專門為多數據中心部署、冗餘、故障轉移和災難恢復而定製的。
  • 可伸縮性:設計為讀寫吞吐量均隨著新機器的添加而線性增加，目標是不停機或不中斷應用程序。
  • 容錯:自動將數據複製到多個節點以實現容錯。支持跨多個數據中心的複製。失敗的節點可以替換為沒有停機時間。
  • 可調一致性:Cassandra通常歸類為一個美聯社系統,這意味著可用性和分區容忍通常被認為是更重要的比一致性在卡桑德拉,寫入和讀取的提供一個可調水平的一致性,從“寫永遠不會失敗”“阻止所有副本可讀,”中間的群體水平。
