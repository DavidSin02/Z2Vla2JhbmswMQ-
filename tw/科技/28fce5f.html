Spring Statemachine 狀態機入門

2020-10-29
| 科技

什麼是有限狀態機

有限個狀態，並且可以在這些狀態之間進行轉移並執行相應的動作的行為模型。

狀態機作用

舉例：十字旋轉門多用於旅遊景地。而且是狀態機的典型案例。

進：刷卡->推動->進->鎖定

我們以前代碼：If(投幣) 解鎖 else if(推) 進入，鎖定。

用了狀態機：我們把狀態和事件進行分離（這是核心）。

State {解鎖，鎖定 }

Event {投幣，推}

狀態機使用場景：可以用枚舉業務場景，用大量if判斷業務狀態並進行下一步業務狀態流轉。

Spring Statemachine 狀態機

系統要求：

Spring Statemachine 是使用JDK 8（所有工件都具有JDK 7兼容性）和Spring Framework構建和測試的。它不需要在其核心繫統內的Spring Framework之外的任何其他依賴項。

配置類

Action：事件被相應後相執行轉換之間的動作

Guard：哨兵 用於是否可以從source到target的轉換，如果true,就執行Action，並且把當前狀態轉到target，如果false,當前狀態還停留在source上。

statecontext包含了上下文信息，你可以傳自定義對象，還可以從中獲取源狀態，目標狀態，當前事件，等

source:源狀態

target:目標狀態

event:相應的事件

withExternal: 外部轉換，有狀態轉移.

運行類

相關名詞解釋

Transition: 轉換是源狀態和目標狀態之間的關係。它可能是複合轉換的一部分，它將狀態機從一個狀態配置帶到另一個狀態配置，表示狀態機對特定類型事件發生的完整響應。

Initial State:初始狀態

End State：業務結束狀態

Fork State: 可以從一個狀態進入兩個以上的目標狀態

Join State：從多個源狀態到一個目標狀態。

狀態機事件

OnStateMachineStart :Statemachine啟動時相應的事件

OnStateEntry：從一個狀態進入時相應的事件

OnStateExit：從一個狀態退出時相應的事件

OnTransition：從一個狀態轉換到另一個狀態的事件

OnStateMachineError：狀態機出錯時事件
