前言

如果你剛開始學習 Vue，想鞏固基礎知識，那麼你可以試試通過這個有趣的練習來創建一個好玩的遊戲。

在這篇文章中，我將逐步教你用 Vue.js 創建一個記憶卡片遊戲。

這篇文章會介紹以下知識點：

- 使用 v-for 命令循環遍歷一個數組對象

- 使用 v-bind 指令動態控制類名和樣式

- 添加 Methods 和 Computed 屬性

- 通過 Vue.set 方法向一個對象動態添加屬性

- 使用 setTimeout 方法延遲 JavaScript 插件加載

- JavaScript 對象的淺拷貝和深拷貝

- 使用 Lodash 工具庫

我們開始學習吧。

匹配和洗牌


對，沒錯！使用這些卡片製作一個記憶遊戲，我們需要每張卡片都正好有跟它匹配的卡片。同時我們也需要在遊戲開始時清洗卡片的排列順序。


在 Vue 實例中添加一個 memoryCards 屬性，存放翻轉過的卡片（也就是說，每張翻轉後的卡片，都有另一張卡片和它背面的圖片）。

匹配


將卡片數組拼接起來，賦值給 memoryCards 屬性，以實現每兩張卡片背面的圖片相同。


改變在 HTML 中的 v-for 指令遍歷對象，讓它從原來的 cards 數據改變到 memoryCards 數組。

然後，修改 created 方法，將數組拼接到 memoryCards 中：

 created(){        this.cards.forEach((card) => {            Vue.set(card,'isFlipped',false)        });        var cards1 = this.cards;        var cards2 = this.cards;        this.memoryCards = this.memoryCards.concat(cards1, cards2);    },


是不是挺簡單？


但這麼做有兩個問題：

- 直接讓 cards1 等於 this.cards，並不能產生一個新的用於匹配的 cards 對象，cards1 指向的仍然是原始對象。

- cards1 和 cards2 指的是同一個對象。


改變 memoryCards 對象中的任何屬性會引起兩個匹配的數組同時變化。

我們可以試試用深拷貝來解決這個問題。

什麼是深拷貝？

對於對象或數組中包含其他對象和數組的情況，要想拷貝這些元素需要通過深拷貝。否則，當改變嵌套引用上的數據時，原始對象和數組中的數據也會發生改變。

進行深拷貝的方法有很多，我們將使用最簡單也是最常用的方法，使用 Lodash 庫。

那麼，什麼是 Lodash 庫?


Lodash 處理了對數組、數字、對象、字符串等類型的一些複雜操作，讓 JavaScript 變得更方便使用。


在這個例子中，Lodash 有一個方法能讓深拷貝的操作變得極其簡單。


首先，通過下載源碼或者使用 CDN 引用將 Lodash 包含在頁面中。


 <script src="https://cdn.jsdelivr.net/npm/lodash@4.17.15/lodash.min.js"></script>


接下來，使用 Lodash 的 cloneDeep 方法對卡片數組進行深拷貝。

 var cards1 = _.cloneDeep(this.cards); var cards2 = _.cloneDeep(this.cards); this.memoryCards = this.memoryCards.concat(cards1, cards2);


洗牌


現在我們將打亂組合起來的數組。Lodash 有一個可以方法同樣可以打亂數組。我們使用這個方法，然後，為了簡化代碼，我們可以將拼合數組和打亂數組寫在一行中。


 created(){        this.cards.forEach((card) => {            Vue.set(card,'isFlipped',false)        });        this.memoryCards = _.shuffle(this.memoryCards.concat(_.cloneDeep(this.cards), _.cloneDeep(this.cards)));    },


現在就能實現洗牌和正常的翻轉啦。


匹配卡片


下一步是匹配翻轉的卡片。用戶一次最多可以翻轉兩張卡片。如果背面的圖片相同，那麼匹配成功。如果不同，再翻轉到正面。


我們給每張卡片增加一個新的屬性，記錄卡片是否已經匹配：


 this.cards.forEach((card) => {    Vue.set(card,'isFlipped',false);    Vue.set(card,'isMatched',false);});


創建一個新的屬性用於存放翻轉的卡片：

這段邏輯很簡單：我們持續往 flippedCards 數組中添加卡片，直到出現兩張一樣的為止。


當出現兩張背面圖片相同的卡片時，我們執行匹配的邏輯：

- 如果兩張卡片的名字一樣，我們通過設置 isMatched 屬性為 true 來標記卡片匹配

- 否則，設置 isFlipped 屬性為 false


之後我們清空 flippedCards 數組。


在匹配的卡片上添加新的 CSS 屬性讓其淡出：


 .matched{   opacity: 0.3;}


該組件容器 class 上綁定屬性，當卡片匹配時設置值為 true：


 :class="{ 'flipped': card.isFlipped, 'matched' : card.isMatched }"


這裡運行時正常，但是邏輯判斷等都發生得太快了，以致於用戶根本不知道發生了什麼。如果卡片不匹配，它們甚至會在用戶看到顯示的卡片背面之前就向後翻轉。


我們使用 JavaScript 的 setTimeout 方法來添加一些延遲效果。

我們在卡片被標記匹配之前添加了 0.4 秒的延遲，以及卡片在返回時添加了 0.8 秒的延遲。

- 卡片已經翻轉

- 用戶已經翻轉兩張卡片
