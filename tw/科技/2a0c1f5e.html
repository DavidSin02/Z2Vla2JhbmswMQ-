淺淡編程入門為什麼要從C語言開始之我見？

2020-11-14
| 科技

談選擇，當然要考慮到選擇出發點、個人情況和環境。是入門還是職業考慮？

入門可以較多地考慮興趣點。如果喜歡弄個網頁，Javacript不錯，如果想做個爬蟲，Python也不錯。如果你是個計算機發燒友，對編程本身又很執迷，當然選擇C了。

如果是職業考慮呢？當然也分當下和為以後做打算，你去看一下大廠的需求或tiobe的排名就較清楚！但是，做為職業玩家，一般都是較精通一門，需要熟悉多名編程語言的。

另外，就編程來說，編程語言是工具、編程思想才是核心，數據結構和算法才是內功，內功是潛在的，於細節處體現你代碼的優雅和高效。

1 從歷史來看

首先要看tiobe.com的最新排名，C本身的排名可不低。

其次，在流行語言中，C的歷史最為悠久，按歷史的脈絡來看，事物的發展總是由簡到繁，而事實也是如此。對於學習一個東西來說，按由簡到繁是一個較佳的選擇。

需要知道的是，C++是C的後續，Java是C的後續。

2 從應用領域來看

為什麼有多種編程語言？因為，現實世界的複雜性，不可能一門語言能擅長（高效）解決各個領域的所有問題。編程語言開發者都是因為某個領域某個點的需要，才在某個語言的基礎上進行改進和完善的。所以，流行的語言都有自己的擅長領域。

C語言是系統開發語言，特別適合理解計算說明的底層。

3 從面向過程與面向對象的編程思想來看

面向過程的程序設計(Procedure-Oriented Programming，簡記為POP)，就是分析出解決問題所需要的數據和整個過程，將過程分解逐層分解為步驟，然後用函數把這些步驟一步一步實現，使用的時候一個一個依次調用就可以了，最有代表性的當然是C語言了。

面向對象程序設計(Object Oriented Programming)是把構成問題事務分解成各個對象，相關的數據體現為對象的屬性（數據成員），讓解決問題的步內聚到對象的行為（或方法，也是函數）中，然後通過對象再去調用其成員數據和函數。最有代表性的當然是c#的當然是java、c#等語言了，而c++則兼有兩者的特點。

POP最先獲得了最廣泛的應用，然後才是OOP。

對比一下兩者的代碼：

類（面向對象最重要的數據抽象）就是一種更高級（或更抽象）的結構體（面向過程最重要的數據抽象）。

在對比中學也是一種有效的學習方法，可以融會貫通，相互借鑑。

對於小程序來說，根本就沒必要OOP。對於大程序來說，OOP較佳。就果生產一個產品，如果是小產品，直接構建，如果是一個複雜的大產品，可以考慮組裝。

另外，OOP因為封裝的需要，天然就有一些性能損耗，所以在嵌入式領域，POP更佳。

從抽象層次來看，OOP是更高層次的抽象，理解起來更困難，如果有了POP的理解，在POP以函數為模塊來構建程序有了一定的認識，再來認識OOP就顯得較容易了。

20世紀60年代以來，編程語言的理論已經出現了幾次進步。編程語言的範式也已經發生了不少更替：

第一代編程語言 (first generation languages)：機器語言(打孔機)

第二代編程語言 (second generation languages)：彙編語言

第三代編程語言 (third generation languages):分為1 面向過程的編程語言 (如C、BASIC、Pascal);2 面向對象的編程語言(如C++、java、c# )

結構化程序設計 (Structured Programming)

功能分解法 (functional decomposition)

自頂向下設計,逐步求精 (top-down design and step-wise refinement)

自底向上設計 (bottom-up design)

迭代式設計 (iterative design)

面對對象程序設計 (object-oriented programming)

組件編程 (components)

編程模式 (programming patterns)

切面編程 (aspect programming)：是Spring框架中的一個重要內容，是函數式編程的一種衍生範型。利用AOP可以對業務邏輯的各個部分進行隔離，從而使得業務邏輯各部分之間的耦合度降低，提高程序的可重用性，同時提高了開發的效率。

第四代編程語言 (fourth generation languages)：數據庫編程語言，如SQL，只需要告訴計算機需要做什麼, 之前的編程語言是告訴計算機工如何做，是一種非過程化程序設計語言。

第五代編程語言 (fifth generation languages)就是自然語言，又被稱為知識庫語言或人工智能語言，目標是最接近日常生活所用語言的程序語言。真正意義上的第五代語言尚未出現，LISP和PROLOG號稱第五代語言，其實還遠遠不能達到自然語言的要求。

4 從瞭解底層看

我們知道，我們現在所使用的計算機都是馮諾依曼機，馮諾依曼機的核心思想是存儲程序控制概念，包括程序存儲和程序控制兩個方面。程序存儲到內存，從內存中讀取數據和代碼來控制流程及對數據的處理。

C是最接近底層的高級語言，有最直接的內存管理，有系統開發語言之稱。看以下的編譯流程，C可以直接彙編到彙編代碼，可以更好地瞭解到代碼在計算機上的運行機制，這是其它高級語言所不能的。

5 從數據結構理解來看

要真正理解數據數據結構，數據的鏈式存儲是很重要的，很難想像，沒有指針的鏈式結構總是覺得很彆扭。可以看到的是，很多關於數據結構和算法的書籍，都是用C或C++來實現的。

6 多種語言的需要

一門語言是通向另外一個世界的一個窗口，計算機語言也是。一個真正的程序員，不可以只會一門語言，甚至兩門語言都不夠，多門語言可以融匯貫通，借鑑各自的編程思想。但不可否認的是，學第一門編程語言難一點，學第二門編程語言會容易一點，學第三名編程語言會更容易一些。

7 綜合瞭解各流行編程語言

Java

Java是一門面向對象編程語言，不僅吸收了C++語言的各種優點，還摒棄了C++裡難以理解的多繼承、指針等概念，因此Java語言具有功能強大和簡單易用兩個特徵。Java語言作為靜態面向對象編程語言的代表，極好地實現了面向對象理論，允許程序員以優雅的思維方式進行復雜的編程。

Java具有簡單性、面向對象、分佈式、健壯性、安全性、平臺獨立與可移植性、多線程、動態性等特點。Java可以編寫桌面應用程序、Web應用程序、分佈式系統和嵌入式系統應用程序等。

Python

Python是純粹的自由軟件，源代碼和解釋器CPython遵循GPL（GNUGeneral Public License）許可。Python語法簡潔清晰，特色之一是強制用空白符（white space）作為語句縮進。

Python具有豐富和強大的庫。它常被暱稱為膠水語言，能夠把用其他語言製作的各種模塊（尤其是C/C++）很輕鬆地聯結在一起。常見的一種應用情形是，使用Python快速生成程序的原型（有時甚至是程序的最終界面），然後對其中有特別要求的部分，用更合適的語言改寫，比如3D遊戲中的圖形渲染模塊，性能要求特別高，就可以用C/C++重寫，而後封裝為Python可以調用的擴展類庫。需要注意的是在您使用擴展類庫時可能需要考慮平臺問題，某些可能不提供跨平臺的實現。

C

C語言是一門通用計算機編程語言，應用廣泛。C語言的設計目標是提供一種能以簡易的方式編譯、處理低級存儲器、產生少量的機器碼以及不需要任何運行環境支持便能運行的編程語言。

儘管C語言提供了許多低級處理的功能，但仍然保持著良好跨平臺的特性，以一個標準規格寫出的C語言程序可在許多電腦平臺上進行編譯，甚至包含一些嵌入式處理器（單片機或稱MCU）以及超級電腦等作業平臺。

C++

C++是一種面向對象的計算機程序設計語言，由美國AT&T貝爾實驗室的本賈尼·斯特勞斯特盧普博士在20世紀80年代初期發明並實現（最初這種語言被稱作“C with Classes”帶類的C）。它是一種靜態數據類型檢查的、支持多重編程範式的通用程序設計語言。它支持過程化程序設計、數據抽象、面向對象程序設計、泛型程序設計等多種程序設計風格。

C++是C語言的繼承，進一步擴充和完善了C語言，成為一種面向對象的程序設計語言。C++這個詞在中國大陸的程序員圈子中通常被讀做“C加加”，而西方的程序員通常讀做“C plus plus”，“CPP”。

Javascript

Javascript[1]是一種由Netscape的LiveScript發展而來的原型化繼承的面向對象的動態類型的區分大小寫的客戶端腳本語言，於是Netscape的瀏覽器Navigator加入了Javascript，提供了數據驗證的基本功能，主要目的是為了解決服務器端語言，比如Perl，遺留的速度問題，為客戶提供更流暢的瀏覽效果。當時服務端需要對數據進行驗證，由於網絡速度相當緩慢，只有28.8kbps，驗證步驟浪費的時間太多。

C#

C#是微軟公司發佈的一種面向對象的、運行於.NET Framework之上的高級程序設計語言。並定於在微軟職業開發者論壇(PDC)上登臺亮相。C#是微軟公司研究員Anders Hejlsberg的最新成果。C#看起來與Java有著驚人的相似；它包括了諸如單一繼承、接口、與Java幾乎同樣的語法和編譯成中間代碼再運行的過程。但是C#與Java有著明顯的不同，它借鑑了Delphi的一個特點，與COM（組件對象模型）是直接集成的，而且它是微軟公司 .NET windows網絡框架的主角。
