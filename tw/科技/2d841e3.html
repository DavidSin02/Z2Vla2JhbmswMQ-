Git實戰005：如何正確的使用同步、獲取和拉取指令

2020-10-29
| 科技

為了更好的管理、跟進、掌控和維護項目開發，我們通常會借用Gitlab等代碼託管平臺來管理項目資源。當然我們不會將項目公開在互聯網上，而是在局域網搭建屬於自己的Git遠程倉庫服務（我司搭建的是Gitlab服務）。在使用Git的使用我們經常需要對項目執行同步、獲取和拉取等操作，那麼我們在實際應用中該如何使用相關指令呢？

克隆資源clone

如果你是剛剛參與項目的開發那麼你需要先獲取當前已經開發的項目資源，這時候就需要從從遠程倉庫克隆資源。執行指令： git clone -b <分支名> <版本庫的網址>；將指定分支克隆到本地新目錄中（不指定分支則默認克隆master分支），執行git clone後會在新目錄下有個.git的隱藏文件夾，所以clone下來的項目不需要進行git init初始化操作。

查看分支branch

在Git中分支一般有本地版本庫分支和遠程版本庫分支，在項目目錄中查看本地版本庫分支可以執行指令：git branch，如果查看遠程版本庫分支可以執行指令：git branch -r，如果想同時查看本地和遠程倉庫的分支情況可以執行指令：git branch -al。通常本地分支顯示為綠色而遠程分支顯示為紅色，前面帶符合“*”表示當前所在的分支。

同步分支

這裡我們可以看到遠程版本庫中還有一個dev分支，而本地版本庫目前只有master分支。現在我想把dev分支也取回，這個我們可以通過指令：git checkout -b <本地分支名> <origin/遠程分支名>將遠程倉庫裡的指定分支拉取到本地，並同時在本地新建了一個和指定遠程分支關聯的dev分支。該指令在本地倉庫新建分支並自動切換到該分支，並自動建立的本地分支會和遠程分支的映射關係。將本地所有分支與遠程版本庫保持同步 git fetch --all，拉取遠程版本庫所有分支 git pull --all。

​獲取狀態fetch

當用戶向遠程倉庫提交更新之後我們需要去獲取當前項目的最新資源，這時候我們需要知道當前資源做了哪些更新。通過指令：git fetch <遠程主機名> <分支名>可以將遠程版本庫的特定分支更新取回，如果不指定分支則將遠程主機的更新全部取回本地本地版本庫。取回更新後會返回一個FETCH_HEAD 指針指向branch在服務器上的最新狀態，我們可以在本地通過指令：git log -p FETCH_HEAD查看當前指針所在分支取回的更新信息。返回的信息包括更新的文件名、更新的作者和時間以及更新的內容，我們可以通過這些信息來判斷是否產生衝突以確定是否將更新merge到當前分支。

​合併分支merge

fetch只是將當前項目分支在遠程倉庫上的最新狀態取回，此時只是將狀態存在了本地暫存庫中並沒有與本地分支進行合併，所以在本地並沒有還看不到遠程倉庫上更新的內容。fetch可以幫助我們查看更新的內容是否與已有資源產生衝突，這樣我們就可以選擇分支更新進行合併。合併分支可以通過指令：git merge <遠程主機名> /<分支名>將指定分支上拉取下來的最新內容合併到當前所在的分支中，如果不放心可以通過指令：git diff <分支名>來比較本地代碼與剛剛從遠程下載下來的代碼的區別。

​拉取分支pull

如果你不需要檢查遠程倉庫的具體更新信息時我們可以直接使用指令：git pull <遠程主機名> <遠程分支名>:<本地分支名>取回遠程倉庫指定分支的更新並與本地分支合併，如果不指定本地分支則默認拉取到本地的當前分支進行合併。git pull相當於執行git fetch後並調用git merge將檢索到的分支頭合併到分支中，相比起來git fetch更安全一些，雖然麻煩些（需要手動合併）但是可以在merge前查看更新情況然後再決定是否合併。

​總結

同步顧名思義就是保持本地版本庫與遠程版本庫的信息一致，當本地有新分支而遠程倉庫沒有、遠程倉庫有新分支而本地沒有、本地刪除了分支時遠程也需要刪除、遠程刪除了分支時本地也同時刪除時就可以用同步來解決。其他的局部資源更新則可以根據實際情況來拉取，如果需要對更新進行查閱那麼就先進行獲取然後再手動合併分支即可。
