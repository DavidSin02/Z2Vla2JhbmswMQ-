1）比較指令


比較數據類型相同的兩個值。該 LAD 觸點比較結果為 TRUE 時，則該觸點會被激活。如果該 FBD 功能框比較結果為 TRUE，則功能框輸出為TRUE。

注：對於 LAD 和FBD：單擊指令名稱（如“==”），以從下拉列表中更改比較類型。單擊“???”並從下拉列表中選擇數據類型。

2）參數的數據類型

參數 數據類型 說明

IN1, IN2 Byte, Word, DWord, SInt, Int, DInt, USInt, UInt, UDInt,

Real, LReal, String, WString, Char, Char, Time, Date, TOD, DTL, 常數 要比較的值

3）比較說明

關係類型 滿足以下條件時比較結果為真 ...

= IN1 等於 IN2

<> IN1 不等於 IN2

>= IN1 大於或等於 IN2

<= IN1 小於或等於 IN2

> IN1 大於 IN2

< IN1 小於 IN2

4）編寫程序段1

4. IN_Range（範圍內值）和 OUT_Range（範圍外值）

1）範圍內值和範圍外值指令

測試輸入值是在指定的值範圍之內還是之外。如果比較結果為 TRUE，則功能框輸出為 TRUE。

注：對於 LAD 和 FBD：單擊“???”並從下拉列表中選擇數據類型。

2）參數的數據類型

參數 數據類型 說明

MIN, VAL, MAX SInt, Int, DInt, USInt, UInt, UDInt, Real, LReal, 常數 比較器輸入

注：輸入參數 MIN、VAL 和 MAX 的數據類型必須相同。

滿足以下條件時 IN_RANGE 比較結果為真：MIN <= VAL <= MAX

滿足以下條件時 OUT_RANGE 比較結果為真：VAL < MIN 或 VAL > MAX

5. OK（檢查有效性）和 NOT_OK（檢查無效性）

1）OK（檢查有效性）和 Not OK（檢查無效性）指令

測試輸入數據參考是否為符合 IEEE 規範 754 的有效實數。

注：對於 LAD 和 FBD：如果該 LAD 觸點為 TRUE，則激活該觸點並傳遞能流。如果該 FBD 功能框為TRUE，則功能框輸出為 TRUE。

2）參數的數據類型

參數 數據類型 說明

IN Real, LReal 輸入數據

3）操作

指令 滿足以下條件時 REAL 數測試結果為 TRUE：

OK 輸入值為有效實數

NOT_OK 輸入值不是有效實數

注：如果 Real 或 LReal 類型的值為 +/- INF（無窮大）、NaN（不是數字）或者非標準化的值，則其無效。非標準化的值是非常接近於 0 的數字。CPU 在計算中用 0 替換非標準化的值。

6. 相同和不同比較指令

S7-1200 CPU 提供了用於查詢 Variant 操作數所指向的變量的數據類型是否與另一個操作數的數據類型相同的指令。此外，S7-1200 CPU 還提供了用於查詢數組元素的數據類型是否與另一個操作數的數據類型相同的指令。在這些指令中，將 <Operand1> 與 <Operand2> 進行比較。 <Operand1> 的數據類型必須為 Variant。<Operand2> 可以是 PLC 數據類型的基本數據類型。在 LAD 和 FBD 中，<Operand1> 是指令上方的操作數。在 LAD 中，<Operand2> 是指令下方的操作數。對於所有指令，如果通過相同或不同測試，則邏輯運算結果 (RLO) 為 1 (true)，否則為 0 (false)。

相同和不同類型比較指令如下所示：

EQ_Type（比較數據類型與變量數據類型是否“相等”）

NE_Type（比較數據類型與變量數據類型是否“不相等”）

EQ_ElemType（比較 ARRAY 元素數據類型與變量數據類型是否“相等”）

NE_ElemType（比較 ARRAY 元素數據類型與變量數據類型是否“不相等”）

1）EQ 和 NE 指令

EQ_Type測試 Operand1 處的變型所指向的變量是否與 Operand2 處的變量具備相同的數據類型。

NE_Type測試 Operand1 處的變型所指向的變量是否與 Operand2 處的變量具備不同的數據類型。

EQ_ElemType測試 Operand1 處的變型所指向的數組元素是否與Operand2 處的變量具備相同的數據類型。

NE_ElemType測試 Operand1 處的變型所指向的數組元素是否與Operand2 處的變量具備不同的數據類型。

2）參數的數據類型

參數 數據類型 描述

Operand1 Variant 第一個操作數

Operand2 位字符串、整數、浮點數、定時器、日

期和時間、字符串、ARRAY、PLC 數據類型 第二個操作數

7. 空比較指令

可以使用指令 IS_NULL 和 NOT_NULL 來決定輸入是否實際上指向對象。對於兩個指令來說，<Operand> 必須為 Variant 數據類型。

1）IS_NULL（查詢等於零的指針）和 NOT_NULL（查詢不等於零的指針）指令

IS_NULL測試 Operand 的 Variant 所指向的變量是否為空，即不指向任何對象。

NOT_NULL測試 Operand 的 Variant 所指向的變量是否不為空，即指向一個對象。

2）參數的數據類型

參數 數據類型 說明

Operand Variant 用於評估是否為空的操作數。

8. IS_ARRAY（檢查數組）

可以使用“檢查數組”指令來查詢 Variant 是否指向 Array 數據類型的變量。<操作數> 必須為 Variant 數據類型。如果操作數是數組，則指令返回 1 (true)。

1）IS_ARRAY（檢查數組）

IS_ARRAY測試 Operand 的 Variant 所指向的變量是否為數組。

2）參數的數據類型

參數 數據類型 說明

Operand Variant 評估是否為數組的操作數。
