Emby+KODI--完美的NAS多媒體方案

2020-10-29
| 科技

轉自：

作者：peterzen

鏈接：https://www.jianshu.com/p/85d00246d1f0

在Ubunt的基本環境（Ubuntu上的軟件基礎--for NAS）搭建好以後，家庭多媒體中心的任務就擺在了面前。 參考了很多資料，Emby Server看起來是最好的選擇，安裝的過程很簡單，網上有很多的教程，重點是我自己安裝後不會用，下面就重點介紹一下安裝以後的幾種在小米盒子上看電影的方法。

Emby Server基本設置

Emby Server安裝好以後，在控制檯（http://192.168.31.2:8096/web/index.html#!/dashboard.html，URL中的ip為Ubuntu server的ip地址）中按照提示一步步設置好，其中重點關注媒體庫和DLNA選項

  1. 在媒體庫設置好的路徑下把相關媒體資源放好
    Snipaste_2018-09-02_20-20-07.png
    放置好資源以後，使用媒體庫中的掃描媒體庫來同步一下數據。
  2. 啟用DLNA服務
    Snipaste_2018-09-02_20-24-52.png

小米盒子 高清播放器（DLNA）

上述設置好以後，打開小米盒子的高清播放器的設備·欄目下就會有1個DLNA設備`，一層層進去就可以正常看到你放的多媒體資源了。

IMG_20180902_203359.jpg


注意如果發現數據未同步，部分新放的電影沒有出現的話，就是在Emby Server的媒體庫中，媒體庫索引未更新；此時需要按照上面第一步的設置中去掃描媒體庫就可以了。

小米盒子+KODI

上述的DLNA方案搭建比較簡單，但是缺點也比較明顯：

  1. 界面看起來比較陽春（看到都是文件夾，界面不美觀）；
  2. DLNA應該是流媒體播放，解碼是在Ubuntu端做，如果是正常的NAS服務器，解碼能力有限；

於是Emby + KODI成為了比較完美的方案，使用Emby提供的多媒體數據，KODI在本地進行解碼，充分使用盒子的解碼能力，這樣比較均衡。

1. 安裝KODI

要使用比較新版本的KODI，剛開始為了簡便，使用當貝市場安裝了一個版本（好像是15.0），版本太久，不知道怎麼安裝emby 插件。 推薦使用官網下載的版本（https://kodi.tv/download），下載後copy到u盤，插到盒子上進行安裝（我的小米盒子是3.0，可以直接接U盤，如果還要舊一點的話，需要使用OTG線轉接）。

2. 安裝Emby插件

這一步非常關鍵，如果不安裝Emby插件的話，KODI只能加載Samba/NFS的遠程文件夾，這樣就失去了Emby Server的意義了。 剛開始是採用參考1的方式，由於參考一種沒有給出那個repository.beta.emby.kodi-1.0.5.zip插件的下載地址（或者是我自己沒找到），於是又是一陣折騰，百度不行就換google，終於在參考2中找到了插件的下載地址（哈哈，世界真是小啊），而且發現參考1應該就是使用參考2的 方法。

下載鏈接在 "參考2 -> 3 Installing Emby for Kodi -> Step 1: Download the Emby repository."的位置。

不過很悲劇的發現，我安裝了1.05以及1.03版本的插件，安裝完以後，最後在插件\視頻插件中都找不到Emby的插件，感覺很崩潰。 到這裡感覺快要進行不下去了，不過Emby+KODI的方案對我誘惑力實在是太大，於是自己繼續折騰，最後自己在在線插件庫中找到了Emby插件，一裝竟然真的可以用（娃哈哈，真的不要太神奇），安裝完，在視頻插件中選擇Emby後，過了幾秒，真的出現了發現Emby Server的彈出提示框，簡直不要太高興……

安裝Emby Server插件的辦法 Add-ons -> Add-on browser -> Install from repository -> All repositories -> Video add-ons -> 往下翻到E開頭，就會找到Emby，目前的版本是Emby.media.3.0.34a

在發現了Emby Server以後，可以按照提示設置好以後，會在本地建立索引，需要一些時間。等索引建立好以後，再進入KODI的Movies，就可以看到Emby Server上的電影都乖乖顯示在這裡了，剩下的就是enjoy your time.

IMG_20180902_210148.jpg


對Emby Server的目錄建立和文件拜訪規則還不是太清楚，這個還需要研究一下，不過多媒體中心建立成功，對NAS的建立又邁進了一步。
