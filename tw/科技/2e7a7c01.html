掌握防禦式編程，讓你的代碼更加健壯！

2020-11-14
| 科技

當你看到防禦式這三個字的時候，腦海裡一定是這樣想象的：

但是，今天我們討論的防禦式，是相當嚴肅的。這是一種編寫代碼時需要掌握而且要一直貫徹的思維，即一個方法或函數不能因為傳入的錯誤數據而被破壞。

▍保護程序不要被輸入的非法數據破壞

程序員在寫代碼的時候，經常會太過於相信上游調用方傳入的數據，比如確信上游不可能傳空值進來，從來不去做參數對象是否為空的判斷。但是，凡事皆有可能，當某一天一個null對象被傳入的時候，程序會被立刻crash。

對於外界的垃圾數據，應該：

  • 檢查所有來源於外部的數據的值。當從文件、用戶、網絡等外部接口獲取數據時，應該檢查所獲得的數據值，以確保它在允許的範圍內。
  • 檢查子程序所有傳入的值。與檢查外部數據一樣，只不過數據來源於其它的子程序。
  • 決定如何處理錯誤的輸入數據。當檢測出錯誤數據時，應該如何處理？

▍斷言

斷言（assert）是指在開發期間使用的，讓程序在運行時自檢的代碼。斷言為真，說明程序正常執行，若為假，則意味著發生了意料之外的錯誤。

一個斷言通常有兩個參數，一個描述假設為真時的布爾表達式，和一個斷言為假時需要顯示的內容。

Java Example of an Assertionassert denominator != 0 : "denominator is unexpectedly equal to 0.";

程序員也可以自己建立或封裝斷言，定義自己的斷言機制。

C++ Example of an Assertion Macro#define ASSERT( condition, message ) { if ( !(condition) ) {  LogError( "Assertion failed: “, #condition, message );  exit( EXIT_FAILURE );  }}

對於斷言的使用，有一些指導意見：

  • 用錯誤處理代碼來處理預期會發生的情況，用斷言來處理絕不應該發生的情況；
  • 避免把需要執行的代碼放在斷言中；
  • 用斷言來註解並驗證前條件和後條件；
  • 對於高健壯性的代碼，應該先使用斷言再使用錯誤處理。

▍錯誤處理技術

對於錯誤的處理，應該考慮以下幾種方式：

  • 返回中立值；
  • 換用下一個正確的數據；
  • 返回與前次相同的數據；
  • 換用最接近的合法值；
  • 把警告信息記錄到日誌中；
  • 返回一個錯誤碼；
  • 調用錯誤處理子程序；
  • 當錯誤發生時顯示錯誤內容；
  • 關閉程序
