自己動手寫操作系統（字符顯示）

2020-11-14
| 科技

自己動手寫操作系統（開篇）

在 開篇一文中，我們實現了簡單的通過使用 8086 彙編語言在裸機（沒有安裝操作系統的機器）上打印顯示了一行文本信息，當然 8086 彙編語言給我們提供了好幾種方式來顯示，下面我們通過代碼來實現其他幾種文本的顯示。



我們知道 8086 可以訪問 1MB 內存。 其中:

0x00000 ~ 0x9FFFF 屬於常規內存，由內存條提供；

0xF0000 ~ 0xFFFFF 由主板上的一個芯片提供，也就是 ROM-BIOS.

這樣一來，中間還有320KB的空洞，也就是0xA0000 ~ 0xEFFFF 的區域， 傳統上，這段地址空間由特定的外圍設備來提供，其中就包括顯卡，由於歷史原因，個人計算機在加電自檢後都會把自己初始化到 80 * 25 的文本模式，這種模式下，屏幕上可以顯示25行，每行80個字符，每屏總共 2000 個字符。

0xB8000~0xBFFFF 這段物理地址空間，都留給顯卡的，由顯卡來提供，用來顯示文本。


文本模式下顯存的位置

通過操作顯存來填滿2000個1

org 07c00h        mov    ax,    cs        mov    ds,    ax        mov ax, 0xb800 ; //把 es 的值設置為 顯存的位置        mov es, ax        mov si, 1999；//要顯示的字符個數        mov di, 0     ；//下標，把顯存看成是一個數組        mov al, 0x31；//字符 ‘1’當然這裡也可以直接寫成 mov al, '1'  也是可以的        mov ah, 0x04；//顏色信息show:        mov [es:di], ax        add di, 2        dec si        jns show        jmp $times    510-($-$$)    db    0dw 0xaa55

說明：

其實就是在物理內存地址為：0xb8000 開始的地方填充了4000個字節

分別填入 0x31 0x04 重複 2000次。


80 * 25 文本模式下的顏色表


字符打印: 一個字符一個字符的打印，直達遇見了 $ 符號後結束， 通過調用 int10h 實現

org 0x7c00    	mov ax, cs     	mov es, ax    	mov ds, ax	    	mov si, weldata    	call DispStr    	jmp $      DispStr:   	mov al, [si]			  cmp al, '$'  	je  Disover    	   mov ah, 0eh    	int 10h    	inc si   	jmp DispStr    Disover:	ret	weldata db 'Welcome Test Os!', '$'times 510-($-$$) db 0db 0x55,0xaa

這個程序的顯示實現主要是靠： ax 的值和 int 10h 實現。

這裡 ax 分為 ah 和 al , 其中 al 裡面保存了要打印的字符，ah 的值設置為 0eh 是告訴 int 10h 執行的是字符打印（顯示）功能。

ah 設置為不同的值，調用 int 10h 可以實現不同的功能。

具體可以參考網絡博文:

https://blog.csdn.net/cherisegege/article/details/79837528


打印字符 ，通過指定長度和調用 int 10h 實現

jmp near start          showstr db 'Hello World!'start:        mov  ax, 0x7c0        mov  ds, ax        mov  es, ax        mov si, showstr        mov cx, start - showstrdisp:        mov al, [si]        mov ah, 0xe        int 10h        inc si        loop disp        jmp $times 510-($-$$) db 0db 0x55, 0xaa


當然也可以直接寫入到顯存中來實現。

jmp near start          showstr db 'Hello World!'start:        mov  ax, 0x7c0        mov  ds, ax        mov  ax, 0xb800        mov  es, ax        mov si, showstr        mov cx, start - showstr        mov di, 0disp:        mov al, [si]        mov [es:di], al        inc di        mov byte[es:di], 0x07        inc di        inc si        loop disp        jmp $times 510-($-$$) db 0db 0x55, 0xaa


說明：通過直接寫入顯存顯示和調用 int 10h 來顯示，在顯示效果上有一個小小的區別 就是 ，通過直接寫顯存的方式來顯示字符不會改變光標的位置，而調用 int 10h 會改變光標的位置.
