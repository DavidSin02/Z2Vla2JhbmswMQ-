Q1。解釋JDK，JRE和JVM？

JDK JRE JVM Java開發工具包。 Java運行時環境。 Java虛擬機。 它是編譯，記錄和打包Java程序所必需的工具。 JRE是指可以在其中執行Java字節碼的運行時環境。 它是一臺抽象機。該規範提供了可以在其中執行Java字節碼的運行時環境。 它包含JRE +開發工具。 它是實際存在的JVM的實現。 JVM遵循三種表示法：規範，實現和 運行時實例。

Q2。在Java中解釋public static void main（String args []）。

Java中的main（）是任何Java程序的入口點。它總是寫為public static void main（String [] args）。

  • public：public是訪問修飾符，用於指定誰可以訪問此方法。Public表示任何類都可以訪問此方法。
  • static：它是java中的關鍵字，標識它是基於類的。main（）在Java中被設為靜態，因此無需創建Class的實例即可對其進行訪問。如果main不是靜態的，那麼編譯器將拋出錯誤，因為JVM在創建任何對象之前調用main（），並且只能通過該類直接調用靜態方法。
  • void：這是方法的返回類型。Void定義了不會返回任何值的方法。
  • main：這是JVM搜索的方法的名稱，它是僅具有特定簽名的應用程序的起點。這是主要執行發生的方法。
  • String args []：這是傳遞給main方法的參數。

Q3。為什麼Java是平臺獨立的？

Java之所以被稱為平臺無關的，是因為Java的字節碼可以在任何系統上運行，而不管其底層操作系統什麼。

Q4。為什麼Java不是100％面向對象的？

Java不是100％面向對象的，因為它使用了不是對象的八種原始數據類型，例如布爾值，字節，字符，整數，浮點數，雙精度型，長型，短型。

Q5。Java中的包裝器類是什麼？

包裝器類將Java原語轉換為引用類型（對象）。每個原始數據類型都有一個專用於它的類。這些稱為包裝器類，因為它們將原始數據類型“包裝”到該類的對象中。請參考下圖，該圖顯示了不同的原始類型，包裝器類和構造函數參數。

Q6。Java中的構造函數是什麼？

在Java中，構造函數是指用於初始化對象的代碼塊。它必須與該類具有相同的名稱。而且，它沒有返回類型，並且在創建對象時會自動調用它。

有兩種類型的構造函數：

  1. 默認構造函數：在Java中，默認構造函數是不接受任何輸入的構造函數。換句話說，默認構造函數是無參數構造函數，如果用戶沒有定義其他構造函數，則默認情況下將創建該構造函數。其主要目的是用默認值初始化實例變量。而且，它主要用於對象創建。
  2. 參數化構造函數： Java中的參數化構造函數是能夠使用提供的值初始化實例變量的構造函數。換句話說，採用參數的構造函數稱為參數化構造函數。

Q7。Java中的單例類是什麼？如何使單例類成為單例？

Singleton類是在一個給定的時間在一個JVM中只能創建一個實例的類。可以通過將構造函數設為私有來使類成為單例。

Q11。 什麼是Java包？列出軟件包的各種優點。

Java中的程序包是捆綁在一起的相關類和接口的集合。通過使用軟件包，開發人員可以輕鬆地將代碼模塊化並優化其重用。而且，包中的代碼可以由其他類導入並重新使用。下面我列出了它的一些優點：

  • 軟件包有助於避免名稱衝突
  • 它們提供了更容易的代碼訪問控制
  • 包還可以包含隱藏類，這些隱藏類對外部類不可見，僅在包內使用
  • 創建適當的層次結構，使查找相關類更加容易

Q12。為什麼在Java中不使用指針？

Java不使用指針，因為它們不安全並且會增加程序的複雜性。由於Java以其簡單的代碼而聞名，因此添加指針的概念將是矛盾的。此外，由於JVM負責隱式內存分配，因此，為了避免用戶直接訪問內存，在Java中不建議使用指針。

Q13。 Java中的JIT編譯器是什麼？

JIT代表Java中的即時編譯器。它是一個程序，有助於將Java字節碼轉換為直接發送到處理器的指令。默認情況下，JIT編譯器在Java中啟用，並且在調用Java方法時被激活。然後，JIT編譯器將被調用方法的字節碼編譯為本地機器代碼，“及時”編譯以執行。編譯完該方法後，JVM會直接調用該方法的已編譯代碼，而不是對其進行解釋。這就是為什麼它經常在運行時負責Java應用程序的性能優化的原因。

Q17。什麼是面向對象編程？

面向對象的編程或通常稱為OOP的編程模型或方法是圍繞對象而不是邏輯和函數來組織程序。換句話說，OOP主要集中在需要操作的對象上，而不是邏輯上。這種方法對於大型和複雜代碼的程序是理想的，並且需要主動更新或維護。

Q18。Java中OOP的主要概念是什麼？

面向對象編程或OOP是一種與以下概念相關的編程樣式：

  1. 繼承：繼承是一個類在其中獲取另一類的屬性的過程。
  2. 封裝： Java中的封裝是一種將數據和代碼打包為單個單元的機制。
  3. 抽象：抽象是向用戶隱藏實現細節並僅向用戶提供功能的方法。
  4. 多態性：多態性是變量，函數或對象採取多種形式的能力。

Q25。什麼是Java字符串池？
