C++面向對象設計

2020-10-29
| 科技

在計算時代初期，編程通常僅限於機器語言編程。機器語言是指特定於特定機器或處理器的那些指令集，格式為0和1。這些是位序列（0100110…）。但是用機器語言編寫程序或開發軟件非常困難。

實際上，要開發在當今場景中使用位序列的軟件是不可能的。這是程序員轉向下一代編程語言，開發彙編語言的主要原因，因為彙編語言與英語非常接近，因此易於理解。這些彙編語言用於微處理器。隨著微處理器的發明，彙編語言在整個工業中得到了廣泛應用，但這還遠遠不夠。程序員再次想出了一些新的東西，即結構化和過程式編程。

結構化編程

結構化編程方法的基本原理是將程序分為功能和模塊。模塊和功能的使用使程序更易於理解和閱讀。它有助於編寫更簡潔的代碼，並保持對功能和模塊的控制。這種方法重視功能而不是數據。它專注於大型軟件應用程序的開發，例如，C用於現代操作系統開發。編程語言：PASCAL（由Niklaus Wirth引入）和C（由Dennis Ritchie引入）遵循這種方法。

過程編程方法

這種方法也稱為自頂向下方法。在這種方法中，程序分為執行特定任務的功能。 此方法主要用於中型應用程序。數據是全局的，所有功能都可以訪問全局數據。 程序編程方法的基本缺點是數據不安全，因為數據是全局的並且可以由任何功能訪問。程序控制流程是通過函數調用和goto語句實現的。編程語言：FORTRAN（由IBM開發）和COBOL（由Grace Murray Hopper博士開發）遵循這種方法。

這些程序結構是在20世紀70年代末和80年代發展起來的，雖然它們滿足了良好結構的程序、軟件等標準，但仍然存在一些問題，沒有達到當時的要求。它們似乎過於籠統，與實時應用程序不相關。

為了解決這類問題，開發了面向對象的方法OOP作為解決方案。

面向對象編程（OOP）方法

結構化編程和過程編程方法與實際應用並不太接近，需求增加了，但仍然使用傳統方法。OOP概念的設計基本上是為了克服上述編程方法的缺點，這種新方法在程序設計方法學領域帶來了一場革命。

面向對象編程（OOP）是一種只允許在特定類和實時對象的幫助下編寫程序的編程方法。我們可以說，這種方法非常接近現實世界及其應用，因為這些類和對象的狀態和行為幾乎與現實世界的對象相同。

什麼是類和對象？

它是OOP的基本概念，是C語言中使用的結構的擴展概念，是一種抽象的、用戶定義的數據類型。它由幾個變量和函數組成。類的主要用途是存儲數據和信息。類的成員定義類的行為。類是對象的藍圖，但也可以說類的實現就是對象。類對世界不可見，但對象是可見的。

Class car {     int car_id;     char colour[4];     float engine_no;     double distance;       void distance_travelled();     float petrol_used();     char music_player();     void display(); }

在這裡，汽車類具有屬性car_id，顏色，engine_no和距離屬性。它類似於具有相同規格的現實世界中的汽車，可以將其聲明為公開（對類以外的所有人可見），受保護和私有（對任何人都不可見）。另外，還有一些方法，例如distance_travelled（），petrol_used（），music_player（）和display（）。在下面給出的代碼中，汽車是類，而c1是汽車的對象。

#include <iostream> using namespace std;   class car { public:     int car_id;     double distance;       void distance_travelled();       void display(int a, int b)     {         cout << "car id is=\t" << a << "\ndistance travelled =\t" << b + 5;     } };   int main() {     car c1; // Declare c1 of type car     c1.car_id = 321;     c1.distance = 12;     c1.display(321, 12);       return 0; }

數據抽象

抽象是指代表重要特徵和特殊特徵而不包含背景細節或對該特徵的解釋的行為。 數據抽象簡化了數據庫設計。

1）物理層：

它描述如何存儲數據，這些數據通常對用戶隱藏。可以用“存儲塊”來形容。

2）邏輯層：

它描述了存儲在數據庫中的數據以及數據之間的關係。程序員通常在此級別上工作，因為他們知道維護數據之間的關係所需的功能。

3）表示層：

應用程序出於安全目的隱藏數據類型和信息的詳細信息。此級別通常在GUI的幫助下實現，並顯示了針對用戶的詳細信息。

封裝

封裝是面向對象編程（OOP）中的基本概念之一。它描述了包裝數據的思想以及在一個單元（例如C++中的類）內處理數據的方法。此概念通常用於從外部隱藏對象的內部狀態表示。

繼承

繼承是一個類繼承另一個類（稱為父類）的功能或屬性的能力。當我們編寫一個類時，我們從其他類繼承屬性。因此，當我們創建一個類時，我們不需要一次又一次地編寫所有的屬性和函數，因為它們可以從擁有它的另一個類中繼承。繼承使用戶可以在可能的情況下重用代碼並減少其冗餘。

多態性

多態是指以多種形式處理數據的能力。它允許以各種方式執行同一任務。它由方法重載和方法覆蓋組成，即，一次編寫方法並使用相同的方法名稱執行許多任務。

#include <iostream> using namespace std;   void output(float); void output(int); void output(int, float);   int main() {     cout << "\nGfG!\n";     int a = 23;     float b = 2.3;       output(a);     output(b);     output(a, b);       return 0; }   void output(int var) { // same function name but different task     cout << "Integer number:\t" << var << endl; }   void output(float var) { // same function name but different task     cout << "Float number:\t" << var << endl; }   void output(int var1, float var2) { // same function name but different task     cout << "Integer number:\t" << var1;     cout << " and float number:" << var2; }

有關OOP的一些重要知識：

1）OOP將數據視為關鍵要素。

2）重點是數據而不是程序。

3）將問題分解為更簡單的模塊。

4）不允許數據在整個系統中自由流動，即本地化控制流。

5）數據不受外部功能的影響。

OOP的優勢

1）它很好地模擬了現實世界。

2）使用OOP，程序易於理解和維護。

3）OOP提供代碼可重用性。已經創建的類可以重複使用，而不必再次編寫它們。

4）OOP有助於程序的快速開發，在這些程序中類的並行開發是可能的。

5）使用OOP，程序更易於測試，管理和調試。

OOP的缺點

1）使用OOP時，有時會過度概括類。

2）類之間的關係有時變得膚淺。

3）OOP設計很複雜，需要適當的知識。此外，還需要為OOP編程做適當的規劃和設計。

4）要使用OOP進行編程，程序員需要適當的技能，例如根據對象和類等進行設計，編程和思考的技能。
