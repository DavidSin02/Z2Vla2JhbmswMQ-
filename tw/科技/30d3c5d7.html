彙編int21h指令以及軟中斷和硬中斷的解釋

2020-11-14
| 科技

CPU的設計者必須在中斷信息和其處理程序的入口地址之間建立某種聯繫，使得CPU根據中斷信息可以找到要執行的處理程序。

我們知道，中斷信息中包含有標識中斷源的類型碼。根據CPU的設計，中斷類型碼的作用就是用來定位中斷處理程序。

比如CPU 根據中斷類型碼 13，就可以找到13號中斷的處理程序。

可隨之而來的問題是，若要定位中斷處理程序，需要知道它的段地址和偏移地址，而如何根據 8位的中斷類型碼得到中斷處理程序的段地址和偏移地址呢？

CPU用 8 位的中斷類型碼通過中斷向量表找到相應的中斷處理程序的入口地址。

那麼什麼是中斷向量表呢？

中斷向量表就是用來存儲中斷向量的列表。

中斷向量表在內存中保存，其中存放著 256箇中斷源所對應的中斷處理程序的入口地址，每個入口地址佔4個存儲單元。如下圖所示：

圖一

中斷向量表在內存中存放，對於8086PC機，中斷向量表指定放在內存地址0處。

從內存0000:0000到0000:03FF的1024個單元中存放著中斷向量表。

從上面的講解中，我們知道，可以用中斷類型碼，在中斷向量表中找到中斷處理程序的入口。

找到這個入口地址的最終目的是用它設置CS和IP，使CPU執行中斷處理程序。

用中斷類型碼找到中斷向量，並用它設置CS和IP，這個工作是由CPU的硬件自動完成的。

CPU 硬件完成這個工作的過程被稱為中斷過程。

int 21h 中斷例程是DOS提供的中斷例程，其中包含了DOS提供給程序員在編程時調用的子程序，DOS中斷的功能主要包括：
1、DOS鍵盤功能調用
2、DOS顯示功內能調用
3、DOS打印功能
4、DOS串行口通信功能調用。

因為int 21h 中斷例程對應的中斷類型碼為21，它的中斷處理程序的入口地址應該從21×4地址單元開始存放，段地址存放在 21×4+2 字單元中，偏移地址存放在21×4字單元中。

經常使用的是 int 21中斷例程的4ch號功能，即程序返回功能，如下：

mov ah,4ch ;程序返回

mov al,0 ;返回值

int 21h

(ah)=4ch表示調用第21h號中斷例程的 4ch號子程序，功能為程序返回，可以提供返回值作為參數。

內中斷的意思是指，CPU不再接著剛執行完的指令向下執行，而是轉去處理這個特殊信息。外中斷是指CPU 在計算機系統中，除了能夠執行指令，進行運算以外，還應該能夠對外部設備進行控制，接收它們的輸入，向它們進行輸出。該程序中的語句

mov ah,7；

int 21h

屬於DOS中斷中的7號功能調用，用來接收鍵盤輸入的信息，屬於軟中斷，也稱內中斷，其過程是按照int 21h指令尋找中斷程序入口地址，這一中斷程序正是用來讀取鍵盤輸入字符的，中斷由int 21h指令觸發；指令

in al,60h

是直接讀入60h號端口的信息，而60h號端口的信息同樣來自於鍵盤輸入，該指令執行時將觸發 int 9H外中斷處理程序，該外中斷可以掃描相關鍵盤上的每一個按鍵所連接的開關電路狀態的變換，從而判斷出被按下的ascii碼鍵值，先送入端口60H，再送入寄存器al，這一中斷過程屬於硬中斷（外中斷）。外中斷是由外部因素引起（某人施加了力量在鍵盤的按鍵上，引起了相應開關電路的變化，這種變化觸發了某個程序的執行，這個被觸發的程序就屬於外中斷）。這個例子可以理解為in al,60h指令先接通了某個開關，允許int 9H中斷執行，但這個中斷是否執行，要看外部電路是否有變化。

指令

mov ah,7；

int 21h

和指令

in al,60h

可以相互替代。

這三條指令的相互替換很好地解釋了軟中斷和硬中斷之間的區別和聯繫。
