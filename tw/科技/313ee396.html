組建局域網完全攻略（1）--協議簡解

一、引子

計算機硬件更新速度越來越快，而且價格也起來越便宜。在校園裡，學生擁有計算機已經很普遍了。試想一下，假如這些計算機沒有組成網絡，而是各自為政，會不會有孤單、寂寞之感嗎？答案是肯定的。在這裡筆者將從最基本的東西入手，帶讀者組建自己的宿舍網。從而實現資源共享、代理上網、遊戲對戰等功能。

二、協議簡解

tcp/ip協議:它是目前最流行的商業化的協議，相對於OSI，它是當前的工業標準或“事實的標準”，在1974年由Kahn提出的。它分為四個層次：應用層、傳輸層、互聯層、主機-網絡層。TCP/IP協議主要用於廣域網，在一些局域網中也有應用，比如在windows 9x和netware網中可以使用，但是在功能和速度方面還有差距，本來TCP/IP的設計目標就和IPX/SPX不一樣。在設置TCP/IP協議的時候，主要有ip地址和子網掩碼，網關（Gateway：在通信中提供網絡協議之間的轉換，適用於不同協議主機之間的信息交換），域名解析服務器（NDS: Domain Name System,即把你輸入的網址轉換為數字型的ip地址）和NetBIOS等等。

IPX/SPX協議：現在解釋一下這種應用很廣的網絡協議。IPX/SPX是基於施樂的XEROX’S Network System（XNS）協議，而SPX是基於施樂的XEROX’S SPP（Sequenced Packet Protocol：順序包協議）協議，它們都是由novell公司開發出來應用於局域網的一種高速協議。它和TCP/IP的一個顯著不同就是它不使用ip地址，而是使用網卡的物理地址即（MAC）地址。它也和TCP/IP協議一樣不能和OSI完全對應。只包含了OSI中傳輸層和網絡層的部分。在實際使用中，它基本不需要什麼設置，裝上就可以使用了。由於其在網絡普及初期發揮了巨大的作用，所以得到了很多廠商的支持，包括microsoft等，到現在很多軟件和硬件也均支持這種協議。

NetBIOS和NetBEUI：NetBIOS由microsoft和IBM開發，是會話層的協議，它建立允許不同計算機上的應用程序通過網絡傳輸的連接會話。它是為只需要與別的在同一個網絡上的計算機會話這一目的話設計的。在win9x&me中，當你在IPX/SPX中選定“希望在IPX/SPX啟用NetBIOS”後，在TCP/IP的設置中可以選定“通過TCP/IP啟用NetBIOS”。而NetBEUI則是IBM為自己作為網絡傳輸站的網絡產品設計的。它是NetBIOS的一個擴充版本，定義了沒有被NetBIOS定義的禎類型，它能在中小規模的單網上傳輸數據。它可以在win9x&me中直接安裝。

組建局域網完全攻略（2）--網線製作

三、網線的製作

RJ-45線的製作：

先準備好我們所需的工具：壓線鉗。

1.先抽出一小段線，然後把外皮剝除一段(長度大約為1.2-1.3釐米);

2.根據排線標準將雙絞線反向纏繞開，現行的接線標準有t568a和t568B標準，平常用得較多的是t568B標準。這兩種標準本質上並無區別，只是線的排序序順不同而已;

T568B標準從1-8的排線順序為：橙白，橙，綠白，藍，藍白，綠，棕白，棕。T568A即在T568B的基礎上，把1<-->3,2<-->6的順序相互換一下即可。

3.用壓線鉗把參差不齊的線頭剪齊;

4.把線插入水晶頭，並用壓線鉗夾緊(使勁)，另一頭也按同一標準接好；

在這裡筆者提醒讀者，在使用HUB、交換機等集線設備時，這兩種標準不能混用。而雙機互聯未使用集線器的情況下，便得一頭採用T568A標準，另一頭採用T568B標準。因為根據規定，網卡的腳1和腳2為發送數據，而腳3和腳6為接收數據引腳。1、3,2、6線互換主要是使一塊網卡1、2腳發送數據，另一塊網卡正好用3、6腳接收。

5.最後使用測試儀測試你的網絡線是否接通，也可以直接用到網絡上進行測試數據能否接通。

細纜線的製作：

在用細纜線打造網絡時，由於計算機之間的距離有長有短，因此網線也要取用適當長度才行。首先準備好你的武器：尖嘴鉗、斜口鉗、扳手、萬用表。

1.首先將鎖式BNC接頭組件中的環形螺絲、金屬環片、像皮墊圈等按順序套上RG58 A/U電纜一端;

2.去掉一小段RG58 A/U同軸電纜外層包線(約1-1.2cm),露出部分導電網，並將導電網部分向外拔開；

3.去掉一小優包覆芯線的絕緣體，露出中心導體。中心導體露出的長度要頂到勉為其難鍍金針頭的針尖端，且後方不宜露出過多的裸線。

4.放入金屬圓錐套。

5.將中心導體的裸線部分插入鍍鎦金針頭後半部，插入後可將針頭按順時方向略為旋轉一不能超過兩圈；

6.將鍍鎦金針頭部分插入金屬套管，同時把橡皮墊圈往接頭方向推進，使翻開的導電網頂住前端的金屬圓錐套。

7.將接頭內的導電網略為整理一下，再將金屬墊片推入，把導電網完全覆蓋在內，最後將環形螺絲往金屬套管的方向推進，並旋緊環形螺絲。

8.製作完成，你可以進行測驗你的網線是否能接通。

組建局域網完全攻略（3）--組網類型

四、組網類型

1.網卡

網卡按接頭種類可分為AUI、BNC和RJ45接頭三種類型，它們分別用來連接三種不同的網絡電纜，即AUI電纜、RG-58A/U電雙絞線[包括非屏蔽線（UTP）和屏敝雙絞線（STP）兩種]。AUI接頭和電纜，由於佈線施工麻煩，已被市場淘汰，本人就不詳敘述。採用BNC接頭和RG-58A/U網絡電纜來構造網絡，不僅價格低廉、且安裝容易，因此備受DIY族的青睞。RJ-45接頭用來連接雙絞線，因易於擴展、系統調試方便，也得到大家普遍採用。

網卡以總線形式又可分為PCI、EISA、PCI、SISC四種，現在普遍採用PCI接口的網卡；還可按數據傳輸頻寬為為10M/S、100M/S，10M-100M、千M網網卡，現用得比較多的是10M-100M自適應網卡。</P><P> 2.網絡類型

一般來說，局域網拓撲大致可分為總線型（BUS）、星型（star)、環形(ring)網絡等幾種。

總線型結構是用一條共用的網線（一般採用細纜線）來連接所有的計算機。它的優點是成本低廉，佈線簡單，但有一個致命的缺點便是整個網絡任何一個節點發生故障，整個網絡將癱瘓。這種拓撲結構逐漸被淘汰。10臺以下計算機比較適合總線型組網，10臺以上便維護麻煩，且易出故障。

星型是所有計算機都接到一個集線器（或是交換機、路由器等），通過集線器在各計算機之間傳遞信號。它的優點便是網絡局部線路故障只會影響局部區域，不會導致整個網絡癱瘓，維護方便。缺點便是成本較高(相對而言)。從上述內容大家可以看出，總線型不用集線器，而星形則至少要有一個集線器，才能使網絡運轉，從而增加了這部分的開支。星型組網比較流行，它適合任意臺計算機組網。

環型結構只有IBM公司採用，目前用的比較少，筆者認為也不是將來的趨勢。

組建局域網完全攻略（4）--從經濟和實用角度分析組網

五、從經濟和實用角度分析建網，可以分為以下幾類：

1、經濟型：這種組網方式一般採用總線型，即用細纜線相連。它的特點是投資少，見效快(何解，還有見效快之說？^_^)，但維護比較麻煩。

我們可以使用帶BNC頭(一定要帶)的ISA或PCI的網卡，建議使用PCI10M網卡。topstar te-3500N便是一種比較好的網卡，它的兼容性好，除了BNS接口以外還帶RJ-45接口，價格只需40元左右。

用細纜連接電腦時，需要注意，便是兩臺計算機的最小長度不得低於0.5米，最大長度不得超過180米，假如小於最小長度或者大於最大長度，均會造成網絡聯接不通的情況。另外，總線的兩端必須安裝50歐母的終端電阻器。

2、實用型：一般採用星型組網，即用HUB與雙絞線與各臺計算機相聯。

在這種組網方案中，筆者還是建議使用topstar te-3500N(PCI 10M)網卡和HUB(根據計算機的多少決定需要多少口的HUB，一般只就多不就少)。它與經濟型相比，維護比較簡便，但你掏也的銀子卻會多許多。

3、發燒型：還是採用星型組網，但是它的網絡速度可以達到100M甚至1000M，當然，代價就是你多掏不秀氣的銀子哦。

網卡可以採用類顯PCI10M/100M自適應的網價，d-link10M/100M自適應的交換機。

第七步：把你需要共享的文件夾或者磁盤共享，雙擊我的電腦，選中你所需共享的文件夾或者磁盤驅動，點右鍵-屬性-共享-確定。

組建局域網完全攻略（7）--在局域網中發短信息和聊天

在internet中，人們利用OICQ、聊天室、BBS論壇與網友進行交流，我們在局域網中同樣也可以實現上述功能。這些功能既可利用windows自帶的程序實現，也可以利用第三方軟件來實現。在這裡，筆者先向大家介紹windows自帶的程序實現局域網上信息交流。

在局域網中發短信息

在windows目錄下，有個名為winpopup.exe的文件，為了方便，可以把它放到啟動組中或者桌面上。運行winpopup，選菜單消息-發送，它可以針對某位用戶或者計算機發送信息，也可以對某一工作組發送。在工作組中輸入workgroup,消息窗口中輸入所示的文字，確定。這些信息即發送到workgroup的工作組當中。當計算機接收時，它會顯示從哪臺計算機發送過來的消息和具體時間。需要指出的是，當幾臺計算機用同一個帳號登錄網絡時，用winpopup發短信息到用戶，只有第一臺登錄網絡的用戶才能接收得到，其它計算機均無法接收到信息。另外windows2000server，不需運行winpopup軟件，也能接收到短信息。

winpopup的優點：它使用的是netbeui協議，對網絡的配置要求較低，它在傳輸過程中，基本上不會影響網絡和其它用戶的工作，更主要的是，它是windows自動的程序，不需要另外安裝。

缺點是隻能發送小容量的文本信息，最大不超過64K,所以用它大容量信息是無法完成。

召開網絡會議

應用windows自帶的netmeeting軟件，在局域網實現聊天、語音交流等功能。

首先檢查計算機是否已經安裝netmeeting，在開始-程序-internet工具-netmeeting處，假如沒有，可以到在控制面板-添加/刪除程序中的windows安裝中找到。

第一步：運行netmeeting,假如第一次運行的話，需要對它進行配置，

第二步：然後輸入個人信息。

第三步：選中“當netmeeting啟動時登錄到目錄服務器”；不要選“不要在目錄中列出我的名字”。

第四步：由於是通過局域網進入netmeeting,所以選擇“局域網”。

第五步：計算機將啟動音頻設置嚮導，對音箱大聲音大小和麥克風進行設置。

設置完成以後，將正式進入netmeeting，馬上可以體會到netmeeting給我們帶來的快感。

進入netmeeting以後，在需要主持會議的計算機（臨時netmeeting服務器）點選呼叫-主持會議,輸入會議名稱，並在會議工具中把“共享、聊天、白板、文件傳送”一併選中，確定。

工作站啟動netmeeting以後，需要加入網絡會議，選呼叫-新呼叫，填上主持會議計算機的IP地址，筆者填的是192.168.0.103.在使用中選擇自動。其它工作站與此選擇一樣。

當有工作站加入到網絡會議當中，netmeeting會在名稱下面顯示加入會議人的姓名(圖715.jpg)，在最底下，有四個選項，分別為共享程序、聊天、白板、傳送文件。

共享程序：可以將程序的控制權傳遞給擁有

NetMeeting 3.0

或更高版本的與會者。不能將程序的控制權傳遞給擁有早期版本的參加者。NetMeeting

將在共享程序者的桌面上顯示消息，詢問是否允許傳遞控制權。如果要自動授予許可，請在“共享”對話框中選中“自動為控制接受請求”複選框。

聊天：即與我們平常上internet的聊天室類似，不過功能沒那麼強大。

聊天界面，在消息下面輸入要發送的信息，在最下面可以選擇你要發送的對象，可以是聊天中的每個人，也可以是私聊。

netmeeting的聊天功能只能傳送文本信息，那麼想傳送圖片怎麼辦？如比做個網頁，網頁設計需要佈局，假如用語言來描述，那需要很大一堆話才能說得清，假如藉助netmeeting的白板，哈哈，一下子便搞定了，一切簡單明瞭。

netmeeting還有一個很實用的功能，便是文件傳送，不知大家是否經常會碰到oicq不能傳送文件，而收電子郵件又延時，這裡便可利用netmeeting，來實時傳送文件。

netmeeting不僅適合局域網用戶，同時也適用於internet。當計算機拔號163上網以後，ISP會自動分配一個IP，我們可以利用這個臨時IP來實現網絡會議。

以上敘述的收發短信息、聊天等是windows自帶的程序，功能並不強大，假如讀者有興趣，可以到http://portal2.eastchina.com/cbuilder/下載免費的局域網聊天室軟件，網站上有相關的軟件使用說明。

組建局域網完全攻略（9）--網絡知識臨時充電

局域網連接測試工具

ping,它經常用於檢測計算機是否可以連通:

它的命令格式為：ping 目的地址 [參數1] [參數2] ......

其中目的地址是指被測試計算機的IP地址或域名或者是計算機名。主要參數有：

a:解析主機地址；

n:數據：發出的測試包的個數，缺省值為4；

l:數值：所發送緩衝區的大小；

t:繼續執行ping命令，直到用戶按ctrl+c終止。

ipconfig，一般用於查看本機IP地址等信息:

它的命令格式為：ipconfig [/參數1] [/參數2] ......

a11:顯示與TCP/IP協議相關的所有細節，包括主機名、節點類型、是否啟用IP路由、網卡的物理地址、默認網關等；

batch [文本文件]：將測試的結果存入指定的文本文件中，以便於逐項查看；

renew:如果計算機啟用 DHCP 並使用 DHCP 服務器獲得配置，請使用 ipconfig /renew 命令開始刷新租約。

另外還有一個與ipconfig相類似的命令，即winipcfg，它在windows環境下運行，具有圖形界面。

netstat,命令顯示協議統計信息和當前的 TCP/IP 連接:

它的命令格式為：netstat [-參數1][-參數2]

-a 命令將顯示所有連接；

-r 顯示路由表和活動連接；

-e 命令將顯示 Ethernet 統計信息；

-s 顯示每個協議的統計信息；

-n，則不能將地址和端口號轉換成名稱。

nbtstat,是解決 NetBIOS 名稱解析問題的有用工具,可以使用 nbtstat 命令刪除或更正預加載的項目：

它的命令格式為：nbtstat [-參數1][-參數2]......

-n 顯示由服務器或重定向器之類的程序在系統上本地註冊的名稱;

-c 顯示 NetBIOS 名稱緩存，包含其他計算機的名稱對地址映射;

-R 清除名稱緩存，然後從 Lmhosts 文件重新加載;

-RR 釋放在 WINS 服務器上註冊的 NetBIOS 名稱，然後刷新它們的註冊;

-a name 對 name 指定的計算機執行 NetBIOS 適配器狀態命令。適配器狀態命令將返回計算機的本地 NetBIOS 名稱表，以及適配器的媒體訪問控制地址;

-S 列出當前的 NetBIOS 會話及其狀態（包括統計）。

組建局域網完全攻略（10）--常見問題FAQ

Q：兩臺計算機，操作系統皆為windows 7,為什麼在網上鄰居里看不到對方？

A：像這種問題，需要綜合考慮，問題可能出現在以下幾種。1.檢測網卡驅動程序是否正確安裝？2.是否安裝了tcp/ip和netbeui協議？3.ip地址是否在同一網段？4.工作組是否相同?5.是否設置了文件共享？6.網絡線有無故障？是否按標準制作的？

Q：兩臺計算機，一臺為windows7，另一臺為windows 8，為何windows 8的計算機能訪問windows 7的資源，而windows 7卻不能訪問8？我已經在8下設置了共享。

A： 8可以訪問7，可以說明協議、工作組、網線都沒問題，你只需要在8的帳戶管理為7新開一個帳號即可，然後你用這個帳號登錄7，看看，一切搞定了吧。

Q： 用sygate作代理，我的工作站為什麼一直無法連上互聯網？

A： 首先你要保證你的服務器端已經正確安裝了sygate軟件，然後你需要在工作站上指定網關為安裝有sygate主機的IP，並設定好DNS，具體操作可參照sygate教程。

Q：還是用sygate作代理，有4臺機器工作站，為什麼老是有兩臺工作站不能上網？而這臺機器又不固定的，有時這臺，有時那臺？

A：很明顯，你所用的sygate系列號只允許3個用戶上網，解決的辦法是找到更多用戶(或者無限制)的系列號，再重新安裝即可。

Q：我想玩對戰遊戲，可是遊戲安裝是提示未找到相關協議？

A：有些遊戲需要tcp/ip協議支持，而有的則需要ipx協議支持，看你的遊戲支持哪種協議，建議你在計算機上把這兩種協議教安裝上，便萬事無憂了。

Q：使用netmeeting，工作站上的netmeeting一直無法拔通主機，不知何故？

A:請先檢查網絡是否連通，再看看主機上是否安裝天網等防火牆。
