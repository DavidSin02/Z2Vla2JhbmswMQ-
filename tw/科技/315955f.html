go語言設計模式之適配器模式

| 科技

適配器模式：將一個類的接口轉換成為客戶希望的另一個接口，Adapter 模式使得原本由於接口不兼容而不能一起工作的類可以一起工作（大話設計模式裡面的，很好的一本書）。

普通話說就是，有一些類的接口是不通用的，但是還需要一起工作，就可以使用這個模式，作為中間的一個適配者。其實在項目中，不止這種情況可以用，有時候為了分層，我們也可以在中間加上這一層，將原來的業務通過適配層分成不同的兩部分。

我們這裡說一個經典的例子，籃球翻譯，加入一個籃球隊都是外國人，他們都使用英語交流，那麼如果隊裡新來了一箇中國人，而且中國人不會英語，這樣他們就沒有辦法一起配合進攻防守，這時就需要一個翻譯，這就是一個典型的適配器模式。我們先看代碼實現：

package mainimport "fmt"type Player interface { 　　Attack() // 進攻方法 　　Defense() // 防守方法}// 前鋒type Forwards struct { 　　Name string}func (this *Forwards) Attack() { 　　fmt.Println("前鋒", this.Name, "正在進攻")}func (this *Forwards) Defense() { 　　fmt.Println("前鋒", this.Name, "正在防守")}// 中鋒type Center struct { 　　Name string}func (this *Center) Attack() { 　　fmt.Println("中鋒", this.Name, "正在進攻")}func (this *Center) Defense() { 　　fmt.Println("中鋒", this.Name, "正在防守")}// 後衛type Guards struct { 　　Name string}func (this *Guards) Attack() { 　　fmt.Println("後衛", this.Name, "正在進攻")}func (this *Guards) Defense() { 　　fmt.Println("後衛", this.Name, "正在防守")}// 外籍中鋒（不會英語的外籍中鋒）type ForeignCenter struct { 　　Name string}// 外籍中鋒進攻方法（認識漢語拼音）func (this *ForeignCenter) jinGong() { 　　fmt.Println("外籍中鋒", this.Name, "正在進攻")}// 外籍中鋒防守方法（認識漢語拼音）func (this *ForeignCenter) fangShou() { 　　fmt.Println("外籍中鋒", this.Name, "正在防守")}type Translator struct { 　　ForeignCenter ForeignCenter}func (this *Translator) Attack() { 　　this.ForeignCenter.jinGong()}func (this *Translator) Defense() { 　　this.ForeignCenter.fangShou()}func main() { 　　var b Player = &Forwards{Name: "巴蒂爾"} 　　b.Attack() 　　var m Player = &Guards{Name: "麥克格雷迪"} 　　m.Defense() 　　// 假設姚明不會英語，就需要一個翻譯，翻譯告訴姚明進攻或者防守 　　var ym Player = &Translator{ForeignCenter{Name: "姚明"}} 　　ym.Attack() 　　ym.Defense()}

隊裡面來了姚明，我們假設他不會英語，不認識Attack和Defense，那麼他就不知道如何配合隊友，不知道何時進攻防守。所以我們請來了一個翻譯Translator，將Attack翻譯成進攻，Defense翻譯成防守，然後告訴姚明，這樣就可以一起配合了。

大家也許會問，姚明自己翻譯多好，就不需要翻譯了，事實上是可以的，但是這不是適配器模式，適配器模式主要是不改變原有的接口，假如姚明又去了一個說法語的隊呢，姚明就需要自己翻譯法語，這樣就再修改姚明的結構體和方法。適配器主要是保持了原有的接口，同時讓不同的接口可以共同使用，這是核心。
