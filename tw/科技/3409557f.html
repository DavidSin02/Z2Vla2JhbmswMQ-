“樹狀數組或二叉索引樹（英語：Binary Indexed Tree），又以其發明者命名為 Fenwick 樹。其初衷是解決數據壓縮裡的累積頻率（Cumulative Frequency）的計算問題，現多用於高效計算數列的前綴和， 區間和。它可以以 O(logn) 的時間得到任意前綴和，並同時支持在 O(logn) 時間內支持動態單點值的修改。空間複雜度 O(n)。”

文章先介紹低位運算（lowbit）的基本知識，再提及如何將一個整數劃分為 logn 個區間的運算過程，進而延展到如何將線性序列以樹行結構進行存取，接著介紹高級數據結構——樹狀數組的兩個基本操作——查詢前綴和與單點增加，最後介紹了樹狀數組的一個應用——求解逆序對數。

樸素算法需要枚舉整數中所有的位，時間複雜度為 O(len)，len 為整數 n 的二進制表示下的位數。

為了高效獲取二進制表示下所有位是 1 的數值，可以利用 lowbit 運算，得到時間複雜度 O(len)，len 為二進制表示下為 1 的位的個數。

比如 n = 10，lowbit(10) = 2；接著令 n = n - lowbit(n) = 10 - 2 = 8，則 lowbit(n) = lowbit(8) = 8；接著令 n = n - lowbit(n) = 8 - 8 = 0，停止。

樹狀數組是基於以上思想的數據結構，基本用途是維護序列的前綴和。

那麼，假設有序列 A = {1,2,3,4,5,6,7,8}，現在的問題就是如何將這個序列劃分成 log n 個小區間。不妨，利用序列的索引值（以 1 為起點開始計數），根據上述計算區間的方式，將其以如下樹形結構展開。

樹狀數組（Binary Indexed Tree） 以樹形結構展開的序列 A


此時，以樹形結構展開的序列 A 中的每一個節點都對應著樹狀數組中的一個值。那麼這個值為以當前節點為根的子樹中所有節點值的總和。

接著，我們看下以樹形結構展開的樹狀數組是什麼樣的。

以樹形結構展開的樹狀數組（Binary Indexed Tree）

  • Index 代表序列 A 中元素的索引，為了方便，以 1 為起點計數
  • Original Value 代表序列 A 中的元素值
  • BIT Value（Binary Indexed Tree Value)代表樹狀數組中的值
  • Binary bit 代表索引值的二進制形式
  • Low bit 代表索引值的二進制形式下的地位

    • 上圖中最大的區別是某些節點中的值發生了變化。這是因為，在以樹形結構展開的樹狀數組中的每一個值代表的是一個區間的總和。這個區間即為我們上述求解的區間，比如一個整數 7，可以將其劃分成 [1，4] ，[5，6] 和 [7，7] 三個小區間。那麼，這三個小區間的右端值作為索引對應的樹狀數組中的值即為當前區間元素的總和。

    比如 Index = 4 對應的樹狀數組的值為（BIT Value）10，它代表 [1，4] 這個區間的和。

    比如 Index = 6 對應的樹狀數組的值為 11，它代表 [5，6] 這個區間的和。

    樹狀數組與逆序對

    對於一個序列 A，如果 i < j，並且 A[i] > A[j] ，那麼則稱 A[i] 與 A[j] 構成逆序對。利用樹狀數組數據結構可以求解序列 A 中的逆序對個數。

    1. 逆序遍歷序列
    2. 利用樹狀數組的性質，使用 query 操作獲取每一個元素的逆序對數
    3. 將當前元素更新（update）到樹狀數組中
    4. 循環迭代上述步驟，直到遍歷所有元素

    C++ 代碼實現

    在每一次更新（update）樹狀數組時，以元素的值作為樹狀數組的索引，更新的值為 +1，代表個數。

    在每一次獲取（query） 逆序對數時，存在於樹狀數組中的元素的索引值都比當前元素的大（逆序遍歷），那麼自然獲取到的樹狀數組的值即為索引值比當前元素的大，且值比當前元素的小的個數。

    注意，上述的求解過程時，如果序列 A 的值範圍較大時，那麼需要離散化處理。
