SpringBoot系列（14）：Spring AOP裝逼指南之實現操作日誌記錄

2020-11-14
| 科技

摘要

AOP ，也稱為"面向切面編程"，其大名早已如雷貫耳，是 Spring 框架的核心特性之一，相信各位小夥伴也早已聽聞過，其最普遍的用法是"記錄應用系統業務模塊的操作日誌"，今天我們就來分享介紹一下如何利用Spring AOP實現業務模塊操作日誌的記錄。

內容

Spring AOP，是"面向切面編程"的簡稱，可以起到"解耦業務模塊"的作用，深層次的作用可以利用網上一位博主所說的一句話進行概括，即：

"AOP 可以實現在不修改源代碼的情況下給程序動態統一添加功能，而不需要破壞某個操作業務模塊代碼的完整性"

對於Spring AOP，在這裡有必要再囉嗦一番其核心要素，"面向切面編程"，一聽就知道其核心要素是"切面"，而"切面"並非只是單純存在的玩意，它由兩大部分組成，"切點PointCut"+"通知Notice"，下面，讓我用"通俗易懂"的語句表達出來吧：

"切點"：在什麼地方觸發我們自定義的操作（定義的是"何處"）

"通知"：在什麼時候執行我們自定義的操作，同時定義好"自定義的操作"具體是什麼！

"切面"：切點 + 通知；即在什麼時候，在哪個地方觸發執行什麼樣的操作！

好了，若需要具體的、深層次的關於專業術語的介紹、剖析，可以上官網或者博客找找相應的介紹吧！下面我們就進入實際的代碼實戰過程！

（1）首先，我們當然需要建立一個"記錄日誌的數據庫表"，sys_log，其DDL定義如下所示：

CREATE TABLE `sys_log` ( `id` bigint(20) NOT NULL AUTO_INCREMENT, `username` varchar(50) DEFAULT NULL COMMENT '用戶名', `operation` varchar(50) DEFAULT NULL COMMENT '用戶操作', `method` varchar(200) DEFAULT NULL COMMENT '請求方法', `params` varchar(5000) DEFAULT NULL COMMENT '請求參數', `time` bigint(20) NOT NULL COMMENT '執行時長(毫秒)', `ip` varchar(64) DEFAULT NULL COMMENT 'IP地址', `create_date` datetime DEFAULT NULL COMMENT '創建時間', PRIMARY KEY (`id`)) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='系統日誌';

然後，採用Mybatis逆向工程生成相應的Entity、Mapper、Mapper.xml，之後便是等待著被調用！

（2）然後，我們定義一個註解LogAnnotation，這個註解將用於應用系統中"所有需要觸發記錄日誌"的操作模塊對應的方法之上，在後面大家會發現，正是這種方式，才實現了所謂的"服務模塊解耦"、"在不修改源代碼的情況下給程序動態統一添加功能"的作用。其完整源代碼如下所示：

@Target(ElementType.METHOD)@Retention(RetentionPolicy.RUNTIME)@Documentedpublic @interface LogAnnotation {	//value將用於存儲 “具體的操作模塊的操作描述” String value() default "";}

（3）緊接著，我們需要定義一個重量級的東西，即"切面"，這個切面是基於AspectJ註解實現的類LogAspect，其完整源代碼如下所示：

/** * 日誌切面 = 切點+通知 * @Author:debug (SteadyJack) * @Link: weixin-> debug0868 qq-> 1948831260 * @Date: 2019/11/7 15:18 **/@Aspect@Componentpublic class LogAspect { private static final Logger log= LoggerFactory.getLogger(LogAspect.class); @Autowired private SysLogMapper sysLogMapper; @Autowired private ObjectMapper objectMapper; //定義切點:發生的時機 - 即一旦加了這個註解，將觸發某些事情 @Pointcut("@annotation(com.debug.springboot.server.annotation.LogAnnotation)") public void logPointCut(){} //定義通知：環繞通知 - 執行核心任務之前 + 執行完核心任務之後 該做的事情 @Around("logPointCut()") public Object around(ProceedingJoinPoint joinPoint) throws Throwable{ long start=System.currentTimeMillis(); //TODO:執行核心任務 Object object=joinPoint.proceed(); long time=System.currentTimeMillis()-start; saveLog(joinPoint,time); return object; } //TODO:保存操作日誌 private void saveLog(ProceedingJoinPoint point,Long time) throws Exception{ log.info("開始觸發-保存操作日誌"); MethodSignature signature= (MethodSignature) point.getSignature(); Method method=signature.getMethod(); SysLog entity=new SysLog(); //TODO:獲取請求操作的描述信息 LogAnnotation annotation=method.getAnnotation(LogAnnotation.class); if (annotation!=null){ entity.setOperation(annotation.value()); } //TODO:獲取操作方法名 String className=point.getTarget().getClass().getName(); String methodName=signature.getName(); entity.setMethod(className + "." + methodName + "()"); //TODO:獲取請求參數 Object[] args=point.getArgs(); entity.setParams(objectMapper.writeValueAsString(args[0])); //TODO:獲取剩餘參數 entity.setTime(time); entity.setCreateDate(DateTime.now().toDate()); entity.setUsername("debug"); entity.setIp("localhost"); sysLogMapper.insertSelective(entity); }}

（3）最後，我們在MultipartSourceController 中寫個請求方法，並在該請求方法上加入日誌註解，用於記錄該請求方法在執行前、後、執行中的軌跡和相應的參數。完整的源代碼如下所示：

 @LogAnnotation("多數據源時-跨事務-日誌操作") @RequestMapping(value = "update",method = RequestMethod.POST,consumes = MediaType.APPLICATION_JSON_UTF8_VALUE) public BaseResponse update(@RequestBody LogUpdateDto dto) throws Exception{ BaseResponse response=new BaseResponse(StatusCode.Success); User user=new User(); user.setName(dto.getName()); user.setCode(dto.getCode()); userMapper.insertSelective(user); return response; }

而LogUpdateDto只是一個簡單的實體類：

/** * @Author:debug (SteadyJack) * @Link: weixin-> debug0868 qq-> 1948831260 * @Date: 2019/11/7 16:14 **/@Datapublic class LogUpdateDto implements Serializable{ @NotBlank private String name; @NotBlank private String code;}
