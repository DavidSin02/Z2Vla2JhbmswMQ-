bee智能化代碼生成器，一分鐘實現前後端系統

2020-11-14
| 科技

beego團隊 GoCN

前言

前不久，我們宣佈了 beego 再出發，要為大家帶來更好用，功能更多的 beego。我們要做的不只是 beego 本身的重構，還有 beego 周邊生態的完善。

眾所周知，beego 是用來構建 go 應用的開源框架，那麼 go 應用有了，相應的管理後臺怎麼辦，怎麼才能只用簡單的配置快速的搭建出一套完整的前後端代碼，從而解放程序員於 crud 之中呢？

bee 1.3

它來了它來了，它帶著大家的福音走來了，經過社區內同學日以繼夜的努力，bee 1.3 正式發佈了，為什麼說它帶來了福音，是因為這個版本新加的命令 bee gen pro，下面就著重介紹一下這個命令都能做什麼。

bee gen pro

上面也提到了，crud 是很多人繞不過的坎，並不是說它有多困難，而是大量的重複工作會讓很多人不勝其煩，beego 社區的同學考慮到了這個問題，所以在 bee 的新版本里推出了這個命令，簡單來說，就是可以使用官方提供的配置模版，或者個性化定製的模版，通過這個命令快速的構建出 crud 代碼，從而節省大量的時間。

你以為這就完了？不不不，在構建出 crud 代碼的同時，我們還可以生成相應的管理後臺，數據庫，並且自動打通前後端，讓你可以直接在管理後臺體驗 crud 的功能。

到這裡如果還覺得不過癮，我們還提供了 Docker 環境的實例，可以幾分鐘之內飛速部署，快速體驗 bee gen pro 帶來的舒爽體驗。

設計

目標

所以我們將Bee設計為CRUD的元數據中心，使CRUD解析數據和代碼模板分開。其優勢在於 • 我們的模板不侷限於go代碼。還可以生成vue，ant design pro等代碼，可以將前後端打通。• 我們提供生成代碼的官方模板，用戶可以提pr，貢獻更多的模板 • 如果官方模板不滿足用戶的需要，用戶可以將代碼倉庫指定到自己的地址或者代碼倉庫，用自定義的生成代碼模板

設計架構

快速體驗

創建項目

  • 更新 bee go get -u github.com/beego/bee
  • 創建一個 beego 項目， bee new beeblog
  • 創建一個 ant design pro4 項目， cd beeblog && mkdir ant && cd ant && yarn create umi
  • 安裝 ant4 的依賴，npm i
  • 創建數據庫， CREATE DATABASE IF NOT EXISTS beeblog DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
  • 創建 beegopro.toml
dsn = "root:@tcp(127.0.0.1:3306)/beeblog"proType = "ant4"[models.blog]    names = ["name","content","created_at","updated_at"]    orms = ["string","string","datetime","datetime"]    comments = ["名稱","內容","創建時間","更新時間"][path]    beego =  "."    ant  = "./ant/src/pages"
  • 注入靈魂，執行生成腳本， bee pro gen

前端添加代碼

  • /ant/package.json 增加 mock 方式 "start:dev": "cross-env REACT_APP_ENV=dev umi dev",
  • ./ant/config/proxy.ts 增加 beego 反向代理
export default {  dev: {    '/api/blog': {      target: 'http://127.0.0.1:8080',      changeOrigin: true,      pathRewrite: { '^': '' },    },    '/api/': {      target: 'https://preview.pro.ant.design',      changeOrigin: true,      pathRewrite: { '^': '' },    },  },  test: {    '/api/': {      target: 'https://preview.pro.ant.design',      changeOrigin: true,      pathRewrite: { '^': '' },    },  },  pre: {    '/api/': {      target: 'your pre url',      changeOrigin: true,      pathRewrite: { '^': '' },    },  },};
  • 在 ./ant/config/config.ts 添加菜單
{ name: '博客',icon: 'RocketOutlined',path: '/blog',component: './blog/list'},{ path: '/blog/info', component: './blog/info'},{ path: '/blog/update', component: './blog/update'},{ path: '/blog/create', component: './blog/create'},

添加 beego 代碼

  • conf/app.conf 增加數據庫配置
appname = .httpport = 8080runmode = dev####################MySQL 數據庫配置###########################db_adapter=mysql# 您的數據庫hostdb_host="127.0.0.1"#您的數據庫端口db_port=3306#您的數據庫用戶名db_username="root"# 您的數據庫密碼db_password=""# utf8或者utf8mb4數據庫。如果數據庫還沒創建，並且賬號有創建權限，將自動創建。db_database="beeblog"
  • main.go 注入數據庫代碼
package mainimport (  _ "github.com/go-sql-driver/mysql"  _ "beeblog/routers"  "fmt"  "github.com/astaxie/beego"  "github.com/astaxie/beego/orm")func main() {  dataSource := fmt.Sprintf("%s:%s@tcp(%s:%s)/%s?charset=utf8mb4&parseTime=true&loc=Local", beego.AppConfig.String("db_username"), beego.AppConfig.String("db_password"), beego.AppConfig.String("db_host"), beego.AppConfig.String("db_port"), beego.AppConfig.String("db_database"))  orm.Debug = true  err := orm.RegisterDataBase("default", "mysql", dataSource)  if err != nil {    panic(err)  }  beego.Run()}

運行項目

  • 運行 Beego bee run
  • 運行 Ant cd ant && npm run start
