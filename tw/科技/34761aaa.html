字符編碼知識整理-漢牛筆記（編程基礎03）

2020-11-14
| 科技

前言

  • 字符

字符(Character)是各種文字和符號的總稱，包括各國家文字、標點符號、圖形符號、數字等。是人類能直接識別和處理的符號。

  • 字符集

字符集(Characterset)是字符的集合。

  • 字符編碼

計算機只能處理二進制信息，所以需要對每一個字符進行二進制編碼。對字符進行編碼的二進制代碼稱為字符編碼。

  • 亂碼問題

在不同的字符編碼標準下，同一個字符的二進制編碼可能不同。如果用A編碼存儲的信息，用B編碼去解讀，就會出現亂碼問題。

常見的字符編碼

  • ASCII

American Standard Code for Information Interchange（ASCII - 美國國家信息交換標準碼）。一種使用7個或8個二進制位進行編碼的方案，最多可以給256個字符(包括字母、數字、標點符號、控制字符及其他符號)分配數值。參見下文《附錄：ASCII編碼集》。

標準ASCII碼是7位編碼，對應的ISO標準為ISO646標準，一個字節中多餘出來的一位（最高位）在計算機內部通常保持為0。

由於標準ASCII字符集字符數目有限，而且是針對英語設計的，當處理帶有音調標號（形如漢語的帶聲調字母）的歐洲文字時就會出現問題，為此，國際標準化組織又制定了ISO2022標準，將ASCII字符集擴充為8位代碼的統一方法，這些擴充字符的編碼均為高位為1的8位代碼（即十進制數128~255），稱為擴展ASCII碼。

  • ANSI

American National Standards Institute（ANSI - 美國國家標準學會），由公司、政府和其他成員組成的自願組織。

為了擴充ASCII編碼，以用於顯示本國的語言，不同的國家和地區制定了不同的標準。這些使用2個字節來代表一個字符的各種漢字延伸編碼方式，稱為ANSI編碼，又稱為MBCS（Muilti-Bytes Charecter Set，多字節字符集）。在簡體中文系統下，ANSI編碼代表GB2312編碼，在日文操作系統下，ANSI編碼代表JIS編碼，在寶島臺灣的繁體中文系統下，ANSI編碼代表BIG5編碼。

  • GB2312

GB2312碼是中華人民共和國國家漢字信息交換用編碼，全稱《信息交換用漢字編碼字符集——基本集》，由國家標準總局發佈，1981年5月1日實施，通行於大陸，新加坡等地也使用此編碼。共收入漢字6763個和非漢字圖形字符682個。

  • BIG5

大五碼（Big5），又稱為五大碼，一種繁體中文漢字字符集，其中繁體漢字13053個，808個標點符號、希臘字母及特殊符號。是1984年由臺灣13家廠商與臺灣地區財團法人信息工業策進會為五大中文套裝軟件（宏碁、神通、佳佳、零壹、大眾）所設計的中文內碼，是使用繁體中文社群中最常用的電腦漢字字符集標準，在在臺灣、香港與澳門地區使用廣泛。

Big5碼的產生背景：當時臺灣不同廠商各自推出不同的編碼，如IBM5550、王安碼等，彼此不能兼容；另一方面，臺灣當時尚未推出官方的漢字編碼，而中國內地所推行的GB2312編碼，亦未有收錄繁體字。

  • GBK

GBK是漢字編碼標準之一，全稱《漢字內碼擴展規範》，Chinese Internal Code Specification，由中華人民共和國全國信息技術標準化技術委員會1995年12月1日製訂。GBK採用雙字節編碼，總計23940個碼位，共收入21886個漢字和圖形符號，其中漢字（包括部首和構件）21003個，圖形符號883個。

  • Unicode

使用ANSI編碼，世界上存在著多種編碼方式，在ANSI編碼下，同一個編碼值，在不同的編碼體系裡代表著不同的字。導致國際電子郵件和網頁經常出現亂碼，就是因為信息的提供者可能是日文的ANSI編碼體系和信息的讀取者可能是中文的編碼體系，他們對同一個二進制編碼值進行顯示，採用了不同的編碼，導致亂碼。

如果有一種編碼，將世界上所有的符號都納入其中，無論是英文、日文、還是中文等，大家都使用這個編碼表，就不會出現編碼不匹配現象。每個符號對應一個唯一的編碼，亂碼問題就不存在了。這就是Unicode編碼。

Unicode是一個很大的集合，規定用4個字節存儲一個符號，現在的規模可以容納100多萬個符號，世界上的每一個符號的編碼都不一樣。

  • UTF

UTF是Unicode Translation Format，即把Unicode轉做某種格式的意思。Unicode固然統一了編碼方式，但是它的效率不高，比如每個英文字母前都必然有三個字節是0，這對存儲和傳輸來說都很耗資源。為了提高Unicode的編碼效率，於是就出現了UTF編碼。目前存在的UTF格式有：UTF-7,UTF-7.5,UTF-8,UTF-16,以及UTF-32。

  • UTF-8

UTF-8是Unicode的一種變長字符編碼，又稱萬國碼，使用1到6個字節編碼UNICODE字符，比如英文字母可以只用1個字節就夠了。

  • UTF-16

UTF-16比起UTF-8，好處在於大部分字符都以固定長度的字節(2字節)儲存，但UTF-16卻無法兼容於ASCII編碼。
