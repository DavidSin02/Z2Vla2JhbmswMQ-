C語言學習|指針的介紹

2020-11-14
| 科技

指針是C語言提供的一種特殊而又有非常重要的數據類型。

聯合使用指針和結構體這兩種數據類型可以有效地表示許多複雜的數據結構，如隊列、堆棧、鏈表、樹、圖等。

指針這個內容為什麼那麼的重要？

1.指針為函數提供修改變量值的手段

2.指針為C語言的內存動態分配系統提供支持

3.指針為動態數據結構（如鏈表、隊列、二叉樹等）提供支持

4.指針可以改善某些子程序的效率

指針的概述

程序是由數據和指令組成的，而數據和指令在執行過程中是存儲在計算機內存中的，變量是程序數據中的一種，因此變量也存儲在內存中。

內存地址：內存中的每字節都有一個唯一的編號，以便內存的管理。

內存地址的編碼方式與操作系統有關，在32為計算機上，內存地址編碼是32位，從0x00000000到0xFFFFFFFF，最多支持4G的內存。任何數據存儲到內存中的過程，都需要記錄兩條信息，一是分配的內存空間的首地址，二是分配的內存空間大小。

如果變量的值已經存儲與內存中，對變量的訪問有兩種方式：直接尋址和間接尋址

直接尋址，顧名思義就是直接訪問變量的值，使用變量名或使用變量的地址都可直接引用變量的值。

例如：

short int a=12；

假設編譯器在編譯程序時為短整型變量分配了兩個字節的存儲單元。如圖

直接尋址示意圖

由於程序運行時變量a被初始化為12，因此變量a當前值是12，也可用變量名a表示變量的值。這種直接按變量名來存取變量值訪問方式。而用地址訪問就必須知道變量的地址，需要使用取地址運算符&來獲取變量的地址。

例如，用&a表示變量a在內存中所佔存儲單元的首地址，而無須關心該地址的具體值是多少。如果通過p=&a賦值操作，將變量a的地址賦值給另一個變量p，那麼我們就獲得了另外一種訪問變量的方法，即通過先訪問變量p獲得變量a的地址值，然後再到該地址值代表的存儲單元中去訪問變量a。當然，這裡的變量p不是普通類型的變量，它是一種特殊類型的變量，即指針類型的變量，簡稱指針變量。

指針變量是C語言中專門用於存放地址型數據的變量。這種通過指針變量間接存取它所指向的訪問方式，稱為間接尋址。間接尋址是通過其他變量來獲取要訪問變量的地址，再進行訪問。

間接尋址示意圖

C語言提供了兩個單目運算符&和*，使用它們可以方便地實現有關地址和指針的運算。

取地址運算符&用於得到變量的地址。

p=&a；

當指針變量p中存儲了變量a的地址時，我們就稱指針變量p指向了變量a。在32位計算機上，由於地址是32位的，因此要用指針變量保存地址就需要佔用4字節的存儲空間。

為了得到一個指針所指對象的內容，可以用*運算符放在指針變量前實現通過指針變量間接訪問它所指向的存儲單元的目的，*稱為指針運算符，也稱間接尋址運算符或解引用運算符，它返回其操作數（即一個指針）所指向的對象的值。

假設指針變量p已指向了變量a，那麼通過*p也可以得到指針變量p所指向的變量a的值，輸出*p的值和輸出a的值是等價的。

以這種方式使用指針運算符*，稱為指針的解引用。對指針變量進行解引用時，要求指針已被正確初始化或已確認它已指向某個確定的內存單元。

如果一個指針變量沒有被初始化，就使其指向內存中某個確定的存儲單元，對這個指針變量解引用，將引起一個致命的運行時錯誤，或者意外地改寫內存中的重要數據，使程序得到一個錯誤的運行結果。

編程小實例：

從鍵盤任意輸入兩個整數

#include<stdio.h>

void swap(int x,int b);

int main()

{

int a,b;

printf("Please enter a and b:");

scanf("%d %d",&a,&b);

printf("Before swap:a=%d,b=%d\n",a,b);

swap(a,b);

printf("After swap:a=%d,b=%d\n",a,b);

return 0;

}

void swap(int x,int y)

{

int temp;

temp=a;

a=b;

b=temp;

}

程序運行的結果如下：

Please enter a and b:36 12（按回車）

Before swap:a=36,b=12

After swap:a=36,b=12

程序運行示意圖如下

由於形參x和y是動態局部變量，離開定義它們的函數swap後，分配給它們的存儲空間就被釋放了，當然保存在變量x和y中的互換結果也就找不到了。這時，主函數中實參a和b的值卻為發生任何的改變，仍然保持原來的值，即a=36，b=12，因此，程序執行第二個printf的輸出結果就是“After swap:a=36,b=12”。顯然swap毫無做功。

由於C語言中函數參數的傳遞方式是“單向傳值”，當用簡單變量作為函數實參進行函數調用時，由於實參向形參的傳值是單向的，實參的值被複制個形參後，對形參值得任何改變都不會影響主函數中對應的實參值，因此，為了在函數調用後得到修改後的參數值，雖然可以利用return語句將其值從被調函數中返回給主函數，但return僅限於從函數返回一個值，但需要得到兩個或多個修改了的值時，就要用傳地址值得方法。

一種方法是用數組名作為函數參數，即將數組在內存中的起始地址傳給形參，這樣做可以得到多個變化的值。這些變化的值存放在實參數組中，它是通過形參數組將其改變的。

如果待修改的數據不多，且不是放在一個數組中，那麼可以用指針變量作為函數參數。由於指針變量中存放的某個變量的地址，因此它傳給形參的是指針所指向的變量所佔存儲單元的地址，即不是直接複製變量的值給形參，而是複製了變量地址給形參，相當於房主配了一把鑰匙給房客，房客在得到房間鑰匙的同時也獲得了修改房間內容的權限。由於實參和形參都表示同一個地址值，相當於它們都指向了同一個內存單元，因此，在這種情況下，被調函數對形參所指向的存儲單元值的任何修改，都可以在主函數中通過相應的實參來間接讀取。
