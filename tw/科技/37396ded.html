數據結構系列：哈希表？這涉及的是“加密/區塊鏈”等技術的核心

2020-11-14
| 科技

之前的幾篇文章分別介紹了線性表和樹結構相關的特點和應用，本文將分享另一種主要應用在加密領域的數據結構，哈希表(散列表)。當然，不僅僅限於加密領域，因為哈希表的效率極其優秀。


什麼是哈希表

官方概念

哈希表（Hash table），是根據鍵（Key）而直接訪問在內存存儲位置的數據結構。也就是說，它通過計算一個關於鍵值的函數，將所需查詢的數據映射到表中一個位置來訪問記錄，這加快了查找速度。這個映射函數稱做散列函數，存放記錄的數組稱做散列表。

非官方理解

哈希表既然作為一種抽象的數據結構，那必然和棧結構或者樹結構一樣，對存放的數據呈現出一定的規律特點。那麼哈希表這種結構的特點是什麼？總結一句話：哈希表可以由數據項的值確定數據存放的位置，每一個存儲位置，稱之為槽，用來保存數據項（value），每個槽應該有唯一的標記(key)

哈希表存儲的是一種映射關係：可以想象假設存在兩張表，一張存儲key，用來做唯一標記；一張存儲value，存放key對應的所有數據項。如果你要想找某個數據項，必須通過key值，key會通過某種你目前所不瞭解的方法被轉化為唯一的哈希值(位置信息)，根據這個唯一值，才能定位到你所需要查找的數據。

就像python中字典的基礎數據結構通過key就可以找到value，但是key值必須唯一

哈希表相關的概念

上面簡單介紹了到底什麼是哈希表，接下來介紹一下學習哈希表需要知道的相關概念

散列技術

數據的存儲位置和他的關鍵字之間建立一個明確的映射關係，使得每個關鍵字key對應一個存儲位置f(key)，查找時，根據這個確定的對應關係找到給定值key對應的映射f（key）

散列值

把任意長度數據的輸入，通過散列函數，變換成固定長度字符串的的輸出，該輸出就是散列值

散列函數

通過算法實現從數據項到存儲槽名稱轉換的函數稱之為散列函數，接收數據項作為參數，返回數據集中唯一的標記（key），這是實現哈希表的核心

完美散列函數

1) 概念

給定一組數據項，如果有一個散列函數能夠把每個數據項映射到不同的槽中，(沒有相同的key),那麼這個函數就稱之為完美散列函數

2) 難點

如果數據集合是不變的，其實很好寫出這樣的函數，但是遇到變化的數據集合，則很難實現通用的完美散列函數。

3) 近似完美散列函數

完美散列函數基本不存在，所以要想辦法實現儘可能的完美，一般包含以下幾個條件：1.對於不同的數據集合，衝突越少越好；2.函數要儘可能的性能高；3.不會使用大量的內存

4) 近似完美函數舉例

MD5：將任何長度的數據變換為固定長度為128位（16字節）的摘要

SHA：SHA-0/SHA-1輸出160位，SHA-256/SHA-224/SHA-512/SHA-384分別輸出後面數字長度的位數

一般對散列函數的要求

  1. 壓縮性：任意長度的數據，得到的散列值的長度是固定的
  2. 易計算性：生成散列值不難
  3. 抗修改性：對原始數據做出更改的話，散列值會發生劇烈變化
  4. 抗衝突性：基本不會生成相同的散列值

散列涉及到的一些特點

  1. 假設存放數據的槽有11個，而數據項求餘數後添加到槽中只填滿了5個，那麼此時就可說負載因子為5/11
  2. 所以要查看一個數據是否在一個數據集合中，如果使用哈希的方法，我們只需要通過標記定位到槽的位置，看下槽中是否由數據項即可
  3. 散列是不可逆的，就是可以通過算法實現數據項到散列值的映射，但是不可能通過散列值求出原數據
  4. 散列函數的輸入和輸出並不是唯一對應的關係，如果兩個散列值相同，那麼兩個輸入有可能是相同的，也有可能是不同的

哈希碰撞

最簡單的散列函數就是依次對數據集中的數據（找一個任意有效數作為除數）求餘數，每個數據項的餘數就是標記數據項的（key），但是這種方式很容易有不同的數據項插入到一個槽中了，這樣就造成了衝突，稱之為哈希碰撞

簡單點說，就是：如果兩個數據項被散列映射到同一個槽，需要一個系統的方法在散列表中保存第二個數據項，這個解決的過程就稱之為解決衝突

目前比較成熟的解決方案

  • 線性探測：為衝突的數據項再找一個開放的空槽來保存數據，最簡單的就是從衝突的槽開始往後掃描，直到碰到一個空槽，如果到散列表尾部還沒找到，則從首部接著掃描，這種尋找空槽的技術稱之為"開放定址"，向後逐個槽尋找的方法則是開放定址技術中的線性探測。線性探測方法的問題在於如果同一個槽衝突的數據項越多的話 ，這些數據就會再槽附近聚集起來，呈現數據項聚集的趨勢。
  • 跳躍式探測：為避免出現聚集的情況，對線性探測的方法提出了改進，就是將數據位置+1的操作改為數據位置+n（n為大於1的有效數字），構成跳躍式探測，針對這個n的取值，不能被散列表大小整除，不然永遠有空槽找不到，一個技巧是把散列表的大小設置為質數
  • 二次探測：不再是固定的n值，而是逐步增加n的大小
  • 數據項鍊：將每個槽的格式更改為可以存儲數據集合的形式，這樣每個槽就可以容納多個數據項，只要有散列衝突，就把新的數據項添加到數據項集合中

散列函數的應用

  1. 數據一致性做校驗，比如從網站上下載的數據是否完整，比如兩個文件的內容是是否完全一致，因為不論多大的數據，生成的散列值是一樣的，只有128位（如果是MD5的話）
  2. 密碼的加密，對散列值進行對比
  3. 防文件篡改
  4. 區塊鏈技術，（區塊鏈是一種分佈式數據庫），通過網絡連接的節點，每個節點保存著整個數據庫的所有數據，最大的特點就是去中心化，每個節點都擁有全部信息。區塊鏈由一個個區塊組成，區塊分為頭和體，頭信息包含元數據和前一個區塊的信息，由於散列函數的特徵，可以保證數據的真實可靠。

哈希函數的一個實現

最基礎的針對非數字的散列函數

如果 slots 當前 hashvalue 位置上的值為key,則用新值替代舊值            if self.slots[hashvalue] == key:                 self.data[hashvalue] = data              else:                # 重新 rehash，實際相當於探測 hashvalue後一個位置                nextslot = self.rehash(hashvalue,len(self.slots))                 # 如果後一個位置不為空，且不等於當前值即被其他值佔用，則繼續探測後一個                while self.slots[nextslot] != None and self.slots[nextslot] != key:                     nextslot = self.rehash(nextslot,len(self.slots))                                    # 如果後一個值為空，則插入；為原來的值，則替換                if self.slots[nextslot] == None:                    self.slots[nextslot]=key                    self.data[nextslot]=data                else:                    self.data[nextslot] = data #replace                    """餘數法計算 hashvalue"""    def hashfunction(self,key,size):          return key%size        """使用 +1 法來重新找位置插入"""    def rehash(self,oldhash,size):        return (oldhash+1)%size 	# 通過key取value    def get(self,key):        startslot = self.hashfunction(key,len(self.slots))         data = None        stop = False        found = False        position = startslot                while self.slots[position] != None and  not found and not stop:            #如果slots當前位置上的值等於 key,則找到了對應的 value            if self.slots[position] == key:                 found = True                data = self.data[position]            else:                 # 否則的話，rehash後繼續搜尋下一個可能的位置                position=self.rehash(position,len(self.slots))            # 如果最後又回到了第一次搜尋的位置，則要找的 key不在 slots中            if position == startslot:                 stop = True        return data     def __getitem__(self,key):        return self.get(key)     def __setitem__(self,key,data):        self.put(key,data)
