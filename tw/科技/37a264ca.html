Java基礎

一：java的基本類型，

java有8種基本類型，分別是int,double ,char,byte，short，long，float,boolean

按照分類來說，其基本數據類型分為：整數類型、浮點類型、字符類型、布爾類型這四個類型。

整數類型：字節型（byte）、短整型（short）、整型（int）、長整型（long）這四種；

浮點類型：分為float（單精度型）和double（雙精度類型）這兩種。float（單精度型）一般在精度要求不高的時候使用。double（雙精度類型）一般在數值精度高的時候使用。

布爾類型：一般用true表示“真”、false表示“假”。用關鍵字boolean來定義；

如果你用double,float等來做金額計算，會發現得不到你想要的結果，甚至可能在不同的場景下等到的結果也會不一樣。

其實這個是阿里巴巴java開發規約中提到的：

【強制】小數類型為decimal，禁止使用float和double。

說明：float和double在存儲的時候，存在精度損失的問題，很可能在值的比較時，得到不正確的結果。如果存儲的數據範圍超過decimal的範圍，建議將數據拆成整數和小數分開存儲

二：參數傳遞、

Java的參數到底倒是值傳遞還是引用地址傳遞？分兩種情況：

1.當基本類型（8種基本類型）作為參數傳遞時，是傳遞值的拷貝，無論你怎麼改變這個拷貝，原值是不會改變的

2、對象作為參數傳遞時，是把對象在內存中的地址拷貝了一份傳給了參數。

例如1，基本類型傳遞；

public class Example1{

public static void main(String[] args) {

int i = 4;

System.out.println("改變之前, n = " + n);

changeData(n);

System.out.println("調用changeData(n)之後, n = " + n);

}

public static void changeData(int number) {

n = 14;

}

}

執行下看打印結果：

改變之前, n = 4

調用changeData(n)之後, n = 3

例如2，Java裡面的String 比較特殊，如果是這樣定義的

String str = “Java按照值傳遞”;

那麼它的傳遞方式和java的基本類型傳遞方式是一樣的，

因為String的API中有這麼一句話：“their values cannot be changed after they are created”， 表示string對象一旦被創建是不能被改變的；

如果是 這樣定義：

String str = new String(“Java按照值傳遞”);

這種方式創建的字符串不放入常量池，是在堆中複製改變量的副本，會將堆中的對象的地址複製給str;；

例如：

例如3，當對象作為參數傳遞的時候，會出現什麼情況呢？

public class Example1{

public static void main(String[] args) {

int n = 4;

System.out.println("改變之前, n = " + n);

Example1 example1 = new Example1();

example1.changeData(n);

System.out.println("調用changeData(n)之後, n = " + n);

}

public void changeData(int number) {

number = 14;

}

}

先看輸出結果：

改變之前, sb = Hello

A調用changeData(n),之後 sb = Hello World!

對象作為參數傳遞時，是把對象的引用傳遞過去，如果引用在方法內被改變了，那麼原對象也跟著改變；

所以：java中沒有引用傳遞，只有值傳遞，引用類型屬於特殊值傳遞（是將它的地址副本給了參數，但是它與基本數據類型不同，如果地址指向的對象發生了變化，因為共享原因，原始對象也會改變）。
