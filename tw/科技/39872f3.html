5分鐘掌握Elasticsearch空間查詢之geo_shape

2020-10-29
| 科技

在筆者之前的文章中，已經介紹過Es對空間類型geo_point的支持了。今天筆者來介紹一下另一種重要的空間類型geo_shape。

1 geo_shape簡介

使用geo_shape類型，可以支持對任意空間幾何形狀的檢索，比如圓形、矩形、多邊形。如果我們存儲的數據是線、面類型，都可以用geo_shape。另外，geo_shape也是可以兼容點的存儲的。

2 geo_shape選項

如果要使用geo_shape類型進行數據存儲，需要在創建mapping時顯式指定。可以支持的選項包括：

  • tree：可以顯示指定前綴樹的實現，支持geohash或者quadtree；
  • precision：存儲精度，默認是50m（對應層級21）;
  • tree_levels：樹的層級，默認21（對應精度50m）；
  • strategy：檢索策略，默認是term，可以指定為recursive；

3 geo_shape設置示例

在插入geo_shape類型的數據之前，首先需要創建mapping（可以理解為關係型數據庫中的表結構）

PUT /test{ "mappings": { "properties": { "location": { "type": "geo_shape", "precision": "5km" } } }}

4 geo_shape數據存儲示例

geo_shape存儲的是空間數據，支持geojson或者WKT格式。

geojson/WKT/es中的空間類型

4.1 存儲Point類型數據——geojson格式

POST /test/_doc{ "location" : { "type" : "point", "coordinates" : [-77.03653, 38.897676] }}

4.2 存儲Point類型數據——WKT格式

POST /test/_doc{ "location" : "POINT (-77.03653 38.897676)"}

4.3 存儲多邊形（polygon）數據

POST /test/_doc{ "location" : { "type" : "polygon", "coordinates" : [ [ [100.0, 0.0], [101.0, 0.0], [101.0, 1.0], [100.0, 1.0], [100.0, 0.0] ] ] }}POST /test/_doc{ "location" : "POLYGON ((100.0 0.0, 101.0 0.0, 101.0 1.0, 100.0 1.0, 100.0 0.0))"}

4.4 存儲線（multilinestring）的數據

POST /test/_doc{ "location" : { "type" : "multilinestring", "coordinates" : [ [ [102.0, 2.0], [103.0, 2.0], [103.0, 3.0], [102.0, 3.0] ], [ [100.0, 0.0], [101.0, 0.0], [101.0, 1.0], [100.0, 1.0] ], [ [100.2, 0.2], [100.8, 0.2], [100.8, 0.8], [100.2, 0.8] ] ] }}
