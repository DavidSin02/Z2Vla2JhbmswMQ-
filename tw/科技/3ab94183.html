零基礎怎樣學好Oracle數據庫

2020-11-14
| 科技

Oracle數據庫作為Java軟件開發中非常重要的一環,一定要非常熟練的掌握。那麼作為零基礎，如何來學習Oracle數據庫呢?下面就隨北京學佳澳小編一起來了解一下。

首先了解下基礎概念

學習Oracle你首先得了解這幾個名詞:數據庫，數據庫管理系統，數據庫系統

數據庫(DataBase DB)：是存放數據的倉庫,所有的數據在計算機存儲設備上保存,而且所有保存的數據會按照一定的格式進行保存。

數據庫管理系統(DataBase Manager System DBMS)：科學的組織和存儲數據,可以高效的獲取和維護數據。

主要功能:數據操作功能,數據庫的事務管理和運行管理,數據定義功能,數據組織,存儲和管理,數據庫的建立和維護功能。

數據庫系統(DataBase System, DBS)：是指在計算機系統中引入數據庫後的系統,一般由數據庫,數據庫管理系統,應用系統,數據庫管理員構成。

Oracle的組成、工作過程與原理又是什麼?

由Oracle數據庫和Oracle實例組成，包括邏輯結構和物理結構。

物理結構:指的是數據庫中的一組組操作系統的文件

分為三類:

數據文件: 用於存儲數據庫數據,如表,索引數據 .dbf文件

控制文件: 記錄數據庫物理結構的二進制文件 .ctl文件

日誌文件: 記錄對數據庫的所有修改信息,用於故障恢復 .arc

邏輯結構:是指數據庫創建之後形成的邏輯概念之間的關係

數據庫- 表空間 - 邏輯對象(模式) - 數據段 - 數據區間 - 數據塊

那Oracle數據庫又有哪些特點呢?

Oracle數據庫作為一種對象關係型的數據庫管理系統,具有以下幾個特點:

1.多用戶/海量數據的事務處理

2.數據安全性和數據完整性 性能優越

3.分佈式數據處理

4.可移植性(跨OS)

接下來是實例了!

[Oracle數據庫基礎用戶]

sys - manager - sysdba

打開數據庫服務器/ 關閉數據庫服務器 / 備份數據庫 / 恢復數據庫 / 日誌歸檔 / 會話限制 / 管理表/視圖

system - change_on_install - sysdba / sysoper

打開數據庫服務器/ 關閉數據庫服務器 / 備份數據庫 / 恢復數據庫 / 日誌歸檔 / 會話限制

普通用戶(scott)

默認的普通權限用戶/ EMP / DEPT

連接身份:

normal:普通身份

sysoper:操作員身份，可以打開，關閉，備份，恢復數據庫等權限

sysdba:管理員身份

[表、表空間、用戶基礎操作語法]

-- Oracle 創建表空間/用戶/授予權限

-- 創建臨時表空間/表空間語法:

create[temporary] tablespace TABLESPACE_NAME

[tempfile | datafile] 'FILE_PATH'

size 10m

autoextend on;

-- 刪除表空間語法:

drop tablespace TABLESPACE_NAME

including contents and datafiles;

-- 創建用戶和密碼並指定表空間語法:

create user USERNAME identified by PASSWORD

default tablespace TABLESPACE_NAME

temporary tablespace TEMP_TABLESPACE_NAME;

-- 刪除用戶語法:

drop user USERNAME cascade;

cascade:刪除這個用戶以及這個用戶下的所有對象了

-- 給用戶授予權限語法:

grant AUTH_NAME to USERNAME;

-- 撤銷權限語法:

revoke AUTH_NAME from USERNAME;

-- 修改用戶密碼語法:

ALTER USER USERNAME IDENTIFIED BY PASSWORD;

-- 鎖定用戶語法:

alter user USERNAME account lock;

-- 解鎖用戶語法:

alter user USERNAME account unlock;

-- 創建表/數據

create table 表名 (

字段名字段類型(容量),

字段名字段類型(容量)

);
