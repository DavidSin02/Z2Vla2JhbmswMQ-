MySQL數據庫字符串類型常用操作

2020-10-29
| 科技

詳細講解MySQL數據庫中的字符串類型，熟練掌握字符串數據的相關操作，能夠在開發過程中靈活處理各種問題

MySQL數據庫字符串類型有很多，常用的包含char、varchar

CHAR數據類型的長度固定為創建表時聲明的長度，插入的數據小於聲明長度自動使用空格補充，插入數據效率高，耗費空間。

VARCHAR類型是變長字符串，根據值的長度而定，插入數據較CHAR類型低，節省空間

創建數據表並插入數據

create table person(

id int(10) NOT NULL AUTO_INCREMENT,

name varchar(20),

address char(128),

card_id varchar(20),

PRIMARY KEY (id)

);

insert into person (name,address,card_id) values('張三丰','遼東省懿州城','123456789');

insert into person (name,address,card_id) values('黃蓉','浙江省桃花島','123412341234');

insert into person (name,address,card_id) values('孫悟空','花果山','5432154321');

1.字符串替換操作，將address字段中的'懿州城'替換為'阜新市'

SELECT id,REPLACE(address,'懿州城','阜新市') FROM person WHERE name= '張三丰';

指定替換字段的位置和長度，設置替換長度為0，則意味著在指定位置插入

select id,name,INSERT(address,4,0,'舟山市') from person where name= '黃蓉';

2.從指定起止位置截取字符串

select id,name,SUBSTRING(card_id,4,9) from person where name= '張三丰';

3.從指定字符串開始截取定長字符串

select id,name,MID(card_id,'4',3) from person where name= '張三丰';

4.截取字段左面/右邊指定長度的字符串

select id,name,left(card_id,3), RIGHT(card_id,3)from person where name= '張三丰';

5.去除字段的左空格/右空格/左右空格

select LTRIM(name),RTRIM(name),TRIM(name) from person where name= '張三丰';

6.獲取字符串長度

select id,name,length(name) from person where name= '張三丰';

7.拼接字符串concat('str','str1',...)

select id,name,CONCAT(name,address) from person where name= '張三丰';
