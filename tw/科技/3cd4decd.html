模擬測試數據的生成方法

應用系統或軟件產品一般都需要進行不同階段的驗證工作，包括原型功能論證、功能測試、性能測試等，這些測試、論證場景都可能涉及到測試數據的準備。測試數據有時可以直接複用歷史數據，但很多情況下，基於歷史數據建立的測試數據可能會出現內容缺失不全、數據量級不夠、數據涉密不能導出、數據已加密無法參與計算等情況，這時就需要根據用戶的業務需求、數據預置約束條件、數據間層級關聯等條件，生成對應的模擬驗證數據。一般來說，按照用戶要求模擬數據應該做到：數據量可控、充分的隨機性、保持數據間特定的邏輯關聯關係（包括直接關聯或隱含關聯）。

單數據表是最常見的數據模擬情況，這種表由主鍵和普通字段組成，也可以由無主鍵的普通字段組成。在單數據表的基礎上，可以進一步進行多數據表的關聯模擬，關聯關係可以歸類分為：外鍵關聯、同維關聯、主子關聯。對於關聯數據生成的原則具體可參見《怎樣生成有關聯的測試數據》一文。對於已經生成的數據，則可以參考《優化 Join運算的系列方法》對模擬生成的數據進行更進一步地深入查詢和分析處理。

一般來說，單數據表模擬數據的難點是對數據表字段內容的靈活生成。而多數據表模擬則是在此基礎上考慮關聯關係（一層或多層）的生成，尤其是需要保證多數據表在進行關聯過濾之後，還能夠有充足、有效的結果數據，從而滿足數據的關聯運算或是其它展現需求。實際情況中，通過編程生成這些模擬驗證數據的難點通常是數據間複雜的邏輯關聯關係。

本文中，我們將介紹一個方便、靈活的模擬數據生成工具——集算器。集算器是一款面向應用程序員和數據分析員，專注於結構化數據分析與處理的快速開發工具，是一套基於 Java 解釋執行的動態語言，採用了先進的計算模型和設計思想，讓開發更易於實現、性能更好。同時，集算器還具備完備的類庫和輕量級架構，足以讓數據模擬生成更加靈活和高效。

一、 引言

集算器具有跨平臺、無框架、易部署的特點，僅需要安裝有JAVA虛擬機的操作系統即可，特別是對於數據模擬生成的過程，可以隨時分步調試查看中間過程數據。

集算器使用SPL程序語言。SPL（Structured Process Language）是一種面向（半）結構化數據計算的程序設計語言，能夠滿足複雜處理和過程計算的數據處理需求。

同時，集算器還可以直接將模擬生成的數據落地在本地磁盤的二進制“集文件”（集算器自定義的一種文件格式）中，免去繁瑣的安裝數據庫的操作，這也符合模擬測試相對臨時性的工作特點。集文件本身具有使用簡單、可追加、支持大數據、可分段並行等特點，而且很容易轉成其它格式，如數據庫、文本等。當然，集算器自身也支持直接或通過集文件將模擬數據導入到任意目標數據庫中，所需的具體外部庫的使用可以參見《外部庫》函數參考章節。

為了方便敘述和驗證，本文默認模擬生成的數據均落地在本地集文件中。

1、 為什麼使用集文件？

在用集算器生成模擬數據時，常用兩種文件格式：文本文件和集文件。

文本是各種數據平臺 / 數據庫都支持的文件格式，具有良好的通用性。但文本文件的查詢性能較差，佔用磁盤空間也較大，而且缺少字段的數據類型定義，有時可能會出現“類型歧義”的錯誤。

針對這些問題，集算器設計了一種二進制格式文件，稱為集文件（文件後綴 btx）。集文件中使用了低 CPU 消耗的壓縮編碼，數據存儲時較文本文件佔用磁盤空間更小，具有較高的查詢性能，並且字段的數據類型也被存儲，避免出現類型歧義。同時，集文件繼承了文本文件支持大數據量、可追加和易於分段並行的特點。因此，在需要使用數據文件時，集文件是更好的選擇。

2、 如何使用集文件？

利用集文件存儲模擬生成的數據，因為其具有與其它數據格式廣泛的互通性，後續就可以靈活進行與目標數據源的雙向轉化，包括 Oracle、DB2、MS SQL、MySQL、PG 等關係型數據庫和 TXT/CSV、JSON/XML、EXCEL 等文件類型。

3) 字段拆分與組合

某些字段的構成需要遵循一定的編碼規則，例如身份證號碼，我們通過公民身份證號碼可以很直接的獲取一個人所在的地區、出生日期、性別，並計算出年齡。這裡就以此為例對字段的拆分和組合做詳細的說明，其它類似的情況可以以此為參考。：

公民身份證號碼是特徵組合碼，由十七位數字本體碼和一位校驗碼組成。排列順序從左至右依次為：六位數字地址碼，八位數字出生日期碼，三位數字順序碼和一位數字校驗碼。具體數據特徵和約束如下：

l 地址碼：編碼對象常住戶口所在縣級（市、旗、區）行政區劃代碼，按 GB/T2260 標準執行。

l 出生日期碼：編碼對象出生的年、月、日，按 GB/T7408 標準執行，年、月、日之間不用分隔符。

l 順序碼：在同一地址碼所標識的區域範圍內，對同年、同月、同日出生的人編定的順序號，其中奇數分配給男性，偶數分配給女性。

l 校驗碼：根據前面十七位數字碼，按照 ISO 7064:1983.MOD 11-2 校驗碼計算出來的檢驗碼。
