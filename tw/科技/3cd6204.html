ESP32 / ESP8266 MicroPython教程：自動連接WiFi

2020-10-29
| 科技

本MicroPython教程旨在闡釋如何在MicroPython上自動連接WiFi網絡，且無需在提示符中插入所有獨立命令。本教程在ESP32（ http://www.dfrobot.com.cn/goods-1359.html ）均進行了測試。

簡介

本MicroPython教程旨在闡釋如何在MicroPython上自動連接WiFi網絡，且無需在提示符中插入所有獨立命令。此程序在ESP32 和ESP8266上均進行了測試。打印結果出自ESP32上的測試。請注意，ESP8266上自動打印的消息與ESP32上的消息不同，因此其結果會與本教程所示的屏幕截圖不同。但兩者採用的方法完全相同。

我們將檢驗兩種方法，一種是從模塊導入函數並在需要連接WiFi網絡時執行該函數，另一種是全自動式，即在啟動後自動將開發板連接到WiFi網絡。

請注意，這兩種解決方案都需要將文件上傳到MicroPython的文件系統。你可以參閱這篇此前的教程以獲取相關詳細說明。此外，有關如何手動連接WiFi網絡的說明，請查看此教程。所有步驟都至關重要，因為我們基本上只是將它們封裝在模塊的函數中。

模塊調用型連接

在第一部分中，我們將闡釋如何在調用模塊中定義的函數時自動連接WiFi網絡。如果我們不想每次使用ESP32 / ESP8266時都連接WiFi網絡，那麼這個方法會非常有用。我們可以通過它決定何時連接WiFi。

要採用此方法，我們將定義一個簡單的Python函數來執行連接程序。我們將此函數稱為connect函數。

def connect():#Python code goes here

然後，導入網絡模塊，這是獲取WiFi網絡連接功能所必須的步驟。為方便起見，我們還要將我們的網絡憑據（ssid和密碼）存儲在兩個變量上。

import networkssid = "yourNetworkName"password = "yourNetworkPassword"

接下來，獲取站點WiFi接口的實例並將其存儲在變量上。然後，檢查我們是否已連接到WiFi網絡。如已連接，則打印警告，完成執行。

station = network.WLAN(network.STA_IF)if station.isconnected() == True:print("Already connected")return

如未連接，則使用存儲在前文所述的變量中的憑證來激活網絡接口並執行實際連接。

station.active(True)station.connect(ssid, password)

由於連接可能需要一些時間，所以我們將通過檢查is connected方法的輸出結果來進行主動等待，直到成功連接為止。請注意，之所以需要pass語句是因為Python語法不起任何作用。

還要考慮到，我們為了簡化操作將無限期地等待連接，舉個例子，如果WiFi憑證出錯，那麼模塊將無限期掛起，處於嘗試連接的狀態。當然，對於更穩健的實際應用場景，我們需要執行某種超時機制。

最後，我們將打印成功消息和WiFi配置

最後，要上傳代碼，只需打開命令行，導航到存儲文件的目錄，然後執行以下命令，將COM5改為你的設備所在的串行端口。

ampy --port COM5 put ConnectWiFi.py

現在，用你選擇的軟件連接Python提示符。本教程所用軟件為Putty。要驗證新文件已正確上載，請執行以下命令：

import osos.listdir()

如圖1所示，該文件應被列出。

圖1 - 成功上傳WiFi連接模塊。

現在，我們只需導入模塊並調用connect函數，如下所示。

import ConnectWiFiConnectWiFi.connect()

結果如圖2所示。請注意，最後會顯示我們的成功消息，表明我們現在已建立連接。

為了確認我們的安全保障運行良好，你可以嘗試再次調用connect函數。它此時應該會在代碼中返回一條消息，表明我們已建立連接。如圖3所示。

圖3 – 已連接WiFi網絡後調用函數時的警告消息。

自動連接

執行前，我們首先需要分析與某些啟動腳本相關的MicroPython特性。

正如之前的一些教程所述，安裝MicroPython時會在文件系統中生成一個boot.py文件。這個特定文件會在開發板供電時運行[1]。它已經包含一些低級代碼，這些代碼應予以保留。

如此處的MicroPython文檔所示，我們可以在此文件放置我們用於連接WiFi網絡的代碼，這樣電路板啟動時會執行該代碼。

不過，我們採用的是另一個方法。除了boot.py外，如果文件系統中有一個名為main.py的文件，那麼該文件會在boot.py腳本[1]完成後運行。因此，我們將創建這個文件並在啟動時通過它自動連接WiFi網絡。請注意，此文件可以為我們的應用執行其他類型的代碼。此外，由於它不是強制性文件，所以我們可以放心使用。

現在，為執行自動連接，我們將重新使用以前的模塊。因此，先創建一個名為main.py的文件。如前所述，這次需要將其命名為main.py（與之前定義的模塊的名稱不同），否則它將不會自動執行。

此處，只需導入ConnectWiFi模塊並調用connect函數，就像我們之前手動操作的那樣。

import ConnectWiFiConnectWiFi.connect()

現在，使用以下命令將文件加載到文件系統（記得將COM5改為你的設備端口）。如果你仍在通過Putty或其他軟件連接，則需要先關閉該連接，否則無法加載文件。

ampy --port COM5 put main.py

現在重新連接Python提示符。你可以使用我們之前執行的os.listdir()調用確認文件已成功加載。

請注意，由於MicroPython已經在運行，因此不會出現任何改變。要檢查我們的main.py是否按指示將開發板連接到WiFi網絡，只需在打開提示符的情況下重置ESP32 / ESP8266。它此時應該會重新啟動並執行我們的函數，如圖4所示。

圖4 – 啟動後自動連接。

我們可以通過導入ConnectWiFi模塊並嘗試調用connect函數來確認我們已建立連接。此處應該會返回“已連接”的警告，如圖5所示。



圖5 - 嘗試再次連接到WiFi網絡時出現警告。
