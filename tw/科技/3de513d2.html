GitHub怎麼了？連續3天出現嚴重宕機情況 微軟未迴應

2020-11-14
| 科技

最近幾周，GitHub連續多次出現了宕機事件，不過微軟官方始終沒有透露原因。特別在過去的三天時間內，有開發者反饋GitHub中斷的情況非常嚴重，甚至於基本的編程任務都無法執行。

正如外媒ZDNet在一篇文章中所指出的那樣，Mary Jo Foley詳細介紹了過去幾天GitHub服務受到的影響：

4月21日，由於某些問題導致部分GitHub服務中止服務，持續了一個半小時。 4月22日，GitHub又出現兩次宕機情況，影響了至少2個小時的時間。而今天，也就是4月23日，GitHub多項服務出現了各種問題，開始時間大約是美國東部時間上午9點20分，持續時間一個半小時。

今天的宕機對Git Operations; API Requests; Webhooks; Issues, pull requests and projects; GitHub Actions; GitHub Packages; GitHub Pages等產品產生影響。在GitHub的狀態網站上，只有Git Operations被列為“ incident”，其他服務被標註為“degraded”。"

在Twitter上，GitHub Education承認他們的服務出現了中斷，尤其是GitHub Classroom和education.github.com。
