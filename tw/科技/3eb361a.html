Python小技巧：3個處理文件路徑的簡單方法

2020-10-29
| 科技

原標題 | Python 3 Quick Tip: The easy way to deal with file paths on Windows, Mac and Linux

作者 | Adam Geitgey

譯者 | 林肯二百一十三（西安交通大學）、Dooria

幾乎所有計算機在文件夾名稱間使用的都是正斜槓，但微軟Windows使用的是卻反斜槓， 這不免造成了編程的一個小煩惱。

這是1980年代早期，計算機史上的一個意外。MS-DOS的第一個版本使用正斜槓來指定命令行選項。當微軟在MS-DOS2.0中增加對文件夾的支持時，因為正斜槓已經被使用了，所以使用反斜槓代替。三十五年後，這種不兼容仍舊困擾著我們。

如果希望你的Python代碼能夠同時在Windows和Mac/Linux上運行，你需要解決這類平臺的特定問題。幸運的是，Python3的新模塊pathlib可以使文件操作變得更簡單。

以下是一些使用pathlib處理文件名的方法，它可以使命名操作變得簡單得多。

從技術上講，這個代碼在Windows上也能運行，因為Python會在調用open函數時識別出任何一種斜槓。即便如此，你也不應該依賴它。不是所有的python庫都可以在錯誤的操作系統上使用錯誤的斜槓，特別是當它們有外部程序或庫接口時。

並且，Python對混合斜槓類型的支持僅限Windows，它無法反向工作。也就是說，代碼中的反斜槓將在Mac上完全失效。

出於上述原因以及其他更多的原因，使用硬編碼的路徑字符串編寫代碼會被其他程序員diss的，應該儘量避免。

因為os.path模塊的大部分函數都像os.path.join這樣笨拙，Python開發者們經常就懶得用這些工具，結果寫了一堆跨平臺bug，還惹惱了一群用戶。

新方法：Python 3 的 pathlib！

Python 3.4 引入了一個用於處理文件和路徑的新標準庫——pathlib，而且很好用！
