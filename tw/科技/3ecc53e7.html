AI-Code代碼生成器

2020-11-14
| 科技

代碼智能化生成框架，平臺化管理生產模板，模板根據需要可以上傳，並在生成時自己選擇，項目提供webUI方式，根據框架模塊化管理，需要那些選擇後自動生成；提供模板開發接口自由定義自己的模板，最終完成AI+模板=70%的代碼量，平臺默認攜帶Spring為核心的各種框架整合細節請看readme介紹。根據需要可以選擇性生成，支持docker 啟動 。

模板分享地址: https://gitee.com/helixin/aicode_template sql腳本中板焊了模板，會在你構建項目時自動拉取到本地，構建代碼完畢後將會刪除，你可以開源你的模板到這個連接中，也可以自己建立私庫自己用都可以，目的就是讓重複的 代碼寫一次後用模板技術和抽象思維服務更多的項目

以上框架為基本常用的框架，目前只需要準備好你的sql腳本在此項目頁上進行創建即可，然後aicode 會自動幫你把數據庫創建出來用於反向生成代碼使用， 自動從你的模板倉庫中下載框架，當然也可以是用我們已經開發好的，代碼會從dao->service->ctrl->frontend 一條龍生成，接口自動根據規則對接完 畢，也就是說你可以生成後直接運行你的項目代碼，另外項目代碼將會根據你的需要配置上git svn 地址後自動提交到對應的倉庫中省去手工拷貝的麻煩，只 需要在開發工具中 git clone git://xxxxx.git即可，如果模板比較優秀可以節省代碼50%左右無需再開發，比如包含了：增，刪，查，改，分頁，條 件查詢，關聯查詢，關聯分頁查詢等等，你需要關心的只是你的邏輯關係是否設置完整，模型關係是否設置完整用於生成時系統進行自動化，我們提供了web-ui管理界面，只需要登錄賬戶即可 完成對代碼的生成管理，你可以通過頁面來完成模型關係的設置，以及對顯示屬性的設置，快速完成從數據庫層到頁面的定製化開發。

特點

ai-code 與其他開源的完整項目不同點在於技術整合框架的抽象，mvc的模型抽象，我們有這樣的場景，有新的業務需求，用同樣的技術框架，通常做法可能就是找一個項目然後刪刪改改提取 出來一個較為乾淨的框架，然後把新的業務模型，實體類，接口等等重新聲明，創建在此基礎上再做很多CRUD的編碼，有時候我們很反感重複這些，但是又不得不做，通常做這些毫無意義又無聊 同意請舉手，於是乎有沒有辦法做一個工具出來讓新的業務模型設計好後拿到sql腳本 通過工具可以選擇各種不同的技術框架組合，就像spring boot一樣整合技術在一起，但是又 可以自動把mybatis mapper,dao/dto,service,ctrl,ui等等通用功能自動開發好呢？把常用的工具，加密算法等等通用的自動集成進去呢？比如swagger,gradle,maven等等 答案是肯定的ai-code就是這個思路，整合有給你定製的可能，業務隨意變化，只要拿到sql腳本一切就是動動手指，點擊生成，再也不用搞來搞去那些重複的不能再重複的無聊工作了。 這就是ai-code的基本思路和思想；

再來看下通常的開源思路是什麼呢？ 完整項目，比如RBAC模型下的管理系統，有權限，區域，部門，日誌等等，這些無可厚非確實通用，但畢竟如果要和自己的項目融合是要調整的，而且編碼習慣不同， 設計規則可能不滿意，維護過慢等等問題，但是也是可以使用的，於是乎就有了各種 商城開源，權限管理開源，區域管理開源，可以關注學習沒問題，非常棒，甚至直接用。

兩種思路從不同的角度來看待重用問題，ai-code 只希望拋磚引玉，讓抽象思維，整合思維，服務未來思維體現出來，期待更好地項目出現，基於ai-code,期待大家貢獻 技術模板，讓那個更多人從中受益，感謝大家的貢獻。

使用方法

連接數據庫：

-首先要做的就是建立ai_code的數據庫(推薦數據庫格式：編碼為utf8mb4 排序為 utf8mb4_general_ci)，建議使用mysql,腳本請用最新的版本日期，你可以在項目根目錄下sql/aicodexxxxx.sql中找到合適的 sql腳本,比如創建數據庫名為ai_code,數據庫的ip為192.168.1.220,賬戶密碼為username=root,password=xxxxx;

-找到項目下 AI/src/main/resources/jdbc.properties 文件將數據庫鏈接修改成剛才初始化的數據庫ip

jdbc.url=jdbc:mysql://192.168.1.220:3306/ai_code?useUnicode=true&characterEncoding=utf-8&zeroDateTimeBehavior=round&allowMultiQueries=truejdbc.username=rootjdbc.password=xxxxx

修改完畢後可以直接啟動,啟動默認為8080端口 瀏覽器中打開輸入：http://127.0.0.1:8080/index.html 默認賬戶:admin 888888嘗試登錄，你可以在數據庫中修改次賬戶數據表明為account

docker 啟動方式(推薦)

docker 依賴dockerhub 鏡像（意思是你的網絡要支持訪問哦，大家明白哈，搞不定的加群），大家可以通過docker的常規命令pullrun 即可，注意：docker鏡像 中不包含數據庫，數據庫腳本請從項目中下載進行初始化，數據庫名稱請使用ai_code命名，以下docker命令操作 dockerhub 詳細操作說明 https://hub.docker.com/r/hegaoye/aicode

直接使用下面的命令即可搞定，數據庫與程序自動link在一起，方便簡單

#檢出數據庫docker pull hegaoye/mysql:1.0-beta#啟動數據庫 設置 --hostname=aicode-db 用於link方式連接docker run -p 3306:3306 -e MYSQL_ROOT_PASSWORD=aicode --hostname=aicode-db --name aicode-db --restart always -d hegaoye/mysql:1.0-beta --lower_case_table_names=1#搜索查看aicode的鏡像是否存在docker search aicode#拉取aicode的鏡像 hegaoye/aicode:1.0-betadocker pull hegaoye/aicode:1.0-beta#啟動容器,注意 host,username,passowrd 要進行修改成自己的mysql主機 使用link 連接aicode-dbdocker run --link aicode-db:aicode-db -e host=aicode-db:3306 -e username=root -e password=aicode -p 8080:8080 --name aicode --restart always -d hegaoye/aicode:1.0-beta#查看日誌docker logs --tail 1000 -f aicode#重啟，關閉，啟動容器命令docker restart|stop|start aicode

注意

關於自己的sql腳本比如如下

/*==============================================================*//* Table: goods *//*==============================================================*/create table goods( id bigint not null auto_increment, code varchar(64) not null comment '編碼', name varchar(32) comment '商品名', stock bigint comment '庫存', status varchar(32) comment '狀態', primary key (id, code));alter table goods comment '產品';/*==============================================================*//* Table: "order" *//*==============================================================*/create table "order"( id bigint not null auto_increment, code varchar(64) not null comment '編碼', order_no varchar(32) not null comment '訂單號',  primary key (id, code, order_no));alter table "order" comment '訂單';

這個sql在執行中失敗，失敗的原因在於 "order" 雙引號不合法將無法執行，所以請檢查後修改為order 的合法sql語句，最好在生成前執行驗證sql 腳本的正確性，比如下面的

/*==============================================================*//* Table: order *//*==============================================================*/create table `order`( id bigint not null auto_increment, code varchar(64) not null comment '編碼', order_no varchar(32) not null comment '訂單號',  primary key (id, code, order_no));alter table `order` comment '訂單';

去處後即可合法使用，一定要保障sql可以正常執行再用來跑代碼，否則會失敗
