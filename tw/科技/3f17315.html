思科訪問控制列表(ACL)設置，控制網絡訪問

2020-10-29
| 科技

一、訪問控制列表的作用：

過濾：通過過濾經過路由器的數據包來管理IP流量

分類：標識流量以進行特殊處理

ACL在作用的時候有兩個動作，即允許和拒絕。

允許或拒絕經過路由器的數據包

允許或拒絕來自路由器或到路由器的vty訪問

如果沒有ACL，所有數據包會發往網絡的所有部分。

配置完以後一定要記得調用ACL的訪問控制列表。

二、訪問列表的分類：

*標準 檢查源地址.通常允許、拒絕的是完整的協議

*擴展 檢查源地址和目的地址、通常允許、拒絕的是某個特定的協議

*進方向和出方向

出方向 先查路由表，如果有去往目的網段的路由，就轉到出接口，如果出接口調用了訪問控制列表ACL，那麼就看是否和ACL的訪問控制列表匹配，是允許permit還是拒絕deny，如果是允許，就根據路由錶轉發數據，如果是拒絕就直接將數據包丟棄了。

入方向 當入接口收到數據包的時候，先跟入接口的ACL訪問控制列表進行匹配，如果允許則查看路由表從而進行轉發，如果拒絕則直接丟棄。

而在這個deny的機制中，有一條隱藏的deny所有報文的語句。也就是說，當接收到的數據包和前面所有的訪問控制列表ACL都不匹配的時候，則就會觸發這條隱藏的命令。

三、如何來標識ACL

標準編號IPV4列表（1-99）可測試源地址的所有IP數據包的條件。擴展範圍是（1300-1999）

擴展編號IPV4列表（100-199）可測試源地址和目的地址、特定TCP/IP協議和目的端口條件。擴展範圍（2000-2699）

命名ACL用字母數字字符串（名稱）標識IP標準ACL和擴展ACL。

四、配置標準訪問控制列表的命令：

標準訪問控制列表可以用來匹配：

1用來匹配源地址和目標地址

2用來匹配遠程登錄

標準訪問控制列表建議配在接近於目標的位置，因為如果配置在接近於源的目標的位置，那麼發出來的報文就會立刻被deny掉了，而且數據包不僅發送不到目標位置，甚至連接近於源的目標位置以外的其他的路由器也發送不到了，都不可以訪問了。所以標準訪問控制列表一定要放在接近於目標的位置。


實驗一：


RouterX(config)#access-listaccess-list-number {permit|deny} sourse [mark]

Permit：允許。Deny：拒絕。Sourse：源地址。Mark：反掩碼。

R3(config)#access-list 1 deny 192.168.1.0 0.0.0.255 意思就是只要是192.168.1.0網段的（1-254）的IP地址發過來的數據包都會被拒絕。

接著要寫一個permit語句去保證允許其他網段的數據報文可以通過。

即：R3(config)#access-list 1 permit 0.0.0.0 255.255.255.255

當然 0.0.0.0 255.255.255.255 代表所有網段也可以用 any 來表示。

即：R3(config)#access-list 1 permit any 可以這樣來寫，也是可以生效的。

隱藏的deny所有的命令：R3(config)#access-list 1 deny any

所以要記得相反的permit允許其他所有的命令：R3(config)#access-list1 deny any

1.然後在接口下來掉用這個訪問控制列表ACL"

RouterX(config-if)#ipaccess-group access-list-number {in|out} 標識是出方向還是入方向

如果用192.168.1.2來ping192.168.2.2則可以是R3的F0/1的入方向或者F0/0的出方向。

R3(config)#interfacef0/1

R3(config-if)#ipaccess-gourp 1 in 同樣建議配置在in方向，因為可以少一次查詢的動作。

到這裡訪問控制列表ACL就調用成功了，這個時候在去ping就會告訴你目的主機不可達。

即：Destination host unreachable。

2.可以通過show access-lists來產看到當前所被deny掉的數據狀態。

3.刪除訪問控制列表：

R3(config)#no access-list 1 它會將之前配置的deny的語句和permit any的語句在訪問控制列表1裡面同時刪除。

4.如果是隻是deny某一個IP地址的數據報文，可以使用完全匹配的方式：

即：R3(config)#access-list 1 deny 192.168.1.2 0.0.0.0 這樣後面的四個零就是完全匹配的意思。當然這個四個零0.0.0.0 也可以用host 來表示：

即：R3(config)#access-list1 deny host 192.168.1.2

而為了能讓其他的數據能夠被轉發，所以必須要加上permit any的命令，以便讓其他的報文可以通過。


即：R3(config)#access-list 1 deny host 192.168.1.2

R3(config)#access-list 1 permit any

此外，因為在輸入訪問控制列表的時候，是按照輸入的順序來進行排列的，所有如果將permit any或者deny any語句放在具體的permit或者deny語句之前，即：

先配置了R3(config)#access-list1 permit any

再配置了R3(config)#access-list1 deny host 192.168.1.2

就會發現，後面配置的"R3(config)#access-list1 deny host 192.168.1.2"完全沒有生效，報文依舊可以轉發。

所以在配置的時候一定要注意先後的順序，嚴謹的放在後面。

實驗二：

如果在R3上配置遠程管理：

R3(config)#line vty 04

R3(config-line)#passwordcisco

R3(config-line)#login

R3(config-line)#exit

R3(config)#enable password cisco

然後不允許192.168.1.2來遠程訪問R3，則:

R3(config)#access-list 1 deny host 192.168.1.2

R3(config)#access-list 1 permit any

但是調用的時候則是在line下面來調用的：

R3(config)#line vty 04

R3(config-line)#access-class 1 in

如果不在line調用的話，也可以在接口下面調用：

R3(config)#interfacef0/1

R3(config-if)#ipaccess-group 1 in

五、擴展訪問控制列表：

擴展訪問控制列表可以用來匹配：

不僅可以匹配源地址還可以匹配協議

還可以匹配目標地址

還可以匹配端口號

擴展訪問控制列表建議配置在接近於源的位置，如果配置在接近於目標位置或其他位置的路由器上面，因為產生的結構都是一樣的，所以會浪費大量的帶寬，而如果配置在接近於源的的目標位置就不會出現這種情況，所以，擴展訪問控制列表一定要配置在接近於源的目標位置。


實驗一：

命令：

建立擴展訪問控制列表：

RouterX(config)#access-listaccess-list-number {permit|deny} protocol source source-wildcard [operator port] Destination destination-wildcard [operator port] [established] [log]

Permit：允許。Deny：拒絕。Protocol：協議。Source: 源地址。source-wildcard：源地址反掩碼 。operatorport：相應端口。Destination：目標地址。destination-wildcard：目標地址反掩碼。

調用擴展訪問控制列表：

RouterX(config-if)#ipaccess-group access-list-number {in|out}

"ping"命令是基於ICMP協議的命令。且下面的命令是deny掉了去的報文和回的報文。

即：

R3(config)#access-list 100 deny icmp192.168.1.0 0.0.0.255 192.168.2.0 0.0.0.255 echo 報文格式

R3(config)#access-list 100 deny icmp 192.168.1.00.0.0.255 192.168.2.0 0.0.0.255 echo-reply 迴應報文格式

R3(config)#access-list 100 permit ip any any 因為三層所用的協議是IP協議，所以是ip anyany，這裡的IP表示的是協議號。

接下來調用，在R1的入方向來調用這個ACL。

R1(config)#interfacef0/1

R2(config-if)#ipaccess-group 100 in

實驗二：

要求：不允許192.168.1.2去telnet遠程連接R3路由器。而telnet的協議是基於tcp協議的。

R1(config)#access-list100 deny tcp host 192.168.1.2 host 23.1.1.3 eq telnet 這裡的eq指的是"等於"

R1(config)#access-list100 permit ip any any

然後去接口調用：

R1(config)#ipaccess-group 100 in


常見的協議端口號：

http 80

dhcp server67 client68

tftp 69

DNS tcp的53號端口和udp的53號端口，而我們平時上網的時候都是用的UDP的53號端口。只有在DNS服務器之間同步DNS服務的時候，才會用到基於TCP的53號端口。或者發的報文比較大的時候也才會用到DNS的第53號端口。

NTP 是網絡時間協議，使用的是123號端口

Email 一個是SMTP的25號端口，還有一個是POP3的110號端口。

FTP 一個是20號端口，一個是21號端口，通常情況下被動模式是使用的是21號端口來建立連接的，而傳輸數據的是隨機端口，而如果設置的是主動模式的話，則是用21號端口來建立連接，用20號端口來傳輸數據。

ACL配置指導：

-標準或擴展代表可過濾經過路由器的內容

-每隔接口、協議、方向只允許有一個ACL

-ACL語句的順序控制著測試，因此最具體的語句位於列表頂部。

-最後的ACL測試始終是隱式拒絕其他所有語句，因此每隔列表需要至少一條permit語句

-在全局範圍內創建ACL，然後將其應用到入站流量或出站流量的接口

-ACL可過濾經過路由器的流量或往返路由器的流量，具體取決於其應用方式

-將ACL置於網絡中時:

*標準ACL應靠近目的地址

*擴展ACL應靠近源地址。

【總結】

-標準IPv4 ACL允許根據源IP地址進行過濾

-擴展ACL允許根據源IP地址、目的IP地址、協議和端口號進行過濾

-命名ACL允許從一條ACL中刪除單挑語句

-可以使用show access-lists和show ipinterface命令排除ACL配置的常見錯誤。
