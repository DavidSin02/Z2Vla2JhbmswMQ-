模式意圖

如果已經有了一種類，而需要調用的接口卻並不能通過這個類實現。因此，把這個現有的類，經過適配，轉換成支持接口的類。

換句話說，就是把一種現有的接口轉變為另一種可用的接口。

模式結構

【類的適配器】

簡要說明：

Target 目標接口

Adaptee 現有的類

Adapter 中間轉換的類，即實現了目標接口，又繼承了現有的類。

代碼示例：

 1 package com.nd.test1; 2 interface Target{ 3 public void operation1(); 4 public void operation2(); 5 } 6 class Adaptee{ 7 public void operation1(){ 8 System.out.println("operation1"); 9 }10 }11 12 class Adapter extends Adaptee implements Target{13 public void operation2() {14 System.out.println("operation2");15 }16 }17 18 public class test {19 public static void main(String[] args){20 Target tar = new Adapter();21 tar.operation1();22 tar.operation2();23 }24 }

對象的適配器

與上面不同的是，這次並不是直接繼承現有的類，而是把現有的類，作為一個內部的對象，進行調用。

 1 package com.nd.test2; 2  3 interface Target{ 4 public void operation1(); 5 public void operation2(); 6 } 7  8 class Adaptee{ 9 public void operation1(){10 System.out.println("operation1");11 }12 }13 14 class Adapter implements Target{15 private Adaptee adaptee;16 public Adapter(Adaptee adaptee){17 this.adaptee = adaptee;18 }19 public void operation1() {20 adaptee.operation1();21 }22 23 public void operation2() {24 System.out.println("operation2");25 }26 27 }28 public class test {29 public static void main(String[] args){30 Target tar = new Adapter(new Adaptee());31 tar.operation1();32 tar.operation2();33 }34 }

使用場景

1 想使用一個已經存在的類，但是它的接口並不符合要求

2 想創建一個可以複用的類，這個類與其他的類可以協同工作

3 想使用已經存在的子類，但是不可能對每個子類都匹配他們的接口。因此對象適配器可以適配它的父類接口。（這個沒理解，以後慢慢琢磨）
