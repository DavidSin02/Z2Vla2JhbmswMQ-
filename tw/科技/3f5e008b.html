Robotstudio軟件：機床上下料工作站機器人主邏輯編寫與仿真運行

2020-11-14
| 科技

前一期介紹了機器人數控機床上下料例行程序的編寫，但是，也只是編寫了機器人的運動程序，並沒有進行信號邏輯控制程序的編寫。本期，就來為大家介紹一下工作站信號控制邏輯的編寫。

概述

機器人例行程序的控制邏輯主要是在Main主程序中完成的，同時在每一個子例行程序中也要對各自的控制信號進行邏輯控制，比如機器人自動給料裝置抓取工件例行程序中需要控制抓手工具抓取工件、輸送帶放料例行程序中需要控制抓手放下工件等。當然，在主程序中還包括前臺與後臺運行程序的信號邏輯交互。

數控機床後臺運行程序

數控機床後臺運行程序沒有改動，這裡不再過多介紹。

完整程序代碼如下。

MODULE MainModule	PROC main()		WHILE TRUE DO            IF RM_C=1 THEN!循環掃描數控機床加工控制信號是否為1,若為1，則向下執行;若不為1,則繼續掃描                Set VM_D_C;!置位數控機床防護門控制信號為1                WaitUntil VM_D_R=1;!等待數控機床防護門反饋信號為1                Set VM_F_C;!置位數控機床法蘭旋轉控制信號為1                Set VM_FD_C;!置位數控機床進給機構控制信號為1                WaitTime 5;!等待5s,用以模擬數控機床加工時間                Reset VM_FD_C;!復位數控機床進給機構控制信號為0                WaitUntil VM_FD_R=1;!等待數控機床進給機構反饋信號為1                Reset VM_F_C;!復位數控機床法蘭旋轉控制信號為0                Reset VM_D_C;!復位數控機床防護門控制信號為0                WaitUntil VM_D_R=0;!等待數控機床防護門反饋信號為0                Reset RM_C;!復位數控機床加工控制信號為0            ENDIF        ENDWHILE	ENDPROCENDMODULE

自動給料裝置後臺運行程序

自動給料裝置後臺運行程序做了一些修改，增加了自動給料裝置A、B交替運行參考標誌可變量reg10的聲明，並對自動給料裝置A、B交替運行標誌flag4、flag5的賦值運算重新定義，同時修改了信號控制邏輯。

完整程序代碼如下。

MODULE MainModule    PERS num reg10:=0;!聲明整型可變量reg10，初始化值為0，作為自動給料裝置A、B交替運行參考標誌，並與前臺程序進行數據交互    VAR bool flag4:=FALSE;!聲明布爾變量flag4，初始狀態為FALSE    VAR bool flag5:=FALSE;!聲明布爾變量flag5，初始狀態為FALSE	PROC main()		WHILE TRUE DO            flag4:=RWA_C=1 AND RWB_C=0 AND reg10>=0 AND reg10<=2;!對自動給料裝置A工件生成控制信號、自動給料裝置B工件生成控制信號、自動給料裝置A、B交替運行參考標誌做邏輯“與”運算，若RWA_C為1且RWB_C為0且0≤reg10≤2，則flag4為1；否則為0            flag5:=RWA_C=0 AND RWB_C=1 AND reg10>=3 AND reg10<=5;!對自動給料裝置A工件生成控制信號、自動給料裝置B工件生成控制信號、自動給料裝置A、B交替運行參考標誌做邏輯“與”運算，若RWA_C為0且RWB_C為1且3≤reg10≤5，則flag4為1；否則為0            IF flag4=TRUE THEN!循環掃描flag4，若為TRUE則向下執行；若不為TRUE則繼續掃描                Reset VM_FYB_C;!循環掃描flag4，若為TRUE則向下執行；若不為TRUE則繼續掃描                WaitUntil VM_FYB_R=0;!等待給料機構B進給反饋信號為0                WaitUntil RWA_R=0;!等待自動給料裝置A工件生成反饋信號為0                Set VM_FYA_C;!置位給料機構A進給控制信號為1                WaitUntil VM_FYA_R=1;!等待給料機構A進給反饋信號為1                Set VM_FAW_C;!置位自動給料裝置A工件生成控制信號為1                WaitUntil VM_FWA_R=1;!等待自動給料裝置A工件生成反饋信號為1                Reset VM_FAW_C;!復位自動給料裝置A工件生成控制信號為0                Reset RWA_C;!復位自動給料裝置A工件生成控制信號為0            ELSEIF flag5=TRUE THEN!循環掃描flag5，若為TRUE則向下執行；若不為TRUE則繼續掃描                Reset VM_FYA_C;!復位給料機構A進給控制信號為1                WaitUntil VM_FYA_R=0;!等待給料機構A進給反饋信號為0                WaitUntil RWB_R=0;!等待自動給料裝置B工件生成反饋信號為0                Set VM_FYB_C;!置位給料機構B進給控制信號為1                WaitUntil VM_FYB_R=1;!等待給料機構B進給反饋信號為1                Set VM_FBW_C;!置位自動給料裝置B工件生成控制信號為1                WaitUntil VM_FWB_R=1;!等待自動給料裝置B工件生成反饋信號為1                Reset VM_FBW_C;!復位自動給料裝置B工件生成控制信號為0                Reset RWB_C;!復位自動給料裝置B工件生成控制信號為0            ENDIF        ENDWHILE	ENDPROCENDMODULE

初始化例行程序

初始化例行程序是在機器人運行之前對機器人關鍵的信號、變量、可變量恢復初始狀態的程序，這裡對機床上下料邏輯判斷標誌位flag6、自動給料裝置A、B交替運行參考標誌reg10分別進行了初始化賦值。對機床上下料邏輯判斷標誌位flag6賦值為TRUE，是告訴機器人這是第一次運行，使得後續機器人機床上下料程序執行時，可以執行第一次放料邏輯。

完整程序代碼如下。

    PROC Init()        flag6:=TRUE;!初始化機床上下料邏輯判斷標誌位flag6為TRUE        reg10:=0;!初始化自動給料裝置A、B交替運行參考標誌reg10為0    ENDPROC

機器人主程序

編寫機器人主程序就是實現整個機器人子例行程序的調用邏輯關係，也就是把機器人工作站的控制邏輯關係轉換為機器人運行程序的過程。機器人工作站的整體運行邏輯關係如下圖所示。

把上述機器人工作站控制邏輯關係轉換為機器人程序，如下圖所示。

完整程序代碼如下。

	PROC Main()        Home;!調用機器人Home點例行程序        Init;!調用機器人初始化例行程序        WHILE TRUE DO            Label1:!程序跳轉標籤Label1            flag1:=reg10>=0 AND reg10<=2;!對自動給料裝置A、B交替運行參考標誌reg10進行判斷，若0≤reg10≤2，則flag1為1；否則為0            flag2:=reg10>=3 AND reg10<=5;!對自動給料裝置A、B交替運行參考標誌reg10進行判斷，若3≤reg10≤5，則flag1為1；否則為0            flag3:=reg10>=6;!對自動給料裝置A、B交替運行參考標誌reg10進行判斷，若reg10≥6，則flag1為1；否則為0            IF flag1=TRUE THEN!循環掃描flag1，若為TRUE則向下執行；若不為TRUE則繼續掃描                Reset RWB_C;!復位自動給料裝置B工件生成控制信號為0                WaitUntil RWB_R=0;!等待自動給料裝置B工件生成反饋信號為0                Pick_FA;!調用自動給料裝置A取料例行程序                reg10:=reg10+1;!自動給料裝置A、B交替運行參考標誌reg10加1運算            ELSEIF flag2=TRUE THEN!循環掃描flag2，若為TRUE則向下執行；若不為TRUE則繼續掃描                Reset RWA_C;!復位自動給料裝置A工件生成控制信號為0                WaitUntil RWA_R=0;!等待自動給料裝置A工件生成反饋信號為0                Pick_FB;!調用自動給料裝置B取料例行程序                reg10:=reg10+1;!自動給料裝置A、B交替運行參考標誌reg10加1運算            ELSEIF flag3=TRUE THEN!循環掃描flag2，若為TRUE則向下執行；若不為TRUE則繼續掃描                reg10:=0;!自動給料裝置A、B交替運行參考標誌reg10賦值為0                GOTO Label1;!跳轉到標籤Label1            ENDIF            Machine_Pick_Drop;!調用數控機床上下料例行程序            IF VC_Hand_R2=1 THEN!循環掃描機器人工具反饋信號2，若為1則向下執行；若不為1則繼續掃描                Conveyor_Drop;!調用輸送帶放料例行程序            ENDIF        ENDWHILE	ENDPROC

數控機床上下料例行程序

數控機床上下料例行程序主要是對機床上下料邏輯判斷標誌位flag6進行判斷，如果flag6為TRUE，也就是機器人第一次運行，那麼就只執行機器人數控機床上料程序；如果flag6為FALSE，機器人就不是第一次運行，那麼機器人就要先把數控機床上加工完成的工件取下來，然後再把新工件安裝到數控機床上。

注：程序中第117、118行代碼重複復位、置位數控機床工件安裝與拆除控制信號VM_W_C，是經過反覆測試後添加的，如果不添加這兩行代碼，機器人無法向下運行。按照正常運行邏輯，這裡只需要復位數控機床工件安裝與拆除控制信號VM_W_C即可。之所以要反覆置位、復位這個信號，個人猜測可能是因為機器人在機床上安裝、拆除工件時使用了不同的TCP，導致機器人內部運行邏輯的不一致，因此，這裡需要再次置位、復位這個信號，使得邏輯保持一致；另一種可能性是仿真軟件的BUG，這裡不再過多討論。

完整程序代碼如下。

    PROC Machine_Pick_Drop()        IF flag6=TRUE THEN!循環掃描flag6，若為TRUE則向下執行；若不為TRUE則繼續掃描            MoveJ Target_100,v500,fine,TCP2\WObj:=wobj0;            WaitUntil RM_R=1;!等待數控機床加工反饋信號為1            MoveJ Target_110,v500,fine,TCP2\WObj:=wobj0;            MoveL Target_80,v500,fine,TCP1\WObj:=wobj0;            MoveL Target_90,v500,fine,TCP1\WObj:=wobj0;            Set VM_W_C;!置位數控機床工件安裝與拆除控制信號為1            WaitUntil VM_W_R=1;!等待數控機床工件安裝與拆除反饋信號為1            WaitTime 1;!等待1s，信號發送、接收時間            Reset VC_Hand_C1;!復位機器人工具控制信號1為0            WaitUntil VC_Hand_R1=0;!等待機器人工具反饋信號1為0            MoveL Target_80,v500,fine,TCP1\WObj:=wobj0;            MoveJ Target_110,v500,fine,TCP2\WObj:=wobj0;            Set RM_C;!置位數控機床加工控制信號為1            flag6:=FALSE;!機床上下料邏輯判斷標誌位flag6賦值為FALSE            MoveJ Target_100,v500,fine,TCP2\WObj:=wobj0;        ELSEIF flag6=FALSE THEN!循環掃描flag6，若為FALSE則向下執行；若不為FALSE則繼續掃描            MoveJ Target_100,v500,fine,TCP2\WObj:=wobj0;            WaitUntil RM_R=1;!等待數控機床加工反饋信號為1            MoveJ Target_110,v500,fine,TCP2\WObj:=wobj0;            MoveL Target_120,v500,fine,TCP2\WObj:=wobj0;            MoveL Target_130,v500,fine,TCP2\WObj:=wobj0;            Reset VM_W_C;!復位數控機床工件安裝與拆除控制信號為0            Set VM_W_C;!置位數控機床工件安裝與拆除控制信號為1            Reset VM_W_C;!復位數控機床工件安裝與拆除控制信號為0            WaitUntil VM_W_R=0;!等待數控機床工件安裝與拆除反饋信號為0            WaitTime 1;!等待1s，信號發送、接收時間            Set VC_Hand_C2;!置位機器人工具控制信號2為1            WaitUntil VC_Hand_R2=1;!等待機器人工具反饋信號2為1            MoveL Target_120,v500,fine,TCP2\WObj:=wobj0;            MoveL Target_80,v500,fine,TCP1\WObj:=wobj0;            MoveL Target_90,v500,fine,TCP1\WObj:=wobj0;            Set VM_W_C;!置位數控機床工件安裝與拆除控制信號為1            WaitUntil VM_W_R=1;!等待數控機床工件安裝與拆除反饋信號為1            WaitTime 1;!等待1s，信號發送、接收時間            Reset VC_Hand_C1;!復位機器人工具控制信號1為0            WaitUntil VC_Hand_R1=0;!等待機器人工具反饋信號1為0            MoveL Target_80,v500,fine,TCP1\WObj:=wobj0;            MoveJ Target_110,v500,fine,TCP2\WObj:=wobj0;            Set RM_C;!置位數控機床加工控制信號為1            MoveJ Target_100,v500,fine,TCP2\WObj:=wobj0;        ENDIF    ENDPROC

邏輯設定

在軟件的“仿真”菜單欄下的“工作站邏輯”中，配置機器人工作站邏輯信號，也就是把仿真工作站中的所有Smart組件控制信號與機器人的I/O信號按照下圖映射關係進行連接。

仿真運行

在“控制器”菜單欄下，將數控機床後臺運行程序、自動給料裝置後臺運行程序的“Type”修改為“Semistatic”，也就是設定後臺運行程序為自動運行模式。

在“RAPID”菜單欄下，機器人前臺運行程序執行“將程序指針設為所有任務中的主例行程序”，即初始化機器人程序運行指針。

在“仿真”菜單欄下，進行仿真設定，全部勾選仿真工作站中的Smart組件對象與機器人對象，並把機器人進入點設置為“主程序Main”。同時手動置位輸送帶運行反饋信號VC_CNY_R，以便於機器人能夠在第一次運行時執行輸送帶放料任務。

全部設置完成後，點擊“播放”按鈕，機器人工作站開始仿真運行。點擊“停止”按鈕，機器人工作站停止仿真運行。

停止運行後要立即點擊“重置”按鈕，把仿真工作站恢復原始狀態。下一次仿真運行之前，首先要執行機器人控制器的重啟，然後再執行“將程序指針設為所有任務中的主例行程序”，最後才能再次運行機器人工作站的仿真；否則會帶來一些不必要的麻煩，切記！
