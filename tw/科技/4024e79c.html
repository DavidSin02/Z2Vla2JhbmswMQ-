理工男必備技能：用小小的滾輪，丈量你周圍的“天下”

2020-11-14
| 科技

Arduino UNO——小型測距輪

現在打開百度地圖，輸入起點和終點，兩地之間的距離馬上顯示在手機屏幕上。相比以往的地圖，現在的電子地圖，地理信息量更大。距離計算的更加精確。可這麼大的面積，是怎麼測量出來的呢？衛星、捲尺還是其他的。衛星能勘探地形、地面資源。但所測距離的精確度不高；捲尺靈活性不夠，所測的距離不夠長。排除以上兩種還有其他測距工具嗎？前段時間在馬路上看見有人手持一個滾輪沿著馬路邊滾動，覺得很是新奇，上前詢問，原來他是地圖工作者，現在正在繪製路線，並測量兩地之間的距離。原來他所用的就是用測距輪。這種測距工具，方便攜帶，靈活性高，測量精準度高。回來之後自己也製作了一個小型的測距輪。

測距輪相比傳統的捲尺，可以隨意滑動，可測正負（向左滑動為正，向右滑動負）。

下面就來看看做的怎麼樣吧。

【配件圖】

【主要配件】

測距輪的主板我採用的DFRobot最新推出的DFRduino UNO R3的主板，他完全兼容Arduino UNO R3版本。性價比也比較高，相比之下，我更鐘意DFRduino UNOR3。因為它採用沉金工藝，外觀精緻，在追求質量的同時，也提高了顏值 。製作測距尺，還需要一個強大的配件，那就是編碼器，這樣才能計數。

在這裡我採用增量式光電旋轉編碼器，它能有效的兼容Arduino，PLC等各類主控器。它具有AB兩相輸出，可以通過旋轉的光柵盤和光耦產生可識別方向的計數脈衝信號。其電路輸出的類型為NPN集電極開路輸出，這樣的輸出類型可以和內部帶上拉電阻的單片機連接，然後實現轉速、角度、角速度等數據的測量。

注意：編碼器在不接設備的情況下，集電極開路輸出，在沒有上拉電阻的時候，是沒有電壓輸出的，如要要示波器示波請在AB兩相輸出上加上兩個上拉電阻。如果實際運用，就不需要加上拉電阻了。

【效果圖】

Reset：此模式為復位模式，切換到此模式，可以清零所測的數據。這樣又可以重新開始測距取值了。

Detection：此模式為測距模式，切換到此模式，可以開始你的表演了，隨意測距，上下左右隨便滾動，完全無壓力。往左滾測得是數值為正，往右滑數值為負（當測距數值為正得時候，往右滑可以看見數值隨著減小）。

Lock：此模式為鎖定，當你測得一定的距離後，選擇到此模式可以鎖定數據，隨意你怎麼滾動，數據都不會改變了，這樣你就可以記錄數據了。方便吧，按三下按鍵就可以了哦。

為了讓攜帶方便，外觀漂亮。我用3D打印機打印了外殼。方方正正的，是不是看起來有點呆板。沒辦法，我能想到的只是如此了。相信朋友們肯定有更好的創意。3D打印的文件我以附件的形式放在文末了。

程序：

#include <Wire.h>

#include "DFRobot_RGBLCD.h"

DFRobot_RGBLCD lcd(16, 2);

//rgb_lcd lcd;

#define A 3

#define B 4

#define Key 12//key

#define D 79 //Diameter 79mm

float C = 0; //perimeter

unsigned int Distance;

int VA = 0;

int VB = 0;

unsigned long Count = 0;//count

unsigned int Count_1 = 0; //Negative count

unsigned char flag = 1, Mark = 0;

unsigned long lasttime = 0, Modetime = 0;

//Length measurement range is ± 6 M

void setup()

{

Serial.begin(9600);

lcd.init();

lcd.setRGB(255, 255,0);

lcd.setCursor(2, 0 );

lcd.print("M:");

lcd.setCursor(2, 1 );

lcd.print("D:");

lcd.setCursor(12, 1 );

lcd.print("cm");

pinMode(A, INPUT_PULLUP); //Pull-up input

pinMode(B, INPUT_PULLUP);

pinMode(Key, INPUT);

attachInterrupt(1, interrupt, RISING);

C = D * PI;

}

void loop()

{

if (millis() - 150 > lasttime)//Detect keys once every 150ms

{

if (digitalRead(Key) == HIGH)

if (digitalRead(Key) == HIGH)

Mark += 1;

if (Mark > 2)

Mark = 0;

while (digitalRead(Key) == HIGH);

lasttime = millis();

}

if (millis() - 100 > Modetime)//Refresh the data every 100ms

{

if (Mark == 0) //Cleared

{

lcd.setCursor(6, 0 );

lcd.print("Reset ");

Distance = C * Count / 40;

flag = 3;

lcd.setCursor(11, 0 );

lcd.print(" ");

}

if (Mark == 1) //Calculate the measured value

{

lcd.setCursor(6, 0 );

lcd.print("Detection");//

lcd.setCursor(4, 1 );

if (Count > 0 && Count < 0xffff)//Determine whether the length is positive

{

lcd.print('-');//The length is negative

Distance = C * Count / 40;

}

else if (Count == 0 && Count_1 == 0 )//Determine whether the length is zero

{

lcd.setCursor(4, 1 );

lcd.print(' ');

Distance = C * Count / 40;

}

else//Length is positive

{

lcd.print('+');

Distance = Count_1 * C / 40;

}

}

else if (Mark == 2) //lock

{

lcd.setCursor(6, 0 );

lcd.print("Lock ");

Count = 0;

Count_1 = 0;

}

Modetime = millis();

}

lcd.setCursor(5, 1 );//Displays the value of Distance

lcd.print(Distance / 10000);

lcd.print((Distance / 1000) % 10);

lcd.print((Distance / 100) % 10);

lcd.print('.');

lcd.print((Distance / 10) % 10);

lcd.print(Distance % 10);

}

void interrupt()//Interrupt handler

{

VB = digitalRead(B);

if (Mark == 1)//Detects whether the current mode is a measurement

{

if (VB == 1)//To determine whether the positive measurement

{

flag = 1;

if (Count > 0xffff)

{

Count_1 -= 1;

}

Count += 1;

}

else//Reverse measurement

{

flag = 0;

Count -= 1;

if (Count > 0xFFFF)

{

Count_1 += 1;

}

}

//Count is cleared over the range

if (Count < 0xFFFF && Count > 0x294A)

Count = 0;

else if (Count < 0xFFFFD6B5 && Count > 0xFFFF)

Count = 0;

}

}
