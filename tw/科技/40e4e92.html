一、企業網絡安全

企業網絡中的網絡設備進行通信，需要保證數據可靠性和安全性。

現網中通過使用硬件防火牆或WAF、流量清洗設備保證企業安全，同時路由交換層面可以使用

ACL確保網絡的安全。

二、ACL功能

1、網絡控制

2、限制流量

3、提高網絡性能

4、防止網絡攻擊

三、ACL實戰一

1、拓撲以OSPF為協議背景

2、基本OSPF關鍵配置

R1 OSPF關鍵配置：

[R1]display current-configuration configuration ospf

ospf 1

area 0.0.0.0

network 1.1.1.1 0.0.0.0

network 10.1.12.1 0.0.0.0

R2 OSPF關鍵配置：

[R2]display current-configuration configuration ospf

ospf 1

area 0.0.0.0

network 2.2.2.2 0.0.0.0

network 10.1.12.2 0.0.0.0

查看OSPF臨接關係正常：

[R2]display ospf peer brief


R1可以學習到2.2.2.2的ospf路由

[R1]ping 2.2.2.2

PING 2.2.2.2: 56 data bytes, press CTRL_C to break

Reply from 2.2.2.2: bytes=56 Sequence=1 ttl=255 time=30 ms

Reply from 2.2.2.2: bytes=56 Sequence=2 ttl=255 time=40 ms

3、ACL配置

[R1]acl ?

INTEGER<2000-2999> Basic access-list(add to current using rules) //基本ACL源IP

INTEGER<3000-3999> Advanced access-list(add to current using rules)//高級ACL源目IP和端口

INTEGER<4000-4999> Specify a L2 acl group//二層ACL，源目MAC以太網幀類型

配置基本ACL

Basic ACL 2000, 1 rule

Acl's step is 5

rule 5 deny source 2.2.2.2 0

接口應用ACL

[R1-GigabitEthernet0/0/0]traffic-filter inbound acl 2000

此時R1無法ping通R2，數據流匹配上ACL

配置高級ACL：

[R1]display acl 3000

rule 5 deny ospf (5 matches)

[R1-GigabitEthernet0/0/0]traffic-filter inbound acl 3000

此時過濾掉OSPF報文，OSPF鄰居關係建立失敗

通過高級ACL過濾源目IP方式：

acl number 3000

rule 5 deny ip destination 224.0.0.5 0

配置二層ACL過濾方式：

acl number 4000

rule 5 deny destination-mac 0100-5e00-0005

ACL規則從上往下匹配，根據配置的規則對數據流量進行過濾。
