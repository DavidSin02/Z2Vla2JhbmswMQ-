計算機啟動過程、啟動故障診斷及啟動盤製作

2020-10-29
| 科技

寫在前面：

我們的祼機是無法直接使用的，所以說通常所說的計算機啟動是指計算機打開電源並運行到操作系統：

I 開機後，要檢查一下硬件是否滿足運行的基本條件。

II 計算機系統安裝在哪裡？軟盤、光盤、硬盤、U盤、網絡？往往一臺計算機可以安裝多個操作系統，那不同的操作系統存放在不同的分區。所以存在存儲設備和活動分區選擇的問題。

III 操作系統要加載到內存才可以運行。

上述過程的完成，需要一個程序最先運行，這個程序比較小，可以固化在一個小的存儲設備中，稱為BIOS。存儲BIOS的芯片由一塊後備電池供電，因此無論是在關機狀態中，還是遇到系統掉電情況，其信息都不會丟失。

一、計算機啟動流程

計算機啟動是一個比較複雜的過程，這裡主要介紹window啟動流程：

MBR主引導記錄由三個部分組成：

（1） 第001-446字節：調用操作系統的機器碼。

（2） 第447-510字節：分區表（Partition table）。

（3） 第511-512字節：主引導記錄簽名（0x55和0xAA）。

當電腦運行完POST後，傳統型BIOS會根據啟動扇區查找開機硬盤中標記"啟動"分區下的BOOTMGR文件；若是UEFI則是Bootmgr.efi文件，接著管理程序會讀取開機配置數據庫（BCD, Boot Configuration Database）下的啟動數據，接著根據其中的數據加載與默認或用戶所選擇的操作系統。

開機配置數據庫是一個存儲啟動配置且獨立於硬件的數據庫。它替代了由舊版Windows使用的NTLDR之開機配置文件案boot.ini。開機配置數據庫的文件結構是與Windows登錄數據庫完全相同的。

Windows Boot Manager通過調用winload.exe程序（作業系統的開機程序）來加載操作系統的核心（ntoskrnl.exe）和基本系統應用程序的。winload.exe用來替代先前Windows中NTLDR程序。

二、電腦不啟動故障診治全攻略

在諸多電腦故障中，無法正常啟動是最令用戶頭痛的事了。

根據啟動過程和故障現象進行診治：

1 系統完全不能啟動，見不到電源指示燈亮，也聽不到冷卻風扇的聲音。

這時，基本可以認定是電源部分故障，檢查電源線和插座是否有電、主板電源插頭是否連好，UPS是否正常供電，再確認電源是否有故障，最簡單的就是替換法。

2 電源指示燈亮，風扇轉，但沒有明顯的系統動作。

這種情況如果出現在新組裝電腦上，應該首先檢查CPU是否插牢或更換CPU，而正在使用的電腦的CPU損壞的情況比較少見（人為損壞除外），損壞時一般多帶有焦糊味，如果剛剛升級了BIOS或者遭遇了CIH病毒攻擊，這要考慮BIOS損壞問題（BIOS莫名其妙的損壞也是有的），修復BIOS的方法很多雜誌都介紹過就不重複了；

確認CPU和BIOS沒問題後，就要考慮CMOS設置問題，如果CPU主頻設置不正確也會出現這種故障，解決方法就是將CMOS信息清除，既要將CMOS放電，一般主板上都有一個CMOS放電的跳線，如果找不到這個跳線可以將CMOS電池取下來，放電時間不要低於5分鐘，然後將跳線恢復原狀或重新安裝好電池即可；

如果CPU、BIOS和CMOS都沒問題還要考慮電源問題：PC機電源有一個特殊的輸出信號，稱為POWER GOOD（PG）信號，如果PG信號的低電平持續時間不夠或沒有低電平時間，PC機將無法啟動。如果PG信號一直為低電平，則PC機系統始終處於復位狀態。這時PC機也出現黑屏、無聲響等死機現象。但這需要專業的維修工具外加一些維修經驗，因此，建議採用替換法；電源沒有問題就要檢查是否有短路，確保主板表面不和金屬（特別是機箱的安裝固定點）接觸。把主板和電源拿出機箱，放在絕緣體表面，如果能啟動，說明主板有短路現象；如果還是不能啟動則要考慮主板問題，主板故障較為複雜，可以使用替換法確認，然後更換主板。

3 電源指示燈亮，系統能啟動，但系統在初始化時停住了，而且可以聽到喇叭的鳴叫聲（沒有顯示畫面）

根據峰鳴代碼可以判斷出故障的部位。

Award BIOS

1短聲：說明系統正常啟動。表明機器沒有問題。

2短聲：說明CMOS設置錯誤，重新設置不正確選項。

1長1短：說明內存或主板出錯，換一個內存條試試。

1長2短：說明顯示器或顯示卡存在錯誤。檢查顯卡和顯示器插頭等部位是否接觸良好或用替換法確定顯卡和顯示器是否損壞。

1長3短：說明鍵盤控制器錯誤，應檢查主板。

1長9短：說明主板Flash RAM、EPROM錯誤或BIOS損壞，更換Flash RAM。

重複短響：說明主板電源有問題。

不間斷的長聲：說明系統檢測到內存條有問題，重新安裝內存條或更換新內存條重試。

AMI BIOS

1短：說明內存刷新失敗。更換內存條。

2短：說明內存ECC較驗錯誤。在CMOS 中將內存ECC校驗的選項設為Disabled或更換內存。

3短：說明系統基本內存檢查失敗。換內存。

4短：說明系統時鐘出錯。更換芯片或CMOS電池。

5短：說明CPU出現錯誤。檢查CPU是否插好。

6短：說明鍵盤控制器錯誤。 應檢查主板。

7短：說明系統實模式錯誤，不能切換到保護模式。

8短：說明顯示內存錯誤。顯示內存有問題，更換顯卡試試。

9短：說明BIOS芯片檢驗和錯誤。

1長3短：說明內存錯誤。內存損壞，更換。

1長8短：說明顯示測試錯誤。顯示器數據線沒插好或顯示卡沒插牢。

4 系統能啟動，有顯示畫面，出現故障提示

這時可以根據提示來判斷故障部位。下面就是一些常見的故障提示的判斷：

4.1 提示“CMOS Battery State Low”

原因：CMOS參數丟失，有時可以啟動，使用一段時間後死機，這種現象大多是CMOS供電不足引起的。

對於不同的CMOS供電方式，採取不同的措施：

a 焊接式電池：用電烙鐵重新焊上一顆新電池即可；

b 鈕釦式電池：直接更換；

c 芯片式：更換此芯片，最好採用相同型號芯片替換。

如果更換電池後時間不長又出現同樣現象的話，很可能是主板漏電，可檢查主板上的二極管或電容是否損壞，也可以跳線使用外接電池，不過這些都需要有一定的硬件維修基礎才能完成。

4.2 提示“CMOS Checksum Failure”

CMOS中的BIOS檢驗和讀出錯；提示“CMOS System Option Not Set”， CMOS系統未設置；提示“CMOS Display Type Mismatch ”，CMOS中顯示類型的設置與實測不一致；提示“CMOS Memory Size Mismatch”，主板上的主存儲器與CMOS中設置的不一樣；提示“CMOS Time & Date Not Set ”，CMOS中的時間和日期沒有設置。這些都需要對CMOS重新設置。

4.3 提示“Keyboard Interface Error”後死機

原因：主板上鍵盤接口不能使用，拔下鍵盤，重新插入後又能正常啟動系統，使用一段時間後鍵盤無反應，這種現象主要是多次拔插鍵盤引起主板鍵盤接口鬆動，拆下主板用電烙鐵重新焊接好即可；也可能是帶電拔插鍵盤，引起主板上一個保險電阻斷了（在主板上標記為Fn的東西），換上一個1歐姆／0.5瓦的電阻即可.

4.4 自檢過程中斷在xxxK Cache處

這表示主板上Cache損壞，可以在CMOS設置中將“External Cache”項設為“Disable”故障即可排除。同理，在自檢主板部件時出現中斷，則可以認為該部件損壞，解決方法一般可以在CMOS中將其屏蔽，如果不能屏蔽該部件最好更換主板

4.5 提示“FDD Controller Failure”

BIOS不能與軟盤驅動器交換信息；提示“HDD Controller Failure”， BIOS不能與硬盤驅動器交換信息。應檢查FDD（HDD）控制卡及電纜。

4.6 提示“8042 Gate A20 Error ”

8042芯片壞；提示“DMA Error”， DMA 控制器壞。這種故障需要更換。

4.7 提示“Display Switch Not Proper”

主板上的顯示模式跳線設置錯誤，重新跳線。

4.8 提示“Keyboard is Lock...Unlock it”

鍵盤被鎖住，打開鎖後重新引導系統。

4.9 IDE接口設備檢測信息為：“Detecting Primary （或Secondary）Master（或Slave）... None”

表示該IDE接口都沒有找到硬盤，如果該IDE口確實接有硬盤的話，則說明硬盤沒接上或硬盤有故障，可以從以下幾方面檢查：

a 硬盤電源線和數據線是否接觸不良，或換一根線試試；

b CMOS設置有無錯誤，進入CMOS將“Primary Master”、“Primary Slave”、“Secondary Master”三項的的“TYPE”都設置成“Auto”；

c 替換法確認硬盤本身有故障。

4.10 IDE接口設備檢測信息下面顯示“Floppy disk(s) fail(40)”出錯信息

表示CMOS所指定的軟盤驅動器有問題。判斷和解決的方法與硬盤相似。

5 系統不能引導。

這種故障一般都不是嚴重問題，只是系統在找到的用於引導的驅動器中找不到引導文件，比如：BIOS的引導驅動器設置中將軟驅排在了硬盤驅動的前面，而軟驅中又放有沒有引導系統的軟盤或者BIOS的引導驅動器設置中將光驅排在了硬盤驅動的前面，而光驅中又放有沒有引導系統的光盤，這個都很簡單，將光盤或軟盤取出就可以了，實際應用中遇到“Disk Boot Failure，Insert System Disk And Press Enter”的提示，多數都是這個原因。

如果是硬盤不能引導的話一般有兩種情況：一種是硬盤數據線沒有插好，另一種就是硬盤數據損壞。前者一般多會出現硬盤容量檢測不正確和引導時出現死機的現象；後者則是乾脆找不到引導文件或提示文件損壞。前者只需重新連接好數據線即可；後者則需要用啟動盤或啟動光盤啟動，根據實際情況來定：

5.1 提示“Invalid partition table”或“Not Found any [active partition] in HDD Disk Boot Failure，Insert System Disk And Press Enter” ，這說明找不到硬盤活動分區，需要對硬盤重新分區。

5.2 提示“Miss operation syste”，說明硬盤活動分區需要重新格式化（format c:/s）。

5.3 提示“Invalid system disk Replace the disk， and then press any key”或顯示“Starting Windows 98 …”時出現死機，說明硬盤上的系統文件丟失了或損壞，使用“sys c:”，命令傳遞系統文件給c盤，再將Command.com拷貝給c盤。

6 硬盤可以引導，但Windows不能正常啟動，也不能進入安全模式。

這種情況表明Windows出現了嚴重的錯誤，首先，用殺毒軟件查殺病毒，看是不是病毒造成的，如果沒有發現病毒可以用以下方法試一試。

6.1 直接將接口卡與各個外設都撥去，再插回去，並調整接口卡上的設置（如果可以的話）來檢查是否是硬件衝突造成，開機看看是否可正常進入Windows。

6.2 檢查CMOS中的設置是否有不正確的地方，若不清楚，可選擇Load Bios Default項目，然後重開機，開機看是否可正常進入Windows。

6.3 在啟動時按下F8鍵，一般會出現6個選項（如果安裝了DOS6.22則出現7個選項）選擇第4項“step－by－step confirmation”進入單步運行方式，按照出現死機的命令選擇處理方法：

a 執行“Process the system registry”計算機就死機，說明是註冊表故障，那麼可以重新啟動按F8鍵，選第4項後，只在Device=c:\windows\himem.sys 這一項上按“Y”，其餘的按“N”後，在DOS提示符下輸入Scanreg ／fix 修復註冊表或者是 scanreg ／restore恢復到以前系統自動備份的註冊表後，再重新啟動即可。

b 在出現“Create a startup log file (BOOTTLOG.TXT) [Enter=Y，Esc=N]”時，選擇Y建立Bootlog.txt這個文件，可以檢查啟動過程中各個系統文件裝載的情況。如果在裝載某一個.vxd文件時死機，可以到其它計算機上覆制該.vxd文件拷貝過來。

c 如果是一啟動就出現“現在可以安全地關閉計算機了”，一般是因為Windows的System目錄的vmm32.vxd被損壞，可以到其它計算機上重新複製一個過來。

d 一啟動就出現藍屏並顯示“VFAT Devcie Initialization Faild”，表示調用Windows目錄下的dblbuff.sys和ifshlp.sys文件出了問題，可以在config.sys文件中手工加入如：

“device=c:\windows\dblbuff.sys”和“devicehigh=c:\windows\ifshlp.sys”，如果還是不行，則表明這兩個文件己損壞，可以到其他計算機上新複製過來一份。

6.4 使用文本編輯程序將Msdos.sys文件中的LOGO參數設置為“0”，關閉後再開機時，看看是否可進入Windows，若還是不行，則只能重新安裝Windows系統了。在重裝Windows時，建議先採用覆蓋原來的Windows的方法，這樣做的好處是：如果故障能夠排除，原來的許多應用程序就不用再安裝了；如果覆蓋式安裝不能解決問題，再採用硬盤格式化之後安裝Windows的方法。

7 Windows無法啟動，但能進入安全模式。

這種故障一般問題不大，修復的機率較高。具體可以參照下面的幾種方法去做：

7.1 在安裝新設備後不能正常啟動，進入安全模式後，在“控制面板”－－“系統”中選擇“設備管理器”選項卡，在列出的所有設備中查找前有一個感嘆號衝突的設備，如果有的話，打開這個設備的“屬性”，查看“資源”選項卡，看看這個設備與其它設備的中斷衝突，然後取消“自動設置”複選框，單擊“更改設置”按鈕，選擇一個沒有使用的中斷號即可。如果還不行，則可以在設備屬性中選擇“常規”選項卡，選中“在此硬件配置文件中禁用”複選框，如果能正常啟動成功，證明這個硬件的驅動程序可能有問題，可以在設備管理器中將它刪除後重裝驅動程序。

7.2 安裝了啟動時自動運行軟的後不能正常啟動，可以將其卸載，待系統可以正常啟動後，嘗試重新安裝。順便說一句，安裝系統啟動時自動運行的軟件不要與其它軟件在同一次啟動中安裝可以減少此類故障。

7.3 電腦啟動時自動運行的文件出現故障造成Windows不能正常啟動，或者前一種情況軟件無法卸載（包括不能完全卸載）的，可以到下面的地方找到並將其刪除：

a “開始”菜單內的“程序”文件夾中的“啟動”文件夾內。

b Config.sys與Autoexec.bat中，若有則在該行前面加上“rem”不讓它運行。通常防病毒程序都會在這兩個文件中加入要執行的程序。

c Window目錄下的Win.ini中[Windows]段中的Run或Load參數後，這裡也可以在該行前加上“Rem”，不讓它執行。

d 註冊表（Registry）中，運行註冊表編輯器(Regedit)，進入編輯器後利用“編輯”中的“查找”功能查找產生故障的自動可執行文件，將不要執行項目的主鍵刪除或將文件更名。注意：修改註冊表時，要事先用“導出註冊表文件”的方法做好備份。

7.4 如果在系統啟動時提示丟失了某些在system.ini文件中的.vxd、.386等文件，你可以到其他計算機上拷貝相應的文件到對應的位置，如果還不行，你還可以備份你的Windows安裝目錄下system.ini文件，然後用文本編輯器打開system.ini文件找到相應的.vxd或.386的那一行，將該行刪除即可，如果修改有誤，再用備份還原即可。

7.5 如果 Windows不能正常啟動是由系統文件損壞造成的，可以運行在Windows\system目錄下的Sfc.exe文件來檢查並恢復遺失或損毀的系統文件，如果產生問題的那個文件是Windows系統需要用到的，則會被修復回來。

7.6 如果Windows不能正常啟動是由文件版本衝突造成的，可以運行Windows目錄下的Vemui.exe文件，找出產生問題的文件後，選擇“恢復所選文件（R）”按鈕，則Windows98會自動用不同號的版本取代目前使用文件，而版本衝突管理器也會給這個被取代的文件製作備份，可以再換回來。

三、啟動盤是怎麼回事？

如果硬盤出現問題（操作系統安裝在硬盤上），即無法啟動到操作系統，此時就可以退而求其次，先啟動到啟動盤（dos操作系統或圖形化用戶界面操作系統的精減版，如PE操作系統）。

這是因為，啟動文件可以保存在任何一種介質上，但當硬盤文件損壞時或無法啟動windows時，就需要用到用光盤、U盤、移動硬盤或磁盤（如有軟盤驅動器的話）的啟動盤了（安裝盤上也有啟動文件，其本身也是一個啟動盤），啟動到dos或PE系統進行安裝或恢復。

如果在DOS上再增加若干款DOS實用程序（工DOS外部命令）的也有稱為系統盤的。

I Boot引導程序

II IO.SYS(輸入輸出處理程序)

III MSDOS.SYS（文件處理程序）

IV command.com（命令處理程序）

常見DOS命令，可在DOS環境下完成磁盤分區、磁盤格式化以及文件拷貝等操作。同時，它還包含了光盤、U盤驅動程序，允許Windows系統不能啟動時，在DOS環境下直接使用光驅a或U盤。

1 為U盤安裝windows啟動代碼

開始→cmd→diskpart：

DISKPART>list disk

DISKPART>select disk 3

選擇你的U盤（注意千萬不要選錯盤，下一步操作會清空選擇磁盤的數據）

DISKPART>clear

清空盤上所有數據和分區（此操作將清空選擇的盤上所有數據，如果U盤上有重要數據，在操作此步前請先備份）

DISKPART>create partition primary

使用U盤所有空間創建一個主分區.

DISKPART>select partition 1

選擇剛剛創建的分區。

DISKPART>format quick fs=fat32

快速格式化剛剛的分區（這裡的fs=fat32指文件系統使用fat32（推薦），你也可以考慮fs=ntfs使用ntfs系統，但是在dos或者linux等環境下不支持）

DISKPART>active

將分區設置為活動分區。

好，到這裡，我們已經為U盤安裝了windows啟動代碼。但是此時的U盤上實際上什麼文件都沒有，如果直接引導開機的話會提示bootmgr missing。

2 安裝windows引導程序bootmgr

打開easybcd（官方下載地址：https://neosmart.net/Download/Register，只要輸入用戶名和郵箱即可以下載個人版）。
切換到BCD Deployment，partition（分區）選擇我們的U盤，然後點擊Install BCD。

等待安裝完成。
彈出提示：安裝完成，是否打開剛剛安裝的BCD，點擊是。
到這裡我們可以打開U盤查看下，一共可以看到2個文件夾和一個bootmgr文件。

（如果只能看到EFI文件夾，請按如下設置顯示所有文件）

組織→文件夾和搜索選項→查看，按紅框裡設置，確定。

這裡說明一下幾個文件的用途：

Bootmgr ---- 我們的legacy模式下的引導文件

EFI文件夾---- 裡邊是UEFI模式下的引導文件（這個版本的easybcd不支持UEFI）

Boot文件夾 --- 裡邊是各種語言包/字體包以及引導配置保存文件BCD

我們後面所有的操作均是對這個BCD文件進行，BCD文件中保存的是啟動菜單和配置信息。

好，我們回到easybcd軟件。

點擊viewsettings查看，顯示我們操作的文件是I:\Boot\BCD，裡邊看到的是空白的，我們什麼都沒有添加。

3 添加啟動項

點擊Add New Entry。

可以看到支持很多種引導方式。

比如，直接引導各種系統windows，linux，mac，甚至可以調用grub。

或者通過加載vhd、iso、wim、img等鏡像引導。

3.1 添加一個wim封裝的win PE系統

引導Wim封裝的winpe我們選擇winpe。

把wim鏡像複製到U盤上，可以放在根目錄，也可以放在某個文件夾下面。

這裡我們在U盤上新建一個pe文件夾，然後把下載下來的PE放在這裡。

（注意U盤引導任何鏡像，請先將該鏡像複製到U盤上，否則只能在本機引導，換個電腦就找不到鏡像文件了）

在easybcd中選擇winpe。

Type選擇WIM Image(Ramdisk) 推薦。

使用內存盤運行-在加載的時候將整個WIM加載到內存中，作為一個虛擬內存盤，加載完成後U盤就可以拔掉了，但wim不能太大超出內存大小。

Name隨便填一個。

Path就是我們剛剛複製到U盤上的PE文件。

點擊Add Entry。

回到viewsettings我們可以看到已經添加了一項。

3.2 也可以再增加一個win7安裝盤

用ultraiso打開一個原版WIN764 ISO鏡像，將sources，support，upgrade目錄解壓縮到U盤根目錄（注意不要解壓所有的文件，我們只要3個文件夾，尤其是不要解壓boot目錄，裡邊也包含bcd文件，會覆蓋我們u盤上剛剛生成的bcd）

然後在easybcd中添加一個winpe，path選擇source下的boot.wim。

3.3 還可以再添加一個DOS 啟動ISO鏡像（可用easyboot製作）

同樣複製到U盤，新建個iso目錄。

選擇ISO文件，選擇LOAD FROM MEMORY(針對小文件兼容性好)，點擊 add Entry。

（不是所有的ISO都能啟動，這個只能測試，比如LINUX安裝盤就無法啟動，太大的ISO可能也不行）

4 啟動菜單設置

到這裡，我們添加了3個啟動項。

在Edit Boot Menu裡可以設置排序，默認項，倒計時時間等等，區域選擇China，否則中文無法顯示，最後別忘了點擊Save Settings，並退出easybcd（不退出可能部分設置不保存）

以上只是描述了一般啟動盤的製作過程。
