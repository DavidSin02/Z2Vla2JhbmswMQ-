linux程序編譯過程簡析

2020-11-14
| 科技



簡介


Linux環境下，平時進行應用程序的編譯操作時，一般都是直接使用gcc編譯生成可執行文件，很少關注gcc編譯的具體實現過程；雖然平時不需要關注編譯過程，但是在出現一些問題的時候，我們需要逐步去分析哪裡出現問題，學習編譯過程可以有助於問題的分析和解決；

Linux應用程序的編譯過程，包含了四個過程：預處理(Prepressing)、編譯(Compilation)、彙編(Assembly)、鏈接(Linking)；這四個過程在使用時，被gcc命令給隱藏起來了；


編譯過程


這裡我們以C語言為基礎，使用軟件界經典的入門程序Hello world，來驗證和學習編譯的四大步驟；

// cat hello.c#include <stdio.h>int main(void){    printf("Hello world!\n");        return 0;}


Linux系統中，使用gcc來編譯C語言程序很容易；

$ gcc hello.c$ ./a.out


以上的編譯過程，是由四個步驟合併而成，這四個步驟分別是預處理、編譯、彙編、鏈接；


1. 預處理


編譯的第一步是預處理，預處理將源代碼文件hello.c和包含的頭文件stdio.h，使用預編譯器cpp（此處的cpp不是C++編譯器）預編譯成.i文件；相當於如下命令操作：

$ gcc -E hello.c -o hello.i

或

$ cpp hello.c > hello.i

預編譯過程主要處理源代碼中以”#“開始的預編譯指令，如：頭文件“#include”、宏定義“#define”、條件編譯“#if”等；主要規則如下：

將所有的“#define”刪除，並且展開所有的宏定義；

處理所有條件預編譯指令，如：#if、#ifdef、#ifndef、#elif、#else、#endif等；

處理“#include”預編譯指令，遞歸地將被包含文件插入到該預編譯指令所在的位置；

刪除所有的註釋；

添加行號和文件名標識，以便於編譯時編譯器產生調試用的行號信息和編譯時產生編譯錯誤或警告的行號；

保留所有的#pragma編譯器指令，編譯器要使用；

經過預編譯後的.i文件不包含任何宏定義，所有的宏和文件已經被插入到.i文件中，可以通過查看預編譯後的文件來確定宏定義是否正確或頭文件是否包含；


2. 編譯


編譯器將高級語言翻譯成機器語言，編譯器的工作過程一般分為六步：掃描、語法分析、語義分析、源代碼優化、代碼生成、目標代碼優化；最終完成從源代碼到最終目標代碼的過程；

編譯過程把預處理完的文件按照編譯過程的六步操作後，生成相應的彙編代碼文件，這部分是程序構建最複雜、最核心的部分，作者能力有限，此處不再進行描述；簡要介紹命令操作如下：

$ gcc -S hello.i -o hello.s

或使用cc1工具，cc1工具將預編譯和編譯合併在一起；

$ cc1 hello.c > hello.s

或者從C語言直接編譯為彙編代碼文件：

$ gcc -S hello.c -o hello.s


現在版本的gcc把預編譯和編譯兩個步驟合併為一個步驟，使用cc1程序來完成編譯操作；


3. 彙編


彙編是由彙編器as來完成的，彙編器是將彙編代碼轉變成機器可以執行的指令，幾乎每一條彙編語句都對應一條機器指令；彙編過程沒有複雜的語法、語義，也不需要指令優化，只是根據彙編指令和機器指令的對照表翻譯；彙編命令如下操作：

$ as hello.s -o hello.o

或

$ gcc -c hello.s -o hello.o

或者直接從C語言經過預編譯、編譯、彙編，輸出目標文件：

$ gcc -c hello.c -o hello.o


4. 鏈接


鏈接由鏈接器ld來完成；鏈接器將由不同源代碼生成的目標文件，和庫文件進行鏈接才能生成最終的可執行程序文件；鏈接過程比較複雜，鏈接的簡化命令如下所示

$ ld *.o -o a.out

經過鏈接之後生成的目標文件，是最終的可執行文件；

鏈接由兩種方法，靜態鏈接和動態鏈接；兩者最大的區別是鏈接的時機不一樣，靜態鏈接是在生成可執行程序前鏈接；動態鏈接是在程序執行時；關於鏈接部分內容可以參考另外一篇文章《靜態鏈接和動態鏈接》；

Linux下進行鏈接的缺省操作是，首先鏈接使用動態庫；即如果同時存在靜態和動態庫文件，不特別指定時，將與動態庫鏈接；

gcc命令是四個步驟的包裝，會根據不同的參數要求去調用預編譯程序cc1、彙編器as、鏈接器ld，經過四個步驟的操作後，最終編譯生成ELF格式的二進制文件；關於ELF二進制文件的解析，可以參考《linux中ELF二進制文件解析》


總結


Linux環境中的編譯過程，分為預處理、編譯、彙編、鏈接四個步驟，每個步驟都有相對應的工具操作完成；gcc命令是由四個步驟包裝得到，gcc命令會根據不同的參數要求調用預編譯程序cc1、彙編器as、鏈接器ld，最終生成可執行文件；
