R中的數據表可視化工具——DT包

2020-11-14
| 科技

小編最近在用業餘時間學習R的一個可視化工具Rshiny，在這個過程當中，接觸到R中的一個數據包叫DT。在自學查資料時，發現網上對這個數據包的介紹十分有限。所以，小編在這裡現學現賣，寫一篇小文給大家介紹一下這個R包。鑑於小編水平有限，有任何錯誤和紕漏，還歡迎各位大神指正。

那好，我們就開始吧！~

有人可能會詫異，一個數據包而已嘛，用得著特別介紹嗎？小編最開始也是這麼想的……然而當我找到官方網頁，打開這個數據包的使用方法介紹的時候，確確實實被裡面紛繁複雜的各類參數給驚到了。一個R包的protocol竟然做出了堪比一個分析方法的protocol的厚重感，這讓我不由得肅然起敬。

扯了這麼多，這個DT包到底是幹什麼用的呢？DT包提供了JavaScript庫DataTables在R中的一個接口，使得R對象（矩陣或者數據框）可以在HTML頁面上顯示為表格，並提供了對數據的篩選、分頁和排序等功能。

翻譯成人話就是，DT包使得JavaScript中的方法能在R中被應用，並將矩陣或者數據表在網頁中可視化成表格，還能實現各種功能。小編先用R中自帶的數據集iris給大家一個直觀感受：

install.packages("DT") #安裝DT數據包

library(DT) # 載入數據包

datatable(iris) # DT包中的核心函數datatable()

注：可視化之前可能還需要加載HTTP和WebSocket服務程序

install.packages("httpuv")

install.packages("yaml")

library(httpuv)

library(yaml)

datatable(iris)


由圖可見，在沒有任何多餘設置的情況下，僅僅使用DT包中的datatable函數，就可以直接可視化表格數據，並且自動完成對數據的分頁顯示，使用戶可以對每一列進行升（降）序排列，控制每頁顯示條目的多少，以及對數據進行檢索等一系列功能。

DT包中最主要的函數就是datatable()函數，完整參數列表如下：

datatable(data, options = list(), class="display", callback = JS("return table;"), rownames, colnames, container, caption = NULL, filter = c("none","bottom", "top"), escape = TRUE, style = "default", width = NULL, height = NULL, elementId = NULL, fillContainer = getOption("DT.fillContainer", NULL), autoHideNavigation = getOption("DT.autoHideNavigation",NULL), selection = c("multiple", "single", "none"), extensions = list(), plugins = NULL, editable = FALSE)

實話說小編看見這個參數列表就已經先慫了。但是參數越多就意味著自主權越多，越能按照自己的心意設置想展現的樣子，這對於可視化來說是很重要的。

所以在這篇小文裡，小編先拋磚引玉，簡要粗淺的講一些我覺得比較好理解的，常用的參數，剩下的參數設置，就請感興趣的各位自行研究啦~

1、表格編輯：

DT::datatable (head (iris) , editable = TRUE)

將參數editable更改為TRUE，即可對呈現出來的表格內容進行編輯。鼠標懸停在數據上方雙擊，即可改變表格內容。

2、行名顯示：

默認顯示行名，通過設置rownames =FALSE，來控制行名不顯示。另外也可以rownames等於一組向量來為行名重新進行賦值。

3、列名設置：

默認顯示列名，可以通過列名編輯來對一個或多個列名進行更改。

以上圖的iris數據集為例：

datatable(head(iris),colnames = c('Here', 'Are','Some', 'New', 'Names'))

將列名修改成瞭如下圖所示：



也可以根據列名來指定某一列進行列名改變，例如：

datatable(head(iris), colnames= c('A Better Name' ='Sepal.Width'))

將列名'Sepal.Width'修改為'A Better Name'，注意，被修改的原名在賦值號後面


也可以根據位置對某一列或某幾列進行指定，對其進行列名更改

datatable(head(iris),colnames = c('Another BetterName' = 2, 'Yet Another Name' = 4))

將第二列的列名更改為'Another Better Name'，將第四列的列名更改為'Yet Another Name'


4、設置數據表標題

datatable( head(iris),caption ='Table 1: This is a simple caption for the table.')

caption參數設置數據表標題

簡單講了以上幾個小參數，接下來我們來看datatable參數中的大boss——options參數列表。這個參數的取值是一系列小參數的list，個別小參數裡還包含更次一級的參數，讓人眼花繚亂。以下小編挑一些給大家介紹一下：

1)設置列寬：

默認情況下，列寬設置選項autoWidth為FALSE，代表表格自動分配寬度值，每一列的列寬度值均一。如果要人為修改某一列或者某幾列的寬度值，則需要先將autoWidth設置從FALSE更改為TRUE，之後用columnDefs參數設置特定列和寬度。例如：

datatable(iris, options = list( autoWidth = TRUE, columnDefs = list(list(width = '200px', targets = c(1, 3)))))

以上代碼表示將第一列與第三列的寬度設置成 ’200px’

效果如下：

原始數據表，列寬均一



第一列和第三列的列寬度改變：



2)一些初始化設置

可以通過option參數列表進行一系列初始化設置：

datatable( head ( iris,20), options = list(columnDefs = list ( list(className = 'dt-left', targets = 1)), pageLength = 5, lengthMenu = c(5, 10, 15, 20)))

上述代碼的意思是顯示iris數據表的前20行，第一列的對齊方式為左對齊（即：dt-left），每頁顯示長度為5條數據，每頁顯示條目長度選項列表為（5,10,15,20），效果如下：



3)排序相關，例如：

datatable(head(mtcars, 30), options= list(order = list(list(4, 'desc'), list(2, 'asc'))))

上述代碼的意思為，取數據集mtcars的前30行，對第四列降序排列，若第四列數值相同，則按照第二列升序進行排列。這種設置只是控制表格的初始呈現方式，之後在頁面上點擊排序按鈕，依舊可以改變排列順序。



4)DOM元素：默認情況下，表格具有以下由單字母表示的DOM元素：l- length (changing inputcontrol) 長度菜單，f-(filtering input) 篩選搜索框，t- (table) 表格主體，i- (information summary) 信息摘要和p- ( pagination control) 分頁控件。可以選擇使用dom選項來組合想要可視化顯示的子集。例如：

datatable(iris, options = list(dom = 'pt')) # 可視化只顯示錶格主體和分頁器

datatable(iris, options = list(dom = 'lpt')) # 可視化顯示錶格主體、分頁器和長度選擇菜單

需要注意的是，表格主體默認顯示，即不管dom選項裡包不包含t，都顯示錶格。


5)選項中的回調:我們還可以在options選項中自定義回調函數（callback function）。例如，可以在選項中使用initComplete回調函數在初始化後將表頭的背景顏色更改成其他顏色：

datatable(head(iris, 20), options = list(

initComplete = JS(

"function(settings, json) {",

"$(this.api().table().header()).css({'background-color':'#000080','color': '#ffff00'});",

"}")))

上述代碼的意思為，將表頭的背景顏色改為藍色，列名改為黃色，所使用的顏色都是RGB顏色代碼。

注：函數JS( ) 表示這個元素不是普通的字符向量，而是JavaScript代碼。這個特殊的功能，使得使用者可以在R中直接編輯JavaScript函數（使用jsonlite）並在瀏覽器中對其解碼。


對於令人頭痛的options參數列表的具體應用，小編可以負責任的告訴大家，想全部掌握，那是——不可能的。粗略估計大小參數大概有幾十個吧。不過大家不用過於憂慮，小編找到一個好東西：https://datatables.net/reference/option/

這個網站對datatable函數中的options參數進行了彙總，並且提供了查詢功能。界面如圖：


可以根據下方黑框內的功能提示，查找自己想要的參數用法。也可以在搜索框內直接輸入參數，比如columnDefs列定義參數，點擊搜索之後會呈現其具體用法和該參數下面所有的二級甚至三級參數設置，一目瞭然，十分方便。
