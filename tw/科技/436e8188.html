[REST]Total Control &易語言的完美結合

剛拿到Total Control的時候，只用了軟件自帶的基礎電腦控制手機功能。後期由於控制的手機越來越多，就想要更多的自動化更效率更便捷的實現更多的功能。本身有一些腳本基礎，便開始研究腳本。幸好TC提供了Rest API，可以實現多種腳本語言的編寫功能，剛好易語言也很容易入門。所以，把自己研究出來的幾種比較常用的功能的腳本分享一下，希望大家喜歡：

易語言概述

易語言是一個自主開發，適合國情，不同層次不同專業的人員易學易用的漢語編程語言。

易語言降低了廣大電腦用戶編程的門檻，尤其是根本不懂英文或者英文了解很少的用戶，可以通過使用本語言極其快速地進入Windows程序編寫的大門。

如何用易語言結合Total Control 多控系統控制手機

Total Control REST API 採用 RESTful Web 服務框架設計，基於原有的 Total Control JS API 開發，其中一些常量被縮短以便於閱讀和編碼。

支持任意腳本語言：如易語言,Java, Node JS, Python, Ruby, C/C++等。

支持各類操作系統：Android, iOS, Windows, macOS。

使用 Total Control REST API 控制設備的一般步驟：

1. 獲取 Total Control 腳本開發所用的開發 token

2. 獲取一個設備 ID 值

3. 通過第二步的一個設備 ID 值（單設備）,對獲取的設備進行操作

[JavaScript] Total Control 用代碼實現通訊錄的導入導出

Total Control 在6.9.0版本提供通訊錄的導入導出 API 啦，我們可以用代碼實現通訊錄的導入導出。

contactLoad 導入通訊錄

原型：contactLoad(fileName, fields, opt, rule)

是否支持多設備：不支持

適用版本: Total Control 6.9.0 及以後版本

描述：導入通訊錄聯繫人信息，可以導入json和excel格式的通訊錄文件。

參數：

path: 字符串，需要導入的通訊錄文件路徑，文件可以是 json 格式，也可以是後綴為 .xls 和 .xlsx 的 excel 格式，例如， "E:/File/mycontact.json"。

要導入的文件，必須包括聯繫人姓名和至少一個聯繫人電話號碼兩個字段。

fields: 字符數組，可選參數，需要導入的字段名稱，不指定字段，則導入全部字段，如果只想導入聯繫人名稱和聯繫人電話號碼，可這樣寫：['name', 'phone1']。

opt：對象，可選參數，設置是否需要異步執行，不填，表示同步執行，如果需要異步執行，這樣表示:{async: true}，異步執行，可以用 JS API "contactCheckStatus(filename)" 獲取通訊錄導入狀態。

rule：整型，恢復通訊錄規則，不填，表示替換重複數據，其中，

1 表示保留重複數據

2 表示替換重複數據

3 表示跳過重複數據

完整示例：導入excel格式的通訊錄文件

// 獲取當前主控設備對象

var device = Device.getMain();

if(!device) {

print("Failed to get device");

} else {

//導入"E:/File/sigmaContact.xls"文件中的聯繫人姓名和聯繫人電話號碼兩欄，{async: true}表示異步執行，3表示跳過重複數據

var ret = device.contactLoad("E:/File/sigmaContact.xls",['name', 'phone1'],{async: true},3);

if (ret == true){

print("Successfully !!!");

} else {

print("Sorry,Failed !!!");

}

}

contactSave 導出通訊錄

原型：contactSave(path)

是否支持多設備：不支持

適用版本: Total Control 6.9.0 及以後版本

描述：將手機通訊錄導出保存到電腦上。

參數：

path：導出的通訊錄文件路徑，例如：E:\File\contact.json

導出的通訊錄聯繫人信息是以 json 格式存儲的，例如：
