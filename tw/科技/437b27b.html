如何從零開始兩天擼一個微信小程序？！（內含源碼）

2020-10-29
| 科技
【程序人生 編者按】新人如何入門小程序？不妨嘗試做一個實用性強的天氣應用小程序吧。下面是作者只用了兩天時間寫的已上線天氣小程序，一起來學學如何做的。


前期準備


天氣數據

因為這是個人的DEMO，開發前就決定選擇免費的天氣數據（個人開發免費），我選擇了百度地圖開放平臺的天氣數據，正好也提供了小程序對應的 SDK，可能相比於其他的天氣 API，百度返回的數據偏少：例如當天 PM2.5、當天和未來三天數據、當天生活指數，但是對於一款簡單的天氣應用小程序來說也夠了。

地理編碼

獲取天氣數據默認返回當前城市的天氣數據，如果要獲取其他的城市的天氣數據，需要傳入經緯度。為了獲取其他城市的經緯度，這裡使用的地圖的地理編碼接口，輸入城市名，輸出經緯度，然後調用獲取天氣數據 API 即可。

運行前準備


  1. 註冊微信小程序，獲取 AppID
  2. 註冊百度地圖開放平臺開發者，創建應用，獲取 ak（其他配置自行查看）
  3. 在 app.js 中替換 globalData 中的 ak 為自己的 ak
  4. Run

具體實現

該應用有五個頁面：首頁、城市選擇頁、設置頁、關於頁、系統信息頁（展示頁）。分別是這樣實現的：

首頁

從上到下依次是：其他城市天氣搜索、當前城市數據展示、當天和未來三天天氣數據展示、當天生活指數展示、footer。下拉刷新會刷新當前地區的天氣數據。其中，頂部城市天氣搜索和生活指數可以在設置中隱藏。屏幕右下角是一個可以移動的懸浮球菜單，點擊後會彈出城市選擇、設置、關於頁面的入口。背景色默認是 #40a7e7 純色，可在設置中更換背景圖，未來三天天氣預報和生活指數分別添加了透明的黑色背景。最終實現圖如下：



主頁面

先定義一個方法獲取當前地區的天氣數據：

init(params) { let that = this let BMap = new bmap.BMapWX({ ak: globalData.ak, }) BMap.weather({ location: params.location, fail: that.fail, success: that.success, })},

ak 請替換為自己的 ak，因為需要獲取用戶的地理位置，所以在 fail 的回調中需要處理用戶拒絕獲取地理位置的邏輯。這裡處理為：提示打開地理位置授權，3000ms 後 wx.openSetting() 跳轉到小程序設置頁，如下：

fail (res) { wx.stopPullDownRefresh() let errMsg = res.errMsg || '' // 拒絕授權地理位置權限 if (errMsg.indexOf('deny') !== -1 || errMsg.indexOf('denied') !== -1) { wx.showToast({ title: '需要開啟地理位置權限', icon: 'none', duration: 3000, success (res) { let timer = setTimeout(() => { clearTimeout(timer) wx.openSetting({}) }, 3000) }, }) } else { wx.showToast({ title: '網絡不給力，請稍後再試', icon: 'none', }) }},

獲取到用戶的地理位置後，執行 success：

success (data) { wx.stopPullDownRefresh() let now = new Date() // 存下來源數據 data.updateTime = now.getTime() data.updateTimeFormat = utils.formatDate(now, "MM-dd hh:mm") let results = data.originalData.results[0] || {} data.pm = this.calcPM(results['pm25']) // 當天實時溫度 data.temperature = `${results.weather_data[0].date.match(/\d+/g)[2]}` wx.setStorage({ key: 'cityDatas', data: data, }) this.setData({ cityDatas: data, })},

success 裡緩存了最新一次獲取的天氣數據+更新的時間 cityDatas，小程序的模板裡無法使用方法，所以數據需要在 js 裡面先格式化。

calcPM 用來計算當前 PM2.5 的質量，返回“優良差”類似字樣，範圍標準可自行搜索。當天的實時溫度並沒有給出獨立的字段，而是混在了 wearther_data[0] 的 data 字段裡："date": "週五 06月29日 (實時：34℃)"，需要自行提取。返回的天氣 icon 和色調不搭，就沒有使用。其他的數據按照按照我們要顯示的格式直接填充即可。

城市天氣搜索

獲取天氣數據傳參為經緯度，所以搜索城市天氣時，需先將城市轉換為對應的經緯度，然後調用獲取天氣數據 API 即可。

返回的數據格式為：

{ "status":0, "result":{ "location":{ "lng":117.21081309155257, "lat":39.143929903310074 }, "precise":0, "confidence":12, "level":"城市" }}

然後直接調用獲取天氣 API 即可。具體代碼可參考文末。

搜索動畫彩蛋

在搜索框裡搜索 520 或 521，會出現從頂部下小心心的動畫，如下：



只需創建了一個 heartbeat 的組件。wxml 結構是遍歷數組，創建多個大小、位置隨機的圖片：

<image wx:for='{{arr}}' wx:key='{{index}}' animation='{{animations[index]}}' class='heart' style='left:{{lefts[index]}}px;top:{{tops[index]}}px;width:{{widths[index]}}rpx;height:{{widths[index]}}rpx;' src='../../img/heartbeat.png'></image>

然後使用的是小程序提供的 wx.createAnimation，動畫的使用比較簡單，創建動畫，然後賦予 animation 屬性即可，比較簡單。

但是也有侷限性，比如沒有直接的動畫結束後的回調，但是可以使用 setTimeout 來實現等。這裡會用到可用窗口寬高，因為多處用到了該參數，所以在 app.js 裡面異步獲取了先。

動畫代碼如下：

dance (callback) { let windowWidth = this.data.windowWidth let windowHeight = this.data.windowHeight let duration = this.data.duration let animations = [] let lefts = [] let tops = [] let widths = [] let obj = {} for (let i = 0; i < this.data.arr.length; i++) { lefts.push(Math.random() * windowWidth) tops.push(-140) widths.push(Math.random() * 50 + 40) let animation = wx.createAnimation({ duration: Math.random() * (duration - 1000) + 1000 }) animation.top(windowHeight).left(Math.random() * windowWidth).rotate(Math.random() * 960).step() animations.push(animation.export()) } this.setData({ lefts, tops, widths, }) let that = this let timer = setTimeout(() => { that.setData({ animations, }) clearTimeout(timer) }, 200) let end = setTimeout(() => { callback && callback() clearTimeout(end) }, duration) },},

首頁搜索特定關鍵詞後，調用組件 dance 方法即觸發小心心動畫。

懸浮球菜單

屏幕右下角的懸浮球提供了三個頁面的入口：城市選擇頁、設置頁、關於頁。菜單彈出、收回會有動畫。

這裡的動畫分為彈出和收起，兩者寫起來基本上一樣的，只是動畫的參數不一樣。這裡貼出彈出的動畫：

// wxml<!-- 懸浮菜單 --><view class='menus'> <image src="../../img/location.png" animation="{{animationOne}}" class="menu" bindtap="menuOne" style='top:{{pos.top}}px;left:{{pos.left}}px;'></image> <image src="../../img/setting.png" animation="{{animationTwo}}" class="menu" bindtap="menuTwo" style='top:{{pos.top}}px;left:{{pos.left}}px;'></image> <image src="../../img/info.png" animation="{{animationThree}}" class="menu" bindtap="menuThree" style='top:{{pos.top}}px;left:{{pos.left}}px;'></image> <image src="../../img/menu.png" animation="{{animationMain}}" class="menu main" bindtap="menuMain" catchtouchmove='menuMainMove' style='top:{{pos.top}}px;left:{{pos.left}}px;'></image></view>// jspopp() { let animationMain = wx.createAnimation({ duration: 200, timingFunction: 'ease-out' }) let animationOne = wx.createAnimation({ duration: 200, timingFunction: 'ease-out' }) let animationTwo = wx.createAnimation({ duration: 200, timingFunction: 'ease-out' }) let animationThree = wx.createAnimation({ duration: 200, timingFunction: 'ease-out' }) animationMain.rotateZ(180).step() animationOne.translate(-50, -60).rotateZ(360).opacity(1).step() animationTwo.translate(-90, 0).rotateZ(360).opacity(1).step() animationThree.translate(-50, 60).rotateZ(360).opacity(1).step() this.setData({ animationMain: animationMain.export(), animationOne: animationOne.export(), animationTwo: animationTwo.export(), animationThree: animationThree.export(), })},

懸浮菜單是可以在屏幕上隨意滑動的，方法也很簡單，監聽 touchmove 事件即可，因為菜單展開方向是在左邊，所以懸浮菜單能往左邊移動的最遠距離要有一段間隔，否則展開的菜單就進入左邊屏幕了，移動到上方同樣邏輯（後期可以改成菜單展開方向隨移動而改變，而不是一味在左邊展開）。

城市選擇頁

城市選擇頁面就是一個城市列表，如下圖：



點擊相應的城市，跳轉到首頁獲取所選城市的天氣數據。這裡的城市數據是這樣的格式無序的列表：

{ "letter": "B", "name": "北京市" }

因為需要按照字母排列進行排序，所以需要先排序再遍歷，代碼如下：

// 按照字母順序生成需要的數據格式getSortedAreaObj(areas) { // let areas = staticData.areas areas = areas.sort((a, b) => { if (a.letter > b.letter) { return 1 } if (a.letter < b.letter) { return -1 } return 0 }) let obj = {} for (let i = 0, len = areas.length; i < len; i++) { let item = areas[i] delete item.districts let letter = item.letter if (!obj[letter]) { obj[letter] = [] } obj[letter].push(item) } // 返回一個對象，直接用 wx:for 來遍歷對象，index 為 key，item 為 value，item 是一個數組 return obj},

點擊城市後，需要通知首頁“我已經切換城市了，麻煩獲取下這個城市的數據謝謝”，這裡使用的是使用 getCurrentPages 獲取頁面堆棧，修改首頁數據的方式。

關於頁

關於頁是一個展示頁，沒有多少交互，使用到的 API 只有複製到剪切板 wx.setClipboardData。“微信快速聯繫”使用的是小程序提供的聯繫客服的方式。有精力的話，可以自己搭建服務，將小程序的消息 Push 到自己的服務上去。效果圖：



設置頁

設置頁的功能看著有點多，其實並不多，只是一堆 API 的調用。這個頁面分了自定義、檢查更新、小工具、清除數據三個部分。各個設置參數保存在 Storage 中。一個一個來說。

1. 自定義

  • 自定義首頁背景

自定義背景是將選取的圖片（wx.chooseImage）保存（wx.saveFile）到本地，然後首頁獲取（wx.getSavedFileList）保存的圖片，在首頁展示出來即可。

長按刪除，則是獲取（wx.getSavedFileList）保存的圖片，然後 wx.removeSavedFile 掉即可。現在設置的是本地只保存一張圖片，所以重新設置其他背景時，會刪除上一張背景圖，然後重新保存新背景圖。

  • 打開頂部城市天氣快捷搜索

該操作只是將首頁的頂部搜索 wx:if 掉而已。switch 組件的樣式可以通過修改默認的類來修改，調一個自己滿意的即可。

.wx-switch-input{width:84rpx !important;height:43rpx !important;}.wx-switch-input::before{width:82rpx !important;height: 38rpx !important;}.wx-switch-input::after{width: 38rpx !important;height: 38rpx !important;}
  • 顯示生活指數信息

同樣 wx:if 。

  • 檢查更新

檢查更新默認關閉。小程序的更新是在冷啟動時去檢查，如果有新版本會異步下載，再次冷啟動時會加載新版本。這裡使用 wx.getUpdateManager，因為該 API 基礎庫支持最低版本是 1.9.90，基礎庫版本低的會提示不支持，顯示的文案也會相應修改。

  • 小工具

1）NFC

使用 wx.getHCEState。

2）屏幕亮度

獲取屏幕亮度、設置屏幕亮度、保持常亮使用的 API 分別是 wx.getScreenBrightness、wx.setScreenBrightness、wx.setKeepScreenOn。完整實現可查看源碼。

3）系統信息

系統信息會跳轉到新頁面。

  • 清除數據

1）首頁懸浮球復位

首頁懸浮球的位置信息是保存本地的變量 pos，復位位置，清除 pos 即可。

2）恢復初始化設置

設置信息是保存本地的變量 setting，復位位置，清除 setting 即可。

3）清除所有本地數據

wx.clearStorage 即可。
