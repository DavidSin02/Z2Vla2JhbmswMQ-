Linux技巧：介紹從目錄路徑獲取文件名和目錄前綴的方法

2020-11-14
| 科技


本篇文章介紹下面三個內容：

  • 使用 dirname 命令獲取路徑名的目錄部分
  • 使用 basename 命令獲取路徑名的文件名部分
  • 使用 bash 的參數擴展獲取文件名的後綴名

使用 dirname 命令獲取路徑名的目錄部分

在 Linux 中，可以使用 dirname 命令獲取路徑名的目錄部分，不包含路徑名最後的文件名。

查看 man dirname 的說明如下：

dirname [OPTION] NAME...

dirname - strip last component from file name.

Output each NAME with its last non-slash component and trailing slashes removed;

if NAME contains no /'s, output '.' (meaning the current directory).

即，dirname 命令可以獲取所給路徑名的目錄部分，也就是從最後一個 ‘/’ 字符往前的部分，不包括 ‘/’ 字符自身。

如果所給的路徑名沒有包含 ‘/’ 字符，返回的目錄部分是點號 ‘.’，認為是當前目錄。

所給的路徑名參數不要求是真實存在的路徑，dirname 只是對所給的路徑名字符串進行處理。

dirname 命令可以提供多個路徑名參數，默認會逐行打印每個路徑名的目錄部分。

如果不想換行打印，可以使用 -z 選項，該選項的說明如下：

-z, --zero

separate output with NUL rather than newline

具體舉例如下：

$ dirname app/Phone.apk src/lib/utils.c

app

src/lib

$ dirname utils.c

.

$ dirname -z app/Phone.apk src/lib/utils.c

app src/lib $

可以看到，dirname app/Phone.apk src/lib/utils.c 命令逐行打印了所給兩個路徑名的目錄部分。

dirname utils.c 命令的路徑名參數沒有包含 / 字符，打印的目錄部分是點號 ‘.’。

dirname -z app/Phone.apk src/lib/utils.c 使用 -z 選項指定在獲取到的目錄部分後面不要加換行符，那麼打印結果都在同一行上，沒有換行。打印結果最後的 $ 字符是命令行提示符。

使用 basename 命令獲取路徑名的文件名部分

在 Linux 中，可以使用 basename 命令獲取路徑名的文件名部分，不包含文件名前面的目錄路徑。

查看 man basename 的說明如下：

basename NAME [SUFFIX]

basename OPTION... NAME...

basename - strip directory and suffix from filenames.

Print NAME with any leading directory components removed.

If specified, also remove a trailing SUFFIX.

即，basename 命令可以獲取所給路徑名的文件名部分，也就是從最後一個 ‘/’ 字符往後的部分，不包括 ‘/’ 字符自身。

當只提供一個路徑名參數時，後面可以提供一個可選的 SUFFIX 參數，該參數指定去掉文件名的後綴部分。

basename 默認只處理一個路徑名參數。

如果想要處理多個路徑名參數，需要使用 -a 選項，該選項的說明如下：

-a, --multiple

support multiple arguments and treat each as a NAME

當使用 -a 選項指定提供多個路徑名參數時，basename 會逐行打印每個路徑名的文件名部分。

如果不想換行打印，可以再使用 -z 選項，該選項的說明如下：

-z, --zero

separate output with NUL rather than newline

當提供多個路徑名參數時，如果想指定去掉文件名的後綴部分，需要用 -s 選項，該選項的說明如下：

-s, --suffix=SUFFIX

remove a trailing SUFFIX

具體舉例說明如下：

$ basename src/lib/utils.c

utils.c

$ basename src/lib/utils.c .c

utils

$ basename -s .c src/lib/utils.c

utils

$ basename -a src/lib/utils.c src/main.c

utils.c

main.c

$ basename -a -s .c src/lib/utils.c src/main.c

utils

main

$ basename -a -z src/lib/utils.c src/main.c

utils.c main.c $

可以看到，basename src/lib/utils.c 命令獲取到所給路徑名的文件名部分，也就是 utils.c。

basename src/lib/utils.c .c 命令指定從獲取到的文件名中去掉 .c 後綴，返回 utils。

而 basename -s .c src/lib/utils.c 命令通過 -s .c 指定從獲取到的文件名中去掉 .c 後綴，要在 -s 選項後面提供去掉的文件名後綴部分。

basename -a src/lib/utils.c src/main.c 命令使用 -a 選項指定處理多個路徑名，可以避免後面的文件名被當成 SUFFIX 參數，獲取到的多個文件名會逐行打印。

basename -a -s .c src/lib/utils.c src/main.c 命令通過 -a 選項指定處理多個路徑名，通過 -s .c 指定從獲取到的文件名中去掉 .c 後綴。

basename -a -z src/lib/utils.c src/main.c 命令通過 -a 選項指定處理多個路徑名，通過 -z 選項指定獲取到的文件名部分後面不要加換行符，那麼打印結果都在同一行上，沒有換行。打印結果最後的 $ 字符是命令行提示符。

使用 bash 的參數擴展獲取文件名的後綴名

在 bash 中，可以使用參數擴展（parameter expansion）表達式來獲取文件名的後綴名。

具體說明如下：

    ${filename##*.}

這個表達式在 filename 變量值中匹配 ‘.’ 這個字符，一直到最後一次匹配為止，然後返回該字符後面的部分，也就是文件名的後綴名，不包含 ‘.’ 這個字符。

注意：${filename##*.} 這個表達式是 bash 的參數擴展表達式，filename 會被當成變量名處理，獲取該變量值以便進行參數擴展。

它不能直接處理字符串，filename 不會被當成字符串處理，必須把字符串賦值給變量，然後把變量名放到該表達式中進行處理。

具體舉例如下：

$ echo ${src/lib/utils.c##*.}


$ filename="src/lib/utils.c"

$ echo ${filename##*.}

c

$ filename="utils.c"

$ echo ${filename##*.}

c

$ filename="utils.1.c"

$ echo ${filename##*.}

c

$ echo '.'${filename##*.}

.c

$ filename="util"

$ echo ${filename##*.}

util

可以看到，echo ${src/lib/utils.c##*.} 命令輸出為空。這裡沒有把 src/lib/utils.c 當成要處理的字符串，而是當成變量名。當前沒有這個變量，變量值為空，匹配結果也為空。

當把 "src/lib/utils.c" 賦值給 filename 變量後，echo ${filename##*.} 命令打印出 filename 變量值對應字符串的後綴名，也就是 c，沒有包含 . 這個字符。

修改 filename 變量值為 "utils.c"，echo ${filename##*.} 命令也打印了對應的後綴名。

把 filename 變量賦值為 "utils.1.c"，包含多個 ‘.’ 字符，${filename##*.} 表達式會一直匹配到最後一個 ‘.’ 字符，並返回該字符後面的部分。

由於這個表達式獲取的後綴名不包含 ‘.’ 字符，如果需要加上 ‘.’ 字符，可以在表達式前面主動加上這個字符。

例如，'.'${filename##*.} 這個寫法會在表達式返回值前面加上 ‘.’ 字符。

要注意的是，當所給變量值中沒有包含 ‘.’ 字符時，這個表達式會返回變量值自身。修改 filename 變量值為 "util"，echo ${filename##*.} 命令打印了這個變量值自身。

當 ${filename##*.} 返回的值等於 $filename 的值時，說明 filename 變量值不包含 ‘.’ 字符，也就是不帶有後綴名。
