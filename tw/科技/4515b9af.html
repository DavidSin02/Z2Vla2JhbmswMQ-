Uber 架構師 Ranny，重構了 Uber 的框架。

Uber 主要使用兩種語言javascript和python，其中 javascript 助我一臂之力進入編程行業。而且也是我個人無論在工作和業餘時間接觸最多的一門語言，所以我對於 javascript 有著特殊的感情。現在 javascript 已經成為了跨平臺的首選語言，瀏覽器端的彙編語言。

  • tchannel
    Ranny 說 Uber 已經從 HTTP+JSON 遷移到基於 TChannel 的 Thrift，TChannel 在 Node.js 中比 HTTP 快 20 倍。基於 RPC 進行服務間函數調用。並且屏蔽掉不同的語言的差異。
  • ringpop
    　Ringpop是為了服務Uber不斷擴大的用戶量而產生的一個分佈式架構，它能自動檢測和恢復單點失敗，並且實現負載均衡。一致性哈希(con.sistenthashing)：將節點映射到哈希環上，給節點分配任務，實現負載均衡。
  • google s2
    Google 的 S2 基於地理地圖數據庫，按四邊形對地圖進行切分。S2 能夠將球體分為小區 cell，每個小區有一個 id，地球大致是一個球形。S2 有兩個重要特性：能夠定義每個cell的分辨率，它能發現需要覆蓋區域的cell。

通過閱讀和整理資料今天給大家分享一下 Uber 的架構



一切從簡單開始，簡化一些 uber 服務就是將司機派遣給乘客。前面是手機，中間是 PHP 負責業務邏輯後面是MySQL 數據。每 4 秒上傳數據更新自己位置，來尋找客服進行匹配來支撐 uber 快速。不過要對應 Uber 業務極速的增長。



為了應對蜂擁而來增長

uber 將 PHP 變成多進程和多線程，同時訪問數據，多個線程同查數據庫，返給用戶。



  • 一個司機兩個顧客：因為每個 php 都是獨立訪問數據庫， 如果鎖的機制沒有做好，一個司機被兩個 PHP 進程同時搜索數據庫將司機派遣給兩個乘客。
  • 兩個司機一輛車 他們用兩個 app，而這個時候如果只能派遣一次就出問題了
  • 一個賬號兩輛車 兩個司機同乘一輛車，一個用戶兩輛車同時登錄一個賬號，用戶位置不斷漂移。


uber 如何應對挑戰



具體化一下，派遣服務後面 MySql 數據庫, 用戶可通過移動設備連接到派遣服務。
首先可以把一些影響派遣服務的邏輯拿出來，例如商業邏輯，用戶綁定手機號，收付款這些業務，商業邏輯直接走 Python API 來調用，從而減輕派遣服務壓力。



進一步加一個消息隊列，可以抵擋大量的請求。消息隊列抵擋壓力，同時將日誌和位置存在mangoDB 中。這樣分出幾層，大家還可以在各個層級如果出現問題就可以重試。


消息怎麼處理 需要消息管理獲得請求就處理消息。
熱升級代碼 hot push code 升級 request Manager 。不關掉服務，以前處理掉一半的消息怎麼辦。


談到改進，改進影響 的商業邏輯綁定手機號，Python API 派遣服務
在這個基礎將派遣服務變為 nodejs 地理位置將派遣數據庫變為 mangodb 。javascript 越來越全面。 不但可以寫前端也能寫服務。我也受益 JavaScript 才進去這一行。
