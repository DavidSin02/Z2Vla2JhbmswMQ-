輸入主題就能生成萬字文章？解密狗屁不通文章生成器！

2020-11-14
| 科技

寫在前面

前段時間，github上出現了一個很火的項目——BullshitGenerator，中文名是狗屁不通文章生成器，到現在已經有13.4k個star，用戶任意輸入一個主題詞，就能生成一篇6000字符的文章，而且看起來還頗有文采。當筆者點進這個項目時，發現項目作者用的方法出奇的簡單，利用幾十行python代碼就能達到如此神奇的效果，想到這裡大家一定躍躍欲試了。今天我們就來解密下這個狗屁不通文章生成器背後的原理，並告訴大家如何自定義自己的文章生成器。

項目描述

解密BullshitGenerator

點進項目主頁發現有10個文件，但是大家其實只需要看2個文件就可以了，一個是data.json，這是語料庫，也是文章廢話的來源；另一個是自動狗屁不通文章生成器.py，這是主文件，運行這個文件就能生成文章。我們先來看下這兩個文件的內容。

項目主頁

首先，單擊data.json，發現裡面的內容長下面這樣：

{"title":"學生會退會","famous":[ "愛迪生a，天才是百分之一的勤奮加百分之九十九的汗水。b", …… ],"bosh":["現在， 解決x的問題， 是非常非常重要的. 所以", …… ],"after":["這不禁令我深思. ", ……],"before":["曾經說過", ……]}

如果大家對python的基礎知識有了解，就會發現這其實就是一個詞典，裡面有五個鍵值對，分別對應主題、名人名言、廢話、名人名言前、後的廢話。大家如果想自定義自己的文章生成器，只需要改變這個data.json裡面的內容就好了。

然後，單擊狗屁不通文章生成器.py，這是個python腳本，裡面的代碼長這樣：

import os, reimport random,readJSONdata = readJSON.讀JSON文件("data.json")名人名言 = data["famous"] # a 代表前面墊話，b代表後面墊話前面墊話 = data["before"] # 在名人名言前面弄點廢話後面墊話 = data['after']  # 在名人名言後面弄點廢話廢話 = data['bosh'] # 代表文章主要廢話來源xx = "學生會退會"重複度 = 2def 洗牌遍歷(列表):    global 重複度    池 = list(列表) * 重複度    while True:        random.shuffle(池)        for 元素 in 池:            yield 元素下一句廢話 = 洗牌遍歷(廢話)下一句名人名言 = 洗牌遍歷(名人名言)def 來點名人名言():    global 下一句名人名言    xx = next(下一句名人名言)    xx = xx.replace(  "a",random.choice(前面墊話) )    xx = xx.replace(  "b",random.choice(後面墊話) )    return xxdef 另起一段():    xx = ". 粗略的瀏覽一遍，發現代碼的主體是三個函數，而且作者為了方便讀者理解，變量名和函數名直接用的中文，相信即使不會python的用戶，也能大概明白什麼意思。這三個函數分別是：洗牌遍歷，來點名人名言，另起一段。三個函數的功能是：

  • 洗牌遍歷：將語料庫中的句子打亂，並隨機生成一句廢話。
  • 來點名人名言：生成一句名人名言，並在名人名言前後墊兩句廢話。
  • 另起一段：輸入句號、換行符和段首縮進。

只需要隨機的排列組合這三個函數，就能生成一篇文采飛揚的文章啦！作者下面給出了使用範例，可以隨機生成一篇6000字的文章。大家只要修改main函數的代碼就能生成自己的文章啦！例如將main函數裡面的6000改為10000，就能生成一篇萬字長文啦！
