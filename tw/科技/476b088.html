從零開始學習比特幣開發：創建錢包重要方法之創建私鑰及公鑰

2020-10-29
| 科技

寫在前面

創建錢包和交易是比特幣最重要的兩方面，涉及到很多很多的內容，遠非一篇文章能概括的完。上一篇從整體上講解了錢包的創建流程，雖然已經很詳細了，但還是漏掉了幾個重要的方法，從本篇開始我們講解這幾個特別重要的方法。對錢包感興趣的朋友一定不要錯誤。

GenerateNewSeed

這個方法主要用來生成私鑰/公鑰，在生成公鑰後，調用錢包對象的 SetHDSeed方法，根據公鑰生成一個CHDChain對象，把公鑰經過 SHA256、RIPEMD160 兩次哈希返回後的 20個字節 180 位字符串作為它的seed_id屬性，從而以後可以衍生更的擴展公鑰。

void CWallet::SetHDSeed(const CPubKey& seed){    LOCK(cs_wallet);    CHDChain newHdChain;    newHdChain.nVersion = CanSupportFeature(FEATURE_HD_SPLIT) ? CHDChain::VERSION_HD_CHAIN_SPLIT : CHDChain::VERSION_HD_BASE;    newHdChain.seed_id = seed.GetID;    SetHDChain(newHdChain, false);}

在創建錢包的過程中，本方法及下面的 TopUpKeyPool方法，可能會被調用兩次，一次在第一次創建錢包時，另一次在用戶明確升級，且錢包支持 HD，但 HD 沒啟用的情況下。

下面，我們開始正式講解這個方法。

  1. 首先，生成並調用私鑰的 MakeNewKey來初始化私鑰。
    CKey key; key.MakeNewKey(true);
  2. 然後，調用 DeriveNewSeed方法，生成並返回私鑰對應的公鑰。
    return DeriveNewSeed(key);

我們先來看 MakeNewKey這個方法。這個方法比較簡單，但是非常重要，因為正是這個方法，生成了真正意義上的私鑰。

void CKey::MakeNewKey(bool fCompressedIn) {    do {        GetStrongRandBytes(keydata.data, keydata.size);    } while (!Check(keydata.data));    fValid = true;    fCompressed = fCompressedIn;}

GetStrongRandBytes方法，正是生成私鑰的過程。這個方法的執行邏輯如下：

  1. 生成一個 SHA512對象和一個無符號字符數組。
    CSHA512 hasher; unsigned char buf[64];
  2. 首先，調用 RandAddSeedPerfmon方法，通過 OpenSSL's RNG 生成隨機數，並進行 SHA512 哈希。
    RandAddSeedPerfmon; GetRandBytes(buf, 32); hasher.Write(buf, 32);
  3. 然後，調用 GetOSRand方法，通過 OS RNG 生成隨機數，並進行 SHA512 哈希。
    GetOSRand(buf); hasher.Write(buf, 32);
  4. 最後，如果 HW 可用，通過 HW 生成隨機數，並進行 SHA512 哈希。
    if (GetHWRand(buf)) { hasher.Write(buf, 32); }
  5. 接下來，合併並更新狀態。
    { WAIT_LOCK(cs_rng_state, lock); hasher.Write(rng_state, sizeof(rng_state)); hasher.Write((const unsigned char*)&rng_counter, sizeof(rng_counter)); ++rng_counter; hasher.Finalize(buf); memcpy(rng_state, buf + 32, 32); }
  6. 然後，把 buf數組中的內容拷貝到輸出參數中，並清空數組中的內容。
    memcpy(out, buf, num); memory_cleanse(buf, 64);

Check方法內部通過調用secp256k1_ec_seckey_verify方法，檢查私鑰的數據是否滿足要求，後者正是橢圓曲線算法相關的。

接下來，我們來看 DeriveNewSeed方法是如何生成公鑰的。方法執行邏輯如下：

  1. 生成當前時間，並用當前時間初始化蜜鑰元數據。
    int64_t nCreationTime = GetTime; CKeyMetadata metadata(nCreationTime);
  2. 調用私鑰的 GetPubKey方法，生成公鑰。方法內部正是通過橢圓曲線算法來生成私鑰對應的公鑰。代碼如下，有興趣的讀者可以自行研究。
    assert(fValid); secp256k1_pubkey pubkey; size_t clen = CPubKey::PUBLIC_KEY_SIZE; CPubKey result; int ret = secp256k1_ec_pubkey_create(secp256k1_context_sign, &pubkey, begin); assert(ret); secp256k1_ec_pubkey_serialize(secp256k1_context_sign, (unsigned char*)result.begin, &clen, &pubkey, fCompressed ? SECP256K1_EC_COMPRESSED : SECP256K1_EC_UNCOMPRESSED); assert(result.size == clen); assert(result.IsValid); return result;
  3. 執行 assert(key.VerifyPubKey(seed));方法，驗證公鑰確實有私鑰相匹配。
  4. 設置蜜鑰元數據的兩個屬性。
    metadata.hdKeypath = "s"; metadata.hd_seed_id = seed.GetID;
    GetID方法，用公鑰的數據經過 SHA256、RIPEMD160 兩次哈希後生成的字符串來生成一個CKeyID對象。
  5. 把密鑰元數據保存在 mapKeyMetadata集合中。
    mapKeyMetadata[seed.GetID()] = metadata;
  6. 調用 AddKeyPubKey方法，把私鑰/公鑰及元數據保存到數據庫。如果出錯，拋出一個異常。
    if (!AddKeyPubKey(key, seed)) throw std::runtime_error(std::string(__func__) + ": AddKeyPubKey failed");
    AddKeyPubKey方法生成一個訪問數據庫的對象，然後調用錢包對象的AddKeyPubKeyWithDB方法來保存密鑰數據。後者的執行流程如下：
    • 如果變量 encrypted_batch為空，那麼設置變量needsDB為真。如果後者為真，那麼設置encrypted_batch屬性為參數指定的對象。
      bool needsDB = !encrypted_batch; if (needsDB) { encrypted_batch = &batch; }
    • 調用 CCryptoKeyStore::AddKeyPubKey方法，保存私鑰和公鑰。如果不成功，進一步，如果變量needsDB為真，則設置變量encrypted_batch為假。
      if (!CCryptoKeyStore::AddKeyPubKey(secret, pubkey)) { if (needsDB) encrypted_batch = nullptr; return false; }
      CCryptoKeyStore::AddKeyPubKey方法內部執行邏輯如下：
      • 如果沒有加密，那麼調用 CBasicKeyStore::AddKeyPubKey來保存私鑰和公鑰，然後返回。
        if (!IsCrypted) { return CBasicKeyStore::AddKeyPubKey(key, pubkey); }
        IsCrypted方法，初始化時是沒有加密的，只有在執行加鎖、解鎖、添加加密密鑰的情況下才會是加密的。當前情景是沒有加密的，所以執行基類的方法來保存私鑰和公鑰。

        CBasicKeyStore::AddKeyPubKey方法，首先把私鑰保存在mapKeys集合中，然後調用ImplicitlyLearnRelatedKeyScripts方法來處理腳本，因為公鑰默認是壓縮的，所以在方法內會生成腳本，並把腳本保存在mapScripts集合中。

        bool CBasicKeyStore::AddKeyPubKey(const CKey& key, const CPubKey &pubkey) { LOCK(cs_KeyStore); mapKeys[pubkey.GetID()] = key; ImplicitlyLearnRelatedKeyScripts(pubkey); return true; }
      • 如果是鎖定的，那麼返回假。
        if (IsLocked) { return false; }
        IsLocked方法，首先調用IsCrypted方法，確定是否是加密的，如果不是加密的，則直接返回假；否則，把vMasterKey集合清空。
        bool CCryptoKeyStore::IsLocked const { if (!IsCrypted) { return false; } LOCK(cs_KeyStore); return vMasterKey.empty; }
      • 接下來，生成加密私鑰。
        std::vector<unsigned char> vchCryptedSecret; CKeyingMaterial vchSecret(key.begin, key.end); if (!EncryptSecret(vMasterKey, vchSecret, pubkey.GetHash, vchCryptedSecret)) { return false; }
      • 最後，調用 AddCryptedKey方法，保存加密私鑰，並返回
        if (!AddCryptedKey(pubkey, vchCryptedSecret)) { return false; } return true;
        AddCryptedKey方法，首先確定是否是加密的，如果沒有加密則返回假。接下來，把私鑰保存在mapCryptedKeys集合中，然後調用ImplicitlyLearnRelatedKeyScripts方法來處理腳本，因為公鑰默認是壓縮的，所以在方法內會生成腳本，並把腳本保存在mapScripts集合中。
        bool CCryptoKeyStore::AddCryptedKey(const CPubKey &vchPubKey, const std::vector<unsigned char> &vchCryptedSecret) { LOCK(cs_KeyStore); if (!SetCrypted) { return false; } mapCryptedKeys[vchPubKey.GetID()] = make_pair(vchPubKey, vchCryptedSecret); ImplicitlyLearnRelatedKeyScripts(vchPubKey); return true; }
    • 如果變量 needsDB為真，則設置變量encrypted_batch為空指針。
    • 調用 GetScriptForDestination方法，獲得公鑰對應的腳本。
      CScript script; script = GetScriptForDestination(pubkey.GetID);
    • 調用 HaveWatchOnly方法，檢查setWatchOnly集合中是否有這個腳本。如果有，則調用RemoveWatchOnly方法，清除公鑰對應的腳本。
      if (HaveWatchOnly(script)) { RemoveWatchOnly(script); }
      RemoveWatchOnly方法執行邏輯如下：
      • 調用 CCryptoKeyStore::RemoveWatchOnly方法來移除腳本。方法內部執行邏輯如下：從setWatchOnly集合中移除對應的腳本；然後，調用ExtractPubKey方法，通過腳本來解析出公鑰，如果可以得到公鑰，則從mapWatchKeys集合中移除對應的公鑰；最後，返回真。
      • 調用數據庫訪問對象的 EraseWatchOnly方法，從數據庫中移除watchmeta和watchs對應的數據。
      • 返回真。
    • 調用 GetScriptForRawPubKey方法，從原始公鑰獲得腳本。
      script = GetScriptForRawPubKey(pubkey);
    • 調用 HaveWatchOnly方法，檢查setWatchOnly集合中是否有這個腳本。如果有，則調用RemoveWatchOnly方法，清除公鑰對應的腳本。方法執行邏輯如上所述，這裡不講。
    • 如果不是加密的，則調用數據庫的訪問對象的 WriteKey方法，把私鑰和公鑰及密鑰元數據寫入數據庫。在這個方法中，首先，以keymeta為鍵，保存對應的元數據；然後，把公鑰/私鑰保存到向量中，以key為鍵，保存對應的公鑰/私鑰。
    • 最後，返回真。
