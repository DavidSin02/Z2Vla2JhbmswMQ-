C語言數據類型

在嵌入式Linux開發《C語言專題(一：不一樣的hello world)》文章中展示了一個最簡單同時又是經典的C代碼例子-hello world.c，通過這個代碼來對C語言有一個整體的認識。這篇文章主要寫C語言中的基本數據類型。

基本數據類型有整型、字符型和浮點型。涉及到short、int、long、char、float、double、signed和unsigne這幾個C語言中的關鍵字。

下面測試一下基本類型在內存中所佔的內存空間大小：

#include <stdio.h>int main(int argc, char** argv){	 printf(" int長度：%d\n short長度：%d\n long長度：%d\n char長度：%d\n float長度：%d\n double長度：%d\n ",sizeof(int),sizeof(short),sizeof(long),		 sizeof(char),sizeof(float),sizeof(double));			return 0;	}

顯示基本數據類型長度

從中可以看到int和long佔4個字節，short佔2個字節，char佔一個字節，float佔4個字節，double佔8個字節。注意這是在32位linux系統中的結果。為什麼要測試類型的長度呢？一方面是為以後定義變量做出合適的選擇，不要讓變量的值超出了變量類型的範圍；另一方面是面試，有些題目直接問int類型所佔的內存空間，不要一口咬定是4個字節，因為在不同的系統中int的長度真不一樣，自己可以試試，

上面的代碼還有需要注意的地方：printf()函數中可以輸出變量的值，類似於printf("a=%d b=%d",a,b);直接將雙引號中的2個%d分別替換成變量a和b的值（前提定義變量a和b為整型：int a=10,b=10;）sizeof是關鍵字不是函數，它可以用來計算類型所佔的存儲空間，比如我們要向內存中申請20個int類型的空間，不要直接寫上malloc(4*20);而應該寫成malloc(sizeof(int)*20) 因為不同的系統中int類型長度未必是4。還有一種用法就是給你一個一維數組名array，計算出此數組中有幾個元素？答案是：sizeof(array)/sizeof(array[0]) 這裡先提一下數組，後續詳細講解。

（1）整型：整型中有短整型(short) 整型 (int) 長整型(long) 都分為有符號和無符號兩種版本。類似於數學中的整數。規定長整型至少和整型一樣長，而整型至少和短整型一樣長。C語言的整型與數學中的整數一樣，用來表示整型值。整型有正數、也有負數對應C語言中的有符號整型數。整型變量的定義為：int a;或者定義並初始化int a=0; int可以換為short、long或者加上unsigned和signed.考慮到頭條中直接放代碼容易亂，我決定放圖片，如下圖顯示：注意看代碼和註釋，自己測試測試

%d就是普通的輸出；%md 將數字按寬度為m，採用右對齊方式輸出，若數據位數不到m位，則左邊補空格，大於或者等於m位正常輸出；%.md與 %0md一樣，採用右對齊方式輸出，若數據位數不到m位，則左邊補0，大於或者等於m位正常輸出；

%d及其變形的以上用法在格式化時間以及在屏幕的規定位置顯示數據的時候比較有用。

（2）字符型：字符型對應ASCII字符。ASCII字符是一種編碼，就是用數字編碼來表示一個符號。字符的本質還是整形，所以可以和整型進行算數運算，只是這些整形數被用來表示為一些字符的ASCII編碼值，所以稱作字符型。字符型一般用一個字節：即8位二進制表示。測試代碼和結果如下圖所示：
