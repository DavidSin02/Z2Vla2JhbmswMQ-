2020年，圖機器學習將走向何方？

2020-10-29
| 科技

選自towardsdatascience

作者：Sergei Ivanov

機器之心編譯

參與：魔王、杜偉

2020 年已經過去了一個多月，但我們已經可以從最近的研究論文中一窺圖機器學習（Graph Machine Learning，GML）的趨勢。機器學習研究科學家 Sergei Ivanov 對 2020 年 GML 的發展趨勢發表了自己的看法，並討論了近期的相關研究論文。

本文的寫作目的不是介紹 GML 的基礎概念，如圖神經網絡（GNN），而是推介頂會前沿研究。作者簡要總結了學術頂會 ICLR 2020 提交論文中圖論文的統計概況：

共有150 篇 GML 研究提交至此次 ICLR 會議，接收率為 1/3，約佔全部接收論文的 10%。


作者閱讀了其中的大部分論文，並列出了他對 2020 年 GML 研究趨勢的判斷：

  1. 對 GNN 有更堅實的理論理解；
  2. 出現新的 GNN 應用；
  3. 知識圖譜更加流行；
  4. 新的圖嵌入框架誕生。

本文作者、機器學習研究科學家 Sergei Ivanov。

現在我們來一一探究（以下以第一人稱視角講述）。

對 GNN 有更堅實的理論理解

我尤其對這一趨勢感到振奮，因為它表明 GML 領域的成熟，先前的啟發式方法正在被新的理論解決方案取代。要想理解圖神經網絡，還有很長的路要走，但關於 GNN 的工作原理已經出現多項重要研究成果。

首先來看我最喜歡的一項研究：Andreas Loukas 的論文《What graph neural networks cannot learn: depth vs width》。這篇論文在技術簡潔性、強大的實踐影響力和意義深遠的理論見解三者之間實現了完美的平衡。

該研究證明，如果我們想讓 GNN 為常見的圖問題（如環檢測、直徑估計、頂點覆蓋等）提供解決方案，則節點嵌入的維度（即網絡寬度 w）與層數（即網絡深度 d）的乘積應與圖大小 n 成正比，即 dw = O(n)。


目前很多 GNN 實現無法達到該條件，因為層數（很多實現中的層數大約為 2–5）和嵌入維度（約 100–1000）與圖大小相差甚遠。另一方面，在當前設置下大型網絡的代價高昂，這就引出了一個問題：如何設計高效 GNN？這是我們未來需要解決的任務。

另外，這篇論文從上世紀 80 年代的分佈式計算模型中汲取靈感，充分證明 GNN 實際上可以做到同樣的事。論文列出了更多結果，推薦大家閱讀。

類似的還有以下兩篇論文，這兩篇文章的主題是 GNN 的力量。

第一篇論文《Graph Neural Networks Exponentially Lose Expressive Power for Node Classification》證明：

在特定權重條件下，當層數增加時，受拉普拉斯譜的影響，GCN 只能學習節點度（node degree）和連通分支（connected component）。


該結果是馬爾可夫過程收斂至唯一均衡這一著名特性的泛化，其中收斂速率由轉移矩陣的特徵值決定。

論文鏈接：https://openreview.net/pdf?id=S1ldO2EFPr

第二篇論文《The Logical Expressiveness of Graph Neural Networks》展示了 GNN 和多種節點分類器之間的聯繫。我們已經知道一些 GNN 和同構的 WL test 一樣強大，即當且僅當兩個節點被 GNN 判斷為同一類別時，WL 賦予二者相同的顏色。

那麼 GNN 具備其他分類功能嗎？例如，當且僅當圖具備孤立頂點時，布爾函數向所有節點分配真值。GNN 具備這樣的邏輯嗎？直觀來看不具備，因為 GNN 使用消息傳遞機制，當圖的兩個部分沒有鏈接時（即兩個連通分量，connected component），二者之間不會出現消息傳遞。

該研究提出了一種簡單的修復方法：在鄰域聚合之後添加 readout 函數，這樣當每個節點更新所有特徵時，它具備圖中其他節點的信息。

論文鏈接：https://openreview.net/pdf?id=r1lZ7AEKvB

其他理論研究還包括：衡量 GNN 中圖信息的使用（Hou 等人的論文《Measuring and Improving the Use of Graph Information in Graph Neural Networks》），以及基於角色和基於距離的節點嵌入的等價性（Srinivasan 和 Ribeiro 的論文《On the Equivalence between Positional Node Embeddings and Structural Graph Representations》）。

GNN 的新應用

看到 GNN 被應用於現實任務也很令人興奮。2020 年 GNN 將應用於修復 Javascript 中的 bug、玩遊戲、回答 IQ 類測試題、優化 TensorFlow 計算圖、分子生成，以及對話系統中的問題生成。

Dinella 等人的研究《HOPPITY: Learning Graph Transformations to Detect and Fix Bugs in Programs》展示了一種檢測出 Javascript 代碼中的 bug 並同步修復的新方法：先將代碼轉換成抽象語法樹，然後用 GNN 執行預處理得到代碼嵌入。其思路是給出一張初始狀態的圖，通過多輪圖編輯操作（添加或刪除節點、替換節點值或節點類型）進行修改。

為了瞭解哪些圖節點需要修改，研究者使用指針網絡（Pointer network），該網絡以圖嵌入和當前編輯歷史為輸入，然後選擇節點。接下來，使用 LSTM 網絡執行修復工作，LSTM 網絡也以圖嵌入和編輯歷史為輸入。論文作者基於 GitHub commit 驗證了該方法，證明其相比其他通用性較差的基線方法有顯著提升。

論文鏈接：https://openreview.net/pdf?id=SJeqs6EFvB

類似的研究還有 Wei 等人的《LambdaNet: Probabilistic Type Inference using Graph Neural Networks》，它主要探討如何為 Python 或 TypeScript 等語言推斷變量類型。

作者展示了一種類型依賴超圖（type dependency hypergraph），圖節點是程序變量，該圖還編碼了變量之間的關係，如邏輯約束（布爾類型）或語境約束（如相似的變量名稱）。先訓練一個 GNN 模型為圖中的變量和可能的變量類型生成嵌入，然後利用嵌入預測具備最高似然的類型。實驗結果表明，LambdaNet 在標準變量類型（如布爾類型）和用戶自定義類型中均獲得了更好的性能。

論文鏈接：https://openreview.net/pdf?id=Hkx6hANtwH

Wang 等人的研究《Abstract Diagrammatic Reasoning with Multiplex Graph Networks》展示瞭如何使用 GNN 在 IQ 測試中進行推理。在瑞文推理測驗（Raven Progressive Matrices，RPM）任務中，研究者為矩陣的每一行構建一個圖（其中邊嵌入通過前饋模型獲得），然後執行圖摘要。由於最後一行有 8 個可能答案，因此創建了 8 個不同的圖，每個圖和前兩行連接，然後通過 ResNet 模型得到 IQ 分數預測結果。

論文鏈接：https://openreview.net/pdf?id=ByxQB1BKwH

圖源：論文《Abstract Diagrammatic Reasoning with Multiplex Graph Networks》

DeepMind 的論文《Reinforced Genetic Algorithm Learning for Optimizing Computation Graphs》提出一種強化學習算法，用來優化 TensorFlow 計算圖的成本。首先通過標準的消息傳遞 GNN 處理圖，生成的離散嵌入對應圖中每個節點的調度優先級。然後將這些嵌入傳輸到遺傳算法 BRKGA 中，由該算法決定每個節點的佈局和調度（placement and scheduling）。該模型用於優化 TensorFlow 圖的真實計算成本。


圖源：論文《Reinforced Genetic Algorithm Learning for Optimizing Computation Graphs》

GNN 的其他有趣應用還包括分子生成（論文《GraphAF: a Flow-based Autoregressive Model for Molecular Graph Generation》）、玩遊戲（論文《Graph Convolutional Reinforcement Learning》）和對話系統（論文《Reinforcement Learning Based Graph-to-Sequence Model for Natural Question Generation》）。

以上三篇論文的鏈接分別如下：

  • https://openreview.net/pdf?id=S1esMkHYPr
  • https://openreview.net/pdf?id=HkxdQkSYDB
  • https://openreview.net/pdf?id=HygnDhEtvr

知識圖譜更加流行

這一年湧現出大量有關知識圖譜的論文。知識圖譜本質上是一種表示事實的結構化方式。與一般的圖不同，知識圖譜的節點和邊均包含意義，如演員姓名或參演電影（參見下圖）。知識圖譜的常見任務是回答複雜的 query，如「2000 年之前，史蒂文·斯皮爾伯格憑藉哪部電影獲得奧斯卡獎？」，這個問題可以轉換成邏輯 query：∨ {Win(Oscar, V) ∧ Directed(Spielberg, V) ∧ ProducedBefore(2000, V) }。


知識圖譜示例。（圖源：論文《A Review of Relational Machine Learning for Knowledge Graphs》）

Ren 等人的論文《Query2box: Reasoning over Knowledge Graphs in Vector Space Using Box Embeddings》提出，將 query 作為矩形框嵌入潛在空間，而不是作為一個點。該方法允許執行自然交運算（即合取 ∧），藉助它可以得到新的矩形框。

但是，建模並運算（即析取 ∨）就沒有那麼直接了，因為它可能導致非重疊區域。此外，要想精確建模任意 query 的嵌入，嵌入之間距離函數的複雜度（由 VC 維度衡量）應與圖中實體數成正比。不過，有一個 trick 可以將析取式 query 替換為 DNF，並運算（union）只在計算圖的最後一步中出現，因此我們只需對每個子查詢執行簡單的距離計算即可。

論文鏈接：https://openreview.net/pdf?id=BJgr4kSFDS


圖源：論文《Query2box: Reasoning over Knowledge Graphs in Vector Space Using Box Embeddings》

在類似的主題中，Wang 等人的論文《Differentiable learning of numerical rules in knowledge graphs》提出了一種處理數值實體和規則的方式。

例如，引用知識圖譜具備規則 influences(Y,X) ← colleagueOf(Z,Y) ∧ supervisorOf(Z,X)∧ hasCitation>(Y,Z)，該規則表示學生 X 受到導師 Z 的同事 Y 的影響，Y 的論文引用量更多。規則右側的每一個關係都可以表示為矩陣，尋找缺失鏈接的過程可以形式為關係的連續矩陣與實體向量相乘，該過程又叫規則學習（Rule Learning）。由於矩陣構建方式的緣故，神經方法僅能處理類別規則，如 colleagueOf(Z,Y)。

該論文的貢獻是：提出了一種高效處理數值規則（如 hasCitation>(Y,Z)）和否定運算符的新方法，該方法證明在現實中沒必要將此類矩陣顯式具體化，從而極大地減少了運行時間。

論文鏈接：https://openreview.net/pdf?id=rJleKgrKwS

在機器學習和 GML 領域中出現頻率更高的另一個主題是重新評估現有模型及其在公平環境中的性能。Ruffinelli 等人的論文《You CAN Teach an Old Dog New Tricks! On Training Knowledge Graph Embeddings》證明，新模型的性能通常依賴於實驗訓練的「小」細節，如損失函數、正則化項和採樣機制的形式。在大型控制變量實驗中，作者觀察到舊有方法（如 RESCAL 模型）在適當調參後可以達到 SOTA 性能。

論文鏈接：https://openreview.net/pdf?id=BkxSmlBFvr

該領域還有很多有趣的研究工作。

Allen 等人的論文《On Understanding Knowledge Graph Representation》基於近期對詞嵌入的見解來更多地理解學得關係和實體表示的潛在空間。

論文鏈接：https://openreview.net/pdf?id=SygcSlHFvS

Asai 等人的論文《Learning to Retrieve Reasoning Paths over Wikipedia Graph for Question Answering》展示了，模型如何基於能夠回答給定 query 的 Wikipedia 圖來檢索推理路徑。

論文鏈接：https://openreview.net/pdf?id=SJgVHkrYDH

Tabacof 和 Costabello 的論文《Probability Calibration for Knowledge Graph Embedding Models》涉及一個重要主題：圖嵌入模型的概率校準（probability calibration）。該研究證明，使用 sigmoid 函數將 logits 轉換為概率的常用嵌入模型 TransE 和 ComplEx 並未得到很好地校準，即事實出現的概率被低估或高估了。該研究提出首先生成受損三元組（corrupted triple），並將其作為負例，然後利用 Platt scaling 方法（一種將模型輸出轉化為基於類別的概率分佈的方法）和保序迴歸方法來校準概率。

論文鏈接：https://openreview.net/pdf?id=S1g8K1BFwS

新的圖嵌入框架

圖嵌入是圖機器學習領域中的老話題了，今年關於如何學習圖表示出現了一些新的觀點。

Deng 等人的論文《GraphZoom: A Multi-level Spectral Approach for Accurate and Scalable Graph Embedding》提出一種新方法，可以為任意無監督嵌入方法縮短運行時間、提升節點分類準確率。其整體思路是：先將原始圖縮小，以便快速計算節點嵌入，然後恢復原始圖的嵌入。

先使用額外邊增強原始圖，這些邊對應基於屬性相似度得到的節點 k 最近鄰之間的鏈接。然後把圖粗糙化：通過局部譜方法將每個節點投影到更低維空間中，並聚合為簇。使用任意無監督圖嵌入方法（如 DeepWalk 或 Deep Graph Infomax）均可獲得縮小圖的節點嵌入。最後，使用平滑算子將獲得的節點嵌入（它們本質上表示簇嵌入）迭代地廣播回去，以防止不同節點具備相同嵌入。

實驗證明，GraphZoom 框架比 node2vec 方法提速 40 倍，準確率比 DeepWalk 方法高出 10%。

論文鏈接：https://openreview.net/pdf?id=r1lGO0EKDH


圖源：論文《GraphZoom: A Multi-level Spectral Approach for Accurate and Scalable Graph Embedding》

多篇論文檢查了圖分類問題的之前結果。Errica 等人的論文《A Fair Comparison of Graph Neural Networks for Graph Classification》就該問題對 GNN 模型進行了公平的重新評估，證明未使用圖拓撲（即僅利用聚合節點特徵）的簡單基線方法的性能與 SOTA GNN 無異。（早在 2015 年 Orlova 等人就發表過這一驚人的現象，但當時並未吸引大量關注。）

該研究的傑出成果包括：提供了模型在常用數據集上的公平 SOTA 結果和 PyTorch-Geometric 代碼，方便以後的論文進行模型對比。

論文鏈接：https://openreview.net/pdf?id=HygDF6NFPB

我們的研究《Understanding Isomorphism Bias in Graph Data Sets》還發現，在常用數據集（如 MUTAG 或 IMDB）中，大量圖擁有同構複印件，甚至節點屬性也是同構的。此外，在這些同構圖中，很多圖的目標標籤不同，從而給分類器引入了標籤噪聲。這表明使用網絡的所有可用元信息（如節點屬性或邊屬性）對提高模型性能的重要性。

論文鏈接：https://openreview.net/pdf?id=rJlUhhVYvS

另一項研究《Are Powerful Graph Neural Nets Necessary? A Dissection on Graph Classification》證明，如果將非線性鄰域聚合函數替換為其線性版本（包含鄰域度和傳播得到的圖屬性），模型性能不會下降。這與之前的說法一致，即很多圖數據集對於圖分類無關緊要，這就引出了一個問題：對該任務提供恰當的驗證框架。

論文鏈接：https://openreview.net/pdf?id=BJxQxeBYwH

結論

隨著頂會提交論文的增長，預計 2020 年 GML 領域將出現很多有趣的研究。我們已經看到該領域從圖深度學習的啟發式應用轉向了更合理的方法和關於圖模型範疇的基礎性問題。GNN 成為很多實際問題的解決方案，不過我認為 GML 才剛剛觸及到圖理論和機器學習交叉領域的皮毛，我們應該更多地關注即將出現的研究成果。
