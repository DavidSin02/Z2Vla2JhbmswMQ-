如何通過向量化78倍速加快您的機器學習算法

2020-10-29
| 科技

給定一個方程式，我們將逐步瞭解如何才能不僅在速度上獲得x78倍的效率更高的代碼，而且僅使用3行代碼！ 讓我們開始吧……

介紹

作為一種解釋型語言，Python for loop本質上比C語言慢。 這對於編程語言是一個很大的瓶頸，因為深度學習和機器學習算法嚴重依賴於矩陣運算，而矩陣運算是通過for循環執行的。

這就是開發人員開發軟件包的原因，例如numpy，他們在numpy數組上提供矢量化操作。 這意味著它將通常在Python中完成的for循環下推到C級，這要快得多。

Python + C級速度=天堂

問題

（如果您對EM算法有疑問，可以跳過解釋部分）

我們想對無監督學習任務使用期望最大化（EM）算法（例如，在MNIST數據集中識別手寫數字），並且我們的數據是二進制的（例如二進制圖像）。 一種自然的方法是將我們的數據建模為伯努利混合模型。 伯努利分佈的加權和，每個分佈都有自己的標量權重π和自己的均值向量μ並表示數據的群集（例如，如果我們的數據是數字2、3和4的圖像，並且我們使用3伯努利進行建模 它們中，一個伯努利將是數字2，另一個是數字4，依此類推）。 總的來說，這使得前者成為向量，而後者成為矩陣。

Bernoulli mixture model (1)


Distribution of one observation x given the cluster k (2)

令N =觀測數，D =一個觀測的維數，K =聚類數。 因為對於我們的問題很重要，所以我們擁有的隨機變量的類型為：

  • X； 我們的數據是一個NxD矩陣（N個圖像數量，D個維數→5張28 * 28的圖像將形成5x784矩陣X）
  • π； 向量K，代表每個分佈的權重的標量。（例如，三個伯努利可以具有π= [0.2，0.75，0.05]加權向量）
  • μ； 每個群集的平均KxD矩陣。（一個圖像的維數為D = 28 * 28 = 784，其中每個圖像代表一個像素值。取屬於同一群集的圖像的每個像素的平均值，例如數字2 ，我們得出的平均向量為784。因此，μ將是KxD的矩陣）

在E步中，我們對潛在變量後驗的期望值或所謂的責任特別感興趣。

E-step of EM algorithm (3)


γ實際上返回屬於聚類k的觀測值（圖像）n的期望值。 γ是NxK矩陣； 對於每個觀察，我們分配屬於每個聚類的概率。 具有最大值的一個就是我們分配給的那個。

為什麼我要告訴所有這些？

"向量化中最重要的事情是瞭解變量的維度。"

責任的計算是我們要向量化的一項

總結：

  • X：NxD矩陣
  • π：1xK向量
  • μ：KxD矩陣
  • γ：NxK矩陣

管道

我們將創建一個函數E_step來運行計算上面的表達式並使用以下代碼對其進行測試

observations = [5, 10, 20, 50, 100, 200, 500, 1000]for n in observations:    X_test = bin_train_data[:n]    D_test, K_test = X_test.shape[1], 10    mu_test = np.random.uniform(low=.25, high=.75,                                 size=(K_test,D_test))    pi_test = np.ones(K_test) / K_test    t0 = time.time()    gamma_test = E_step_1(X_test, mu_test, pi_test)    runtime = time.time() - t0    assert gamma_test.shape == (n, K_test)


隨意先嚐試一下！

嘗試№1

第一次嘗試時，我們將使用for循環編寫所有內容； 在矢量/矩陣運算中，僅標量。

通過查看方程式，我們可以看到存在3個循環。 每個示例N一個，每個群集K一個，每個對象D每個維一個，我們將按此順序循環。 因此，我們將一次用一個元素填充矩陣γ。

def E_step(X, mu, pi):    N, D = X.shape    K = pi.shape[0]    gamma = np.zeros((N, K))    for n in range(N):        for k in range(K):            m = 1            for i in range(D):                m *= mu[k][i]**X[n][i] * (1-mu[k][i])**(1-X[n][i])            gamma[n][k] = m * pi[k]        gamma[n] /= gamma[n].sum()    return gamma

我們肯定可以做得更好！

嘗試№2

最好從內部循環開始，然後逐步發展到外部循環。 而這正是我們要做的！

我們要擺脫for循環D。 因此，每個依賴於D的項現在都應成為向量。 在此for循環中，我們有兩個變量； μ和x（請參閱等式（2））。 因此x和μ→向量。 問題; 它是μ** x，是另一個矢量的冪的矢量，很難計算。 如果我們能解決這個問題……

有一個函數可以將冪運算轉換為乘法。 是的，是對數！ 因此，讓我們對表達式取對數，然後得出結果的指數！

首選對數概率運算，因為它們提供數值穩定性

即使在我們的情況下它沒有任何影響，但是每次使用日誌時，請在表達式內部使用常量epsilon以保持穩定性（使用-inf時不要為零）。

因此，我們將不得不逐元素矢量相乘。 簡單 ;）

def E_step(X, mu, pi):    N, D = X.shape    K = pi.shape[0]    gamma = np.zeros((N, K))    for n in range(N):        for k in range(K):            log_gamma = np.log(pi[k]) + (X[n] * np.log(mu[k]) \                        + (1 - X[n])*np.log(1 - mu[k])).sum()            gamma[n][k] = np.exp(log_gamma)        gamma[n] /= gamma[n].sum()    return gamma

那是巨大的勝利！ 與Algor.1相比，x軸看起來更像1！ 但是，我們可以做得更好;）

嘗試№3

一次循環一圈：K圈！

在矢量化過程中，我們正在朝以下方向發展：

標量→向量→矩陣

隨著我們用numpy數組替換越來越多的循環，越來越多的代碼將在C→更快更乾淨的代碼上運行。

我們採用之前的實現，並且想要刪除K for循環。 因此，每個依賴於K的標量都將變成一個向量，而每個向量都將變成一個矩陣。 這意味著X將保持不變，而μ將變成矩陣，而π和γ將成為向量。 注意最後一個； 隨著γ字段的逐行傳播，我們表達式的結果現在必須是向量！ 因此，μ和X的運算必須產生1xK向量，快速指示符是（i）它們必須與向量π相加，這也是1xK（ii）結果是矩陣γ的一行，也是1xK 向量。

得出的結果是：

def E_step(X, mu, pi):    N, D = X.shape    K = pi.shape[0]    gamma = np.zeros((N, K))    for n in range(N):        log_gamma = np.log(pi) + np.log(mu) @ X[n] \                    + np.log(1 - mu) @ (1 - X[n])        gamma[n] = np.exp(log_gamma)        gamma[n] /= gamma[n].sum()    return gamma

驚人！ 在n = 1000的情況下，我們成功完成了一半的時間！ 與Algor.1確實沒有可比性。 但是，我們可以做得更好嗎？

嘗試№4

我們還有一個循環。 我們可以有一個無Python的計算循環嗎？ N，你的時間到了！

在將矩陣*向量運算轉換為矩陣@矩陣運算時，我們需要採用前者的轉置矩陣（@是常規矩陣乘法）。 請記住，現在我們的輸出必須是整個γ矩陣。 我認為到現在為止，您對它的運行方式有了想法；）。

所以我們的代碼是

def E_step(X, mu, pi):    gamma = np.exp(np.log(pi) + X @ np.log(mu.T) \            + (1 - X) @ np.log(1 - mu.T))        gamma /= gamma.sum(axis=1)[:, np.newaxis]    return gamma


沒有一個循環！ 該代碼看起來很優雅，只有三行！ 現在要獲得結果，鼓掌吧…

就是這樣，再也無法得到任何改善！ 對於n = 1000，我們僅使用三行代碼就從11.688→0.012的運行時開始運行！

摘要

因此，當您要對表達式進行向量化時，您需要做什麼：

  • 瞭解矩陣的尺寸。
  • 筆和紙：寫下公式，從求和到求和，然後將其轉換為等效的矩陣運算。 總是要考慮任何表達式必須返回的維數； 觀察鄰居求和操作，因為它們的維數相同
  • 逐個循環，逐步：標量→向量→矩陣
  • 取對數並確保引入歸一化常數epsilon
  • 對方法的向量化版本進行編碼，並進行光澤：D
