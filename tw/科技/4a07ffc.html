一、適配器模式的定義

適配器模式解決類的不兼容問題，在一些場景下，接口定義的與接口使用方可能存在不一致的地方，但是又需要複用這些接口，就需要就行接口的轉換，自己定義一個相同功能的接口作為轉化。讓實現類同時擁有多個相同方法的接口，這樣就可以達到兼容的目的。

適配模式分為三種形式的適配：類適配器、對象適配器、接口適配器

適配器模式

二、3種適配器示例

1.類適配器

類適配器顧名思義就是適配一個類，我們示例分為以下流程

1） 創建待適配的對象，我此我們創建一個Peple對象

public class People {
public void drive(){
System.out.println("這個人在開車~");
}
}

對象有一個行為，駕駛

2） 創建創建需要適配的接口Action

public interface Action { /** * 會開火車 */ void drive(); /** * 可以走路 */ void walk(); /** * 還可以亂跑 */ void run();}

drive()方法是需要與people適配的

3） 創建適配器

/*** * @description :  * 類適配器， people是待適配的對象， action是適配的目標， * 這樣就將待適配的對象擴展了帶有目標的功能 * 目標也有了待適配對象的功能 * 通常在源於目標有重疊功能時 */public class ClassAdapter extends People implements Action { @Override public void walk() { System.out.println("這個人在步行"); } @Override public void run() { System.out.println("這個人在跑步~"); }}

通過以上適配的過程，Action擁有了People的dive ()行為，很多時候還有不同的表示方式，在此沒有在ClassAdapter的dive（）做任何的修飾，當然也可以重寫，改變其行為模式。

2.對象適配器模式

類適配採用繼承的方式實現，而對象適配採用成員變量的方式實現，如果不加任何修飾，有點像靜態代理模式，如果加以修飾，就變成了裝飾模式，我們來看看具體的代碼，我們仍然用類適配器中的People對象與Action接口

public class ObjectAdapter implements Action { private People people; public ObjectAdapter(People people){ this.people = people; } @Override public void drive() { people.drive(); } @Override public void walk() { System.out.println("具有了行走了能力"); } @Override public void run() { System.out.println("具有跑的能力"); } public static void main(String[] args) { Action action = new ObjectAdapter(new People()); action.walk(); action.run(); action.drive(); }}

我們把Action的drive()行為直接代理到People成為去實現，達到了轉換的目的

3. 接口適配

適配接口，我們在定義一個HighAction作為高級行為，和Action中有共同的run()行為一致

public interface HighAction { /** * 可以飛，能上天 */ void fly(); /** * 可以跑，能下海 */ void run();}

我們在整合兩種行為在一起，對其進行適配

public class AbstractAction implements Action, HighAction { @Override public void drive() { System.out.println("駕駛"); } @Override public void walk() { System.out.println("走路"); } @Override public void run() { System.out.println("奔跑大家都會，蝸牛都可以狂奔"); } @Override public void fly() { System.out.println("可能會飛了~"); }}

那麼他們就都有了run()的行為了，行為一致

適配器模式的目的就是要將不同對象的行為轉化成一致，不同的場景下用不同的方式去實現適配。
