1) 可視化思考

圖畫思維

親自去做一做，動作思考。

觀察的六個方式：誰/什麼？ WHO

在哪裡？ WHERE

什麼時候？ WHEN

有多少？ HOW MANY

3) 六頂帽思考-愛德華·德·博諾

思考一個問題或看待一件事情的時候，不再是單純地思考做還是不做、黑或者白、好還是壞，而是進行全面的思考和總結多種可能性。

白色思考帽——代表事實，關注客觀的事實和數據；

綠色思考帽——代表創造，運用創造性思考並採取行動；

黃色思考帽——代表價值與肯定，尋找事物的亮點和長處

黑色思考帽——代表否定和懷疑，從事物的缺點、危險來看待問題；

紅色思考帽——代表感性，象徵從情緒、感性和直覺角度來看待問題；

藍色思考帽——代表理性，意味著控制和調節。



4) 探索式測試思維模型

CPIE模型-Erik Petersen



整理（Collation）：盡最大可能收集關於被測產品的信息，去了解和理解它們。

排序（Prioritization）：確定所有測試任務的優先級。

調查（Investigation）：對即將執行的測試任務進行仔細的分析並確定測試輸入和預期輸出。

實驗（Experimentation）：實際地去測試，驗證我們的預測是否正確，檢查我們在整理階段獲取到的信息是否正確。根據實驗結果，測試人員將收集更多的信息，並調整測試任務的優先級。

KAET模型-James Bach

知識：掌握產品特性、開發技術、測試技術和領域規則等測試需要的知識。

分析：分析產品風險、測試覆蓋、測試方法、測試先知 和產品缺陷等測試相關因素。

實驗：配置、操作、觀察和評估被測產品。

測試故事：用測試計劃、測試報告和可工作的產品等組成測試報告，以準確地反映測試狀態和產品質量。

6) 獨立思考模型

正確的定義問題 Define Question

我們的大腦常常不聰明，大腦提出的問題 未必是真正需要解決的問題。

建立邏輯樹 Set up Logical（Decision）Tree

由所問的問題展開的要解決這個問題需要考慮的因素（把問題作為結果，即是會影響結果的因素）

奧多姆剃刀 Dirty Filter

為確保高效簡潔和操作性，保留核心，大膽刪除不相關部分或者相關度低的部分。

提出假設

在用海量數據進行分析之前，根據自己的常識，感覺以及少量的數據，進行判斷，提出一個假設

確定數據 Define Data

寫出數據來源 List Data Source

收集數據 Collect Data

開始著手收集需要的信息，並在這個過程中不斷調整計劃。

7) 辨別信息的思維模型

實證模型

科學的思考方法的一個重要基石就是實證主義，即任何一個理論或者理念要被確認為正確的話，首先他必須是能被觀察的，能現實存在的，能夠通過客觀事實來證實的。否則就無法被科學地承認。

可證偽模型

當這句話/理論是為真時，發生A，如果不為真時，發生B。A和B互斥。

能解釋一切的東西往往不具可證偽性。

可證偽是有強度的。強度不同，可信度也不同。

信息來源模型

信息是否準確，一個重要的衡量標準是信息的來源是否可靠。這個可靠包含兩層含義。第一層是這個信息的根源是否可信。第二層含義是信息宣傳的來源渠道 是否可信。

個案見證模型

一件事情如果只能靠“個案”或者“見證者”是無法證明其真實性和有效性的。那麼哪類信息是可信的呢？是擁有大量樣本的 統計數據。

在判斷一個論斷，或者一個發現的時候，只需要分辨這個結論的依據是大量的實驗或者統計數據，還是僅有的幾個個案。

個案見證模型可以幫我們分辨信息的代表性，從而判斷是否需要採用。

具象模型

大部分的信息都是抽象的，如果將之具體化形象化，簡稱具象，會加深這個信息給我們的印象，增大在我們心中的影響。具象模型分成2個部分，分別是“鮮 活形象效應”和“當下發生效應”。

8) 尋找真相的思維模型

因果關聯模型

因果關聯模型就是在腦中建立一個報警器，當看到兩個高度相關的事件或者數據時，敏銳的識別出他們究竟是因果關係還是僅僅只有相關性，以防止我們被表 面現象或者所謂專家舉出的別有用心的數據所迷惑。

第三因素模型

AB相關並不代表因果，那麼嘗試進一步思考，AB究竟為何會呈現如此高的相關性呢？背後的原因是什麼？

答案是第三因素。也就是說AB之間存在一個隱藏的關係C，其實是AB和C發生因果關係。

If…So…”控制模型

確保其他因素不變，找到發生影響的根源。

交互影響模型

當看到一個數據/信息的時候，將該數據/信息放入過濾器過濾，判斷是屬於單一因素起決定作用，還是眾多因素相互 影響共同作用。如果是交互影響，又有哪些因素共同作用，哪些作用大？哪些作用久？哪些又對其他因素產生影響？

安慰劑效應模型

錯誤地認為自己所採取的行動對結果產生了積極的作用。

因果方向模型

看待一個數據/信息時，需要首先找到它的假設前提。許多假設前提都是隱藏的，但卻決定了作者基於假設的觀點正確與否。因果方向模型提供了 解構和質疑假設前提的一種思維方式。

9) 3C戰略

成功戰略有三個關鍵因素，在制定任何經營戰略時，都必須考慮這三個因素，即公司自身（Corporation）、公司顧客（Customer）、 競爭對手（Competition）。 只有將公司、顧客與競爭者整合在同一個戰略內，可持續的競爭優勢才有存在的可能。

10) 競爭戰略模型

基本競爭戰略有三種：成本領先戰略、差異化戰略、集中戰略。企業必須從這三種戰略中選擇一種，作為其主導戰略。

22) PDCA循環

Plan(計劃)、Do(執行)、Check(檢查)和Act(修正)。

每一件事情先做計劃，計劃完了以後去實施，實施的過程中進行檢查，檢查結果以後，再把檢查的結果進行改進，進行實施，進行改善，這樣把沒有改善的問題又放到下一個循環裡面去，就形成一個一個的PDCA循環。
