做Android開發的我們肯定離不開網絡請求，無論是做數據展示還是文件的上傳下載都要請求網絡。在各大網絡請求框架的幫助下，我們很容易就能實現這一功能，但是你有思考過數據在網絡中是如何度過他的一生的嗎？

計算機網絡概念

計算機網絡的定義

計算機網絡是利用通信設備與通信鏈路或者通信網絡，互聯位置不同、功能自治的計算機系統，並遵循一定的規則事項的計算機系統之間的信息狡猾。

概括：計算機網絡是互聯的、自治的計算機的集合。

協議的定義

協議是網絡通信實體之間在數據交換過程中需要遵循的規則或約定，包括三個基本要素：

  1. 語法：實體之間交換信息的格式與結構。

  2. 語義：實體間交換信息中需要發送哪些控制信息，這些信息的具體含義，以及針對不同含義的控制信息，接受信息端應如何響應。

  3. 時序：實體之間交換信息之間的順序以及如何匹配或適應批次的速度。

計算機網絡概念

  • 硬件資源共享

  • 軟件資源共享

  • 信息資源共享

計算機網絡的分類

  1. 按照覆蓋範圍：個域網、局域網、城域網、廣域網

  2. 按拓撲結構：星型、總線型、環形、網狀、樹形、混合拓撲結構

  3. 按交換方式：分組交換網絡、電路交換網絡、報文交換網絡

  4. 按網絡用戶屬性：公用網、私有網


計算機網絡結構

  • 網絡邊緣如：電腦、手機

  • 接入網絡如：電話撥號、非對稱用戶數字線ADSL、混合光潛同軸電纜HFC、局域網、移動接入）

  • 網絡核心

    主要作用是數據轉發，由通信鏈路互聯的分組交換設備構成。

數據交換技術

分組交換

將完整報文拆分為若干個分組，每次傳輸一個分組，交換節點每次接收一個分組，當一個分組完全接收完成後開始轉發，各節點之間以存儲-轉發方式傳送

優點：

  1. 對交換設備存儲的容量要求低

  2. 交換的速度更快

  3. 可靠傳輸效率高

  4. 更加公平

缺點：

  1. 分組長度與延遲時間的平衡

  2. 分組長度與與碼率的平衡

報文交換

已報文為單位在交換網絡各節點之間以存儲-轉發方式傳送

優點：不需要建立連接，只有當報文被轉發時才佔用相應的信道

缺點：交換節點需要緩衝存儲，報文需要排隊，增加了延時

電路交換

在電路交換網絡中，首先需要通過中間交換節點為兩臺主機間建立一條專用的通信線路，稱為電路，然後在利用該電路進行通信，通信結束後再拆除電路。

優點：實時性高，時延和時延抖動較小

缺點：對於突發性數據傳輸，信道利用率低，傳輸速率單一

計算機網絡性能

速率與帶寬

速率：指網絡單位時間內傳送的數據量，也稱為傳輸速率或數據速率，單位為 bit/s（位每秒）也叫 bps

帶寬：帶寬原本指的是信號具有的頻帶寬度，是信號成分的最高頻率與最低頻率之差，單位為Hz（赫茲）

有時帶寬也被用來描述速率，但是我們可以以描述單位區分所指的具體是什麼，如單位為Hz（赫茲）指帶寬，如單位為 bps 指速率。

時延

時延是指數據從網絡中的一個節點到達另一個節點所需要的時間。

分組的每跳傳輸付哦成主要會產生四類時間延遲：

  1. 節點處理時延

  2. 排隊時延

  3. 傳輸時延

  4. 傳播時間

時延帶寬積

一段物理鏈路的傳播時延與鏈路帶寬的乘積，便是一段鏈路可以容納的數據位數，也稱為以位為單位的鏈路長度。

丟包率

丟包率常被用於評價和衡量網絡性能的指標，在很大程度上可以反映網絡的擁堵程度。

丟包率 = 丟失的分組總數 / 發送的分組總數

吞吐量

單位時間內源主機通過網絡向目的主機實際發送的數據量。經常用於度量網絡的實際數據傳送能力。

計算機網絡分層體系結構

分層思想

計算機網絡完成的所有功能可以劃分為若干層，每一層完成一部分子功能，每層在完成相應功能時與另一通信實體的相同層按照某種協議進行信息交換。

體系結構定義

計算機網絡所劃分的層次以及各層協議的集合稱為計算機網絡體系結構

OSI 參考模型

全稱：開放系統互連（Open System Interconnection，OSI） 由國際標準化組織 （International Organization for Standardization，ISO） 發起的，其任務是制定國際計算機通信標準，特別是促進不兼容系統間的互聯。

OSI 模型分為七層：物理層、數據鏈路層、網絡層、傳輸層、會話層、表示層、應用層

TCP/IP 四層參考模型

實際應用最多的參考模型，

TCP/IP 模型分為四層：

網絡接口層：對應 OSI 的物理層、數據鏈路層

網際層：對應 OSI 的網絡層，使用 IP（網際協議地址） 協議實現

運輸層：對應 OSI 的傳輸層，使用 TCP（傳輸控制協議） 、UDP（用戶數據報） 協議實現

應用層：對應 OSI 的會話層、表示層、應用層，使用 HTTP（超文本傳輸協議）、SMTP（電子郵件服務）、NDS（域名系統）、RTP 等協議實現（實時傳輸協議）

TCP/IP 五層參考模型

五層參考模型：物理層、鏈路層、網絡層、傳輸層、應用層
