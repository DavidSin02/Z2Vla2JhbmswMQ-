那些讓人欲罷不能的複製構造函數

2020-11-14
| 科技


缺省構造函數:

即為系統提供的，參數表為空的構造函數。


帶默認參數的構造函數:

當我們需要構造函數中的某些參數值不被改變時，可以將其定義為帶默認參數的構造函數。
如


class Test{
int x,y;
public:
Test(int i=0,int j=0):x(i),y(j){}
……
};


Test類的工作函數即為帶有默認參數的工作函數，在這裡定義數據成員i,j時就已經將它們賦值為零。這就是帶有默認參數的構造函數。


重載構造函數:

構造函數是可以進行重載的，一個類中可以定義多個構造函數，（一個類只能有一個缺省構造函數）但是這些構造函數同樣也必須滿足函數重載的條件——即所帶參數的個數或類型不同，以便定義對象調用構造函數時不能嚴格匹配產生二義性。
當有多個構造函數時，系統通過形參表來匹配構造函數。


如:11 / 30


Test a(1,2);與Test b;這兩條語句，很明顯，在定義對象時調用的構造函數是不一樣的，第一條語句應該調用有兩個參數的構造函數，而第二條語句則應該調用缺省構造函數。


如上Test b;在調用缺省構造函數時要注意區分Test b;和Test b();


前者是創建Test類的對象a;而後者是聲明函數a,此函數函數的返回值是Test類型，也就是把Test當作數據類型作為返回值，同樣可以類比C語言中數據結構。


複製構造函數:

複製構造函數是一種特殊的構造函數，其作用是用一個已經存在的對象去初始化一個新的同類的對象，簡單來說，就是


用已知對象創建同類對象


複製構造函數的一般形式:


類名::類名（const 類名 &引用對象名）{
函數體
}


複製構造函數的特點:

* 只有一個形參，並且該參數是該類的對象的引用，即形式中的引用對象名。

* 每一個類中都必須有一個複製構造函數。如果用戶沒有聲明類的複製構造函數，系統會自動生成一個缺省複製構造函數。


複製構造函數的功能:

把引用對象的每個數據成員的值對應賦給新建立的對象的數據成員。


複製構造函數是如何被調用的呢？

根據一般形式，先定義一個對象a,再定義一個對象b,用缺省複製構造函數將a的數據值傳遞複製給b。
即
Test a (1,2);
Test b(a);


顯然Test b(a)這條語句調用了缺省複製構造函數，將對象a的數據都複製傳遞給了b。
其實這條語句調用的是系統提供的缺省複製構造函數，必要時，我們也可以自定義複製構造函數。

定義形式如下:

類名::類名（const 類名 &引用對象名）{﻿﻿﻿﻿
函數體﻿﻿﻿﻿
};

自定義複製構造函數與缺省複製構造函數的調用語句並無區別。


什麼時候需要調用複製構造函數呢？


* 當用已有對象去初始化該類的另一個新對象時。


Test a (1,2);
Test b(a);


* 如果函數的形參，是類的對象，調用函數進行形參和實參的結合。


Test a;
fun（a）;


調用函數fun()時實參是Test類的對象，故調用函數時將複製一個新對象obj。


* 如果函數的返回值是類的對象，函數調用完成時。


Test a;
a=fun();


這種情況下，在fun返回對象時，也會調用複製構造函數，複製出臨時對象，然後將臨時對象的值賦給a。


我們為什麼要用到複製構造函數呢？


用第一個例子做對比:
調用了複製函數的語句:
Test a (1,2);﻿﻿
Test b(a);


直接用對象複製和複製的語句
Test a,b;
b=a;


我們可以清楚的看到，不調用複製構造函數的語句明顯語句更多，更不清晰，而且程序運行過程中需要執行賦值運算符重載函數。


類型轉換構造函數:
除複製構造函數外，只有一個參數的構造函數一般都可以稱作類型轉換構造函數，起到類型自動轉換的作用。


如Test(int i);這個構造函數就可以稱為Test類的類型轉換構造函數。
定義對象之後就可以調用Test (int i )函數，來完成對象的初始化。
