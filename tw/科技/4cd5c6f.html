React的組件動態參數使用Underscore和Context來傳遞

2020-10-29
| 科技

通常情況下，子組件明確地知道從props中傳入的屬性，甚至要對傳入的屬性進行限定。但父組件向子組件傳遞參數時，有時參數名稱是作為變量出現的，無法預先明確下來，這就是動態的參數傳遞。動態參數傳遞的方法是使用展開屬性的方法，直接傳遞一個屬性對象，但需要加入特殊的標識符“…”，如下：

return <Component {...this.props} more="values" />;

有時把所有屬性都傳下去是冗長且不安全的。這時可以使用ES 6規範解構賦值中的剩餘屬性特性，把未知屬性批量提取出來。下面的示例將this.props中除了checked屬性之外的其他屬性複製到other變量中再傳遞給組件，但這裡的“…”表示剩餘屬性，與上例中出現的“…”展開屬性是不一樣的。

var { checked, ...other } = this.props;

return <Component {...other} attrMore="values" />;

如果上面的other對象中也含有attrMore屬性，則會被覆蓋。這裡的順序很重要，把{...other}放到JSX props前面會使它不被覆蓋。

在數據通過組件樹逐層傳遞時，有時某些屬性會從上層組件手動逐層傳遞到最底層的組件，比如A組件為了給包含在B組件中的C組件傳遞一個prop，需要把參數在組件中傳遞兩次才能最終將A組件中的prop傳遞給C組件。這樣的傳遞既煩瑣又無聊，而且使B組件在中間參與了與自己無關的邏輯。針對這個問題，我們可以使用Context特性，其能實現組件樹上的數據越級傳遞。

Context是React 0.14版本新增的一個高級的、實驗性的機制，將來的API細節可能會有一些更改，不推薦頻繁使用。如果確實需要，應儘量保持在小範圍內使用，並且避免直接使用Context的API，以便於以後的升級。

將上面的例子改為使用Context機制進行數據傳遞，重點關注加粗部分和相應註釋。

在示例代碼中，在下級組件Button中定義contextTypes，並通過this.context訪問數據，在上層組件MessageList通過添加childContextTypes和getChildContext（）提供數據，React自動向下傳遞數據，使得組件樹中上層組件的任意層級的下級組件都能獲得上層組件中的數據。Context機制在React的路由管理框架router中也有應用。
