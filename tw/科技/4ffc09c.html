IP地址基礎知識和子網規劃，一文讀懂

2020-10-29
| 科技

在弱電調試中，設置最多的是ip地址、掩碼、網關，調試必設置工作之一，你設置的ip是如何計算出來的，下面我們來一起了解ip地址。

IP地址基礎

1.IP地址介紹

在 TCP/IP 環境中，各種各樣的終端、工作站能同服務器、其它工作站無縫連接，是因為每一網絡節點都使用了在全網範圍內能夠唯一標識本節點的IP地址。

在互聯網上，報文的發送經常基於名稱，而不是枯燥的 IP 地址。如果在通訊過程中使用名字而不是 IP 地址，則需先將名字轉化成 IP 地址，然後再發送。

在互聯網上每個公司被看作單個網絡，在能訪問到公司的主機之前必須先能訪問到它所在的網絡。每個網絡有一個全網唯一的網絡號，在該網絡中的各主機共享這個網段號（網段地址）。同時，各主機有可唯一標識自己的主機號。

2.IP地址

IP 地址有 32 位比特位，分為兩部分：

網段地址部分；

主機地址部份。

地址採用點分十進制格式：

如 172.16.112.204

八位組每位有二進制權（128，……，4，2，1）

八位組最大值為 255

八位組最小值為 0

IP 地址的分配由地址分配中心管理。

3.IP地址類型

當初開發 IP 協議時，IP 地址並沒有分類。後來為了管理上的需要，對IP 地址進行了分類：

有 126 個 A 類網絡，每個 A 類網絡包括大約 1600 萬個 IP 地址；有16000 餘個 B 類網絡，每個 B 類網絡包括 65534 個 IP 地址；有 200 多萬個 C 類網絡，每個 C 類網絡包括 254 個 IP 地址。

這種地址分配原則允許地址管理機構基於網絡大小來分配地址。

D 類地址從 224.0.0.0 開始，為多播使用。E 類地址從 240.0.0.0 開始，用於實驗目的。

4.IP地址範圍

IP 地址最高五位決定了IP地址的種類。

A 類地址：

地址從 1.0.0.0 到 126.0.0.0 ，共 16777214 個主機地址；

B 類地址：

地址從 128.1.0.0 到 191.254.0.0 ，共 65534 個主機地址；

C 類地址：

地址從 192.0.1.0 到 223.255.254.0 ，共 254 個主機地址；

D 類地址：

從 224.0.0.0 到 239.255.255.254。

5.IP地址辨別

圖中第一個八位組決定了 IP 地址的類型。

IP 地址以這種方法定義，使用時可以很快從中抽出主機地址和網段地址部分。當決定將一個分組發往何處時，路由器使用地址的網段部分。路由器因為能高效地提取網段地址從而達到很高的報文轉發速度。

6.主機地址

每一設備和接口必須有一個主機地址（即 IP 地址的主機部分不能為全0 或全為 1）。

全 1 主機地址保留給 IP 廣播使用。

全 0 主機地址意味著這個網絡本身，有些早期的網絡把用它作為廣播地址使用。

路由表中包含網段地址，它通常不包含主機地址。

在接口上的 IP 地址和子網地址實現三個功能：

使系統收發報文；

指定設備地址；

指定共享同一線路的設備地址範圍。

7.無子網編址

對於沒有子網的 IP 地址組織，外部將該組織看作單一網絡，不需要知道內部結構。例如，所有到地址 172.16 .X.X 被認為同一方向，不考慮地址的第三和第四個 8 位分組，這種方案的好處是減少路由表的項目。

但這種方案沒法區分一個大的網絡內不同的子網網段，這使網絡內所有主機都能收到在該大的網絡內的廣播，會降低網絡的性能。另外也不利於管理。

比如，一個 B 類網可容納 65000 個主機在網絡內。但是沒有任何一個單位能夠同時管理這麼多主機。這就需要一種方法將這種網絡分為不同的網段。按照各個子網段進行管理。

8.帶子網編址

利用子網，網絡地址的使用會更有效。對外仍為一個網絡，對內部而言，則分為不同的子網。

如圖：網絡172.16.0.0 分為兩個網段：172.16.4.0、172.16.8.0。

如果公司的財務部使用 172.16.4.0 子網段；公司的工程部使用 172.16.8.0 子網段。這樣可使路由器根據目的子網地址進行路由，從而限制一個子網的廣播報文發送到其它網段，不對網絡的效率產生影響。

9.帶子網編址舉例

從地址分配的角度來看，子網是網段地址的擴充。網絡管理員根據組織增長的需要決定子網的大小。

網絡設備使用子網掩碼決定IP地址中哪部分為網絡部分，哪部分為主機部分。

子網規劃

1.子網規劃

在這個例子中，網段地址是一個 C 類地址：201.222.5.0 。假設需要 20 個子網，其中每個子網 5 個主機，就要把主機地址的最後一個八位組分成子網部分和主機部分。

子網部分的位數決定了子網的數目。在這個例子中子網部分佔有5位，最大可提供 30（25 -2）個子網。剩餘 3 位為主機部分。一共有 8 個（23）值。主機部分全是 0 的 IP 地址，是保留地址；主機部分全是 1 的 IP 地址是本子網的廣播地址。這樣就剩餘 6 個主機地址。可以滿足需要。

2.B類子網規劃實例

對於 B 類網絡來說，如果子網有八位，則能提供 254 個子網，每個子網可容納 254 臺主機。

2.C類子網規劃實例

對於圖中 C 類網絡來說，如果子網有五位，則能提供 30 個子網，每個子網可容納 6 臺主機。

廣播地址

Internet 網支持廣播地址。廣播信息是那些要求每臺主機都要收到的信息。廣播地址有兩種：

直接廣播地址：有網絡號但主機部分是全 1，可由路由器轉發。

有限廣播地址：全 1 的 IP 地址，即 255.255.255.255。不能被路由器傳遞，只能在本網段內廣播。

IP 地址配置及檢驗

配置 IP 地址

路由器是通過掩碼來識別 IP 地址的網絡部分、主機部分。例如：路由器以太網口的 IP 地址是 129.9.30.42，掩碼是 255.255.0.0，將 IP 地址與掩碼相與，可知路由器以太網接口的 IP 地址的網絡部分是 129.9.0.0，主機部分是 30.42。

靜態域名解析

配置靜態域名解析：

Quidway(config) # host quidway1 129.102.10.1

當用戶在 Quidway 路由器上配置了該命令，用戶就可以使用 quidway1 來代替 129.102.10.1 這個 IP 地址了。

網絡檢測工具

下面這些工具使您能檢驗IP地址配置的正確性：

Ping：是一種檢驗物理層、鏈路層、網絡層連通性的測試工具。

Tracert：是一種用來對遠端網絡或子網上設備和主機間的路由進行跟蹤、探測的工具。
