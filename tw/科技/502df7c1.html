Java基礎：Java容器之LinkedList

2020-11-14
| 科技

定義

實現List接口與Deque接口雙向鏈表，實現了列表的所有操作，並且允許包括null值的所有元素，對於LinkedList定義我產生了如下疑問：

  • 1.Deque接口是什麼，定義了一個怎樣的規範?
  • 2.LinkedList是雙向鏈表，其底層實現是怎樣的,具體包含哪些操作?

下文將圍繞這兩個問題進行，去探尋LinkedList內部的奧祕,以下源碼是基於JDK1.7.0_79。

結構

類結構

LinkedList的類的結構如下圖所示:

通過上圖可以看出，LinkedList繼承的類與實現的接口如下：

  • 1.Collection 接口： Collection接口是所有集合類的根節點，Collection表示一種規則，所有實現了Collection接口的類遵循這種規則
  • 2.List 接口： List是Collection的子接口，它是一個元素有序(按照插入的順序維護元素順序)、可重複、可以為null的集合
  • 3.AbstractCollection 類： Collection接口的骨架實現類，最小化實現了Collection接口所需要實現的工作量
  • 4.AbstractList 類： List接口的骨架實現類，最小化實現了List接口所需要實現的工作量
  • 5.Cloneable 接口： 實現了該接口的類可以顯示的調用Object.clone()方法，合法的對該類實例進行字段複製，如果沒有實現Cloneable接口的實例上調用Obejct.clone()方法，會拋出CloneNotSupportException異常。正常情況下，實現了Cloneable接口的類會以公共方法重寫Object.clone()
  • 6.Serializable 接口： 實現了該接口標示了類可以被序列化和反序列化，具體的 查詢序列化詳解
  • 7.Deque 接口： Deque定義了一個線性Collection，支持在兩端插入和刪除元素，Deque實際是“double ended queue(雙端隊列)”的簡稱，大多數Deque接口的實現都不會限制元素的數量，但是這個隊列既支持有容量限制的實現，也支持沒有容量限制的實現，比如LinkedList就是有容量限制的實現,其最大的容量為Integer.MAX_VALUE
  • 8.AbstractSequentialList 類： 提供了List接口的骨幹實現，最大限度地減少了實現受“連續訪問”數據存儲(如鏈表)支持的此接口所需的工作，對於隨機訪問數據(如數組)，應該優先使用 AbstractList，而不是使用AbstractSequentailList類

基礎屬性及構造方法

基礎屬性

public class LinkedList<E>

extends AbstractSequentialList<E>

implements List<E>, Deque<E>, Cloneable, java.io.Serializable

{

//長度

transient int size = 0;

//指向頭結點

transient Node<E> first;

//指向尾結點

transient Node<E> last;

}

如上源碼中為LinkedList中的基本屬性，其中size為LinkedList的長度，first為指向頭結點，last指向尾結點，Node為LinkedList的一個私有內部類，其定義如下，即定義了item(元素)，next(指向後一個元素的指針)，prev(指向前一個元素的指針)

private static class Node<E> { //元素 E item; //指向後一個元素的指針 Node<E> next; //指向前一個元素的指針 Node<E> prev; Node(Node<E> prev, E element, Node<E> next) { this.item = element; this.next = next; this.prev = prev; }}

那麼假如LinkedList中的元素為[“A”,”B”,”C”],其內部的結構如下圖所示

可以看出一個節點中包含三個屬性，也就是上面源碼中定義的屬性，可以清晰的看出LinkedList底層是雙向鏈表的實現

構造方法

在源碼中，LinkedList主要提供了兩個構造方法，

  • 1.public LinkedList() ：空的構造方法，啥事情都沒有做
  • 2.public LinkedList(Collection<? extends E> c) : 將一個元素集合添加到LinkedList中

底層實現

在2.2.1中的LinkedList內部結構圖，可以清晰的看出LinkedList雙向鏈表的實現，下面將通過源碼分析如何在雙向鏈表中添加和刪除節點的。

添加節點

通常我們會使用add(E e)方法添加元素，通過源碼我們發現add(E e)內部主要調用了以下方法 //在鏈表的最後添加元素

void linkLast(E e) {

final Node<E> l = last;

final Node<E> newNode = new Node<>(l, e, null);

last = newNode;

if (l == null)

first = newNode;

else

l.next = newNode;

size++;

modCount++;

}

其實通過源碼可以看出添加的過程如下

  • 1.記錄當前末尾節點，通過構造另外一個指向末尾節點的指針l
  • 2.產生新的節點：注意的是由於是添加在鏈表的末尾，next是為null的
  • 3.last指向新的節點
  • 4.這裡有個判斷，我的理解是判斷是否為第一個元素(當l==null時，表示鏈表中是沒有節點的)， 那麼就很好理解這個判斷了，如果是第一節點，則使用first指向這個節點，若不是則當前節點的next指向新增的節點
  • 5.size增加 例如，在上面提到的LinkedList[“A”,”B”,”C”]中添加元素“D”，過程大致如圖所示

LinkedList中還提供如下的方法，進行添加元素，具體邏輯與linkLast方法大同小異，就不在這裡一一介紹了。

刪除節點

LinkedList中提供了兩個方法刪除節點，如下源碼所示

//方法1.刪除指定索引上的節點

public E remove(int index) {

//檢查索引是否正確

checkElementIndex(index);

//這裡分為兩步，第一通過索引定位到節點，第二刪除節點

return unlink(node(index));

}

//方法2.刪除指定值的節點

public boolean remove(Object o) {

//判斷刪除的元素是否為null

if (o == null) {

//若是null遍歷刪除

for (Node<E> x = first; x != null; x = x.next) {

if (x.item == null) {

unlink(x);

return true;

}

}

} else {

//若不是遍歷刪除

for (Node<E> x = first; x != null; x = x.next) {

if (o.equals(x.item)) {

unlink(x);

return true;

}

}

}

return false;

}

通過源碼可以看出兩個方法都是通過unlink()刪除，在方法一種有個方法要介紹下，就是node(index)該方法的作用就是根據下標找到對應的節點，要是本人去寫這個方法肯定是遍歷到index找到對應的節點，而JDK提供的方法如下所示

  • 1.首先確定index的位置，是靠近first還是靠近last
  • 2.若靠近first則從頭開始查詢，否則從尾部開始查詢，可以看出這樣避免極端情況的發生，也更好的利用了LinkedList雙向鏈表的特徵

Node<E> node(int index) {

// assert isElementIndex(index);

if (index < (size >> 1)) {

Node<E> x = first;

for (int i = 0; i < index; i++)

x = x.next;

return x;

} else {

Node<E> x = last;

for (int i = size - 1; i > index; i--)

x = x.prev;

return x;

}

}

下面會詳細介紹unlink()方法的源碼,這是刪除節點最核心的方法

E unlink(Node<E> x) { // assert x != null; final E element = x.item; final Node<E> next = x.next; final Node<E> prev = x.prev;  //刪除的是第一個節點，first向後移動 if (prev == null) { first = next; } else { prev.next = next; x.prev = null; }  //刪除的是最後一個節點，last向前移 if (next == null) { last = prev; } else { next.prev = prev; x.next = null; } x.item = null; size--; modCount++; return element;}
  • 1.獲取到需要刪除元素當前的值，指向它前一個節點的引用，以及指向它後一個節點的引用。
  • 2.判斷刪除的是否為第一個節點，若是則first向後移動，若不是則將當前節點的前一個節點next指向當前節點的後一個節點
  • 3.判斷刪除的是否為最後一個節點，若是則last向前移動，若不是則將當前節點的後一個節點的prev指向當前節點的前一個節點
  • 4.將當前節點的值置為null
  • 5.size減少並返回刪除節點的值

至此介紹了LinkedList添加、刪除元素的內部實現。

4.對比

在ArrList詳解中講解了ArrayList的相關的內容，下面將對ArrayList與LinkedList進行對比，主要從以下方面進行

4.1 相同點

  • 1.接口實現：都實現了List接口，都是線性列表的實現
  • 2.線程安全：都是線程不安全的

4.2 區別

  • 1.底層實現:ArrayList內部是數組實現，而LinkedList內部實現是雙向鏈表結構
  • 2.接口實現：ArrayList實現了RandomAccess可以支持隨機元素訪問，而LinkedList實現了Deque可以當做隊列使用
  • 3.性能：新增、刪除元素時ArrayList需要使用到拷貝原數組，而LinkedList只需移動指針，查找元素 ArrayList支持隨機元素訪問,而LinkedList只能一個節點的去遍歷

自己動手寫個LinkedList

自己動手實現一個這樣的LinkedList：

 public class MyLinkList<T> implements Iterable<T> { private int size; private int modeCount = 0; private Node<T> beginMarker; private Node<T> endMarker; public MyLinkList() { doClear(); } private void doClear() { size = 0; modeCount++; beginMarker = new Node<T>(null, null, null); endMarker = new Node<T>(null, beginMarker, null); beginMarker.next = endMarker; } public boolean add(T data) { Node<T> node = new Node<>(data, endMarker.prev, endMarker); endMarker.prev.next = node; endMarker.prev = node; modeCount++; size++; return true; } public T remove(int index) { Node<T> node = getNode(index); node.prev.next = node.next; node.next.prev = node.prev; size--; modeCount--; return node.data; } public Node<T> getNode(int index) { Node<T> p; if (index < 0 | index >= size) { throw new IndexOutOfBoundsException("out of index"); } if (index < size / 2) {//從左邊開始找 p = beginMarker.next; for (int i = 0; i < index; i++) { p = p.next; } } else {//從右邊開始找 p = endMarker; for (int i = size; i > index; i--) { p = p.prev; } } return p; } public T get(int index) { return getNode(index).data; } @Override public Iterator<T> iterator() { return null; } @Override public void forEach(Consumer<? super T> action) { } @Override public Spliterator<T> spliterator() { return null; } static class Node<T> { private T data; private Node<T> prev; private Node<T> next; public Node(T data, Node<T> prev, Node<T> next) { this.data = data; this.prev = prev; this.next = next; } }}
