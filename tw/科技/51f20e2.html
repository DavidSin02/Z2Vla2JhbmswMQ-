3、優先級

如果有多箇中斷同時響應，搶佔優先級高的就會搶佔優先級低的優先得到執行，如果搶佔優先級相同，就比較子優先級。如果搶佔優先級和子優先級都相同的話，就比較他們的硬件中斷編號，編號越小，優先級越高。

串口

USART_ITConfig(DEBUG_USART1, USART_IT_RXNE, ENABLE); /* 使能串口接收中斷 */ USART_ITConfig(DEBUG_USART1, USART_IT_RXNE, DISABLE); /* 關閉串口接收中斷 */

TIM_ITConfig(TIM4, TIM_IT_Update, ENABLE); /* 使能定時器更新中斷 */ TIM_ITConfig(TIM4, TIM_IT_Update, DISABLE); /* 關閉定時器更新中斷 */

6、搶斷優先級、響應優先級說明

l 搶斷優先級高的，能打斷搶斷優先級低的；

l 當搶斷優先級相同時，執行順序由響應優先級決定；

l 當搶斷優先級、響應優先級都相同時，則按中斷向量表的順序執行。

如果搶斷優先級一樣，則不會打斷，而是等待完成後執行。

比如有ABCD四個中斷：

一、當A正在執行，又來了中斷B，由於B的搶斷優先級更高，所以會打斷A的運行；

二、當A正在執行，又來了中斷C，由於C搶斷優先級<=A，所以會等待A完成後，再執行C；

三、當A正在執行，同時來了C和D，由於ABC優先級都相同，所以不會打斷，而C的響應優先級比D高，所以在A完成後，會先執行C，再執行D。


7、定時器說明

STM32F42xxx 系列控制器有2 個高級控制定時器、10 個通用定時器和2 個基本定時器，還有2 個看門狗定時器。

控制器上所有定時器都是彼此獨立的，不共享任何資源。各個定時器特性參考下表：

注意：其中最大定時器時鐘可通過RCC_DCKCFGR 寄存器配置為90MHz 或者180MHz。


7.1基本定時器

主要兩個功能，第一就是基本定時功能，生成時基，第二就是專門用於驅動數模轉換器(DAC)。

基本定時器的內部時鐘(CK_INT)頻率默認設置為90MHz。

7.2高級/通用定時器

高級控制定時器(TIM1 和TIM8)和通用定時器在在基本定時器的基礎上引入了外部引腳，可以輸入捕獲和輸出比較功能。高級控制定時器比通用定時器增加了可編程死區互補輸出、重複計數器、帶剎車(斷路)功能，這些功能都是針對工業電機控制方面。

一般情況下，我們都是使用內部時鐘，內部時鐘CK_INT 即來自於芯片內部，等於180M。
