1-SpringMVC簡介

長期以來 Struts2 與 Spring 的結合一直存在很多的問題，比如兼容性和類脆腫。加之近年來 Struts2 漏洞問題頻發，導致使用率大減。與此同時，生於 Spring Web 項目的 MVC (ModelView Controller）框架走到了我們的面前， Spring MVC 結構層次清晰，類比較簡單，並且與 Spring 的核心 IoC 和 AOP 無縫對接，成為了互聯網時代的主流框架。

MVC 模式把應用程序（輸入邏輯、業務邏輯和 UI 邏輯）分成不同的方面，同時提供這些元素之間的鬆搞合。

• Model （模型），封裝了應用程序的數據和由它們組成的 POJO 。

• View （視圖），負責把模型數據渲染到視圖上，將數據以一定的形式展現給用戶。

• Controller （控制器），負責處理用戶請求，並建立適當的模型把它傳遞給視圖渲染。
在 Spring MVC 中還可以定義邏輯視圖，通過其提供的視圖解析器就能夠很方便地找到對應的視圖進行渲染，或者使用其消息轉換的功能，比如在 Controller 的方法內加入註解@ResponseBody 後，SpringMVC 就可以通過其消息轉換系統，將數據轉換為 ISON，提供給前端 Ajax 請求使用。SpringMVC 中的重點在於它的流程和一些重要的註解，包括控制器、視圖解析器、視圖等重要內容。
2-最流行的 NoSQL一－Redis
Redis 是當前互聯網世界最為流行的 NoSQL ( Not Only SQL) 。 NoSQL 在互聯網系統中的作用很大，因為它可以在很大程度上提高互聯網系統的性能 。它具備一定持久層的功能，也可以作為一種緩存工具。對於 NoSQL 數據庫而言，作為持久層，它存儲的數據是半結構化的，這就意味著計算機在讀入內存中有更少的規則，讀入速度更快。對於那些結構化、多範式規則的數據庫系統而言 ， 它更具性能優勢。作為緩存，它可以支持大數據存入內存中，只要命中率高，它就能快速響應，因為在內存中的數據讀／寫 比數據庫讀／寫磁盤的速度快幾十到上百倍，其作用如圖 1-6 所示 。


目前 NoSQL 有很多爭議，有些人認為它可以取代數據庫，而筆者卻不這麼認為，因為數據庫系統有更好的規範性和數據完整性，功能更強大，作為持久層更為完善，安全性更高。而 NoSQL 結構鬆散 、 不完整 ，功能有限 ， 目前尚不具備取代數據庫的實力，但是作為緩存工具，它的高性能、高響應等功能，使它成為一個很重要的工具。

當前 Redis 已經成為了主要的 NoSQL 工具，其原因如下 。

·響應快速： Redis 響應非常快，每秒可以執行大約 110 000 個寫入操作，或者 8 1 000個讀操作，其速度遠超數據庫 。 如果存入一些常用的數據，就能有效提高系統的性能。

·支持 6 種數據類型： 它們是字符串 、哈希結構、列表、集合、可排序集合和基數 。比如對於字符串可 以存入一些 Java 基礎數據類型，哈希可以存儲對象，列表可以存儲 List 對象等 。 這使得在應用中很容易根據自己的需要選擇存儲的數據類型，方便開發。對於 Redis 而言，雖然只有 6 種數據類型 ，但是有兩大好處 ： 一方面可以滿 足存儲各種數據結構體的需要；另外一方面數據類型少，使得規則就少，需要的判斷和邏輯就少，這樣讀／寫的速度就更快。

·操作都是原子的：所有 Redis 的操作都是原子的，從而確保當兩個客戶同時訪問Redis 服務器時，得到的是更新後的值（最新值）。在需要高併發的場合可以考慮使用 Redis 的事務，處理一些需要鎖的業務。

• MultiUtiIity工具： Redis 可以在如緩存、消息傳遞隊列中使用（ Redis 支持“發佈＋訂閱”的消息模式〉，在應用程序如 Web 應用程序會話、網站頁面點擊數等任何短暫的數據中使用。

正是因為 Redis 具備這些優點，使得它成為了目前主流的 NoSQL 技術，在 Java 互聯網中得到了廣泛使用。一方面，使用 NoSQL 從數據庫中讀取數據進行緩存，就可以從內存中讀取數據了 ，而不像數據庫一樣讀磁盤。現實是讀操作遠比寫操作要多得多，所以緩存很多常用的數據，提高其命中率有助於整體性能的提高，並且能減緩數據庫的壓力，對互聯網系統架構是十分有利的。另一方面，它也能滿足互聯網高併發需要高速處理數據的場合，比如搶紅包 、商品秒殺等場景，這些場合需要高速處理，並保證併發數據安全和一致性。

3-SSM+Redis 結構框圖及概述
在 Java 互聯網中，以 Spring+Spring MVC+MyBatis ( SSM ）作為主流框架， SSM+Redis的結構框圖，如圖 1 -7 所示 。

下面簡單介紹它們各自承擔的功能。

• Spring IoC 承擔了一個資源管理、整合、即插即拔的功能。

• SpringAOP 可以提供切面管理，特別是數據庫事務管理的功能。

• SpringMVC 用於把模型、視圖和控制器分層 ， 組合成一個有機靈活的系統。

• MyBatis 提供了 一個數據庫訪問的持久層，通過 MyBatis-Spring 項目 ， 它便能和Spring 無縫對接。• Redis 作為緩存工具，它提供了高速度處理數據和緩存數據的功能，使得系統大部分只需要訪問緩存，而無須從數據庫磁盤中重複讀／寫；在一些需要高速運算的場合中，也可以先用它來完成運算，再把數據批量存入數據庫，這樣便能極大地提升互聯網系統的性能和響應能力。
