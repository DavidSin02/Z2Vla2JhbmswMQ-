90、熱交換

Spring Boot 支持熱交換。本節回答關於它如何工作的問題。

90.1、重新加載靜態內容

熱重新加載有幾種選擇。推薦的方法是使用 spring-boot-devtools，因為它提供了額外的開發時特性，例如，支持快速應用程序重啟和 LiveReload，以及合理的開發時配置（例如模板緩存）。Devtools 通過監視類路徑的更改來工作。這意味著必須 “構建” 靜態資源更改才能使更改生效。默認情況下，當你保存更改時，這將在 Eclipse 中自動發生。在 IntelliJ IDEA 中，Make Project 命令觸發必要的構建。由於默認的重新啟動排除，對靜態資源的更改不會觸發應用程序的重新啟動。然而，它們確實觸發了實時重新加載。（https://docs.spring.io/spring-boot/docs/2.1.6.RELEASE/reference/html/using-boot-devtools.html#using-boot-devtools-restart-exclude ）

或者，在 IDE 中運行（特別是在調試時）是一種很好的開發方法（所有現代 IDE 都允許重新加載靜態資源，通常也允許熱交換 Java 類更改）。

最後，可以配置 Maven 和 Gradle 插件（參見 addResources 屬性）以支持從命令行運行，並直接從源代碼重新加載靜態文件。如果使用更高級的工具編寫代碼，則可以將其用於外部 css/js 編譯器進程。（https://docs.spring.io/spring-boot/docs/2.1.6.RELEASE/reference/html/build-tool-plugins.html ）

90.2、重新加載模板而不重新啟動容器

Spring Boot 支持的大多數模板技術都包含一個禁用緩存的配置選項（本文檔稍後介紹）。如果你使用 spring-boot-devtools 模塊，這些屬性將在開發時自動為你配置。

90.2.1、Thymeleaf 模板

如果使用 Thymeleaf，請將 spring.thymeleaf.cache 設置為 false。有關其他 Thymeleaf 的自定義選項，請參見 ThymeleafAutoConfiguration。（https://github.com/spring-projects/spring-boot/tree/v2.1.6.RELEASE/spring-boot-project/spring-boot-autoconfigure/src/main/java/org/springframework/boot/autoconfigure/thymeleaf/ThymeleafAutoConfiguration.java ）

90.2.2、FreeMarker 模板

如果使用 FreeMarker，請將 spring.freemarker.cache 設置為 false。有關其他 FreeMarker 的自定義選項，請參見 FreeMarkerAutoConfiguration。（https://github.com/spring-projects/spring-boot/tree/v2.1.6.RELEASE/spring-boot-project/spring-boot-autoconfigure/src/main/java/org/springframework/boot/autoconfigure/freemarker/FreeMarkerAutoConfiguration.java ）

90.2.3、Groovy 模板

如果使用 Groovy 模板，請將 spring.groovy.template.cache 設置為 false。有關其他 Groovy 的自定義選項，請參見 GroovyTemplateAutoConfiguration。（https://github.com/spring-projects/spring-boot/tree/v2.1.6.RELEASE/spring-boot-project/spring-boot-autoconfigure/src/main/java/org/springframework/boot/autoconfigure/groovy/template/GroovyTemplateAutoConfiguration.java ）

90.3、快速應用程序重新啟動

spring-boot-devtools 模塊包括對自動應用程序重啟的支持。雖然不像 JRebel 等技術那樣快，但它通常比 “冷啟動” 要快得多。在研究本文檔後面討論的一些更復雜的重新加載選項之前，你可能應該嘗試一下。

有關更多詳細信息，請參見第 20 章：開發者工具。（https://docs.spring.io/spring-boot/docs/2.1.6.RELEASE/reference/html/using-boot-devtools.html ）

90.4、重新加載 Java 類而不重新啟動容器

許多現代 IDEs（Eclipse、IDEA和其他）支持字節碼的熱交換。因此，如果你所做的更改不影響類或方法簽名，則它應該乾淨地重新加載，而不會產生任何副作用。
