滲透測試新手入門之DVWA（Command Injection命令行注入）篇

2020-10-29
| 科技

命令行注入（Command Injection）：

是指在某些需要輸入數據的位置，還構造了惡意的代碼破壞了原先的語句結構。而系統缺少有效的過濾，最終達到破壞數據、信息洩露甚至掌控電腦的目的。許多內容管理系統CMS存在命令注入漏洞。


命令連接符：

&&：代表首先執行命令a在執行命令b，但是前提條件是命令a執行正確才會執行命令b，在a執行失敗的情況下不會執行b命令。所以又被稱為短路運算符。

&：代表首先執行命令a在執行命令b，如果a執行失敗，還是會繼續執行命令b。也就是說命令b的執行不會受到命令a的干擾，在執行效率上來說“&&”更加高效。

||：代表首先執行a命令在執行b命令，如果a命令執行成功，就不會執行b命令，相反，如果a命令執行不成功，就會執行b命令。

|：代表首先執行a命令，在執行b命令，不管a命令成功與否，都會去執行b命令

解決亂碼問題：

當我們輸入文本時候後顯示亂碼，


在DVWA-master\dvwa\includes目錄下找到dvwaPage.inc.php文件中所有的”charset=utf-8”，修改”charset=gb2312”，即可。

low級：

源代碼：

<?phpif( isset( $_POST[ 'Submit' ]  ) ) {    // Get input    $target = $_REQUEST[ 'ip' ];    // Determine OS and execute the ping command.    if( stristr( php_uname( 's' ), 'Windows NT' ) ) {        // Windows        $cmd = shell_exec( 'ping  ' . $target );    }    else {        // *nix        $cmd = shell_exec( 'ping  -c 4 ' . $target );    }    // Feedback for the end user    echo "<pre>{$cmd}</pre>";}?>

我們通過看源代碼發現，沒有進行任何過濾，我們只要使用“&&”、“&”、“|”即可完成注入。


1.使用ping 192.168.35.132&&dir，得到結果是（前面是我的主機地址）


2.使用ping 192.168.35.132&net user，得到結果是


3.使用ping 192.168.35.132|dir，192.168.35.132|net user得到結果是


Medium級：

源代碼:

<?phpif( isset( $_POST[ 'Submit' ]  ) ) {    // Get input    $target = $_REQUEST[ 'ip' ];    // Set blacklist    $substitutions = array(        '&&' => '',        ';'  => '',    );    // Remove any of the charactars in the array (blacklist).    $target = str_replace( array_keys( $substitutions ), $substitutions, $target );    // Determine OS and execute the ping command.    if( stristr( php_uname( 's' ), 'Windows NT' ) ) {        // Windows        $cmd = shell_exec( 'ping  ' . $target );    }    else {        // *nix        $cmd = shell_exec( 'ping  -c 4 ' . $target );    }    // Feedback for the end user    echo "<pre>{$cmd}</pre>";}?>

我們通過觀察代碼發現，medium等級的命令行注入增加了一些過濾，通過查看源碼可以看到，他將&&和：過濾成了空字符，所以我們依然可以進行注入。

輸入”192.168.35.132 & dir”時，同樣可以攻擊，表明沒有對”&”過濾。

但是”&&”和”&”是有區別的，”&&”是短路運算符，只有前一步執行成功才會執行後一步，而”&”則兩個表達式都會執行。

我們輸入”192.168.35.132&;& net view”時，也是可以的，因為過濾一次後相當於”192.168.35.132&& net view”。

還有很多方法可以進行命令行輸入，比如:“|”、“||”等等

High級：

<?phpif( isset( $_POST[ 'Submit' ]  ) ) {    // Get input    $target = trim($_REQUEST[ 'ip' ]);    // Set blacklist    $substitutions = array(        '&'  => '',        ';'  => '',        '| ' => '',        '-'  => '',        '$'  => '',        '('  => '',        ')'  => '',        '`'  => '',        '||' => '',    );    // Remove any of the charactars in the array (blacklist).    $target = str_replace( array_keys( $substitutions ), $substitutions, $target );    // Determine OS and execute the ping command.    if( stristr( php_uname( 's' ), 'Windows NT' ) ) {        // Windows        $cmd = shell_exec( 'ping  ' . $target );    }    else {        // *nix        $cmd = shell_exec( 'ping  -c 4 ' . $target );    }    // Feedback for the end user    echo "<pre>{$cmd}</pre>";}?>

這關看似沒什麼漏洞了，但是也是有一個小bug的。。引號下多了個空格，所以“|”還是可以用的。


Impossible級：

<?php if( isset( $_POST[ 'Submit' ]  ) ) {     // Check Anti-CSRF token     checkToken( $_REQUEST[ 'user_token' ], $_SESSION[ 'session_token' ], 'index.php' );     // Get input     $target = $_REQUEST[ 'ip' ];     $target = stripslashes( $target );     // Split the IP into 4 octects     $octet = explode( ".", $target );     // Check IF each octet is an integer     if( ( is_numeric( $octet[0] ) ) && ( is_numeric( $octet[1] ) ) && ( is_numeric( $octet[2] ) ) && ( is_numeric( $octet[3] ) ) && ( sizeof( $octet ) == 4 ) ) {         // If all 4 octets are int's put the IP back together.         $target = $octet[0] . '.' . $octet[1] . '.' . $octet[2] . '.' . $octet[3];         // Determine OS and execute the ping command.         if( stristr( php_uname( 's' ), 'Windows NT' ) ) {             // Windows             $cmd = shell_exec( 'ping  ' . $target );         }         else {             // *nix             $cmd = shell_exec( 'ping  -c 4 ' . $target );         }         // Feedback for the end user         echo "<pre>{$cmd}</pre>";     }     else {         // Ops. Let the user name theres a mistake         echo '<pre>ERROR: You have entered an invalid IP.</pre>';     } } // Generate Anti-CSRF token generateSessionToken(); ?>

可以看到，Impossible級別的代碼加入了Anti-CSRF token，同時對參數ip進行了嚴格的限制，只有諸如“數字.數字.數字.數字”的輸入才會被接收執行，因此不存在命令注入漏洞。
