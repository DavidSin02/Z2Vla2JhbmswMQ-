Bash技巧：使用參數擴展獲取變量的子字符串和字符串長度

2020-11-14
| 科技

在 Linux bash shell 中，通常使用 ${parameter} 表達式來獲取 parameter 變量的值，這是一種參數擴展 (parameter expansion)。
Bash 還提供了其他形式的參數擴展，可以對變量值做一些處理，起到操作字符串的效果。例如：

  • ${parameter:offset:length} 從 parameter 變量值的第 offset 個字符開始，獲取 length 個字符，得到子字符串。
  • ${#parameter} 獲取 parameter 變量值的字符串長度。

參考 man bash 的 “Parameter Expansion” 小節的說明，具體舉例說明如下。

${parameter:offset} 和 ${parameter:offset:length}

查看 man bash 對 ${parameter:offset} 和 ${parameter:offset:length} 的說明如下：

Substring Expansion.

Expands to up to length characters of the value of parameter starting at the character specified by offset.


If length is omitted, expands to the substring of the value of parameter starting at the character specified by offset and extending to the end of the value.


length and offset are arithmetic expressions.
Note that a negative offset must be separated from the colon by at least one space to avoid being confused with the :- expansion.

即，${parameter:offset:length} 表達式從 parameter 變量值的第 offset 個字符開始，一直獲取 length 個字符，得到一個子字符串，會包括第 offset 個字符自身。

${parameter:offset} 表達式省略了 length 參數，會從 parameter 變量值的第 offset 個字符開始一直獲取到末尾。

這裡的 length 和 offset 可以是算術表達式。
字符串的 offset 從 0 開始。

如果 offset 的數值小於 0，那麼這個值被用作 parameter 變量值的末尾偏移，從後往前獲取字符。
如果 length 數值小於 0，它會被當成 parameter 變量值的末尾偏移，而不是當作總的字符數目，且擴展後的結果是在這兩個偏移之間的字符。

注意：一個負數的偏移量必須用至少一個空格和冒號分割開，以避免和 :- 擴展產生混淆。
即，這種情況下的冒號 : 和 負號 - 之間至少要有一個空格，類似於 : - 的形式。

具體舉例說明如下：

$ value="This is a test string."

$ echo ${value:5}

is a test string.

$ echo ${value:5:2}

is

$ echo ${value:5: -4}

is a test str

$ echo ${value: -7:3}

str

$ echo ${value: -7: -1}

string

可以看到，${value:5} 獲取從 value 變量值的第 5 個字符開始，一直到末尾的全部字符。
注意偏移量是從 0 開始，獲取到的子字符串包括第 5 個字符自身。

${value:5:2} 從 value 變量值的第 5 個字符開始，獲取包括該字符在內的兩個字符，也就是 "is" 字符串。

當所給的 length 參數值為負數時，負號和冒號之間要用空格隔開。
此時 length 參數不表示要獲取的字符個數，而是表示對應 parameter 變量值從後往前的偏移，而且這個偏移是從 1 開始。
${value:5: -4} 表示從 value 變量值的第 5 個字符開始，一直獲取到倒數第 4 個字符為止，不包括倒數第 4 個字符。

當所給的 offset 參數值為負數時，負號和冒號之間要用空格隔開。
此時 offset 參數對應 parameter 變量值從後往前的偏移，而且這個偏移是從 1 開始。
${value: -7:3} 表示從 value 變量值的倒數第 7 個字符串開始，獲取包括該字符在內的三個字符，也就是 "str" 字符串。

${value: -7: -1} 表示從 value 變量值的倒數第 7 個字符串開始，一直獲取到倒數第 1 個字符為止。
包括倒數第 7 個字符，不包括倒數第 1 個字符。
具體得到的是 "string" 字符串，不包含最後的點號 ‘.’。

${#parameter}

查看 man bash 對 ${#parameter} 的說明如下：

Parameter length.

The length in characters of the value of parameter is substituted.
If parameter is * or @, the value substituted is the number of positional parameters

即，如果 parameter 變量值是字符串，則 ${#parameter} 可以獲取到對應字符串的長度。

具體舉例說明如下：

$ value="123456"

$ echo ${#value}

6

注意：在 bash 的參數擴展中，數字屬於位置參數（positional parameters），可以用數字來引用傳入腳本或者函數的參數。
當用在當前表達式時，就表示獲取所傳參數的字符串長度。
例如 $1 對應傳入的第一個參數，那麼 ${#1} 對應所傳入第一個參數的字符串長度。
具體舉例如下：

$ function param_length() { echo ${#1}; }

$ param_length 123456

6

可以看到，param_length 函數使用 ${#1} 獲取到所給第一個參數值的長度。
