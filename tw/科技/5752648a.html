HCS系列故障：報F2077,F8069，F8070等

力士樂伺服驅動控制器故障

一、 力士樂伺服驅動控制器故障
　　伺服驅動由伺服驅動控制器DKC 及程序模塊組成。伺服驅動控制器部分也是通過逆變、整流等過程，實現對伺服軸的控制。機床曾出現過伺服驅動控制器的逆變硅橋損壞的故障。故障是由於冷卻風扇將潮溼物質帶入逆變單元的硅橋中，造成短路引起的。因為冷卻風扇位於驅動器逆變單元硅橋的後方，雖然有防護網防止灰塵進入伺服驅動電路，但若電器櫃密閉不嚴，若現場環境惡劣，則風扇處極易堆積灰塵，而夏季空氣潮溼，隨著風扇的轉動，潮溼物質被帶入逆變單元的硅橋後會造成短路故障。因此注意系統保養，定期清洗系統板，清除系統冷卻風扇扇葉上的附著物，保持電器櫃內的空氣乾燥，對於降低伺服系統故障，具有重要意義。
二、上電後，程序模塊沒有任何顯示
　　ECODRIVE03伺服驅動系統中包括伺服驅動控制器DKC及FWA-ECODR3-SMT-02VS-MS程序模塊兩部分組成。而FWA-ECODR3-SMT-02VS-MS程序模塊又包括參數模塊、硬件模塊兩部分，如下圖所示。
　　由於帶有完備的自診斷功能，上電後在H1 處顯示-0、-1、-4，使系統自動進行硬件檢查。然後顯示C2-bb-AF，即命令激活--操作準備—驅動使能(進入預設的操作模式)。但機床上電後，H1處沒有任何顯示。為此需判斷是程序模塊的故障還是DKC 的問題。因機床的左、右兩主軸控制方式一致，只是參範圍不一樣，所以將FWA-ECODR3-SMT-02VS-MS硬件進行了互換。互換後，顯示F276(絕對值編碼器的值溢出)這說明驅動控制器DKC 是好的，所以更換FWA-ECODR3-SMT-02VS-MS 模塊。安裝模塊後，H1處首先顯示UL，這時需要進行參數傳輸。進行參數傳輸時需要準備：
　　計算機
　　傳輸線纜
　　力世樂伺服驅動程序
　　機床伺服參數備份盤
　　按下圖做好傳輸線纜，以便在計算機及伺服驅動器連接後，進行參數傳輸。
　　用傳輸線纜將計算機的9 針COM1 接口與DKC 的RS232 接口連接後，首先雙擊驅動程序Drive Top 13V02 圖標，在計算機屏幕上顯示
　　c on line using RS232/485(Drive)
　　c online using SERCANS
　　c offline SIMULATION
　　c No CONNECTION
　　選擇 on line using RS232/485(Drive)後，屏幕出現操作窗口，顯示 File Parameter Setup Overview Extras Options Help 菜單;然後選擇Options 菜單，下拉菜單選擇connection 項，進行通訊接口及傳輸波特率的
　　設定。選擇 COM=1 Baud=9600 ;
　　然後選擇Extras 菜單，下拉後選擇Parameter mode項;
　　然後選擇File 菜單，下拉後選擇Load，單擊“Change”鈕，改變驅動盤，若備份數據在A 盤，則選擇驅動器A 及所需文件，單擊“Load”鈕後，參數被自動裝入。
三、數據傳輸後，仍然顯示UL
　　UL表示數據不匹配。由於與MKD、MHD、MKE型伺服電機相關的電機參數被存儲在反饋數據區，上電後驅動器將根據存儲的電機參數與所連接的電機進行自動比較,如果不一樣，需重新從反饋存儲區裝入參數。此時按下模塊的S1 鍵,基本裝入命令被執行。進行在線檢測時,在Parameter菜單下可以看到Controller Type (控制版本號)及電機型號。繼續檢查參數時發現JOG 參數S-0-0033 設置錯誤,修正後,故障消除。
　　機床出現故障後，傳輸正確參數很重要，所以應在機床正常工作時，做好備份工作。
　　備份方法：進行硬件連接後，通過RS232 口進行在線通訊。選擇File菜單的Save 項，單擊“Change”鈕，改變驅動盤符，如選擇A 盤，然後單擊“Save”鈕即可。
