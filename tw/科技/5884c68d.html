概述

1.1背景

隨著市場競爭的日趨激烈，傳統的基於PC的各類信息系統已經明顯的表現出不足。比較典型的一些情況就是，銷售人員不能在銷售現場下單工作效率低下、營銷方案因領導出差不能及時審批而錯失商機等等，這些情況都嚴重的制約了企業的發展，急需得到改善。

而解決這些問題的最直接的手段，就是對企業的的各類網絡應用系統進行在手機上的延伸，實現手機辦公。當用戶在非固定場所，辦公室之外的場所，如機場、火車、輪船上、行走中，或突發情況下，可通過手機終端，在有GPRS、3G（WCDMA）等移動網絡覆蓋的情況下進行移動辦公操作，訪問OA、ERP、郵件、業務運營等系統，這種方式方便快捷、應用簡單，而且不需要改變現有信息系統。通過把客戶PC應用移動化，讓用戶突破時間和空間的限制，在享受信息科技進步的同時提升工作效率。

1.2必要性

手機辦公市場急需挖掘，並且已有客戶提出了對原有辦公系統進行手機延伸的需求。目前針對客戶需求採取項目制運作，由SI提供項目實施，該方式具有較多弊端，一是成本高，面對其他運營商不具有競爭力，二是分散式項目建設不利用後期的管理與維護，因此造成了手機辦公業務發展的瓶頸。

建設手機OA集中接入平臺可以有效地解決上述問題，並具有以下優勢：

n 降低客戶接入成本，增強競爭力 ；

n 易實施，部署週期短 ；

n 優化完善產品結構（桌面＋WAP＋客戶端）；

n 探索行業應用產業鏈多種運作方式。

1.3系統目標

依託聯通WCDMA 上網速度快、技術成熟度高的特點，通過平臺的建設為全省客戶推出手機辦公的應用產品,利用個性化的應用產品發展客戶，增加客戶黏性。其主要建設目標為：

n 平臺一期支持100個單位客戶，5000個最終使用用戶的高效運營，並支持後期的平滑擴容。

n 平臺應該具有如下功能：

手機辦公業務開通、註銷、接入鑑權、試用等的方便、快捷；

手機辦公業務開發、測試、發佈等整個流程的可管理、可控制。

1.4建設原則

1.4.1開放性

整個平臺遵循開放性架構的設計要求，採用標準開放的接口協議與開發平臺，為用戶提供統一的、開放的各種能力調用。並且整個業務的維護與發展不依賴於設備廠商，可以保證在現有平臺上對業務進行持續升級和發展。平臺建設應該充分考慮“標準和開放”的原則，要支持各種相應的軟硬件接口，使之具有靈活性和延展性。

1.4.2可靠性

支持連續7×24小時不間斷地工作。應用軟件中的任一構件更新、加載時，在不更新與上下構件的接口的前提下，不影響業務運轉和服務。

系統具備靜態過負荷控制和動態過負荷控制兩種過負荷控制的能力，能根據CPU、內存、處理能力進行過負荷控制。

軟件故障情況下，系統具備故障守護恢復機制，即當發生一般性軟件障礙時，平臺具有自糾恢復能力。當發生全平臺中斷或電源中斷恢復後，平臺能迅速地自動再啟動運行。

平臺能夠正確識別外圍系統發的錯誤請求及重複請求，避免不可預測的結果。

1.4.3安全性

手機辦公是一個無線應用業務，與傳統互聯網業務的安全機制略有不同，平臺應該提供完備的安全體系與安全技術，包括了終端數據安全、無線網絡傳輸數據加密、服務器數據安全等。

1.4.4可移植性

平臺基於成熟的J2EE架構構建，是系統能夠在不改動或少改動的情況下，就可以在不同的主流UNIX（IBM、HP、SUN等）及Linux 平臺下方便的移植。

平臺對於存儲設備、備份設備及各種網絡設備具有完全無關性。

平臺應該支持在不同主流數據庫平臺（ORACLE、INFOMIX、DB2、SYBASE等）的移植。移植時不修改業務邏輯構件，並儘可能少地修改直接操作數據庫的信息服務構件。

1.4.5高效性

平臺的數據操作應保證在規定的響應時間內完成，如：每秒完成任務數≥50次，應用適配處理效率≤1.5秒。圖片優化處理效率≥1000張/秒，辦公附件轉換引擎效率5M/秒。

1.4.6易用性

平臺部署簡單方便；操作簡單、使用方便，易維護、易管理，以減少用戶在使用中的疑問與投訴。

n 平臺應具備圖形化的可維護界面；

n 平臺具備方便且可定期執行、分析結果的業務測試功能；

n 平臺易於修改，對某一個子模塊的修改，不影響其他模塊的正常運行；

n 平臺具備自管理和監控功能，能夠實時監控各模塊的執行；

n 平臺具備在線升級協議及版本的功能，在不中斷業務的情況下支持對系統的外部接口協議進行在線升級、對修改後的系統版本進行在線升級。

2平臺體系架構

2.1平臺概述

手機OA集中接入平臺為河南聯通的企業行業客戶提供手機辦公解決方案。該產品基於聯通WCDMA網絡通訊通信能力，使用戶無論何時何地，只要有聯通無線網絡信號覆蓋的地方，就可以利用手機等移動終端設備收發郵件、信息、處理公文，進行信息溝通和數據傳遞，提高辦公工作效率。

平臺主要包括三種網元：手機辦公平臺、客戶信息系統以及手機辦公應用終端。

n 手機辦公平臺部署在聯通專業機房，面向IT基礎較好、已配置辦公自動化系統的大中型企業，為企業已有的辦公自動化系統提供移動化延伸服務。手機辦公平臺與企業信息系統可以通過Internet連接，可以通過物理專線+VPN方式連接。

n 客戶信息系統為客戶已有的需要移動化的應用系統，例如：OA，ERP，CRM等。

n 手機辦公應用終端是手機客戶端應用軟件，也可以是手機WAP瀏覽器。

3平臺業務功能

3.2.1.5.1.

3.1概述

根據系統建設目標和系統體系架構，將系統劃分為5個模塊：

n 平臺運營管理模塊，實現手機OA應用產品開發商（以下簡稱SI）、客戶、終端用戶、應用運營、客戶接入系統的管理，以及手機OA應用產品的計費和相關的統計分析。

n 應用接入支撐模塊，主要負責每個手機OA應用產品運行的技術支撐，是手機OA應用產品底層服務。

n 應用開發模塊，根據每個客戶的業務需求，完成手機OA應用產品的分配、開發、測試、發佈等。

n 平臺維護模塊，主要包括用戶權限管理、業務管理參數的配置維護，是保障平臺的平穩運行輔助性功能。

n 平臺接口模塊，完成與APN、WAP網關、客戶信息系統、VAC系統、短信/彩信網關、BSS系統、手機定位業務、視頻監控系統等的無縫集成，實現系統間數據的交互、共享。

3.2功能模塊

3.2.1平臺運營管理模塊

平臺中主要有3類單位，6類角色：

n 平臺人員，包括：平臺管理員、平臺支撐人員

n 開發商（SI），開發商管理員，開發商所屬的開發者

n 客戶，客戶管理員，客戶所屬的終端用戶

其中平臺人員是平臺建設時已經內置的角色，開發商以及客戶是逐步申請和開通的。

3.2.1.1 SI管理

平臺應提供SI的註冊功能，SI完成註冊並且審核通過後，具有如下功能：

n 可承接客戶的應用開發任務

n 可開啟多個開發者帳號，供其所屬的開發者使用；

n 可分配應用開發給所屬的開發者

3.2.1.2客戶管理

平臺提供客戶信息、權限的新建、修改、刪除等功能，同時可為客戶添所要接入的信息系統（如OA），並創建客戶管理員。

客戶管理員具有維護客戶及所屬用戶數據的功能，可完成所屬用戶手機設備號、郵箱的綁定等操作，具有查看自身應用業務日誌使用信息記錄的權限。

3.2.1.3用戶管理

平臺提供用戶基本信息的新建、修改、刪除等功能，可建立用戶、手機號碼、手機串碼等的綁定關係，同時提供用戶查詢功能。

3.2.1.4應用（產品）管理

平臺提供應用（客戶接入系統對應的手機服務端文件和手機客戶短安裝包）的上傳、映射、發佈、查詢等功能，保障客戶可以正常使用。

同時建立產品與客戶、用戶的映射關係。

3.2.1.5 鑑權中心

鑑權中心負責確認用戶是否可以使用手機OA應用（產品），即判定用戶的訂購關係是否正常。

以手機號碼、手機串號（IMEI）或移動用戶識別碼（IMSI）為介質，訂購關係為判定依據，確定是否允許接入集中平臺或使用手機辦公業務。其中：

手機號碼，來源於WAP網關或手機OA集中平臺（接入模塊）；

訂購關係， 來源於VAC系統，包括新增、停用、退定、銷戶等，以上數據可以通過FTP方式到VAC系統獲取。

產品，來源於集中平臺、WAP服務平臺（如：地市的WAP接入前置機）、其他SI服務提供平臺。

3.2.1.6短信中心

短信中心負責完成各個應用產品在實際使用過程中的短信收發功能，連接短信行業網關，可以實現對聯通手機，以及它網手機發送短信。

接入一個客戶，需要給客戶申請相對應的行業網關接口，系統提供接入行業網關的適配。

3.2.1.7接入系統管理

接入系統指的是接入平臺的客戶信息系統（如OA）。

按角色權限的不同，平臺可實現對其狀態控制：

n 平臺管理員的控制：在業務欠費、資源使用過高等情況下，平臺管理員有權停止接入業務系統的使用，平臺管理員停止後，接入業務系統處於關閉狀態。客戶管理員對處於關閉狀態的業務系統無法操作。另外，平臺管理員針對某項應用，可按照具體需求對該應用開通/關閉/新增各項通信能力，如新增短信通信能力、關閉定位能力、註銷該應用等操作。

n 客戶管理員控制：客戶管理員可針對具體情況（如應用上線、下線；發現適配BUG等），暫停、恢復接入業務系統。

平臺提供接入系統的監控功能，可查看每個客戶信息系統的接入情況，如：併發請求數和在線用戶數等。

平臺提供接入系統告警記錄、查詢功能：

n 平臺在使用中，與客戶原信息系統交互出現的異常情況以告警日誌的形式記錄下來，包括處理狀態、企業簡稱、業務系統、請求地址、請求方式、狀態碼、請求時間、描述等等，管理員可查看其詳細信息來分析異常出現的問題所在，以便快速有效的排查解決問題。

n 平臺可通過企業簡稱、應用名稱、操作用戶、請求地址、響應狀態、處理狀態、起始時間、結束時間等關鍵信息進行告警信息的查詢

3.2.1.8用戶和應用對應關係管理

根據VAC系統提供的訂購關係（一小時一個文件），在集中平臺也建立、維護相應的用戶與產品的對應關係；此對應關係供鑑權中心進行判定用戶是否可以使用產品。

用戶與產品對應關係的維護變更提供歷史軌跡的保留。

3.2.1.9統計分析

平臺可提供多類統計報表，以便各類管理員實時分析並統計平臺的各項運行維護的情況，包括但不限於以下幾類：

n 客戶活躍用戶統計報表(查詢該客戶在特定時間段內的活躍用戶信息)

n 客戶用戶數統計（可以通過客戶名稱及時間段進行檢索）

n 客戶接入應用統計（可以通過客戶名稱及時間段進行檢索）

n 註冊客戶統計（可以通過時間段進行檢索）

n 使用情況統計表（可以通過應用系統名稱及時間段進行檢索）

同時，平臺提供BSS系統所需的基礎元素，如：客戶信息、產品（應用）信息、用戶信息、使用時長、手機號碼等信息。

系統推送時，可以根據用戶是否在線，是否使用客戶端軟件，是否方便接收消息等信息綜合判斷，選擇最適合的方式將數據推送到用戶終端

3.2.3應用開發模塊

主要供手機OA應用產品開發使用。

3.2.3.1開發環境

提供全真模擬的開發環境，可使用PC模擬器進行開發，程序效果與在手機顯示、運行上保持一致，並且模擬器的分辨率，字體大小以及字間距均可以調整，這樣手機的屏幕與字體均可以模擬出來。

平臺提供了一個虛擬的測試環境，開發完成以後在平臺提供的虛擬環境中試運行，在測試階段，每個手機應用業務，平臺應提供5個用戶的免費授權，供用戶完成適配。

3.2.3.3封裝環境

開發者只需要上傳應用包，設置相關參數，選擇終端後，平臺將自動生成對應終端的程序安裝包。

3.2.3.4基礎應用子業務

n 手機拍照，可調用手機拍照功能，同時，形成圖片後可瀏覽、保存、發送。

n 短信發送，可將此功能集成到公文處理，通知，公告的整個環節，以及日程安排、郵件到達，實現及時提醒。短信的發送有兩種方式：一種是立即發送，一種是定時發送。平臺同時提供群發功能。

n 收發公文，指的是發起一個公文審批流程，或接收一個內部、外部來的公文。

n 待辦待閱，待辦用於對用戶所新增或接收到一個事物（一般指公文、申請等）進行處理，處理之後的記錄將自動從待辦列表中消失；待閱事宜是指未查看的公告、通知、郵件。查看之後的記錄將自動從待閱列表中消失。

n 手機郵件，用於郵件的查收和發佈。包括收件箱和發郵件兩部分

n 公告通知，查看、發佈公告、通知等公共信息。

n 日程安排，提供個人日程安排的新增、修改、查詢等。

n 通信錄，管理聯繫人的信息，可以搜索已有聯繫人，添加新聯繫人，一般情況下用於對單位或部門人員信息的查詢。

3.2.4平臺維護模塊

3.2.4.1管理參數配置

管理參數配置功能提供平臺管理員維護如短信網關、彩信網關、定位系統等接口系統的相關參數配置。

平臺提供對系統數據、日誌，以及業務數據的保存和刪除的時間設置。

3.2.4.2數據備份

平臺提供數據庫備份功能，可以設置備份文件存儲在服務器的路徑及文件名。

3.2.4.3日誌管理

平臺提供日誌功能，平臺日常運行的各項操作、違規行為等都會以日誌的形式記錄下來，供平臺管理員實時查看分析平臺的運行情況。

日誌類型包括：操作日誌、運行日誌、安全日誌、用戶日誌。

3.2.4.4角色權限管理

平臺可將多權限（或者策略）可以組織成為角色（Role）,角色可以賦給用戶，用戶可以擁有多個角色。

賦權操作必須由相應的管理員進行操作，同時記錄詳細的賦權操作日誌，以備日後審計使用。

3.2.4.5性能參數

平臺應提供對應服務器的性能數據（磁盤使用、CUP使用、內存使用）狀況分析以及服務器運行總時間，可實時刷新。

平臺應提供性能門限值設置，可以在平臺的服務器的性能達到告警不同門限時發出告警。如：CPU佔用率、內存佔用率和磁盤佔用率。

3.2.5平臺接口模塊

3.2.5.1客戶信息系統接口

平臺與客戶信息系統接口，主要採用表現層對接的方式，以捕獲網頁的形式，將原系統表現層的內容轉化到平臺上，再由平臺將解析的內容傳到手機終端上。

此接入方式，尤其適用於客戶基於B/S結構的信息化系統，它的最大優勢在於，無需客戶原系統開發商進行任何配合，只需在原有業務系統相關帳號及其使用權限的情況下，即可由移動辦公提供商針對業務進行頁面分析和配置，並最終將系統通過配置文件和插件的方式提供全業務的接入，整個過程，原業務系統無需任何改造。同時，新業務系統對接上線整個過程不會對以後移動化業務系統造成任何影響。

3.2.5.2 BSS接口

平臺與BSS系統的接口可採用數據庫、FTP方式，提供用戶名，企業名，手機號，套餐類型，訪問業務等各種信息，用以滿足BSS系統的數據同步、計費要求。

3.2.5.3短信接口

平臺與短信網關接口，實現對用戶的短信方式的通知，支持SGIP協議及其他業務擴展規範協議。

3.2.5.4彩信接口

平臺與彩信網關的接口，實現對用戶的彩信方式的通知。支持MMS接入，採用符合《中國聯通數字蜂窩移動通信網多媒體消息業務（MMS）CP/SP業務開發規範》的接口與MMSG互聯。

3.2.5.5手機定位系統接口

平臺與手機定位系統的接口，實現對用戶位置的獲取。接口方式待定。

3.2.5.6視頻監控系統接口

平臺與視頻監控系統的接口，實現以手機方式共享視頻監控系統的實時視頻數據。接口方式待定。

4系統安全

4.1安全策略

從以下幾方面制定安全策略來確保系統的機密性、完整性、可用性、可控性與可審查性

4.1.1機密性

確保信息不暴露給未授權的實體或進程。

4.1.2完整性

不同級別的權限維護不同數據，並且能夠判別出數據是否已被篡改。

4.1.3可用性

得到授權的實體在需要時可訪問數據，即攻擊者不能佔用所有的資源而阻礙授權者的工作

4.1.4可控性

可以控制授權範圍內的信息流向及行為方式

4.1.5可審查性

對出現的網絡安全、數據安全問題提供調查的依據和手段

4.2網絡安全

在網絡傳輸設計中，安全可靠性能主要有以下方面：

4.2.1訪問控制

為了保障系統安全，避免非法入侵，需要由防火牆將系統與外部不可信任的訪問隔離，對與外部網絡交換數據的內部網絡及其主機、所交換的數據進行嚴格的訪問控制

4.2.2數據加密

數據加密是在數據傳輸、存儲過程中防止非法竊取、篡改信息的有效手段。

在平臺服務器與手機客戶端之間，平臺必須提供基於數字證書的SSL傳輸加密機制，數字證書用以表明客戶所屬員工的網上身份。

根據客戶需要，移動辦公服務器與企業系統之間，可以通過物理專線接入，同時附加虛擬專用網（VPN）的傳輸層安全機制，建立臨時安全連接，以保證數據的安全傳輸。避免業務數據經過Internet所造成的風險。

4.2.3安全審計

安全審計是識別與防止網絡攻擊行為、追查網絡洩密行為的重要措施之一。具體包括兩方面的內容，一是採用網絡監控與入侵防範系統，識別網絡各種違規操作與攻擊行為，即時響應（如報警）並進行阻斷；二是對信息內容的審計，可以防止機密或敏感信息的非法洩漏。

4.3硬件與設備安全

重要部件熱插拔，系統地的應用服務器、文件服務器、數據庫服務器等的硬盤都應支持熱插拔，當出現故障時，可以在不斷電、不停機的情況下替換這些部件。從而保證局部的故障不會影響整個系統的運行，同時也方便的系統的維護。

4.4應用安全機制

平臺可以提供多種應用層安全手段的組合使用，提供多種應用安全機制，具體包括：

4.4.1多種身份校驗機制

可根據需要將用戶賬號密碼與手機號碼，設備號綁定，提供多重身份校驗機制。

4.4.2炸彈消息

當用戶丟失手機或手機使用權發生更變的時候，手機OA平臺可推送出炸彈消息，利用炸彈消息清空用戶手機中關於手機辦公相關的所有資料。

炸彈消息的原理是平臺向用戶手機發送二進制短信激活手機端程序來清空手機中手機辦公相關資料。

當用戶丟失手機或手機使用權發生更變時，可到平臺管理員處登記，由管理員通過平臺發出炸彈消息。

4.4.3停用鎖止

當用戶手機丟失，可報告管理員，由管理員在系統中將丟失手機列入黑名單，平臺鎖定用戶所有的數據，並阻止用戶登錄。保證用戶的機密信息不會洩漏。

4.4.4防密碼窮舉攻擊

平臺提供必要的的機制，如：只允許用戶每次登錄嘗試三次密碼，假如還不對，將自動中止與用戶的連線。使得無法完成對用戶密碼的窮舉攻擊。
