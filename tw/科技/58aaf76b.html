Flink 基礎學習(七) 窗口 Window

2020-10-29

1 前言

前面講了時間 Time 的概念和實際解決問題後，本篇來看下經常搭配使用的另一個關鍵工具：窗口 Window。

窗口也有三種類型可供選擇使用：

  • Tumbling Windows：滾動窗口
  • Sliding Windows：滑動窗口
  • Session Windows：會話窗口

友情提示，本篇主要翻譯自官網以及參考了 wuchong 大神的博客，內容比較乾貨，介紹這三種窗口的概念以及使用場景，希望看完能對 Flink 的窗口概念加深理解。

2 Window 窗口是什麼

Windows 是處理無限流的核心。Windows 將流分成有限大小的“存儲桶”，我們可以在其上應用計算。Flink 是一個優秀的流計算引擎，數據是源源不斷的，它認為批處理 Batch 是一種特殊的流計算，在流中分割出一個個窗口，每個窗口相當於有限大小的空間，匯聚了待處理的數據。

我們舉個具體的場景來形象地理解不同窗口的概念。假設，淘寶網會記錄每個用戶每次購買的商品個數，我們要做的是統計不同窗口中用戶購買商品的總數。下圖給出了幾種經典的窗口切分概述圖（圈中的數字代表該用戶本次購買的商品個數）：



從上圖能夠看出，Raw stream data 是原始數據流，數據輸入方向為【從左到右】，根據左側不同的窗口分類，將會得到右邊相應的窗口，依據設定的窗口大小和時間間隔，每個窗口將會蒐集到相應的數據（一個顏色塊表示一個窗口）。

下面將分別介紹這三種窗口類型，其中計數 Count 和時間 Time 窗口可以選擇使用滾動 Tumbling 或滑動 Sliding 功能，區別在於統計窗口計算的觸發時機和窗口是否有重疊。

3.1 Time Windows

顧名思義，Time Window 按時間分組流元素。例如，一分鐘的滾動時間窗口將收集一分鐘的元素，並在一分鐘後將功能應用於窗口中的所有元素。

在前面的文章中講過 Flink 中的時間概念，分別是 Event Time 事件時間、Processing Time 處理時間和 Ingestion Time 注入時間，其中 Event Time 可能引起的亂序事件，使用 Watermark 去兼容處理，在 Flink 中，時間類型和窗口機制是解耦的，所以設定不同的時間類型，不需要修改窗口的計算邏輯，就能達到想要的計算結果。

3.1.1 Tumbling Time Window 滾動時間窗口

例如下面例子中，表示蒐集 5s 內的事件，做統計 sum 計算。

DataStream<T> input = ...;// 時間屬性的設定，EventTime 模式，用基於數據中自帶的時間戳env.setStreamTimeCharacteristic(TimeCharacteristic.EventTime);input    .keyBy(<key selector>)    .window(TumblingEventTimeWindows.of(Time.seconds(5)))    .sum(1);

在上面時間屬性的設定 TimeCharacteristic.EventTime，需要與後面算子 window 操作的 TumblingEventTimeWindows.of(Time.seconds(5)) 保持一致。後面例子中，將會省去時間屬性的設定，貼出核心的代碼

3.1.2 Sliding Time Window 滑動時間窗口

有時候我們會需要每 5min 後，統計前面 60min 內事件的聚合值。

這時需要滑動窗口，保存上一次時間窗口的輸入值，給下一個窗口使用重疊的部分。所以一個數據在滑動窗口中，可以出現在多個窗口中。

DataStream<ItemViewCount> windowData = pvData    .keyBy("itemId")    .timeWindow(Time.minutes(60), Time.minutes(5))    .aggregate(new CountAgg(), new WindowResultFunction());

3.2 CountWindows

Apache Flink 還具有計數窗口。滾動計數窗口為 100 時，將在一個窗口中收集 100 個事件，並在添加第 100 個元素時觸發該窗口的計算。

在 DataStream API 中，滾動和滑動計數窗口的定義如下：

3.2.1 Tunmling Count Window

例如每次想要統計 100 個事件的合計值，只要從數據源處，累加滿 100 個事件就觸發計算，可以使用下面的 API

 DataStream<WordWithCount> windowCounts = text    .flatMap(...)    .keyBy("word")    .countWindow(100)    .sum(1);

3.2.2 Sliding Count Window

在一些場景下，有可能來了 5 個數據後，要統計前 100 個數據的合計值，每個數據可以被分配到多個窗口中，那麼就可以使用滑動窗口實現。

 DataStream<WordWithCount> windowCounts = text    .flatMap(...)    .keyBy("word")    .window(SlidingEventTimeWindows.of(5, 100)    .sum(1);

3.3 Session Window

Session Window 會話窗口分配器 Assigner 按活動會話對元素進行分組。

與 Tumbling 滾動窗口和 Sliding 滑動窗口相比，會話窗口不重疊且沒有固定的開始和結束時間。

相反，當會話窗口在一定時間段內未接收到元素時，即在發生不活動間隙時，它將關閉。會話窗口分配器可以配置有靜態會話間隔，也可以配置有會話間隔提取器功能，該功能定義不活動的時間長度。 當此時間段到期時，當前會話將關閉，隨後的元素將分配給新的會話窗口。

DataStream<T> input = ...;input    .keyBy(<key selector>)    .window(EventTimeSessionWindows.withGap(Time.minutes(10)))    .<windowed transformation>(<window function>);

其中，withGap 方法中的入參，表示會話間隔的大小，表示超過這個時間沒有新數據進入，該會話就會關閉，之後的數據就會進入下一個窗口。

4 Window 開放的三大核心 API

核心有三個組件：窗口分配器 Window Assigner 、觸發器 Trigger、’驅逐者’ Evictor，下面來 一一介紹

4.1 Window Assigner

該組件主要功能是決定數據該分發到哪個窗口，它的作用可以類比於 Spring MVC 中的 Dispatcher。



上圖左側是 Flink 窗口實現的包結構，三大組件在對應的目錄下，清晰明瞭。

底部是 Window Assigner 的繼承類，在調用 WindowedStream<T, KEY, W> window(WindowAssigner<? super T, W> assigner) 這類方法時，可以傳入上述的窗口分發器，在裡面實現自定義的窗口分發邏輯。

4.2 Trigger

每個窗口都有一個觸發器，該觸發器決定何時評估或清除該窗口。

對於每個插入到窗口中的元素以及先前註冊的計時器超時時，將觸發該觸發器。

上圖展示了觸發器的繼承類，從中可以看出，它可以根據時間或者計數來觸發，表示這個窗口的數據已收集完成，可以觸發計算邏輯。

4.3 Evictor

直譯為 ‘驅逐者’，作用類似於過濾器 fliter，在 trigger 後執行，如果設定了 evictor，將會去除不符合條件的數據（默認是不設定的，不會驅逐）



通過這三大組件，可以實現自定義窗口邏輯，決定數據如何分配、何時觸發計算以及哪些數據要被提前去除，詳細使用例子可以參考官網的示例：https://flink.apache.org/news/2015/12/04/Introducing-windows.html

5 剖析 Window 實現機制

前面介紹了三大組件，接下來就來看下在實際應用中，它們的流程是如何串起來。

Flink 的內置時間和計數窗口涵蓋了各種常見的窗口用例。

但是，當然有些應用程序需要自定義窗口邏輯，而 Flink 的內置窗口無法解決這些邏輯。為了也支持需要非常特定的窗口語義的應用程序，DataStream API 公開了其窗口機制內部的接口。

這些接口可以非常精細地控制窗口的構建和評估方式。下圖描述了 Flink 的窗口機制，並介紹了其中涉及的組件。



到達窗口運算符的元素將傳遞給 WindowAssigner。 WindowAssigner 將元素分配給一個或多個窗口，可能會創建新窗口。窗口本身只是元素列表的標識符，並且可以提供一些可選的元信息，例如在使用 TimeWindow 時的開始和結束時間。請注意，可以將元素添加到多個窗口，這也意味著元素可以同時存在於多個窗口（圖中的元素 6 就存在於第一個和第三個窗口中）。

每個窗口都有一個觸發器 Trigger，該觸發器決定該窗口何時被計算或清除。對於每個插入到窗口中的元素以及先前註冊的計時器超時時，將觸發對應的 Trigger。

對於每個事件，觸發器都可以決定觸發（即計算），清除（刪除窗口並丟棄其內容），或者觸發然後清除窗口。僅觸發的觸發器會計算窗口並保持其原樣，即所有元素保留在窗口中，並在下次觸發時再次計算。一個窗口可以被計算多次，並且一直存在，直到被清除為止。請注意，在清除窗口之前，它會一直消耗內存。

觸發 Trigger 時，可以將窗口元素列表提供給 Evictor （如果有的話）。驅逐者可以遍歷列表，並決定從列表的開頭刪除一些元素，即刪除一些首先進入窗口的元素。 其餘元素進入到下一步計算函數 Evaluation Function。 如果未定義 Evictor，則觸發器將所有窗口元素直接移交給計算函數。

關於驅逐者 Evictor，可以參考綠色線（元素 5 被移除了）和橙色線（沒有定義 Evicotr，所有元素流入計算函數）。

Evaluation Function 計算函數接收一個窗口的元素（可能由 Evictor 過濾），併為該窗口計算一個或多個結果元素。DataStream API 接受不同類型的計算函數，包括預定義的聚合函數，例如 sum（），min（），max（） 以及 ReduceFunction，FoldFunction 或 WindowFunction。WindowFunction 是最通用的計算函數，它接收窗口對象（即窗口的元數據），窗口元素列表以及窗口鍵（在鍵控窗口的情況下）作為參數。

以上就是一個完整的窗口計算流程，經歷了 Window Assigner -> Trigger -> Evictor -> Evaluation Function 的過程，最終獲得結果。

6 使用例子

這裡需要翻出前面寫的 HelloWorld 的例子，參考以下 Demo


public class SocketWindowWordCount {    public static void main(String[] args) throws Exception {        String hostName = "127.0.0.1";        int port = 9000;        final StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();        DataStream<String> text = env.socketTextStream("localhost", port, "\n");        DataStream<WordWithCount> windowCounts = text                .flatMap(new FlatMapFunction<String, WordWithCount>() {                    @Override                    public void flatMap(String value, Collector<WordWithCount> out) {                        for (String word : value.split("\s")) {                            out.collect(new WordWithCount(word, 1L));                        }                    }                })                .keyBy("word")                .timeWindow(Time.seconds(5), Time.seconds(1))                .reduce(new ReduceFunction<WordWithCount>() {                    @Override                    public WordWithCount reduce(WordWithCount a, WordWithCount b) {                        return new WordWithCount(a.getWord(), a.getCount() + b.getCount());                    }                });        // print the results with a single thread, rather than in parallel        windowCounts.print().setParallelism(1);        env.execute("Socket Window WordCount");    }}

打開 9000 端口，往窗口發送數據，程序接收到數據進行解析，接著使用 timeWindow 時間窗口蒐集數據，窗口大小為 5s，滑動統計的時間間隔為 1s，於是就有了下圖的輸出結果：



每秒輸出當前 5s 內蒐集到的數據，時間窗口統計正確運行。

7 總結

Time 時間和 Window 窗口是 Flink 的亮點，所以這兩篇對它們的學習是十分必要的，瞭解它們的概念和原理，可以更好的去使用它們。

Flink 是一個流處理器，提供了非常強大的運算符，幫助我們計算、聚合數據。同時提供了窗口機制，將流劃分成一個個區間，對於區間，也就是窗口中的元素進行計算，達到批處理的作用。
