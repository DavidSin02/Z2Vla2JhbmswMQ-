一、基本網絡結構

1.1 整體架構

5G系統由接入網（AN）和核心網（5GC）組成（38.300）。若考慮NSA（非獨立組網）場景，則還需要考慮4G的網元。

AN有兩種：

gNB, 為UE提供NR用戶面和控制面協議終結點。

ng-eNB, 為UE提供E-UTRA的用戶面和控制面協議的終結點。

1.3 AN網絡結構（38.401）

每個邏輯gNB 由一個gNB-CU和若干個gNB-DU組成。每個gNB-CU和gNB-DU通過F1邏輯接口連接。

圖3：gNB Overall architecture

一般來說一個gNB-DU只連接一個gNB-CU。但是為了實現的靈活性，每個gNB-DU也可能連接到多個gNB-CU。

一個gNB CU中的控制面和用戶面是分離。一般只有一個CP，但是允許有多個UP。要注意的是，gNB-CU及連接的若干gNB-DU作為一個整體邏輯gNB對外呈現的，只對其他的gNB和所相連的5GC可見。

二、系統結構（23.501）

2.1 系統設計原則

  1. 5G系統結構採用NFV/SDN以支持數據連接和業務靈活部署, 促使基於業務的控制面網絡功能和概念互動:
  2. 用戶面功能和控制面功能分離, 允許獨立的可擴展性,可演進性及可靈活部署.比如可選擇採用集中式或者分佈式的方式.
  3. 功能設計模塊化, 比如採用靈活高效的網絡切片.
  4. 無論在什麼地方適用,都可以將相應的過程(網絡功能之間的互動)定義為業務, 以便可以複用他們.
  5. 如果有需要都可以使相應的網絡功能和其他的NF互動. 5G結構並沒有排除中間功能幫助路由控制面信息. (比如 DRA).
  6. 縮小接入網絡(AN)和核心網絡(CN)的依賴性.這個結構是一個公共接入網AN連接匯聚的核心網. 而AN可以是不同的接入類型比如3GPP接入網和non-3GPP接入網.
  7. 支持統一的鑑全架構.
  8. 支持 “stateless” 網絡功能NF, 這些網絡功能的計算資源和存儲資源解耦.
  9. 支持能力開放
  10. 同時支持本地和集中化的業務. 為了支持低延遲業務並訪問本地數據網絡,用戶面功能部署需要儘可能的靠近介入網絡AN.
  11. 在VPLMN支持基於LBO和歸屬地路由的漫遊方式.

2.2 設計的功能實體

5G系統結構定義瞭如下網元功能實體 (NF)：

  1. UE: 用戶終端設備User Equipment (UE)
  2. RAN: 接入網絡(Radio) Access Network (RAN)
  3. AMF: 接入及移動性管理功能Access and Mobility Management Function
  4. UPF: 用戶面功能User plane Function (UPF)
  5. AUSF: 鑑權服務功能Authentication Server Function
  6. DN: 數據網絡Data Network (DN), 比如運營商業務，互聯網接入或者第三方業務等。
  7. UDFS: 非結構性數據存儲功能Unstructured Data Storage Function
  8. NEF: 網絡業務呈現功能Network Exposure Function (NEF)
  9. NRF: 網元數據倉庫功能NF Repository Function (NRF)
  10. NSSF: 網絡切片選擇功能Network SliceSelection Function (NSSF)
  11. PCF: 策略控制功能Policy Control function (PCF)
  12. SMF: 進程管理功能Session Management Function (SMF)
  13. UDM: 統一數據管理功能Unified Data Management (UDM)
  14. UDR: 統一數據倉庫功能Unified Data Repository (UDR)
  15. AF: 應用層功能Application Function (AF)
  16. EIR: 5G設備標識註冊設備5G-Equipment Identity Register (5G-EIR)

2.3 設計的接口

2.3.1 基於業務service-based，控制面相關的網元功能實體能夠授權其他網元來訪問它的業務。

5G系統結構中包含如下基於業務service-based的接口

  1. Namf: Service-basedinterface exhibited by AMF.
  2. Nsmf: Service-basedinterface exhibited by SMF.
  3. Nnef: Service-basedinterface exhibited by NEF.
  4. Npcf: Service-basedinterface exhibited by PCF.
  5. Nudm: Service-basedinterface exhibited by UDM.
  6. Naf: Service-basedinterface exhibited by AF.
  7. Nnrf: Service-basedinterface exhibited by NRF.
  8. Nnssf: Service-basedinterface exhibited by NSSF.
  9. Nausf: Service-basedinterface exhibited by AUSF.
  10. Nudr: Service-basedinterface exhibited by UDR.
  11. Nudsf: Service-based interfaceexhibited by UDSF.
  12. N5g-eir: Service-based interface exhibited by 5G-EIR.

2.3.2 基於參考點reference point ，點對點網元功能業務之間互作用點。

5G結構中定義瞭如下參考點：

  • N1: Referencepoint between the UE and the AMF.
  • N2: Referencepoint between the ®AN and the AMF.
  • N3: Referencepoint between the ®AN and the UPF.
  • N4: Referencepoint between the SMF and the UPF.
  • N6: Referencepoint between the UPF and a Data Network.
  • N9: Referencepoint between two UPFs.
  • N5: Referencepoint between the PCF and an AF.
  • N7: Referencepoint between the SMF and the PCF.
  • N8: Referencepoint between the UDMand the AMF.
  • N10: Referencepoint between the UDM and the SMF.
  • N11: Reference point between the AMF andthe SMF.
  • N12: Reference point between AMF andAUSF.
  • N13: Reference point between the UDM andAuthentication Server function the AUSF.
  • N14: Reference point between two AMFs.
  • N15: Referencepoint between the PCF and the AMF in case of non-roaming scenario, PCF in thevisited network and AMF in case of roaming scenario.
  • N16: Reference point between two SMFs,(in roaming case between SMF in the visited network and the SMF in the homenetwork).
  • N17: Referencepoint between AMF and 5G-EIR.
  • N18: Referencepoint between any NF and UDSF.
  • N22: Reference pointbetween AMF and NSSF.
  • N24: Referencepoint between the PCF in the visited network and the PCF in the home network.
  • N27: Reference pointbetween NRF in the visited network and the NRF in the home network.

3.3 空口

3.4 UE和5GC接口

N1 NAS信令的終結點為UE和AMF，一個NAS信令連接用於註冊管理/連接管理(RM/CM)和會話管理（SM）。NAS協議由NAS-MM和NAS-SM兩部分組成；此外UE和5GC間還有多個其它協議（SM、SMS、UE policy、LCS等），這都協議都是通過N1 NAS-MM進行透傳的。

RM/CM NAS消息和其它類型的NAS消息是解耦的，也就是AMF負責RM/CM，其它的消息就透傳給對應的模塊去處理。

位於AMF的NAS-MM負責：

1）維護處理RM/CM的狀態和對應流程處理，

2）提供安全的NAS消息傳輸通道（也即NAS層的加密和完保），

3）透傳其它類型的NAS消息（SM、SMS、UE Policy、LCS）。

如果UE同時通過3GPP和non-3GPP接入網接入5GC，那麼每個接入模式下都有一個N1 NAS信令連接。

3.4.1 UE—AMF

NAS-MM: NAS-MM協議負責註冊管理、連接管理、用戶面連接的激活和去激活操作，負責NAS消息的加密和完保。5G NAS協議定義在 TS 24.501.

5G-AN Protocol layer: 接入網的協議棧取決於具體的接入網類型；如果從eNB接入，則對應的空口協議棧定義在TS36.300，如果從gNodeB接入，則對應的空口協議定義在TS38.300，如果從non-3GPP網絡接入，則對應的協議棧定義在TS23.501 8.2.4章節。

3.4.2 UE—SM

NAS-SM: NAS-SM消息支持用戶面PDU會話的建立、修改、釋放；NAS-SM消息通過AMF傳輸，且其對AMF是透明的（也就是AMF負責透傳SM消息、不對其進行解析處理）。具體的消息和流程見於協議TS 24.501。

3.4.2 UE PDU會話的用戶面協議棧

N3口使用GTP-U協議，這和3G/4G一樣，GTP-U定義於TS29.281；5G-AN用戶面協議棧取決於具體的接入網類型。
