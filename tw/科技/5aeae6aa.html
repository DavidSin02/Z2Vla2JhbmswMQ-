分析MTK校準調試的原理

2020-11-14
| 科技

一般產線校準主要是4種：

1、電池電壓校準（ADC）

2、26MHz AFC校準

3、RX校準

4、TX 功率校準

1、電池電壓校準（ADC）

通過電池電壓校準主要是讓系統知道當前的電池電壓是多少，這樣系統可以正確的表示出顯示出電池格數圖標。MTK方案在校準電池電壓的時候，取電源的兩個電壓值：4.2V和3.4V，利用這兩個值對應的ADC值，來算出斜率，通過固定的算法來計算出各個電壓時對應的ADC值。

2、26MHz AFC校準

目的：使得在室溫下TCXO穩定工作在26MHz情況的ADC值和斜率slope（Hz/ADC）。

由於TCXO本身就有誤差，再加上老化等原因，TCXO的輸出不會在26MHz，因此需要通過適當改變基帶IC控制信號AFC的電壓（ADC值），使得TCXO能工作在26MHz，滿足ETSI規範要求。

具體算法：

1）、在test模式下，打開TX，設置ARFCN

2）、設置DACmin、DACmax，並計算出對應delta Fmin和delta Fmax

3）、根據上圖可以計算出相應的斜率slope

4）、把DAC值在DACmin和DACmax中移動，計算出最小的delta F和對應的DAC

5）、保存DAC和slope

3、RX校準

RX校準：包括兩部分：RX Gain和信道補償

信道補償主要是因為在高、中、低信道的時候RX Gain會有一定的差異，因此需要一定的補償來彌補這種差異，免得在高、低信號的值接近甚至超出規範的範圍。

但是在產線具體生產的時候由於RX校準的信道補償計算量比較大，比較耗時，一般不做這項校準。

RX Gain校準：

基於下圖的線性關係，每一RX Input Level都有相對的RX Lev的值：－110＝RX Input Level－RX Lev。

具體實現方式：

按照兩條直線來分解，目的是因為小功率的時候誤差相對比較大。分解點為

（－70，40）上

line1：x<=-70

line2：x>=70

依據：y=ax+b，只要知道兩個點就能算出a和b

注意：

校準RX只是讓RX Lev的值與RX Input level 的值完全對應，只要補償的值在一定範圍內就不會影響系統的靈敏度

4、TX 功率校準：包括功率等級校準、信道補償、溫度補償、電壓補償等

溫度補償和電壓補償一般產線不實行，一般通過統計然後把補償的ADC值直接寫道程序中，以節約生產時間。

信道補償只是對最大功率等級有效。

功率等級校準：不同的PA一般校準算法也不同。

我們知道，在GSM中，控制手機PA在不同等級的信號是基帶IC控制信號Vapc，每一等級的發射功率都有對應的Vapc，Vapc大表示功率大。因此我們需要校準Vapc的offset值，使得發射功率到目標功率值上去。

Vapc的波形：

每一個Vapc的ADC值都會有對應的PA輸出功率值，一般情況下這種關係不是線性關係（ADL5552是線性關係的，因此校準算法相對來說比較簡單），如RFMD的PA。

我們以RFMD的PA為例，來說明功率等級校準的算法：

1）、通過一下公式把PA的輸出功率轉換：

使得Pout[V]與Vapc的ADC值在一定範圍 內變成線性關係

這樣就可以算出slope：

slope＝

（Pou[V]2-Pou[V]1）/（ADC1- ADC2）

一般GSM：ADC1為PL17對應的ADC值

ADC2為PL6對應的ADC值

DCS：ADC1為PL13對應的ADC值

ADC2為PL1對應的ADC值

然後計算出每一等級的誤差功率：

Error[V]=

Targer_power[V]－Measured_power[V]

計算出每一功率等級對應的offset值：

PL_offset=Erro[V]/slope

把PL_offset 保存到EEP中
