C++｜類封裝如何提升安全和可維護性，以智能指針封裝裸指針為例

2020-11-14
| 科技

我們知道，對於C風格的動態字符串，需要程序員自己管理內存，也沒有直觀的操作符運算（只能使用C風格的string函數），封裝成類就不一樣的，字符串類簡單一點來說就是封裝一個字符指針以及一些成員函數和運算符重載函數，可以有邊界檢查。複雜一點就是再封裝一個智能指針，如包含引用計數的智能指針。

智能指針就是對裸指針的封裝，用來解決內存洩漏的問題。我們知道類的析構可以自動清理動態內存（當然要有顯式的delete(或free)語句包含在內，類對象在離開作用域後會自動調用析構函數），作用域可以自動管理棧上內存，如果一個棧上的指針指向了一個堆上的動態內存塊，如何利用類的析構與棧上內存作用域的機制來避免內存洩漏？再考慮到堆上動態內存塊是否共享的問題，這就是auto_ptr、unique_ptr、shared_ptr等智能指針的概念。

如果一個棧上的指針（如nudePtr）指向了一個堆上的動態內存塊，如果沒有delete(或free)，當nudePtr離開作用域後，釋放了nudePtr本身，並沒有釋放堆上的內存（需要delete(或free)）。如果將nudePtr封裝成smartPtr（模板類實現），當smartPtr指向堆上內存塊時，在smartPtr離開作用域後，作為類對象，會自動調用析構函數，如果在smartPtr的析構函數中有delete(或free)釋放掉堆上內存，則內存洩漏就不是問題了，當然還有堆上內存共享的問題（通過複製、賦值產生的），此內存所有權的問題有以下方法：

引用計數

寫時複製、賦值即釋放對象擁有權限

控制權限轉移（移動語義）

擁有不同的方法對應著不同類型的智能指針，比如C++11在STL中引入的shared_ptr就實現了引用計數的功能，已經被C++11摒棄的auto_ptr實現了賦值即釋放對象擁有權限，C++11引入的unique_ptr則實現了控制權限的轉移功能。

auto_ptr 主要有兩大問題：

複製和賦值會改變資源的所有權，不符合人的直覺。

在 STL 容器中無法使用auto_ptr ，因為容器內的元素必需支持可複製（copy constructable）和可賦值（assignable）。

unique_ptr通過移動語義來解決上述問題。

以下就是一個unique_ptr實現的簡單雛形：

#include <iostream> template<class T>class Unique_ptr{	T* m_ptr;public:	Unique_ptr(T* ptr = nullptr):m_ptr(ptr)	{} 	~Unique_ptr()	{		delete m_ptr;	} 	// Copy constructor -- no copying allowed!	Unique_ptr(const Unique_ptr& a) = delete; 	// Move constructor	// Transfer ownership of a.m_ptr to m_ptr	Unique_ptr(Unique_ptr&& a)		: m_ptr(a.m_ptr)	{		a.m_ptr = nullptr;	} 	// Copy assignment -- no copying allowed!	Unique_ptr& operator=(const Unique_ptr& a) = delete; 	// Move assignment	// Transfer ownership of a.m_ptr to m_ptr	Unique_ptr& operator=(Unique_ptr&& a)	{		// Self-assignment detection		if (&a == this)			return *this; 		// Release any resource we're holding		delete m_ptr; 		// Transfer ownership of a.m_ptr to m_ptr		m_ptr = a.m_ptr;		a.m_ptr = nullptr; 		return *this;	} 	T& operator*() const { return *m_ptr; }	T* operator->() const { return m_ptr; }	bool isNull() const { return m_ptr == nullptr; }}; class Resource{public:	Resource() { std::cout << "Resource acquired\n"; }	~Resource() { std::cout << "Resource destroyed\n"; }}; int main(){	Unique_ptr<Resource> res1(new Resource);	Unique_ptr<Resource> res2; // Start as nullptr 	std::cout << "res1 is " << (res1.isNull() ? "null\n" : "not null\n");	std::cout << "res2 is " << (res2.isNull() ? "null\n" : "not null\n"); 	//res2 = res1; // auto_ptr 	Resource* nudePtr(new Resource);// 裸指針	delete nudePtr; //如果忘記delete，則nudePtr超出作用域時自動釋放，	// 釋放的是自身，其指向的動態內存卻再也沒有機會釋放，造成動態內存洩漏	// res2超出作用域時會自動在棧上釋放，同時會調用析構函數，delete m_ptr，釋放堆上內存	// 封裝的指針（智能指針）在釋放自身（棧）的同時釋放了其成員指針指向的動態內存（堆內存）	// 裸指針釋放自身（棧）的同時卻讓其指向的動態內存失去了釋放的機會（如果沒有delete）	// delete的不是nudePtr本身（本身超出作用域會自動在棧上釋放），	// delete針對的是其指向的堆內存，其實本質上也沒有delete，只是標識為可重新鏈接的堆內存。	return 0;}/*Resource acquired res1 is not null res2 is null Resource acquired Resource destroyed Resource destroyed */
