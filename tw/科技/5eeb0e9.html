PLC故障排查經驗和技巧，一步步教你輕鬆搞定PLC維修

2020-10-29
| 科技

今天小編教大家一些查找PLC故障的方法和技巧，小編相信好的方法和技巧能夠讓你在查找PLC故障的時候事半功倍。


一、PLC的故障類型

第一，外部設備故障

首先第一，外部設備是什麼？就是指與PLC工作過程直接聯繫的各種開關、 傳感器、 執行機構、 負載等。

如果這些設備發生故障,就會直接影響 PLC 系統的控制功能。

所以維修 PLC的故障之前 ,首先要分清是外部設備故障,還是 PLC 本身故障。

第二，系統故障

  1. 系統故障：影響 PLC 系統運行的全局性故障。
  2. 系統故障：分為固定性故障、偶然性故障。
  3. 故障發生之後,如果可以重新啟動使系統恢復正常,不然就是偶然性故障。
  4. 固定性故障：一般需要更換硬件或軟件, 系統才能夠恢復正常。

第三，硬件故障

硬件故障：系統中的模板、電路損壞而造成的故障。

第四，軟件故障

軟件故障包含軟件錯誤、操作錯誤等。

軟件故障一般可以通過本身 的自診斷測試功能或者軟件來查看。

所以，可以看到一般PLC常見故障有電源指示燈不亮，PLC人機無法通訊，PLC IO模塊燈亮異常,PLC觸點無輸出,PLC POWER電源指示燈不亮，PLC模組指示燈不亮,PLC無電源，PLC不通電等。


二、查找一般的PLC故障的基本步驟

維修時,先插好編程器,將開關調到 RUN 位置,然後根據六步法步驟查找:

（1）假如該信號操控在一個計數器 ，首要查看操控復位的邏輯，然後是計數器信號。按上述2到5步進行。

（2）假如施耐德PLC中止在某些輸出被鼓勵的當地，一般是處於中間狀況，則查找引起下一步操作發作的信號（輸入，定時器，線川，鼓輪操控器等）。編程器回顯現那個信號的ON/OFF狀況。

（3）輸入信號，將編程器顯現的狀況與輸入模塊的LED指示作比較，結果不共同，則替換輸入模塊。假如發現在擴展架上有多個模塊要替換，那麼，在替換之前，應先查看I/O擴展電纜和她的銜接情況。

（4）假如輸入狀況與輸入模塊的LED指示指示共同，就要比較一下發光二極管與輸入設備（按鈕、限位開關等）的狀況。假如二者不同，丈量一下輸入模塊，如發現問題，需求替換I/O設備，現場接線或電源；不然，要替換輸入模塊。

（5）假如信號是定時器，並且停在小於999.9的非零值上，則要替換CPU模塊。

（6）如信號是線川，沒有輸出或輸出與線川的狀況不同，就得用編程器查看輸出的驅動邏輯，並查看程序清單。查看應按從左到右進行，找出N個不接通的觸點，如沒有通的那個是輸入，就按第二個和第三步查看該輸入點，日是線川，就按第四步和第五步查看。要承認使主控繼電器不影響邏輯操作。

三、PLC系統維護與故障排除的基本流程

總體診斷：根據總體檢查流程圖，找出故障點的大致方向,然後逐漸細化找出具體的PLC故障。

第一，電源故障診斷

如果電源燈不亮需要對供電系統以及電源燈本身進行檢查。

第二，運行故障診斷

電源正常, 運行指示燈不亮, 則說明系統可能因某種異常原因而終止正常運行。

第三，輸入輸出故障診斷

  1. 輸入輸出單元是 PLC 與外部設備進行信息交流的通道;
  2. 輸入輸出是否正常工作,還與連接配線、接線端子、熔斷器等元器件狀態有關。


四、維修的基本順序

第一，先問再做

  1. 維修 PLC 時,不要立即直接動手,而是先詢問故障發生前後的情況、故障現象;
  2. 如果對生疏的 PLC 維修,應先了解其工作原理。

第二，先清再修

在維修PLC 之前,先打開機器清潔 PLC之後，再進行維修工作 。

第三，PLC檢查先外後內

維修PLC 時,先檢查外部現象與原因,如果外部正常,則然後檢查 PLC 內部。

第四，先無後通

在沒有通電的情況下，判斷熔絲是否損壞，在不通電的情況下就可以判斷出故障點。

如果不能夠通過這種方法判斷出問題的話,再通電進行檢查 PLC的各項 。
