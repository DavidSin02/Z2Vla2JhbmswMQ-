我國嵌入式技術及應用現狀分析

2020-10-29
| 科技

自上個世紀七十年代提出嵌入式系統的概念以來，嵌入式系統以其高性能、低功耗、低成本和小體積等優勢，獲得了飛速的發展與廣泛的應用，然而，由於受到當時硬件水平的制約，嵌入式系統的軟件開發方式與以 PC 為代表的通用計算機系統的軟件開發方式一開始就完全不同。當時的嵌入式系統硬件平臺一般都是基於 8 位機的簡單系統，系統的硬件設計者通常也是系統的軟件開發者，編程語言絕大多數也以彙編語言為主，硬件設計者在進行硬件系統的設計時，同時會考慮程序如何編寫，而在進行程序設計時，也會考慮與硬件的配合，因此，嵌入式系統的設計開發人員一般都非常瞭解系統的底層軟、硬件細節。

由於軟、硬件條件的制約，這些系統一般都用於實現一個或幾個簡單的數據採集與控制功能，程序結構多使用簡單的控制循環，配合中斷服務來實現對外界請求的相應，不採用操作系統，所以，系統開發者對通用計算機基於操作系統的軟件開發工具和技術的理解與掌握比較有限。而與此同時，另一大批人則對以 Microsoft Windows 、 Linux 為代表的操作系統技術深深著迷，醉心於在這些操作系統上的應用程序開發，他們感興趣的東西是如何使用諸如 VB 、 VC 、 Java 之類的高級編程語言，在操作系統平臺上實現更復雜和更精彩的應用，而不太關心繫統的更多的硬件細節。傳統的觀點一般將前者歸類為硬件開發人員，而後者通常被稱之為軟件程序員。這兩大群體在很長一段時間幾乎也是井水不犯河水，各自有著不同的開發應用領域。

然而，隨著計算機軟、硬件技術的飛速發展和人們的應用需求越來越高，特別是隨著以 ARM 、 PowerPC 、 MIPS 等為代表的高性能嵌入式微處理器普遍使用，使傳統意義上的嵌入式系統的概念發生了很大的變化。以基於 8 位機的嵌入式系統為例，當應用系統隨著需求的增加變得越來越複雜、使用範圍變得越來越廣泛時，系統每增加一項新的功能，都會帶來應用軟件結構的變化，甚至需要完全重新設計，而這種問題在基於操作系統的通用計算機上就小得多，顯然，沒有操作系統的支持已經成為傳統的嵌入式系統的最大的缺陷，但由於 8 位機系統有限的運算能力和存儲容量，不足以支持操作系統的運行，因此，在很長一段時間以來，人們儘管已經意識到這個問題的存在，但也沒有更好的解決辦法。

自上個世紀九十年代以來，嵌入式系統的設計提出了 “ 片上系統 ” （ SoC ， System on Chip ）的概念，目前，嵌入式系統的硬件設計已進入以 SoC 為核心的階段，顯然， SoC 為高性能的嵌入式系統開發的提供了功能豐富的硬件平臺，而這些平臺強大的運算能力和存儲能力，足以支持複雜嵌入式操作系統（ EOS ， Embedded Operating Systems ）的運行。

嵌入式操作系統的發展是伴隨的嵌入式微處理器性能的不斷提高而逐漸發展起來的，最近幾年來，嵌入式操作系統的應用領域不斷擴大，其原因主要是嵌入式系統軟件開發複雜度的增加，同時，高性能的嵌入式微處理器的推廣使用，為嵌入式操作系統的可靠運行提供了硬件平臺的保障。目前，在嵌入式系統中使用的操作系統種類也非常繁多，其中，市場份額佔有比較高的有 Linux 、 Windows CE 、 VxWorks 、 QNX 、 eCos 、 uC/OS II 等。嵌入式操作系統主要可分為兩大類，一類是商業級的嵌入式操作系統，這類操作系統一般都經過較長時間的使用，且都由一些大公司開發與維護，在性能或服務上都有較好的保證，如 Windows CE 、 VxWorks 、 QNX 等，但這類操作系統一般價格比較昂貴，且都不公開核心代碼，這就與嵌入式系統用戶定製性較強、升級換代快等特點相違背，因此其應用範圍受到了很大的限制。另一類則是源代碼開放的嵌入式操作系統，如 Linux 、 eCos 、 uC/OSII 等，儘管這類系統在技術支持和服務上有所欠缺，且對開發人員有較高的要求，但由於用戶可定製性好、具有可持續開發的優勢，從目前看來，其應用範圍反而比商業嵌入式操作系統廣泛得多。
