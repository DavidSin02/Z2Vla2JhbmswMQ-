3.1 ACL下發失敗故障處理

3.1.1 故障描述

用戶下發 ACL 失敗，具體分兩種失敗情況：

• 執行下發命令後設備提示資源不足。

• 執行下發命令後設備無任何錯誤提示，但 ACL 不起作用。

3.1.2 故障處理流程

圖3-1 ACL 下發失敗故障診斷流程圖

3.1.3 故障處理步驟

1. 查看下發時是否報硬件資源不足

下發 ACL 配置時如果界面打印出“Reason: Not enough hardware resource”字樣，則表明 ACL

下發失敗是由硬件資源不足導致。通過 display qos-acl resource 命令可以進一步確認 ACL 硬件

資源使用情況。如果顯示信息中 Remaining 條目為 0 了，則表示 ACL 硬件資源已用盡，設備無法

再下發 ACL。如果下發時沒有報“Reason: Not enough hardware resource”字樣，則根據下發方

式進行步驟選擇：

• 通過 MQC（QoS 策略）方式下發，請進行步驟 2：

• 通過包過濾（packet-filter）方式下發，請進行步驟 3。

2. 檢查QoS策略配置是否正確

通過下面命令分別檢查不同使用情況下 QoS 策略的配置情況：

• （僅 Release 24xx 和 Release 26xx 版本支持）顯示以太網服務實例 QoS 策略配置信息，

display qos policy l2vpn-ac

• （僅 Release 26xx 版本支持 accounting 和 remarking 關鍵字）顯示端口上 QoS 策略配置信息， display qos [ accounting | remarking ] policy interface

• 顯示 VLAN 上 QoS 策略配置信息， display qos vlan-policy

• 顯示全局 QoS 策略配置信息， display qos policy global

• 顯示控制平面上 QoS 策略配置信息， display qos policy control-plane slot slot-number

• 顯示用戶上線後 User Profile 下應用的 QoS 策略配置信息， display qos policy user-profile如果 QoS 策略中缺少流分類和流行為關聯的配置，則補充相應配置。否則可通過以下兩個命令分別檢查下 QoS 策略中的類和流行為是否配置正確。

• 顯示配置的類信息， display traffic classifier user-defined

• 顯示配置的流行為信息， display traffic behavior user-defined

如果沒有正確配置，則進行正確配置，否則進行步驟 4。

3. 檢查packet-filter配置是否正確

可以通過 display packet-filter 命令檢查 packet-filter 配置是否正確，如果不正確，則進行正確配置，否則進行步驟 4

4. 檢查ACL配置是否正確

可以通過 display acl 命令檢查 ACL 是否配置正確，如果不正確，則進行正確配置，否則進行步驟5。

5. 尋求技術支持

如果上述檢查完成後故障仍無法排除，請聯繫 H3C 的技術支持工程師。

3.1.4 故障診斷命令

設備提供瞭如下診斷命令，需要注意的是：

• 僅 Release 26xx 版本支持顯示 QoS 策略的診斷信息。

• 僅 Release 26xx 版本支持指定 accounting、 remarking。

• 僅 Release 24xx 和 Release 26xx 版本支持顯示 L2VPN AC 承載的以太網服務實例上 QoS 策略的配置信息和運行情況。
