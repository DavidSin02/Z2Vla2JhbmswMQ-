20個對Java程序員有用的開源庫

2020-10-29
| 科技

優秀且經驗豐富的Java開發人員的特點之一是對API的廣泛瞭解，包括JDK和第三方庫。我花了大量的時間學習API，特別是在閱讀了 Effective Java 3rd Edition之後，約書亞·布洛赫(Joshua Bloch)建議如何使用現有的API進行開發，而不是為普通的東西編寫新的代碼。

在本文中，我將分享一些Java開發人員應該熟悉的最有用、最重要的庫和api。然而，不包括例如Spring和Hibernate框架。

通常，我將為日常項目包括有用的庫，包括Log4j之類的日誌庫、Jackson之類的JSON解析庫，以及JUnit和Mockito之類的單元測試api。如果您需要在項目中使用它們，那麼，您可以在項目的類路徑中包含這些庫的jar來開始使用它們，或者可以使用Maven進行依賴管理。

當您使用Maven進行依賴管理時，它將自動下載這些庫，包括它們所依賴的庫，稱為傳遞依賴。

例如，如果下載Spring框架，它還將下載Spring依賴的所有其他jar，例如Log4j。

您可能沒有意識到，但是擁有合適的依賴jar是一件令人頭疼的事情。如果您有錯誤的JAR版本，那麼您將獲得ClassNotFoundException、NoClassDefFoundError或UnsupportedClassVersionError等。

下面是我收集的一些有用的第三方庫，Java開發人員可以在他們的應用程序中使用它們來執行許多有用的任務。為了使用這些庫，Java開發人員應該熟悉它們，這就是本文的重點。如果你有一個新需求，你可以研究決定使用哪個類庫。

1.日誌庫

日誌庫非常常見，因為在每個項目中都需要它們。對於服務器端應用程序來說，日誌是最重要的，因為日誌只放在您可以看到應用程序上發生了什麼事情的地方。儘管JDK附帶了自己的日誌庫，但是還有更好的選擇，例如Log4j、SLF4j和LogBack。

Java開發人員應該熟悉日誌庫的優缺點，知道為什麼使用SLF4j比使用普通的Log4j要好。

2.JSON解析庫

在當今的web服務和物聯網世界中，JSON已成為將信息從客戶機傳輸到服務器的首選協議。它們已經取代了XML，成為以獨立於平臺的方式傳輸信息的最理想方式。

不幸的是，JDK沒有JSON庫。但是，有許多優秀的第三方庫允許您解析和創建JSON消息，如Jackson和Gson。

Java web開發人員應該至少熟悉其中一個庫。

3. 單元測試類庫

單元測試是區分普通開發人員和優秀開發人員的最重要的一點。程序員經常會得到不編寫單元測試的藉口，但是避免單元測試的最常見藉口是缺乏流行的單元測試庫的經驗和知識，包括JUnit、Mockito和PowerMock。

我的目標是在2018年提高單元測試和集成測試庫的知識，比如JUnit 5、Cucumber、Robot framework等等。

4. 通用類庫

Java開發人員可以使用一些好的、通用的第三方庫，比如Apache Commons和谷歌Guava。我總是在我的項目中包含這些庫，因為它們簡化了很多任務。

正如約書亞·布洛赫(Joshua Bloch)在《有效Java》(Effective Java)一書中所言，沒有必要重新發明輪子。我們應該更喜歡使用經過測試的庫，而不是時不時地編寫我們自己的例程。

對於Java開發人員來說，熟悉Google Guava 和Apache Commons庫是很有幫助的。

5. HTTP Libraries

我不喜歡JDK的一點是它不支持HTTP。雖然可以使用java.netpackage中的類建立HTTP連接，但是使用開源、第三方庫(如Apache HttpClient和HttpCore)並不容易，也不無縫。

儘管JDK 9支持HTTP 2.0和更好地支持HTTP，但我強烈建議所有Java開發人員熟悉流行的HTTP客戶機庫，包括HttpClient和HttpCore。

XML Parsing Libraries

有許多XML解析庫，包括Xerces、JAXB、JAXP、Dom4j和Xstream。Xerces2是Apache Xerces家族中的下一代高性能、完全兼容的XML解析器。Xerces的新版本引入了Xerces本機接口(XNI)，這是構建解析器組件和配置的完整框架，非常模塊化，易於編程。

Apache Xerces2解析器是XNI的引用實現，但是其他解析器組件、配置和解析器可以使用Xerces本機接口編寫。Dom4j是Java應用程序的另一個靈活的XML框架。

7. Excel 讀寫處理 庫

不管你信不信，實際應用中的所有應用程序都必須以某種形式與Microsoft Office交互。許多應用程序需要提供在Excel中導出數據的功能，如果需要從Java應用程序中導出數據，則需要使用Apache POI API。這是一個非常豐富的庫，允許您從Java程序中讀寫XLS文件。

8. Bytecode Libraries

如果您正在編寫生成代碼或與字節碼交互的框架或庫，那麼您需要一個bytecode庫。

它們允許您讀取和修改應用程序生成的bytecode碼。Java世界中流行的bytecode碼庫是javassist和Cglib Nodep。

9.數據庫連接池 Libraries

如果您是從Java應用程序中與數據庫交互，而不是使用數據庫連接池庫，那麼您就錯過了一些東西。

由於在運行時創建數據庫連接需要時間，並且請求處理速度較慢，所以建議使用DB連接庫。一些流行的共享池和DBCP。

在web應用程序中，web服務器通常提供這些功能，但是在核心Java應用程序中，您需要將這些連接池庫包含到類路徑中，以使用數據庫連接池。

10. Messaging Libraries

與日誌記錄和數據庫連接類似，消息傳遞也是許多實際Java應用程序的共同特性。

Java提供了JMS或Java消息傳遞服務，這不是JDK的一部分。對於這個組件，您需要包含一個單獨的jms.jar。

類似地，如果您使用第三方消息傳遞協議，如Tibco RV，則需要在應用程序類路徑中引用第三方JAR - tibrv.jar .

11. PDF 類庫

與Microsoft Excel類似，PDF庫是另一種無處不在的格式。如果需要在應用程序中支持PDF功能，比如在PDF文件中導出數據，可以使用iText和Apache FOP庫。

兩者都提供了有用的PDF相關功能，但iText更豐富、更好。

12.日期和時間相關庫

在Java 8之前，JDK的數據和時間庫有很多缺陷，因為它們不是線程安全的、不可變的和容易出錯的。許多Java開發人員依賴JodaTime實現他們的日期和時間需求。

從JDK 8開始，沒有理由再繼續去使用Joda，因為您在JDK 8的新日期和時間API中可以使用所有這些功能，但是如果您使用的是舊的Java版本，那麼JodaTime是一個值得學習的庫。

如果您想了解更多關於新的日期和時間API的信息，我建議您去搜索引擎查找Java 8課程的最新內容。它很好地概述了Java 8的所有重要特性，包括日期和時間API。

13. 集合類庫

儘管JDK有一個豐富的集合庫，但是有一些第三方庫提供了更多的選項，比如Apache Commons collections、Goldman Sachs collections、谷歌collections和collections。

Trove library特別有用，因為它為Java提供了高速的常規和原始集合。

FastUtil是另一個類似的API。它通過提供特定於類型的映射、集合、列表和優先級隊列來擴展Java集合框架，並且具有較小的內存佔用、快速訪問和插入;它還為二進制文件和文本文件提供了大型(64位)數組、集和列表，並提供了快速、實用的I/O類。

14. Email APIs 庫

javax.mail和Apache Commons電子郵件都提供了一個API，用於從Java發送電子郵件。它構建在JavaMail API之上，目的是方便使用。

15. HTML Parsing 庫

與JSON和XML類似，HTML是我們許多人不得不處理的另一種常見文檔格式。謝天謝地，我們有了JSoup，它極大地簡化了在Java應用程序中使用HTML的工作。

使用JSoup，您不僅可以解析HTML，還可以創建HTML格式文檔

16.Cryptographic 庫

Apache Commons Codec包包含簡單的編碼器和各種格式的解碼器，如Base64和十六進制。

除了這些廣泛使用的編碼器和解碼器之外，編碼器包還維護語音編碼實用程序的集合。

17. 嵌入式SQL數據庫庫

我非常喜歡內存中的數據庫，比如H2，您可以將它嵌入到Java應用程序中。它們非常適合測試SQL腳本和運行需要數據庫的單元測試。然而，除了H2數據庫之外，您還可以選擇Apache Derby和HSQL。

J18.DBC故障診斷庫

有一些好的擴展JDBC庫可以使調試更容易，比如P6spy。

這是一個庫，可以無縫地攔截和記錄數據庫數據，而無需對應用程序進行代碼更改。您可以使用它們來記錄SQL查詢及其時間。

例如，如果在代碼中使用PreparedStatment和CallableStatement，這些庫可以使用參數記錄精確的調用，以及執行所需的時間。

19. 序列化類庫

谷歌Google Protocol Buffers是一種以高效但可擴展的格式編碼結構化數據的方法。它比Java序列化更豐富、更好。我強烈推薦有經驗的Java開發人員學習谷歌Protobuf。

20. 網絡相關類庫

一些有用的網絡庫是Netty和Apache MINA。如果您正在編寫一個應用程序，需要執行低級網絡任務，請考慮使用這些庫。如果您想了解更多關於Java網絡編程的知識，請參考Java網絡編程——TCP/IP套接字編程（ Java Network Programming - TCP/IP Socket Programming）。

以上就是所有Java開發人員都應該使用的一些常用的類庫。Java應用領域是非常廣泛的，您能找到大量的庫用於處理不同的需求。

如果您想在Java中做任何事情，那麼您很可能會找到一個類庫來滿足的你的需求。與往常一樣，搜索引擎是您尋找有用Java庫的最佳朋友，但您也可以查看Maven中央存儲庫，以找到與手頭任務相關的一些有用庫。
