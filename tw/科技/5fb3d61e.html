1.5編寫一個簡單的項目程序

這個項目將向讀者演示在本章學到的C++語言和項目管理的特性。該項目將使用若干源代碼文件,以便讀者可以瞭解到依賴項的作用,以及構建工具是如何管理源代碼變更的。這個項目非常簡單,它將要求你輸入名字,然後在命令行上打印輸出名字以及相關的時間和日期。

1.5.1項目結構

該項目包含3個函數: main函數,它會調用其他另外兩個函數print-name和print_time 。它們分別位於3個獨立的源代碼文件中,因為 main函數將調用位於其他源代碼文件中的兩個函數,這意味著main 函數源代碼文件中必須包含這些函數的原型聲明。在本示例中,這意味著引用每個文件的頭文件。該項目還會使用一個預編譯頭文件,這表示存在一個源代碼文件和一個頭文件。總之,項目中將包含3個頭文件和4個源代碼文件。

1.5.2創建預編譯頭文件

該代碼將使用C++標準庫的流對象進行信息的輸入和輸出,因此我們將使用<iostream)頭文件。代碼中將使用C++的字符串類型處理信息輸入,因此還將使用<string> 頭文件。最後,它會訪問C運行時庫的time 和date 函數,因此代碼中還將用到<ctime>頭文件。這些都是標準的頭文件,我們在進行程序開發時無需對它們進行修改,因此它們是預編譯的理想目標。

在Visual Studio中創建一個C++頭文件,然後在其中添加如下代碼:

#include <iostream>#include <string>#include <ctime>將它另存為utils.n

新建一個C++源代碼文件,添加一行代碼來引用剛才創建的頭文件:

#include "utils.h"

將其另存為utils.cpp ,我們將需要為這個項目創建一個 makefile ,因此在新建文件對話框中,選擇文本文件作為文件類型。添加以下規則來構建預編譯頭文件:

utils pch utils.obj:: utils cpp utils. hCl /EHsc /c utils.cpp/ Ycutils h

將它另存為文件名為"makefile."的文件,注意要在文件末尾加一個句點( . )符號。因為我們是以文本文件的形式創建該文件的, Visual Studio將自動為它添加一個txt文件後綴,但是因為我們不希望使用該文件後綴,所以需要添加一個句點以表明該文件無後綴名。其中第一行表示utils.pch和utils.obj依賴於特定的源代碼文件和頭文件。第二行(以一個製表符作為前綴)告知編譯器去編譯C++文件,但是不調用鏈接器,同時還告知編譯器將預編譯代碼保存到utils.n文件中。該命令將創建utils.pch和

utils.obj文件,即聲明的兩個目標。

當make實用程序發現有兩個目標時,默認的動作(當目標和依賴項之間使用單個冒號時)是為每個目標都調用一次命令(你可以使用宏來決定哪個目標需要被構建)。這意味著相同的編譯器命令將執行兩次。這是我們不希望看到的,因為兩個目標是通過調用一次命令構建的。雙冒號:是一個變通方案,它告知nmake不要採用為每個目標調用命令的行為。最終的結果就是,當make程序調用命令一次,並生成了utils.pch文件,然後它會嘗試生成utils.0bj文件,不過它發現該文件已經被生成了,所以不需要再次調用命令。

現在測試輸出結果。在命令行中,導航到項目文件夾下,輸入nmake命令。

如果沒有提供makefile的名稱,程序維護工具將自動使用名為makefile的文件(如果你希望使用一個其他名字的makefile ,可以使用/f開關並指定文件名) :

C:\Beginning C++\Chapter_01\Code>nmakeMicrosoft (R) Program Maintenance Utility Version 14.00.24210.0Copyright (C) Microsoft Corporation. All rights reserredcl/EHsc /c utils.cpp /Ycutils.hMicrosoft (R) C/C++ Optimizing Compiler Version 19,00.24210 for x86Copyright (C) Microsoft Corporation. All rights reservedutils.cpp

查看文件目錄列表,以確認utils.pch和utils.obj文件是否已經被創建。

1.5.3創建主文件

現在創建一個C++源代碼文件並添加如下代碼:

#include "utils,h"#include "name,h"#include "time.h"void main(){print_name();print_time();}

將此文件另存為main.cpp。

其中引用第一個文件是標準庫頭文件的預編譯頭文件。其他兩個文件提供了被main 函數調用的另外兩個函數的函數原型聲明。

現在用戶需要為main文件添加一條規則到makefile文件。將下列加粗顯示的代碼行添加到文件頂

部

main.obj : main.cpp name.h time.h utils.pchcl/EHsc /c main.cpp /Yuutils.hutils.pch utils.obj: utils.cpp utils.hc1 /EHsc /c utils.cpp /Ycutils.h

新增的一行表示目標文件main.obi 依賴於兩個頭文件:一個源代碼文件main.cpp和一個預編譯頭文utils.pch 。目前 main.cpp文件將無法編譯,因為相關的頭文件還不存在。所以我們可以測試

makerile文件,創建兩個C++頭文件。在第一個頭文件中,添加函數原型聲明:

void print_name();

將該文件另存為name.n 。在第二個頭文件中,添加函數原型聲明,例如:

void print_time();

將該文件另存為time.h。

現在我們可以運行make 程序,它將只編譯main.cpp文件。測試它的輸出結果:通過在命令行中輸入del main.obj utils.obj utis.pch命令來刪除所有目標文件,然後再次運行make程序。這次,用戶將發現make 程序首先編譯了utils.cpp文件,然後編譯了main.cpp文件。這樣的編譯順序是因為首個目標是main.obj文件,但是因為它依賴於utils.pch文件,所以 make程序在返回創建main.obj文件規則之前,移動到了下一個規則並採用此規則生成了預編譯頭文件

需要注意的是,我們還沒有定義print_name 和print_time函數,不過編譯器並不會對此有異議。因此編譯器只創建對象文件,鏈接器負責鏈接函數。頭文件中的函數原型聲明滿足了編譯器的要求,該函數將在其他對象文件中被定義。

1.5.4輸入和輸出流

目前為止,我們已經瞭解瞭如何通過cout 對象將數據輸出到控制檯。標準庫還提供了一個cin流對象,允許讀取命令行中輸入的值。

創建一個C++源代碼文件,並添加如下代碼:

#include "utils.h"#include "name.h" void print_name(){std::cout << "Your first name? ";std:;string name;std::cin >> name;std;;cout << name;}

將該文件另存為name.cpp.

其中首先引用的文件是預編譯頭文件,這將引用兩個標準庫頭文件,即<iostream)和<string> ,因此我們可以在這些文件中使用類型聲明。

函數中第一行的含義是在控制檯上打印輸出字符串"Your first name? ".

注意,在該問題末尾有一個空格,因此光標將保持在同一行,等待輸入信息。

接下來的一行聲明瞭一個C++字符串對象變量。字符串可以包含0個或者多個字符,並且每個字符都會佔用內存。字符串類會處理字符串相關的分配和釋放內存的所有工作,該類將在第9章詳細介紹。cin重載了運算符》>以便獲取控制檯的輸入信息。當用戶按下回車"鍵之後,運算符>>將把輸入內容傳遞給變量name (將空格字符視為分隔符)。

該函數會將變量name中的內容打印輸出到控制檯,並且不帶換行符。

現在為該源代碼文件添加一條規則到makefile中,即添加如下代碼到該文件的頂部:

name.obj: name, cpp name.h utils,pchCl /EHSC /c name.cpp /Yuutils.h

保存該文件,然後運行make工具,以確認它是否生成了name.obj目標文件。

1.5.5 time函數

最終的源代碼文件將會獲取時間,並將它打印輸出到控制檯。創建一個C++源代碼文件並添加如下代碼

行：

#include "utils.h"#include "time.h"void print_time(){std::time_t now = std::time(nullptr);std::cout <<", the time and date are"<< std::ctime(&now) << std::endl;}

sta::time 和std::gmtime 這兩個函數是C函數,並且std:time_t 是一個C類型,所有這些都是通過C++標準庫獲得的。std:time 函數獲取的時間是1970年1月1日午夜以來的秒數。該函數會返回一個std::time_t類型的值,即一個64位整數。如果用戶通過指針傳遞變量在存儲中的存儲位置,則該函數可以將上述整數值拷貝到另外一個變量中。

在這個示例中,我們不需要這個工具,因此我們傳遞一個C++的nullptr 給該函數,以聲明不需要執。行拷貝操作。

接下來,我們需要將秒數轉換成包含時間和日期的字符格式,以方便用戶理解。這也是std::ctime函數的主要用途,它會接收一個指向保存秒數變量的指針作為參數。變量now保存了秒數,運算符&用於獲取變量的內存地址。第4章將詳細介紹變量和指針的細節。 std::ctime函數會返回一個字符串,不過我們還沒有為該字符分配任何內存,也不應該嘗試為該字符串分配內存。 std:ctime 函數創建了一個靜態分配內存緩衝區,它將被運行在當前執行線程的所有代碼共享使用。每次在相同執行線程上調用std:ctime函數時,使用的內存地址是一樣的。不過內存中的內容可能會發生變化。

此函數說明檢查幫助手冊,查看誰負責分配和釋放內存是非常重要的。第4章將詳細介紹分配內存的細

節。

從std:ctime返回的字符被打印輸出到控制檯,並且是通過調用若干次運算符<<對輸出結果進行格式化的。

現在給makefile添加一條構建規則,即添加如下規則到該文件頂部:

time.obj: time. cpp time.h utils.pchC1 /EHsc /c time.cpp /Yuutils.n

保存該文件並運行make工具,然後確認該構建過程是否生成了time.obj 目標文件。

1.5.6構建可執行文件

現在我們已經擁有了項目所需的所有對象文件,因此下一個任務是將它們鏈接到一起。為此,將下列代碼添加到makefile文件頂部

time_test.exe: main.obj name.obj time.obj utils.objink /out:$@$**

這裡的目標是可執行文件,並且依賴項是4個對象文件。命令行中為了構建可執行文件會調用鏈接器並使用特殊的語法。標識符$@會被make工具解析為使用目標,所以/out開關實際的內容是

/out:time_test.out 。標識符$**會被make工具解析為使用所有依賴項,因此所有依賴項都將被鏈接。

保存該文件並運行make程序。用戶將發現只有鏈接器被調用,並且它將鏈接所有對象文件繼而創建可執行文件。

最後,添加一條規則來清理項目。提供一種機制移除編譯過程中生成的文件,只保留源代碼文件,從而保持項目結構的整潔是一個非常好的習慣。在鏈接對象文件的代碼行之後,添加如下代碼:

time_test.exe: main.obj name.obj time.obj utils.objlink /out:@$**clean: @echo Cleaning the project...del main.obj name.obj time.obj utils.obj utils.pckdel time_test.exe

clean任務是一個偽目標,實際上並沒有生成文件,因此也不存在依賴項。這說明了make程序的個特性,如果調用mmake工具,並指定目標名稱,那麼該程序將只生成該目標。如果沒有聲明目標,那麼該程序將生成makefile提及的第一個目標,在這種情況下是time_test.exe

clean 偽目標包含3個命令:第一個命令會將"Cleaning the project.打印輸出到控制檯,標識符e會告知命令行不要將該命令打印輸出到控制檯;第二個和第三個命令會調用命令行工具de1刪除文件。現在可以在命令行上輸入 mmake clean 命令清理項目,然後確認項目目錄下是否只包含頭文件、源代碼文件和makefile文件。

1.5.7 測試代碼

再次運行make程序,以便構建可執行文件。在命令行中,可以通過輸入time_test命令運行示例程序。你將被要求輸入姓名;執行此操作,然後按回車"鍵,此時將發現自己的名字、當前時間和日期被打印輸出到控制檯上:

C:Beginning_C++Chapter_01>time_testYour first name? RichardRichard, the time and date are Tue Sep 6 19:32:23 2016

1.5.8 修改項目

現在讀者對基本的項目結構有所瞭解,通過makefile ,你可以對文件進行修改,並確保重新構建項目時,只對發生變更的文件進行編譯。為了說明這一點,將修改name.cpp 中的 print_name 函數,以更客氣的方式詢問你的姓名。修改函數體中第一行代碼,比如下列代碼中加粗顯示的代碼行:

void print_name(){std::cout<<"Please type your first name and press [Enter] ";std;;string name;

保存該文件,然後運行make工具。這一次只有源代碼文件main.cpp被編譯,生成的name.obj文件會與已有的對象文件鏈接到一起。

現在修改頭文件name.h ,並在其中添加一個註釋信息:

//更客氣的版本void print_name();

構建該項目。讀者發現了什麼?這一次,有兩個源代碼文件被編譯,即name.cpp和main.cpp 。並且它們與已有的對象文件鏈接到一起,從而創建了可執行文件。為了細究這兩個文件被編譯的原因,可以查看makefile 中的依賴項規則。唯一發生變更的文件是name.h,並且該文件的名字出現在了ame.obj和main.obj依賴項列表上,因此這兩個文件被重新構建。由於這兩個文件出現在了time_test.exe的依賴項列表上,所以該可執行文件也將被重新構建。
