服務器安裝網站教程「網站三件套——LNMP」

2020-11-14
| 科技

本文是說購買了VPS或雲主機之後，使用LNMP建站方法。

具體購買雲主機的流程可以看[ 阿里雲域名註冊與備案、服務器ECS購買與登錄使用]。其中[阿里雲服務器ECS官網鏈接如下] (https://www.aliyun.com/minisite/goods?userCode=yos4xyvp)


簡單說說建網站相關的事情

  1. 比較久遠的時候，以前都是在Linux環境下，在黑洞洞的命令行窗口，安裝建立網站常用的軟件。建立網站肯定離不開三件套嘛——http服務器、數據庫、服務器運行語言（俗稱後臺）。
  2. 安裝這三件套有各種各樣的組合。後來逐步形成LAMP（Linux+Apache+MySQL+PHP）和LNMP（Linux+Nginx+MySQL+PHP）幾個組合嘛。
  3. 這三件套一個個安裝需要逐個輸入命令，還有軟件版本對應和依賴問題，以及權限及相關配置，很麻煩。後來就有人就編寫了LAMP或者LNMP一鍵安裝命令腳本。

SSH登錄VPS：

SSH登錄使用putty軟件。

在登錄VPS之前，設置putty的keep alive時間，就是心跳時間，意思是每隔多久發送一下報文，來保持putty軟件和VPS通信，免得超時斷開。

在putty -> Connection -> Seconds between keepalives ( 0 to turn off )，默認為0，改為60。

使用putty客戶端進行遠程登錄。其中IP地址是VPS地址，SSH端口是VPS端口（注意不一定是默認的22端口，每個VPS不一樣）。


連接後，跳出對話框，因為是首次登錄，提醒沒有緩存祕鑰，點擊是。

使用SSH登錄後，進入命令行界面，輸入root賬號及密碼（密碼見前面安裝操作系統時候那一串不規則字符）。輸入密碼過程中是沒有光標及**星號顯示的，這是linux的傳統。輸入完之後敲擊回車登錄成功。#標誌表示root賬號登錄。

修改密碼 。因為原本root密碼太複雜難記，修改一下比較方便，如果不想修改那就跳過吧。在命令行輸入命令passwd,輸入新密碼。因為輸入的密碼比較弱，linux提出了提醒，可以不管它，繼續輸入第二遍確認密碼。



改斷開連接時間：

因為SSH登錄後如果長時間不操作會斷開。但是系統默認斷開連接時間太短，因此進行修改。如果不想改也可以跳過。

修改需要用到vim編輯器，首先安裝vim。輸入命令 yum install vim，等待安裝成功。

輸入命令vim /etc/profile

進入了vim界面，在最後一行添加 TMOUT=18000。(即斷開時間時間為18000)

在這裡簡單介紹vim使用方法——

  1. 按insert鍵進入編輯模式，左下角會顯示insert模式。
  2. 移動光標到最後一行添加TMOUT=18000，跟平常文本編輯一樣。
  3. 編輯完畢按esc鍵退出編輯模式。
  4. 按:鍵進入命令模式，左下角會顯示:符號即命令模式。
  5. 輸入wq保存退出.

使用WinScp進行SFTP文件傳輸。

SFTP 是 Secure File Transfer Protocol 的縮寫，安全文件傳送協議，可以為傳輸文件提供一種安全的網絡的加密方法。SFTP與 FTP 有著幾乎一樣的語法和功能。

SFTP 是 SSH 的一部分，在 SSH 軟件包中，已經包含了一個叫作 SFTP 的安全文件信息傳輸子系統， 同樣是使用加密傳輸認證信息和傳輸的數據.簡而言之，SFTP 不需要在 VPS 上額外安裝任何東西，我們購買 VPS 後就能直接使用 SFTP 進行文件傳輸，非常方便，所以這裡介紹基於 SFTP 協議的文件傳輸方法。


安裝screen命令。

因為安裝LNMP時間較長，所以用screen命令操作不會中斷。

需要先安裝screen命令，命令行輸入yum install screen

新建lnmp會話，發現putty軟件左上角已經變為[screen 0: bash]，即已經切換至screen會話了。

screen -S lnmp


搭建LNMP環境

wget -c ftp://soft.vpser.net/lnmp/lnmp1.3-full.tar.gz && tar zxf lnmp1.3-full.tar.gz && cd lnmp1.3-full && ./install.sh lamp


安裝過程中需要輸入密碼、選擇相應版本、輸入通知郵箱。全都按默認處理。

出現這個說明安裝結束

打開瀏覽器訪問主機IP，出現這個則說明成功安裝

建議安裝PHP緩存加速類擴展

在安裝wordpress前，推薦安裝兩個：OPcache和Memcached。

首先，需要進入LNMP解壓目錄lnmp1.3-full：

cd /root/lnmp1.3-full

回車，接下來安裝Opcache：

./addons.sh install opcache

回車，再回車。

當出現 Opcache installed successfully, enjoy it! 字樣時，即表示安裝成功。

接著安裝Memcached：

./addons.sh install memcached

回車，選擇2，回車，再回車。

當出現 “Memcached installed successfully, enjoy it!” 字樣時，即表示安裝成功。

安裝好擴展包後，我們看到lnmp有哪些命令。有Nginx、mysql、mariadb、pureftpd、httpd進程的操作。

其中，Vhost是域名管理、Database是數據庫管理。


添加域名 、

lnmp vhost add

回車，提示輸入域名：XX.com(如果沒有則輸入IP地址)

回車，提示是否添加多個域名，輸入y繼續綁定綁定帶www的域名www.XX.com.

回車，顯示網站目錄。默認 /home/wwwroot/XX.com 即可。注意：若是綁定二級域名，必須手動輸入完整的目錄路徑。例如：將tools.XX.com綁定到網站根目錄下的tools文件夾，則應輸入：/home/wwwroot/XX.com/tools

回車，選擇是否日誌記錄，默認跳過即可。

會車後，輸入網站管理者郵箱。

添加數據庫

繼續回車，提示數據庫名和數據庫用戶名是否保持一致。輸入y表示一致。

回車，輸入root用戶的數據庫密碼（輸入過程中密碼不會顯示）。

回車，輸入數據庫名，自行設置，本處寫wp

回車，設置數據庫密碼。

回車，再回車。

當出現下圖所示畫面時候，說明添加域名與數據庫成功。

安裝Wordpress

首先，進入添加的域名目錄：

cd /home/wwwroot/xx.com

瀏覽器中打開WordPress中文站點，下載程序壓縮包：

wget https://cn.wordpress.org/wordpress-5.0.3-zh_CN.tar.gz

等待下載完之後，解壓壓縮包：

tar -zxvf wordpress-5.0.3-zh_CN.tar.gz

接下來，將解壓出來的wordpress文件夾內全部文件移動到當前的域名目錄下（別忘了後面的.符號）。

mv wordpress/* .

回車。然後，可以選擇刪掉空文件夾wordpress。

rm -rf wordpress

為避免因權限的問題導致安裝出錯，比如wp-config.php無法創建、需要提供FTP用戶密碼以及主題和插件不能更新等，建議賦予根目錄文件的可寫權限。以後每添加一個域名，都要執行一次以下兩步操作。

chmod -R 755 /home/wwwrootchown -R www /home/wwwroot

Wordpress 配置

在瀏覽器地址欄輸入域名或IP地址，跳出wordpress配置頁面

點擊“現在就開始”，跳轉到數據庫配置頁面。按照前面面板添加站點生成的數據庫配置信息填入。

數據庫配置完畢後，設置wordpress站點標題及賬戶信息。


添加完畢，登錄wordpress賬號及密碼，進入該網站頁面。

此時網站建立完畢。後續就是網站內容豐富和美化的事情。
