編程｜實例（分書問題）瞭解數據結構、算法（窮舉、遞歸、回溯）

2020-10-29
| 科技

任何一個可以用計算機解決的問題所需的計算時間都與其規模有關係，這也就意味著，通常情況下問題規模越大，所耗費的時間和計算資源越多；而問題的規模越小，所需的時間和計算資源越小，問題的求解也越容易，因此，在處理一些困難問題的時候，我們會考慮通過縮小問題的規模而使得困難問題更加容易求解。在充分研究這類算法的規律之後，人們將這些算法總結成縮小規模算法，如遞歸法、分治法、動態規劃法、貪心法等。（分治法、動態規劃法、回溯法都是基於遞歸的算法）

1 窮舉法

在實際解題過程中，可以發現並不是任何問題都可以直觀地分解成若干小問題來進行求解，往往最容易想到的解題策略是枚舉法（也稱為窮舉法或蠻力法），即對要解決問題的所有可能情況，一個不漏地進行檢驗，從中找出符合要求的答案，因此枚舉法是通過犧牲時間來換取答案的全面性。如果在搜索解的過程中加上判定條件來加快搜索速度，則按搜索策略的不同有回溯法與分支限界算法。

2 回溯法

回溯法（探索與回溯法）是一種選優搜索法，又稱為試探法，按選優條件向前搜索，以達到目標。但當探索到某一步時，發現原先選擇並不優或達不到目標，就退回一步重新選擇，這種走不通就退回再走的技術為回溯法，而滿足回溯條件的某個狀態的點稱為“回溯點”。擴大當前候選解的規模，並繼續試探的過程稱為向前試探。也就是如果當前候選解除了不滿足規模要求外，滿足其他所有要求時，繼續擴大當前候選解的規模，並繼續試探。如果當前的候選解滿足包括問題規模在內的所有要求時，該候選解就是問題的一個解。

回溯算法實際上是一個類似枚舉的搜索嘗試過程，主要是在搜索嘗試過程中尋找問題的解，當發現已不滿足求解條件時，就“回溯“返回，嘗試別的路徑。可以參考一下走迷宮的過程，一開始會隨機選擇一條道路前進，一直到走不通之後就會回頭直到找到另外一條沒有試過的道路前進。實際上，走迷宮的算法就是回溯法的經典問題。

回溯法實際上也是一種試錯的思路，通過不斷嘗試解的組合來達到求解可行解和最優解的目的。雖然都有窮搜的概念蘊含其中，但是回溯法和窮舉查找法是不同的。對於一個問題的所有實例，窮舉法註定都是非常緩慢的，但應用回溯法至少可以期望對於一些規模不是很小的實例，計算機在可接受的時間內對問題求解。

回溯，正如去某一個地方要回到原點一樣，需要記住回溯點的狀態，棧或遞歸棧就是適合的一個方式。

許多複雜的規模的問題都可以使用回溯法，有”通用解題方法”的美稱。分書問題和八皇后都是典型的回溯法問題。

3 遞歸算法及代碼執行流程

在遞歸函數中，遞歸分為兩個階段，遞推與迴歸，遞推執行遞歸函數自調用處之前的代碼，迴歸時執行遞歸函數自調用處之後的代碼。

4 分書問題

有編號為0，1，2，3，4的5本書，準備分給5個人A，B，C，D，E，寫一個程序，輸出所有皆大歡喜的分書方案。

每個人的閱讀興趣用一個二維數組like描述：

Like[i][j] = true i喜歡書j

Like[i][j] = false i不喜歡書j

設計一個函數trynext(int i)給第i個人分書。

用一個一維數組take表示某本書分給了某人。take[j]=i+1;//把第j本書分配給第i個人

依次嘗試把書j分給人i。

如果第i個人不喜歡第j本書，則嘗試下一本書，如果喜歡，並且第j本書尚未分配，則把書j分配給i。

如果i是最後一個人，則方案數加1，輸出該方案。否則調用trynext(i+1)為第i+1個人分書。

如果對第i個人枚舉了他喜歡的所有的書，都沒有找到可行的方案，那就回到前一個狀態i-1，讓i-1把分到的書退回去，重新找喜歡的書，再遞歸調用函數，尋找可行的方案。

6 回溯的執行流程

6.1 第一個回溯點（遞歸函數迴歸點）：i=4; j=4;

take[4] = 0;

j++;

j=5;不滿足j<5的條件，回到下一個遞歸點。

6.2 第2個回溯點（遞歸函數迴歸點）：i=3; j=3;

take[3] = 0;

j++;

j=4;

j++;

j=5;不滿足j<5的條件，回到下一個遞歸點。

6.3 第3個回溯點（遞歸函數迴歸點）：i=2; j=1;

take[1] = 0;

j++;

j=2;

like[2][2]=1、take[2] = 1

j++;

j=3

like[2][3]=0

j++;

j=4;

like[2][4]=1、take[4] = 0

take[4] = 2+1 = 3

i=4 不成立；

遞歸：trynext(3)

開始遞推：

i = 3; j=0;

j++

j++

j++

take[3] = 3+1

i=4 不成立；

遞歸：trynext(4)

開始遞推：

i = 4; j=0;

j++

take[1] = 4+1

至此，又得到了一個新的方案。

然後再次在棧中進行迴歸（或遞歸、遞推）……
