你辛辛苦苦做出的Excel數據模型同事不會用？參數表格可以拯救你

2020-11-14
| 科技

需求場景

假設你用微軟的Excel做了一個牛叉到沒朋友的報告模板，有各種複雜的計算。這個模板使用的N個原始文件是從某個系統導出的。如果這個模板只供你一個人使用，毫無問題；當你需要把這個模板提交給其他夥伴使用時，他們就需要自己從系統中導出N個原始文件然後修改模板的PowerQuery代碼，這將是一場你無法想象的災難。怎麼避免這樣的災難發生？

避免災難的方法是：儘可能將模板代碼封裝，只讓其他夥伴導出系統文件放到某個特定文件夾，然後把模板文件丟到原始文件所在的文件夾。然後打開模板文件刷新即可。

這就是參數表格發揮作用的地方。

不知道“Parameter Table”確切翻譯是什麼，我直譯為“參數表格”。參數表格的意思是：PowerQuery的參數可以根據用戶輸入來實現動態化。

首先在Excel中創建參數表格並進行初始化

這個表格只有兩列：參數和值。因為這整個思路是我抄的英文資源的，所以我偷懶照搬英文列名：“Parameter”和“Value”。然後用插入表格或套用表格的方式，將區域轉化成表格，記得將表格命名為“Parameters”【圖1】。注意左上角的“表名稱”。Value列就是需要用戶輸入的地方。

現在進行參數表格初始化：由於需要用的數據源有多個且放在了一個固定的文件夾內（假設叫"原始數據"），因此，在Parameter列下面添加一個”FolderPath"。Value的值裡填入下面的公式：

=LEFT(CELL("filename",A1),FIND("[",CELL("filename",A1),1)-1)&"原始數據\"

然後把參數表格引入到PowerQuery

選擇“從表格”創建然後新建一個查詢，這一步是創建包含參數值的查詢，便於後面從這個查詢裡引用參數值。創建過程比較簡單，一路默認和確定即可，不再截圖演示。因為是從已經命名的表格創建的，所以查詢名自動繼承了表格名字“Parameters”。

創建引用參數表格值的函數

現在要創建一個函數，因為只有函數才會擁有動態變化的值。直接點擊“新建查詢-從其他源-空白查詢”，創建一個空白查詢，點擊“高級編輯器”，刪除裡邊的所有代碼，把下面代碼粘貼進去然後保存，並把該查詢命名為“fnGetParameter”。

//括號和等號加右箭頭是函數的標誌，括號裡的是函數的參數，或者叫變量。如果把第一行拿掉，其實就是一個完整的查詢。加上第一行就把這個查詢封裝成為一個函數了。

(ParameterName as text) =>

let

//獲取剛才創建的Parameters表格

ParamSource = Excel.CurrentWorkbook(){[Name="Parameters"]}[Content],

//這一步其實是讓參數表格的行可以無限擴充，不再侷限於本例中的三行。注意在“in”前的都是一條語句，並且有縮進，我這裡偷懶沒縮進。整條語句蠻抽象，我自己也還沒完全弄懂其內部的運作原理，直接照搬了。

ParamRow = Table.SelectRows(ParamSource, each ([Parameter] = ParameterName)),

Value=

if Table.IsEmpty(ParamRow)=true

then null

else Record.Field(ParamRow{0},"Value")

in

Value

現在可以創建你牛叉的數據模型了

接下來創建你的數據模型，無需考慮原始文件的引用路徑是否硬編碼。進行各種複雜的Shaping也好，Calculating也好，然後加載到數據模型，。根據需要添加計算列、計算字段，創建數據透視表、數據透視圖乃至PowerView。十八般武藝樣樣上吧。具體步驟不贅述。

把你模型中原始文件的引用路徑全部用引用參數表格函數來代替

報告創建好之後，你同事能夠輕鬆使用你的牛逼報告模板的關鍵一步來了：將上一步寫死的源文件位置，替換成參數表格裡的值。類似於下面這樣的：

源 = Excel.Workbook(File.Contents(fnGetParameter("FolderPath")), null, true),

其中的FolderPath就是你同事電腦上保存源文件的路徑。但不需要你同事自己寫路徑，模型文件會自動獲取，這就是它牛叉的地方。

保存該Excel文件。

你同事不費吹灰之力就可以使用你的牛叉模板

同事拿到這個Excel數據模型文件後，放到ta想放的位置。在同一個文件夾下新建一個叫“原始數據”的空文件夾，把系統中的原始數據文件都丟到這個叫“原始數據”的文件夾中。

然後打開模板文件，點擊“刷新”，大概幾秒到幾分鐘就獲取到最新的數據了（取決於數據量大小和模型的複雜程度），你模型中的數據透視表、圖表及其他各種豐富多彩的可視化結果都自動更新好了。
