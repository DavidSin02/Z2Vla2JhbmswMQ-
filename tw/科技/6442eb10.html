1-MyBatis

（1）MyBatis的前身是Apache的開源項目iBatis。iBatis一詞來源於“internet ”和“abatis“的組合，是一個基於Java的持久層框架。 2010年這個項目由 Apache software foundation遷移到Google code ，並更名為 MyBatis。2013年11 月，MyBatis遷移到 GitHub 上，目前由GitHub提供維護。

（2）MyBatis 的優勢在於靈活，它幾乎可以代替JDBC ，同時提供了接口編程。目前MyBatis的數據訪問層 DAO (Data Access Objects ）是不需要實現類的，它只需要一個接口和 XML（或者註解〉。MyBatis 提供自動映射、動態 SQL 、級聯、緩存、註解、代碼和SQL 分離等特性，使用方便，同時也可以對 SQL 進行優化。因為其具有封裝少、映射多樣化、支持存儲過程、可以進行 SQL優化等特點，使得它取代了 Hibernate成為了 Java 互聯網中首選的持久框架。

(3)Hibernate作為一種十分流行的框架，它有其無可替代的優勢，這裡我們有必要討論一下它和MyBatis 的區別 。由於MyBatis和Hibernate 都是持久層框架，都會涉及數據庫，所以首先定義一個數據庫表一一角色表（t_role ），其結構如圖所示。

根據這個角色表，我們可以用一個POJO (Plain Ordinary Java Object）和這張表定義的字段對應起來，如代碼清單所示。

package com.learn.chapterl.pojo;public class Role implements java.io.Serializable { private Integer id; private String roleName; private String note; /** setter and getter **/}

2-Hibernate簡介

（1）要將POJO和數據庫映射起來需要給這些框架提供映射規則，所以下一步要提供映射的規則，如圖所示。

（2）在MyBatis或者Hibernate中可以通過XML或者註解提供映射規則， 這裡討論的是XML方式，因為在MyBatis中註解方式會受到一定的限制，所以MyBatis通常使用XML方式實現映射關係。

（3）我們把POJO對象和數據庫表相互映射的框架稱為對象關係映射（Object Relational Mapping , ORM，或O/RM ，或O/R mapping）框架。無論MyBatis 或者Hibernate都可以稱為ORM框架， 只是Hibernate的設計理念是完全面向POJO 的， 而MyBatis則不是。Hibernate基本不再需要編寫SQL就可以通過映射關係來操作數據庫，是一種全表映射的體現：而MyBatis則不同，它需要我們提供SQL去運行。

Hibernate是將POJO和數據庫表對應的映射文件，如代碼清單所示。

首先，對POJO 和表t_role進行了映射配置，把兩者映射起來了。然後，對POJO進行操作，從而影響t_role 表的數據， 比如對其增、刪、查、改可以按照如代碼清單所示方式操作。

這裡我們沒有看到SQL ，那是因為Hibernate會根據映射關係來生成對應的SQL ，程序員不用精通SQL ，只要懂得操作POJO就能夠操作對應數據庫的表了。這在管理系統時代是十分有利的。因為對於管理系統而言，首先在於實現業務邏輯，然後才是性能，所以Hibernate成為了那個時代的主流持久框架。
