多態的概念及C++例子

關於多態，之前整理的什麼是面向對象？這篇文章有說到：

多態按字面的意思就是多種形態。當類之間存在層次結構，並且類之間是通過繼承關聯時，就會用到多態。多態意味著調用成員函數時，會根據調用函數的對象的類型來執行不同的函數。比如關於多態的C++的例子（該C++代碼來自菜鳥教程）：

#include <iostream> using namespace std;// 基類  class Shape {   protected:      int width, height;   public:      Shape( int a=0, int b=0)      {         width = a;         height = b;      }      virtual int area()      {         cout << "Parent class area" <<endl;         return 0;      }};​// 派生類Rectangleclass Rectangle: public Shape{   public:      Rectangle( int a=0, int b=0):Shape(a, b) { }      int area ()      {          cout << "Rectangle class area" <<endl;         return (width * height);       }};​// 派生類Triangleclass Triangle: public Shape{   public:      Triangle( int a=0, int b=0):Shape(a, b) { }      int area ()      {          cout << "Triangle class area" <<endl;         return (width * height / 2);       }};​// 程序的主函數int main( ){   Shape *shape;   Rectangle rec(10,7);   Triangle  tri(10,5);    // 存儲矩形的地址   shape = &rec   // 調用矩形的求面積函數 area   shape->area();    // 存儲三角形的地址   shape = &tri   // 調用三角形的求面積函數 area   shape->area();      return 0;}

代碼中用到了一個關鍵字：virtual，這是C++的關鍵字。基類中用virtual關鍵字修飾的函數叫做虛函數。這虛函數有點像弱定義的感覺，先定義一個弱的/虛的函數，其它地方再定義同名的真的函數，實際用的是真的函數。

該例中，在派生類中重新定義基類中定義的虛函數area時，會告訴編譯器不要靜態鏈接到該函數，而是根據所調用的對象類型來選擇調用真正的函數。
