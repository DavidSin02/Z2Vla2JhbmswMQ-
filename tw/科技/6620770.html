工業控制(PLC）專業術語

2020-10-29
| 科技

BCD

BCD（二進制編碼的十進制）是計算中的數字代碼，其中十進制數的每一位均單獨進行二進制編碼。

CAPI

CAPI（公共應用編程接口）是一種符合 ISDN 的標準化編程接口。通過 CAPI 接口，可以使用 ISDN 的軟件，而無需知道所用的製造商特定的 ISDN 卡。

CIR

CiR（RUN 模式下的組態）是在運行期間的組態更改。對於不允許關閉的系統，可在系統運行期間對某些組態進行更改，前提是已組態了相應的保留。

CM

CM（通信模塊）是執行通信任務的模塊，在自動化系統中用作 CPU 的擴展接口。

CP

CP（通信處理器）是用於擴展通信任務的模塊，可為 CPU 提供附加接口類型或通信選項。

CPU

CPU（中央處理單元）是自動化系統的硬件設備，包含控制和算數邏輯單元、存儲器、操作系統和編程設備接口。

DHCP

DHCP（動態主機組態協議）是計算機技術中的通信協議。服務器通過它為客戶端分配網絡組態。這意味著連接到同一網絡的設備（客戶端）均由 DHCP 服務器自動分配 IP 地址。

DNS

DNS（Domain Name System，域名系統）是分佈在全世界幾千臺服務器上的數據庫，可管理 Internet 上的名稱空間。

DNS 服務器

DNS 服務器（域名系統服務器）是管理 Internet 上 IP 地址的字母數字名稱的服務器。

DP 從站

DP 從站是構成 DP 主站系統一部分的模塊，它通過 PROFIBUS 與 DP 主站通信。DP 從站本身沒有總線訪問權限，這意味著它們只能確認已收到的消息，或者應 DP主站的請求向主站傳送消息。

DP 主站

DP 主站是 DP 主站系統的中央控制器。DP 主站與其 DP 從站之間的數據傳輸由該主站按定義的順序或重複發生順序自動處理。DP主站按 Profibus DP 標準 (EN50170) 運行。

DP/PA 耦合器

DP/PA 耦合器是 PROFIBUS DP 和 PROFIBUS PA 之間的連接模塊。

EMC

EMC（Electromagnetic Compatibility，電磁兼容性）是指電氣和/或電子設備彼此互不干擾的狀態。

EN

EN（Enable，使能）是每個 STEP 7 塊都包含並可在調用塊時置位的輸入。EN =TRUE 時，調用塊；EN = FALSE 時，不調用塊。

ENO

ENO（Enable output，使能輸出）是每個 STEP 7 塊都包含的一個輸出。可在調用塊後立即使用 ENO 檢查塊中的全部操作是否均已正確完成，或者是否錯誤發生。

FBD

函數塊圖 (FBD) 是在 EN 61131-3 (IEC 61131-3) 中定義的一種語言，用於對可編程邏輯控制器 (PLC) 編程。函數塊圖使用與布爾代數相同的邏輯框表示邏輯功能，特別適合邏輯控制器。

FTP

FTP（File Transfer Protocol，文件傳輸協議）是兩臺計算機通過 Internet 傳輸文件所用的網絡協議。其中一臺計算機必須是 FTP 服務器，另一臺是 FTP 客戶端。

GPRS

GPRS（General Packet Radio Service，通用分組無線業務）是一項基於數據包的服務，用於 GSM 網絡中基於 IP 的數據傳輸。GPRS 數據包也可以通過 Internet 傳輸。

GRAPH

GRAPH 是一種用於創建順序控制系統的圖形編程語言，其組態和編程方式遵循IEC 61131-3 和 DIN EN 61131 標準。順控程序極大提高了編程的速度與直觀易用性。通過將過程分解為多個步，而且每個步都有明確的功能範圍，然後再將這些步組織到順控程序中。在各個步中定義待執行的動作，以及步之間的轉移條件。這些轉移條件包括切換到下一步的條件。

GSM

GSM（Global System for Mobile Communication，全球移動通信系統）是全數字移動通信網絡標準，主要用於電話但也用於線路和包交換數據的傳輸以及短消息。

HMI

HMI（Human Machine Interface，人機界面）是用來實現自動化過程可視化與控制的設備。

HOLD 模式

在“HOLD”操作模式中，將暫停執行用戶程序。只有在使用編程設備進行測試時，才使用“HOLD”操作模式。

HTTP

HTTP（Hypertext Transfer Protocol，超文本傳輸協議）是 Internet 上網頁的傳輸協議。

HTTPS

HTTPS（Secure Hypertext Transfer Protocol，安全超文本傳輸協議）是 Internet 上加密數據的傳輸協議。HTTPS 擴展了 HTTP 協議的內容，使用 SSL 安全傳輸保密數據。

I/O

I/O 是指遠離中央機架安裝的模擬量和數字量模塊。

IO 控制器

IO 控制器是 PROFINET 系統的中央設備，如 CPU 或 PC。IO 控制器可建立與 IO 設備的連接，與這些設備進行數據交換，以及對 PROFINET 系統進行監控。

IO 設備

IO 設備是受 IO 控制器監控的 PROFINET 系統中的分佈式 I/O 設備。

IPE

IPE（Inter Project Engineering，項目間工程組態）支持跨項目數據交換。原項目中的控制器數據可以通過“設備代理數據”對象複製並導出到目標對象。

IPE 文件

IPE（Inter Project Engineering，項目間工程組態）文件包括來自源項目的 CPU 控制器數據，這些數據可通過“設備代理數據”對象導出並傳送到目標項目的 CPU 中。

ISDN

ISDN（Integrated Services Digital Network，綜合業務數字網）是傳輸數字數據和語音的數字遠程通信網絡的國際標準。

ISDN 適配器

ISDN 適配器（或 ISDN 終端適配器 ISDN-TA）是一種 ISDN 設備，它連接到串行 PG/PC 接口來通過數字網絡傳送數據。

ISDN 調制解調器

ISDN 調制解調器（調製器/解調器）是具有集成模擬調制解調器功能的 ISDN 適配器。

LAD

LAD（梯形圖）是指在 IEC 標準 EN 61131-3 中標準化的一種圖形編程語言，尤其適合邏輯控制器。梯形圖與電路圖表示相對應。

MAC 地址

MAC 地址（“介質訪問控制”地址）是全球以太網設備獨一無二的設備標識。MAC地址由製造商分配，其中 3 字節為供應商 ID，另外 3 字節（以連續數字表示）為設備 ID。

MRP

MRP（Media Redundancy Protocol，介質冗餘協議）可用來實現冗餘網絡。在這些網絡內，環網拓撲結構中的冗餘傳輸路徑可確保在一個傳輸路徑故障的情況下，可以使用備用通信路徑。構成此冗餘網絡的 PROFINET 設備一起組成一個 MRP域。

NTP

NTP（Network Time Protocol，網絡時間協議）規定了通過工業以太網同步自動化系統中的時鐘的標準。

OPC / OPC UA

“開放平臺通信”的簡稱（正式名稱為“過程控制 OLE”），是一種應用於自動化技術領域的標準軟件接口。基於 OPC，只要接口驅動程序符合 OPC 規範，不同製造商的應用程序之間都可 OPC 進行數據通信。後續的 OPC UA（“OPC 統一架構”）中已忽略了之前與 Windows 操作系統的鏈接。此外，OPC UA 中還包含有安全功能和功能擴展性等重要特性。

OPC- / OPC UA 服務器

基於 OPC/OPC UA 標準，帶有服務器功能的設備可與 OPC 或 OPC UA 客戶端進行數據通信。

PLC 變量表

PLC 變量表是用於定義在整個 CPU 範圍有效的變量的表格。

PLC 數據類型

PLC 數據類型是指由用戶定義、在程序中可多次使用的數據結構。PLC 數據類型可由多個元素組成，每個元素可以有不同的數據類型。

PROFIBUS

PROFIBUS（Process Field Bus，過程現場總線）是自動化技術中的一種總線通信標準。

PROFIBUS DP

PROFIBUS DP 是支持 DP 協議（Decentralized Peripherals，分散型外設）的 PROFIBUS（過程現場總線），符合 EN 50170。它用於通過中央控制器對傳感器和執行器尋址以及對多個控制器進行聯網。從用戶程序角度看，分佈式 I/O 與集中式 I/O 的尋址方式完全相同。

PROFINET

PROFINET（Process Field Network，過程現場網絡）是基於以太網的通信系統，用於分佈式自動化系統。

PROFINET 端口

PROFINET 端口可用來從物理上連接充當 PROFINET 節點的設備。

PROFINET 設備

PROFINET 設備是帶 PROFINET 接口（電氣、光學、無線）的設備。

RUN 模式在“RUN”操作模式中，CPU 執行用戶程序、更新輸入和輸出、並處理中斷和錯誤消息。

SCL

SCL（Structured Control Language，結構化控制語言）是一種基於 PASCAL 的高級

編程語言。這種語言基於標準 DIN EN 61131-3（國際標準為 IEC 1131-3）。根據該標準，可對用於可編程邏輯控制器的編程語言進行標準化。SCL 編程語言實現了該標準中定義的 ST 語言（結構化文本）的 PLCopen 初級水平。

SMS

SMS（短消息服務）是長度受限（最長 160 個字符）的消息，可通過移動網絡發送至 GSM 接收器。

SNMP

SNMP（Simple Network Management Protocol，簡單網絡管理協議）是用於以太網網絡基礎結構的診斷和參數分配的標準化協議。

SSL

SSL（Secure Sockets Layer，安全套接層）是用以在 Internet 上實現安全數據傳輸的加密協議。

STL

語句表 (STL) 是一種文本編程語言，與機器代碼相似。如果程序採用 STL 編寫，則各條語句代表 CPU 執行程序的步驟。

STOP 模式

STOP 模式下不執行用戶程序。所有輸出均設置為替換值，從而將受控的過程置於安全的工作模式。

TeleService

藉助 TeleService，可通過遠程連接對分佈式設備進行集中管理、控制和監視。要通過 TeleService 建立遠程連接，必須使用 TS Adapter 或遙控 CP。

TS Adapter

TS Adapter 用於通過調制解調器將自動化系統連接至電話網絡，從而為使用 Tele-Service 準備電話網絡。TS Adapter 有兩種型號，即 TS Adapter MPI 和 TS AdapterIE。

TS Adapter IE

TS Adapter IE 泛指所有具備以太網接口的 TS Adapter。

TS Adapter MPI

TS Adapter MPI 泛指所有具備 MPI/DP 接口的 TS Adapter。

半雙工

半雙工是指通信技術中的一種數據交換方式，即數據可以在兩個方向上交替發送但不能同時發送。

保持性存儲器

保持性存儲器是指 CPU 內，在斷電後和在上電後從 STOP 切換為 RUN 時，其內容仍然保留的存儲區。保持性存儲器也稱為保持性存儲區。

保持性數據

保持性數據包括那些存儲在 CPU 保持性存儲區中，即使在電源關閉時仍然保留的變量、數據或塊。

保護等級

保護等級是多級保護理念的一部分，用於對 SIMATIC CPU (S7-300/400) 的功能和數據進行保護。保護等級分為“無保護”（允許訪問所有功能）、“寫保護”（只能讀取但不能更改數據）以及“讀/寫保護”（不允許訪問數據）。

報警

報警是事件驅動型信息，通過報警可以檢測到自動化系統中的過程控制錯誤、定位這些錯誤並將其清除。我們對程序報警、系統診斷報警和用戶診斷報警進行了

區分。

報警組態

報警組態可用於創建、編輯和編譯事件驅動型報警及其文本和屬性，並在顯示設備中顯示。

背板總線

自動化系統的背板總線為插入的模塊提供內部工作電壓，並允許在模塊間交換數據。

背景數據塊

背景數據塊存儲函數塊的形參和靜態數據。可以將背景數據塊分配給函數塊調用或函數塊的調用層級。

邊沿

邊沿是指數字信號（例如，輸入）的信號狀態變化。信號狀態從“0”變為“1”時，出現上升沿。出現下降沿時，信號狀態從“1”變為“0”。

邊沿檢測

需要藉助邊沿評估來檢測和評估信號狀態變化。

編程設備

編程設備 (PG, Programming Device) 是具有緊湊型且工業兼容設計的 PC。編程設備完全適合對 SIMATIC 自動化系統進行編程。

編程語言

編程語言用於創建用戶程序並以圖形或文本形式的指令提供用於創建用戶程序的已定義語言子集。用戶使用編輯器輸入這些指令，並將其編譯為可執行用戶程序。

標識數據

標識數據是保存在模塊中的一些信息，用戶可用以檢查工廠組態以及查找硬件變更或錯誤。可使用標識數據在線明確識別模塊。

撥號方法

撥號方法是接通電話的方法。有音頻撥號和脈衝撥號兩種方法。對於脈衝撥號電話，在撥打電話時會聽到咔嗒聲；如果電話連接使用的是音頻撥號，則聽到的是一連串的高頻音。

波特率

波特率（符號率）以波特為單位度量一秒內傳輸信號的狀態數。

參數實例

參數實例是一種用於將正在調用的塊的實例作為 in/out 參數傳送至調用塊的塊調用。

操作

操作是程序語句的一部分並指示處理器將執行的動作。

操作數

操作數是程序語句的一部分並指示處理器將處理的對象。

操作系統

操作系統將與特定控制任務不相關的所有 CPU 功能和順序組合在一起。

常數

常數是代碼塊中常數值的佔位符。常數用於提高程序的可讀性。示例：在函數塊中是指定佔位符，例如“Max_loop_iterations”，而不是直接指定一個值（例如，10）。當調用此佔位符時，會指定這個常數的值（例如，10）。

程序報警

程序報警用於報告程序同步事件，每個都指定給一個塊。在程序編輯器中創建，在報警編輯器中編輯。

程序段

程序段在代碼塊內劃分用戶程序。對於要使用 STL、LAD 或 FBD 編程的代碼塊，必須包含至少一個程序段。

程序段數據

通過過程映像或使用直接訪問在中央處理單元和信號模塊、功能模塊和通信模塊之間交換程序段數據。程序段數據可包括：信號模塊的數字量與模擬量輸入/輸出

信號，功能模塊的控制和狀態信息。

初始化字符串初始化字符串是由 AT 命令（用於調制解調器的準標命令）組成的字符串，用於初始化連接到 TS Adapter 的調制解調器。

觸點可以使用觸點創建或中斷兩個元件之間的載流連接。電流從左向右傳遞。可以使用觸點查詢操作數的信號狀態或值，並根據電流的結果對其進行控制。

觸發點

觸發點是在用戶程序中定義的點，例如，週期開始、週期結束、RUN --> STOP 模式切換時。觸發點確定在用戶程序中何時監視或修改所選變量。

觸發器

觸發器啟動測試功能並由觸發點和觸發條件定義。

觸發條件

觸發條件可用來決定要對所選變量監視或修改一次，還是永久性監視或修改。

從屬性結構

從屬性結構顯示程序中每個塊與其它塊的從屬關係。

錯誤 OB

錯誤 OB 是用戶可以用來對錯誤反應進行編程的組織塊。

錯誤反應

錯誤反應是操作系統對運行錯誤的反應。可能的錯誤反應包括：將自動化系統切換到 STOP 模式、調用用戶可在其中對反應進行編程的組織塊，或者顯示錯誤。

代碼塊

代碼快包含部分用戶程序的塊。

單個背景

單個背景是指被調用塊將其數據存儲到自身背景數據塊中的塊調用。

地址

地址是用戶程序中操作數或操作數區域的標識。示例：輸入 I12.1，存儲器字MW25，數據塊 DB3。

多重背景

多重背景是一種調用的塊將其數據存儲在調用函數塊的背景數據塊中的塊調用。

訪問保護

通過分配項目密碼來設置項目和庫的訪問保護。

訪問級別

訪問級別是多級保護理念的一部分，用於對 SIMATIC CPU (S7-1200/1500) 的功能和數據進行保護。訪問級別分為“無保護”（允許訪問所有功能）、“寫保護”（只能讀取但不能更改數據）以及“讀/寫保護”（不允許訪問數據）。

分佈式 I/O

分佈式 I/O 是指遠離中央機架安裝的模擬量和數字量模塊。

分配列表

分配列表顯示是否通過訪問從 S7 程序中分配了地址或是否已將地址分配給 SIMATICS7 模塊。分配列表概述了字節中的哪個位用於存儲區輸入 (I)、輸出 (Q) 和位存儲器 (M)。

副本模板

副本模板是一種庫元素。副本模板可用來創建彼此獨立的庫元素副本。

工業以太網

工業以太網 (IE, Industrial Ethernet) 是在工業環境中安裝以太網的安裝準則。與標準以太網的主要區別在於各個組件的機械強度、載流能力和抗干擾能力。

工作存儲器

工作存儲器是執行用戶程序時處理器所訪問 CPU 上的 RAM。

工作模式

操作模式用於說明 CPU 的相關狀態。CPU 可能具有以下幾種工作狀態：RUN、STOP、啟動、ERROR 和維護 (MAINT)。

功能框

功能框是具有複雜功能的程序元素。但空功能框除外。可以使用空功能框作為佔位符，在其中可以選擇所需的運算。

共享調試

在共享調試中，多個工程組態系統作為團隊工程組態的一部分同時對同一個 CPU進行調試。

固件

固件是內嵌在電子設備中的軟件。固件通常保存在閃存（如 EPROM、EEPROM 或ROM）中，並且只能由用戶使用特殊工具或功能進行替換。固件總是連接到硬件，二者相互依存，因此而得名。

固件更新

固件更新是指更新模塊固件。更新可以解決現有錯誤並增加新功能。

廣播計算機網絡的廣播是將其包含的數據包從一點傳輸到信息網絡中所有設備的消息。

廣播地址

廣播地址是網絡中的 IP 地址，所有設備均將讀取發送到該地址的請求。

軌跡和邏輯分析器功能可以使用軌跡和邏輯分析器功能來記錄設備變量並評估這些記錄。

過程映像

過程映像是 CPU 系統存儲器中的一個存儲區，其中包含數字輸入/輸出模塊的信號狀態。過程映像分區分為過程映像輸入 (PII) 和過程映像輸出 (PIQ) 兩種。

過程映像輸出

在執行用戶程序和讀取過程映像輸入之前，操作系統將過程映像輸出 (PIQ) 傳送到輸出模塊。

過程映像輸入

在啟動用戶程序之前，操作系統從輸入模塊中讀取過程映像輸入 (PII, process imageinput)。

過程中斷

過程中斷是指由過程中的事件而觸發的中斷。

函數根據 IEC 1131-3 標準，函數 (FC) 是不含存儲區的代碼塊。通過函數可在用戶程序中傳送參數。因此，函數特別適合取代頻繁出現的複雜結構，如計算等。

函數塊

根據 IEC 1131-3，函數塊是指帶有靜態數據的代碼塊。函數塊可用來在用戶程序中傳送參數，並且擁有存儲區（背景數據塊）。

函數塊圖

函數塊圖 (FBD) 是在 EN 61131-3 (IEC 61131-3) 中定義的一種語言，用於對可編程邏輯控制器 (PLC) 編程。函數塊圖使用與布爾代數相同的邏輯框表示邏輯功能，特別適合邏輯控制器。

互鎖

互鎖是指可在一個步內設定的條件。互鎖會影響單項動作的執行。

監視變量

使用“監視變量”功能，可在執行用戶程序期間在指定點監視用戶程序的變量，並讀取編程設備上各個變量的當前值。

監視表

使用監視表可以監視和修改用戶程序中的變量。在監視表中，還可以在 STOP 模式下為 CPU 的各外圍設備輸出分配固定值。

交叉引用列表

交叉引用列表總括了程序中 I、Q、M、P 和 DB 存儲區中操作數的使用情況，以及DB、FC 和 FB 訪問情況。

接口模塊

接口模塊通過現場總線將分佈式 I/O 系統連接到 CPU，併為 I/O 模塊準備數據。

結構

結構 (STRUCT) 是複雜的數據類型，它由不同類型的數據元素組成。

靜態數據

靜態數據是存儲在背景數據塊中功能塊的局部數據，因此，在下一次處理功能塊之前會一直保留。

局部數據

局部數據是指分配給代碼塊的、已在接口中聲明的數據。它包括形參、靜態數據和臨時數據。

可編程邏輯控制器

可編程邏輯控制器 (PLC, Programmable logic controller) 是指其功能作為程序存儲在

控制設備中的電子控制器。因此，設備的安裝和接線與控制器功能無關。

可編程模塊

可編程模塊是中央處理單元 (CPU)、功能模塊 (FM) 和通信模塊 (CP) 的通稱。

客戶端

客戶端是從服務器接收數據和服務的程序或計算機。

庫

庫用於存儲可重複使用的程序組件。例如，站、塊、PLC 變量表、過程畫面或畫面元素。可選擇一個項目庫和全局庫。

跨項目數據交換

跨項目數據交換可通過項目間工程組態 (IPE) 功能實現。可以通過 IPE 文件或項目文件將 CPU 的控制器數據從源項目不變地傳送到目標項目的 CPU 中。

塊

塊可將用戶程序分成獨立的部分。塊可分為：組織塊 (OB)、函數塊 (FB)、函數

(FC) 和數據塊 (DB)。

塊保護

塊保護（專有技術保護）可為各個塊分配密碼以防止它們遭受未經授權的訪問。

塊參數

塊參數是塊接口中的佔位符，調用相應的塊時會將其替換為當前值。

塊堆棧

塊堆棧是 CPU 系統存儲器中的存儲區。調用塊時，它會保存相應的有效數據塊的開始地址和返回地址。

塊接口

塊接口包含塊參數的聲明和代碼塊的本地數據。

塊調用

塊調用是指一個塊調用另一個塊的操作。

塊註釋

塊註釋包含有關未裝載到 CPU 工作存儲器中的塊的可選附加信息。

類型

類型是一種庫元素。可以使用類型獲取和使用相應庫元素的實例。這些實例與各自的類型關聯，對一個實例的修改也將改變所有其它現有實例。

連接表

連接表是用於定義網絡中可編程模塊之間的通信連接的表格。

臨時局部數據

臨時局部數據是在塊處理期間臨時存儲的塊的局部數據，但在接下來的處理中不再可用。

令牌

令牌是指對總線限時訪問的授權。

邏輯運算結果

邏輯運算結果是指處理器中的當前信號狀態，該狀態將用於後續的二進制信號處理。根據上一個邏輯運算結果執行某些操作。

模塊參數

模塊參數是用於設置模塊行為的參數。

目標系統

目標系統是運行用戶程序的自動化系統。

啟動 OB

啟動 OB 是一種當 CPU 從 STOP 進入 RUN 模式後立即由 CPU 操作系統調用的組織塊。例如，啟動 OB 可包含在系統掉電後進行啟動的預設默認設置。如果存在多個啟動 OB，則將按它們的 OB 編號順序進行調用。

啟動模式

啟動模式是 CPU 從 STOP 模式到 RUN 模式的過渡。冷啟動、啟動和暖啟動之間有所差異。

啟動事件

啟動事件是指導致操作系統啟動相應組織塊的預定義事件，如故障/錯誤或中斷。

啟動信息

啟動信息是指在特定組織塊啟動時操作系統提供的信息，該信息可在用戶程序中進行評估。

起始值

起始值是指系統在啟動時分配給變量的值。

強制

強制用於給 CPU 的各個外圍設備輸出永久性地分配固定的值。必須在強制表格中輸入變量及其強制值。

強制表

通過強制表，您可以在用戶程序中監視和強制變量。強制用於給 CPU 的各個外圍設備輸出永久性地分配固定的值。

全雙工

全雙工是指通信技術中的一種數據交換方式，即可以在兩個方向上同時發送數據。

設備

TIA Portal 中的設備是指控制器 (CPU)、HMI 設備和 PC 系統等各種不同的對象。設備的配置、參數分配和聯網在硬件和網絡編輯器中進行。

設備代理數據藉助“設備代理數據”對象，可實現在源項目和目標項目之間實現 CPU 控制器跨項目數據交換的一致性，而無需任何冗餘組態。

設備地址

設備地址是用來對網絡中的設備（例如，PG）或可編程模塊（例如，CPU）進行

尋址的地址。

時間戳

時間戳是在塊中生成的關於上一次更改的條目。

時鐘存儲器

時鐘存儲器是以固定的頻率週期性更改其信號狀態的位存儲器。

實參當調用功能塊 (FB) 或功能 (FC) 時替換形參的值。實例：由實參“I3.6”替換形參“Start”。

實例

背景是指調用分配了背景數據塊的函數塊。

輸出

輸出是 CPU 系統存儲器中的存儲區。

輸出參數

輸出參數是將數據和計算結果從程序中的塊傳送到調用塊中的參數。

輸入

輸入是指 CPU 系統存儲器內的存儲區（過程映像輸入），或者是指到輸入模塊的連接。

輸入/輸出參數

輸入/輸出參數用於將數據傳遞到被調用塊並將結果返回給調用塊。

輸入參數

輸入參數用於將數據傳遞到被調用塊以進行處理。

數據塊

數據塊用於存儲用戶數據。數據塊分為可由所有代碼塊訪問的全局數據塊，以及分配給特定功能塊調用的背景數據塊。

數據類型

數據類型指定在用戶程序中如何使用變量值或常數。

數據類型，複雜

複雜數據類型通過鏈接基本數據類型構成。結構和數組之間存在差別。結構包含不同的基本數據類型，而數組包含同一個數據類型的多個相似元素。

數據類型，基本

基本數據類型是根據 IEC 1131-3 預定義的數據類型。示例：數據類型“BOOL”定義了二進制變量（“位”），數據類型“INT”定義了 16 位定點數變量。

數據流控制

數據流控制可控制數據流，並藉助軟件或硬件協議指定在出現短期數據流中斷的情況下如何進行數據傳輸。

數組

數組是由相同類型的數據元素組成的複雜數據類型。

雙工

雙工是指通信技術中的一種數據交換方法，與通信通道的傳輸方向無關。全雙工和半雙工之間有區別。對於全雙工，數據可以在兩個方向上傳輸；對於半雙工，數據可以在兩個方向上交替傳輸，但不能同時傳輸。

順控程序

順控程序採用 GRAPH 編程語言，用於用戶程序編程。順控程序中的程序可分成幾個獨立的步驟，在轉換中指定前進到下一個步驟的條件。

順序控制系統

順序控制系統是一種使用順序命令或條件逐步執行程序的控制系統。

梯形圖

梯形圖 (LAD) 是指在 IEC 標準 EN 61131-3 中標準化的一種圖形編程語言，尤其適合邏輯控制器。梯形圖與電路圖表示相對應。

調用層級

調用層級是組織塊中塊調用的順序和嵌套。

調用結構

調用結構描述了 S7 程序中塊的調用層級。它提供了以下幾個方面的概要信息：所用的塊、到塊的使用點的跳轉、塊的本地數據要求和塊的狀態。

調制解調器

調制解調器（調製器/解調器）是允許計算機通過電話線發送和接收數據的通信設備。用於將計算機數字脈衝轉換為模擬信號，或將模擬信號轉換為數字脈衝。

調制解調器連接

調制解調器連接是通過調制解調器線路建立的連接。在安裝了 TeleService 的PG/PC 與通過 MPI/DP 或以太網接口插入了 TS Adapter 的自動化系統之間，通常使用這種連接。使用 TeleService，通常使用調制解調器連接。

通信處理器是指用於實現通信任務（例如，聯網、點對點鏈接）的可編程模塊。

通信連接

在待交換數據的兩個設備之間建立通信連接。通信連接要求連接到一個公用硬件媒介（例如，總線系統）上。在此基礎上，才可建立邏輯通信連接（軟件）。

團隊工程組態

團隊工程組態支持多個工程組態系統 (ES) 的多個編輯器同時訪問同一個共享CPU。

維護請求

維護請求是具有以下含義的維護信息：短期內必須更換相關硬件。

位存儲器

位存儲器是指 CPU 系統存儲器中的存儲區，用於保存中間結果。允許對位存儲器進行讀/寫訪問。位存儲器也可以被定義為保持性存儲器，從而在斷電後或在上電後從 STOP 切換為 RUN 時，仍然能保留其中的內容。

系統存儲器

系統存儲器集成在 S7-CPU 上並用作 RAM。系統存儲器包含操作系統內部所需的操作數區（例如，位存儲器）和數據區（例如，用於通信的緩衝區）。

系統錯誤

系統錯誤是指自動化系統中可能出現的錯誤，例如，CPU 中的程序錯誤或模塊故障。

系統控制點

系統控制點 (SCP, System control points) 是循環程序執行中的定義點，在該點上不對塊進行處理而是執行操作系統功能。

系統診斷

系統診斷是指在系統出現錯誤時，由操作系統啟動的對診斷信息的檢測和評估。

系統診斷報警

系統診斷報警是指由組態相關模塊事件觸發的報警。系統報警在硬件配置期間激活或禁用。這類報警在報警編輯器中只能查看，不能編輯。

顯示設備

顯示設備是用於顯示處理結果的設備。

現場設備

現場設備是連接到自動化系統或 I/O 設備的傳感器或執行器。

線圈

線圈用於控制二進制操作數並根據邏輯運算結果的信號狀態置位或復位操作數。

項目副本

項目副本是指與進行團隊工程組態時通過結構化主項目創建的項目副本；項目副本可分配到參與的工程組態系統中進行編輯。每個工程組態系統作為參與共同調試的成員，僅處理分配到的具體項目副本部分。各項目副本在共同調試結束時再次集成到主項目中。

項目間工程組態

項目間工程組態 (IPE, Inter Project Engineering) 支持跨項目數據交換。源項目中的控制器數據可以通過“設備代理數據”對象複製並導出到目標對象。

項目文本

項目文本是指項目內的所有文本（例如，報警文本或註釋）。可翻譯為相應的項目語言。

協議

協議是指傳送系統為實現有效的數據交換而使用的一種同步方法，例如 MRP。

信號模塊

信號模塊 (SM, Signal module) 構成了過程和自動化系統之間的接口。它們包括輸入模塊、輸出模塊和輸入/輸出模塊（數字量與模擬量）。

形參

形參是代碼塊的參數。它用作實際參數（調用時傳送到代碼塊）的佔位符。

修改變量

使用“修改變量”功能，可在執行用戶程序期間在指定點修改用戶程序變量，併為各個變量分配永久值。

需要維護

需要維護是具有以下含義的維護信息：在可預期的時間內必須更換相關硬件。

尋址

尋址是向操作數或操作數區域分配地址。示例：輸入 I12.1；存儲器字 MW25。

尋址，符號符號尋址使用名稱而不使用絕對地址。示例：將為輸入 |1.0 分配符號名稱“啟動信號”(Start signal)。

尋址，絕對在絕對尋址中，將指定要處理的操作數地址。示例：地址 Q4.0 指定了過程映像輸出的字節 4 的 0 位。

循環時間

循環時間是 CPU 執行一次用戶程序的所需時間。

循環中斷

循環中斷以恆定的時間間隔來啟動程序。

延時中斷

延時中斷是在用戶程序中啟動的延時到期後生成的中斷。

一致性數據

一致性數據是指在內容方面屬於一個整體，並且在數據傳輸過程中不可分開的數據。

以太網

以太網是連接到本地網絡中的設備之間以數據包形式進行數據交換的技術。

硬件

硬件 (HW) 是系統（例如，計算機系統）的機械設備和電子設備的統稱。

硬件限位開關

硬件限位開關是一種機械限位開關，例如，可限制軸的最大允許行程。

用戶程序

用戶程序解決獨立的控制任務。可將其分配給可編程模塊，並藉助各個塊進行構建。

用戶診斷報警

用戶診斷報警用於將用戶條目寫入 CPU 的診斷緩衝區。用戶可在報警編輯器中創建和編輯用戶診斷報警。

優先級

優先級定義當前正在運行的用戶程序的可中斷性，即高優先級的事件可中斷低優先級的事件。

語句表 (STL)

語句表 (STL) 是一種文本編程語言，與機器代碼相似。如果程序採用 STL 編寫，則各條語句代表 CPU 執行程序的步驟。

遠程連接使用 TeleService 通過電話網絡撥號連接遠程工廠時，即建立了遠程連接。通過遠程連接，可以按照常規方式操控遠程工廠。

遠程通信

遠程通信（也稱為遠程連接）是工廠和 PG/PC 之間建立的異步連接。使用指定的電話號碼，TS 適配器可以建立到 PG/PC 的遠程鏈路/遠程連接。

運行錯誤

運行錯誤是指在自動化系統中執行用戶程序時發生的錯誤。

增量式編碼器

增量式編碼器是指以數值形式輸出增量位置變化的位置編碼器。

站

站是指可以在沒有附加模塊的情況下運行的任何設備。例如，不能將單個的 DP從站稱為站，這是因為如果沒有相關聯的 DP 主站就無法對其進行操作。而智能DP 從站（帶有 CPU）則是站，因為它可以進行單獨操作。

診斷

診斷可檢測、提示和定位錯誤並確定適當的補救措施。在診斷中，系統診斷、過程錯誤診斷和用戶自定義診斷之間有所區別。

診斷報警

舉例來說，CPU 發送到顯示設備的預先準備的診斷事件便是診斷報警。

診斷緩衝區

診斷緩衝區是 CPU 中的備份存儲器，用於按照診斷事件的發生順序存儲一定數量的診斷事件。

診斷事件

診斷事件會導致在帶有診斷緩衝區的 CPU 或 CP/FM 的診斷緩衝區中創建一個條目。診斷事件包括模塊故障、過程接線錯誤、工作模式切換錯誤和 CPU 系統錯誤。

診斷數據

診斷數據是包含在錯誤消息中的信息，如診斷事件或時間戳。

診斷條目

診斷條目是藉助診斷事件輸入診斷緩衝區中的條目。

診斷中斷

診斷中斷是具有診斷功能的模塊通知 CPU 發生系統錯誤的中斷。

直接 I/O 訪問

直接 I/O 訪問是指不使用過程映像直接訪問 I/O 的指令。

直接訪問

直接訪問是指 CPU 通過 I/O 總線直接訪問模塊，而無需使用過程映像。

直接連接

直接連接是指通過 TS Adapter 在安裝了 TeleService 的 PG/PC 與自動化系統之間建立的直接連接。直接連接用於分配 TS Adapter 的參數。

指令

指令是用戶程序中最小的獨立單元。指令採用相應的編程語言編寫，代表處理器的指示，在用戶程序處理過程中會對其進行執行。

中斷

中斷是一種導致 S7-CPU 操作系統調用已分配組織塊（中斷 OB）的事件，用戶可在該組織塊中對所需反應進行編程。

中央處理單元

在自動化系統的中央處理單元 (CPU, central processing unit) 中存儲和執行用戶程序。它包含操作系統、處理單元和通信接口。

重新啟動

重新啟動是 CPU 的一種啟動類型。CPU 啟動時，會初始化所有非保持性系統數據和用戶數據。

主項目

主項目是支持共同調試的基礎項目。它是根據特定規則構建的項目，包括已完整組態的硬件配置以及全部所需的變量和塊。該項目裝載到共同使用的 CPU 中，然後作為“主項目”通過項目副本分發到參與的工程組態系統。每個工程組態系統僅處理分配到的具體項目副本部分。然後，再次將這些項目副本集成到主項目中。

專有技術保護

專有技術保護（塊保護）可用以為塊分配密碼的選項，防止塊遭受未經授權的訪問。

裝載存儲器

裝載存儲器是可編程模塊的一個組件。它包含在編程設備中創建的對象（裝載對象）。它既可以作為可插入存儲卡，也可以作為永久集成存儲器。

資源

程序信息中的“資源”顯示每個所用編程對象的 CPU 硬件資源、CPU 中存儲區的分配以及現有 I/O 模塊上所使用的輸入和輸出。

子網

子網包括所有未通過網關互連的網絡設備。可以包含中繼器。

自動化系統

自動化系統是由中央控制器、CPU 和各種輸入/輸出模塊組成的可編程邏輯控制器(PLC, programmable logic controller)。

字符延遲時間

字符延遲時間用於監視數據傳輸階段的字符到達情況。

總線

總線是多個設備之間通過共享傳輸路徑進行數據傳輸的系統。

總線系統

總線系統是一個數據傳輸系統，其中自動化系統的所有站都通過總線進行物理連接。

組態

組態是對自動化系統的組件進行選擇和組合，以及安裝所需軟件和根據特定用途

定製自動化系統。

組織塊

組織塊構成了 CPU 的操作系統與用戶程序之間的接口。在組織塊中指定用戶程序的執行順序。

最大循環時間

最大循環時間用於監視用戶程序的允許執行時間。如果用戶程序的執行時間超出了指定的最長循環時間，則操作系統會生成錯誤報警並將 CPU 切換為 STOP 模式。

最小循環時間

最小循環時間是指循環程序運行所需的時間，包括高優先級組織塊中所有嵌套程序的運行時間。
