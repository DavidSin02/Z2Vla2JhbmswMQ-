從技術上講，它實際上不是語言，而是規則及其結構可能非常複雜。 圍繞清潔CSS進行的討論和社區活動，以及關於清潔代碼的ter不休。

當開發人員瞭解CSS時，我們通常會學習如何在工作狀態下使用CSS。 這就像弄清楚如何在不真正瞭解JavaScript的情況下使Angular或React應用程序正常工作一樣。

為什麼？

因為大多數時候，它都可以正常工作。

基礎教程通常不會涉及選擇器的工作原理，如果選擇器可以工作，那麼儘管它們是CSS的基礎主題之一，但通常很快就會被淘汰。

或者以大量文本的方式完成。

事不宜遲，這是我對您對CSS選擇器的更深刻理解的貢獻。

選擇器快速短路

選擇器是CSS的一部分，它告訴瀏覽器目標代碼的哪一部分。 基本選擇器的目標是HTML標籤，例如h1，p和img標籤。 定位根標記後，您應用的所有樣式都會成為"全局"樣式的一部分。

這是通過CSS選擇元素的最簡單方法。

許多剛接觸CSS的開發人員往往會做的事情，尤其是當事情進展不順利時，是在CSS文件中進一步添加了另一個選擇器規則塊。

由於級聯，所寫的內容從上到下依次應用。 這會產生覆蓋效果，因此不建議使用，因為它會創建垃圾代碼。 這也使跟蹤問題的根本原因變得困難。

編寫CSS規則時，僅在一個地方的特定情況下為特定選擇器提供一個屬性。

這意味著，如果您已經聲明瞭帶有color屬性的p選擇器，則不應稍後在文件中更改它，除非它處於其他狀態，例如：hover

隨著CSS樣式的增長，這將省去跟蹤選擇器及其屬性的麻煩。

您還可以通過類和id選擇器來定位HTML的特定塊。 如今，ID只能使用一次，並且通常保留給JavaScript掛鉤。 類通常是HTML塊的一種更標準的方法，並且本質上比id更可重用。

類通過HTML中的class屬性聲明，並由CSS使用調用。 作為前面的符號。 選擇特定的id是相同的概念，其中id是屬性，而＃是前面的符號。

在DOM中，彼此直接相鄰的標記稱為同級。 這意味著h1和p是同級，而h1和ul不是同級。 兄弟姐妹必須彼此相鄰，否則放下h1 + ul將會打擊存在的每個可能的ul，無論其接近程度如何。

使用+號時，可以創建選擇條件。 它告訴您的瀏覽器選擇相鄰的兄弟姐妹，即直接位於另一個指定元素旁邊的元素組合對。

屬性選擇器

屬性選擇器通過其屬性定位HTML。

HTML屬性僅特定於特定元素，並且使用額外的元信息來豐富HTML。 它們通過目標元素作為目標，然後是由[]包圍的相關數據

以下是一些通過HTML屬性作為目標的常用且有用的屬性選擇器。

· a [href * =" twitter"] —定位具有href值且包含單詞" twitter"的元素。

示例： Twitter

· a [href ^ =" https：//"]-目標元素的href值以http：//開頭

示例：點綴的松鼠

· a [href $ ="。co"] —目標元素的href值以.co結尾

示例：公司與公司

· button [disabled]-定位已應用了disable屬性的按鈕元素

例如：<button type =" button" disabled>點擊我！</ button>

· input [type =" submit"]-如果輸入元素具有類型屬性，且其確切值為Submit，則將其定位

偽元素

偽元素將元素之前和之後的"空格"作為目標。 HTML中並不存在實際的空間，但是CSS插入了一個空容器，為您提供了一個可操作的空間。

讓我們看一下CSS以及將內容插入到哪裡。

.item:before { content: "icon.jpg"; }.item:after { content: "anotherIcon.jpg"; }

：before和：after偽元素通常用於顯示特殊的前置項目，例如圖標。 對於偽元素，content屬性是必需的，但可以保留為空白。 它們像其他任何元素一樣對待，可以相應地設置樣式。

雜類

像現有的一切一樣，也有一些雜亂無章的人不太適合一個類別。 儘管如此，仍然很高興知道他們的存在。

與：target和#someLinkHere有所不同

在HTML中，您可以鏈接頁面內的內容而無需刷新它。 您可以通過和href屬性執行此操作。 當在假定的鏈接前放置＃時，它將在DOM中查找相關的ID，並將頁面向下滾動到適當的位置。

代碼看起來像這樣：

<a href="#sectionOne">Section One</a><a href="#sectionTwo">Section Two</a>… some more code here …<div id="sectionOne">some content here</div>………<div id="sectionTwo">some content here</div>

當您單擊第一部分時，瀏覽器將查找href部分中設置的相應ID，並將您帶到頁面上的該特定位。 在樣式方面，：target可讓您配置選擇並導航至頁面上目標鏈接後的視覺效果。

因此，您的CSS可能看起來像這樣：

:target { border:solid 2px black; }

這將導致目標宿主元素（又稱div，基於上面的示例）周圍的邊框。 當您單擊其他目標元素時，樣式將從原來的樣式中刪除，並應用於新的目標。
