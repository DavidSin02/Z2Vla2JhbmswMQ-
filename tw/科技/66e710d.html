RF 433MHz發送器/接收器模塊使用方法，程序詳解

| 科技

使用Arduino控制RF 433MHz發送器/接收器模塊的完整教程

這篇文章是有關帶Arduino的用的相對比較多的RF 433MHz發送器/接收器模塊的指南。我們將解釋它們如何工作並分享一個Arduino項目示例，您可以將其應用到自己的項目中。



描述

在本教程中，我們將使用 FS1000A發送器和相應的接收器，但是這裡的講解說明也適用於以類似方式工作的其他433MHz發送器/接收器模塊。這些RF模塊在Arduino愛好者中非常流行，並用於需要無線控制的各種應用中。

這些模塊非常便宜，您可以將它們與任何微控制器一起使用，無論是Arduino，ESP8266還是ESP32。

規格射頻433MHz接收器

  • 頻率範圍：433.92 MHz
  • 調製方式：ASK
  • 輸入電壓：5V

    • 規格射頻433MHz發射機

  • 頻率範圍：433.92MHz
  • 輸入電壓：3-12V

    • 哪裡買（這個？）



    您只需花費幾快錢即可購買這些模塊。你可以比較多家商店的RF 433MHz發射器/接收器，並找到最優惠的價格。

    RF 433MHz發射器/接收器模塊的Arduino應用

    在本節中，我們將構建一個簡單的示例，該示例使用433 MHz將消息從Arduino發送到另一個Arduino板。一塊Arduino開發板將連接到433 MHz發射器，併發送" Hello World！"。信息。另一個Arduino開發板將連接到433 MHz接收器以接收消息。

    所需零件

    此示例需要以下組件：

  • 2個Arduino
  • RF 433MHz接收器/發射器
  • 麵包板
  • 跳線

    • 安裝RadioHead庫

    該庫提供了一種簡單的方法來與Arduino控制的433MHz的發射器/接收器模塊的應用。請按照以下步驟在Arduino IDE中安裝該庫：

    1. 下載該庫，您的下載文件夾中應該有一個.zip文件夾。

    2. 解壓縮RadioHead 庫。

    3. 將RadioHead庫文件夾移至Arduino IDE安裝庫文件夾。

    4. 重新啟動您的Arduino IDE

    RadioHead庫很棒，它可以與市場上幾乎所有的RF模塊一起使用。

    發射電路

    按照這個示意圖將發射器模塊連接到Arduino。

    將以下代碼上傳到將用作發送器的Arduino板。

    #include <RH_ASK.h>

    #include <SPI.h>

    RH_ASK driver;

    void setup()

    {

    Serial.begin(9600);

    if (!driver.init())

    Serial.println("init failed");

    }

    void loop()

    {

    const char *msg = "Hello World!";

    driver.send((uint8_t *)msg, strlen(msg));

    driver.waitPacketSent();

    delay(1000);

    }

    發射模塊程序的工作原理

    首先，包括RadioHead ASK庫。

    #include <RH_ASK.h>

    該庫需要SPI庫才能工作。因此，您還需要包括SPI庫。

    #include <SPI.h>

    之後，創建一個稱為driver的RH_ASK對象。

    在setup（）中，使用init（）方法初始化RH_ASK對象。

    Serial.begin(9600);

    if (!driver.init())

    Serial.println("init failed");

    在loop（）中，我們編寫併發送消息。該消息保存在msg變量中。請注意，該消息必須為char類型。

    const char *msg = "Hello World!";

    此消息包含" Hello World！" 消息，但是您可以發送任何所需的內容，只要它是char格式即可。

    最後，我們發送消息如下：

    driver.send((uint8_t *)msg, strlen(msg));

    driver.waitPacketSent();

    該消息每秒發送一次，但是您可以調整此延遲時間。

    delay(1000);

    接收電路

    按照下一個原理圖將接收器模塊連接到另一個Arduino。

    接收模塊程序

    將以下代碼上傳到連接到接收器的Arduino。

    #include <RH_ASK.h>

    #include <SPI.h>

    RH_ASK driver;

    void setup()

    {

    Serial.begin(9600);

    if (!driver.init())

    Serial.println("init failed");

    }

    void loop()

    {

    uint8_t buf[12];

    uint8_t buflen = sizeof(buf);

    if (driver.recv(buf, &buflen))

    {

    int i;

    Serial.print("Message: ");

    Serial.println((char*)buf);

    }

    }

    接收模塊程序如何工作

    與之前的草圖類似，您首先需要包含必要的庫：

    #include <RH_ASK.h>

    #include <SPI.h>

    您創建一個稱為driver的RH_ASK對象：

    RH_ASK driver;

    在 setup（）中，初始化RH_ASK對象。

    void setup(){

    Serial.begin(9600);

    if (!driver.init())

    Serial.println("init failed");

    }

    在loop（）中，我們需要設置一個與接收到的消息大小匹配的緩衝區。"你好，世界！" 有12個字符。您應該根據收到的消息調整緩衝區大小（空格和標點符號也算在內）。

    uint8_t buf[12];

    uint8_t buflen = sizeof(buf);

    然後，檢查您是否收到了有效的消息。如果收到有效消息，請在串行監視器中打印。

    if (driver.recv(buf, &buflen)) {

    int i;

    Serial.print("Message: ");

    Serial.println((char*)buf);

    }

    寫到最後

    使用此模塊時，您需要有一些現實的期望。當接收器和發射器彼此靠近時，它們可以很好地工作。如果將它們分開太遠，則會失去通信。通訊範圍會有所不同，這取決於您要向發射機模塊提供多少電壓，環境中的RF噪聲以及是否使用外部天線。
