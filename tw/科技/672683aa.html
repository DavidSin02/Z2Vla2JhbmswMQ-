自定義Laravel命令行make指令生成的文件

2020-11-14
| 科技

前言

使用Laravel或其他框架開發的時候，通常會創建一個Model基類，讓所有的Model繼承這個基類，但是使用 Laravel Artisan 控制檯指令創建的Model並不是繼承於我所創建的基類，為了避免每次都要手工修改，需要重寫源碼的生成方法

閱讀源碼並分析

服務提供者是 Laravel 應用啟動的中心，所有服務提供者都是通過配置文件 config/app.php 中進行註冊，那麼我們就直奔 config/app.php 文件找到控制檯的服務提供者


打開 ConsoleSupportServiceProvider.php 後，發現有三個服務提供者，目標文件是Artisan 控制檯服務提供者，無視其他兩個文件



閱讀 ArtisanServiceProvider.php 代碼後，找到了註冊創建Model命令的方法 registerModelMakeCommand



最後打開 ModelMakeCommand.php 文件，找到了Model生成器文件 stubs/model.stub



分析：只需要修改 stubs/model.stub 文件即可，但不提倡直接修改源碼，這裡選擇創建新的ModelMakeCommand服務，重寫 getStub 方法，重新綁定到服務容器

自定義模型創建命令服務

在項目根目錄下創建 Customize 目錄，用於存放自定義框架類，這裡我根據源 ModelMakeCommand.php 文件的存放路徑， 創建 Customize/Foundation/Console/ModelMakeCommand.php 文件，並繼承源類

<?phpnamespace App\Customize\Foundation\Console;class ModelMakeCommand extends \Illuminate\Foundation\Console\ModelMakeCommand{    protected function getStub()    {        if ($this->option('pivot')) {            return parent::getStub();        }        return __DIR__ . '/stubs/model.stub';    }}複製代碼

在當前文件目錄下創建 stubs/model.stub 文件

<?phpnamespace DummyNamespace;use App\Customize\Database\Eloquent\Model; // 自定義的Model基類class DummyClass extends Model{    //}複製代碼

重新綁定模型創建命令服務

兩種方式：

  1. 直接在 AppServiceProvider.php 中重新綁定新的ModelMakeCommand服務
  2. 創建一個新的服務提供者，並將其追加到配置文件 config/app.php 的 providers 數組中

測試

使用 php artisan make:model Test 創建一個Model，確認過眼神，是繼承於自定義Model基類的Model
