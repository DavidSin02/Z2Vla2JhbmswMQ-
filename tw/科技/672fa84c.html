1. 變量

程序就是對數據進行處理(運算)，比如"2+3"就是對數據2和3進行加法運算。

然而程序數據的值在編寫程序時並不是就能完全確定的，比如一個程序用於統計某個班級某門課的成績信息，

程序員並不知道這個程序將用於什麼班級？學生人數是多少？學生將來的考試成績是多少？

這些信息都是不可預知的，然後程序必須能適用不同情況下的班級課程成績的統計分析，就需要在程序中

用某種方式能表示不同情況下的“人數”“所有學生成績”“班級名稱”等信息，即需要在程序中有一塊內存

用於表示人數，一塊內存用於存放所有學生成績...

這些保存未知數值的不同類型數據的內存塊，應該可以通過比如名字來區分它們，來定位它們。

因此，需要給保存不確定數據的內存塊起一個名字，改名字叫作變量。

比如語句“double r;” 就定義了一個叫作r的數據類型是double（浮點實數類型）的變量。

變量r就表示了一塊可以存放“浮點實數類型double”的內存塊。

在計算機中double類型變量一般佔據8個字節，但不同操作系統佔用內存大小也許會不同。

#include <iostream>

int main() {

double r;

std::cin>>r; //等待用戶從鍵盤輸入一個實數給變量r

std::cout << 3.1415926*r*r << std::endl;

return 0;

}

這個程序中，變量r的數值來自用戶輸入，是會變化的。

而3.1415926是一個直接給出確定數值的量，稱之為“文字量”。

我們看到一個程序中的數據由 "變量"和“文字量”構成。變量對應一塊內存，

而文字量直接編碼在代碼中，沒有獨立的內存。

2. 類型

2.1 變量或文字量的類型

C++中每個變量和文字量都有確定的類型，類型決定了數據的內容（是整數、實數還是字符）

、類型也決定了對這種類型的數據能進行什麼運算，比如整數可以進行“取模(求餘數)”運算，

但實數則不能。對於變量來說，類型還決定了這種類型的數據佔據多少內存。

比如int整數類型通常佔用4個字節，而double實數類型則通常佔用8個字節，表示字符的char類型佔用1個字節

#include <iostream>

void main_1() {

int i = 3; //i和3都是int整型, i是變量，3是文字量

char ch = 'A'; //ch和'A'都是char字符型

double radius = 2.56; //radius和2.56都是double浮點實數類型

bool ok = false; //ok和false都是bool邏輯類型

std::cout << i << std::endl;

std::cout << ch << std::endl;

std::cout << radius << std::endl;

std::cout << std::boolalpha; //用來改變輸出bool型的值的形式

std::cout << ok << std::endl;

}

上述程序中定義了4個不同類型的變量，分別用對應類型的文字量“初始化”這些變量的值。

即變量i的內存塊裡存儲的數值是3。

2.2 sizeof運算符

我們可以用C++自帶的sizeof運算符來查詢一個變量或類型在不同操作系統中實際佔用的內存的大小

#include <iostream>

int main() {

int i = 3;

char ch = 'A';

double radius = 2.56;

bool ok = false;

std::cout <<"int整型佔用內存： " << sizeof(int) <<"個字節" << std::endl;

std::cout <<"i佔用內存： " << sizeof(i) <<"個字節" << std::endl;

std::cout <<"ch佔用內存： " << sizeof(ch) <<"個字節" << std::endl;

std::cout <<"radius佔用內存： " << sizeof(radius) <<"個字節" << std::endl;

std::cout <<"ok佔用內存： " << sizeof(ok) <<"個字節" << std::endl;

}

2.3 //變量的定義及初始化

從前面我們看到，定義變量時需要說明變量的類型，還可以給變量一個初始值。

多個同類型的變量可以在一個語句中定義，它們之間用逗號隔開。

如：

double radius,;

int i, i2 = 3, i3; //定義了3個int型的變量i, i2, i3，其中i2有一個初始值3。

bool _num_1;

變量名是一個滿足下列條件的"標識符"

a) 只能以字母和下劃線開頭

b) 只能包含字母、下劃線和數字

c) 不能與C++的關鍵字(也稱保留字)相同。比如"bool"、"int"、"sizeof"等

都是被C++語言已經使用的標識符，變量名不能和它們同名！

定義一個變量時，可以用=，(),{ }等它1個初始值。

其中{}初始化稱為“統一初始化”或“列表初始化”。

double d;

double d1{3.5};

double d2 = {3.5};

double d3 = 3.5;

double d4(3.5);

列表初始化{}： 不允許導致“信息損失” 的初始化

int i;

int i1{3.5};

int i2 = {3.5};

int i3 = 3.5;

int i4(3.5);

因為文字量3.5是double型，用它對int型變量初始化時，會進行"double"到"int"的類型轉換，

這會導致小數部分丟失，編譯器對於列表初始化{}進行檢查，不允許這種“信息損失”的

列表初始化。

請編譯下列程序，看看編譯錯誤是什麼？

#include <iostream>

int main() {

double d;

double d1{3.5};

double d2 = {3.5};

double d3 = 3.5;

double d4(3.5);

int i;

int i1{3.5};

int i2 = {3.5};

int i3 = 3.5;

int i4(3.5);

std::cout<<d<<std::endl;

std::cout<<d1<<std::endl;

std::cout<<i<<std::endl;

std::cout<<i1<<std::endl;

}

再看一個例子：

#include <iostream>

using namespace std;

int main {

double val = 7.9;

int val2 = 1025;

int x2 = val; // x2 等於 7

char c2 = val2; // c2 等於 1

int x3{ val }; // 錯: 截斷“信息損失”

char c3{ val2 }; // 錯: 收縮 “信息損失”

char c4{ 24 }; // OK:

char c5{ 264 }; // 錯：264超出char的值的範圍

int x4{ 2.0 }; // 錯： double到int的收縮 “信息損失”

}

注意其中c4和c5都是用一個int型值初始化，

但因為佔用1個字節的char只能表示2的8次方即256個不同的值，

表示數的範圍是[-127,127]或[0,255]，而264超出了其表示值的範圍了！

3. C++中的數據類型

3.1 基本類型

C++語言本身自帶了一些數據類型如int、char、double、bool等，這些C++語言自帶的數據類型

叫做“內在類型”(built-in類型)。C++還允許程序員定義自己的數據類型，如我們前面的cout就是

程序員自定義的ostream類型，用來表示一個“輸出流”對象。

C++內在類型分為“基本類型”和“複合類型”。

基本類型又分為“算術類型”和"void類型"。其中算術類型又包括“整型”和“浮點類型”。

整型又包括：

布爾類型：bool。表示邏輯值"true"和"false"

字符類型:

char: 字符類型，如'A'

wchar_t: 寬字符類型，如L'A'

char16_t： UniCode寬字符類型，如u'A'

char32_t： UniCode寬字符類型，如U'A'

整型：

short: 短整型,如3s

int: 整型

long: 長整型，如3L

long long: 長長整型,如3LL

浮點類型包括

float 單精度實數，如3.14f

double 雙精度實數，如3.14, -3.14e3

long double 長雙精度實數，如3.14L

3.2 signed和unsigned

除bool和擴展字符類型， 其他整型都區分: 符號signed和無符號unsigned。

因此結合signed 和unsigned，又派生出更多的基本類型，

不同類型變量佔用內存大小和值的表示範圍可見下表或網上搜索！

不同類型佔內存大小和取值範圍

3.3 auto

C++11引入了"auto"關鍵字，當定義一個變量有初始值時，不需要明確指定類型，

在變量前用auto關鍵字，編譯器就能自動從變量的初始值推斷出該變量的數據類型。

如：

#include <cmath>

#include <iostream>

#include <typeinfo>

using namespace std;

int main() {

auto b = true; // a bool

auto ch{'x'}; // a char

auto i = 123; // an int

auto d{1.2}; // a double

auto x = d+i; //從表達式d+i的值推斷x的數據類型

cout << typeid(b).name() << endl;

cout << typeid(ch).name() << endl;

cout << typeid(i).name() << endl;

cout << typeid(d).name() << endl;

cout << typeid(z).name() << endl;

}

其中用到了一個c++運算符typeid可獲得一個變量的類型信息，

然後通過".name()"調用返回類型對象的成員函數輸出類型的名字

3.4 類型轉換

“用運算符對不同類型的數據進行運算時”或 “定義變量的初始化值類型和變量類型不一致時”，

C++編譯器會將它們轉換為同一種類型。比如下列程序：

#include <iostream>

using namespace std;

int main() {

bool b = 42; // int型值42轉換為bool型值true,再對b初始化，b的值就是true

std::cout << boolalpha << b << std::endl;

int i = b; // b的值true轉化為int型值1，再初始化int變量i。

std::cout << i << std::endl;

i = 3.14; // double型值3.14轉換為int型值3，再對變量i賦值，i的值就是3

std::cout <<i << std::endl;

unsigned char c = -1; //給unsigned類型變量

//賦超出範圍的值時，取餘數

// -1關於[0,255]的餘數為

// -1 % 256 = 255

signed char c2 = 256; //給signed類型變量賦

//超出範圍的值時，結果不確定

std::cout <<c << std::endl;

std::cout << c2 << std::endl;

return 0;

}

混合int和unsigned類型時，會用同樣的“取模法”(餘數法)將負整數轉化為unsigned.

int main(){

unsigned u = 0;

u = u-1; //如果unsigned是32位整數: [0, 4294967295]

// (-1+4294967296) % 4294967296

std::cout << u << std::endl;

int i = -42;

cout << i + i << endl; // 輸出 -84

cout << u + i << endl; // 如果是32位整數，則輸出4294967264

// (-42+4294967296)%4294967296 = 4294967254

}

int main(){

int a = -1,b = -1;

unsigned c = 1;

cout <<a*b <<endl << a*c << endl;

}

4. 聲明與定義

聲明就是說存在這個名字的“類型、對象（變量）和函數”，但沒有完整的信息

定義是這個名字“類型、對象（變量）和函數”完整的描述

變量(對象)的定義不但說明這個對象的類型和名字還會分配一塊內存，

而對象的聲明則僅僅說明其類型和名字，並不分配內存

變量聲明：說明變量類型和變量名

變量定義：除“說明變量類型和變量名外，還會分配內存塊，並可能提供初始值。

變量聲明，需要用關鍵字extern 開頭且不能初始化。

extern int i; //說明存在叫作i的int型變量，但聲明並沒有為它分配內存

int j; //定義了叫作j的int型變量，併為它分配了一塊內存

extern double r = 2.5; //定義了叫作r的double型變量，併為它分配了一塊內存

//其初始化是2.5。包含初始值的儘管前面有extern，也屬於變量定義！

當然“變量的定義”也包含了“變量的聲明”，反之不然！

“變量可以聲明多次，但只能定義1次”，如：

extern int a;

extern int a;

int main(){

extern int a;

extern int a;

int b;

return 0;

}

可以將變量的聲明和定義放在不同文件裡，一般的聲明放在所謂的頭文件中，

如下面的頭文件util.h中包含了變量a的聲明

//util.h

extern int a;

//...

而變量a的定義在util.cpp文件中：

//util.cpp

int a = 0;

其他程序文件如test.cpp如果需要用到變量a，可以包含(#include)這個頭文件

//test.cpp

#include <iostream>

#include "a.h"

int main() {

a = 3*5;

std::cout << a;

}

5. 外部變量(全局變量)與局部變量

定義在某個函數內部的變量叫做“局部變量”或“內部變量”，反之，稱為“全局變量”或“外部變量”

對於基本類型的變量： 定義外部變量如果沒有給初始值，則默認為0.而對於內部變量，其值是不確定的。
