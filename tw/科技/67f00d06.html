學習廖雪峰的JAVA教程---異常處理（使用斷言）

2020-11-14
| 科技

斷言（Assertion）是一種調試程序的方式。在Java中，使用assert關鍵字來實現斷言。

我們先看一個例子：

public static void main(String[] args) { double x = Math.abs(-123.45); assert x >= 0; System.out.println(x);}

語句assert x >= 0;即為斷言，斷言條件x >= 0預期為true。如果計算結果為false，則斷言失敗，拋出AssertionError。

使用assert語句時，還可以添加一個可選的斷言消息：

assert x >= 0 : "x must >= 0";

這樣，斷言失敗的時候，AssertionError會帶上消息x must >= 0，更加便於調試。

Java斷言的特點是：斷言失敗時會拋出AssertionError，導致程序結束退出。因此，斷言不能用於可恢復的程序錯誤，只應該用於開發和測試階段。

對於可恢復的程序錯誤，不應該使用斷言。例如：

void sort(int[] arr) { assert arr != null;}

應該拋出異常並在上層捕獲：

void sort(int[] arr) { if (x == null) { throw new IllegalArgumentException("array cannot be null"); }}

當我們在程序中使用assert時，例如，一個簡單的斷言：

// assertpublic class Main { public static void main(String[] args) { int x = -1; assert x > 0; System.out.println(x); }}

斷言x必須大於0，實際上x為-1，斷言肯定失敗。執行上述代碼，發現程序並未拋出AssertionError，而是正常打印了x的值。

這是怎麼肥四？為什麼assert語句不起作用？

這是因為JVM默認關閉斷言指令，即遇到assert語句就自動忽略了，不執行。

要執行assert語句，必須給Java虛擬機傳遞-enableassertions（可簡寫為-ea）參數啟用斷言。所以，上述程序必須在命令行下運行才有效果：

$ java -ea Main.javaException in thread "main" java.lang.AssertionError	at Main.main(Main.java:5)

還可以有選擇地對特定地類啟用斷言，命令行參數是：-ea:com.itranswarp.sample.Main，表示只對com.itranswarp.sample.Main這個類啟用斷言。

或者對特定地包啟用斷言，命令行參數是：-ea:com.itranswarp.sample...（注意結尾有3個.），表示對com.itranswarp.sample這個包啟動斷言。

實際開發中，很少使用斷言。更好的方法是編寫單元測試，後續我們會講解JUnit的使用。

小結

斷言是一種調試方式，斷言失敗會拋出AssertionError，只能在開發和測試階段啟用斷言；

對可恢復的錯誤不能使用斷言，而應該拋出異常；

斷言很少被使用，更好的方法是編寫單元測試。
