1、矢量字體引入

使用點陣字庫顯示英文字母、漢字時，大小固定，如果放大縮小則會模糊甚至有鋸齒出現，為了解決這

個問題，引用矢量字體。

矢量字體形成分三步：

① 確定關鍵點，

② 使用數學曲線（貝塞爾曲線）連接頭鍵點，

③ 填充閉合區線內部空間。

如果需要放大或者縮小字體，關鍵點的相對位置是不變的，只要數學曲線平滑，字體就不會變形。


2、Freetype 介紹

Freetype 是開源的字體引擎庫，它提供統一的接口來訪問多種字體格式文件，從而實現矢量字體顯示。我們只需要移植這個字體引擎，調用對應的 API 接口，提供字體文件，就可以讓 freetype 庫幫我們取出關鍵點、實現閉合曲線，填充顏色，達到顯示矢量字體的目的。

關鍵點(glyph)存在字體文件中，Windows 使用的字體文件在 c:\Windows\Fonts 目錄下，擴展名為 TTF的都是矢量字庫，本次使用實驗使用的是新宋字體 simsun.ttc。

給定一個字符，怎麼在字體文件中找到它的關鍵點？

首先要確定該字符的編碼值：比如 ASCII 碼、GB2312 碼、UNICODE 碼。如果字體文件支持某種編碼格式(charset)，就可以使用這類編碼值去找到該字符的關鍵點(glyph)。有些字體文件支持多種編碼格式(charset)，這在文件中被稱為 charmaps(注意：這個單詞是複數，意味著可能支持多種 charset)。

以 simsun.ttc 為值，該字體文件的格如下：頭部含有 charmaps，可以使用某種編碼值去 charmaps 中找到它對應的關鍵點。下圖中的“A、B、中、國、韋”等只是 glyph 的示意圖，表示關鍵點。

Charmaps 表示字符映射表，字體文件可能支持哪一些編碼，GB2312、UNICODE、BIG5 或其他。如果字體文件支持該編碼，使用編碼值通過 charmap 就可以找到對應的 glyph，一般而言都支持 UNICODE 碼。


有了以上基礎，一個文字的顯示過程可以概括如下：

① 給定一個字符可以確定它的編碼值(ASCII、UNICODE、GB2312)；

② 設置字體大小；

③ 根據編碼值，從文件頭部中通過 charmap 找到對應的關鍵點(glyph)，它會根據字體大小調整關鍵點；

④ 把關鍵點轉換為位圖點陣；

⑤ 在 LCD 上顯示出來


下載到“freetype-doc-2.10.2.tar.xz”，下圖中的文件就是官方文檔：

參照上圖中 step1，step2，step3 裡的內容，可以學習如何使用 freetype 庫，總結出下列步驟：

① 初始化：FT_InitFreetype

② 加載(打開)字體 Face：FT_New_Face

③ 設置字體大小：FT_Set_Char_Sizes 或 FT_Set_Pixel_Sizes

④ 選擇 charmap：FT_Select_Charmap

⑤ 根據編碼值 charcode 找到 glyph_index：glyph_index = FT_Get_Char_Index（face，charcode）

⑥ 根據 glyph_index 取出 glyph：FT_Load_Glyph（face，glyph_index）

⑦ 轉為位圖：FT_Render_Glyph

⑧ 移動或旋轉:FT_Set_Transform

⑨ 最後顯示出來。


上面的⑤⑥⑦可以使用一個函數代替：FT_Load_Char(face, charcode, FT_LOAD_RENDER)，它就可以得到位圖。


3、在 LCD 上顯示一個矢量字體

1. 使用 wchar_t 獲得字符的 UNICODE 值

要顯示一個字符，首先要確定它的編碼值。常用的是 UNICODE 編碼，在程序裡使用這樣的語句定義字符串時，str 中保存的要麼是 GB2312 編碼值，要麼是 UTF-8 格式的編碼值，即使編譯時使用“-fexec-charset=UTF-8”，str 中保存的也不是直接能使用的 UNICODE 值：

char *str = “中”;

如果想在代碼中能直接使用 UNICODE 值，需要使用 wchar_t，寬字符，示例代碼如下：

01 #include <stdio.h>02 #include <string.h>03 #include <wchar.h>0405 int main( int argc, char** argv)06 {07 wchar_t *chinese_str = L"中 gif";08 unsigned int *p = (wchar_t *)chinese_str;09 int i;1011 printf("sizeof(wchar_t) = %d, str's Uniocde: \n", (int)sizeof(wchar_t));12 for (i = 0; i < wcslen(chinese_str); i++)13 {14 printf("0x%x ", p[i]);15 }16 printf("\n");1718 return 0;19 }}

以 UTF-8 格式保存 test_wchar.c，編譯、測試命令如下：

book@book-virtual-machine:~/10_freetype/01_wchar$ gcc -o test_wchar test_wchar.cbook@book-virtual-machine:~/10_freetype/01_wchar$ ./test_wcharsizeof(wchar_t) = 4, str's Uniocde:0x4e2d 0x67 0x69 0x66

每個 wchar_t 佔據 4 字節，可執行程序裡 wchar_t 中保存的就是字符的 UNICODE 值。

注意：如果 test_wchar.c 是以 ANSI(GB2312)格式保存，那麼需要使用以下命令來編譯：

gcc -finput-charset=GB2312 -fexec-charset=UTF-8 -o test_wchar test_wchar.c

4. 編譯

編譯命令(如果你使用的交叉編譯鏈前綴不是 arm-buildroot-linux-gnueabihf，請自行修改命令)：

$ arm-buildroot-linux-gnueabihf-gcc -o freetype_show_font freetype_show_font.c -lfreetype


它會提示如下錯誤：

freetype_show_font.c:12:10: fatal error: ft2build.h: No such file or directory#include <ft2build.h> ^~~~~~~~~~~~compilation terminated.

我們不是已經編譯過 freetype 並且把頭文件複製進工具鏈裡了嗎？怎麼還有這個錯誤？

我們編譯出 freetype 後，得到的 ft2build.h 是位於 freetype2 目錄裡，我們把整個 freetype2 目錄

複製進了工具鏈裡。

但是包括頭文件時，用的是“#include <ft2build.h>”，要麼改成：

#include <freetype2/ft2build.h>

要麼把工具鏈裡 incldue/freetype2/*.h 複製到上一級目錄，我們使用這種方法：跟 freetype 文檔保持一致。執行以下命令：

book@PC$ cd /home/book/100ask_stm32mp157_pro-sdk/ToolChain/arm-buildroot-linux-gnueabihf_sdk-buildroot/arm-buildroot-linux-gnueabihf/sysroot/usr/includebook@PC$ mv freetype2/* ./

然後再次執行以下命令：

$ arm-buildroot-linux-gnueabihf-gcc -o freetype_show_font freetype_show_font.c -lfreetype


5. 上機

將編譯好的 freetype_show_font 文件與 simsun.ttc 字體文件拷貝至開發板，這 2 個文件放在同一個目錄下，然後執行以下命令。

[root@board:~]# ./freetype_show_font ./simsun.ttc 或[root@board:~]# ./freetype_show_font ./simsun.ttc 300

如果實驗成功，我們將在屏幕中間看到一個藍色的“繁”字。
