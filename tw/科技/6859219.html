「4」MySQL中操作數據表的SQL彙總

2020-10-29
| 科技

在學完建表知識後，自然會想到該如何操作表的事情了，這就是本文的內容。

【0】引入問題


我們還是按照從問問題的老套路開始，羅列下表操作有哪些。

  • 如何增加與刪除表的列（請條友移步上一篇瞄一眼）？
  • 數據庫中有多少表？
  • 如何查看錶結構？
  • 表名如何修改？
  • 字段名如何修改？
  • 表的字段順序是否可以調整？
  • 表的字段類型是否可以修改？如何修改？
  • 表的存儲引擎如何查看與修改？
  • 表的主鍵如何創建？
  • 表的外鍵如何創建與刪除？

下面我們就這些問題一一展開。

【1】查看數據庫表

連接到數據庫之後，我們的第一反應當然是想看看有多少張表，每張表的表名如何。

show tables;


【2】查看錶結構

MySQL提供了兩種查看方式：一是查看基本表結構，一是查看詳細結構。

describe student_info; -- 基本表結構show create table student_info; -- 詳細表結構

【3】修改表名

-- 將student_info表改名為student_info2alter table student_info rename student_info2;

【4】修改字段名

-- 將字段grade更名為 nianjialter table student_info2 change grade nianji varchar(10);

注：此命令中change關鍵詞不能丟，另外新字段名後面要帶上字段類型。

【5】修改字段類型

-- 將字段nianji類型修改為varchar(32)alter table student_info2 modify nianji varchar(32);

【6】字段順序調整

-- 將字段nianji調為第一個字段alter table student_info2 modify nianji varchar(32) first;

-- 將字段nianji調為另一個字段後面，當然就能實現放到最後一位alter table student_info2 modify nianji varchar(32) after address;

【7】表的存儲引擎

如果仔細點我們就能發現在查看錶詳細結構的時候就看到了該表的存儲引擎。


-- 修改表的存儲引擎為MyISAMalter table student_info2 ENGINE=MyISAM;


此處我們發現，由於目標表存儲一個外鍵，所以導致引擎修改失敗。我們在接下來再解決這個問題。

【8】主鍵

一般主鍵是在建表的時候指定。

當然也提供了再建好表之後再建主鍵的功能，並且可以用多個字段組合創建主鍵。

-- 用兩個字段做主鍵ALTER TABLE bbs_user ADD CONSTRAINT pk_bbs_user1 PRIMARY KEY(uID,userName);

-- 刪除主鍵（不執行）ALTER TABLE bbs_user drop PRIMARY KEY;

【9】外鍵

通過核查，我們發現student_info2表的字段sno是表score_info的外鍵。此時我們嘗試刪除該外鍵。

alter table score_info drop FOREIGN KEY score_info_ibfk_1;

細心的人可能發現一個問題：刪除主鍵時不用指定主鍵名，而刪除外鍵時需要指定外鍵名？那是因為一個表只有一個主鍵，而可能有多個外鍵。

接著我們可以嘗試再次修改表的存儲引擎，看看是否可行。

當然也是可以恢復刪除的外鍵。

alter table score_info add FOREIGN KEY score_info_ibfk_1(sno) references student_info2(sno);

總結

本文羅列了八種表操作的情況，肯定不能覆蓋全部的情況，更多地希望起到拋轉引玉的效果。
