首先，我們先來回顧下環境搭建完畢後，我們用來測試環境搭建是否成功的例子：

WebDriver driver = new FirefoxDriver();Driver.get("http://youdao.com");WebElemnet element = driver.findElement(By.id("translateContent"));element.sendKeys("selenium");

我們先來看一下每一行代碼是什麼意思？他是幹什麼的？（不要嫌麻煩哦~）

WebDriver driver = new FirefoxDriver(); //初始化一個FirefoxDriver類型的對象，名叫driver
Driver.get("http://youdao.com"); //使用driver對象打開有道首頁
WebElemnet element = driver.findElement(By.id("translateContent")); //使用driver對象找到Html屬性id=translateContent的元素，也就是有道搜索的輸入框，名叫element
element.sendKeys("selenium"); //向element輸入框對象中輸入“selenium”關鍵字

從上面我們可以看出，自動化的語法是要先找到需要操作的元素對象，比如說driver和element；然後這個對象調用相應的API的方法，比如說get和sentKeys；然後再傳入數據，比如說By.id("translateContent")和"selenium"；最後進行我們所預設的自動化的操作。

第一行：瀏覽器driver對象，調用get方法，打開有道首頁

第二行：瀏覽器driver對象，調用findElement方法，傳入數據By.id("translateContent")，找到輸入框元素

第三行：輸入框元素element，調用sendKeys方法，傳入selenium關鍵字數據，進行搜素

WEB元素定位語法

在每一個web頁面中，充滿了大量的web元素，比如說每一個按鈕、連接、圖片、文字等等都是web元素，更準確的說他們是Html元素。

在seleniumwebdriver中，每一個頁面上的web元素都可以對應到一個WebElement對象；WebElement對象則可以通過WebDriver實例來獲得。在這裡selenium提供了一個尋找web元素的方法：findElement 。

在Java中，查找元素的方法通過一個定位器來定位web元素，這個定位器叫做By。

接下來我們聊聊元素定位的幾種方法。

首先，元素可以通過ID來定位。通過元素ID是元素定位最有效和最為推薦的方法。

元素（element）ID

頁面中的格式：<element id = "value">

我們來看一下有道首頁輸入框的元素ID如何來查看（PS：下面元素查看方法相同，我的文章風格比較詳細、囉嗦，大神可以忽略。）

a、訪問有道首頁，選中輸入框，點擊右鍵，選擇“檢查”（谷歌瀏覽器）或“查看元素”（火狐瀏覽器）

b、點擊“檢查”或“查看元素”，元素ID如下圖所示

找到了元素ID，我們就可以通過"translateContent"這個ID來找到輸入框元素。在selenium中的寫法：

WebElement element = driver.findElement(By.id("translateContent")) //driver對象調用findElement方法，傳入數據(By.id("translateContent"))

這裡的By就是selenium的定位器，By.id就是通過id來定位元素；很容易聯想，還有其他定位元素的方法。

再來看，元素可以通過Name來定位。

頁面中的格式：<element name= "value">

同樣我們依舊可以用有道搜索框的例子來查看元素name，如下圖：

在selenium中的寫法：

WebElement element = driver.findElement(By.name("q"));

這裡可以看到，我們傳入的數據變成了By.nama("q")，簡單說就是使用name定位器來定位name=q的元素。

繼續看，元素可以通過Class Name來定位。

頁面中的格式：<element class= "value">

同樣按照之前的步驟，我們在Html查看器中，找到【精選閱讀】tab，查看搜索框元素class屬性的值，如下圖：

因此，使用class定位器在selenium中的寫法是：

WebElement element = driver.findElement(By.className("active"));

另外，selenium還支持通過超鏈接文本來定位元素。

頁面中的格式：<element href= "value">LinkText</element>

在頁面中通常有大量的包含連接的元素，一個包含連接的元素通常是這樣的格式：

那麼我們要找的是連接的文本，也就是“下載詞典客戶端”，通過這幾個漢字我麼也可以找到這個元素：

在selenium中的寫法是：

WebElement element = driver.findElement(By.linkText("下載詞典客戶端"));

那麼有時候超鏈接的文本非常的長，如果使用一長串的文字來定位會使我們的代碼顯得不夠優雅，那麼我們可以使用缺省超鏈接文本來定位元素。

頁面中的格式：<element href= "value">PartialLinkText</element>

還是剛才的例子，我們找到“下載詞典客戶端”這個元素的位置：

這次我們只使用“客戶端”這三個漢字來定位元素，對應在selenium中的使用方法：

WebElement element = driver.findElement(By.partialLinkText("客戶端"));

在實際的自動化測試工作中，我們發現在同一個頁面會有多個元素擁有同樣的屬性，那麼我們通過這個屬性定位到的元素是不是就有很多呢？

沒錯，在selenium中我們可以獲取同樣屬性的所有元素。我們以缺省超鏈接文本定位方法為例，我們來獲取一下所有包含“有道”這兩個漢字的超鏈接文本元素。在selenium中的寫法如下：

List<WebElement> elemenets = driver.findElements(By.partialLinkText("有道"));

特別注意：我們元素定位定位調用的方法變成了“findElements”，該方法返回的是一個元素集合的list。

為了便於觀察，我們把獲取到所有元素的文本都打印出來：

對應到頁面上的我們可以找到這些超鏈接文本的位置：

在頁面中，每個元素都有一個tag，簡單可以理解為html元素類型；我們也可以使用這個tag來找到對應的元素。

頁面中的格式：<tag class= "value"></tag>

我們依舊使用有道搜索框作為例子，我們這次不是使用id或者name等屬性來定位元素，從下圖我們可以看到，這個元素本身是一個input類型的元素，那麼我們就用input類型這個tag本身來定位元素

在selenium中的寫法：

WebElement element = driver.findElement(By.tagName("input"));

另外，剛才提到了在頁面中通常會有多個元素有著同樣的tag，比如同樣是input類型的元素，頁面上有多個

那麼我們上面使用input類型來定位的元素，最終會定位到那個元素上呢？

在元素定位的過程中，如果頁面上存在多個滿足條件的元素，findElement方法只會返回頁面樹狀結構中自上而下的第一個滿足條件的元素。

下面的代碼，是模擬id、classname、和paartialLinkText三種方法定位元素的例子，大家可以看一下：

import java.util.List;import org.openqa.selenium.By;import org.openqa.selenium.JavascriptExecytor;import org.openqa.selenium.WebDriver;import org.openqa.selenium.WebElement;import org.openqa.selenium.firefox.FirefoxDriver;public class Elementlocating{ static WebElement highlighted = null; public static void main(string[] args){ //初始化firefoxdriver對象 WebDriver driver = new FirefoxDriver(); //開啟有道首頁 driver.get("http://www.youdao.com/"); //找到有道logo頁面元素對應的WebElement對象 WebElement logo = driver.findElement(By.id("logo")); //highlight方法為selenium高級應用，用於高亮顯示頁面元素 highlight(driver,logo); //找到“精選閱讀”頁面元素對應的WebElement對象 WebElement read = driver.findElement(By.className("active")); highlight(driver,read); //找到含有“有道”的頁面鏈接對應的對象列表 List<WebElement> elemenets = driver.findElements(By.partialLinkText("有道")); for(WebElement element:elements){ highlight(driver,elements); } //關閉瀏覽器 driver.quit(); }}public static void highlight(WebDriver driver, WebElement element) { JavascriptExecutor js = (JavascriptExecutor) driver; js.executeScript("element = arguments[0];" + "original_style = element.getAttribute('style');" + "element.setAttribute('style', original_style + \";" + "background: blue; border: 2px solid red;\");" + "setTimeout(function(){element.setAttribute('style', original_style);}, 1000);", element);}

其實從以上我們看出，web自動化的核心就是找到一個又一個webelement元素，無論是輸入框、按鈕、選擇框等頁面元素，都有著id、name、class等公共屬性，以及getid、click、sedkeys等公共方法；總結這些規律，定義一個公共的抽象類，這便是selenium webdriver的核心思想WebElement類的由來。
