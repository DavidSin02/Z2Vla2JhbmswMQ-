基於雙向匹配的陌生人社交業務、策略及算法思考

2020-10-29
| 科技

作為連接人的工具，社交產品的價值不言而喻；熟人社交領域，微信已佔據絕對霸主地位，但是在陌生人社交領域還未出現類似絕對地位的產品，今天就以探探和Tinder為例，跟大家聊聊陌生人社交。本文主要通過對業務策略的分析，引入一些數據、算法、架構體系設計的思考，希望可以讓大家在做算法和數據過程中瞭解到對業務洞察的必要性。

我們常用的資訊、短視頻、電商、音樂等，屬於典型的以單向匹配及滿意度為終結的產品，比如電商產品中的用戶->商品、短視頻產品中的用戶->短視頻等；而陌生人社交則是以雙向匹配滿意度為終結的產品。最大的不一樣就是滿意度體現在單向還是雙向匹配的完成度上，正因為這種不一樣，建模就異常複雜。所以，通過本文的梳理過程可以為我們帶來非常大的啟發 ( 其實招聘、打車、外賣等都存在一定的雙向匹配問題，有感興趣的小夥伴歡迎在文末留言探討交流 )。接下來，我們將開啟今天的探索之旅：

圖1 在線交友約會雙向匹配圖

交友匹配/推薦在廣義上，也是撮合交易的一種，在平臺產品上的展現形態，也以推薦、排序、匹配聊天為主。但交友匹配有極其獨特的特點，那就是人是無庫存的唯一品和非標準品。唯一指的是每個人幾乎各不相同，長相、性格、職業等等，完全不同於商城的商品推薦。非標是指，需求方對另一方是有要求的，而且在不同人、時間也不同，是非標準品；所以在非標、唯一的情況下，我們需要考慮當下單向交互、中期雙向匹配、長期群體成功率。

02

陌生人社交的特點及思考

在陌生人社交領域有哪些不同的特點呢，我們逐一來看看：

1. 用戶定位及標籤明顯

Tinder等陌生人社交產品的用戶定位及標籤明顯，用戶大多數情況會在單身、無聊、xxx等情況下安裝使用軟件。這是常態：在需要時下載安裝，用了幾天或幾個小時就卸載，當再有需求時又重新安裝使用。在熟人社交平臺中用戶往往會展現自己正向的那一面，極力保持個人良好的形象，而由於陌生人社交平臺有著鮮明的標籤特色 ( 比如Tinder、探探、陌陌都逃不過YP的標籤 )，用戶的訴求往往會有較大的差異，並且隱私要求更高 ( 不希望過多暴露真實信息 )；在陌生人社交平臺中很多用戶都會突破原有禁錮，以新的形象/不同於生活中的另一面，展現和認識一些人。

2. 雙向匹配及轉化鏈路長

上面也說過，大多數交友類APP最終的用戶體驗訴求是匹配成功 ( 即用戶開始聊天、交流、約會、成為男女朋友等 )，這個跟大部分互聯產品不一樣的地方是非單向網絡，雙向網絡存在雙向匹配問題，單向滿意效用低。雙向匹配網絡中，如果有用戶發現了很多心儀的對象，右滑表示喜歡以後，如果不能有一定比例的迴音，用戶可能很快流失，這也說明單單從一方來決定最終的展現是有問題的，而這些產品往往非常依賴於智能匹配技術，這就給推薦系統提出了新的挑戰。

圖7 典型的雙向匹配

在雙向匹配問題中，從展示到喜歡，再到被喜歡方交互和迴應，這裡面的轉化鏈非常長，而且存在用戶是否在線的問題；就算最終匹配成功後，用戶可以進行聊天和約會，我們還需要評估聊天的成功率；這裡面可能存在有很長的時間反饋延遲，用戶的體驗和收益需要被喜歡人反饋後才能真正體現。所以，鏈路長也帶來了成功率的下降。

3. 網絡效應和流動性

互聯網的本質就是一張信息網絡，互聯網產品普遍具有網絡效應，而社交類產品則更加依賴於網絡效應，邊際成本下降明顯，邊際收益上升明顯。那某個社交產品如何評估網絡效應的強弱，或者說達到多少才是合適的呢。前兩天跟一個朋友交流陌生人社交，他們有一個觀點就是在可互動 ( 比如基於LBS的產品，可互動指的是距離不能大於某個值 ) 的用戶每日用戶數需要達到2000以上，7日留存在40%以上，這個產品是初具網絡效應的，值得投入持續拉新最大的 ( 當然上述2000和40%會被具體產品的用戶活躍情況、性別比例、產品特性決定 )。比如可以分析每個區塊 ( 比如同城 ) 中人數對整體活躍度的影響，最終可以畫出如下的圖表，隨著用戶增長，拐點會出現 ( 拐點可能就是magic number )，規模臨界點前後的差異巨大。當然下圖需要在用戶體量增加時做好匹配，不然用戶量的上升可以使體驗下降，繼而帶來留存的下降。

圖10 上線用戶數與留存的關係

因為交友類產品存在地理位置的限制，如果某個地區匱乏的話，無法解決這個地區足夠數量的潛在匹配問題，一個用戶在若干次交互後再無新的用戶可交互。這種情況下，無論用戶體驗多麼有趣或參與度很高，這個地區的用戶就容易流失，網絡效應需要分地區。所以初期基於地域的屬性可以集中力量做片區發展，在某個或某幾個地區走通並驗證，則可以逐個地區鋪開。

陌生人社交就像我前面介紹的，用戶可能會有反覆下載和卸載的過程，並且不像熟人社交，用戶天然具有流動性。這是業務天然的優點和缺點，缺點是說今天用戶就不會像使用微信一下長期高頻使用，比如找到女朋友了。優點是每天有新的供給和需求在輪替。有足夠多的流動性，可以帶來新鮮感，才能產生更多的匹配與聊天。

由於流動性和網絡效應在雙向匹配中的影響，必然存在單邊失衡的情況，這時候出現了需求多供給少的情況，需要做好流量的平衡，動態維持一個比例，比如當性別失衡時 ( 突然大量湧入一波男性用戶 )，需要保證核心的用戶的體驗。活躍、留存高的用戶不一定是你的核心用戶，活躍、留存高、付費、且被別人喜歡的用戶才是你的核心用戶 ( 付費、或者被其他人消費 )。核心要素是找到一個生態系統良性運轉的本質，有流入有流出，動態平衡。這中間就是長期與短期目標的確定，長期需要考慮生態性問題，短期一般是可度量可迭代的。很多社交產品一開始做得很好，卻因為沒有很好的一套生態體系，逐漸走向末路。很多用戶的個性優勢、興趣需求互相不匹配，甚至感覺到干擾。只有當用戶自然流失等於或小於平臺自然流入，產品運營團隊才可以把更多精力放在服務用戶身上，而非拉新促活。

4. 哪方才是驅動力

Tinder的男女比例接近 ( 國內2款頭部陌生人社交產品男女比大概為8:2、7:3 )，那到底是男性還是女性才是社交類產品的驅動力呢？哪方的存在可以更多地拉昇產品整體的體驗，或者哪方對產品影響更大。目前幾款產品都是認為女性更重要，至少在冷啟動階段體現的特別明顯，兩方市場率先都引入了女性作為種子用戶，因為在當下從社會男女比例以及性別心理差異化方面看，男性更容易被異性吸引，且引流成本更低。雖然是雙向匹配，但是還是存在供給與需求的差異，冷啟動階段女生的角色更偏向於供給的角色。比如從國內較大的陌生人社交產品瞭解到的數據，用戶看了每100張照片，女生會選擇6%喜歡 ( 右滑 )，男生會選擇60%喜歡 ( 右滑 )，和現實中選擇態度很相似，男生對於陌生匹配的反感程度更低，耐受度更高，男生天然更開放，只要這裡異性多，對男生而言就是最好的服務了。維持這種生態系統運作的基本機制，通常是對女性提供更加開放友好的環境，吸引她們入駐，再以此輻射男性群體。這中間需要以女性視角關注產品 ( 男生希望快速觸達，女生希望更有安全感 )，所以相互匹配才能交流，從這個角度來看是一個偏向於女性的功能。

從另一個視角來看這個問題，在匹配效率達到一定值以後，核心用戶會被服務好，並且有不錯的留存。這時候用戶體量的外擴是兩方持續博弈的過程。在新用戶進入或老用戶流失過程中，永遠是非平衡態，一方的快速增長，需要快速推動另一方的新用戶進入。如果更深入地從匹配角度來看，假設產品的匹配效率和推薦沒問題，在現有用戶群體下，理想情況下互相滿意的匹配完成後，剩餘了一群人，這群人的性別比例才是根本。因為不是兩方數量達到一定比例就可以了，還需要考慮質量，如果男性需求過剩，需要新的女生進入產生供給，這部分女生不是引入對應的數量就可以了，而是要滿足上述過剩男生需求的女生，才能使得生態達到平衡。

當然最終哪個群體才是某個階段的核心驅動力呢？其實就是看供需關係、看貢獻，供小於需的那部分群體就是需要重要考慮和保護的，差體驗用戶 ( 賣貨、不法交易 ) 需要被限制，這部分人的存在和留存高反而會嚴重拉低其它用戶體驗和留存；不同用戶價值不同，留存的高低對系統影響也就應該區別對待。

5. 資源獨佔/頭部效應&有限供給

在電商、內容等領域，一個商品、短視頻可以海量地分發給各個用戶，但是在社交特別是交友領域，這肯定是不行的，道德問題和雙向匹配滿意度問題都限制了分發的有限性。每個用戶作為供給，只能完成一定次數的被匹配 ( 而且這個次數一般不會大 )，同一人能與多人匹配並聊天，但是一般只能與一人成為男女朋友談戀愛。如果不考慮道德問題，用戶的交際帶寬也是有限的，無法做到人人有反饋 ( 例如在探探中發現有個女生在幾天之內被喜歡了30w次 )。優質用戶也只能提供有限的供給，這和書籍或者商品很不一樣，並不是只要有庫存就可以同時推薦給很多用戶。美女帥哥在平臺上從來不缺被喜歡，很容易成為產品的頭部用戶，他們也可以為產品吸引更多的用戶進來；如果我們過度推薦這些用戶可能會讓這些用戶收到到上萬個喜歡的轟炸，但他們無法提供高的反饋率，比如下圖中的頭部用戶拿到了幾倍於普通用戶的喜歡，但是回覆率和匹配率卻是顏值一般的人的五分之一水平 ( 來自於OkCupid Blog )。

圖11 顏值與收信數的關係，回覆率與顏值的關係

頭部效應不受控的話，海量匹配會被沉沒，帶來了雙向匹配成功率的快速下降，而且佔用了大量的曝光機會。不能大部分人沒人喜歡，沒有曝光，需要平衡。所以這裡面就要求在流量分發的過程中做流控，但是如何在這有限次展現中獲得優質匹配是個大學問。既要儘量為這個用戶匹配到合適的用戶，但是又不能海量地將其展現給各類用戶，就算Ta可能被大部分喜歡。

圖12 匹配相關的流量宏觀調控系統

舉個例子，今天有2個候選人分別為A和B，現在有個用戶C使用產品，他對A和B均比較滿意，但是A跟C的匹配度更高，這時候給C首先展現A好像比較自然。但是如果現在用戶D進來了，他對A比較滿意，對B不滿意，這時候是不是在C來的時候，為其展現B更合適，因為後面D來可以匹配A，這樣匹配度為100%，不然的話就是50%。上述問題中，有效供給、時間先後效應，全局最優匹配度問題均暴露了出來。好的很容易被匹配，不要過早匹配，而是留給另一個還不錯且更苛刻的用戶？那前提是你知道ta是好的，並且為了還有匹配機會，你才會在初期匹配度較高的情況下，不對ta進行匹配。比如頭部的用戶往往是相對優質的用戶，但是他們也是單點，存在有限供給問題。推薦系統中存在趨熱問題，召回打壓，排序如何打壓，最終控制減少頭部效應。比如引入每個用戶的受歡迎程度指標 ( 可以通過仿真系統模擬其未來的被喜歡數和匹配概率等 )，可以為匹配時是否進行平緩限流有影響。因為有限供給，如果在達到限流條件後，其被匹配方若無迴應或迴應負向，則可以再次釋放這個用戶進入新的匹配。有限供給下的問題，是全局優化問題，但是看看能否拆解到局部優化，每次匹配儘量實時，這個比較重要，如果能拆，那就好辦非常多了。你有一盤不錯的貨，上午來了很多人，詢價了問你賣不賣，賣了感覺下午還有人過來，不賣下午可能匹配不好，大概這個意思。對用戶分層，並構建流量配比實驗。這中間如何分層，從被喜歡最多的群體裡面反推他們喜歡的人，如果這群人沒有再往外推，那麼自成一個群體；剩餘人群繼續這個操作。

6. 新人

契合上面提到的流動性問題，在池子中存在流出情況，並且被需求方用戶消費過的用戶無法做二次展現 ( 不能給一個人反覆看他之前看過的 )，這時候新用戶的快速引入就非常有必要。由於匹配概率的準確率依賴於數據，所以如何讓新人在只有靜態信息時 ( 地區、設備、年齡、職業等 )，快速定位其喜歡和被喜歡的人群，這中間對新用戶進行聚類是一個很重要的點；通過靜態信息初步匹配後，在後續的左右滑過程中 ( 平衡好Exploit & Explore )，儘量多樣的積累用戶信息非常重要。這裡需要兼顧精準與多樣，利用用戶實時的正反饋 ( 比如右滑 ) 來快速修正其初期的群體偏好，當然這中間存在匹配不精準，流量的浪費情況。

初期新用戶對匹配或對高顏值的期待，會非常有別於老用戶，比如往往初次體驗對顏值期待更高，當然通過數據分析，我們發現顏值也是測試偏好和拉新的手段。但長期來說，匹配聊天成功率是普遍的終極需求。新用戶的首次匹配成功，開啟聊天的時間間隔，會對長期留存產生較大的影響。

7. 心理學效應

序列效應：這個效應與幼鵝效應/錨定效應類似。用戶在瀏覽產品內的其他用戶時，對其中某個用戶的喜歡或不喜歡往往會受前面女生的影響。舉個例子，比如用戶看到的其他用戶的顏值順序是一般、一般、一般、一般和美、美、美、一般，兩個序列中第四個人為同一人，有可能在第一個序列中用戶4會被喜歡，在第二個序列中用戶4可能不會被喜歡。推薦的整體序列問題，給一群難看的加一個一般的，和一群不好看的加一個好看的。比如在流量分發過程中通過list-wise方式的強化學習建模，目標提升累計的長期收益。

8. 風控

社交類產品，第一肩負著較大的"輿論與監管的壓力"，第二劣質流量將破壞氛圍，所以建立一套風控體系既可以降低產品監管下架風險，也可以提升體驗，並且降噪 ( 劣質用戶行為數據基本無價值 )。

一個好用戶好不容易在平臺上匹配到了一個其他用戶，結果這個用戶上來就賣貨，這樣的經歷基本可能就讓用戶卸載了APP。除了上述這種情況，還有些用戶使用網紅甚至明星的非本人照片來做頭像，有些用戶不是來交友的，他們用非本人信息匹配完成後，開始騷擾用戶，類似惡意用戶無非就是各類拖、賣貨、色情等。我們可利用圖像技術識別用戶的信息真偽性 ( 比如頭像是否被其他人使用，如果是明星存在被其它使用的可能 )；也可以利用NLP技術識別用戶聊天階段是否出現違禁詞、是否存在兜售商品或者騷擾性詞彙等。

當然這裡面推薦也可以發揮價值，對於風險高但不明確的用戶分發過程中不做海量分發，雖然可能短時間內有較好的顏值和被喜歡率，並且反饋及時；也可以開通用戶舉報反饋渠道，當存在用戶投訴情況時，將被投訴用戶送入審核通道，不再推薦上給予流量。

9. 為了匹配效率，進行全面的數據收集

為了讓匹配效率更高，並且識別出隱藏的惡意用戶或低價值用戶，我們需要收集更多的數據進行預測和篩選。比如前置顯性數據收集，可以在用戶進入APP後，加一些對自己的描述，或者對另一半要求標籤，當然這個功能男女生使用頻率差異大。當然也需要記住的是用戶不會告訴你一切，並且用戶可能也無法準確描述自己的需求。這時候就需要收集一些"隱式"的數據信息，比如當新用戶進來時，在為其推薦了一批用戶後，Ta的交互 ( 喜歡or不喜歡，是否查看某個用戶的詳細標籤信息、是否有較長的停留等 )，這些都可以反映出用戶感興趣的對象和不感興趣的人，這背後利用算法和數據技術進行分析就可以得到一個隱式的用戶偏好。

數據非常有用，但也需要進行降噪，比如上述的偏好建模過程中，需要進行噪聲數據的識別，比如滑動速度與是否是謹慎可以用來判斷用戶隨意性。比如用戶滑動速度小於1s，基本不做停留，而且基本大量右滑，這時候可能需要丟棄這個數據。去噪類的方法看上去很簡單，但是最終線上的效果卻可能很好。

10. 社交產品中的感性因素

社交場景中存在很多感性因素，對於這些感性因素的把握，也可能會帶來不錯的收益，比如：

持續刺激：短時高強度給予被匹配用戶改為持續間隔刺激對用戶刺激及留存有較大的影響。比如頭部、中腰部用戶獲取到的關注、匹配率、被喜歡率均高出其他人很多，這時候可以引入蓄水池，在這些匹配完成後 ( 比如有100個match )，不是在用戶打開APP的瞬間完成全部展現，而是在用戶使用APP的過程中，每隔一段時間 ( 比如5分鐘或者更久 ) 或者用戶滑卡疲勞時，彈出匹配，讓用戶在體感上有在線持續有匹配，在這個產品上自己很受歡迎，類似的體驗可以拉昇用戶活躍和留存。

地理位置可能勝過顏值：由於陌生人社交有較強的可得性需求，所以對於LBS也是較敏感的，在距離上的貼近可以帶來用戶對於可得性的滿足。比如在顏值相當或者略差的情況下，用戶可能對於近距離的匹配也會更感興趣，這個在陌陌上特徵明顯。

快匹配和慢匹配：陌陌單向可發起匹配和交流，Tinder和探探需要雙向匹配才可發起聊天；陌陌屬於瀑布流形式，一屏可以看到若干用戶，直接可進行比較，而Tinder和探探則是單圖滑卡型；雖然後者匹配速度慢，難度大，但是後者對用戶的騷擾更少，匹配質量會高於前者。作為Soul這類不看顏值的匹配，就更加突出了效率低和質量高。

男女匹配策略差異：前面也講了男女用戶在匹配過程中注意力的差異，男看女只關注外貌，均勻分配給男生；女生看男生完全不一樣，顏值曲線波動特別大，證明推薦和個性化維度會有更強的效能。

群體有偏與誰先說話：根據前面哪個是核心驅動力的基礎上，我們可以對匹配進行改造，非核心用戶與核心用戶完成匹配後，非核心用戶發起第一次聊天后才為核心用戶展示已匹配成功。比如男生A與女生B完成匹配，男生會先收到匹配信息，然後如果男生說了第一句話，才給女生展示已經匹配的信息，這樣女生的體驗會更好。

1. 挑戰

業務特點：流動性 ( 用戶卸載與下載循環，非強需求 )、新用戶、消耗速度快 ( 用戶消費的就是用戶 )、有限的供給關係。

指標的非獨立性和聯動：首先，指標之間不可能完全獨立，我們在優化一個指標的同時很容易對另一個指標產生副作用。比如互動率、留存、付費率等。一個產品上線初期 ( 其實中後期也很難 )，不可能在太複雜的業務邏輯下，完全依靠算法達成產品目標，所以初期可能需要加入一些規則進行宏觀控制，在機制運行過程中進行調整糾偏。

3. 相關數據指標及特徵

擬合一個目標最好的方式之一就是將這個目標的歷史先驗信息加入進來，比如一個被喜歡率很高的用戶，在未來任然高的概率比歷史被喜歡率低的要高。所以一個用戶的歷史被喜歡率、喜歡率、匹配率都可以加入特徵去影響匹配。

反饋率/在線率：這個指標需要計算用戶在收到一個被喜歡後，多久會反饋，因為雙向匹配網絡中，兩個人在互相右滑的間隔中，越短的間隔可以帶來更好的首滑用戶體驗。反饋率=被喜歡後進行喜歡操作的時間間隔，在線率=一天24小時內在線次數 ( 時間間隔小於30分鐘記為1次 )。

顏值分：用戶會在註冊使用產品中上傳照片，通過圖像識別進行顏值打分，這個分偏向於大眾分，屬於大部分人的審美；在匹配階段使用的圖像Match分更加重要，畢竟蘿蔔青菜各有所愛。

用戶標籤：當然用戶雙方選擇並不是完全看顏值，畢竟大部分都是 ( 長相 ) 普通的人，所以可以通過標籤及用戶生活圈層的相似程度來為每個用戶建立一套算法作為推薦參考。比如某大陌生人社交平臺上有超過70%的人填了各種標籤，平均每人填了15個以上的標籤。這些都是在照片背後的信息，可以用來為匹配增加更多信息決策依據，而且越詳細的標籤也代表了用戶的認真程度，可以加分不少，也是一個特徵。在前期匹配，標籤數據不一定有顏值這麼有影響，但是長遠來說，跟圖片無關的文本型的信息，但是對聊天的影響非常大 ( 比如職業、學歷、背景相同可能話題更多，聊天更多 )。

其它特徵還有很多，比如用戶的年齡、性別、使用頻次、活躍時間段、地理位置 ( 城市、小區 )、星座、生日、畢業院校、工作行業、工作崗位、歷史匹配數/率、左右滑卡數等等。對於統計類指標及樣本進入模型是需要降噪，異常低頻和高頻的需要進行處理。

4. 精準與去中心化

雙向匹配算法中有兩個要素：一個是精準度，一個是去中心化。既要控制頭部的用戶的超級火爆，第二個就是需要精準扶貧，對匹配困難戶扶一把。這兩個點真正做好的非常少，因為這兩個一定程度上是相互衝突的。

這裡面需要突出的是分別對供需建模 ( 供需關係 )、決策匹配建模 ( 匹配率預估 )、分配建模 ( 流控、運籌規劃 )、匹配時效建模 ( 時間預估 )。

5. 核心系統

針對上述的問題，需要構建起幾大系統對網絡、社會環境進行建模分析，比如仿真系統、計劃系統、控制系統、指標系統、溯源系統。

指標系統：業務的核心指標統計分析中心。

控制系統：實際業務中的召回、排序、業務邏輯、流量控制等模塊。

溯源系統：通過日誌埋點，記錄歷史每個時刻各個系統的狀態，方便未來回溯分析。

仿真系統：即"沙盤模擬環境"，交友類產品要求用戶之間深度參與服務過程，線上試錯成本極高。仿真系統主要是覆蓋主流程，對於細節部分可以忽略，一般精度在8-9成，結合溯源回放歷史過程對業務建模、用戶行為模擬等方法模擬線上場景，仿真通過選取指標系統中的核心指標輸出報告，做到離線預估，當然這個系統是逐步完善的。

計劃系統：利用全局歷史數據、單粒度用戶歷史數據進行預測後，進行規劃儘量逼近全局最優的流量和曝光分配方案，並通過控制系統在實際系統中產生影響，當然這個過程也是在實時調整的，客觀和虛擬的實時融合，通過數據反饋時刻都在推演未來，就像下圍棋一樣，每一步的落子前多往前看幾步棋，別人落子後，重新推演計算下一步方案。

總結

社交類產品的發展壯大一定是感性與理性交織，個體匹配 ( 微觀 ) 和全局匹配 ( 宏觀 ) 兼顧的。儘可能從微觀出發，利用數據和算法，得到宏觀，並通過類似財務對賬機制，保證兩者的一致性，一致才能驅動優化。

社交類產品異常複雜，如何制定正確的優化指標真的很難，需要構建閉環，並縮短業務迭代的閉環，提升迭代效率，小步快跑試錯。數據質量是地基，保證好質量，並多做覆盤分析。業務理解 > 工程實現；數據 > 系統 > 算法。
