postman教程-05-強大的斷言功能

2020-11-14
| 科技

如果沒有斷言，我們只能做接口的功能測試，但有了斷言後，就為我們做自動化提供了條件，並且在postman中的斷言是非常方便和強大的 。

斷言簡述

我們先來了解下postman斷言的一些特點 ，具體如下

  • 斷言編寫位置：Tests標籤
  • 斷言所用語言：JavaScript
  • 斷言執行順序：在響應體數據返回後執行 。
  • 斷言執行結果查看：Test Results

在上面我們介紹到，編寫的斷言代碼是JavaScript，那如果不會寫怎麼辦 ？ 不用擔心，因為postman已經給我們內置了一些常用的斷言 。用的時候，只需從右側點擊其中一個斷言，就會在文本框中自動生成對應斷言代碼塊 。

斷言

常用斷言

接下來就讓我們瞭解一些常用斷言，還是按響應的組成來劃分，分別是狀態行，響應頭，響應體。

狀態行中又包括狀態碼，狀態消息 。在postman也可以對這兩個進行斷言

狀態行中的斷言：

  • 斷言狀態碼：Status code: code is 200
pm.test("Status code is 200", function () {    pm.response.to.have.status(200);        //這裡填寫的200是預期結果，實際結果是請求返回結果});
  • 斷言狀態消息：Status code：code name has string
pm.test("Status code name has string", function () {    pm.response.to.have.status("OK");   //斷言響應狀態消息包含OK});

響應頭中的斷言

  • 斷言響應頭中包含：Response headers:Content-Type header check
pm.test("Content-Type is present", function () {    pm.response.to.have.header("Content-Type"); //斷言響應頭存在"Content-Type"});

斷言響應體(重點)

  • 斷言響應體中包含XXX字符串：Response body:Contains string
pm.test("Body matches string", function () {    pm.expect(pm.response.text()).to.include("string_you_want_to_search");});     //註解pm.expect(pm.response.text()).to.include("string")      獲取響應文本中包含string
  • 斷言響應體等於XXX字符串：Response body : is equal to a string
pm.test("Body is correct", function () {    pm.response.to.have.body("response_body_string");});//註解pm.response.to.have.body("response_body_string");   獲取響應體等於response_body_string
  • 斷言響應體(json)中某個鍵名對應的值：Response body : JSON value check
pm.test("Your test name", function () {    var jsonData = pm.response.json();    pm.expect(jsonData.value).to.eql(100);});//註解var jsonData = pm.response.json()   獲取響應體，以json顯示，賦值給jsonData .注意：該響應體必須返會是的json，否則會報錯pm.expect(jsonData.value).to.eql(100)  獲取jsonData中鍵名為value的值，然後和100進行比較

響應時間(一般用於性能測試)

  • 斷言響應時間：Response time is less than 200ms
pm.test("Response time is less than 200ms", function () {    pm.expect(pm.response.responseTime).to.be.below(200);   //斷言響應時間<200ms});

對單個請求斷言

以下案例是我們最常見的在單個請求中編寫的斷言，具體如下 。

案例說明：

針對以下接口返回的數據進行斷言：

{    "cityid": "101120101",    "city": "濟南",    "update_time": "2020-04-17 10:50",    "wea": "晴",    "wea_img": "qing",    "tem": "16",    "tem_day": "20",    "tem_night": "9",    "win": "東北風",    "win_speed": "3級",    "win_meter": "小於12km/h",    "air": "113"}
  • 斷言響應狀態碼為200
  • 斷言city等於濟南
  • 斷言update_time包含2020-04-17

為集合斷言

除了可以為單個請求斷言，postman還提供了非常方便的為集合斷言，它的使用場景就是當多個請求的返回參數和參數值相同時，我們就可以編寫一個通用的斷言，最後達成的效果就是對每個請求的某個值進行斷言 。

編寫通用斷言的位置是在集合或集合的文件夾中 。具體位置如下圖：

打開集合斷言

案例說明：

  • 對項目中每個接口返回的響應狀態碼進行斷言，同時對用戶管理模塊下每個接口的code進行斷言。

實現步驟：

  1. 選擇其中一個集合，進行編輯，選擇Tests標籤，在文本域內輸入斷言響應狀態碼的代碼塊
  2. 選擇用戶管理文件夾，進行編輯，選擇Tests標籤，
  3. 批量運行該集合，就會查看到每個用例中都會
