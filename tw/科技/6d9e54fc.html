dbutils+反射實現自定義ORM框架，完成數據庫切換過程

2020-10-29
| 科技

背景

產品開發過程中，由於數據庫性能問題，需要更換數據庫，從一個 NoSQL 數據庫切換到 RDB 數據庫。

原 ORM 框架的 DAO 層抽象了一個頂層的父類，換庫方案是以此類為基礎，重寫一整套操作，可以將換庫工作量降低到最少。那麼工作量就落在瞭解決 ORM 映射問題上，原來使用的 BSONObject 工具包，直接完成Java對象到 JSON 對象的轉換。現在需要利用原始的 JDBC、重寫一套 ORM 映射機制。

本文來介紹 ORM 的實現過程，基本思路是：自定義一個 MyBeanHandler ，利用 Java 反射機制完成 JDBC 的 ResultSet 和 JavaBean 的轉換。

實現流程

基本思路：將數據庫的一條查詢結果，轉換成Java實體類的實例，自定義一個ResultHandler，接受一個類型 Class。

由於 select 列可能不是 Java 全部的屬性，所以不能遍歷 Java 屬性列表，在逐個從 ResultSet 獲取該列的值，一旦某一列沒有被查出來，ResultSet 的 getObject(entityField) 的時候直接報異常了。

所以轉換思路，以查詢結果列，反向找到該列對應的 Java 類的屬性，再用反射調用屬性的 setter 方法，完成實例的組裝。

此外，需要注意 Java 實體類屬性的類型和數據庫表列的類型，如果存在不一致的話，需要特殊處理。例如：Java 中 boolean 類型的屬性，對應到數據庫中是 tinyint 。

還有一個特別重要的問題就是 ResultSet 中獲取的元數據信息中，所有的數值列都是 Java的封裝類型，即：tiny/int/bigint 對應的 Class 是 Integer.class 。

Java 的封裝類型和基本類型的 Class 是不同，int/long/byte/char/float 的 Class 是對應的類型名稱 .class ，如 int.class，long.class。因此，調用 setter 方法觸發反射的類型一定要傳遞 Java 實體屬性的類型。

反射獲取類屬性

開發過程中有些模塊的實體有公共屬性，所以有不少實體類有基礎父類屬性，這些實體類對應的表字段，除了有自己定義的屬性外，還必須包括父類的屬性。

反射獲取實體字段方法時，必須要獲取父類所有字段，保證表字段的完整：

此外，數據庫表列名稱和 Java 屬性 setter 直接的映射關係，可以自定義的，通用的方法是二者名稱一致，對應的就是 setXXX,getXXX,isXXX(boolean類型）。

如果數據庫列表是下劃線，那麼就是另一種映射關係，這些都可以自定義映射規則的。那麼獲取列的 getter/setter 方法為：

/**	 * 獲取某個屬性的get方法名稱	 * 	 * @param name	 * @return	 */	public static String obtainGetterName(String name,boolean isBooleanType) {		//特殊處理boolean類型，統一為isXXX		if(isBooleanType){			return "is" + name.substring(0, 1).toUpperCase()+name.substring(1);		}				return "get" + name.substring(0, 1).toUpperCase()+name.substring(1);	}	/**	 * 獲取某個屬性的set方法名稱	 * 	 * @param name	 * @return	 */	public static String obtainSetterName(String name) {		return "set" + name.substring(0, 1).toUpperCase()+name.substring(1);	}

自定義BeanHandler

按上述思路修正原來實現的 BeanHandler 如下：

public class MyBeanHandler<T> implements ResultSetHandler<T> {	private Logger logger = Logger.getLogger(MyBeanHandler.class);	private Class<T> entityClass;	public MyBeanHandler(Class<T> entityClass) {		this.entityClass = entityClass;	}	@Override	public T handle(ResultSet resultSet) {				try {			while (resultSet.next()) {				T result = null;				result = entityClass.newInstance();				ResultSetMetaData metaData = resultSet.getMetaData();				int columnCount = metaData.getColumnCount();				for(int col = 1 ; col <= columnCount ; col++){					String name = metaData.getColumnName(col);					Object oldValue = resultSet.getObject(col);					Object columnValue = oldValue;					if(columnValue==null){//空，直接過濾掉。。。。						continue;					}										//獲取該字段名稱對應的Java實體類字段聲明：updated on 2017-09-05					Field field = IntrospectUtil.getFieldOfEntity(entityClass,name);					if(field==null){					       continue;					}										Class<?> columnType = field.getType();					if (ArrayList.class == columnType || List.class == columnType) {						// 數據庫存儲的是JSON，迴轉成List對象						List<Object> list = JSONObject.parseArray(columnValue.toString(), Object.class);						columnValue = list;					} else if (IntrospectUtil.isSimpleTypeField(columnType)) {						// non op 普通屬性，直接設置值					} else if(IntrospectUtil.isBooleanTypeField(columnType)){						columnValue = (Integer)oldValue==1 ? Boolean.TRUE:Boolean.FALSE;					}else {						// 複雜對象，且非List，通過JSON轉換為對應類型的對象						columnValue = JSONObject.parseObject(columnValue.toString(), columnType);					}					//找到對應簡單字段的原始字段類型，作為setter的方法類型					String setter = IntrospectUtil.obtainSetterName(name);					Method method = entityClass.getMethod(setter, columnType);					method.invoke(result, columnValue);				}								return result;			}		} catch (Exception e) {			logger.error("MyBeanHandler error", e);		}		return null;	}}

啟示錄

反射應用時，Class 類的 getDeclaredField 能獲取當前類的所有屬性，靜態、非靜態的都能獲取到。如果插入操作需要根據實體屬性集合類定義 insert 後面的字段，需要排除掉靜態成員變量。

此外，父類的成員變量即使定義成 protected 的，也不能直接被子類用 getDeclaredField 方法獲取到；但是 getMethod 方法不僅能獲取到自己的方法，同時也能獲取父類繼承來的方法。
