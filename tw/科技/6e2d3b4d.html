泰克信號發生器應用之串行總線故障模擬

2020-10-29
| 科技

串行總線通訊由於其良好的抗干擾能力和連接的便利性，在汽車電子、航空、航天、通訊電子以及工業現場得到了廣泛的應用。

串行總線通訊採用在導線上串發送數據的通訊方式，即一個信號可能包括地址信息、控制信息、數據信息和時鐘信息。如果串行總線設備設計錯誤，或串行系統連接不正確、不可靠，又或者應用環境複雜的情形下，總線通訊將不可避免的出現錯誤。處理這些錯誤在組建串行通訊系統或設計基於串行總線的儀器設備中是一個重要問題，研發人員和用戶都希望能對 相應錯誤處理機制進行儘可能完善、接近真實的測試， 此時就需要一個便捷的故障注入設備。

使用泰克任意波形發生器可以產生各種形式的任意波形，但信號發生器通常驅動能力有限，在輸出幅度和輸出形式上也可能無法達到要求。因此泰克和方案合作伙伴共同開發了串行總線故障信號模擬選件。它包括運行在 PC 機上的串行總線故障信號模擬軟件和信號隔離和電平適配器。

串行總線故障信號模擬軟件可以控制任意波形發生器方便地產生用戶所需的串行總線通訊故障信號，模擬總線上出現的通訊錯誤；信號隔離和電平適配器將任 意波形發生器產生的信號轉換為符合總線電氣要求的電平，並提供電氣隔離的保護功能。總線故障信號模擬選件可以使用戶產生應用中不易出現但致命的總線通訊錯誤情況，從而加深對總線通訊過程的理解，並設計更加安全的總線（設備）系統故障處理機制。

泰克方案介紹：

泰克科技與合作伙伴基於信號發生器產品提供了串行總線模擬仿真方案，無論是通用的RS232總線，還是有特定應用場景的CAN、FLEXRAY，1553B總線， 都可以使用泰克函數發生器，配合專用的信號隔離適配器，在軟件控制下產生相應的總線信號，並模擬各種總線故障條件下的輸出，信號隔離適配器可以將函數發生器的輸出端與被測設備進行隔離，並增大信號的輸出驅動能力。同時可以根據被仿真信號的特點， 改變信號的輸出形式 ( 比如單端信號轉差分信號 )。

泰克測試方案特點：

支持串行總線協議包括：MIL-STD-1553B，USB( 低速 / 全速 )，CAN，RS-232，I2C等；

基本參數設置包括電平幅度，碼速率，截止位和奇 偶教研方式等；

可以設置獨立的傳輸內容，包括幀長度和每個數據 幀的具體內容；

可以仿真特定的故障問題，比如同步誤差，停止位錯誤或奇偶校驗位錯誤，編碼錯誤等；

配置專用的信號隔離適配器，將函數發生器產生的 基礎信號變為對應的差分信號或其他類型所需要的信號形式；

連接方案

串行總線故障模擬方案

使用泰克信號發生器AFG3000 系列配合信號隔離適配器產生 CAN 信號，信號隔離適配器可以將AFG 3000 產生的模擬信號轉變為符合 CAN 標準的差分信號進行輸出。

軟件可以對1553B 總線協議中的任意字節內容進行設置和故障模擬(包括同步錯誤，奇偶校驗錯誤和曼徹斯特編碼錯誤)。

信號隔離適配器的輸出端口，BUS口為三同軸接口，可以直接輸出差分信號，OUT1/OUT2可以與示波器連接進行信號觀測。

使用MDO 3000示波器的串行總線解碼功能對產生的總線信號進行解碼檢驗

總線仿真的內容：

可以仿真的總線項目包括：

MIL-STD-1553B 總線

正常通訊：數據字 / 狀態字 / 命令字

故障類型：奇偶校驗錯誤 / 同步錯誤 / 曼徹斯特錯誤

USB 總線（低速 / 全速）

正常通訊：SOP/EOP/ 暫停 / 恢復 / 復位 / 令牌包 / 數據包 / 握手包

故障類型：PID錯誤 /CRC5/CRC16錯誤 / 毛刺錯誤 / 位填充錯誤 /SE1錯誤 / 毛刺錯誤

CAN 總線

正常通訊：遠程幀 / 數據幀 / 過載幀 / 錯誤幀

故障類型：確認錯誤 /DLC 錯誤 /CRC 錯誤

FlexRay 總線

正常通訊：啟動幀/空幀/同步幀/標準幀/非啟動 幀 / 非空幀 / 非同步幀

故障類型：標題CRC錯誤 / 幀CRC錯誤

總線事件：喚醒 /TSS/BSS/FES/DTS

RS-232 總線

正常通訊：故障類型：奇偶校驗錯誤 / 停止位錯誤

I2C 總線

正常通訊：啟動/重新啟動/停止/EEPROM數據 讀取

故障類型：確認丟失 / 地址字段無確認
