不容錯過的 5 個微型 Linux 發行版

2020-10-29
| 科技

這些微型 Linux 發行版可以讓你的老爺機復活，可以啟動一個損壞的系統，或者是確保在公共電腦上進行安全的操作。

-- Seth Kenlon（作者）

可供日常使用的 Linux 發行版比比皆是，不過其中有一些發行版常常被我們忽視，因為它們實在是太小了。但這些微型 Linux 發行版其實是一種非常強大的創新：使用一套完整的操作系統驅動一臺只有不到 1 GB 存儲空間和 512 MB 內存的計算機，真的是終極的黑客作風。

微型發行版的用法有很多種，比如說：

  • 從垃圾桶邊挽救回那些又老又慢的電腦。你可以繼續使用那些本來已經計劃報廢的機器，直到它們徹底解體（而不是在剛開始感覺有點兒慢的時候就扔掉）。
  • 使用 U盤啟動一個損壞的系統來恢復數據或者修復啟動分區。
  • 確保在安全和隱私的操作環境下使用公共電腦。如果使用 U 盤啟動酒店大廳或者圖書館裡的一臺公共電腦，你是可以確定操作環境是安全的。

輕量級發行版有很多種，比如說 Lubuntu 、 Peppermint OS 和 Bodhi ，但是那些真正微型的發行版又有一些獨到之處。下面就是你不容錯過的五個微型發行版：

Tiny Core


Tiny Core Linux


Tiny Core Linux 小得近乎不可思議：終端版本只有 11 MB，圖形界面版本只有 16 MB。我翻了一下之前收集的舊 U盤，最小的一個是 128 MB 的，也有 Tiny Core 鏡像文件的八倍之大呢。

Tiny Core 默認包括只包括了基本的操作系統，你需要通過以太網下載需要的應用程序。由於設計得極端精簡，甚至安裝完整操作系統的應用程序都沒有被包含在內（不過需要的話可以從 Tiny Core 的軟件倉庫下載）。

我使用過一個 128 MB 的 U盤在一臺只有 512 MB 內存的機器上運行了 Tiny Core，對於一個只有 16 MB 的操作系統來說，效果算是非常棒了。只有在使用網頁瀏覽器的時候速度才會變慢，但這主要是由於大部分現代網站太過複雜，而不是 Tiny Core 的問題。

如果不使用圖形界面，運行 Tiny Core 就只需要 64 MB 的內存了。

安裝

下載 Tiny Core 並使用 dd 或者 Etcher 寫入 U盤。

你只需要點擊屏幕底部啟動欄上的 Apps 圖標下載 tc-install 或者 tc-install-GUI 應用，就可以輕鬆安裝 Tiny Core了。


Tiny Core installer


安裝 Tiny Core 有幾種不同的方式。你可以把它安裝在一個格式化為 Linux 驅動器的 U盤裡（這要求你的電腦支持使用 USB 驅動啟動。大多數現代電腦都支持，但是在老一些的電腦上不太常見），或者安裝在微軟 FAT 文件系統的 U 盤裡（這對於大多數不支持從 USB 驅動啟動的電腦來說非常管用），或者甚至安裝在一個現有 Linux 分區的一個文件夾裡。

安裝過程非常快，完成之後就可以重啟計算機，進入到 Tiny Core Linux 系統中啦。

應用程序

由於系統自帶程序基本上只有一個文本編輯器和一個終端，你所要做的第一件事情就應該是安裝一些應用程序。底部啟動欄上的 Apps 圖標展示了 Tiny Core 提供的所有軟件包。Apps 軟件倉庫同時包含了一些重要的驅動程序，對於使用 WiFi 網卡或者是打印機等等都很有幫助。

在安裝一個新的應用程序或者實用程序的時候，你可以選擇在 Tiny Core 啟動的時候就加載軟件包，或者是需要的時候才加載。如果選擇啟動時加載，那麼不僅該軟件立即就可以使用，並且（不出所料地）下次重啟之後也依然可用；如果選擇需要時加載，那麼在軟件包下載完成之後仍然可以馬上使用，但是重啟之後就不會被自動加載到內存中了。這樣可以保持很快的開機速度，並且只佔用很少的內存，但同時也意味著每次開機之後，該應用的軟件包只有在第一次被使用的時候才會被加載到內存中。

可供選擇的應用程序同時包括像 office 和圖像應用之類的用戶端應用，以及像 Samba 和網站服務器這種的服務端應用。

當然了，隨著你在 Tiny Core 上添加的應用程序越來越多，它就不那麼“微型”了。不過在 Tiny Core 的網站上我們可以看到，即使是包括了所有 WiFi 驅動程序的 Tiny Core Plus 鏡像文件也只有大約 100 MB，所以“不那麼微型”也仍然很可能比 256 MB 要小很多。

結論

Tiny Core 非常適合性能不佳的老爺機、用來通過網絡啟動的鏡像文件，以及任何更看重應用而不是操作系統的人。Tiny Core 可以作為一個很好的週末工程來實踐：從 16 MB 開始一步步搭建操作系統，直到你感覺這個操作系統已經足夠滿足你的需求了。

SliTaz


SliTaz Linux


SliTaz Linux 的鏡像文件有大約 51 MB 大小，差不多是 Tiny Core 的四倍，但是包含一整套出色的驅動程序和應用程序。事實上，如果事先不知道的話，你可能會以為是通過一個 1 GB 的 Ubuntu 鏡像啟動的，因為能想到的任何一個基本啟動鏡像應該有的東西都在這兒：文本編輯器、網頁瀏覽器、繪畫工具、表格工具等等。

我使用過一個 128 MB 的 U盤 在一個 512 MB 內存的機器上運行了 SliTaz，效果非常不錯。瀏覽複雜網站的時候性能會下降，但是系統包含的輕量級瀏覽器 Midori 可以快速加載絕大多數網站。

你可以在啟動的時候選擇進入沒有圖形界面的 SliTaz，這樣在僅僅只有 64 MB 的機器上也可以很好地運行。

安裝

可供下載的 SliTaz 有很多種，因為它的開發者和社區針對可能存在的限制提供了非常多的版本。比如說，有一種低內存版本可以在只有 24 MB 內存的機器上運行；有一種版本使用 Firefox 而不是 Midori；還有一種版本沒有包含額外的應用程序，等等。

如果你挑花了眼，只想趕緊選擇一個版本嘗試一下的話，那就 下載滾動發佈版本吧 。這個版本有差不多 50 MB 大小，每週都會更新。如果你愛上了 SliTaz，而滾動發佈版本又更新得 過快 了的話，可以再選擇一個更符合你需求的版本。

下載好你選擇的 SliTaz 鏡像文件之後，你就可以用 dd 或者 Etcher 將它寫入 U 盤，然後重啟。

將 SliTaz 安裝在 U 盤或者硬盤上需要通過 TazPanel 這個應用程序來實現。它會引導你對硬盤進行需要的分區，然後將 SliTaz 安裝在你選擇的地方。


SliTaz installer


應用程序

SliTaz 的控制中心是 TazPanel 這個應用程序。如果你喜歡 OpenSUSE 或者 Mageia (最初被稱為 Mandrake)，那 TazPanel 對你來說應該不會陌生（至少在核心思想上）：包括系統設置、硬件監測、用戶和用戶組的管理、系統升級、安裝應用程序在內的這些功能，都在這一個應用程序內實現。

SliTaz 提供的應用程序可以滿足大多數基本需求，如果你不是非常在意完成某一項任務必須使用哪一個應用程序的話，那麼在 SliTaz 的軟件倉庫裡應該可以找到你想要的應用。如果你有一些特別的需求（比如說想要使用 GIMP 2.10 而不是 GIMP 2.8），那麼就需要學習如何生成 SliTaz 軟件包了。好消息是，tazpkg 命令支持從好幾種軟件包格式轉換過來，包括：

  • Debian 軟件包（.deb，.udeb）
  • RPM 軟件包（.rpm）
  • Slackware 軟件包（.tgz）
  • Puppy 軟件包（.sfs，.pet）
  • NuTyX 軟件包（.cards.tar.xz）
  • Arch 和 Alpine Linux 軟件包（.apk，.pkg.tar.gz，.pkg.tar.xz）
  • OpenWrt 軟件包（.ipk，.opk）
  • Paldo 軟件包（.tar.bz2）
  • Void 軟件包（.xbps）
  • Tiny Core 軟件包（.tce，.tcel，.tcem， .tcz）

結論

SliTaz 是一個快速而小巧的 Linux 發行版，並且非常容易上手（因為有一箇中心化的控制面板）。由於它的軟件包工具支持從其它格式的 Linux 軟件包轉換成自己的格式，它的應用程序理論上來說是非常豐富的，你可以很容易地使用喜歡的工具搭建自己的工作環境。SliTaz 很小，但是也非常具有殺傷力，正如它的蜘蛛 logo 所暗示的那樣。

Porteus


Porteus Linux


Porteus 提供了不同的桌面環境可供選擇，最小的鏡像文件大約在 270 MB 左右，最大的有 350 MB。它是微型 Linux 中鏡像文件最大的一個，但是這些額外的空間都被用來確保一個非常順暢的 Linux 桌面環境的體驗，以至於你很可能會忘了自己是在使用一個 live 版本。如果將 Porteus 安裝到 SSD 或者是在啟動的時候加載到內存裡的話，你就會得到一個如此天衣無縫地順暢的環境，以至於不會相信你的操作系統所佔用的空間只有不到半個 CD-ROM 的大小。

Porteus 的基礎鏡像文件相對來說比較小，因此被稱為是“微型”，但是根據你選擇的桌面環境版本，Porteus 有可能會需要 1 GB 之多的內存才可以運行。儘管其它微型 Linux 發行版傾向於通過精簡應用程序來節約空間和資源，Porteus 卻希望你像普通發行版一樣來使用它。忘掉你是在使用一個微型的壓縮根文件系統，盡情安裝所有你喜歡的應用程序吧。

安裝

可以在 離你最近的 Porteus 鏡像網站 上下載 Porteus，並且從 MATE、LXQT、LXDE、OpenBox、XFCE、Cinnamon 或者 KDE 裡選擇自己喜歡的桌面環境。如果沒有特殊偏好，MATE 或者是 KDE 桌面都是不錯的選擇，他們可以提供熟悉的桌面環境體驗，並且鏡像文件又不至於太大。


Porteus installer


你可以根據 官方的安裝指南 將 Porteus 安裝到一個 U盤 或者是內部硬盤裡。這兩種方式非常相似，都會使用一個不可變的壓縮根文件系統。這是一種穩定的、受限制的文件系統，會根據你的使用被修改。你所做的變更和安裝的應用程序在重啟的時候都會被加載到內存裡，從而還原你關機前的使用環境。

應用程序

應用程序在 Porteus 裡被稱為“模塊”，由 Slackware 軟件包統一管理器 （USM）提供。USM 的資源涵蓋五個不同的 Slackware 軟件倉庫，所以可供選擇的應用還是很豐富的。

結論

Porteus 可以提供完整的 Linux 使用體驗，卻只使用了正常 Linux 所需要空間的一小部分。這是一個配備了很多種可供選擇的桌面環境和很多應用程序的出色的便攜式 Linux 發行版。

Bodhi Linux


Bodhi Linux


Bodhi Linux 的 ISO 鏡像文件有 740 MB 大小，初看之下並不是很“微型”，不過一旦安裝完成之後，你就會驚訝於它是多麼微型了。Bodhi 在 512 MB 大小的內存上也可以順暢運行，並且它的桌面環境看起來就像是來自未來一樣。Bodhi 使用的是 Enlightenment 桌面，這是一個精心製作的優美的用戶界面，小巧而強悍。

不過 Bodhi 並不只是簡單地使用 Enlightenment，而是在此基礎上增色不少。Bodhi 在配置型應用程序和系統設置面板上都進行了界面處理，避免了 Enlightenment 有時顯得過於繁複的選項。Bodhi 替你做了一些很好的默認選擇，並且只顯示全部選項的一部分。如果你是一個 Enlightenment 狂熱分子，那麼 Bodhi 這樣的做法對你來說可能顯得不是很純粹，但是對於大多數用戶來說，Bodhi 這樣做可以讓人更加專注於 Enlightenment 桌面本身。

安裝

下載 Bodhi Linux ，通過 dd 或者 Etcher 寫入 U盤，然後重啟。

Bodhi 安裝器可以在 設置 頁面的 應用程序 菜單裡找到。安裝程序用的是 Ubiquity，所以整個過程和安裝 Ubuntu 是一樣的。如果你沒有安裝過 Ubuntu 也不必擔心，因為這是最好安裝的發行版之一了。


Bodhi installer


應用程序

Bodhi 是基於最新的 Ubuntu 長期維護髮布版的，所以可供使用的應用程序簡直數不勝數。只要是在 Ubuntu 上可以使用的應用，Bohdi 上就同樣可以找到。

結論

Bodhi Linux 相比一個標準的 Ubuntu 來說要小不少，但是相比其它微型 Ubuntu 環境來說又好一些（因為使用了 Enlightenment）。如果你在找一個比大多數發行版更輕量的 Linux 發行版，但是又不想使用 OverlayFS 或者是應用程序模塊的話，那麼 Bodhi 就是一個不錯的選擇了。

Puppy Linux


Puppy Linux


早在 Tiny Core、SliTaz、 AntiX 或者是 Porteus 誕生之前，就已經有 Puppy Linux 了。作為最早的微型 Linux 發行版之一，Puppy 已經歷經了十五年風霜，並且無論是對於老爺機還是新用戶來說始終都是一個可靠的、可啟動的操作系統。

為了保證正常運行，Puppy 會在第一次啟動之後引導用戶完成必要的設置步驟。整個過程涉及很多個窗口，但是一旦完成，你就會對一切功能瞭如指掌，然後再決定是否需要安裝。

Puppy 幾乎有 300 MB 大小，並且在我測試的 1 GB 內存的機器上並不能正常運行，所以它並不是一個特別微型的 Linux 發行版。儘管如此，它仍然是一個非常棒的 1 GB 以下的操作系統，並且在該類系統裡算是非常友好的一個。

安裝

下載 Puppy Linux ，然後通過 dd 或 Etcher 寫入 U 盤，或者是刻錄到 CD 或者 DVD 裡，然後重啟。


Puppy installer


Puppy 幾乎可以安裝在支持任何一種數據格式的載體上。你可以在頂部啟動欄裡找到 Puppy Installer 安裝程序，這個程序負責安裝 Puppy 以及 Puppy 的應用程序。

Puppy 安裝器會一步步引導你將系統安裝在你提供的任何一種媒介上。Puppy 可以從 U盤、光盤、硬盤，或者甚至是 SD 卡上啟動。我曾經在一臺沒有硬盤、光驅出了故障，並且也無法從 USB 啟動的計算機上成功運行了 Puppy。由於 Puppy 支持在任何載體上寫入你的配置選項，我甚至可以在一個擁有長期數據存儲的外部設備上使用它。

應用程序

Puppy 安裝器 這個應用同樣被用來在 Puppy 上安裝應用。由於 Puppy 是基於 Ubuntu 的，它的軟件倉庫幾乎不會缺少任何一個 Linux 軟件包，並且如果真的出現了這種情況的話，你也可以使用 Flatpak 。

結論

Puppy 是最早的微型 Linux。儘管它已經不是最微型的了，卻是目前最易用的一個。

附贈：Silverblue


SilverBlue, not tiny, but tiny-adjacent


微型 Linux 這個概念是隨著時間不斷變化的。很久以前，微型 Linux 發行版意味著需要下載到 CD-R 裡，從光驅啟動，然後將修改寫入外部媒介中。後來，你可以從 U 盤啟動它，並且有專門用來記錄永久修改的空間。現在的微型 Linux 不僅支持上面兩種方法，還可以被直接安裝在內部驅動或者文件夾裡。

大家都沒有想到 Linux 開創了容器的熱潮 —— 容器裡應用程序是在半虛擬化的環境中運行的一套獨立的 Linux 系統。曾經只是屬於喜歡優化硬盤空間或者重新利用老爺機的人們的小眾愛好，很快成為了那些想要開發容器但又不想在應用程序上添加太多負載的人的強烈需求。那些在極簡化的、不起眼的 Linux 發行版上所付出的辛苦，一夜之間以一種意想不到的方式得到了回報。

立足於根文件系統這個概念，Fedora 項目發起的 Silverblue 試驗旨在創造一個不可修改的操作系統。該操作系統主要通過容器的形式來更新系統以及安裝應用，系統本身永遠不會改變。

2.1 GB 的 Silverblue 可不是一個微型 Linux 發行版，但是從某種程度上來說，它是微型 Linux 和容器運動的產物。

安裝

下載 Silverblue ，然後通過 dd 琥或 Etcher 寫入 U 盤，或者是刻錄到 CD 或者 DVD 裡，然後重啟。

啟動到 Silverblue 之後，使用 Anaconda （標準的、友好的 Fedora 安裝器）將它安裝在一個內部硬盤裡。


Anaconda installer


應用程序

Silverblue 安裝應用的方式和傳統意義上不同：它是在基礎操作系統之上運行容器。具體來說，它使用 Flatpak 運行 GUI 應用程序，使用 Toolbox 運行命令。

由於 Flatpak 並非像傳統的 Fedora RPM 軟件包一樣常見，Silverblue 也提供了一種可以將 Fedora RPM 軟件包轉換成 Silverblue 形式的方法：軟件包分層。

結論

Silverblue 可能是一個用來嘗試前沿科技的有趣實驗，或者也可能是桌面操作系統的未來。它之所以被稱為微型，只是因為根文件系統的大小不會隨著系統升級或者安裝應用而改變。不過，透過 Silverblue 來看看對微型 Linux 的迷戀在帶領著 Linux 社區和行業往哪個方向走，也是一件挺有意思的事情。對了，走之前不要忘了向 11 MB 大小的微型 Linux 先驅們脫帽致敬。
