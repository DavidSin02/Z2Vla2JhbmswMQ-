Java之適配器模式（Adapter）

設計模式-適配器模式（Adapter）

1.設計意圖

適配器模式是將一個類的接口轉換成另一個類想要的接口，適配器讓原本兩個不兼容的類能夠兼容。

2. 演示案例

現實生活中我們到處都可以看到適配器

  • 1.如果你需要將SD卡中的數據傳輸到電腦上，為了將數據傳輸到電腦上，你可能需要一個與計算機USB接口兼容的讀卡器，以便將SD卡連接到計算機。此時的讀卡器就是一個適配器。
  • 2.如果你需要給手機充電，Type-c接口和普通充電接口無法對接，此時你需要一個充電轉換器或者電源適配器，以便能夠對接充電，這裡的轉換器就是一個適配器。
  • 3.同聲傳譯人員將英文話翻譯成中文話，翻譯人員就是一個“適配器”

簡而言之

適配器模式允許你在適配器中包裝一個原本不兼容的對象，使其與另外一個類兼容

維基百科：

在軟件工程中，適配器模式是一種軟件設計模式，它允許現有類的接口作用於另外一個接口。它通常用於在不修改源代碼的前提下使現有的類能夠與其他類一起工作。

3. 程序示例

我們就以SD卡通過讀卡器將數據傳輸到電腦中為案例來講解適配器模式。

首先我們創建一個存儲卡類和一個數據傳輸接口。存儲卡類中有一個現實內存大小的方法：memory();數據傳輸接口中定義兩個方法：read()和write();

SDCard.java

package com.ramostear.pattern.adapter;/** * @author ramostear * @create-time 2019/1/4 0004-19:41 * @modify by : * @info:[存儲卡類] * @since: */public class SDCard { public void memory(){ System.out.println("8GB"); }}

DataTransfer.java

package com.ramostear.pattern.adapter;/** * @author ramostear * @create-time 2019/1/4 0004-19:54 * @modify by : * @info:[數據傳輸接口] * @since: */public interface DataTransfer { void read(); void write();}

接下來我們將定義一個讀卡器類，並將SD卡 “插入” 讀卡器中，且實現數據讀寫接口

CardReader.java

package com.ramostear.pattern.adapter;/** * @author ramostear * @create-time 2019/1/4 0004-19:56 * @modify by : * @info:[讀卡器] * @since: */public class CardReader implements DataTransfer{ private SDCard sdCard; public CardReader() { this.sdCard = new SDCard(); } @Override public void read() { sdCard.memory(); System.out.println("read data from sdCard..."); } @Override public void write() { sdCard.memory(); System.out.println("write data to sdCard..."); }}

然後我們定義一個計算機類，預留“USB”的接口，並設置數據讀寫入口

Computer.java

package com.ramostear.pattern.adapter;/** * @author ramostear * @create-time 2019/1/4 0004-20:01 * @modify by : * @info:[計算機類] * @since: */public class Computer { private DataTransfer dataTransfer; public Computer(){} public Computer(DataTransfer dataTransfer){ this.dataTransfer = dataTransfer; } public DataTransfer getDataTransfer() { return dataTransfer; } public void setDataTransfer(DataTransfer dataTransfer) { this.dataTransfer = dataTransfer; } public void read(){ dataTransfer.read(); } public void write(){ dataTransfer.write(); }}

最後，讓我們將裝入SD卡的讀卡器插入到計算機的USB口中，啟動計算機並對SD卡中的數據進行讀寫

App.java

package com.ramostear.pattern.adapter;/** * @author ramostear * @create-time 2019/1/4 0004-20:04 * @modify by : * @info:[測試類] * @since: */public class App { public static void main(String[] args){ /** * 1. 將SD卡插入讀卡器 * 2. 將讀卡器插入電腦USB接口 * 3. 電腦開機 * 4. 從SD卡中讀入數據到電腦，或者將輸入從電腦中寫入SD卡 */ Computer computer = new Computer(new CardReader()); computer.read(); computer.write(); }}

至此，適配器模式設計實現完成。

4. 適用場景

當滿足(不限於)以下應用場景時：

  • 你想複用現有的類，但它所提供的接口與你期望的接口不匹配時
  • 你希望創建一個可重用的類，並讓該類與其他不可預見的類（不一定具有兼容接口的類）協作時
  • 大多數使用第三方庫的應用程序使用適配器作為應用程序和第三方庫之間的中間層，以將應用程序與庫分離。如果必須使用另一個庫，則只需要新庫的適配器，而不必更改應用程序代碼
  • 你需要使用幾個現有的子類，但通過對每個子類進行子類化來調整它們的接口是不切實際的

5. 使用建議

5.1 類適配器

由於類適配器是適配者類的子類，因此可以在適配器類中調整適配者類中的方法，使得適配器的靈活性更強。但是如果在Java這樣的單繼承語言中，一次最多隻能適配一個適配者類，且目標抽象類只能為接口，不能為類。

5.2 對象適配器

把多個不同的適配者適配到同一個目標類，同一個適配器可以把適配者和它的子類都適配到一個目標接口。
