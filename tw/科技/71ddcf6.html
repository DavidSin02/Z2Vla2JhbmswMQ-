兩條消息的時間戳（timestamp）是一樣的，這是因為 datetime.now 只執行了一次，也就是它只在函數定義的時候執行了一次。參數的默認值，會在每個模塊加載進來的時候求出，而很多模塊都是在程序啟動時加載的。包含這段代碼的模塊一旦加載進來，參數的默認值就固定不變了，程序不會再次執行 datetime.now。

在 Python 中若想正確實現動態默認值，習慣上是把默認值設為 None，並在文檔字符串（docstring）裡面把None所對應的實際行為描述出來。編寫函數代碼時，如果發現該參數的值是 None，那就將其設為實際的默認值。

現在，兩條消息的時間戳就不同了。

我們本以為 foo 和 bar 會表示兩份不同的字典，每個字典裡都有一對鍵和值，但實際上，修改了其中一個之後，另外一個似乎也會受到影響。這種錯誤的根本原因是：foo 和 bar 其實都等同於寫在 default 參數默認值中的那個字典，它們都表示的是同一個字典對象。

要點

  • 參數的默認值，只會在程序加載模塊並讀到本函數的定義時評估一次。對於 { } 或 [ ] 等動態的值，這可能會導致奇怪的行為。
  • 對於以動態值作為實際默認值的關鍵字參數來說，應該把形式上的默認值寫為 None，並在函數的文檔字符串裡面描述該默認值所對應的實際行為。
