CC++語言19｜靜態成員（對象共享數據成員及其成員函數）

2020-11-14
| 科技

類中可以包含靜態數據成員和靜態函數成員，統稱為靜態成員。默認情況下，靜態成員的作用域是類範圍內且是外部鏈接的。靜態成員在內存中只有一個副本，不是類對象的一部分，而是獨立的對象。因此，靜態數據成員的聲明不是定義，聲明在類作用域內完成，而定義在文件作用域內完成。

不管這個類有多少個對象，靜態數據成員都只有一份拷貝。靜態數據成員一般用來保存整個類所有對象共享的信息。

1 靜態成員變量

假設類SavingAccount專門用於存放存款帳戶，它包括存戶的姓名、地址、存款額、利率等成員變量：

class SavingAccount{char m_name[20]; //存戶姓名char m_addr[60]; //存戶地址double m_total; //存款額double m_rate; //利率…};

這家銀行採用浮動利率，每個帳戶的利息根據當天的掛牌利率計算。這時候如果m_rate就不適合成為每個帳號一個資料，否則每天一開市，光把所有帳戶內容調出來，修改m_rate的值，就花掉不少時間。m_rate應該獨立於各對象之外，成為獨一無二的資料。怎樣實現呢？就是聲明為各對象共享的數據，使用關鍵字static。

對於靜態成員數據：

1）在編譯階段就分配空間，對象還沒有創建；

2）必須在類中聲明，在類外初始化（或定義）。

由於靜態數據成員要在類聲明與定義時實際分配空間，因此不能在類聲明中進行初始化。靜態數據成員初始化在類的外部進行，且與一般數據成員初始化不同，它的格式如下：

<數據類型><類名>::<靜態數據成員名>=<值>

3）歸同一個類的所有對象所有，共享同一個靜態變量。在為對象分配的空間中不包含靜態成員所佔空間。

4）有權限限制：private靜態變量和 public靜態變量。

private靜態變量；在類（print類）外不能訪問和操作。

類外訪問：print::a;//erro 類外操作：print::a = 6;//erro

public靜態變量；在類（print類）外可以訪問和操作。

類外訪問：print::a;//yes 類外操作：print::a = 6;//yes

5）const 靜態成員變量

const static int a = 8;

定義靜態變量 const 靜態成員變量時，最好在類的內部初始化。

static 成員變量不屬於對象的一部份，而是類別的一部份，所以程序可以在還沒有誕生任何對象的時候就處理此種成員變量。但首先你必須初始化它。不要把static 成員變量的初始化動作安排在類別的構造式中，因為構造式可能一再被調用，而變量的初值卻只應該設定一次。也不要把初始化動作安排在頭文件中，因為它可能會被包含許多地方，因此也就可能被執行許多次。你應該在實作檔中且類別以外的任何位置設定其初值。例如在main 之中，或全域函數中，或任何函數之外：

double SavingAccount::m_rate = 0.0075; // 設立static 成員變量的初值void main() { ... }

為靜態成員分配空間稱為靜態成員的定義。

但靜態數據成員屬於類，因此定義對象時並不為靜態成員分配空間。

靜態成員的定義一般出現在類的實現文件中。如在SavingAccount類的實現文件中，必須要如下的定義：

double SavingAccount::rate = 0.05;

該定義為rate分配了空間，並給它賦了一個初值0.05。

如果沒有這個定義，連接器會報告一個錯誤。

可以通過作用域操作符從類直接調用。如： SavingAccount::rate

但從每個對象的角度來看，它似乎又是對象的一部分，因此又可以從對象引用它。如有個SavingAccount類的對象obj，則可以用：obj.rate

由於是整個類共享的，因此不管用哪種調用方式，得到的值都是相同的。

2 靜態變量的調用

用類名加域作用符: print::a;

用對象名加取址符： class p1; p1.a;

3 靜態成員函數

與靜態變量一樣，在沒有創建對象前即可通過類名調用。

1）靜態成員函數只能訪問 "靜態成員變量"，不能訪問普通成員變量。

2）靜態成員函數的使用規則和靜態成員變量一樣。

3）靜態成員函數也有使用權限。類外無法訪問私有靜態函數。

4）普通成員函數可訪問"靜態成員變量"，也可訪問"非靜態成員變量"

class BufferedOutput{public: short BytesWritten() { return bytecount;} static void ResetCount() { bytecout = 0;} static long bytecount;};long BufferedOutput::bytecount = 8;

定義靜態成員函數的主要目的是訪問靜態的數據成員。

如在SavingAccount類中，當利率發生變化時，必須修改這個靜態數據成員的值。為此可以設置一個靜態的成員函數：

static void SetRate(double newRate) {rate = newRate;}

靜態成員函數可定義為內嵌的，也可在類外定義。在類外定義時，不用static。

靜態成員函數的訪問：可以通過類作用域限定符或通過對象訪問

類名::靜態成員函數名();

4 static關鍵字的作用

1）函數體內static變量的作用範圍為該函數體，不同於auto變量，該變量的內存只被分配一次，因此其值在下次調用時仍維持上次的值。

2）在模塊內的static全局變量可以被模塊內所用函數訪問，但不能被模塊外其他函數訪問。

3）在模塊內的static函數只可以被這一模塊內的其他函數調用。這個函數的使用範圍被限制在聲明它的模塊內。

4）在類中的static成員變量屬於整個類所擁有，對類的所有對象只有一份拷貝。

5）在類中的static成員函數屬於整個類所擁有，這個函數不接收this指針，因而只能訪問類的static成員變量。

6) 對於靜態（static）對象，當對象誕生時其構造式被執行；當程序將結束時（此對象因而將遭致毀滅）其析構式才被執行，但比全域對象的析構式早一步執行。

5 常見問題

1) 常量數據成員和靜態常量數據成員有什麼區別？如何初始化常量數據成員？如何初始化靜態常量數據成員？

常量的數據成員指得是那些在對象生成時給定了初值，在整個對象的生命週期中，該數據成員的值是不能變的。常量數據成員的值必須在構造函數的初始化列表中進行初始化。靜態的常量數據成員是整個類所有對象共享的一個常量。對整個類而言，不管定義了多少個對象，該成員永遠只有一份拷貝。靜態常量數據成員的值是在定義類時給定。

2) 靜態成員函數不能操作非靜態的數據成員，那麼非靜態的成員函數能否操作靜態的數據成員？為什麼？

因為靜態成員函數沒有隱含的this指針，所以不能夠操作非靜態的數據成員。靜態的數據成員是所有對象共享的數據成員，在邏輯上是屬於每一個對象的，所以非靜態的成員函數可以操作靜態的數據成員。
