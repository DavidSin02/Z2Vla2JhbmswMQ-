一個很基礎的問題：new一個子類，會另外new一個父類嗎？

2020-10-29
| 科技

前言

前幾天有一個小夥伴給我留言，提出了2個疑問：

是關於這篇文章的：Java中，實例化子類會實例化父類麼

  • 子類實例為什麼會有父類的成員變量？
  • 它的初始化是通過哪種方式訪問了父類對象中的表？（我理解這句話的意思是：子類如何訪問到從父類中繼承的變量或方法）

實話實說，看到這個小夥伴的問題時…我一下子懵住了，很久沒有看JVM的內容，一時有些語塞…今天特地抽了些時間重新整理了一下。

因為自己在JVM這一塊並沒有特別深入的瞭解，所以這裡權當一起探討交流~歡迎一起討論。

正文

問題一：子類實例為什麼會有父類的成員變量？

這個問題，怪我並沒有get到點…為什麼會有？因為繼承吶，繼承在Java規範中的語義就是能夠擁有父類中protect及以上權限的變量和方法。規範要求，不得不從~

我猜這個小夥伴可能是問另一個意思：子類通過什麼樣的方式擁有了父類的成員變量。

如果是這個問題，那麼其實和問題二是差不多的。所以讓我們從問題二中找尋答案…

問題二： 它的初始化是通過哪種方式訪問了父類對象中的表？

寫了倆個很簡單的子/父類：

public class Son extends Father { public String nick;}public class Father { public String name;}

然後javap -v看一下Son.class：

注意看常量池（Constant pool）裡的#3。#3被映射到#14上，而#14怎是我們的Father的CONSTANT_Class_info。通過此就可以定位到我們Father，因此也就可以找到到其中的屬性/方法。

換一種理解，子類的實例中已經包含了父類屬性，為什麼？因為此時至始至終都是一塊內存空間，我們可以用一小段代碼，驗證這個問題：

public class Son extends Father { public String nick; public Son(){ System.out.println(super.equals(this)); }}

run起來，你會發現是true。也就是說super的引用和this的引用是同一個。既然是同一個，那就不可能是兩個內存空間。
