循環鏈表操作分析

1. 添加

a) 空鏈表添加第一個節點

b) 正常添加節點

i. 利用輔助節點，找到鏈表的最後一個節點

ii. 將新節點添加到最後

2. 刪除

a) 刪除首節點

i. 如果刪除首節點後 size為0，說明鏈表為空。

ii. 正常刪除首節點

注意：需要先找到當前鏈表的最後一個節點，改變尾結點的next指向新的首節點。

b) 刪除正常節點

約瑟夫環

據說著名猶太歷史學家 Josephus有過以下的故事：在羅馬人佔領喬塔帕特後，39 個猶太人與Josephus及他的朋友躲到一個洞中，39個猶太人決定寧願死也不要被敵人抓到，於是決定了一個自殺方式，41個人排成一個圓圈，由第1個人開始報數，每報數到第3人該人就必須自殺，然後再由下一個重新報數，直到所有人都自殺身亡為止。然而Josephus 和他的朋友並不想遵從。首先從一個人開始，越過k-2個人（因為第一個人已經被越過），並殺掉第k個人。接著，再越過k-1個人，並殺掉第k個人。這個過程沿著圓圈一直進行，直到最終只剩下一個人留下，這個人就可以繼續活著。問題是，給定了和，一開始要站在什麼地方才能避免被處決？Josephus要他的朋友先假裝遵從，他將朋友與自己安排在第16個與第31個位置，於是逃過了這場死亡遊戲。

解決約瑟夫環

修改測試代碼：

/** * All rights Reserved, Designed By https://www.tulingxueyuan.com/* @Title: CircleLinkedListDemo.java* @Package com.tuling.linked* @Description: * @author 北京圖靈學院* @date 2019年11月19日* @version V1.0 */package com.tuling.linked;import com.tuling.linked.CircleNodeManager.Node;/** * @ClassName: CircleLinkedListDemo * @Description: * @author 小白 * @date 2019年11月19日 * */public class CircleLinkedListDemo {	/**	 * @Title: main 	 * @Description: 	 * @param args 	 * @return void 	 * @throws	 */	public static void main(String[] args) {		CircleNodeManager cnm = new CircleNodeManager();				int nums = 41;//總人數		int space = 3;//間隔				//--------------循環依次上循環鏈表中插入數據，構成環-----------------		for(int i = 1; i <= nums; i++) {			cnm.add(i);		}						//--------------下面模擬自殺-----------------		//需要一個輔助節點，當做遊標，不停移動。		Node currNode = cnm.getFirst();//從第一個人開始數數。				/**		 * 如果：currNode.getNext() == currNode 說明鏈表中只有一個元素		 */		while(currNode.getNext() != currNode) {//還有人沒死，繼續數數			//利用循環，根據 space 的值，不停的移動遊標，也就是 currNode			for(int j = 1; j < space; j++) {				currNode = currNode.getNext();			}						/**			 * 循環結束，currNode 指向要自殺的節點			 */			System.out.print(cnm.del(currNode.getData()).getData() + ", ");			//刪除完成，遊標一定要記得後移			currNode = currNode.getNext();		}		System.out.println();		System.out.println("自殺完成！");		cnm.show();	}}
