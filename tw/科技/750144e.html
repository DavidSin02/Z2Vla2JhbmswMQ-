LTE下VOLTE和CSFB介紹

2020-10-29
| 科技

LTE下有兩種通話方式：

1.VoLTE(Voice over Long-Term Evolution)即在LTE下通過數據域進行通話；

2.CSFB（circuit Switched Fallback）即從LTE回落到3G或者2G下進行通話，默認聯通是回落到3G下通話，移動是回落到2G下通話

下面分別介紹這兩種方式下的通話機制：

volte流程介紹：

VOLTE是基於SIP協議的語音通話，所有與IMS交互的信令全部為SIP信令，在理解VOLTE信令方面必須對SIP信令進行了解，EPC只是做為業務承載體

SIP 提供了發現機制，如果用戶要發起和另一個用戶的會話，SIP 必須發現可到達目的用戶的當前主機，註冊將記錄地址 URI 和一個或者多個聯繫地址相關聯，這樣才能進行呼叫等業務。

嚴格意義上說，SUBSCRIBE和NOTIFY過程不屬於註冊過程，但由於該過程在註冊完成後緊跟著出現，所以本文將該過程放在註冊流程中進行說明。用戶的註銷過程與註冊過程相似。用戶的註銷過程與註冊過程相似，主要就是註銷請求中,expire值為0，所以本文中不再進行單獨說明，註銷過程無SUBSCRIBE信令，是因為UE註冊時已有SUBSCRIBE。

註冊流程介紹：

IMS註冊過程

信令說明如下： 

1. UE進行Attach，建立QCI=9的默認承載，並使用IMS APN建立PDN連接； 

2. 建立立QCI=5的默認承載，用於傳送SIP信令；

3. UE通過QCI=5的默認承載向IMS發起註冊請求； 

4. P-CSCF通過HSS獲知用戶信息不在數據庫中，便向終端代理回送401

Unauthorized 質詢信息，其中包含安全認證所需的令牌； 

5. 終端將用戶標識和密碼根據安全認證令牌加密後，再次用REGISTER消息報Unauthorized 質詢信息，其中包含安全認證所需的令牌； 

6. P-CSCF將REGISTER 消息中的用戶信息解密，驗證其合法後，IMS核心網將該用戶信息登記到數據庫中，並向終端返回成功響應消息200 OK；

7. 用戶向IMS訂閱註冊事件包；

8. 服務器應答訂閱成功；

9.IMS 服務器發送notify消息，由於訂閱的用戶已經註冊，所有IMS服務器迴應Notify消息中狀態為active，同時攜帶XML信息

通話流程各個階段介紹

1、用戶發起呼叫，CM 通知IMS ，IMS 創建併發送INVITE 消息

Reliable provisional responses (100 Rel)

SDP for audio codec support

2、當LTE棧接收到該消息時，它啟動一個服務請求過程。 RRC層執行RACH過程以進入RRC連接狀態併發送該消息。

3、LTE棧發送INVITE 消息

a、INVITE到達P-CSCF1。 它通過I-CSCF路由到服務UE的S-CSCF。 S-CSCF確定目的地可通 過IMS（不是CS /固定電話用戶）到達並轉發請求到相應的I-CSCF。 該消息最終被傳遞給服務UE2的PGW通過其P-CSCF（P-CSCF2）。

b、UE2在E-UTRAN上尋呼，並執行RACH過程以響應LTE尋呼

c、INVITE消息被髮送到UE2並被路由到IMS客戶端2。

4、P-CSCF1向IMS客戶端1發送100：Trying，以指示INVITE已被轉發到目的地。

5、IMS client2 響應180 Ringing

180（Ringing）消息在SDP中僅包含一個編解碼器選擇

180（Ringing）消息通過INVITE消息之後的反向路徑到達IMS客戶端1。

6、當P-CSCF1和P-CSCF2收到180（振鈴）消息時，它們觸發專用承載的網絡啟動推送。 P-CSCF通過與PCRF建立Diameter會話觸發網絡發起的專用EPS承載創建。 這是通過發送一個包含以下內容的AARequest完成的：

由IMS PDN分配給UE的IP地址

媒體組件描述，它在180（Ringing）消息中指定來自SDP的編解碼器和業務流模板（TFT）

7、響應於步驟6中的AA-Requsest消息，PCRF使用從P-CSCF獲得的IP地址來識別IP連通性接入網絡（IP-CAN）會話。 它使用在IP-CAN會話建立期間從HSS /用戶簡檔庫（SPR）下載的QoS策略信息和從P-CSCF接收的媒體描述來確定用於承載語音業務的專用EPS承載的QoS和過濾器信息。 它通過發送帶有QoS信息和FlowDescription字段的（直徑）RAR命令來執行IP-CAN會話修改

QoS-Information specifies QCI, ARP, GBR/MBR parameters.

For GPRS Tunneling Protocol (GTP), this command reaches PGW.

8、在GTP的情況下，PGW通過基於接收到的RAR命令的Flow-Description字段安裝分組過濾器來創建S5 EPS承載。 它向SGW發送創建承載請求。 該消息包含EPS承載QoS，S5 / S8-Tunnel-ID，TFT和LBI（專用承載鏈接到的默認承載的EPS承載ID）。

9、SGW向MME發送包含EPS承載QoS，S1-Tunnel-ID，TFT和LBI的創建承載請求。

10、MME為專用承載選擇EPS bearer ID。 它創建NAS消息激活專用EPS承載上下文請求消息，並在稱為承載建立請求的MME消息中將其發送到eNB。 重要的領域是：

承載建立請求，其中包含EPS承載ID，EPS QoS，S1-Tunnel-ID和NAS消息

激活專用EPS承載上下文請求消息，指定EPS承載ID，TFT，QoS參數和LBI的NAS消息。

11、eNB將EPS承載QoS映射到無線承載QoS。 它發送RRC連接重配置消息，該消息包含鏈接到EPS承載ID的數據無線承載（DRB），該承載ID能夠在無線電級別保證QoS。 RRC連接重配置也包含來自MME的NAS消息。

12、當UE使用RRCConnection重配置完成確認無線承載激活時，eNB向MME發送承載建立響應，指示可以分配EPS承載QoS。 該消息包含EPS承載ID和S1-Tunnel-ID。

如果eNB沒有資源，它會迴應失敗。

13、UE中的NAS層以UL信息傳輸消息中的NAS消息激活專用EPS承載上下文接受進行響應。 而且，當網絡發起的專用承載被創建時，LTE棧：

使用NAS消息/ RRC消息的內容來創建到無線承載的業務流之間的映射。

預計安裝的UL TFT在SDP應答中具有目的地IP地址/端口，並且在SDP報價中具有廣告的源IP /端口的DL TFT。

14、eNB在UL NAS傳輸中將NAS消息轉發給MME。

15、The MME sends a Create Bearer Response message to the SGW

16、The SGW sends a Create Bearer Response message to the PGW.

17、The PGW sends an RAR response message to the PCRF

18、收到RAR響應消息後，PCRF向P-CSCF發送AA-Answer命令，指示專用承載建立成功。

19、UE1中的IMS客戶端發送PRACK SIP消息。 該PRACK消息通過與步驟3中相同的一組節點到達UE2上的IMS客戶端。

20、UE2中的IMS客戶端在接收PRACK時以200（OK）響應。 此響應確認PRACK並接受SDP-Offer。

UE2中的IMS客戶端在接收到確認180（振鈴）的PRACK之後提醒用戶。

21、當目標用戶應答呼叫時，IMS客戶端2向源IMS客戶端1發送200（OK）消息。 該消息用作初始INVITE消息的200（OK）。

22、IMS客戶端1用ACK確認200（OK），媒體流已準備好建立

CSFB流程介紹

主叫CSFB流程說明

1） UE發起CS Fallback語音業務請求。UE語音撥打時，會發一條extended service request，消息裡會攜帶CSFB信息。其中service-type信元指示業務類型為始發CSFB語音業務，同時攜帶該UE在聯合附著過程中CS域給它分配的TMSI。之後會在基站的輔助下回落至2G。

2） MME發送Initial Context Setup Request消息給eNodeB，包含CS Fallback Indicator。該消息指示eNodeB，UE因CS Fallback業務需要回落到UTRAN/GERAN。

3） eNodeB要求UE開始系統的小區測量，並獲得UE上報的測量報告，確定重定向的目標系統小區。然後向UE發送目標系統具體的無線配置信息，並釋放連接。LTE網絡通過RIM流程（無線消息管理流程）提前獲取2G目標小區廣播信息，將2G網絡的廣播信息一併填充至RRC Release消息中下發，省去終端讀取2G廣播信息的時間（約省1.83秒）

4） UE接入目標系統小區，發起CS域的業務請求CM Service Request。如果CM業務請求消息中攜帶“CSMO”標誌，則MSC Server記錄本次呼叫是移動始端CSFB呼叫。

5） 如果目標系統小區歸屬的MSC Server與UE附著EPS網絡時登記的MSC Server不同，則該MSC Server收到UE的業務請求時，由於沒有該UE的信息，可以採取隱式位置更新流程，接受用戶請求。如果MSC Server不支持隱式位置更新，且MSC Server沒有用戶數據，則拒絕該用戶的業務請求。如果MSC Server拒絕用戶的業務請求會導致UE發起一個CS域位置更新流程。終端發起位置更新請求，且位置更新請求消息中的Additional update parameters信元中攜帶CSMO標識，同時該標識有效，則MSC Server會記錄本次呼叫是CSFB呼叫。（CS fallback緊急呼叫流程中，CM_SERVICE_REQUEST消息前無需位置更新。）

6）完成位置更新後UE再次在 CS域建立語音呼叫流程。

7） 通話結束後，MSC Server向主叫回落到的BSC發送的拆線消息CLEAR_COMMAND消息中攜帶CSFB Indication信元，指示BSC拆除空口連接並指示UE回到LTE網絡。 或者MSC Server向主叫回落到的RNC發送IU_RELEASE_COMMAND消息，攜帶End Of CSFB信元，指示RNC拆除空口連接並指示UE回到LTE網絡。

8） MSC收到BSC的CLEAR_COMPLETE消息/RNC的IU_RELEASE_COMPLETE消息表示呼叫結束，A口拆鏈完成。接入側在指示終端重選網絡時只針對CSFB用戶通話前攜帶LTE頻點，實現CSFB終端快速返回LTE，快速回落過程也稱為FastReturn（用戶不可及時間可縮短為1-2秒。）。

2.6.2 CSFB被叫流程

MSC Server收到對UE的被叫語音請求，通過存在的SGs關聯和MME信息，向該MME發起尋呼請求。MME通過eNodeB在空口尋呼該UE，並指示UE回落到目標GERAN/UTRAN網絡。UE接入到目標網絡後，在電路域繼續進行語音呼叫。

被叫CSFB流程說明

1） GMSC Server向被叫用戶歸屬HLR發送取路由信息請求。

2） HLR收到該SRI消息後，向被叫用戶當前附著到的old MSC Server獲取漫遊號碼。

3） old MSC Server為該次呼叫分配漫遊號碼MSRN1，並返回給HLR。

4） HLR將該漫遊號碼發送給GMSC。

5） GMSC收到該漫遊號碼後，進行號碼分析，根據分析結果將呼叫路由到old MSC Server。

6） MSC Server收到IAM入局（例如中繼ISUP入局）消息後，根據存在的SGs關聯和MME信息，發送SGsAP-PAGING-REQUEST(攜帶IMSI，TMSI，Service indicator ，CLI，LAC)消息給MME。

7） MME發送Paging消息給eNodeB。eNodeB發起空口的Paging流程。

8） UE建立連接併發送Extended Service Request消息給MME（消息中攜帶“CSMT”移動終端標識）。

9） MME發送SGsAP-SERVICE-REQUEST消息給MSC Server。MSC Server收到此消息，不再向MME重發尋呼請求消息。為避免呼叫接續過程中，主叫等待時間過長，MSC Server收到包含空閒態指

示的SGs Service Request消息，先通知主叫，呼叫正在接續過程中。

10） MME發送Initial UE Context Setup消息給eNodeB，包含CS Fallback Indicator。該消息指示eNodeB，UE因CSFB業務需要回落到UTRAN/GERAN。

11） UE回落到CS域之後，UE檢測到當前的位置區信息和存儲的位置區不同，將發起位置更新。MSC Server收到UE發送的LOCATION_UPDATE_REQUEST消息。這種情況下，UE不需要回Paging Response給MSC Server，UE直接發送SETUP消息建立呼叫。如果位置更新消息中攜帶“CSMT”標誌，則MSC Server記錄本次呼叫是CSFB呼叫。

12） 伴隨著空口、A/Iu-CS接口連接的建立，UE回Paging Response消息給MSC Server，該消息中攜帶CSMT標識，即使BSC/RNC沒有向該UE發起過尋呼請求，也需要能處理UE的尋呼響應。如果尋呼響應消息中的位置區信息和VLR中保存的不一致，則VLR在業務接入成功之後將SGs關聯置為非關聯。

13） 建立CS呼叫。

14）通話結束後，指示BSC/RNC拆除空口連接並指示UE回到LTE網絡。

15） MSC收到BSC的CLEAR_COMPLETE消息/RNC的IU_RELEASE_COMPLETE消息表示呼叫結束。接入側在指示終端重選網絡時只針對CSFB用戶攜帶LTE頻點，實現CSFB終端快速返回E-UTRAN。FastReturn方案也需要網絡的支持，如果網絡不支持，則通過網絡優先級的方式返回LTE（一般為最高優先級）

VoLTE和CSFB的有缺點：

csfb需要2g3g網絡和4g網絡重疊覆蓋，這樣駐留4g的手機在撥語音呼叫zhidao時候可以回滾到2g3g
大多數手機都支持csfb，lte網絡初始階段都要csfb方式支持語音，但是呼叫時延長
volte是在lte網絡覆蓋達到絕大部分區域時候部回署，優點是使用4g一張單獨的網絡就支持語音和數據業務，不需要2g3g網絡的幫助，呼叫時延短。但是需要手機端支持。
