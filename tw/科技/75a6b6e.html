手寫“棧”數據結構

2020-10-29
| 科技

棧作為一個常用的數據結構（如果有不太理解什麼是數據結構的請點這裡），其使用範圍還是非常廣泛的。今天我們就一起了解一下“棧”。

簡介

棧是線性結構當中一種特殊的線性表，它會限制元素的插入和刪除，並且是一個有序的列表。棧存取數據都是在線性表的一端進行的，存取數據的一端稱為“棧頂”，另一端則是“棧底”。這種存取方式就造就了棧具有先入後出的特點，也就是第一個存儲的數據會在最後一個取出來。舉個例子來說棧就像是一個彈夾，棧中的數據就好比是一枚枚的子彈。開槍射出的第一枚子彈都是最後一個壓入彈夾的。這樣說大家是不是大概都能明白棧是一種什麼樣的結構？如果還是不太清楚的話請看下圖。

通過上面的描述和圖片相信大家對棧這個數據結構有了一個大致的概念。學習一個新東西肯定是為了使用，那麼它有什麼用途呢？別急別急，拿好小板凳讓們來了解一下。

應用場景

1、子程序的調用：在跳往子程序前，會先將下個指令的地址存到棧中，直到子程序執行完再將地址取出，以回到原來的程序中。

2、處理遞歸調用：和子程序調用類似，除了存儲下一個指令的地址外，也將參數、區域變量等數據存入棧中。

3、表達式的轉換【中綴表達式轉後綴表達式】。

4、二叉樹的遍歷。

5、圖形的深度優先（depth-first）搜索法。

以上就是棧比較常見的一些應用場景。在之後的文章中我也會為大家仔細介紹這些場景下是如何運用棧的。

下面重頭戲來嘍，讓我們一起來動手實現一個“棧”！

代碼實現

通過數組實現棧：

public class ArrayStack {

// 棧的最大容量

private int maxSize;

// 存放數據（使用數組）

private int[] stack;

// 棧頂指針

private int top = -1;

/**

* 構造方法

*

* @param maxSize 棧的最大容量

*/

public ArrayStack(int maxSize) {

this.maxSize = maxSize;

// 初始化數組

this.stack = new int[this.maxSize];

}

}

判斷棧空的方法：

public boolean isEmpty() {

// 當棧頂指針為-1時代表當前棧還沒有存入元素

return this.top == -1;

}

判斷棧滿的方法：

public boolean isFull() {

// 當棧頂的指針等於棧最大容量減一時，代表棧滿了（指針指向的是stack的索引，是從0開始的）。

return this.top == this.maxSize - 1;

}

壓棧（入棧）的方法：

public void push(int value) {

// 驗證是否還可以壓入數據

if (this.isFull()) {

System.out.println("棧滿了！");

return;

}

// 先移動棧頂指針

this.top++;

// 保存數據

this.stack[this.top] = value;

}

彈棧（出棧）的方法：

public int pop() {

// 驗證是否還可以取出數據

if (this.isEmpty()) {

throw new RuntimeException("棧空了！");

}

// 取出棧頂數據

int value = this.stack[this.top];

// 將棧頂指針向下移動

this.top--;

return value;

}
