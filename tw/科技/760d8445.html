你的前端知識框架，該如何搭建？

2020-11-14
| 科技

前端工程師是研發體系中的重要崗位之一。可是，與此相對的是，系統性的教學方案几乎找不到。大部分前端工程師的知識，其實都是來自於實踐和工作中零散的學習。

前端技術的更新如此迅速，在這樣的大環境下，前端工程師保持自學能力就顯得尤其重要了。因此，“學什麼”“怎麼學”就是我們要迫切解決的問題。

所以，我想帶你梳理一套前端知識框架，幫你把知識點重新做個遍歷，讓你在學習的路上走的更輕鬆一些。這份知識框架，也是我在《重學前端》專欄裡幫助大家重構前端架構的核心內容。

0 基礎入門前端

對於 0 基礎入門的同學，我建議你先去讀幾本經典的前端教材，打好起知識基礎的地基。我們可以先從《JavaScript 高級程序設計》、《精通 CSS》看起。然後去閱讀一些參考性質的網站，很推薦：MDN。


《JavaScript 高級程序設計》

簡介：《JavaScript 高級程序設計》被稱為“程序員的紅寶書”。幾乎涵蓋了 JavaScript 的大部分內容，有大量的例子，理解起來也並不太費解。


《精通 CSS》

簡介：這本書將有用的 CSS 技術彙總在一起，還總結了 CSS 設計中的極具實踐。CSS 看似簡單，但真正精通 CSS 絕非易事。

如果你已經有 1 年以上的工作經驗了，我們可以一起建立起前端的知識框架，再去把知識點做個遍歷，這其中有原理和背景的部分，我去看知識的原理和背景就夠了。如果沒有的話，我們就去看整理和記憶這部分知識的方法，這樣，即使你遇見無法一下子記住的知識，也可以很容易地查閱參考手冊和標準來解決。


前端必備知識框架

我們先來講講什麼叫做知識架構？我們可以把它理解為知識的“目錄”或者索引，它能夠幫助我們把零散的知識組織起來，也能夠幫助我們發現一些知識上的盲區。

我們把前端知識在總體上分成基礎部分和實踐部分，基礎部分包含了 JavaScript 語言、CSS 和 HTML 以及瀏覽器的實現原理和 API，這三個模塊涵蓋了一個前端工程師所需要掌握的全部知識，學完這三個部分，你再結合基本的編程能力，就可以應對基本的前端開發工作了。

JavaScript 知識架構圖


在 JavaScript 的模塊中，首先我們可以把語言按照文法、語義和運行時來拆分，這符合編程語言的一般規律：用一定的詞法和語法，表達一定語義，從而操作運行時。

接下來，按照程序的一般規律，可以把運行時分為數據結構和算法部分——數據結構包含類型和實例（內置對象），算法是執行過程。

類型部分中，對象比其它所有類型加起來都複雜，所以我們會用較長的時間來學習對象，包括它的一些歷史和設計思路。

執行過程應按照從大結構到小結構的角度來學習，從最頂層的事件循環和微任務，到函數、再到語句級的執行。

實例部分也很關鍵，但對 JavaScript 來說類似基礎庫，JavaScipt 的內置對象多達 150 以上，在MDN會有細緻全面的講解。

文法中的語法和語義基本是一一對應關係，在 JavaScript 標準中有一份語法定義表，建議隨時拿出來看一看。


HTML 和 CSS 知識架構圖


在 HTML 的部分，我們按照功能和語言來劃分它的知識，HTML 的功能主要由標籤來承擔，所以首先要把標籤做一些分類。

我們按照承擔的不同功能，把標籤分成下面幾種。

  1. 文檔元信息：通常是出現在 head 標籤中的元素，包含了描述文檔自身的一些信息；
  2. 語義相關：擴展了純文本，表達文章結構、不同語言要素的標籤；
  3. 鏈接：提供到文檔內和文檔外的鏈接；
  4. 替換型標籤：引入聲音、圖片、視頻等外部元素替換自身的一類標籤；
  5. 表單：用於填寫和提交信息的一類標籤；
  6. 表格：表頭、表尾、單元格等表格的結構。

除了標籤之外，你還應該把 HTML 當作一門語言來了解下。但標記語言跟編程語言不太一樣，比較基礎的 HTML 的語法和幾個重要的語言機制：實體、命名空間，你一定要掌握。

HTML 還有一個補充標準也很關鍵：ARIA，它是 HTML 的擴展，在可訪問性領域，它有至關重要的作用。

CSS 部分，按照慣例，我們按照語言和功能劃分。

  • 語言部分，@rule、選擇器、單位是三個要關注的部分。
  • 功能部分，我們可以重點關注佈局、繪製和交互類。
  • 在佈局類的兩個最常用的佈局：正常流和彈性佈局。
  • 繪製類，建議你分成圖形相關的和文字相關的繪製。

瀏覽器的實現原理和 API


瀏覽器部分中瀏覽器的實現原理，是我們深入理解 API 的基礎。

你可以從一般的瀏覽器設計出發，按照解析、構建 DOM 樹、計算 CSS、渲染、合成和繪製的流程來學習瀏覽器的工作原理。

在 API 部分，可以從 W3C 零散的標準中挑選幾個大塊的 API 來詳細講解，主要有：事件、DOM、CSSOM 幾個部分，他們分別覆蓋了交互、語義和可見效果，這是我們工作中用到的主要內容。
