詳解繼承、重載、重寫和多態的關係

2020-10-29
| 科技

一提到面向對象OOP就會想到三大特性，繼承、封裝、多態；多態的實現主要是重載和重寫實現的；多態表現為父類對象引用子類實現子類的向上轉型，這樣父類就能通過指向不同的子類實現不同的子類的方法，子類的方法重寫或者重載父類的方法實現；同樣子類向上轉型被父類引用，還可以再向下轉型放到子類對象

public class Animal {    public void eat(){        System.out.println("animal eat");    }   public void run(){        System.out.println("animal run");    }} public class Cat extends Animal{     public void eat(){         System.out.println("貓愛吃魚");    }    public void run(){        System.out.println("貓會跳");    }} public class Dog extends Animal{     public void eat(){         System.out.println("狗愛吃骨頭");    }    public void run(){        System.out.println("狗會跑");    }} public class Main {     public static void main(String[] args) {         Animal animal = new Cat(); //向上轉型        animal.eat(); //貓愛吃魚        Cat c = ((Cat) a); //向下轉型        c.eat()     //貓會跳         animal = new Dog();//向上轉型        animal.eat();  //狗愛吃骨頭        Cat c1 = ((Cat) a1);  //向下轉型        c1.eat();  //狗會跑    } }

向上轉型子類定義但是父類沒定義的的方法會丟失，只能用重載、重寫和繼承的方法；這樣減少代碼的重複性，提高了擴展性；向下轉型是父類對象轉為子類的引用指向，但是隻能轉為子類對象本身的實際對象也就是cat類型的animal只能是cat不能是其他類型


所以多態特性的實現必須要有繼承，子類繼承父類或者接口；重載或者重寫，子類重寫父類的方式去實現不同的方法形式；父引用指向子類實例，實現父類引用調用子類方法。多態的特性帶來的好處主要是對代碼的替換(子類方式去實現方法，父類很容易切換調用不同的子類對象去調用不同的方法)、接口靈活性(通過interface定義方法簽名，由不同的子類覆蓋實現，子類之間完全沒有相互依賴，調用的時候可以自由選擇切換)
