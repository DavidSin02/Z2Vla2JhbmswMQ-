函數第二部分：動態參數是自由主義的體現-Python基礎前傳（11）

2020-10-29
| 科技

編程語言就是大家一起定個規則大家一起玩。規則是人定的，但也是定給人的。我們之前學習函數的參數，都遵循著一個原則就是：形式參數有幾個，實際參數就只能傳幾個；我們從開發 python 這門編程語言的角度思考，我們要是真的就是規定形式參數有幾個，實際參數就有幾個的原則，是不是不能滿足實際工作需要，在實際工作中誰又能保證每一次都預先設定好參數的個數呢？這個時候，作為編程語言的設計人員，就會引進動態參數，我們想想如果不引入動態參數，誰還能使用這個編程語言呢。

（一）動態參數1-一個星號變元組

  • 動態參數存在的意義？

    • 函數的作者有時候也不知道這個函數到底需要多少個參數，這時候動態參數就有存在的意義了

  • 動態參數創建-加 *

    • 底層原理是：把數值型或其他數據類型變成了元組類型，是元組了就可以多加實際參數了（一個星號變元組，傳的參數就是元組的元素）

#不用動態參數
def f1(a):
print(a,type(a))
f1(123)
>>> 123 <class'int'>

#使用動態參數
def f1(*a):
print(a,type(a))
f1(123，456，789)
>>> (123,456,789) <class'tuple'>
f1(123,[11,22,33],{"key1":"values2"})
>>>(123,[11,22,33],{"k1":"values2"}) <class 'tuple'>

（二）動態參數2-兩個星號變字典

def f1(**a):
print(a,type(a))
f1(k1=123,k2=456)

>>>{'k1':123,'k2':456} <class 'dict'>

說到這裡，大家有沒有注意到一個問題：就是動態1（1個星號）是元組只能是元組，動態2（2個星號）是字典只能是字典，我們有沒有什麼辦法，將動態1和動態2組合起來，讓我們參數既可以是元組，又可以是字典呢？

def f1(*a,**aa):
print(a,type(a))
print(aa, type(aa))
f1(11,22,33,k1=123,k2=456)
>>>(11, 22, 33) <class 'tuple'>
>>>{'k1': 123, 'k2': 456} <class 'dict'>

  • 這裡jacky要給大家強調的是：*a 一定要放在 **a 前面，順序不能顛倒，這是固定寫法

（三）動態參數的規範書寫

  • 一般情況下，把動態參數設置成如下規範形式

# *args, **kwargs
def f1(*args,**kwargs` )

  • 為動態參數傳入列表、字典、元組

def f1(*args):
print(args,type(args))
li = [11,22,33]
f1(li)
f1(*li) #加*相當於for循環遍歷了列表裡的元素

>>>([11,22,33],)<class 'tuple'>
>>>(11,22,33)<class 'tuple'>

def f1(**kwargs):
print(kwargs,type(kwargs))
dic = {"k1":123}
f1(k1=dic)
f1(**dic)

>>>{'k1':{'k1':123}}<class 'dict'>
>>>{'k1':123}<class 'dict'>
