【威脅預警】iSCSI未授權訪問漏洞，數萬iSCSI可能受影響

2020-10-29
| 科技

概述

iSCSI（Internet Small Computer System Interface）Internet小型計算機系統接口，又稱為IP-SAN，是一種基於因特網及SCSI-3協議下的存儲技術，由IETF提出，並於2003年2月11日成為正式的標準。

2019年4月17日，白帽匯安全研究院（以下簡稱白帽匯）經過FOFA系統分析，發現存在許多未授權訪問的iSCSI面臨安全風險，任意的攻擊者都可對未授權訪問的iSCIS進行訪問、修改、刪除等操作，對用戶造成數據洩露、數據丟失等風險。

根據FOFA系統近一年統計顯示，目前全球共有34564個iSCSI協議對外開放（非漏洞情況）。其中，中國第一，共有8726個對外開放；第二是韓國，共有3688個；第三是美國，共有3662個；第四是俄羅斯，共2602個。

iSCSI 全球分佈情況（僅為分佈情況，非漏洞影響情況）

中國大陸地區對外開放的共有5337個。其中，廣東省第一，共有768個；第二是北京市，共有699個；第三是江蘇省，共有511個；第四是浙江省，共有492個；第五是上海市，共有365個。

iSCSI 中國大陸地區分佈情況（僅為分佈情況，非漏洞影響情況）

漏洞原理與危害

iSCSI利用了TCP/IP的port 860 和 3260 作為溝通的渠道。透過兩部計算機之間利用iSCSI的協議來交換SCSI命令，讓計算機可以透過高速的局域網集線來把SAN模擬成為本地的儲存裝置。該漏洞主要由配置不當導致的未授權訪問漏洞。任意的攻擊者都可以使用linux系統或windows系統（筆者使用的是Windows10和Windows server2012 版本，其他Windows版本可自行查看）的軟件進行連接，連接成功後，即可管理遠程的磁盤，查看、刪除、修改文件等操作。

如果要連接iSCSI，可使用Windows系統自帶的iSCSI發起程序進行連接，linux系統則可以安裝相應軟件（管理軟件有多個，可自行搜索。如：iscsi-initiator-utils。）來進行連接。另外需要注意的是，Windows使用iSCSI，需要先啟動msiscsi服務。第一次使用iSCSI發起程序會進行提示，點擊是可直接啟動服務。

通過Windows iSCSI發起程序的快速連接功能進行快速連接或通過iSCSI的發現門戶功能添加IP地址，根據列出的門戶目標，選擇目標名稱即可連接，連接成功。連接成功後，即可在計算機管理的磁盤管理中看到磁盤。這裡磁盤可能出現為脫機狀態或者為初始化狀態等情況，在Windows資源管理器中是不會顯示磁盤盤符，這時只需將硬盤聯機、進行初始化、分配磁盤盤符等來查看磁盤文件。同時Wnidows系統中的iSCSI支持命令行，啟動msiscsi服務後可使用iscsicli命令對iSCSI來進行管理。

linux系統下有許多工具，使用搜索引擎可以搜到相關工具和使用說明。由於筆者還未使用，這裡不做說明，大家可自行嘗試。

Windows Server 2012 的iSCISI連接成功截圖

漏洞影響

通過通信的協議特徵，直接可通過FOFA搜索到存在未授權訪問的網絡資產。其未授權訪問的標誌為authmethod=None。目前，FOFA系統近一年數據顯示，全球共有17733個存在iSCSI存在未授權訪問漏洞，並可能被攻擊者利用。其中，中國第一，共有5166個；第二是美國，共有2020個；第三是韓國，共有1514個；第四是俄羅斯，共1026個。

可訪問 http://t.cn/EXY0o4w 查看全球受影響情況(如連接有問題，可通過https://fofa.so 網站搜索protocol=="iscsi" && banner="authmethod=None" 查看)。

iSCSI 全球存在未授權特徵分佈情況

中國大陸地區共有4084個存在iSCSI存在未授權訪問漏洞，並可能被攻擊者利用。其中，廣東省第一，共有600個；第二是北京市，共有563個；第三是江蘇省，共有372個；第四是浙江省，共有362個；第五是上海市，共有304個。
