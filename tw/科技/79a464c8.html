乾貨｜|一文帶你瞭解世界上最先進的分佈式版本控制系統Git

2020-11-14
| 科技


Git，是目前世界上最先進的分佈式版本控制系統。

版本控制系統（version control system），是一種記錄一個或若干文件內容變化，以便便將來查閱。

特定版本修訂情況的系統。版本控制系統不不僅可以應⽤用於軟件源代碼的⽂文本文件，而且可以對任何類型的文件進行版本控制。


就像哆啦A夢的時光機，它記錄了了我們每一個時間段做過的每一件事情，因此我們可以隨時通過時光機回到過去。

版本控制系統除了了 Git 這種分佈式以外，還有一種集中式。

集中式

最早期的時候，為了了讓不同系統上的開發者能夠協同工作，誕生了集中式版本控制系統（CVCS）。

CVCS 都有一個單一的集中管理理的服務器，用來保存所有文件的修訂版本。而協同工作的人們都通過客戶端連接到這臺服務器，獲取最新的文件或者提交更新。


集中式的缺點

CVCS 有一個最顯而易見的缺點，就是中央服務器的單點故障問題。如果負責集中管理的服務器死翹翹了，那麼就會導致誰都無法提交更新的，也就⽆無法協同工作，那我們只能集體去喝茶聊人生了；

如果磁盤發生故障，而備份又不夠即時，那麼就有丟失數據的風險，最壞的情況是丟失整個項目的歷史更改記錄，那我們只能以身相…不，不是，是以死謝罪了了！

分佈式

正是由於集中式版本控制系統的缺點，分佈式版本控制系統問世了（DVCS）。

DVCS 沒有“中央服務器”，每個人的電腦上都是一個完整的版本庫，這樣，我們在工作的時候，就不需要聯網了，因為版本庫就在自己的電腦上。

但是既然每個人電腦上都有一個完整的版本庫，那多人之間如何協作呢？例如你在自己電腦上改了文件A，你的同事也在他的電腦上改了文件A，這時，你們倆之間只需把各自的修改推送給對方，就可以互相看到對方的修改了。


關於 Git

Linus 在 1991 年年創建了開源的 Linux，從此，Linux 系統不斷髮展，成為最大的服務器系統軟件了。


Linus 雖然創建了 Linux，但 Linux 的壯大是靠全世界熱心的志願者參與的，世界各地的程序員都在為Linux 編寫代碼，那麼這麼多的代碼怎麼管理？那個時期集中式版本控制系統 CVS、SVN 等已經誕生了，但是 Linus 義正言辭斬釘截鐵鏗鏘有力……的拒絕了！

因為集中式版本控制系統不但速度慢，而且必須聯網才能使用，Linus 表示：不！喜！歡！

到 2002 年，BitMover 公司出於人道主義精神，授權 Linux 社區免費使用他們公司本來要收費的BitKeeper 版本控制系統。Linus 表示：很喜歡嘻嘻嘻….

但是安定團結的大好局面在 2005 年被打破了。

因為 Linux 社區中開發 Samba 的 Andrew 試圖破解BitKeeper 的協議，結果被 BitMover 公司發現了，於是 BitMover 公司怒了，收回了了 Linux 社區對於

BitKeeper 的免費使用權。

正常劇情接下來應該是 Linus 向 BitMover 公司道個歉，然後大家握手言和執手相看淚眼……

但是，Linus 表示：呸！老子就不！

於是 Linus 花了兩週時間自己用 C 語言寫了一個分佈式版本控制系統，Git 就此誕生了！一個月之內，Linux 系統的源碼就已經由 Git 管理了。

Git 工作流


Workspace：工作區

Index（Stage）：暫存區

Repository：倉庫區（本地倉庫）

Remote：遠程倉庫

Git 的安裝配置

Git 的安裝 So Easy，直接從 Git 官網 下載安裝包安裝即可。

初次運行 Git 前的配置一般在新的系統上，我們都需要先配置下⾃自⼰己的 Git 工作環境。配置⼯工作只需⼀一次，以後升級時還會沿用現在的配置。

當然，如果需要，你隨時可以用相同的命令修改已有的配置。

Git 本地倉庫的使用，添加文件到倉庫


要將一個本地文件添加到本地倉庫，主要分為四步：

1. 創建本地倉庫（版本庫）

2. 將本地文件添加至工作區

3. 將⼯工作區文件添加至暫存區

4. 將暫存區文件提交至倉庫

創建版本庫

版本庫，中⽂文名又叫"倉庫"，英文名叫"repository"。它其實就是⼀一個本地的目錄文件，用來保存我們項目中的所有的文件，然後統一交給 Git 管理。

創建一個版本庫非常簡單，主要分為兩步：

第一步，選擇一個合適的地方，創建一個空目錄（例如我在桌面創建了一個 testgit ⽂文件）。

第二步，在命令執行工具中進入目錄路路徑，執行 git init 命令。


執行完成後，就可以看到在剛剛的空目錄文件中生成了一個 .git 文件，這個 .git 就是我們的本地倉庫。


注意： .git 是一個隱藏文件，有些電腦默認不顯示隱藏文件，需要去更改一下設置。

工作區

這個包含了 .git 文件的目錄（就是我們一開始創建的空目錄，我創建的 testgit 目錄），就是我們的工作區目錄。

如果後續我們需要向本地倉庫提交文件的話，這個文件就必須在工作區目錄中。

例如，我們想要提交一個 readme.md 的文件至本地倉庫。

那麼首先需要在工作區中添加該文件。


添加文件至暫存區

當我們想要把工作區的文件交給 Git 來管理時，就需要將文件提交到 Git 倉庫，但是在提交到倉庫前還有一個步驟，就是先將文件添加到暫存區。

執行命令 git add 文件名 （ 文件名 輸入我們要添加的文件名，例如： readme.txt ），將文件添加至暫存區。


沒有任何提示則說明添加成功。

提交文件至倉庫


當暫存區文件添加完成後，就可以執行命令 git commit -m "提交說明信息" 將暫存區的所有文件提交到倉庫。


以後工作區再有文件需要提交至倉庫，就只需要第3、4步即可。

修改文件

查看工作區文件狀態git status 命令可以用來查看工作區文件的一些狀態。

例如當我們將所有文件都提交至倉庫後，執行一下 git status 命令。


提示"nothing to commit"，沒有任何文件未提交。

接下來我們修改一下工作區的文件。

例如我在 readme.md 中加一個 Hello。

然後我們再執行一下 gitstatus 命令。

提示"Changes not staged for commit"，有修改未添加至暫存區。




查看修改內容

git diff 文件名 命令可以查看當前文件的修改內容。




顯示 +Hello 添加了了一個 Hello。

提交修改

如果要繼續提交修改後的文件，依然按照前面的步驟，先 add 到暫存區，再 commit 到倉庫。




刪除文件

當我們需要刪除一些文件，首先在工作區目錄中將文件刪除（例如我將工作區的 readme.md 文件刪掉），然後可以查看一下文件狀態。




這個時候，倉庫中的文件還沒有被刪除。

恢復被刪除的文件

如果工作區中的文件被誤刪後，可以通過命令 git checkout -- 文件名 從倉庫中恢復。


刪除倉庫中的文件

當我們從工作區將文件刪除後，倉庫中的文件存在，如果希望將倉庫中的文件也刪除，依然通過 add和 commit 兩個命令。

例如我希望將倉庫中的 readme.md 也刪掉。




版本回退

當我們對文件做了多次修改後，又想要回退到之前的某一個版本。




例如這裡我對 readme.md 文件進行了三次修改。

我們可以通過命令 git log 查看修改記錄。




git log 命令顯示從最近到最遠的操作日誌，顯示太多可以通過上下鍵來查看，按 q 鍵退出日誌。

如果我們現在想要回退到上一個版本，可以通過 git reset --hard HEAD~1 來實現。




想要回退到上面幾個版本，命令最後的數字就寫幾。例如我們想要回退到前 100 個版本，就執行命令git reset --hard HEAD~100 。

但是，實際操作中，我們不可能挨著挨著去數要回退的是第幾個版本。

因此，我們還可以通過 commitid 來進行指定版本的回退。




例如我現在要回退到第一次修改。可以通過命令 git reset --hard b83bd 來實現。




命令最後跟的就是 commit id ，不需要寫全，寫前幾位 Git 就會自動去查找。

以上，就是 Git 在本地倉庫中的使用了。
