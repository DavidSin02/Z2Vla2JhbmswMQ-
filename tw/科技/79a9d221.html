Bash技巧：使用參數擴展轉換字符串的大小寫和替換字符串

2020-11-14
| 科技

在 Linux bash shell 中，通常使用 ${parameter} 表達式來獲取 parameter 變量的值，這是一種參數擴展 (parameter expansion)。

Bash 還提供了其他形式的參數擴展，可以對變量值做一些處理，起到操作字符串的效果。例如：

  • ${parameter^^pattern} 把 parameter 變量值中匹配 pattern 模式字符的小寫字母轉成大寫。
  • ${parameter,,pattern} 把 parameter 變量值中匹配 pattern 模式字符的大寫字母轉成小寫。
  • ${parameter/pattern/string} 把 parameter 變量值中匹配 pattern 模式的部分替換為 string 字符串。

參考 man bash 的 “Parameter Expansion” 小節的說明，具體舉例說明如下。

${parameter^^pattern} 和 ${parameter,,pattern}

查看 man bash 對 ${parameter^pattern}, ${parameter^^pattern}, ${parameter,pattern}, ${parameter,,pattern} 的說明如下：

Case modification.

This expansion modifies the case of alphabetic characters in parameter. The pattern is expanded to produce a pattern just as in pathname expansion.


The ^ operator converts lowercase letters matching pattern to uppercase; the , operator converts matching uppercase letters to lowercase.

The ^^ and ,, expansions convert each matched character in the expanded value; the ^ and , expansions match and convert only the first character in the expanded value.


If pattern is omitted, it is treated like a ?, which matches every character.


If parameter is @ or *, the case modification operation is applied to each positional parameter in turn, and the expansion is the resultant list.

If parameter is an array variable subscripted with @ or *, the case modification operation is applied to each member of the array in turn, and the expansion is the resultant list.

即，這四個表達式會在 parameter 變量值中匹配 pattern 模式，並對匹配的字符進行大小寫轉換：

  • ^ 操作符把小寫字母轉換為大寫，且只轉換開頭的第一個字符
  • , 操作符把大寫字母轉換為小寫，且只轉換開頭的第一個字符
  • ^^ 操作符把小寫字母轉換為大寫，會轉換每一個匹配的字符
  • ,, 操作符把大寫字母轉換為小寫，會轉換每一個匹配的字符

這裡的 pattern 模式可以使用通配符進行擴展，注意不是用正則表達式。

注意：^ 和 , 不是轉換第一個匹配到的字符，而是隻轉換 parameter 變量值的首字符。
所給的 pattern 模式必須和 parameter 變量值的首字符匹配才會轉換，不會轉換字符串中間的字符。

具體舉例說明如下：

$ value="This Is a Test String."

$ echo ${value^t}

This Is a Test String.

$ echo ${value^^t}

This Is a TesT STring.

$ echo ${value,T}

this Is a Test String.

$ echo ${value,,T}

this Is a test String.

可以看到，使用 ${value^t} 不會把 value 變量值中間小寫的 t 字符換行為大寫。
因為這個表達式只匹配和轉換 value 變量值的首字符，value 變量值並不是以小寫字母 t 開頭，不做轉換。

而 ${value^^t} 表達式會匹配 value 變量值中的每一個小寫字母 t，並轉換為大寫。
所以輸出結果裡面不再有小寫的 t 字符。

類似的，${value,T} 表示把 value 變量值開頭的大寫 T 轉換為小寫的 t。
${value,,T} 表示把 value 變量值所有的大寫 T 轉換為小寫的 t。

如果省略 pattern 模式，則表示匹配任意字符，但並不表示會轉換所有字符，^ 和 , 操作符還是隻轉換首字符。

以上面的 value 變量值舉例如下：

$ echo ${value^}

This Is a Test String.

$ echo ${value^^}

THIS IS A TEST STRING.

$ echo ${value,}

this Is a Test String.

$ echo ${value,,}

this is a test string.

可以看到，${value^} 只會把 value 變量值首字符變成大寫，由於原本就是大寫，所以輸出結果跟 value 值一樣。
${value^^} 把所有字符都轉換為大寫。
${value,} 把value 變量值首字符變成小寫。
${value,,} 把所有字符都轉換為小寫。

注意：如果要匹配多個字符，要用方括號 [] 把字符串括起來，進行 pathname expansion，才會得到多個可匹配的字符。
直接把 pattern 模式寫成字符串並不能匹配該字符串中的每一個字符。

以上面的 value 變量值舉例如下：

$ echo ${value,TI}

This Is a Test String.

$ echo ${value,,TI}

This Is a Test String.

$ echo ${value,,[TI]}

this is a test String.

$ echo ${value,[TI]}

this Is a Test String.

可以看到，當所給模式寫為 TI 時，無論是使用 , 還是 ,, ，都不能把大寫的 T 和 I 轉換為小寫。
${value,TI} 甚至都不能轉換開頭的 T 字符。

而寫為 ${value,,[TI]} 就會把所有大寫的 T 和 I 都轉換為小寫。
[TI] 就是 pathname expansion 的一種寫法，表示匹配方括號 [] 裡面的每一個字符。
基於字符匹配，不是基於字符串匹配。
寫為 ${value,[TI]} 表示把首字符 T 或者首字符 I 轉換為小寫，只匹配首字符。

關於 pathname expansion 的具體寫法可以查看 man bash 的 Pathname Expansion 部分。

最常見的就是用 * 通配符匹配零個或多個任意字符，用 ? 匹配任意單個字符。
上面說明中提到，如果省略 pattern 模式，就相當於寫為 ?。
即 ${parameter^^} 等價於 ${parameter^^?}。

${parameter/pattern/string}

查看 man bash 對 ${parameter/pattern/string} 的說明如下：

Pattern substitution.

The pattern is expanded to produce a pattern just as in pathname expansion.

Parameter is expanded and the longest match of pattern against its value is replaced with string.


If pattern begins with /, all matches of pattern are replaced with string. Normally only the first match is replaced.

If pattern begins with #, it must match at the beginning of the expanded value of parameter.

If pattern begins with %, it must match at the end of the expanded value of parameter.


If string is null, matches of pattern are deleted and the / following pattern may be omitted.


If parameter is @ or *, the substitution operation is applied to each positional parameter in turn, and the expansion is the resultant list.

If parameter is an array variable subscripted with @ or *, the substitution operation is applied to each member of the array in turn, and the expansion is the resultant list.

即，${parameter/pattern/string} 表達式可以替換 parameter 變量值的字符串。

所給的 pattern 模式會按照文件名擴展 (pathname expansion) 的方式來擴展，然後對 parameter 變量值進行擴展。
其值中最長匹配 pattern 的部分會被替換成 string 指定的字符串。

如果 pattern 模式開始於 /，所有匹配 pattern 模式的地方都被替換成 string 字符串。
通常僅僅替換第一個匹配的地方。

如果 pattern 模式開始於 #，它必須從頭開始匹配 parameter 變量值。
如果 pattern 模式開始於 %，它必須從後往前匹配 parameter 變量值。

如果 string 字符串是空，匹配 pattern 模式的地方會被刪除，且跟在 pattern 模式之後的 / 字符可以省略。

具體舉例說明如下：

$ value="This is a test string. This is a new test"

$ echo ${value/test/TEST}

This is a TEST string. This is a new test

$ echo ${value//test/TEST}

This is a TEST string. This is a new TEST

$ echo ${value/#test/TEST}

This is a test string. This is a new test

$ echo ${value/#This/THIS}

THIS is a test string. This is a new test

$ echo ${value/%test/TEST}

This is a test string. This is a new TEST

$ echo ${value/test}

This is a string. This is a new test

$ echo ${value//test}

This is a string. This is a new

可以看到，在 ${value/test/TEST} 表達式中，value 變量值是要被替換的原始字符串。
中間的 test 是要被替換的模式，且只替換第一個出現的 "test" 字符串，不會替換所有的 "test" 字符串。 後面的 TEST 是替換之後的內容。
最終輸出的結果是把 value 變量值中的第一個 "test" 字符串替換成了 "TEST"，第二個 "test" 字符串沒有被替換。

${value//test/TEST} 表達式的 "/test" 模式以 / 開頭，表示替換所有出現的 "test" 字符串。
輸出結果所有的 "test" 字符串都替換成了 "TEST"。

${value/#test/TEST} 表達式的 "#test" 模式以 # 開頭，表示要從 value 變量值的第一個字符開始匹配。
由於 value 變量值不是以 "test" 開頭，所以匹配不到，並沒有做替換。

要使用 ${value/#This/THIS} 來把 value 變量值開頭的 "This" 替換成 "THIS"。
${value/%test/TEST} 表達式的情況類似，要求從 value 變量值的末尾往前匹配 "test" 字符串。
這兩者都是從最後一個字符往前開始匹配。

${value/test} 表達式沒有提供替換後的 string 參數，表示從 value 變量值中刪除第一個出現的 "test" 字符串。
${value//test} 表達式的 "/test" 模式以 / 開頭，表示從 value 變量值中刪除所有出現的 "test" 字符串。

上面提到 "最長匹配" 部分會被替換。
所謂的 "最長匹配" 是指被 pattern 模式括起來的最長部分。
常見於用通配符匹配多個字符形成嵌套的情況。

具體舉例如下：

$ value="This is a |test string|new test|, check it"

$ echo ${value/|*|/NEW STRING}

This is a NEW STRING, check it

$ echo ${value/|*|}

This is a , check it

可以看到，所給的匹配模式是 |*|。使用 * 通配符來匹配在兩個 | 之間的任意字符串。
在所給的 value 變量值裡面，"|test string|"、"|test string|new test|" 這兩種形式都匹配這個模式。
實際被替換的是最後一種，也就是最長匹配。
由於該模式沒有以 / 開頭，只處理第一個匹配的地方，所以 "|new test|" 不會被匹配到。

即，當 pattern 模式的擴展結果是不定長的字符串時，它會有一個前綴部分、中間變長部分、後綴部分。
那麼最長匹配是從前綴部分開始匹配，一直到最後一個匹配的後綴部分為止，而不是遇到第一個匹配的後綴部分就停止。
中間變長部分可以包含多個前綴部分和後綴部分。

下面再舉例說明如下：

$ value="This is a test string, first check it"

$ echo ${value/t*st}

This is a check it

可以看到，在所給的 value 變量值裡面，t*st 模式的後綴部分 "st" 匹配到了 "first" 字符串後面的 "st"。
而不是匹配到 "test" 字符串的 "st"。
最終結果取最長匹配的部分。
