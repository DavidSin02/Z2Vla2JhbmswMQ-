找到了！物聯網開發“祕笈”原來在這

2020-11-14
| 科技

傳統物聯網應用模式通常為相互獨立的“煙囪式”系統，應用之間的終端類型、通信協議和數據存儲格式難以統一標準，單個系統的承載和可擴容能力有限，當終端數量和類型出現爆發式增長時，應用系統往往不堪重負，尤其隨著5G進程的提速，大量物聯網應用所管理的終端數將快速增加，以上問題可能頻繁出現。



作為5G產業發展的重要參與方之一，中國電信致力於幫助客戶解決5G物聯網應用的後顧之憂，整合自身的雲、網和生態等優勢資源，打造了中國電信物聯網開放服務平臺（CTWing），可承載海量物聯網終端接入，並向開發者提供網絡管理、雲資源、產品渠道、平臺能力等服務，其中應用使能服務板塊承上啟下，能將設備層和應用層解耦，提供便捷的終端開發、對接方案功能，還支持多種開發方式和開發工具，可助力物聯網應用的快速構建和高效運營。


01

設備便捷對接


CTWing支持多元異構網絡的設備接入，即支持設備通過多種網絡制式和多種通信協議和平臺對接，進行數據的交互，主要支持的網絡和通信協議如下：


網絡制式

CDMA/EVDO、4G （LTE Cat.4/6）、LTE Cat.1、WIFI、藍牙、光纖/寬帶。

通信協議

LWM2M、MQTT、T-Link（中國電信自定義）、TCP、Http/Https、ModBus等，另外支持部標車機協議JT/T808。

開發方式

提供設備對接接口、設備端開發SDK、開發板、開發Sample等。

設備安全

提供特徵串認證、SM9認證、證書認證等設備認證方式，支持一機一密、一行一密等安全類型。可提供SM2、DTLS等數據加密方式。


02

應用快速構建


CTWing可提供SDK和API兩種Web、APP等應用開發方式，也支持C/S架構的桌面應用程序開發。


SDK

可提供適用於Java、.Net（C#）、Python等主流Web應用開發平臺的SDK。

API

以Restful API的形式提供標準接口，用於平臺數據和能力的調用。

應用部署方式

可提供應用託管及容器化部署方案，還可在平臺上開通雲主機等服務用於部署應用，也可以是運行在其它物理機或雲服務器上的獨立應用。


03

平臺能力支撐


CTWing平臺將物聯網開發過程中常見的比較繁瑣、機械重複但又非常耗費開發成本的一些操作（如設備管理和數據處理）等進行了提煉和封裝，形成了通用性的服務能力，開發者可以使用這些能力較為高效地實現設備的適配、管理以及數據的清洗、流轉、存儲等動作，有效降低開發難度。


設備管理


生命週期管理


提供設備生命週期的管理，包括創建、刪除、註銷、激活設備等動作。

狀態管理


提供設備上/下線變更通知和主動查詢接口，可以實時讀取設備的最新狀態。

物模型


對設備進行數字化建模，促進物聯網開發的協議標準化和數據結構化，提高開發的敏捷度。

在線調試


支持遠程調試真實設備，模擬指令下發，也可以通過虛擬設備模擬數據上報和指令下發。

固件升級


支持LWM2M協議的FOTA和SOTA升級。

消息追蹤


在設備登錄、命令下發和數據上報等過程中出現故障時，提供快速的故障定位和原因分析手段。


數據處理和流轉


支持HTTP（S）和MQ兩種方式進行消息訂閱推送。

支持通過規則引擎進行數據處理和流轉到任意物理機或雲服務器/存儲。

支持基於規則引擎的場景聯動和告警管理。


基於CTWing的物聯網應用開發流程


04

雲產品服務


CTWing平臺的雲產品是基於中國電信的天翼雲提供的融合服務，目前在CTWing上可訂購的雲產品包括瞭如下幾類：


主機類

雲主機、容器服務、雲堡壘機、虛擬私有云。

存儲類

對象存儲、雲硬盤。

數據庫類

Mysql、SqlServer、PostgreSQL。

其它類

分佈式消息、彈性IP、彈性負載均衡等。


CTWing平臺的雲產品與中國電信的物聯網業務不為強相關關係，通過CTWing購買的雲資源，既可用於在CTWing上開發的應用部署和數據存儲，也可以用作其它用途。


CTWing平臺提供的雲產品


05

實時技術支持


CTWing平臺的文檔中心提供了全面的平臺使用說明和開發樣例，為平臺的用戶提供詳細的開發指引，並可通過QQ群，聯繫電話等方式向開發者提供技術支持，此外，CTWing打造了開發者社區，為開發者提供技術交流平臺，並配備專業技術支持人員為開發者提供諮詢服務。
