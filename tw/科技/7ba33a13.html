對大數據與商務智能的粗淺認識

2020-11-14
| 科技

進入21世紀，全球信息革命的一個重要標誌就是大數據時代的到來。究竟什麼是大數據？為什麼在21世紀興起？大數據對人類社會的發展有什麼影響？我們將如何應對？本文將帶著大家一起沿著信息革命的歷程來探究這些問題的答案。

什麼是大數據？

大數據（Big Data）實際上是一種“俗稱”，就如“雲計算”的命名一樣，都是為了讓社會民眾更好的瞭解這些專業領域的新生事物而起的一種通俗的名稱。

大數據實際上指的就是信息系統產生的海量數據，因為此類數據具有規模龐大、組成複雜、來源多樣、格式各異、持續湧現的特性，在如何處理和發揮其價值方面存在諸多困難、面臨很多挑戰，我們才用“大”這個詞對其進行修飾。大數據的“大”既體現了其規模大（Big Volume），也體現了其價值“大”(Great Value)，更說明了它是個不容易對付的“大”問題(Big problem)。

目前大數據主要產生和應用的領域包括：

  • IT、通信領域及物聯網領域
  • 電子商務及互聯網領域
  • 交通與物流領域
  • 商業服務領域
  • 金融領域
  • 醫療健康領域
  • 氣象服務領域
  • 以上領域都是和民眾日常工作、生活息息相關的業務領域，還有很多專業領域如天文、宇航領域、工業控制領域等等，也是大數據應用的主戰場。

大數據為什麼在21世紀興起？

大數據出現的根本原因，也是其發展的基本動力，就是信息技術的進步及普及應用，也可以說是信息化發展的結果。

隨著信息技術的普及應用以及各行各業信息化的發展，信息系統所產生的數據越來越多，產生數據的速度越來越快，不斷積累的海量數據逐漸成為組織運營的負擔，同時海量數據中蘊含的“規律”、“經驗”等知識也逐步引起大家的興趣和重視，“大數據”隨之應運而生。可以說，我們對大數據的愛與恨、悲與欣是交織在一起的。

“大數據”並非在21世紀出現，早在20世紀90年代就初現端倪。數據倉庫之父比爾·恩門（Bill Inmon）在1991年出版的“Building the Data Warehouse”（《建立數據倉庫》）一書中就已經提醒社會對海量數據的重視。從那時起，對大數據的研究就進入了一個小高潮，突出表現在數據倉庫和商務智能技術的誕生。當時社會對大數據的認識主要集中在專業領域，沒有達到現在“全民化、全球化”的“現象級”水平。

但是隨之幾年後進入低潮期，主要原因是受到當時IT資產性能水平和性價比的限制，很多技術、功能“看上去很美”但無法實現，無法普及應用。直至21世紀IT資產的性能水平大幅提升、建設成本顯著下降，大數據技術的應用普及逐步走入尋常百姓之家。

大數據會對人類社會產生哪些影響？

任何事物對人類社會的影響都分為正反兩個方面，而且可以相互轉化，大數據也不例外。
  • 大數據首先給我們帶來的就是沉重的管理負擔。

這一點從數據容量單位上就可以看出來。20世紀90年代以Mb計量數據就已經是天量的感覺了，後來短短几年就上升到Gb、Tb、Pb、Eb、Zb，數據量級呈幾何級數增長，社會組織在數據存儲和管理方面付出了高額的代價。以銀行業為例，每TB單位就要花費10萬元的運維成本，設想一下，在電子銀行成為主流服務形式的時代，銀行業每天產生的海量數據將會給銀行增加多少費用支出？有機構統計，銀行業每創收100萬，平均就會產生約1TGB的數據，僅數據運維成本就佔到總收入的10%！

  • 大數據的泛濫會給我們帶來極大的社會風險

大數據來源廣泛，內容各異，良莠不齊，會製造很多社會動亂。目前互聯網謠言四起、小道消息滿天飛，不但影響了社會組織的正常運營，也給民眾生活帶來很多困擾，小則誤導人們做出錯誤判斷，大至影響國家安定團結和世界格局的穩定。以當年日本福島核電危機為例，“食用海鹽防輻射”的謠言就讓“粗粒海鹽”搶購一空，各家各戶囤積海鹽，不但影響了民眾的日常生活，而且成了全球的笑柄。在金融領域，“虛假消息”的快速蔓延對股市、匯市、期貨市場的影響更加嚴重，引發全球經濟波動，“蝴蝶效應”凸顯，全球經濟危機一觸即發。

  • 大數據是管理創新、技術創新的重要驅動力

大數據的價值之一就是決策支持。在大數據的支持下，組織管理決策將獲得更多的信息支持，保證管理者更加全面地認識問題、分析問題，做出更加準確的判斷和決策，從而大幅提高組織的經營管理水平和運營效率，提升組織的競爭力。

大數據的價值之二就是智能化。所謂智能化就是應用人工智能技術，讓組織的業務流程可以自行運轉，提高業務處理的效率和質量。無人辦公、無人商場、自動駕駛、智能交通、智能醫療就是非常典型的應用。而人工智能的核心是機器學習，機器學習脫離了大數據的支持，就成了無源之水、無本之木。近幾年人工智能膾炙人口，與大數據的發展與支持是息息相關的。

  • 大數據能夠提供更好、更多的民生服務¶

民眾的日常生活離不開信息服務，更高質量、更加豐富的信息服務使我們的日常生活變得更加便捷。在交通、旅遊、購物、餐飲、教育、醫療等關鍵民生領域，我們目前無時無刻不在享受大數據的服務。就以旅遊服務為例，以前我們採取的基本模式就是通過旅行社獲得服務，既不靈活也不自由成本也高，原因在於我們獲得信息的渠道太少。如今自駕遊、自由行基本成為民眾旅遊的主要形式，這全依賴於豐富的旅遊服務信息和交通服務信息，從選擇景點到購買門票、預訂酒店、預訂車票都可以在出行前一氣呵成，回退到十幾年前，這是不可想象的。

如何應對大數據？

將大數據比作垃圾不太合適，但是如果沒有正確的處理，大數據就會變成信息世界的垃圾，引發嚴重的環保問題。

幸運的是，在大家的共同努力下，目前我們處理大數據的思想、方法、技術和工具不斷湧現，不斷進步，為我們處理這個“大麻煩”提供了極大的幫助，我們正經歷“變廢為寶”的革命歷程。以下我們將從平臺級、工具級、應用級分別介紹應對大數據的各種優秀措施。

  • 大數據平臺級處理

大數據平臺級處理主要完成對大數據的存儲管理、檢索支持。這一方面與我們熟悉的數據庫平臺極為相似。

在平臺級應用中，首當其衝的就是Hadoop平臺。Hadoop是大數據領域主流的平臺級應用。Hadoop實現了一個分佈式文件系統（Hadoop Distributed File System），簡稱HDFS。HDFS有高容錯性的特點，並且設計用來部署在低廉的（low-cost）硬件上；而且它提供高吞吐量（high throughput）來訪問應用程序的數據，適合那些有著超大數據集（large data set）的應用程序。目前全球知名的互聯網信息服務基本採用了這一技術。Hadoop本身還是一個大數據處理工具的大家庭，內置了很多優秀的數據處理和數據分析的工具。

其次是文檔數據庫和內存數據庫平臺。這兩個領域代表性的產品為MongoDB文檔數據庫和Redis內存數據庫。

MongoDB是一個介於關係數據庫和非關係數據庫之間的分佈式非結構化數據庫產品，是非關係數據庫當中功能最豐富，最像關係數據庫的。它支持的數據結構非常鬆散，是類似json的bson格式，因此可以存儲比較複雜的數據類型。Mongo最大的特點是它支持的查詢語言非常強大，其語法有點類似於面向對象的查詢語言，幾乎可以實現類似關係數據庫單表查詢的絕大部分功能，而且還支持對數據建立索引。目前深為中小型大數據應用所青睞。

Redis 是一個高性能的key-value內存數據庫，支持對高頻訪問數據的高速訪問。它支持string(字符串)、list(鏈表)、set(集合)、zset(sorted set --有序集合)和hash（哈希類型）數據類型。為了保證效率，數據都是緩存在內存中，同時redis會週期性的把更新的數據寫入磁盤或者把修改操作寫入追加的記錄文件，並且在此基礎上實現了master-slave(主從)同步，從而保證數據在服務器斷電後的恢復。

  • 大數據工具級處理

大數據工具級處理主要完成對數據的採集、清洗、變換和加載，也就是平時所說的ETL操作。採集操作主要從各類數據源採集所需要的數據，這些數據源包括：文本文件、數據庫和在線實時數據；清洗操作主要用於清除數據中的“垃圾數據”，這些數據包括：非法數據（數值超出合法範圍）、噪聲數據（因採集方法或數據源本身的原因而產生的干擾數據，比如數據在傳輸過程中出現的畸變、測量誤差等）和嚴重殘缺數據（信息內容不完整，無法推測補全的數據）；變換則根據數據格式和取值類型的標準，對多種表示的同類數據進行重新表達，以便後續處理；加載則完成將數據輸送到應用處理環節，或者補充到數據庫中。

目前ETL方面的工具非常多，有產品級的（如Pentaho），也有構架級的(如Storm和flink，見hadoop體系結構圖)等，可以滿足不同場景的數據ETL處理需要。

大數據應用級處理

大數據應用級處理分類比較多，我們在此將其分為基本信息服務和商務智能兩個領域。（不算完整、嚴謹的一種狹義分類）

基本信息服務專指將數據彙集在一起，提供基本的信息檢索服務，比如天氣預報、交通信息服務等等，在此不再贅述。

商務智能(Business Intelligence),也稱商業智能。業界普遍認為商務智能的概念在1996年最早由加特納集團（Gartner Group）提出，加特納集團將商業智能定義為：商業智能描述了一系列的概念和方法，通過應用基於事實的支持系統來輔助商業決策的制定。商業智能提供使企業迅速分析數據的技術和方法，包括收集、管理和分析數據，將這些數據轉化為有用的信息，然後分發到企業各處。

商務智能目前分為聯機分析處理和數據挖掘兩個專業領域。其中：

  • 聯機分析處理（OLAP）：支持複雜的分析操作，側重決策支持，並且提供直觀易懂的查詢結果。OLAP使分析人員、管理人員或執行人員能夠從多角度對信息進行快速、一致、交互地存取，從而獲得對數據的更深入瞭解。OLAP的目標是滿足決策支持或者滿足在多維環境下特定的查詢和報表需求。OLAP的基本多維分析操作有鑽取（roll up和drill down）、切片（slice）和切塊（dice）、以及旋轉（pivot）等，並將結果以圖表的形式進行直觀展現（數據可視化）。目前市面上的商務智能產品多指該類應用。我們此次開設的《商務智能分析師初級進階課程》就面向聯機分析處理。

  • 數據挖掘（Data Mining）：是數據庫知識發現（Knowledge-Discovery in Databases，簡稱：KDD)中的核心環節。一般特指從大量的數據中通過算法搜索隱藏於其中的規則、模式等高級規律性知識的過程。這些挖掘成果被模型化、軟件化後，可以內置到信息系統中，實現業務流程的智能化和自動化。目前數據挖掘技術在風險控制、醫療服務、交通物流等領域廣泛應用，是大數據應用的熱點。數據挖掘通常與計算機科學有關，並通過數理統計、機器學習、專家系統（依靠經驗法則）和模式識別等諸多方法來實現上述目標。我們此次開設的《商務智能分析師高級進階課程》專門面向數據挖掘領域。
