CPU體系結構

寄存器的考察比較多

CPU Central Proccessing Unit 中央處理單元

微處理器 Microprocessor

運算器 控制器 寄存組 內部總線



（這是百度圖片找到的構架圖，輔助參考理解）

控制器

程序計數器 指令寄存器 數據寄存器 指令譯碼器

1. 程序計數器 PC 指出下一條指令在主存中的存放地址指令是數據 PC增加指向下一條地址（理解待命狀態）

2. 指令寄存器 IR 保存當前正在執行的指令寄存器的位數決定指令字長

3. 地址寄存器 AR 用於存放CPU當前正訪問的內存單元的地址

4. 數據寄存器 暫存內存器中讀出/寫入的指令數據（跟後面的數據寄存器的區別）

5. 指令譯碼器 對獲取的指令譯碼 執行一系列微操作信號 控制終端機各個部件

運算器

算數邏輯單元 通用寄存器 數據暫存器

程序狀態字寄存器 執行控制器發來的命令 相應操作對數據的加工處理

1. ALU 各種算數邏輯

2. 通用寄存器存放操作數中間結果 各種地址信息 一些列存儲單元

a. 數據寄存器

AX 累加寄存器 算數運算的重要寄存器

BX 基址寄存器

CX 計數寄存器 串操作循環控制

DX 數據寄存器

b. 地址寄存器

SI 源變地址寄存器

DI 目的變地址寄存器

SP 堆棧寄存器

BP 基址指針寄存器

c. 累加寄存器

AR AR為ALU提供區域

3. 數據暫存器 DR 用來暫存 不能放通用裡 會破壞原有內容

4. 程序狀態字寄存器

（以上需要系統學習，很多東西較為模糊，現在只能理解記憶）

CPU指令的執行

指令機器 語句二進制

操作碼：操作性質功能

地址碼：操作數的地址

操作碼必須有 地址碼可以很多

指令譯碼器測試操作碼

指令寄存器中 操作字段的輸出 就是指令譯碼器的輸入

譯碼後操作控制器 相應信號

CPU指令的執行三個步驟

1. 取指令

根據程序計數器（提供地址） 主存儲器中命令送主存數據緩衝（主存和高速緩衝的關係後面會介紹）

送往CPU內指令寄存器 更改程序計數器 （指向）下一條

2. 取操作數

無操作數直接下一過程

有操作數尋址方式計算地址存儲器中 取操作數

3. 執行操作

根據操作碼完成相應 操作數尋址方式保存結果

指令執行週期緊密相關

a. 取指週期 PC給出地址取指令

PC自動遞增 出現轉移指令 地址執行週期就會被改

此週期解決計算機操作數地址並取出操作數

b. 執行週期 完成指令操作碼規定動作

傳送結果/記錄狀態保留狀態信息

c. 指令週期 一條指令從取出到執行完成

指令週期是完成一條指令的時間指令週期

劃分幾個不同階段 階段時間為機器週期

CPU工作週期或者基本週期 時間週期是時鐘頻率的倒數 是處理操作的基本單位

一個指令週期由若干機器週期組成 每個機器週期由若干時鐘週期完成

CPU指令系統

1. 複雜指令集Complex Instruction Set Computer CISC

各條指令順序串行 控制簡單 利用率低 速度慢 為了兼容仍在使用

2. 精簡指令集Reduced Instruction Set Computr RISC

由CISC發展出來 指令使用頻率 概率（信息論哈夫曼編碼 可以瞭解一下）

精簡指令系統 超標量 超流水結構（不太理解）

指令格式統一 種類少 尋址簡單 速度高

與CISC軟硬件不兼容 高檔服務器RISC

兩者在以下幾方面具有區別：

a. 指令系統指數目

b. 變成便利性

c. 尋址方式

d. 指令長度

e. 控制器的複雜性

CPU主要性能指標

1. 主頻時鐘頻率 GHz 重要參數之一 但不是唯一參數

2. 位和字長單位時間一次處理二進制位數 （32位與64位系統的區別？）

3. 緩存 CPU與主存之間 內存小 速度快 接近CPU速度

解決CPU與主存之間速度不匹配問題緩存結構大小

L1 Cache 數據緩存指令 緩存跟CPU面積有關 很小

L2 Cache 內部/外部兩種 內部接近主頻 外部50%主頻速度 2-4MB

（日常購買時的重要參數之一）

L3 Cache 大數據量計算使用為數值計算

流水線技術

流水線指令 分解小步驟

讓不同指令操作步驟重疊 指令並行處理 加速程序速度

（像是工廠的流水線工人負責不同的部分 而不是一個人 完成整條流水任務）

執行命令最長的時間 變成了瓶頸

流水線週期總時間取決於操作步驟中最長時間的那個操作

流水線的性能指標

吞吐量 單位時間內可以處理的任務指令個數

加速比 某任務採用串行與流水模式的比值

效率 流水線中各個部件的利用率（不能有閒人）

內存結構和尋址

兩大類 主存儲器 輔助存儲器 也就是內存和外存

主存儲器類型：

1. RAM Random Access Memory CPU對RAM隨機使用存儲與物理位置無關 斷電後信息丟失

DRAM SRAM DRAM定期刷性 SRAM不斷電不丟棄

2. ROM 隨機存取ReadyOnly Memory 固定在存儲器內的信息

3. 順序存取存儲器Sequential Access Memory SAM 按照某種順序時間跟物理位置有關磁帶機

4. 直接存取存儲器Direct Access Memory 直接向整個存儲器某個範圍按照順序檢索DAM比SAM快點

5. CAM Content AddressableMemory 基於數據內容 CAM選擇一部分內容檢索地址標記/比較並行進行速度比較快

高速緩存的概念

CPU與主存之間過度信息 調度和傳送都是硬件完成

（日常生活中硬件解碼 比軟件解碼快）

硬件首先自動取址 進行譯碼 如果需要的數據已經在高速存儲器 主存儲器地址映射為高速存儲器 命中 如果不在 脫靶

硬件把數據從主存儲器調出高速存儲器

高速緩存的命中率

（沒太吃透，不輕易筆記）

內存地址編址

內存單元編號 十六進制從小到大順序排列

每個內存單元8it

（十六進制轉換）

虛擬內存

主存不夠在輔助存儲劃分一定空間 速度慢 但是可以滿足特殊要求

屬於計算機內存管理技術

數的表示與計算

定點數浮點數（不太理解）

定點整數定點小數

原碼 二進制直接表示第一位符號位 0正1負

反碼 正數本身負數除了符號各位取反

補碼 正數本身負數除了符號各位取反 +1

移碼 符號位取反的補碼

總線與中斷

BUS是連接計算機 一組信號線 傳輸信號的通道

通過BUS各部件可以互相通信 網狀的不是點對點互聯

1. 內部總線 CPU內部連接寄存器

2. 外部總線 CPU與外部設備連接傳輸信號：

a. 數據總線Data Bus 雙向總線各個部件之間數據傳輸

b. 地址總線Address Bus CPU 主設備地址信號

c. 控制總線CPU Control Bus CPU與存儲器接口控制信號

地址線的根數和存儲器的容量大小密切關係

中斷的原理

主存與外設之間進行數據傳輸 程序控制 程序中斷控制 DMA

1. 程序控制 CPU執行相應的程序代碼控制 依賴代碼和CPU 效率低

2. 中斷控制 緊急事件發生 CPU終止當前轉入事件子程序 保存寄存器/保存現場 緊急事件處理完後恢復 緊急事件的處理方式中斷 中斷程序的入口就是中斷向量（雖然我們感知任務是同時進行的）

a. 單級中斷 不容許打斷完成 才能打斷

b. 多級中斷 多個不同優先級 高級可以斷低級 堆棧方式

中斷提供CPU處理緊急事件手段但是保護現場

3. DMA CPU管理 主存與外設直接數據通路 大數據處理不中斷、使用DMA

CPU初始化 地址空間轉變 而後DMA控制

完成後告訴CPU CPU不需要全程參與 簡單的控制DMA得到分權
