如何挑選一款字體顯示出色的顯示器？這些問題值得注意

2020-11-14
| 科技

編注：本文標題《Time to upgrade your monitor》，少數派經作者 Niki Tonsky 授權翻譯轉載。文章題圖素材由 Yulia Prokopova 繪製。

文中關於顯示器的選擇標準是作者從個人需求出發，結合系統字體渲染技術得出的結論，大家請根據自己的實際情況選擇，本文在 Hacker News 上也引發了大量討論。文章最後部分推薦的顯示器可能並未在國內發售，價格也不盡相同，僅供參考。

我是一名程序員，我不會去和數碼繪畫、照片後期、視頻剪輯等等工作打交道，我也不是很在意廣色域甚至正確的色彩還原。我大部分時間裡都呆在文本瀏覽器、文字編輯器和文本終端裡，看著幾乎不動的字母。

為此，我將我的工作設備進行優化，好能讓它們顯示非常非常優質的文字。從這一方面來說，一個優秀的顯示器是非常重要的 —— 不是 nice-to-have，而是必需品。另外，我這裡的「優秀」指的是 —— 越強越好，上不封頂。那麼，下面就是我從我自己的個人經驗角度出發，來談談什麼樣子的顯示器最適合編程開發工作。

低像素密度顯示器

根據我在 程序員、開發者中的調查，我發現 43% 的人依舊在使用屏幕顯示密度低於 150 PPI 的顯示器。

作者在 Twitter 上的投票調查

為什麼這是一個問題呢？因為如果我們想要得到顯示效果更好的字符，那麼唯一途徑就是為每個字符堆疊更多的像素 —— 就是這麼簡單。在過去，我們顯示器的像素總數很小，所以我們學會了去忍受這一事實，甚至我們還發明瞭許多巧妙的技巧來改善我們的觀感體驗。但需要注意的兩個重要事情是：

  • 低分辨率顯示器的時代已經結束了！現在高分屏早已成為大眾消費品；
  • 無論是過去還是現在，那些優化低分辨率屏幕字體觀感的技巧都不能真正提升顯示效果，它們只是讓低分屏上顯示的文本看起來不那麼糟糕，但它們依舊很糟糕。

如果你覺得你可以用什麼辦法讓你的 1080p 分辨率的屏幕通過一些奇淫巧計來渲染清晰的文本，那麼現在我要告訴你：這是完全不可能的。越早接受這一事實，我們才能越早開始尋找真正的解決方案。

為了讓我的結論更有支撐，我們來看看文本在低分屏上的顯示究竟是什麼樣子的，以及我們到底能為這樣的事實用哪些措施進行彌補。（抱歉，但是能做的並不多！）

像素不夠用

首先，低分屏上用來渲染一個字符的像素就是不夠用的。我們以微軟專為開發者設計的 Consolas 字體為例：微軟為了讓 Consolas 在低分屏上有更好的顯示做了非常多的調優工作。我們把 Consolas 設置成 VS Code 默認字體大小 —— 14px（而且有些人甚至調的更小！）：

Consolas 14px 大小字符顯示（macOS）

在這一字體大小下，大寫字母 B 在屏幕上僅僅佔用了 6×9 的像素點。而小寫字母則僅僅有 7 個豎排像素點來渲染自己。這真的太少了，還沒有我的手指頭多。無論我們字體的設計有多好，當我們只能用 7 個像素點的時候，我們都幾乎不能顯示任何東西。任何比「T」和「H」更復雜的字母都會被渲染成一團糟的像素塊。

看上面這張圖裡的小寫字母 g。我們甚至都不太好判斷筆畫從哪裡開始、從哪裡結束，甚至都看不清有多少筆畫。上面那渲染的僅僅是隨機的灰度噪聲、國際象棋盤，完全不是一個字母。真正字母應該是什麼樣的？應該是這樣的。

Consolas 168px 大小字符顯示

看這優美的線條和精緻的細節被強行壓縮成 7×10 的像素塊，我心裡真的很難受。

令人害怕的字體 hinting

為了防止所有字母在低分屏上直接變成一團灰，Windows 上會採用非常激進的字體 hinting 策略。字體 hinting 的原理基本上就是把字符裡的筆畫進行彎折和平移，讓它們更靠近最臨近的像素，好能顯示更清晰的邊緣。

確實，這個方法有用！字體在 hinting 策略下的渲染確實比沒有 hinting 渲染的好一點。

沒有字體 hinting 渲染（macOS）→ 有字體 hinting 渲染（Windows）

但是你也別抱太大期望：這字體在 hinting 策略下渲染出來的效果依舊是一團糟。字體 hinting 並不會讓字體本身看起來更優秀。它確實讓字體渲染變好了一些，但是它還是看起來很糟糕。

字體 hinting 最大的問題，其實是這個渲染策略會損失字符本身的形狀。像素點不會被渲染到其本該在的位置，而是被渲染在像素格子裡面。一個例子：

Verdana (k) 和 Times New Roman (z) 在 13px 大小柵格化之前的渲染效果

基本理念是：如果我們把字體的每個像素都渲染在它本該在的位置上，渲染得到的字體看起來會更好。

但是就算我們只看豎向 hinting 策略的豎排字體筆畫，我們也會發現字符的形狀被改變了太多：

看到豎向的字符筆畫跟實際矢量圖格式的子圖相差多大了嗎？這裡的誤差幾乎有 ¼ 個像素那麼大。

但是！如果我們之前就從來沒見過高分辨率屏幕上 Consolas 的顯示效果，那麼誰還會在意字母 g 顯示的形狀是不是正確的呢？如果我們也根本不知道字母裡豎向筆畫原先應該在什麼位置，那麼誰又會在意筆畫位置是不是錯誤的呢？不過，有時候問題其實還是比較明顯的：本應是圓圈的渲染出來卻不是圓圈，本應距離相當的渲染出來距離不一樣，渲染得到的比例完全不同，本應比較小渲染出來變得特別大（或者反過來）等等。比如下面這個井號：

當我們將字符裡面橫向的筆畫移動到和像素網格對齊時（平移 ½ 個像素的寬度！），Windows 就已經很難把剩下的 7 個像素點平均分開成三個等間距了。可惜令人遺憾的是，我們另一種選項也不大盡如人意。

從我 構建 Fira Code 字體 的個人經驗來說，我已經見過太多的「把邊緣跟最近的像素點對齊就好」這種簡單粗暴的解決辦法出大問題的情況了：

把一個像素分開渲染

我們可以渲染一條比一個像素點還細的標準直線嗎？

可以的。其實方法還挺簡單的，真的。我們顯示器的每個像素點都包含三個垂直的子像素，每個子像素負責呈現自己的顏色。我們可以單獨的讓每個子像素點顯示，從而實現三倍從前的標準橫向分辨率。

當然，實際上我們並不能這樣幹，因為這麼渲染的話我們就渲染出來跟聖誕掛燈一樣的一團糟了：

所以我們又得妥協了（套娃妥協！），通過限制某個像素點顯示的顏色距離黑色有多遠來彌補這樣的現象：

這也就意味著我們字母的形狀並不是 3 倍更加清晰，而可能是 1.5 倍更加清晰，但總體來說依舊挺糊的。

最後，這樣的渲染策略雖然給閱讀文本的體驗帶來了一點提升，但是與此同時，白底黑字周圍就會出現一點青色和橙色的光暈。雖然不太影響，但是你還是能注意到的。

我想表達的主要思想是：所有的這些技巧都有用，有這些技巧比沒有的話嚴格意義上來說還是強的，對於低分屏來說這些策略還是必須的，但是與此同時這些策略還是跟我們當年沒有高分辨率屏幕的時代背景緊密相關的。現在呢？我們已經有高分屏了，這些策略是時候放一放了。

視網膜 Retina 屏幕的 MacBook

Retina MacBook 的屏幕確實可以把字符渲染的非常漂亮。但是，我們必須首先幹這樣的兩件事情。

關閉字體平滑

首先，在「系統設置 - 通用」中關閉「使用平滑字體（可用時）」的選項：

我不太確定現在默認設置是什麼，但是確保這個關掉就行。（從反饋來看，這個選項默認是開啟的，所以記得確定它是關掉的！）

這個設置的名字比較誤導人。原先這個選項叫做「LCD 字體平滑」，表示的是次像素抗鋸齒。但是 Apple 在 2018 年就將次像素抗鋸齒的功能從 macOS 中移除了，並在同一時間淘汰了最後一代非視網膜屏 MacBook。

另外這個名字還暗示了你的字體本身可能就不是平滑的，當然這也不是事實。

這個選項實際上會把你的字體變得稍微粗一些：

關閉「平滑字體」效果

打開「平滑字體」效果

所以，為什麼我們應該把這個選項關掉呢？因為事實上並沒有自動把字體變粗的方法。通常情況下字體的字重是由專業字體設計師設計的。這個設計的過程是一個涉及到上百萬精細的約束的過程。如果你是通過給字母外圍增加一圈輪廓來模擬粗體的話，得到的結果會非常糟糕：

真實粗體 VS 通過增加輪廓模擬的粗體

但這正是 macOS 「字體平滑」這一功能所做的事情！這裡是另外一個例子，開啟「字體平滑」的 macOS 會將原本清晰絲滑的像素邊緣渲染成一片模糊：

想象一個字體設計師非常認真的平衡每個字母筆畫，將每個點的擺放位置精確到 1/100 個像素的程度，結果到最後還是被自作聰明的軟件直接忽視。

那麼這對於我們開發者來說意味著什麼呢？如果我們拿一個專為某個特定像素大小徒手設計優化的字體來看（大多數編程字體，比如 11px 大小的 Input 字體、12px 大小的 Monoid 字體，都是這樣的），那麼這些字體儘管做了種種努力，依舊會被渲染得非常模糊。

與此同時所有的其他字體，包括系統自帶字體，都會變得比他們原本更加模糊。

另外：Chris Morgan 在 這個評論中 提到了這個設置選項可能就是許多設計師在 自己的網頁默認字體 裡面把 font-weight: 300 作為默認值的一個原因。他們就是為了彌補 macOS 這一字體加粗設置的坑啊！

整數倍縮放

當我在 2012 年買了我的第一臺（也是蘋果首款）視網膜屏幕 MacBook Pro 時，其屏幕表現就像宣傳所說的一樣：2 倍縮放，每個邏輯像素都被渲染為 2×2 的屏幕真實像素。2800×1800 分辨率的屏幕會根據 1400×900 的邏輯分辨率源進行渲染。

可惜， Apple 自那時起就失了智，並在之後的某個時刻，MacBook 開始使用奇怪的非整數分辨率作為默認選項。比如：2880×1800 分辨率的屏幕會有 1680×1050 的邏輯分辨率。這算下來就是 1.7142857143… 倍數的縮放比例，也就是 12/7.

為什麼會出現這種情況呢？我猜想：Apple 內部有人覺得我們屏幕顯示空間越大，產品就賣得越好。但是問題在於，你增加的也並不多啊 —— 也就變大了 15%。我是說，15% 挺好，但光加大 15% 也並非是顛覆性的改變呀。最糟糕的是，你加大 15% 之後，非常容易導致我們根本無法渲染出任何像素清晰的圖片。

我們來看看。12/7 的縮放比例意味著對於每 7 個邏輯像素點，會有 12 個與之對應的屏幕物理像素點。也就是，對於每 7 個像素，我們只有畫一個 7 個像素高的長方形，才能將其跟像素網格對齊。

往上或者往下平移一個像素點 —— 你就輸了。給它變高或者變矮一個像素點 —— 你也輸了。

想畫一條像素完美的直線？除非你將 7/12 寬度的像素設定為線條寬度，否則不可能。更糟糕的是，每個一個像素點的線條都根據其所處的豎直位置而看起來完全不同：

所以當我們發現現代化的圖標主要都是由單像素寬度的筆畫構成的時候就完全不應該驚訝：

上：2 倍縮放；下：同樣 2 倍縮放並按照 12/7 比例縮小

很難想象為什麼會有人故意想把界面渲染成這樣子。（我也不知道為什麼右下角的像素點都沒顯示。）

那文本呢？情況也不太樂觀。首先系統會將文本以兩倍分辨率渲染得每個像素都非常清晰，之後又會將其按照 85.7142857143…% 比例進行縮小，使得其能夠跟真實的物理像素塊對齊：

12px 大小的 Monoid。上：2× 放大，下：同樣按照 12/7 比例縮小

是的，系統 UI 都不是按照這個奇奇怪怪的目標分辨率進行渲染的。每個 Mac 應用都以為自己是按照 2 倍分辨率進行渲染，直到系統直接把它縮小到目標分辨率。這兩步的放大和縮小過程中，我們會損失非常多的精度和細微差別。

在我看來，沒有什麼操作對 UI 渲染的傷害能比這大了。就算是老式的低分辨率 UI 都比這樣子渲染強，至少它們的線條跟像素點對得齊！

另外不要忘記：這樣的渲染設置是默認的！每一臺 MacBook 出廠就被設置成這樣的渲染策略。數百萬的人們都直接這樣使用 MacBook 而完全不知道自己被奪走了視網膜屏幕的快樂。

對我們來說，幸運的是，這個事情（暫時）還是比較好處理的。在 macOS 的「系統設置 - 顯示器」裡面，取消勾選「默認」並選擇二倍分辨率的選項：

這樣會將屏幕上所有東西都變得稍微大一些，導致你的屏幕顯示內容變得（稍微！）小一點。當然這是預期之中的事情。我個人覺得，一個筆記本按照其產品定義，本身就是在一個受限的環境下使用的。多餘的 15% 的顯示面積並不會將其直接神奇的變成超大的桌面環境。但是至少我們還是可以享受美麗的屏幕和像素清晰的字體。否則你為什麼還要特意去購買視網膜屏幕對吧？

Windows 上的 ClearType 技術

前面我們說了這麼多關於 ClearType 的弊端，以及為什麼 ClearType 是一個對於低分辨率屏幕必須的技術，那麼我們應該為了換上 4K 顯示器而關掉它嗎？理論上來說，應該關掉。但是實際上，我們不該關。

首先，Windows 根本就沒有關閉 ClearType 的選項。我是說，你看這個選項框：

但是即使你在這裡取消勾選，你還是需要走一遍設置 ClearType 的流程。這裡根本沒有「確定」的按鈕。¯\_(ツ)_/¯

而且如果你用這個方法關掉 ClearType，那麼有些地方確實會停止使用 ClearType，但是另外一些地方並不會發生改變。我猜那些地方使用的 API 是另一套 API，部分 API 按照這裡的設定選擇使用 ClearType 與否，而另外的地方則不管這裡的設定。

同時最重要的是，如果我們不使用 ClearType 渲染文本內容，那文本看起來很糟。而它本不該是這樣的（macOS 上渲染的就非常好），在 Windows 上沒有 ClearType 尤其無法忍受。我猜 Windows 的人根本不把關閉 ClearType 作為一個選項進行測試：

買個顯示器

首先我來表達我的觀點。畢竟這是我的博客。我個人覺得筆記本並不適合開發工作。它們機動性和便捷性很好，對於某些人來說，這種說法確實可能勝過其他一切理由，我也接受這個觀點。但是即使是這樣，一個桌面顯示器 + 一個外接鍵盤的組合永遠都比單一臺筆記本電腦好。你可能有其他不購買顯示器的理由，但是我希望沒有人會辯解一個有顯示器的環境確實是一個更好的開發環境。

好了，現在我們解釋完為什麼需要買顯示器，接下來的問題就立刻出現：我們應該買哪款顯示器？根據我們之前討論的內容，下面的兩件事情需要我們認識清楚：

  • 至少應該是 4K 顯示器。5K 和 6K 顯示器當然更好 1 ；
  • 必須使用整數倍的縮放比例。

也就是說，如果你有一臺 4K 的顯示器（3840×2160），然後使用兩倍縮放比例，那麼你得到的就是 1920 × 1080 的等價邏輯分辨率。所以，從屏幕顯示內容的角度來說，你基本上就得到的是一臺 1080p 的屏幕，但是顯示的 UI 和文本都將變得更加清晰。

其實，你可能非常想要去使用比如 1.5 倍縮放等類似的顯示比例。這樣的話，你機會得到一個 2560×1440 的等價邏輯分辨率。你可能會覺得這樣做會更好，但是事實上你不應該這樣做！4K 顯示器的理念不是單純的為了堆砌像素，而是為了得到一個像素完美且密集的 UI 渲染效果，否則一個普通的 1440p 顯示器可能會表現得更好。需要記得一個簡單的規則：像素排列比其他所有因素都重要。1440p 分辨率的顯示器顯示 1440p 的內容比 2160p 分辨率的顯示器顯示同樣的內容效果好。

另外，其實直接讓 4K 顯示器顯示原生的 3840×2160 分辨率也是可以的。當然，這一點取決於顯示器的大小，但是以我的經驗來說，即使是 27 英寸的顯示器以 1× 的比例顯示內容也太小了。UI 將會被渲染的非常小。

關於 Apple 專利保護的 PPI 的神話

有一些文章 暗示所有的 Apple 電腦都應該只用 220 PPI 的顯示器進行配套使用，因為這一數據正是 Apple 自己的所有 MacBook 和 iMac 顯示屏幕的顯示密度。有時候有些人還甚至會說 macOS 完全無法搭配其他不同 PPI 的屏幕進行使用。

我覺得是這樣的。PPI 定義的是屏幕的物理像素大小（220 PPI 表示每英寸有 200 個像素，或者一個像素的大小是 1/220 英寸寬），所以 Apple 保證其產品線上的所有設備都有同樣的像素尺寸。那這意味著 macOS 控件的所有的物理大小都相同嗎？在 MacBook 開始默認使用非整數的縮放比例之後就已經不是這樣的情況了。

但是，其實我們幾乎不可能保證用戶看到的控件都是一致大小的，因為每個用戶到屏幕的距離都是不一樣的。比如，平均下來我到筆記本電腦屏幕的距離是 33cm，但是到桌面顯示器距離則是 68cm。這裡有足足兩倍的差距啊！

這意味著 1/220 的 MacBook 像素角直徑是和 1/110 的桌面顯示器像素等價的。也就是說我在一個 4K 27 英寸的桌面顯示器上看到的像素尺寸甚至要比 15 英寸 MacBook Pro 上面看到的像素尺寸小！

Apple 自己就意識到這個問題了！它們的 iPhone 的顯示屏幕擁有比 MacBook 更高的 PPI，正是因為用戶看 iPhone 時離屏幕的距離往往要比離 MacBook 近。

總體來說，我覺得 24 英寸 4K 顯示器或者甚至 27 英寸顯示器都沒什麼問題。這兩種屏幕都有用來跟 macOS 搭配使用，而且我個人都非常喜歡，沒有遇到什麼問題。當然，5K 或者 6K 顯示器是更好的，但是這些選項就是屬於 nice-to-have 類別的了。4K 是必須的，也應該是任何跟文本打交道的人所選擇的最低標準。

上 120Hz 高刷新率

顯示器世界曾經劃分為兩大陣營：高分辨率陣營和高刷新率陣營。前者適合文字工作，而後者適合遊戲，這其中卻缺少過渡的中間陣營。如果你喜歡玩競技性的遊戲，那兩種都得買（還得買個大桌子）。遊戲玩家沒理由直接購買 4K 的顯示器，因為沒有合理的遊戲會以 4K 120Hz 絲滑運行，然後專業創造者們也沒有理由為了照片後期、文章撰寫而購買 120Hz 高刷新的顯示器。我從 2014 年開始就堅守高分辨率陣營，當然，我也肯定不會為了幾乎注意不到的刷新率升級而去換掉我的視網膜屏幕。

HP Z27 (4k) v. LG 34GL750-B (120 Hz)

現在呢，這一割裂已經不存在了！不久之前我們就可以直接買到二者兼得的顯示器了！我們可以購買 4K 的顯示器並讓其以 120Hz 的刷新率進行顯示。事實上，我寫這篇文章的主要動機也是因為我發現可以購買到同時支持 120Hz 高刷新率的 4K 高分辨率顯示器。

為什麼是 120Hz？

如果你像我一樣，跟文字打交道，那麼你可能會覺得自己升級 120Hz 完全沒必要。確實，120Hz 高刷新率這個東西是一個 nice-to-have 的特性，但是如果你希望尋找提升自己體驗的方法，那麼上高刷新率絕對是一個非常好的辦法。

120Hz 會給你帶來這樣的幾個非常明顯的提升：

  • 動畫特效會變得更加絲滑，甚至看起來會是一整段完整的動效而不是一個非常快的幻燈片放映的感覺；
  • 特別的，滾動的感覺會變得非常非常順暢，比如在瀏覽器裡瀏覽、代碼編輯器裡面滾動效果的提升都很明顯；
  • 整個系統的反饋都變得非常及時；
  • 你可以用同一塊顯示器來玩遊戲和工作。

我當然是沒辦法直接展示給你 120Hz 刷新率的感受，但是你可以用這個辦法去自己體驗一下：把當前顯示器的刷新率調整為 30Hz 然後去嘗試用這個設置工作一段時間。

你會發現 30Hz 刷新率下，所有的動畫都變得非常糟糕，整個系統的反應也變得異常遲鈍。這是因為 30Hz 下顯示器更新內容的時間間隔變為 32ms，與之相應的 60Hz 下時間間隔則是 16ms。這也意味著每當你在電腦上執行任何操作（比如按按鈕、動鼠標等），顯示器顯示你的操作的延遲將可能會長達 32ms。

32ms 是一段挺長的時間，而且也非常容易被人眼發現。在 60Hz 刷新率下這段時間被砍掉一半：你所需要等待的最長時間是 16ms。在 120Hz 刷新率下這段時間在此基礎上再次被砍掉一半：從 16ms 降低至 8ms。從絕對數字的角度來說，你失去了整整 8ms，也就意味著從 60Hz 升級為 120Hz 的體驗提升大概是從 30Hz 升級為 60Hz 的體驗提升的一半。儘管如此，我覺得這個升級還是值得的。

需要注意的事情（macOS）—— 續篇

如果剛剛的事情還不夠，那這裡還有更多！

首先，你的 MacBook 必須得有一張獨立顯卡（我覺得）3 ，各種 Intel UHD / Iris 核顯都不能用，eGPU 能用。

但是即使你有合適的 Mac，你的 Mac 有合適的端口和合適的線纜，這也不夠。每次我打開 Mac 之後，我都必須經過一系列操作才能把我的顯示器切換到 120Hz 模式：

  • 完整重啟進入 macOS，此時顯示模式默認設置為 60Hz；
  • 進入「系統設置 » 顯示」；
  • 按住「Alt / Option」按鍵時（⌥ 按鍵），選擇分辨率下的「縮放」；

  • 為了能進入設置屏幕刷新率設置的選項裡面，勾選「顯示低分辨率模式」的選項；

  • 點擊「刷新率」，大多數情況這裡面只有最高 60Hz 的選項；

  • 把顯示器關閉；
  • 等一下（幾秒）；
  • 再把顯示器打開；
  • 再去看「刷新率」的選項，如果順利那麼這裡就會出現 119.88 Hertz 的選項；

  • 在「刷新率」選項裡面選擇「119.88 Hertz」；
  • 搞定收工！

這裡為什麼是 119.88Hz 而不是 120Hz 呢？我也不知道，但看起來工作方式都是一致的。為什麼 macOS 不能記住這一選項呢？我也不知道。為什麼我得把顯示器關閉再打開才能讓 macOS 識別到 120Hz 呢？我也不知道！最重要的問題是，這個 120Hz 的選項不一定總會出現，但是經過這裡的一系列操作之後，它可能會識別到 120Hz 的選項，而且如果一旦識別到了，那大概率設置上就好了。

這整個情況讓我回想起 2014 年買 4K 顯示器的情形：僅有一款型號支持，端口令人迷惑，Apple 對其支持真的糟糕。希望五年之後，120Hz 的顯示器會成為標準。直到那時候，我們都應該感激 Apple 讓我們雖然極為不方便，但是至少能給 macOS 用上現代顯示器。謝謝 Apple！

下一步是什麼？

每個人都應該有個夢想。在未來的某個時刻，4K 120Hz 顯示器將成為大眾消費品，而且我們甚至會看到 5K 120Hz 或者更多。我們甚至可能可以看到 21:9 顯示比例的視網膜屏幕 2 或者 32:9 的顯示器（更多的橫向空間），這些也當然永遠都是更好的選項。

不過就算是今天，我們也可以向未來看。如果我們有 $4000 閒錢可以花的話，我們完全可以直接購買這款世界上第一臺也是唯一一臺 8K 顯示器 —— Dell UP3218K：

就算是戴爾的 8K 顯示器宣傳頁面，也只放了張 1 倍縮放的圖

這臺顯示器的像素密度細緻的（280 PPI）以至於最好使用 300% 縮放比例才能日常使用（而這一縮放比例 macOS 不支持，Windows 才支持）。另外，這臺顯示器需要兩根 DisplayPort 線纜並列連接才能用，當然，這一點 Mac 也是不支持的。

但是即使是 300% 的縮放比例我們得到的邏輯分辨率也足足有 2560×1440，這也比目前主流 4K 顯示器的等效 1920×1080 要多很多。超多的像素密度和超大的分辨率！害，做做夢好了。

結論

總結一下，下面這些是對於一個開發者來說比較合理優秀的配置：

  • 文字在低分辨率顯示器上就無法顯示好；
  • 高 PPI 顯示器現在已經是大眾消費品了，是時候更新換代了；
  • 筆記本是可以，但是獨立桌面顯示器永遠是更好的選擇；
  • 4K 顯示器只有在 2 倍 / 200% 縮放下才比較合理；
  • 如果你想更進一步，那現在可以買一些價格合理的 4K 120Hz 顯示器。
