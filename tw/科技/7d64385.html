故障處理工具

其實java已經給我們提供了很多的故障處理工具，只是我們沒有使用而已，我們一起來看看java都給我提供了哪些故障工具

這是jdk1.8 bin目錄，如果有人使用過這裡面的工具的話就知道，jps和jinfo都是java提供給我們工具。

注意：如果需要在jdk1.5的虛擬機上田間監控功能，需要在程序啟動的時候添加參數：`-Dcom.sum.management.jmxremote`，否則大部分的工具都無法使用，如果jdk在1.6或以上的話，JMX管理是默認開啟de.suoyi不要在啟動的時候添加參數。

jps（JVM Process Status）

jps：虛擬機進程狀況工具

他的功能和ps命令有點類似，可以展示出正在運行的虛擬機進程，並展示虛擬機執行的主類（Main函數所在的位置）名稱以及這些進程的本地虛擬機唯一ID（LVMID），他是日常開發中最長使用的命令之一，當本地虛擬機唯一ID是非常重要的，我們需要通過它來確定我們需要監控的是哪一個虛擬機進程。對於本地來說，虛擬機進程與操作系統的進程是一致的，如果同時啟動了多個虛擬機進程，無法通過進程名來定位時，就需要通過jps命令來展示主類才能區分。

好了，我們一起來看看jps的基本語法。

參數介紹




這個工具的命令還是很簡單的，卻也是最常使用的一個。

jstat(JVM Statistics Monitoring Tool)

jstat位於java的bin目錄下，主要利用JVM內建的指令對Java應用程序的資源和性能進行實時的命令行的監控，包括了對Heap size和垃圾回收狀況的監控。

格式：jstat [option vmid [ interval [ s | ms ] [ count ] ] ]

如果是本地虛擬機進程，vmid就是虛擬機ID（LVIMID），如果是遠程虛擬機，那VMID的格式應該是：

[protocol : ] [ // ] lvmid [@hostname:[:port] /servername]

interval：查詢的時間間隔。

count：查詢的次數。

比如下面這行命令

jstat -gc 27088 1000 5

表示一秒查詢一次進程27088的垃圾收集情況，一共執行5次。

原想option表示查詢的虛擬機信息，一共分為三種：`類加載`、`垃圾收集`、`運行期編譯狀況`，參數詳情如下：



這裡隨便查詢一條信息

jstat -class 5276

jinfo：配置信息工具

jinfo的作用是實時查看和調整虛擬機的各項參數，其實上面說的jps -v也能查詢到jvm的參數列表，但是它僅僅只能查到顯式指定的參數列表，而jinfo可以查詢到違背指定的顯式參數列表，如果我們需要查看系統的默認值，我們可以通過jinfo的flag查詢。

jinfo的基本格式

jinfo -[option] pid

比如我要查詢某個進程jvm的參數，可以這樣寫

jinfo -flags 13012



查詢java的系統參數

jinfo -sysprops 13012

這些都是查詢，剛才說到jinfo可以實時修改參數，怎麼修改呢》

jinfo -flag <name> pid：輸出指定JVM的參數值

jinfo -flag [+|-]<name> pid：設置指定JVM參數的布爾值

jinfo -flag <name>=<value> pid：設置指定JVM參數的值


如果我現在想設置Xms

jinfo -flag  Xms=64m  13012


我靠，居然報錯了，提示Xms不能被修改，好吧，可能是windows平臺的限制，大家可以使用linux試試。


jmap：內存映像工具

jmap命令用於生成堆轉儲快照（一般稱為heapdump轉dump文件）。

當然了，jmap的作用並不僅僅是為了獲取堆轉儲的快照，它還可以查詢finalize執行隊列、java堆和方法區的詳細信息，如空間使用率、當前使用的是哪種收集器等。

jmap有些功能在windows平臺是受限制的，所以在使用的時候推薦linux。

jmap使用格式

jmap [option] vmid

vmid：上面講過了，可以理解為id。


jstack：堆棧跟蹤工具

jstack命令用於生成虛擬機當前時刻的線程快照。線程快照就是當前虛擬機內每一條線程正在執行的方法堆棧集合，生成快照的目的通常時定位線程出現長時間停頓的原因，比如線程之間的死鎖、死循環、請求外部資源時間過長等等，都是導致線程長時間卡頓的原因之一，我們可以在線程卡頓的時候通過jstack來查看各個線程的調用堆棧，就可以知道沒有響應的線程在後臺幹了啥見不得人的壞事。

使用格式

jstack [option] vmid

選項作用-F將正常輸出的請求不被響應時，強制輸出線程堆棧。-l除堆棧外，顯示關於鎖的附加信息-m如果調用本地方法的話，可以顯示c/c++的堆棧

比如：

jstack -l 13012
