基於企業架構的端到端業務流程管理理念及實踐

2020-11-14
| 科技

新一代信息技術與航空業務的融合已經成為航空公司贏得未來競爭優勢的關鍵途徑，讓 “IT 引領業務發展”不再是一句空話。隨著東航公司戰略轉型及業務發展，如何有效地管理、 優化業務流程，如何科學地、合理地提升業務運行品質成為 IT 面臨的重要課題。實踐證明， 構建“端到端業務流程”管理體系是一種有效地流程管理及優化的方法，在夯實公司信息化 建設的同時，為公司挖掘出更多的業務價值。

一、 業務流程與“企業架構”

所謂“業務流程”是為達到特定的價值目標而由不同的人分別共同完成的一系列活動， 是 IT 系統建設的基本實現單元，特別是東航這樣超大規模的傳統企業，業務流程像毛細血 管一樣密集、複雜，遍佈於公司生產運營的各個環節中。 在 IT 建設初期，大多數人對業務流程的理解是狹義的、局部的，普遍認為只要通過大 量 IT 系統建設，實現了某領域某些業務流程的線上操作，用戶滿意了，項目成功了，就會 給企業創造了業務價值。

但伴隨著近幾年東航同國際先進航空公司達美公司全方位對標，公司戰略也在不斷調整，公司信息化規模也日益龐大，東航 IT 在認真思考著一些問題：

1. 東航到底有多少業務流程？ 還有多少業務流程未實現信息化？是不是所有業務流 程都實現信息化了，就可以達到了公司戰略要求？

2. 需不需要將東航所有業務流程作為一種資產管理起來？

3. 各業務領域之間是否可以實現無縫銜接，協調運作？

4. 各生產領域業務運行品質如何？業務流程是否需要優化？

5. 不同領域類似業務流程是否具有共性？類似業務流程是否在應用系統中固化？ 僅僅站在某些領域或某個部門的角度，即便是所覆蓋的業務流程已經相當成熟、邏輯非 常清晰，也很難從業務流程層面看到公司的業務全貌，同樣也很難回答上述問題。

在 TOGAF 企業架構思想引入東航之後，將“業務架構”的企業架構理念不斷融入到 IT 生產過程中，基於公司業務需求，建立覆蓋公司各單位、各條線的總體業務架構，採用 IT 語言，定義符合公司發展運營的信息化總體業務視圖，構建“端到端”業務流程管理體系， 可以實現業務流程與業務戰略的一體化，為公司精細化管理及信息化建設提供決策依據。

二、 構建東航“端到端”業務管理體系

針對民航業務的複雜性，結構化的業務剖析顯的尤為關鍵，在具體業務流程管理上，一 定要講究策略和方法，才能保證業務流程與業務戰略的一致性。在“端到端”業務流程管理體系中，任何一條業務流程，不論是領域內的，還是跨領域的，均應是一個“端到端”的過 程，即一條完整的業務流程應是從“業務發起” 至 “業務終結” 的全過程，涉及各級職 能單位、不同崗位角色人員的參與，其中信息的流轉，保證了業務上的連續性，其本質上實 現了業務流與信息流的端到端整合。 作為一個涉及高端服務和高技術需求的企業，東航有著自身鮮明的業務特點，構建成熟、 穩定的“端到端”業務流程管理體系，有利於促進公司業務健康發展。緊扣 TOGAF 企業架 構開發方法論（ADM），東航“端到端”業務管理模型包含三層含義：

第 1，“端到端”業務流程管理理念，是結合東航業務特點及公司業務規劃，參考 TOGAF 業務架構層次參考模型，逐步形成從業務戰略、業務領域、業務能力到業務流程的 4 層業務 架構管理體系，從中體現出業務流程分層、分類和關聯關係，保證了公司業務頂層設計和規 劃。

第 2，“端到端”的業務流程管理體系，更加強調業務流程是一個“端到端”的有序銜 接，保證了業務流程間的關聯性，消除了“業務孤島”，加強了各單位間或各單位內業務流 程間的協作能力。

第 3 ，“端到端”的業務流程管理體系，通過設置“業務檢查點”的自測模式，對“端 到端”的業務閉環進行檢查，及時測評各業務環節的健康度及運行品質，及時優化業務流程， 調整企業管理策略及 IT 規劃路線。

三、 東航業務流程管理實踐 下水方知水深淺，探索真理靠實踐。

東航 IT 以“端到端”業務流程管理理念為指導思 路，深刻分析東航自身的業務環境和業務特點，實踐出“理清楚、管起來、持續優化”的落 地步驟，最終實現了對東航業務流程可量化、可分析、可優化的管控目的。

由信息部規劃管理部架構組牽頭，在各產品中心領域架構師（DA）的共同參與 下，對各業務部門的所有業務板塊進行詳細的分析和整理，並依據東航“端到端” 業務流程管理策略，對業務流程逐條檢查、審核、確認、歸檔，最終將公司整體業 務設計成 7*4 的矩陣結構，即 7 個頂級業務分類，4 層業務架構。且截至 2016 年 6 月份，經統計分析，共梳理出 32 個子領域、232 項業務能力、1140 條業務流程。 如下圖表:

2、 管起來

將業務流程梳理的成果作為企業固定資產統一管理起來,是為了統一流程管理 標準，規範流程基本要素，集中反映和逐級落實公司總體業務管控要求，達到流程 統一，高效利用的目標。東航 IT 通過以下措施實現業務流程管理：

1) 由架構組負責統一業務流程發佈、變更、版本管理

2) 採用 EPROS 工具與架構開放平臺對業務流程進行設計和管理

3) 全員（各產品中心）參與業務流程更新工作，由架構組負責規範性審核、 歸檔

4) 發佈《東航業務流程管理辦法》（計劃中），各產品中心遵照執行

3、 持續優化

在實現業務流程統一管理後，並不意味著業務流程梳理工作就結束了，因公司 戰略在不斷調整，業務在持續發展、創新，業務流程管理不是“一次性革命”，而需 追求長治久安，持續改進。因此，為保證業務流程的持續優化和落實，東航 IT 需要 定期更新和維護業務流程梳理的成果，並對其進行常態化、動態化管理。除日常維 護外，東航 IT 當前以 2 次/年的頻度進行大規模的版本控制，進行業務變更情況（新 增、刪除、細化、更新、合併等）統計分析，並保證每次調整結果及時反映到生產 中，保持業務規劃與 IT 建設同步，使業務與 IT 得到進一步融合。

四、 東航業務流程管理的價值 “

戰略執行，流程落地”，業務戰略只有落實到具體運轉的業務流程中才算是進入了執行 階段。也只有在執行階段，才會發現，整個公司的戰略舉措及目標的價值，其實就是在層層 業務流程實現過程中創造的。 東航 IT 充分利用業務流程梳理成果，逐步形成從 IT 管控、運營生態圈閉環、業務檢查 點識別、業務報表分析到運行品質監控的業務價值鏈條，並從中挖掘出極大的使用價值。

1、 理清業務流程，支撐 IT 應用規劃

東航 IT 充分利用業務流程梳理成果，實現公司全部業務流程分類和分級的結構化管理， 構建出各領域各級業務架構體系，並以業務流程基本要素為起點，設計 IT 應用模塊，逐步 形成以業務為中心的 IT 規劃路線，加大東航 IT 建設與業務規劃的融合力度，切實做到了 IT 規劃、投資預算“不偏題，不跑題，合理建設”。

2、 計算業務自動化率，提高信息化水平

在業務流程梳理成果基礎上，逐條業務分析，逐條判斷是否需實現自動化（線上操作）， 通過計算業務自動化率及移動化率，直觀反映東航 IT 信息化水平，即業務自動化率越高， 信息化程度越高；移動化率越高，業務移動辦公水平越高。比如，截止 2015 年年底，東航 業務流程總數 1244 條，其中應實現自動化的流程數為 1149 條，已實現業務自動化的流程數 為 1112，業務自動化率為 96.78%；應實現業務移動化的業務流程數為 461 條，已實現業務 移動化的流程數為 424，移動化率為 91.98%。 備註：隨著各領域業務不斷優化、調整，不同統計時間口徑，自動化率及移動化率數值 會有所變化。 ※ 業務自動化率=已實現自動化業務流程數/業務流程總數 ※ 業務移動化率=已實現移動化業務流程數/應移動化的業務流程數

3、 構建運營生態圈，實現業務自查

“端到端”的業務流程架構，促成在領域間或領域內，形成業務流程閉環，通過識別業 務閉環上的關鍵業務點，設置業務檢查指標，分析出不同業務閉環之間“指標數據與外圍數 據”的依賴關係，找出業務斷層，實現業務自查，最終構建出跨領域、相互影響、相互協作 的“生產運營生態圈”。比如上圖 4-4，在東航客艙子領域業務閉環上，共識別出 16 個業務 檢查點，在每個檢查點上，定義出“及時率、完成率”等不同健康指標，再經數據分析，這 些指標數據依賴於營銷領域、運行領域、機務領域提供的數據服務，其中紅色標註的業務節 點，因未得到外圍數據支持，導致業務斷層，是檢查出的問題業務節點，後續需要改進。

4、 直觀運行品質，優化生產管理

站在企業管理角度，各級管理人員更需要關注各領域生產運營情況。東航 IT 對業務流 程清單進行橫向分析，識別出關鍵業務流程，在此基礎上，歸納總結出可以反映具體領域業 務健康度及運行品質的性能指標，並設置告警閥值，通過大屏實時監控數值變化情況，直觀 生產運營狀態，及時發現問題，及早決策，優化生產管理。比如上圖 4-5，在該管理大屏上， 可以通過實時監控進出港航班數、旅客數、不正常航班數，實現航班保障業務的高度集成， 為現場保障人員提供決策支持。 綜上所述，信息化是建立在業務流程上的信息化，業務流程是信息化的基礎，東航 IT 已成為推動業務轉型、管理變革和戰略落實的重要力量。深刻理解 TOGAF 業務架構管理精 髓，不斷完善東航“端到端”的業務流程管理體系，實現公司全部業務分類和分級結構化管 理，進一步將業務流與信息流融合，使得東航業務架構更加穩固，業務優化更加常態化，最 終為公司持續發展挖掘更多潛在業務價值。
