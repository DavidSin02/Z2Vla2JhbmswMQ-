SQL 2大觸發器類型及應用實例

2020-11-14
| 科技

觸發器複雜語法：

create trigger trigger_name on { table | view } [ with encryption ] --用於加密觸發器 { { { for | after | instead of } { [ insert ] [ , ] [ update ] } [ with append] [ not for replication] as [ { if update( column ) [ { and | or } update ( column ) ] [ ...n ] | IF ( columns_update ( ) { bitwise_operator } updated_bitmask ) { comparison_operator } column_bitmask [ ...n ] } ] sql_statement [ ...n ] } }

注：

  1. 觸發器最多可以嵌套 32 層。
  2. for和after同義。
  3. delete 觸發器不能捕獲truncate table 語句
  4. 觸發器中不允許以下 Transact-SQL 語句： alter /create/drop/load database 、disk init/resize、 load log 、reconfigure 、restore database/log 等。
  5. 觸發器也可以根據數據修改前後的表狀態，再行採取對策。一個表中的多個同類觸發器（INSERT、UPDATE 或 DELETE）允許採取多個不同的對策以響應同一個修改語句。

SQL Trigger 觸發器的分類

1.數據操作語言 (DML) 觸發器

DML Trigger 是指觸發器在數據庫中發生DML事件時所觸發的操作。

DML事件即指在表或視圖中修改數據的insert、update、delete語句。

DML觸發器包含兩種觸發器：

  1. after觸發器：執行DML事件後觸發，只能定義在表上。
  2. instead of 觸發器：不執行DML事件，僅執行觸發器本身。

2. 數據定義語言 (DDL) 觸發器

DDL Trigger 是指當服務器或數據庫中發生DDL事件時所觸發的操作。

DDL事件即指在表或索引中的create、alter、drop語句。

3. 登錄觸發器

登錄觸發器一般應用在備份登錄或註冊信息。

DML觸發器的實現使用兩個邏輯表（虛表）：deleted表和inserted表

1.兩表位置：

deleted表和inserted表，建立在數據庫服務器的內存中。

2.兩表權限：

只讀權限。

3.表結構：

和觸發器所在的數據表的結構一樣。

4.表作用：

  1. deleted表

用於存放你在操作 insert、update、delete語句前，你創建觸發器表中數據庫。

2. inserted表

用於存放你在操件insert、update、delete語句後，更新的記錄。

5.表時效：

當觸發器執行完成後，它們也就會被自動刪除。

SQL 觸發器查看

  1. 查看單個觸發器
exec sp_helptext 'TriggerName'

2.查看所有觸發器

select * from sysobjects where xtype='TR'

慎用觸發器

  1. 過多的觸發器使得數據邏輯變得複雜
  2. 數據操作比較隱含，不易進行調整修改
  3. 觸發器的功能逐漸在代碼邏輯或事務中替代實現，更符合OOP思想。
