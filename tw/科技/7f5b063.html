MySQL的字段類型之數字數據類型（必修）

2020-10-29
| 科技


精確值類型

整數類型

TINYINT (M)：

所佔內存空間：1個字節

取值範圍：-2^7~2^7-1,即 -128到127

SMALLINT(M):

所佔內存空間：2個字節

取值範圍：-2^15~2^15-1


MEDIUMINT(M):

所佔內存空間：3個字節

取值範圍：-2^23~2^23-1


INT(M):

所佔內存空間：4個字節

取值範圍：-2^31~2^31-1


BIGINT(M):

所佔內存空間：8個字節

取值範圍：-2^63~2^63-1


Unsigned(無符號)屬性修飾整數類型的意義：

相當於將區間右移，比如TINIINT，右移，起始點由-128右移到0的位置，最終範圍變成了0~255。


定點數類型DECIMAL(M,D)

取值範圍：

DECIMAL類型定義的時候可以包含一個最大有效位M和一個小數位D，也就是精度和小數位數。M的取值範圍為1~65，D的取值範圍為0~30，並且D的值不能超過M。M和D都是可選的，D的默認值為0，M的默認值為10。


注意，最大有效位=整數部分+小數部分


也就是說，整數部分的長度加上小數部分的長度就是最大有效位。

假設我們定義的為：DECIMAL(4,1),那麼表示最大有效位為4，其中小數有1位，因此整數部分有3位。所以得出這個列的取值範圍為：-999.9~999.9。


還有一個需要注意的問題：unsigned修飾DECIMAL類型的時候，是捨去了整個負數部分。

假設定義：DECIMAL(4,1) UNSIGNED,那麼它的範圍就變成了0~999.9


存儲空間：

取決於小數點左右兩側數字個數，每一側每9位數字需要4個字節，最後剩下的數字需要1~4個字節。存儲空間是小數點左右兩側數值所需存儲空間的總和。

剩下的數字需要1~4個字節具體為:

剩餘位數：字節數

0:0

1:1

2:1

3:2

4:2

5:3

6:3

7:4

8:4

9:4

即假如剩餘的數字的位數為6，則還需要3個字節來存儲。


一般涉及到金錢這種斤斤計較的東西（不允許有誤差），都用decimal。


整數類型和定點數類型的特點就是不會自動進行四捨五入，計算也是十分精確的。

浮點類型

單精度類型FLOAT(M,D)

存儲空間：4個字節

取值範圍：

最小非0值：±1.175494351E-38

最大非0值：±3.402823466E+38

雙精度類型DOUBLE(M,D)

存儲空間：8個字節

取值範圍：

最小非0值：±2.2250738585072014E-308

最大非0值：±1.7976931348623157E+308


浮點類型說明：

unsigned修飾浮點類型:

該類型的負數部分將會去掉，只保留0到正數的部分。


省略M(最大有效位)和D(小數位數)：

省略M和D的時候會根據硬件能支持的最大精度來存儲。


超過精度會四捨五入。


FLOAT(p):

這種語法也是可以的。p的取值範圍為0~53，如果p在0~24範圍內，則該列為單精度，如果p在25~53範圍內，則該列為雙精度類型。


位域類型BIT(M)

取值範圍：

0~2^m-1,1≤m≤64

存儲空間：

取決於m,大約（m+7）/8 個字節

m為一個1到64的整數，默認值為1


加入我們創建了一個表：

Create table test (b BIT(3));

插入數據的時候格式要注意，格式為b'values'，並且value只能由0和1組成：

Insert into test (b) values(b'101');

查詢的時候也要注意，BIT列查出來的數值不能打印顯示，需要加上0或者使用函數如cast(),bin(),oct()或者hex()

如： select b+0 from test;

查出來的結果為5

Select bin(b) from test;

查出來結果為101


顯示寬度M說明:

如果沒有指定顯示寬度，則會以該列最長值的長度作為顯示寬度。


我們創建表的時候經常會寫這樣的，如下：

Create table user(id int(10) zerofill,age int(3))charset utf8;

那麼int(10)中的數字10就代表顯示寬度。

它表示整數再數據庫中顯示的數字+符號的個數。


特點：

不影響數據的大小，只改變顯示。可以利用zerofill，使數字變為指定格式。

Zerofill只會在數字面前補0，因此只有無符號的類型能用。
