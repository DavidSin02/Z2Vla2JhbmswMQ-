一、8086的內存分佈

高於 1M的是高位內存區和擴展內存。80386,80486,奔騰等最高尋址4G空間。

二、8086的啟動流程

  1. 計算機上電後，CPU跳到0xFFFF0位置
  2. 跳轉F000:E05B運行BIOS
  3. BIOS 運行完，最後一步檢查0盤0道1扇區，如果這個扇區最後兩字節是0x55,0xaa，bios就把這個扇區的內容加載到0x7c00，使用語句 jmp 0:7c00。
  4. 系統進入 MBR，即運行0x7c00處程序。

下面就寫一段程序加載到7c00，讓計算機來引導看是否能啟動。

三、編譯引導程序

org 07c00h ; 告訴編譯器程序加載到7c00處mov ax, csmov ds, axmov es, axcall DispStr ; 調用顯示字符串例程jmp $ ; 無限循環,$代表當前彙編行的地址DispStr:mov ax, BootMessage    ; 把BootMessage的首地址傳給寄存器axmov bp, ax ; ES:BP = 串地址mov cx, 16 ; CX = 串長度mov ax, 01301h ; AH = 13,  AL = 01hmov bx, 000ch ; 頁號為0(BH = 0) 黑底紅字(BL = 0Ch,高亮)mov dl, 0int 10h ; 10h 號中斷retBootMessage: db "Hello, MY OS!" ; 想要開機後在屏幕上顯示的字符串times  510-($-$$) db 0 ; 填充剩下的空間，使生成的二進制代碼恰好為512字節 $$表示一個section的開始處彙編後地址dw  0xaa55 ; 結束標誌

把代碼保存到nasm的目錄，編譯：

nasm boot.asm -o boot.bin

六、調試引導程序的方法

修改上面代碼，把org 07c00h改成 org 0100h ， 就可以編譯成一個.com文件在DosBox下運行。


為便於以後的調試，下面使用nasm的預編譯宏切換org語句加載的地址：

;%define _BOOT_DEBUG_   ; 做Boot Sector時把這行註釋掉						; 啟用這行就用nasm Boot.asm -o Boot.com生成.com文件用於調試						%ifdef _BOOT_DEBUG_   org  0100h%else   org 07c00h%endifmov ax, csmov ds, axmov es, axcall DispStr ; 調用顯示字符串例程jmp $ ; 無限循環DispStr:mov ax, BootMessagemov bp, ax ; ES:BP = 串地址mov cx, 16 ; CX = 串長度mov ax, 01301h ; AH = 13,  AL = 01hmov bx, 000ch ; 頁號為0(BH = 0) 黑底紅字(BL = 0Ch,高亮)mov dl, 0int 10h ; 10h 號中斷retBootMessage: db "Hello, MY OS!" ; 想要開機後在屏幕上顯示的字符串times  510-($-$$) db 0 ; 填充剩下的空間，使生成的二進制代碼恰好為512字節dw  0xaa55 ; 結束標誌
