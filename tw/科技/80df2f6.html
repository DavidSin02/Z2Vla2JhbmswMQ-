c++基礎之「多態」

2020-10-29
| 科技

什麼是多態?

同一操作作用於不同的對象，可以有不同的解釋，產生不同的執行結果，這就是多態性。簡單的說就是基類的引用指向子類的對象。

為什麼要用多態呢?

1.封裝可以隱藏實現細節，使得代碼模塊化

2.繼承可以擴展已存在的代碼模塊(類)，代碼重用

3.解決項目中緊耦合的問題，提高程序的可擴展性

多態的實現

1.重載

靜態多態，就是重載函數，編譯時已經生成

2.虛函數

簡單地說，每一個含有虛函數(無論是其本身的，還是繼承而來的)的類都至少有一個與之對應的虛函數表，其中存放著該類所有的虛函數對應的函數指針。

struct B

{

long b;

virtual void foo(){}

virtual void bar(){}

};

其內存佈局

vptr --------> B::foo

b B::bar

struct D:public B

{

long d;

virtual void bar(){}

virtual void quz(){}

};

其內存佈局

vptr ---------->B::foo

b D::bar

d D::quz

其中:B的虛函數表中存放著B::foo 和B::bar兩個函數指針，D的虛函數表中存放的既有繼承自B的虛函數B::foo,又有重寫了基類虛函數B::bar的D::bar,還有新增的虛函數D::quz
