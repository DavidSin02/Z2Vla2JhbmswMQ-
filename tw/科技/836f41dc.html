探寶之交換機上行口查詢

2020-11-14
| 科技

第1章探寶之路

1.1寶藏是什麼

某客戶機房有臺24網口千兆以太網交換機，交換機可console口管理但無任何配置，交換機23個網口接入23臺普通PC（普通PC自動獲取IP地址）、1個網口作為上聯口，24個接口都是點亮正常狀態。如何挖取“交換機上行口”的寶藏呢？

1.2探寶之路

寶藏就在那裡，尋找寶藏的路徑卻有很多條，以下是我繪製的藏寶圖，各位看好探寶了。

1、與客戶溝通確認，PC設置地址的方式，靜態配置or自動獲取；

2、將交換機一個接口拔掉，用網線連接PC，根據實際情況設置PC的IP地址；

3、PC使用串口線口與交換機console口連接；

4、打開PC的cmd命令行，使用arp -a查詢PC的網關地址對應的mac地址；

5、在交換機命令行界面，使用查詢mac地址命令，查詢網關地址對應的mac地址從哪個接口學到的，從而可判斷交換機的上行口，即探寶成功。

1.3探寶之路演繹

按圖索驥，不如演繹來的實在。

1.3.1演繹拓撲圖


備註說明：

1、拓撲圖由1臺核心交換機、兩臺接入交換機和兩臺PC組成。

2、核心交換機配置兩個地址段192.168.20.0/24（vlan 20，用戶地址段，自動下發地址）、192.168.10.0/24（vlan 10，設備管理址段，靜態配置）。

3、JR01管理地址192.168.10.1，JR02管理地址192.168.10.2，核心管理地址192.168.10.254。


1.3.2探寶之路演繹

1、PC01自動獲取IP地址


2、使用arp -a查詢PC的網關地址對應的mac地址


3、在交換機命令行界面，使用查詢mac地址命令，查詢網關地址對應的mac地址，即探寶成功。


1.3.3設備配置說明

1.3.3.1HX交換機

#

vlan batch 10 20

#

lldp enable


dhcp enable

ip pool vlan20

gateway-list 192.168.20.254

network 192.168.20.0 mask 255.255.255.0

#

aaa

local-user admin password simple admin

local-user admin privilege level 15

local-user admin service-type telnet

#

interface Vlanif10

ip address 192.168.10.254 255.255.255.0

#

interface Vlanif20

ip address 192.168.20.254 255.255.255.0

dhcp select global


interface GigabitEthernet0/0/1

port link-type trunk

port trunk allow-pass vlan 10 20

#

interface GigabitEthernet0/0/2

port link-type trunk

port trunk allow-pass vlan 10 20

#

user-interface vty 0 4

authentication-mode aaa

第2章探寶總結

寶藏就在那裡，尋找寶藏的路徑卻有很多條，有沒有其它辦法了呢？一般問這樣的問題，答有，絕對得分。

第1種：拔線操作，具體不陳述了，大家都懂得。

第2種：查看每個接口的流量狀態，一般上行口的接口流量最大。


第3中：利用LLDP協議。LLDP協議不但能查看接口信息，而且能查看到設備信息。


第4種：交換機配置管理地址並可ping通網關地址，查看交換機的arp表項


回到最初的問題，基於此生成另外一個寶藏，“交換機可console口管理但無任何配置”，存在ARP表項嗎？歡迎一起溝通交流。
