什麼是文件？

在計算機沒有被髮明出來之前，它通常以紙質的形式存在，通過文字和筆墨將人們口述或者大腦裡的想法記錄下來的實體。


在記事本中編寫的手


文件可以是竹簡、紙張、牛羊皮等等，它記錄了人類的智慧結晶和一切生產行為，如果沒有文件存在並進行傳承，那整個人類的文明就無從談起了。

計算機和編程語言本質上就是人類的智慧結晶，高度抽象化的產物。

那麼在計算機中，文件是如何產生並存在的呢？

我們都知道計算機中有兩個東西必不可少，內存和硬盤。

簡單的說我們在計算機中產生文件其實就是將內存中的數據存儲到硬盤裡，

大家肯定在編寫文檔的時候隨時選擇保存防止數據丟失吧。

那你現在大概已經知道了當你敲擊鍵盤每次產生的數據都臨時放在內存中，直到你點擊菜單選擇保存時計算機會將內存中的數據存儲到硬盤中。


今天我們就來講講Python的文件管理、對象序列化相關的內容。

通過本章的學習我們將掌握如何讀寫文件，自定義文件內容相關知識。

知識點:

  • Python文件讀取、寫入、刪除
  • Python的對象序列化和反序列化
  • 自定義nb文件格式

Python文件讀取

讓我們先來準備一個文本文件並且寫一段代碼來讀取它。


首先我在電腦上寫了一個叫hello.txt的文本文件並保存了下來，其中有兩行內容。

f = open('hello.txt', 'r')print(f.read())

現在我們來看看程序運行的結果。


結果跟你想的一樣嗎？

通過這兩行行代碼和輸出結果我們會學到幾個知識點。

  1. Python裡通過open()內置函數打開一個文件，其中open()函數接收的第二個參數表示打開文件的模式，例如我只想讀取文件，就傳一個 "r" 字符就行了，它是英文 read 的意思。
  2. open()函數 打開文件後會得到一個 文件對象，我們定義了一個變量f來接收它。
  3. 通過文件對象的read()函數會獲取全部內容，由print() 函數打印出來。

關於文件打開模式這裡有一個表格，希望大家保存記住，便於隨時查閱。


現在我們再想想剛才的代碼，其實有個缺陷，那就是我們能不能根據自己的需求一行一行的讀取文件內容，而不是一下就把它打印了出來。

我們來看看如何讀取文件的每一行內容吧。

f = open('hello.txt', 'r')for i in f.readlines():    print(f"文件內容:{i}")


我們通過文件對象的 readlines() 函數對文件按行讀取內容，實際上readlines()函數會得到一個列表，列表裡的每個元素就是每一行的實際內容。


通過以上例子我們可以發現readlines() 函數其實會獲得一個列表對象。


不過唯一有一點疑惑的就是為什麼“第一行”後面會多一個 "\n"，而且它叫換行符，表示遇到它時就該換行了。

因為Python的print()函數在輸出內容時本來就會進行自動換行，所以我們會發現在之前的輸出結中。

文件內容：第一行文件內容：第二行

這個輸出文件內容之間多了一個換行，其實就是這個 "\n" 換行符起了作用。


Python文件寫入

瞭解了讀取文件信息，現在我們來看看怎麼用Python寫入文件信息的。

f = open('hello.txt', 'r+')for x in range(1, 10):    line_txt = f"第{x}行內容\n"    f.write(line_txt)f.close()

現在來解讀一下代碼。

  1. 我們用 "r+" 模式進行讀寫文件
  2. 生成一個可迭代對象，從1循環到10
  3. 通過文件對象的write()函數寫入文件內容
  4. 寫入文件完成後，通過文件對象的close()函數關閉文件，它的作用是關閉文件對象，不再接受新的輸入信息，這個時候如果再寫入文件內容，程序就會報錯了，需要重新用open函數打開文件才能繼續寫入。

Python文件刪除

關於如何刪除文件，在Python裡有很多種方式，在這裡我們只介紹一種。

import osos.remove(文件名)

通過引入 os 這個Pyhton自帶的庫，我們可以完成很多跟文件相關的操作，其中的remove()函數就是用來刪除文件的，當我們remove()一個不存在的文件或者文件夾時，程序就會會報錯。


Python的對象序列化

在講序列化之前先想想，我們的計算機裡除了剛才的 hello.txt 這類的txt文本文件之外是不是還有 xls、png、avi、mp3等等各種類型的文件格式？

它們是如何產生的？它們存儲的格式是怎麼樣的呢？

這些文件的存儲格式不再是文本格式，而是字節流信息，換句話說就是二進制信息。

我們知道程序在運行時向內存申請資源，然後開始計算，將計算結果暫存在內存裡的各種對象裡。對象包括圖片信息、視頻信息、電子表格信息等等。

程序將對象通過二進制的方式從內存存儲到硬盤文件的過程就叫做序列化。

本章的前言裡說到人類通過紙張將大腦裡的信息用筆通過文字錄下來，變成了文件。

那麼在計算機裡把對象序列化的這個過程其實就是人類用文字和筆記錄下來的過程。



現在我們來試試Python怎麼進行對象序列化的。

import picklef = open('object_file', 'wb')list1 = [1,2,3,4,5,6]pickle.dump(list1, f)

以上代碼我們引用到了Python自帶的庫pickle。

pickle是Python中專門用來做對象序列化操作的庫。

通過它我們可以把對象進行序列化或者反序列化，簡單的說就是把對象存儲到文件裡，或者從文件裡直接讀取出對象信息到程序裡。

而pickle的dump()函數的作用就是 將一個對象直接輸出到硬盤文件裡。

現在回到代碼裡來理解一下執行過程。

  1. 程序先是用 "wb" 的方式新建一個文件，叫做 object_file。"wb" 模式表示寫入一個二進制文件。
  2. 創建了一個1-6的列表內容
  3. 通過dump()函數把對象直接寫入到文件裡，dump()第一個參數是需要存儲的對象，第二個參數是需要存儲的文件。

對於初學者來說這就是一堆莫名其妙的符號。

那麼如何正確的解析這些數據呢？這個時候就需要用到反序列化了。


Python的對象反序列化


通過以上代碼，我們利用pickle的load() 函數進行反序列化，把剛才保存在文件裡的二進制信息又變成了列表對象了。

反序列化本身其實沒什麼好講的，它就是將文件裡的信息讀取出來並轉換為程序裡可用的對象，反序列化顧名思義就是它所做的事情和序列化的過程完全相反。

使用pickle的注意事項

在使用pickle庫進行序列化的時候要注意最好不要用它來存儲大容量的對象信息（例如1億個字符串），如果有大容量數據的需求，可以選擇用別的庫來實現，pickle庫只是應對於常見情況。

同時也要注意在load時確保來源數據文件是可信的，因為在load時Python會自動加載模塊構造對象，如果攻擊者惡意製造一個破壞系統信息的對象文件，我們去load時就會有麻煩了。
