可重入鎖ReentrantLock

2020-11-14
| 科技

0. 說明

在多線程變成的時候，當多個線程需要對臨界資源進行操作的時候，這個時候就需要通過加鎖的機制，保證在同一時間只能有一個線程操作臨界資源，從而保證數據一致性。

這裡我們先介紹下什麼叫做可重入，在Java中可重入的意思就是當前線程已經獲得了鎖，但是還可以再次獲得（當然，其他線程不能再次獲得，至於獲得什麼，本文當時是獲得鎖了，鎖也算是一種臨界資源）。

  • 不可重入鎖：當前線程已經獲得鎖了，當再次獲得鎖的時候，將獲取不到，容易出現死鎖。
  • 可重入鎖：當前線程已經獲得鎖了，當再次獲得鎖的時候，能夠獲取的到，但是在釋放的時候，需要釋放多次。


鎖分類


1. 不可重入鎖

話不多說，直接上代碼。

package cn.ideacs.learn.lock;class NoReentrantLock { private boolean lock; public Boolean lock() { if (!lock) { lock = true; return lock; } else { return false; } } public Boolean unLock() { if (lock) { lock = false; return true; } else { return false; } } @Override public String toString() { return "NoReentrantLock{" + "lock=" + lock + '}'; }}public class NoReentrantLockTest { private NoReentrantLock lock; public NoReentrantLockTest(NoReentrantLock lock) { this.lock = lock; } public void methodA() { while (!lock.lock()) { System.out.println("A正在要鎖"); } methodB(); System.out.println("A假裝進行了一波騷操作"); lock.unLock(); } public void methodB() { while (!lock.lock()) { System.out.println("B正在要鎖"); } System.out.println("B假裝進行了一波騷操作"); lock.unLock(); } public static void main(String[] args) { NoReentrantLock noReentrantLock = new NoReentrantLock(); NoReentrantLockTest test = new NoReentrantLockTest(noReentrantLock); test.methodB(); System.out.println(noReentrantLock); test.methodA(); }}


運行結果：

B假裝進行了一波騷操作NoReentrantLock{lock=false}B正在要鎖B正在要鎖B正在要鎖B正在要鎖B正在要鎖...


可見B一直在試圖獲得鎖，但是卻因為是不可重入鎖，所以一直不能獲取的到。


2. 可重入鎖

一樣，話不多說，直接上代碼

package cn.ideacs.learn.lock;class MyReentrantLock { private int lock = 0; private Thread lockThread; public boolean lock() { if (lockThread == null) { lockThread = Thread.currentThread(); lock++; return true; } else if (lockThread == Thread.currentThread()) { lock++; return true; } else { return false; } } public boolean unLock() { if (lockThread == null || lock <= 0) { return false; } lock--; if (lock <= 0) { lockThread = null; } return true; } @Override public String toString() { return "MyReentrantLock{" + "lock=" + lock + ", lockThread=" + lockThread + '}'; }}public class MyReentrantLockTest { private MyReentrantLock lock; public MyReentrantLockTest(MyReentrantLock lock) { this.lock = lock; } public void methodA() { while (!lock.lock()) { System.out.println("A正在要鎖"); } System.out.println("methodA:" + lock); methodB(); System.out.println("A假裝進行了一波騷操作"); lock.unLock(); System.out.println("methodA unLock:" + lock); } public void methodB() { while (!lock.lock()) { System.out.println("B正在要鎖"); } System.out.println("methodB:" + lock); System.out.println("B假裝進行了一波騷操作"); lock.unLock(); System.out.println("methodB unLock:" + lock); }  public static void main(String[] args) { MyReentrantLock lock = new MyReentrantLock(); MyReentrantLockTest test = new MyReentrantLockTest(lock); test.methodB(); System.out.println(lock); test.methodA(); }}


運行結果：


methodB:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}B假裝進行了一波騷操作methodB unLock:MyReentrantLock{lock=0, lockThread=null}MyReentrantLock{lock=0, lockThread=null}methodA:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}methodB:MyReentrantLock{lock=2, lockThread=Thread[main,5,main]}B假裝進行了一波騷操作methodB unLock:MyReentrantLock{lock=1, lockThread=Thread[main,5,main]}A假裝進行了一波騷操作methodA unLock:MyReentrantLock{lock=0, lockThread=null}


可見，沒有產生死鎖。


3. 總結

本文只是介紹可重入鎖的概念，以及簡單的實現，可重入鎖，主要有可能會又在循環調用或者遞歸的時候，這裡面還是有很多問題的，其實都是線程安全問題，在JDK的ReentrantLock中會有更全面的考慮，以後我們在做分析。希望本文對你理解可重入鎖有幫助。
