1---多態概述

某一個事物，在不同時刻表現出來的不同狀態。

舉例：

貓可以是貓的類型。貓 m = new 貓();

同時貓也是動物的一種，也可以把貓稱為動物。

動物 d = new 貓();

在舉一個例子：水在不同時刻的狀態

多態前提和體現

有繼承關係

有方法重寫

有父類引用指向子類對象

父 f =new 子；

案例-1

  1. /*
  2. 多態基礎案例
  3. */
  4. class Fa{
  5. public void show(){
  6. System.out.println("show Fa");
  7. }
  8. }
  9. class Zi extends Fa{
  10. public void show(){
  11. System.out.println("show Zi");
  12. }
  13. }
  14. class DuotaiDemo{
  15. public static void main(String[] args){
  16. Fa f = new Zi();
  17. f.show();
  18. }
  19. }

這就是多態。

3---多態的好處和弊端

多態的好處：

A:提高代碼的維護性(繼承體現)

B:提高代碼的擴展性(多態體現)

案例-5

  1. /*
  2. 多態的好處：
  3. 提高代碼的擴展性
  4. */
  5. class Animal{
  6. public void eat(){
  7. System.out.println("吃飯");
  8. }
  9. public void sleep(){
  10. System.out.println("睡覺");
  11. }
  12. }
  13. class Cat extends Animal{
  14. public void eat(){
  15. System.out.println("貓吃飯");
  16. }
  17. public void sleep(){
  18. System.out.println("貓睡覺");
  19. }
  20. }
  21. class DuotaiDemo4{
  22. public static void main(String[] args){
  23. Cat c = new Cat();
  24. c.eat();
  25. c.sleep();
  26. System.out.println("-------------");
  27. Cat c1 = new Cat();
  28. c1.eat();
  29. c1.sleep();
  30. System.out.println("改進後");
  31. System.out.println("-------------");
  32. //這就是擴展性
  33. useCat(c);
  34. useCat(c1);
  35. }
  36. public static void useCat(Cat c){
  37. c.eat();
  38. c.sleep();
  39. }
  41. }

多態的弊端：

父不能使用子的特有功能。

案例-2 24行已演示過 ，這裡提供解決辦法

案例-6

  1. /*
  2. 解決不能訪問子類特有功能
  3. 那麼我們如何才能訪問子類的特有功能呢?
  4. */
  5. class Fu{
  6. public void show(){
  7. System.out.println("show fu");
  8. }
  9. }
  10. class Zi extends Fu{
  11. public void show(){
  12. System.out.println("show zi");
  13. }
  14. public void sleep(){
  15. System.out.println("sleep zi");
  16. }
  17. }
  18. class DuotaiDemo5{
  19. public static void main(String[] args){
  20. Fu f = new Zi();
  21. f.show(); //打印的為子類的方法 所以運行看右
  22. System.out.println("--------------");
  23. //第一種解決辦法 創建子類對象（這種方法太佔用內存空間）
  24. Zi z = new Zi();
  25. z.sleep();
  26. //第二種解決辦法 將父類引用賦值給子類引用 （向下轉型）
  27. Zi z1 = (Zi)f;
  28. z1.sleep();
  29. }
  30. }
