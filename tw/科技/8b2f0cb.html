跟老廖(廖雪峰)學習區塊鏈（5）教程：公鑰和地址

2020-10-29
| 科技

公鑰

比特幣的公鑰是根據私鑰計算出來的。

私鑰本質上是一個256位整數，記作k。根據比特幣採用的ECDSA算法，可以推導出兩個256位整數，記作(x, y)，這兩個256位整數即為非壓縮格式的公鑰。

由於ECC曲線的特點，根據非壓縮格式的公鑰(x, y)的x實際上也可推算出y，但需要知道y的奇偶性，因此，可以根據(x, y)推算出x'，作為壓縮格式的公鑰。

壓縮格式的公鑰實際上只保存x這一個256位整數，但需要根據y的奇偶性在x前面添加02或03前綴，y為偶數時添加02，否則添加03，這樣，得到一個1+32=33字節的壓縮格式的公鑰數據，記作x'。

注意壓縮格式的公鑰和非壓縮格式的公鑰是可以互相轉換的，但均不可反向推導出私鑰。

非壓縮格式的公鑰目前已很少使用，原因是非壓縮格式的公鑰簽名腳本數據會更長。

我們來看看如何根據私鑰推算出公鑰：

const bitcoin = require('bitcoinjs-lib');let    wif = 'KwdMAjGmerYanjeui5SHS7JkmpZvVipYvB2LJGU1ZxJwYvP98617',    ecPair = bitcoin.ECPair.fromWIF(wif); // 導入私鑰// 計算公鑰:let pubKey = ecPair.getPublicKeyBuffer(); // 返回Buffer對象console.log(pubKey.toString('hex')); // 02或03開頭的壓縮公鑰


構造出ECPair對象後，即可通過getPublicKeyBuffer()以Buffer對象返回公鑰數據。

地址

要特別注意，比特幣的地址並不是公鑰，而是公鑰的哈希，即從公鑰能推導出地址，但從地址不能反推公鑰，因為哈希函數是單向函數。

以壓縮格式的公鑰為例，從公鑰計算地址的方法是，首先對1+32=33字節的公鑰數據進行Hash160（即先計算SHA256，再計算RipeMD160），得到20字節的哈希。然後，添加0x00前綴，得到1+20=21字節數據，再計算4字節校驗碼，拼在一起，總計得到1+20+4=25字節數據：

對上述25字節數據進行Base58編碼，得到總是以1開頭的字符串，該字符串即為比特幣地址，整個過程如下：

使用JavaScript實現公鑰到地址的編碼如下：

const bitcoin = require('bitcoinjs-lib');let    publicKey = '02d0de0aaeaefad02b8bdc8a01a1b8b11c696bd3d66a2c5f10780d95b7df42645c',    ecPair = bitcoin.ECPair.fromPublicKeyBuffer(Buffer.from(publicKey, 'hex')); // 導入公鑰// 計算地址:let address = ecPair.getAddress();console.log(address); // 1開頭的地址

計算地址的時候，不必知道私鑰，可以直接從公鑰計算地址，即通過ECPair.fromPublicKeyBuffer構造一個不帶私鑰的ECPair即可計算出地址。

要注意，對非壓縮格式的公鑰和壓縮格式的公鑰進行哈希編碼得到的地址，都是以1開頭的，因此，從地址本身並無法區分出使用的是壓縮格式還是非壓縮格式的公鑰。

以1開頭的字符串地址即為比特幣收款地址，可以安全地公開給任何人。

僅提供地址並不能讓其他人得知公鑰。通常來說，公開公鑰並沒有安全風險。實際上，如果某個地址上有對應的資金，要花費該資金，就需要提供公鑰。如果某個地址的資金被花費過至少一次，該地址的公鑰實際上就公開了。

小結

比特幣的公鑰是根據私鑰由ECDSA算法推算出來的，公鑰有壓縮和非壓縮兩種表示方法，可互相轉換。

比特幣的地址是公鑰哈希的編碼，並不是公鑰本身，通過公鑰可推導出地址。

通過地址不可推導出公鑰，通過公鑰不可推導出私鑰。
