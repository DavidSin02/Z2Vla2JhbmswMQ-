springboot配置文件詳解

2020-10-29
| 科技

關於配置文件

springboot有兩種格式的配置文件，即yaml和properties，兩種配置文件在功能上完全等效。
相對於 properties而言，yaml更加簡潔明瞭，而且使用的場景也更多。除了簡潔，yaml還有另外一個特點，就是yaml中的數據是有序的，properties 中的數據是無序的
關於yaml以及properties詳細使用方法，我們稍後開貼專門說明，這裡不再贅述。

配置文件加載順序

默認情況下，springboot應用會依次從以下位置加載配置文件：

  1. 項目根目錄下的config文件夾
  2. 項目根目錄
  3. resources目錄下的config文件夾
  4. resources目錄

這四個是默認的位置，我們可以簡單驗證一下。

在項目根目錄下config文件夾下的application.yaml，設置端口為6666

server:  port: 6666

啟動項目，我們可以看到，默認端口已經是6666

自定義配置文件位置

除了上述的四個默認加載配置文件的位置，我們也可以自定義文件位置。通過

pring.config.location

即可指定加載配置文件位置

Idea調試時動態指定配置文件位置

舉個栗子，我在resources文件夾下面建立一個blog文件夾，並將 application.yaml 文件放到blog文件夾下，同時設置端口為6666

  1. 加入創建配置文件，並按照如下設置
  2. 修改Idea配置信息
spring.config.location=classpath:/blog/

一定要注意配置文件位置，最後的/不能少，否則會報錯

  1. 運行程序，我們會發現端口變成了 6666

運行jar包是動態指定配置文件位置

除了在Idea調試，我們終歸要將程序發不成jar包，那麼jar包如何加載配置文件呢？

我們是以上面的程序進行說明，此時，我們將配置文件單獨拿出來，同時將resource\blog文件夾刪掉。並通過

mvn package

將程序打包成jar包，打包後的jar包名稱為properties-0.0.1-SNAPSHOT.jar ，然後我們將jar包及application.yaml文件放到一起。

我們通過

java -jar properties-0.0.1-SNAPSHOT.jar --spring.config.location = ./

運行程序，通過控制檯，我們可以看到此時端口仍然是6666，證明已經加載我們的配置文件。

自定義配置文件名

自定義配置文件名的方式，與自定義配置文件的方式類似，同時兩個命令可以一起使用。

spring.config.name

這裡只展示一下用java -jar命令運行的方式，我把application.yaml文件改成blog.yaml，內容不變。

java -jar properties-0.0.1-SNAPSHOT.jar --spring.config.location=./ --spring.config.name=blog


可以看到，程序依然以6666端口成功啟動。

自定義配置文件加載順序

上面說了工程內的配置文件就自定義配置文件，如果自定義配置文件與默認配置文件同時存在，那麼springboot如何加載呢。

  1. 新建工程
  2. 修改application.propertes端口號改成1234
  3. package打包工程
  4. 使用java -jar 命令加載配置文件，並制定端口為4321

我們看到，程序先加載了環境變量中的配置信息
