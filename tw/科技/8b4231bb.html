懶人：使用 idea 插件 Easy Code 自定義 MybatisPlus 模板一鍵快速生成所需代碼

2020-11-14
| 科技

每一個成功人士的背後，必定曾經做出過勇敢而又孤獨的決定。

放棄不難，但堅持很酷~

之前無意中瞭解到了 idea 中的 Easy Code 插件，說是能快速生成 entity 、mapper、service、controller 等文件，避免很多簡單重複性的創建工作，大大提高 MySQL 增刪改查的開發效率。

正好今天要做對 MySQL 的增刪改查，想著試試這個插件，沒想到，特別好用，但也需要自己定製，所以就有了這篇文章，分享如何使用 idea Easy Code 插件配置 Mybatis Plus 模板來提高對 MySQL 的開發效率的。

一、idea 安裝 Easy Code 插件

安裝完成後，需要重啟 idea 生效。

二、使用 idea 連接 MySQL 數據庫

配置連接數據庫步驟：

View --> Tool Windows --> Database

然後，新建 MySQL 連接，最後如下圖所示：

連接成功後，這時候我們可以選擇其中一個表，右鍵：EasyCode --> Generate Code，來快速生成 entity 、mapper、service、controller 等文件。如下圖所示：

但是，這樣會生成挺多文件，挺多內容的，亂七八糟。有的內容我並不想要，所以我們需要配置 Easy Code 自定義宏操作模板。

三、配置 Easy Code 生成模板

點擊 File --> Settings --> Other Settings --> Easy Code --> Template Setting，如下圖所示：

我們可以新建 Group，創建宏操作來自動生成 entity 、mapper、service、controller、mapper.xml 等文件。

四、添加類型映射

點擊 File --> Settings --> Other Settings --> Easy Code --> Type Mapper，如下圖所示：

在我們生成類文件之前，我們也可以在 idea 的 Database 中的 某個表 中，右鍵：EasyCode --> Config Table，來修改字段類型和字段備註等。

五、快速生成代碼

點擊 idea 的 Database，選擇其中一個表，右鍵：EasyCode --> Generate Code，來快速生成 entity 、mapper、service、controller 等文件。如下圖所示：

其中 Package 路徑為 Application 類的根路徑。點擊 "OK"，實現代碼的快速生成。

這個 Easy Code 插件，配合著自己定義的宏操作，用的確實太爽了，解放勞動力啊。生成完代碼之後，我們只需要在其中寫業務代碼即可。
