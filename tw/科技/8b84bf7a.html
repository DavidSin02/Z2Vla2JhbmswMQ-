CC++語言21｜函數模板與類模板

2020-11-14
| 科技

C++是一種強類型語言，它要求程序中每一個對象的類型在編譯階段就能確定。一方面，這可以在很大程度上保證程序不會出現因類型問題而導致的錯誤。但另一方面，這種對類型的強力約束也限制了編碼的靈活性，並且有可能導致編碼效率的低下。泛型編程技術的引入使得兩方面得到 了很好的平衡。

C++語言中有函數重載（overload）的功用，可以讓相同功能（相同函數名）的函數處理不同的數據類型：

#include <iostream>using namespace std;int adder(int a, int b){	return a+b;}double adder(double a, double b) //重載adder(){	return a+b;}void main(){	cout<<adder(3,4)<<endl;	cout<<adder(3.3,4.4)<<endl;	 system("pause");}/*77.7*/

不過這樣還是會造成程序代碼重複（函數體重複）的問題，這時就可以使用模板。所謂模板就是以相同程序代碼處理不同數據類型的函數或類，在模板中以參數來代替數據類型（類型泛化），如此一來，就能在使用模板時，再設置數據類型來取代參數，產生一個新的函數或類。

模板根據不同的應用，分為函數模板（function template）和類模板（class tmeplate）：

1 函數模板

函數模板就是函數中的某個參數或返回值的類型是不確定的，是可變的，這些不確定的類型稱為模板參數。如果給函數模板的模板參數指定了一個具體的類型，就得到了一個可以執行的函數，這個函數稱為模板函數。函數模板可以節省程序員的工作量，若干個被處理的數據類型不同，但處理流程完全一樣的函數可以寫成一個函數模板。

函數模板是指在函數聲明之前加上模板的聲明，如此一來，就能以相同的函數，使用不同的數據類型。

函數模板的原型為：

template <數據類型參數表> 返回值類型 函數名(參數表){}template <class T, class U> U func(T a, int b, U c){;}//class也可以用typename代替

一般的寫法：

template <class 模板參數行>返回數據類型 函數名稱(參數行){ 定義函數模板 return 返回值;}template <class T>void swap(T* a, T* b) //可以是T{ ...}

實例1：

#include <cstdio>#include <cstdlib>template<class T>T add(T m, T n){	return m + n;}int main(){	float i;	i = add<float>(4.1, 7);	printf("%f\n",i);	system("pause");	return 0;}//11.100000

實例2

#include <stdio.h>#include <stdlib.h>template<typename T>T max(const T arr[], int size) {	T maxElem=arr[0];	for (int i=0;i<size;i++)		if(arr[i]>maxElem) maxElem=arr[i];		return maxElem;}int main(){	int a[] = {1,2,4,8,7};	int b = max(a,sizeof(a)/sizeof(int));	printf("最大數組元素：%d",b);	system("pause");	return 0;}//最大數組元素：8//函數模板也可顯示實例化：template int max<int>(const int[],int);

實例3：函數模板重載：

2 類模板

類模板是先聲明模板，將數據類型以模板參數取代，就可以在使用時才指定數據類型。

template <class 模板參數行>class 類名稱{ 定義類模板}

類模板可以實例化成一個類，一個類才可以實例化成一個具體的對象。

類模板的產生對象的方式：

類名稱<數據類型> 對象名稱; //聲明一般對象

或

類名稱<數據類型> 對象名稱(); //此對象名稱為對象和構造函數的合併聲明

在C++14之前的版本中，模板可以是函數模板或類模板。C++14還也可以創建變量模板。模板的一般規則，包括特化都適用於變量模板的聲明和定義。

實例：

#include <iostream>using namespace std;template <class T>class ptr{private:	T *ip;//指針public:	ptr(T *p);	~ptr();};template <class T>ptr<T>::ptr(T *p){	ip=p;//初始化	cout<<"調用指針管理類構造函數"<<endl;}template <class T>ptr<T>::~ptr(){	ip=NULL;//釋放	cout<<"指針管理類析構函數被調用"<<endl;}void main(){	int a=10;//自定義變量a	int b;	{		ptr<int> p(&b);//自定義指針類對象	}//超出作用域自動析構，釋放指針	system("pause");}/*調用指針管理類構造函數指針管理類析構函數被調用*/

3 相關問題

3.1 什麼是模板的實例化？

類模板只是個設計圖紙，不是一個真正的類。要使得類模板變成一個真正的類，必須用真正的類型名或常量替換類模板的形式參數。這個過程稱為類模板的實例化。實例化後，類模板成為了一個真正的類，可以定義這個類的對象了。

3.2 為什麼要定義模板？定義類模板有什麼好處？

有了類模板，可以將一組功能類似、存儲方式也類似的類定義成一個類模板，可以進一步減少程序員的工作量。

3.3 同樣是模板，為什麼函數模板的使用與普通的函數完全一樣，而類模板在使用時還必須被實例化？

在函數模板中，如果模板的形式參數出現在函數形式參數表中，那麼當函數調用時，編譯器可以根據函數的實際參數的類型確定模板的實際參數，然後對函數模板進行實例化。在定義類模板的對象時，無法確定模板形式參數對應的實際參數值，因此只能在程序中顯式地指出模板實際參數的值。

函數模板在調用時中，會有賦值操作，而類模板對象時，沒有涉及到這樣的賦值操作，所以要顯示指定類型，就像基本數據類型的定義或類實例化一樣，前面要有類型信息。

函數模板的類型可以通過函數調用進行推斷。

3.4 什麼時候需要用到類模板的聲明？為什麼？

一般來說，當定義一個類模板是另一個類模板的友元時必須要用到類模板的聲明。當類模板A聲明類模板B是它的友元時，編譯器必須知道有這樣的一個類模板B存在。如果類模板B的定義出現在類模板A的定義前面，則沒有問題。但如果類模板B定義在類模板A後面時，編譯器就不知道B是什麼，也無法確定類模板A中對類模板B的名是否合法，這時可以通過類模板的聲明來告訴編譯器B是一個類模板。

3.5 類模板繼承時的語法與普通的類繼承有什麼不同？

類模板繼承時，凡是涉及到基類的地方，都必須在基類名後面跟上模板的形式參數名。即：基類名<形式參數1，形式參數2，……>。

3.6 定義了一個類模板，在編譯通過後為什麼還不能確保類模板的語法是正確的？

由於類模板包含有模板參數，這些模板參數對應的實際參數在編譯時尚未確定，所以編譯器無法確定那些類型為模板參數的數據或函數的用法是否正確，只好暫時不檢查。
