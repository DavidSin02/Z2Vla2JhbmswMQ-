C#中的多態

2020-10-29
| 科技


(一)里氏替換原則(LSP)

1、在一個軟件系統中，子類對象可以替換所有使用的父類對象，且程序行為沒有變化


(二)is與as操作符

1、is 檢查對象是否與指定類型兼容


2、as 用於在兼容的類型之間執行轉換


3、一種類型轉換方式：將empls集合的元素轉換為SE對象；轉換失敗返回null，

用as操作符進行類型轉換不會產生異常，但這並不代表不需要異常處理。

(二)總結

1、用虛方法實現多態的步驟

2、子類重寫父類的虛方法

3、兩種方式

4、創建父類變量，實例化子類對象

5、把父類類型作為參數類型，該子類及子類對象作為參數傳入

6、運行時，根據實際創建的對象決定執行哪個方法

(三)為什麼需要抽象方法

1、交通工具類是抽象的概念，根本不可能實例化，Run()方法不具有實際意義！

(四)抽象類和抽象方法

1、抽象類用來列舉一個類所需要的行為

2、抽象類不明確提供具體實現方法

3、抽象類必須由其子類實現它的抽象方法（除非子類也具有抽象性）

4、由abstract聲明的方法稱為抽象方法。抽象方法只有方法簽名但沒有方法體，定義格式如下：

訪問修飾符 abstract 返回類型 方法名(參數列表);

示例：

public abstract void doing();

5、由abstract關鍵字聲明的類稱為抽象類，定義格式如下：

訪問修飾符 abstract class 類名

{

類體

}

6、說明：

(1)當類中出現一個抽象方法時，該類就必須定義為抽象類。

(2)抽象類也可以沒有抽象方法。

(3)若子類的父類是抽象類，則子類必須實現父類中的抽象方法，若不實現則該子類必須也定義為抽象類。

7、抽象類不能被實例化


8、抽象類不能是密封或者靜態的


9、說明：

(1)靜態方法不能聲明為抽象方法。

(2)private修飾的方法不能聲明為抽象方法。


10、將光標放在“：”後的父類名上，按快捷鍵Shift+Alt+F10+Enter，系統會給出抽象方法的默認實現


(五)抽象類的應用場合

1、父類提供一系列規定，約束子類的行為


2、抽象類和抽象方法實現多態性


(六)對比虛方法

1、抽象類和抽象方法

抽象類通常代表一個抽象的概念

抽象方法約束子類對象的行為

比如：抽象的動物類，具有抽象方法吃()、喝()


2、虛方法

具體類的默認實現，提供其子類擴展實現方式

比如：數據庫連接類，Connect()方法實現與SQL Server的連接，數據庫連接子類繼承並重寫它，實現與其它數據庫的連接

(七)抽象方法與虛方法

抽象方法 虛方法

用 abstract 修飾 用 virtual 修飾

不允許有方法體 要有方法體，哪怕是一個分號

必須被子類 override 可以被子類 override

只能在抽象類中 除了密封類都可以寫

(八)

抽象類中可以包括抽象方法和非抽象方法

(九)面向對象的三大特性

1、封裝：隱藏內部實現，穩定外部接口

2、繼承:子類繼承父類成員，實現代碼複用


3、多態:不同子類對同一個消息作出不同的反應
