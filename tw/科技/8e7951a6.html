面向對象編程設計的類與對象

2020-10-29
| 科技

前天在面向對象程序設計中，我們用比較通俗的語言解釋了面向對象是什麼意思。

不過，再通俗的語言在面對專業生產時，也必須進行專業的解釋，通俗的語言是讓大家便於理解，而專業的術語是在今後的程序開發中，必須掌握的。

類和對象

類是對象的來源和模板，對象是類的實例。

在面向對象編程中，一切都是對象，每個對象都有屬於它的靜態特徵（屬性）和動態特徵（行為）。對象的屬性和行為決定了，每一個對象都是獨特的。對象一定是屬於某個類。

當把一大批對象的屬性和行為總結起來，那就可以稱之為類。

定義類

定義類是用關鍵字 Class 來進行定義，用前面學過的函數的方法來進行定義。

class Animal(object):

# __init__是一個特殊方法用於在創建對象時進行初始化操作

self.name = name

創建和使用對象

創建和使用對象可以使用之前我們學過的 字符和符號

例如：# 創建動物對象並指定姓名和年齡

駝峰命名法

對於這麼長的時間裡，有關於代碼的寫法，我們一直使用的是駝峰命名法。

什麼是駝峰命名法？
