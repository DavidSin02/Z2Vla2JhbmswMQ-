介紹

前面的文章介紹了在Eclipse中如何配置和關聯Tomcat源碼，我們也已經獲得了Tomcat源碼並配置好了。

現在讓我們以HttpServlet為例來看看Eclipse的大綱視圖的用法。

大綱的英文是outline，從百度翻譯可以看到該單詞作為名詞的中文意思是：

梗概，大綱，提綱，草稿，要點，主要原則; 外形，輪廓，輪廓線，輪廓畫法，略圖（畫法）;

所以，顧名思義，大綱視圖就是用來查看某個東西的要點的。

對於Java類來說，要點就是屬性域和方法。

打開大綱視圖

如果沒有打開大綱視圖，可以通過以下方法打開：

  • 菜單欄 -> Window -> Show View -> Outline

大綱視圖內容

可以看到，大綱視圖默認顯示在主界面右側，包括標題欄、工具欄和視圖主界面：

  • 最上面是標題、最小化和最大化按鈕，大家可以嘗試點擊一下。

最小化會隱藏起來變成一根豎條放在主界面最右側；

最大化會佔據整個主界面，其他視圖變成豎條放置在兩側；

最小化和最大化之後都會有一個恢復（Restore）按鈕，點擊即可恢復。

  • 下一行是工具欄，從左到右依次是：
  1. Focus on Active Task：未知。
  2. Collapse All：將視圖中的樹狀節點全部摺疊起來。
  3. Sort：視圖中的樹狀節點按照字母順序排序，但是也先歸類再排序，大致是先把內部類（以後介紹）排序，然後是靜態屬性、靜態方法（這篇文章介紹了什麼是靜態成員）、非靜態屬性、構造方法、非靜態方法等等。
  4. Hide Fields：隱藏屬性。
  5. Hide Static Fields and Methods：隱藏靜態成員。
  6. Hide Non-Public Members：隱藏非公有成員。
  7. Hide Local Types：未知。
  8. 最右端的倒三角：其他工具按鈕。
  • 工具欄下面是視圖的主界面，列舉了Java類源碼文件的要點，主要就是類的各個成員（內部類、屬性、方法等），是按樹形結構組織的。

HttpServlet大綱視圖舉例

按照之前文章的方法：先把鼠標移到HttpServlet這個類上，（Windows系統下）按住鍵盤的Ctrl鍵（有些筆記本可能沒有這個鍵）同時摁下鼠標左鍵，這時候Eclipse的編輯器會打開另外一個標籤頁，如下圖所示：

編輯器打開了HttpServlet類的源碼，大綱視圖顯示了HttpServlet類的成員。

大綱視圖中，各個成員都有特殊的符號來表示，我嘗試著在HelloWorld這個Servlet添加了擁有不同訪問級別的屬性和方法，有些還加了static、final等修飾符，效果如下：

可以發現還是有些規律的：

  • 首先，類、接口、枚舉（enum）、註解的符號是比較大的圓圈，裡面是字母C、I、E、@。
  • 綠色圓形表示public、黃色圓角菱形表示protected、藍色三角形表示包訪問級別、紅色正方形表示private。
  • 屬性的符號最小（裡面好像是個加號+），方法的符號中等。
  • 符號右上角一般表示static、final、abstract等修飾符，直接是修飾符的首字母，構造器是字母C（取自構造器英文單詞Constructor）。
  • 類、接口、枚舉（enum）、註解的符號右下角表示的是訪問級別。
  • 符號左下角表示該成員有錯誤或警告等信息。

關聯大綱視圖和編輯器

Eclipse默認是點擊大綱視圖的某個節點，編輯器就會定位到相應的代碼。但這個功能可以關閉/打開，就是在大綱視圖的工具欄右側的下拉菜單中的：

  • Link with Editor

查看源碼

以HttpServlet為例，在上圖中可以看到它的源碼，右側是它的大綱。

首先，我們看這個類的定義：

public abstract class HttpServlet extends GenericServlet

可見它又繼承了另外一個類GenericServlet，我們可以用同樣的方法查看該類的源碼：

public abstract class GenericServlet implements Servlet, ServletConfig, java.io.Serializable

再用同樣的方法查看Servlet、ServletConfig、Serializable的源碼：

public interface Servletpublic interface ServletConfigpublic interface Serializable

總結一下就是：

  • HttpServlet和GenericServlet都是抽象類（我們後面討論）
  • Servlet、ServletConfig、Serializable都是接口（我們後面討論）
  • 它們形成繼承的層次，可以用UML中的類圖來更加直觀清晰的表示（我們後面討論）

其次，拖動編輯器的滾動條到最底端，可以看到其源碼將近1000行，沒有辦法一下子全部看完。

再次，在重要的地方都添加上了JavaDoc註釋，這就是我們把鼠標放到該類或該類的成員上會顯示其JavaDoc文檔的原因。

最後，我們看它的大綱視圖，可以看到大部分方法都是protected的方法，且名稱都類似doXXXX，參數都是HttpServletRequest和HttpServletResponse這兩個類型。

這些方法都有默認的實現，然而我們顯然要覆蓋某個方法，添加我們自己的處理HTTP請求的邏輯，然後使用HttpServletResponse生成網頁併發送。

這些方法就是用戶（開發HttpServlet子類的開發人員）需要覆蓋的方法，比如我們開發的HelloWorld這個Servlet就覆蓋了doGet方法。

從這個例子我們可以看到類的繼承/擴展機制的作用，需要功能擴展的就覆蓋父類方法或添加新方法；剩下的複用父類的方法。

總結

  • 工具的使用不必死記硬背，用的多了就記住了。
  • 大家可以多點點大綱視圖的工具欄各個按鈕，看看有什麼效果。
  • 類可以形成繼承層次。
  • 繼承/擴展機制是一種很強大的代碼複用（即消除重複）、功能擴展的技術。
