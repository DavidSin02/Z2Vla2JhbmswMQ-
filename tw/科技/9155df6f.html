第十六章 泛型

2020-11-14
| 科技

16.1 概述

我們在編寫程序時，經常遇到兩個模塊的功能非常相似，只是一個是處理int數據，另一個是處理string數據，或者其他自定義的數據類型，但我們沒有辦法，只能分別寫多個方法處理每個數據類型，因為方法的參數類型不同。有沒有一種辦法，在方法中傳入通用的數據類型，這樣不就可以合併代碼了嗎？泛型的出現就是專門解決這個問題的。

泛型的意思就是Generic,它所代表的是通用類型，可以代替任意的數據類型，使得類型參數化，從而達到了只需要一個方法就可以操作多種數據的目的。

泛型不僅僅可以實現代碼的重用，減少了裝箱和拆箱的過程。泛型還是避免損失的有效方法。

16.2 泛型定義

泛型在定義時，用一個通用的數據類型[T]來作為一個佔位符，等待在實例化時用一個實際的類型來代替。

eg:

public class Stack<T>{}//泛型類

注意：

  • T僅僅是一個佔位符，可以使用任意你所喜歡的佔位符進行替換。
  • 泛型定義和使用時，必須引用命名空間：using System.Collections.Generic;

【常用泛型類型集合】

C#常用的泛型類型有：

List<T>是泛型類中最常用的集合，可以動態的調整數據。

stack<T>棧是以先進後出的方式維護數據的集合，包含pop()和push(),從棧內壓入或移除數據。

Quenue<T>隊列是以先進先出的方式訪問數據，使用Enqueue()和Dequeue()添加數據和移除數據。

Dictionary<T>字典是以鍵值對的形式訪問數據，可以動態的調整數據。

以List集合為例，如下代碼：

16.3 泛型類

泛型類定義方式十分簡單，泛型類分為【非靜態泛型類】和【靜態泛型類】；

【非靜態泛型類】

eg:

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Collections.Generic;​namespace MSN{    public class Compare<T> where T : IComparable    {        public static T compareGeneric(T t1, T t2)        {            if (t1.CompareTo(t2) > 0)            {                return t1;            }            else            {                return t2;            }        }    }​    class MainClass    {        public static void Main(string[] args){            Console.WriteLine(Compare<int>.compareGeneric(3, 4));            Console.WriteLine(Compare<string>.compareGeneric("abc", "a"));        }    }}

泛型不僅僅可以實現代碼的重用，減少了裝箱和拆箱的過程。泛型是避免損失的有效方法。

如下所示代碼測試了使用泛型和不適用泛型的執行時間：

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Diagnostics;using System.Collections;using System.Collections.Generic;​namespace MSN{    class MainClass    {        //非泛型方法        public static void TestNonGeneric()        {            Stopwatch stopwatch = new Stopwatch();            ArrayList arrayList = new ArrayList();​            stopwatch.Start();​            for (int i = 1; i < 10000000; i++)            {                arrayList.Add(i);            }​            stopwatch.Stop();​            TimeSpan ts = stopwatch.Elapsed;​            string elapsedTime = String.Format("{0:00}:{1:00}:{2:00}.{3:00}",                ts.Hours,ts.Minutes, ts.Seconds, ts.Milliseconds / 10);            Console.WriteLine("非運行時間是" + elapsedTime);        }​        //泛型方法        public static void TestGeneric()        {            Stopwatch stopwatch = new Stopwatch();            List<int> gelist = new List<int>();​            stopwatch.Start();​            for (int i = 0; i < 10000000; i++)            {                gelist.Add(i);            }​            stopwatch.Stop();​            TimeSpan ts = stopwatch.Elapsed;​            string elapsedTime = String.Format("{0:00}:{1:00}:{2:00}.{3:00}",                ts.Hours,ts.Minutes, ts.Seconds, ts.Milliseconds / 10);​            Console.WriteLine("運行時間是" + elapsedTime);​        }​        public static void Main(string[] args){            MainClass.TestGeneric();​            MainClass.TestNonGeneric();        }    }}

從結果可以看出，向泛型數組中加入元素的效率遠高於非泛型數組，因為非泛型的Add操作中，參數為object類型，當int傳入的時候，會發生裝箱操作，從而導致性能的損失，時間變長。

【靜態泛型類】

在普通靜態類中，靜態數據類型是屬於類型的，如果定義了一個靜態字段x，則不管之後創建了多少個該類的實例，也不管派生多個實例，都只存在一個。泛型並非如此，每個封閉泛型都具有自己的靜態數據。

eg:

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Collections.Generic;​namespace MSN{    /*靜態泛型類*/    public static class GenericClass<T>    {        /*每個封閉類型的泛型類型都有一個靜態構造函數*/        static GenericClass()        {            Console.WriteLine("泛型的靜態構造函數被調用，實際類型" + typeof(T));        }        public static void Print()        {        }    }​    /*普通靜態類*/    public static class NoGenericClass    {        static NoGenericClass()        {            Console.WriteLine("靜態構造函數被調用");        }        public static void Print()        {        }    }•​    class MainClass    {        public static void Main(string[] args){            GenericClass<int>.Print();            GenericClass<string>.Print();​            NoGenericClass.Print();            NoGenericClass.Print();        }    }}

16.4 泛型方法

有時候，我們需要將泛型T的作用範圍縮小，直接將其應用在方法上實現泛型方法無疑是一種非常明智的選擇。

eg：

16.4.1 泛型類中的方法重載

方法的重載在.Net Framework中被大量應用在泛型類中，由於通用類型T在類編寫時並不確定，所以在重載時有些注意事項，這些事項我們通過以下的例子說明：

eg:

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Collections.Generic;​namespace MSN{    public class Node<T, V>​    {​        public T add(T a, V b)          //第一個add​        {            Console.WriteLine("第一個add");            return a;​        }​        public T add(V a, T b)          //第二個add​        {            Console.WriteLine("第二個add");            return b;​        }​        public int add(int a, int b)    //第三個add​        {            Console.WriteLine("第三個add");            return a + b;​        }​    }•    class MainClass    {        public static void Main(string[] args){            Node<int, int> node = new Node<int, int>();            object num = node.add(2, 11);            Console.WriteLine("num = {0}", num);        }    }}​

注：

使用如下代碼進行測試時：

Node<int, int> node = new Node<int, int>();​object num = node.add(2, 11);

Node的實例化引起了三個add方法具有同樣的簽名，但卻能調用成功。原因在於編譯器優先匹配了第三個add方法。但如果刪除了第三個add方法，上面的調用代碼則無法編譯通過，提示方法產生的混淆，因為運行時無法在第一個add方法和第二個add方法之間選擇。

Node<string, int> node = new Node<string, int>();​object x = node.add(2, "11");

這兩行調用代碼可正確編譯，因為傳入的string和int，使三個add方法具有不同的簽名，當然能找到唯一匹配的add方法。

16.5 泛型全面解析

泛型分為開放泛型和封閉泛型。其中開放類型是指包含類型參數的泛型，但是未綁定的類型；封閉類型是指已經為每一個類型參數都傳遞了數據類型。

檢測泛型類別時可使用【ContainsGenericParameters】屬性來判斷類型對象是否包含未被實際類型替代的類型參數。存在則返回true，表示為開放類型，不存在則返回false，表示為封閉類型。

eg：

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Collections.Generic;​namespace MSN{    public class DictionaryStringKey<T> : Dictionary<string, T>    {​    }​    class MainClass    {        public static void Main(string[] args){            Type t = typeof(Dictionary<,>);            Console.WriteLine("是否為開放類型：" + t.ContainsGenericParameters);​            t = typeof(DictionaryStringKey<int>);            Console.WriteLine("是否為開放類型：" + t.ContainsGenericParameters);        }    }}

16.6 泛型約束

在16.3小節實現的泛型之中，我們使用了where T:IComparable的代碼，其中where語句用來使類型繼承於IComparable接口，從而對類型參數進行約束。泛型約束共有六大類別，下面一一進行詳解。

16.6.1 基類約束

基類約束有兩個功能。首先，它允許在泛型類中使用自由約束指定的基類所定義的成員。通過提高基類約束，編譯器將知道所有的類型實參都擁有由指定的基類所定義的成員。基類約束的第二個功能是，確保只使用支持指定基類的類型實參。這意味著對於任意給定的基類約束，類型實參必須是基類本身或者是派生於該基類的類。基類約束使用如下形式的where子句：

    where T：base-class-name

其中，T是類型形參的名稱，base-class-name是基類的名稱。只能指定一個基類。

16.6.2 接口約束

接口約束是指定某個類型實參必須實現的接口。它的兩個主要功能與基類約束一樣，允許開發人員在泛型類中使用接口的成員；確保只能使用實現了特定接口的類型實參。也就是說對任何給定的接口約束，類型實參必須是接口本身或者實現了該接口的類。接口約束使用的where子句具有以下形式：

    where T:interface-name

其中，T是類型形參的名稱，interface-name是接口的名稱。可以使用逗號分隔開指定的多個接口。若某個約束同時包含基類和接口，則需先指定基類列表，再指定接口列表。

16.6.3 new()構造函數約束

new()構造函數約束允許開發人員實例化一個泛型類型的對象。new()約束要求類型實參必須提供一個無參數的公有構造函數。使用new()約束時，可以通過調用該無參數的構造函數來創建對象。new()構造函數約束的形式為：

    where T:new()

使用new()約束時應當注意3點：

1.new()約束可以與其他約束一起使用，但必須位於約束列表的末端

2.new()約束僅允許開發人員使用無參數的構造函數構造一個對象，即使同時存在其他的構造函數也是如此。即不允許給類型形參的構造函數傳遞實參。

3.不可以同時使用new()約束和值類型約束。因為值類型都隱式的提供了一個無參公共構造函數。就如同定義接口時指定訪問類型為public一樣，編譯器會報錯，因為接口一定是public的。

16.6.4 引用類型約束

引用類型約束將一個類型形參限定為引用類型。引用類型一般是用戶定義的類型，包含類、接口、委託、字符串和數組類型。引用類型約束使用class關鍵字，它的通用形式為

    where T:class

在這個where子句中，class關鍵字是指定T必須是引用類型。因此，嘗試對T使用值類型，將會導致編譯錯誤。

16.6.5 值類型約束

值類型約束將一個類型形參限定為值類型。值類型派生於System.ValueType類型。基元和結構都是值類型。值類型約束使用struct關鍵字，它的通用形式為：

    where T:struct

在該形式中，struct關鍵字指定T必須是值類型。因此，嘗試對T使用引用類型，將導致編譯錯誤。

16.6.6 組合約束

同一個類型形參可以使用多個約束。這種情況下，需要使用一個由逗號分隔的約束列表。在該列表中，第一個約束必須是引用類型約束或者值類型約束，或者是基類約束。指定引用類型約束或值類型約束的同時也指定基類約束是非法的。接下來必須是所有的接口約束，最後是new()約束。下面是一個合法的聲明：

class Test<T>where T:MyClass,Iinterface,new()

{ }

在上述聲明中，用於替換T的類型實參必須繼承MyClass類，實現Iinterface接口，並且擁有一個無參數的構造函數。

在使用兩個或者更多的類型形參時，可以使用多條where子句分別為它們指定約束。

  • where T: struct

類型參數必須為值類型。

  • where T : class

類型參數必須為類型。

  • where T : new()

類型參數必須有一個公有、無參的構造函數。當於其它約束聯合使用時,new()約束必須放在最後。

  • where T : <base class name>

類型參數必須是指定的基類型或是派生自指定的基類型。

  • where T : <interface name>

類型參數必須是指定的接口或是指定接口的實現。可以指定多個接口約束。接口約束也可以是泛型的。

16.7 泛型委託

與方法一樣，委託也可以是泛型的。因為泛型內在的類型安全性，無法將不兼容的方法賦給委託。聲明泛型委託的形式為：

delegate ret-type delegate-name<type-parameter-list>(arg-list);

類型形參的聲明緊跟在委託的名稱之後。泛型委託的優點在於，它允許開發人員以類型安全的方式定義一種通用形式，該形式可用於匹配任意兼容的方法。

// #region << 版 本 注 釋 >>// /*----------------------------------------------------------------// // Copyright (C) 2019 極客部落// // 版權所有。 // //// // 文件名：Program.cs// // 文件功能描述：// //// // // // 創建者：GeekTribe// // 時間：14:05// //----------------------------------------------------------------*/// #endregionusing System;using System.Collections.Generic;​namespace MSN{    class MainClass    {        public delegate T MyGenericDelegate<T>(T obj1, T obj2);​        int AddInt(int x, int y)        {            return x + y;        }​        string AddString(string s1, string s2)        {            return s1 + s2;        }​        public static void Main(string[] args){            MainClass p = new MainClass();                       MyGenericDelegate<int> intDel;            intDel = p.AddInt;            Console.WriteLine("int代理的值是{0}", intDel(100, 200));                           MyGenericDelegate<string> stringDel;                 stringDel = p.AddString;            Console.WriteLine("string代理的值是{0}", stringDel("Hello", "World"));         }    }}​

16.8 泛型接口

除了定義泛型類和泛型方法外，還可以定義泛型接口。泛型接口的定義與泛型類基本相同。泛型接口的聲明形式為:

interface Iinterface<T>{   }

使用泛型時，要注意一下限制：

  • extern修飾符不能用於泛型方法。
  • 屬性、運算符、索引器和事件不能泛型化。這些項仍可以用作泛型中，並且可以使用類的泛型類型形參。
  • 指針類型不能用作類型實參。
  • 如果泛型類包含一個static字段，那麼每一個構造類型都會有該字段的獨立副本。這意味著同一個構造函數類型的所有實例都會共享同一個static字段。然而，不同的構造類型使用不同的字符副本。因此，static字段並不是由所有的構造類型共享。
