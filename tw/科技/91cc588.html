什麼是堆棧？

2020-10-29
| 科技

堆與棧是兩種數據結構，並不是一種數據結構，堆是堆，棧是棧。

1、棧：是一種只能在一端進行插入和刪除的數據結構。

允許插入與刪除的一端被稱為棧頂，另一端被稱為棧底。

按照先進後出的順序存儲數據。最先進入的數據存放在棧底，最後進入的數據存放在棧頂。

當棧裡的元素為空時，稱為空棧。

在函數調用的時候用於存儲斷點，遞歸的時候，也需要用到棧。

棧一般包含以下兩個方面的信息：

(1)函數的返回地址和參數。

(2)臨時變量：包括函數的非靜態局部變量及編譯器自動生成的其他臨時變量。

在彙編語言中，棧頂由esp寄存器進行定位，壓棧的操作使得棧頂的地址減小，彈出的操作使得棧頂的地址增大。

2、堆：是一種動態的存儲結構，實際上是數據段中的自由存儲區，常用語存儲、分配動態數據。

堆中存入的數據地址向增加方向變動。

堆可以不斷的進行分配直到沒有堆空間為止，也可以隨時進行釋放、在分配，不存在順序問題。

在C語言中，堆空間的分配通常通過malloc()、calloc()、realloc()三個函數來實現的，而堆的釋放是通過free()函數來進行釋放分配的空間。

3、棧的體驗

如何通過棧將數組裡面的數據通過print函數打印出來呢？

print函數沒有參數，應該在print函數裡面填充什麼代碼，可以實現將arr數組裡面的數據全部打印出來呢？

#include <stdio.h>void print(){}int main(){ int a = 1; int b = 2; char c = 'c'; int arr[] = { 11,12,13,14,15,16,17 }; print(); return 0;}

那麼這就涉及到彙編的一些東西了，具體彙編如下：

1、push操作先移動棧頂指針，之後將信息入棧。

2、eps為棧頂指針，壓棧的操作使得棧頂的地址減小，彈出的操作使得棧頂的地址增大。

3、ebp是32位的bp，是基址指針。bp為基指針寄存器，用它可以直接存取堆棧中的數據，它在調用函數時保存esp，以便函數結束時可以正確返回。

4、默認的函數內部變量的壓棧操作為：從上到下，從左到右，採用4字節對齊，數組壓棧方法略有不同，即從最後一個元素開始，直到起始元素為止，即採用從右到左的方法壓棧。

在main()函數和print()函數的開頭都有如下兩句彙編指定：

main函數


print函數

第一個push指針的操作步驟是，首先移動棧頂指針esp，然後將ebp內容壓棧，注意此時壓棧的ebp的值為上一個函數的esp的值，而esp恰好就是上一個函數的棧底。

所以每個函數一開始push指令就是保存上一個函數的棧底。

由於esp是當前的棧頂指針，所以該指針的作用就是保存當前棧頂指針的值。

由此就可以得出，ebp存放的就是此刻棧頂的地址，也就是說，ebp是一個指針，指向棧頂，而棧頂存放的數據就是上一個函數的ebp的值，就上一個函數的棧底。

所以我們只需要在print函數中得到main函數的棧底，就可以去取出數組中的每一個元素了。

具體實現方法如下：

void print(){ unsigned int _ebp; __asm { mov _ebp, ebp; } int *p = (int *)(*(int *)_ebp - 4 - 4 - 4 - 4 * 7 ); for (int i = 0; i < 7; i++) { printf("%d\t", p[i]); }}

char類型只佔一個字節，為什麼在函數中是-4而不是減1呢？

這就涉及到字節對齊的問題，因為壓棧操作為4字節對齊，所以這裡必須是減4，而不是減1。如果你寫成減一的話，得出來的數據就是一段亂碼！
