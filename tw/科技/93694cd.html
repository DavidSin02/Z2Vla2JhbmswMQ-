STM32中斷的注意事項

2020-10-29
| 科技

歡迎訪問小編的網站：182.61.136.210:1024

  1. 高優先級的搶佔優先級是可以打斷正在進行的低搶佔優先級中斷的。
  2. 搶佔優先級相同的中斷，高響應優先級不可以打斷低響應優先級的中斷。
  3. 搶佔優先級相同的中斷，當兩個中斷同時發生的情況下，哪個響應優先級高，哪個先執行。
  4. 如果兩個中斷的搶佔優先級和響應優先級都是一樣的話，則按照中斷向量表的順序執行。

Reset、NMI、Hard Fault 優先級為負(高於普通中斷優先級)且不可調整。

PRIMASK位：只允許NMI和hard fault異常，其他中斷/異常都被屏蔽(當前CPU優先級=0)。  

FAULTMASK位：只允許NMI，其他所有中斷/異常都被屏蔽(當前CPU優先級=-1)。

在STM32固件庫中(stm32f10x_nvic.c和stm32f10x_nvic.h) 定義了四個函數操作PRIMASK位和FAULTMASK位，改變CPU的當前優先級，從而達到控制所有中斷的目的。

下面兩個函數等效於關閉總中斷：

void NVIC_SETPRIMASK(void)；

void NVIC_SETFAULTMASK(void)；

下面兩個函數等效於開放總中斷：

void NVIC_RESETPRIMASK(void)；

void NVIC_RESETFAULTMASK(void)；

上面兩組函數要成對使用，但不能交叉使用。

常常使用：

NVIC_SETPRIMASK();     // 關閉中斷

NVIC_RESETPRIMASK();  // 開啟中斷

可以用：

#define CLI()  __set_PRIMASK(1)  //關閉總中斷

#define SEI()  __set_PRIMASK(0)  //打開總中斷

來實現開關總中斷的功能。

1、一般來說單片機的中斷髮生有兩個條件，一是中斷標誌位置位，二是中斷允許，如果這兩個條件都滿足則進入中斷，因為正常情況下中斷一直是允許的，那麼只能通過標誌位來區分是否有中斷掛起，

2、如果進入中斷不清除標誌位，那麼這一中斷服務程序結束後由於標誌位還是置位的並且中斷是允許的，那麼還會再次進入該中斷，就會發生一直在執行中斷程序的情況

3、週期性中斷是由定時器輔助完成的，只要定時時間到，硬件就會自動置位標誌位，這時中斷髮生，定時器還在運行，與中斷處理是並行的沒有衝突，因此週期性的中斷不是從中斷標誌位清零開始的。

4、如果中斷程序沒有執行完並且下一個中斷又來了，那麼硬件還是會自動置位標誌位的，如果在這之前已經清零標誌位那麼中斷結束後會再進入一次該中斷，如果這時還沒有清除中斷標誌位，則第二個1毫秒中斷無效。
