架構師：系統運行環境的集成與設計有哪幾種？

2020-11-14
| 科技

在設計一個新的系統時，設計者必須考慮目標系統的運行環境問題，人們往往認為軟件應該能夠在任何環境中運行，常常看到這樣的系統，硬件已經升級了多次，而軟件還是原來的軟件。軟件的運行環境是指系統運行的設備、操作系統和網絡配置。

如下是軟件運行的幾個典型環境

1．集中式系統

早期的計算機系統沒有什麼可以選擇的，除了集中式系統。所有的操作都集中於一臺主機中，而操作員必須在主機的附近操作，結果也在附近給出。這種系統仍然廣泛地應用於批處理應用系統及更大的分佈式系統的一部分。

集中式系統常見於銀行、保險、證券行業，它們含有大規模的處理應用。而現在流行的電子商務又給大型處理機注入了新的活力，人們發

現電子商務要面對大量的事務，需要大型處理機來處理。但是，實踐中很少單獨使用集中式系統，因為大量的系統需要處理在地理上分佈得很遠的連接請求，這些請求有的需要實時響應，並可能要發送到其他某個地方的一個集中式系統。所以，在現代的系統中，集中式系統通常是某個分佈式系統的一個環節。

集中式系統由以下幾個部分組成。

  • （1）單計算機結構：這種結構簡單、容易維護，但是處理能力受到限制。
  • （2）集群結構：由多個計算機組成，這些計算機具有類似的硬件平臺、操作系統等。通常採用負載均衡、資源共享等方式實現更大的處理能力和容量。
  • （3）多計算機結構：由多個計算機組成，這些計算機之間操作環境可能不同。適用於當系統可以分解成多個不同的子系統時。

2．分佈式系統

分佈式系統由於網絡的普遍延伸，費用的不斷降低而越來越成為大型系統的首選環境。分佈式系統必須基於網絡，這個網絡可以是在一個地域內的局域網，也可以是跨越不同城市乃至國家的廣域網。對比集中式的計算機環境，分佈式系統有著多種多樣的形式。

3．C/S 結構

系統由提供服務的服務器和發起請求、接受結果的客戶機構成。這種結構是一種可以使用很多方式實現的通用結構模型。並非只限於數據庫的 C/S 結構，典型的還有網絡打印服務系統，現在流行的網絡遊戲也顯然是基於這種結構的。

4．多層結構

這種結構是 C/S 結構的擴展，典型的分為由存儲數據的數據庫服務器作為數據層、實現商業規則的程序作為邏輯層、管理用戶輸入輸出的視圖層所組成的三層結構。當系統更復雜時，可以再增加其他層次構成多層結構。

多層結構形式複雜，功能多樣。實現多層結構常常需要來實現不同層次間通信的專門程序——管件，也稱為中間件。中間件大多數實現遠程程序調用、對象請求調度等功能。

5.Internet、Intranet 和 Extranet

  • Internet 是全球的網絡集合，使用通用的 TCP/IP 協議來相互連接。Internet 提供電子郵件、文件傳輸、遠程登錄等服務。
  • Intranet 是私有網絡，只限於內部使用，也使用 TCP/IP協議。
  • Extranet 是一個擴展的 Intranet。它包括企業之外的和企業密切相關合作的其他企業。

Extranet 允許分離的組織交換信息並進行合作，這樣就形成了一個虛擬組織。現在的 VPN技術允許在公用網絡上架構只對組織內部開發服務。

Web 同樣基於 C/S 結構，實際上 Web 接口是一個通用的接口，不是隻能使用瀏覽器的協議，它同樣能夠在普通的程序中使用。Internet 和 Web 已經給設計者提供了一個非常富有吸引力的選擇方案。它的優勢在於：它們已經成為網絡的事實上的標準，支持它們的軟件已經廣泛地存在於全世界的計算機中，而且通信費用已經下降到很有競爭力的水平。從某種程度上來說，企業可以把 Internet 當作自己廉價的廣域網。沒有它們，電子商務還是水中月。
