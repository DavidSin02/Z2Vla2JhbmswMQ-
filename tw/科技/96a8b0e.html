磁盤及文件系統

硬盤的物理構成：

扇區（sector）、磁道（track）、磁頭、柱面(cylinder)

分區：就是記錄每一個分區的起始柱面和結束柱面。分區信息存放在0柱面0磁頭1扇區上：MBR 主引導記錄(446字節)+DPT磁盤分區表（64字節）+結束標誌（2個字節）

硬盤的主引導記錄（MBR）是不屬於任何一個操作系統的，它先於所有的操作系統而被調入內存，並發揮作用，然後才將控制權交給主分區（活動分區）內的

操作系統，並用主分區信息表來管理硬盤。

文件系統是操作系統用於明確存儲設備（常見的是磁盤）或分區上的文件的方法和數據結構；即在存儲設備上組織文件的方法。

block ：數據存儲的最小單元

inode:索引節點，全局唯一編號，除了記錄文件的屬性外，同時還具有指針功能，指向文件內容放置的快

（裡面保存的是文件的權限，所有者，所屬主等基本信息）

1、文件的擁有者與用戶組（owner/group）

2、文件的訪問模式（read/write/excute）

3、文件的類型（type）

4、文件建立或狀態改變的時間（ctime)、最近一次的讀取時間（atime）、最近修改時間（mtime）

5、文件的大小

6、定義文件屬性的標誌（flag），如setUID...

7、文件真正內容的指針（pointer）

創建目錄：分配一個inode和至少一個block

inode記錄該目錄的相關屬性，並指向分配到的那個快；

block記錄在這個目錄下的相關文件（或目錄）的關聯性

創建文件：分配至少一個inode與相對於該文件大小的快數量

inode不記錄文件名，而是記錄文件的相關屬性，文件名記錄在目錄所屬的block區域

例如：讀取/etc/crontab的流程如下：

1、操作系統根據根目錄（/）的相關數據可獲取/etc目錄所在的inode，並讀取/etc這個目錄所有相關屬性

2、根據/etc的inode的數據，可以獲取/etc目錄下所有文件的關聯數據是放置在哪一個block中，並前往該block讀取文件的關聯性內容

3、由上步驟的block中，可以知道crontab文件的inode所在地，並前往該inode

4、由上步驟的inode中，可以獲取crontab文件的所有屬性，並前往由inode所指向的block區域，獲取crontab文件內容

塊組：block group 對用戶是不可見的，子邏輯區域，有個超級標示區

文件系統構成：（dumpe2fs 可以查看）

superblock(超級塊)：記錄整個文件系統相關信息

1、block與inode的總量

2、未使用和已使用的inode/block數量

3、文件系統的載入時間、最近一次寫入數據的時間、最近一次檢驗磁盤（fsck）的時間等文件系統的相關信息

4、有效位的值，已載入為0，未載入為1

group description(組描述)：記錄此block由何處開始記錄

block bitmap(快位圖)：此處記錄block是否使用

inode bitmap(inode位圖)：此處記錄inode是否使用

inode table(inode表)：為每個inode的數據存放區

data block(數據塊)：為每個block的數據存放區

metadata bitmap （塊位圖)dentry 目錄項

創建文件：先在目錄下創建一個條目，把名字和iNode對應上，回到源數據，指定塊位置

刪除文件：刪除條目裡面的名字和iNode對應關係，將對應的iNode標記為0

通過inode號碼來刪除文件：find ./ -inum 400935 -exec rm -rf {} \;

extundelete： linux下高效數據恢復工具 http://extundelete.sourceforge.net/ 安裝時需要的依賴包 e2fsprogs-lib e2fsprogs-devel

複製文件：不同的inode和block

剪切文件：同一個分區，inode號和磁盤塊位置都沒有變，只是把條目裡面的名字改變了一下

鏈接：硬鏈接和軟鏈接（符號鏈接），多個文件指向同一個iNode，硬鏈接；符號連接，指向的是源文件的路徑，而不是文件塊

ln (link) [-s -v] 源文件 鏈接

硬鏈接：在某個目錄下的block中增加一個文件關聯數據，不會用到inode與磁盤空間

只能對文件進行創建，為了避免循環引用

不能跨文件系統，可以在不同目錄下

創建硬鏈接可以增加文件被連接的次數

符號鏈接：建立一個獨立的文件，這個文件會讓數據的讀取指向它連接的文件內容

可以對目錄創建

可以跨文件系統

不會增加被鏈接文件的鏈接次數

其大小為指定路徑所包含的字符個數

Linux： 查看VFS結構圖

VFS:virtual filesystem ,interface:system call

當前，除了Linux標準的文件系統Ext2/Ext3/Ext4外，還有很多種文件系統，比如reiserfs, xfs, Windows的vfat NTFS，

網絡文件系統nfs 以及flash 文件系統jffs2, yaffs/yaffs2 ubifs。

linux通過叫做VFS的中間層對這些文件系統提供了完美的支持。

VFS存在的意義

1. 向上，對應用層提供一個標準的文件操作接口；

2. 對下，對文件系統提供一個標準的接口，以便其他操作系統的文件系統可以方便的移植到Linux上；

3. VFS內部則通過一系列高效的管理機制，比如inode cache, dentry cache 以及文件系統的預讀等技術，

使得底層文件系統不需沉溺到複雜的內核操作，即可獲得高性能；

4. 此外VFS把一些複雜的操作儘量抽象到VFS內部，使得底層文件系統實現更簡單

VFS 是一種軟件機制，也許稱它為 Linux 的文件系統管理者更確切點，與它相關的數據結構只存在於物理內存當中。

所以在每次系統初始化期間，Linux 都首先要在內存當中構造一棵 VFS 的目錄樹(在 Linux 的源代碼裡稱之為 namespace)，

實際上便是在內存中建立相應的數據結構。

VFS: virtual filesystem 中間層
