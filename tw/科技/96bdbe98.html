無公網IP？不知道怎麼訪問家裡的服務器？今天教大家只需3分鐘

2020-11-14
| 科技

如今社會網絡發展迅速，越來越多服務都是依賴網絡，但是隨著網絡的普及，網絡依賴的ipv4地址已經耗竭。而為了解決這一問題，衍生出了NAT技術。目前家用寬帶很多都是經過2次NAT轉換甚至3次NAT轉換才到達公網實現上網的目的，還有更多的地方是直接不讓更改光貓模式。這就造成很多家庭用戶無法通過端口映射的方式在外網訪問家裡的監控、NAS等。

今天就教大家來簡單的設置，只需三步就可以訪問到家裡的服務器。（以下是我用虛擬機模擬公網來遠程家裡的win7系統舉例）

首先我們需要準備一臺服務器，我這裡用虛擬機來做演示（原因是我因為我窮[捂臉]）。真實環境請大家在各大雲廠商購買有云主機，系統最好是用Linux，原因這裡就不多說了，自行某度。

第一步；搭建frp服務端：

1、編輯服務端的frps.ini配置文件

[common]

bind_port = 7000 監聽端口

token=123456 客戶端認證密碼

2、創建frpsi啟動服務

vim /etc/systemd/system/frps.service

[Unit]

Description=frps

After=network.target

[Service]

ExecStart=/opt/frp_0.33.0_linux_amd64/frps -c /opt/frp_0.33.0_linux_amd64/frps.ini

[Install] #路徑請根據自己壓縮包的解壓路徑

WantedBy=multi-user.target

3、啟動服務以及添加開機自啟動

systemctl start frps.service

systemctl enable frps.service

4、檢查監聽端口是否開啟

lsof -i:7000

COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME

frps 1814 root 3u IPv6 25695 0t0 TCP *:afs3-fileserver (LISTEN)

有以上信息證明端口已經開啟

第二步；配置客戶端：

1、編輯frpc.ini配置文件

[common]

server_addr = server ip #這裡填寫你frps服務器的地址

server_port = 7000 #監聽端口與服務器一致

token=123456 # 認證密碼，加強安全

[mstsc]

type = tcp

local_ip = 127.0.0.1

local_port = 3389 #填寫內網的端口

remote_port = 33890 #填寫外網的端口

2、啟動客戶端服務


這裡看到有success的字樣就證明啟動成功
