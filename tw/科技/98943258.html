18.1. 存儲特點

  • Set集合中，沒有下標的概念。
  • Set集合，是一個去重複的集合。 在Set集合中不會添加重複的元素的！
    在向一個Set集合中添加元素的時候， 會先判斷這個元素是否已經存在了。 如果存在， 則不再添加。
  • Set集合中， 數據的存儲是無序的。
    無序： 所謂的無序， 其實指的是元素的添加順序和存儲順序是不一致的。
    無序， 並不意味著隨機！

Set接口， 是繼承自Collection接口的。 Set接口中的方法， 都是從Collection接口中繼承下來的， 並沒有添加新的方法。

18.2. 哈希表

Set集合的兩個實現類HashSet與LinkedHashSet，底層實現都是哈希表。

  • Hash，一般翻譯做“散列”，也有直接音譯為“哈希”的，它是基於快速存取的角度設計的，也是一種典型的“空間換時間”的做法。顧名思義，該數據結構可以理解為一個線性表，但是其中的元素不是緊密排列的，而是可能存在空隙。
  • 散列表（Hash table，也叫哈希表），是根據鍵值碼值(Key value)而直接進行訪問的數據結構。也就是說，它通過把鍵值碼值映射到表中一個位置來訪問記錄，以加快查找的速度。這個映射函數叫做散列函數。
  • Hash表的組成是”數組+鏈表”這些元素是按照什麼樣的規則存儲到數組中呢。一般情況是通過hash(key)%len獲得，也就是元素的key的哈希值對數組長度取模得到。

比如下圖哈希表中，12%16=12,28%16=12,108%16=12,140%16=12。所以12、28、108以及140都存儲在數組下標為12的位置

18.3. 二叉樹

  • 二叉樹是一種非常重要的數據結構，它同時具有數組和鏈表各自的特點：它可以像數組一樣快速查找，也可以像鏈表一樣快速添加。但是他也有自己的缺點：刪除操作複雜。
  • 二叉樹：是每個結點最多有兩個子樹的有序樹，在使用二叉樹的時候，數據並不是隨便插入到節點中的，一個節點的左子節點的關鍵值必須小於此節點，右子節點的關鍵值必須大於或者是等於此節點，所以又稱二叉查找樹、二叉排序樹、二叉搜索樹。
  • 叉樹遍歷分為三種
    • 先序遍歷
      首先訪問根，再先序遍歷左子樹，最後先序遍歷右子樹
    • 中序遍歷
      首先中序遍歷左子樹，再訪問根，最後中序遍歷右子樹
    • 後序遍歷
      首先後序遍歷左子樹，再後序遍歷右子樹，最後訪問根

18.4. TreeSet的排序

18.4.1. TreeSet的簡介

TreeSet是一個Set接口的實現類，底層實現是二叉樹。這樣的集合，會對添加進集合的元素進行去重的處理。 同時， 這個集合會對添加進入的元素進行自動的升序排序。

18.4.2. Comparable接口

如果某一個類實現這個接口， 表示自己實現了一個可以和自己的對象進行大小比較的規則。 此時， 這個類的對象就可以直接存儲進TreeSet集合中了。 因為此時TreeSet集合已經知道了怎麼對兩個這個類的對象進行大小比較。

/** * @Description */public class Person implements Comparable<Person> {    String name;    int age;    int score;    int height;    int weight;    public Person(String name, int age, int score, int height, int weight) {        this.name = name;        this.age = age;        this.score = score;        this.height = height;        this.weight = weight;    }    @Override    public String toString() {        return "Person{" +                "name='" + name + '\'' +                ", age=" + age +                ", score=" + score +                ", height=" + height +                ", weight=" + weight +                '}';    }    /**     * 制定了大小比較的規則     * @param o 和this進行比較的Person對象     * @return 比較結果     *      > 0 :   this > o     *      ==0 :   this == o     *      < 0 :   this < o     */    @Override    public int compareTo(Person o) {        return this.age - o.age;    }}class TreeSetUsage2 {    public static void main(String[] args) {        // 1. 實例化一個TreeSet對象        TreeSet<Person> sets = new TreeSet<>();        sets.add(new Person("xiaoming", 10, 100, 165, 50));        sets.add(new Person("xiaohong", 11, 99, 164, 50));        sets.add(new Person("xiaolan", 9, 98, 164, 51));        sets.add(new Person("xiaolv", 8, 98, 166, 48));        sets.add(new Person("xiaozi", 10, 97, 159, 46));        // 2. 遍歷集合        sets.forEach(System.out::println);        System.out.println(sets.size());    }}

18.4.3. Comparator接口

在實例化TreeSet集合對象的時候，可以通過Comparator進行實例化。 此時， 這個集合有著自己的排序的依據， 與集合中存儲的元素對應的類無關。 此時集合中存儲的元素對應的類， 可以不實現Comparable接口， 依然可以完成排序。 即便這個類真的實現了Comparable接口， 最終的排序結果依然以構造方法中的Comparator為準。

/** * @Description 通過Comparator進行比較 */public class TreeSetUsage3 {    public static void main(String[] args) {        // 1. 實例化一個TreeSet對象        //    可以通過Comparator進行TreeSet的實例化。使用指定的規則進行大小比較。        //    此時，集合中存儲的元素可以不實現Comparable接口        TreeSet<Person> sets = new TreeSet<>((p1, p2) -> p1.height - p2.height);        sets.add(new Person("xiaoming", 10, 100, 165, 50));        sets.add(new Person("xiaohong", 11, 99, 164, 50));        sets.add(new Person("xiaolan", 9, 98, 164, 51));        sets.add(new Person("xiaolv", 8, 98, 166, 48));        sets.add(new Person("xiaozi", 10, 97, 159, 46));        // 2. 遍歷集合        sets.forEach(System.out::println);        System.out.println(sets.size());    }}

18.4.4. Comparable與Comparator的使用場景

  • 如果這個對象， 在項目中大多數的情況下， 都採用相同的大小比較的方式。 比如： 一個Person類， 在大多數情況下， 都是按照年齡進行大小比較的。 此時就可以讓Person類實現Comparable接口。
  • 如果某一個類的對象， 在臨時進行大小比較的時候， 使用的與默認的比較不一樣的規則。 比如： 一個Person類， 大多數情況下， 都是使用的年齡進行大小比較的， 但是臨時需要使用身高進行一次比較， 此時就可以使用 Comparator 臨時完成了。 而且， Comparator的優先級要高於Comparable。
