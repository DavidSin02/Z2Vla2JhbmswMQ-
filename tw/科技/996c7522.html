MySQL觸發器的簡單操作

2020-11-14
| 科技

觸發器是MySQL的數據庫對象之一，該對象與編程語言中的函數非常類似，都需要聲明、執行等。但觸發器的執行不是有程序調用的，也不是有手動啟動的，而是有事件來觸發、激活從而實現執行。

1、為什麼要用觸發器呢？

先看兩個實例：

*在學生表中，有用字段學生名字，字段學生總數，每當添加一條關於學生記錄時，協商的總數就必須同時改變。

*在客戶信息表中，有字段客戶名字，字段客戶電話和字段客戶地址縮寫，每當添加一條客戶記錄時，都需要檢查客戶電話號碼是否正確，客戶地址縮寫是否正確。

上述實例，業務邏輯不同，但有共同之處，即在表發生更改時，自動進行一些處理。這時就可以使用觸發器數據庫對象，例如第一個實例，可以創建一個觸發器，在添加一條記錄時，就執行一次計算學生總數的操作，這樣就可以保證每次添加後數據一致。

MySQL軟件在觸發如下語句時，就會自動執行所設置的操作：

*DELETE 語句。

*INSERT 語句。

*UPDATE 語句。

其他SQL語句在不會觸發。

2、創建觸發器

在MySQL中，創建觸發器的SQL語句如下：

create trigger trigger_name BEFORE|AFTER trigger_EVENT ON TABLE_NAME FOR EACH ROW trigger_STMT

上述語句中，trigger_name 參數表示要創建的觸發器名字，不能與已經創建的重複，BEFORE和AFTER參數指定了觸發器執行時間，BEFORE是在觸發器事件前，BEFORE是在觸發器事件後，trigger_EVENT參數表示觸發器執行條件，包含DELETE、INSERT、UPDATE語句，TABLE_NAME表示觸發事件操作表的名字，FOR EACH ROW表示任何一條記錄的操作都觸發，trigger_STMT參數表示激活觸發器後被執行的語句。

實例：

(1)創建兩個表tab1和tab2

DROP TABLE IF EXISTS tab1;

CREATE TABLE tab1(

tab1_id int(10)

);

DROP TABLE IF EXISTS tab2;

CREATE TABLE tab2(

tab2_id int(10)

);

(2)創建觸發器，實現tab1添加數據時tab2自動添加

CREATE TRIGGER tri_tab1_to_tab2

AFTER INSERT ON tab1

FOR EACH ROW

insert into tab2(tab2_id) values(new.tab1_id);

(3)向tab1中插入數據

INSERT INTO tab1(tab1_id) values('0001');

(4)查看結果

3、查看觸發器

在MySQL中不能有同名的觸發器，所以在新建觸發器之前，必須要查看觸發器，可以使用如下命令：

SHOW TRIGGERS \G;

運行效果如下：

在執行完SHOW TRIGGERS \G命令後，會顯示一個列表，在該列表中，會顯示出觸發器的所有信息。

4、刪除觸發器

在MySQL中，可以使用DROP TRIGGER trigger_name刪除觸發器，其中trigger_name是想要刪除發觸發器名稱。

執行完成後，再次使用命令查看觸發器，我們發現已經沒有所建的觸發器了
