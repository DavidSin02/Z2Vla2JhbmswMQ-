Spring Cloud Feign/Ribbon 超時時間設置和Istio故障注入測試

2020-10-29
| 科技

很多小夥伴在引入Spring Cloud Feign後，都會碰到如下故障：

feign.RetryableException: Read timed out executing GET http://test-order/orderInfo?id=1] with root cause java.net.SocketTimeoutException: Read timed out

由於SpringCloudFeign的客戶端負載均衡是通過SpringCloudRibbon實現，而SpringCloudRibbon的默認連接超時時間為2秒，默認讀取超時時間為5秒，所以，當調用超過2秒的時候，就會報錯，如圖：

Ribbon默認超時時間

超時例子

圖中所示的例子，是有微服務"test-user"發起，調用微服務"test-order"，我們利用istio的故障注入，使得調用"test-order"微服務的http請求都會延時7秒返回，如下：

apiVersion: networking.istio.io/v1alpha3kind: VirtualServicemetadata:  name: test-order  namespace: econspec:  hosts:    - test-order  http:    - route:        - destination:            host: test-order            subset: v1            port:              number: 8259      fault:        delay:          percentage:            value: 100.0          fixedDelay: 7s

針對這種情況，我們可以手動調長ribbon的延時時間，在服務調用發起方的application.yaml里加入，注意超時時間必須大於istio的故障注入延遲時間：

#--------------Feign負載均衡配置 配置全局超時時間 毫秒單位   根據業務酌情配置ribbon:  ConnectTimeout: 10000 #請求連接的超時時間    ReadTimeout: 10000 #請求處理的超時時間

或者是在kubernetes deploy的env變量里加入：

- env:

- name: ribbon_ConnectTimeout

value: "10000"

- name: ribbon_ReadTimeout

value: "10000"

加入之後，效果變成，7秒鐘之後，請求結果就順利返回：


花費時間7秒鐘，結果順利返回
