算法｜遞歸：參數、局部變量的順序迭代與逆序迭代

2020-10-29
| 科技

迭代使用單一關係式，要考慮迭代次數或結束條件，分精確迭代（如最大公約數）和近似迭代（如求平方根）。

遞推的遞推變量和遞推關係式（包含初始表達式，已知條件或結果，形成初始或邊界條件），分為順推（如斐波那契數列）、逆推（如五猴分桃）。

由編譯器隱式實現一個函數調用棧，函數可以自己調用自己，參數之間相互迭代，所以使用遞推法能解決的問題也可以通過遞歸法來解決，遞推形成遞歸的一個階段。遞歸調用語句前的部分，一般包含返回的邊界條件及其它部分，可用一個局部變量及參數順序迭代的循環來簡單代替，即遞推階段；遞歸語句本身即迴歸階段；遞歸調用語句的後面部分，可用一個局部變量及參數逆序迭代的循環來代替，但需要程序員自己維護一個顯式棧來記憶參數、局部變量和返回值。

#include <stdio.h>void counting(int n){    int lv=10;    lv*=n;    if(n>3)        return;    printf("遞歸調用語句前面部分數據的順序輸出，參數n和局部變量lv：%d,%d\n",n,lv);    counting(n+1);    printf("遞歸調用語句後面部分數據的逆序輸出，參數n和局部變量lv：%d,%d\n",n,lv);}int main(){    counting(1);    getchar();    return 0;}/*遞歸調用語句前面部分數據的順序輸出，參數n和局部變量lv：1,10遞歸調用語句前面部分數據的順序輸出，參數n和局部變量lv：2,20遞歸調用語句前面部分數據的順序輸出，參數n和局部變量lv：3,30遞歸調用語句後面部分數據的逆序輸出，參數n和局部變量lv：3,30遞歸調用語句後面部分數據的逆序輸出，參數n和局部變量lv：2,20遞歸調用語句後面部分數據的逆序輸出，參數n和局部變量lv：1,10*/

在逆序時為什麼能夠記憶參數值和局部變量？

函數調用時編譯器使用了一個隱式棧，每一個函數使用一個棧幀來記錄需要返回的函數地址、參數值、局部變量值。為了能夠在出發後經過一系列調用能回到最初的出發地，需要一個後進先出的數據結構，這就是棧。

當遞歸用遞推的循環來實現時，對於遞歸調用語句後面的部分，需要由程序員使用一個顯式棧來記錄參數值和局部變量，並逆序輸出：

#include <stdio.h>#include <stack>using namespace std;typedef struct Node{     int para1;    int lv1;}node; // 用一個結構體為保存參數與局部變量void counting2(int n){    stack<node> sf;    int init = n;    while(n<=3)    {        int lv=10;        lv*=n;        printf("數據的順序輸出，參數n和局部變量lv：%d,%d\n",n,lv);        node newnode = {n,lv};        sf.push(newnode); // 壓棧操作，後面要逆序輸出數據        n++;    }    while(!sf.empty())    {        node newnode = sf.top();        printf("數據的順序輸出，參數n和局部變量lv：%d,%d\n",newnode.para1,newnode.lv1);        sf.pop();     }    }int main(){    counting2(1);    getchar();    return 0;}/*數據的順序輸出，參數n和局部變量lv：1,10數據的順序輸出，參數n和局部變量lv：2,20數據的順序輸出，參數n和局部變量lv：3,30數據的順序輸出，參數n和局部變量lv：3,30數據的順序輸出，參數n和局部變量lv：2,20數據的順序輸出，參數n和局部變量lv：1,10*/

附整數與浮點數的表示與存儲：

對於浮點數的存儲，卻是用記階法表示的，如：

+1001.011B = + 0.1001011B×2^ 100

-0.0010101B = -0.10101B×2^－10

可見，任一個二進制實數 N 均可表示為：

N=±S×2^P

其中， ±是該數的符號； S是N 的尾數；P是N的階碼。

因此，32位的單精度浮點數在計算機中可表示為：

由於指數（階碼）可以選用不同的編碼（原碼、補碼等），尾數的格式和小數點位置也可以有不同的規定，因此早期計算機中浮點數的表示方法互不相同。

現代計算機中，一般都以IEEE 754標準存儲浮點數，這個標準的在內存中存儲的形式為：

對於不同長度的浮點數，階碼與小數位分配的數量不一樣，如對於32位的單精度浮點數，數符分配是1位，階碼分配了8位，尾數分配了是23位：

符號位：0表示正；1表示負；

偏移階碼e：e=指數的實際值+127。

假有一個浮點數10110010.001，則指數是7，階碼就要用7+127的二進制數表示，也就是：111+01111111 = 10000110

尾數使用原碼錶示，絕對值在1與2之間，其中1和小數點都是隱含的，並不直接表示。

根據這個標準，我們來嘗試把一個十進制的浮點數轉換為IEEE754標準表示。

例如：178.125

先把浮點數分別把整數部分和小數部分轉換成2進制：

整數部分用除2取餘的方法，求得：10110010

小數部分用乘2取整的方法，求得：001

合起來即是：10110010.001

轉換成二進制的浮點數，即把小數點移動到整數位只有1，即為：1.0110010001 * 2^111，111是二進制，由於左移了7位，所以是111

把浮點數轉換二進制後，這裡基本已經可以得出對應3部分的值了：

數符：由於浮點數是正數，故為0(負數為1)。

階碼 : 階碼的計算公式：階數 + 偏移量, 階碼是需要作移碼運算，在轉換出來的二進制數裡，階數是111(十進制為7)，對於單精度的浮點數，偏移值為01111111(127)[偏移量的計算是：2^(e-1)-1, e為階碼的位數，即為8，因此偏移值是127]，即：111+01111111 = 10000110

尾數：小數點後面的數，即0110010001

可能有個疑問：小數點前面的1去哪裡了？由於尾數部分是規格化表示的，最高位總是“1”，所以這是直接隱藏掉，同時也節省了1個位出來存儲小數，提高精度。

浮點數的二進制顯示可以使用以下代碼：

//位運算結合union輸出int和float的二進制位#include <iostream>using namespace std;union { //用於將浮點數的二進制位解析為int位輸出    float input;    int output;} data;void int2binary(unsigned n)//位運算只能用於整形{    unsigned b32 = 1<<31;//32位二進制，最高位是1，其它位是0    cout<<((n&b32)>>31)<<" ";//最高位與運算，移位後最高位輸出    for(int i=1;i<32;++i)    {        n=n<<1;//循環左移一位，用於最高位的與運算        cout<<((n&b32)>>31);//最高位與運算，移位後最高位輸出        if(i==7)            cout<<" ";        if(i>8 && (i-7)%8==0)            cout<<" ";    }    cout<<"\n";}void float2binary(unsigned n){    unsigned b32 = 1<<31;//32位二進制，最高位是1，其它位是0    cout<<((n&b32)>>31)<<" ";//最高位與運算，移位後最高位輸出    for(int i=1;i<32;++i)    {        n=n<<1;//循環左移一位，用於最高位的與運算        cout<<((n&b32)>>31);//最高位與運算，移位後最高位輸出        if(i%8==0)            cout<<" ";    }    cout<<"\n";}void main(){    while(1)    {        int n;        cout<<"please input a int:";        cin>>n;        int2binary(n);        cout<<endl;        cout<<"please input a float:";        cin>>data.input;        float2binary(data.output);        cout<<endl;    }}/*please input a int:1780 0000000 00000000 00000000 10110010please input a float:178.6250 10000110 01100101 01000000 0000000please input a int:-1781 1111111 11111111 11111111 01001110please input a float:-178.6251 10000110 01100101 01000000 0000000please input a int:1340 0000000 00000000 00000000 10000110please input a float:10110010的階數是7，最高位總是1，直接隱藏掉；階碼=階數+偏移量=階數+2^(e-1)-1=階數+2^(8-1)-1=7+127=134(10000110) */

要區別整數的補碼和浮點數的符號位存儲，雖然兩者都是以高位0表示正數，1表示負數，但兩者規定的規則卻是不一樣的，負整數是通過正整數的二進制自然碼（最高位是0）取反+1而來的。而負浮點數卻是直接取1而來的，階碼、尾數都是用自然碼來表示的。
