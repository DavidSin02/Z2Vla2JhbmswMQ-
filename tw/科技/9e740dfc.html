彙編語言入門（四）

2020-11-14
| 科技

一個完整的彙編語言源程序需要增加段定義偽指令和定義數據存儲單元偽指令等必須有的偽指令。偽指令與 C 語言等高級語言中的說明性語句的含義類似，起到說明作用。有了偽指令，系統在彙編該程序時，就會正確的將源程序翻譯、連接成可執行文件。

用戶編寫的源程序要經彙編程序 MASM 彙編（翻譯）後生成二進制目標程序，文件名默認與源程序同名、擴展名為 .OBJ；再經過 LINK 連接生成可執行程序，文件名默認與源程序同名、擴展名為 .EXE。

執行 MASM 和 LINK 命令時需要按多次回車。

執行步驟：

（1）輸入彙編命令 masm abc.asm

注意：此處顯示 0 個警告性錯誤，0 個嚴重性錯誤。如果有錯誤，必須對源程序進行修改。有關出錯提示信息參見教材附錄 B。

（2）輸入連接命令 link abc.obj

注意：此處有一個警告性錯誤：no stack segment 沒有堆棧段。這個提示是由於我們的程序中沒有定義堆棧段，此錯誤可以忽略，不影響生成 .exe 文件。如果出現嚴重性錯誤，必須再檢查一下源程序有無錯誤。

（3）用 dir 查看一下

已經生成了 abc.exe 可執行程序了。如果程序有錯誤，再回到記事本修改錯誤；之後需要重新彙編和連接。

（4）接著運行該程序 abc.exe（運行的方式是直接在 dosemu 中輸入 abc.exe），可以看到沒有顯示任何結果就退回到 DOS 下了。
