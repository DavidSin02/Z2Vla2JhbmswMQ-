整理 | MUI項目中使用標題欄進度條或正在加載控件

2020-11-14
| 科技

在APP項目開發中，我們都會使用一些提示性的控件來給用戶優質的體驗。使用MUI開發APP時，其頁面使用H5來構建，這時我們需要在頁面初始化的時候使用各類提示性組件（如彈窗、正在加載等）給用戶友好的提示，否則會因為各類原因（如網絡、設備等影響）會影響用戶的體驗，或者交互。對於這種情況，我所使用的是，在跳轉頁面時，一種方式使用繪製原生導航控件時設置窗口標題欄控件顯示進度參數；另一種使用H5+ API的nativeUI的showWaiting和closeWaiting方法，在plusReady中使用showWaiting，在頁面渲染完成後使用closeWaiting。這兩種方式都是基於mui.openWindow之後的。

標題欄進度條

我們可以通過在mui.openWindow的styles節點中設置titleNView節點的相關參數。

				mui.openWindow({					url: "home.html", //打開窗口頁面					id: "home",					styles: {						titleNView: { //窗口標題欄控件							titleText: "新聞詳情頁", //標題欄文字							titleColor: "#FF0000", //字體顏色							titleSize: "18px", //字體大小							progress: { //標題欄控件的進度條樣式								color: "#00FF00", //進度條顏色								height: "2px", //進度條高度							}						}					}				})

正在加載提示控件

使用加載提示控件，相對來說，更加明確，當加載提示控件顯示時，說明數據正在渲染中。

			mui.plusReady(function() {				plus.nativeUI.showWaiting("正在加載...");				//頁面渲染數據方法				var list = [{					"id": 1,					"title": "第一條新聞"				}, {					"id": 2,					"title": "第二條新聞"				}];				var data = {					news: list				};				var html = template('newsl', data);				document.getElementById('newsList').innerHTML = html;				plus.nativeUI.closeWaiting();			})

兩種方式各有特點，可以根據實際情況來使用。

示例
