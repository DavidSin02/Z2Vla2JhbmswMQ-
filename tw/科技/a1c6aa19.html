sql server複製技術，本地發佈和本地訂閱數據庫實時同步

2020-11-14
| 科技

sql server複製技術是挺不錯的，今天就詳細寫一篇關於sql server複製的文章。

sql server複製技術應用場景

  • 數據庫備份

複製技術，用於同數據庫備份，多一個選擇備份的方法

  • 跨庫數據庫實時同步

本地客戶端數據庫和雲端服務端數據庫同步，用於零售行業居多，客戶端如果連接雲端數據庫延遲大，打開速度慢，可以在客戶端本地電腦安裝數據庫，再和雲端數據庫實時同步，這樣就完美解決延遲大的問題，實現連鎖。

準備工具，sql server2008標準版

開始配置

設置服務端本地發佈

1.在服務端電腦創建個文件夾，命名快照，再創建個admin管理員賬號，設置admin的密碼，再把快照這個文件夾共享，共享選擇admin這個用戶可讀寫↓

2.打開sql server配置管理器-sql server服務-sql server代理，點擊本用戶，瀏覽選擇admin這個賬號，輸入密碼確定，sql server代理要設為自動啟動↓

3.打開SQL Server Management Studio工具，登錄，點擊複製-本地發佈--新建發佈↓

4.下一步↓

5.輸入剛剛創建的文件夾網絡地址（IP換成自己服務器的IP）↓

6.選擇要發佈的數據庫↓

7.選擇合併發佈，合併發佈意思是服務端數據庫有更新，客戶端數據庫會跟著更新，客戶端數據庫有更新，服務端數據庫也會更新↓

8.下一步↓

9.選擇要發佈的表↓

10.下一步↓

11.點擊更改，選擇更新的頻率，一般選擇每天間隔5分鐘更新一次就行↓

12.點擊安全設置，設置代理服務器↓

13.按圖選擇，輸入數據庫賬號密碼，點擊確定↓

13.最後點擊完成，輸入發佈名稱，點擊確定就完成了服務端數據庫發佈配置↓

14.查看是否發佈成功，點擊複製-本地發佈-選擇剛剛發佈的名稱，右鍵選擇查看快照代理狀態↓

這樣就表示發佈成功↓

設置客戶端本地訂閱

1.在客戶端電腦創建個admin管理員賬號，設置admin的密碼和服務端admin密碼一致，用於讀取服務端的共享文件

2.打開sql server配置管理器-sql server服務-sql server代理，點擊本用戶，瀏覽選擇admin這個賬號，輸入密碼確定，sql server代理要設為自動啟動↓

3.打開SQL Server Management Studio工具，登錄，點擊複製-本地訂閱--新建訂閱↓

4.下一步↓

5.查找sql sever發佈服務器↓

6.服務器名稱只支持填入計算機名，局域網直接輸入計算機名稱，輸入數據庫賬號密碼，不需記住密碼，點擊連接↓

（注意，對於遠程公網，要把計算機名和公網IP綁定到hosts就可以了）

這是對於公網綁定hosts方法，局域網不需要綁定hosts↓

C:\Windows\System32\drivers\etc目錄下的hosts文件，用文本格式打開

7.選擇上面創建的發佈數據庫，點擊下一步↓

8.下一步↓

9.選擇要訂閱的數據庫，點擊下一步↓

10.單擊這個位置配置連接↓

11.按圖選擇，連接發佈服務器那裡輸入服務端的數據庫賬號密碼，點擊確定↓

12.下一步↓

13.這裡選擇定義計劃，一般選擇每天間隔5分鐘更新一次↓

14.下一步↓

15.最後點擊完成，就完成了本地訂閱配置↓

16.查看是否訂閱成功，複製-本地訂閱，點擊剛剛創建的訂閱右鍵，查看同步狀態↓

這樣表示訂閱成功↓
