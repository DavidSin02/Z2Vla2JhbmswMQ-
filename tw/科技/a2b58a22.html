編程中變量還有生命長短？C語言四種不同的存儲類變量

2020-11-14
| 科技

存儲類定義C程序中變量和/或函數的範圍（可見性）和生命週期。它們位於它們修改的類型之前。我們在C程序中有四種不同的存儲變量

  • 自動
  • ·寄存器
  • 靜態的
  • ·外部

自動存儲類

該自動存儲類是所有局部變量的默認存儲類。

{

int mount;

auto int month;

}

上面的示例在同一存儲類中定義了兩個變量。'auto'只能在函數中使用，即局部變量。

寄存器存儲類

該寄存器存儲類是用來定義應當被存儲在寄存器中，而不是RAM局部變量。這意味著變量的最大大小等於寄存器大小（通常是一個單詞），並且不能將一元“＆”運算符應用於它（因為它沒有內存位置）。

{

register int miles;

}

該寄存器只應用於需要快速訪問的變量，例如計數器。還應注意，定義“寄存器”並不意味著變量將存儲在寄存器中。這意味著它可能會根據硬件和實現限制存儲在寄存器中。

靜態存儲類

該靜態存儲類指示編譯器在程序而不是創建和每次進入和超出範圍時摧毀它的壽命時間內保持一個局部變量存在。因此，使局部變量靜態允許它們在函數調用之間維護它們的值。

靜態修飾符也可以應用於全局變量。完成此操作後，它會將該變量的範圍限制為聲明它的文件。

在C編程中，當靜態用於全局變量時，它只會導致該成員的所有對象共享該成員的一個副本。

#include <stdio.h>

/* function declaration */

void func(void);

static int count = 5; /* global variable */

Int main() {

while(count--) {

func();

return 0;

}

return 0;

}

/* function definition */

void func( void ) {

static int i = 5; /* local static variable */

i++;

printf("i is %d and count is %d\n", i, count);

}

編譯並執行上述代碼時，會產生以下結果 -

i is 6 and count is 4

i is 7 and count is 3

i is 8 and count is 2

i is 9 and count is 1

i is 10 and count is 0

外部存儲類

該的extern存儲類是用來給一個全局變量，那就是所有程序文件可見的參考。當您使用'extern'時，無法初始化變量，但是它將變量名稱指向先前已定義的存儲位置。

當您有多個文件並且定義了一個全局變量或函數（也將在其他文件中使用）時，extern將在另一個文件中用於提供已定義變量或函數的引用。僅僅為了理解，extern用於在另一個文件中聲明全局變量或函數。

當有兩個或多個文件共享相同的全局變量或函數時，最常用的是外部修飾符，如下所述。

第一個文件：main.c

#include <stdio.h>

int count ;

extern void write_extern();

Int main() {

count = 5;

write_extern();

return 0;

}

第二個文件：support.c

#include <stdio.h>

extern int count;

void write_extern(void) {

printf("count is %d\n", count);

}

這裡，extern用於在第二個文件中聲明count，因為它在第一個文件main.c中定義了它。現在，編譯這兩個文件如下 -

$gcc main.c support.c

它將生成可執行程序a.out。執行該程序時，會產生以下結果 -

count is 5
