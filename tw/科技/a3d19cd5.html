GitHub連續3天出現嚴重宕機情況 微軟未迴應

2020-11-14
| 科技

【點擊右上角加'關注'，全國產經信息不錯過】

4月24日消息 在COVID-19大流行期間，更多的人在遠程工作。但在過去的三天裡，全球範圍內的GitHub開發者都遭遇到了服務宕機的問題，導致他們無法執行基本的編程任務。

4月21日，多個GitHub服務被問題困擾了至少一個半小時。4月22日，再次出現的中斷影響了眾多開發者至少兩個小時的時間。而4月23日，多個GitHub服務也遇到了各種問題的影響，開始時間大約是美國東部時間上午9點20分，持續了近三個小時。和最近的其他中斷事件一樣，官方沒有提供任何原因，也沒有公佈恢復過程的信息。

今天的故障影響到了Git運營；API請求；Webhooks；問題、拉取請求和項目；GitHub行動；GitHub包；GitHub頁面等。在 GitHub 狀態網站上，只有 Git Operations 被列為“故障”，其他服務被標註為 "已降級"。

雖然GitHub狀態網站上確實定期更新了微軟試圖修復各種故障的情況，但並沒有提供任何關於問題的細節。在Twitter上，一些開發者一直在要求微軟提供更多關於GitHub問題的詳細事後說明。

IT之家瞭解到，外媒ZDNET已經向微軟發去置評請求，詢問為什麼本週GitHub出現這麼多的宕機問題，目前微軟還沒有迴應。

今天早上，GitHub教育賬號在Twitter上承認GitHub Classroom和education.github.com上出現了宕機。

"啊，每天的GitHub中斷，"4月23日，Mijndert在推特上寫道。

"Github這個月中斷時間高達159分鐘.....但誰在計算呢？"4月23日，Alex Atkinson在推特上寫道。

"對那些今天早上抱怨@github今天早上又發生了一次#outage（繼本週早些時候的一次中斷之後）的開發者們來說...... 就當這是又一次提醒大家，#雲並不神奇...........它只是別人的電腦，"Kellen Murphy在4月23日的推特上寫道。
