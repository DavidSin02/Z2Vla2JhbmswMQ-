前景提要(文末提供源碼下載）

今天我們繼續來分享工作表拆分的操作，相對於昨天來說，我們今天的這個場景可能會比較常見一些，因為此類問題有幾個小夥伴一起提出過類似的情況，所以今天我也是專門合併在一起來實現下，今天的這個拆分場景想比較昨天的案例，又是一個升級版，不僅僅有空行，還有空列，數據都是獨立成塊，我們要一塊塊一片片的進行拆分，來看看案例

場景說明

今天的數據是這樣的，這裡肯定會有小夥伴們抱怨了，這個數據完全看不清內容，其實今天數據源的內容反而不是主要的，主要是數據的格式，大致一眼看過去，就能夠看到數據都是被分成一塊一塊的，一個區域一個數據源，這樣的數據構造影像中是比較常見，有很多時候我們為了一次性展示數據，通常會選擇這樣的方式來實現，但是數據分析和保存的時候，誰都不喜歡看這樣的數據，那麼這樣的數據如何進行拆分呢？

代碼區

來看看代碼吧

Sub swws()Dim sth As Worksheet, sthn As WorksheetSet sth = ActiveSheetRowl = sth.UsedRange.Rows.CountColl = sth.UsedRange.Columns.CountFor i = 1 To Rowl For j = 1 To Coll If Cells(i, j) <> "" Then Rcount = Cells(i, j).CurrentRegion.Rows.Count Ccount = Cells(i, j).CurrentRegion.Columns.Count Worksheets.Add after:=Worksheets(Worksheets.Count) Set sthn = ActiveSheet sth.Cells(i, j).CurrentRegion.Copy sthn.Cells(1, 1) sth.Activate j = j + Ccount - 1 End If Next j i = i + Rcount - 1Next iEnd Sub

簡短了很多，來看看過程

額~什麼都不用選擇，按照當前的場景，直接執行程序就可以進行拆分了，我們來看看結果

剛好8個板塊的數據被拆成了8個區域，滿足了我們的要求，今天的代碼就比較的簡單了，因為數據模式比較的規則，來看看代碼的分析

代碼分析

今天的代碼的重點就不在代碼本身，而在代碼邏輯本身的思路上面

Rowl = sth.UsedRange.Rows.CountColl = sth.UsedRange.Columns.Count

我們這裡先得到當前區域的總行數和總列數，usedrange代表的是所有的使用區域，無視空行空列

有了總行數和總列數之後，我們就可以開始構造循環了，這裡的循環和我們之前的循環又有一點線不同了，這裡是兩個循環同時進行

For i = 1 To Rowl For j = 1 To Coll **** Next jNext i

i代表的是行數，j代表的則是列數，再循環行的時候，同時循環列

我們先看第一次循環

If Cells(i, j) <> "" Then Rcount = Cells(i, j).CurrentRegion.Rows.Count Ccount = Cells(i, j).CurrentRegion.Columns.Count Worksheets.Add after:=Worksheets(Worksheets.Count) Set sthn = ActiveSheet sth.Cells(i, j).CurrentRegion.Copy sthn.Cells(1, 1) sth.Activate j = j + Ccount - 1 End If

cells（1,1）不為空，是由數值的，所以這裡既可以進入判斷內部了，得到當前這個這個小的區域的範圍，如何得到CurrentRegion

就是這一小塊區域，以空行空列為標準，非空的填充數據的都是CurrentRegion的範圍，然後我們可以得到這個單位的總行數總列數了，有了範圍，直接將他們丟到新的工作表中就可以了。這裡就很簡單了。

然後我們繼續，這個時候要注意循環的過程，這裡時候我們正在進行j就是列的循環，這個時候我們的列數j是1

我們再次循環是從2開始嗎？

但是第2列已經是我們第一個數據區的範圍了，剛剛已經複製過去了

如果你想到這一點了，那就證明你已經理解CurrentRegion的範圍了，既然已經複製過了，我們自然不能再次複製了，那麼怎麼辦，直接跳躍

之前我們也用過跳躍式循環，j = j + Ccount - 1

當我們複製完一個區域之後，順帶的將j的數值變成我們剛剛複製的區域的總行數

然後再次循環，j就來到了空列，然後進入判斷，cells(i,j)是否為空，為空，跳過j+1,就這樣成功的進入了下一個數據區

這樣又來到了下一個循環了，這個循環和上面的循環過程值一樣的，不過這個已經到了j的結尾了，所以就會自動跳出j的循環了。

然後又回到i的循環，i在前面的兩個循環中一直都是1.所以這裡他也要選擇和j一樣的操作，我們不需要再循環之前已經複製的數據區了，直接讓他跳躍式前進。

i = i + Rcount - 1

所以本節的代碼是很簡單，關鍵的重點是大家要理解這個代碼工作的思路和邏輯，以及各個變量的作用和跳躍範圍。
