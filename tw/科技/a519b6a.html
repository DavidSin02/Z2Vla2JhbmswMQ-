前言

老顧之前介紹過多個子項目如何設計配置文件，以及多環境下如何配置文件；小夥伴如果仔細想一下，之前的文章的解決方案還有有些問題。

第一：配置多環境還是比較麻煩，Maven插件配置麻煩

第二：配置信息暴露給了開發人員

第三：不能動態改變配置內容，如：生成環境中日誌級別一般為warn，但如果線上報錯了，需要排查問題，有時候是需要把日誌級別調整一下；一般是需要把應用服務停止，改下日誌級別後再啟動應用服務；這種就比較麻煩，那能不能不需要重啟服務的情況下，就能改變日誌級別呢？

外置配置

老顧這裡利用之前介紹過的nacos配置中心方案，小夥伴們就一步步跟這老顧的思路。首先我們是否可以利用nacos解決配置信息暴露的問題。

我們首先需要在POM文件中引入nacos的config依賴包

<dependency>  <groupId>com.alibaba.cloud</groupId>  <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId></dependency>

然後我們在項目配置文件bootstrap.yml，必須是bootstrap.yml文件哦

spring:  application:    name: config-service  cloud:    #nacos相關配置    nacos:      #註冊中心服務地址      discovery:        register-enabled: true        server-addr: 127.0.0.1:8848      #配置中心服務地址      config:        server-addr: 127.0.0.1:8848        file-extension: yml

我們一般項目都會在application.yml中配置datasource數據源信息，以及redis分佈式緩存的配置信息，如：

spring:  datasource:    name: rainbow    url: jdbc:mysql://localhost:3306/rainbow    username: root    password: 123456  redis:     database: 0    host: localhost    password:     port: 6379

我們會發現一些重要的配置信息暴露了出來，如：url鏈接地址，用戶名，密碼

如果我們通過application-dev.yml,application-sit.yml,application-uat.yml,application-prod.yml方式實現多環境；

那dev環境，sit環境，uat環境，prod環境都會暴露出來。

那我們有沒有什麼方法不讓開發人員輕易發現信息，或者說如果外置這些配置信息。我們通過nacos的加載配置文件機制解決這個問題

nacos會自動加載${prefix}-${spring.profile.active}.${file-extension}配置文件信息，

1、prefix 默認為 spring.application.name 的值，也可以通過配置項spring.cloud.nacos.config.prefix 來配置。

2、spring.profile.active 即為當前環境對應的 profile，可以通過配置項 spring.profile.active 來配置。

3、file-exetension 為配置內容的數據格式，可以通過配置項 spring.cloud.nacos.config.file-extension 來配置。目前只支持 properties 和 yaml 類型。

注意：當 spring.profile.active 為空時，對應的連接符 - 也將不存在

dataId 的拼接格式變成 prefix.{file-extension}

那我們可以利用spring.profile.active屬性，設置db，以及redis配置信息

spring:  application:    name: config-service  cloud:    #nacos相關配置    nacos:      #註冊中心服務地址      discovery:        register-enabled: true        server-addr: 127.0.0.1:8848      #配置中心服務地址      config:        server-addr: 127.0.0.1:8848        file-extension: yml  profiles:    active:    - db    - redis

我們需要在nacos配置中心新建文件

這樣的配置後，就無須在application.yml中配置db和redis配置信息了，啟動時nacos客戶端先去nacos配置中心讀取配置，並加載初始化一些Bean。

這樣就很好的起到了配置信息外置的方式，而且我們可以很方便的修改信息。

多模塊共享配置

因為很多子模塊都會需要鏈接數據庫以及redis，而且他們的配置信息是一樣的，那我們就不需要每個模塊都要配置一次，我們只要利用nacos的共享配置機制

spring:  application:    name: config-service  cloud:    #nacos相關配置    nacos:      #註冊中心服務      discovery:        register-enabled: true        server-addr: 127.0.0.1:8848      #配置中心服務      config:        server-addr: 127.0.0.1:8848        file-extension: yml        shared-configs:        - db.yml        - redis.yml

這樣每個子模塊都這樣配置，就只需要共享此配置文件。

用了shared-configs屬性

動態刷新日誌級別

那我們如何在線刷新日誌級別呢？其實也很簡單

spring:  application:    name: config-service  cloud:    #nacos相關配置    nacos:      #註冊中心服務      discovery:        register-enabled: true        server-addr: 127.0.0.1:8848      #配置中心服務      config:        server-addr: 127.0.0.1:8848        file-extension: yml        shared-configs:        - db.yml        - redis.yml  profiles:    active:    - log

日誌配置內容

業務中的日誌代碼

log.error("這是一條error");

log.warn("這是一條warn");

log.info("這是一條info");

log.debug("這是一條debug");

直接通過nacos的配置，動態可以更新應用服務的日誌級別

多環境配置

多環境配置推薦利用nacos的命名空間方案，可以很好的實現。

可以在外部啟動jar時，通過外部參數傳入

java -jar config-server.jar --spring.cloud.config.namespace=a1a8bb58-18a6-4e3e-a584-6adb109d7a85

以上就是利用配置中心實現多項目模塊，配置外置的方法，也能夠實時刷新日誌級別。
