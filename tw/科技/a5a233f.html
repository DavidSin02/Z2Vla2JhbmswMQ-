1.1. 多態（掌握）

多態時面向對象三大特徵：封裝、繼承、多態。

在繼承關係，是一種”is A”的關係，也就說子類是父類的一種特殊情況，有如下代碼：

public class Animal{}

public class Dog extends Animal{}

public class Cat extends Animal{}

那麼我們可以認為狗和貓都是一種特殊的動物，那麼可以使用動物類型類來表示狗或貓。

Dog d = new Dog(); //創建一隻狗對象,賦給子類類型變量

Animal a = new Cat(); //創建一隻貓對象,賦給父類類型變量

此時對象（a）具有兩種類型:

n 編譯類型：聲明對象變量的類型——>Animal

n 運行類型：對象的真實類型 ——>Dog

當編譯類型和運行類型不一致的時候，此時多態就產生了:

注意：編譯類型必須是運行類型的父類或接口。

所謂多態，表示一個對象具有多種形態。

Animal a = null;

a = new Dog(); // a此時表示Dog類型的形態

a = new Cat(); // a此時表示Cat類型的形態

多態的前提，可以是繼承關係（類和類），也可以是實現關係（接口和實現類），在開發中，一般都指接口和實現類之間的關係，什麼是多態？

一言以蔽之：父類引用變量指向於子類對象，調用方法時實際調用的是子類的方法。

我家有一種動物，你猜它的叫聲是怎麼樣的，猜不到，因為這個動物有多種形態。

n 如果該動物是狗，叫聲是：旺旺旺...

n 如果該動物是貓，叫聲是：妙妙妙...

多態操作有兩種定義格式和操作語法：

n 操作繼承關係（開發中不是很多）：

父類 變量名 = new 子類();

變量名.方法();

n 操作實現關係（開發中最頻繁）：

接口 變量名 = new 實現類();

變量名.方法();

1.1.4. 類型轉換和instanceof運算符（瞭解）

自動類型轉換：把子類對象賦給父類變量（多態）

Animal a = new Dog();

Object obj = new Dog(); //Object是所有類的根類:

強制類型轉換：把父類類型對象賦給子類類型變量（前提：該對象的真實類型應該是子類類型）

Animal a = new Dog();

Dog d = (Dog) a; //正確

Cat c = (Cat) a; //錯誤

instanceof 運算符：判斷該對象是否是某一個類的實例，在開發中運用不是很多

語法格式：boolean b = 對象A instanceof 類B; //判斷 A對象是否是 B類的實例？如果是,返回true

代碼如下：

Animal a = new Dog();

System.out.println(a instanceof Animal); //true

System.out.println(a instanceof Dog); //true

System.out.println(a instanceof Cat); //false
