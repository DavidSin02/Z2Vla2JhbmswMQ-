深入淺出排序算法

2020-11-14
| 科技

1. 冒泡排序

時間複雜度：O(n2)

穩定性：穩定

算法描述

  1. 從頭到尾比較相鄰兩個元素，如果第一個比第二個大，則交換位置；
  2. 依次往後，直到最後一對，這樣在數組末尾應該是當前最大值；
  3. 除了後面已經排好的序列重複上面的步驟，直到排序結束。

特點

比較適合小數據的排序。但是，由於算法複雜度較高，在數據量大的時候不適合使用。

在數據完全有序的時候展現出最優時間複雜度，為O(n)

O(n)，其他情況下，幾乎總是O(n2)

O(n2)。

動圖演示

代碼

def bubbleSort(array): for i in range(len(array)): for j in range(len(array) - i - 1): if array[j] > array[j + 1]: array[j], array[j + 1] = array[j + 1], array[j] return array

代碼優化：增加一個swap標誌，若一次循環中沒有任何值交換位置，則說明排序已經完成，跳出即可。

def bubbleSort2(array): for i in range(len(array)): swap = False for j in range(len(array) - i - 1): if array[j] > array[j + 1]: array[j], array[j + 1] = array[j + 1], array[j] swap = True if not swap: break return array

2. 選擇排序

時間複雜度：O(n2)

穩定性：不穩定

算法描述

  1. 在未排序的序列中找出最小的元素，存放到數組的開始位置；
  2. 重複上述操作，直到排序結束。

特點

在各種情況下複雜度波動小，因此一般是優於冒泡排序的。在所有的完全交換排序中，選擇排序也是比較不錯的一種算法。

但是，由於固有的O(n2)複雜度，選擇排序在海量數據面前顯得力不從心。因此，它適用於簡單數據排序。

動圖演示

代碼

def selectSort(array): for i in range(len(array) - 1): min_idx = i for j in range(i + 1, len(array)): if array[j] < array[min_idx]: min_idx = j array[i], array[min_idx] = array[min_idx], array[i] return array

3. 插入排序

時間複雜度：O(n2)

穩定性：穩定

算法描述

  1. 將序列分為已排序和未排序兩段，每次將未排序的第一個元素插入前面已排序的正確位置；
  2. 重複上述操作，直到排序結束

特點

插入排序由於O(n2)

O(n2)的複雜度，在數組較大的時候不適用。但是，在數據比較少的時候，是一個不錯的選擇，一般做為快速排序的擴充。

動圖演示

代碼

def insertionSort(array): for i in range(len(array) - 1): cur, pre_idx = array[i + 1], i while pre_idx >= 0 and cur < array[pre_idx]: array[pre_idx + 1] = array[pre_idx] pre_idx -= 1 array[pre_idx + 1] = cur return array

4. 希爾排序

時間複雜度：O(nlogn)

O(nlog⁡n)

穩定性：不穩定

算法描述

  1. 設置一個增量序列，最後一個為1；
  2. 分別將序列按照第一個增量分為若干字列，對每個子列進行插入排序；
  3. 對下一個增量做同樣的插入排序，直到為1時排序結束。

例子：對於待排序列 {44，12，59，36，62，43，94，7，35，52，85}，我們可設定增量序列為 {5，3，1}。

特點

希爾排序雖然快，但是畢竟是插入排序，其數量級並沒有後起之秀——快速排序O(nlogn)

O(nlog⁡n)快。在大量數據面前，希爾排序不是一個好的算法。但是，中小型規模的數據完全可以使用它。

代碼

本代碼中動態定義間隔序列的算法是《算法（第4版》的合著者 Robert Sedgewick 提出的。

def shellSort(array): length = len(array) gap = 1 while gap < length // 3: gap = gap * 3 + 1 while gap > 0: for i in range(gap, length): cur, pre_idx = array[i], i - gap while pre_idx >= 0 and cur < array[pre_idx]: array[pre_idx + gap] = array[pre_idx] pre_idx -= gap array[pre_idx + gap] = cur gap //= 3 return array

5. 歸併排序

時間複雜度：O(nlogn)

O(nlog⁡n)

穩定性：穩定

算法描述

採用分治法的思想，將原始序列分為兩段，兩段分別歸併排序後，再合併。合併時將兩個已排序序列的首部開始比較，總是將小的那個加入到結果中，如果有一個序列已經遍歷結束，則將另一個序列剩餘的部分直接添加進來。

特點

歸併排序在數據量比較大的時候也有較為出色的表現，但是，其空間複雜度O(n)

O(n)使得在數據量特別大的時候幾乎不可接受。考慮到有的機器內存本身就比較小，因此，採用歸併排序一定要注意。

動圖演示

代碼

def mergeSort(array): def merge(left, right): res = [] i, j = 0, 0 while i < len(left) and j < len(right): if left[i] < right[j]: res.append(left[i]) i += 1 else: res.append(right[j]) j += 1 return res + left[i:] + right[j:] if len(array) <= 1: return array mid = len(array) // 2 left, right = mergeSort(array[:mid]), mergeSort(array[mid:]) return merge(left, right)

6. 快速排序

時間複雜度：O(nlogn)

O(nlog⁡n)

穩定性：不穩定

算法描述

  1. 從當前數組中挑出一個數，稱為“基準”(pivot)；
  2. 比較所有剩餘的元素，小於pivot的放到左邊，大於pivot的放到右邊，再對左右分別快速排序。

特點

快速排序尤其在數據量大的時候性能優越性更加明顯。但是在必要的時候，需要考慮優化以提高其在最壞情況下的性能。

動圖演示

代碼

def quickSort(array): if len(array) < 2: return array pivot = array[0] left, right = [], [] for i in array[1:]: if i <= pivot: left.append(i) else: right.append(i) left = quickSort(left) right = quickSort(right) return left + [pivot] + right

7. 堆排序

時間複雜度：O(nlogn)

O(nlog⁡n)

穩定性：不穩定

堆

堆是一種特殊的完全二叉樹，二叉堆一般分為兩種：

最大堆：每個父節點的值都大於或等於其子節點的值（若存在）

最小堆：每個父節點的值都小於或等於其子節點的值（若存在）

算法描述

堆排序的原理就是把最大堆堆頂的最大數取出，將剩餘的堆繼續調整為最大堆，再將堆頂的最大數取出，這個過程持續到剩餘數只有一個時結束。

調整最大堆：對於一個指定不滿足最大堆條件的結點進行“下沉”使其滿足最大堆條件；

建立最大堆：從一個堆的最後一個非葉子結點開始進行調整，往前再調整前一個非葉子結點……

堆排序：創建最大堆——取出最大堆堆頂元素與堆尾元素互換位置（為了節省存儲空間）——對堆頂元素進行調整最大堆——取出堆頂元素——……

特點

堆排序在建立堆和調整堆的過程中會產生比較大的開銷，在元素少的時候並不適用。但是，在元素比較多的情況下，還是不錯的一個選擇。尤其是在解決諸如“前n大的數”一類問題時，幾乎是首選算法。

代碼

def heapSort(array): def adjustHeap(arr, idx, size): """調整二叉堆為最大堆 idx: 當前結點需要調整 size: 二叉堆的大小（後面的不用調整，因為用來存放排好序的結果）""" lchild = 2 * idx + 1 rchild = 2 * idx + 2 largest = idx if lchild < size and arr[lchild] > arr[largest]: largest = lchild if rchild < size and arr[rchild] > arr[largest]: largest = rchild if largest != idx: arr[largest], arr[idx] = arr[idx], arr[largest] adjustHeap(arr, largest, size) def buildHeap(arr, size): # 從最後一個非葉子結點開始建立 for i in range(size // 2)[::-1]: adjustHeap(arr, i, size) size = len(array) buildHeap(array, size) for i in range(size)[::-1]: array[0], array[i] = array[i], array[0] # 將最大堆頂元素放入後面，節省了存儲空間 adjustHeap(array, 0, i) return array

8. 計數排序

時間複雜度：O(n+k)

O(n+k)，其中k

k為max(array)−min(array)+1

max(array)−min(array)+1

穩定性：穩定

算法描述

  1. 將原始數組映射到count範圍為[0,N+1]
  2. [0,N+1]中，每個位置存放不大於該數字的數字出現的次數；
  3. 若不大於i
  4. i的數有s
  5. s個（包括了i
  6. i本身），那麼i
  7. i應該放在s−1
  8. s−1的位置上。

特點

排序目標要能夠映射到整數域，其最大值最小值應當容易辨別。另外，計數排序需要佔用大量空間，它比較適用於數據比較集中的情況。

動圖演示

代碼

def countSort(array): min_num = min(array) count = [0] * (max(array) - min_num + 1) # 將原始[min, max]映射到[0, N+1] res = [0] * len(array) for i in array: count[i - min_num] += 1 # 將數i減去最小值即為正確位置 for i in range(1, len(count)): count[i] += count[i - 1] # count中i位置變為array中不大於i的數量 for i in array: res[count[i - min_num] - 1] = i # 因為不大於i的數有count[i - min_num]個，所以i應該放在count[i - min_num] - 1的位置上 count[i - min_num] -= 1 # 放好一個數，就將對應的位置數量-1 return res

9. 桶排序

算法描述

桶排序又叫箱排序，是計數排序的升級版，它的工作原理是將數組分到有限數量的桶子裡，然後對每個桶子再分別排序（有可能再使用別的排序算法或是以遞歸方式繼續使用桶排序進行排序），最後將各個桶中的數據有序的合併起來。

  1. 找出數組的最大值max_num和最小值min_num；
  2. 桶的數量設置為(max_num - min_num) // length + 1；
  3. 將數組按照桶的個數等分，然後把每個元素放入對應的桶；
  4. 對每個桶內元素進行排序；
  5. 將所有的桶拼接起來。

特點

在分桶和從桶依次輸出的過程是穩定的，但桶排序的穩定性還依賴於桶內的排序算法。桶排序可用於最大最小值相差較大的數據情況，但桶排序要求數據的分佈儘量均勻，否則可能導致數據都集中到一個桶中。

圖片示意

代碼

def bucketSort(array): min_num, max_num, length = min(array), max(array), len(array) num_bucket = (max_num - min_num) // length + 1 buckets = [] # 此處不能直接使用buckets = [[]] * num_bucket，會導致內存地址指向同一處 for _ in range(num_bucket): buckets.append([]) for i in array: buckets[(i - min_num) // length].append(i) array.clear() for i in buckets: array.extend(quickSort(i)) return array

10. 基數排序

算法描述

基數排序是桶排序的擴展，主要思想為將非負整數按位數切割成不同的數字，然後按每個位數分別比較。

基數排序有兩種：

  1. LSD（Least significant digital，次位優先法）：從低位開始進行排序
  2. MSD（Most significant digital，主位優先法）：從高位開始進行排序

特點

基數排序要求較高，排序元素必須是非負整數。

動圖演示

代碼

該代碼只支持非負整數的基數排序。

def radixSort(array): # LSD實現基數排序 div = 1 max_length = len(str(max(array))) buckets = [[] for _ in range(10)] for _ in range(max_length): for i in array: buckets[i // div % 10].append(i) i = 0 for bucket in buckets: while bucket: array[i] = bucket.pop(0) i += 1 div *= 10 return array
