Excel VBA 7.18 Excel報表順序不一致如何合併？給我一個按鈕

2020-10-29
| 科技

前景提要(文末提供源文件下載）

這個也是小夥伴們在閱讀了之前的文章之後提出的問題，之前我們分享了將多個規範的工作薄數據合併在一個工作表的操作的方法，本來也沒有想過特殊的數據需求裡面會有數據表頭順序不一致的情況，不過既然小夥伴們已經提出了這樣的場景需求，那證明還是有可能會碰到的，我今天就針對這個情況在分享下我自己的解法

場景模擬

我們這裡還是那之前的數據作為一個模擬參考對象

A班是標準格式

B班我們將python的成績移動到第二列

C班我們將姓名移動到最後一列

還有其他的工作薄的順序就不更改了，只要更改了一個工作薄的的所有的工作表的順序，也能夠代表表頭順序不同的情況了，來上代碼

代碼區

我們今天直接用數組的寫法吧，因為小夥伴們表示數據可能會比較大，所以我們用數組吧，數組能夠實現一次寫入，避免了多次寫入導致卡頓的情況出現

Sub test()Dim rng As Range, sth As Worksheet, arr(), book As WorkbookApplication.DisplayAlerts = FalseSet book = ActiveWorkbookSet rng = Application.InputBox("請選擇要合併的列名", "表頭的確定", , , , , , 8)With Application.FileDialog(msoFileDialogFolderPicker) If .Show = -1 Then pathn = .SelectedItems(1) End IfEnd Withf = Dir(pathn & "\")Do While f <> "" Workbooks.Open pathn & "\" & f For Each sth In Worksheets l = Cells(Rows.Count, 1).End(xlUp).Row On Error Resume Next maxl = UBound(arr, 2) k = 0 For Each a In rng k = k + 1 num = a.Column ReDim Preserve arr(1 To rng.Columns.Count, 1 To maxl + l - 1) For i = 2 To l j = j + 1 arr(k, j) = sth.Cells(i, num) Next i If k <> rng.Columns.Count Then j = maxl End If Next a Next sth If ActiveWorkbook.Name <> book.Name Then ActiveWorkbook.Close False End Iff = Dir()Loopbook.Worksheets(1).Cells(2, 1).Resize(UBound(arr, 2), rng.Columns.Count) = WorksheetFunction.Transpose(arr)End Sub

來看看操作步驟

依然實現選擇合適的文件夾

然後程序就會自動執行合併操作，看看最後的結果

依然是非常的整齊，表頭就算是順序不一致，依然可以成功的按照正確的表頭來合併數據

代碼分析

其實今天的代碼就比較的簡單了，因為基本上都是前面學習過的知識點，我就來簡單的說下代碼實現的原理和步驟吧

If k = 1 Then sth.UsedRange.Copy tsth.Cells(1, 1) arr = tsth.UsedRange.Rows(1)

首先我們通過遍歷循環文件夾的方式打開第一個工作薄，然後如果是第一個操作的話呢，我們會將當前的工作表的所有已用區域全部直接複製到新建的工作表中

為什麼要這樣操作呢，因為在這樣的話，我們就有了一個表頭對不對，這個表頭就是我們後面複製的參照物了。

然後就是簡單的循環，然後利用數組的方式保存數據了，當然相對於上一節的代碼，這裡又有一點不同了，因為上節我們是第一次就開始就開始使用數組了，有明確的參照物

而本次是第二次才開始使用的，所以數組的構造上面就需要花費一點功夫了。這裡有又回到了7-15講述的最大下標的問題了，

大家首先要有一個邏輯概念，就是我們構造的數組是一個多維數組，多行多列的，那麼有多少行，多少列呢？

首先列數是確定的，因為是根據表頭來確定的，表頭是多少列，那麼就是多少列了，所以這個列數我們可以直接確定了

1 To UBound(arr1, 2)，就是確定數組的列數了

然後確定行數，行數是根據我們統計的工作表的數據來確定的，怎麼確定呢？當前數組的最大小標+當前工作表總共有多少行，代碼就出來了。

1 To l + Count - 1

l是當前工作表的總行數，count是當前數組的最大小標，因為後續不需要彙總表頭這一行了，所以要-1

合起來就是最終獲得動態數組的代碼了。

ReDim Preserve arrt(1 To UBound(arr1, 2), 1 To l + Count - 1)

本節換了一種說法，不知道大家會不會理解的更加清楚一點~~

如果大家確實比較難理解的話，可以像這樣子，通過斷點+調試的方式來關注每次數組的變化情況，慢慢的理論數組的運用

最後就是直接將數組整體寫入我們新建的工作表中，得到我們想要的結果了。
