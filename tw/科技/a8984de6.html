面向對象程序設計三大重要機制之多態

2020-11-14
| 科技

多態性是面向對象程序設計的三大重要機制之一。其與類（數據封裝），類的繼承（繼承性）構成了面向對象程序設計的核心。

這三大機制相互關聯，類是基礎，繼承是核心，而多態便是補充。而且，多態性必須存在繼承的環境之中。

所謂的多態是指對於同一種消息，多種對象成員有完全不同的行為表現。舉個例子：同時送禮物給不同的人，每個人的表現不會完全相同，有人開心，有人疑惑，也有人無所謂。在C++中消息主要是函數成員的調用（也就是上例中的送禮物）而不同的行為表現則是指成員函數的調用，而不同的行為表現則是指成員函數不同的實現（也就是上例中的人的表現）。這樣，也與現實中的處理方式符合。

在C++中多態的實現又以下幾種：

  • 函數的重載
  • 運算符的重載
  • 虛函數

其中，函數和運算符的重載，這兩種都是類的簡單多態性。而虛函數是多態性的主要表現。

虛函數

C++中採用虛函數來實現動態綁定，虛函數是動態綁定的基礎。如果一個函數被聲明為虛函數，那麼其成員函數在子類中可以有不同的實現。

聲明格式：

virtual 數據類型 函數名（參數表）

其中，virtual是關鍵字。用其聲明虛函數。

實例分析：

#include <iostream>

using namespace std;

class Base1{

public:

virtual void display() const; //虛函數

};

void Base1::display() const{

cout<<"Base1::display()"<<endl;

}

class Base2:public Base1{

public:

void display() const; //覆蓋基類的虛函數

};

void Base2::display() const{

cout<<"Base2::display()"<<endl;

}

class Derived:public Base2{

public:

void display() const; //覆蓋基類的虛函數

};

void Derived::display() const{

cout<<"Derived::display()"<<endl;

}

void fun(Base1 *ptr){ //參數為指向基類對象的指針

ptr->display(); //“對象指針->成員名”

}

int main() {

Base1 base1;

Base2 base2;

Derived derived;

fun(&base1);

fun(&base2);

fun(&derived);

return 0;

}

注意：

  1. 類的構造函數也可以調用虛函數（注意是調用不是聲明），不過其採用靜態綁定的方式（當基類構造函數調用虛函數時，不會調用派生類的虛函數）。
  2. 類的析構函數可以是虛函數（注：父類是虛析構函數，子類的析構函數也為虛不管是否用 virtual 聲明）
  3. 只要類的成員函數才可以被聲明為虛函數。因為虛函數只適用於有繼承關係的類對象So 普通函數不能被說明為虛函數
  4. 靜態成員不能是虛函數因為其不受限於某個對象
  5. 內聯函數不能是虛函數因為內聯函數是不能在運行過程中動態確定其位置的，即使虛函數在類體內實現，編譯時，依然將其看作非內聯的。
