JAVA語言程序設計（一）

2020-10-29
| 科技

面向對象的程序設計方法是目前軟件開發的主流技術。Java語言是由美國Sun公司開發的，是一種完全面向對象的程序設計語言。它具有面向對象、跨平臺、安全、健壯、分佈式等特點。由於Java語言從根本上解決了Internet的異質、代碼交換以及安全性等諸多問題，因此，Java語言非常適合於企業網絡和Internet環境下的應用開發，自1995年誕生以來，很快就風靡全球，現已成為Internet中最受歡迎、最有影響力的程序設計語言。Java語言已經成為最卓越的程序設計語言之一，並進入了主流計算模式，對傳統的計算模式提出了新的挑戰，對整個計算機軟件行業的發展產生了極其重大而深遠的影響。

1999年，Sun公司把Java技術分成J2SE、J2EE和J2ME，2005年分別更名為Java SE、Java EE和Java ME。其中，Java SE為創建和運行Java程序提供了最基本的環境；Java EE和Java ME建立在Java SE基礎上，Java EE為分佈式的企業應用提供開發和運行環境，Java ME為嵌入式應用提供開發和運行環境。可見，不論是從事Java EE開發，還是從事Java ME開發，要想成為Java方面的卓越工程師，就必須具備紮實的Java SE知識結構。

Java語言的簡單性、完全面向對象等特點，使得它非常適合大學中面向對象程序設計的教學。本書就是提供了一本零起點、全面介紹Java SE的知識結構體系和講解面向對象程序設計的教程。讀者只需具有一些計算機文化基礎知識，就可學習本教程，而不必具有編程經驗。通過學習本教程，可使讀者掌握Java應用程序和Java Applet的開發方法和技術、熟練使用一種開發工具進行Java程序設計、學會面向對象的思維方式及其編程風格。為進一步學習各種Java技術，從事Java EE開發或Java ME開發打下堅實的基礎。

1.JAVA概述

隨著計算機和Internet的發展，迫切需要面向網絡的計算機語言。大家想一想，在PC上用Windows編寫的程序，不做修改就直接拿到UNIX系統上運行，行嗎？顯然不行，因為程序必須轉換成計算機硬件的機器指令才能執行，而專門為某種計算機硬件和操作系統編寫的程序是不能夠直接放到另外的計算機硬件上執行的，至少要做移植工作。而在網絡環境下，網絡上的計算機是五花八門的，即計算機的類型和操作系統是不一樣的。例如，SUN工作站的硬件是SPARC體系，軟件是UNIX中的Solaris操作系統；而PC的硬件是INTEL體系，操作系統是Windows或Linux。因此，要想讓程序在網絡中各種計算機上都能正常運行，這就要求程序設計語言能夠跨越各種軟件和硬件平臺。Java就是在這種需求背景下誕生和流行的。微軟的比爾·蓋茨（Bill Gates）認為：“Java是長期以來最卓越的程序設計語言。”如今，它已經成為最流行的網絡編程語言。

1.JAVA語言簡介

1.1 JAVA發展歷史

1991年，美國Sun Microsystems公司（以下簡稱Sun公司）的Jame Gosling、Bill Joy等為了能夠在消費電子產品上開發應用程序，積極尋找合適的編程語言。由於消費電子產品種類繁多，所採用的處理芯片不相同，存在著跨平臺的問題。當時，Sun公司的研究人員採用C＋＋語言來編寫消費電子產品的應用程序，但是研究表明，對於消費電子產品而言，C＋＋語言過於複雜和龐大，並不適用，安全性也並不令人滿意。於是，Bill Joy先生領導的研究小組就著手設計和開發出一種新語言，稱之為Oak。Oak是一種與平臺無關、可靠性強、小而靈活的編程語言，但在當時，該語言並沒有引起人們的注意。

1994年下半年，隨著Internet的迅猛發展，人們發現Oak這種中性平臺及安全性高的面嚮對象語言恰恰就是全球信息網所期待的語言。為了使之適應網絡需求，開發人員對Oak進行了一系列的改進，1995年，面向對象的Java語言誕生了。其響亮的口號是“Write Once，Run Anywhere”。Java的誕生對傳統的計算模型提出了新的挑戰。如軟件4A目標要求軟件能達到任何人在任何地方和任何時間對任何電子設備都能操作。

Java的發展歷史

1.2 Java語言的特點

Sun公司在“Java白皮書”中對Java的定義是：“Java：A simple，object－oriented，distributed，interpreted，robust，secure，architecture－neutral，portable，high－performance，multi－threaded，and dynamic language．”按照這個定義，Java語言是一種簡單的、面向對象的、分佈的、解釋的、健壯的、安全的、體系結構中立的、可移植的、高性能的、多線程的以及動態執行的程序設計語言。

1．簡單易學

Java語言簡單易學。其語法與C語言和C＋＋語言很接近，但它丟棄了C＋＋中複雜、不安全的特性，如指針與內存管理、操作符重載和多繼承等。

2．面向對象

面嚮對象的技術是當今軟件開發的主流技術之一。Java語言是一個徹底的純面向對象的程序設計語言。它具有面向對象的封裝、繼承和多態三大特點。Java語言通過類實現封裝，在類之間實現單繼承，在接口之間實現多繼承，並全面支持動態綁定來實現多態。

3．安全性

除了Java語言具有的許多安全特性以外，Java提供了字節碼校驗器、文件訪問限制機制、類裝載器和運行時內存佈局四級安全保證機制。

4．跨平臺（體系結構中立）

Java程序能夠在網絡上任何地方執行；完全統一的語言版本，實現平臺無關性；字節代碼與平臺無關性；訪問底層操作系統功能的擴展類庫，不依賴於具體系統等。

5．多線程

Java環境本身就是多線程的。特別地，Java提供了對多線程的語言級支持，程序員能很方便地編寫多線程應用程序。

6．動態性

Java所需要的類是運行時動態裝載的，也可從網絡載入。在分佈環境中動態地維護應用程序和類庫的一致性，類庫的更新，不需重新編譯程序，不影響用戶程序的執行。

2.JAVA程序工作原理

2.1Java程序工作流程

學習Java語言，有必要了解Java的工作流程，這將更有助於理解Java語言的特點。

對於運行在Internet上的網絡應用程序，需要有良好的可移植性。因為Internet是由各種各樣不同類型的終端、服務器和PC等硬件設備組成的，而且在這些設備上運行的軟件系統也是多種多樣的，所以Internet上的網絡應用程序應該具有在各種不同的軟硬件平臺上都能正常工作的能力。Java的工作流程使得它具有了這樣的能力，如下圖所示。

Java的工作流程

Java的工作流程是：

（1）編輯Java源文件。首先，編程人員使用Java語言編寫好源代碼，形成源文件。文件名的後綴為．java。

（2）編譯Java源文件。然後，使用Java編譯器來編譯Java源文件，生成一種二進制的中間碼，稱為字節碼（byte code），形成字節碼文件。文件名的後綴為．class。

（3）運行Java程序。最後，使用Java解釋器來解釋執行編譯生成的字節碼，完成Java程序要實現的功能。

2.2 Java虛擬機

Java虛擬機（Java Virtual Machine，JVM）是一個抽象的計算機處理器，負責運行Java字節碼文件，負責把字節碼解釋成具體平臺上的機器指令進行執行。它也稱為Java解釋器。

Sun公司對Java虛擬機規範進行了嚴格定義和控制。Java虛擬機規範規定了：

（1）Java虛擬機由6個部分組成：一組指令集、一組寄存器、一個類文件格式規定、一個棧、一個無用單元收集堆（Garbage－collected－heap）和一個方法區域。

（2）虛擬機能夠認識的字節代碼以及能實現的功能。

但是Java虛擬機規範沒有規定Java虛擬機組織結構及其功能是如何實現的。而這些必須在真實機器上以某種方式實現，既可以用軟件實現，也可以用硬件實現。因此，對於不同的軟硬件平臺（指處理器和操作系統），Java虛擬機就要作專門的實現。但是，在不同軟硬件平臺上的Java虛擬機，只要符合Java虛擬機規範，將保證字節碼文件的正確執行。Java語言編譯執行的過程，如圖1－2所示。

Java平臺由Java虛擬機和Java應用編程接口（Application Programming Interface，API）構成。Java應用編程接口為Java應用提供了一個獨立於操作系統的標準接口，可分為核心部分和擴展部分。核心Java API中封裝了程序設計所需要的主要應用類。在硬件和操作系統平臺上安裝一個Java平臺之後，Java應用程序就可運行。現在Java平臺幾乎已經嵌入到所有的操作系統。這樣，Java程序只編譯一次，就可以在各種系統中運行。

Java語言編譯執行的過程

2.3 Java語言的平臺無關性

Java程序的跨平臺主要是指字節碼文件可以在任何具有Java虛擬機的計算機或者電子設備上正確運行。其原因主要是：

（1）Java語言是與平臺無關的。它沒有“依據機器的不同而不同”或“由編譯器決定”等字眼。因此，Java語言編寫的程序在不同平臺上都是完全一樣的。

（2）字節代碼是與平臺無關的。由Java編譯器產生的字節代碼是二進制碼，它與具體的計算機處理器代碼無關。

（3）Java虛擬機隱藏了不同平臺的差異。字節碼文件並不是直接運行在計算機平臺上，而是運行在Java虛擬機上。Java虛擬機將字節碼文件與操作系統及硬件分開，保證了字節碼文件在不同平臺上運行的正確性。

Java的平臺無關性

由此可見，Java語言實現了二進制代碼級的平臺無關，在網絡上實現了跨平臺的特性。而C語言是實現了源程序代碼級的跨平臺。

Java語言這種“一次編寫，到處運行（write once，run anywhere）”的方式，有效地解決了目前大多數高級程序設計語言需要針對不同系統來編譯產生不同機器代碼的問題，即硬件環境和操作平臺的異構問題，大大降低了程序開發、維護和管理的開銷。

2.4 Java平臺的分類

目前，Java分為三個體系Java SE（Java2Platform Standard Edition，Java平臺標準版），Java EE（Java 2Platform Enterprise Edition，Java平臺企業版），Java ME（Java 2Platform Micro Edition，Java平臺微型版）。

（1）Java SE：Java SE以前稱為J2SE。它允許開發和部署在桌面、服務器、嵌入式環境和實時環境中使用的Java應用程序。Java SE包含了支持Java Web服務開發的類，併為Java EE提供基礎。

（2）Java EE：這個版本以前稱為J2EE。企業版本幫助開發和部署可移植、健壯、可伸縮且安全的服務器端Java應用程序。Java EE是在Java SE的基礎上構建的，它提供Web服務、組件模型、管理和通信API，可以用來實現企業級的面向服務體系結構（Service－Oriented Architecture，SOA）和Web 2．0應用程序。

（3）Java ME：這個版本以前稱為J2ME。Java ME為在移動設備和嵌入式設備（比如手機、PDA、電視機頂盒和打印機）上運行的應用程序提供一個健壯且靈活的環境。Java ME包括靈活的用戶界面、健壯的安全模型、許多內置的網絡協議以及對可以動態下載的連網和離線應用程序的豐富支持。

3.JAVA開發環境

3.1 Java SE的開發工具包

1995年Sun雖然推出了Java，但它只是一種語言，而要想開發複雜的應用程序，必須要有一個強大的開發庫支持才行。因此，Sun在1996年1月發佈了JDK 1．0。此後，Sun平均以兩年一個新版本的速度不斷更新JDK。目前，較新的版本是JDK 6。

JDK（Java SE Development Kits）就是Sun公司免費提供的Java SE開發工具包。JDK是一種開發環境，開發者利用它可以編譯、運行和調試Java程序。

Java SE開發工具包可以從站點http：／／java．sun．com免費下載。下載完成後，在JDK根目錄下，有bin、jre、lib、demo、include、src．zip等子目錄和一些文件，其含義如下。

（1）開發工具

位於bin子目錄中。指工具和實用程序，可幫助用戶開發、執行、調試和保存以Java語言編寫的程序。

（2）運行時環境

位於jre子目錄中。Java SE運行時環境的實現，由JDK使用。該運行時環境包含Java虛擬機、類庫以及其他文件，可支持執行以Java語言編寫的程序。

（3）附加庫

位於lib子目錄中。包括開發工具需要的附加類庫和支持文件。

（4）演示applet和應用程序

位於demo子目錄中。帶有源代碼的Java平臺編程示例，包括使用Swing和其他Java基類以及Java平臺調試器體系結構的示例。

（5）C頭文件

位於include子目錄中。支持使用Java本機界面、JVMTM工具界面以及Java SE平臺的其他功能進行本機代碼編程的頭文件。

（6）源代碼

位於src．zip文件中。該文件保存著核心API中所有類的源代碼（即java．＊、javax．＊和某些org．＊包的源文件，但不包括com．sun．＊包的源文件）。參閱這些源代碼，可以幫助開發者更好地學習和使用Java程序設計語言、使用任一常用的zip實用程序或者使用JDK的bin目錄中的Jar實用程序，即使用命令jar xvf src．jar。

可以對這些文件進行解壓，提取所有的源代碼文件。

3.2 Java集成開發環境

雖然JDK中提供了一些編譯、運行和調試程序的工具，但是其命令行的工作方式讓用戶感覺不方便。因此，很多廠商推出了一些Java集成開發環境（Integrated Development Environment，IDE）。這些IDE是集成了開發一種語言程序所需的各種工具，集源代碼的編輯、編譯、調試、部署和管理等功能於一體，同時還提供友好的用戶界面，可以幫助程序員生成應用程序框架，減少程序員的重複勞動，提高軟件開發效率。

目前較為常用的Java集成開發環境有：Borland公司開發的JBuilder，Xinox Software公司開發的Jcreator，IBM公司開發的Eclipse，SUN公司開發的NetBeans等。下面對JCreator 和Eclipse作簡要介紹。

1．JCreator簡介

JCreator是Xinox公司開發的功能強大的Java集成開發環境，JCreator分為Le和Pro兩個版本，Le版本是免費的，功能不及Pro的全面。而Pro版本需要購買註冊才能使用。

JCreator是適合各個層次的Java開發人員的IDE工具。它為用戶提供了相當強大的功能，包括：

（1）項目管理功能，項目模板功能，可個性化設置語法高亮屬性、行數、類瀏覽器、標籤文檔、多功能編繹器、嚮導功能以及完全可自定義的用戶界面。

（2）支持無限撤銷、重做、代碼自動縮進、自動類庫方法提示、按照所選內容自動定位查閱Java API文檔等功能。

（3）通過JCreator，不用激活主文檔而直接編繹或運行用戶的Java程序。

（4）可以開發JSP、HTML，並支持Ant和CVS（版本控制）。

（5）JCreator能自動找到包含主函數的文件或包含Applet的HTML文件，然後它會運行適當的工具。

（6）可以通過一個批處理同時編譯多個項目。

（7）第一次啟動時提示設置JavaJDK主目錄及JDKJavaDoc目錄，軟件自動設置好類路徑、編譯器及解釋器路徑，還可以在幫助菜單中使用JDKHelp。

（8）JCreator的設計接近Windows界面風格，用戶對它的界面比較熟悉。其最大特點是與用戶機器中所裝的JDK完美結合，是其他任何一款IDE所不能比擬的。

總之，JCreator小巧、易用、美觀，是Java初級程序員的理想IDE。缺點是不能進行企業級Java EE開發應用。

2．Eclipse簡介

Eclipse是一個開放源代碼的、基於Java的可擴展集成開發平臺。就其本身而言，它只是一個框架和一組服務，用於通過插件組件構建開發環境。幸運的是，Eclipse附帶了一個標準的插件集，包括Java開發工具（Java Development Tools，JDT）。在Eclipse中還可以集成數據庫開發（比如MySQL、Oracle等）和Java EE容器（如Tomcat、JBoss和Weblogic）。可以方便地使用Eclipse進行快速高效的Java企業級應用程序的開發。

Eclipse項目分成3個子項目：

（1）平臺———Platform；

（2）開發工具箱———Java Development Toolkit（JDT）；

（3）外掛開發環境———Plug－in Development Environment（PDE）。

這些子項目又細分成更多子項目。例如Platform子項目包含數個組件，如Compare、Help與Search。JDT子項目包括三個組件：User Interface（UI）、核心（Core）及排錯（Debug）。

Eclipse Platform是一個開放的可擴展IDE，提供了一個通用的開發平臺。JDT支持Java開發。PDE用來支持插件開發。

Eclipse是著名的跨平臺的自由集成開發環境（IDE）。最初主要用來進行Java語言開發，但是有人通過插件使其作為其他計算機語言（比如C＋＋和Python）的開發工具。Eclipse的本身只是一個框架平臺，但是眾多插件的支持使得Eclipse擁有其他功能相對固定的IDE軟件很難具有的靈活性。許多軟件開發商以Eclipse為框架開發自己的IDE。

Eclipse的插件體系架構決定了Eclipse的高度可擴展性，作為IDE，在各種插件的幫助下，它不僅支持各種語言（Java、C／C＋＋、PHP、Perl、Python等）的開發，還支持軟件開發過程中各種開發活動（設計建模、測試、編譯構建；插件開發、J2EE開發、GUI開發、數據庫設計等），甚至能成為圖片繪製的工具。

Eclipse SDK（軟件開發者包）是Eclipse Platform、JDT和PDE所生產的組件合併，它們可以一次下載。這些部分在一起提供了一個具有豐富特性的開發環境，允許開發者有效地建造可以無縫集成到Eclipse Platform中的工具。Eclipse SDK由Eclipse項目生產的工具和來自其他開放源代碼的第三方軟件組合而成。Eclipse項目生產的軟件以CPL發佈，第三方組件有各自自身的許可協議。

總之，Eclipse是一款非常受歡迎的Java開發工具，使用它的Java程序員是最多的。其缺點是比較複雜，對初學者來說，理解起來比較困難。

3.3 Java學習與幫助資源———JDK文檔

JDK文檔包含有關Java API、Java命令行開發工具等的一系列說明和幫助文檔。其中，API的全稱是Application Programming Interface應用程序編程接口。在JDK文檔中，程序員可以瞭解類的繼承結構、成員方法、成員變量、構造方法、靜態成員的詳細說明和描述信息。JDK文檔是Java程序開發不可缺少的編程詞典，是Java語言學習者最權威的參考資料。我們要逐漸習慣用Java文檔自行解惑，通過Java文檔深入地學習和感悟Java語言。同時，Java開發工具包（JDK）提供了“Java”、“Javac”、“Javadoc”、“appletviewer”等命令，在Java幫助文檔中也對此進行了詳細的介紹。

Java文檔有多個版本，我們應該找到與所用JDK匹配的那個版本。另外，它有CHM和HTML兩種格式，皆有中英文的版本。在查閱方式上，可選擇在線查閱，也可將文檔下載後在本地檢索使用。可在Sun中國技術社區下載中文版的Java文檔，也可以直接到Sun公司的Java官方網站下載英文版的Java文檔。
