適配器模式

適配器模式（Adapter Pattern）是作為兩個不兼容的接口之間的橋樑。這種類型的設計模式屬於結構型模式，它結合了兩個獨立接口的功能。適配器模式有三種：類適配器、對象適配器、接口適配器

適配器解決的問題

主要解決在軟件系統中，常常要將一些"現存的對象"放到新的環境中，而新環境要求的接口是現對象不能滿足的。

命令模式模式角色

目標（Target）接口：當前系統業務所期待的接口，它可以是抽象類或接口。

適配者（Adaptee）類：它是被訪問和適配的現存組件庫中的組件接口。

適配器（Adapter）類：它是一個轉換器，通過繼承或引用適配者的對象，把適配者接口轉換成目標接口，讓客戶按目標接口的格式訪問適配者。

適配器模式和裝飾著模式的區別

適配器模式的意義是要將一個接口轉變成另一個接口，它的目的是通過改變接口來達到重複使用的目的。而裝飾器模式不是要改變被裝飾對象的接口，而是恰恰要保持原有的接口，但是增強原有對象的功能，或者改變原有對象的處理方式而提升性能。

代碼實現：220v電壓適配成5v電壓

/** * 目標（Target）接口 */public interface Targetable { public int v5();}
/** * 適配者（Adaptee）類 */public class Adaptee { public int v220(){ return 220; }}

1.基於對象的適配器，組合適配者對象

/** * * 採用對象聚合的方式進行適配 * * 對象的適配器模式 * */public class ObjectAdapterMode implements Targetable { private Adaptee adaptee; public ObjectAdapterMode(Adaptee adaptee) { this.adaptee = adaptee; } public int v5() { return adaptee.v220()/44; }}

2.基於類的適配器，繼承適配者類

public class ClassAdapterMode extends Adaptee implements Targetable { public int v5() { return super.v220() / 44; }}

3.基於接口的適配器，同時實現目標接口和適配者接口

public interface Adapteeable { public int v220();}public class InterfaceAdapterMode implements Targetable ,Adapteeable{ public int v220() { return 220; } public int v5() { return v220()/44; }}

InputStreamReader是通過對象適配的方式進行適配，Reader是目標，StreamDecoder是適配者。

優缺點

優點：可以讓任何兩個沒有關聯的類一起運行，提高了類的複用，增加了類的透明度，靈活性好。

缺點：過多地使用適配器，會讓系統非常零亂，維護成本高。

生活中的命令模式

中國標準輸出電壓220v，手機充電器5v電壓，無法改變標準輸出，可以增加一個充電插頭，它負責將220v的電壓轉化成5v電壓，充當了一個適配器的作用。

手機內存讀卡器，將內存卡轉換成USB接口，方便筆記本電腦讀取手機內存卡中的內容。手機圓孔耳機和方塊耳機，可以通過轉換器進行轉換。
