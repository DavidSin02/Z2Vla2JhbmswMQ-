彙編語言之我的第一個彙編程序

2020-11-14
| 科技

昨天老師講了如何在電腦上運行彙編程序。現在就寫出具體操作步驟記錄下來。

需要的工具

要編寫第一個彙編程序，我們首先要有能夠與相應編譯器兼容的操作系統。鑑於64位windows10操作系統與編譯器不兼容，所以我們使用dos。dos是英文Disk Operating System的縮寫，意思是“磁盤操作系統”。dos是個人計算機上的一類操作系統。我們可以通過多種方式讓我們的計算機進入dos。這裡我選擇老師推薦的dosbox。 DOSBox是一個軟件。它是當前在 Windows 系統運行 DOS 遊戲的較為完美的解決方案。大家可以下一下這個可以在windws環境下模擬dos系統的軟件。

dosbox的桌面圖標

有了系統我們還需要相應的編譯器，這裡是用微軟的masm5.0，這個編譯器與我的windows不兼容，害我搞了好長時間。這個masm中包含了我們將要使用的masm.exe和link.exe以及能夠查看各個寄存器和主存內容以及修改寄存器和主存內容用來調試程序的debug.exe文件。

工具的位置

在我們電腦上的D盤根目錄下創建一個asm文件夾並將debug.exe放進去。再在D盤下創建masm文件夾，並且將masm.exe和link.exe放進去。這樣我們需要的工具就徹底準備好了。準備開始實驗。

開始搞事情

我們的工具搞定了，就下來就是打開我們的虛擬操作系統dosbox。

dosbox剛打開的界面

打開後我們需要輸入以下東西：mount c d:\ 這幾個字母和符號的作用就是表明現在的dos系統的C盤就是你自己電腦上的D盤，比如在你的windows電腦上D盤裡有asm文件夾，在dos裡要訪問就需要鍵入C:\asm，因為此時你的D盤就是dos的C盤。

輸入mount c d:\並按回車之後

然後我們可以先試一試debug的使用。上面已經說了debug的作用就是可以調試程序或者說它可以看到你的電腦的現在的運行狀態。

先打開C盤（其實是你電腦上的D盤），打開asm文件夾，注意在打開文件夾時要在文件夾前面加上cd兩個字母和空格。不然就無法進入文件夾。

現在我們打開了asm文件夾

打開了asm文件夾後我們啟動debug程序。輸入debug按下回車。會出現一個短橫線，

已經啟動debug，等待輸入命令

現在我們輸入查看寄存器內容的命令，輸入r，按下回車。

r命令可以顯示各個寄存器的內容

常用的debug命令：

a (Assemble) 逐行彙編 a [address]

c (Compare) 比較兩內存塊 c range address

d (Dump) 內存16進制顯示 d [address]或 d [range]

e (Enter) 修改內存字節 e address [list]

f (fin) 預置一段內存 f range list

g (Go) 執行程序 g [=address][address...]

h (Hexavithmetic) 制算術運算 h value value

i (Input) 從指定端口地址輸入 i pataddress

l (Load) 讀盤 l [address [driver seetor>

m (Move) 內存塊傳送 m range address

n (Name) 置文件名 n filespec [filespec...]

o (Output) 從指定端口地址輸出 o portadress byte

q (Quit) 結束 q

r (Register) 顯示和修改寄存器 r [register name]

s (Search) 查找字節串 s range list

t (Trace) 跟蹤執行 t [=address] [value]

u (Unassemble) 反彙編 u [address ]或range

w (Write) 存盤 w [address[driver sector secnum>

一段程序的開始

要想在dos裡運行彙編程序，我們首先要寫好代碼。寫彙編代碼我們可以用很多工具，edit，wps，word或者記事本都可以，我在這裡用的是文本文檔（txt），不過我們在保存時要保存為ASM格式。打開一個txt，在裡面寫下這些代碼。

DATAS SEGMENT

STRING DB 'Hello World!',13,10,'$'

DATAS ENDS

ASSUME CS:CODES,DS:DATAS

CODES SEGMENT

START:

MOV AX,DATAS

MOV DS,AX

LEA DX,STRING

MOV AH,9

INT 21H

MOV AH,4CH

INT 21H

CODES ENDS

END START

就像我一樣

在txt中輸入的代碼

寫完後保存成後綴名為asm的文件。放入masm文件夾裡。

現在我們已經完成了代碼。接下來就要開始編譯程序。

首先我們要退出debug程序，輸入q按回車。然後輸入cd..返回上一級目錄，千萬別忽略cd後面的兩個點

退出debug然後返回上一級目錄

在dos裡面打開masm文件夾（輸入cd masm）

打開masm文件夾

打開masm編譯程序

打開masm編譯器之後

首先會彈出一些程序有關的信息，然後下面會提示輸入ASM格式文件名，因為我們已經將文件放到masm文件夾下所以不需要在寫絕對路徑，如果你的helloworld的asm文件放到了其他位置，就需要寫絕對路徑，但千萬不要忘記你的D盤是現在的C盤。

我的helloworld文件名為2.asm，所以我直接輸入2，如果你的文件後綴名為txt，那麼就連後綴名一起輸入，然會回車

輸入文件名之後

接下來還是要求輸入不用管，直接回車，

回車之後

這個也不用管，繼續回車。

第二次回車之後

這個依然不用管，繼續回車。

總共三次回車之後

這樣我們的編譯就完成了，提示我們0 warning，0 error。說明我們的代碼沒有任何錯誤。現在在我們的masm文件夾中我們可以看到多了一個2.obj文件。

接下來是連接過程，輸入link用來啟動link.exe程序。

啟動了link之後

下面提示我們輸入obj格式的文件，正是我們剛剛生成的那個文件，所以輸入文件名2，

link開始

輸入文件名開始連接，然後同樣和之前的步驟一樣，一直按三次回車，直到link完成，我們的masm文件夾下多出了一個2.exe文件，這個就是最終可執行文件。。

程序執行完成

這樣我們可以看到，已經打印出了Hello World ！這樣我們就完成了通過彙編語言寫出的第一個程序helloworld。
