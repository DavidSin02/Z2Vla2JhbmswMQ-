文件系統基礎總結

2020-10-29
| 科技

1 相關概念

文件系統的管理功能，是通過把它所管理的程序和數據組織成一系列文件的方法來實現的。文件是指具有文件名的若干相關元素的集合。元素通常是記錄，而記錄又是一組有意義的數據項的集合。因此，基於文件系統的概念，可以把數據分成數據項，記錄和文件三級。

2 文件的邏輯結構

2.1 有結構的文件

記錄式文件中，每個記錄都用於描述實體集中的一個實體，各記錄有著相同或不同數目的數據項。記錄的長度可以分為定長和不定長兩類。根據用戶和系統管理上的需要，可採用多種方式來組織這些記錄，形成下述的幾種文件：

（1）順序文件

由一系列記錄按某種順序排列所形成的文件。其中的記錄通常是定長記錄，因而能用較快的速度查找文件中的錄。

優點：順序文件應用的場合是對記錄進行批量存取，也只有順序文件才能存儲在磁帶上。

缺點：

a）在交互應用的場合，如果用戶程序要求查詢或者修改某個記錄，文件系統需要順序查找目標記錄，大文件場景下，性能可能很差。

b）增加或者刪除困難。為了解決這一問題，可以為順序文件設置一個運行記錄文件（Log File），或稱為事務文件（Transaction File），把試圖增加，刪除或者修改的信息記錄於其中，規定每隔一定時間，將運行記錄文件與原來的主文件加以合併，產生一個按關鍵字排序的新文件。

（2）索引文件

當記錄為可變長度時，通常為之建立一張索引表，併為每個記錄設置一個表項，以加快對記錄的建索速度。索引表是按照記錄鍵排序的，因此索引表本身是一個定長記錄的順序文件，從而也就可以方便地實現直接存取。

查找過程中，首先根據用戶提供的關鍵字，並利用二分查找的方法檢索索引表，從中找到相應的表項：再利用該表項中給出的指向記錄的指針值，去訪問所需的記錄。每當要向索引文件中增加一個記錄時，變須對索引表進行修改。

由於索引表有較快的檢索速度，故它主要用於對信息處理的及時性要求比較高的場合。這種管理方式的缺點：提高了存儲的費用。

（3）索引順序文件

將數據進行分組，為文件建立一張索引表，為每一組記錄中的第一個記錄設置一個表項。

這種文件保持了順序文件的特點，記錄按照某種順序進行排列。克服了變長記錄的順序文件不能隨機訪問，以及不便於插入和刪除的缺點。它引入了兩個特徵：

a）引入了文件索引表，通過該表可以實現對索引順序文件的隨機訪問；

b）增加了溢出文件，用來記錄新增加的，刪除和修改的記錄。

對於文件比較大的場景，建索速度依然比較慢，此時可以建立多級索引，加快速度。

2.2 無結構文件

大量的源程序，可執行文件，庫函數等都使用無結構文件的形式，即流式文件。unit系統中，所有的文件全被看做是流式文件，即使是有結構文件，也被視為流式文件，系統不對文件進行格式處理。

3 文件目錄

為了能夠對文件進行正確的操作，需要有一定的管理結構，我們稱之為"文件控制塊"。把文件控制塊的有序集合稱為文件目錄，即一個文件控制塊就是一個文件目錄項。通常，一個文件目錄也是一個文件，稱為目錄文件。

3.1 FCB

文件控制塊，簡稱FCB，主要包括如下信息：

（1）基本類型信息

  • 文件名
  • 邏輯結構（流式文件還是記錄式文件）
  • 物理結構（磁盤上的存儲結構：順序文件，索引文件或者索引順序文件）
  • 物理位置（磁盤地址，包括設備名，起始盤塊號等）

（2）存取控制類信息

包括文件主的存取權限，核准用戶的存取權限以及一般用戶的存取權限；

（3）使用信息

文件創建日期，文件上一次修改時間，當前使用信息：當前打開文件的進程數，是否被其他進程鎖住，文件已經被修改但尚未刷新到磁盤上。

3.2 索引節點

文件檢索過程中需要查找文件目錄，由於文件目錄也存儲在磁盤上。數據量比較大的場景，檢索文件目錄需要讀取的磁盤的次數會比較多。為了優化，unix系統將文件名和文件描述信息拆分，即將文件描述信息單獨形成一個稱為索引節點的數據結構，簡稱I節點。因此，文件目錄中的每個目錄項僅由文件名和指向該文件所對應i節點的指針構成。

（1）磁盤上的索引節點

每個文件都包含唯一的一個磁盤索引節點，包括以下內容：

  • 文件主標識符，標識文件的擁有者
  • 文件類型：正規文件，目錄文件或者特殊文件
  • 存取權限：各類用戶對文件的存取權限
  • 物理地址：每個索引節點包含13個地址項，具體功能如下：

其中iaddr[0]-iaddr[9]來存放直接地址。如果每個盤塊大小為4KB，當文件大小不大於40KB時，便可直接從索引節點中讀出該文件的全部盤塊號。

iaddr[10]提供一次間接地址，一級索引到間址塊，系統將配置給文件的多個盤塊號記錄其中，文件最大允許長度4MB；

iaddr[11]提供二次間接地址，文件最大允許的長度是4GB；iaddr[12]提供三次間接地址，所允許的最大文件長度是4TB；

  • 文件長度：以字節為單位的長度
  • 文件連接數：所有指向該文件名的指針計數
  • 文件存取時間：本文件最近被進程存取的時間，最近被修改的時間以及索引節點最近被修改的時間；

（2）內存索引節點

  • 索引節點的編號，用於標識內存索引節點
  • 狀態，i節點上是否被上鎖或被修改
  • 訪問計數，每當有一進程訪問i節點時，將訪問計數加1，訪問完再減1
  • 文件所屬文件系統的邏輯設備號
  • 鏈表指針，設置有指向空閒鏈表指針和散列隊列指針

3.3 目錄結構

（1）單級目錄結構

在整個文件系統中建立一張目錄表，每個文件佔一個目錄項。

存在的問題：

a）查找速度慢；

b）不允許重名；

c）不便於實現文件共享

（2）兩級目錄

為每個用戶建立一個單獨的文件目錄UFD（User File Directory），在系統中再建立一個主文件目錄MFD（Master File Directory）。在主文件目錄中，每個用戶目錄文件都佔有一個目錄項，包括用戶名和指向該用戶目錄文件的指針。

解決了單級目錄的問題，但不能滿足多個用戶共享文件的需求；

（3）多級目錄

對於大型文件系統中，通常採用三級或者三級以上的目錄結構，主目錄在此被稱為根目錄。

多級目錄較兩級目錄而言，查找速度更快；但在多級目錄中查找一個文件，需要按照路徑名逐級訪問中間節點，這就增加了磁盤的訪問次數，無疑影響查詢速度。目前大多數操作系統如UNIX，Linux和Windows系列都採用了多級目錄結構。

4 外存的分配方式

4.1 連續分配

連續分配（Continuous Allocation）要求為每一個文件分配一組相鄰接的盤塊，這種分配方式保證了邏輯文件中的記錄順序與存儲器中文件佔用盤塊的順序的一致性。隨著文件建立時空間的分配和文件刪除時空間的回收，將使磁盤空間分割成許多小塊，此即外存的碎片。

優點：

a）順序訪問容易

b）順序訪問速度快

缺點：

a）要求有連續的存儲空間，產生較多的外部碎片，嚴重降低了外存空間的利用率；如果定期地利用緊湊的方法來消除碎片，則又需要話費大量的機器時間；

b）必須實現知道文件的長度

4.2 鏈接分配

分為隱式鏈接和顯示鏈接兩種形式

4.2.1 隱式鏈接

在文件目錄的每個目錄項中，都必須含有指向鏈接文件的第一個盤塊和最後一個盤塊的指針。每個盤塊中都包含有一個指向下一個盤塊的指針，如圖所示：

主要問題：

（1）只適用於順序訪問，隨機訪問比較低效；

（2）只通過一個鏈接指針來將一大批離散的盤塊鏈接起來，其可靠性較差，只要有一個指針出現問題，將導致整個鏈的斷開。

4.2.2 顯示鏈接

將鏈接文件各個物理塊的指針，顯示放到一張鏈接表中，該表在整個磁盤上僅設置一張。凡是屬於某個塊的第一個盤塊號，均作為文件地址被填入FCB的物理地址字段中，這種方式大大降低了訪問磁盤的次數。

由於分配給文件的所有盤塊號都放在該表中，故把該表稱為文件分配表FAT(File Allocation Table)；

4.2.3 FAT12

微軟早期的MS-DOS中，所使用的是12位的FAT12文件系統，後來為16位的FAT16文件系統。Windows95和98操作系統中則升級為32位的FAT32。Windows NT，2000，和XP系統又進一步發展為新技術文件系統NTFS（New Technology File System）。上述幾種文件系統，均採用顯示鏈接法。

對於FAT12文件系統，接下來我們計算其容量：

每個FAT表項位12位，因此最多允許的表項4096個。如果採用以盤塊作為基本分配單位，盤塊大小512B，因此FAT12最大支持的磁盤容量：4096*512B= 2MB。同時一個物理磁盤支持4個邏輯磁盤分區，所以相應的磁盤最大容量僅為8MB。（每個分區中都配又兩個文件分配表FAT1和FAT2，其中一個作為備份）隨著技術的發展，8MB的磁盤容量不能再滿足需求。

優化方法：

盤塊以簇為基本單位，簇是一連串的扇區，在FAT中它是一個虛擬的扇區，簇的大小一般為2n個盤塊。在MS-DOS的實際運用中，簇的容量可以僅有一個扇區（512B），兩個扇區（1KB），四個扇區（2KB），八個扇區（4KB）等。

因此，如果簇包含一個扇區，磁盤容量最大時8M；包含兩個扇區時，可以到達16M；包含8個扇區時，可以到達64M。

存在的問題：

a）以簇為基本的分配單位，能夠降低FAT表的項數，從而減少存儲空間的佔用，提高文件系統的效率。但這也會造成較大的簇內零頭。

b）允許的最大存儲空間有限，這是最大的問題；

4.2.4 FAT16

支持的最大磁盤空間：

FAT表項最大64 * 1024 = 65536，簇內允許的盤塊數為4，8，16，32，64，則FAT16最大支持的存儲空間：65536 * 64 * 512 = 2048M。

存在的問題：

FAT16對FAT12的侷限性有所改善，但改善很有限。當磁盤容量迅速增加時，如果再繼續使用FAT16，由此所形成的簇內碎片所造成的浪費也越大。如要求存儲空間到達8G，則每個簇大小為128K，這就意味著內部的零頭最大可達128K。

FAT12和FAT16均不支持長文件名，受到了8個字符文件名和3個字符文件擴展名的長度限制。為了解決該問題，Windows 95之後的系統，對FAT16進行了擴展，通過一個長文件名佔用多個目錄項的方法，使得文件名的長度可以長達255個字符，這種擴展的FAT16也稱為VFAT。

4.2.5 FAT32

FAT32是FAT系列文件系統的最後一個產品，每一簇在FAT表中的表項佔據4個字節，FAT32可以表示4*1024* 1024* 1024 （即4G個）。FAT32中採用較小的簇，每個簇都固定為4KB，即用8個盤塊代替FAT16的64個盤塊，每個盤塊仍為512B。FAT32最大支持的磁盤空間為4KB * 4G = 2TB。

三種文件系統的最大分區以及所對應的塊大小如下圖所示：

由上圖所述，FAT32比FAT16支持更小的簇和更大的磁盤容量，同一個磁盤容量，FAT32的磁盤簇以及簇內碎片更新，磁盤利用率更高。FAT32主要應用於Windows98及後續的Windows系統，它增強了磁盤性能，增加了可用磁盤空間，同時也支持長文件名。

存在的問題：

a）由於文件分配表比較大，運行速度比FAT16格式要慢；

b）FAT32有最小管理空間的限制，每個FAT32卷至少要有65537（64 * 1024）個簇，不支持容量小於512MB的分區。因此對於小分區，仍然需要FAT16或FAT12；

c）FAT32的單個文件長度也不能大於4GB；

d）不能保持向下兼容

4.2.6 NTFS

NTFS（New TechnoloFile System）是一個專門為Windows NT開發的，全新的文件系統，並適用於Windows 2000、XP、2003。特點如下：

（1）使用64位磁盤地址，理論上可以支持2的64次方字節的磁盤分區；

（2）NTFS中可以很好低支持長文件名，單個文件名限制在255個字符內，全路徑名為32767個字符；

（3）具有系統容錯功能，即在系統出現故障或差錯時，仍能夠保證系統正常運行。

（4）提供數據的一致性；

磁盤組織

同FAT文件系統一樣，採用簇作為磁盤空間分配和回收的基本單位。

文件組織

NTFS中，以卷為單位，將一個卷中的所有文件信息，目錄信息以及可用的未分配空間信息，都以文件記錄的方式記錄在一張主控文件表MFT（Master File Table）中。該表示卷結構的中心，從邏輯上講，卷中的每個文件都作為一條記錄，在MFT表中佔有一行，其中還包括MFT自己的這一行。每行大小固定1KB，每行稱為改行所對應文件的元數據，也稱為文件控制字。

在MFT表中，每個元數據將將其所對應的文件的所有信息，包括文件的內容等，都被組織在所對應文件的一組屬性中。對於元數據1KB的空間，可能記錄不下文件的全部信息，因此當文件較大時，元數據僅記錄該文件的一部分熟悉，其餘屬性，如文件的內容等，可以記錄到卷中的其他可用簇中。

存在的問題：

只能被Windows NT所識別，NTFS文件系統可以存取FAT等文件系統的文件，反之不行，因此NTFS缺乏兼容性。Windows的95，98，98SE和Me版本都不能識別NTFS文件系統。

4.3 索引分配

鏈接分配雖然解決了連續分配的問題，但又出現了下述另外兩個問題：

a）不能支持高效的直接存取，要對一個較大的文件進行直接存取，必須首先在FAT中順序查找許多盤塊。

b）FAT需要佔用較大的內存空間；

4.3.1 單級索引

打開一個文件，理論上只需要將該文件對應個盤塊號等信息調入內存即可，不需要將整個FAT表加載至內存。索引分配方法就是基於該想法所形成的一種分配方法。它為每個文件分配一個索引塊（表），再把分配給該文件的所有盤塊號都記錄在該索引塊中，因而該索引塊就是一個包含許多盤塊號的數組。建立一個文件時，只需要為之建立的目錄項中填上指向該索引塊的指針。如圖所示：

索引分配方式，支持直接訪問，當要文件的第i個盤塊時，可以方便地直接從索引塊中找到第i個盤塊的盤塊號。

存在的問題：

可能要花費較多的外存空間，用於保存索引塊的信息。對於小文件採用索引分配方式，其索引塊的利用率是極低的。

4.3.2 多級索引

當文件較大時，索引塊可能不止一個盤塊，這種場景下，使用單級索引效率是比較低的。作為優化，一般採用多級索引，二級，三級等。

加入每個盤塊大小為1KB，每個盤塊號佔4個字節，則在一個索引塊中可存放256個盤塊號。如此，兩級索引時，最多可包含的盤塊號總數：256 * 256 = 64K個盤塊。因此採用兩級索引，所允許的文件最大長度是64MB。倘若盤塊大小為4KB，則採用單機索引所允許的最大文件長度是4MB，兩級索引最大允許4GB。如果所示：

4.3.3 混合索引

unix系統中採用了這種方式，共設置13個地址項，即iaddr(0)-iaddr(12)，如圖所示：

（1）直接地址

iaddr(0)-iaddr(9)，存放直接地址，這裡的每項中所存放的是該文件數據所在的盤塊的盤塊號。如果每個盤塊4KB，當文件不大於40KB時，便可直接從索引節點中讀出該文件的全部盤塊號。

（2）一次間接地址

用iaddr(10)來提供一次間接地址，即一級索引，每個盤塊號如果4個字節，則最大允許的文件是4MB。

（3）多次間接地址

用iaddr(11)提供二次間接地址，即兩級索引，文件最大允許長度為4GB；同理，iaddr(12)三次間接地址，所允許的文件最大長度是4TB。
