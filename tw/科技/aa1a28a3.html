網絡專有名詞複習筆記

2020-10-29
| 科技

一、概述

Internet ：指當前全球最大的、開放的、有眾多網絡相互連接而成的特定計算機網路， 它採用TCP/IP協議族。

1、因特網：從硬件和軟件方面來說：數以百萬計的互聯的計算設備（主機=端系統，通信鏈路 communication link ，運行網絡應用）；從分佈式應用提供服務的聯網基礎設施：通信基礎設施使能分佈式應用，提供給應用通信服務。

2、協議：定義了在兩個或多個通信實體之間交換的報文格式和次序，以及在報文傳輸和/或接受或其他事件方面所採取的動作。一組控制數據通信的規則。

3、網絡組成：網絡邊緣（應用與主機）、網絡核心（路由器，網絡的網絡），接入網。

4、網絡邊緣：面向連接服務——TCP（ transmission Control protocol）：可靠的，有序的字節 流數據傳送 （丟包： 確認和重傳），採用流控制 （發送方不能過載接收方），擁塞控制（當網絡擁塞時發送方“降低發送速率”）。

5、網絡邊緣：無連接服務——UDP（User Data protocol ）用戶數據報協議，無連接，不可靠的數據傳送，無流控，無擁塞控制。

6、網絡核心：電路交換（circuit switching ）和分組交換（packet switching ）。

7、電路交換：為“呼叫”預留端到端資源，在電路交換網絡中，沿著端系統通信路徑，為端系統之間通信所提供的資源在 （緩存、鏈路傳輸速率）在通信會話期間會被預留。（非共享）。將鏈路帶寬劃為“片”，FDM和TDM。

8、FDM（ frequency-divisionmultiplexing ）頻分多路複用，該鏈路在連接期間為每條連接專用一個頻段。TDM（ time-divisionmultiplexing）時分多路複用，時間被劃分為固定區間的幀，並且每幀又被劃分為固定數量的時隙，一個時隙可用於傳輸該連接。

9、分組交換（統計多路複用statistical multiplexing）：每個端到端數據劃分為分組，分組交換使用按需的方式分配鏈路。

10、分組交換與電路交換的對比：分組交換允許更多的用戶使用網絡；對突發數據極為 有效，資源共享，較簡單，無呼叫建立。但是分組交換需要可靠數據傳送、擁塞控制協議。

11、分組交換——存儲轉發：從源到目的地通過路由器移動分組，幾種路徑選擇算法（數據報網絡，虛電路網絡）。

12、數據報網絡：分組中的目的地址決定下一跳，在會話中路由可以變化。不是面向連接的，而是無連接的。

13、虛電路網絡：每個分組攜帶標籤（虛電路ID），標籤決定下一跳，固定的路徑在呼叫建立時決定，在呼叫期間保持不變，路由器保持呼叫狀態。

14、分組交換中丟包和時延出現的原因：分組到達鏈路的速率超過輸出鏈路能力，分組 排隊，等待交換，如果無空閒緩存則到達的分組丟失，出現丟包現象。分組時延的4種 來源：（ 1）、 節點處理 （檢查比特差錯） ；（ 2）、 排隊 （等待輸出鏈路傳輸的時間，取決 於路由器擁塞的等級）；（ 3）、 傳輸時延 ： R=鏈路寬帶（bps）， L=分組長度（比特） ，發 送比特進入鏈路的時間=L/R；（ 4）、傳播時延 ：d=物理鏈路的長度，s=在媒體中傳播的速 度（ ~2*10 8m/sec ），傳播時延=d/s 。

15、協議分層： 每一層實現一種服務，經他自己的層內動作，依賴由下面層次提供的服務。分層是為了處理複雜系統，明確的結構使得能夠標識複雜構件的關係，模塊化易於維護、系統的更新。

16、因特網協議棧：應用（支持網絡應用， FTP，SMTP，HTTP）；運輸（主機到主機數 據傳輸， TCP，UDP）； 網絡 （從源目的地數據報的選路，IP，選路協議） ； 鏈路 （在鄰近 網元之間傳輸數據，PPP，以太網） ；物理 （“在線上 ”的比特）。

第二章 應用層

1、應用體系結構：客戶機/ 服務器，對等（P2P），客戶機 /服務器與P2P 的混合。

2、客戶機 / 服務器體系結構：服務器 （總是 打開的主機，永久的IP地址，可擴展為服務器池）；客戶機 （與服務器通信，可以間歇地連接，可以具有動態的IP 地址 ，彼此不直接地通信）。

3、純 P2P 體系結構：無總是打開的服務器，任意的端系統直接通信，對等方間歇地連接， 改變 IP地址。

4、混合：例——Napster （文件傳送P2P，集中式文件搜索：對等方在中心服務器上註冊內 容對等方查詢相同的中心服務器以定位內容）。

5、進程通信——進程：在一臺主機上運行的程序。客戶機進程（發起通信的進程），服務器 進程（等待聯繫的進程）。進程通過一個稱為套接字 的軟件接口在網絡上發送和接收報文。 套接字是同一臺主機內應用層與運輸層之間的接口。

網絡應用從運輸層協議獲得的服務：TCP和 UDP

6、TCP：面向連接 （客戶機和服務器之間所需的建立）；

可靠傳輸 （在發送和接收進程之間）；

流控制 （發送方不會淹沒接收方） ；

擁塞控制（當網絡過載時抑制發送方）；

並不提供定時，最小帶寬保證。

7、UDP：在發送進程及接收進程之間的不可靠數據傳輸，不提供建立連接建立、可靠性流控制、擁塞控制、定時或帶寬保證。

8、HTTP（超文本傳輸協議）和Web 應用：使用TCP，進程 端口為80。

HTTP連接分為：非持久連接 （每個請求 / 響應是經一個單獨的TCP連接發送）和持久連接 （所有的請求及相應的響應經相同的TCP連接發送）。

RTT（往返時延）：從客戶機到服務器發送一個小分組並返 回所經歷的時間。

9、HTTP 請求報文格式：請求行、首部行和實體主體。

方法類型：GET、 POST、 HEAD。

GET 和 POST的區別在於：使用GET方法時實體主體為空，使用POST方法時實體主體看到用戶 在表單中所填的內容。

10、 HTTP響應報文格式：

11、 WEB 緩存：滿足客戶機請求而不涉及起始服務器。

12、FTP（文件傳輸協議）：使用兩個並行的TCP連接來傳輸文件，一個是控制連接（帶外傳 送）， 端口號為21；一個是數據連接（帶內連接），端口號為20。

13、電子郵件：三個主要部分（用戶代理、郵件服務器和SMTP（簡單郵件傳輸協議））。

14、電子郵件： SMTP（簡單郵件傳輸協議），端口號25，使用持久連接， 要求報文以7bitASCII格式，交付 / 存儲到接收方服務器。

15、HTTP與 SMTP 比較 ：

HTTP：拉，每個對象封裝在其自己的響應報文中；

SMTP：推，多 個對象在多方報文中發送。兩者都有ASCII命令 / 響應交互，狀態碼。

17、MIME（ multipurpose Internet mail extension）多用途因特網郵件擴展：為發送非ASCII文本的內容，發送方的用戶代理必須在報文中使用附加的首部行。在報文首部的附加行聲明 MIME 內容類型。

18、郵件訪問協議：從服務器獲取郵件。POP3：郵局協議；IMAP：互聯網郵件訪問協 議。基於Web 的電子郵件使用的是HTTP協議，例如：Hotmail ，Yahoo 等。

19、DNS（ Domain Name System）域名系統： 一個由分層的DNS 服務器實現的分佈式數據庫，一個允許主機查詢分佈式數據庫的應用層協議，主要任務是實現主機名到IP 地址 轉換的目錄服務。

識別主機的方式：通過主機名或者IP 地址。 DNS 協議運行在UDP 上， 使用 53 號端口。

頂級域(TLD)服務器 ：負責com,org,net,edu 等，以及所有頂級國家域 uk,fr,ca,jp 。

權威 DNS 服務器 ：組織的DNS 服務器為組織的服務器提供對IP 的權威主機 名。查詢過程分為遞歸查詢和迭代查詢。從請求主機到本地DNS 服務器的查詢是遞歸的， 其餘的查詢是迭代的。

20、P2P 文件共享，在對等方區域中組織和搜索索引的3 種方法：

集中式索引（當用戶啟動 P2P 文件共享應用程序時，該應用程序將他的IP 地址以及可供共享的文件名稱通知索引服務器，該索引服務器從每個活動的對等方那裡收集這些信息，從而建立一個集中式的動態索引，將每個文件拷貝映射到一個IP 地址集合。

缺點是：單點故障，如果索引服務器崩潰， 則整個P2P 應用也就隨之崩潰；性能瓶頸和基礎設施費用）；查詢洪泛 （查 詢報文經現有的TCP連接，對等方轉發查詢報文，查詢命中經反向路徑發送）；層次覆蓋（結合前兩者的優點，與洪泛查詢類似，不使用專門的服務器來跟蹤和索引文件，但不同的是，在層次覆蓋設計中並非所有對等方都是平等的）

第三章 運輸層

1、運輸層協議為運行在不同主機上的應用進程之間提供了邏輯通信 （logic communication ）， 進程間的邏輯通信，端到端傳輸。運輸層協議是在端系統中而不是在網絡路由器中實現的。在發送方，運輸層將接收到的來自發送應用進程的報文轉換成運輸層分組，用因特網術語稱其為運輸層報文段（ segment ），可能的方法是，將應用報文劃分為較小的塊， 併為每塊加上一個運輸層首部來創建運輸層報文段。然後，在發送方端系統中，運輸層將這些報文段傳遞給網絡層，網絡層將其封裝成網絡層分組並向目的地發送。注意：網絡路由器僅作用於該數據報的網絡層字段，即他們不會檢查封裝在該數據報的運輸層報文段的字段。在接收方，網絡層從數據報中提取運輸層報文段，並將該報文段向上交給運輸層。

2、運輸層VS 網絡層 ：網絡層是主機間的邏輯通信；運輸層是進程間的邏輯通信，依賴、強化網絡層服務。

3、將主機間交付擴展到進程間交付，成為運輸層的多路複用（multiplexing ）和 多路分解（demultiplexing ）。

多路複用（ multiplexing）：從源主機的不同套接字中搜集數據塊，併為每個數據塊封裝上首 部信息從而生成報文段，然後將報文段傳遞到網絡層的工作。

多路分解（demultiplexing）：將運輸層報文段中的數據交付到正確的套接字的工作。

4、無連接分解： UDP 套接字由二元組標識（目的地IP 地址，目的地端口號）；當主機接收UDP 段時：在段中檢查目的地端口號，將UDP段定向到具有該端口號的套接字。

5、面向連接分解：TCP套接字由四元組標識（源IP 地址，源端口號，目的IP 地址，目的端 口號）；接收主機使用這四個值來將段定向到適當的套接字。

6、無連接傳輸——UDP：“盡力而為 ”服務，UDP段可能丟包或者對應用程序交付失序。首部只有4個字段 ，每個字段兩個字節（即共8個字節）。UDP檢驗和提供了差錯檢測功能，即檢驗和用於確定當UDP 報文段從源到達目的時，其中的比特是否發生了改變。

7、可靠數據傳輸的原理（停等協議和流水線協議）：不可靠信道的特點決定了可靠數據傳輸協議（rdt）的複雜性。使用有限狀態機（FSM）來定義發送方和接收方。

Rdt1.0：經可靠信道的可靠傳輸（底層信道非常可靠：無比特差錯，無分組丟失）。

Rdt2.0：具有比特差錯的信道，該數據傳輸協議採用了差錯檢測、肯定確認與否定確認。類似於 Rdt2.0 的協議被稱為停等協議（ stop-and-wait ）。自動重傳請求 （ automatic repeat request ， ARQ）協議。

Rdt2.1：如果 ACK/NAK 受損 ，將會出現發送方不知道在接收方發生了什麼情況，不能只是重傳，可能導致冗餘。處理冗餘：發送發對每個分組增加序列號，如果ACK/NAK 受損，發送方重傳當前的分組，接收方丟棄冗餘分組。

Rdt2.2：一種無NAK 的協議 ，代替NAK，接收方對最後正確接收的分組發送ACK。

Rdt3.0：具有差錯和丟包的信道。解決方法：發送方等待ACK 一段“合理的”時間，需要倒計時定時器 。其接收FSM 和Rdt2.2 的一樣。

8、流水線協議：發送方允許發送多個、“傳輸中的 ”、還沒有應答的報文段（序號的範圍必須增加，發送方和 / 或接收方設有緩衝） ；有兩種形式（回退N幀法 go-back-N和 選擇重傳S-R）。

9、Go-Back-N：對失序的分組 （丟棄，不緩存，沒有接收緩衝區；重新確認具有按序的分組）

10、 SR協議 通過讓發送方僅重傳那些他懷疑在接收方出錯的分組而避免了不必要的重傳。

窗口長度必須小於或等於序號空間大小的一半 。

11、TCP報文段結構 ：由首部字段（一般是20 字節）和一個數據字段組成。

MSS（ maximum segment size）最大報文段長；

MTU（ maximum transmission unit ）最大傳輸單元。 MSS 通常根據最初確定的最大鏈路層幀長度來設置。

12、TCP可靠數據傳輸：TCP在 IP 不可靠服務的基礎上創建可靠數據傳輸服務，流水線發送給報文段，累計確認，TCP使用單個重傳計時器，重傳超時事件和重複ACK 事件觸發。

快速重傳 ：如果對相同數據，發送方收到3 個 ACK，假定被確認的報文段以後的報文段丟失了，在定時器超時之前重傳。

13、 流量控制 ：發送方不能發送太多、太快的數據讓接收方緩衝溢出。

14、 擁塞控制 ：擁塞（太多的源發送太多太快的數據，使網絡來不及處理），表現為丟包和 長延時。擁塞控制的兩類方法（端到端的擁塞控制和網絡輔助的擁塞控制）。

15、 TCP擁塞控制 ：端到端控制。三個機制（AIMD 加增倍減算法、慢啟動和超時事件後的 保守機制）。慢啟動（當連接開始的時候，速率呈指數式上升，直到第一次報文丟失事件發 生為止或到達閥值Threshold）。

第四章 網絡層

1、關鍵的網絡層功能：轉發（forwarding ）和選路（routing ）。

轉發：將分組從路由器的輸入移動到適當的路由器輸出。

選路：決定分組從源到目的地所採用的路由。

2、 網絡層連接和無連接服務：數據報網絡提供網絡層無連接服務；虛電路網絡提供網絡層連接服務 。

3、 一條 VC 的組成：從源到目的地的路徑；VC 號是標誌沿路徑每條鏈路的號碼；沿路徑路由器中轉發表中的項。

4、 數據報網絡：無呼叫建立，路由器（沒有端到端連接的狀態），分組使用目的主機地址轉發（在相同源和目的對可能採用不同的路徑）。採用最長前綴匹配。

5、 路由器體系結構概述：路由器的兩個關鍵功能（運行選路算法 / 協議 RIP 、OSPF、BGP；從入鏈路到出鏈路轉發數據報）。

6、 IP 分片和重新組裝

7、 IP 編址 ： IP 地址（對主機、路由器接口的32bit 標識符）。採用無類型域間選路（CIDR， classless inter-domain routing任意長的地址的子網部分；地址格式 : a.b.c.d/x,其中x是地址子網部分的比特長度）。

8、子網：從主機或路由器分離每個接口，生成孤立網路的島，每個孤立的網絡被稱為一個子網。 IP 地址（子網部分——高階比特；主機部分——低階比特）。

9、主機得到一個IP地址—— DHC（P 動態主機配置協議，Dynamic Host Configuration Protocol ）：動態地從服務器得到地址，“即插即用 ”。

10、網絡從他的ISP的地址空間得到IP 地址的子網部分；ISP通過 ICANN 得到地址塊。

11、NAT（網絡地址轉換，network address translation ）

12、ICMP（互聯網控制報文協議）：用於主機和路由器彼此交互網絡信息。其功能時： 差錯報告；回聲請求/ 回答

13、IPV6：數據報格式（固定長度40 字節首部） 。與 IPv4 的變化： IPv6 沒有檢查和、選項， IPv6 不允許在中間路由器上進行分片與重新組裝。從 IPv4到 IPv6 的遷移，引入了隧道（在IPv路由器之間 IPv6 數據報作為IPv4數據報的負載） 。

14、選路算法：選路信息必須在兩個方向流動（從一組參與路由器到一個非參與路由器； 非參與路由器必須將隱藏網絡的信息傳給該組）。全局選路算法（所有路由器具有完全 的拓撲、鏈路費用信息，具有全局狀態信息的算法常被稱作鏈路狀態（LS）算法； 分佈式選路算法 （以迭代的、分佈式的方式計算出最低費用路徑，距離向量（DV） 算法））。

15、LS算法可能振盪。

16、DV 算法會增加毒性逆轉。

17、RIP（選路信息協議）：距離矢量算法；每30秒在鄰居之間經響應報文（又稱通告）交換；每個通告在AS中包括多大24 個目的網絡的列表。

18、OSPF（開放最短路優先）：使用鏈路狀態算法；公共可用；OSPF攜帶每個鄰居路由 器一個項；通告散佈到整個AS。

19、BGP（邊界網關協議）：為每個AS 提供了一種手段：從相鄰AS獲得子網可達性信 息；向AS內部的所有路由器傳播可達性信息；基於可達性信息和策略，決定到子網的 好路由。

20、廣播選路（ broadcast routing ）：網絡層提供了從一個源節點到網絡中的所有其他節 點交付分組的服務。

21、多播選路（ multicast routing ）：使單個源節點能夠向其他網路節點的一個子集發送分組的拷貝。

22、IGMP： Internet 組管理協議。

第五章 鏈路層

1、鏈路層服務：成幀（將數據報封裝進幀，加上首部和尾部）；

鏈路接入（位於幀首部的“ MAC”地址標識符、目的地）；

相連節點間的可靠交付 ；

流量控制（相鄰發送和接收 節點間的步調一致）；

差錯檢測 （差錯由信號衰減、噪聲所致， 接收方檢測差錯的存在， 信號發送方負責重傳或丟棄幀）；

糾錯 （接收方識別和糾正比特差錯，而不採取重傳）；

半雙工和全雙工（使用半雙工，鏈路的兩端節點能夠傳輸，但不能同時）。

2、差錯檢測：奇偶校驗，循環冗餘檢測

3、多路訪問協議：決定節點怎樣共享信道的分佈式算法，如何定何時節點能夠傳輸。

三大類： 信道劃分 （將信道劃分為較小的“段”——時隙、頻率和編碼；為節點分配一部分專用）；

隨機訪問 （不劃分信道，允許碰撞，從“碰撞”恢復）；

輪流 （節點輪流，但有更多信息要發送的能夠輪流的較長時間）。

4、信道劃分MAC協議： TDMA（時分多路訪問） 、 FDMA（頻分多路訪問） 。

5、隨機訪問MAC 協議：時隙ALHOHA；ALOHA；CSMA（載波偵聽多路訪問）；CSMA/CD（具有碰撞檢測的載波偵聽多路訪問）。

6、CSMA/CD：在短時間內檢測到碰撞，碰撞的傳輸流產，減少信道浪費

7、“輪流” MAC 協議 ： 輪詢協議 （主節點以循環的方式輪詢每個節點）； 令牌傳遞協議（當一個節點收到令牌時，僅當他有一些幀要發送時，他才持有這個令牌，否則，他立即向下一個節點轉發該令牌）。

8、鏈路層編址—— MAC 地址：長度6 字節，固定一個MAC 地址的前24 比特，讓公司自己為每個適配器生成後24 比特的唯一組合。

9、ARP（地址解析協議） 。

10、 交換機 ：鏈路層設備，存儲並轉發以太網幀；檢查幀首部並基於MAC 目的地址選擇性地轉發幀；當幀在網段上轉發時，使用CSMA/CD 訪問網段；透明，主機不知道交換機的存在；即插即用，自學習，交換機不必配置。
