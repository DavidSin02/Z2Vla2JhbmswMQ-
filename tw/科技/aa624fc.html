實戰PyQt5: 083-圖形視圖框架中的座標系

2020-10-29
| 科技

圖形視圖基於笛卡爾座標系，圖元在場景中的位置和幾何形狀由x座標和y座標兩個數字表示。使用未做變換的視圖觀察場景時，場景中的一個單位等於屏幕上的一個像素。

注意：“圖形視圖”使用Qt的座標系，因此不支持反轉的Y軸座標系（向上增長）。

圖形視圖中有三個有效的座標系在起作用：圖元座標，場景座標和視圖座標。為了簡化實現，“圖形視圖”提供了方便的函數，允許在三個座標系之間進行映射。渲染時，圖形視圖的場景座標對應於QPainter的邏輯座標，視圖座標與設備座標相同。

圖片來源(doc.qt.io)

圖元(Item)座標

圖元位於它們自己的座標系中。它的座標都以點(0,0)為中心點，這也是所有變換的中心點。在圖元座標系中的幾何圖元，經常被稱為圖元點，圖元線，圖元矩形。

當創建一個自定義的圖元時,必須考慮圖元的座標。QGraphicsScene與QGraphicsView可以執行所有轉換，這使得實現自定義的圖元變得容易。例如，假如收到鼠標按下或是拖動輸入事件，事件的位置以圖元座標的形式給出。當某個點的位置在指定的圖元內時，QGraphicsItem.contain()返回True,否則返回False。這個點參數使用圖元座標，相似地，圖元的邊界矩形與形狀也使用圖元座標。

圖元位置指的是圖元的中心點在其父親的座標系中的座標，有時稱其為父座標。從這個意義上說，場景被視為所有無父親圖元的‘父圖元’，因此頂層的圖元位置就是在在場景座標中的位置。

子座標是相對於父座標的，假如子圖元未經變換，則子座標與父座標之間的差值等於在父座標系下，父圖元與子圖元之間的距離。例如，假如一個未經變換的子圖元位置與其父圖元的中心重合，那麼這兩個圖元的座標系統完全相同。如果子圖元的位置是（10，0），那麼子圖元座標系中的(0,10)點，對應於父座標系中的（10，10）點。

因為圖元的位置與變換是相對於父圖元的，子圖元的座標不會被父親的變換影響，儘管父圖元的變換隱含地對子圖元做了變換。在上面的例子中，即使父圖元旋轉，縮放，子圖元的(0,10)點依然對應於父圖元的(10,10)點。然而，相對於場景來講，子圖元會遵循父圖元的變換。假如父圖元被縮放(2X,2X),子圖元 的位置在場景中的座標是（20，0），它的（10，0）點則與場景中的（40，0）對應 。

除QGraphicsItem.pos()函數以外，QGraphicsItem的函數在圖元座標中操作，例如一個圖元的邊界矩形總是以圖元座標的形式給出。

場景(Scene)座標

場景代表所有圖元的基本座標系,場景座標系統描述了每個最頂級圖元的位置，也是從視圖向場景傳遞所有場景事件的基礎。場景中的每個圖元有場景位置與邊界矩形（QGraphicsItem::scenePos(),QGraphicsItem::sceneBoundingRect()), 另外，它有自己本地圖元位置與邊界矩形。場景位置描述了圖元在場景座標下的位置，它的場景邊界矩形則用於QGraphicsScene決定場景中哪塊區域發生了變化。場景中的變化通過QGraphicsScene.changed()信號來通知，它的參數是場景矩形列表。

視圖(View)座標

視圖座標是部件的座標，視圖座標中每個單位對應一個像素。這種座標的特殊之處在於它是相對於部件或是視口的，不會被所觀察的場景所影響。QGraphicsView的視口的左上角總是（0，0），右下角總是(視口寬，視口高）。所有鼠標事件和拖放事件最初都是作為視圖座標接收的，因此需要將這些座標映射到場景座標以便與圖元進行交互。

座標映射

通常在處理場景中的圖元時，要在場景與圖元之間，圖元與圖元之間，視圖與場景之間進行座標映射。舉例來講，當在QGraphicsView的視口中點擊鼠標時，應該通過調用QGraphicsView.mapToScence()與QGraphicsScene.itemAt()來獲知光標下是場景中的哪個圖元。假如想獲知一個圖元位於視口中的什麼位置，應該先在圖元上調用QGraphicsItem.mapToScene(), 然後調用QGraphicsView.mapFromScene()。最後，假如想知道在一個橢圓中有哪些圖元,則可以將QPainterPath傳遞到mapToScene(),然後再把映射後的路徑傳遞給QGraphicsScene.items()。

演示代碼

使用多種顏色繪製螺旋線，操作下方的滑動條，可旋轉這個彩色的螺旋線。完整代碼如下:

import sys,mathfrom PyQt5 import QtCore, QtGui, QtWidgetsfrom PyQt5.QtCore import Qt, QPointFfrom PyQt5.QtGui import QPen, QBrush, QFont, QTransformfrom PyQt5.QtWidgets import (QApplication, QWidget, QGraphicsScene, QGraphicsView,                             QVBoxLayout, QSlider, QGraphicsLineItem) class DemoGvCoordinate(QWidget):    def __init__(self, parent=None):        super(DemoGvCoordinate, self).__init__(parent)                   # 設置窗口標題        self.setWindowTitle('實戰PyQt5: Graphics View 框架座標系統演示')              # 設置窗口大小        self.resize(480, 480)              self.initUi()            def initUi(self):        #場景部分        scene = QGraphicsScene()                        scene.setBackgroundBrush(QBrush(Qt.black))                #繪製螺旋線        colors=[Qt.red, Qt.darkMagenta, Qt.blue, Qt.green, Qt.yellow, Qt.darkCyan]        x0 = 0        y0 = 0        ratio = 0.35        deg = 59 * math.pi / 180        for i in range (360):            line = QGraphicsLineItem()            line.setPen(QPen(colors[i%6]))            x1 = x0 + i * ratio * math.cos(i * deg)            y1 = y0 + i * ratio * math.sin(i * deg)            line.setLine(x0, y0, x1, y1)            scene.addItem(line)            x0 = x1            y0 = y1                      self.view = QGraphicsView()        self.view.setScene(scene)                #滑動條        slider = QSlider(Qt.Horizontal, self)        slider.setRange(-360, 360)        slider.setPageStep(10)        slider.setValue(0)        slider.valueChanged.connect(self.onRotateValueChanged)                layout = QVBoxLayout()        layout.addWidget(self.view)        layout.addWidget(slider)                self.setLayout(layout)        def onRotateValueChanged(self, value):        #是個累積效應，先對變化矩陣進行復位操作        self.view.setTransform(QTransform())        self.view.rotate(value)    if __name__ == '__main__':    app = QApplication(sys.argv)    window = DemoGvCoordinate()    window.show()    sys.exit(app.exec())
