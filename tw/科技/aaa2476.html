「五分鐘讀懂」芯片常用數字接口（5） QSPI

2020-10-29
| 科技

【QSPI概述】

QSPI接口，是Queued SPI的縮寫。和之前談到的SPI一樣都是出自Motorola。QSPI在SPI基礎上做了一些增強，且向下兼容SPI。QSPI相對SPI最顯著的差異就是增加了發送接收數據隊列，Queued的稱呼就是這麼來的。這樣做的好處就是，無需每次數據傳輸都需要CPU參與，可以降低CPU的資源佔用。

【QSPI應用框圖】


圖1 QSPI應用框圖

圖1為QSPI應用框圖，同SPI最明顯的區別就是多了隊列Buffer和片選變為4個。QSPI通過片選直連的方式，最多可接4個外設，如果外掛4-16譯碼器可以擴展至16個外設。

【QSPI接口信號】

表1 QSPI接口信號

如表1所示，QSPI在SPI原有4根信號基礎上增加了4個外設片選信號PCS0~3，其中PCS0同SSn複用。因此QSPI總共7根信號線。

【QSPI新增功能】

QSPI是在SPI基礎上進行擴展，其包含了SPI所有特性。其和SPI相同的特性，本文不再贅述，感興趣的同學請點鏈接（常用數字接口（2） SPI ）。QSPI相對於SPI新增如下功能：

1， 可編程Buffer

圖2 QSPI 可編程Buffer

如圖2所示，QSPI提供總共80字節的可編程Buffer。其中Tx Data Buffer 32字節，Tx CMD Buffer 16字節，Rx Data Buffer 32字節。Rx Data buffer存儲接收數據， Tx Data buffe存儲發送數據，Buffer深度均為16深度，數據位寬取決於單次訪問位寬配置，配置範圍8~16bit。Tx CMD Buffer存儲發送命令，Buffer深度為16，位寬8bit。

圖3 QSPI Tx CMD Buffer存儲命令格式

Tx CMD Buffer每條命令格式如圖3所示。

CONT：片選保持使能。1’b1，本次傳輸完成後PCS[3:0]輸出保持本次傳輸時的狀態；1’b0，PCS[3:0]輸出與之複用的GPIO輸出配置值。

BITSE：傳輸長度控制。1’b1，根據配置長度傳輸（可配置8~16bit）；1’b0，固定傳輸8bit長度。

DT：每次傳輸完成後是否等待。1’b1，本次傳輸完成後等待一個可配置的時間；1’b0，不等待。

DSCK：PCS和SCK相對相位調整。1’b1，從PCS有效到SCK延遲一定時間（時間可配置）；1’b0，固定為1/2SCK延遲。

PCS[3:0]: QSPI外設片選信號狀態。當PCS0~3一對一連接至從設備時， QSPI Master通過該命令可以選擇訪問4個外設中的哪個。當PCS0~3外接4-16譯碼器時，QSPI Master可以管理最多16個從設備。

Tx CMD Buffer和Tx Data Buffer共享指針，即Tx CMD Buffer中的命令和Tx Data Buffer中的數據一一對應。通過配置Tx CMD Buffer可以定義了每一次發送數據的行為，如發送數據長度、發送間隔進行定製。這種針對每次數據傳輸定義不同訪問間隔、訪問長度的方式，在訪問不同類型外設時可以大大提高訪問效率。

2， 隊列指針可編程

如圖2所示，QSPI依據ENDQP（隊列尾指針）和NEWQP（隊列首指針）的配置，從NEWQP指定的行開始，按照Tx CMD Buffer對應行中指定的行為發送Tx DATA Buffer中對應行的數據，直到發送/執行到ENDQP指定的行結束。該過程中，CPU僅需在發送開始前對Buffer中內容進行編輯，之後無需佔用CPU資源由QSPI自動完成。當ENDQP比NEWQP小時，QSPI先從NEWQP對應的行開始執行，當執行到第15行時卷繞至第0行，然後再遞增執行到ENDQP行停止。

3， 自動回捲（Wraparound）功能

該功能使能後，QSPI按照EDNQP，NEWQP確定的傳輸隊列重複發送數據。這種功能在對外設進行定時掃描時非常有用。

4， 連續傳輸模式

該模式使能時，QSPI可在一次數據傳輸中，進行多達256bit的數據傳輸。

從上述4條增強功能可以看出，QSPI所有新增功能都是圍繞儘量降低CPU資源佔用的理念進行的。

【後記】

在寫這篇文章時，順手在網上搜了一下QSPI，發現大部分提到Queued SPI的文章講的都是Quad SPI。Quad SPI是SPI的四線模式，一般用於SPI Flash的訪問，而Queued SPI是隊列SPI，這倆不是一回事兒。
