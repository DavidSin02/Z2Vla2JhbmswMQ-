自動生成源代碼工具-MoliCode介紹

2020-11-14
| 科技

前言

作為一個Java開發人員，或者前端開發人員，我們常常需要生成重複的代碼。如最常見的CRUD(增刪改查)功能前後臺的代碼。這些都是模板化的，重複性的操作，沒有任何的業務意義，非常的耗時。

在目前市面上，也有很多的代碼生成工具，但是我經常發現現在的代碼工具存在的問題：

  1. 和業務系統耦合在一起：如jeecg, 其生成的代碼是和其基礎工程有很強的代碼耦合性，離開了其基礎工程很難通用；
  2. 不提供模板市場：一般模板都是內置，沒有一個可以共享的模板市場可以下載和共享自己的代碼模板；
  3. 模板開發不友好：不太清楚其模板開發方式；

所以，MoliCode 雲模板代碼工具正是為了解決以上問題而應運而生，讓我們一起來了解一下。

MoliCode自動代碼工具簡介

MoliCode 是基於 java 開發的一種代碼生成器，當然其除了生成 Java 代碼亦可生成其它代碼。 它和其它代碼生成器的最主要區別在於，其關注點不在模板本身。 MoliCode不關注生成了什麼樣的代碼，而是作為一個基礎工具方便大家編寫和共享使用代碼模板。為了方便大家管理和分享代碼模板，建議將模板託管到git倉庫，使用的時候只需要知道git地址和分支即可。

編寫模板雖然花點時間，但是可以被重複利用，節省自己和大家的工作時間。任何重複性的工作按道理都可以代碼模板來解決，期望大家一起發揮自己的才智。

開發模板的原則

  1. 通用性：不要和自己的公司業務邏輯耦合太深，這樣不方便別的公司開發人員使用；
  2. 共享性：代碼模板在節省了自己的時間的同時，最好能共享出來，節省大家的時間，一起享受快速編碼的快感；
  3. 充分說明：代碼模板如何用，和哪個基礎工程使用需要詳細說明，減少別人使用的疑惑；
  4. 適度配置：給與適度的可配置功能，方便他人進行配置；

它是如何工作的？

事實上，moliCode 的兩個核心模塊是：數據模型處理器 + 模板資源包。

  1. 數據模型處理器：主要用於對數據進行加工處理，如將 XML 文本解析為 tableModel, JSON 文本轉換為 FastJson 對象。
  2. 模板資源包：按一定規則組織配置文件和模板文件, 並推送存儲在 git 倉庫。使用時，只需要知道 git 地址並拉取相關資源，配置到相關路徑即可。
  3. 通過交互式界面，配置相關信息，然後選擇相關模板即可一鍵化生產目標代碼。

數據模型處理器

在讀取到源數據之後，需要對基本的原始數據進行處理，以方便模板能方便讀取和使用。 其中最重要的是依據用戶前臺交互界面選擇的數據模型類型，選擇合適的數據模型處理器。

除工具自帶的 數據庫表模型、Java 源碼模型、JSON 數據模型、XML 數據模型、CSV 文件 等基礎的數據處理模型之外，代碼模板資源包開發人員可以通過約定的數據處理腳本進行數據處理。 以此來擴展不同的，定製化的數據模型處理。

模板資源包

一種按約定組織的代碼模板資源包，用於將多個模板文件集中存儲在一個文件夾下，同時該資源包擁有代碼模板的描述性 XML 文件，用於描述模板在該文件夾中的相對路徑和輸出信息。 在使用模板資源時，只需要從 git 倉庫中拉取代碼模板，解析倉庫中的模板描述性 XML 文件生成前臺交互式界面。
