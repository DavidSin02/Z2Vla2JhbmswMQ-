java8的Stream全解析

2020-11-14
| 科技

Lambda表達式

先從基礎的lamda表達式開始講起

java8新增的語言級特性,和javascript等函數式編程語言不同。在java中，lambda表達式依然是一個對象。它必須依附於一種特殊的對象類型functional interface。（稱為方法引用或者函數式接口）

語法

(arg1, arg2...) -> { body }(type1 arg1, type2 arg2...) -> { body }
  1. 簡單的說，可以看成是沒有訪問修飾符、返回值聲明和名字的方法
  2. 參數類型可以省略，自動推斷
  3. 只有一個參數時，（）可以省略
  4. 匿名函數的返回類型與代碼塊的返回類型一致，若沒有返回則為空
  5. 函數體，只有一條語句時，可省略{}
  6. 每個 Lambda 表達式都能隱式地賦值給函數式接口比如 Runnable就是一個函數式接口(用@FunctionalInterface註解修飾)Runnable r = () -> System.out.println("hello world");
  7. 當不指明函數式接口時，編譯器會自動解釋這種轉化new Thread( () -> System.out.println("hello world")).start();

雙冒號(::)操作符

另一種將常規方法轉化為 Lambda 表達式的方法

與匿名類的區別

一大區別在於關鍵詞的使用。

對於匿名類，關鍵詞 this 解讀為匿名類，而對於 Lambda 表達式，關鍵詞 this 解讀為使用 Lambda 的外部類。

另一不同在於兩者的編譯方法。

Java 編譯器編譯 Lambda 表達式並將他們轉化為類裡面的私有函數，它使用 Java 7 中新加的 invokedynamic 指令動態綁定該方法


方法引用或函數接口

上面提到了一個註解FunctionalInterface,可翻譯為方法引用或函數接口.

java8新增,用於指明該接口類型聲明是根據 Java 語言規範定義的函數式接口。函數式接口只能有一個抽象方法

//定義一個函數式接口@FunctionalInterfacepublic interface WorkerInterface {  public void doSomeWork();}public class WorkerInterfaceTest {public static void execute(WorkerInterface worker) {  worker.doSomeWork();}public static void main(String [] args) {  //使用匿名類的寫法  execute(new WorkerInterface() {      @Override      public void doSomeWork() {          System.out.println("Worker invoked using Anonymous class");      }  });  //使用lambda表達式  execute( () -> System.out.println("Worker invoked using Lambda expression") );}}



常用API

Collection接口的

Collection接口提供了 stream()方法

我們執行的任何操作都不會對源集合造成影響，你可以同時在一個集合上提取出多個 stream 進行操作。

靜態方法

of

構造一個Stream對象

Stream<String> s1 = Stream.of("a", "b");
empty

創建一個空的 Stream 對象。

contact

連接兩個 Stream ，不改變其中任何一個 Steam 對象，返回一個新的 Stream 對象。

Stream<String> s1 = Stream.of("a", "b");Stream<String> s2 = Stream.of("c", "d");Stream<String> s3 = Stream.concat(s1, s2);
generate

創建一個無限Stream，一般用於隨機數生成

Stream<Double> s5 = Stream.generate(Math::random);
iterate

創建一個無限Stream。可以添加初始元素和生產規則

Stream<Integer> s4 = Stream.iterate(1, n -> n + 2);


實例方法

返回Stream的
peek

對其中每個元素進行處理，返回的是一個新的Stream

主要用於debug使用，打印下當前元素

map

一般用這個對每個元素進行處理。比如從一個類型，轉換為另一個類型

List<String> stringList = integerStream.map(n -> "我是" + n).collect(Collectors.toList());
peek和map的區別

peek一般只用於debug，打印一下信息使用

peek的參數的Consumer接口,它是沒有返回值的

@FunctionalInterfacepublic interface Consumer<T> {  void accept(T t);

map的參數是Function接口,必須給返回值

@FunctionalInterfacepublic interface Function<T, R> {  R apply(T t);
還有一個疑問
integerStream.peek或者map(item ->        System.out.println("========="));

這一句話的打印是不會執行的，沒有後面的collect等終止就不會執行.

也就是說，流方法真正執行是在終止方法觸發

mapToInt

將元素轉換成int類型，後面一般跟sum,max，min,average

mapToLong

略

mapToDouble

略

limit

限制個數

distinct

去重功能。判斷是根據元素的equals方法和hashCode方法。(兩個都需要重寫)

基本元素

Stream<Integer> integerStream = Stream.of(2, 5, 100, 5);List<Integer> collect = integerStream.distinct().collect(Collectors.toList());System.out.println(JSONUtil.toJsonStr(collect));

[2,5,100]

對象

@Getter@Setterclass User {  private String name;  private int age;}@Testpublic void distinct() {  User a = new User();  a.setName("yun");  a.setAge(20);  User b = new User();  b.setName("yun");  b.setAge(20);  Stream<User> userStream = Stream.of(a, b);  List<User> userList = userStream.distinct().collect(Collectors.toList());  System.out.println(JSONUtil.parse(userList));}

只寫了getter/setter方法時

[{"name":"yun","age":20},{"name":"yun","age":20}]

加上@EqualsAndHashCode

[{"name":"yun","age":20}]


sorted

排序,基本元素可以使用默認排序方法

Stream<String> strStream = Stream.of("ba", "bb", "aa", "ab");strStream.sorted().forEach(item -> System.out.println(item));

aa ab ba bb

也可自定義排序方法

自定義根據第二個字母排序

      Stream<String> strStream = Stream.of("ba", "bb", "aa", "ab");      Comparator<String> comparator = (x, y) -> x.substring(1).compareTo(y.substring(1));      strStream.sorted(comparator).forEach(item -> System.out.println(item));

ba aa bb ab

filter

過濾

Stream<Integer> integerStream = Stream.of(2, 5, 100, 5);integerStream.filter(item -> item > 10).forEach(item -> System.out.println(item));

100


終止類型的方法
max

獲取Stream中的最大值

Stream<Integer> integerStream = Stream.of(2, 5, 100, 5);Integer max = integerStream.max(Integer::compareTo).get();
min

獲取最小值

findFirst

獲取第一個元素

Integer i = integerStream.findFirst().get();
findAny

隨機獲取一個元素。串行情況下，就是第一個。並行不一定，先獲取誰就是誰。

count

返回流中元素個數

long count = integerStream.count();


collection

將最終的Stream彙總為collection,Collectors已經為我們提供了很多拿來即用的收集器。經常用到Collectors.toList()、Collectors.toSet()、Collectors.toMap()。

另外高級用法還有比如Collectors.groupingBy()用來分組

// 返回 userId:List<User>Map<String,List<User>> map = user.stream().collect(Collectors.groupingBy(User::getUserId));// 返回 userId:每組個數Map<String,Long> map = user.stream().collect(Collectors.groupingBy(User::getUserId,Collectors.counting()));
toArray

collection是返回列表、map 等，toArray是返回數組

      Stream<Integer> integerStream = Stream.of(2, 5, 100, 5);      // Object[] objects = integerStream.toArray();      Integer[] toArray = integerStream.toArray(Integer[]::new);

如果無參,是返回Object數組.可以加參數Integer[]::new,可以返回Integer數組

forEach

也是對每一個元素進行處理

和map的區別是，forEach不會返回Stream，直接消費掉了

forEachOrdered

功能與 forEach是一樣的，不同的是，forEachOrdered是有順序保證的，也就是對 Stream 中元素按插入時的順序進行消費。

在使用並行的時候，兩者會有區別。

reduce

具體可以學習map/reduce計算模型.

Stream<Integer> integerStream = Stream.of(1,2,3);      Integer sum = integerStream.reduce(100, (x, y) -> x + y);      System.out.println(sum);

提供初始值100,然後開始累加

106

我們直接使用reduce較少,但是Collectors好多方法都用到了 reduce，比如 groupingBy、minBy、maxBy等等。


並行方法

創建並行Stream

Stream.of(1,2,3).parallel();或List<Integer> src = Arrays.asList(1, 2, 3);Stream<Integer> integerStream = src.parallelStream();

並行 Stream和普通Stream支持的api基本一樣

並行 Stream 默認使用 ForkJoinPool線程池，當然也支持自定義，不過一般情況下沒有必要。ForkJoin 框架的分治策略與並行流處理正好契合。

雖然並行這個詞聽上去很厲害，但並不是所有情況使用並行流都是正確的，很多時候完全沒這個必要。

比如

      Stream<Integer> integerStream = Stream.of(1,2,3).parallel();      Integer sum = integerStream.reduce(100, (x, y) -> x + y);      System.out.println(sum);

這個結果就變成306了
