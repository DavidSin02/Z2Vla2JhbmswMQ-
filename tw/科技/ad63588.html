01、內部存儲器

存儲體系

存儲系統——是計算機的重要組成部分,用來存儲計算機工作需要的信息（程序和數據）的部件，構成計算機的信息記憶功能。

存儲體系：由內部存儲器和外部存儲器兩個層次構成。

CPU、內存儲器、外存儲器的系統連接關係：

內存儲器

通過總線或存儲器專用通道連接；

均為半導體存儲器，但分為隨機存儲器和只讀存儲器兩種；

外存儲器

通過接口連接到主機系統；

分為磁性存儲器、光存儲器和半導體存儲器三種。

內部存儲器的作用及主要技術參數

▶ 內部存儲器：CPU通過總線或存儲器通道，按照地址訪問（讀取或寫入），訪問速度很快（相比訪問外部存儲器）。根據不同的用途，可以用RAM（隨機存儲器）或ROM（只讀存儲器）實現，RAM和ROM分別佔據不同的地址區段。

▶ 內部存儲器訪問：

◿ 讀取——將存儲器指定地址的一個或多個字節傳送到CPU；

◿ 寫入——將CPU中一個或多個字節傳送到存儲器指定地址。

▶ RAM的作用

◿ 程序運行，由外部存儲器傳送到RAM；

◿ 運行數據，程序中的數據部分；

◿ 與外部設備間數據交換的緩衝存儲。

▶ ROM的作用

◿ 固化在硬件中的基本輸入輸出程序；

◿ 硬件相關的初始數據、靜態數據。

● 存儲容量：存儲器可以容納的二進制信息量稱為存儲容量。以字節(B:Byte)為單位。

1KB = 1024B

1MB = 1024KB = 1,048,576B

1GB =1024MB = 1,048,576KB = 1,073,741,824B

● 速度：讀取時間=從發出讀出命令到信息穩定在存儲器輸出端的時間，一般單位為ns(納秒，10-9秒)。

DRAM芯片一般為幾十ns，SRAM芯片為幾ns。

其他速度包括寫入速度（一般與讀取速度差別不大）、擦除速度等。

● 突發訪問支持：向存儲器發出一次地址，可以連續讀/寫多個字節。目前由DRAM芯片構成的內存條（模塊）突發傳送模式下讀寫速度可以達到幾個ns。

● 帶寬：（存儲器位數/8）×讀取速度峰值，單位為MB/s。

● 錯誤校驗：內存在讀寫過程中檢測和糾正錯誤的能力，常用的錯誤校驗方式有奇偶校驗和ECC兩種。

◿ 奇偶校驗（Parity）：一般每個字節增加一位，用於奇校驗或偶校驗，只有檢錯能力。

◿ ECC：一般每64位增加8位。具有數據檢錯和糾錯能力，ECC可以糾正存儲器訪問的絕大多數錯誤。

半導體存儲器的組成及工作原理

● SRAM

SRAM芯片：內部由存儲矩陣、地址譯碼器、存儲控制邏輯和雙向緩衝器組成。

WR: 寫控制 (WRite)

OE: 輸出允許 (Output Enable)

CE: 片選 (Chip Enable)

● DRAM

DRAM的位存儲電路：為MOS管+電容器動態存儲電路，其記憶信息的機理是依靠電容器C存儲電荷的狀態，電容器C有電荷時，為邏輯“1”，沒有電荷時，為邏輯“0”。

單元讀寫：只有行選擇信號和列選擇信號同時有效時才選中該存儲單元，再根據數據線狀態和控制電路完成對電容電壓的讀取（讀）或對電容的充放電（寫）。

刷新：由於電容器存在漏電，因此需要定期對電容器充放電，每隔一定時間（一般2ms左右的刷新週期）就要刷新一次。

DRAM芯片結構：存儲陣列為多頁面結構，地址線為行地址和列地址分別傳送，由行選通（RAS）信號和列選通（CAS）信號控制。數據線分為輸入和輸出，WE有效為寫，無效為讀。

● ROM

◿ 只讀存儲器ROM：

信息只能讀取，也不會在掉電時丟失。

◿ ROM器件的特點：

只讀，寫入需要先擦除信息，再通過特殊操作寫入；

結構簡單，所以密度高，且成本低；

非易失，可靠性高；

相比較DRAM和SRAM，讀取速度慢。

◿ ROM的技術發展過程：

掩膜ROM → PROM → EPROM → EEPROM → Flash Memory

02、總線與芯片組

總線——概念

總線的定義——是嚴格定義一組的信號線的集合，用於實現計算機各部件之間的信息傳輸的公共通道。

總線特徵——公共性、標準性、可擴展性。

總線連接——通過總線可以實現點對多點連接（主從連接）或多點連接。只能實現點對點連接的通道概念上不是總線，但習慣上可能稱作總線。

總線信息

數據——傳輸的數據內容

地址——傳輸的存儲位置，如存儲器地址、端口地址等

控制——傳輸對類型，如存儲器讀/寫、端口讀/寫等

其他——時鐘、復位、電源等

總線寬度指的是可以同時傳送的數據位數。如USB、SPI、I2C位寬是1，FSB是64位，PCI是32或64位。

● 總線時鐘頻率總線的傳送時鐘頻率。總線的最高時鐘頻率可能隨著版本的提高而增加。

03、接口與通信設備

● 接口——用於完成計算機主機系統與外部設備之間的信息交換。接口由接口硬件（接口電路、連接器、連接電纜等)和接口軟件(程序)組成。

● 接口的指標

→ 傳輸速度、傳輸距離、傳輸媒介類型；

→ 可靠性（差錯控制） 、安全性（保密）；

→ 資源佔用（包括CPU時間、中斷、DMA等）；

→ 使用方便性和高層協議功能。

接口的功能

● 接口的基本功能

→ 數據傳送：CPU執行輸入/輸出指令與外部設備交換數據。

→ 數據緩衝：用於實現輸入輸出過程中的暫存。接口電路中存儲機制可以是寄存器（1~2個字節）、FIFO存儲器（數十到數百字節）、或是普通數據存儲器。

→ 信號變換：完成計算機數字信號與I/O設備信號（如模擬信號、開關信號、計數脈衝等）的相互轉換。

→ 中斷：接口電路一般有中斷功能，以提高接口程序的效率。

● 接口的高級功能

→ 差錯控制：實現檢錯或糾錯。

→ 高層通信協議：實現呼叫、數據加密、流量控制等。

→ 即插即用、電源管理、動態配置等。

接口軟件與硬件

● 接口軟件、I/O總線、接口電路和連接的外部設備的關係

● 接口電路構成

● 接口電路的構成——控制和狀態寄存器

由地址譯碼器確定控制和狀態寄存器地址，不同的接口電路佔用不同的I/O端口地址。

● 數據緩衝存儲

接口電路在發送和接收過程中，都有緩衝存儲機制。多數緩存機制為FIFO方式。

● 數據變換機制

數據變換可以是電平變換（如RS-232串行接口）、脈衝變換（如紅外接口）、差分變換（如USB），也可以是數模變換（如音頻）等。

● 接口的操作：程序對接口的訪問（讀/寫）方式。不同的接口電路支持不同的操作方式。

查詢控制方式：在程序的主動控制下，通過讀取狀態寄存器瞭解接口的情況，完成相應的程序操作。為了及時瞭解接口的狀態，需要時間密集的查詢操作。CPU效率低。


中斷控制方式：當接口出現需要程序干預的事件，通過中斷通知CPU，CPU再讀取狀態寄存器，確定事件的種類，以便執行不同的代碼處理。CPU效率高而且及時。


DMA（Direct Memory Access）控制方式：CPU與接口的數據傳送採用DMA傳送，即傳送的具體過程由硬件（DMA控制器）完成，傳送速度比通過CPU快，尤其是在批量傳送時效率很高。


串行接口

● 串行接口——RS-232C基本特性

RS-232C接口：

→ 傳輸速率：75~115,200（128,000）b/s；

→ 傳輸距離：最長電纜15米（低速率時）；

→ 電纜/連接器：採用9線或25線信號傳輸，當前多用9線。採用D型連接器；

→ 協議簡單：無高層協議；

→ 在Pentium 4時代作為PC的標準配置，目前在PC上的應用逐漸減少。


● 串行接口——RS-232C信號

異步串行傳輸：異步串行傳輸中以字節為傳輸單位，傳輸一個字節包括起始位、數據位和終止位。

● RS-232C信號電平：採用雙極方式，負電平（-3 ~ -15V）代表邏輯1，正電平（+3 ~ +15V）代表邏輯0。一般採用±5V 或±12V。

● 串行接口——RS-232C傳輸參數

RS-232C接口應用參數：在使用RS-232C通信之前必須設置接口參數，包括速率、數據位數、校驗、停止位數和流控方式等。Windows下從“我的電腦”—“設備”— “端口”可以查看串行接口參數。

● 串行接口——RS-232C的接口信號

RS-232C接口信號線：連接的兩端分別為DTE（ Data Terminal Equipment ）和DCE（ Data Communication Equipment ）端。

● 串行接口——USB擴展RS-232C

通過USB擴展RS-232C：現代計算機大多已經沒有RS-232-C接口，但可以通過USB接口擴展，實現與RS-232C接口的設備連接。安裝驅動程序後，在操作系統呈現“虛擬串行口”。

USB

● USB——採用先進技術的新型通用接口

→ 新型：高度複雜的接口電路的集成芯片實現高速差分傳輸技術，使得傳輸速度大幅度提高。

→ 複雜：規定了複雜的高層協議，使用簡單但編程複雜。

→ 通用：適合計算機中大多數中、低速傳輸需求。操作系統和芯片組全面支持USB。

→ 版本發展：

● USB物理接口（USB 1.1/2.0）

→ USB採用4線傳輸，其中兩條信號線（差分傳輸），兩條電源線。

→ 標準USB連接器分為A和B兩種，A連接器用於主機，B連接器用於外設。

→ mini USB連接器，用於微型設備。

● 傳輸模式和速度

● USB同步機制

→ NRZI（Non Return to Zero Invert，不歸零反向碼）編碼：輸入為0時輸出反向，輸入為1時輸出不變。

→ 位填充：如果在數據流中出現連續傳輸6個1的情況，則強制在NRZI編碼的數據流中加入一個0 。

● USB——連接方式

PC機的USB接口可以連接外設、複合設備和集線器。最多可以連接5級集線器，最多可連接127臺外設和集線器。

● USB的四種傳輸模式（USB 2.0）

→ 控制傳輸（Control transfer）

支持外設與主機之間的控制、狀態等信息的傳輸，為外設與主機之間提供一個控制通道。

→ 等時傳輸（Isochronous transfer ）

適合數據連續不間斷、實時的、帶寬要求恆定的傳輸。該類型無差錯校驗。適合音、視頻設備。

→ 中斷傳輸（Interrupt transfer）

適合數據量小，無週期性，但對響應時間敏感的傳輸。

→ 數據塊傳輸（Bulk transfer ）

適合傳輸的數據量大，但沒有實時要求的傳輸，USB在滿足帶寬的情況下才進行該類型的數據傳輸。適合外存儲設備。

● USB 2.0的傳輸延（理論最大值）

→ 低速、全速模式：1ms

→ 高速模式：125μs

● USB2.0的數據傳輸速率（KB/s，理論最大值）

● USB 3.0

→ 在USB 2.0的基礎上，增加一個更高的速度模式——SuperSpeed，該模式的位速率為5Gbps，全雙工模式；

→ 與USB 1.1/2.0並存，USB 1.1/2.0設備在USB 3.0主機正常工作。USB 3.0設備在USB 2.0主機上可能不能工作。

IEEE-1394接口——基本特點

● IEEE-1394：是一蘋果公司開發的高速串行方式接口，又名FireWire（火線）、i.Link（Sony名稱）、Lynx（德州儀器名稱）。

● IEEE-1394連接：包括兩個雙向的數據差分線對（TPA和TPB）和電源線對。1394的電源為8～40VDC，1.5A供電能力。數據傳輸有兩種模式。

● 1394a：

標準定義了多種傳輸速率，12.5、25、50、100、200、400Mb/sec的傳輸速率。電纜和連接器可採用帶供電的6線制和不帶供電的4線制兩種。

● 1394b：

增加了800Mb/sec速率，電纜和連接器採用9線，鏈接器與1394a不同，但其中的差分對信號與1394a兼容。

● 1394連接器

有標準6線和不帶電源的4線（微型）兩種。

藍牙

● 藍牙（Bluetooth）：是一種近距離、共享無線信道方式的無線數據通訊技術標準。由Ericsson、IBM、Intel、NOKIA和Toshiba等5家公司組成了藍牙特別興趣小組（Special Interest Group，SIG），負責制定藍牙標準。


● 藍牙技術

→ 採用2.402GHz~ 2.480GHz頻段，劃分成79個間隔1MHz的頻道。使用跳頻頻譜擴展技術，跳頻速率1600次/秒；

→ 相對傳統無線傳輸，抗干擾、保密性好。多個不相關的藍牙通信可在同一個區域共存，相互的干擾只是造成實際通信速率下降。

→ 一臺藍牙設備可同時與其它七臺藍牙設備建立連接，構成微微網；

● 藍牙應用

→ 發現藍牙設備：藍牙設備啟動搜索，發現周圍處於“可被發現”狀態的藍牙設備；

→ 獲取服務：藍牙服務包括藍牙協議定義的各種應用，包括藍牙局域網、撥號網絡、文件傳輸、對象交換、單聲道音頻、高保真音頻、打印、串行通信等；

→ 配對：通過密碼實現配對；

→ 連接：針對特定服務的連接。

遠程通信設備

遠程通信需要經過遠程通信設備，即Modem（MOdulator -DEModulator，調制解調器 ）。大多遠程通信通過公用有線或無線通道實現。

● 電話網Modem

普通電話網Modem利用公共電話網的撥號和音頻傳輸能力實現計算機遠程通信。


調製：將數字變換成符合特定傳輸通道特性的信號；

解調：將調製信號恢復成原數字信號。


電話公網Modem速率：目前電話網Modem的最高數據速率約56Kb/s。

計算機可能集成了Modem，對外提供電話接口（RJ11連接器）。也可以通過擴展卡（Modem卡）、外接Modem、USB擴展Modem等方式擴展電話網Modem。

筆記本計算機、平板電腦可能集成無線公網Modem，即支持3G。也可以外部擴展無線公網Modem，大多數擴展模塊接口是筆記本專用接口（如CardBus、Express Card等接口）或USB接口。

● 寬帶Modem

→ 寬帶Modem：指通信帶寬超過音頻帶寬的數據傳輸Modem。用於普通上網的方案主要有通過電話線實現的ADSL Modem和通過電視電纜實現的Cable Modem。

→ ADSL（Asymmetric Digital Subscriber Line，非對稱數字用戶專線）：設備僅實現用戶到電話局的傳輸，通過在普通電話線上採用高頻段複用方式實現數據傳輸。

→ ADSL標準：由ITU（International Telecommunication Union，國際電信聯盟 ）制定，目前的最高版本是ITU G.992.5，上行速率3.3Mb/s，下行速率24Mb/s。

ADSL Modem一般通過局域網接口與計算機直接連接，也可以通過集成路由器的ADSL Modem組成局域網。


Cable Modem：通過有線電視電纜實現家庭寬帶連接的設備。

傳統的有線電視網只有下行傳輸，經過雙向改造的有線電視網才能實現寬帶數字傳輸。

一路電視信號帶寬為8MHz，可以實現30Mb/s的下行數據傳輸和2.048Mb/s的上行數據傳輸。

Cable Modem可以是機頂盒設備，也可以是獨立設備或計算機總線擴展卡。


● 以太網

以太網：目前的局域網（Local Area Network，LAN）採用以太網（Ethernet）組網方式，物理層符合IEEE 802.3標準。

局域網連接：其物理層連接採用雙絞線，雙絞線為8芯四對雙絞線。連接器為RJ45標準。

以太網速率：以太網速率標準的發展過程經歷了10Mb/s、100Mb/s和1000Mb/s等不同速率標準。

10Mb/s以太網採用10BASE-T（10Mbps，BASE代表基帶傳輸，T代表雙絞線）標準的3類雙絞線。在之後的100Mb/s局域網中，電纜標準為100BASE-TX的5類雙絞線。千兆以太網採用1000BASE-T/TX的超5類或6類雙絞線。

RJ45線序：在局域網連接中使用兩種線序不同的電纜，即直通線和交叉線。10Mb/s和100Mb/s連接僅使用其中兩對雙絞線，1000Mb/s連接使用全部4對雙絞線。

直通線：兩端的連接器線序相同，用於連接不同級別設備；

交叉線：一端的連接器線序符合T568A，另一端的連接器線序符合T568B。用於相同級別的設備連接。

局域網適配器（LAN adapter）：計算機網絡接口的硬件部分，實現網絡協議的物理層和數據鏈路層。更高協議在軟件中實現。

MAC（Media Access Control）地址：用於實現計算機的唯一訪問標識，是一個48位的二進制數，是全球唯一的地址。

● 交換機（Switch）：交換機為多端口網絡設備，用於連接多臺計算機。交換機都可以實現數據包的定向轉發，將數據包發送到目標設備。

與廣播式轉發的集線器相比，定向轉發的交換機使得局域網的轉發效率提高。目前使用交換機更廣泛。

部分交換機可以實現差錯控制。

● 路由器（Router）：路由器是一個比交換機更復雜的多端口網絡設備，傳統的路由器用於連接多個局域網到一個廣域網（Wide AreaNetwork，WAN），如Internet。

● Wi-Fi（Wireless Fidelity,無線高保真）：是一種無線局域網通信協議，標準是IEEE 802.11b / 802.11g /802.11n。採用2.4GHz頻段的短距離無線通信技術，是以太網的無線擴展。

基本性能：Wi-Fi速率最高可達11/54/600Mb/sec，電波的覆蓋範圍可達100米左右，支持多點接入，如果有多個用戶同時通過一個點接入，帶寬被多個用戶分享。

● Wi-Fi的兩種組網方式：一種是ad-hoc模式，用於組成無中心的自組網無線網絡。另一種是帶有AP（Access Point，接入點）設備的方式，稱之為基礎網絡（Infrastructure Network）模式。

● Wi-Fi設備：Wi-Fi設備包括專用的AP接入設備、帶有AP接入功能的寬帶Modem、帶有AP接入的交換機，以及用於計算機的Wi-Fi擴展部件等。
