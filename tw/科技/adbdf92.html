一、C/C++的開發過程

1.編輯（Edit）：使用文本編輯工具錄入程序代碼,生成源程序*.c，*.cpp；

2.編譯（Compile）：語法分析查錯，翻譯生成目標程序*.obj；

3.鏈接（Link）：與其它目標程序或庫鏈接裝配,生成可執行程序*.exe；

4.執行(Run)：運行最終的可執行文件。

2>. VC++常見文件

以Hello項目為例，它的組成文件如下

.dsw文件：工作區(Develop Studio Wrokspace)文件,多個項目組成一個工作區,並用擴展名為.dsw的文件存儲有關工作區的設置情況。

.dsp文件：項目(Develop Studio Project)文件,主要存儲編譯和連接這個項目的一些信息。

.h文件：c/c++頭文件,用於定義被.cpp或.c文件使用的一些標識符號，比如變量類型說明和函數原型說明。

.c或.cpp文件：程序的實現文件,即程序的源文件。

StdAfx.h文件：標準(Standard Afx)的頭文件.通常包含Windows系統頭文件。

StdAfx.cpp文件：對應StdAfx.h，用於產生對StdAfx.h所包含文件的編譯動作。

.rc文件：資源(Resource)描述文件

resource.h：資源頭文件

.ncb文件：No Compile Browser文件，包含語法分析過程中產生的文件。智能庫文件，幫助信息。

.plg文件：Build 日誌(Build Log)文件,每次build,plg文件都會被最新的build信息覆蓋。該文件的內容實際上也出現在build窗口中。

.opt文件：工作區選項(Wrokspace Options)文件,存儲用戶為工作區設置的所有選項。記憶性保存。

.res文件：.rc資源文件編譯後生成的二進制資源文件，用於同其他目標文件連接。

Vc60.idb：用於最小重建和增量編譯信息。

Vc60.pdb：數據庫程序文件，用於調試信息

.pch文件：預編譯頭文件

.obj文件：目標文件

.lib文件：庫文件

.ilk文件：增量連接信息文件(Incremental Link Information File) 。

.exe文件：最終的可執行文件。

三、 編寫C語言程序

C語言程序的3中基本結構。

順序：從頭到尾一步一步執行

分支：程序執行的過程中遇到支流

循環：程序執行的過程中，會重複執行一段代碼

下面來看第一個程序

#include <stdio.h> //編譯預處理：頭文件

int main(void) //主函數，有且只有一個

{

printf("hello world\n"); //輸出

return 0; //返回

}

1.10 C程序的基本結構

由上面的第一個程序，我們可以總結：

#include <stdio.h> 表示頭文件包含-->標準的輸入輸出頭文件

main-->表示主函數，每一個C語言程序都必須有且僅有一個main函數，main前面的int表示該函數的返回值類型。主函數後面跟一對{}，C語言的函數後面都要有一對{}。我們前面講C語言的編程風格時說到：每一條語句結束後面都要加;。除了函數的{}、頭文件（預編譯）和常量的定義外。eg:void fun(){ ... }, #include <stdlib.h>, #define PI 3.1415926等。

在main函數的{}內部的代碼是函數體部分，其中printf是一個函數調用，調用了系統的庫函數，用於輸出數據，該函數後面會講。

return 0;表示程序的返回值，對應main前面的int.

//和/* */表示C語言的註釋部分，對程序沒有任何作用，不參與編譯。

另外，C語言是嚴格區分大小寫的。
