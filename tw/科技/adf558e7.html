查看 Golang、Lua、JS、Rust、Python等語言生成的彙編代碼

2020-11-14
| 科技

喜歡的話可以收藏轉發加關注

為什麼寫這篇文章？

昨天在技術群上，有人問了個問題：

如果一個結構體, 只是讀裡面的成員, 在 golang 裡面傳值的時候, 不傳遞指針, golang 編譯器會幫你優化成 const & 麼?

隨便一猜：golang 肯定是直接 copy 整個結構體。

為了確認是否真的是這樣，最直白的方式就是直接看 golang 生成的彙編代碼。

從圖中的彙編代碼中，我們可以清楚的看到：golang 的確是執行了完整的結構體 copy 。

然後群友給了這樣的反饋...

看著自己日益升高的髮際線，我陷入了沉思...

好了，進入正題。

本文將以 1 + ... + 100 的代碼為例，介紹以下幾種語言查看“彙編代碼”的方式。

（這裡的“彙編代碼”只是個統稱，大家不用太計較）

  1. GolangLuaJavaScript（V8）RustPython等等 ...

1. Golang 生成彙編代碼

源碼

package mainfunc main(){	var sum = 0	for i:=1 ; i <= 100; i++ {		sum = sum + i	}}

查看方式

go tool compile -S .\test.gogo tool objdump .\test.o

分析

行 1 : 表示 將數值 1 放到 AX 中

行 2 : 表示 跳轉到 行 4

行 3 : 表示 對 AX 中的數值執行 + 1 操作

行 4 : 比較 AX 是否在 100 以內。（0x64 是 數值 100 的 16 進制）

行 5 : 跳轉到 行 3

嗯... 怎麼感覺就是在空循環，我們的 sum 變量哪裡去了？

事實上：golang 檢測到 sum 沒有被使用，直接就幫我們優化掉了，只留下一個空循環。

（但它沒有徹底的幫我們把這個空循環也刪掉...)

如果我們把代碼改成這樣子：

package mainfunc main(){}func getSum()int{	var sum = 0	for i:=1 ; i <= 100; i++ {		sum = sum + i	}	return sum}

那麼 sum 累加部分的彙編就是可以正常的顯示出來，如圖所示：

2. Lua 生成彙編代碼

源碼

function getSum() local sum = 0 for i=1, 100 do sum = sum + i end return sumend

查看方式

luac.exe -l test.lua

分析

Lua 虛擬機是基於寄存器來實現的。這段彙編代碼讀起來，就不像golang那麼好理解了。

（關於寄存器虛擬機的相關內容，我在文章的評論中補充了一些。有興趣的話，可以看看）

這裡我就簡單的分析下：

行 1-4：將常量表裡的 0，1，100，1分別加載到寄存器中。

LOADK 指令後面的跟著 2 個參數，分別是：參數1 寄存器索引，參數2 常量表索引

分號後面的數值是： 具體的常量值

這四個數字中：

第一個數字 0 ，就是 sum 的初始值 。

後面三個數（1， 100， 1）：表示了循環從 1 開始，到 100 結束，步長為 1

（ 也就是說 i 從 1 開始，每次循環自動+1，直到達到 100）

行 5-7：執行循環

Lua 通過 FORPREP、FORLOOP 兩條指令來實現循環。

它們的第一個參數：表示指向 循環所需的三個數字 的起始寄存器索引，也就是 寄存器 1。

（這樣虛擬機就知道了，循環所需的三個數字：1，100，1，從而準確的控制循環的邏輯）

它們的第二個參數，表示PC指令跳轉的距離。

注意：FORLOOP 會把 i+1 的結果 放在寄存器 4 中，對應 add 的第 3 個參數

行 6：執行 Add 操作

Add 後面跟著 3 個參數。含義如下：

參數1：存放結果的寄存器索引

參數2、參數3： 分別是兩個加數的索引位置

既然已經分析了 golang 和 lua 兩個語言生成的彙編代碼，後面的語言就不再詳細的分析了，基本大同小異。

3. JavaScript 生成彙編代碼

function getSum(){ let sum = 0; for(let i = 1 ; i <= 100; i++){ sum += i } return sum}

查看方式

node --print-bytecode .\test.js

4. Rust 生成彙編代碼

rust 就比較有意思了（和 c++ 差不多），值得稍微提一下。

fn main() { println!("{}", get_sum());}fn get_sum() -> i32{ let mut sum = 0; for i in 1..=100 { sum += i } return sum}

查看方式

rustc -O --emit asm=test_rust.s .\test_rust.rs

加入了 -O 優化之後，生成的彙編代碼是這個樣子，它直接用 5050 來賦值，省略了計算的過程。

最後

那麼瞭解彙編層面的東西，有什麼用呢？

嗯...對於絕大多數開發者來說，的確沒什麼用...

但是如果你有深入底層的追求，那麼瞭解彙編的知識，會讓你對程序理解的更加透徹。

而且，不管是 Rust，還是 C++ 都支持 內聯彙編：可以在代碼裡嵌入彙編語言，直接控制 CPU，從而獲取極致的底層操作、以及性能上的提升。

比如：騰訊開源的C++協程庫：Tencent/libco 就用到了該技術方案。
