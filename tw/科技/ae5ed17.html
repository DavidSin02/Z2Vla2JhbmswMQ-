大前端進階-同構應用

2020-10-29
| 科技

概述

前端領域中，隨著Vue，React，Angular等框架的流行，前端工程化、模塊化成為了當下主流技術方案。這類框架構建的SPA單頁應用具有用戶體驗好、渲染性能好、可維護性高等有點，但是也存在以下兩個方面的問題：

  1. 首屏加載時間長

SPA應用採用客戶端渲染，用戶需要等待客戶端js解析完成之後才能看到頁面，這樣會導致首屏加載時間變長，用戶體驗差。

  1. 不利於SEO

由於SPA應用採用客戶端渲染，在js未完成解析之前，網站HTML是沒有內容的，這樣導致搜索引擎爬取站點HTML時獲取不到內容。

為了解決這兩個問題，業界提出了一種新的解決方案，如下圖：

利用服務端渲染解決首屏加載慢和不利於SEO的缺陷，首屏渲染完成之後，客戶端渲染接管頁面重新成為單頁應用以保證良好的用戶體驗。這種方式稱為現代化服務端渲染方式或者同構渲染。

與傳統服務端渲染區別

傳統服務端渲染，如JSP可以總結為以下幾步：

  1. 客戶端發送請求
  2. 服務端根據請求查找模板並獲取數據。
  3. 執行渲染
  4. 生成html返回客戶端展示

這種傳統服務端渲染方式，會存在以下缺點：

  1. 前後端完全耦合，不利於開發維護。
  2. 前端發揮空間小。
  3. 服務端壓力大。
  4. 用戶體驗一般。

而同構渲染只是在首屏渲染的時候和傳統服務端類似，都是返回渲染好的html頁面，但是，同構渲染中，當客戶端展示渲染好的html頁面後，客戶端渲染會接管頁面的控制權，也就是後續的渲染都是由客戶端進行的，這樣可以保證良好的用戶體驗。

同構渲染缺點

與單頁應用相比，由於首屏渲染採用的是服務端渲染，所以存在以下缺點：

  1. 開發條件有限，開發有限制（服務端只能使用nodejs，而且並不是所有的工具包都能在服務端渲染中使用）。
  2. 涉及構建和部署的要求高（需要部署客戶端和服務端，不能再和單頁應用一樣，部署靜態站點即可）。
  3. 更多的服務端負載。

Nuxt.js

Nuxt.js是基於vue技術棧的一種同構應用解決方案，它屏蔽了vuejs構建同構應用的難點。

基本使用

搭建nuxtjs同構應用相對比較簡單，通過以下四步即可搭建最簡單的同構應用：

  1. 創建項目文件夾。
  2. npm或者yarn安裝nuxt。
  3. 添加pages文件夾並在其中添加index.vue文件。
  4. 執行npx nuxt。

基礎路由

pages文件夾下存放所有的路由頁面vue文件，nuxt會根據pages文件夾自動生成路由。

如pages目錄如下：

pages/--| user/-----| index.vue-----| one.vue--| index.vue

那麼nuxt會自動生成如下的路由結構：

router: {  routes: [    {      name: 'index',      path: '/',      component: 'pages/index.vue'    },    {      name: 'user',      path: '/user',      component: 'pages/user/index.vue'    },    {      name: 'user-one',      path: '/user/one',      component: 'pages/user/one.vue'    }  ]}

動態路由

nuxt在解析路由結構時，如果遇到以_為前綴的vue文件時，會將其定義為帶參數的動態路由。

如下面的文件結構：

pages/--| _slug/-----| comments.vue-----| index.vue--| users/-----| _id.vue--| index.vue

生成的路由表為：

router: {  routes: [    {      name: 'index',      path: '/',      component: 'pages/index.vue'    },    {      name: 'users-id',      path: '/users/:id?',      component: 'pages/users/_id.vue'    },    {      name: 'slug',      path: '/:slug',      component: 'pages/_slug/index.vue'    },    {      name: 'slug-comments',      path: '/:slug/comments',      component: 'pages/_slug/comments.vue'    }  ]}

嵌套路由

在vuejs中，我們可以通過子路由實現路由的多層嵌套，同樣在nuxtjs中，可以通過特殊的文件結構實現嵌套路由。

創建嵌套路由，需要添加一個Vue文件，同時添加一個與該文件名稱相同的文件夾用於存放子路由組件（在Vue文件中需要添加類似route-view的nuxt-child節點。）

如下面的文件結構：

pages/--| users/-----| _id.vue-----| index.vue--| users.vue

生成的路由表如下：

router: {  routes: [    {      path: '/users',      component: 'pages/users.vue',      children: [        {          path: '',          component: 'pages/users/index.vue',          name: 'users'        },        {          path: ':id',          component: 'pages/users/_id.vue',          name: 'users-id'        }      ]    }  ]}

自定義路由

nuxtjs不僅可以通過pages文件夾結構自動生成路由表，同樣提供了配置文件的方式配置自定義路由。

nuxtjs的配置文件和vue.config.js一樣，在項目根目錄下創建nuxt.config.js文件，此文件默認導出一個配置對象，配置對象有一個 router 屬性，可以在此屬性下配置自定義路由。

export default {    router: {        extendRoutes(routes, resolve) {            // 可以採用此方式清空默認生成的路由            // routes.splice(0)            routes.push(...[                {                    path: '/',                    component: resolve(__dirname, 'your.vue'),                }            ])        }    }}

模板

nuxtjs是基於html模板生成html文件的，我們可以在項目中修改此模板。

定製模板需要在項目根目錄下創建app.html文件，默認的html結構如下：

<!DOCTYPE html><html {{ HTML_ATTRS }}>  <head {{ HEAD_ATTRS }}>    {{ HEAD }}  </head>  <body {{ BODY_ATTRS }}>    {{ APP }}  </body></html>

我們可以在head中添加相應自定義的css、js引用。

asyncData

在vuejs項目中，某個組件如果想要從服務端獲取數據，一般都會在created聲明周期函數中發起請求。在同構應用中，如果想要頁面內容能夠SEO，那麼就不能採用此方式。

在nuxt中為vue組件添加了asyncData方法，此方法會在服務端被執行，執行後的結果會被合併到當前組件的data對象中。

如下例，在服務端獲取所有的文章和標籤並渲染：

async asyncData() {    const [articleRes, tagRes] = await Promise.all([      getAllArticles({        offest: 0,        limit: 10,      }),      getAllTags(),    ])    const { articles, articlesCount } = articleRes.data    const { tags } = tagRes.data    return {      articles,      articlesCount,      tags,    }  }

需要注意的是，此方法雖然是vue組件的一個方法，但是由於其在服務端被調用，所以方法內部不能通過this對象獲取vue實例，如果想要獲取實例上的某些信息，可以用上下文對象。

上下文對象

asyncData方法包含一個參數context，context對象上包含路由等常用信息。

route: 當前路由對象store: store對象params, query: 路由參數req, res: 請求，響應對象redirect: 用於重定向

身份驗證

在同構應用中，當用戶登錄之後，其身份信息需要在客戶端和服務端都能夠被獲取到，因此可以將信息保存在cookie中。

客戶端獲取信息並存入cookie：

methods: {    onSubmit() {        let request = this.isLogin ? login : register        request(this.user)            .then(({ data }) => {                // 獲取到用戶信息                const { user } = data                if (user) {                    const Cookie = process.client ? require('js-cookie') : undefined                    // 將獲取到的用戶信息保存在cookie中                    Cookie.set('user', user)                    this.$store.commit('setUser', user)                }            })            .catch((err) => {            })    }}

服務端讀取cookie並存入store供客戶端使用：

在store的action中可以添加一個名稱為 nuxtServerInit 的異步方法，該方法會在nuxt應用啟動的時候被調用。

nuxtServerInit({ commit }, { req }) {    let user = null    if (req && req.headers.cookie) {        const cookieParser = process.server ? require('cookieparser') : undefined;        const parsed = cookieParser.parse(req.headers.cookie)        try {            user = JSON.parse(parsed.user)        } catch (err) {        }    }    commit('setUser', user)}

插件

如果想要在vuejs程序運行之前執行某些js操作，如註冊第三方組件庫，此時可以利用插件。

所有的插件全部放在plugins目錄下，如果想要在nuxt應用中使用elementui組件庫，可以在plugins文件夾下添加element-ui.js文件：

import Vue from 'vue'import Element from 'element-ui'import locale from 'element-ui/lib/locale/lang/en'Vue.use(Element, { locale })

然後在nuxt.config.js的plugins屬性數組中添加相應文件路徑：

plugins: [    '@/plugins/element-ui',]

之後，就可以在vue組件中正常使用element-ui。
