2、概論

2.1、物聯網平臺是什麼、能幹什麼

阿里雲物聯網平臺為設備提供安全可靠的連接通信能力，向下連接海量設備，支撐設備數據採集上雲；向上提供Open API，服務端通過調用Open API將指令下發至設備端，實現遠程控制。
物聯網平臺也提供了其他增值能力，如設備管理、規則引擎等，為各類IoT場景和行業開發者賦能。

2.2、設備接入

物聯網平臺支持海量設備連接上雲，進行穩定可靠的雙向通信。
• 支持蜂窩（2G/3G/4G/5G）、NB-IoT、LoRaWAN、Wi-Fi等不同網絡設備接入，解決企業異構網絡設備接入管理痛點。
• 支持MQTT、CoAP、HTTP/S等多種協議的接入，既滿足長連接的實時性需求，也滿足短連接的低功耗需求。
• 開源多種平臺設備端SDK，提供跨平臺移植指導，賦能企業基於多種平臺做設備接入。

2.3、設備管理

物聯網平臺提供完整的設備生命週期管理功能，支持設備註冊、功能定義、數據解析、在線調試、遠程配置、固件升級、遠程維護、實時監控、分組管理、設備刪除等功能。
• 提供設備物模型，簡化應用開發。
• 提供數據存儲能力，方便用戶海量設備數據的存儲及實時訪問。
• 提供OTA升級能力，賦能設備遠程升級。
• 提供設備電子緩存機制，將設備與應用解耦，解決不穩定網絡下的通信不可靠痛點。

2.4、規則引擎

物聯網平臺規則引擎包含以下功能：
• 服務端訂閱：訂閱某產品下所有設備的某個或多個類型消息，您的服務端可以通過AMQP客戶端或消息服務（MNS）客戶端獲取訂閱的消息。
• 雲產品流轉：物聯網平臺根據您配置的數據流轉規則，將指定Topic消息的指定字段流轉到目的地，進行存儲和計算處理。
• 場景聯動：配置簡單規則，即可將設備數據無縫流轉至其他設備，實現設備聯動。

2.5、Open API

物聯網平臺提供豐富的API接口，方便您管理設備或者給設備發送控制指令。同樣的，Open API 也有多種語言版本的SDK。

3、基礎概念

3.1、設備三元組

• 一個設備包含4個信息：productKey、productSecret、deviceName、deviceSecret，其中productKey、deviceName、deviceSecret我們稱為三元組信息，也叫做設備證書，在創建設備時由物聯網平臺頒發。
• productKey、productSecret 是產品信息，產品是一類設備的集合，productKey 是產品的唯一性標識符，productSecret 是產品密鑰，後面動態註冊會用到。
• deviceName、deviceSecret 是設備信息，創建產品後創建設備，productKey+deviceName 是設備的唯一性標識符，deviceSecret 是設備密鑰，設備接入時進行身份驗證要用到。
• 三元組這個概念是我們在使用物聯網平臺的過程中一直會出現的概念，也是阿里雲iot最重要、最基礎的概念。三元組是平臺唯一的，是物聯網平臺識別一個設備的重要標識。

3.2、接入協議

物聯網平臺支持MQTT、CoAP、HTTP/S等多種協議的接入，其中MQTT是使用最廣泛的一種接入協議，本次講解設備接入就是基於MQTT來接入的。
MQTT 接入的幾個概念
• host：設備接入的域名，${YourProductKey}.iot-as-mqtt.${YourRegionId}.aliyuncs.com:1883
• keepalive：心跳時間，30秒~1200秒，物聯網平臺 1.5 倍心跳時間內沒有收到心跳會斷開設備連接
• cleanSession：會話清除，0 或 1，設置為 1 時無法接收離線消息

3.3、通信機制

MQTT 是基於對 topic 的訂閱和發佈（sub／pub）來通信的。
• topic：物聯網平臺提供 topic 定義功能，用戶可以自定義topic的字段，也可以使用平臺默認定義好的topic。消息發佈者往topic發送消息，消息訂閱者從topic接收消息，兩者是異步的。物聯網平臺的topic包含productKey和deviceName，所以可以做到設備粒度的權限控制。設備只能往自己的topic發佈消息，也只能訂閱自己的topic。topic有發佈權限和訂閱權限，這個是針對設備視角的。
• qos：消息質量，物聯網平臺支持 qos=0 和 qos=1 的消息質量。qos=0 意味著消息最多到達一次，可能丟失，設備離線無法接收；qos=1 意味著消息至少到達一次，可能收到多次，設備離線期間的消息在設備上線後可以接收到。當設備端接入時設置了cleanSession=1，那麼即使qos=1的消息，設備重新上線後也是收不到的。

4、設備接入及消息通信

本例將完成如下步驟，涉及的demo代碼包通過課程【前言】提供的鏈接下載即可。開發環境的搭建不再贅述，是普通 maven 工程。
設備端代碼：com.aliyun.alink.demo.devicesdk.chapter01.HelloWorld
服務端代碼：com.aliyun.alink.demo.openapi.chapter01.MessageDownlink
• 創建一個產品，名為：一個HelloWorld
• 創建一個設備，名為：device_0
• 使用設備端SDK讓設備 device_0 上線
• 使用設備端SDK讓設備 device_0 發送消息給物聯網平臺
• 使用 Open API 發送消息給設備 device_0

4.1、創建產品

官網文檔：https://help.aliyun.com/document_detail/73728.html

4.2、創建設備

官網文檔：https://help.aliyun.com/document_detail/73729.html

4.3、設備端 SDK 接入

啟動：com.aliyun.alink.demo.devicesdk.chapter01.HelloWorld，填寫設備三元組信息後直接啟動。
整體流程：初始化設備信息-->註冊下行監聽-->初始化SDK-->等待初始化完成-->訂閱topic。
注意：最後一步訂閱的topic是/${productKey}/${deviceName}/user/get，下面 Open API 消息下行要用到。


設備狀態：
• 未激活：設備剛創建，從來沒有連接過平臺是未激活狀態；設備連接到平臺即可激活
• 在線：設備和平臺建立連接
• 離線：設備從平臺斷開連接

4.4、設備端 SDK 消息上報

接上一步，設備端調用publish接口上報消息，本例發送的topic=/${productKey}/${deviceName}/user/update，消息內容為 hello iot。
整體流程：初始化設備信息-->註冊下行監聽-->初始化SDK-->等待初始化完成-->發佈消息到topic
發送消息後查看日誌。

4.5、Open API 消息下行

啟動：com.aliyun.alink.demo.openapi.chapter01.MessageDownlink，填寫賬號信息和設備信息後直接啟動。
整體流程：初始化賬號信息-->初始化SDK-->發送消息給設備。
本例發送的topic=/${productKey}/${deviceName}/user/get，消息內容為 hello world。
發送消息後查看日誌。
