1 從數據與函數的關係看面向對象與面向過程

面向過程的程序設計思路：分析解決問題所需步驟，用函數把這些步驟依次實現。面向過程是在面向對象產生之前及之後，甚至時至今日還應用於程序開發中的程序設計思想。

面向對象的程序設計思路：把構成問題的事務分解為各個對象，將數據(數據結構)和對數據的操作(方法或函數)抽象、封裝到類和對象。建立對象目的，不是完成一個步驟，而是描述某個事務在解決整個問題步驟中的行為。面向對象技術是一種以對象為基礎，以事件或消息來驅動對象執行處理的程序設計技術。它具有抽象性、封裝性、繼承性及多態性等。

C語言是面向過程的編程，它最重要的特點是函數，通過主main函數來調用各個子函數。程序運動的順序都是程序員事先決定好的。C++是面向對象的編程，類是它的主要特點，在程序執行過程中，先由主main函數進入，定義一些義，根據需要執行類的成員函數，過程的概念被淡化了（實際上過程還是有的，就是主函數的那些語句），以類驅動程序運行，類就是對象，所以我們稱之為面向對象程序設計。

形象地理解，面向過程就是把所有的功能全部在一個大的類裡定義出來，當系統龐大時，功能多了，各種操作之間的調用關係也很複雜，當需要修改一個功能時就可能引發一連串的改動，使修改和維護成本很昂貴。

面向對象是把功能以對象為基本單位進行分類，這就是面向對象程序設計所常說的類。當需要添加功能時，只需修改相應的類和極小的其他部分，即可達到目的。

面向對象在分析和解決問題的時候，將涉及的數據和數據的操作封裝在類中，通過類可以創建對象。例如，可以構建一個Circle類(表示圓的類)，它是半徑不同的所有圓的一個抽象描述。通過類可以創建任意多個對象，這些對象之間是相互獨立的。可以由Circle創建圓c1和圓c2，而c1和c2是彼此獨立的對象。這樣，對於事務的管控力度，就由某個具體的對象擴大到一個類，符合人們認識事務的習慣。

class Circle{	private:	double m_dRadius;	public:	void SetR(double r)	{	m_dRadius = r;	}	double GetArea()	{	return(3.14*m_Radius * m_dRadius);	}}

“面向對象程序設計是一種方法，這種方法為數據和函數提供共同的獨立內存空間，這些數據和函數可以作為模板以便在需要時創建類似模塊的拷貝。”從該定義可以看出，一個對象被認為是計算機內存中的一個獨立區間，在這個區間中保存著數據和能夠訪問數據的一組操作。因為內存區間是相互獨立的，所以對象可以不經修改就應用於多個不同的程序中。

程序設計的重點在於數據而不是過程；程序被劃分為所謂的對象。

數據結構為表現對象的特性而設計；函數作為對某個對象數據的操作，與數據結構緊密地結合在一起。

數據被隱藏起來，不能為外部函數訪問；對象之間可以通過函數溝通；

新的數據和函數可以在需要的時候輕而易舉地添加進來。

在程序設計過程中遵循由下而上的設計方法。

2 面向過程與面向對象的差異

面向過程程序設計的實質是從計算機處理問題的觀點（信息流及轉換）來進行程序設計工作：輸入－運算－輸出。需要一開始就設計全面的、自上而下的整個應用程序的架構，因此要求程序設計者對問題域有全面的瞭解。

面向對象程序設計是一種自下而上的結構化的程序設計方法，往往從問題的一部分著手，一點一點地構建整個程序。面向對象設計以數據為中心，類作為表現數據的工具，成為劃分程序的基本單位。
