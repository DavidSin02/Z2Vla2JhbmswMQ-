告別百度網盤和微信，不要錯過這 10 款傳輸神器

2020-10-29
| 科技

即便是現在手機成為工作生活中作為重要的效率工具，但我們在很多時候依舊離不開電腦——有著無可比擬的性能優勢，同時鍵鼠操作文本編輯處理上所帶來高效率，當然這也帶給我們一個問題，如何讓手機和電腦發揮各自的優勢，相互之間可以互傳文件，讓兩者的信息互通有無？

同屬蘋果的 Mac 和 iPhone 一直以來在互通性上做的相當健全，通過「接力」和「隔空投送」功能你可以將文件發送給局域網中的蘋果設備， 甚至可以通過 Apple ID 在 Mac 上實現短信的收發以及通話，從而將手機和電腦兩者的優勢充分的整合起來，高效利用。

但對於並沒有「蘋果生產力解決方案」的用戶而言，手機和電腦之間的協同操作是否就不可行了呢？事實上我們依舊可以通過一些第三方解決方案，讓不同平臺的手機和電腦依舊可以互通有無。

最全面的跨平臺消息傳輸服務：Pushbullet

在移動端和桌面端的協同操作上早就有了相當全面的解決方案，比如說 Pushbullet 這款服務可以說是早期不少 Android 用戶實現跨平臺信息發送的「終極解決方案」，其實從名稱 Push 就不難看出，通知是其最為核心的功能。

Pushbullet 在跨平臺上做的相當不錯，移動端支持 iOS 和 Android，而桌面端則支持 Windows，雖然目前尚不支持 Mac 客戶端，但可以通過安裝對應的瀏覽器擴展來彌補，目前支持的瀏覽器有 Chrome、Opera、Firefox 以及 Safari。

你可以在每一個 Pushbullet 客戶端都想象成一款聊天 App，所以可以通過客戶端向自己的設備發送文件或者網頁鏈接，當然這樣的發送文件就不可能實時點對點發送，所以 Pushbullet 發送文件就像是 QQ 上的離線文件，不過 Pushbullet 接受文件會自動下載，相對而言更為智能一些。

至於剪貼板同步功能在 Pushbullet 也支持，比如在電腦上選擇一段文本「CTRL C」，在手機端長按選擇粘貼就可以將文本貼過來，只不過這項功能需要付費才可以使用。目前 Pushbullet 採用按照時間的訂閱付費的模式（約 39.99 美元/年），付費用戶傳輸文件時單個文件大小可以增大到 1GB，可以對手機鏡像過來的通知直接進行操作，以及前面提到的剪貼板同步功能。其他和免費版賬戶基本無差。

Pushbullet 優勢在於良好的跨平臺性，功能全面（文件傳輸，通知傳輸，短信/電話跨平臺等等），尤其是你使用的是 Android 手機的話，大而全的功能將大部分可能的需求全部包括。當然也有不少缺點，比如文件傳輸上，即便是局域網內部也無法實現點對點傳輸，必須要先存到 Pushbullet 然後再下載，文件大小也有最高 1GB 的限制，此外就是 Pushbullet 網絡問題，畢竟服務器在海外，在國內網絡訪問對網絡質量有較高的要求。

只要文件傳輸的首選：Send Anywhere

有些時候我們只需要在不同的平臺之間傳輸文件，而不需要協同操作功能，那麼Send Anywhere 或許是你的首選，比起那些功能豐富的應用，做好一件事（傳文件）其實實屬不易。

Send Anywhere 是一款跨平臺文件傳輸服務，在 iOS/Android，Windows/masOS /Linux 上都有對應的客戶端。其功能就兩點：點對點即時文件傳輸以及文件暫存服務。點對點即時文件傳輸用起來很簡單，在發送端中發送文件時會生成一組六位數的配對碼，然後接受客戶端只要輸入配對碼，雙方就開始傳輸，而這個過程是不經過第三方服務器中轉，兩個客戶端直接點對點傳輸的，所以傳輸速度依賴於你當前局域網或者互聯網的上傳帶寬。

當然 Send Anywhere 也提供了類似文件離線傳輸功能，不過由於上傳服務器在國外所以訪問速度有點慢，而 Send Anywhere 的收費點也主要是離線傳輸，所以在國內使用就直接免費賬戶使用點對點傳輸即可。

為 Android 用戶量身打造的跨平臺協同利器：Join

跨平臺協同其實主要是為 Android 手機用戶提供解決方案，畢竟蘋果的生態系統為 iOS 和 macOS 提供了最基本的協同操作服務，而要說到為 Android 服務，首選當然是全部建立在 Google 生態圈之上的 Join。

Join 提供的跨平臺的形式更為有趣，首先支持的移動端上當然只有 Android，而桌面端則可以通過 Chrome 擴展提供服務，換言之只需要你的電腦安裝有 Chrome 瀏覽器，那麼你就可以輕鬆實現不同設備之間的跨平臺協同操作。當然 Join 還提供了 Windows UWP 客戶端，但需要單獨收費下載。

和 Pushbullet 類似，Join 提供也是一整套的協同操作方案，在跨平臺的通知同步推送（在桌面端接受手機端的推送通知），跨平臺剪切板同步（跨平臺複製粘貼文本和鏈接）以及文件互傳，總的來說覆蓋了大部分的跨平臺設備之間的操作需求。

不過 Join 也並非沒有缺點，首先由於服務完全依賴 FCM ，因此通知同步推送等需要較好的網絡環境，另外就是客戶端設置相對比較複雜，幾乎所有的功能點都需要手動設置。 另外 Join 是一款收費應用，Play Store 上售價 4.99 美元，一次買斷沒有內購。

簡單易用的跨平臺協同工具 AirDroid

與 Pushbullet 一樣提供「大而全」服務的還有 AirDroid，早期主要是針對 Android 設備（從名字其實就不難看出），而現在移動端也加入了對 iOS 的支持，而桌面端則支持 Windows、macOS，並且提供了一個 web 端，換言之只要通過瀏覽器就可以使用他的服務。

而在功能上 AirDroid 主要提供了跨平臺的通知鏡像（支持直接在桌面端操作鏡像通知），跨平臺的信息發送（支持桌面端直接使用手機端的即時聊天工具），此外當然還少不了跨平臺的文件傳輸，除了點對點的文件傳輸之外，高級版本甚至還支持文件夾傳輸，並且同樣也支持離線文件暫存和下載（文件體積有限制）。如果要說缺點的話，就是 AirDroid 不支持跨設備的剪貼板同步功能，因此相互之間發送文本或者鏈接必須手動通過客戶端像聊天軟件那樣進行發送。

除了上面的這些功能，AirDroid 還提供了一些特色的功能，比如說你可以通過桌面客戶端鏡像手機來進行遠程控制，或者遠程控制來調用手機攝像頭來作為移動監控設備使用。

AirDroid 同樣提供了免費版和高級版，兩者之間的主要差別是設備數量限制，部分功能（傳輸文件夾、遠程攝像頭、去除廣告）以及文件傳輸大小之間的差別，高級版採用訂閱收費方式，價格為 12元/月或者 120元/年。

來自錘子家的高效跨平臺協同工具：Handshake

不得不說錘子在軟件創新上確實更加註重使用效率問題，閃念膠囊，大爆炸等都在一定程度上解決了手機使用上的痛點，而在跨平臺協同領域也同樣提供了一個極為優秀的工具 Handshake。

作為錘子手機內置的一項功能，Handshake 可以非常方便使用桌面設備來訪問和管理錘子手機的內容，最為核心的當然就是文件之間的互傳，管理相冊、文件、音樂等文件，而操作方式也非常直觀，就像使用資源管理器那樣拖動到不同的管理窗口即可。

其他功能上，Handshake 還支持通過歡喜雲同步來管理閃念膠囊，並且支持查看錘子/堅果的剪切板，以及通過 HandShaker 輸入文字發送到手機剪貼板。作為錘子手機的跨平臺協同服務來說，功能還是比較全面的。

Handshake 桌面端支持 Mac 和 Windows，移動端則支持 Android 手機，不過由於是錘子開發的服務，因此其他 Android 手機可能有部分服務無法正常使用，此外由於眾所周知的原因，Handshake 也已經兩年沒有更新，所以保不齊後面可能會出現兼容性等問題，所以如果你使用的是非錘子的 Android 手機建議選用其他的服務。

大文件傳輸最佳解決方案：奶牛快傳

跨平臺之間的文件傳輸似乎是一個極為重要的使用痛點，而非常弔詭的是提供這種單一服務的特別少，而說到簡單易用上估計沒有什麼服務可以和奶牛快傳相比，個人甚至認為其效果可以不遜於海外的 Mega 網盤服務。

奶牛快傳簡單來說提供了一種大文件暫存服務，無需使用客戶端，通過瀏覽器就能直接用，而且使用上也沒有需要強制註冊賬號一說，只要通過瀏覽器直接上傳即可，上傳完成之後自動生成分享二維碼，然後手機掃碼就可以下載。

不註冊賬戶最高能傳 2GB 的文件，文件保存 72 小時，而註冊登錄則不限制文件大小，保存時間可以到 168 小時（頗有點像 QQ 文件中轉站的意思），此外奶牛快傳還有 Pro 付費賬號，主要是支持自定義二級域名等等，其實如果主要是提供跨平臺大文件傳輸的話，付費也就可有可無了。

長的像聊天應用的跨平臺文件傳輸工具：Feem v4

多個設備之間的跨平臺文件傳輸其實有時候不需要那麼麻煩，比如這個名為 Feem v4 的應用就提供了簡單的文件點對點傳輸和文本鏈接發送功能，而且使用起來就像你在使用聊天軟件一樣。

簡單來說，Feem v4 傳輸文件的步驟非常簡單，打開客戶端找到局域網中需要接受文件的設備，然後點擊發送文件就好了，而接收端就會自動將文件保存下來，自動配對也就沒有了繁瑣的掃碼配對過程。

此外 Feem v4 還可以像聊天軟件一樣，向局域網中直接發送文本和鏈接，接收端收到文本或者鏈接可以使用應用上覆制按鈕一鍵粘貼到剪貼板上，可以說是非常的簡單高效。

Feem v4 提供了iOS 和 Android 客戶端，桌面端支持 macOS 和 Windows，因為功能只有這麼幾個，所以相對還算比較輕量的應用，唯一缺點可能是應用界面略微有點兒醜，此外部分平臺傳輸文件時會有些卡頓，並且僅支持局域網內文件傳輸。Feem v4 也提供了付費方案，主要是設備終端的數量限制（低於五臺就可以免費使用）。

只做跨平臺的剪貼板同步服務：Clip Cloud

其實對於很多朋友而言，經常使用的垮平臺協同操作其實是複製粘貼的同步，也就是我們說的剪貼板同步——在手機上覆制文本，然後在桌面端粘貼，這樣比起那些文件傳輸或者文本鏈接發送要更有效率。

Clip Cloud 就是這樣的一款應用，簡單到只有剪貼板同步這樣一個服務，手機上安裝客戶端，然後在 Chrome 瀏覽器上安裝對應的擴展，然後在應用中使用自己的 Google 賬戶登錄就可以實現跨端剪貼板同步，並且後面無需進行任何額外的設置，非常高效。

要說到這款服務的缺點的話，就是重度依賴 FCM，國內使用的話對網絡上要求比較高，此外 Clip Cloud 是付費服務，4.99 美元直接買斷。

那些充當跨平臺文件傳輸服務的聊天應用：QQ、微信以及 Telegram

即便有著上面說的林林總總的跨平臺協同服務，我們依舊可以選擇一些現在經常使用的的應用來實現類似功能，例如文件傳輸上，我們最長使用的 QQ、微信或者 Telegram 都可以實現跨平臺文件傳輸。

微信有「文件傳輸助手」可以實現文本，鏈接的發送，文件當然也可以發送只不過都是離線存到雲端，然後再從另一個客戶端從雲端下載下來，簡單來說就是不支持點對點實時傳輸文件。微信傳輸圖片一不留神還會「被壓縮」，所以如果是工作文件，選擇微信來做文件傳輸並不算靠譜。

QQ 就相對正經的多，雖然同樣也不支持點對點傳輸文件（桌面端之間支持點對點傳輸），但勝在圖片發送默認不會壓縮，所以如果傳輸文件的話，QQ 依舊是聊天應用中做跨平臺文件傳輸最好用的。

可以充當文件傳輸工具的聊天應用當然不僅僅只是騰訊系的 QQ 和微信，什麼 Telegram 這類的當然也是沒問題的，實現的方式也都大同小異，這裡就不繼續展開說了。簡單來說目前大部分聊天應用都支持跨平臺的文件傳輸，如果手裡沒個專門的跨平臺文件傳輸的應用，找個順手救急完全沒問題。

結語

即便是手機性能越發強大的今天，電腦無論是在性能上還是從辦公效率上依舊完勝手機，這也是為什麼我們希望為兩者建立一條橋樑，讓手機和電腦可以充分的發揮各自優勢並最終高效完成工作和學習任務。
