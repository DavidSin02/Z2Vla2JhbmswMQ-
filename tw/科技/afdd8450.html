醫療大數據應用遭遇的困境

隨著醫療信息化水平的提高和人工智能技術的發展，醫學大數據的應用範圍逐漸擴大。越來越多的人開始瞭解醫療大數據的魅力，而對於大數據而言，巨量的數據來源是其分析準確性的根本保證。當數據量大到一定程度之後，想要保證數據的準確度就會變得異常艱難，相應的分析結果也就很難保障，造成大數據應用的失敗。Google流感預測和IBM Watson在醫療大數據領域遭遇重創，甚至被認為這是人工智能在醫療領域的倒退。

Google——流感疾病預測

2009年美國H1N1爆發前幾周Nature上面有一篇文章《Detecting influenza epidemics using search engine query data》論述了Google基於用戶的搜索日誌（其中包括搜索關鍵詞、用戶搜索頻率以及用戶IP地址等信息）的彙總信息，成功“預測”了流感病人的就診人數。截止到文章發表出來的時候，Google的預測還是準確的，不過到後來就發生了很大的偏差，偏差最大甚至高出了標準值（CDC公佈的結果）將近一倍[3]。

Google預測的失敗，一方面模型過度地依賴於數據，導致很多被忽略了的因素對預測的結果產生了很大的影響。找準數據是前提，數據並非越大越好，數據來源是像搜索引擎這種複雜度本身就很高的地方，那分析就會變得更困難，每天都有上百萬的與CDC的數據相關的搜索詞彙，其中的許多搜索都會與流感趨勢緊密相關，但其中也會存在著很多與流感相關的搜索實際上並不是由真正的流感病例引起或者與流感趨勢有關。另一方面，對客觀世界進行預測需要模型，模型首先來自於理論構造，其次需要數據對模型進行訓練對模型進行優化完善。大數據觀點強調模型對數據訓練的依賴，而儘可能地忽略理論構造這一部分的意義，因此忽略或脫離流行病的基礎理論，這就有可能帶來隱患。後來有人用CDC的歷史數據並依據流行病學定義流感預測的相關控制指標構建數據模型，歷史數據標準化處理後保證其數據質量，預測準確率顯著提高。

IBM——沃森(Watson)

IBM Watson曾被認為是“引領認知革命”的先驅，能夠從海量的醫學文獻和病歷中提取醫生臨床診斷經驗，通過機器學習讓計算機掌握臨床診斷方法，目標是可以替代人類醫生看很多疑難雜症。沃森在全球7個國家（中國、美國、韓國、泰國、新加坡、印度、荷蘭）落地並服務了近萬例患者。早在2016年8月，WFO就進入了中國醫療領域，IBM與中國北京、上海、廣州、浙江、福建、雲南等省市的21家醫院達成了關於Watson腫瘤解決方案的合作意向。2018年裁員舉措意味著IBM沃森在醫療健康領域遭遇重挫[5]。

沃森為什麼在實際應用上和預期的效果差別如此之大呢？首先到目前為止，沒有一種AI可以匹配人類醫生的理解和洞察力，沃森只做到了求同，卻做不到求異。像糖尿病、高血壓、腫瘤等非標準化的疑難雜症領域，導致疾病的成因有環境、遺傳、基因突變、生活習慣等各種因素，頗為複雜，醫生需要吸納各種信息後進行綜合判斷。因此，如果僅僅依靠機器閱讀文獻獲得醫療方案很難令人信服。

另外，在數據處理上，除了收集的樣本數據量不足，最大的問題便是在數據質量不高，本來應該依據大量的真實數據從而找到新的治療手段，由於罕見病例數據的極端匱乏，沃森所用的訓練數據不足，訓練數據量最高的肺癌只有635例，而最低的卵巢癌只有106例，更多采用的是患者的假想數據來做訓練。這種通過假想數據學出來的AI，準確性可想而知。瞭解其中數據的質量以及如何使用人工智能算法來適應數據質量問題將變得很重要。

醫療領域，歷經長期信息系統標準不統一，臨床術語開發無法跟進，健康數據碎片化、非結構化等因素困擾，越過數據質量談人工智能確實有點天方夜譚，紮實做好數據治理基本功，保障數據質量才是醫療人工智能的唯一出路。

醫療大數據現狀分析

醫療數據有數據質量參差不齊、分散的特質，造成醫療機構內或區域集成平臺眾多異源異構應用系統的數據形成信息孤島，不能達到數據互聯互通、共享和應用的要求。醫療大數據的應用主要面臨以下問題：

  1. 數據不可直接讀：系統原始存儲命名、格式等差異大，表名、數據元名稱等系統差異大。需要將英文、拼音或縮寫等數據庫命名根據真實數據解析、標註、釋義。
  2. 核心業務數據需要辨識：醫院信息系統全量數據中混雜系統數據、日誌數據及大量空值數據等應用價值小的數據對主要業務數據形成干擾。需要剝離核心應用業務數據。
  3. 患者身份未歸一、無關聯：醫院中各自獨立的醫療信息系統未將患者身份統一，數據關聯性差、一致性不高，導致跨系統、跨院、跨區域應用時存在障礙。需要建EMPI等主數據管理，將分散的數據按主數據集成在一起。
  4. 系統模塊和流程不統一：各醫院各系統的業務模塊、分類和命名差異大、數據分散，查找和應用困難。需要建立標準的數據集分類和數據集，使不同來源的異構數據按統一的結構存儲，方便查找和應用。
  5. 數據元不統一：信息系統的數據的表結構設置和命名體差異很大，辨識和掌握所有醫院數據結構成為數據應用的障礙。需要統一的數據元標準，統一對數據元進行命名和定義，並建立相同數據元概念的映射關係，打破差異化。
  6. 數據元值域編碼不一致：數據元值存儲多為代碼形式，而各醫院數據編碼體系不同，導致同一值域有多個不同編碼，不能直接讀取應用。需要建立統一的數據字典標準，統一編碼，保證數據有效利用。
  7. 醫學術語命名和編碼差異大：診斷、藥品、診療項目等醫學術語，是醫療數據的焦點信息，存在各地應用等標準不統一、醫生命名習慣與標準命名不統一等問題。需要統一的醫學術語分類編碼標準，建立不同標準術語間、自然語言與標準術語間、醫療術語大概念與小概念間等多種映射關係的臨床術語標準體系，滿足不同數據應用需求。
  8. 數據表示格式不統一：數據元值域的表示格式多樣、數據元合併拆分不一致、數據表示不一致等。需要按數據類型規範數據表示格式，按統一的數據表示形式存儲和展示，提高應用效率。

面對上述複雜、多樣的數據質量問題，需要構建標準化的數據治理流程和規範。醫療數據治理就像是一個“修高速公路”的過程，人工智能和大數據挖掘應用是“跑車”，而醫學本身的脈絡體系就是支撐高速公路的“鋼架”。近年來眾多新興互聯網企業進入到健康醫療大數據的產業中，同時也越來越清楚意識到數據質量在不同應用模塊的建設中的重要作用，數據標準化對應用模型的建設起到了頂層架構的作用。按數據管理能力成熟度評估模型來看，目前國內的醫療大數據剛剛從初始級向受管理級轉化，尚未缺少穩定、成熟的數據標準化管理流程和規範，因此出現了“跑車”尚無匹配的優質賽道的現狀，醫療大數據應用迫切需要一套標準化的數據治理路徑，為提高醫療大數據的數據質量保駕護航。

國內外醫學信息標準和規範

國際早在上世紀八、九十年代就已經誕生了醫學信息化的標準，用於解決醫療在傳輸、匯聚和共享層面面臨的不同類型數據質量問題。主要的標準見下表：

我國自2009年以來參照國際通用標準，陸續推出多項國家衛生信息標準，包括基礎類標準、數據類標準、技術類標準、應用與服務類標準、管理類標準等，逐步形成面向國內數據現狀的衛生信標準體系（見下圖）。

存在問題

  1. 標準體系建設晚於信息系統 我國的醫療信息化起步於上世紀90年代末，經歷了10的快速發展期，這個過程缺乏衛生信息標準的支撐，全國有幾千家信息系統廠商有自己的數據標準，造成同一家醫院有多種、多個不同廠商系統，形成數據不能互聯互通、信息孤島的局面。同時國內衛生信息標準更新節奏慢，不像國際標準根據實際應用情況有固定的更新週期：按月、季、（半）年等，使得標準很難跟上實際應用的需要。
  2. 標準體系與實際數據業務應用差異大 國內衛生信息標準是從科技角度和方法論角度建標準，大部分是參考原則，缺少可執行的落地路徑、規則和解決方案。同時各信息化廠商和醫院有自己固有的業務流程體系，造成標準體系與實際信息系統數據和應用有差距。例如，2018年9月13日，國家衛生健康委員會印發《國家健康醫療大數據標準、安全和服務管理辦法（試行）的通知》。《通知》針對標準管理、安全管理、服務管理、管理監督進行了詳細的規定。在安全管理方面明確瞭如何界定主體責任人和監管責任人，但對於真實數據中各類業務數據如何定義數據安全等級、開放和共享等級尚無明確的參考依據。
  3. 大數據技術與應用需求先行 目前大數據技術有突飛猛進的發展，同時在互聯網醫療、藥物研發、健康管理、保險等方向應用已經展開積極探索。而國內醫療大數據標準尚存在很多空白，尤其是數據應用服務方向缺乏標準指導但缺乏可執行的標準背書，導致企業應用標準不一，各自為政，甚至遊走在法律風險的邊緣。醫療大數據應用遍地開花的今天，數據應用、服務、安全等方面方面尚缺乏可直接參考的標準成為影響其發展的瓶頸。
  4. 標準執行缺少監理、驗收、評估 現行已頒佈的信息化標準，因缺乏強制、有效的執行措施，出現各信息化廠商、機構和單位推遲執行和不願意執行的情況，導致標準執行不一致和雖然有標準但並沒有真正發揮其價值的亂象。同時缺乏有效的驗收、評估管控標準，以保證按標準執行的質量。

可見，行業內信息化標準尚不完備，醫療信息化標準直接用於醫療大數據的數據治理和應用還有很長的一段路要走。而基於標準化體系支撐的數據彙集、數據治理和應用，實現為每個人創建一份健康檔案，確保獲得完整和準確的醫療健康記錄，支持最佳的醫療服務、醫療協同和數據應用已經是迫在眉睫的需求。因此近幾年各醫療大數據公司和機構都在置重金構建適合自己應用的標準體系，以提高數據質量，支撐自身不同數據應用和服務。

聯仁健康醫學信息標準建設規劃

聯仁健康作為健康醫療大數據的國家隊，目標是圍繞大數據資源優勢，構建產業價值鏈開放體系，快速形成國內健康醫療大數據產業匯聚高地，成為國內最大的健康醫療數字化服務運營商。而面對健康醫療大數據的數據現狀，公司在搭建“一湖三臺”產品架構和打造健康醫療大數據、互聯網醫療健康、健康醫療產業園三大板塊，佈局多種醫療大數據和互聯網業務的同時，已經開始積極規劃和投入構建規範化數據標準和醫學知識庫體系，它是平臺數據匯聚、治理、共享和應用的核心支柱——“數據指揮中樞”。

聯仁健康數據標準和醫學知識庫體系依託於國家標準、行業標準和政府發佈的資源目錄標準等，形成資源目錄“基礎級”數據標準，用於數據採集和整合（包括電子病歷和健康檔案等）；同時面向數據應用需求構建數據治理細分“應用級”數據標準（包括單病種、藥物研發、核保理賠、健康管理等），以形成聯仁企業級數據標準。數據標準包括：數據集分類標準、數據集標準、數據元標準、數據元值域字典和醫學術語集標準等。同時構建基於醫學專業知識、循證醫學、真實世界數據的醫學知識庫和知識圖譜。目標是打造能支撐科研服務、人工智能、健康管理、互聯網醫療等不同方向的規範化的數據標準和知識庫體系，也希望未來在健康醫療大數據行業樹立標杆，形成企業影響力。
