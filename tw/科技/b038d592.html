屆時，我相信我們能夠輕而易舉地、規範地寫出我們看見的任意一個靜態頁面。

1 CSS 相關定義

CSS（Cascading Style Sheets 層疊樣式表）：用來控制 HTML 的表現。

CSS 決定頁面好不好看、動畫效果酷不酷炫。

HTML+CSS 是一個靜態頁面的基本組成。就好比一個女生在有基本的身體五官（HTML）後，又對自己打扮了下：化了妝、梳了髮型、穿了好看的衣服（CSS）。

2 CSS 工作原理

再次複習文章：《老生常談的從 URL 輸入到頁面展現背後發生的事》

  • 首先，當我們在瀏覽器上輸入一個 URL 並回車後，瀏覽器向服務器發了一個請求，服務器就把相應的數據——HTML 返回給瀏覽器。
  • 然後，瀏覽器拿到這個數據後就加載這個頁面，去解析這個 HTML。怎樣去解析呢：瀏覽器得到的字符串，它會對其做一些編碼等的分析，把字符串解析成“樹狀”的結構，創建一個 DOM，叫 DOM 樹（因為它是一個文檔結構，類似“樹”這樣一個效果）。
  • 接著，在解析 HTML 的過程中，瀏覽器會去加載相應的 CSS。比如，當解析到以下框選的代碼時：

瀏覽器會再次去向服務器發請求，去獲取這個 style.css 文件。

  • 繼而，同步的這個 CSS 文件就下載下來了。下載下來之後，瀏覽器同時去解析這個 CSS 文件，讓它變成一個 “CSS 樹”。然後瀏覽器再去做一個計算，把 “DOM 樹”和“CSS 樹”對應起來，最終得到一個新的“樹”。
    這個“樹”裡有兩個節點，一個是它的元素是什麼東西；另一個是這個元素的具體參數：寬、高、位置、大小、邊框等。
  • 瀏覽器得到這個“樹”之後，再去“渲染”。
    相當於我們得到一個很豐富的數據，這個數據告訴了我們頁面上每個元素的位置、每個元素之間的關係——父子、鄰居等。這個時候，我就可以拿塊畫板進行“繪製”。

3 CSS 在 HTML 中的引入方式

3.1 外部樣式表：通過 <link> “鏈入” CSS（如上例）

在 <head> </head> 裡邊，緊接著 <title> 標籤下邊加 <link> 標籤。

  • <link> 標籤中， rel="stylesheet" ，“rel 屬性”指定了 HTML 文件與所鏈接的文件的關係，這裡指我們要鏈接一個“樣式表”——stylesheet。
  • 樣式表放入這個 href（Hypertext Reference）——這裡可以用相對鏈接（上例用的一個 .css 文件，此文件將所有與之相關的 CSS 全部包含在裡邊，這裡注意文件路徑要寫對），也可以用一個完整的 URL。

3.2 外部樣式表：通過 @import “導入”樣式

<style>  @import url("index.css");      /* ① */  @import url(index.css);        /* ② */  @import "index2.css";          /* ③ */  @import url("landscape.css") screen and (orientation:landscape); /* ④ */</style>

單從形式上， @import 這種方式與 <link> “鏈入” CSS 的方式差不多，都是從外部引入 CSS 文件。

格式上 ①②③ 都可以，效果等同。需要注意的是，對於多個 .css 文件或 URL 的“導入”時，分號 ; 一定不能省。

本例 ④ 中後半段加的是“媒體查詢限制條件”：

  • screen 指“在有瀏覽器屏幕下”；
  • and (orientation:landscape) 針對移動端設備的“顯示方向 orientation ”是“橫屏（landscape）”。
    “豎屏”英文單詞是：portrait。

3.3 內部樣式表：放在 head 元素內的 <style> 標籤中

如上篇文章的例子：

<!DOCTYPE html><html><head>  <meta charset="utf-8">  <title>HTML 表單</title>  <style>    #txa {      width: 300px;      height: 200px;      margin-left: -12px;    }  </style></head><body>……

3.4 內聯樣式：直接寫在元素開始標籤的 style 屬性中

<p style="background: yellow; font-family: sans-serif;">Oli 是個好人!</p>

注：

  • 這種寫法注意分號、前後的引號；
  • 一般不推薦使用，在特定情況下使用。

總結：一般情況下推薦使用 <link> “鏈入” CSS。

4 CSS 選擇器初識

CSS 選擇器用於“定位”我們想要給予“樣式”、“規則”的 HTML 元素。

對於 CSS 的“樣式”、“規則”，我們可以拆分理解：

h1 {color: red; background: yellow;}

4.1 元素選擇器

p {  color: yellow;}div {  background: yellow;}p, div {  font-family: sans-serif;}

4.2 ID 選擇器

id 定位頁面上唯一的元素（如上篇文章的例子）：

<!DOCTYPE html><html><head>  <meta charset="utf-8">  <title>HTML 表單</title>  <style type="text/css">    #txa {                                        /* ① */      width: 300px;      height: 200px;      margin-left: -12px;    }  </style></head><body>…  <div>    <label for="txa">評論：</label>    <textarea id="txa" name="article">是個好人！    <!-- ② -->      </textarea>                           </div>…

註釋：

  • 對於 ② 中 id="txa" ：“id 屬性”是一種標識元素“唯一”的方法，帶 id 的元素有一個特性：你可以直接鏈接這些元素，給它們單獨加樣式。
    ❗️但注意：id 是唯一的，一個元素不能有多個 id，另外頁面上不允許多個元素都有相同的 id。
    id 取名時，不允許出現空格。
  • 對於 ① 中 #txa ：按 id 來選擇一個元素，需要在 id 名前面使用一個 # 字符。

4.3 類選擇器

源碼及效果展示
https://jsbin.com/sijuminino/edit?html,css,output

“類”：可直接理解為一個足球俱樂部，在這個俱樂部（類）裡，所有成員（元素）都必須遵循統一的規範——如出席商業活動時都必須著同樣的裝束。

創建一個“類”分兩步：

  • 為 HTML 元素增加一個 class 屬性，這樣就可以將這個元素加到這個“類”中。
    ❗️注意：一個元素可以加入多個類，即一個人可以加入多個俱樂部，寫法上各個“類名”用“空格”隔開。
<ul>  <li class="first">fine</li>  <li class="second">thank you</li>  <li class="third done">and you</li></ul>
  • 在 CSS 中選擇這個“類”（❗️注意：格式為在 class 類名前面用一個 . 字符）。
.first {  font-weight: bold;}.done {  text-decoration: line-through;}

小總結：想要對多個元素使用某個樣式時，就把這些元素都加入到這個“類”裡，就會用到 class；但如果只有一個元素需要加這個樣式，那就用 id。

4.4 通配選擇器

這個選擇器可以與任何元素匹配，就像一個“通配符”。如：

* {  color: yellow;}

這個“規則”可以讓文檔中所有元素的 color 值都指定為 yellow。

4.5 屬性選擇器

“屬性選擇器”根據元素的屬性和屬性值來匹配元素，“屬性”英文單詞為：attribute。

4.5.1 [attr]

源碼及效果展示
https://jsbin.com/voyipikura/edit?html,css,output

即直接用 [] 符號將元素的屬性名框起來，用於根據元素屬性來選擇元素：

li[class] {  color: red;  background: yellow;}

選擇有 class 屬性（值不限）的所有 li 元素，使其文本為 red，背景色為 yellow。

4.5.2 [attr=val]

源碼及效果展示
https://jsbin.com/togaluxiso/edit?html,css,output

即直接用 [] 符號將“屬性”和與之對應的“屬性值”框起來，用於根據具體屬性及其值來選擇元素：

li[class="second"] {  color: red;  background: yellow;}

選擇有 class 屬性值為 second 的所有 li 元素，使其文本為 red，背景色為 yellow。

❗️注意：這種選擇格式要求必須與“屬性值”完全匹配，不多不少，不能有任何形式的簡寫。

例：選擇有 class 屬性值為 third done 的所有 li 元素，使其文本為 red，背景色為 yellow。

li[class="third done"] {  color: red;  background: yellow;}

✘不能簡寫為：

li[class="third"] {  color: red;  background: yellow;}

4.5.3 [attr~=val]

源碼及效果鏈接
https://jsbin.com/niluyupero/edit?html,css,output

即直接用 [] 符號將“屬性”、符號 ~ 和與之要加樣式的“屬性值”框起來，用於“根據部分屬性值”來選擇元素：

li[class~="done"] {  color: red;  background: yellow;}

選擇有 class 屬性值包含 done 的所有 li 元素，使其文本為 red，背景色為 yellow。

4.5.4 ^ $ *

1. ^

源碼及效果展示
https://jsbin.com/ruqererafe/edit?html,css,output

li[class^="se"] {  color: red;  background: yellow;}

選擇 class 整個屬性值以 se 開頭的所有 li 元素，使其文本為 red，背景色為 yellow。

2. $

源碼及效果展示
https://jsbin.com/zasiyadoqa/edit?html,css,output

li[class$="ne"] {  color: red;  background: yellow;}

選擇 class 整個屬性值以 ne 結尾的所有 li 元素，使其文本為 red，背景色為 yellow。

3. *

源碼及效果展示
https://jsbin.com/jikeyetola/edit?html,css,output

li[class*="ir"] {  color: red;  background: yellow;}

選擇 class 屬性值中包含字串 ir 的所有 li 元素，使其文本為 red，背景色為 yellow。

助記：
^ 是笑臉（^^）的左邊開始，所以記憶為屬性值以某字符開頭；
$ 美元符號很像一個搖擺的尾巴，所以可記憶為屬性值以某字符結尾；
* 星號猶如星星，都是一串一串的，所以可記憶為屬性值中包含某字符串。

4.5.5 語言值匹配

源碼及效果展示
https://jsbin.com/doqibureci/edit?html,css,output

<!DOCTYPE html><html lang="zh">                        <head>                                     <meta charset="utf-8">                      <title>表示“謝謝”的語言</title>            </head>                                 <body>                                    <h1>表示“謝謝”的語言</h1>                <dl>    <dt>英語</dt>     <dd lang="en">Thank you.</dd>    <dt>日語</dt>     <dd lang="ja">ありがとう.</dd>    <dt>法語</dt>    <dd lang="en">Merci.</dd>    <dt>澳洲英語</dt>    <dd lang="en-au">Thank you very much.</dd>  </dl>        </body>                                           </html>

當需要對以上代碼中：選擇 lang 屬性等於 en 或者以 en- 開頭的所有元素，使其文本為 red，背景色為 yellow。

*[lang|="en"] {  color: red;  background: yellow;}

❗️注意：上面這種屬性選擇器常常用於匹配語言值。
