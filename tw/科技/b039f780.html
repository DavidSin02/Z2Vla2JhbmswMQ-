Android通知形式 Toast、Notification、Dialog對話框

2020-11-14
| 科技

通知系統

1.Toast

彈出形式： 界面的下半部分 彈出一段時間後自動消失

不會因為界面關閉而消失 不會響應用戶事件

應用場景： 提示不需要用戶響應的信息 eg:雙擊退出

e

實現

//得到通知對象 （1.上下文對象 2.內容 3.時長）

Toast.LENGTH_LONG ：3.5s Toast.LENGTH_SHORT:2s

Toast toast = Toast.makeText(this, "再次點擊，退出應用！", Toast.LENGTH_LONG);

//顯示通知

toast.show();

另一種寫法：

Toast.makeText(this, "再次點擊，退出應用！", Toast.LENGTH_LONG).show();


Notification 狀態欄上的通知

形式：顯示在狀態欄上 顯示提示信息 點擊跳轉到 詳情界面

下拉顯示詳細內容

應用場景：全局通知

eg:短信提醒 歌曲播放歌曲通知


2.3 API

實現步驟：

//1.獲得通知管理器對象

manager = (NotificationManager) getSystemService(Context.NOTIFICATION_SERVICE);

//3.設置通知上升信息（參數1.上升動畫 2.提示信息 3.系統時間）

Notification notification=new Notification(R.drawable.ic_launcher, "10086通知來了！", System.currentTimeMillis());

//7.設置點擊通知自動消失的標誌位

notification.flags=Notification.FLAG_AUTO_CANCEL;

//6.意圖 設置點擊通知跳轉的方向（界面）

Intent intent=new Intent(this, MainActivity.class);

//5.延時意圖

PendingIntent contentIntent=PendingIntent.getActivity(this, 0, intent, 0);

//4.設置下拉信息（1.上下文對象2.標題3.詳細內容4.延時意圖）

notification.setLatestEventInfo(this, "詳細內容標題 10086", "內容欠費 100元", contentIntent);

//2.發送通知

manager.notify(1, notification);


//取消通知

manager.cancel(1);


Dialog 對話框

彈出形式：

在界面的正中央 ，打斷用戶的操作

應用場景： 需要立即用戶做出響應 eg: 檢查更新 雙擊退出

警告對話框AlertDialog

實現步驟：

//1.獲得對話框輔助類 建造者模式

AlertDialog.Builder builder = new AlertDialog.Builder(this);

//2.設置標題

builder.setTitle("檢查更新")

//3.設置內容

.setMessage("1.解決了XXXBug \n 2.新增了XXX功能")

//4.設置圖標

.setIcon(R.drawable.ic_launcher)

//5.設置積極按鈕

.setPositiveButton("確定", new OnClickListener() {

@Override

public void onClick(DialogInterface dialog, int which) {

Toast.makeText(MainActivity.this, "開始下載新版本", Toast.LENGTH_SHORT).show();

}

})

.setNegativeButton("取消", null)//消極按鈕

.setNeutralButton("中立按鈕", null)//中立按鈕

.show();

//獲得對話框對象 彈出方式二

選項菜單OptionMenu

應用必有的功能，但一又不是經常使用


創建菜單的方式一：xml中創建

1.重寫onCreateOptionsMenu


2添加菜單文件.xml

添加菜單

<item

android:id="@+id/action_settings" //菜單Id

android:orderInCategory="80" //菜單顯示順序

android:showAsAction="ifRoom"

android:title="設置"/>//設置內容


ifRoom:如果有空間就顯示在 ActionBar

always：總是顯示在ActionBar

never：總是不顯示在ActionBar

UI控件

1.CheckBox 多選框

應用場景：問卷調查 免責聲明 是否記住密碼

xml:

<CheckBox

android:id="@+id/checkBox1"

android:layout_width="wrap_content"

android:layout_height="wrap_content"

android:checked="true" true:選中 false:沒有選中（默認）

android:text="記住密碼" /> 文本


code:

//設置CheckBox選中狀態

mBox.setChecked(true);

//獲得CheckBox是否選中

boolean checked = mBox.isChecked();

//獲得CheckBox的文本

String string = mBox.getText().toString();
