最簡WebGL教程

2020-10-29
| 科技

現代 OpenGL（以及名為WebGL的擴展）與我過去學習的傳統 OpenGL 有很大不同。我瞭解柵格化的工作原理，所以對這些概念很滿意。但是我所閱讀的每篇教程都介紹了抽象和輔助函數，這使我很難理解哪些部分是 OpenGL API 的真正核心。

明確地說，在實際的應用程序中，把位置數據和渲染功能分離到單獨的類這樣的抽象很重要。但是，這些抽象把代碼分佈到了多個區域，並且由於模板的重複以及邏輯單元之間的數據傳遞而導致大量的開銷。而我的最佳學習方式是線性代碼流，其中每一行都是手頭主題的核心。

首先，本文要歸功於我所學過的教程。從這個基礎開始，我剝離了所有抽象，直到有了一個“最小可行的程序”為止。希望這將幫助你使用現代OpenGL入門。這就我們要做的：



一個等邊三角形，頂部為綠色，左下為黑色，右下為紅色，中間有過渡顏色

初始化

要使用 WebGL，需要用 canvas 進行繪製。你肯定會想包括一些常用的 HTML 骨架、某些樣式等，但是 canvas 才是最關鍵的。加載 DOM 後，我們將能夠用 Javascript 訪問畫布。

<canvas id="container" width="500" height="500"></canvas><script>  document.addEventListener('DOMContentLoaded', () => {    // 所有的 Javascript 代碼將會出現在這裡  });</script>

我們可以通過畫布的可訪問性獲得 WebGL 的渲染上下文，並將其初始化為透明色。 OpenGL 的世界中的顏色是RGBA，每個分量都在 0 和 1 之間。透明色是用於在重新繪製場景的幀的開始時繪製畫布的顏色。

const canvas = document.getElementById('container');const gl = canvas.getContext('webgl');gl.clearColor(1, 1, 1, 1);

在實際的程序中，還可以進行更多的初始化。需要特別注意的是啟用了“深度緩衝區（depth buffer）”，這將允許基於 Z 座標對幾何圖形進行排序。對於只包含一個三角形的最簡程序，我們將會忽略這種情況。

編譯著色器

OpenGL 的核心是柵格化框架，在這裡我們可以決定如何實現除柵格化之外的所有內容。這需要在 GPU 上至少運行兩段代碼：

  1. 為輸入所執行的頂點著色器，每個輸入都會對應輸出一個3D位置（實際上是齊次座標中的4D）。
  2. 為屏幕上的每個像素所執行的片段著色器，負責輸出這個像素應該是哪種顏色。

在這兩個步驟之間，OpenGL 從頂點著色器獲取幾何圖形，並確定這個幾何圖形實際上覆蓋了屏幕上的哪些像素。這是柵格化部分。

兩種著色器通常都是用 GLSL（OpenGL 著色語言）編寫的，然後將其編譯為 GPU 的機器代碼。機器代碼隨後被髮送到 GPU，因此可以在渲染過程中運行。我不會把太多時間花在 GLSL 上，因為我只是在展示基礎知識，但是這種語言與 C 很接近，著足以讓大多數程序員感到熟悉。

首先，我們編譯頂點著色器並將其發送到GPU。此處著色器的源代碼被存儲在字符串中，但是也可以從其他位置加載。最終，該字符串被髮送到 WebGL API。

const sourceV = `  attribute vec3 position;  varying vec4 color;  void main() {    gl_Position = vec4(position, 1);    color = gl_Position * 0.5 + 0.5;  }`;const shaderV = gl.createShader(gl.VERTEX_SHADER);gl.shaderSource(shaderV, sourceV);gl.compileShader(shaderV);if (!gl.getShaderParameter(shaderV, gl.COMPILE_STATUS)) {  console.error(gl.getShaderInfoLog(shaderV));  throw new Error('Failed to compile vertex shader');}

在這裡的 GLSL 代碼中有一些需要提到的變量：

  1. 一個名為 position 的屬性。屬性本質上是一個輸入，並且為每個這樣的輸入調用著色器。
  2. 一種稱為 color 的 varying。這既是頂點著色器的輸出（每個頂點著色器都有一個），也是片段著色器的輸入。值被傳遞到片段著色器時，將根據柵格化的屬性對值進行插值計算。
  3. gl_Position 值。本質上是頂點著色器的輸出，如任何存在變化的值。這很特別，因為它用於確定需要去繪製哪些像素。

還有一個稱為 uniform 的變量類型，該變量類型在多次調用頂點著色器時將會保持不變。這些 uniform 用於變換矩陣之類的屬性，對於單個幾何圖形上的頂點來說，它們都是恆定的。

接下來，我們用片段著色器執行相同的操作，將其編譯併發送到 GPU。注意，片段著色器現在可以讀取頂點著色器中的 color 變量。

const sourceF = `  precision mediump float;  varying vec4 color;  void main() {    gl_FragColor = color;  }`;const shaderF = gl.createShader(gl.FRAGMENT_SHADER);gl.shaderSource(shaderF, sourceF);gl.compileShader(shaderF);if (!gl.getShaderParameter(shaderF, gl.COMPILE_STATUS)) {  console.error(gl.getShaderInfoLog(shaderF));  throw new Error('Failed to compile fragment shader');}

最後，頂點著色器和片段著色器都被鏈接到單個 OpenGL 程序中。

const program = gl.createProgram();gl.attachShader(program, shaderV);gl.attachShader(program, shaderF);gl.linkProgram(program);if (!gl.getProgramParameter(program, gl.LINK_STATUS)) {  console.error(gl.getProgramInfoLog(program));  throw new Error('Failed to link program');}gl.useProgram(program);

我們告訴 GPU，上面所定義的著色器就是我們要運行的著色器。所以剩下事情的就是創建輸入，並讓 GPU 在這些輸入上進行運算。

將輸入數據發送到 GPU

輸入的數據將會存儲在 GPU 的內存中，並從那裡進行處理。與其對每個輸入進行單獨的繪製調用（一次僅傳輸一個相關數據），不如將整個輸入傳輸到 GPU 並從那裡讀取。 （傳統 OpenGL 一次只能傳輸一份數據，從而導致性能下降。）

OpenGL 提供了一種被稱為“頂點緩衝對象”（VBO）的抽象。我仍在試圖完全弄清楚它的工作原理，但是最終，我們將會使用抽象來進行以下操作：

  1. 將一系列字節存儲在 CPU 的內存中。
  2. 用通過 gl.createBuffe() 創建的唯一緩衝區和 gl.ARRAY_BUFFER 的綁定點（binding point）將字節傳輸到 GPU 的內存。

儘管在頂點著色器中每個輸入變量（屬性）都有一個 VBO，但也可以把一個 VBO 用於多個輸入。

const positionsData = new Float32Array([  -0.75, -0.65, -1,   0.75, -0.65, -1,   0   ,  0.65, -1,]);const buffer = gl.createBuffer();gl.bindBuffer(gl.ARRAY_BUFFER, buffer);gl.bufferData(gl.ARRAY_BUFFER, positionsData, gl.STATIC_DRAW);

通常你將會用對程序有意義的任何座標來指定幾何圖形，然後在頂點著色器中使用一系列轉換將它們轉換為 OpenGL 的“剪輯空間（clip space）”。我不會介紹剪輯空間的詳細信息（它們與同構座標有關），但是現在，X 和Y 在 -1 到 +1 之間變化。由於頂點著色器僅按原樣傳遞輸入數據，因此可以直接在剪輯空間中指定座標。

接下來，我們還會把緩衝區與頂點著色器中的變量之一相關聯：

  1. 從上面創建的程序中獲取 position 變量的句柄。
  2. 告訴 OpenGL 從 gl.ARRAY_BUFFER 綁定點讀取數據，每批 3 個，其特殊參數如 offset 和 stride 為零。
const attribute = gl.getAttribLocation(program, 'position');gl.enableVertexAttribArray(attribute);gl.vertexAttribPointer(attribute, 3, gl.FLOAT, false, 0, 0);

請注意，我們可以創建 VBO 並將其與“頂點著色器”屬性相關聯，因為要一個接一個地做。如果我們將這兩個功能分開（例如一次性創建所有 VBO，然後將它們與各個屬性相關聯），則需要在將每個 VBO 與對應的屬性相關聯之前調用 gl.bindBuffer(...)。

繪製！

最後，按照我們想要的方式設置 GPU 內存中的所有數據，我們可以告訴 OpenGL 清除屏幕並在設置的陣列上運行程序。作為柵格化的一部分（確定哪些像素被頂點覆蓋），我們告訴 OpenGL 將 3 個一組的頂點視為三角形。

gl.clear(gl.COLOR_BUFFER_BIT);gl.drawArrays(gl.TRIANGLES, 0, 3);

以線性方式進行設置確實意味著可以一次就能使程序運行。在任何實際的應用中，我們都會以結構化的方式存儲數據，在數據發生變化時將其發送到 GPU，並在每一幀進行繪製。

將所有內容放在一起，下圖顯示了在屏幕上顯示第一個三角形的最小概念集。即使這樣，該圖還是被大大簡化了，所以你最好配合本文所介紹的 75 行代碼放在一起進行研究。



完整的處理流程：首先創建著色器，通過 VBO 將數據傳輸到 GPU，把兩者關聯在一起，然後 GPU 在再將所有內容組裝成最終的圖像。

最後的步驟，儘管經過了簡化，但完整描述了三角形所需的步驟順序

對我而言，學習 OpenGL 的難點在於獲得屏幕上最基本圖像所需的大量模板。由於柵格化框架要求我們提供 3D 渲染功能，並且與 GPU 的通信非常冗長，所以有很多概念需要預先學習。我希望本文所展示的基礎知識比其他教程更簡單！
