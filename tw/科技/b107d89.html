結合案例深入解析適配器模式（一）

2020-10-29
| 科技

一、基本概念

將一個類的接口，轉換成客戶期望的另一個接口。適配器讓原本接口不兼容的類可以合作無間。

可以將適配器理解為我們日常用的電腦充電器: 家庭電壓為220V，而電腦充電頻率是20V左右，所以需要適配。

二、結構

適配器可以分為兩種: 對象適配器和類適配器。

對象適配器:

從用戶的角度看不到被適配者。

用戶調用適配器轉換出來的目標接口方法。適配器再調用被適配者的相關接口方法。

用戶收到反饋結果，感覺只是和目標接口交互。

類適配器:

通過多重繼承目標接口和被適配者類方式來實現適配。

三、案例

案例: 將火雞冒充成鴨子。

1、對象適配器模式

邏輯圖:

代碼組織結構圖:

被適配者火雞Turkey:

public interface Turkey { void gobble(); // 火雞叫聲 void fly();}/** 野火雞 */public class WildTurkey implements Turkey{ @Override public void gobble() { System.out.println("Go Go!"); } @Override public void fly() { System.out.println("I am Flying a short distance!"); }}

目標對象Duck:

/** 鴨子的接口 */public interface Duck { void quack();//鴨子叫聲 void fly();}

適配器TurkeyAdapter:

/** * 在外面表現是 鴨子(目標)，但是實質是火雞(被適配者) */public class TurkeyAdapter implements Duck { //實現目標的接口 private Turkey turkey; //這種對象型適配器必須要組合 被適配者，也就是要有適配者的引用 public TurkeyAdapter(Turkey turkey) { this.turkey = turkey; } // 實際是火雞在叫 @Override public void quack() { turkey.gobble();//外面表現是quack,但是內部是turkey.gobble() } @Override public void fly() { //由於火雞飛的短，所以多飛幾次，讓火雞更像鴨子 for(int i = 0; i < 6; i++){ turkey.fly(); } }}

測試:

public class MyTest { public static void main(String[] args) { WildTurkey turkey = new WildTurkey(); Duck duck = new TurkeyAdapter(turkey); duck.quack(); //看似是鴨子，其實內置是火雞 duck.fly(); }}

輸出:

Go Go!I am Flying a short distance!I am Flying a short distance!I am Flying a short distance!I am Flying a short distance!I am Flying a short distance!I am Flying a short distance!

2、類適配器模式

基本結構圖:

雖然Java不支持多繼承，但是可以實現的同時繼承。

只有TurkeyAdapter有一些代碼變動，其他不變:

/** * 和 對象適配器模式唯一的不同就是 : 適配器直接繼承 被適配者 (而不是組合) */public class TurkeyAdapter extends WildTurkey implements Duck { @Override public void quack() { super.gobble(); //直接繼承 被適配者 } @Override public void fly() { //讓火雞飛6次，飛的像鴨子 super.fly(); super.fly(); super.fly(); super.fly(); super.fly(); super.fly(); }}

測試:

public class MyTest { public static void main(String[] args) { TurkeyAdapter duck = new TurkeyAdapter();//直接new即可 duck.quack(); duck.fly(); }}

輸出和上面對象適配器一樣。

3、Java中從以前枚舉器Enumeration到迭代器Iterator的適配

從Enumeration到Iterator適配的結構圖:

一個實例代碼:(下面EnumerationIterator就是一個適配器)

/** 對象適配器的舉例 java中的枚舉到迭代器 */public class EnumerationIterator implements Iterator<Object> { private Enumeration enumeration; //枚舉 : 被適配者 public EnumerationIterator(Enumeration enumeration) { this.enumeration = enumeration; } @Override public boolean hasNext() { return enumeration.hasMoreElements(); //實際調用的是 被適配者 的方法 } @Override public Object next() { return enumeration.nextElement(); } //這個可以說是適配器的缺點， 有些不能適配，比如兩個插孔的插頭不能適配為三個插孔的插頭 @Override public void remove() { throw new UnsupportedOperationException(); }}

四、總結

對象適配器和類適配器使用了不同的方法實現適配，對象適配器使用組合，類適配器使用繼承。
