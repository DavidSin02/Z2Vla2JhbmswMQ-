一文弄懂Python面向對象的三大特性：繼承、封裝、多態

2020-10-29
| 科技

前面給大家介紹了兩個關於Python面向對象編程的實戰案例，相信大家對面向對象的理解更進一步了。這裡，小編再給大家介紹一個重點——面向對象的三大特性：繼承、封裝、多態，只有充分的理解了這三大特性，在做面向對象編程時才會得心應手

基本概念

  • 繼承：兩個類之間具備一定的關係，這種關係允許一個類訪問另外一個類的方法和屬性。也就是我們常說的父類和子類：父類是指被訪問的類；子類是指訪問的類
  • 封裝：對類的自身屬性和方法進行保護，不被外部類訪問。形象來說就是用個“箱子”裝起來
  • 多態：一種接口，多種形態。通過一種接口，實現不同的目的。

案例分析：繼承特性

# 定義一個父類class Father:    def __init__(self,name,age):        print("姓名是：｛｝，年齡是：｛｝".format(name,age))    def playGame(self,gameName):        print("喜歡玩的遊戲是｛｝".format(gameName))

首先，我定義了一個父類Father，它有兩個屬性，分別是姓名和年齡。然後，我們給父類初始化1個方法：將姓名和年齡打印出來；同時我們還給這個類定義了另外一個方法：playGame，將喜歡玩的遊戲打印出來。

father1=Father("父親1","50")father1.playGame("DOTA")

接下來，我們給這個類實例化一個對象father1，並給它傳入兩個屬性，姓名：“父親1”，年齡：“50”，然後我們通過這個對象調用Father類的方法playGame。

# 定義一個子類class Children(Father):    def playLive(self):        print("喜歡看直播")child1=Children("兒子1",20)child1.playGame("王者榮耀")

現在，我們再定義一個子類Children，讓它繼承父類Father，然後定義一個方法玩直播playLive。接下來我們再給子類實例化一個對象child1，因為它繼承了父類，所以我們可以給它傳入父類的屬性，姓名：“兒子1”，年齡：“20”；同時我們還可以調用父類的方法，比如playGame，這就是面向對象的繼承特性。在一些不是個性化的方法需求上，我們可以通過繼承使用代碼重用，提高複用率。

這裡大家思考一下，我們是可以通過子類調用父類的屬性和方法，假如我們需要一個類，不允許別的類訪問自己的屬性和方法等該怎麼辦呢？

案例分析：封裝特性

class Father:    # 私有屬性    __smoking=True    # 公有屬性    working=True

一般情況，每個類都有自己公有屬性和私有屬性，也就是像每個人都有自己可以公開的信息和不能公開的信息。那麼，如果不想公開自己的私有屬性，那就可以將“私有屬性”進行封裝。比如，我們在父類下定義一個“私有屬性”smoking（以雙下劃線開頭），子類是不能直接訪問的，而working屬性是可以被直接訪問的。

    # 賦值    def setSmoking(self,smoke):        self.__smoking=smoke    # 獲得值    def getSmoking(self):        print(self.__smoking)

子類想訪問父類的屬性，無非就是兩種情況：獲得父類的屬性值；給父類的屬性重新賦值。所有如果要想訪問“私有屬性”，就可以這樣進行封裝：分別定義兩個方法，一個用來獲取父類的屬性值，一個用來給父類的屬性重新賦值，子類只需要調用父類的這兩個方法，就可以訪問父類的屬性了。

child1.setSmoking("False")child1.getSmoking()

訪問__smoking這個私有屬性時，我們調用setSmoking這個方法，並給它寫入屬性False，然後再調用父類getSmoking這個方法，就可以看到__smoking的值被修改為False，即child不smoking：

child2=Children("兒子2",15)child2.setSmoking(True)child2.getSmoking()

案例分析：類的多態

多態性是面向對象在一定條件下才會出現的一種特性：

  • 多態一定是子類和父類之間
  • 多態子類重寫了父類方法

下面的代碼定義了Children3類 、Children4和Children5類，三者都繼承於Father1類，並且都重寫了父類的payType方法，所有它具備多態性。

class Father1():    def payType(self,goodsType):        print("不同的人買東西的方式")        goodsType.buy(self)class Children3(Father1):    def buy(self,payType):        print("孩子3買東西的方式：現金")class Children4(Father1):    def buy(self,payType):        print("孩子4買東西的方式：支付寶")class Children5(Father1):    def buy(self,payType):        print("孩子5買東西的方式：微信")child=Father1()child.payType(Children3())child.payType(Children4())child.payType(Children5())

在調用Father1類的payType方法時，程序並不關心為該方法傳入的參數對象children是3、4、5的那哪，只要求此參數對象children包含payType方法即可，而調用者傳入的參數對象是來源於哪個類，程序並不在乎，這就是面向對象的多態性。
