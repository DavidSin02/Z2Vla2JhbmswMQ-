SQL Server——遊標詳解

2020-11-14
| 科技

遊標的應用

查詢語句可能返回多條記錄，如果數據量非常大，需要使用遊標來逐條讀取查詢結果集中的記錄。應用程序可以根據需要滾動或者瀏覽其中的數據。

遊標的定義

遊標是一種數據訪問機制，一種處理數據的方法，它允許用戶訪問單獨的數據行。主要用於存儲過程、觸發器和Transac-SQL腳本中，他們使結果集的內容可用於其他Transac-SQL語句。在查看或處理結果集中的數據時，遊標可以提供在結果集中向前或者向後瀏覽數據的功能。類似於C語言中的指針，他可以指向結果集中的任意位置。當要對結果集進行逐行單獨處理時，必須聲明一個指向該結果集的遊標變量。

Sql Server中的數據操作結果都是面向集合的，並沒有一種描述biao單一記錄的表達形式。

遊標的優點

1.用戶可以對每一行進行單獨處理，從而降低系統開銷和潛在的阻隔情況，用戶也可以使用這些數據生成SQL代碼並立即執行或輸出。

2.允許程序對有select查詢語句返回的行集中地每行執行相同或不同的操作，而不是對整個集合執行同一個操作。

3.提供對基於遊標位置的表中的行進行刪除和更新的能力。

4.遊標作為數據庫管理系統和應用程序設計之間的橋樑，將兩種處理方式連接起來。

遊標類型

1.Transac-SQL遊標

基於declare cursor語法，主要用於Transact-SQL腳本、存儲過程和觸發器。Transact-SQL遊標在服務器上實現，並由從客戶端發送到服務器的Transact-SQL語句管理。他們還可能包含在批處理、存儲古城或觸發器中。

2.應用程序編程接口（API）服務器遊標

支持OLEDB和ODBC中的API遊標函數，API服務器遊標在服務器上實現。每次客戶端應用程序調用API遊標函數時，SQL Server Native ClientOLEDB 訪問接口或者ODBC驅動程序會把請求傳輸到服務器，以便對API服務器遊標進行操作。

3.客戶端遊標

由SQL Server Native Client ODBC 驅動程序和實現ADO API 的DLL在內部實現。 客戶端遊標通過在客戶端高速緩存所有結果集中的行來實現。每次客戶端應用程序調用API遊標函數時，SQL Server Native Client ODBC驅動程序 或ADO DLL 會對客戶端上高速緩存的結果集中的行執行遊標操作。

由於Transact-SQL遊標和API服務器遊標都在服務器上實現，所以他們統稱為服務器遊標。ODBC 和ADO 定義了Microsoft SQL Server 支持的4種遊標類型。

SQL Server 支持的4種API服務器遊標類型

1.只進遊標

只進遊標不支持滾動，他只支持遊標從頭到尾順序提取。由於遊標無法向後滾動，因此在提取行後對數據庫中的行進行的大多數更改通過遊標均不可見。

2.靜態遊標

SQL Server 靜態遊標始終是隻讀的。其完整結果集在打開遊標時建立在tempdb中。靜態遊標總是按照打開遊標時的原樣顯示結果集。

3.由鍵集驅動的遊標

該遊標中各行的成員身份和順序是固定的。由鍵集驅動的遊標由一組唯一標識符（鍵）控制，這組鍵成為鍵集。鍵是根據以唯一方式標識結果集中各行的一組列生成的。鍵集是打開遊標時來自負荷select語句要求的所有行中的一組鍵值。由鍵集驅動的遊標對應的鍵集是打開該遊標時在tempdb中生成的。

4.動態遊標

動態遊標與靜態遊標相對。當滾動遊標時，動態遊標反映結果集中所做的所有更改。結果集中的行數據值、順序和成員在每次提取時都會改變。所有用戶做的全部update、insert和delete語句均通過遊標可見。如果API函數（如SQLSetPos）或者Transact-SQL where current of 子句通過遊標進行更新，他們將立即可見。在遊標外部所做的更新知道提交時才可見，除非將遊標的事務隔離級別設置為未提交讀。

聲明遊標

遊標主要包括遊標結果集和遊標位置兩部分，遊標結果集是由定義遊標的select語句返回的行集合，遊標位置則是指向這個結果集中的某一行的指針。

語法格式：

declare cursor_name cursor [ local | global ]  [ forward_only | scroll ]  [ static | keyset | dynamic | fast_forward ]  [ read_only | scroll_locks | optimistic ]  [ type_warning ]  for select_statement  [ for update [ of column_name [ ,...n ] ] ]

注：

  1. cursor_name: 是所定義的Transact-SQL 服務器遊標的名稱。
  2. Local：對於其中創建的批處理、存儲過程或觸發器來說，該遊標的作用域是局部的。[Local | Global]:默認為local。
  3. Global：作用域為全局，由連接執行的任何存儲過程或批處理中，都可以引用該遊標。
  4. Forward_Only:指定遊標智能從第一行滾到最後一行。Fetch Next是唯一支持的提取選項。如果在指定Forward_Only是不指定Static、KeySet、Dynamic關鍵字，默認為Dynamic遊標。如果Forward_Only和Scroll沒有指定，Static、KeySet、Dynamic遊標默認為Scroll，Fast_Forward默認為Forward_Only
  5. Static:靜態遊標
  6. KeySet：鍵集遊標
  7. Dynamic：動態遊標，不支持Absolute提取選項
  8. Fast_Forward：指定啟用了性能優化的Forward_Only、Read_Only遊標。如果指定啦Scroll或For_Update，就不能指定他啦。
  9. Read_Only:不能通過遊標對數據進行刪改。
  10. Scroll_Locks：將行讀入遊標是，鎖定這些行，確保刪除或更新一定會成功。如果指定啦Fast_Forward或Static，就不能指定他啦。
  11. Optimistic：指定如果行自讀入遊標以來已得到更新，則通過遊標進行的定位更新或定位刪除不成功。當將行讀入遊標時，sqlserver不鎖定行，它改用timestamp列值的比較結果來確定行讀入遊標後是否發生了修改，如果表不行timestamp列，它改用校驗和值進行確定。如果已修改改行，則嘗試進行的定位更新或刪除將失敗。如果指定啦Fast_Forward,則不能指定他。
  12. Type_Warning:指定將遊標從所請求的類型隱式轉換為另一種類型時向客戶端發送警告信息。
  13. select_statement: 是定義遊標結果集的標準select語句。
  14. For Update[of column_name ,....] :定義遊標中可更新的列。

聲明靜態遊標

--聲明靜態遊標use TestSchoolgodeclare cursor_Adress cursor forselect Id,PCity ,Pcode from Adress;

聲明動態遊標

--聲明動態遊標declare TestData_cursor cursor localscrollDynamicread_onlyfor select Value from TestData where Id=111;

打開遊標

語法：

open [ Global ] cursor_name | cursor_variable_name

注：

  • Global：指定cursor_name是全局遊標。
  • cursor_name：已聲明的遊標名稱。
  • cursor_variable_name：遊標變量的名稱，該變量引用一個遊標。

打開靜態遊標cursor_Adress

--打開靜態遊標cursor_Adressuse TestSchool;go open cursor_Adress;

打開動態遊標cursor_TestData

--打開動態遊標cursor_TestDatause TestSchool;goopen TestData_cursor;

讀取遊標中的數據

fetch命令可以讀取遊標中的某一行數據。

fetch語法：

Fetch [ [ Next | prior | Frist | Last | Absoute {n | @nvar } | Relative {n | @nvar }] from ]{ { [ Global ] cursor_name } | @cursor_variable_name } [ into @variable_name [,....] ]

注：

  • Frist：結果集的第一行
  • Prior：當前位置的上一行
  • Next：當前位置的下一行
  • Last：最後一行
  • Absoute { n | @nvar }：從遊標的第一行開始數，第n行。
  • Relative {n | @nvar} ：從當前位置數，第n行。
  • Into @variable_name[,...] : 將提取到的數據存放到變量variable_name中。

檢索Adress表中數據

--檢索Adress表中的記錄。use TestSchool;Gofetch next from cursor_Adress while @@FETCH_STATUS =0 --提取成功，進行下一條數據的提取操作begin		fetch next from cursor_Adressend

取數據：

fetch next from cursor_Adressfetch next from cursor_Adressselect * from Adress where Id=25

關閉遊標

遊標打開後，服務器會專門為遊標分配一定的內存空間存放遊標操作的數據結果集，同時使用遊標也會對某些數據進行封鎖。所以遊標一旦用過，應及時關閉，避免服務器資源浪費。

語法：

close [ Global ] cursor_name | cursor_variable_name

注：

  • Global：指定cursor_name是全局遊標。
  • cursor_name：已聲明的遊標名稱。
  • cursor_variable_name：遊標變量的名稱，該變量引用一個遊標。

關閉遊標實例：

close cursor_Adress;

釋放遊標（刪除遊標）

遊標操作的結果集控件雖然被釋放了，但是遊標結構本身也會佔用一定的計算機資源，所以在使用完遊標之後，為了收回被遊標佔用的資源，應該將遊標釋放。

語法：

--釋放遊標語法deallocate [ Global ] cursor_name | cursor_variable_name

釋放遊標實例:

--釋放遊標deallocate cursor_Adress;deallocate cursor_TestData;

遊標的使用步驟

  1. 聲明遊標變量
  2. 創建遊標
  3. 打開遊標
  4. 為遊標變量賦值
  5. 從遊標變量中讀取值
  6. 判斷fetch語句是否執行成功
  7. 讀取遊標變量中的數據
  8. 關閉遊標
  9. 釋放遊標

遊標實例

--使用遊標變量use TestSchoolgodeclare @varcursor Cursor --聲明遊標變量declare cursor_Adress cursor for --創建遊標select Id,PCity ,Pcode from Adress;open cursor_Adress; --打開遊標set @varcursor = cursor_Adress --為遊標變量賦值fetch next from @varcursor --從遊標變量中讀取值while @@FETCH_STATUS = 0 --判斷fetch 語句是否執行成功begin		fetch next from @varcursor --讀取遊標變量中的數據endclose @varcursor --關閉遊標deallocate @varcursor --釋放遊標
