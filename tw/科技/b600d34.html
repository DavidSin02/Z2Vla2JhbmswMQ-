1 遺留 bug

那啥，不知道大家有沒有發現上一篇有一個 bug，開始之前還是先處理一個 遺留bug 吧，具體請看下圖：

首次新建筆記報錯

具體表現為當我們新建第一篇筆記的時候，會報錯 TypeError: Cannot read property 'title' of undefined，可是明明我們已經在data中定義了為什麼還 undefined ？

async init() {  await this.getFileList()  const [firstFileItem] = this.fileList  this.fileItem = firstFileItem  this.activeIndex = 0}

其實關鍵就在於我們 init() 方法初始化那裡了，在inti()中我們是直接將獲取到的筆記列表第一項賦值給當前的this.fileItem，那麼當筆記列表為空的時候，賦值後會發生什麼呢？firstFileItem會是 undefined，最後this.fileItem也變成 undefined 了，對象this.fileItem中的title字段也就不用我說了……

找到原因了，那麼解決方案自然也就隨之而來了，只要添加一行代碼就能完美解決這個報錯問題了：

async init() {  await this.getFileList()  if (this.fileList.length === 0) return  const [firstFileItem] = this.fileList  this.fileItem = firstFileItem}

即在inti()初始化方法中，添加一行判斷，當獲取到筆記列表數據後，如果筆記列表this.fileList為空的時候（即長度等於 0），則直接 return 掉，不需要為this.fileItem重新賦值。

2 右鍵菜單展示

有道雲筆記右鍵菜單動態展示

我們再來細看一下有道雲筆記的右鍵菜單的展示體驗，可以明顯看到，在不同的位置右鍵，菜單也就在不一樣的地方出現，也就是說右鍵菜單的位置是不固定（但是菜單左上角一定在當前選中的筆記內），所以菜單的定位值top和left是不能寫死的，只能是通過動態計算得出。從而我們可以得出大概流程就是：

  • 右側菜單使用ul列表展示
  • 使用v-if控制菜單的顯示隱藏
  • 使用鼠標右鍵事件contextmenu，計算菜單的定位

有了方案流程，就直接開幹吧！

2.1 列表展示

我們右鍵菜單是在FileList組件中的，我們就直接改造FileList組件即可。

<div class="filelist-container">  <!-- el-scrollba 組件內容已省略 -->  <ul v-if="visible" :style="{ left: `${left}px`, top: `${top}px` }" class="contextmenu">    <slot name="menu"></slot>  </ul></div>
// 菜單列表樣式.contextmenu {  position: absolute;  margin: 0;  padding: 5px 0;  background: #fff;  border-radius: 4px;  border: 1px solid #ebeef5;  box-shadow: 0 2px 12px 0 rgba(0, 0, 0, 0.1);  list-style-type: none;  color: #333;  font-size: 12px;  font-weight: 400;  z-index: 3000;  li {    margin: 0;    padding: 7px 16px;    cursor: pointer;    &:hover {      color: #66b1ff;      background: #ecf5ff;    }  }}

看上面代碼，我們新建一個菜單列表ul和el-scrollbar組件同級，再用一個 div 標籤包裹起來，菜單的定位直接使用style行內樣式控制。

  • visible變量是控制菜單的顯示與隱藏
  • left變量是控制菜單的左側定位
  • top變量是控制菜單的頂部定位

這裡我們使用了vue的具名插槽，即v-slot指令，具名其實就是給插槽起個名字（簡單粗暴的理解，哈哈哈~），還不理解的話請自行參考 vue 官網解釋吧。

可能有人會問：我們現在就兩三個菜單，直接寫兩三個li將事件拋出去就好了，搞什麼插槽呢？咋一聽沒毛病，我們現在確實只有兩三個菜單，那麼後面如果我們再新增兩三個菜單呢？再新增十個菜單呢？是不是一直要改FileList組件，不斷地將菜單事件拋出去，就問你怕不怕？而使用插槽，我們就可以杜絕此種狀況，而且從組件的設計角度來說，這裡也應該使用插槽會更好一點，不僅是便於後期維護，也能將業務代碼抽離出來。組件歸組件，業務代碼歸業務代碼，儘可能的不要將兩者糅雜混在一起，當然這也是個人建議，有更好想法的請給我留言，大家一起學習一起進步~~

2.2 右鍵邏輯

在筆記列表中添加鼠標右鍵事件contextmenu，將當前筆記對象fileTtem、index和$event傳遞到函數openMenu，傳遞index是為了動態切換當前高亮選中的筆記，這裡也使用到了 vue 事件綁定的語法糖.prevent。點擊openMenu就是我們執行右鍵事件的所有邏輯了：

  • 計算右鍵菜單的位置
  • 控制菜單展示出來
  • $emit將筆記數據傳回父組件執行相關業務

函數openMenu的具體代碼為：

openMenu(fileItem, index, e) {  const menuMinWidth = 105  const { offsetWidth } = this.$el // 獲取容器的寬  const maxLeft = offsetWidth - menuMinWidth // 左側邊界  const left = e.clientX + 15 // 15 => margin right  this.left = left > maxLeft ? maxLeft : left  this.top = e.clientY  this.visible = true  this.$emit('update:active', index)  this.$emit('update:selectedFile', fileItem) // 將當前右鍵選中的文件返回父組件}

menuMinWidth是為了計算防止菜單超出筆記列表的右側。

還需要在props中添加selectedFile: Object，到時候父子組件通過該變量傳遞當前右鍵菜單所選中的筆記信息。

菜單展示的邏輯我們寫完了，但是關閉的邏輯我們要怎麼寫呢，正常的右鍵菜單都是有兩個場景的關閉邏輯，一是當我們點擊執行其中一個菜單選擇的時候則關閉菜單層，二是當我點擊其它任意位置的時候也執行關閉菜單層。非常常見的操作，這點相信大家都不會有什麼疑問，那麼我們需要怎麼去代碼實現呢？

我們先來寫一個關閉菜單的方法，方法很簡單，只要將visible設置為false即可：

closeMenu() {  this.visible = false}

然後我們通過 watch 監聽變量visible，根據visible來動態添加/移除事件監聽方法，visible為true時添加點擊事件監聽，visible為false時移除點擊事件監聽，具體看下面代碼：

watch: {  visible(value) {    if (value) {      document.body.addEventListener('click', this.closeMenu)    } else {      document.body.removeEventListener('click', this.closeMenu)    }  }}

至此，FileList組件的右鍵菜單就算改造完畢了。

2.3 業務應用

組件搞完了，那就開始我們的實際業務使用了。我們的組件使用了具名插槽，直接使用v-slot即可：

<file-list :fileList="fileList" :active.sync="activeIndex" :selectedFile.sync="selectedFile">  <template v-slot:menu>    <li>置頂</li>    <li>刪除</li>  </template></file-list>

data中定義selectedFile對象，當右鍵菜單執行的時候，對應的筆記信息會傳遞過來，我們就可以操作我們具體的業務邏輯了：

data() {  return {    selectedFile: {} // 右鍵菜單選中的文件數據  }}

寫完這一步再來看看我們的客戶端頁面，是不是菜單已經出來了呀

3 右鍵菜單-刪除

刪除筆記的邏輯也很簡單，有道雲筆記是直接就刪除的，我們這裡做個小優化，加上一個二次確認的操作，防止誤刪除，畢竟我們現在數據沒有存雲上，刪除了就真的沒了……添加fileDetele方法，根據子組件傳回來的selectedFile對象拿到數據id，調用nedb api 執行：

fileDelete() {  this.$confirm('此操作將永久刪除該筆記, 是否繼續?', '提示', {    confirmButtonText: '確定',    cancelButtonText: '取消',    type: 'warning'  })    .then(() => {      const { _id } = this.selectedFile      this.$db.markdown        .remove({ _id })        .then(num => {          this.$message.success(`刪除了${num}個項目`)          this.init()        })        .catch(() => {          this.$message.error('刪除失敗')        })    })    .catch(() => {})}

this.init()，刪除成功後重新獲取列表數據，初始化一下即可。
