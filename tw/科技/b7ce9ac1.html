java併發編程之進程狀態

2020-11-14
| 科技

java線程中，線程狀態是如何轉換的呢？這一次我們一起來學習下。

線程狀態:

NEW:

線程創建之後，還沒有啟動。這時候它的狀態就是NEW

RUNNABLE:

正在Java虛擬機下跑任務的線程的狀態。在RUNNABLE狀態下的線程可能會處於等待狀態， 因為它正在等待cpu的時間片。

BLOCKED:

線程處在阻塞狀態，共分幾種情況，

同步阻塞：運行的線程在獲取對象的同步鎖時，若該同步鎖被別的線程佔用，則JVM會把該線程放入鎖池中。

其他阻塞：運行的線程執行sleep()或join()方法，或者發出了I/O請求時，JVM會把該線程置為阻塞狀態。

當sleep()狀態超時、join()等待線程終止或者超時、或者I/O處理完畢時，線程重新轉入就緒狀態。sleep方法不釋放持有的鎖

WAITING:

等待狀態，wait方法進入，無限期等待，等待其它線程顯式地喚醒，否則不會被分配 CPU 時間片。

進入原因:

1.沒有超時timeout參數的Object.wait()

2.沒有超時timeout參數的Thread.join()

3.LockSupport.park()

TIMED_WAITING:

超時等待狀態，無需等待其它線程顯式地喚醒，在一定時間之後會被系統自動喚醒。

1.Thread.sleep()方法

2.設置了超時timeout參數的Object.wait()方法

3.設置了超時timeout參數的Thread.join()方法

4.LockSupport.parkNanos()方法

5.LockSupport.parkUntil()方法

TERMINATED:

線程中止的狀態，這個線程已經完整地執行了它的任務。

“阻塞”與“等待”的區別：

（1）“阻塞”狀態是等待著獲取到一個排他鎖或資源(如IO輸入等)，進入“阻塞”狀態都是被動的，離開“阻塞”狀態是因為其它線程釋放了鎖；

（2）“等待”狀態是在等待一段時間 或者 喚醒動作的發生，進入“等待”狀態是主動的。

Java線程調度:

協同式:

線程的執行時間由線程本身來控制，線程把自己的工作執行完了之後，要主動通知系統切換到另外一個線程上。

搶佔式:

每個線程將由系統來分配執行時間，線程的切換不由線程本身來決定，Java採用這種方式調度，但我們可以"建議"操作系統，給予都些重要的線程更多的執行時間。如何做呢？就是通過“線程優先級”

線程優先級

java的線程Thread類中，有幾個常量定義優先級，

static int MAX_PRIORITY

線程可以具有的最高優先級10。

static int MIN_PRIORITY

線程可以具有的最低優先級1。

static int NORM_PRIORITY

分配給線程的默認優先級5。

而實際優先級為1-10，但使用除上面之外的其他優先級需要關注，對應操作系統可能與對應優先級別，建議使用上面三個。
