Wasm介紹之5：控制指令 | 火星技術帖

WebAssembly（簡稱Wasm）控制指令一共有11條，其中unreachable指令（操作碼0x00）和nop指令（操作碼0x01）比較簡單，不介紹。call指令（操作碼0x10）已經在上一篇文章裡介紹，call_indirect指令（操作碼0x11）將在下一篇文章裡介紹。本文重點討論block（操作碼0x02）、loop（操作碼0x03）、if（操作碼0x04）、br（操作碼0x0C）、br_if（操作碼0x0D）、br_table（操作碼0x0E）和return（操作碼0x0F）這7條指令。

block

block指令相當於一個無參的內聯（inline）函數調用。函數的返回值類型，也就是block指令的結果類型（Result Type，在後面的示意圖中簡稱rt）編碼後存儲在指令的第一個立即數參數裡。函數的指令（可能有很多條）編碼後存儲在第二個立即數參數裡。block指令必須以end指令（操作碼0x0B）結尾。由於end指令和後面將要介紹的else指令（操作碼0x05）只起到標記作用，沒有任何執行效果，所以沒有把這兩條指令計入控制指令。

Wasm1.0規範規定block指令的結果不能超過一個，所以rt可以用一個字節表示：0x40表示沒有結果、0x7F表示i32類型、0x7E表示i64類型、0x7D表示f32類型、0x7C表示f64類型。根據討論可知，block指令在執行時不會使用棧上已經存在的任何操作數，執行完畢後可能會在棧頂留下一個操作數，下面是它的示意圖：

bytecode:...][ block ][ rt ][ instrs ][ end ][...stack:|           |          |           | |           |         ➘|   ?(rt)   | # exec(instrs)|     d     |          |     d     | |     c     |          |     c     | |     b     |          |     b     |  |     a     |          |     a     | └───────────┘          └───────────┘

下面是一個非常簡單的WAT例子，展示了block指令的用法：

(module  (func (result i32)    ;; ... other instructions    (block (result i32)      (i32.const 100)    )  ))

loop

loop指令和block指令非常相似，區別僅在於如何跳出控制塊。後文介紹br指令時會進一步討論這個區別，下面先給出一個跳轉示意圖：

...][ block ][ rt ]...[ br ]...[ end ][...                         |        ↑                         └────────┘...][ loop ][ rt ]...[ br ]...[ end ][...       ↑               |       └───────────────┘

if

和block指令類似，if指令也類似於一個內聯函數。區別主要有兩點。第一，if內聯函數帶一個i32類型的參數。第二，if內聯函數帶有兩份代碼（兩條分支），中間用else指令隔開。if指令執行時，會先從棧頂彈出這個i32類型的參數，如果參數值不等於0，則執行分支1代碼，否則執行分支2代碼。下面是if指令的示意圖：

bytecode:...][ if ][ rt ][ instrs1 ][ else ][ instrs2 ][ end ][...stack:|           |          |           | |   e(i32)  |➚        ➘|   ?(rt)   | # exec(e!=0?instrs1:instrs2)|     d     |          |     d     ||     c     |          |     c     | |     b     |          |     b     |  |     a     |          |     a     | └───────────┘          └───────────┘

也可以把if指令的else分支省略，但是這種情況下if指令不能有任何結果。下面是省略else分支時if指令的示意圖：

bytecode:...][ if ][ 0x40 ][ instrs1 ][ end ][...stack:|           |          |           | |   e(i32)  |➚         |           | # e==1?exec(instrs1)|     d     |          |     d     ||     c     |          |     c     | |     b     |          |     b     |  |     a     |          |     a     | └───────────┘          └───────────┘

下面這個WAT例子展示瞭如何用if指令實現max函數：

(module  (func $max (param $a i32) (param $b i32) (result i32)    (i32.gt_s (local.get $a) (local.get $b))    (if (result i32)      (then (local.get $a))      (else (local.get $b))    )  ))

br

br指令（可以理解為break，或者branch）可以進行無條件跳轉。和傳統彙編語言裡的JUMP指令不同，br指令並不能跳轉到任意位置，而只能跳出（相對於block和if指令而言是跳出，相對於loop指令而言是重新開始，後面不再強調）其他控制指令產生的控制塊。br指令帶有一個u32類型（32位無符號整數）的立即數參數，指定跳轉的層數：0表示跳出當前循環，1代表跳出2層循環，以此類推。下面是一個WAT例子，展示了嵌套的block，以及br指令的用法：

(module  (func (export "main") (result i32)    (i32.const 100) (block (result i32)      (i32.const 200) (block (result i32)          (i32.const 300) (block (result i32)           (i32.const 123) (br n)        ) (i32.add)      ) (i32.add)    ) (i32.add)  ))

假設上面例子中的main函數已經執行到了br指令這裡，下圖展示了跳轉層數分別為0、1、2、3時操作數棧的變化情況：

bytecode:...][ br ][ depth ][...stack:|           |  |           |  |           |  |           |  |           ||           |  |           |  |           |  |           |  |           ||    123    |  |    123    |  |           |  |           |  |           ||    300    |  |    300    |  |    123    |  |           |  |           ||    200    |  |    200    |  |    200    |  |    123    |  |           ||    100    |  |    100    |  |    100    |  |    100    |  |    123    |└───────────┘  └───────────┘  └───────────┘  └───────────┘  └───────────┘ (before br)        n==0           n==1           n==2           n==3

br_if

br_if指令從棧頂彈出一個i32類型的操作數，如果操作數的值為0，則不跳轉，否則執行br邏輯。下面是br_if指令的示意圖：

bytecode:...][ br_if ][ depth ][...stack:|           |          |           | |   e(i32)  |➚         |           | |     d     |          |     ?     | # e!=0?br(depth)|     c     |          |           | |     b     |          |           |  |     a     |          |           | └───────────┘          └───────────┘

下面是一個稍微複雜一些的WAT例子，展示瞭如何用loop和br_if指令實現sum函數：

(module  (func $sum (param $from i32) (param $to i32) (result i32)    (local $n i32)    (loop $l      ;; $n += $from      (local.set $n (i32.add (local.get $n) (local.get $from)))      ;; $from++       (local.set $from (i32.add (local.get $from) (i32.const 1)))      ;; if $from

請注意loop指令並不會自動循環，必須和br等跳轉指令配合使用。

br_table

不管是br還是br_if指令，都只有一個立即數參數，能指定一個跳轉深度。br_table指令打破了這種限制，可以帶N+1個立即數參數，指定N+1個跳轉深度。br_table指令執行時，會從棧頂彈出一個i32類型的操作數n。如果n小於或等於N，則按第n個深度跳轉，否則按照最後一個深度跳轉。下面是br_table指令的示意圖：

bytecode:...][ br_table ][ labels... ][ default ][...stack:|           |          |           | |   n(i32)  |➚         |           ||     d     |          |     ?     | # d

return

return指令可以認為是br指令的特殊形式：直接跳出最外層循環（也就是整個函數）。return指令沒有立即數參數，不畫示意圖了。下面的WAT例子展示瞭如何用block、br_table和return指令實現Go等高級語言中的switch-case語句：

(module  (func $select3 (param $n i32) (param i32 i32 i32) (result i32)    (block      (block        (block          (local.get $n)          (br_table 0 1 2)        )        (return (local.get 1))      )      (return (local.get 2))    )    (return (local.get 3))  ))

如果把上面例子中定義的select3函數翻譯成Go語言代碼的話，應該是下面這樣：

func select3(n, a, b, c int32) int32 {  switch n {    case 0 : return a    case 1 : return b    default: return c  }}
