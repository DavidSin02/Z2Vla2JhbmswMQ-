Linux之系統移植介紹

2020-10-29
| 科技

在進行 Linux 驅動開發之前肯定需要先將Linux 系統移植到開發板上去。如果學習過UCOS/FreeRTOS 應該知道，UCOS/FreeRTOS 移植就是在官方的 SDK 包裡面找一個和自己所使用的芯片一樣的工程編譯一下，然後下載到開發板就可以了。


問： 那麼 Linux 的移植是不是也是這樣的呢？下載 Linux 源碼，然後找個和我們所使用的芯片一樣的工程編譯一下就可以了呢？？？

答：很明顯不是的！！！因為 Linux 的移植要複雜的多，在移植 Linux之前我們需要先移植一個 bootloader 代碼，這個 bootloader 代碼用於啟動 Linux 內核，bootloader有很多，常用的就是 U-Boot。移植好 U-Boot 以後再移植 Linux 內核，移植完 Linux 內核以後Linux 還不能正常啟動，還需要再移植一個根文件系統(rootfs)，根文件系統裡面包含了一些最常用的命令和文件。所以 U-Boot、Linux kernel 和 rootfs 這三者一起構成了一個完整的 Linux 系統，一個可以正常使用、功能完善的 Linux 系統。
