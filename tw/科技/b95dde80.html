Linux技巧：在代碼中設置終端字符顯示顏色和移動光標位置

2020-11-14
| 科技

本篇文章介紹如何在代碼中使用 ANSI 轉義碼來設置終端的字符顯示顏色、移動光標位置等，並實現一個進度條百分比跳變的效果。

ANSI 轉義碼

在 Linux 中，可以使用 ANSI 轉義碼（ANSI escape codes）設置終端的字符顯示顏色、移動光標位置、清除字符顯示等。

ANSI 轉義碼是由終端自身支持，獨立於編程語言之外，可以在 C 語言、Java、Python、或者 Shell 中使用。

下面以 bash shell 為例來說明如何使用 ANSI 轉義碼。

ANSI 轉義碼格式

ANSI 轉義碼由一串 ASCII 編碼的字符串組成，要求以 ASCII 編碼的 Escape 字符和 [ 字符開頭，後面跟著具體的轉義碼，指定相應的操作。基本格式如下：

Esc[escape code

Escape 字符也就是 Esc 鍵對應的字符。

由於按 Esc 鍵，不會得到一個可顯示的字符，需要用具體的編碼值來表示這個字符。

在不同編程語言中，表示字符編碼值的寫法可能不一樣。一般常用 \e 轉義字符來表示 Esc 字符。

使用 echo 命令測試 ANSI 轉義碼

在 bash shell 中，可以使用 echo 命令的 -e 選項來測試 ANSI 轉義碼。

查看 man echo 對 -e 選項說明如下：

-e

enable interpretation of backslash escapes.

If -e is in effect, the following sequences are recognized:

\e

escape

\0NNN

byte with octal value NNN (1 to 3 digits).

即，在 echo 命令中，-e 選項可以指定處理轉義字符。

\e 轉義字符表示 escape 字符。

\0NNN 轉義字符使用八進制來獲取 NNN 編碼值對應的字符。

在 ASCII 編碼中，Escape 字符對應的八進制值是 033。

則在 echo 命令中，\033 表示 escape 字符。

使用 echo 命令測試 ANSI 轉義碼時，可以寫為 echo -e "\033[31m"。

這裡的 31m 轉義碼錶示要把終端字符的前景色設成紅色。

Linux 的 printf 命令也可以輸出 ANSI 轉義碼，而且不需要加 -e 選項，例如寫為 `printf "\e[31m"。

注意：這裡需要用雙引號、或者單引號把 \033[31m 括起來，避免 bash 自身對 \ 進行轉義，會去掉 \ 字符，導致 echo 命令收不到 \ 字符，無法處理轉義字符。

也可以寫為 echo -e "\e[31m"，\e 也表示 escape 字符。

後面測試的時候，統一使用 \e 的形式，少輸入一些字符。

使用ANSI轉義碼設置終端的字符顯示顏色

執行 echo -e "\e[31m" 命令後，終端的提示字符會變成紅色，之後輸入的字符也都會變成紅色。

即，終端的默認字符顏色變成了紅色。

執行 echo -e "\e[0m" 命令重置終端屬性，讓終端的字符顏色變成原來的默認顏色。

這裡的 0m 轉義碼錶示重置字符顯示屬性。

一般來說，為了不影響終端自身的顯示，使用 ANSI 轉義碼設置某個字符串的顯示顏色後，建議隨後使用 0m 轉義碼來重置為原來的顏色。

舉例說明如下：

在上面命令中，\e[31m 是一個 ANSI 轉義碼，表示設置終端字符顏色為紅色。

\e[0m 也是一個 ANSI 轉義碼，表示重置終端的顏色屬性，會恢復成原來的顏色。

在這兩個轉義碼中間的字符串會顯示在終端上。

執行該命令後，終端的提示符會顯示為原來的顏色。

設置終端字符顏色的 ANSI 轉義碼

下面詳細說明設置終端字符顏色的 ANSI 轉義碼，其基本格式如下：

Esc[Value;...;Valuem

這裡的 Value 可以提供多個值，不同值之間用分號 ‘;’ 隔開。

這些值可以分別指定字符的前景色、背景色、字符屬性（粗體、下劃線、反轉）。它們之間的順序不限。

轉義碼最後以 m 字符結尾。

可以看到，\e[31;44m、\e[44;31m 這兩個轉義碼設置的字符顏色效果是一樣的。所給的前景色、背景色沒有要求先後順序。

目前的大部分終端都支持 256 色，可以使用 Esc[38;5;Valuem 來設置終端字符為 256 色。這裡的 Value 取值是 0-255。

例如，echo -e "\e[38;5;111mAAAAAA\e[0m" 命令設置為 111 對應的顏色。

具體的顏色取值可以查看 256 色的顏色表。網上的很多文章都有說明。這裡不再列舉。

使用 ANSI 轉義碼移動終端光標

ANSI 轉義碼可以用來移動終端的光標位置，從而改變字符的輸出位置。

具體舉例如下：

$ echo -e "123456789\e[4Dabc"12345abc9

在這個命令中，\e[4D 轉義碼錶示把光標往左移動 4 列。

可以看到，光標移動 4 列後，位於字符 6 所在的位置，重新輸出 abc，覆蓋了原來的 678 三個字符。

移動光標的具體轉義碼說明如下：

移動終端光標位置的ANSI轉義碼

上面所說的終端位置指的是終端可見的窗口位置，不包括緩衝區位置。

即，窗口顯示不會發生滾動，只在當前可見的窗口區域跳轉光標。

注意：由於 echo 命令默認會輸出換行符，導致移動光標後再次換行，會對光標移動效果造成干擾。

在測試移動光標的轉義碼時，建議用 printf 命令測試。該命令默認不會輸出換行符。

由於 bash 裡面需要按下回車才執行命令，會影響光標的左右移動效果，建議在 printf 自身輸出的內容中左右移動光標。

實際測試發現，光標右移 n 列，光標會位於第 n 列的後面，之後輸出的字符串會從 n+1 列開始。

Esc[C、Esc[0C、和 Esc[1C 的效果相同，都是光標右移 1 列。

類似的，Esc[D、Esc[0D、和 Esc[1D 的效果相同，都是光標左移 1 列。

使用 printf 命令測試如下：

$ printf "123456789\e[1Da\n"12345678a$ printf "123456789\e[0Da\n"12345678a$ printf "123456789\e[Da\n"12345678a

可以看到，使用 \e[D、\e[0D、\e[1D 往左移動光標，然後輸出字符 a，都是覆蓋同一個字符 9。

這三個轉義碼的光標移動效果相同。

$ printf "123456789\e[4Da\n"12345a789$ printf "123456789\e[4D\e[Ca\n"123456a89$ printf "123456789\e[4D\e[0Ca\n"123456a89$ printf "123456789\e[4D\e[1Ca\n"123456a89

\e[4D 把光標左移 4 列，移動到字符 6 的位置。

\e[C、\e[0C、\e[1C 都是往右移動光標到下一列，到字符 7 的位置，輸出字符 a，覆蓋了字符 7。

通過移動光標實現進度百分比的效果

我們可以通過移動光標實現進度百分比的效果。假設有一個 progress.sh 腳本，內容如下：

#!/bin/bashfor ((i = 0; i <= 100; ++i)); do    printf "\e[5D%3d%%" $i    sleep 0.1sdoneecho

這裡使用 printf 命令進行輸出，以便格式化字符串。printf 命令也是使用 \e 來表示 escape 字符。

\e[5D 轉義碼錶示把光標左移 5 列。

由於所輸出的字符不超過 5 個字符，每次光標左移 5 列，都會移動到最左邊，從第一列開始輸出。

那麼後面輸出的內容會覆蓋前面輸出的內容，達到在同一行重複輸出的效果。

sleep 0.1s 命令表示暫停 0.1 秒。添加這個語句，以便清楚地看到進度百分比跳變。否則執行過快，百分比很快就跳到 100%。

執行 progress.sh 腳本的結果如下：

$ ./progress.sh100%

這裡不是動圖，看不到進度百分比跳變。實際執行就能看到。

從結果來看，在 for 循環中多次打印信息，這些信息都打印在同一行，並覆蓋前面的輸出。而不是換行打印。

通過移動光標實現進度條的效果

下面通過移動光標實現進度條的效果。假設有一個 progressbar.sh 腳本，內容如下：

#!/bin/bashfunction print_chars(){    # 傳入的第一個參數指定要打印的字符串    local char="$1"    # 傳入的第二個參數指定要打印多少次指定的字符串    local number="$2"    local c    for ((c = 0; c < number; ++c)); do        printf "$char"    done}declare -i end=50for ((i = 1; i <= end; ++i)); do    printf "\e[80D["    print_chars "#" $i    print_chars " " $((end - i))    printf "] %3d%%" $((i * 2))    sleep 0.1sdoneecho

這個腳本定義了一個 print_chars 函數，可以多次打印同一個字符。

printf "\e[80D[" 語句把光標左移 80 列。由於這個進度條的字符總長度小於 80，會移動最左邊，總是從第一列開始輸出。

在 ‘80D’ 後面的 ‘[’ 字符是進度條的開頭第一個字符。

print_chars "#" $i 語句遞增打印多個 # 字符，形成進度條往前移動的效果。

print_chars " " $((end - i)) 語句打印多個空格，填充到指定的最後一列，讓進度條的結束字符總是打印在同一列。

printf "] %3d%%" $((i * 2)) 語句打印進度條的結束字符 ]、以及進度條百分比。

sleep 0.1s 語句暫停 0.1 秒，避免執行過快，看不到進度條的移動效果。

執行 progressbar.sh 腳本的結果如下：

$ ./progressbar.sh[##################################################] 100%

這裡不是動圖，看不到進度百分比跳變。實際執行就能看到。

使用 ANSI 轉義碼清屏、清除字符

下面的 ANSI 轉義碼可以用於清屏、清除光標往後的字符。

ANSI轉義碼清除終端字符

注意：上面的 J、K 都是大寫字母。

具體舉例說明如下：

$ printf "123456789\e[5D\e[K\n"1234$ printf "123456789\e[5D\e[1K\n"     6789$ printf "123456789\e[5D\e[2K\n"

printf "123456789\e[5D\e[K\n" 命令先光標左移 5 列，停在字符 5 的位置，然後用 \e[K 轉義碼從光標位置往後清除所有字符，只保留了前面的 1234 字符串。

printf "123456789\e[5D\e[1K\n" 命令使用 \e[1K 轉義碼從光標位置往前清除所有字符，只保留了後面的 6789 字符串。

printf "123456789\e[5D\e[2K\n" 命令使用 \e[2K 轉義碼清除光標所在的整行內容，輸出內容為空。
