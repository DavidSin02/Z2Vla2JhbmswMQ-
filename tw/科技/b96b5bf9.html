如何正確認識Java多態

2020-11-14
| 科技

多態在Java技術具有很重要的位置，在Java面試中是必問的知識點，那我們如何正確的認識和理解多態呢？

多態的理解

所謂多態，就是指同一個行為具有多個不同表現形式或形態的能力。也可以理解為，多態是指通過指向父類的指針，來調用在不同子類中實現的方法。

多態存在的必要條件

  1. 繼承
  2. 重寫
  3. 父類引用指向子類對象（即向上轉行）

例如：

 Parent p = new Child();

當使用多態方式調用方法時，首先檢查父類中是否有該方法，如果沒有，則編譯錯誤；如果有，再去調用子類的同名方法。

多態的實現方式

1、重寫父類方法

父類和繼承該父類的一個或多個子類對某些方法的重寫，多個子類對同一方法的重寫可以表現出不同的行為

例如：

class Animal {	public void eat() {		System.out.println("動物吃飯");	}}class Cat extends Animal {	//重寫父類方法	public void eat() {		System.out.println("貓吃飯");	}}class Demo_Test1 {	public static void main(String[] args) {		Animal am = new Cat();		am.eat();//打印“貓吃飯”	}}

2、接口實現

interface Animal1 {	public void eat();}class Pig implements Animal1 {	//實現父類	@Override	public void eat() {		System.out.println("豬吃飯");	}}class Dog implements Animal1 {	//實現父類	@Override	public void eat() {		System.out.println("狗吃飯");	}}public class Demo_Test2 {	public static void main(String[] args) {		Animal1 am = new Pig();		am.eat();//打印“豬吃飯”		Animal1 am1 = new Dog();		am1.eat();//打印“狗吃飯”	}}

3、實現抽象類的抽象方法

abstract class Animal1 {		public abstract void eat();		public void play() {		System.out.println("動物在玩");	}}class Pig extends Animal1 {	//實現父類抽象方法	public void eat() {		System.out.println("豬吃飯");	}	//重寫父類方法	public void play() {		System.out.println("豬在玩");	}}public class Demo_Test2 {	public static void main(String[] args) {		Animal1 am = new Pig();		am.eat();//打印“豬吃飯”		am.play();//打印“豬在玩”	}}

後記：很多人認為重載屬於多態，我個人不認同這個觀點，重載是指方法名相同，參數不同的多個方法，這叫重載，重載首先和繼承沒有任何關係，更不涉及到覆蓋父類的屬性或者方法，也和父類動態的去引用子類對象沒有任何關係，所以說，重載不是多態！
