01 共享單車租賃需求迴歸係數分析

本文中，我們將構建一個簡單直觀的模型，並使其與不同的環境因素進行交互，進而瞭解這些環境因素是如何影響自行車租賃需求的。對用戶來說，這會是一種很好的方式，因為這種方式可以讓他們確認關於什麼會讓他們的用戶租賃或不租賃自行車的直觀判斷。

當然，在有些情況下，他們也會感到很驚訝（比如，冬天騎車的人居然會比夏天還多——本文中，我們將讓你親眼看到這個現象）。

這個Web應用背後的“大腦”是一個線性迴歸模型（linear regression model）。它能夠發現歷史數據集與模型輸出結果之間的線性關係。利用這個原理，通過線性迴歸模型，我們就可推斷隨著時間推移和不同環境參數的變化，自行車租賃在需求上的變化。最終我們希望看到的是，這個模型能否幫助我們預測未來的自行車租賃需求。

記住一點，無論何時，只要你打算將一個Python模型擴展到Web應用，那麼在添加任何構建Web應用程序所需的額外層之前，必須解決該模型中當前的所有問題和缺陷（bug）。在將任何應用或內容移至雲端前，請先把所有簡單問題解決掉！遵循這條建議，後續會減少你很多不必要的麻煩。

02 探索共享單車原始數據集

雖然是一種興起不久的服務，共享單車已經受到極大歡迎。根據人工智能與決策支持實驗室的Hadi Fanaee-T的說法（來自UCI機器學習數據集信息庫講義筆記）：

與公共汽車或地鐵等其他交通服務相反，共享單車模式中，騎行時長、出發和到達位置在系統中都被明確記錄。這一功能將自行車共享系統變成了虛擬傳感器網絡，利用這個網絡，可以感知到一個城市的移動性。因此，通過監測這些數據，城市中發生的很多重要事件都有希望被監控到。

下載的數據集包括兩部分：hour.csv和day.csv，特徵細節描述如下：

  • instant：記錄索引
  • dteday：日期
  • season：季節（1：春　2：夏　3：秋　4：冬）
  • yr：年份（0：2011，1：2012）
  • mnth：月份（1～12）
  • hr：小時（0～23）
  • holiday：是否是假期
  • weekday：工作日
  • workingday：如果既不是週末也不是假期則值為1，否則為0
  • weathersit：
  • 1. 晴朗，雲層很少，部分多雲
  • 2. 薄霧+多雲，薄霧+短暫雲，薄霧+少雲，薄霧
  • 3. 小雪，小雨+雷雨+散雲，小雨+散雲
  • 4. 大雨+冰粒+雷暴+薄霧，雪+霧
  • temp：標準化溫度（攝氏度）
  • 計算公式：(t-t_min)/(t_max-t_min), t_min = -8, t_max = +39（僅在小時範圍內）
  • atemp：標準化體表溫度（攝氏度）
  • 計算公式：(t-t_min)/(t_max-t_min), t_min = -16, t_max = +50 (僅在小時範圍內)
  • hum：歸一化溼度。值除以100（最大值）
  • windspeed：歸一化風速。值除以67（最大值）
  • casual：臨時用戶數
  • registered：註冊用戶數
  • cnt：租賃自行車總數，包括臨時用戶和註冊用戶

03 數據建模準備工作

在大多數數據科學項目中，存在一個數據預處理階段，在這個階段會進行數據評估和清理，以實現數據的“模型就緒”。在清理階段，我們已經放棄了一些無用的數據特徵，同時也清除了可能存在的空值，我們也不用擔心相關性或多重共線性，因為在最終模型中，我們僅會使用四個簡單的特徵。

1. 迴歸建模

在統計分析中，迴歸模型試圖預測變量之間的關係。它主要用於分析獨立變量與依賴變量的關係，而擬合模型（fitted model）則可用於預測依賴變量的新變化。

2. 簡單線性迴歸

線性迴歸可能是最簡單的建模算法。它試圖解釋一個因變量和一個或多個自變量之間的關係。基本的線性迴歸方程如圖13所示。

▲圖13 基本線性迴歸方程

在上述方程中，y是因變量，β0是常數，β1是迴歸係數，x是自變量。

3. 簡單線性迴歸模型

這裡，從一個簡單的多線性迴歸模型開始，我們輸入所有變量並獲得均方根誤差（RMSE）。RMSE最終轉化到與結果變量（也稱為y軸標籤）相同的單位來表示誤差（即RMSE值與y值具有相同量綱），因此很容易看出模型在學習/預測自行車租賃方面表現如何，而誤差是置信區間的一種表現形式。

你所關注的是儘可能希望獲得較低的RMSE值，因此我們的目標是不斷調整數據和模型，直到RMSE值停止繼續降低為止。本文中，我們所有建模工作都基於Python scitkit-learn /sklearn庫。這是一個很了不起的Python庫，幾乎可以滿足大多數Python用戶的建模需求。

儘管我們只打算進行簡單的線性迴歸，但我們還是用到了sklearn庫中的三個函數：train_test_split函數從原始數據中創建兩個隨機數據集，並從結果中分離特徵；linear_model函數運行我們的模型；mean_squared_error函數評估模型的學習效果（代碼清單⑧）。

  • 代碼清單⑧ 將數據集拆分為訓練和測試兩部分的代碼片段
outcome = 'cnt'# create feature listfeatures = [feat for feat in list(bike_df_model_ready) if feat not in[outcome, 'instant']]# split data into train and test portionsfrom sklearn.model_selection import train_test_splitX_train, X_test, y_train, y_test = train_test_split(bike_df_model_\ready[features],bike_df_model_ready[['cnt']],test_size=0.3,\ random_state=42)

train_test_split()函數基於用戶指定的種子參數將原始數據集拆分為兩個隨機數據集。無論何時，如果想要測試不同的方法，並希望確保始終使用相同的拆分以進行不同方法的公平比較，則設置random_state種子參數是個不錯的選擇。

test_size參數設置測試拆分的大小，在這裡我們將它設置為0.3或30%，因此最終結果將是數據集中70%的數據分配給了訓練集，而剩餘30%的數據分配給測試集（代碼清單⑨）。

  • 代碼清單⑨ 線性迴歸代碼
from sklearn import linear_model model_lr = linear_model.LinearRegression()# train the model on training portionmodel_lr.fit(X_train, y_train)

我們事先聲明一個LinearRegression()模型，然後調用fit()函數，使用訓練數據和訓練標籤來訓練模型。執行上述代碼後，模型model_lr即經過訓練並準備好進行預測（代碼清單⑩）。

  • 代碼清單⑩ 預測並獲取均方根誤差RMSE的代碼

輸入：

predictions = model_lr.predict(X_test)from sklearn.metrics import mean_squared_errorprint("Root Mean squared error: %.2f" % \ sqrt(mean_squared_error(y_test,predictions)))

輸出：

Root Mean squared error: 143.08

最後，我們調用函數predict()，並將分配為測試集的剩餘30%數據傳入函數，然後將預測標籤值輸入函數mean_squared_error()求取均方根誤差值。我們最終得到的RMSE值為143.08，同時我們將其作為後續預測的基準參考值。

上述結果，也就是我們基於目前選擇的數據特徵和拆分種子值（我們在train_test_split函數上應用的種子，這可以確保我們每次都得到相同的數據分割）所得到的結果。

理解這個RMSE值的一種方式，就是我們的模型預測的偏差是143輛自行車（因為它與我們的結果變量處於相同的量綱）。考慮到每小時自行車平均租賃數量大約是190輛，因此，與簡單採用自行車租賃的全局平均值相比，我們的模型做得更好。

我們能否通過幾種技術手段（主要包括多項式、非線性建模和利用時間序列），獲取更好的預測模型？由於篇幅有限，關於此問題的討論請見《機器學習即服務》一書第2.4節。
