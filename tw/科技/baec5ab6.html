PHP進階教程-設計模式之裝飾器模式

2020-11-14
| 科技

裝飾器模式（Decorator Pattern）允許向一個現有的對象添加新的功能，同時又不改變其結構。這種類型的設計模式屬於結構型模式，它是作為現有的類的一個包裝。

其實說明白點就是在不動原來類點基礎上去擴展一些新的功能，在客戶端使用的過程中不會感覺到使用裝飾器和沒有使用裝飾器的區別，但是它們的結果會改變。

在生活中有很多裝飾器模式的例子存在，例如有些人給手機貼膜和安裝手機殼、女孩子化妝購買漂亮的衣服打扮自己、給房子進行裝修等等這些操作實際上都是一個裝飾器模式的實現。

給手機貼膜並沒有改變手機的功能和原本的屬性結構，女孩子化妝並沒有動刀子給自己整容，給房子裝修在房子的基礎骨架上進行美觀。所以說裝飾器模式是在不改變一個類的結構但是需要給它添加新的功能。

意圖

動態地給一個對象添加一些額外的職責。就增加功能來說，裝飾器模式相比生成子類更為靈活。

解決問題

降低訪問複雜系統的內部子系統時的複雜度，簡化客戶端與之的接口，避免出現太多的派生子類。

UML圖

該模式中包含的角色及其職責

抽象構件（Component）

定義一個抽象接口以規範準備接收附加責任的對象，也就是最原始的對象。

具體構件（ConcreteComponent）

實現抽象構件，通過裝飾角色為其添加一些職責。通過繼承實現Component抽象類中的抽象方法。是最核心、最原始、最基本的接口或抽象類的實現，我們要裝飾的就是它。

抽象裝飾類（Decorator）

繼承抽象構件，幷包含具體構件的實例，可以通過其子類擴展具體構件的功能。

具體裝飾類（ConcreteDecorator）

實現抽象裝飾的相關方法，並給具體構件對象添加附加的責任。

優缺點

優點

  • 裝飾模式與繼承關係的目的都是要擴展對象的功能，但是裝飾模式可以提供比繼承更多的靈活性。
  • 可以通過一種動態的方式來擴展一個對象的功能，通過配置文件可以在運行時選擇不同的裝飾器，從而實現不同的行為。
  • 通過使用不同的具體裝飾類以及這些裝飾類的排列組合，可以創造出很多不同行為的組合。可以使用多個具體裝飾類來裝飾同一對象，得到功能更為強大的對象。
  • 具體構件類與具體裝飾類可以獨立變化，用戶可以根據需要增加新的具體構件類和具體裝飾類，在使用時再對其進行組合，原有代碼無須改變，符合“開閉原則”

缺點

  • 比繼承更加靈活機動的特性，也同時意味著更加多的複雜性。
  • 裝飾模式會導致設計中出現許多小類，如果過度使用，會使程序變得很複雜。
  • 多層裝飾比較複雜，排除問題需要逐級排除增加了排除的難度。

應用場景

  • 不通過繼承來擴展功能的時候。
  • 當對象的功能要求可以動態地添加，也可以再動態地撤銷時。
  • 當現有的功能有比較多複雜擴展，但是不能通過繼承難以實現的時候。

案例背景

現在我們成了億萬富翁，買了一套大別墅，在入住之前我們需要把房子裝修。現在的裝修風格有很多種，將使用裝飾器模式來實現一個房子裝修的案例。

示例代碼

HouseComponent.php 抽象構件（Component）房子的抽象

interface HouseComponent{    public function decoration();}

HouseConcreteComponent.php 具體構件（ConcreteComponent）具體的房子

class HouseConcreteComponent implements HouseComponent{    public function decoration(){        echo "房子裝修\n";    }}

Decorator.php 抽象裝飾類（Decorator）裝修風格抽象

abstract class  Decorator implements HouseComponent{    protected $component;    /**     * Decorator constructor.     * @param $component     */    public function __construct(HouseComponent $component){        $this->component = $component;    }}

ChineseStyle.php 具體裝飾類（ConcreteDecorator）中式風格

class ChineseStyle extends Decorator{    public function decoration(){        echo "中國風格------";        $this->component->decoration();    }}

EuropeanStyle.php 具體裝飾類（ConcreteDecorator）歐式風格

class EuropeanStyle extends Decorator{    private function doing(){        echo "歐式風格------";    }    public function decoration(){        $this->doing();        $this->component->decoration();    }}

調用代碼：

// 原本的樣子$house1 = new HouseConcreteComponent();$house1->decoration();// 採用中式風格$house2 = new ChineseStyle(new HouseConcreteComponent());$house2->decoration();// 採用歐式風格$house3 = new EuropeanStyle(new HouseConcreteComponent());$house3->decoration();

輸出結果：

房子裝修中國風格------房子裝修歐式風格------房子裝修

上面的內容就是經過包裝之後輸出的內容。

第一個沒有任何處理所以默認類裡面的是什麼就是什麼。

第二個採用了 ChineseStyle 來裝飾，所以輸出中國風格。

第三個採用了 EuropeanStyle 來裝飾，所以輸出歐洲風格。

房子本質不變，採用不同的裝飾器可以有不同的變化，這就是裝飾器的魔力所在。給大家留一個作業，可以去思考一下用裝飾器模式去構造一下手機、變形金剛、汽車、商品等等業務的實現。

一句話總結：千變萬化，本質不變。
