西門子 PCS 7 系統結構

2020-10-29
| 科技

西門子 SIMATIC PCS 7 系統是現代 DCS 的一個實例（分佈式控制系統），它採用了當前的LAN （局域網） 技術、久經考驗的西門子 PLC（可編程序控制器）和現場總線技術。整個系統由大量的西門子硬件組件組成，包括儀表、執行機構、模擬和數字信號模塊，控制器、通訊處理器、工程師站和操作員站等。所有硬件組件由 PCS 7 軟件工具支持和組態。軟件系統高度地滿足了各種工業場所不同自動化任務的設計、組態和調試方面的要求，如化工，石化，製藥，水處理和電廠等行業。


典型的系統配置

下圖顯示了一個典型的PCS 7系統，圖中ES為工程師站，OS為操作員站，AS 為自動化站。


ES：工程師站

PCS 7 項目是在PCS 7工程師站上設計的。工程師站安裝有PCS 7組態工具，可以和自動化站和操作員站進行通訊。

PCS 7 ES 提供了強大的組態工具，例如：

  • SIMATIC Manager（SIMATIC管理器）：項目創建、庫創建、項目管理和診斷等。
  • PH: Plant Hierarchy（工廠層級）。用於工廠層級的設計。
  • HW Config：Hardware Configuration Environment（硬件配置環境）。用於配置 CPU、 通訊處理器、外圍設備和現場總線等。
  • CFC：Continuous Function Chart（連續功能圖）。用於設計庫、自動化邏輯、聯鎖、算法和控制等。
  • SFC：Sequential Function Chart（順序功能圖）。用於設計順序控制、邏輯和聯鎖等。
  • SCL：Structured Control Language（結構化控制語言）。用於編寫算法程序和創建功能塊等。
  • IEA: Import Export Assistant（導入導出助手）。用於生成控制模型、過程標籤類型和副本。
  • WinCC：Windows Control Centre（Windows 控制中心）。PCS 7 操作員界面和可視化。
  • Graphi cs Desi gner edi t or（圖形編輯器）：圖片、圖形對象和動畫的設計。
  • Commissioning Wizard（調試嚮導）：當 PC 重新啟動時，調試嚮導自動檢測新近安裝的 SIMATIC 模塊（即插即用），並一步步指導用戶安裝和配置 PC 站。
  • Configuration Console（配置控制檯）：使用配置控制檯可以在 Commissioning Wizard (調試嚮導) 運行後改變設置。用於配置訪問點。
  • Station Configurator（站配置器）：顯示 PCS 7 系統中建立的實際的 PC配置。
  • Multiproject（多項目）：在 SIMATIC 管理器中，您可以創建項目（單項目）或多項目。一個多項目可以包含幾個項目和一個主數據庫。
  • Master data library（主數據庫）：一個主數據庫是與一個多項目聯繫在一起的。它不同於其它系統或應用專用庫，一個主數據庫是存在於多項目中，並收集該多項目使用的所有功能類型。
  • Process Object View（過程對象視圖）：組態中會創建許多對象。過程對象視圖包含了一個項目的所有組態內容。可在視圖中顯示這些對象並進行編輯。
  • SIMATIC NET：調試嚮導，組態控制檯和站配置器是 SIMATIC NET 的接口。SIMATIC NET 是一個 SIMATIC項目中配置網絡和總線系統的平臺。


在ES中組態PCS 7項目可以分成2個階段：AS engineering（AS組態）和 OS engineering （OS組態）。AS 組態包括了工廠層級、功能塊、CFC、SFC 和硬件及通訊組件配置的設計。使用“Compiling OS（編譯OS）”功能 ，可以自動得到OS組態的項目數據。OS 組態是設計操作功能和圖形。


AS: 自動化站

一個自動化站可由電源（ （PS），中央處理單元（CPU），通訊處理器（CP）和輸入和輸出模塊組成的。CPU 處理操作系統和程序。PCS 7 CPU 是從西門子 SIMATIC S7 400 CPU 系列中選擇的。它們通過 System/Palnt Bus（系統/工廠總線）與 ES 和/或 OS 服務器進行通訊。AS也有一個通訊端口通過 Profibus DP 與現場設備進行通訊。下圖 是一個連接到分佈式I/O的自動化系統典型例子。


現場設備

PCS 7 系統的一個很大的優點是：採用現場總線技術把各種現場設備和儀表無鏠集成於它們的中央控制系統中。西門子公司本身和各種設備供應商提供了多系列遵守 Profibus 協議的驅動器、變送器、傳感器和儀表。Profibus 支持本質安全型儀表，並具有 HART 和 AS 模塊的接口。下圖 顯示了 PCS 7 系統設備級的概況。


Profibus DP 和 Profibus PA

Profibus DP 的設計是來替代製造自動化中傳統的 24 V 並行信號傳輸，過程自動化中 4 – 20 mA 或 HART 模擬信號傳輸。Profibus PA 協議集定義了典型現場設備如測量變送器或定位器的參數和性能。PA 協議集適用於 4 – 20 m A 和 Hart的模擬量傳輸。把 Profibus 技術與傳統的電纜技術相比較，好處是顯而易見的。分離器、終端和分佈設備由一個 Profibus 系統所取代。圖形化界面和全面的診斷工具的應用可以高效地安裝總線系統。通過光纖，比如建築物之間的長距離通訊，開始成為切實可行的。Profibus 的簡易性大大減少了調試和服務的工作量。下圖 顯示了使用 Profibus 技術的簡易性。


用於現場設備的 PCS 7 組態工具（PDM）


PDM（Process Device Manager 過程設備管理器）能集成於 PCS 7 組態系統中，或作為單機控制檯使用。SIMATIC PDM是一種用於現場設備和自動化組件調試、維護、診斷和顯示的工具。下圖 顯示了可以在其中校驗設備，在總線中設定設備地址，以及與設備進行在線通訊的軟件環境。



PCS 7 系統也提供庫功能，把設備集成至自動化設計中。下圖顯示了通過功能塊 PA_AI把壓力變送器讀數讀入CFC 中，並在 OS 中顯示數值。


OS: 操作員站

OS 可以是指 OS 服務器、OS 客戶機、OS 項目。“服務器”通常是指物理機器。一個 OS 項目可以是 Single-user（單用戶）項目、Multi-user（多用戶）項目或者 Client（客戶機）項目。“單用戶或多用戶系統”是指一個 OS 項目的類型。

（1）OS 組態

OS 組態是在一個 ES 中實施的。它包含了下列功能：

• 圖形對象的設計（按鈕、幻燈、趨勢、面板等）。

• SFV：Sequential Function chart Visualisation（SFC可視化）。用於在 OS 運行系統中顯示 SFC。

• 數據歸檔（變量和消息）以及長期數據存儲的設計。

• 報表的設計。系統和過程數據的打印輸出。

• 用戶管理。針對不同的操作任務，分配和控制用戶訪問權限。

• 冗餘。配置與主服務器連接的第二個 OS 服務器。如果兩個服務器之一發生故障，第二個服務器即承擔整個系統的控制。故障服務器恢復工作後，所有消息和過程文檔的內容被複制和同步。

• 時間同步。一個 OS 在運行時間裡作為時間主站，對連接到系統總線和終端總線的所有其它 OS 和 AS， 以當前時間進行時間同步控制。

• 設備狀態監視。設備狀態監視是用於持續地監視獨立系統（OS 和 AS），並在運行系統中以屏顯的方式顯示其結果。

? 與其它應用軟件的連接。PCS 7 OS 為用戶的解決方案提供開放式接口。這樣，就可以把 PCS 7 OS 集成於複雜的、全

公司範圍內的自動化解決方案中。

（2）單用戶項目

一個單用戶項目是用於一個獨立的操作站。適合於在一臺 PC 上運行服務器和客戶機功能的小系統場合。

（3）多用戶項目

一個多用戶項目是用在服務器/客戶機環境中，項目數據庫位於服務器中，而客戶機可以訪問數據庫。

（4）OS 服務器

OS 服務器與 AS 和 OS 客戶機進行通訊。一臺服務器最多可以被 32 臺客戶機所訪問。一臺服務器可以訪問大約 256 k的變量（≈ 85,000 過程對象），這些變量是由 AS 提供和生成的。OS 客戶機的操作員也能夠發出指令並將其輸入到 AS中。

項目數據如圖片和數據歸檔存儲在服務器上，並可由客戶機所使用。

如果一個項目需要幾臺 OS 服務器，那么，服務器根據工廠層級而排列。可將一個或多個層級分配給一個服務器。這一點與 SCADA 系統有著根本上的區別，SCADA 系統中服務器是根據歸檔任務分配的，比如消息（報警）服務器，過程-趨勢（過程變量）服務器和圖片（圖形對象）服務器。下圖顯示了 PCS 7 OS 服務器根據工廠區域的分佈情況。

（5）冗餘服務器

此處，冗餘位於 OS 級。PCS 7 提供其它級的冗餘，比如， AS 級，系統總線級和外圍設備級等。冗餘服務器功能上與服務器相同，與服務器並行運行。在正常工作時，過程數據服務器完全是並行運行的。每臺服務器有著自己的過程連接和數據歸檔。AS 把過程數據和消息發送到雙方冗餘服務器中。如果一個服務器發生故障，客戶機將自動從故障服務器切換到正常運行的服務器上。故障服務器恢復在線後，冗餘功能將履行故障期間的歸檔同步，傳送丟失的數據到故障服務器來填補由於故障而產生的歸檔缺口。該運行均衡了服務器。

（6）OS 客戶機

OS 客戶機基於Windows 客戶機/服務器技術與OS服務器之間有網絡連接。客戶機不能直接訪問系統總線和 AS。它們是通過 OS 服務器進行項目數據的訪問。


系統總線


PCS 7 系統總線是基於 IEEE 標準 802.3，數據傳輸率為 10 或 100 Mbps 的工業以太網。多達1024 個站可以被連接到工業以太網上。

傳輸媒體可以是：

• 三同軸電纜 （屏蔽同軸電纜）

• I TP（Industrial Twisted Pair 工業雙絞線）

• 光纜(FOC)

網絡可以使用光鏈路模塊（OLM）。一個 OLM 的實例如下圖所示，有三個工業雙絞線（ITP）端口和兩個光學端口。使用 ITP端口，可以連接多達三個終端或其它 ITP 段。使用光纜端口，可以建立冗餘環形網絡。在一個環中，最多可連接 11個 OLM。

對於性能要求極高的中型至大型系統，系統總線採用Fast Industrial Ethernet 快速工業以太網（以後簡稱為 Fast Ethernet快速以太網）。快速以太網是標準以太網的進一步發展，其基本特性與一般的以太網標準相似。它們都使用相同的數據格式、CSMA/CD （帶有衝突檢測的載波偵聽多路存取）訪問程序、玻璃纖維光纜以及雙絞電纜（非三同軸電纜）。快速以太網-網絡結構的構造主要基於光學數據的傳輸。工業以太網 OSM（Optical Switch Modules光學交換模塊）使您能夠使用工業雙絞線（傳輸速度為 10/100 Mbps）和以玻璃光纖（傳輸速度為100 Mbps）為快速以太網主幹的交換技術，對工業以太網網絡進行配置。一個工業以太網 OSM 實例具有六個工業雙絞線接口（ITP 端口）和兩個光學接口。由此，您可以把多達六個 ITP 終端或

其它ITP 網絡連接到 ITP 端口上。您可以將工業以太網總線組態為冗餘網絡。所使用的冗餘概念如下：


• 冗餘光學環

• 帶 REDCONNECT（冗餘連接）的傳輸介質冗餘

一個環形結構允許電纜一次失效，如：電纜被切開或斷開。傳輸介質冗餘是指：一個 ES 或 OS有兩根連接到冗餘系統總線的電纜。

從組態到運行

項目組態完成後， 把程序（AS - 專用數據塊：DB、FC、FB、和 OB 等）從ES下載到 AS中去執行。項目的 OS 部分（OS-專用數據：圖片，消息和文件歸檔）被下載到 OS 服務器和客戶機中。然後在服務器和客戶機上激活OS 項目。項目就處於 PCS 7運行系統的控制之下。
