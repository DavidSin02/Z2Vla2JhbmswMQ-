適配器模式（Adapter Pattern）是作為兩個不兼容的接口之間的橋樑，將一個接口轉換成客戶希望的另一個接口，適配器模式使接口不兼容的那些類可以一起工作。這種類型的設計模式屬於結構型模式，適配器模式既可以作為類結構型模式，也可以作為對象結構型模式。例如：讀卡器是作為內存卡和筆記本之間的適配器。您將內存卡插入讀卡器，再將讀卡器插入筆記本，這樣就可以通過筆記本來讀取內存卡。


在軟件系統中，常常要將一些"現有的對象"放到新的環境中，而新環境要求的接口是現有對象不能滿足的，所以適配器模式應用場景：

1、系統需要使用現有的類，而此類的接口不符合系統的需要

2、想要建立一個可以重複使用的類，用於與一些彼此之間沒有太大關聯的一些類，包括一些可能在將來引進的類一起工作，這些源類不一定有一致的接口

3、通過接口轉換，將一個類插入另一個類系中


適配器的種類：

1、類適配器


public class Adapter extends Adaptee implements Target {

public void request() {

super.specificRequest();

}

}


2、對象適配器


public class Adapter extends Target {

private Adaptee adaptee; //維持一個對適配者對象的引用


public Adapter(Adaptee adaptee) {

this.adaptee=adaptee;

}


public void request() {

adaptee.specificRequest(); //轉發調用

}

}


Target：目標抽象類

Adapter：適配器類

Adaptee：適配者類


適配器模式案例：

現需要設計一個可以模擬各種動物行為的機器人，在機器人中定義了一系列方法，如機器人叫喊方法cry()、機器人移動方法move()等。如果希望在不修改已有代碼的基礎上使得機器人能夠像狗一樣叫，像狗一樣跑，使用適配器模式進行系統設計。


package com.test;

public class Test {


public static void main(String[] args) {


Robot robot= new DogAdapter();

robot.cry();

robot.move();

}


}


interface Robot

{

public void cry();

public void move();

}


class Dog

{

public void wang()

{

System.out.println("狗叫！");

}


public void run()

{

System.out.println("狗跑！");

}

}


class DogAdapter extends Dog implements Robot

{

public void cry()

{

System.out.print("機器人模仿：");

super.wang();

}


public void move()

{

System.out.print("機器人模仿：");

super.run();

}

}

適配器模式優點：


1、將目標類和適配者類解耦，通過引入一個適配器類來重用現有的適配者類，無須修改原有結構

2、增加了類的透明性和複用性，提高了適配者的複用性，同一個適配者類可以在多個不同的系統中複用

3、靈活性和擴展性非常好

適配器模式缺點：

1、過多地使用適配器，會讓系統非常混亂

2、由於 JAVA 至多繼承一個類，所以至多隻能適配一個適配者類，而且目標類必須是抽象類。
