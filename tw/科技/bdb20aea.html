C｜實例說明變量存儲類別的時間存續期和空間作用域

2020-11-14
| 科技

變量的值可能在整個程序運行期間一直存在，也可能是臨時分配存儲空間，使用結束，變量就不存在了（再次使用時，再重新分配釋放）。變量值的存在時間是由變量的存儲方式來決定的。

變量的存儲方式分為靜態存儲方式和動態存儲方式。

採用靜態存儲方式的變量，在程序運行期間，變量由系統在靜態存儲區為其分配存儲空間，且存儲空間在程序運行期間是不釋放的，即變量可以一直使用為其分配的存儲空間。

採用動態存儲方式的變量，只有在函數被調用時，系統才為其分配動態存儲空間，函數調用結束則系統釋放該變量的存儲空間，變量的值也就不復存在了。

在C語言中，變量的存儲類別可以通過關鍵字auto、static、extern、register來指定。用register聲明的變量其值存入在寄存器中（一種比內存速度更快的存儲器），使用寄存器變量只為提高數據的存取速度，但由於機器性能越來越高，目前已不常用。

auto

#include <stdio.h>

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

}

system("pause");

return 0;

}

void func()

{

auto int j = 0;

++j;

printf("\t%d",j);

}

//輸出結果：1 1 1 1 1

auto聲明的變量j是自動變量，為局部變量，定義在代碼塊內部（ auto關鍵字可以省略)。函數調用時對其賦值（只有在函數被調用時，系統才為其分配動態存儲空間，函數調用結束時則系統釋放該變量的存儲空間），函數調用一次重新賦值一次。

需要補充說明的是，函數的形式參數也同樣被看作局部變量。另外，主函數main語句塊的變量的時效性與空間性與其他函數處於同等地位。

static

#include <stdio.h>

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

}

system("pause");

return 0;

}

void func()

{

static int j = 0;

++j;

printf("\t%d",j);

}

//輸出結果：1 2 3 4 5

static定義的變量j為靜態變量，因為位於代碼塊內，所以也是局部的，所以屬於局部靜態變量。編譯時賦初期，無論函數調用多少次只賦值一次。雖然是局部變量，因為static的靜態聲明，函數調用結束後與自動變量不同的是，空間不釋放，值不消失，變量值保留到下次函數調用，調用時的值為下次函數調用結束時的值。當然，因為變量定義在塊內，雖然函數調用結束後其值仍存在，但其他函數不能使用其值，即變量的作用域只能是當前函數。

（所謂靜態就是一直存在，動態就是調用時存在，調用完消失。靜態如果是對全局的修飾，其實是對全局的限制，在多文件源程序中，通過靜態限制到定義該變量的文件自身，而不是全部文件。）

extern

#include <stdio.h>

extern j;

int main()

{

int i;

void func();

for(i=1;i<=5;i++)

{

func();

j++;

}

system("pause");

return 0;

}

int j = 0;

void func()

{

++j;

printf("\t%d",j);

}

//輸出結果：1 3 5 7 9

上例聲明和定義的j處於語句塊外，屬於外部變量，也叫全局變量，屬於靜態存儲方式。外部變量只能定義一次，可以在文件的開頭，也可以在任意兩個函數定義之間，但作用域只從定義點開始到文件結束。全局變量若不在文件開頭定義，可在當前文件任意位置用extern對全局變量進行聲明（可以多次聲明），這樣也就擴大了變量的作用域，變量的有效區域從定義處擴充到了extern聲明處。也就是說，extern只是用來說明該變量已經在其他地方定義了。當然，如果全局變量定義在其它文件中，則本文件需要加extern進行聲明。

(extern也是隱含在一般的函數定義前面的，表面函數的全局性。)

全局變量（外部變量）的說明之前再冠以static就構成了靜態的全局變量。未冠static的全局變量和靜態的全局變量二者都是靜態存儲方式，在單文件程序中二者無任何區別。區別在於當一個源程序由多個源文件組成時，非靜態的全局變量的作用域是整個源程序，當一個源程序由多個源文件組成時，非靜態的全局變量在各個源文件中都是有效的，而靜態的全局變量則限制了作用域，即只在定義該變量的源文件內有效，在同一源程序的其他源文件中不能使用它。由於靜態全局變量的作用域侷限於一個源文件內，只能為該源文件內的函數公用，因此可以避免在其他源文件中引起錯誤。從以上分析可以看出，把局部變量增加靜態聲明後是改變了它的存儲生存期。把全局變量增加靜態聲明後是限制了它的使用範圍（作用域）。因此static這個說明符在不同的的地方所起的作用是不同，應該是語言開發者為避免使用更多的關鍵字而增加一些關鍵字在不同的上下文環境中的功能。

如是一個全局變量被局部變量所屏蔽，可以通過一元作用域運算符::（兩個半角冒號）來訪問這個全局變量。
