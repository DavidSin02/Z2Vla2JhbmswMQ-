項目中比較棘手的oracle表中有數據並且修改字段類型

2020-10-29
| 科技

有一個表名為tb_test，字段段名為tb_name，數據類型VARCHAR2(4000)。

1、假設字段數據為空，則不管改為什麼字段類型，可以直接執行：

alter table tb_test modify (tb_name nvarchar2(20));

2、假設字段有數據，則改為nvarchar2(20)可以直接執行：

alter table tb_test modify (tb_name nvarchar2(20));

3、假設字段有數據，則改為varchar2(40)執行時會彈出：“ORA-01439:要更改數據類型,則要修改的列必須為空”，這時要用下面方法來解決這個問題：

/*修改原字段名tb_name為tb_name_tmp*/

alter table tb_test rename column tb_name to tb_name_tmp;

/*增加一個和原字段名同名的字段name*/

alter table tb_test add tb_name varchar2(40);

/*將原字段tb_name_tmp數據更新到增加的字段tb_name*/

update tb_test set tb_name=trim(tb_name_tmp);

/*更新完，刪除原字段tb_name_tmp*/

alter table tb_test drop column tb_name_tmp;

注意點：

1、當字段沒有數據或者要修改的新類型和原類型兼容時，可以直接modify修改。

2、當字段有數據並用要修改的新類型和原類型不兼容時，要間接新建字段來轉移。
