網絡中怎樣通過安全控制列表ACL實現安全控制，實現遠程登錄訪問

2020-10-29
| 科技

一、需求

1.路由器名為:cisco1、cisco2;

2.cisco1的S0端口與cisco2的S0端口相連，IP地址如圖所示,在cisco2上設置特權密碼為cisco，線路密碼為cisco；

3.從cisco1使用PING命令測試到cisco2的連通性，結果可達，但卻不可以 TELNET到cisco2。

二、拓撲

三、步驟

有兩種方法實現

方法一:使用擴展ACL

1.檢測基本配置

cisco2(config)#enable secret cisco

cisco2(config)#line vty 0 4

cisco2(config-line)#password cisco

cisco2(config-line)#login

cisco1#telnet 192.168.12.2

Trying 192.168.12.2 ... Open

User Access Verification

Password:

cisco2>------------驗證成功！

2.創建ACL

cisco2(config)#access-list 101 deny tcp host 192.168.12.1 any eq 23

注：telnet屬於TCP協議。代表絕對匹配一個主機。telnet協議的端口號是23。

cisco2(config)#access-list 101 permit ip any any

cisco2(config-if)#ip access-group 101 in

3.檢驗效果

cisco1#telnet 192.168.12.2

Trying 192.168.12.2 ...

% Destination unreachable; gateway or host down

cisco1#ping 192.168.12.2

Type escape sequence to abort.

Sending5, 100-byteICMP Echos to 192.168.12.2, timeout is 2 seconds:

!!!!!

Success rate is 100 percent (5/5), round-trip min/avg/max = 24/156/364 ms

注：我們只拒絕了TELNET協議，但是ICMP協議我們是放行的。

方法二:使用標準ACL

1. 刪除先前的配置

cisco2(config)#int s0

cisco2(config-if)#no ip access-group 101 in

cisco1#telnet 192.168.12.2

Trying 192.168.12.2 ... Open

User Access Verification

Password:

Password:

cisco2> ---------------------Cisco2上的配置被刪除了

2.創建ACL

cisco2(config)#access-list 1 deny host 192.168.12.1

cisco2(config)#access-list 1 permit any

注：標準的ACL編號範圍是1到99，和1300到1999。標準的ACL只能檢測源地址。

3.應用ACL

cisco2(config)#line vty 0 4

cisco2(config-line)# access-class 1 in

注：在VTY線路中應用ACL。

4.驗證效果

cisco1#telnet 192.168.12.2

Trying 192.168.12.2 ...

% Connection refused by remote host

小結：問設置了2種ACL，但是得到的效果卻不一樣。如果是使用了擴展的ACL，那麼它的提示是“% Destination unreachable; gateway or host down”,說明23號端口根本不可達。如果是使用了標準的ACL放置在VTY 線路中，則提示“% Connection refused by remote host”，說明的確是到達了23號端口，只不過被拒絕了。在實際使用中最好使用擴展的ACL，減少23號端口的負擔！
