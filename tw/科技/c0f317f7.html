python窗口編程-1：窗口編程的基本概念與框架

2020-10-29
| 科技

雖然用著同樣的語言，當我們執行不同的編程任務時，就如同身處不同的世界。我們學習python語言基礎時，所處的是最簡單的控制檯世界。現在，讓我們進入一個稍微複雜一些，但更具表現力的世界：窗口世界。窗口世界獨立於語言之外而存在著，很多語言都可以進行窗口編程，有了前面關於python語言的基本概念，我們已經可以用python語言來理解這個世界了。

讓我們首先了解關於窗口編程的基本概念和框架。

窗口編程的世界中有四個基本概念：窗口、控件、佈局、事件。使用windows（或類似）操作系統的朋友，很容易直觀地理解它們。

窗口：打開一個桌面應用（比如我們使用的瀏覽器），一般都會以一個窗口的形式出現。

控件：所謂控件，就是顯示在窗口上，用來向用戶提供信息或者供用戶控制窗口的組件。窗口上一般可以看到多個控件。還以瀏覽器為例，我們可以看到的控件是：標籤頁、地址輸入欄、若干按鈕和菜單，以及佔窗口面積最大的：網頁內容顯示區。根據我們使用的具體瀏覽器與版本的不同，控件可能不完全一樣。

窗口的控件，看似豐富，其實真正常用的只有十幾種，包括按鈕、輸入框、文字、圖片、複選框、單選框、下拉框、列表、菜單、標籤頁、進度條，等。

佈局：所謂佈局，就是指控件的大小和在窗口上的相對座標。當然最準確的方法是逐一指定窗口上每個控件的大小和座標位置。但在很多應用場景下，我們也用一些更簡單快捷的方法來指定窗口的佈局。

事件：如果說前面三個是窗口的靜態描述，事件就是對窗口的動態描述。用戶對控件的每一個有含義的操作都可以算一個事件。比如點擊按鈕，是一個事件；在下拉列表中進行選擇，又是一個事件；在窗口上雙擊鼠標，當然也是事件。窗口編程的核心就是編寫這些事件響應的代碼。

下面我們通過一個例程，來進入窗口編程的世界。在此之前，必須安裝一個第三方的python庫，在我們的課程學習中，以及你自己的編程實踐中，這將會是一個常見的動作。

在windows下打開命令提示符黑色窗口，粘貼這行命令並運行：

pip install pysimplegui -i https://pypi.doubanio.com/simple/

它將安裝PySimpleGUI庫到python當前環境中，這是一個非常好的GUI庫，重點突出，屏蔽很多無須關注的細節，讓我們把注意力放在編程本身。 -i 後面的部分，是為了加速。

安裝必須聯網，過程有幾秒鐘即完成，如果沒有彩色的提示文字，就說明你安裝正確了。

然後粘貼下面這段代碼到你的編輯器中，先運行一下看看效果：

由於加了很多註釋，顯得代碼有點長。註釋中對每個涉及到的知識點進行了介紹，在正文中我們講解一下重點內容：

引用

先看代碼的最開頭位置。本課程的窗口編程，必須引用PySimpleGUI這個庫。import語法前面的課程簡單介紹了一下，這裡是一個變種版，由於這個庫的名字比較長，我們通過as語句把它簡化為一個簡單的縮寫，方便編程。

import PySimpleGUI as sg

主程序

再看代碼最後的位置。

if __name__ == '__main__':	simpleTest1()

當我們寫一個python程序的時候，考慮到其中的功能有可能被其它python程序使用。比如這段程序就提供了一個比較實用的功能：彈出窗口獲得用戶輸入。其它的程序有可能會用到這個功能，如果你把這個程序保存為win1.py，那麼只須在其它程序的開頭import win1（詳見python編程基礎2），並在使用的位置用類似這樣的語句a=win1.getNumber()就可以了。但我們大概來不希望引入這個程序的同時它就彈出一個窗口讓我們輸入一個數字並對它進行因子分解！，所以，我們這個功能放在if __name__ == '__main__':的條件後面，這個判斷語句的含義是：判斷是正式運行還是被引入，如果是正式運行，則執行後面的語句；否則什麼都不做。

測試函數

simpleTest1其實算是一個測試函數，它就是我們前面課程中講過的通過遍歷獲得整數全部因子的算法（詳見python編程基礎3）。唯一不同的就是變量a的值，不是在代碼中指定，而是通過一個函數，彈出窗口，讓用戶輸入。所以我們重點看getNumber函數。

窗口靜態描述（控件、佈局、窗口）

我們這裡看一下教程前面所講的窗口相關概念在代碼中是怎樣落實的。看getNumber函數定義。

layout = [		[sg.Text('請輸入：'), sg.InputText()],		[sg.Button('  確定  '), sg.Button('  取消  ')],	]

這一段是窗口中的控件和佈局。layout變量被定義成一個列表，這個列表中有兩個元素，而每個元素都是一個列表，每個列表中又有兩個元素。它的含義是：窗口的佈局分成兩行

第一行包括兩個控件：一個文本顯示控件，顯示的內容是“請輸入”；後面跟著一個輸入框控件。

第二行包括兩個按鈕：第一個顯示“確定”，第二個顯示“取消”。

我們從運行的截圖上也可以看到，確實是這樣的。由於我們對控件的具體大小、準確位置並不關心，所以這樣簡單設置一下，就足夠了。

再看窗口

window = sg.Window('輸入獲取窗口', layout=layout)

這句話，設置了窗口的標題，並把前面定義好的控件和佈局，佈置到這個窗口上。我們對窗口的大小也不關心，所以交給系統自己判斷，能把這些控件都顯示出來就好了。

窗口動態描述（事件）

看代碼

	event, values = window.read()	window.close()

在這裡，用window.read()函數打開窗口，並等待一個窗口事件。在這裡，只有按鈕事件。可能發生的事件有三個：關閉窗口（點擊右上角的×）、點擊確定按鈕、點擊取消按鈕。一旦任何一個按下，窗口就關閉了（window.close()）。

究竟發生了什麼事件，保存在event變量中。如果是關閉窗口，event變量為空，在python中用（None）表示。

而在輸入框控件裡輸入了什麼內容，則保存在values變量中。values變量是一個字典，裡面保存窗口所有能夠輸入內容的控件的值。由於我們沒有為輸入框控件指定特別的鍵值，系統就默認為0號。

其它值得注意的知識點

1、如果你仔細觀察就會發現，我們為了讓按鈕上的文字顯示得比較寬鬆，在兩邊留了空格。所以事件名不是"確定"而是" 確定 "，我們這裡使用了字符串處理函數strip()來取消兩邊的空格。

2、用戶輸入的不一定是數字，如果不完全是數字，我們的轉換會出錯，所以我們先用另一個字符串處理函數isdigit()進行了判斷，只有字符串完全由數字組成它才為真。

字符串處理函數常用的大概有二十幾個，我們在課程中會逐步介紹，並在適當的時候進行總結。

3、如果你點擊了取消按鈕，或者直接關閉窗口，或者輸入的不是一個整數，都會導致系統認為你的輸入為空，於是會彈出這樣的提示。

這是一個簡單的格式化的窗口功能，一個無標題的窗口，上面有兩個控件：文字顯示和OK按鈕，分兩行佈局。事件處理很簡單：點擊按鈕就關閉窗口。它的作用就是提示用戶某個信息，通過這樣的代碼來實現：

sg.popup('您沒有輸入數字')

更多細緻的知識點內容，請仔細閱讀代碼中的註釋。
