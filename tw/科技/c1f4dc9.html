內存管理

  • 內核空間：1GB（對32位操作系統來說）

  • 用戶空間：3GB（對32位操作系統來說）

    比如一個程序經過編譯的形成的地址空間是一個虛擬地址的空間，但是程序最終還是要運行在物理內存中的，虛地址轉換成物理地址需要分頁操作。如果給出的頁表不同，那麼虛擬地址由MMU轉換成物理地址就會不同。所以我們對每個進程都建立不同的頁表，在某一時刻一個CPU中只能有一個進程在運行，當進程發生切換的時候，將頁表更換成相應的頁表，這就形成了各個進程都有自己的虛擬空間而相互不影響。所以在任意時刻，對於一個CPU來說只需要有當前的頁表就能實現從虛地址到物理地址的轉換。

    內核空間從3G---4G，但映射到物理內存空間卻是從0x00000000開始的。

虛存區中的頁會與磁盤發生關係

  • 1.磁盤交互區：當內存不夠用時候，一些久未使用的頁就會交換到磁盤的交換區，騰出物理頁給那些急需的進程使用（交換機制）

    2.虛存映射：Linux提供mmap函數，這個系統調用可以將一個打開的文件映射到進程的用戶空間，此後就可以像訪問內存一樣來訪問這個文件。

mmap函數

  • 1.需要映射的文件通過mmap函數映射到用戶空間（虛擬地址），然後虛擬地址通過MMU地址轉換到物理內存地址。

    2.邏輯地址MMU機制轉換成物理地址（邏輯地址通過分段機制轉換成虛擬地址，虛擬地址通過分頁機制轉換成物理地址）

    3.4當進程第一次訪問頁時候，頁沒有在內存中，這就需要請頁機制，當內存不夠用時候，一些久未使用的頁就會交換到磁盤的交換區，騰出物理頁給那些急需的進程使用（交換機制）

mmap

void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);

addr：指定映射的起始地址, 通常設為NULL, 由系統指定

length：映射到內存的文件長度

prot：映射區的保護方式, 可以是:

PROT_EXEC: 映射區可被執行

PROT_READ: 映射區可被讀取

PROT_WRITE: 映射區可被寫入

flags：映射區的特性, 可以是:

MAP_SHARED:寫入映射區的數據會複製迴文件, 且允許其他映射該文件的進程共享。

MAP_PRIVATE:對映射區的寫入操作會產生一個映射區的複製(copy-on-write), 對此區域所做的修改不會寫回原文件

fd：由open返回的文件描述符, 代表要映射的文件

offset：以文件開始處的偏移量, 必須是分頁大小的整數倍, 通常為0, 表示從文件頭開始映射。

  • lseek函數

一個用於改變讀寫一個文件時讀寫指針位置的一個系統調用

off_t lseek(int fd, off_t offset, int whence);

fd：由open返回的文件描述符

offset：偏移量，該值可正可負，負值為向前移

whence：搜索的起始位置，有三個選項：

(1).SEEK_SET: 當前位置為文件的開頭，新位置為偏移量大小

(2).SEEK_CUR: 當前位置為文件指針位置，新位置為當前位置加上偏移量大小

(3).SEEK_END: 當前位置為文件結尾，新位置為偏移量大小whence：

#include <errno.h>

內存分配和回收

  • 夥伴算法工作原理

    Linux把空閒頁面分為10個鏈表，每個鏈表包含2的冪次這樣的頁面，第0個鏈表中包含的都是2^0大小的頁面，依次類推。假設要求分配的塊大小為128個頁面，該算法首先會在2^7這個鏈表裡面查找，如果找到就分配，如果找不到就向2^8（256）鏈表尋找，如果找到就把256平分，一個分配一個128的頁面，另一個128大小的頁面插入到2^7（128）的鏈表；如果2^8（256）鏈表也沒有，還繼續往上尋找2^9（512），如果找到就把512分成128和384,128分配，把384再分成128和256，然後把128和256依次插入到2^7和2^8鏈表中；如果512也沒找到，該算法就放棄分配，併發出出錯信號。（上面的數字代表多少個頁面）

  • 物理頁面的回收

    page_idx：是page結構在mem_map數組中的索引（mem_map是一個全局變量，描述系統中的全部的物理頁面）

    order：指出要回收的頁塊的大小為2的order次冪個物理頁（2^order）

    1.如果page_idx+1不能被2^(order+1)整除，那麼page_idx+2^order就是buddy_idx（其夥伴）

    2.如果page_idx+1能被2^(order+1)整除，那麼page_idx-2^order就是buddy_idx（其夥伴）

  • vmalloc和kmalloc區別

    內核空間分佈（3G-4G）：

    PAGE_OFFSET-------->high_memory--------->VMALLOC_START-------->4G

    CPU訪問的都是虛擬內存，其中PAGE_OFFSET為3G，high_memory為保存物理地址最高值的變量，VMALLOC_START為非連續區的起始地址。

    vmalloc和kmalloc都是在內核代碼中用來分配內存的函數

    kmalloc分配的是從3G----high_memory之間，這一段內核空間和物理內存映射是一一對應的（保證分配的內存在物理上是連續的）

    vmalloc分配的是從VMALLOC_START----4G之間，這一段內核空間和物理內存映射可能是非連續的（保證在虛擬地址空間上的連續）

中斷和異常

  • linux對256個向量分配如下

    1-31 的向量對應異常和非屏蔽中斷

    32-47 的向量（由I/O設備引起的終端）分配給屏蔽中斷

    48-255 的向量用來標識軟中斷，linux用了128即0x80用於系統調用

  • 中斷向量表

    也叫中斷描述符表IDT，其中每個表項叫做一個門描述符，主要的門描述符有三種

    中斷門：類型碼為110，請求特權級為0，用戶態不能訪問

    陷阱門：類型碼為111，不關中斷

    系統門：請求特權級為3，系統調用就是通過系統門進入內核的

  • 中斷和異常的硬件處理

    CPL是當前執行任務的特權等級，存儲在cs和ss的第0位和第1位（0~3）四個等級

    DPL表示門或者段的特權等級，存儲在門（中斷描述符IDT）和段描述符GDT的DPL字段中

    1.確定發生中斷或異常的向量i（0~255之間）

    2.讀取IDTR寄存器找到IDT表，讀取IDT的第i項

    3.分兩步進行有效性檢查，“段”級檢查和“門”級檢查

    4.檢查是否發生特權級的變化，當中斷髮生在用戶態（特權級為3），中斷處理程序運行在內核態（特權級為0），所以會引起堆棧的更換，從用戶態切換到內核態。如果中斷髮生在內核態級不會發生堆棧的切換

  • 中斷上下部處理機制

    下半部運行時是允許中斷請求的，上半部運行時是關中斷的

  • 小任務機制

    是指對要推遲執行的函數進行的函數進行組織的一種機制

  • 工作隊列

    是另外一種將工作推後執行的一種形式，工作隊列可以把工作推後，交由一個內核線程去執行，也就是說這個下半部分可以在進程上下文中執行。什麼時候用小任務機制什麼時候用工作隊列呢？如果推遲的任務需要睡眠那就使用工作隊列，如果不需要睡眠就使用小任務機制
