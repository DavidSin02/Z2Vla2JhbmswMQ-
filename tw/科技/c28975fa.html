JAVA基礎&繼承(上)

繼承： extends

Java繼承(Java inheritance)

　　Java繼承是使用已存在的類的定義作為基礎建立新類的技術，新類的定義可以增加新的數據或新的功能，也可以用父類的功能，但不能選擇性地繼承父類。這種技術使得複用以前的代碼非常容易，能夠大大縮短開發週期，降低開發費用。比如可以先定義一個類叫車，車有以下屬性：車體大小，顏色，方向盤，輪胎，而又由車這個類派生出轎車和卡車兩個類，為轎車添加一個小後備箱，而為卡車添加一個大貨箱。 　　JAVA不支持多重繼承，單繼承使JAVA的繼承關係很簡單，一個類只能有一個父類，易於管理程序，同時一個類可以實現多個接口，從而克服單繼承的缺點。 　　在中運用繼承原則，就是在每個由一般類和特殊類形成的一般—特殊結構中，把一般類的對象實例和所有特殊類的對象實例都共同具有的屬性和操作一次性地在一般類中進行顯式的定義，在特殊類中不再重複地定義一般類中已經定義的東西，但是在語義上，特殊類卻自動地、隱含地擁有它的一般類(以及所有更上層的一般類)中定義的屬性和操作。特殊類的對象擁有其一般類的全部或部分屬性與方法，稱作特殊類對一般類的繼承。 　　繼承所表達的就是一種對象類之間的相交關係，它使得某類對象可以繼承另外一類對象的數據成員和成員方法。若類B繼承類A，則屬於B的對象便具有類A的全部或部分性質(數據屬性)和功能(操作)，我們稱被繼承的類A為基類、父類或超類，而稱繼承類B為A的或子類。 　　繼承避免了對一般類和特殊類之間共同特徵進行的重複描述。同時，通過繼承可以清晰地表達每一項共同特徵所適應的概念範圍——在一般類中定義的屬性和操作適應於這個類本身以及它以下的每一層特殊類的全部對象。運用繼承原則使得比較簡練也比較清晰。

class Person

{

String name;

int age;

}

class Student extends/*繼承*/ Person

{

void study()

{

System.out.println(name+"...study run..."+age);

}

class Worker extends Person

{

void work()

System.out.println("work run");

}

1、提高了代碼的複用性

2、讓類與類之間發生了關係(正是因為這種關係的存在，才讓類出現了多態)

java只支持單繼承。對於多繼承，java保留這種機制，並進行了改良，避免了多繼承調用時的不確定性。

java支持多層繼承

對於使用一個繼承體系：如果java中已經建立一個繼承體系，那麼當我們想用這個體系中的功能，那麼這個時候我們就應該先查閱最頂層類中的共性內容瞭解該體系的基本功能。然後創建該體系的最子類對象。

什麼時候使用繼承？當事物之間存在著所屬關係(is a)時，就可以使用繼承。所屬： xxx是yyy中的一種 xxx extends yyy

當子類繼承了父類之後，那麼子類就擁有了父類的所有功能，但是當父類有10個功能，而子類只具備其中某些功能，那麼這個時候子類是不能繼承父類的，一定要記住，如果讓子類繼承了父類，那麼子類就會自動繼承了父類中不屬於自己的功能的方法。堅決不能這麼做。如果真的要繼承，那麼就把這兩個類中的共性抽取出來，從新定義一個類，讓這兩個類來繼承抽取出來的這個類。

子父類中的成員變化

1、成員變量

當子父類中出現同名的成員變量時，可以使用super關鍵字來區分父類的成員變量。super關鍵字和this關鍵字的用法很相似。this 代表的是本類一個對象(引用)。super 代表的是父類的所屬的那個空間。父類中的私有成員，子類可以繼承過來，只是訪問權限有限制。就相當於子類中存在父類中這個私有成員變量，但這個成員變量前面有標識，子類沒有權限使用，而當通過父類的提供的方法的時候就可以操作父類中私有的成員變量。但是這種情況非常少見。

注意面試的時候會出現子父類中同名的成員變量。注意他們的細節。

2、成員函數：如果成員函數私有，別說子類沒有訪問權限，就父類自己也無法在本類之外的其他類中訪問。

當子父類中出現一模一樣的函數時，就會出現調用時運行子類的方法的情況。這種情況，形象的稱為覆蓋或者重寫或者叫做覆寫。

什麼時候使用覆蓋？

為了對phone類中的來電顯示功能進行改變。

定義了一個子類，保留了父類中功能的聲明，但是重新定義了功能的內容。這就是覆蓋的應用。

當對一個類中的功能的內容進行重新定義的時候，可以定義該類的子類，並保留該類中的功能聲明重新定義功能內容，並建立子類對象調用該功能。

覆蓋的小細節：1）子類的權限必須大於等於父類權限；2）靜態只能覆蓋靜態，或者被靜態覆蓋

3、構造函數：構造函數不可能覆蓋的，函數最起碼不一樣。

子父類中的構造函數的特點：

在運行時，發現父類的構造函數方法先運行了，然後才是子類構造函數。

原因：是子類的所有的構造函數中，都有一句默認的隱式語句super(),會默認調用父類中的空參數的構造函數。

為什麼要有super默認語句呢？因為子類對象在初始化的時候，必須要到父類先去初始化。

為什麼非要到父類中去初始化呢？那是因為，子類繼承了父類，可以獲取到父類中的數據。所以在使用父類數據之前，必須要明確父類是如何對自己的成員進行初始化的，如果父類中的沒有空參數的構造函數，那麼子類的構造函數就必須手動的通過super語句或者this語句指定要訪問的父類中的構造函數或者子類的其他構造函數。如果在構造函數出現了this()那麼就沒有了super()，不管this調用本類那個構造函數，最後那個構造函數也會去父類的構造函數。注意：super語句必須要定義在構造函數的第一行。因為子類繼承了父類，子類要初始化，就必須明確父類是如何初始化的。

對象的初始化：

1、加載對象所屬的字節碼文件。

2、通過new操作符在堆內存中開闢空間。明確首地址值。

3、對該對象空間中的屬性進行默認初始化

4、調用對應本類構造函數進行初始化。

5、本類構造函數中有隱式super，必須到父類中進行初始化

6、當父類中的內容初始化完畢後，在進行屬性的顯示初始化

7、在進行構造代碼塊的初始化

8、在執行本類對應的構造函數中的自定義

當在查閱API文檔的時候：(小技巧)

1、當一個類沒有構造函數說明的時候，說明這個類不能創造對象。

2、此時說明這個類中的所有內容全部靜態的。

3、如果這個類中沒有構造函數，還有非靜態成員，這個類肯定會有靜態方法返回該類對象。

final 關鍵字

繼承的弊端：打破了封裝性。

class Fu

{

void method(){ // code 訪問底層代碼 }

}

class Zi extends Fu

{

void method() { sop("haha"); } 覆寫了父類的method方法 這樣做是很危險的，可能導致程序無法與windows進行交互

}

當我們寫了一個類之後，不想讓別人隨隨便便就覆寫掉，那麼這個時候我們可以將這個類聲明為最終類，這個時候就需要用final來修飾這個類。

final關鍵字的特點：

1、final是一個修飾符，可以修飾類，方法，變量（成員變量，局部變量）。

2、final修飾的類不可以被繼承。

3、final修飾的方法不可以被覆蓋。當在一個類中有部分功能不能讓覆蓋時，就用final修飾。

4、final修飾的局部變量是一個常量，只能被賦值一次。不能再修改。

通常把用final修飾的變量名的所有字母都大寫，目的真假閱讀性。當變量名由多個單詞組成時需要用下劃線隔開 PERSON_AGE。為了增加閱讀性，只要數據是固定的，就將這個數據用一個名稱來表示，並用final來修飾。

5、被final 修飾的成員變量必須手動初始化值。不能默認初始化。

原因是成員變量在堆內存中，他們有默認初始化值，但是我們定義成員變量，一般都不是想使用他們的默認初始化值，所以必須給他賦值。final 修飾的變量是常量值，那麼這個時候可以再加上靜態來修飾，反正這個值是固定不變的，不管哪個對象使用，值都不會變，沒有必要讓每個對象都擁有這個值，這個時候可以把這個值靜態修飾，讓所有對象共享。但是需不需要用public修飾根據具體情況來確定。 在寫單例的時候一定要注意，在寫餓漢式的時候需要用final修飾 private static final Single s = new Single();
