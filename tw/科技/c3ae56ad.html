詳解英特爾以太網800系列網絡適配器關鍵技術

2020-11-14
| 科技

在前幾周的系列介紹中，我們從不同角度瞭解到英特爾最新推出了Ethernet 800系列網絡適配器如何滿足高性能存儲的高速傳輸需求，使用具有ADQ加速技術的英特爾Ethernet 800系列網卡，如何提供更高的性能和更低的延遲，今天我們將繼續詳解Ethernet 800系列網絡適配器的關鍵技術。

關鍵特性

☞QSFP28，單口或雙口

☞PCle3.0 和 4.0，×16

☞以太網端口配置工具(EthernetPort Configuration Tool，EPCT)

☞應用設備隊列(ApplicationDevice Queues，ADQ)。

☞動態設備個性化(Dynamic DevicePersonalization，DDP)。

☞支持RDMA iWARP和RoCEV2

Ethernet 800系列通過創新性地提高應用效率和網絡性能，以優化高性能服務器工作負載，如NFV、存儲、HPC-AI以及混合雲。

提升雲應用性能

為雲工作負載(包括邊緣服務、web服務器、數據庫應用程序、緩存服務器和存儲目標)按需交付高帶寬，並提升應用程序吞吐量。

◆ADQ使用增強流量導向技術提升了應用程序響應時間的可預測性

◆DDP增強了包分類能力，對某些雲工作負載，可提供高達3倍的吞吐量

◆同時支持RDMA iWARP和RoCE v2傳輸協議，為存儲目標提供高速低延遲連接

優化電信類負載

為高帶寬網絡和通信工作負載(包括移動核心（mobile core）、5G RAN和網絡設備)提供包分類和排序優化

◆DDP支持通信領域現有的和新的特定協議，能將某些NFV工作負載的包處理效率提高3倍

◆支持IEEE 1588v2精準時間協議（Precision Time Protocol），在5G RAN部署中能夠實現高精度時間同步

◆增強的數據平面開發工具包(DataPlane Development Kit，DPDK)支持，提高了包處理速度

用EPCT靈活配置多端口

E810-CQDA1和CQDA2網絡適配器支持多種系統配置，以滿足用戶所需的工作負載需求。更多可用端口與速度的組合簡化了驗證和部署。

利用EPCT，英特爾E810(雙端口或單端口)可以編程為許多不同的物理網絡適配器，其最大吞吐量為100Gbps

英特爾Ethernet 800系列網絡適配器關鍵技術:

ADQ：提高橫向擴展的可預測性

隨著現代數據中心的橫向擴展，提供彈性並可預測的應用程序級性能成為關鍵挑戰之一。ADQ技術通過為關鍵負載建立專用隊列，大幅減少了性能抖動，顯著提高了應用程序的可伸縮性和可預測性。


更低的抖動增加了應用響應時間的可預測性，這不僅能給任務分配更多的計算服務器，還能讓更多的用戶訪問系統，並提供更好的最終用戶體驗。即使應用規模不大也能從更高的一致性中獲益，讓其能夠更容易滿足SLA。

ADQ使用一種優化應用程序線程到設備數據路徑的方法來實現針對特定應用程序的數據控制、傳輸和速率限制。這種專用隊列和調整網絡流量的能力不僅提高了應用性能，還降低了延遲並提升吞吐量。

利用動態設備個性化(DDP)提高包處理效率

DDP通過對數據包過濾，並結合增強的DPDK，可支持雲和NFV等場景下的高級數據包轉發和高效的包處理。

英特爾 Ethernet 700系列首次引入了DDP技術，一些協議可以添加到固件中的默認協議集中。而英特爾Ethernet 800系列固件在驅動程序初始化階段就加載了一個增強DDP配置文件，以及許多用於特定工作負載的協議，從而獲得更高的靈活性。當系統中存在多個800系列適配器時，每個適配器上的流水線都可以使用不同的DDP配置文件進行獨立編程。

IEEE1588精準時間協議(PTP)

英特爾Ethernet 800系列支持IEEE 1588 PTP v1和v2兩個版本，提供納秒級時間精度，能夠準確報告每個數據包的接收時間。這種級別的時間精度有助於確保從5G RAN到金融服務、工業自動化和能源監控等領域網絡部署的精確同步。

通過遠程直接內存訪問(RDMA)提高吞吐量和降低延遲

RDMA通過消除TCP/IP堆棧進程、內存拷貝以及應用程序上下文切換等三個主要的網絡資源開銷，為現代高速以太網提供高吞吐量和低延遲性能。英特爾Ethernet 800系列網絡適配器支持所有主流存儲傳輸協議，包括iWARP、RoCE v2和NVMe over TCP等。

RoCE（RDMA over Converged Ethernet）：

RoCE v2用以太網替代InfiniBand物理層和數據鏈路層，運行在UDP/IP之上，並可在IP網絡上路由。

iWARP，基於IETF標準協議:

在常見的TCP/IP協議之上交付RDMA。iWARP RDMA通過標準網絡和傳輸層運行，並與所有以太網網絡基礎設施一起工作。TCP提供流控制和擁塞管理，無需建立無損以太網。iWARP是一個可高度路由和可擴展的RDMA實現。

保護、檢測、恢復

零信任基於這樣一種理念設計，即在授予訪問權之前，您需要對任何試圖連接或獲得准入的內容進行驗證。

英特爾Ethernet 800系列通過保護、檢測和恢復三種安全機制以及硬件信任根（Hardware Root of Trust）為平臺提供彈性保護。通過這種設計，其內建的故障檢測可保護固件和關鍵設備設置，並進行自動設備恢復以確保設備回到其最初的編程狀態。

英特爾Ethernet 800系列適配器基於Ethernet Controller E810，總體特性如下：

主機接口

  • 符合目前0.9版本的PCle 4.0標準
  • 256個non-posted請求的併發

軟件界面

  • 基本模式VF兼容Intel Adaptive Virtual Functions Specification（Intel自適應虛擬功能規範）
  • Tx / Rx隊列

-2048Tx隊列和2048 Rx隊列

-隊列動態分配到functions和VSls

  • 中斷

-2048箇中斷向量，以靈活的方式分配給隊列和其他用例

-多重中斷調節方案

-20,000,000次中斷/秒

  • 控制隊列(Control Queues，又名管理隊列，Admin Queues)

-針對PF-VF和驅動-驅動的郵箱隊列

-針對軟件-固件控制流的管理隊列

-針對軟件訪問E810內部IP的邊帶隊列（Sideband Queues）

  • 256個Tx門鈴(DB)隊列
  • 512個Tx完成隊列
  • 每Tx隊列有一個量子描述符(QD)隊列，量子信息也嵌入到Tx門鈴中
  • 可編程的Rx描述符字段

包處理

  • 增強的數據平面開發工具包(DPDK)
  • 通用功能

-解析、交換、ACL、分類、包修改等階段

-可編程數據包處理流水線

-基於配置文件

-可編程操作

-階段之間傳播優先級

  • 解析器

-可解析高達504B包頭

-基於解析圖

-基於會話的解析

-可編程解析引擎

  • 二進位分類器(VEB Switch)

-768交換端口(VSls)

-可編程的轉發規則

-風暴控制

  • ACLs

-8K可編程TCAM條目

-平鋪能力寬度可達n*40b

  • 分類過濾器

-基於哈希的統計分佈

-基於英特爾 EthernetFD流的分類

-基於流的iWARP和RoCE流識別

-可編程規則

  • 修改

-插入(Tx)，刪除(Rx)，和包VLAN修改

-L3和L4校驗和與CRC

虛擬化

  • 通過VMDQ和SR-IOV實現主機虛擬化
  • 最多256個SR-IOV虛擬功能
  • 隧道包的無狀態卸載(網絡虛擬化支持)
  • 惡意VF保護
  • 虛擬機負載均衡(VMLB)
  • 高級的信息包過濾
  • VLAN支持VLAN標籤插入、剝離和包過濾，最多可支持4096個VLAN標籤
  • 帶有網絡服務標頭(NetworkService Headers，NSH)的VXLAN,GENEVE, NVGRE, MPLS, VXLAN-GPE
  • 英特爾以太網自適應虛擬功能驅動程序（Intel Ethernet Adaptive Virtual Function drivers）

RDMA

  • 支持iWARP and RoCEv2
  • 256K隊列對
  • 發送隊列推送模式

注意:當E810配置為>4端口操作時，不支持RDMA。

QoS

  • WFQ傳輸調度程序有九個可編程層
  • 共享流水線，避免starvation
  • 通過802.1p PCP或差異化服務代碼點(Differentiated Services Code Point，DSCP)值的QoS
  • 包調整

可管理性

  • SMBus操作可達1Mb/s
  • 符合DMTF的NC-SI 1.1接口，100Mb/s
  • MCTP over PCIe與SMBus
  • 通過本地BMC實現企業級管理方案
  • SNMP和RMON統計計數器
  • 看門狗定時器
  • PLDM over MCTP；PLDM監控；PLDM固件升級：PLDM for RDE
  • 固件管理協議支持

電源管理

  • 支持PCI電源管理狀態D0， D3hot, D3cold
  • Do, D3hot和D3cold中的APM WoL支持

時鐘同步

  • 每個Rx包的時間戳
  • 可選擇的Tx包時間戳
  • 支持IEEE PTP1588v1/2
  • 與其他本地平臺同步時間戳

Pre-Boot

  • 與HTTPS啟動兼容的簽名UEFl選項ROM

安全機制

  • 基於硬件的信任根
  • 基於NVM Read與Power On之上的身份驗證
  • 內置固件/關鍵設置故障檢測與自動設備恢復
