密碼學基礎知識概述

研一上學期上了信息安全這門專業課，因為本科的讀的專業是計算機科學與技術，所以對密碼學這方面接觸的其實並不多，不過對這方面還是比較感興趣的，所以在學習完之後根據課本以及自己的理解以博客的方式整理下學習的內容，課本使用的是《Cryptography and Network Security》，作者是Behrouz A. Forouzan<美>。本文主要梳理密碼學的知識體系，大概的內容框架，讓大家對密碼學有個簡單的瞭解，不涉及具體的算法等。

一、導論

隨著計算機的發展，信息已經成為了一種資產，和其他人任何一種資產一樣具有價值。因此，如何保護信息避免攻擊也是當今人們一直在研究的問題。

1.1 安全目標

  • 機密性(Confidentiality)
  • 對信息的訪問和公開進行授權限制，包括保護個人隱私和私密信息。機密性缺失的定義是信息的非授權洩露。
  • 完整性(Integrity)
  • 防止對信息的不恰當修改或破壞，包括確保信息的不可否認性和真實性。完整性缺失的定義是對信息的非授權修改和毀壞。
  • 可用性(Availability)
  • 確保對信息的及時和可靠的訪問與使用。可用性的缺失是對信息和信息系統訪問與使用的中斷。


1.2 攻擊

  • 偵聽(Snooping)：非授權實體訪問或攔截信息。
  • 流量分析(Traffic Analysis)：通過監控在線流量來獲取其他信息。
  • 篡改(Modification)：在攔截到消息之後攻擊者對信息進行修改，把消息改成對自己有利。
  • 偽裝(Masquerading)：假裝成別的實體。
  • 重放(Replaying)：將獲得的消息再次發送。
  • 否認(Repudiation)：分為兩種，發送發發送消息之後否認曾發送過消息或者接收方接受消息之後否認接受過消息。
  • 拒絕服務攻擊(Denial of Service)：減慢或者完全中斷系統的服務。

1.2.1 攻擊類型

根據危害上述三種安全目標可以將攻擊分為三類：危害機密性的攻擊、危害完整性的攻擊、危害可用性的攻擊。

1.2.2 被動攻擊與主動攻擊

攻擊類型還可以分為主動攻擊與被動攻擊，其中危害機密性的攻擊為被動攻擊，危害完整性以及可用性的攻擊為主動攻擊。被動攻擊僅是獲取信息但並不修改信息，因此被動攻擊可以防止但是很難檢測，而主動攻擊對數據進行修改或者偽造數據，與被動攻擊相反，主動攻擊難以絕對預防但易於檢測。

1.3 密碼分析攻擊

在介紹相關加密知識之前，先簡單介紹下密碼學中用到的一些術語。

  • 明文：原始消息
  • 密文：加密後的消息
  • 加密：從明文到密文的變換過程
  • 解密：從密文到明文的變換過程
  • 密鑰：相用來完成加解密等過程的祕密信息

密碼分析攻擊可分為唯密文攻擊、已知明文攻擊、選擇明文攻擊、選擇密文攻擊。

  • 唯密文攻擊：攻擊者知道加解密算法並且攔截到了密文，嘗試對密文進行還原。比如蠻力攻擊、統計攻擊、模式攻擊等方法。
  • 已知明文攻擊：攻擊者知道加解密算法，並且事先已經掌握了一些明文-密文對，通過這些信息嘗試破解攔截到的密文。
  • 選擇明文攻擊：這種方法和已知明文攻擊很類似，區別是攻擊者掌握的明文-密文對由攻擊者選擇。比如攻擊者得到了發送方用來加密的電腦，他可以輸入一些自己想輸入的明文來獲取相應的密文。
  • 選擇密文攻擊：和選擇明文攻擊很類似，區別是攻擊者通過輸入密文來獲取相應的明文。如果攻擊者得到了接收方用來解密的電腦可以實現。
  • Kerckhoff原理：我們在設計密碼的時候應該假設攻擊者知道加解密算法，僅依靠密鑰來抵抗攻擊，也就是說我們的密鑰應該足夠難以攻破，沒必要隱藏加解密算法。

二、對稱密鑰加密

對稱加密，在加解密過程中使用的密鑰相同，是20世紀70年代公鑰密碼產生之前唯一的加密類型，即便在今天，它的使用也十分廣泛。對稱密鑰加密的最大優勢是加/解密速度很快，適合於對大數據量進行加密，但是他的缺點是密鑰的管理與分配，也就是說，如何把安全可靠的把密鑰發送到正確的人的手裡而不被攔截是個問題。

2.1 代換密碼

傳統的對稱密鑰密碼可根據加密方式分為兩類：代換密碼、換位密碼。在代換密碼中，我們在加密的過程中將一個需要加密的符號用其他符號來替換，而在換位密碼加密過程中，我們則是將加密信息進行重新排序。在實際加密過程中，我們通常是採用兩種方法結合使用以提高密碼的安全性。

代換密碼又可分為單碼代換和多碼代換，在單碼代換中，同一個符號總是被另外相同的符號代換，而在多碼代換中，不同位置的同一個符號可以被不同符號代換。

傳統的單碼代換密碼有加法密碼、乘法密碼、仿射密碼等。下面以仿射密碼進行舉例：

加密過程：C = (P * k1 + k2) mod 26

解密過程：P = ((C -k2) * k1-1) mod 26

其中各個符號意義如下（之後的表示也均按照如下）：

C：密文

P：明文

k：密鑰（其中k1表示第一個密鑰，k2表示第二個密鑰）

k-1：k的乘法逆元，k在模p下的乘法逆元表示為 k * k-1 ≡ 1（mod p），並且k是大於0小於p的整數

仿射密碼相當於加法密碼和乘法密碼的結合，在加密過程中一共用到一個密鑰，一個用於加法加密，一個用於乘法加密。下面是仿射密碼一個具體的例子：

用仿射密碼對消息“hello”進行加解密，密鑰對（k1，k1）為（7，2）：

首先按照字母表的順序對字母進行賦值，從00到25，即a對應的值為00，z對應的值為25。

加密過程：

P:h➡07 加密：(07*7+2)mod26=25 C:25➡Z

P:e➡04 加密：(04*7+2)mod26=04 C:04➡E

P:l ➡11 加密：(11*7+2)mod26=01 C:01➡B

P:l ➡11 加密：(11*7+2)mod26=01 C:01➡B

P:o➡14 加密：(14*7+2)mod26=22 C:22➡W

解密過程：

根據乘法逆元的定義，7在模26下的乘法逆元為15，因為 7 * 15≡ 1（mod26）

C:Z➡25 解密：((25-2) * 15)mod26=07 P:07➡h

C:E➡04 解密：((04-2) * 15)mod26=04 C:04➡e

C:B➡01 解密：((01-2) * 15)mod26=11 C:11➡l

C:B ➡01 解密：((01-2) * 15)mod26=11 C:11➡l

C:W➡22 解密：((22-2) * 15)mod26=14 C:14➡o

多碼代換在加密過程中不同位置的同一符號加密得到的結果不一定相同，這是因為和單碼代換加密過程中密鑰不改變不一樣，在多碼代換加密過程中，往往密鑰是在不斷改變的，比如提前設定好一個密鑰流或者前一位將前一位的明文作為後一位加密時使用的密鑰。具體有Autokey Cipher、Playfair Cipher、Vigenere Cipher等加密方案。下面以Autokey Cipher為例：

P=P1P2P3… C=C1C2C3… k=(k1,P1,P2…)

加密：Ci = (Pi + ki) mod 26 解密：Pi = (Ci - ki) mod 26

2.2 換位密碼

換位密碼得到的密文就是被打亂順序的明文，比如可以將明文按照固定的寬度水平的寫在紙上，然後按垂直方向讀出得到密文，解密時則將密文按相同寬度垂直的寫在紙上，按水平方向讀出。比如我們將"meet me at the park"這句話以水平寬度為4寫出按垂直方向讀出得到的密文則為"mmtaeehreaekttp".

2.3 流密碼與分組密碼

還可以根據每次加密信息的大小將密碼分為流密碼與分組密碼。流密碼又稱序列密碼，明文稱為明文流，密鑰稱為密鑰流，在加密過程中，每次對明文流中單個比特位進行加密變換。分組密碼又稱塊密碼，加密之前先將明文分成大小相同的n個組，每組大小為m ( m > 1)，若有不足位則進行補全，在加密過程中逐次依組進行加密。前面介紹到的一些算法都是以前所使用的一些基礎簡單的算法，在現在的環境下很容易就被破解，十分不安全。目前主流的對稱加密方案主要時DES和AES，兩者都屬於分組密碼。

三、非對稱密鑰加密

非對稱密鑰加密，又稱公鑰加密，這個概念最早在1976年被人提出。和對稱加密不一樣的是公鑰加密有兩個密鑰，公鑰和私鑰，公鑰即為公開的密鑰，私鑰則不公開。當人們需要加密發送信息時，發送方使用接收方的公鑰對數據進行加密，接收方收到加密信息之後用自己的私鑰進行解密即可得到信息。與對稱加密相比，由於公鑰加密加解密過程中使用的密鑰不一樣，所以相對而言安全性更高，但算法更加複雜因此加解密速度更慢，不適用與對大量數據進行加密，因此，在實際應用中，我們需要對大量數據進行加密時，往往採用對稱加密的方法，再用公鑰加密對對稱加密的密鑰進行加密。接收方收到信息之後首先用自己的私鑰進行解密得到對稱加密的密鑰，然後再對消息進行解密。這樣既保證了加解密的速度同時又提高了安全性，一舉兩得。

RSA算法是目前使用最為廣泛的公鑰加密算法，由Ron Rivest、Adi Shamir以及Leonard Adleman三人在1977年提出，下面簡單介紹該算法及其證明。

密鑰生成過程：

  • 選取兩個不相等的大素數p和q
  • 計算n = p * q
  • 計算φ(n) = (p-1) * (q-1)
  • 選取 e 使得 1 < e < φ(n)，且e和φ(n) 互素
  • 計算e模φ(n) 的逆d，即求d使得 ed ≡ 1 (mod φ(n))，也就是ed = k*φ(n) + 1,其中k為整數
  • (e，n)公鑰，d為私鑰

加密： C = Pe mod n

解密 ：P = Cd mod n

證明：

證明過程使用到了歐拉定理：如果n = pq,a<n,並且k為整數，則有 akφ(n)+ 1 ≡ a (mod n)

假設接收方用私鑰解密出的明文為P1，證明P1 = P

P1 = Cd mod n = (Pe mod n)d mod n = ped mod n

ed = kφ(n) + 1，所以 P1 = ped mod n = P kφ(n)+ 1 mod n = P mod n = P

四、哈希函數與數字簽名

4.1 哈希函數

哈希函數H可將任意長度的信息M作為輸入，同時產生固定長度的哈希值h=H(M)。常見的哈希函數有MD5、SHA1等。當哈希函數應用於密碼學中，它需要滿足以下三個標準：

  • 原像穩固(單向性)：通過給定的哈希值很難找到原文。即對於給定的哈希值 h，很難找到任何M使得h=H(M)。
  • 第二原像穩固：給定明文和哈希值，很難找到與給定明文不同但哈希值相同的另一中明文。即給定M以及h=H(M)，很難找到M‘使得M≠M’但H(M)=H(M’)。
  • 抗碰撞：在不給定任何信息條件下，找到兩種明文不同但哈希值相同的明文。即找到M和M’使得M≠M’但H(M)=H(M’)。

4.2 數字簽名

數字簽名，又稱公鑰數字簽名，是公鑰密碼的一種應用，該過程使用的密鑰分為公鑰和私鑰，和公鑰加密不一樣的是，公鑰加密過程中使用的是接收方的公鑰進行加密，接收方的私鑰進行解密。而在數字簽名過程中則是使用發送方的私鑰進行加密得到數字簽名，之後再使用發送方的公鑰進行解密得到哈希摘要。數字簽名是保證消息來源和完整性的重要方法。

數字簽名過程如下圖所示：

  1. A首先用B的公鑰對明文加密得到密文，再用哈希函數對明文進行哈希得到哈希摘要，之後使用A的私鑰對摘要進行加密得到數字簽名。
  2. A將數字簽名和密文一起發給B。
  3. B收到密文和簽名後，首先使用B的私鑰對密文解密得到明文，然後用哈希函數哈希明文得到摘要。之後用A的公鑰對數字簽名解密得到消息摘要 ，對得到的兩個摘要進行對比，若一致則表明消息沒有被篡改，否則拒絕接受消息。

五、密鑰管理

通過前面的介紹我們便能明白密鑰對信息安全起著多麼重要的作用，因此，如何高效安全的對密鑰進行管理一直是人們研究的重點。

5.1 對稱密鑰分配

由於公鑰加密效率不高，因此在對大量數據進行加密時我們往往都是使用對稱加密，然而，對稱加密要求雙方都享有密鑰，也就是說，對於N個人的團體，如果他們使用公鑰加密進行通信的話，則一共需要的密鑰數量為N(N-1)/2。而且雙方交換彼此的密鑰風險很大，因此我們需要一個可信任的第三方幫助我們對密鑰進行管理，也就是我們的密鑰分配中心(Key-Distribution Center)。

5.1.1 密鑰分配中心(KDC)

在這種模式下，每個用戶和KDC都有進行交流的密鑰，比如Alice和KDC的共享密鑰為KAlice，Bob的則為KBob。當Alice想要和Bob進行交流時，過程如下：

  1. Alice首先發送想要和Bob的交流請求給KDC。
  2. KDC收到請求之後告知Bob。
  3. 如果Bob同意則KDC為兩者創建一個會話密鑰。

5.1.2 Kerberos

Kerberos是目前比較流行的一種網絡認證協議，同時也是一個KDC。它可分為以下及部分：

  • Authentication Server(AS)：用戶身份認證、管理用戶密碼對、提供用戶和TGS的會話密鑰和TGS票據。
  • Ticket-Granting Server(TGS)：提供票據許可服務，提供用戶和Real Server的會話密鑰以及Real Server票據。
  • Real Server：為用戶提供服務。

5.2 公鑰分配

若用公鑰加密時，我們需要將自己的公鑰發佈出去，讓其他人都知道，但這很容易被別人偽造我們的公鑰以及被中間人攻擊。因此我們同樣需要一個可信的第三方。
