一.AOP概述

AOP(Aspect Oriented Programming)意為:面向切面編程，通過預編譯方式和運行期間動態代理實現程序功能的統一維護的一種技術。AOP是OOP(面向對象編程)的延續，是軟件開發中的一個熱點，也是Spring框架中的一個重要內容，是函數式編程的一種衍生範型。利用AOP可以對業務邏輯的各個部分進行隔離，從而使得業務邏輯各部分之間的耦合度降低，提高程序的可重用性，同時提高了開發的效率。

AOP底層是採用動態代理模式實現的，主要是JDK的動態代理與CGLIB動態代理。

二.AOP舉例描述

比如，我們在開發中要打印一個方法執行的日誌：某某方法開始執行，某某方法執行結束。

一般我們會在每個方法中都加入，這樣方法一多，日誌就多，如果說某天，這樣的日誌不要了，處理起來將會是很糟糕的。

此時我們就可以使用AOP面向切面編程來實現。

首先溫習一下動態代理的實現

  • (1)DAO接口及其實現
/** * 目標接口 * @author Administrator */public interface TargetDAO {  /**目標方法*/  String print();}public class TargetDAOImpl implements TargetDAO{  @Override  public String print() {     return "TargetDAOImpl";  }}
  • (2)創建調用處理器
//動態代理處理器public class OwnInvocationHandler implements InvocationHandler{   private TargetDAO targetDao;//目標類   //無參構造器   public OwnInvocationHandler() {     super();   }   //有參構造器   public OwnInvocationHandler(TargetDAO targetDao) {     super();     this.targetDao = targetDao;   }   /**    * @param proxy 動態生成的代理對象    * @param method 目標方法    * @param args 方法參數    * @return 目標方法返回值    */   @Override   public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {     System.out.println("某某方法開始執行");     //1.調用反射類Method的invoke,就是調用方法     Object returnObj = method.invoke(targetDao, args);     if(returnObj!= null){       return (String)returnObj+"--invoke";     }     System.out.println("某某方法執行完畢");     return returnObj;  }}
  • (3)測試動態代理
/** * 測試 * @author Administrator */public class Test {  public static void main(String[] args) {   //創建目標方法   TargetDAO target = new TargetDAOImpl();   //創建處理器   OwnInvocationHandler handler = new OwnInvocationHandler(target);   //生成動態代理類   TargetDAO TargetDAOProxy = (TargetDAO)Proxy.newProxyInstance(target.getClass().getClassLoader(),           target.getClass().getInterfaces(), handler);   //執行方法   String printStr = TargetDAOProxy.print();   System.out.println(printStr); }}

三、AOP編程術語

  • 1.Aspect（切面）：泛指交叉業務邏輯，比如上面舉例中的日誌打印，再比如事務處理、登錄驗證等，都可以理解為切面。常用切面通知顧問 實際上，切面的存在是對業務邏輯處理的一種增強。
  • 2.Target（目標對象）：目標對象就是要被AOP作用的對象，也就是要被增強處理的對象。
  • 3.Weaving（織入）：織入是一個過程，就是將切面插入到目標對象的一個過程。
  • 4.Joint point（連接點）：連接點指可以被切面織入的方法，也泛指業務接口中的方法，也就是所有的方法都可以看做是連接點。
  • 5.Pointcut（切入點）：指切面具體織入的方法。比如一個A類中有a(),b()兩個方法，我們要對a()方法使用切面，那麼a()方法就是切入點(也是連接點)。b()僅為連接點。
  • 6.Advice（通知）：切面的一種簡單的具體實現。其中定義了切面織入到目標切入點的時間，也就是在目標方法之前還是之後執行。可以認為切入點定義織入的位置；通知定義織入的時間。
  • 7.Advisor(顧問)切面的一種複雜的具體實現，能夠將Advice（通知）包裝為更復雜的切面。
