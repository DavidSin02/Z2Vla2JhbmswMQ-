什麼是HTML，看完這篇文章就懂了

2020-11-14
| 科技

本文創建於2020年8月，以下為正文：

在這互聯網發達的21世紀，HTML作為網絡的通用語言，一種簡單、通用的全置標記語言，你竟然還沒有掌握？

HTML是什麼

1.超文本標記語言（Hyper Text Mark-up Language）

2.用來設計網頁的標記語言

3.用該語言編寫的文件，以.html或.htm為後綴

4.由瀏覽器解釋執行

5.不區分大小寫，建議小寫

HTML標籤標籤

1.HTML用於描述功能的符號成為“標籤”

2.標籤都封裝在一對尖括號“<...>”之中，如<html>就是一個標籤

3.封閉類型標記（也叫標記），必須成對出現，如<p></p>

非封閉類型標記，也叫空標記，或者但標記，如

web瀏覽器的作用是讀取HTML文檔，並以網頁的形式顯示出它們。瀏覽器不會顯示HTML標籤，而是使用標籤來解釋網頁的內容。

HTML元素

指的是從開始標籤（start tag）到結束標籤（end tag）的所有代碼

HTML 元素語法

HTML 元素以開始標籤起始 以結束標籤終止

元素的內容是開始標籤與結束標籤之間的內容

某些HTML元素具有空內容（empty content）

空元素在開始標籤中進行關閉（一開始標籤的結束而結束）

大多數HTML元素可擁有屬性

HTML屬性與值

1.屬性是用來修飾元素的

屬性必須位於開始標籤裡

一個元素的屬性可能不止一個，多個屬性之間用空格隔開

2.每個屬性都有值

屬性和屬性的值之間用等號鏈接

屬性的值包含在引號當中

屬性總是以名稱/值對的形式出現

< p align="center">段落一</p>

其中align是屬性 ceenter是屬性值

註釋

1.為代碼添加適當的註釋是一種良好的編輯習慣

2.註釋只在編輯文本情況下可見，在瀏覽器展示頁面是並不會顯示

3.註釋標籤不支持任何屬性

4.語法：<!--註釋的文本內容-->

<1."<!--"和：-->"之間的任何內容都不會顯示在瀏覽器種

<2.註釋不可以嵌套在其他註釋中

<3.註釋不可以位於嵌在<>中

html的基本結構

< !DOCTYPE html>

以下文檔類型聲明

<html lang="en">

     <head>        <meta charset="UTF-8">        <title>網頁標題</title>  文檔標題，一個網頁只能有一個標題    </head>

<body> ​ <!--網頁內容，可以使文本、圖像等--> ​ </body> </html>

什麼是DTD文檔模型，為什麼寫在HTML當中？

DTD文檔模型也稱DOCTPE文檔說明，它是Document Type Definition 的英文縮寫，意思是文檔類型定義，在HTML文檔中，用來指定頁面所使用的HTML（或者XHTML）的版本。要想製作符合標準的頁面，一個必不可少的關鍵組成部分就是DOCTYPE聲明。只有確定了一個正確的DOCTYPE，HTML裡的標識和CSS才能正常生效。它一般被定義喜愛頁面的第一行，html標籤之前。

xhtml和html 的區別

XHTML是HTML向xml的一個過渡語言，在最初W3C組織希望把HTML變成更為嚴謹的標記語言（比如XML），但HTML的已經應用的太為廣泛，全部換掉不現實。因此產生了XHTML這樣一種過渡形式。它比HTML更嚴謹，基本標籤都還是沿用了HTML，但廢除了“變現層”標籤，同時要求標籤的嚴格嵌套，標籤結束等等。

隨著互聯網的發展，以及對HTML5人才需求的增加，HTML5相關工作的信息也是水漲船高，前景可謂是一片光明。最近幾年，HTML5不斷的發展，各大瀏覽器也在不斷兼容HTML5技術，如今的微信小程序，手機移動端的使用都離不開HTML5，對於這種現象，HTML5應用的範圍已經拓展到各個行業中，所以每個行業對HTML5的人才需求都是很大的。
