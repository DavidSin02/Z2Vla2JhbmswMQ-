Windows系統安全管理

2020-11-14
| 科技

1. Windows系統安全體系結構

Windows系統採用的是層次性的安全架構，整個安全架構的核心是安全策略，完善的安全策略決定了系統的安全性。Windows系統的安全策略明確了系統各個安全組件如何協調工作，Windows系統安全開始於用戶認證，是其它安全機制能夠有效實施的基礎，處於安全框架的最外層。常見的認證機制，登錄口令和令牌。

加密和訪問控制處於用戶認證之後，是保證系統安全的主要手段，加密保證了系統與用戶之間的通信及數據存儲的機密性；訪問控制則維護了用戶訪問的授權原則。審計和管理處於系統的內核層，負責系統的安全配置和事故處理，審計可以發現系統是否曾經遭受過攻擊或者正在遭受攻擊，並進行追查；管理則是為用戶有效控制系統提供功能接口。

Windows系統的安全性主要圍繞安全主體展開，保護其安全性。安全主體主要包括用戶、組、計算機以及域等。用戶是Windows系統中操作計算機資源的主體，每個用戶必須先行加入Windows系統，並被指定唯一的賬戶，組是用戶賬戶集合的一種容器，同時組也被賦予了一定的訪問權限，放到一個組中的所有賬戶都會繼承這些權限；計算機是指一臺獨立計算機的全部主體和客體資源的集合，也是Windows系統管理的獨立單元；域是使用域控制器(DC, Domain Controller)進行集中管理的網絡，域控制器是共享的域信息的安全存儲倉庫，同時也作為域用戶認證的中央控制機構。



Windows系統的安全性主要是由它的安全子系統來提供，安全子系統即可以用於工作站，也可以用於服務器，區別只在於服務器版的用戶賬戶數據庫可以用於整個域，而工作站版的數據庫只能本地使用。

如圖6-6所示，Windows系統的安全性服務運行在兩種模式下，安全參考監視器（Security Reference Monitor，SRM）運行在內核模式下，作為Windows Executive的一部分；而用於與用戶進行交互的主要安全服務即本地安全機構主要包括身份認證、訪問控制和事件審計等。



安全引用監視器（SRM）是Windows系統所有安全服務的基礎，運行在內核模式下，負責檢查一個用戶是否有權限訪問一個客體對象或者是否有權利完成某些動作，對每個客體對象的訪問必須得到內核層SRM的有效性訪問授權，否則訪問無法完成。SRM的另一個功能是與LSA配合來監視用戶對客體對象的訪問，並生成事件日誌傳送給事件記錄器保存，為管理員的事件審計提供依據。

LSA安全服務主要由本地安全機構子系統（Local Security Authority Subsystem，LSASS）登錄模塊WinLogon兩個服務來完成。WinLogon是系統啟動時自動加載的一個進程，監視整個登錄過程，同時可以加載Gina（Graphical Identification and Authentication）進程，提供圖形化的認證界面。LSASS主要包括LSA服務、安全賬戶管理（Security Account Management，SAM）、網絡登錄服務NetLogon等基本組件。

LSA服務是用戶與系統的交流通道，它提供了許多服務程序幫助用戶完成許多工作，主要包括提供交互式登錄認證服務、創建用戶的訪問令牌、儲存和映射用戶權限、設置和管理審核策略等。LSA與用戶交涉涉及許多服務進程，與WinLogon合作完成用戶登錄；調用Msv1_0.dll支持NT LanMan認證服務，調用Kerberos.dll支持Kerberos認證的服務，可見LSA可以微系統提供豐富的認證機制。

SAM是實現用戶身份認證的主要依據，在SAM數據庫保存著用戶賬號和口令等數據，為LSA提供數據查詢。NetLogon是進行域登錄的重要部件，首先通過安全通道與域中控制器建立連接，然後再通過安全的通道傳遞用戶的口令，完成域登錄。

圖6-7是Windows登錄認證流程，SSPI（Security Support Provider Interface）是微軟公司提供的公用API接口，第三方利用該接口獲得不同的安全性服務而不必修改協議本身。



2. Windows系統的訪問控制

Windows系統的訪問控制是其安全性的基礎構建之一。訪問控制模塊有兩個主要的組成部分分筆試訪問令牌（Access Token）和安全描述符（Security Descriptor），它們分別被訪問者和被訪問者持有。

通過訪問令牌和安全描述符的內容，Windows可以確定持有令牌的訪問者能否訪問持有安全描述符的對象。Windows中的每個賬戶或賬戶組都有一個安全標示符（Security Identity，SID），系統的Administrator、Users等賬戶或者賬戶組在Windows內部均使用SID來標識，每個SID在同一個系統中都是唯一的。

訪問令牌是一個被保護的對象，每一個訪問令牌都與特定的Windows賬戶相關聯，訪問令牌包含該賬戶的SID、所屬組的SID以及賬戶的特權信息。

當一個賬戶登錄的時候，LSA會從內部數據庫裡讀取該賬戶的信息，然後使用這些信息生成一個訪問令牌。當用戶試圖訪問系統資源時，需要將擁有的令牌提供給SRM，SRM會檢查用戶試圖訪問的對象的訪問控制列表。

每個被訪問的客體對象都與一個安全描述符相關聯，安全描述符用來描述客體對象的屬性及安全規則，包含客體對象所有者的SID和ACL。ACL包含自主訪問控制列表（Discretionary Access Control List，DACL）和系統訪問控制列表（System Access Control List，SACL）。

DACL由多個訪問控制項（Access Control Entry，ACE）組成，每個訪問控制項的內容描述了允許或拒絕特定賬戶對這個歌對象執行特定操作。

SACL是為系統審計服務的，它的內容決定了當特定賬戶對該客體對象執行特定操作時，其行為是否會被記錄到系統日誌中。

圖6-8所示，Smith的進程Thread A訪問客體對象FILE.txt，則SRM依據Smith的訪問令牌的信息和FILE.txt的安全描述符進行審核，由於安全描述符中DACL包含有訪問控制項ACE1，其內容是拒絕Smith的讀操作Read、寫操作Write和執行操作Execute，因此SRM拒絕Thread A；而SRM根據DACL的內容允許Thread B訪問FILE.txt。



3. Windows活動目錄與組策略

Windows網絡管理中有兩個非常重要的技術，活動目錄（Active Directory，AD）和組策略（Group Policy，GP）,它們協調工作有效提升了Windows網絡的安全性。AD存儲有關網絡對象的信息，讓管理員和用戶輕鬆查找和使用這些信息。

AD是一個面向網絡對象管理的綜合目錄服務，網絡對象包括用戶、用戶組、計算機、打印機、應用服務器、域、組織單元和安全策略服務。AD是各種網絡對象的索引集合和數據存儲的視圖，把分散的網絡對象建立索引目錄，存儲在活動目錄的數據庫內。

圖6-9為AD的管理劃分模型，AD把整個域作為一個完整目錄進行管理，域模式要求用戶進行網絡登錄，用戶只要在域中有一個賬戶，登錄成功可以在整個域網絡中漫遊。

同時，AD把域劃分為若干個組織單元（Organizational Unit，OU），OU是域中用戶和組、文件、打印機等資源對象以及其它OU集合。可見，OU可以劃分下級OU，下級OU能夠繼承父OU的訪問權限。每一個OU有自己的管理員，負責OU的權限管理，從而實現AD的多層次管理。

AD的功能包括基於目錄的用戶和資源管理、基於目錄的網絡服務和基於網絡的應用管理。基於目錄的用戶和資源管理為用戶提供網絡對象的統一視圖，基於目錄的網絡服務主要包括DNS、WINS、DHCP、證書服務等，基於網絡的應用管理包括管理企業通訊錄、用戶組管理、用戶身份認證、用戶授權管理和應用系統支撐等。



AD是Windows網絡中重要的安全管理平臺，組策略（Group Policy，GP）是其安全性的重要體現。GP是依據特定的用戶或計算機的安全需求定製的安全配置規則。圖6-10所示，管理員針對每個組織單元OU定製不同的GP，並將這些GP存儲在AD的相關數據庫內，可以強制推送到客戶端實施GP。AD可以使用GP命令來通知和改變已經登錄的用戶的GP，並執行相關安全配置。

註冊表是Windows系統中保存系統應用軟件配置的數據庫，很多配置都是可以自定義設置的，但這些配置發佈在註冊表的各個角落，如果是手工配置，可想是多麼困難和繁瑣。GP可以將系統中重要的配置功能彙集成一個配置集合，管理人員通過配置並實施GP，達到直接管理計算機的目的。簡單點說，實施GP就是修改註冊表中的相關配置。GP分為基於活動目錄的和基於本地計算機的兩種：AD GP存儲在域控制器上AD的數據庫中，它的定製實施由域管理員來執行；本地GP存放在本地計算機內，由本地管理員來定製實施。

AD GP實施的對象是整個組織單元OU；本地GP只負責本地計算機。GP和AD配合GP部署在OU、站點或域的範圍內，也可以部署在本地計算機上。部署在本地計算機時，GP不能發揮其全部功能，只有和AD配合，GP才可以發揮出全部潛力。


4. Windows系統安全管理

Windows安全服務依託安全管理功能實現，包括賬戶安全、文件安全和主機安全。

Windows 系統賬戶安全

Windows Server 2003操作系統的合法賬戶，才能訪問網絡上的資源，基於Internet的非法入侵是從尋找賬戶的漏洞開始的。Windows依靠賬戶來管理用戶，控制用戶對資源的訪問，每一個需要訪問網絡的用戶都要有一個賬戶。Windows用戶分為域用戶賬戶、本地用戶賬戶和內置用戶賬戶。

①域用戶賬戶。

域用戶賬戶是用戶訪問域的唯一憑證，該賬戶在域控制器上建立，作為活動目錄的一個對象保存在域的數據庫中。用戶在域中的任何一臺計算機登錄到域中的時候必須提供一個合法的域用戶賬戶，該賬戶將被域控制器所驗證。

SAM作為保存域用戶賬戶的數據庫，位於域控制器的\%systemroot% \NTDS\NTDS.DIT文件中。每個賬戶都被Windows簽訂一個唯一的SID，保證賬戶在域中的唯一性，SID作為賬戶的屬性不能被修改，賬戶被刪除SID將不復存在。Windows系統中SID對應用戶權限，因此只要SID不同，新建賬戶不會繼承原有賬戶的權限與組的隸屬關係。

②本地用戶賬戶。

Windows Server 2003在工作組模式或作為域中的成員服務器時，計算機操作系統中存在的是本地用戶和本地組。本地用戶賬戶的作用範圍僅限於在創建該賬戶的計算機上，以控制用戶對該計算機上的資源的訪問。

當用戶訪問在工作組模式下的計算機時，必須具有被訪問計算機的本地賬戶，本地賬戶存儲在%SystemRoot%\system32\config\Sam數據庫中，這些賬戶在被訪問計算機上必須是唯一的。

本地用戶賬戶驗證由創建該賬戶的計算機進行，因此這種類型賬戶的管理是分散的，並且也有一個唯一的SID標誌，記錄賬戶的權限和組的隸屬關係。

③內置用戶賬戶。

Windows Server 2003自帶賬戶，系統安裝好後這些賬戶已經存在，並且賦予相應的權限以完成某些特定的工作。Windows內置用戶賬戶包括Administrator和Guest，內置賬戶不允許被刪除，Administrator不允許被屏蔽，但內置賬戶允許被更名。

Administrator賬號被賦予在域中和在計算機中不受限制的權利，用於管理本地計算機或域，具體地說，創建其它用戶賬號和組、實施安全策略、管理打印機和分配用戶對資源的訪問權限等。Guest用於在域或計算機中的用戶臨時訪問時使用的，默認情況下不允許對域或計算機中的設置和資源做永久性的更改。

在規劃Windows Server 2003域時，注意考慮賬戶的命名約定和賬戶的密碼約定。

賬號命名約定包括：域用戶賬號的用戶登錄名在AD中必須惟一；域用戶賬號的完全名稱在創建該用戶賬號的域中必須惟一；本地用戶賬號在創建該賬號的計算機上必須惟一；如果用戶名稱有重複，則應該在賬號上區別出來；對於臨時僱員應該做出特殊的命名，以便標示出來。

賬戶密碼約定包括：儘量避免帶有明顯意義的字符或數字的組合，最好採用大小寫和數字的無意義混合；對於不同級別的安全要求，確定用戶的賬號密碼是由管理員控制還是由賬號的擁有者控制；定期更改密碼，儘量使用不同的密碼.有關密碼的策略可以由系統管理員在密碼策略管理工具中加以規定，以保護系統的安全性。

Windows文件系統安全

Windows Server 2003使用NTFS文件系統格式，該結構提供數據文件訪問控制機制。NTFS權限是基於NTFS分區實現，支持用戶對文件的訪問權限，支持對文件和文件夾的加密，NTFS權限可以實現高度的本地安全性。

①通過對用戶賦予NTFS權限可以有效地控制用戶對文件和文件夾的訪問。

在NTFS分區上的每一個文件和文件夾都有一個ACL列表，該表記錄每一個用戶和組對該資源的訪問權限。在默認情況下，NTFS權限具有繼承性，即文件和文件夾繼承來自上層文件夾的權限。

NTFS權限分為特殊NTFS權限和標準NTFS權限兩類，標準NTFS權限是特殊NTFS權限的特定組合，特殊NTFS權限規定了用戶訪問資源的所有行為。

Windows將一些常用的特殊NTFS權限組合為標準NTFS權限，當需要分配權限時將一個標準NTFS權限分解為多個特殊NTFS權限，簡化權限的分配和管理。

②NTFS權限的使用原則。

一個用戶可能屬於多個組，組對某種資源被賦予了不同的權限，NTFS權限法則判斷用戶對資源有何種訪問權限。

權限最大原則，當一個用戶同時屬於多個組，而這些組又有可能被對某種資源賦予了不同的訪問權限，則用戶對該資源最終有效權限是在這些組中最寬鬆的權限，即加權限，將所有的權限加在一起即為該用戶的權限。

文件權限超越文件夾權限原則，當用戶或組對某個文件夾以及該文件夾下的文件有不同的訪問權限時，用戶對文件的最終權限是用戶被賦予訪問該文件的權限，即文件權限超越文件的上級文件夾的權限，用戶訪問該文件夾下的文件不受文件夾權限的限制，而只受被賦予的文件權限的限制。

拒絕權限超越其它權限原則，當用戶對某個資源有拒絕權限時，該權限覆蓋其它任何權限，即在訪問該資源的時候只有拒絕權限是有效的。

當有拒絕權限時權限最大法則無效。因此對於拒絕權限的授予應該慎重考慮。

③NTFS權限的繼承。

在同一個NTFS分區內或不同的NTFS分區之間移動或拷貝一個文件或文件夾時，該文件或文件夾的NTFS權限會發生不同的變化。在同一分區內移動的實質就是在目的位置將原位置上的文件或文件夾搬過來，因此文件和文件夾仍然保留有在原位置的一切NTFS權限。

在不同NTFS分區之間移動文件或文件夾，文件和文件夾會繼承目的分區中文件夾的權限，實質就是在原位置刪除該文件或文件夾，並且在目的位置新建該文件或文件夾。

在同一個NTFS分區內拷貝文件或文件夾，文件和文件夾將繼承目的位置中的文件夾的權限。在不同NTFS分區之間拷貝文件或文件夾，文件和文件夾將繼承目的位置中文件夾的權限。

④共享文件夾權限管理。

共享文件夾是被用來向網絡用戶提供對文件資源的訪問，可以包括應用程序、公用數據或用戶個人數據。共享文件夾的讀權限：用戶可以顯示文件夾名稱、文件名、文件屬性，運行程序文件，對共享文件夾內的文件夾作出改動。修改權限：用戶可以創建文件夾、向文件夾中添加文件、改變文件中的數據、向文件中添加數據、改變文件屬性、刪除文件夾和文件。完全控制權限：用戶可以改變文件權限、獲取文件的所有權、並執行修改權限允許的所有任務。

⑤文件的加密與解密。

在Windows Server 2003的NTFS文件系統中內置了EFS加密系統，利用EFS加密系統可以對保存在硬盤上的文件進行加密，其加密和解密過程對應用程序和用戶而言是完全透明的。文件或文件夾被加密後，未經許可加密文件或文件夾進行物理訪問的入侵者都無法閱讀這些文件或文件夾中的內容。通常將要加密的文件置於一個文件夾中，在對該文件夾加密，可以一次加密大量的文件。在該文件夾下創建的所有文件和子文件夾都會被加密。

Windows系統主機安全

Windows Server 2003主機安全是針對單個主機設置的安全規則，保護計算機上的重要數據。系統安全策略定義了用戶在使用計算機、運行應用程序和訪問網絡等方面的行為，通過這些約束避免了各種對網絡安全的有意或無意的傷害。

安全策略是一個事先定義好的一系列應用於計算機的行為準則，應用這些安全策略將用戶有一致的工作方式，防止用戶破壞計算機上的各種重要配置，保護網絡上的敏感數據。在Windows Server 2003中安全策略是以本地安全設置和組策略兩種形式出現的。

本地安全設置是基於單個計算機的安全性而設置的。對於較小的組織，或者在網絡中沒有應用活動目錄的網絡，適用於本地安全設置。而組策略可以在站點、組織單元或域的範圍內實現，通常在較大規模並且實施活動目錄的網絡中應用組策略。

①實施本地安全設置。

本地安全設置只能在不屬於某個域的計算機上實現，其中可設定的值較少，對用戶的約束也較少。如果要在整個網絡中約束用戶使用計算機的行為，則必須在每一個計算機上實施本地安全設置。本地安全設置包括賬戶策略、本地策略、公鑰策略和IP安全策略。

②配置並實施組策略。

在Windows Server 2003活動目錄中，組策略主要作用是規定用戶和計算機的使用環境，還應用於成員服務器、域控制器以及管理範圍內的其它計算機。組策略設置定義了系統管理員需要管理的用戶桌面環境的各種組件。要為特定用戶組創建特殊的桌面配置，可使用組策略對象編輯器創建組策略對象，組策略對象又與選定的活動目錄對象相關聯。

組策略包括兩部分：用戶配置策略，是指定對應於某個用戶賬戶的策略，這樣不論該賬戶在域內哪個計算機上登錄，其工作環境都是一樣的。計算機配置策略，是指定對應於某臺計算機的策略，這樣不論哪個賬戶在該計算機上登錄，其工作環境都是一樣的。

③使用預定義安全性模板。

Windows Server 2003包含多個安全性模板適用於不同安全需求，利用這些模板網絡管理人員可以簡化策略的設定和實施操作。

預定義的安全模板包括4種安全級別：基本級別的模板為Windows定義的默認的安全級別，包括以下設定：默認的工作站、默認的服務器、 默認的域控制器，可以在\systemroot\security\templates文件夾中找到這幾個模板。

兼容標準的模板包括商用應用程序的所有功能，使之仍然可以有效的運行,該模板為兼容工作站或服務器模板。安全模板有可能會影響到一些商用應用程序的某些功能的運行，主要包括安全的工作站或服務器、安全的域控制器。

高度安全模板，不會考慮應用程序是否會受到這些設定的影響，在通常情況下這類模板要慎重使用，主要包括：高安全的工作站或服務器、高安全的域控制器。
