c語言入門4，製作可以變形的積木，函數的形參和實參

2020-11-14
| 科技

上一節介紹瞭如何自己製作積木——定義自己的函數，並且給出了例子。應該能記得，上一節中自定義的函數既沒有返回值，也沒有參數，但是實際應用中，要解決複雜的問題，函數應該能夠根據外界的輸入做出相應的變化。例如，我們需要打印出 4+6 的值，函數可以如下定義：


void add(){ printf("4+6=%d\n", 4+6);}

的確，add 函數能夠打印出 4+6 的值，解決了問題。但是，可能還會有類似，但不相同的問題出現，比如需要打印出 3+4 的值，這時上面定義的 add 函數就沒有辦法解決問題了，只能重新定義一個函數。這非常麻煩。事實上，C 語言中的函數可以接收外界傳入的數據，我們如下修改 add 函數：

void add(int a, int b){ printf("%d+%d=%d\n", a, b, a+b);}

這樣，add 函數解決的就不是一個問題了，而是一類問題，打印 4+6 的值只需調用 add(4, 6)，打印出 3+4 的值，只需調用 add(3, 4)。


形參和實參

在之前一節，我們介紹了全局變量和局部變量，知道了函數的參數也屬於局部變量。在 C 語言中，函數的參數分為“形參”和“實參”，以上面定義的 add 函數為例，我們調用 add(4, 6) 時，函數中的參數 a 就代表 4，b 就代表 6。確切的說，當我們討論函數中的 a 這個參數時，我們所說的“參數”是指“形參”，當我們討論傳一個參數 4 給函數時，我們所說的“參數”是指“實參”。

但是我習慣都叫參數而不習慣總把形參、實參這兩個文縐縐的詞掛在嘴邊兒（事實上大多數人都不習慣），可根據上下文判斷指的到底是形參還是實參。記住這條基本原理：形參相當於函數中定義的變量，調用函數傳遞參數的過程相當於定義形參變量並且用實參的值來初始化。
int main(){ int p1 = 4, p2 = 6; add(p1, p2); return 0;}

上面在 main 中調用了 add 函數，就相當於在 add 函數中執行了如下語句：

int a = p1;int b = p2;printf("%d+%d=%d\n", a, b, a+b);


main 函數的變量 p1 和 add 函數的參數 a 是兩個不同的變量，只不過它們各自的存儲空間中存了相同的值 4，因為變量 p1 的值賦給了參數 a。同理，變量 p2 的值賦給了參數 b。C語言中在調用函數時，每個參數都需要得到一個值，函數定義中有幾個參數，在調用中就需要傳幾個參數，不能多也不能少，每個參數的類型也必須對應上。

事實上，C 語言中也定義了一些參數不定的函數，比如 printf，它可以接收 1 個或多個參數。以後遇到需要定義這樣的函數時，再詳細介紹。

易錯點

請看如下代碼：

void add(int a){ a=a+1;}int main(){ int p = 2; add(p); printf("p=%d\n", p); return 0;}

該程序員原計劃讓 add 函數實現參數 p 加一，但是卻失敗了，以上代碼輸出的是 p=2。這是因為 add 函數中的參數 a 和 main 函數中的 p 根本就是兩個變量。add 函數在執行完畢後，它所在的內存就被收回了。關於這一點，《一文弄懂C語言的全局變量和局部變量，它們會衝突嗎》一節說的非常清楚，可以再回顧一下。


上一節，我們把自定義函數比作自己做積木，那自定義帶參數的函數，就相當於製作可以變形的積木，更加靈活，所以能夠完成更多工作。
