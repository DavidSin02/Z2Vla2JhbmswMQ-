安防國標學習筆記8 NodeJS實現GB28181信令服務端

2020-10-29
| 科技

一、實現步驟

通過前面的幾次學習，一個能接收攝像機註冊的GB28181服務端信令服務器功能基本上可以使用了。本文只是對前面的內容進行總結匯總。

1. 整體架構

  • NodeJS實現信令服務器
  • ZLMediaKit 用作媒體服務器

2. 部署過程

I 克隆項目GB28181_NODE_HTTP到本地

  1. 通過npm i安裝依賴
  2. 修改配置文件，將 sip_config.js.template 修改為 sip_config.js

配置文件說明：

/** * 配置文件 */var config={    // 設備信息，沒有寫在這裡的將註冊不了，不允許匿名註冊    registry : {        '34020000001110000001'  : { password : '12345678', online:false},   // 測試，這個是設備的account，需要密碼        '34020000001320000001'  : {}                                        // 測試，這個是通道的account，不需要密碼    },    http:{        // http restful接口端口        port        : 8777    },    sip:{       server:{           // sip服務端口號           port     : 5060,           // sip 信令服務器域           realm    : '3401030000',           // sip信令服務器ip地址，一般是本機ip           ip       : '148.70.110.25',           // sip服務端 account           account  : '34010300002000002250'       }     },    logger:{        // 日誌地址        path:'./sip.log'    },    // 生成Tag使用的函數    rstring:function() { return Math.floor(Math.random()*1e6).toString(); }};module.exports=config;

啟動項目，使用node sip_app.js也可以使用pm2啟動，使用pm2 start sip_app.js

II 部署ZLMediaKit

這裡使用CentOS7，使用docker部署

a. 安裝docker

b. 下載並啟動鏡像

# 創建dockerdocker run -id -p 10000:10000/udp -p 1935:1935 -p 8080:80 -p 554:554 -p 332:332 -p 9000:9000 gemfield/zlmediakit:20.04-runtime-ubuntu18.04

c. 設置webhook

MediaServer可以把內部的一些事件通過http post 第三方http服務器的方式通知出去，以下是相關的默認配置：

[hook]enable=1admin_params=secret=135c73f7-bb6b-4889-a715-d9eb2d1925ddtimeoutSec=10on_flow_report=https://127.0.0.1/index/hook/on_flow_reporton_http_access=https://127.0.0.1/index/hook/on_http_accesson_play=https://127.0.0.1/index/hook/on_playon_publish=https://127.0.0.1/index/hook/on_publishon_record_mp4=https://127.0.0.1/index/hook/on_record_mp4on_rtsp_auth=https://127.0.0.1/index/hook/on_rtsp_authon_rtsp_realm=https://127.0.0.1/index/hook/on_rtsp_realmon_shell_login=https://127.0.0.1/index/hook/on_shell_loginon_stream_changed=https://127.0.0.1/index/hook/on_stream_changedon_stream_none_reader=https://127.0.0.1/index/hook/on_stream_none_readeron_stream_not_found=https://127.0.0.1/index/hook/on_stream_not_foundon_server_started=https://127.0.0.1/index/hook/on_server_started

如果是鑑權事件，且訪問IP是127.0.0.1或者鑑權url參數與admin_params一致，那麼會直接鑑權成功(不會觸發鑑權web hook)。

設置on_stream_changed:

on_stream_changed=https://sip信令服務器地址:8777/index/hook/on_stream_changed

設置以後，推送成功的消息就會傳到我們的NodeJS程序裡。

在Express裡接收此消息，並解析出播放的url;

on_stream_changed解釋:

rtsp/rtmp流注冊或註銷時觸發此事件；此事件對回覆不敏感。

觸發請求：POST /index/hook/on_stream_changed HTTP/1.1Accept: */*Accept-Language: zh-CN,zh;q=0.8Connection: keep-aliveContent-Length: 118Content-Type: application/jsonHost: 127.0.0.1Tools: ZLMediaKitUser-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_12_1) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/57.0.2987.133 Safari/537.361234567891011
{   "app" : "live",   "regist" : true,   "schema" : "rtsp",   "stream" : "obs",   "vhost" : "__defaultVhost__"}

請求參數詳解：

參數名參數類型參數解釋appstring流應用名registbool流注冊或註銷schemastringrtsp或rtmpstreamstring流IDvhoststring流虛擬主機

默認回覆:

二、使用過程

1. 設置IPC

找一臺大華或海康攝像機，設置：

  • sip服務器： 設置為sip信令服務器地址
  • sip服務端口號： 5060
  • sip服務賬號： 與上面sip_config.js裡的sip.server.account設置保持一致
  • sip客戶端賬號： 與上面sip_config.js registry裡值保持一致
  • sip通道賬號： 與上面sip_config.js registry不帶密碼的值保持一致

2. 訪問列表api

http://信令服務器ip:8777/api/v1/device/list

3. 訪問播放api

http://信令服務器ip:8777/api/v1/device/invite?account=34020000001320000001

4. 獲得播放地址

http://信令服務器ip:8777/api/v1/device/query?account=34020000001320000001從中獲取streamid值。

5. 使用rtsp播放

rtsp://媒體服務器地址/rtp/streamid
