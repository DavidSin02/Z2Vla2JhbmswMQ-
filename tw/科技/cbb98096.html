一個完整的項目管理流程包括什麼？

2020-10-29
| 科技

首先，我們先了解一下什麼是「項目」和「項目管理」。

「項目」是為完成某一獨特的產品或服務所做的臨時性努力。在項目的實施過程中，通常都會遇到一些問題，這些問題會阻礙項目的實施。而「項目管理」就是指在項目活動中運用專門的知識、技能、工具和方法，使項目能夠在限定的資源條件和計劃的時間內高質量、高效率地實現或超過設定的需求和期望的過程。

「項目管理」的五個過程組

一個完整的項目管理流程包括五個階段：啟動階段、規劃階段、執行階段、監控階段、收尾階段，其中監控階段貫穿項目管理的整個過程。


1. 啟動階段

這是所有項目的開始階段，是新的項目識別和開始的過程。在這個階段，最關鍵的是要確定項目的價值及可行性。通常情況下，在項目獲得批准或拒絕之前，需要給利益相關方提供以下工作：

  • 商業案例：證明項目的必要性，分析投資回報
  • 可行性研究：概述項目的目標和需求，預估項目完成所需的資源和項目的成本，以及項目是否具有財務和商業意義

2. 規劃階段

項目獲得批准之後，下一步就是規劃項目的管理工作，以確保項目能在規定的時間和預算的範圍內實現預期的目標。在這一階段，又可以分為幾個主要的部分：

• 範圍說明：明確業務的需求、項目的目標和可交付成果，確定項目的管理範圍

• 工作分解：創建工作分解結構（WBS，Work Breakdown Structure），將任務細分為小組件，以便於管理，並確定任務的優先級及關聯性

• 進度計劃：估算任務的持續時間，可以利用甘特圖將所有的任務和截止日期以時間軸的格式直觀地表示出來，在重要的時間節點設置里程碑，制定進度計劃

• 成本管理：預估每一項工作所需要的成本，制定預算表，制定成本管理方案

• 溝通計劃：確定任務進行中負責人和成員之間的溝通計劃，制定高效的溝通方式，並確定好項目的例會制度風險管理計劃：識別項目進行過程中可能遇到的風險，並對其進行分析以確定風險等級，制定相應的風險應對方案

• 風險管理計劃：識別項目進行過程中可能遇到的風險，並對其進行分析以確定風險等級，制定相應的風險應對方案

3. 執行階段

在對項目進行了規劃之後，就可以開始啟動項目了。雖然我們已經在規劃階段做了很多準備，但是這並不意味著前方會一路平坦，在執行階段還是有很多問題需要注意：

• 任務進行過程中會積累很多的經驗、知識，這些都是寶貴的財富，需要對其進行管理，同時也要注意對任務的開展質量進行管理

• 項目的最終成果取決於每一個小任務的完成，而團隊中每一個成員的工作決定了任務的質量，所以需要切實地建設和管理團隊的積極性及溝通方式，確保每一位成員高效工作

• 項目執行過程中如果遇到風險，要及時實施風險應對策略以確保項目正常進行

ONES Project 中的看板視圖幫助團隊快速同步信息、溝通進度


4. 監控階段

為了能確保項目計劃的順利實施，必須實時對項目的方方面面進行監控和評估，並及時作出必要的調整。其中幾個主要的監控方面為：

• 實時跟蹤項目的整體進度，掌握可能導致項目延期的情況，及時對進度表進行調整以保持正常運行

• 監控相關方需求的變化、市場的變化，並及時處理變更

• 監控資源使用和成本使用，確保資源合理分配，及時維護成本基準

• 監控溝通效果，確保成員之間、成員和相關方之間的溝通是高效的

• 監控風險並識別新的風險，對風險管理及時作出更新，評估風險管理的效果

使用 ONES Project 中的燃盡圖追蹤項目進度

ONES Project 提供多維度報表跟蹤項目進度


5. 收尾階段

實現了項目的目標，產出了一個可交付成果，但並不意味著項目結束了，這個時候還要進行項目的最後一個階段：收尾階段。在這個階段，首先要審查項目的實現情況，根據評估規定確定達到要求後，移交項目成果給相關方，同時還要總結項目的相關經驗結果並完成信息歸檔，以便實現持續改進。

項目管理是一個極其複雜的過程，若要使項目的所有任務都能按時順利地實施，藉助於項目管理軟件會是更好、更高效的選擇。而且在使用雲端的項目管理軟件時，數據和協作可以實時同步，能更準確地反映項目情況並有助於進行正確的決策。

ONES Project 是一款項目進度管理工具，涵蓋了儀表盤、項目管理、任務管理、篩選器、迭代管理、進度報表查看、通知、插件等功能，支持項目的全流程管理。ONES 的產品矩陣還包括 ONES Plan（項目集管理）、ONES Wiki（知識庫管理）等，此產品矩陣可以提升團隊的效率，提高項目成果的交付質量！
