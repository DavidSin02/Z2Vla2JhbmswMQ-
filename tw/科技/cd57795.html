Python 的動態類型系統

2020-10-29
| 科技

本系列文章譯自Python之父 Guido van Rossum 的系列博客“The History of Python”。這個博客系列對我們理解Python及其演變很有幫助，經Guido同意，在這裡翻譯推薦給大家，希望大家喜歡，也請大家多多指教！

Python 與 ABC 的一個重要區別在於其類型系統。

ABC 採用靜態類型，編譯器會檢查程序中的變量類型是否保持一致，如果不一致，程序就無法運行。並且，ABC與當時大多數靜態語言不同，採用的是類型推導（和 Haskell 一樣），而不是類型聲明（比如 C 語言）。而 Python 採用動態類型，所有類型檢查都是在程序運行過程中，而不是編譯時進行的。

不過，Python 的類型系統與 ABC 也沒有大家所想的那麼大的區別。

與其它靜態語言不同，ABC 除了在編譯時進行類型檢查，在運行時還會再次檢查。在ABC的原型中，並沒有實現完整的編譯時類型檢查，於是，運行時類型檢查可以作為一種驗證方式。同時，運行時類型檢查可以拋出清楚的異常信息，對debug也很有幫助，否則，語言內核可能會因為一些類型錯誤直接崩潰。

ABC 採用運行時類型檢查的最主要的原因，在於它是一種交互式語言。

在交互式場景中，用戶輸入的每一句代碼，都是立即執行的，因此，完全有可能創建一個數字變量，刪除它，又創建一個同名的字符串變量。如果這樣的賦值過程出現在同一段代碼中，顯然是無法編譯通過的，但在交互式場景中，禁止這種操作也不合理。作為妥協，ABC 對全局變量採用動態類型檢查，而局部變量採用靜態類型檢查——為了簡化實現，乾脆所有變量都在運行時再次檢查。

因此，實際上 Python 的類型檢查系統實現得很簡單——只是完全拋開 ABC 的編譯時類型檢查而已。這很符合 Python 的“抄近道”哲學，因為所有的類型錯誤都在執行前檢查過了，所以這個簡單的實現完全不影響程序的可靠性。

不過，一旦採用動態類型系統，就和靜態類型徹底告別了。

在 ABC 中，精心設計的內置操作可以根據輸入內容的具體形式來推斷其類型（譯註：ABC採用類型推導而不是類型聲明），比如說，根據“x^y”，編譯器可以推斷，x 和 y 都是字符串，這個操作的結果也一定是字符串。而在 Python 中則沒法做這樣的推斷，比如“x + y”有可能是字符串拼接，也可能是數字相加，也可能是用戶自定義類型重載加號之後的某種操作。
