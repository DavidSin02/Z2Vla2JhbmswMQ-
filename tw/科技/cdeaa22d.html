巧妙結合spring與簡單模板方法實現代碼簡化

2020-11-14
| 科技

在之前的一個項目中，有遇到過分步處理的問題，簡單的來說就是極簡的工作流，一般是指處理步驟在三步以內。

需求場景是這樣的：公司訂單線的系統需要升級重構，原本兩個代碼比較亂的服務（其中一個是用php寫的）需要拆分成三個新服務（全部用java重構）

服務A作為前置服務，直接和前端做交互；服務B作為中間調度服務，承載所有的訂單調度業務；服務C作為交易服務，負責所有訂單的支付業務。

我負責的是服務B，其中有一個環節是：服務A將用戶的的下單信息推送到服務B，服務B接收後會生成訂單信息，然後對訂單做業務調度（正向成單，逆向退單等等），最後會將該訂單的信息推送到各個關聯繫統（大概有三十多個），但是這些關聯繫統不一定要全量的訂單信息（如果所有關聯繫統都全量推，消息隊列也扛不住），所以需要對這些信息做個性化處理。

我是使用了簡單的模板方法+spring事件來實現的

首先，對於上述的場景，我們可以拆分為兩個步驟

  1. 數據組裝。由於各個關聯繫統比較雜亂，有的是go寫的，有的是php，還有不同的報文格式要求，所以數據組裝這一步需要自定義，可以是簡單的json字符串報文，可以是xml，也可以是自定義格式的報文
  2. 數據推送。雖然公司有統一的rpc框架，但是對於go，php編寫的系統不夠友好，所以這類的系統就統一使用http推送，但是一部分用java編寫的系統，經討論後還是考慮使用rpc遠程調用，最後測試下來也確實比用http接口調用要更有效率

ok，到這裡基本上就算是做好了分析了，接下來就是針對分析結果做設計，很自然的我想到了使用模板方法來解決這個問題，主要是基於以下兩點考慮：

  1. 步驟簡單，雖然有現成的工作流框架可以使用，但是自己實現的話應該會更輕量
  2. 只需要定義兩個模板，並且藉助spring可以很好的管理

接下來就上代碼

AbstractDataGroupFunc這個類可以看作是數據組裝的基類，通過模板方法實現了註冊的功能實現了spring的增強接口InitializingBean，在spring啟動的時候會自動註冊所有繼承該基類的實現類
package cn.magicnian.demo1022.group;import org.slf4j.Logger;import org.slf4j.LoggerFactory;import org.springframework.beans.factory.InitializingBean;import org.springframework.context.ApplicationEvent;import org.springframework.util.Assert;import java.util.Map;import java.util.concurrent.ConcurrentHashMap;public abstract class AbstractDataGroupFunc implements InitializingBean {    private static Logger logger = LoggerFactory.getLogger(AbstractDataGroupFunc.class);    private static final Map<Class<? extends ApplicationEvent>, AbstractDataGroupFunc> DATA_GROUP_MAP = new ConcurrentHashMap<>();    //初始化    protected void register(Class<? extends ApplicationEvent> eventClass, AbstractDataGroupFunc dataGroup) {        DATA_GROUP_MAP.put(eventClass, dataGroup);    }    public static AbstractDataGroupFunc getDataGroupFunc(Class<? extends ApplicationEvent> eventClass) {        Assert.isTrue(eventClass != null, "事件class不能為null!");        return DATA_GROUP_MAP.get(eventClass);    }    /**     * 數據組裝-按需自由處理     * 如可以組裝成json字符串或者是自定義報文格式     * @param event     * @return     */    public abstract String doDataGroup(ApplicationEvent event);}

AbstractDataPushFunc數據推送的基類，設計原理和數據組裝基類一致
package cn.magicnian.demo1022.push;import org.slf4j.Logger;import org.slf4j.LoggerFactory;import org.springframework.beans.factory.InitializingBean;import org.springframework.context.ApplicationEvent;import org.springframework.util.Assert;import java.util.Map;import java.util.concurrent.ConcurrentHashMap;public abstract class AbstractDataPushFunc implements InitializingBean {    private static Logger logger = LoggerFactory.getLogger(AbstractDataPushFunc.class);    private static final Map<Class<? extends ApplicationEvent>, AbstractDataPushFunc> DATA_PUSH_FUNC_MAP = new ConcurrentHashMap<>();    protected void register(Class<? extends ApplicationEvent> eventClass, AbstractDataPushFunc dataPushFunc) {        DATA_PUSH_FUNC_MAP.put(eventClass, dataPushFunc);    }    public static AbstractDataPushFunc getDataPushFunc(Class<? extends ApplicationEvent> eventClass) {        Assert.isTrue(eventClass != null, "事件class不能為null!");        return DATA_PUSH_FUNC_MAP.get(eventClass);    }    /**     * 數據推送-自由處理     * 可以是簡單的http接口，或者是rpc調用，自定義報文協議通過netty發送等     * @param param     */    public abstract void doDataPush(String... param);}
