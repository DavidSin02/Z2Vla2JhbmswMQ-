開源 | Umajs：輕量級 Node.js Web 框架

2020-11-14
| 科技


開源二期項目專題系列

（二）

1.開源項目名稱：Umajs


2.GitHub地址：

https://github.com/wuba/Umajs


3.簡介：Umajs 是58同城推出的一款輕量級 node web 框架。它的中文含義是大熊座，北斗七星都是它的組成部分；正如同 Umajs 也是由不同的 package 所組合在一起。我們希望 Umajs 的每一部分，都是優秀的、閃耀的、經受的住各種大型項目檢驗的。

Umajs開源項目具備以下特點：

  • 內置豐富的裝飾器 (decorator)，提供自定義裝飾器機制；
  • 依賴注入 (IOC)，模塊依賴不再需要引入和實例化；
  • 面向切面 (AOP)，基於裝飾器的 AOP 可以很方便的使用在各種方法上；
  • 統一返回 (Result)，讓編碼更簡單清晰；
  • 提供插件機制，靈活擴展框架功能，兼容 Koa2 中間件；
  • 高穩定高性能，源碼 TypeScript 開發、單元測試覆蓋全。


項目背景

Nodejs 對於前端工程師的重要性不言而喻，它的出現使 JavaScript 成為了最受歡迎的 web 開發語言之一，各類 node web 框架層出不窮，諸如 Express、Koa 等等；但是在應用到實際業務場景時我們還需要額外做一些功能的擴展，為此相應的社區也提供了大量的中間件。

與此同時，隨著 JavaScript 承擔了更多的大型複雜應用的開發，TypeScript 也漸漸進入了我們的視野：做為 JavaScript 的超集，它的靜態類型檢查無疑是構建大型 JavaScript 項目的強大助力。
有鑑於此，我們推出了使用 TypeScript 開發的 node web 框架：Umajs 。這裡主要和大家分享一下 Umajs 在搭建過程中的設計及其應用的介紹。


架構設計

  • 用戶請求達到 router；
  • router 解析請求穿過 plugin 中的中間件，然後到達 controller；
  • controller 可以通過 IOC 調用 service 和 resource；
  • controller、service、resource 等可以通過 AOP 的 Aspect 進行切面開發；
  • controller 返回 Result，Umajs 解析 Result 按 Koa 框架格式返回數據；
  • AOP 可以對 controller、service、resource 進行切面開發，還可以將 middleware 封裝成 Aspect.around 對 controller 進行切頁面開發；
  • 中間件（middlware）有提供兩種形式使用，一種是插件配置(plugin.config.ts)，一種是封裝成Aspect.around 以裝飾器形式使用；
  • Plugin 有兩種形式進行擴展，一種中間件形式、一種複合形。

AOP

AOP(Aspect Oriented Programming) 面向切面編程，是一種不修改原代碼也能為其動態擴展功能的編程方式。

Umajs 內置了以下五種切面：

  • before：在目標方法之前執行；
  • after：在目標方法之後執行；
  • around：在最開始調用時執行，會將目標方法作為參數傳入，可對目標方法進行攔截；
  • afterThrowing：當執行目標方法出現異常時調用；
  • afterReturning：當目標方法有返回值之後執行，在 after 之後執行。


切面可以作用在 class 或者 method 上；一個 class 或者 method 上可以有多個切面，執行順序如下圖所示：


使用 AOP 能夠有效的隔離業務中不同部分的邏輯。例如原有目標方法如下：





getList(params){    // do something    return list; }

假設隨著業務的發展，我們需要為這些方法添加埋點統計。此時我們可以通過 AOP 的方式快速擴展目標方法：

@Aspect.before('report')getList(params){    // do something    return list; }

相比直接修改 getList 方法，AOP 的方式分離了上報邏輯與數據獲取邏輯、降低了耦合度，在保持對業務代碼零侵入的情況下完成了功能的擴展。同時該上報方法能夠多處複用，提高了代碼的可複用性與可維護性。


Result統一返回

統一返回是 Umajs 的另一個特色。我們知道，在大多數 node web 框架中，controller 在處理完業務邏輯後以如下的方式返回處理結果：


// 返回json數據getList(param){    // do something    ctx.body = { list };}// 返回viewindex(){    ctx.render('index.tpl');}

而 Umajs 提供瞭如下快捷返回方式：

  • Result.send(val, status)：快捷返回文本內容；
  • Result.json(data)：返回json數據；
  • Result.jsonp(data, callback)：以jsonp的形式返回數據；
  • Result.view(templatePath, data)：通過渲染模板的方式將數據返回；
  • Result.stream(data, fileName)：將文件以流（stream）的方式返回；
  • Result.download(filePath, opts)：下載文件。

Result 封裝了常見的返回值快捷方式，有效的提升了開發效率。同時 Result 可以通過插件進行擴展，為業務提供更多可能性。Result 的意義不僅於快捷方式，在結合了切面、自定義裝飾器等功能後，我們可以很輕鬆的實現對返回值的定製修改。


自定義裝飾器

Umajs中大量的功能都通過內置裝飾器來提供，諸如：

  • 切面裝飾器：@Aspect
  • IOC裝飾器：@Resource、@Inject
  • 路由裝飾器：@Path、@Private

同時針對高頻場景提供了自定義參數裝飾器 createArgDecorator，例如通過該方法創建的裝飾器能夠快速到獲取到 query 上的參數：

const Query = createArgDecorator((argKey, ctx) => ctx.query[argKey]);

crateArgDecorator用途不止於此，可以利用它進行參數聚合、參數校驗、參數轉換等等參數相關的操作。下文我們會詳細介紹其強大之處。

在提供豐富的內置裝飾器和自定義參數裝飾器的同時，框架的使用者也可以自定義符合規範的其它類型裝飾器，可以參考 TypeScript 裝飾器，此處不再贅述。


應用介紹

上文介紹了Umajs的幾個特色，接下來我們一起探索它們在業務場景下的應用。

業務需要根據用戶ID查詢該用戶下的列表數據，API如下:

GET /api/list?userId=userId

對應的controller方法可以如下實現:

getList(){    const { userId } = this.ctx.query;    const list = this.service.getListByUid(userId);    return Result.json(list)}

雖然上述代碼實現了查詢功能，但是這種實現是不符合開發規範、存在風險的：

首先我們期望的返回值內存在狀態碼而不僅僅是數據；其次userId為必填項，此處應該進行參數校驗；最後沒有進行異常處理。

首先我們創建一個自定義的參數裝飾器用於類型檢查：

const IdCheck = createArgDecorator((key: string, ctx: IContext) => {    const id = ctx.query[key];        if (id === undefined || id === null || id === '') {        return Result.json({            code: ErrorType.Empty,            msg: `參數 ${key} 不能為空`,        });    }    // 其它校驗邏輯    return id;});export IdCheck;

然後創建一個切面，該切面內實現一個用於異常處理的around切點：

export default class Demo implements IAspect {    async around(proceedPoint: IProceedJoinPoint) {        try {            const { proceed, args } = proceedPoint;            const result = await proceed(...args);            return result;        } catch (e) {            // 可用 afterThrowing            return Result.json({                code: ErrorType.Error,                msg: e.toString(),            });        }    }}

改造後的方法如下:

@Aspect('demo')getList(@IdCheck('userid') uid)){    const list = this.service.getListByUid(uid);    return Result.json(list)}

通過以上簡單的示例我們可以看到：Umajs 的代碼在保持業務邏輯清晰的同時，以輕量的擴展實現了參數校驗、異常處理、返回值包裝等等擴展功能。而這些擴展的裝飾器是可以複用在其它方法上的。

對於大型的、複雜度高的工程來說，使用 Umajs 能夠在獲得良好開發體驗的同時，大幅提升工程的可擴展性、可維護性、可複用性，從而提升開發效率。


核心優勢

  • 功能強大：內置豐富的裝飾器，諸如IOC 裝飾器、AOP 裝飾器、路由裝飾器等等，也可以自定義裝飾器；
  • 擴展靈活：通過插件機制擴展框架，提供多種方式兼容 Koa 中間件；
  • 快捷返回：框架內置了常用的返回類型，同時支持擴展；結合切面及裝飾器能夠覆蓋絕大多數業務場景；
  • 開發效率：Umajs內置了大量功能能夠提升開發效率；同時 Koa 項目可以平滑的遷移到 Umajs。

未來規劃

Umajs會進一步的完善插件機制，使其更易於使用。未來將會增加對於模型驅動開發的支持。我們會持續關注業內的新技術、新趨勢、新特性，在合適的情況下會將其引入框架。與此同時，我們會根據社區的反饋不斷的完善框架。
