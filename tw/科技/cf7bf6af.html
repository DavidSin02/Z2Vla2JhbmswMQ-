android內存管理

2020-11-14
| 科技

android 內存優化是作為一個android開發人員必須要掌握的知識之一，它能夠讓我們的app更加的流程增加用戶的體驗感。

java內存分配策略：

1.靜態存儲區（方法區）：主要存放靜態數據、全局static數據和常亮。這塊內存在程序編譯時就已經分配好，並且在整個程序運行間都存在。

2.棧區：方法執行時被執行，方法體內的局部變量都在棧上創建，並在方法執行結束時這些局部變量所持有的內存將被自動釋放。因為棧內存分配運算內置於處理器指令集中，效率很高，但是分配的內存是有限的。

3.堆區：又稱為動態分配區，通常是值在程序運行時直接new出來的對象。這部分內存在不使用的時候將由java垃圾回收期來負責回收（new 出來的對象以及該對象中的所有成員變量都在堆內存中）。

tips:堆與棧的區別

棧中的變量生命週期隨著方法的結束而結束，系統自動回收

堆內存由GC負責回收

舉個例子：

public class Demo {

private final String TAG = Demo.class.getSimpleName();//常量， 方法區

private static int <em>m </em>= 0;//靜態變量， 方法區

private int n = 1;//成員變量， 堆區

private void learn(){

int k = 2;//局部變量， 棧區

Demo demo = new Demo();// demo局部變量， 棧區; Demo實例，堆區

}

}
