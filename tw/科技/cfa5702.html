public ParamNameResolver(Configuration config, Method method) {final Class<?>[] paramTypes = method.getParameterTypes();//獲取參數類型,對於上述例子則是String,int,Stringfinal Annotation[][] paramAnnotations = method.getParameterAnnotations();//這裡獲取到的則是@Param,getParameterAnnotations方法也用到了動態代理.final SortedMap<Integer, String> map = new TreeMap<Integer, String>();int paramCount = paramAnnotations.length;// get names from @Param annotationsfor (int paramIndex = 0; paramIndex < paramCount; paramIndex++) {if (isSpecialParameter(paramTypes[paramIndex])) { //過濾其內部一些特殊類型// skip special parameterscontinue;}String name = null;for (Annotation annotation : paramAnnotations[paramIndex]) {if (annotation instanceof Param) {hasParamAnnotation = true;name = ((Param) annotation).value();//獲取到'name'值break;}}if (name == null) {// @Param was not specified.if (config.isUseActualParamName()) {//默認為true,因此編譯後參數都是args0,args1之類,因此這裡獲取的也是args0...name = getActualParamName(method, paramIndex);}//當上面配置為false的時候這裡才會使用0,1代替,因此如果未開啟則會報錯if (name == null) {// use the parameter index as the name ("0", "1", ...)// gcode issue #71name = String.valueOf(map.size());//上述都沒的話則世界使用map的index.}}map.put(paramIndex, name);}names = Collections.unmodifiableSortedMap(map);}

那麼執行完畢後對於上述例子,names裡面如下圖所示,由於config.isUseActualParamName()為true,所以#{0}這種寫法這裡並不支持,而且也不建議這種寫法,無可讀性.

接下來執行method.convertArgsToSqlCommandParam(args)獲取到實際輸入的參數,對於上面例子我獲取到的是個Map集合,如下圖所示,對於單一實體例如User那麼獲取到的就是該實體.

再看我所用的sql寫法,那麼這裡只能獲取到name的值,sql處理時就會報錯.

由此可見針對多參數的輸入,最佳解決方案是用@Param註解,其次為使用Map集合包裹參數,這樣的話method.convertArgsToSqlCommandParam(args)得到的則是該Map集合.

到了接下來轉到執行器,使用DynamicContext構造動態sql所需要的上下文,對其構造函數分析

執行到這裡的話參數只有三種情況

  1. null,無任何參數傳入

  2. Map類型，對於多參數,或者參數本身就是map再或者輸入單一參數集合類型,數組類型都會轉換為map

  3. 單一POJO類型.

那麼針對上面的例子,這裡先是去boundSql中的addtionParameters中獲取參數,該參數一般是sql解析時動態生成的,比如foreach生成的_frch_xx,獲取不到的話再去原始的ParamsObject中獲取,該處的解析為遞歸形式了.

總結

Mybatis的SQL解析總體流程如下:

  1. 構造ParamtersMap,保存輸入參數.

  2. 構造ContextMap,為OGNL解析提供數據.

  3. 讀取xml.使用SqlSource與SqlNode解析xml中的sql,設置參數值到boundSql的addtionParameters中,其為ContextMap的一個副本.

  4. 根據boundSql.parameterMappings獲取到參數,從addtionParameters與ParamtersMap中讀取參數設置到PreparedStatement中

  5. 執行sql

    本文只分析了總體流程,其中有很多細節都忽略了,如遇到問題再看也不遲.
