c、c++三種傳參方式比較

2020-11-14
| 科技

C/C++中傳參方式可分三種：值傳遞（傳副本）、指針傳遞（傳地址）、引用傳遞（傳別名）。

（1）值傳遞：其實是一種賦值操作，實參傳遞給形參時會產生額外的副本，形參只是實參的拷貝，所以函數內對形參的操作不會更新到源實參。在函數結束時，形參作為局部變量會被釋放，對實參不會產生任何影響。若為類的對象會調用拷貝構造，這種深拷貝操作會影響到傳參效率。（可理解為“單向接口”）

（2） 指針傳遞：因為傳遞的是實參的地址，指針指向的內存中同一個對象，所以函數內部對形參得操作會“同步更新”到實參。（可理解為“雙向接口”）

（3）引用傳遞（C++特有）：傳遞的是實參的別名，傳參時形參被綁定到實參對象上，因此函數內部對形參的操作也都會“同步更新”到源實參。（可理解為“雙向接口”）

注：關於指針傳遞和引用傳遞，形參改變對實參的改變是一種副作用，我們利用這種副作用可以方便修改傳遞的實參，另外，這種方式避免了值傳遞中的拷貝，對於大數據的實參可以有效提升效率。因此，對於想提升傳參效率又不想影響實參的情況，可以使用const：

bool isShorter(const string &s1,const string &s2)//const修飾指針傳參會有同樣的效果

{

return s1.size()<s2.size();

}//這也是為何在我們不想不修改實參時儘量使用const引用（指針）而不是值傳遞的原因。
