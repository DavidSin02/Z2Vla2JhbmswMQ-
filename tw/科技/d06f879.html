1. SIFT介紹

SIFT（Scale Invariant Feature Transform），又稱尺度不變特徵轉換匹配算法，是在計算機視覺任務中的特徵提取算法。

SIFT可以幫助定位圖像中的局部特徵，通常稱為圖像的“關鍵點”。這些關鍵點是比例尺和旋轉不變量，可用於各種計算機視覺應用，例如圖像匹配，物體檢測，場景檢測等。

還可以將通過SIFT生成的關鍵點用作模型訓練期間的圖像特徵。與邊緣特徵或單一特徵相比，SIFT特徵的主要優勢在於它們不受圖像大小或方向的影響。

例如，這是埃菲爾鐵塔的另一張圖片以及它的較小版本。第一張圖像的關鍵點與第二圖像中找到的關鍵點相匹配。當另一幅圖像中的對象稍微旋轉時，兩幅圖像也是如此。是不是很厲害？

瞭解如何識別這些關鍵點，以及用於確保比例和旋轉角度不變的技術是什麼。廣義上講，整個過程可以分為四個部分：

  • 構造比例空間：確保要素與比例無關
  • 關鍵點本地化：確定合適的特徵或關鍵點
  • 方向分配：確保關鍵點是角度不變
  • 關鍵點描述符：為每個關鍵點分配獨一的指紋

最後，用這些關鍵信息進行特徵匹配！

2. 創建尺度空間（Scale Space）

識別給定圖像中最鮮明的特徵，同時忽略任何噪點。另外，確保特徵不依賴於比例。這些是關鍵概念，我們將逐一討論。

2.1（Gaussian Blur）

使用高斯模糊技術（Gaussian Blur）來降低圖像中的噪點。

因此，對於圖像中的每個像素，高斯模糊技術會基於其相鄰像素計算一個值。以下是應用高斯模糊之前和之後的圖像示例。如圖所示，紋理和次要細節將從圖像中刪除，並且僅保留諸如形狀和邊緣之類的相關信息：

高斯模糊成功地去除了圖像中的噪點，強調了圖像的重要特徵。現在，需要確保這些功能一定不能與比例相關。這意味著通過創建“比例空間”來在多個比例上搜索這些功能。

比例空間是從單個圖像生成的具有不同比例的圖像的集合。

因此，這些模糊圖像是針對多個比例創建的。要創建一組不同比例的新圖像，我們需要拍攝原始圖像並將其比例縮小一半。對於每個新圖像，我們都會創建模糊版本。

這是一個以更好的方式理解它的例子。我們有尺寸為（275，183）的原始圖像和尺寸為（138，92）的縮放圖像。對於這兩個圖像，將創建兩個模糊圖像：

你可能在想——需要縮放圖像多少次，並且每個縮放圖像需要創建多少後續的模糊圖像？理想的數量應為四個，並且對於每個均，模糊圖像的數目應為五個。

2.2 高斯的差異

到目前為止，我們已經創建了多個比例的圖像（通常由σ表示），並對每個圖像都使用了高斯模糊，以減少圖像中的噪點。接下來，嘗試使用稱為高斯差異（DoG）的技術來增強特徵。

高斯差異是一種特徵增強算法，涉及從原始圖像的另一個模糊版本中減去原始圖像的一個模糊版本。

DoG通過從前一個圖像中以相同比例尺減去每個圖像，為每個octave創建另一組圖像。這是如何實現DoG的直觀說明：

在右側，通過減去連續的高斯圖像生成了四個圖像。結果令人驚訝！

現在我們有了一組新的圖像，使用它來找到重要的關鍵點。

3. 關鍵信息本地化

創建圖像後，下一步就是從圖像中找到可用於特徵匹配的重要關鍵點。即找到圖像的局部最大值和最小值。分為兩個步驟：

1.找到局部最大值和最小值
2.刪除低對比度的關鍵點（關鍵點選擇）

3.1 局部極值

為了定位局部最大值和最小值，仔細檢查圖像中的每個像素，並將其與相鄰像素進行比較。

當我說“鄰近”時，它不僅包括該圖像的周圍像素（像素所在的像素），還包括八度中上一張和下一張圖像的九個像素。

這意味著將每個像素值與其他26個像素值進行比較，以確定是否為局部最大值/最小值。例如，在下圖中，我們從第一個八度獲得了三個圖像。將標記為x的像素與相鄰像素（綠色）進行比較，如果它是相鄰像素中最高或最低的像素，則將其選擇為關鍵點：

現在，我們有代表圖像的潛在關鍵點，並且尺度不變。對選中的關鍵點進行最後檢查，以確保這些是代表圖像的最準確的關鍵點。

3.2 關鍵點篩選

哇哦！到目前為止，我們已經成功地生成了尺度不變的關鍵點。但是這些關鍵點中的一些可能對噪聲沒有魯棒性。這就是為什麼需要進行最終檢查以確保我們擁有最準確的關鍵點來表示圖像特徵的原因。

因此，將消除對比度低或非常靠近邊緣的關鍵點。

為了處理低對比度關鍵點，將為每個關鍵點計算二階泰勒展開（second-order Taylor expansion）。如果結果值小於0.03（大小），則剔除該關鍵點。

那麼，如何處理其餘關鍵點呢？再次檢查以確定位置不佳的關鍵點。這些是具有高邊緣度但對少量噪點無魯棒性的關鍵點。使用二階Hessian矩陣來識別此類關鍵點。可以在這裡瞭解其背後的數學原理。

現在我們已經執行了對比測試和邊緣測試來剔除不穩定的關鍵點，現在為每個關鍵點分配一個方向值以使旋轉角度不變。

4. 定位任務

在此階段，為圖像提供了一組穩定的關鍵點。現在，為每個關鍵點指定一個方以使它們不變。再次將該步驟分為兩個較小的步驟：

1.計算幅度和方向
2.創建大小和方向的柱狀圖

4.1 計算幅度和方向

看下面的例圖：

假設要找到紅色像素值的大小和方向。為此，通過提取55和46與56和42之間的差值來計算x和y方向上的梯度。得出的分別是Gx = 9和Gy = 14。

一旦有了梯度，就可以使用以下公式找到幅度和方向：

Magnitude = √[(Gx)2+(Gy)2] = 16.64

Φ = atan(Gy / Gx) = atan(1.55) =57.17

大小表示像素的強度，方向表示像素的方向。

現在，假設我們具有這些大小和方向值，就可以創建柱狀圖。

4.2 創建大小和方向的柱狀圖

在x軸上，有一個角度值的區間，例如0-9、10 – 19、20-29，最大為360。由角度值為57，它會落在第6個區間中。第6個bin值與像素的大小成正比，即16.64。我們將對關鍵點周圍的所有像素執行此操作。

這樣就得到了下面的柱狀圖：

該柱狀圖將在某個點達到峰值。可以看出峰值的位置將是關鍵點的方向。此外，如果存在另一個顯著的峰值（在80 – 100％之間），則將生成另一個關鍵點，其大小和比例與用於生成直方圖的關鍵點相同。角度或方向將等於具有峰值的新bin。

在這一點上，關鍵點的數量可能會有所增加。

5. 關鍵信息描述符

這是SIFT的最後一步。到目前為止，我們有穩定的關鍵點——不變的比例以及旋轉角度。在本部分中，我們將使用相鄰像素，它們的方向和大小為該關鍵點生成一個唯一的指紋，稱為“描述符”。

另外，由於我們使用周圍的像素，因此描述符對於圖像的照度或亮度部分不變。

首先在關鍵點周圍採用16×16的鄰域。將該16×16區域進一步劃分為4×4子塊，對於這些子塊中的每一個小塊，使用幅度和方向生成柱狀圖。

在此階段，bin的大小增加，只佔用8個bin(不是36個)。每一個箭頭代表8個bin，箭頭的長度定義了幅度。因此，每個關鍵點總共有128個bin值。

例如：

import cv2

import matplotlib.pyplot as plt

%matplotlib inline

#reading image

img1 = cv2.imread('eiffel_2.jpeg')

gray1 = cv2.cvtColor(img1, cv2.COLOR_BGR2GRAY)

#keypoints

sift = cv2.xfeatures2d.SIFT_create()

keypoints_1, descriptors_1 = sift.detectAndCompute(img1,None)

img_1 = cv2.drawKeypoints(gray1,keypoints_1,img1)

plt.imshow(img_1)

6. 特徵匹配

現在，使用SIFT功能進行特徵匹配。為此，本人下載了兩張埃菲爾鐵塔的圖像，它們是從不同位置拍攝的。你可以嘗試使用任意兩個圖像。

如下為本人所用的兩張圖像：

import cv2

import matplotlib.pyplot as plt

%matplotlib inline

# read images

img1 = cv2.imread('eiffel_2.jpeg')

img2 = cv2.imread('eiffel_1.jpg')

img1 = cv2.cvtColor(img1, cv2.COLOR_BGR2GRAY)

figure, ax = plt.subplots(1, 2, figsize=(16, 8))

ax[0].imshow(img1, cmap='gray')

ax[1].imshow(img2, cmap='gray')

現在，對於這兩個圖像，生成SIFT特徵。首先，必須創建一個SIFT對象，然後使用函數detectAndCompute來獲取關鍵點。它將返回兩個值——關鍵點和描述符。

確定關鍵點並得到每個圖像中找到的關鍵點總數：

接下來，嘗試將圖片1中的特徵與圖片2中的特徵進行匹配。使用BFmatcher（強力匹配）模塊中的match（）功能。同樣，在兩個圖像中都匹配的要素之間畫線。可以使用OpenCV中的drawMatches函數來完成。

import cv2

import matplotlib.pyplot as plt

%matplotlib inline

# read images

img1 = cv2.imread('eiffel_2.jpeg')

img2 = cv2.imread('eiffel_1.jpg')

img1 = cv2.cvtColor(img1, cv2.COLOR_BGR2GRAY)

img2 = cv2.cvtColor(img2, cv2.COLOR_BGR2GRAY)

#sift

sift = cv2.xfeatures2d.SIFT_create()

keypoints_1, descriptors_1 = sift.detectAndCompute(img1,None)

keypoints_2, descriptors_2 = sift.detectAndCompute(img2,None)

#feature matching

bf = cv2.BFMatcher(cv2.NORM_L1, crossCheck=True)

matches = bf.match(descriptors_1,descriptors_2)

matches =sorted(matches, key=lambda x:x.distance)

img3 = cv2.drawMatches(img1, keypoints_1, img2, keypoints_2, matches[:50], img2, flags=2)

plt.imshow(img3),plt.show()
