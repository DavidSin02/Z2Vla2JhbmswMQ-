TCP請求發送和接收，如果接收端終止，發送端繼續發送會出什麼錯

2020-11-14
| 科技

我利用nodejs構造了一對TCP服務器和客戶端之後，

客戶端每隔5秒向服務器端發送一個TCP請求：

function doRunLoopDo() {log.info('timeout') pushNotificationTest(); setTimeout(() => { doRunLoopDo(); }, 5000);}function pushNotificationTest() { let pushMessage = { command: 'newElement', method: 'broadcastToGroup', data: {group: "gui" }, id: Math.floor(Math.random() * 1e100), dateTime: Date.now(), value: Math.floor(Math.random() * 1000) } TCPClient.sendMessage(pushMessage);}

然後我終止服務器端，

在TCP客戶端我加上了如下的錯誤處理分支：

async function startClient(ip = configJson.TCP.client.ip, port = configJson.TCP.client.port) {	client = new net.Socket();	await client.connect(port, ip, () => { console.log(" in tcpclient.js connect callback!!!");	}); client.on('error', (error) => { log.info('TCP Client: Connection error - destroy Connection '+error); })
