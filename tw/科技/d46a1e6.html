計算機網絡複習知識點

基本知識點

1.OSI參考模型（七層體系結構）

物理層 - 數據鏈路層 - 網絡層 - 運輸層 - 會話層 - 表示層 - 應用層

2.TCP/IP體系結構（四層體系結構）

網絡接口層 - 網際層IP - 運輸層(TCP/UDP) - 應用層(FTP/SMTP/HTTP/TELNET...)

物理層

1.奈氏準則

理想低通信道最高碼元傳輸速率 = 2W Baud（即每赫茲理想低通信道最高碼元傳輸速率每秒2個碼元）

2.香農定理

信道的極限傳輸速率C = Wlog2(1+S/N)，W為帶寬，S為平均功率，N為高斯噪聲功率，S/N也稱為信噪比

3.物理層之下的傳輸媒體

雙絞線 同軸電纜 光纜

4.調製器和解調器

調製器：基帶數字信號波形轉為模擬信號的波形

解調器：將經過調製器變換的模擬信號恢復成原來的數字信號

5.採樣定理

只要採樣頻率不低於電話信號最高頻率的2倍，就可以從採樣脈衝信號無失真地恢復出原來的電話信號。

6.信道複用技術

頻分複用FDM 時分複用TDM

碼分多址CDMA：每個用戶可以在同樣的時間使用同樣的頻帶進行通信。

7.單工 半雙工 全雙工

8.基帶和寬帶

基帶信號指將數字0,1直接用兩種不同電壓來表示，然後運到線路上傳輸。寬帶信號指將基帶信號調製後形成頻分複用模擬信號。

9.基帶信號 - 曼徹斯特編碼信號 - 差分曼徹斯特編碼

曼徹斯特編碼：一個碼元拆2個，碼元1前高後低，碼元0相反

差分曼徹斯特編碼：一個碼元拆2個，碼元1前半個與上一個碼元的後半個一樣，碼元0相反。

數據鏈路層

1.鏈路和數據鏈路（除了一條物理鏈路外還需要加一些通信協議控制數據的傳輸）

2.數據鏈路層3個基本問題 幀定界 透明傳輸 差錯檢測

3.幀定界

幀的數據部分的上限 <= MTU（最大傳輸單元），SOH（0x01）和EOT(0x04)分別指示幀的開始和結束。

4.透明傳輸（數據的比特組合必須是不受限制的）

方法1：字節插入

SOHEOTESCESC xESC yESC z

方法2：比特插入

5.差錯檢測

誤碼率：傳輸錯誤的比特與傳輸總比特數的比率

CRC(Cyclic Redundancy Check)：循環冗餘檢驗

CRC是檢錯方法並不能糾錯，FCS（Frame Check Sequence）是冗餘碼。

計算冗餘碼（餘數R）的方法：先補0（n個）再對生成多項式取模。

CRC只能表示以接近1的概率認為它沒有差錯。但不能做到可靠傳輸。可靠傳輸還需要確認和重傳機制。

生成多項式P(X)：CRC-16，CRC-CCITT，CRC-32

6.停止等待協議：解決死鎖使用超時計時器。對重複幀的處理：丟棄，再次發送確認幀。向上面的網絡層提供了可靠傳輸。是一種ARQ的差錯檢測方法。

7.發送窗口大於1的ARQ：連續ARQ和選擇重傳ARQ

連續ARQ又稱Go-back-N ARQ，意思是出現差錯必須重傳時，要回走n個幀，即使是已經傳送正確的數據幀也要重傳。（僅因這些正確幀那面有一個錯誤）

滑動窗口：（解決連續ARQ中一旦某個幀出錯要重傳之後許多幀，浪費時間，增大開銷，因此我們對發出去但未被確認的幀的數目加以限制）包括髮送窗口和接收窗口。為了減少開銷，連續ARQ還執行捎帶確認的機制。發送窗口Wt<= 2^n-1(n代表用幾比特編號)，接收窗口Wr=1。

選擇重傳ARQ：需要緩存空間，一般不使用。

接收窗口Wr<=2^n/2 當接收窗口Wr為最大值時 Wt=Wr=2^n/2

這種收發兩端窗口按規律向前移動，也稱為滑動窗口協議。（連續ARQ，選擇重傳ARQ，停止等待都是）

局域網

2.以太網

以太網提供的服務是不可靠的交付，即盡最大努力交付。因為採用無連接，數據幀不編號，也不要求對方確認。對有差錯的幀的重傳有高層來決定。

CSMA/CD協議(Carrier Sense Multiple Access with Collision Detection)：載波監聽多點接入/碰撞檢測

用於協調同一時間只能允許一臺計算機發送信息，否則就會相互干擾的問題。

電磁波在1km電纜的傳播時延約5us。使用CSMA/CD協議，一個站不能同時發送和接收，因此不能實現全雙工。

爭用期：2τ 51.2us 最短有效幀長為64字節

截斷二進制指數退避算法：基本退避時間為2τ k=min{重傳次數，10} r=random(0~2^k-1) 重傳所需時延為r倍的基本退避時間

5.IP地址和硬件地址

網絡層的IP數據報交給數據鏈路層被封裝成MAC幀，MAC幀的源地址和目的地址變為硬件地址

6.ARP（IP地址 -> MAC地址）

使用IP地址並不能直接用來通信，因為IP地址只是主機在抽象的網絡層中的地址。最終要傳到數據鏈路層封裝成MAC幀才能發送到實際的網絡。

因此不管使用什麼協議最終需要的還是硬件地址。

每個主機擁有一個ARP高速緩存（存放所在局域網內主機和路由器的IP地址到硬件地址的映射表）

舉例：A發送B

(1).A在ARP Cache中查到B的MAC地址，寫入MAC幀發往此B

(2).沒查到，A向本局域網廣播ARP請求分組，內容包括自己的地址映射和B的IP地址

(3).B發送ARP響應分組並寫入A的映射（單播的方式）

ARP Cache映射項目具有一個生存時間。

固定部分 20字節

標識：一個計數器，用來產生數據報的標識。

標誌字段：3位 MF=1後面還有分片 DF=0允許分片

IP分片 MF DF 片偏移

協議：佔1個字節

首部檢驗和：反碼算數運算，不採用CRC

8.劃分子網：IP地址：網絡號 - <子網號 - 主機號>（子網號是在兩級分類編址中的主機號中劃出的）

子網掩碼：網絡號子網號置1(相當於CIDR中的網絡前綴)主機號置0，這樣子網掩碼與IP地址相與即可算出網絡地址。

劃分子網下的路由器轉發算法：

（1）從收到的數據報的首部提取目的IP地址D

（2）先判斷是否為直接交付。對與路由器相鄰的子網進行檢查。用各網絡子網掩碼與D相與，查看是否匹配。若是，直接交付（D

轉化為物理地址，數據報封裝成幀發送出去。）轉發結束。若否，為間接交付，執行（3）

（3）若路由表中有目的地址D的特定主機路由，則將數據報傳給該路由，否則，執行（4）

（4）對路由表的每一行（目的網絡地址、子網掩碼、下一跳地址），將其中的子網掩碼與D相與，結果為N。若N與該行目的網絡地址匹配，則

將該數據報傳給該行指明的下一跳路由器，否則，執行（5）

（5）若路由表中有一個默認路由，則傳給默認路由，否則，執行（6）

（6）報告分組轉發出錯

9.無分類編址CIDR（CLassless Inter-Domain Routing 無分類域間路由選擇）

無分類的兩級編址：格式 <網絡前綴 - 主機號>

路由聚合也稱為構造超網因為一個CIDR地址塊可以表示很多地址。

最長前綴匹配：從匹配結果中選擇具有最長網絡前綴的路由

10.ICMP（Internet Control Message Protocol）因特網控制報文協議

包括ICMP差錯報文和ICMP詢問報文

ICMP差錯報文的類型：終點不可達 源點抑制 時間超過 參數問題 改變路由

ICMP詢問報文的類型：回送請求和回答 時間戳請求和回答 掩碼地址請求和回答 路由器詢問或通告

應用層服務Ping使用了ICMP回送請求和回答報文，是應用層直接使用網絡層ICMP的例子。沒有通過運輸層的TCP或UDP

11.路由選擇協議

自治系統AS：自治系統有權自主地決定在本系統類採用何種路由選擇協議。

路由選擇協議分為兩類：內部網關協議IGP（Interior Gateway Protocol）主要有RIP和OSPF

外部網關協議EGP（External Gateway Protocol）主要是BGP-4

RIP（Routing Information Protocol）：分佈式的基於距離向量的路由選擇協議

RIP的距離也稱為跳數，RIP允許一條路徑上最多包含15個路由器，因此距離為16相當於不可達。

RIP僅和相鄰路由交換信息，交換的是當前路由器知道的全部信息，即自己的路由表。按固定的時間間隔交換。

RIP協議是收斂的，所謂收斂指自治系統中的所有結點都得到正確的路由選擇信息的過程。最終使得每一個路由器到

每一個目的網絡的路由都是最短的。

優點：實習Ian簡單，開銷簡單。缺點：“好消息傳得快，壞消息傳得慢”。

RIP使用運輸層UDP（520端口）進行傳送，因此RIP的位置在應用層。但轉發用戶數據報的過程在網絡層完成的。

OSPF（Open Shortest Path First開放最短路徑優先）：分佈式的基於鏈路狀態協議

1.使用洪範法向本自治系統內所有路由器發送信息。

2.發送的信息是與本路由器相鄰的所有路由器的鏈路狀態。

3.只有鏈路狀態發生變化時，路由器才使用洪泛法向所有路由器發送信息（而RIP是不管網絡拓撲有無變化，路由器間都要定期交換路由表信息）

由於頻繁交換鏈路狀態信息，因此所有路由器最終都能建立一個鏈路狀態數據庫，這個數據庫實際上就是全網的拓撲結構圖，在全網範圍內是一致的。

OSPF更新過程收斂的快。

OSPF不用UDP而直接用IP數據報發送。可見OSPF的位置在網絡層。

負載均衡：在代價相同的路徑上分配通信量，這叫多路徑間的負載均衡。

外部網關協議BGP（Border Gateway Protocol:邊界網關協議）：基於路徑向量路由選擇協議。

目標：力求選擇一條比較好的路由。並非尋找最佳。

BGP發言人需要建立TCP連接（端口179），然後在此連接上交換報文以建立BGP會話。

BGP發言人需要同時運行BGP協議和內部網關協議

BGP解決了基於距離向量路由選擇算法中的“好消息傳得快，壞消息傳得慢”這一問題。

BGP-4一共4種報文：打開包文 更新報文 保活報文 通知報文

運輸層

運輸層提供應用進程間的邏輯通信。它使應用進程看見的就好像是在兩個運輸層實體間一條端到端的邏輯通信信道。

當運輸層採用TCP時，儘管下面的網絡是不可靠的，但這種邏輯通信信道相當於一條全雙工的可靠信道。可以做到報文的無差錯、

按序、無丟失、無重複。

單單面向連接只是可靠的必要條件，不充分。還需要其他措施，如確認重傳，按序接收，無丟失無重複。

UDP重要

UDP的優點：

1.發送之前無需建立連接，減小了開銷和發送數據的時延

2.UDP不使用擁塞控制，不使用可靠交付，因此主機不需要維護複雜的參數表、連接狀態表

3.UDP用戶數據報只有8個字節的首部開銷，而TCP要20字節。

4.由於沒有擁塞控制，因此網絡出現擁塞不會使源主機的發送速率降低（IP電話等實時應用要求源主機以恆定的速率發送數據）

UDP的過程：

1.服務器進程運行著，等待TFTP客戶進程的服務請求。客戶端TFTP進程啟動時，向操作系統申請一個臨時端口號，然後操作系統為該進程創建2個隊列，

入隊列和出隊列。只要進程在執行，2個隊列一直存在。

2.客戶進程將報文發送到出隊列中。UDP按報文在隊列的先後順序發送。在傳送到IP層前給報文加上UDP首部，其中目的端口後為69。然後發給IP層。

出隊列若溢出，則操作系統通知應用層TFTP客戶進程暫停發送。

3.客戶端收到來自IP層的報文時，UDP檢查報文中目的端口號是否正確，若正確，放入入隊列隊尾，客戶進程按先後順序一一取走。若不正確，UDP丟棄該報文，並請ICMP發送”端口不可達“差錯報文給服務器端。入隊列可能會溢出，若溢出，UDP丟棄該報文，不通知對方。

服務器端類似。

UDP首部：源端口 - 目的端口 - 長度 - 檢驗和，每個字段22字節。

IP數據報檢驗和只檢驗IP數據報的首部，而UDP的檢驗和將首部和數據部分一起都檢驗。

TCP報文段是面向字節的數據流。

窗口：A根據TCP緩存空間的大小確定自己的接收窗口大小。A發送給B的窗口字段寫入該值。作為B的發送窗口的上限。

接收端窗口rwnd（receiver window）：接收端根據目前接收緩存大小設置的窗口值，是來自接收端的流量控制

擁塞窗口cwnd（congestion window）：是發送端根據自己估計的網絡擁塞程度設置的窗口值，是來自發送端的流量控制

發送端的窗口=Min(rwnd, cwnd)

慢開始算法：cwnd設置為1個MSS，每收到一個確認，將cwnd+1，逐步增大cwnd，使分組注入網絡的速率更加合理。

慢開始門限：ssthresh，當cwnd < ssthresh,執行慢開始算法；cwnd > ssthresh，改用擁塞避免算法。 cwnd = ssthresh時，都可以。

慢開始算法擁塞窗口增長速率緩慢很多。（加法增大）

每發送一個報文段，就對這個報文段設置一次計時器。新的重傳時間=γ*舊的重傳時間。

TCP連接建立和釋放的過程

SYN置1和FIN的報文段要消耗一個序號。

客戶端連接狀態變遷：CLOSED -> 主動打開,發送SYN=1 -> SYN_SENT -> 收到服務器的SYN=1和ACK時,發送三次握手的最後一個ACK

-> ESTABLISHED -> 數據傳送 -> 主動關閉 -> 發送FIN=1,等待確認ACK的到達 -> FIN_WAIT_1 -> 收到確認ACK後時，一向連接關閉 -> FIN_WAIT_2

->收到服務器發送的FIN=1報文，響應，發送四次揮手的的最後一個確認ACK -> 這時另一條連接也關閉了，進入TIME_WAIT狀態

-> 經過2倍報文壽命，TCP刪除連接記錄 -> 回到CLOSED狀態

服務器端連接狀態變遷：CLOSED -> 被動打開 -> LISTEN -> 收到SYN=1的報文，發送SYN=1和確認ACK -> 進入SYN_RCVD -> 收到三次握手

的最後一個確認ACK -> ESTABLISHED -> 數據傳送 -> 數據傳送完畢，收到FIN=1 -> 發送確認ACK並進入CLOSED_WAIT -> 發送FIN=1給客戶端 -> LAST_ACK

-> 收到客戶端四次揮手的最後一個確認ACK -> 刪除連接記錄 -> 回到CLOSED狀態
