Bash技巧：介紹重定向輸入輸出等十種重定向操作符的用法

2020-10-29
| 科技


在 Linux bash shell 中，執行一個命令之前，可以使用重定向操作符對該命令的輸入、輸出進行重定向，從其他文件獲取輸入，把命令輸出寫到其他文件。
具體說明可以查看 man bash 的 REDIRECTION 小節。下面舉例說明十種重定向操作符的具體用法。

重定向的概念

在 REDIRECTION 小節對重定向概念說明如下：

Before a command is executed, its input and output may be redirected using a special notation interpreted by the shell.


Redirection allows commands' file handles to be duplicated, opened, closed, made to refer to different files, and can change the files the command reads from and writes to.


If the file descriptor number is omitted, and the first character of the redirection operator is <, the redirection refers to the standard input (file descriptor 0).


If the first character of the redirection operator is >, the redirection refers to the standard output (file descriptor 1).


Redirections using file descriptors greater than 9 should be used with care, as they may conflict with file descriptors the shell uses internally.

即，在執行一個命令之前，可以使用重定向操作符對該命令的輸入、輸出進行重定向。
從其他文件獲取輸入，把命令輸出寫到其他文件。

在 Linux 中，打開文件後，會得到一個文件描述符（file descriptor）。
文件描述符的有效值是大於或等於 0 的整數，可以使用這個整數值來讀寫對應的文件。

重定向可以修改文件描述符對應的文件，從而在讀寫同一個文件描述符時，可以讀寫到其他文件。

重定向輸入會把指定的文件描述符關聯到以讀模式打開的文件。
那麼讀取指定文件描述符，就會讀取文件內容。

重定向輸出會把指定的文件描述符關聯到以寫模式打開的文件。
那麼寫入內容到指定文件描述符，就會寫入文件。

即，重定向前後會關聯兩個文件。
在重定向之前，指定的文件描述符對應一個已經打開的文件。
重定向之後，該文件描述符會對應另一個文件、或者對應另一個文件描述符。
而另一個文件描述符也是對應另一個已經打開的文件。

當下面文件名用於重定向時，bash 會進行特殊處理：

  • /dev/fd/fd：如果 fd 是一個已經打開的文件描述符整數值，則 /dev/fd/fd 文件會複製到文件描述符 fd
  • /dev/stdin：複製到文件描述符 0，也可以寫為 /dev/fd/0。文件描述符 0 一般對應標準輸入
  • /dev/stdout：複製到文件描述符 1，也可以寫為 /dev/fd/1。文件描述符 1 一般對應標準輸入
  • /dev/stderr：複製到文件描述符 2，也可以寫為 /dev/fd/2。文件描述符 2 一般對應標準錯誤輸出

基於上面說明，可以使用 /dev/fd/5 來複制到文件描述符 5。
文件描述符 5 必須對應一個打開的文件，否則會報錯。
也就是要有某個已打開文件的文件描述符是 5，/dev/fd/5 才能複製到文件描述符 5。

當使用大於 9 的文件描述符時，要特別小心。
shell 自身可能也使用了某個大於 9 的文件描述符，會導致衝突。

重定向的先後順序

在使用重定向，要注意重定向的先後順序，這是很重要的點，REDIRECTION 小節的說明如下：

Redirections are processed in the order they appear, from left to right.
Note that the order of redirection is significant. For example, the command

    ls > dirlist 2>&1

directs both standard output and standard error to the file dirlist, while the command

    ls 2>&1 > dirlist

directs only the standard output to file dirlist, because the standard error was duplicated from the standard output before the standard output was redirected to dirlist.

即，當指定了多個重定向操作時，按照重定向出現的順序，從左到右處理。

對 ls > dirlist 2>&1 命令來說，第一個 > 表示重定向 ls 命令的標準輸出到 dirlist 文件。
第二個 2>&1 表示複製標準錯誤輸出到標準輸出。
由於標準輸出已經被重定向到 dirlist 文件，所以 dirlist 文件會包含 ls 命令的標準輸出和標準錯誤輸出。

而 ls 2>&1 > dirlist 命令只重定向標準輸出到 dirlist 文件，標準錯誤輸出只會打印到終端。具體說明如下：

  • 第一個 2>&1 表示複製標準錯誤輸出的文件描述符 2 到標準輸出的文件描述符 1。
    實際上是把標準錯誤輸出寫入到標準輸出對應的文件。
    此時標準輸出還沒有重定向，會關聯到終端，所以標準錯誤輸出會寫入到終端。
  • 第二個 > 表示把標準輸出重定向到 dirlist 文件，會把文件描述符 1 指向 dirlist 文件。
    這不會影響文件描述符 2，標準錯誤輸出的文件描述符 2 還是指向終端。
    所以， 標準錯誤輸出沒有重定向到 dirlist 文件。

具體舉例測試如下：

$ ls not_exist 2>&1 > dirlist

ls: cannot access not_exist: No such file or directory

$ ls not_exist > dirlist 2>&1

$ cat dirlist

ls: cannot access not_exist: No such file or directory

在當前目錄下，沒有 not_exist 文件。那麼 ls not_exist 會報錯，提示找不到該文件。

如前面說明，ls not_exist 2>&1 > dirlist 命令不能把標準錯誤輸出重定向到 dirlist 文件，所以報錯信息直接打印在終端上。

ls not_exist > dirlist 2>&1 命令可以把標準錯誤輸出重定向到 dirlist 文件，終端上沒有看到報錯信息。
用 cat dirlist 命令輸出 dirlist 文件內容，可以看到報錯信息寫入到了該文件。

使用 exec 命令對當前 shell 進行重定向

在 bash 中執行命令，會啟動子 shell 來執行。此時的重定向隻影響執行命令的子 shell，不影響當前 shell。
如果要對當前 shell 進行重定向，可以使用 bash 的 exec 內置命令。

查看 man bash 裡面對 exec 命令的說明如下：

Note that the exec builtin command can make redirections take effect in the current shell.


exec [-cl] [-a name] [command [arguments]]
If command is specified, it replaces the shell. No new process is created. The arguments become the arguments to command.
If command is not specified, any redirections take effect in the current shell, and the return status is 0.
If there is a redirection error, the return status is 1.

即，當 exec 沒有指定要執行的命令時，可以直接在當前 shell 進行重定向。
例如，把當前的 shell 標準輸入重定向到文件，不再從鍵盤讀取輸入。

下面具體舉例說明如何使用 exec 命令來進行重定向。

  • 重定向當前 shell 的標準輸出到文件（如果終端自身不支持保存log功能，可以使用這個方法來把命令輸出都保存到文件）：

$ exec 1>output.txt

$ ls

$ cat output.txt

cat: output.txt: input file is output file

$ exec 1>/dev/stdin

$ cat output.txt

testfile

在這個例子中，exec 1>output.txt 命令把當前 shell 的標準輸出重定向到 output.txt 文件。
那麼標準輸出不再關聯到終端。
執行 ls 命令，終端上看不到任何輸出，ls 命令的輸出結果被寫入到 output.txt 文件。

此時，無法在當前 shell 上輸出 output.txt 文件的內容。
執行 cat output.txt 命令，提示輸入文件是輸出文件，會形成循環，不會在終端上輸出 output.txt 文件內容。

此時要查看 output.txt 文件的內容，可以在另外的終端上查看，或者取消重定向當前 shell 的標準輸出到 output.txt 文件。
exec 1>/dev/stdin 命令把當前 shell 的標準輸出重定向到 /dev/stdin 文件，會關聯到終端。
執行 cat output.txt 命令，就能輸出 output.txt 文件內容到終端上。

注意：這裡有一個比較古怪的地方，需要執行 exec 1>/dev/stdin 命令把對應標準輸入的 /dev/stdin 文件關聯到文件描述符 1，才能重新輸出到終端上。
執行 exec 1>/dev/stdout 命令並不能重定向標準輸出到終端上。

在重定向中使用 /dev/stdout 文件，會複製到文件描述符 1。
而文件描述符 1 之前已經重定向到 output.txt 文件。
所以 exec 1>/dev/stdout 命令還是讓文件描述符 1 關聯到 output.txt 文件，並不是關聯到終端。

在重定向中使用 /dev/stdin 文件，會複製到文件描述符 0。
而文件描述符 0 沒有重定向，關聯到終端。
所以 exec 1>/dev/stdin 命令會讓文件描述符 1 關聯到終端。

  • 重定向當前 shell 的標準輸入到文件：

$ cat input.txt

date

sleep 2

$ exec 0<input.txt

$ date

2019年 11月 28日 星期四 13:14:11 CST

可以看到，用 cat input.txt 命令打印 input.txt 文件內容。
這個文件裡面有兩行，分別是 date 命令和 sleep 2 命令。
這個 sleep 命令用於延遲終端退出，否則執行後面命令，終端會很快關閉，終端窗口一閃而過。

執行 exec 0<input.txt 命令，會把當前 shell 的標準輸入重定向到 input.txt 文件。
則 shell 會開始逐行讀取該文件內容，並把每行內容當作命令來執行。
所以後面看到執行了 date 命令，這個命令並不是通過鍵盤手動輸入。

自動執行 date 命令後，會再執行 sleep 2 命令，等秒 2 秒，接著就會退出當前 shell。

這是因為 bash shell 遇到 EOF 就會退出。
例如打開一個 shell，輸入 CTRL-D，就會退出 shell。
重定向標準輸入到 input.txt 文件後，當前 shell 讀取完該文件內容，就會遇到 EOF，所以會退出 shell。

重定向輸入

查看重定向輸入（Redirecting Input）的說明如下：

Redirection of input causes the file whose name results from the expansion of word to be opened for reading on file descriptor n, or the standard input (file descriptor 0) if n is not specified.
The general format for redirecting input is:

    [n]<word

即，[n]<word 用讀模式打開 word 文件，並關聯到文件描述符 n 上。
如果沒有提供文件描述符 n，默認關聯到文件描述符 0 上，也就是標準輸入。
重定向輸入後，從文件描述符 n 讀取內容，會讀取到所關聯的 word 文件內容。

例如，重定向標準輸入後，word 文件關聯到文件描述符 0。
被執行的命令讀取標準輸入，也就是讀取文件描述符 0，從而讀取到 word 文件的內容，不需要從鍵盤進行輸入。

重定向輸入的例子說明如下：

$ cat < input.txt

date

sleep 2

$ exec 3< input.txt

$ cat /dev/fd/3

date

sleep 2

$ read -u 3 line

$ echo $line

date

在 cat < input.txt 命令中，執行 cat 命令且沒有提供文件名參數，那麼 cat 命令默認讀取標準輸入。
< input.txt 表示把 input.txt 文件關聯到文件描述符 0，也就是關聯到被執行命令的標準輸入。
cat 命令讀取標準輸入時，讀取的就是 input.txt 文件的內容。
這隻影響被執行命令的標準輸入，不影響當前 shell 的標準輸入。

執行 exec 3< input.txt 命令把 input.txt 文件關聯到文件描述符 3 的輸入。
之後，讀取文件描述符 3，讀取的就是 input.txt 文件的內容。
用 cat /dev/fd/3 命令讀取文件描述符 3，獲取到的內容確實和 input.txt 文件內容一致。

read -u 3 line 命令使用 -u 選項指定從文件描述符 3 讀取一行，保存到 line 變量。
打印 line 變量值，讀取到的內容就是 input.txt 文件的第一行。

重定向輸出

查看重定向輸出（Redirecting Output）的說明如下：

Redirection of output causes the file whose name results from the expansion of word to be opened for writing on file descriptor n, or the standard output (file descriptor 1) if n is not specified.
If the file does not exist it is created; if it does exist it is truncated to zero size.
The general format for redirecting output is:

    [n]>word

即，[n]>word 用寫模式打開 word 文件，並關聯到文件描述符 n 上。
如果沒有提供文件描述符 n，默認關聯到文件描述符 1 上，也就是標準輸出。
重定向輸出後，往文件描述符 n 寫入內容，會寫入到所關聯的 word 文件。

如果 word 文件不存在，會創建該文件。
如果 word 文件已經存在，則把它的文件大小截斷成 0，會丟棄原有的內容。

重定向標準輸出是重定向最常見的用法之一。
重定向輸出的例子說明如下：

$ echo "redirection output" > output.txt

$ cat output.txt

redirection output

$ exec 5> output.txt

$ cat input.txt > /dev/fd/5

$ cat output.txt

date

sleep 2

echo "redirection output" > output.txt 命令把 echo 命令的標準輸出重定向到 output.txt 文件。
那麼 echo 命令的輸出結果就是寫入到該文件。

用 cat output.txt 命令查看 output.txt 文件內容，可以看到重定向標準輸出寫入的字符串。

exec 5> output.txt 命令把 output.txt 文件關聯到文件描述符 5 的輸出。
之後，往文件描述符 5 寫入內容，就是寫入到 output.txt 文件。

cat input.txt > /dev/fd/5 命令把標準輸出重定向到文件描述符 5。
那麼所輸出的 input.txt 文件內容會被寫入到文件描述符 5。

用 cat output.txt 命令輸出 output.txt 文件內容，已經不再是 "redirection output" 字符串。
而是跟 input.txt 文件的內容一致。

追加重定向輸出

查看追加重定向輸出（Appending Redirected Output）的說明如下：

Redirection of output in this fashion causes the file whose name results from the expansion of word to be opened for appending on file descriptor n, or the standard output (file descriptor 1) if n is not specified.
If the file does not exist it is created.
The general format for appending output is:

    [n]>>word

即，[n]>>word 命令用追加寫入模式打開 word 文件，並關聯到文件描述符 n 上。
如果沒有提供文件描述符 n，默認關聯到文件描述符 1 上，也就是標準輸出。
重定向輸出後，往文件描述符 n 寫入內容，會追加寫入到所關聯的 word 文件。

如果 word 文件不存在，會創建該文件。
如果 word 文件已經存在，會把內容追加寫入到文件末尾，不會丟棄原有的內容。

追加重定向輸出的例子說明如下：

$ cat output.txt

date

sleep 2

$ echo "append" >> output.txt

$ cat output.txt

date

sleep 2

append

先用 cat output.txt 命令查看 output.txt 文件的內容。
然後使用 echo "append" >> output.txt 命令追加重定向標準輸出到 output.txt 文件。

再次查看 output.txt 文件，原有的內容還在，並在文件末尾追加了寫入的 "append" 字符串。

當重定向的輸出文件不存在時，> 和 >> 都會新建該文件。
當輸出文件已經存在時，> 會清空該文件的內容，然後再寫入新的內容（如果有的話）。
而 >> 會將新寫入的內容追加到該文件的末尾，文件原來的內容還會保留。

同時重定向標準輸出和標準錯誤輸出

查看同時重定向標準輸出和標準錯誤輸出（Redirecting Standard Output and Standard Error）的說明如下：

This construct allows both the standard output (file descriptor 1) and the standard error output (file descriptor 2) to be redirected to the file whose name is the expansion of word.
There are two formats for redirecting standard output and standard error:

    &>word

and

    >&word

Of the two forms, the first is preferred. This is semantically equivalent to

    >word 2>&1

When using the second form, word may not expand to a number or -.
If it does, other redirection operators apply (see Duplicating File Descriptors below) for compatibility reasons.

即，可以使用 &>word、或者 >&word 來同時重定向標準輸出和標準錯誤輸出。

建議採用 &>word 這個寫法。
在語義上，這個寫法等同於 >word 2>&1。
2>&1 這個寫法表示複製文件描述符。後面會具體說明。

如果使用 >&word 這種形式來同時重定向標準輸出和標準錯誤輸出，那麼 word 文件名不能是數字，也不能是連字符 -。
因為這種寫法是複製文件描述符，而複製文件描述符時，數字和 - 具有特殊含義，不會當成文件名處理。

當重定向標準輸出到文件後，如果命令執行出錯，錯誤信息會打印到標準錯誤輸出，會在終端上看到這些打印。
如果確實需要把標準錯誤輸出也重定向到文件，就可以使用 &>word 來同時重定向標準輸出和標準錯誤輸出。這裡不再舉例。

同時追加重定向標準輸出和標準錯誤輸出

查看同時追加重定向標準輸出和標準錯誤輸出（Appending Standard Output and Standard Error）的說明如下：

This construct allows both the standard output (file descriptor 1) and the standard error output (file descriptor 2) to the appended to the file whose name is the expansion of word.
The format for appending standard output and standard error is:

    &>>word

This is semantically equivalent to

    >>word 2>&1

即，可以使用 &>>word 來同時追加重定向標準輸出和標準錯誤輸出。
這個命令在語義上，等同於 >>word 2>&1。

Here Documents

前面說明的重定向輸入需要提供文件名，會有一個單獨的文件。
編寫 shell 腳本時，如果需要進行重定向輸入，這個腳本就會依賴一個外部的文件，增加耦合。

我們可以使用 Here Documents 機制來重定向標準輸入為指定的字符串，不需要提供外部文件名。
查看 Here Documents 的說明如下：

This type of redirection instructs the shell to read input from the current source until a line containing only delimiter (with no trailing blanks) is seen.
All of the lines read up to that point are then used as the standard input for a command. The format of here-documents is:


<<[-]word

here-document

delimiter


No parameter expansion, command substitution, arithmetic expansion, or pathname expansion is performed on word.
If any characters in word are quoted, the delimiter is the result of quote removal on word, and the lines in the here-document are not expanded.
If word is unquoted, all lines of the here-document are subjected to parameter expansion, command substitution, and arithmetic expansion.
In the latter case, the character sequence \<newline> is ignored, and \ must be used to quote the characters , $, and `.
If the redirection operator is <<-, then all leading tab characters are stripped from input lines and the line containing delimiter.
This allows here-documents within shell scripts to be indented in a natural fashion.

即，可以使用 here-documents 格式來重定向標準輸入：

  • word 指定一個字符串。如果字符串帶有空格，需要用引號括起來。
  • here-document 內容會被重定向到標準輸入。
    當某一行內容完全等於 word 指定的字符串時（行首、行末的空白字符也會用來比較），則停止重定向。
  • delimiter 對應 word 指定的字符串，會去掉 word 中的引號。
    所給的 delimiter 不會被重定向到標準輸入。

這裡的 word 字符串不會進行參數擴展、命令替換、算法括號、或路徑名擴展，意味著不能用 $var 的形式來獲取 var 變量值。
如果 word 寫為 $var，對應的 delimiter 字符串就是 $var。

當 word 字符串的任意字符被引號括起來時，對應的 delimiter 字符串不會包含這些引號。
而且中間的 here-document 字符串不會進行擴展。
例如，不能在 here-document 中用 $var 的形式來獲取 var 變量值。

當 word 字符串中沒有引號時，中間的 here-document 字符串可以進行參數擴展、命令替換、算法括號，不進行路徑名擴展。

對 <<word 這個寫法來說，中間 here-document 字符串的任意字符都會被保留，包括行首、行末的空白字符。
而寫為 <<-word 時，中間 here-document 字符串行首的 tab 字符會被去掉，行首的空格還是會保留。

下面舉例說明各種情況的 here-documents 格式寫法。

  • 在 here-document 中不進行擴展

假設有一個 testhere.sh 腳本文件，內容如下：

（不好意思，近期網頁版文章的代碼塊排版錯亂，後臺諮詢確認網頁版不支持。下面用四個 ‘----’ 代替四個空格來進行縮進和隔行顯示。後面的代碼塊會類似處理。如果需要複製代碼到本地驗證，麻煩以四個 ‘----’為單位，替換成四個空格。非常抱歉。）

#!/bin/bash

cat -A <<"The end"

This is a here-document.

----The end

The end

$((3+7))

The end

該腳本執行 cat -A 命令，-A 選項會在輸出的行末打印一個 $ 字符，以便看到行末結尾。
這裡用 <<"The end" 指定 here-documents 類型的重定向輸入。
cat 命令會從 here-documents 獲取標準輸入，直到讀取 "The end" 字符串為止。

在 << 後面的 "The end" 字符串帶有空格，要用引號括起來。
如果不加引號，會執行報錯。

在 $((3+7)) 上一行的 "The end " 字符串後面有兩個空格。
在測試的時候要注意在行末加上空格。

執行 testhere.sh 腳本，打印結果如下：

$ ./testhere.sh

This is a here-document.$

----The end$

The end $

$((3+7))$

可以看到，here-documents 的第二行 " The end"，在行首帶有空格。
這一行不完全匹配 "The end"，沒有停止重定向。

第三行內容是 "The end "，在行末有兩個空格，也是不完全匹配 "The end"，沒有停止重定向。

第四行內容是 "$((3+7))"，這是一個算術擴展表達式。
但是這裡的 word 參數值是 "The end"，帶有引號，不會進行算術擴展，會原樣輸出 "$((3+7))" 字符串。

第五行內容是 "The end"，完全匹配 "The end"，停止重定向。

停止重定向後，cat -A 命令會打印從標準輸入讀取到的所有內容，並在行末打印 $ 字符。

最後一行的 "The end" 字符串沒有被重定向輸入到 cat 命令。

  • 在 here-document 中進行擴展

前面提到，如果 <<word 的 word 沒有包含引號，則 here-document 中的內容可以進行參數擴展、命令替換、算術擴展，不進行路徑名擴展。
修改 testhere.sh 腳本為下面的內容，測試這種情況：

#!/bin/bash

number="1234567"

cat <<end

number = $number

$(date)

$((3+7))

test*

end

這裡的停止重定向字符串是 end，沒有加引號。
裡面的 here-document 用 $number 獲取 number 變量值。
用 $(date) 執行 date 命令並獲取改命令的輸出，也就是命令替換。
用 $((3+7)) 進行算術擴展。

執行修改後的 testhere.sh 腳本，打印結果如下：

$ ./testhere.sh

number = 1234567

2019年 11月 28日 星期四 16:38:36 CST

10

test*

可以看到，打印結果裡面確實獲取到了 number 變量值，打印了 date 命令的執行結果，並進行算術擴展。
但是 test* 沒有進行路徑名擴展，沒有擴展成當前目錄下以 "test" 開頭的文件名。

  • word 參數不進行擴展

在 <<word 中，word 本身不進行參數擴展、命令替換、算法括號、或路徑名擴展，所給的字符串會保持不變。
無法通過獲取變量值的方式來指定停止重定向的字符串。

修改 testhere.sh 腳本內容為下面的內容：

#!/bin/bash

number="12345"

cat <<$name

What is the name?

12345

$name

這裡指定停止重定向的字符串為 $name，name 變量值是 "12345"。
在 here-document 中提供了 "12345" 這一行，之後是 $name 這一行。

執行修改後的 testhere.sh 腳本，打印結果如下：

$ ./testhere.sh

What is the name?

12345

可以看到，輸出結果裡面包含了 "12345" 這一行，說明這一行沒有停止重定向。
<<$name 並不會獲取 name 變量值來作為停止重定向的字符串，而是保持停止重定向的字符串為 $name 不變。

Here Strings

前面說明的 here-documents 可以重定向多行到標準輸入。如果只需要重定向一行，也可以使用 Here Strings。
查看 Here Strings 的說明如下：

A variant of here documents, the format is:

    <<<word

The word undergoes brace expansion, tilde expansion, parameter and variable expansion, command substitution, arithmetic expansion, and quote removal. Pathname expansion and word splitting are not performed.
The result is supplied as a single string to the command on its standard input.

即，here-strings 會把所給的 word 重定向到命令的標準輸入，word 可以進行擴展。

跟 here-documents 不同，here-strings 的 word 無論是否加引號，都會進行參數擴展、命令替換、算術擴展、大括號擴展、波浪號擴展，並移除引號。
但是不進行路徑名擴展和單詞拆分。

如果 word 帶有空格，需要用引號括起來，否則可能會報錯。
對 <<<here strings 這種寫法來說，其實是 <<<、here、strings 三個參數。
如果被執行命令不能處理這三個參數，就會報錯。
如果想要把 here strings 當成一個參數，需要用引號括起來，例如 <<<"here strings"。

具體舉例說明如下：

$ number=1 2 3 4 5

$ grep 2 <<<$number

12345

$ grep t <<<test string

grep: string: No such file or directory

這裡的 <<<$number 會獲取 number 變量值來重定向到標準輸入。
grep 命令沒有提供文件名參數時，默認讀取標準輸入。
經過重定向，讀取到 number 變量值。

在 grep t <<<test string 命令中，test string 中帶有空格，沒有用引號括起來。
那麼 <<<test 是 here-strings 類型的重定向輸入，後面的 string 是另外的參數，會被 grep 命令當成文件名處理。
當前目錄下沒有該文件，執行報錯。

在 <word、<<word、<<<word 這三種形式中，word 的類型有如下差異：

  • 在 <word 中，word 是文件名，讀取文件內容作為重定向輸入的來源。即使用引號把 word 括起來，也還是當成文件名，不會當成字符串處理。
  • 在 <<word 中，word 是字符串，不會被當成文件名處理，也不會進行擴展。
  • 在 <<<word 中，word 是字符串，不會被當成文件名處理，會進行擴展。具體擴展類型如前面說明。

複製文件描述符

查看複製文件描述符（Duplicating File Descriptors）的說明如下：

The redirection operator

    [n]<&word

is used to duplicate input file descriptors. If word expands to one or more digits, the file descriptor denoted by n is made to be a copy of that file descriptor.
If the digits in word do not specify a file descriptor open for input, a redirection error occurs.
If word evaluates to -, file descriptor n is closed. If n is not specified, the standard input (file descriptor 0) is used.

The operator

    [n]>&word

is used similarly to duplicate output file descriptors. If n is not specified, the standard output (file descriptor 1) is used.
If the digits in word do not specify a file descriptor open for output, a redirection error occurs.
If word evaluates to -, file descriptor n is closed.
As a special case, if n is omitted, and word does not expand to one or more digits or -, the standard output and standard error are redirected as described previously.

即，[n]<&word 把文件描述符 n 複製到文件描述符 word。
如果文件描述符 word 沒有對應以寫模式打開的文件，則重定向報錯。
如果 word 的值是連字符 -，則會關閉文件描述符 n。
如果沒有提供文件描述符 n，默認會使用文件描述符 0，也就是標準輸入。

類似的，[n]>&word 把文件描述符 n 複製到文件描述符 word。
如果文件描述符 word 沒有對應以讀模式打開的文件，則重定向報錯。
如果 word 的值是連字符 -，則會關閉文件描述符 n。
如果沒有提供文件描述符 n，默認會使用文件描述符 1，也就是標準輸出。

有一種特別情況是，當 [n]>&word 中的 word 擴展結果不是數字、也不是連字符 - 時，word 會被當成文件名處理。
也就是前面說明的“同時重定向標準輸出和標準錯誤輸出”這種場景。

這個複製文件描述符的行為類似於 Linux 的 dup() 系統調用函數。
這兩個文件描述符會指向同一個文件表（file table），共享文件偏移指針。
所以它們寫入的內容是交錯開的，不會發生覆蓋的現象。

在編譯大型項目代碼時，就經常用到 2>&1 這個寫法。
例如可以使用下面命令全編譯 Android 源碼：

make -j16 2>&1 | tee build_android.log

這個命令使用 2>&1，把標準錯誤輸入複製到標準輸出。
那麼寫入到標準錯誤輸出的內容也會寫入到標準輸出，然後通過管道 | 把標準輸出重定向給下一個命令。

當使用管道 | 來重定向前一個命令的輸出到後一個命令的輸入時，只會重定向標準輸出，不會重定向標準錯誤輸出。
如果想重定向標準錯誤輸出，可以寫為 2>&1 |。
這種寫法會先將標準錯誤輸出重定向到標準輸出上，然後再一起輸出到管道上。

實際上，更簡單的寫法是 |&。
查看 man bash 的 Pipelines 小節提到了這一點，具體描述如下：

If |& is used, the standard error of command is connected to command2's standard input through the pipe; it is shorthand for 2>&1 |.
This implicit redirection of the standard error is performed after any redirections specified by the command.

移動文件描述符

查看移動文件描述符（Moving File Descriptors）的說明如下：

The redirection operator

    [n]<&digit-

moves the file descriptor digit to file descriptor n, or the standard input (file descriptor 0) if n is not specified. digit is closed after being duplicated to n.

Similarly, the redirection operator

    [n]>&digit-

moves the file descriptor digit to file descriptor n, or the standard output (file descriptor 1) if n is not specified.

即，[n]<&digit- 把文件描述符 digit 複製到文件描述符 n，然後關閉文件描述符 digit。
後面的連字符 - 必須提供。
如果沒有提供文件描述符 n，默認使用文件描述符 0，也就是標準輸入。

類似的，[n]>&digit- 把文件描述符 digit 複製到文件描述符 n，然後關閉文件描述符 digit。
後面的連字符 - 必須提供。
如果沒有提供文件描述符 n，默認使用文件描述符 1，也就是標準輸出。

這個移動文件描述符的行為和 Linux 的 dup2() 系統調用函數有所區別。
dup2() 函數將老的文件描述符（後面稱之為 oldfd）複製到新的文件描述符（後面稱之為 newfd）。
如果 newfd 之前是打開的，會先關閉 newfd，但是這個函數不會關閉 oldfd。
而 [n]<&digit-、[n]>&digit- 會在複製文件描述符之後，關閉 digit 對應的文件描述符。

注意：在 [n]<&digit-、[n]>&digit- 中，如果省略了 digit，就變成 [n]<&-、[n]>&-。
這是前面“複製文件描述符”的寫法，會關閉文件描述符 n。

以讀寫模式打開文件描述符

前面說明的文件描述符都對應到一個已經打開的文件。例如，n>word 是把 word 文件關聯到已經打開的文件描述符 n 上。
而下面介紹的重定向方式會打開一個新的文件描述符。

查看以讀寫模式打開文件描述符（Opening File Descriptors for Reading and Writing）的說明如下：

The redirection operator

    [n]<>word

causes the file whose name is the expansion of word to be opened for both reading and writing on file descriptor n, or on file descriptor 0 if n is not specified.
If the file does not exist, it is created.

即，[n]<>word 會以讀寫模式打開 word 文件，且打開後的文件描述符為 n。
如果沒有文件描述符 n，默認會打開到文件描述符 0 上。

如果 word 文件不存在，會創建該文件。
如果 word 文件已經存在，且往文件描述符 n 寫入內容，則會把清空 word 文件原有的內容，然後寫入新的內容。

具體舉例說明如下：

$ cat output.txt

date

sleep 2

$ echo "new string" 3<> output.txt > /dev/fd/3

$ cat output.txt

new string

這裡先用 cat output.txt 命令查看 output.txt 文件內容，說明該文件存在，且自身內容不為空。

在 echo "new string" 3<> output.txt > /dev/fd/3 命令裡面，3<> output.txt 以讀寫模式打開 output.txt 文件，並關聯到文件描述符 3 上。
後面的 > /dev/fd/3 表示把 echo 的輸出結果重定向到 /dev/fd/3 文件，也就是重定向到文件描述符 3，那麼會寫入到 output.txt 文件。

之後再次用 cat output.txt 命令查看 output.txt 文件內容。可見文件原有內容已經丟失，變成 echo 命令輸出的字符串。
