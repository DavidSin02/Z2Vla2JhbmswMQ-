從網絡架構演進的前世今生詳解5G各NF網絡功能體

2020-11-14
| 科技

一、引言

以前從來沒關注電信無線上網網絡的具體架構（也即PS域架構），現在開始學5G接觸這些東西時，理解起來很痛苦，資料也少，於是一方面到處找人諮詢，一方面到處查資料，最後發現應該從3G、4G時代的架構及功能開始學習，這樣知識有個演進過程，瞭解起來就很簡單了。廢話不說，下面我們來分別看看3G、4G、5G流量上網的網絡能力架構。

二、3G時代PS域架構

3G時代的PS域結構非常簡單，請見下圖：


上圖中：

  • RAN：無線接入網，其中的NodeB是基站，RNC是無線網絡控制器（Radio Network Controller，也稱為基站控制器），RNC負責移動性管理、呼叫處理、鏈路管理和移交機制
  • 無線核心網：3G核心網由電路域（CS域，CS即Circuit Switched，電路交換）和分組域（PS域,PS即分組交換Packet Switch，有時又稱分組業務Packet Service）兩部分組成，分別負責傳統語音業務和分組域業務（數據和多媒體業務），無線上網只和PS域相關。3G的PS域核心網主要由SGSN和GGSN兩部分組成。SGSN和GGSN統稱為GSN。
    其中SGSN(Serving GPRS Support Node，服務GPRS支持節點)執行用戶和GGSN之間的數據路由（用戶面功能），並進行用戶接入鑑權控制、移動性管理（包括附著、去附著和位置管理）、會話管理（包括激活、修改或者刪除PDP上下文等功能）這些控制面功能。
    GSGN（Gateway GPRS Support Node，網關GPRS支持節點，GSN網關）作為整個GPRS網絡的網關，位於GPRS網絡和外部分組交換網絡（Internet）之間，負責在核心網和外部網絡之間交換數據和執行地址轉換。

三、4G時代PS域架構

4G時代PS域的發展出現了較大的變化，PS域整體架構如下：

上圖中：

  • MME：英文全稱Mobile Managenment Entity， 即移動管理實體，主要負責M移動性管理和控制，包含用戶的鑑權、尋呼、位置更新和切換等等
  • SGW：Serving GW，即服務網關，用戶面接入服務網關，主要負責手機上下文會話的管理和數據包的路由和轉發
  • PGW：Packet data network Gateway ，即PDN網關，主要負責連接到外部網絡，包括會話和承載管理(承載管理即控制用戶設備和網絡的連接狀態，一個承載可以看做是一條分配了IP地址的數據包傳輸通道)、用戶IP地址分配、計費支持

可以看到，4G PS域的架構相對3G PS域架構進行了演進：

  • 將SGSN的移動性管理等控制面功能劃分給了MME
  • 將SGSN的路由管理等用戶面功能劃分給了SGW
  • PGW相當於原有的GGSN
  • PCRF是新增的策略控制網元

在2016年，受5G規範的CUPS架構（控制面用戶面分離架構）的影響，3GPP對SGW/PGW進行了一次拆分，把這兩個網元都進一步拆分為控制面（SGW-C和PGW-C）和用戶面（SGW-U和PGW-U）。如圖：


SGW-C和PGW-C主要根據終端所在位置及漫遊信息選擇合適的SGW-U/PGW-U的路由，從而建立數據傳輸承載信道，路由建立後，MME將SGW-C/PGW-C返回的路由信息傳至基站eNodeB，eNodeB自動將移動終端數據包在的SGW-U/PGW-U等組成的承載信道中直接傳輸，對於無需出本地的數據流量在本地直接分流。

三、5G時代PS域架構

5G標準制定時，為了提高系統響應性能，在整個PS域進行了比較大的架構優化，架構方面的優化主要有如下：

  • SBA（Service Based Architecture）架構：實際上就是中心化和微服務化架構，傳統網元的功能進行整合後再按功能單一化劃分為多個NF，然後NF再被分解為多個“網絡功能服務”，支持容器化部署，目標是實現敏捷開發、變更控制範圍、可動態伸縮、提升可靠性。所有基於服務的NF都接入SBI（Service Based Interface）消息總線，SBI消息總線基於HTTP/2，採用RESTful API設計原則
  • CUPS(Control and User Plane Separation）：控制與用戶面分離，用戶面可以集中部署也可分佈式部署，在分佈式部署時將用戶面下沉到更接近用戶，提升對用戶請求的響應速度，而控制面集中管理、統一集群部署，提升可維護性和可靠性
  • NS(Network Slice )：網絡切片，基於網絡功能虛擬化、軟件化的基礎上，把網絡切成多個虛擬且相互隔離的子網絡，分別應對有不同要求的不同服務，再將網絡功能進一步細粒度模塊化，實現靈活組裝業務應用

5G PS域相關的整體架構如下：


相關網元及功能如下：

  • AMF：Access and Mobility Management Function，接入及移動性管理功能，AMF接收UE的請求並處理所有與連接和移動性管理有關的任務如鑑權，並通過N11接口向SMF轉發會話管理需求。AMF通過詢問NRF確定哪一個SMF是最適合處理該連接請求
  • SMF: Session Management Function ，會話管理功能，例如會話建立，修改和釋放，包括UPF和AN節點之間的隧道維護、UE IP地址分配和管理、選擇和控制UPF功能、計費數據收集和計費接口支持等
  • UPF: User plane Function，用戶面功能，包括用戶數據包的路由和轉發、與外部數據網DN的數據交互、用戶平面的QoS處理、流控規則實施（例如門控、重定向、流量轉向）等
  • UDM：Unified Data Management，統一數據管理功能，負責用戶標識、簽約數據、鑑權數據的管理、用戶的服務網元註冊管理（比如當前為終端提供業務的AMF、SMF等，如當用戶切換了訪問的AMF時，UDM還會向舊的AMF發起註銷消息，要求舊的AMF刪除用戶相關信息）
  • AUSF：Authentication Server Function，鑑權服務功能，AUSF用於接收AMF對UE進行身份驗證的請求，通過向UDM請求密鑰，再將UDM下發的密鑰轉發給AMF進行鑑權處理
  • PCF：Policy Control function ，策略控制功能，支持統一的策略框架去管理網絡行為，提供策略規則給網絡實體去實施執行，訪問統一數據倉庫（UDR）的訂閱信息
  • NRF：NF Repository Function ，網絡存儲功能，用來進行NF登記、管理、狀態檢測，實現所有NF的自動化管理，每個NF啟動時，必須要到NRF進行註冊登記才能提供服務，登記信息包括NF類型、地址、服務列表等
  • AF：Application Function ，應用功能，指應用層的各種服務，可以是運營商內部的應用如Volte AF(類似4G的Volte As）、也可以是第三方的AF（如視頻服務器、遊戲服務器），如果是運營商內部的AF，與其他NF在一個可信域內，可以直接與其他NF如PCF交互訪問，而第三方的AF則不在可信域內，必須通過NEF訪問其他NF
  • NEF：Network Exposure Function ，網絡開放功能，位於5G核心網和外部第三方應用功能體之間（可能也有部分內部AF），負責管理對外開放網絡數據的，所有的外部應用，想要訪問5G核心網內部數據，都必須要通過NEF。NEF提供相應的安全保障來保證外部應用到3gpp網絡的安全，提供外部應用Qos定製能力開放、移動性狀態事件訂閱、AF請求分發等功能
  • NSSF:Network Slice Selection Function， 網絡切片選擇功能
  • DN: 數據網絡Data Network (DN), 比如運營商業務，互聯網或者第三方業務等

在這些功能型的NF後面，還有兩個重要的NF，它們是：

  • UDR：Unified Data Repository ，統一數據存儲/倉庫，用於UDM存儲訂閱數據或讀取訂閱數據以及PCF存儲策略數據或者讀取策略數據
  • UDSF：Unstructured Data Storage network function，非結構化數據存儲功能，所有NF的非結構化數據都存儲於UDSF

從以上架構可以看出，無論怎麼演進，網絡架構中的用戶管理、會話管理、移動性管理、接入和鑑權管理的控制面功能和數據包轉發的用戶面功能都是PS域的必選基礎功能，從上面這些NF的功能可以看出，5G的基礎功能相關NF是從4G的網元進行功能拆分再部分整合後形成的功能體：

  • MME中負責接入和移動性管理的功能獨立出來，成了5G的AMF
  • SGW-C和PGW-C關於會話管理的功能合併成為SMF
  • MME和HSS中用戶鑑權的功能被抽取出來，合併成為5G的AUSF，HSS中剩餘的用戶數據管理功能獨立成為UDM，和AUSF配合工作來完成用戶鑑權數據相關的處理
  • PCRF成了5G中的PCF
  • SGW-U和PGW-U合併成為5G的用戶面功能UPF
  • AF和DN基本功能保持不變就是4G網絡中的AS和互聯網

同時為了適應新的SBA架構和業務要求，新增了NSSF、NRF、UDR、UDSF及NEF以及SBA總線。

四、小結

經過查閱各種互聯網上分享的知識和閱讀部分3GPP規範，並多處求教，同時結合3G、4G的PS域組網架構，老猿終於將5G PS域各功能體的大致功能弄清楚了。這個關於5G各NF的功能介紹，應該是目前網上相關文章中最全的，並且由於以前老猿本身對核心網並不熟悉，因此介紹時寫得還算比較通俗易懂，這些內容有助於有一定IT基礎大眾的理解。

我們看到，從3G到4G、4G到5G的演進過程中，核心功能是一脈相傳，但實現的方式在不停演進，系統實現演進的目的就是為了提升用戶上網響應的性能、減少不必要的流量傳輸、提升系統的可複用性、可靠性和擴展性，並節約投資。
