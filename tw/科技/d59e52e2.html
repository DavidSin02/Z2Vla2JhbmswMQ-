視圖，索引，事務，備份與還原

2020-11-14
| 科技



視圖

  • 視圖就是一張虛擬表表示一張表的部分或多找表的綜合數據它的結構和數據是建立在對錶的查詢的基礎上
  • 視圖不存放數據數據存放在視圖所引用的原始表
  • 一個原始表根據不同的用戶需求可以創建不同的視圖

好處

  • 篩選表中的數據
  • 防止未經許可的用戶訪問敏感數據
  • 降低數據庫的複雜程度
  • 將多個物理的數據庫抽象成一個邏輯的數據

語法

創建

create view 視圖名 as select語句

使用：作為表使用

刪除

drop view 視圖名

視圖使用的注意事項

- 視圖中可以使用多個表 - 一個視圖可以嵌套另一個視圖 - 對視圖數據進行添加、更新和刪除操作直接影響所引用表中的數據 - 當視圖數據來自多個表時，不允許添加和刪除數據 - 使用視圖修改數據會有許多限制，一般在實際開發中視圖僅用作查詢

索引

- 索引是一種有效組合數據的方式，為快速查找到指定記錄 - 作用 - 大大提高數據庫的檢索速度 - 改善數據庫性能 - MySQL索引按存儲類型分類 - B-樹索引：InnoDB、MyISAM均支持 - 哈希索引

常用索引類型

- 普通索引 - 基本索引類型 - 允許在定義索引的列中插入重複值和空值 - 唯一索引 - 索引列數據不重複 - 允許有空值 - 主鍵索引 - 主鍵列中的每個值是非空、唯一的 - 一個主鍵將自動創建主鍵索引 - 複合索引 - 將多個列組合作為索引 - 全文索引 - 支持值的全文查找 - 允許重複值和空值 - 空間索引 - 對空間數據類型的列建立的索引

索引的使用

- 創建索引

 CREATE [UNIQUE|FULLTEXT|SPATIAL] INDEX index_name ON table_name (column_name[length]…);

- 刪除所有

 DROP INDEX index_name ON table_name; -- 刪除表時，該表的所有索引同時會被刪除

建索引的指導原則

- 按照下列標準選擇建立索引的列 - 頻繁搜索的列 - 經常用作查詢選擇的列 - 經常排序、分組的列 - 經常用作連接的列（主鍵/外鍵） - 請不要使用下面的列創建索引 - 僅包含幾個不同值的列 - 表中僅包含幾行

事務

- 為什麼需要事務？轉賬問題

什麼是事務

- 事務(TRANSACTION)是作為單個邏輯工作單元執行的一系列操作 - 多個操作作為一個整體向系統提交，要麼都執行、要麼都不執行 - 事務是一個不可分割的工作邏輯單元

事務的特性

- 原子性（Atomicity）：事務是一個完整的操作，事務的各步操作是不可分的(原子的)，要麼都執行，要麼都不執行 - 一致性（Consistency）：當事務完成時，數據必須處於一致狀態 - 隔離性（Isolation）：併發事務之間彼此隔離、獨立，它不應以任何方式依賴於或影響其他事務 - 持久性（Durability）：事務完成後,它對數據庫的修改被永久保持

事務的策略

- 默認情況下，每條單獨的SQL語句視為一個事務 - 關閉默認提交狀態後，可手動開啟、關閉事務 - 關閉/開啟自動提交狀態

   SET autocommit = 0|1;   -- 值為0：關閉自動提交   -- 值為1：開啟自動提交

- 關閉自動提交後，從下一條SQL語句開始則開啟新事務，需使用COMMIT或ROLLBACK語句結束該事務

事務的使用

- 開始事務：

 BEGIN ; 或 START TRANSACTION;

- 提交事務：

 COMMIT ;

- 回滾事務：

 ROLLBACK ;
數據庫的備份

- 使用mysqldump命令備份數據

 mysqldump [options] –u username –h host –ppassword dbname[tbname1[,tbname2……]]>filename.sql

- 使用mysql命令恢復數據庫

 mysql –u username –p [dbname] < filename.sql

- 1. mysql為DOS命令 - 2.在執行該語句之前，必須在MySQL服務器中創建新數據庫，如果不存在恢復數據庫過程將會出錯 - 使用source命令恢復數據庫 - 通過複製文件實現數據備份和恢復 - 優點：操作簡單 - 缺點： - 複製數據前停止服務 - 不適合InnoDB數據庫 - MySQL版本不同不兼容

- 表數據導出到文本文件，語法：

SELECT * FROM tablename      [WHERE contion]     INTO OUTFILE 'filename' [OPTION]

- 將文本文件中的數據導入到表中，語法：

 LOAD DATA INFILE filename INTO TABLE tablename [OPTION]

- 導入數據前應確保目標表已存在！

- 使用工具navicat備份與導入 - 對著數據庫，右鍵轉儲 - 對著數據庫，右鍵運行
