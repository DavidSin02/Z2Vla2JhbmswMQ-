線程與進程概念

在開始前，有必要重複下操作系統中的進程與線程的概念:

進程是指一個操作系統內存中獨立運行的應用程序，每個進程都有自己獨立的一塊內存空間，進程包含一個或多個線程。

線程是指進程中的一個執行流程，是cpu的最小調度單元。

運行java程序，整個jvm就是一個進程，而你寫的main就是啟動一個線程，而GC是另一個線程。也就是說，一個jvm進程至少包含了兩個線程。

為何要用多線程？

可以說，多線程的出現，大半是搞硬件的甩鍋。

java中的進程和線程實現

在 java 中，創建進程有直接關係的類為 Java.lang.Process，和實現該抽象類的 ProcessImpl 類，如果用戶創建了一個進程，那麼肯定會伴隨著一個新的 ProcessImpl 實例。

在 JDK 的代碼中，只提供了 ProcessImpl 類來實現 Process 抽象類。而這個類引用了 native 的 create, close, waitfor, destory 和 exitValue 方法來實現進程相關操作。

眾所周知，java中的native方法是c/c++實現的與平臺相關的實現，如在windows中，native的create方法會調用windows的api接口的CreateProcess和CreatePipe，由操作系統來創建一個真正的進程。正是由於眾多native方法，才能讓java有平臺無關的特性。

總結，java中創建進程，會通過native方法調用操作系統級的命令，創建一個真正的進程。

線程實現 Runnable 接口或者繼承 Thread類，一個 Java 線程的創建根本上就對應了一個本地線程（native thread）的創建，兩者是一一對應的。

最終調用的本地方法

JVM_ENTRY(void, JVM_StartThread(JNIEnv* env, jobject jthread))

…

native_thread = new JavaThread(&thread_entry, sz);

…

總結，java中創建線程程，會通過native方法調用操作系統創建一個真正的線程的命令，創建一個真正的線程。
