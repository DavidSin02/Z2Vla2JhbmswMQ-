程序猿福利：支持12種數據庫的自動分頁插件

2020-10-29
| 科技

MyBatis-Pageable是一款自動化分頁的插件，基於MyBatis內部的插件Interceptor攔截器編寫完成，攔截Executor.query的兩個重載方法計算出分頁的信息以及根據配置的數據庫Dialect自動執行不同的查詢語句完成總數量的統計。

支持的數據庫

MyBatis-Pageable目前支持的主流數據庫：

  • DB2
  • Derby
  • DM、
  • H2、
  • HSQL、
  • InforMix、
  • Mariadb、
  • MySQL、
  • Oracle、
  • Postgres、
  • SqlLite、
  • SqlServer2000以上版本

快速開始

MyBatis-Pageable可以在傳統的Spring項目內使用，也可以在最新SpringBoot項目內使用。

SpringBoot集成

MyBatis-Pageable目前已經上傳到Maven中央倉庫，你可以隨時配置依賴使用，如果你是Maven管理的項目，複製如下代碼到你的pom.xml配置文件內：

<dependency> <groupId>com.gitee.hengboy</groupId> <artifactId>spring-boot-starter-mybatis-pageable</artifactId> <version>{lastVersion}</version> </dependency>

如果你使用的Gradle管理的項目，那麼你需要複製如下的代碼到你們的build.gradle文件內：

compile 'com.gitee.hengboy:mybatis-pageable:{lastVersion}'

注意：上面內容需要把{lastVersion}改成目前最新版本的MyBatis-Pageable版本號

配置插件

添加完成 starter依賴後，你可以在配置文件application.yml或者application.properties內完成配置，下面是yml格式配置：

hengboy: pageable: dialect: mysql

在使用pageable時你僅僅需要配置一個數據庫方言，如果你使用MySQL或者MariaDB數據庫你可以不配置數據庫方言。

傳統Spring集成

使用mybatis-config.xml 配置插件

如果你項目中配置使用了mybatis-config.xml，那麼你需要添加如下代碼到你的配置文件：

<plugins> <plugin interceptor="com.gitee.hengboy.mybatis.pageable.interceptor.MyBatisExecutePageableInterceptor"> <!-- 配置數據庫方言 --> <property name="dialect" value="mysql"/> </plugin></plugins>

使用application.xml 配置插件

當然你如果不使用mybatis-config.xml配置文件配置插件，也可以通過SqlSessionFactoryBean的插件列表來完成集成MyBatis-Pageable，如下配置：

<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean"> <!-- 配置插件列表 --> <property name="plugins"> <array> <bean class="com.gitee.hengboy.mybatis.pageable.interceptor.MyBatisExecutePageableInterceptor"> <property name="properties"> <!-- 配置數據庫方言 --> <value> dialect=mysql </value> </property> </bean> </array> </property></bean>

怎麼使用？

目前MyBatis-Pageable使用比較簡單，有一個分頁請求類PageableRequest來完成自動分頁操作，我們來看個簡單的示例：

Page<UserEntity> page = PageableRequest.of(1, 5).request(() -> userMapper.selectAll());
  • of 配置分頁的當前頁碼以及每頁的限制條數
  • request 該方法需要傳遞一個夜晚邏輯方法，也就是你需要執行分頁的方法

Page對象詳解

在上面簡單的一行代碼就可以完成自動分頁以及讀取出分頁相關的信息，分頁執行後我們通過Page對象都可以獲取到什麼內容呢？

  • data 分頁後的數據列表，具體的返回值可以使用Page<T>泛型接收
  • totalPages 總頁數
  • totalElements 總條數
  • pageIndex 當前頁碼
  • pageSize 每頁限制條數
  • offset 分頁開始位置
  • endRow 分頁結束位置
  • hasNext 是否存在下一頁，true：存在，false：不存在
  • hasPrevious 是否存在上一頁，true：存在，false：不存在
  • isFirst 是否為首頁，true：首頁，false：非首頁
  • isLast 是否為末頁，true：末頁，false：非末頁

翻頁查詢

實際開發過程中存在這種情況，雖然傳遞的分頁頁碼為1，但是種種判斷過後我需要查詢上一頁或者下一頁、首頁的數據，這時候你就可以PageableRequest對象的next()、previous()、first()方法來處理這種事情的發生，如下示例：

Pageable pageable = PageableRequest.of(2, 10); if (xx = xx) { pageable.next(); } Page<UserEntity> page = pageable.request(()->userMapper.selectAll());

上面是翻頁到下一頁的查詢示例，當然這個功能是為了儘可能的方便分頁的使用，同樣的previous()、first()方法都可以這麼使用。
