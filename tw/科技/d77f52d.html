C語言：數據類型與運算符

2020-10-29
| 科技

一、數據類型

除void類型以外，基本的數據類型之前都可以加各種各類修飾符。在基本類型前加類型修飾符可以更加準確地對類型進行定義。

1）signed———“無符號”之意，可以修飾int、char基本類型。

注意：默認的int型定義為有符號整數，因此對int型使用signed是冗餘的。signed的重要用途是在char類型默認為無符號時，使用signed修飾char，表示有符號字符型。

2）unsigned——“無符號”之意，可以修飾int、char基本類型。

3）long——“長型”之意，可以修飾int、char基本類型。

4）short———“短型”之意，可以修飾int基本類型。

5）signed和unsigned也可以用來修飾long int和short int，但不能修飾double和float類型。

6）當類型單獨使用（即將修飾的基本類型省略）時，系統默認其為int類型，因此下面幾種用法是等價的。

signed 等價於 signed int

unsigned 等價於 unsigned int

long 等價於 long int

short 等價於 short int

計算機處理的最小單元就是位（bit），一個位的值要麼是0，要麼是1，只有兩種可能，因此稱為二進制數（Binary digit），bit就是Binary digit得縮寫。8位構成1字節（Byte）。有符號和無符號整數之間的區別在於怎樣解釋整數對應於內存中的二進制數的最高位。對於無符號數，其最高位被C編譯器解釋為數據位。而對於有符號數，C編譯器將其最高位解釋為符號位，若符號位為0，表示該數為正數；若符號位為1，表示該數為負數。對於計算機硬件而言，是沒有數據類型的概念的，它更不區分內存中的二進制數那些是有符號數，那些是無符號數。可以說是，你把它看成什麼，它就是什麼。例如，你把它的最高位看作符號位，它就是一個有符號數；你把它的最高位看作數據位，那麼它就是一個無符號數。

對於具有相同字節數的整數而言，因為有符號數的數據位比無符號數的數據位少了一位，且少的這一位恰好是最高位，因此有符號整數中的最大數的絕對值僅為最大無符號整數的一半。比如，假設編譯系統為int性數分配2字節的存儲空間，即16個二進制位中的最高位是符號位，以32767為例，其內存中的存儲方式如下：

為什麼解釋成-1呢？

答：在計算機內存中，負數是以二進制補碼形式表示和存放。

為什麼要使用補碼？

原因分兩方面：一方面可以將減法運算變成加法運算來處理問題；

另一方面可以對0這個數形成統一的表示。

補碼

在保持符號位不變的情況下，將負數的原碼中得的1變成0、0變成1，得到的就是該負數的反碼。負數的補碼就是，在保持符號位不變的情況下，先求得求其反碼，然後再將其結果加1。

正數的反碼、補碼亦如此。

由於-1的二進制補碼全為1，因此若將最高位的1解釋為符號位（有符號數），則這個數就是-1；若將最高位的1解釋為數據位（無符號數），則這個數就是65535.

為什麼用補碼可以統一的形式來表示0，並且可以將減法運算變成加法運算？

對於雙字節整型數，+0的源代碼都是0000 0000 、0000 0000，-0的原碼為1000 0000 0000

0000，其反碼為1111 1111 1111 1111，將這個反碼加1後得到0000 0000 0000 0000（其中舍掉了最高位的進位），顯然，+0和-0的補碼是相同的，即0的表示是唯一的，而+0和-0

的原碼卻是截然不同的，也就是0的原碼錶示是不唯一的。

二、運算符

C語言既具有高級語言的特點，又具有低級語言的功能，如支持位運算就是這種特點的具體表現。這是因為，C語言最初是為取代彙編語言設計系統而設計，因此它必須支持位運算等彙編操作。位運算是對字節或二進制數位進行測試、抽取、設置或移位等操作。操作對象不能是float和double等其他複雜的數據類型，只能是標準的char和int數據類型。
