Elastic Stack 7.3.0 發佈，引入數據幀

2020-10-29
| 科技

Elastic Stack 是一系列開源產品的合集，包括 Elasticsearch、Kibana、Logstash 以及 Beats 等等，能夠安全可靠地獲取任何來源、任何格式的數據，並且能夠實時地對數據進行搜索、分析和可視化。

7.3.0 版本已經發布，具體更新如下：

實體化視圖？以實體為中心的索引？引入數據幀

利用這項新功能，可以動態透視 Elasticsearch 數據，創建以實體為中心的活動索引，它打開了全新分析世界的大門，包括新的機器學習分析，諸如異常值檢測、聚類、分類等。

藉助數據幀功能，可以新建一個以實體為中心的索引，每個唯一的 IP 地址都有一個文檔，用於跟蹤每個感興趣的指標 — 在這個用例中為總請求數、每個響應狀態的計數和傳輸的字節數。最重要的是，數據幀支持連續處理，這意味著當有新文檔添加到輸入索引中時，這一轉換後的以實體為中心的索引將自動更新。

Elastic SIEM 增加了異常檢測 — 因為只有規則還不夠

用戶現在可以從 SIEM 應用中輕鬆啟用和運行一組機器學習異常檢測作業，用以專門檢測特定的網絡攻擊行為。檢測到的異常可以方便地顯示在 SIEM 應用的託管和網絡視圖上。

不想使用預配置作業的用戶可以使用 Machine Learning 應用輕鬆添加自定義異常檢測作業。

通過此項集成，現在利用機器學習來幫助檢測攻擊行為變得前所未有的容易。

Elastic 地圖已正式發佈

位置一直是搜索中十分重要的一部分，無論是調查您網絡中的攻擊源、診斷特定區域內應用程序響應時間較慢的原因、實時跟蹤送貨卡車，還是隻想找到附近最好吃的墨西哥捲餅，都離不開位置搜索。

此外，Elastic 地圖還添加了幾項新功能，其中最令人興奮的是能夠將特性、形狀和圖層從 GeoJSON 文件上傳到地圖中。其他諸如在繪製自定義圖標和可視化過去已知位置方面的功能改進，增強了整體的用戶體驗。有關完整詳情，請閱讀 Elastic 地圖博文

還有非常多的功能：

  • Elasticsearch 引入了只支持選舉的主節點，添加了期待已久的稀有詞彙聚合、新的快照/恢復管理 UI、動態更新的同義詞及其他更多內容
  • Kibana 提供了對 Kerberos、自動完成和 KQL 的支持來過濾聚合，並提供了畫布工作板模板，可以更輕鬆地構建漂亮的顯示內容
  • Beats 增加了對一系列新數據源的支持（包括 Oracle 和 Amazon RDS 這樣的關係數據庫、針對 Kubernetes kube-proxy、kube-scheduler 以及 kube-controller-manager 的指標）、對 GCP VPC 流日誌的網絡流支持，並改進了對 Amazon Kinesis Data Streams 和帶有 Functionbeat 的 Amazon Cloudwatch 的支持。
  • Logstash 現在默認包含 JMS 支持
  • Elastic APM .NET Agent 現已正式發佈。此外，Elastic APM 添加了一種新的方法來查看帶有聚合服務分解圖的服務、來自 Kibana 的代理採樣率配置，以及更多內容
  • Elastic Uptime 改進了多位置監測器在監測摘要和詳情方面的界面外觀
  • Elastic Logs 添加了關鍵字高亮顯示，以及基於 trace.id 跳轉到 APM 痕跡的功能
  • Elastic Infrastructure 應用中的 Metrics Explorer 現在可以在生產系統中使用，您可以根據時序指標快速執行聚合可視化。此外，Elastic Infrastructure 還改進了對其他核心 Kubernetes 服務的監測，並在 AWS 模塊中新增了 RDS 指標集
