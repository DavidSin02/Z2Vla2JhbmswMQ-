一個簡單的小程序入門信息安全

本文通過一個簡單的密碼驗證的小程序來演示在輸入密碼錯誤的情況下仍然可以通過驗證的方法。其中用到了緩衝區溢出的原理，通過該文章的學習，即可成功入門信息安全。

本文需要用到一些基礎的彙編知識。

首先看程序的源碼

程序源碼

#include <string.h>#include <stdio.h>int main(void){    char buff[15];    int pass = 0 ;               printf("\n please input the password\n");    gets(buff);    if(strcmp(buff,"hello")){        printf("\npassword wrong\n");    }         else{        printf("\npassword right\n");        pass = 1;    }         if(pass){        printf("\ncongratulations\n");    }         return 0;}

該代碼首先分配了15個字節的buff和4個字節的pass。
接著使用gets函數接受輸入存在buff中。（gets函數是不安全的，工作中使用一定要慎重哦！！）
接著該代碼使用strcmp進行字符串比較判斷輸入的密碼是否正確，如果正確則打印出password right，並打印出congratulations，如果錯誤，則打印出password wrong。

而我們的目的是在密碼輸入錯誤的情況下，讓程序也能打印出congratulations。

接下來，我們就開始分析該程序。

首先看下面這行：

sub    rsp,0x20

將rsp-0x20，相當於給棧區分配了32個字節的空間。0x20=32（十進制）。

我們的代碼中，實際上只有15+4=19個字節的大小，但是因為內存對齊，實際上編譯器分配了32個字節的空間。

char buff[15];int pass = 0 ;

接下來

mov    DWORD PTR [rbp-0x4],0x0

這一行實際上就是給pass進行賦值，pass=0。

接著往下可以看

   0x0000000000400696 <+25>:	lea    rax,[rbp-0x20]   0x000000000040069a <+29>:	mov    rdi,rax   0x000000000040069d <+32>:	call   0x400570 <gets@plt>

在get函數之前，有這麼一句rbp-0x20,這個實際上就是buff的地址。

看到這裡我們基本就對buff和pass存儲的位置有了印象。

由於程序中使用的是gets函數，而gets函數是可以進行緩衝區溢出的，那麼如果我們一次性將32個字節的內存區域全部填滿，那麼就可以將pass賦值為別的值，那麼就有可能跳過密碼檢查。
根據猜想，我們輸入：

aaaaaaaaaaaaaaaBBBBBBBBBBBBBBBBBBBBB

如下所示：

# root @ localhost in /home/code/test [11:05:32] $ ./test please input the passwordaaaaaaaaaaaaaaaBBBBBBBBBBBBBBBBBBBBBpassword wrongcongratulations

發現雖然打印出password wrong 但是還是打印出了congratulations。
