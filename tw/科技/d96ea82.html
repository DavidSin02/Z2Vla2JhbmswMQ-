第一部分 機房工程

第一章 概述

在現代科學技術高度發展的社會裡，電子計算機機房這個概念將越來越廣泛地應用於各個領域，近年來信息技術正迅猛發展，但是計算機設備只有通過穩定、可靠的運行才能發揮其效益，而計算機設備的穩定、可靠運行要依靠電子計算機房的嚴格的環境條件，即機房溫度、溼度、潔淨度、噪聲、承重、振動、電磁屏蔽、防靜電、不間斷供電、安保、防雷、防火、防漏水、防竊、防鼠蟲害等條件及其控制精度，因此計算機機房系統工程是一項多學科緊密結合的複雜系統工程，其設計與施工也日益被人們所重視起來。

計算機機房工程是一種涉及到空調及新風技術、供配電技術、自動檢測與控制技術、抗干擾技術、綜合佈線及弱電技術、淨化、消防、建築、裝潢等多種專業的綜合性產業。我公司在吸取了國內外十多年來計算機機房建設的設計、施工方面的經驗教訓的基礎上，根據現場資料和實際需求，對影響計算機設備穩定、可靠運行的各種因素作了較全面的分析，在設計過程中採用先進的設計思想及理念。

2、設計理念

為了將本計算機機房建設成為具有當今領先水平的智能化數據處理和通信中心，使其成為一個能滿足當前及今後若干年發展需要的、系統功能完善、高可靠性的標準化的GB標準A類機房，設計過程中遵循以下原則：

1）整體規劃

政務外網機房改造項目是一個整體，在設計中充分考慮各系統設備設置的協調性，裝修格調及效果的整體性，以及後期擴容的方便靈活性。

2）合理佈局

機房可使用的面積共80平米，空間是有限的，在設計中在瞞足規範的前提下充分利用現有的面積，將網絡機房和監控室進行合理設置，空間合理佈局。

3）技術先進

在滿足可靠，適用的前提下，採用新技術新材料，及先進的施工、工藝、和成熟穩定性價比高的動力環境設備，為計算機系統、網絡系統的運行提供保障，同時為工作人員提供先進的高科技工作環境。

4）安全可靠

機房建設項目從設計，設備選型到施工，把安全可靠作為該項目的核心，並充分考慮建築結構的安全，計算機網絡設備運行的安全和在機房內工作的技術人員的身心健康的安全，從建築結構的荷載、裝修材料的防火、環保、節能、方面和供配電系統、防雷接地系統，空調及通風系統，消防系統，安全監控系統設備運行的可靠性，穩定性等方面採取有效措施，確保政務外網設備的運行環境以及技術人員的工作環境。

5）節能環保

節能、環保是我們國家的國策，計算機、網絡機房中的空調系統和照明系統是主要耗能系統。因此，在機房圍護結構的處理，空調通風設備的選型以及照明光源的選型、控制等方面予以重點考慮，同時在裝修材料的選擇上選用有檢測報告、甲醛含量低，合格產品，保證建成後整體空氣質量合格。

6）經濟適用

保證項目的重點環節，在考慮日後維護費用的同時，使工藝與造價兩者兼顧，性價比的最優化。

3、機房環境設計指標

3.1 開機時的溫度和溼度

溫度：22±20C，最佳：220C；關機時3~350C

開機時相對溼度：45％－65％，最佳:55％；關機時40％－70％

變化率<5％；不結露

3.2 空氣潔淨度

在靜態條件下測試，每升空氣中大於等於0.5μm的塵粒數,應小於10000粒/dm3。

3.3 噪聲

5點測試平均值應小於70dB(A)。

3.4 電磁干擾

磁場干擾：≤800A／m.

3.5 無線電干擾

在頻率為0．15～1000mHz時,≤120dB。

3.6 振動

在系統停機條件下，地板表面垂直及水平方向的振動加速度不大於500mm／s².

3.7 靜電

地面、吊頂、吊頂及工作臺面的靜電洩露電阻，符合現行國家標準《計算機機房用活動地板技術條件》的規定；

3.8 照度

機房區域平均照度為≥300勒克斯（離地面800㎜處），輔助房間的平均照度為≥200勒克斯（離地面800mm處），應急照明的平均照度為≥５勒克斯（離地面800mm處）。

3.9 電壓、頻率

允許變動範圍；單相220V＋5%～220V－5%；50Hz±0.2Hz。波形失真<±5％。

3.10 瞬間變動電壓

不能超過220V±15％。

第二章 裝飾工程設計說明

計算機機房應建成安全性、可靠性要求最高、最重要的部位，是數據交換、儲存和通訊樞紐，是項目的重要組成部分。主機房內放置的昂貴的計算機設備和網絡通訊設備等，不僅因為高科技產品需要一個非常嚴格的環境，更重要的是它能否正常可靠運作，對系統工作是至關重要的。因此，機房的基本結構組合必須達到以下目的及水準。

重點目標有防火、防塵、防靜電、防漏水、防電磁干擾、防鼠蟲害、防竊、隔熱、保溫、和適當的屏蔽等。

該機房的裝飾部分的主旨是：既要與現代化的計算機通信設備相匹配，又能通過精良而獨特的設計構思，真正體現"安全、可靠、現代、高雅、美觀、適用"的整體形象。

1、裝飾設計基本原則

體現特點――體現出計算機機房高科技部門的室內裝潢特點。

突出重點――在充分考慮服務器、網絡通訊、空調、UPS等設備的安全性、可靠性、先進性的前提下，達到高雅、大方的風格。

宜於健康――在材料的選用方面，要以自然材料為主，充分考慮環保因素，採用安全、環保和對人體無害的材料。

2、平面功能分區設計思路

機房區域出入口共有1個，作為進出的人員的出入口，人員進出機房時在出入口換鞋後進入機房，保證機房的潔淨度和安全性，機房門採用防火屏蔽門，完全滿足消防規範的要求。機房內部隔斷採用12mm厚防火玻璃，將機房分為值班室和網絡中心機房兩部分，透過玻璃隔斷，展示出高科技整齊美觀IT設備的靚麗外觀，同時使整體機房通透可視，實現機房區域內設備運轉和人員流通互不干擾。

電源室的獨立，主要考慮作為機房區域的UPS配電中心，同時保證了設備在維護和管理上的便利性，考慮到UPS蓄電池組的重量較大，因此蓄電池組安裝在加固位置處。

3、材料選擇原則

為保證機房的潔淨度，裝修材料應選用氣密性好、不起塵、易清潔、安全、環保和對人體無害的飾面材料。

應避開強電磁場的干擾及保障電腦系統信息安全，要採取有效的電磁屏蔽措施，滿足機房內抗無線電干擾場強要求和抗磁場干擾場強要求。

為滿足機房的防火要求，裝修材料必須全部採用不燃、阻燃材料。

所選天花板、地板、牆板、防火防盜門均為防火性能高的材料,滿足機房的防火要求。各裝飾材料均通過國家建築裝飾材料檢測中心檢測合格，並通過國家消防部消防材料檢測中心檢測合格。

4、機房標高

2、新/排風機組配置方案

2.1 新風機組的選型

合適的新風即可以滿足工作人員對氧氣的需求，又可以保持機房的正壓，防止外界的塵土及潮氣侵入。新風都需要經過處理才能進入機房。一般性機房，需經過初效、中效、亞高效過慮器，才能滿足要求。

新風必須送至機房精密空調上方，與室內迴風混合後，再通過機房專用空調過濾處理，送風至主機房各個區域。

新風機組的選擇：選擇風量為 400 m３/ｈ新風機一臺

2.2 新風系統的佈置

新風機安裝在機房室外，通過新風管道將室外新風送至機房各區域，保證新風進入機房的清潔度，室外新風先通過進風口引至新風機進行兩級過濾，經新風機處理後的新風還未達到機房要求，為此這些新風再經過機房專用空調處理，最後送入地板下靜壓箱。

新風機通風管道必須安裝手動、電動關斷閥門（防火閥）並與消防控制系統聯動。當機房出現火警時，新風機電源被切斷。新風管道閥門關閉防止火勢蔓延擴大。

2.3排風機組的選型

在本空調系統中，排風系統主要考慮換氣排風，同時保持空調機房內一定的正壓。

排風機組的選擇：選擇風量為 350 m３/ｈ排風機一臺。

2.4 排風系統的佈置

排風機安裝在機房吊頂內。通過排風管道將機房各區被汙染的空氣排出室外，排風口設在機房北側吊頂內，排風處為大樓外，在機房外牆上做防水百葉。為了防止雨水和鼠蟲沿排風管進入機房，在室外排風處安裝防水百葉和防蟲網。

第四章 供配電系統工程

計算機房的建設必須建立一個良好的供電系統，在這個系統中不僅要解決計算機設備（主機、網絡、主控、電腦、終端等）用電的問題，還要解決保障計算機設備正常運行的其它附屬設備（計算機房空調、照明系統、安全消防系統等）的供配電問題。我公司倡導的機房供電方案是端到端無單點故障的高可靠性方案。為保障設備的長期穩定安全運行，計算機、網絡設備，機房空調通風設備按一級負荷供電。機房照明及輔助用電設備，按二級負荷供電。

1、供配電系統

1.1電力分配

本項目低壓供電系統均採用TN-S系統，交流50HZ，380/220V電源。計算機房及輔助區域用電設備包括：網絡設備、計算機及其它相關設備；動力設備、空調與新風設備、排風設備以及一般照明和應急照明設備等。

市電經過低壓配電櫃配電後分別向UPS、精密空調、維修插座、照明等設備提供市電電源。經過UPS逆變整流後電源輸出到UPS輸出配電櫃、為機房內服務器、網絡設備、監控設備、消防設備等提供電源。

進線ATS必須與消防信號實現聯動，當火災出現時，該ATS自動分離，此時空調自動停機，門禁自動打開，以避免火災的蔓延。

在UPS輸入配電櫃內專門設置了帶鎖維修旁路斷路器，該帶鎖維修旁路斷路器由專人進行管理，用於當UPS出現災難性故障時，需要將UPS退出運行並保證整臺UPS無電時以便檢修，通過切換該帶鎖維修旁路斷路器，可臨時在不間斷供電的情況下將機房內計算機負載全部轉移於為市電供電。以滿足在檢修UPS時，能夠用市電直接向計算機供電。

1.2 UPS系統

對機房內計算機設備提供乾淨、不間斷的電源供應是極端重要的。但公用供電系統由於電網受各種因素的影響，時常有不正常的現象發生，往往會對計算機系統造不利的作用，為此，採用UPS不間斷電源極為重要，它不但能提供穩定可靠的高質量的電源，沒有瞬變和諧波，即使當電網斷電時，它也可由後備電池支撐，繼續供電，使計算機設備有一定的時間進行處理。

1.3機櫃的供電

在主機房內設置UPS輸出配電櫃，配電櫃分別向每臺機櫃輸出2條ZR-VV-3*4mm²電纜，即實現雙迴路向各計算機供電。上述的供電每一臺計算機機櫃的兩路電源中的每路電源採用一對一的方式與饋出斷路器連接。

機櫃內安裝帶有防浪湧專用PDU電源插座。

以上供電系統是一可靠性非常高的方式，整個供電系統任意一點（如市電同時停，配電櫃故障，某一電纜故障，或計算機負載短路跳閘等等）均不會影響機房其他計算機或機房設備的正常供電。

另外為防止UPS輸出的三相電源不平衡，必須合理均勻分佈UPS輸出配電櫃的各路輸出，使三相電源在正常工作時，負載趨於平衡。由於三相UPS如果三相負載不平衡，則該UPS會因一相超過額定值而引起UPS過載跳閘，大大減小UPS的帶載容量。另外當三相電流不平衡時使中性線電流增大，造成高次諧波干擾增大和中性點電位漂移。

1.4空調設備電源

每個功能間內的空調由市電配電櫃提供電源。當消防動作時，空調電源自動切斷，以利及時消除災情，防止火災的蔓延。

機房區域的空調統一由動力配電櫃提供電源並控制。

1.5照明、市電插座電源

照明、市電插座電源統一由動力配電櫃提供電源並控制，日常採用市電供電，當市電停電後自動轉換為UPS供電。

1.6電線電纜

機房內的電線電纜除具備相應的載流量以外，還要考慮到線纜阻燃特性等要求。

導線截面選擇：根據負荷大小、允許電壓損失、導線長時間允許溫升及導線機械強度等因素進行選擇，載流量留有30%的備用能力，以備將來擴容。

機房內所有電線電纜均於地板下沿熱鍍鋅金屬橋架和金屬電線管敷設，鋼管和金屬橋架內徑留有餘量。

天花照明電源線在天花吊頂上穿鍍鋅鋼管敷設，然後就近經軟管敷設至燈具。

一般插座和照明開關電線在輕鋼龍骨牆面通過鍍鋅鋼管鋪設至相應位置。

為防止漏電危及人身安全，並防止電磁干擾，機房內所有金屬線管、線槽、電氣設備外殼等不帶電的金屬部分都可靠接地。強電管道的佈置必須考慮與弱電管道的間距和通風情況。

機房內強電管道和弱電管道均採用：國標金屬300×75、250*50鍍鋅橋架，DG25金屬電線管、DG20金屬軟管、DG20金屬接頭、86型接、分線盒。

2、電氣安裝要求

電氣元器件的安裝整齊、牢固、正確、標誌明確、外觀良好、內外清潔。電氣接線盒（箱）內無殘留物，蓋板整齊，同類設備安裝高度一樣，電氣裝置的臺座必須安裝牢固、幹線採用壓接端子連接。

電源相線、零線、保護接地線、直流工作接地線的顏色有所區分。管內穿線總面積不超過管內截面積40%。不同迴路、不同電壓的線不能穿在同一管內。機房內電源線主幹部分採用鍍鋅橋架。

3、輔助供配電系統

為防止機房內輔助用電設備在運行時可能對計算機系統造成的干擾，一般將計算機系統中的輔助設備用電自成系統，稱為輔助供配電系統。計算機系統中的輔助用電設備主要包括各類空調、照明燈具、維護設備以及輔助插座等。

輔助插座主要有以下作用：

1)為計算機的通用測試設備提供臨時電源；

2)為臨時局部照明提供臨時電源；

3)為清掃機房的吸塵器等設備提供臨時電源；

這些設備通常採用單相電源，電壓為220V。這樣每個輔助插座至少應具有二孔和三孔各一個，總容量一般小於15A，每12-15m2左右設置一組。在機房內通常將該類輔助插座安裝在活動地板上或利用暗線安裝在牆上；

4、照明系統

計算機機房主要依靠人工採光，計算機房照明質量的好壞不僅會影響計算機操作人員和軟硬件維修人員的工作效率和身心健康，因此要對機房照明系統進行科學合理的設計。

機房照明系統又分為普通照明和應急照明兩大類。

4.1普通照明

機房各類房間對照度的要求各不相同。據有關統計資料提供的數據，目前國內新建的機房照度平均照度可按400LX取值；基本工作間、第一類輔助工作間的平均照度按300LX取值；第二、三類輔助工作間及辦公室應按國家建委標準TJ34---79（工業企業照明設計標準）執行。與此同時，設計中對機房照明的均勻度、穩定性、光源的顯色性、眩光和陰影等指標也要認真考慮。

針對該機房工程的具體情況，機房區域設計採用600×600嵌入式格柵燈具。

燈具位置排列根據機櫃位置進行排列，使機櫃前後均有較高的照度，避免了燈具設置在機櫃正上方。

4.2 應急照明

應急照明又稱事故照明，是在正常照明因故障或停電熄滅時提供的照明。國家標準《電子計算機機房設計規範》中規定：主機室及已記錄的媒體存放間中距地面0.8米處的測量照度不應低於150LX。

此外，應急照明的電源控制要能快速、可靠地自動投入，且與正常照明電源聯動，即在正常照明斷電後應急照明的電源應能迅速、可靠地接通。

4.3 事故照明

在機房大門內側門框上方分別安裝一套電子式出口指示燈，以作消防事故發生後的消防通道標記。

5、防雷接地系統

計算機的接地系統是消除公共阻抗的耦合，以及防止寄生電容耦合的干擾，保護設備和人身的安全，保證計算機系統穩定可靠地運行的重要手段。因此為了能保證計算機安全、可靠、穩定地運行，保證設備、人身的安全，針對不同類型計算機的不同要求，設計出適當形式的接地系統。

在國家標準《計算機站場地及時要求》中多計算機站接地系統的要求做了具體的規定。

計算機機房一般具有以下幾種地：

1) 交流工作接地;

2) 計算機系統的弱電接地;

3) 安全保護接地;

4) 防雷保護接地；

5) 防靜電接地；

接地電阻及相互關係：

① 交流工作地的接地電阻應不大於4Ω；

② 安全保護地的接地電阻應不大於4Ω；

③ 防雷保護地的接地電阻應不大於10Ω；

④ 計算機直流工作接地電阻的大小、接法以及諸地之間的關係，應依不同計算機系統的要求而定。一般要求該電阻不大於1Ω；

所謂接地，既把電路中的某一點或某一金屬殼體用導線與大地連在一起，形成電氣通路。其目的是讓電流易於流到大地，因此從這個意義上講，希望接地電阻是越小越好。另外計算機系統的接地，還希望在接地電流受某種外界條件影響而數值發生變化時使接地點的電位隨之變化而產生的噪音應儘量減少。所以接地電阻也是越小越好。

5.1 機房接地方案

基於該工程的接地系統現狀和實際情況以及地當今計算機的特點，我公司設計採用以下接地方案：

(1)防雷接地： 防雷接地系統使用大樓的防雷接地系統。

(2)交流設備保護地：從大樓市電總低壓配電屏沿五芯供電電纜的地線將大樓綜合地送入計算機房，在機房形成安全保護接地母線。然後將機房的天花板、牆板、防靜電地板等通過分支接地線接入防靜電接地母線，將所有的機櫃外殼、機架、金屬橋架、金屬電線管等不導電的金屬部分通過分支接地線接入安全保護接地母線。在機房區的抗靜電地板下設置一組接地銅排以實現等電位接地，避免各種地在機房內不同點形成電位差。

(3)防靜電釋放地

機房地板下采用30*0.3銅皮做1800*1800網格，地板支腳壓於交匯點處。機房選用S型等電位接地網絡，從大樓市電總低壓配電屏沿五芯供電電纜的地線將大樓綜合地送入計算機房，在機房形成防靜電釋放地接地母線。然後用網格編織多股銅線對天棚龍骨、金屬吊頂、抗靜電地板支架、牆體金屬龍骨、金屬風管、隔斷金屬骨架等可能產生靜電的金屬、非金屬件通過分支接地線接入機房內分區設置專用接地端子匯接箱。機房內的各種箱體、殼體、機架等金屬組件均分區接到端子匯接箱中。周邊採用機房原有就銅排做等電位銅排。

5.2 電源防雷

防雷最重要的就是等電位連接，把各種設備利用防雷器、線材、鋼筋和銅排進行設備的電氣處理。使各種設備在雷電流入侵時行成等電位體，這樣設備與設備、設備與大地、設備與大樓整體結構之間就不會產生電位差。利用這些材料和設備把雷電流引入大地，從而保護設備。

電源防雷應保護配電系統、UPS、用電設備等。電源防雷採用三級分流法。第一級採用70KA的防雷器，將較大部分的雷電源分流到地下。它安裝在大樓進線的總配電櫃的後面。第二級防雷採用40KA的防雷器，它能進一步的衰減雷電流。它的安裝位置在UPS輸入配電櫃中。第三級防雷採用10KA的防雷器，它能更進一步的衰減雷電流，接進正常值。它的安裝位置在UPS輸出配電櫃中。

針對該工程的具體情況，我司整體設計時採用如下設計方案：

設置電源應急開關，在火災發生時立即切斷機房總配電櫃電源。該應急開關既可與自動報警系統聯動，也可以手動。

在選擇機房內設備、元器件、導線、電纜等的品種、規格時充分考慮了它們的過載能力和防火阻燃性能，防止因過熱導致溫升過高而引起火災。

選擇機房內材料時充分考慮了其阻燃能力。

2、設計思路

依據該機房的氣體防護區面積，及使用功能不同進行分隔，故將整個機房劃分為三個氣體防護區。網絡機房、值班室。當報警設備感應到火災信號後啟動滅火裝置，聯動門禁系統開門、配電櫃斷電。

3、工作原理

氣體滅火控制器在接收到感煙或感溫探測器中任意一個發出的火災信號後，立即發出報警聲以作出警報。同時向聲光報警器發出傳輸信號，令聲光報警器發出聲光預警信號——指示燈亮。當保護區內的溫度和煙的濃度都達到響應值時，即感煙和感溫探測器同時都動作時，氣體滅火控制器會立即向聯動設備發出聯絡信號，發出聲光報警，並啟動延時器延時30秒鐘。30秒鐘以後，氣溶膠滅火裝置開始啟動，氣體過導流系統進入保護區將火撲滅。同時釋放顯示燈也動作，指示防護區內在進行滅火，防止人員進入。整個氣體噴放過程在60秒內完成。在需要人工啟動或停止啟動時，還可以使用安裝在防護區外的手動控制盒以實現現場手動控制功能。

4、控制方式

本系統主要有自動、手動、機械應急手動和緊急啟動/停止四種控制方式。

（1）自動控制方式：本滅火控制器配有感煙火災探測器和定溫式感溫火災探測器。控制器上有控制方式選擇鎖，當將其置於"自動"位置時，滅火控制器處於自動控制狀態。當只有一種探測器發出火災信號時，控制器即發出火警聲光信號，通知有異常情況發生，而不啟動滅火裝置釋放滅火劑。如確需啟動滅火裝置滅火時，可按下"緊急啟動按鈕"，即可啟動滅火裝置釋放滅火劑，實施滅火。當兩種探測器同時發出火災信號時，控制器發出火災聲、光信號，通知有火災發生，有關人員應撤離現場，併發出聯動指令，關閉風機、防火閥等聯動設備，經過一段時間延時後，即發出滅火指令，打開電磁閥，啟動氣體打開容器閥，釋放滅火劑，實施滅火；如在報警過程中發現不需要啟動滅火裝置，可按下保護區外的或控制操作面板上的"緊急停止按扭"，即可終止控制滅火指令的發生，不啟動滅火裝置，釋放滅火劑，實施滅火。

（2）手動控制方式：將控制器上的控制方式選擇鎖置於"手動"位置時，滅火控制器處於手動控制狀態。這時，當火災探測器發出火警信號時，控制器即發出火災聲、光報警信號，而不啟動滅火裝置，需經人員觀察，確認火災已發生時，可按下保護區外或控制器操作面板上的"緊急啟動按鈕"，即可啟動滅火裝置，釋放滅火劑，實施滅火。但報警信號仍存在。

無論裝置處於自動或手動狀態，按下任何緊急啟動按扭，都可啟動滅火裝置，釋放滅火劑，實施滅火。同時控制器立即進入滅火報警狀態。

（3）應急機械啟動工作方式：用於控制器失效時，當職守人員判斷為火災時，應立即通知現場所有人員撤離現場，在確定所有人員撤離現場實施滅火。

（4）緊急啟動/停止工作方式：用於緊急狀態。情況一，當職守人員發現火情而時氣體滅火控制器未發出聲光報警信號時，應立即通知現場所有人員撤離現場，在確定所有人員撤離現場後，方可按下緊急啟動/停止按鈕，系統立即實施滅火操作；情況二，當氣體滅火控制器發出聲光報警信號時並正處於延時階段時，如發現為無報火警時可立即按下緊急啟動/停止按鈕，系統將停止實施滅火操作避免不必要的損失。

第六章 動力環境集中監控系統

1、概述

隨著計算機技術的發展和普及，計算機系統數量與日俱增，其配套的環境設備也日益增多，計算機房已成為各大單位的重要組成部分。機房的環境設備（如供配電、UPS、空調、消防、安保等）必須時刻為計算機系統提供正常的運行環境，一旦機房環境設備出現故障而又得不到及時的處理，就會影響到計算機系統的運行並對數據傳輸、存儲及系統運行的可靠性構成威脅，尤其是對於稅務機構這種需要實時交換數據的機房，其機房內設備的正常運做是保障國家利益的關鍵，所以機房管理就顯得更為重要，一旦系統發生故障，造成的影響和經濟損失將是不可估量。

因此，對機房的監控和管理就成為了一個新的難題，而智能化機房監控系統就可以很好的解決這些問題，實現以下幾個功能：

為相關設備提供全面的保障

減少管理人員的工作負擔

3、系統選型

3.1系統分析

機房工程機房約80m²，監控系統將通過智能化途徑、開放型數據接口對機房進行全方位、全天候的密集監控，包括機房動力監控、機房環境監控和機房保安監控三個子系統共七類監控對象。同時在集中監控中心要對機房做到4種管理方式：性能管理、告警管理、配置管理和安全管理。

監控系統還必須滿足無地域查看信息和接收報警的要求，這樣用戶就可以在任何有網絡的地方、任何可以使用無線電話的地方實時接收查看到被監控對象的信息和報警，並且可以通過網絡對設備進行操作控制。

3.2機房環境監控

（1）空調監控系統

通過空調自帶智能通訊接口，系統可實時、全面地監控空調各部件（壓縮機、風機、加熱器、加溼器、去溼器、濾網等）的運行狀態與參數，並可遠程修改空調設置參數（溫度和溼度），並能實現空調的遠程開關機。

（2）漏水監測系統

漏水檢測系統。當發生漏水時，系統會自動啟用對外報警系統，根據用戶選擇，系統會自動撥打用戶預先設定的電話號碼和發送短消息等。

漏液檢測，由一條檢測液體洩漏的感應線纜和報警控制器兩部分組成。當洩漏發生時，感應線纜與水或其它液體接觸，就會立即將信號傳給報警控制器，紅燈亮，並同時報警。正常情況下，控制器綠燈亮，安全無事故；黃燈亮時，表明線纜有故障，請時維護。當洩漏清理好後，線纜很快便乾爽而報警器即回覆監察工作。

3.3機房動力設備監控

（1）電源參數監測系統

實時監控機房內市電輸入櫃的三相電的各重要參數，如電壓、電流、功率（有功、無功視在）功率因數、頻率電度（有功、無功）等。現在，配電櫃的供電參數通過智能電量儀表把市電的運行狀態以數據的形式傳輸給監控系統，對其進行監視，使現場達到無人值守的目的。

配置智能電量檢測儀，實時監視進線櫃的三相電壓、電流,頻率,功率因數，有功功率,無功功率。系統管理員和操作員可以通過歷史曲線圖中見到每天的電壓、頻率、有功、無功的最大值、最小值、當前值及電壓、電流峰值。使能清楚地知道三相電壓、電流是否均衡。

（2）開關狀態監測系統

實時監測機房內配電櫃的各重要的配電開關，其狀態監控是十分必要的，一旦開關跳閘，計算機系統可能會立即崩潰，需要儘快報警處理。

機房電源室配電櫃，重要開關共計6路，設計通過ICS-D8H與I-7052模塊對信號進行採集和轉換後，將其全部納入開光狀態監測系統中。

（3）UPS監控系統

通過由 UPS廠家提供的通訊協議及智能通訊接口，對機房內 UPS 進行監控，對UPS內部整流器、逆變器、電池、旁路、負載等各部件的運行狀態進行實時監視，一旦有部件發生故障，系統會自動報警。並且實時監視UPS的各種電壓、電流、頻率、功率等參數，並有直觀的圖形界面顯示。

3、線纜布放

布放線纜的規格、路由和位置應符合施工圖的規定，線纜排列必須整齊，外表無損傷。線纜在轉彎處應注意彎曲率半徑應大於50mm。

建議統一採用暗敷形式的鋼管和金屬橋架敷設線纜，鋼管之間、橋架之間、鋼管與橋架之間應保持可靠的電氣連接，並做跨接地線，橋架的規格應當保證在放完線纜後有的填充度不超過60％。

在從橋架至室內信息點底盒的管道選擇上，建議每組信息點中，三個UTP統一放在直徑為25mm的金屬管內，防靜電活動地板下的線纜，應順直不凌亂，避免交叉，並且不得堵住送風管道。

網絡部分

第一章 項目設計原則

根據業務應用環境及需求，本次網絡系統平臺規劃採用以下基本原則：

安全性、可靠性

網絡系統應嚴格按照國家有關信息安全標準進行建設，可以阻斷非法用戶訪問，防止合法用戶的越權訪問。通過防火牆、入侵檢測、入侵防禦、漏洞掃描、安全審計、主機安全管理、上網行為管理、網頁防篡改以及防病毒等不但能保證主機安全，網絡安全而且還能確保應用安全及數據安全。

安全的前提在於設備可靠、可用，一方面，當網絡服務不可用時，不管是何種原因，網絡就失去了實際價值。另一方面，當網絡服務中斷恢復時間不可控時，說明網絡系統已經不可靠、不可用。

電信級高可靠的網絡設備，設備關鍵功能部件，包括電源、路由協議處理、交換和控制等均具備1+1備份能力，規避單點故障，提供設備可靠性；設備所有關鍵部件支持熱插拔，不但便於維護，而且能保證網絡業務的連續性。

合理性

信息技術的飛速發展，合理的網絡架構對網絡系統規劃和設計尤為重要，分層網絡架構設計不但能做到建網成本合理化，而且還能充分挖掘設備潛力，實現合理、靈活組網的目的。

先進性、成熟性、穩定性

基礎網絡建設具有投資大、週期長等特性，因此網絡規劃設計時必須保證網絡在相當長的時間內不會由於技術、標準等原因而被淘汰。採用先進的、成熟的、符合標準的技術不但能滿足網絡性能和穩定要求，而且還能兼容技術未來發展。

開放性和互連、互通性

開放性意味著遵循大多數網絡系統所共同遵循的標準，以實現內部各系統之間，以及與其它系統以及與第三方設備的互連互通，開放性還意味著更多的選擇和最佳的性能價格比，有利於在眾多滿足開放標準要求的硬件、軟件系統中選擇最符合要求的產品。

網絡規劃設計應採用國家、國際標準化組織制定的標準規範，符合開放系統互連互通，便於業務功能及網絡的統一管理和維護。

可管理性和可維護性

網絡管理已經越來越受到人們的重視，因為它關係到網絡系統的使用效率、維護、監控甚至系統資源的再分配；因而一體化完整的系統管理能力至關重要。網絡管理包括配置管理、故障管理、性能管理等基本網絡管理功能，同時應具有強大的用戶管理和安全控制策略，並將各種接入設備、用戶終端設備等納入統一管理的範疇。

可擴展性

網絡系統的可伸縮性已成為網絡規劃設計非常重要的一環，良好的規劃設計應能方便地對網絡系統規模或技術進行擴展。用戶對網絡資源的需求經常隨著應用而發生變化，系統應具有一定的靈活性，為滿足用戶的不同需求而作靈活的系統配置和資源的再分配。網絡系統擴展性主要包括：

1）網絡規模的擴展，包括網絡的地理分佈，用戶數；

2）應用內容的擴展，網絡平臺將不僅承載數據業務傳輸，而且也肩負語音、語音會議、視頻會議等業務的傳輸。這就要求網絡系統平臺必須具有多種業務支持的能力。

3）網絡容量的擴展，隨著規模和應用的擴展，網絡容量也必須能具備擴展能力。

4）網絡設備的擴展，模塊化架構的網絡設備在可伸縮性上亦有著固定式系統無法比擬的優越性。整個系統的性能將能隨著模塊數量的增加而得到相應的增加，因此也就更能適應不同規模網絡對設備的要求。模塊化的網絡設備在多種技術的適應能力上也具有相當大的靈活性。

網絡系統規劃設計應當考慮系統平臺平滑升級的能力，使系統能滿足各種用戶對應用處理不同程度的需求，以及逐步升級的發展規劃，以節約投資避免系統性能的閒置和浪費。

第二章 網絡規劃方案

1、總體思路

信息系統網絡以應用為核心，多種通信方式並存，跨平臺、支持分佈式處理的計算機網絡系統。針對網絡應用和特點，採用分區、分層、分類的模塊化設計思想保證網絡架構的安全可靠、高擴展能力。

分類：信息系統基於業務功能分為內網系統、外網系統、監控應急系統、無線網等4套網絡，4套網絡物理隔離。

內網系統：承載公文流轉、OA系統、視頻會議、IP電話系統等，實現內部各級部門互聯

外網系統：與Internet互聯，承載內部上網、對外公共信息服務等業務

監控應急系統：承載安防視頻監控、報警系統、溫控、動環、應急管理、廣播系統、智能停車系統等；

無線網：主要對大樓部分區域進行無線覆蓋，方便辦工和多動終端的接入。

分層

4套網絡均採用核心、接入兩層分層架構。核心層負責數據的高速交換，提供快速路由和高交換能力，並提供相應的策略控制等，如安全控制，路由、地址聚合等；接入層根據用戶需求，接入最終用戶，接入層也可實現部分策略控制，如VLAN劃分，安全准入控制等。

分區

根據用戶和業務的不同及安全要求，將網絡劃分域便於安全控制和統一管理，如安全檢測域、安全接入域、服務器域、互聯域等。

分類、分層、分區模塊化設計的優點：

(1) 可擴展性：網絡規模或者業務增長不會導致網絡大規模改動，局部或者增加部件即可滿足網絡增長要求；

(2) 簡單性：通過將網絡分成許多功能模塊，降低了網絡的整體複雜性，使故障排除更容易，能隔離廣播風暴的傳播、防止路由循環等潛在的問題；

(3) 靈活性：使網絡容易升級到最新的技術，升級任意層次的網絡不會對其他層次造成影響，無需改變整個環境；

(4) 可管理性：層次結構使單個設備的配置的複雜性大大降低，更易管理。

針對無線網絡中存在的射頻管理難題，提供基於集中化或內置於無線控制設備的可視化智能 RF 管理工具。藉助該管理工具所提供的強大功能，無線控制設備可以及時發現無線網絡中由於接入點失效而產生的射頻覆蓋黑洞，並通過經優化設計的 RF 算法協調失效 AP 周邊的其它 AP 資源來完成對覆蓋黑洞的有效彌補。同時無線控制設備還可以發現無線網絡中存在的射頻干擾問題，繼而利用經優化設計的 RF 算法協調整網射頻信道資源，從而為受干擾 AP 選擇新的可用信道。該新的信道信息會自動下發到受干擾的 AP 並得到執行，從而快速解決射頻干擾問題。當在現有無線網絡中為滿足覆蓋需求而增加新的 AP 時，無線控制設備也可以通過優化的 RF 算法自動決策並指導新的AP 完成無線信道及射頻功率的調整，以此實現無線接入點的快速部署。智能化的管理特性可顯著提高網絡管理者對無線網絡的部署和管理效率。

1、安全背景

經過近二十年的發展，我國醫院信息系統(Hospital Information System，HIS)建設初具規模，日趨完善。信息系統的發展經歷了從單機系統、局部網絡系統到整個醫院信息系統的多個階段，可以說，HIS系統是醫療信息化的最核心資產。從目前醫院的網絡結構上看，可以分為兩大部分，用於日常醫療信息交換的業務專網以及實時獲取Internet信息資源的辦公網。其中，醫院業務專網是醫院業務開展的重要平臺，承載著核心業務，同時具備線路出口與醫保等其他機構交換數據；而辦公網主要對外提供信息發佈門戶，對內提供Internet網絡接入等服務。

在服務患者方面，醫院基於信息化平臺，為患者提供了更多的便捷服務。隨著醫院信息化建設的深入，醫療智慧化、信息化給醫院管理、醫療服務和研究帶來巨大的創新契機。伴隨而來的的安全問題也日益突出，尤其是隨著網絡規模的不斷擴大，網絡應用項目越來越豐富，涉及到的人員越來越來越龐雜，部署策略越來越繁瑣，整個系統變得越複雜，醫院面對的安全風險也越來越大。如何有效地降低安全風險、降低安全成本，安全的策略顯得尤為重要。醫院的HIS系統是關鍵業務系統，需要系統不間斷運行。即使發生短暫的業務中斷，也會導致難以估量的經濟和名譽損失。

2、建設目標

鑑於當前的信息安全形勢，如何保證醫院的網絡正常運行和網絡安全已成為迫切需要關注的問題。

隨著醫療行業信息化發展的要求，《全國衛生信息化2003-2010年發展規劃綱要》中對信息安全提出了明確的要求：

加強與衛生信息化相關的法律、法規、政策體系的建設；

提高信息安全意識，加強計算機和網絡安全培訓，防範、打擊計算機與網絡犯罪；

在衛生信息系統建設的同時，要進行信息安全的總體設計和信息系統安全工程建設；

在系統驗收時必須對信息系統安全進行測評認證；

對於已建的衛生信息系統，要採取信息安全加固措施，進行系統安全測評認證；

衛生信息系統建設中信息安全投資應占系統投資的一定比例。

通過本次網絡系統及網絡安全規劃與建設，將延津縣人民醫院的網絡建設成為一個具有現代化、多功能、結構化、智能化的綜合性網絡系統。醫院網絡系統是為醫院內部提供網絡信息系統應用的IT基礎平臺，包括PACS/RIS系統、HIS/OA信息系統、管理、科研和多業務的綜合信息服務網絡系統。

同時，按照等級保護的要求，將滿足"安全訪問，內外隔離，分期建設"三點基本原則，安全訪問包含兩方面，即從內網可以安全訪問互聯網，從互聯網也能夠安全訪問內網；而在內外隔離上則是要保證內網數據與互聯網之間保持邏輯或物理隔離；分期建設則是建設的思路，是根據醫院的實際情況，分步驟從基礎到深入，從滿足最迫切的安全需求慢慢上升到管理安全上來。以下此方案將針對這三點進行詳細的需求描述與解決思路陳述。

第二章 需求分析

1、安全技術需求分析

1.1網絡和通信安全

外部網絡邊界

主要指業務網所有外部網絡邊界，包括：

專網邊界：該邊界位於院內業務網與其他單位互聯的專網之間。專網邊界兩側都是內部用戶，網絡環境、應用環境、用戶身份及規章制度都很接近，所面臨的網絡安全風險及相應的需求也基本類似。主要需要考慮防止內部的非法訪問以及病毒、蠕蟲等惡性安全事件的快速蔓延。

互聯網邊界：互聯網接入主要用於提供對外基於互聯網的WEB業務、內部人員的互聯網訪問，因此在同一個邊界具備兩種屬性。外網邊界直接面臨社會各界用戶，使用環境複雜面臨的安全風險極大。各類複合網絡攻擊手段以及針對網站的流量攻擊均是常見的安全威脅。應予以嚴格的安全防護手段在此邊界進行設防，維護整個內部網絡不被外部侵入。

內部安全域邊界

在內部網絡中，可以劃分處多個網絡安全域，包括多個服務器區、業務區等等。這些安全域之間存在著內部邊界，為能更加有針對性的對各安全域進行防護，在不同的邊界應採取不同的邊界防護措施。

數據傳輸安全

對於外部單位用戶的接入存在兩種方式：專線方式、互聯網方式。通過互聯網進行接入傳輸的數據屬於醫院內部的信息，這些敏感的數據信息在互聯網上十分容易被非法竊取、篡改或刪除。因此必須採用技術手段保證數據的機密性、完整性以及可用性。

1.2應用和數據安全

首先分析業務網。業務網承載著最核心HIS系統、LIS系統、PACS系統等，整個安全建設將圍繞保障業務系統的安全運行為目標。從整體上可分為外部和內部兩大類。

外聯接入

業務網外部邊界通常有兩個，一個是與醫保、銀聯等外聯單位的邊界，一個是與辦公網的邊界。需要採取措施維護業務網整體的高度獨立與安全，並同時能夠與醫保等相關外聯單位以及業務網進行必要的數據交換。

內部網絡

"超過70%的安全問題來自於網絡內部"。內部威脅比外部更復雜多樣。

病毒擴散：嚴重威脅著廣大終端和HIS業務系統的運行安全。

終端管理：終端用戶能否正確使用終端系統關係到HIS系統是否正常運行和日常業務的正常開展。誤操作、非法拷貝、系統漏洞、非法程序等均可導致終端自身和HIS系統的非正常運行。

內部邊界：HIS系統不僅面臨著外部人員的安全威脅，內部人員則更加容易威脅到業務系統的安全，因此核心區域邊界必須能夠進行從網絡層到應用層的攻擊過濾。

業務監控：解決了各類安全威脅後，業務系統自身的運行狀態還缺乏有效監控。實時監控業務系統的健康狀態，對已經發生過的運行問題要能夠回溯並發現原因，為改善系統的能力提供依據。

對於存在於醫院局域網平臺上的眾多應用系統，同樣存在著多種類型安全需求，包括：

數據庫審計

由於用戶的計算機相關的知識水平參差不齊，一旦某些安全意識薄弱的管理用戶誤操作，將給信息系統帶來致命的破壞。有必要進行基於數據庫的審計。從而威懾那些心存僥倖、有惡意企圖的少部分用戶，以利於規範正常的網絡應用行為。

系統風險評估管理

對於外網平臺上大量的服務器系統，因此，對服務器各項服務的安全配置就顯得尤為重要，如果有一點疏忽也會直接造成審計署信息系統被攻擊。我們建議定期/不定期的全面掌握網絡設備、安全設備、主機、應用系統、數據庫系統的風險情況，並以此在安全事件發生前進行加固，全面提高抗風險能力。

應用安全管理

從用戶角度看，其業務系統的正常運轉是最關心的核心問題，而業務系統能否實施良好的監控管理則是關鍵因素之一。因此需要技術手段對應用系統的狀況進行全面監控，能夠全盤呈現業務環境，實施主動監控，進行運行趨勢分析，及時發現存在的問題。

病毒是對計算環境造成危害最大的隱患，當前病毒威脅非常嚴峻，特別是蠕蟲病毒的爆發，會立刻向其他子網迅速蔓延，這樣會大量佔據正常業務十分有限的帶寬，造成網絡性能嚴重下降甚至網絡通信中斷，嚴重影響正常業務開展。因此必須採取有效手段進行查殺，阻止病毒的蔓延危害整個數據中心。

2、安全管理需求分析

"三分技術、七分管理"更加突出的是管理層面在安全體系中的重要性。除了技術管理措施外，安全管理是保障安全技術手段發揮具體作用的最有效手段，建立健全安全管理體系不但是國家等級保護中的要求，也是作為一個安全體系來講，不可或缺的重要組成部分。

安全管理體系依賴於國家相關標準、行業規範、國際安全標準等規範和標準來指導，形成可操作的體系。

1.1設計原則

等級保護建設原則

醫院的核心業務系統屬國家重要信息系統，其安全建設不能忽視國家相關政策要求，在安全保障體系建設上最終所要達到的保護效果應符合《信息系統安全等級保護基本要求》。

體系化的設計原則

系統設計應充分考慮到各個層面的安全風險，構建完整的安全防護體系，充分保證系統的安全性。同時，應確保方案中使用的信息安全產品和技術方案在設計和實現的全過程中有具體的措施來充分保證其安全性。

產品的先進性原則

本次建設對所需的各類安全產品提出了很高的要求。必須認真考慮各安全產品的技術水平、合理性、先進性、安全性和穩定性等特點，共同打好工程的技術基礎。

按步驟有序性建設原則

信息安全保障體系的建設是一項長期的工程，並非一蹴而就解決所有安全問題。因此，在實際建設過程中要根據實際情況分輕重緩急，分期、分批的進行部署。

1.2網絡結構

根據業務性質以及流程的不同，劃分為辦公網安全域和內網、專網安全域。

按照等級保護基本要求，對醫院網絡劃分為兩大安全域、三張網絡。兩大安全域為：辦公網安全域和內網、專網安全域；三張網絡為：互聯網、內網和專網。採用橫向隔離、縱深防禦的基本策略，對網絡安全進行全面的安全防護。

橫向隔離：橫向隔離分為物理隔離（強邏輯隔離）與邏輯隔離，（即外網區與內網區物理隔離，辦公網與業務專網邏輯隔離）。

縱深防禦：分區分域防護，辦公網安全域分為互聯網接入、無線接入、終端接入和DMZ服務區；內網、專網安全域分為數據業務區、安全管理區、外聯接入區和終端接入區。

2、網絡技術體系

2.1邊界防火牆

辦公互聯網出口部署防火牆，防火牆技術是目前網絡邊界保護最有效也是最常見的技術。採用防火牆技術，對醫院局域網重要節點和網段進行邊界保護，可以對所有流經防火牆的數據包按照嚴格的安全規則進行過濾，將所有不安全的或不符合安全規則的數據包屏蔽，杜絕越權訪問，防止各類非法攻擊行為。

防火牆對應用能夠精確識別，感知網絡安全態勢，智能調整安全策略達到主動防禦的目的，同時集成DOS防禦、用戶認證、應用控制、入侵防禦、站點分類過濾、病毒過濾、web應用防禦、數據防洩密等多種安全防禦手段為用戶提供集成式的多方位的安全防護，利用人工智能技術對異常流量、病毒木馬變種進行檢測，不再依賴特徵對威脅進行匹配和判斷，而是通過機器學習模型判定安全威脅，能夠檢測木馬變種和未知威脅攻擊、0day攻擊等。

2.2VPN通信傳輸

使用VPN設備或採用PKI/PMI平臺體系中的完整性校驗功能進行完整性檢查，保障通信完整性及通信過程中敏感信息字段或整個報文的保密性。

2.3入侵防禦系統

在互聯網邊界部署了防火牆，對每個安全區進行嚴格的訪問控制。但是，防火牆通常不具備內容檢測的能力，不具備檢測新型的混合攻擊和防護的能力。而此邊界是最容易成為入侵目標的部分。為了確保局域網內部各服務器區的安全訪問，必須建立一整套的安全防護體系，進行多層次、多手段的檢測和防護。入侵防護系統（IPS）就是安全防護體系中重要的一環，它能夠及時識別網絡中發生的入侵行為並實時報警並且進行有效攔截防護。

入侵防禦系統可在針對WEB的攻擊前期：

抗躲避檢測——採用載體還原技術，重新組合攻擊會話，可以發現那些割裂攻擊會話攻擊企圖。

擬態識別——採用過程再現技術，模擬目標環境實現攻擊語言解析，可以發現那些企圖以多變shell或字符替換方式進行的攻擊企圖。

在攻擊分析階段：

多因子協議識別——在識別網絡數據的協議時，並不單純採用RFC的端口標準規範，而是綜合協議指紋技術，融合協議的常見字段、參數和慣用用法，對沒有采用常規端口的協議進行準確識別。

協議伺服器——支持隨時添加對新型協議的支持，而不影響原有協議識別機制。

在攻擊判定階段：

會話內和會話間的時序關聯——有一些攻擊事件可能並不是一個會話，而是由多個會話組合，這就要求檢測設備可以將多個會話關聯起來分析。

多個參數組合判斷事件——對於一些複雜的事件，需要組合多個邏輯參數變量進行判斷，任何一個單一變量都不是攻擊，但多個組合在一起，就形成完整的攻擊。

基於漏洞機理的判斷機制——對某一個漏洞而言，可能會存在上百種攻擊方法，無需為每一種新的攻擊變種定製單獨的檢測規則，而是針對漏洞的攻擊機理進行分析，不論是什麼變種，都可以及時發現。

上述技術綜合起來，確保入侵防禦產品的最大特點：精確阻斷攻擊行為，採用Bypass技術，不增加網絡單點故障在內網、專網安全域的數據業務服務器區部署入侵防禦系統，在防火牆進行訪問控制，保證了訪問的合法性之後，IPS動態的進行入侵行為的保護，對訪問狀態進行檢測、對通信協議和應用協議進行檢測、對內容進行深度的檢測。阻斷來自內部的數據攻擊以及垃圾數據流的泛濫。

2.4上網行為管理

隨著互聯網的發展，越來越多的個人利用互聯網瀏覽、發佈信息，其中必然存在大量黃、賭、毒、邪黑客等不良信息，嚴重地危害了國家安全、社會穩定以及醫院的正常業務開展。

在互聯網出口部署上網行為管理，對網站瀏覽進行控制審計，記錄所有網站訪問的記錄；對外發信息以及郵件進行審計，在出現一些安全事件的時候，能夠及時完整的進行溯源和追蹤；同時根據上網的不同需求，針對特定的應用程序進行流量管控，避免出現網絡訪問擁塞的情況出現。

2.5外聯防火牆

醫院因為業務需要，將會與醫保、新農合、遠程會診等進行對接，對於外部網絡，有很大的不安全因素，因此在外聯接入區部署防火牆，對來自外部的網絡進行精細粒度的訪問控制，防範來自外部網絡的安全威脅。

2.6安全隔離網閘

辦公網需要連接互聯網，對於內網、專網安全域將帶來大量的安全隱含，根據等級保護要求，不同安全域之間必須進行物理隔離。防火牆無法高度保證傳輸內容、協議、數據的安全性，在兩大安全域之間部署安全隔離網閘，對不同安全級別的網絡進行安全防護。

辦公網的用戶和應用系統通過安全隔離網閘訪問內網單個部門服務器安全域的相應數據庫完成業務處理，並將業務處理結果，按照用戶部門的不同，存儲在單個部門服務器安全域中、訪問用戶所在的部門的數據庫中，完成用戶通過互聯網對相關業務服務器的訪問。同時內網用戶通過網閘可以在內網服務器區的相應服務器上提交數據供本醫院同互聯網上的數據進行交換和同步。

通過這種方式，可以為訪問提供更高的安全性保障。安全隔離網閘兩側網絡之間所有的TCP/IP連接在其主機系統上都要進行完全的應用協議還原，還原後的應用層信息根據用戶的策略進行強制檢查後，以格式化數據塊的方式通過隔離交換矩陣進行單向交換，在另外一端的主機系統上通過自身建立的安全會話進行最終的數據通信，即實現"協議落地、內容檢測"。這樣，既從物理上隔離、阻斷了具有潛在攻擊可能的一切連接，又進行了強制內容檢測，從而實現最高級別的安全。

3、業務安全體系

3.1數據中心防火牆

訪問控制是網絡系統安全防範和保護的主要策略之一，它的主要任務是保證系統資源不被非法使用，是系統安全、保護網絡資源的重要手段。而安全訪問控制的前提是必須合理的建立安全域，根據不同的安全訪問控制需求建立不同的安全域。

通過對系統網絡的邊界風險與需求分析，在網絡層進行訪問控制部署防火牆產品，同時設置相應的安全策略（基線），對所有流經防火牆的數據包按照嚴格的安全規則進行過濾，將所有不安全的或不符合安全規則的數據包屏蔽，杜絕越權訪問，防止各類非法攻擊行為。

數據中心區域部署防火牆，防火牆技術是目前網絡邊界保護最有效也是最常見的技術。採用防火牆技術，對醫院局域網重要節點和網段進行邊界保護，可以對所有流經防火牆的數據包按照嚴格的安全規則進行過濾，將所有不安全的或不符合安全規則的數據包屏蔽，杜絕越權訪問，防止各類非法攻擊行為。

防火牆對應用能夠精確識別，感知網絡安全態勢，智能調整安全策略達到主動防禦的目的，同時集成DOS防禦、用戶認證、應用控制、入侵防禦、站點分類過濾、病毒過濾、web應用防禦、數據防洩密等多種安全防禦手段為用戶提供集成式的多方位的安全防護，利用人工智能技術對異常流量、病毒木馬變種進行檢測，不再依賴特徵對威脅進行匹配和判斷，而是通過機器學習模型判定安全威脅，能夠檢測木馬變種和未知威脅攻擊、0day攻擊等。

3.2數據庫防護與審計系統

網絡審計系統綜合起來講，有兩大功能：數據庫審計和系統操作審計。

網絡審計功能充當了網絡攝像機的功能，真實地記錄用戶的操作；並且，當發現不符合規定的越權操作時，能及時告警並阻斷；通過系統提供的審計記錄能迅速地查找出違規者的真實身份。

網絡審計系統是通過網絡旁路偵聽的方式對網絡數據流進行採集、分析和識別，並對應用層協議進行完整還原，根據制定的安全審計策略進行審計響應。

網絡旁路監聽技術的優勢是：不影響用戶的網絡結構；不影響用戶的網絡性能；不影響用戶的應用業務。

網絡審計系統具體的功能包括：支持網絡管理行為審計如針對FTP、TELNET協議的審計；數據庫行為審計如查詢（Select）、插入（Insert）、刪除（Delete）、創建（Create）等SQL命令以及存儲過程的執行進行審計和分析；以及其他一些網絡行為的審計如HTTP、NETBIOS、SMTP、POP3等進行審計。

針對內網的關鍵服務器，將通過部署數據庫審計系統，對用戶行為、用戶事件及系統狀態加以審計，從而把握數據庫系統的整體安全。

在核心數據服務器區部署數據庫審計系統，包括數據庫審計插件以及數據庫審計控制檯。審計控制檯採用分級模式，可以實現向下一級或引擎下達管理策略，同時處理來自下一層監控中心的信息與事件。

面對網絡中的用戶操作行為（如讀、寫記錄等）、用戶事件（如用戶帳號的創建、刪除等）和系統狀態（如數據庫的啟動和關閉等）加以審計，審計信息作為安全事件分析和追蹤的基礎。通過抓取網絡中的數據包，並對抓到的包進行分析、匹配、統計，通過數據庫遠程連接進行分析，從數據庫訪問操作入手，對抓到的數據包進行語法分析，從而審計對數據庫中的哪些數據進行操作，可以對特定的數據操作制定規則，產生報警事件。

4、終端安全體系

在進行業務訪問和數據處理的過程中，內部洩密和內部攻擊已經成為威脅網絡安全應用的最大隱患。在內部辦公終端主機上統一部署內網安全管理系統，通過對終端和訪問行為進行限制和保護，達到安全業務訪問的目的。同時，需要在內網安全管理服務器區中部署內網安全管理系統的管理主機服務器、控制檯、數據庫，對內網終端主機和必要的其他終端主機進行統一的管理。

在內部安全域的計算機終端上面部署內網終端管理系統，可實現終端安全加固、網絡接入控制、非法外聯控制、資產管理、I/O接口管理、終端配置維護、終端審計監控等功能。

4.1終端安全加固

實現補丁管理，對內網終端計算機補丁狀態進行定期檢測並自動安裝與更新。

實現防病毒軟件監測，判斷終端計算機是否安裝了防病毒軟件、防病毒軟件運行是否正常以及病毒庫是否保持最新等情況，並對於未進行防病毒軟件部署的主機進行內網接入限制

實現主機防火牆功能，有效防範網絡入侵和攻擊行為

4.2網絡接入控制

對接入內網的終端計算機進行身份鑑別或者安全狀態檢查，阻止未授權或不安全的終端計算機接入內網和訪問內網資源。

4.3非法外聯控制

通過控制外接設備的使用和終端計算機的撥號行為進行網絡非法外聯控制，充分保證內網計算機安全性

4.4資產管理

實現終端計算機的硬件配置（包括CPU類型、主頻、內存、硬盤、顯示卡、網卡等等）的自動登記，使網管人員在控制檯的機器上，可以觀察到各個機器的配置信息。

能夠自動將終端計算機的操作系統、安裝的軟件、運行的程序和服務、系統日誌、共享資源、以及補丁、端口等信息統計彙總，並可進行分類管理。

4.5移動存儲管控

移動存儲設備管理是對通用移動存儲設備和專用U盤從註冊、使用、銷燬整個生命週期提供完整的授權、控制和管理，有效地保證了移動存儲介質在其整個生命週期中的可控性、安全性和可審計性。

4.6終端配置維護

通過終端管理系統，IT管理人員可獲得終端計算機各種相關信息，如主機名、IP地址、網絡參數、帳戶信息等。

IT管理人員可以響應遠程終端計算機的協助請求，臨時接管遠程終端計算機，進行本地化操作。

4.7主機監控審計

監控功能包括服務監控、進程監控、硬件操作監控、文件系統監控、打印機監控、非法外聯監控、計算機用戶賬號監控等。

審計功能包括文件操作審計、外掛設備操作審計、非法外聯審計、IP地址更改審計、服務與進程審計等。

5、運維管理體系

5.1運維堡壘機

運維審計系統是針對業務環境下的用戶運維操作進行控制和審計的合規性管控系統。它通過對自然人身份以及資源、資源賬號的集中管理建立"自然人—資源—資源賬號"對應關係，實現自然人對資源的統一授權，同時，對授權人員的運維操作進行記錄、分析、展現，以幫助內控工作事前規劃預防、事中實時監控、違規行為響應、事後合規報告、事故追蹤回放，加強內部業務操作行為監管、避免核心資產（服務器、網絡設備、安全設備等）損失、保障業務系統的正常運營。

運維審計系統由WEB模塊、協議代理模塊、行為審計模塊和應用發佈模塊和存儲模塊組成。

WEB模塊：為用戶提供web方式訪問系統的界面。管理員用戶在界面中進行運維用戶管理、設備及帳號管理、用戶授權管理和運維審計管理等管理功能；運維用戶在界面中進行資源單點登錄等操作。

協議代理模塊：實現對主機、數據庫、網絡設備維護過程中的協議數據包代理轉發、行為還原及記錄、高危/違規行為阻斷等功能。

行為審計模塊：實現對行為操作的審計功能，包括實時高危/違規行為的告警、實時監控、歷史數據檢索及報表統計等功能。

應用發佈模塊：安裝在Windows服務器上，用於發佈非標準協議或應用客戶端，如IE、plsql、sqlplus等。可實現對應用客戶端工具的自動調出、密碼代填和操作審計功能。

5.2漏洞掃描

建立漏洞掃描管理體系。在系統中部署漏洞掃描軟件，結合後期的安全服務，制定相應的管理制度，用戶可以建立自己的主動安全防護體系。漏洞掃描軟件可以裝在移動平臺上或裝在幾臺固定的主機上，定期對網絡進行掃描，提前發現漏洞提前修補。網絡漏洞掃描與評估系統，包括網絡模擬攻擊、漏洞檢測、報告服務進程、提取對象信息、風險評估和安全建議等功能，幫助用戶控制可能發生的安全事件，最大可能地消除安全隱患。該系統具有強大的漏洞檢測能力和檢測效率、貼切用戶需求的功能定義、靈活多樣的檢測方式、詳盡的漏洞修補方案和友好的報表系統，並支持在線升級。

同時，漏洞掃描的部署，可以與入侵檢測系統配合，構成網絡安全評估系統，實施的將系統安全漏洞信息與入侵事件相結合進行分析。

5.3日誌審計

為了解決網絡不斷出現的安全問題，部署了大量安全設備，同時也具備了專業的技術隊伍，對信息系統進行安全維護和保障。但由於IT環境中存在較多的安全設備和大量的日誌信息，安全管理人員面對眾多的控制檯界面、告警窗口和日誌信息，往往束手無策，導致工作效率低下，難以發現真正的安全隱患。而部署的各個安全設備只能分別防範來自某個方面的安全威脅，這樣形成了一個個安全防禦的孤島，無法產生協同效應。

新頒佈的《網絡安全法》中，對系統日誌做了明確要求，要求對日誌必須要有不少於180天的保存週期。

日誌審計系統，通過SYSLOG被動接受和主動探測資產手段，對網絡中的大量設備進行探測，並且收集日誌信息。通過對網絡安全設備的安全事件以及服務器主機的syslog進行深度分析和關聯審計，為醫院提供全方位的安全防護服務。通過其它安全產品的主動防禦手段和聯動協調手段，在網絡內部建立一個立體的防禦體系，為用戶提供一個安全的網絡環境。通過對網絡基礎設施的安全日誌事件，對海量的安全日誌事件進行智能分析，評估網絡安全風險，定位系統脆弱點，提前進行預警。

1、信息化建設的重要性

全國醫療制度的改革正在把病人-醫院的二元關係改變為病人-院-險機構-府監督的多元關係。醫院要強化自身內部的管理，醫院要加強醫療服務質量的自我監督、自我控制的能力。國內醫院分級評審的一個重要考查項目，國內每個醫院的信息系統將成為我國廣域醫療信息網絡的基礎。

國家即將進行衛生資源區域規劃，實現局部和整體相結合的衛生醫療資源共享。為此，建立醫院管理信息系統才能適應當代社會發展的需要，為將來納入社會化管理、共享衛生資源奠定基礎。

數字化醫院建設可以優化各種工作環節，良好的信息化系統對醫務人員來說能將他們從繁重的事物性工作中解脫出來，可以將精力集中到臨床業務的研究和提高中；對病人來說能加快就診速度，緩解 "三長一短"（掛號、收費、取藥時間長、醫生看病時間短）的矛盾，同時住院病人費用能做到了"日日清"；對醫院來說一方面利用信息化手段，改善了醫院的服務質量，加強了患者監督能力，贏得了聲譽，同時可以利用規範化的流程，把難以量化的東西全部量化，堵住了收費、藥品管理中的漏洞，降低了成本，減輕了病人負擔，也提高了醫院的競爭力。

2.1超融合軟件定義計算設計

超融合設備的特點在於在通用x86服務器通過安裝虛擬化軟件和分佈式存儲軟件，構建計算存儲網絡相融合的雲平臺環境，實現軟硬件一體化交付。對客戶來說，這種開箱即用的解決方案可以幫助用戶快速搭建虛擬化平臺，快速部署任何基於虛擬機的業務，真正實現業務的快速上雲。

另一方面，超融合系統使用戶擺脫了對於傳統共享存儲的依賴。超融合設備基於分佈式存儲系統，最大化利用了x86服務器的本地存儲空間，構建成一個統一的存儲資源池為虛擬機提供存儲空間。超融合架構相比於傳統存儲架構的優勢在於，無需購買昂貴的存儲設備以及軟件服務，而且可以真正的將計算資源和存儲資源進行統一化的管理，從資源調度的角度來說，這將是極大的便利。

總體來說，超融合設備基於服務器虛擬化和軟件定義存儲技術實現對計算、存儲以及網絡資源的池化，並通過相關虛擬化技術保障業務的連續性和冗餘性。滿足用戶想要快速新建私有云數據中心，快速實現業務上雲，降低運維複雜度，提高資源交付效率的實際需求。

醫院應用系統比較豐富，傳統IT架構中除HIS、PACS等核心業務之外的應用系統，容災保障機制較少；且隨著現代化智慧醫療的發展，醫院也需要有隨之準備新業務上線的準備。超融合架構的特點非常完美的解決的醫院信息化發展的難題。

超融合優化功能示例

CPU綁定

通過CPU綁定選項，可以精確控制虛擬機CPU在主機的物理核心之間分佈的方式。使用CPU綁定可以向虛擬機分配特定處理器，通過此操作，可以將虛擬機只分配給多處理器系統中特定的可用處理器。對於某些重要業務來說，令其獨佔CPU資源，解決了計算資源爭用的問題。

CPU/內存熱添加

由於前期調研或規劃不足導致虛擬機的CPU或內存資源緊缺，因此需要擴容CPU或內存資源。若需擴容的虛擬機運行是業務連續性要求比較高的系統，可以通過CPU或內存熱添加功能在不停機的情況下調整CPU或內存資源。這是一種能夠在虛擬機運行期間不中斷業務便可以動態增加計算資源的管理功能。

CPU QoS

CPU的QoS只在各虛擬機競爭計算資源的時候才發揮作用，如果沒有競爭情況發生，有需求的虛擬機可以獨佔物理CPU資源。CPU份額是在多個虛擬機競爭物理CPU的時候按比例分配計算資源。以一個主頻為2.8GHz的單核物理機為例，如果上面運行有三臺單核的虛擬機。三個虛擬機A、B、C，它們的份額分別設置為低，正常，高。當三個虛擬機內部都運行滿CPU負載的應用時，Hypervisor會根據三個虛擬機的份額按比例分配計算資源，份額為低的虛擬機A的計算能力約為400MHz，份額為正常的虛擬機；B獲得的計算能力約為 800MHz，份額為高的虛擬機；C獲得的計算能力約為1600MHz。（以上舉例僅為說明CPU份額的概念，實際應用過程中情況會更復雜）。CPU份額可以控制一個服務器上的各個虛擬機在競爭CPU資源時的資源佔用率，以便針對不同等級的用戶提供服務。

裸磁盤映射

裸磁盤映射通過將SCSI LUN或本地硬盤以塊設備的方式直接映射到虛擬機中，可以為虛擬機提供接近物理磁盤的性能。當虛擬機對裸磁盤執行讀寫操作時，虛擬機內的塊設備前端驅動將對應的I/O操作傳遞給半虛擬化驅動後端，後端驅動再將這個I/O操作作用在實際的物理存儲設備，並將其應答返回給虛擬機，為高I/O的應用提供支持，保證應用的讀寫性能，發揮應用的最大優勢

裸磁盤映射一般用於虛擬機的大容量存儲需求。

虛擬磁盤快照

虛擬磁盤管理支持磁盤快照功能，採用寫時重定向技術（Redirect on Write），實現當快照後的磁盤文件被修改時，不修改原磁盤文件，而是將修改內容直接操作於一個快照文件中，並將該快照文件的父節點指向原磁盤，使得從快照文件讀取數據時，可以從原磁盤獲取數據；通過虛擬磁盤快照管理，既可以保存虛擬磁盤任意時刻的狀態，又可以恢復任意快照時刻的虛擬磁盤狀態。藉助於磁盤快照功能與磁盤移動和複製的操作，為虛擬機提供存儲數據的冷/熱遷移以及鏈接克隆等功能。

虛擬網絡調優

對於大多數的虛擬化業務場景，使用基於OVS的虛擬交換機都可以滿足其業務需求，但某些對於網絡要求較高的業務則需要更高性能的網絡技術予以支持。

I支持多種不同的虛擬化網絡，包括SR-IOV網絡，MAVTAP網絡和Pass-through直通網絡。相比於虛擬交換機，這些網絡性能更強，但是無法實現類似於虛擬交換機的網絡策略，其靈活度不如虛擬交換機。

SR-IOV

支持虛擬機掛載SR-IOV網卡，SR-IOV 技術是一種基於硬件的虛擬化解決方案，SR-IOV實現了多個虛擬機共享使用一個物理網卡的功能，並且達到直接分配的目的，提供了網絡交換的性能。

SR-IOV 使用兩種功能： 物理功能（Physical Functions，PF）是完整的帶有 SR-IOV 能力的PCIe設備，PF能像普通 PCI 設備那樣被發現、管理和配置；虛擬功能（Virtual Functions，VF）具有簡單的PCIe功能，VF只能處理I/O，每個VF都是從PF中分離出來的。每個物理硬件都有一定數目的VF可以配置，一個PF能被虛擬成多個VF用於分配給多個虛擬機，Hypervisor 能將一個或者多個VF分配給一個虛擬機。在某一時刻，一個VF只能被分配給一個虛擬機，一個虛擬機可以擁有多個VF。在虛擬機的操作系統看來，一個VF 網卡看起來和一個普通網卡沒有區別。

SR-IOV特性的使用需要硬件平臺 Intel VT-d 或者 AMD IOMMU 的支持，在使用此功能時，這些特性必須在BIOS中被啟用，如果BIOS中有SR-IOV的使能配置，那麼也需要進行開啟操作，系統支持虛擬機在使用SR-IOV類型的網卡時，進行熱添加，不支持SR-IOV類型網卡的熱刪除以及虛擬機使用SR-IOV類型網卡時的遷移功能。

MACVTAP

支持虛擬機掛載MACVTAP虛擬網卡，MACVTAP是Linux引入的一種新的網絡設備模型，用以代替傳統的Linux TAP設備加Bridge設備組合，同時支持新的虛擬化網絡技術。MACVTAP可以直接繞過內核協議棧，有效提高數據從物理網卡到虛機之間的性能，並簡化虛擬化環境中的交換網絡。MACVTAP 的實現基於傳統的 MACVLAN，和TAP設備一樣，每一個MACVTAP設備擁有一個對應的Linux字符設備，並擁有和 TAP 設備一樣的IOCTL接口，因此能直接被KVM/Qemu使用，方便地完成網絡數據交換工作。

Pass-through

網卡Pass-through也稱為網卡直通，通過Pass-through，虛擬機可以使用 I/O 內存管理單元直接對物理網卡進行獨佔使用，簡單理解就是在虛擬機添加網卡時直接將物理網卡以透傳的方式直接給虛擬機使用，即支持客戶機以獨佔方式訪問宿主機上的物理網卡，客戶機對該物理網卡的I/O交互操作和實際的物理設備操作完全一樣，不需要或者很少需要KVM的參與。

網卡的Pass-through需要硬件平臺 Intel VT-d 或者 AMD IOMMU 的支持，在使用此功能時，這些特性必須在BIOS中被啟用，系統支持虛擬機在使用網卡直通時，進行熱添加，不支持Pass-through類型網卡的熱刪除以及虛擬機使用Pass-through類型網卡時的遷移功能。

資源調度機制

負載均衡

負載均衡是對集群中主機資源的優化管理，對當前集群資源利用進行綜合評估，並採取對集群中虛擬機的合理調度，達到對集群中資源的負載均衡，以及在保證性能的前提下通過主機的待機操作（將主機電源關閉）減少能耗。

DRS：提供資源負載均衡機制（默認開啟）：

DRS提供可配置的CPU、內存利用閾值，進行定製化的配置；

DRS週期性的對集群資源利用情況進行檢測評估，並依據評估結果合理調度虛擬機，實現整個集群資源的負載均衡；

DRS提供優秀的調度策略，當集群中主機資源利用率超過閾值，DRS調度算法會對該主機上的虛擬機進行遷移預算評估，並從中最佳的虛擬機遷移組合方案，在保證能把主機利用率降到閾值的前提下，選取最小的資源遷移量；

可靠的遷移過程，DRS在遷移虛擬機過程中，會對生成的遷移策略進行排序調度，保證單個主機在同一時刻的遷入/遷出線性執行，防止併發遷移給主機增加過大的遷移壓力。

節能調度

DPM（電源管理）：在DRS基礎上提供節能調度機制：

DPM提供可配置的CPU、內存利用率低閾值；

DPM同樣會針對集群的資源利用情況，產生待機請求（即: 將會把主機電源關閉的請求），或對待機的主機進行電源打開操作；

當集群負載較低時，DPM會向DRS提出將利用率低於閾值的主機待機的請求，DRS收到待機請求，同樣會根據集群中主機的負載的情況，進行資源預判評估。從利用率低於閾值的主機中選出合理的一個，在保證剩餘主機負載不超過閾值的情況下，將該主機上的虛擬機均衡的遷移到其他主機上，並且成功處理完遷移請求之後，DPM會在等待一個週期的資源穩定之後，將該主機進行下電待機，節約能耗；

當集群負載較高時，且當前集群中的主機不足以滿足資源需求時，DRS會向DPM發出打開待機主機電源的請求，DPM會給出候選的待機主機列表，DRS會對給出的待機主機進行資源評估，並選取合適的主機，由DPM進行電源打開操作。

虛擬機備份與恢復

虛擬機備份功能旨在允許用戶從虛擬機備份恢復由於服務器硬件故障損壞或用戶誤操作損壞的虛擬機。

虛擬機快照備份

提供了一種簡便的快照機制，無需額外配置嚮導。藉助快照機制，可以在給定的時間內生成虛擬機磁盤信息和元數據的快照。生成快照時，可在需要時臨時停止IO以確保捕獲自身一致的磁盤鏡像。通過創建虛擬機的快照，捕獲虛擬機磁盤數據備份和虛擬機配置，並可以通過虛擬機快照生成虛擬機模板或生成新的虛擬機。虛擬機的快照包含該虛擬機所有的磁盤信息和虛擬機的配置信息，可以導出並還原這些信息並進行備份。

支持基於磁盤的快照和回滾，同時也支持基於內存的快照及回滾技術來保證交易的完整性。在原快照的基礎上，還支持多分支的連續快照以最大限度保證業務的連續性。

的虛擬機快照包含兩個部分：虛擬磁盤文件和虛擬機配置文件。支持靜態快照和動態快照兩種快照。要創建靜態快照，則生成虛擬機磁盤快照；要創建動態快照，則選擇生成虛擬機磁盤和內存快照，但是需要更長的創建時間。

出現故障時，通過快照恢復/還原，可以將虛擬機還原為創建快照時的狀態。快照還原將會放棄自生成快照以來對虛擬機進行的所有更改。用戶還可以將快照另存為新模板，如果用來創建快照的原始虛擬機已刪除，也可以將孤立快照另存為新模板導出。當出現故障時，重新導入模板即可完成虛擬機的恢復工作。

虛擬機備份

支持虛擬機備份功能，為虛擬機在可用存儲上創建虛擬機某一時刻的備份。特定類型的虛擬機不支持虛擬機備份：包含裸磁盤的虛擬機，包含物理網卡或SRIOV網卡的虛擬機。支持為啟動和關閉狀態的虛擬機創建虛擬機備份。

支持的備份存儲類型：本地存儲，CFS存儲，NFS存儲。有厚置備磁盤的虛擬機不支持備份到NFS存儲。由於本地存儲位於特定主機，所以不推薦使用本地存儲作為備份存儲，推薦使用CFS存儲或NFS存儲進行備份（如果虛擬機沒有厚置備磁盤）。

系統支持設置自動為虛擬機創建備份功能，通過此功能，系統可以在指定時間為指定的虛擬機創建備份。可以設置只創建一次或週期性執行。設置調度任務後， 系統會在設置的時間自動為指定的虛擬機創建備份，自動備份任務的成功與否依賴備份時的主機狀態和存儲狀態， 如果主機，網絡和存儲狀態良好，則可以成功生成虛擬機備份。

虛擬機恢復

用戶使用備份恢復功能，用指定的虛擬機備份恢復生成新虛擬機。恢復備份生成的虛擬機與源虛擬機配置相同，虛擬磁盤內的數據狀態為源虛擬機創建備份時的文件系統狀態。

恢復備份產生的虛擬機是一個全新的虛擬機（擁有全新的虛擬機名稱和ID），恢復備份過程不依賴源虛擬機狀態，生成的新虛擬機與源虛擬機互不影響。

2.2 HIS應用服務器建設

HIS應用是醫院信息化系統的中樞神經，可採用兩臺物理服務器作物HIS應用專區，單獨承載HIS應用系統，且兩臺服務器可做雙機熱備，保證業務系統的連續性。

高可用集群(HA/雙機熱備技術)就是保障用戶核心應用系統24*365小時持續穩定的運行。

集群技術是將多臺服務器聯在一起，組成一個透明的系統，這些服務器之間互相共享資源，如IP、數據或應用軟件等。對最終用戶來說，可把這個集群系統當作一個虛擬的服務器來使用。當集群中的某臺服務器由於軟件或硬件原因發生故障時，集群系統可以把IP、應用軟件等資源切換到其他健康的服務器上，即另外的服務器可立即取代該故障機的職責，繼續為用戶提供服務，使整個系統能連續不間斷的對外提供服務，從而為企業24x365的關鍵業務提供了可靠的保障。

2.3 平臺服務器集群建設

RAC服務器建設

保障數據安全與穩定是信息化建設的核心之一，為保障數據庫的性能與穩定，採用兩臺數據庫服務器做RAC集群。

Oracle RAC高可用原理

Oracle Real Application Clusters (RAC) 是數據庫高可用性 (HA) 的基礎。Oracle RAC 體系以 Oracle Grid Infrastructure 作為自己的管理框架，該框架提供計劃、意外以及運行時和按需容量管理方法，因此可確保數據中心針對各種應用程序的操作不會中斷。

Oracle ASM/CloudFS 和 Oracle Clusterware 的 Oracle Grid Infrastructure 以及包含 Oracle Real Application Clusters (RAC) 選件的 Oracle 數據庫共同構成了 Oracle RAC 體系.Oracle RAC 採用共享磁盤架構，因此卷管理和用於存儲數據庫數據的文件系統必須支持集群識別。Oracle 自動存儲管理 (ASM) 是為 Oracle 數據庫推薦的（集群）卷管理器。 Oracle ASM 管理所有數據，如Oracle 數據庫文件、Oracle 集群件文件和非結構化數據，如 二進制文件、外部文件和文本文件（通過 Oracle CloudFS 進行管理）。憑藉其較低的開銷、便捷的管理以及較高的性能特點，Oracle ASM 成為管理 Oracle 數據庫的理想的存儲技術。

數據庫系統高可靠

Oracle數據庫服務器在選型時具備以下五個原則：高性能、高可靠、高可擴展、安全性以及可管理性等。對於企業而言，Oracle數據庫已經是必不可少的重要IT組份之一了 。

Oracle支持非結構化數據，其二進制XML大大改進了存儲效率並且XML索引的速度，另一方面，Oracle增加了服務器和客戶端的結果緩存技術，使得PL/SQL性能提高了兩倍，JAVA性能提高了11倍，這意味著基礎硬件的要求也越來越嚴格。由於Oracle 具有RAC(真正應用集群)技術，可以分佈式的將數據庫部署在眾多服務器中以避免單臺服務器效率補足並保證冗餘，x86平臺的通用性和靈活性符合他們數據庫的伸縮性特徵。

Oracle數據庫對於服務器的要求總結為：虛擬化性能高、內存容量大、可靠性好，而4路服務器的性能和RAS特性可以為Oracle數據庫提供完備的硬件能力支撐。
