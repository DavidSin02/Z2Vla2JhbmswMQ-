讀書筆記-java高級編程之面向對象程序設計

2020-10-29
| 科技

圖文無關


Java特性：一次編譯，到處運行。

封裝是一個重要的面向對象程序設計特徵，是設計類的一個基本思想。

面向過程（Process-Oriented，PO）的程序設計強調的是模塊化、結構化的程序設計方法——通過函數實現。使用面向過程的程序設計語言設計程序時，不能很好地體現現實世界中事物的本來面目，丟失了數據本來應該攜帶的一些邏輯信息，增加了程序維護和重用的難度。

面向對象（Object-Oriented，PO）是對現實世界模型的自然體現和延伸，它認為現實世界的各種實體（甚至是虛構的事物）都可以看成對象，對象之間通過消息相互作用。

一個對象需要有兩部分描述：第一部分是它的“數據”，包括該對象的特性、狀態等靜態信息；第二部分是它的“行為”，包括對該對象的操作、對象的功能等能動信息。數據和行為是對象的兩個基本特徵。對象是可存儲數據的變量和可提供操作的方法的集合，對象的核心是變量和方法。

對於現實世界中存在的類似的多個實體，我們可以將其抽象概括為某一類事物；對於在數據和行為上類似的多個對象（Object），可以將其抽象為一個類（class）來表示。類是對象的抽象。類（抽象的）是對象（具體的）的模板，而對象是類的實例。類具有對象的兩個要素：一是描述對象的數據元素，稱為類的屬性（attribute）；另一個是描述對象的行為和功能，稱為類的方法（method），它用於操作對象或執行相關對象之間的交互。屬性和方法一起稱為類的成員（member）。

類的聲明/定義：

[<修飾符>] class <類名>{

[<屬性聲明>]

[<構造器聲明>]

[<方法聲明>]

}

類聲明中所包含的數據稱為屬性，語法為：

[<修飾符>] <類型> <屬性名稱>

方法的聲明：

<修飾符> <返回值類型> <方法名稱> ([<參數列表>]){

[<語句>]

}

構造器是Java類中一個特殊的方法，它用於創建該類的一個新對象。換句話說，對象是構造的，不調用構造器，就不能創建一個新對象。構造器是一段無論何時使用new關鍵字都要運行的代碼。一個類可以聲明一個或者多個構造器。通常在構造器中進行一些初始化工作，如初始化類的實例變量的狀態。

<修飾符> <類名> ([<參數列表>]){

[<語句>]

}

類名應做到“見名知意”，通常類名應該是名詞，第一個字母和名字內其他所有單詞的第一個字母應該大寫，其他字母小寫。屬性名通常為名詞，方法名通常為動詞，第一個單詞全部小寫，後面每一個單詞的第一個字母大寫，其他字母小寫。

構造器和方法的區別：

構造器沒有返回值，而一般的方法必須指明返回值，即使是void；構造器的名稱和類名相同，而一般的方法名稱則沒有這個限制。

在聲明類時如果沒有聲明構造器，Java編譯器將自動生成默認的構造器。默認的構造器沒有參數，並且構造器代碼為空。如果在聲明類時顯式聲明瞭其他的構造器，那麼編譯器將不再提供沒有參數的默認構造器。建議顯式的聲明一個無參數的構造器，否則使用默認構造器創建對象會引起編譯錯誤。

信息的隱藏和封裝

使用private關鍵字，只允許在同一個類中對屬性值進行訪問，而不允許在其它類中直接對其操作，這樣就實現了信息隱藏。這種方法常用於需要對外隱藏敏感信息的場合。

封裝（Encapsulation），即為客戶端提供訪問這些元素的公有接口——聲明為public的方法。封裝可以使我們在設計代碼中做修改而不中斷使用該代碼的其他人的代碼，這是封裝的主要優點。

封裝的一般做法：1.保護實例變量（用訪問修飾符，通常是private）2.建立公有的讀取器和設置器方法，強調調用代碼使用這些方法而不是直接訪問被保護的類成員3.對於讀取器和設置器方法，分別使用JavaBeans命名規則get<someProperty>和set<someProperty>。

Java源文件結構：

[<包聲明>]

<導入聲明>*

<類聲明>+

說明：<>裡面是描述條款；[]代表可選項；*代表有0或多個；+代表有1或多個。一個源文件中可以聲明一個或多個類，但只能有一個聲明為公有的。源文件的名稱必須和聲明為公有的類的名稱相同。

為了創建一個包，需要在聲明瞭類和接口的Java源文件的開頭寫上package語句。點號“.”指明瞭包的層次關係。如果在Java源文件中不使用package語句，則表明所聲明的類和接口位於一個默認包中。源程序中的package語句實際上是告訴編譯器屬於哪個包。包的命名通常採用全部小寫的字母表示。Java語言要求包聲明的層次和實際存放類文件（class文件）的目錄結構存在對應關係，以便將來使用該類時能夠通過包名（也就是目錄名）查找到所需要的類文件。

import語句告訴編譯器用到的類位於哪個包中。

import <包名> [.<子包名>].<類名>

import <包名> [.<子包名>].*

import語句的作用只是使當前類能訪問其他包中的類（有了import語句以後就可以使用短類名，即使用類時不用加上包名，在有些條件下只能使用長類名），它不會導致編譯器在內存中加載任何附加的內容，因此上面兩種導入包的方式是一樣的，不會對程序的編譯和運行速度有影響。它和C及C++中的#include語句是不同的。

這裡的package語句和import語句都只是針對class文件而言的，它指的是類文件的層次結構安排，而不是針對Java文件。package語句和import語句可以應用於源文件中所有的類和接口。

Java編譯器默認為所有Java源程序都引入java.lang中所有的類（import.java.lang.*），無需顯示引入。

編譯文件：javac -d 目錄 文件名.java
