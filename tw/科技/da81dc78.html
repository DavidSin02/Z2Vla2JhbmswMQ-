1 校準終測的基本原理

1.1校準 、終測的目的

現在生產的相同型號手機雖然使用都是相同器件,但這相同器件還是有的一定的偏差,由此組合的手機就必然存在著差異,但這差異是在一定的範圍,超出了就視為手機不良。因此校準的目的就是將手機的這種差異調整在符合國標的範圍，而終測是對於校準的檢查，因為校準無法對手機的每個信道，每個功率級都進行調整，只能選擇有代表性的（試驗經驗點）進行，所以校準通過的手機並不能肯定它是良品，只有通過終測檢驗合格的才算是，我們現在生產線上的校準終測測試程序都是將這兩個部分合並（除了DA8和EMP平臺）。

1.2手機的基本校準、測試項的介紹

1、Battcal（電池校準）：是對手機的電池模擬使用的調整，分兩種情況（4.2V和3.5V）。

恆9系列和Florence平臺的校準相似，先調整手機電池處在4.2V時的偏置值，使其衝手機讀取的電壓表示值在4.2±0.1v的範圍，然後將電池的電壓調至3.5v，看電壓是否還處於3.5±0.1v的範圍，是就將這偏置值存入手機。

2、TxCal（發射機校準）：不同的平臺有不同的校準方法，但其大致的原理是一樣的。就是通過一定的方法調整在一個或者幾個試驗經驗點（全部功率級）的功率值的表示值，使其符合國標的要求。這表示值可以是一個單一的數字，也可以是一組，像A6/A8系列的就是多個經驗點（GSM900有10,60,105,1000這4個信道，DCS1800有570,700,800這3個信道）全功率級（即GSM900有5-19，DCS1800有0-15）單一的數值，而恆9系列和Florence平臺則是單個經驗點（GSM900有62，DCS1800有698）的全功率級代表該功率級的一組功率曲線的表示值。

在這就目前使用的兩種PA將校準做個詳細的介紹

一）RFMD

a)、發射機及其校準原理

在發射機中，從CSP產生的已調信號，經過HD155148的混頻、射頻放大，再經功率放大器（PA）放大、濾波後從天線發送出去。發送信號的功率和形狀（burst shape）由PA決定，這裡採取功率控制環來控制發送信號的功率和形狀。Tx校準原理就是通過測量計算得到一系列TXP值，去控制PA的增益，使得不同PCL的發射信號滿足規範的要求（絕對功率大小、相連PCL的功率、切換頻譜、Burst Shape等）。如圖Figure 1 所示。

校準時，我們先根據寫入手機的TXP值和測量得到的功率值PM，計算得到TXP和PM的關係曲線L，再根據L對每個PCL所要求的功率值Prequired計算出相應的TXP值存到NVM即可。校準的關鍵是找到TXP和PM的關係曲線L，根據實驗得到TXP和PM相應的電平V存在線性關係，因此我們只需要兩個PCL的對值（TXP，V）即可得到L。

b)、校準方法及公式

發射信號的形狀如圖Figure 2 所示，它包括三部分：Ramp Up、Mid-Burst、Ramp Down。其中Mid-Burst 為平坦部分，決定著信號的功率。校準過程中，Mid-burst 可由TXP和PM的關係得到Mid-TXP，而Ramp Up和Ramp Down用正弦曲線來逼近，見圖Figure 3。

校準時，待發射信號直接通過cable耦合到測試儀器，負載為50Ohm，因此輸出信號電平V（v）和輸出信號功率P（mW）滿足Formula_1。

Ramp UP和Ramp Down（Burst Shape除去Mid-Burst後的形狀）用0到Pi的三次正弦函數模擬。如圖Figure 3，我們在該曲線上按時間均勻取32個點（element），element1到element5的值為0，element6到element15的值按正弦函數給出。其他element的值對稱得到。Formula_2給出了element5到element15的值，其中Te=（e-15）*（48/13/2）us （每個element佔半個bit，每個bit為48/13us），t=T-Te 。

線性曲線L的斜率m和常數c由Formula_4計算得到，TXPH和TXPL由推薦值m0和c0計算得到（plH→Prequired→Vrequired→TXPH），VH和VL由Formula_1得到（PMH→VH）。

c)、校準步驟

1. 確定TXP vs V曲線中的m和c：

先根據推薦值m0和c0, 由給定plH(pl 6 at EGSM; pl 1 at DCS)得到Prequired，計算Vrequired，從而TXPH=m0*Vrequired+c0 ，同時從儀器讀出手機的輸出功率PM，計算VH 。同樣方法得到TXPL和VL 。

再根據（TXPH，VH）和（TXPL，VL），計算m和c 。

2. 對每個頻帶每個功率級進行校準：

根據上一步得到的m和c，算出每個功率級的Ramp Data並存入RAM。最後存入NVM。

3. 驗證校準結果：

其實RFMD的校準方式就是計算出實際發射功率的電壓值與手機RAM存儲的功率表示值的關係直線。而SKY校準與這不同，下面將詳細介紹。

二）SKY

這種校準方式比較花時間，但可以將各個功率級的功率校的很精確。

a）、將手機儀器都設置在TestMode；

b）、設置儀器一個合適的信道，發送PTE指令控制手機處於連續發射狀態，信道與儀器對應；

c）、寫入該功率級的默認值，調整默認值（或加或減）使手機的發射功率達到期望值，將此數據換算的Ramp值存入手機，對該信道的各個功率級都進行這樣計算、存值。

3、RxCal（接收機校準）：

由於恆9系列和Florence平臺的Rx射頻接收模塊不一樣，因此校準也不一樣。我們分開介紹，

關鍵的校準步驟：

1）、設置工作點（Setworkingpoint）

Setworkingpoint主要的目的是尋找CSP.ADC處信號電平在-20dBfs時且通道增益已定義的情況下的Cellpower。這樣，通道增益就包含有前置低噪放大器的增益，直接變頻器的增益和22dB的HD155141TF內部的基帶增益，但不包括CSP的基帶增益，其手機接受的控制詞（RxGainA = 0，RxGainB = 202）。選一個任意的測試信道，例如：A9/A90的就選擇975信道，這是由於低信道的決定了頻率邊帶的補償。

1/發送PTE指令ReadRSS,

975,0,0,202,975,0,0,202,975,0,0,202,975,0,0,202

2/讀取結果算出平均值並轉換成dBfs單位，公式：10 log ((Reading1 + Reading2 + Reading3 + Reading4) / (4 * 32767))

3/如果計算出的結果偏離-20dBfs的0.2dB，就得調整信號發生器的信號電平，然後重複測試。

4/記錄下信號發生器的信號電平

2）、測試邊帶信道（Measure band ripple for EGSM band）

目的是計算在相同的通道增益和相同的信號強度條件下，其通道放大器對不同頻段的響應的補償，即將偏差範圍較大的響應曲線糾正在一定的範圍如上圖

1/發送PTE指令ReadRSS,Chan No,0,0,202,Chan No,0,0,202,Chan No 0,0,202,Chan No,0,0,202

2/讀取結果並將其轉換為dBfs單位（取平均值）

3/測試每個邊緣信道，根據公式：Sub–Band Average = (Average Lower Channel + Average Upper Channel) / 2算出這個頻段的平均值，再根據公式：Overall Band Mean = Sum of Sub–Band Mean Column / 8計算出這8組頻段的平均值。

4/計算頻段補償，根據公式：（每個頻段的平均值-8組頻段的平均值）*16,並記錄下參數

3）、測試Rx基本通道增益(LNA開啟)(Calculate the EGSM Basic Rx Path Gain for LNA on case)

已知：Channel=62 ；取基本通道增益BB_Gain =12 ，因此基本通道開關增益為12-（-28）=40（因為通道增益關閉時增益為-28dBm）

1/根據已知的-54.5dBm 對應的BB_Gain數值202 ，換算成BB_Gain數值162時，信號發生器的信號電平

公式：-54.5 + ((202-2)/8) - ((162-2)/8)

2/發送PTE指令，ReadRSS,62,0,0,162,62,0,0,162,62,0,0,162,62,0,0,162

3/讀取結果並將其轉換成dBfs單位，（Mean）

4/取出原先計算的現在測試信道所對應的頻段補償

5/計算基本通道增益，Basic Rx Path Gain= Mean-換算出來的信號發生器的信號電平-基帶部分的開關增益-頻段補償

6/將其結果乘以16，保存下來。

4）、測試基本通道的補償值（Calculate the BB_Gain compensation values）

已知：Channel=62，BB_Gain=-16 dB (RxGainA = 0, RxGainB = 50)，LNA = on

設置信號發生器的信號電平能保證CSP的ADC處的信號強度在-20dBfs，通過SetWorkingPoint部分完成

例如：現在的信號發生器的信號電平為：Signal level at antenna = – 24.375dBm

1/發送PTE指令：ReadRSS, 62,0,0,50,62,0,0,50,62,0,0,50,62,0,0,50

2/讀取結果並計算其CSP處的信號強度Mean=10 Log ((327 + 326 + 326 + 326) / 4) / 7FFF) = – 20.0188 dBfs

那麼，Rx基帶通道增益=Mean-信號發生器的信號電平-基帶開關增益-頻帶補償

重複以上的操作，將BB_Gain設為-2，26，40，54dB，得到以下結果：

Basic Rx Path Gain for – 16dB Base–Band gain = – 8.7688

Basic Rx Path Gain for – 2dB Base–Band gain = – 8.8134

Basic Rx Path Gain for + 12dB Base–Band gain = – 8.7678

Basic Rx Path Gain for + 26dB Base–Band gain = – 8.851

Basic Rx Path Gain for + 40dB Base–Band gain = – 9.2095

Basic Rx Path Gain for + 54dB Base–Band gain = – 9.5656

3/分別計算各個測試的BB_Gain值與-8.7678的差值，並取整

4/通過PTE指令將值存入手機，SetGainCalBBIF %d,%d,,%d,%d,%d,%d

BB_Gain[0] = 16 * (– 8.7678 + 8.7688) = 0.256 which is rounded to 0

BB_Gain[1] = 16 * (– 8.7678 + 8.8134) = 0.7296 which is rounded to 1

BB_Gain[2] = 16 * (– 8.7678 + 8.7678) = 0

BB_Gain[3] = 16 * (– 8.7678 + 8.851 ) = 1.3312 which is rounded to 1

BB_Gain[4] = 16 * (– 8.7678 + 9.2095) = 7.0672 which is rounded to 7

BB_Gain[5] = 16 * (– 8.7678 + 9.5656) = 12.7648 which is rounded to 13

5）、計算Rx基帶通道增益（LNA關閉）（Calculate the EGSM Basic Rx Path Gain for LNA off case）

已知：Channel 62，BB_Gain = 52dB (RxGainA = 0, RxGainB = 1536 + 322=1858 , RG0 and RG1 位設置為1，使LNA完全關閉)，LNA = off

設置信號發生器的信號電平能使CSP處的信號強度保持在-20dBfs

例如：

此時信號發生器的信號電平，Signal level at antenna = – 39.86dBm

1/ 發送PTE指令：ReadRSS, 62,0,0,1858,62,0,0,1858,62,0,0,1858,62,0,0,1858

2/讀取結果並將其轉換為dBfs單位，Mean = 10 Log ((323 + 323 + 325 + 323) / 4) / 7FFF) = – 20.0556 dBfs

3/其頻帶增益：Sub–Band Compensation = 1.125dB

4/當前的BB_Gain=52dB，那麼其通道開關增益 BB switched gain = 52 + 28 = 80dB

5/由於BB_Gain=52dB，在46dB和62dB之間，其基帶補償：BB_Gain compensation = 13/16 = 0.8125dB，即使用BB_Gain[5]的補償

6/現在我們可以計算出在LNA關閉的情況下的Rx基帶通道增益：Basic Rx Path Gain =– 20.0556 – (Sub–Band Compensation) – (BB_Gain compensation) – (Level at antenna) – (BB switched gain)= – 20.0556 – 1.125 – 0.8125 + 39.86 – 80 = – 62.1331dB

7/將其乘以16轉換回去，並分成兩部分存儲：Round(– 62.1331 * 16)=-994 或者0xFC1E(MSB=252,LSB=30)
