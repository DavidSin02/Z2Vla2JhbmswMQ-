SpringBoot 集成StateMachine實現狀態機

2020-10-29
| 科技

前言：

Spring StateMachine框架是Spring.io提供給開發者一種可以簡化狀態機實現的過程，讓狀態機結構更加層次化。本次狀態機的實現Showcase主要是模擬一個商品購買的場景，從下單到收貨這個業務流程下所涉及的事件、狀態來描述Spring StateMachine如何使用與運作。

正文：

下面詳細的介紹整個實現過程：

States

States.java完整代碼如下：

package com.example.statemachine.enums;/** * @package: com.example.statemachine.enums * @description: 狀態枚舉類 * @author: @IT講壇 * @create: 2018-12-16 17:27 **/public enum States { MAKE_AN_ORDER, // 生成訂單 UNPAID, // 待支付 PAYING, // 支付中 PAYED, // 已支付 WAITING_FOR_RECEIVE, // 待收貨 CUSTOMER_RECEIVED, // 已收貨 DONE // 訂單結束}

說明：

事件會觸發狀態的轉變，例如：ORDERING事件可以出發狀態從MAKE_AN_ORDER轉變為UNPAID，後面對於StateMachine的配置上將會體現。

3.創建StateMachine的配置類，這個配置類也是Spring Boot的自動裝配的必備，也是大家從xml配置轉為bean配置的最不適應的地方

StateMachineConfig.java完整代碼如下：

StateMachineConfig

package com.example.statemachine.config;import com.example.statemachine.enums.Events;import com.example.statemachine.enums.States;import com.example.statemachine.persist.InMemoryStateMachinePersist;import org.springframework.beans.factory.annotation.Autowired;import org.springframework.context.annotation.Bean;import org.springframework.context.annotation.Configuration;import org.springframework.statemachine.config.EnableStateMachine;import org.springframework.statemachine.config.EnumStateMachineConfigurerAdapter;import org.springframework.statemachine.config.builders.StateMachineStateConfigurer;import org.springframework.statemachine.config.builders.StateMachineTransitionConfigurer;import org.springframework.statemachine.persist.DefaultStateMachinePersister;import org.springframework.statemachine.persist.StateMachinePersister;import java.util.EnumSet;/** * @package: com.example.statemachine.config * @description: 狀態機配置類 * @author: @IT講壇 * @create: 2018-12-16 17:17 **/@Configuration@EnableStateMachine(name="stateMachine") //可以通過設置名稱來定義若干個狀態機的Config，並可以通過制定狀態機配置名稱來使用制定的狀態機 @EnableStateMachine(name="testMachine")public class StateMachineConfig extends EnumStateMachineConfigurerAdapter<States, Events> { @Autowired private InMemoryStateMachinePersist inMemoryStateMachinePersist; @Bean(name = "myPersister") public StateMachinePersister<States, Events, String> getPersister() { return new DefaultStateMachinePersister<>(inMemoryStateMachinePersist); } /** * * @title 初始化狀態機及初始狀態 * * @description * * @see org.springframework.statemachine.config.AbstractStateMachineConfigurerAdapter#configure(org.springframework.statemachine.config.builders.StateMachineStateConfigurer) * @param states * @throws Exception */ @Override public void configure(StateMachineStateConfigurer<States, Events> states) throws Exception { states .withStates() .initial(States.MAKE_AN_ORDER) //初始化狀態為UNPAID .states(EnumSet.allOf(States.class)); } /** * * @title 事件及狀態流轉配置 * * @description * * @see org.springframework.statemachine.config.AbstractStateMachineConfigurerAdapter#configure(org.springframework.statemachine.config.builders.StateMachineTransitionConfigurer) * @param transitions * @throws Exception */ @Override public void configure(StateMachineTransitionConfigurer<States, Events> transitions) throws Exception { transitions .withExternal() .event(Events.ORDERING) //當事件為ORDERING時，源狀態從MAKE_AN_ORDER變為UNPAID .source(States.MAKE_AN_ORDER).target(States.UNPAID) .and() .withExternal() .event(Events.PAY) //當事件為PAY時，源狀態從UNPAID變為PAYING .source(States.UNPAID).target(States.PAYING) .and() .withExternal() .event(Events.PAYED) //當事件為PAYED時，源狀態從PAYING變為PAYED .source(States.PAYING).target(States.PAYED) .and() .withExternal() .event(Events.RECEIVE) //當事件為RECEIVE時，源狀態從PAYED變為WAITING_FOR_RECEIVE .source(States.PAYED).target(States.WAITING_FOR_RECEIVE) .and() .withExternal() .event(Events.RECEIVED) //當事件為RECEIVED時，源狀態從WAITING_FOR_RECEIVE變為CUSTOMER_RECEIVED .source(States.WAITING_FOR_RECEIVE).target(States.CUSTOMER_RECEIVED) .and() .withExternal() .event(Events.FINISH) //當事件為FINISH時，源狀態從CUSTOMER_RECEIVED變為DONE .source(States.CUSTOMER_RECEIVED).target(States.DONE); }}

說明：

1)註解：@Configuration和@EnableStateMachine，一個是表明此類為一個Spring的Bean配置類，另一個則是表明當前的配置是要啟用狀態機的自動裝配。

2)父類EnumStateMachineConfigurerAdapter<States,Events>，為狀態機的自動裝配提供自定義的配置重寫，需要重寫configure方法。

3)可以暫時忽略本類中的關於狀態持久化的配置，因為，這個配不配不影響一個狀態機Showcase的實現，例如：InMemoryStateMachinePersist

4)configure(StateMachineStateConfigurer<States, Events> states)throws Exception 方法是用來定義狀態機的初始化狀態以及所能包含的狀態集合。

5)configure(StateMachineTransitionConfigurer<States, Events> transitions)throws Exception 方法是用來定義狀態機由哪些事件導致狀態從哪個狀態遷移到哪個狀態，其中命名中我們很容易理解每一個遷移動作，source是源狀態，target是目標狀態，event為觸發事件。

4.配置事件處理者，用於監聽狀態變化，允許當狀態發生變化時則執行對應的方法來進行相關業務處理

EventProcessor.java:

EventProcessor

package com.example.statemachine.handle;import lombok.extern.slf4j.Slf4j;import org.springframework.statemachine.annotation.OnTransition;import org.springframework.statemachine.annotation.WithStateMachine;/** * @package: com.example.statemachine.handle * @description: 事件處理類 * @author: @IT講壇 * @create: 2018-12-17 09:17 **/@WithStateMachine@Slf4jpublic class EventProcessor{ /** * * @title 自定義事件執行方法——1.創建訂單 * * @description 當狀態為MAKE_AN_ORDER，並轉為UNPAID狀態就執行此方法 * */ @OnTransition(source="MAKE_AN_ORDER",target = "UNPAID") public void makeAnOrder() { log.info("訂單已創建，等待用戶支付"); } /** * * @title 自定義事件執行方法——6.完成訂單 * * @description 當狀態為RECEIVED，並轉為DONE狀態就執行此方法 * */ @OnTransition(source = "CUSTOMER_RECEIVED", target = "DONE") public void finishOrder() { log.info("訂單流程已完成"); }}

EventProcessor2.java

EventProcessor2

package com.example.statemachine.handle;import lombok.extern.slf4j.Slf4j;import org.springframework.statemachine.annotation.OnTransition;import org.springframework.statemachine.annotation.OnTransitionEnd;import org.springframework.statemachine.annotation.OnTransitionStart;import org.springframework.statemachine.annotation.WithStateMachine;/** * @package: com.example.statemachine.handle * @description: 事件處理類 * @author: @IT講壇 * @create: 2018-12-17 09:17 **/@WithStateMachine@Slf4jpublic class EventProcessor2 { /** * * @title 自定義事件執行方法——2.開始支付 * * @description 當狀態為UNPAID，並轉為PAYING狀態就執行此方法 * */ @OnTransitionStart(source = "UNPAID", target = "PAYING") public void paying() { log.info("用戶支付中....."); } /** * * @title 自定義事件執行方法——3.完成支付 * * @description 當狀態為PAYING，並轉為PAYED狀態就執行此方法 * */ @OnTransition(source = "PAYING", target = "PAYED") public void payed() { log.info("用戶完成支付"); } /** * * @title 自定義事件執行方法——並行 4.1.收到支付成功短信 * * @description 當狀態為PAYED，並轉為WAITING_FOR_RECEIVE狀態就執行此方法 * */ @OnTransitionEnd(source = "PAYED", target = "WAITING_FOR_RECEIVE") public void syncNotifyFromSMS() { log.info("收到用戶支付成功短信，準備發貨"); } /** * * @title 自定義事件執行方法——並行 4.2.發貨中 * * @description 當狀態為PAYED，並轉為WAITING_FOR_RECEIVE狀態就執行此方法 * */ @OnTransitionEnd(source = "PAYED", target = "WAITING_FOR_RECEIVE") public void waitingForReceive() { log.info("等待用戶簽收貨品......"); } /** * * @title 自定義事件執行方法——5.用戶完成收貨 * * @description 當狀態為WAITING_FOR_RECEIVE，並轉為RECEIVED狀態就執行此方法 * */ @OnTransitionEnd(source = "WAITING_FOR_RECEIVE", target = "CUSTOMER_RECEIVED") public void received() { log.info("用戶已簽收貨品"); }}

說明:

其實一個EventProcessor類就可以完成需要的處理，但為什麼要這裡要分成兩個Processor類來實現，主要原因就是想簡單設計一下對於處理過程的抽象，把開始與結束放在一個類中，具體過程放在另外一個類中；此類有幾點需要注意：

1)註解@WithStateMachine，定義此類為一個狀態監聽者，@OnTransition(source="",target = "")，主要是定義一個方法的執行時機，也就是當源狀態是A，目標狀態是B時，則執行這個方法

2) ILog接口，這個接口是自己為了方便打印日誌，自己封裝的一個Logback logger的Handler，對於狀態機的showcase沒有影響，各位可以直接忽略。

5.最後創建主入口類來完成整個Showcase的流程

StateMachineApplication.java

StateMachineApplication

package com.example.statemachine;import com.example.statemachine.enums.Events;import com.example.statemachine.enums.States;import lombok.extern.slf4j.Slf4j;import org.springframework.beans.factory.annotation.Autowired;import org.springframework.beans.factory.annotation.Value;import org.springframework.boot.CommandLineRunner;import org.springframework.boot.SpringApplication;import org.springframework.boot.autoconfigure.SpringBootApplication;import org.springframework.statemachine.StateMachine;import org.springframework.statemachine.persist.StateMachinePersister;import javax.annotation.Resource;@SpringBootApplication@Slf4jpublic class StatemachineApplication implements CommandLineRunner { @Value("${spring.application.name}") private String proName; /** 狀態機對象 */ @Resource //使用制定的狀態機配置進行注入，例如：@Resource(name="testMachine"); 這樣可以有多個狀態機實例 private StateMachine<States, Events> stateMachine; /** 狀態機持久化 */ @Autowired private StateMachinePersister<States, Events, String> persist; public static void main(String[] args) { SpringApplication.run(StatemachineApplication.class, args); } /** * * @title 事件驅動業務流程主控方法 * * @description * 1.啟動狀態機 * 2.發送事件 * 3.將某一事件進行持久化 * 4.打印狀態機的當前狀態 * 5.取回被持久化的狀態並打印  * * * @see org.springframework.boot.CommandLineRunner#run(java.lang.String[]) * @param args * @throws Exception */ @Override public void run(String... args) throws Exception { log.info("==========================================Project name : {}" , proName); stateMachine.start(); stateMachine.sendEvent(Events.ORDERING); stateMachine.sendEvent(Events.PAY); stateMachine.sendEvent(Events.PAYED); persist.persist(stateMachine, "MyRegion"); log.info("Store the state in MyRegion ================1: {}",stateMachine.getState().getId()); stateMachine.sendEvent(Events.RECEIVE); stateMachine.sendEvent(Events.RECEIVED); stateMachine.sendEvent(Events.FINISH); log.info("The state Now ================2: {}",stateMachine.getState().getId()); persist.restore(stateMachine, "MyRegion"); log.info("Restore the state from MyRegion ================3: {}",stateMachine.getState().getId()); }}

說明：

在run方法中，定義了流程的驅動處理過程，其中start()是根據之前的定義觸發狀態機啟動這個訂單流程，並一步步的sendEvent來持續驅動，直至最終的FINISH事件發出。

注意幾點：

1)persist.persist和persist.restore這兩個方法調用是關於持久化和獲取這個流程狀態的，可以不用關心，除非想了解如何持久化狀態機內瞬時狀態及狀態獲取的。

2)直接運行這個Main函數、JUnit或使用maven客戶端來執行spring boot cli命令，例如： clean compile spring-boot:run

至此，就完成了一個模擬訂單下單場景的狀態機驅動過程的Showcase。
