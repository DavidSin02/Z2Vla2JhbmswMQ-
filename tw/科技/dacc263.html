MySQL數據庫中的事務以及創建表的注意事項

2020-10-29
| 科技

數據庫事務：

一組sql完成一個工作

例如 轉賬

update account set money= money-1000 where id =1;

update account set money= money+1000 where id= 2;

這兩個語句完成一個工作 ----- 這就是一個事務

數據庫事務的四個基本特徵:

A 原子性

這個事務不能再拆分 如果再拆開 就不能完成相應的工作

要保證一個事務中的多個sql語句 同時成功 或 同時失敗

C 一致性

通常指數據的一致性 事務執行前後 數據的總和不變

I 隔離性

多個事務之間不能互相影響

D 持久性

事務提交以後 數據要真正的持久化到數據庫

對於MySQL 默認情況下 每個DML語句獨自執行在一個事務中 默認自動提交事務

其他數據庫對象：

1 視圖 （view）：

什麼是視圖？ 由一個表 或多個表的部分數據組成的一個新的數據庫對象

視圖是一張虛擬表

表示一張表的部分數據或多張表的綜合數據

其結構和數據是建立在對錶的查詢基礎上

為什麼有視圖： 1 最主要就是限制數據訪問

2 使得複雜的查詢變的簡單

create view v_emp as select id,name,birthday from employee

create view v2_emp as select id emp_id,name emp_name,birthday emp_birthday from employee

create view v_emp_dept as

select e.id,e.name emp_name,e.salary,d.name dept_name

from employee e left outer join department d on e.department_id = d.id

查看所有視圖：

USE information_schema;

SELECT * FROM views\G;

簡單視圖 數據來自一個表 形成視圖的查詢語句中

簡單視圖可以做DML（因為對視圖進行DML時 要對形成視圖表同樣做DML操作）

複雜視圖 數據來自一個或多個表 形成視圖的查詢語句中要有 函數或分組

複雜視圖不能做DML（增刪改）

內建視圖： 內建視圖不是數據庫對象 只是存放在內存中，當查詢語句執行結束

內建視圖也就不存在了

MySQL中必須取別名

子查詢 放在where條件中叫做子查詢

放在from後，叫做內建視圖

select t.name,t.salary

from (select id,name,birthday,salary from employee) t

刪除視圖： drop view 視圖名

2 索引：

在列上添加索引----- 索引的目的就是為了提高查詢效率

主鍵約束 唯一性約束 都會在對應的列上添加索引

根據主鍵列和有唯一性約束的列查詢數據效率高於普通列

用戶可以自定義索引

create index 索引名 on 表名(列名)

加索引可以絕對可以提高查詢效率 但為什麼不在所有列上都添加索引

添加索引會提高查詢效率 但有可能降低增刪改的效率

因為對數據進行增刪改時 同樣需要維護索引

在什麼樣的列上添加索引合適？？？？ 經常根據這個列進行查詢

比較少做增刪改操作

在student表的studentName列創建普通索引

USE myschool;

CREATE INDEX `index_student_studentName`

ON `student`(`studentName`);

刪除索引 drop index 索引名 on 表名

按照下列標準選擇建立索引的列

頻繁搜索的列

經常用作查詢選擇的列

經常排序、分組的列

經常用作連接的列（主鍵/外鍵）

請不要使用下面的列創建索引

僅包含幾個不同值的列

表中僅包含幾行

查詢時減少使用*返回全部列，不要返回不需要的列

索引應該儘量小，在字節數小的列上建立索引

WHERE子句中有多個條件表達式時，包含索引列的表達式應置於其他條件表達式之前

避免在ORDER BY子句中使用表達式

查看索引：

SHOW INDEX FROM 表名;

Table：創建索引的表

Non_unique：索引是否非唯一 0代表唯一 1代表不是唯一

Key_name：索引的名稱

Column_name：定義索引的列字段

Seq_in_index：該列在索引中的位置

Null：該列是否能為空值

Index_type：索引類型

語句來創建約束：

主鍵約束

外鍵約束

非空約束

唯一性約束

1 在建表階段創建

create table 表名(

列名 數據類型,

...

列名 數據類型

)

create table dept(

id int primary key auto_increment,

name varchar(200) not null,

address varchar(200)

)

create table emp(

id int primary key auto_increment,

name varchar(200) not null,

email varchar(200) unique,

dept_id int,

foreign key emp(dept_id) references dept(id)

)

2 表創建後，添加

create table dept(

id int,

name varchar(200) not null,

address varchar(200)

)

表創建完成後再添加主鍵

alter table dept add constraint primary key(id)

create table emp(

id int,

name varchar(200) not null,

email varchar(200),

dept_id int,

)

創建完表後添加主鍵約束，添加外鍵約束，添加唯一性約束

alter table emp add constraint primary key(id)

alter table emp add constraint unique(email)

alter table emp add constraint foreign key emp(dept_id) references dept(id)
