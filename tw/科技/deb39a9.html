一次記錄ResponseBodyAdvice 導致系統故障問題的排查心得

2020-10-29
| 科技

背景:

ResponseBodyAdvice 顧名思義是返回body體通知器 ,其作用是讓我們在寫controller層時,可以不用每個路由都寫相對統一的返回對象.類似:

	@Override	public Object beforeBodyWrite(Object body, MethodParameter returnType, MediaType selectedContentType, Class<? extends HttpMessageConverter<?>> selectedConverterType, ServerHttpRequest request, ServerHttpResponse response) {		if(body instanceof HttpResult){			return body;		}		return HttpResult.success(body);	}	@Override	public boolean supports(MethodParameter returnType, Class<? extends HttpMessageConverter<?>> converterType) {		return true;	}
public class HttpResult<T> implements Serializable {  private int status;    private String info;    private T data;}

問題:

當返回結果為String時,會報錯:

    @GetMapping("test")    public String test() {        return "i am a string";    }


報錯信息:

17:48:22.403 [XNIO-1 task-1] ERROR ServiceExceptionHandler: - 【ServiceExceptionHandler - Exception】 [java.lang.ClassCastException: class com.gaodun.storm.passport.common.HttpResult cannot be cast to class java.lang.String (com.gaodun.storm.passport.common.HttpResult is in unnamed module of loader 'app'; java.lang.String is in module java.base of loader 'bootstrap')	at org.springframework.http.converter.StringHttpMessageConverter.getContentLength(StringHttpMessageConverter.java:43)	at org.springframework.http.converter.AbstractHttpMessageConverter.addDefaultHeaders(AbstractHttpMessageConverter.java:260)

分析:

很顯然,問題出在消息轉換器上,為什麼會用到 StringHttpMessageConverter 而不是我們配置的,如MappingJackson2HttpMessageConverter 轉換器,或者是fastjson轉換器.

貼出我們自定義的消息轉換器配置代碼:

    @Bean    public MappingJackson2HttpMessageConverter mappingJackson2HttpMessageConverter() {        MappingJackson2HttpMessageConverter mappingJackson2HttpMessageConverter                = new MappingJackson2HttpMessageConverter();        //設置日期格式        ObjectMapper objectMapper = new ObjectMapper();        SimpleDateFormat smt = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");        objectMapper.setDateFormat(smt);        objectMapper.setPropertyNamingStrategy(PropertyNamingStrategy.UPPER_CAMEL_CASE);        mappingJackson2HttpMessageConverter.setObjectMapper(objectMapper);        //設置中文編碼格式        List<MediaType> mediaTypes = new ArrayList<>();        mediaTypes.add(MediaType.APPLICATION_JSON_UTF8);        mediaTypes.add(MediaType.TEXT_HTML);        mappingJackson2HttpMessageConverter.setSupportedMediaTypes(mediaTypes);        return mappingJackson2HttpMessageConverter;    }


跟進源碼發現:

spring 會根據 mediaType 來 選取最合適的消息轉換器來轉換消息體.

有一個消息轉換器集合,從第一個開始,依次遍歷,只要在遍歷的過程中,發現有mediaType符合條件,即選定消息體轉換器,哪怕不是我們想要的,哪怕後面還有更合適的消息轉換器,都不會再執行了.

很不幸,我們的 StringHttpMessageConverter 被選中了,於是乎就出現了篇頭的那個錯誤.


解決方案1:

比較拙劣,不改變那個錯誤的消息體轉換器,在 beforeBodyWrite 方法內將返回值強行用json轉換為String. 可以解決問題,但是手段拙劣,不推薦.

解決方案2:

spring 其實通過 WebMvcConfigurer留了一個擴展點給開發人員自定義消息體集合的操作

至此,問題解決.希望可以幫助到遇到相同問題的小夥伴。
