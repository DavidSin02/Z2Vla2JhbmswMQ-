算法連載之求解逆序對問題

2020-10-29
| 科技

什麼是逆序對

如果存在正整數 i, j 使得 1 ≤ i < j ≤ n 且 A[i] > A[j], 則 <A[i], A[j]> 這個有序對, 稱為A的一個逆序對, 也稱作逆序數。

舉例

數組a：2，3，8，6，1

從左邊第一個數開始向右逐個數遍歷，第一和第五個數可以組成一個逆序對<2,1>，即1<5，a[1]>a[5]。數組a共有5個逆序對，分別是：<2,1> <3,1> <8,6> <8,1> <6,1>

數組a：1，2，3，4，5

對於升序數組，沒有符合規則的逆序對。

數組a：5，4，3，2，1

對於降序數組，共有n*(n-1)/2個逆序對。即數組a共有10個逆序對。

暴力求解

上述算法，即為暴力求解逆序對。

時間複雜度是：O(n²)

逆序對就是從左向右，比較兩個數的大小。我們發現，對於插入排序和冒泡排序，當按照升序排序時，交換的兩個數符合逆序對規則。但他們的時間複雜度都是O(n²)。

我們可以利用歸併排序，來獲取更優的性能。

歸併排序求解

使用改進的歸併排序求解逆序對。

對於原歸併排序算法，我們只是加入了一句number += mid - start - leftStart;在合併左右兩個已經排序的數組時，若左邊數大於右邊數，則符合逆序對規則，並且我們發現，因為左邊序列已是有序序列，因此當左邊數大於右邊數時，左邊數後邊的所有數都可以和右邊數組成逆序對。新增加的語句的時間複雜度是O(1)。

歸併排序的時間複雜度是O(nlgn)，所以歸併排序求解逆序對的問題，時間複雜度優化為O(nlgn)。

性能比較

分別測試暴力求解和改進的歸併排序求解逆序對的性能，100000個隨機排列100以內的數字進行統計逆序對。

There are 2477065358 inversions by Violence solve, using time is 5060 milliseconds.

There are 2477065358 inversions by Merge solve, using time is 22 milliseconds.

從輸出結果檢查，改進的歸併排序求解逆序對的性能符合我們的預期。
