「圖型計算架構」GraphTech生態系統2019-第2部分：圖形分析

2020-11-14
| 科技

這篇文章是關於GraphTech生態系統系列文章的一部分「圖型計算架構」GraphTech生態系統2019-第1部分：圖形數據庫。這是第二部分。它涵蓋了圖形分析領域。第一部分是關於圖形數據庫，第三部分將列出現有的圖形可視化工具。


Actors of the graph analytics landscape in 2019

第二層是另一個後端層：圖形分析或計算框架。它們由一組工具和方法組成，這些工具和方法是為了從以圖形形式建模的數據中提取知識。它們對於許多應用程序都至關重要，因為處理複雜連接數據的大數據集在計算上具有挑戰性。

對大規模分析的需求

圖論領域已經產生了多種算法，分析人員可以依賴這些算法來發現隱藏在圖表數據中的見解。從Google著名的PageRank算法到遍歷和路徑查找算法或社區檢測算法，都有大量的計算可以從圖表中獲得見解。

我們在上一篇文章中提到的圖形數據庫存儲系統擅長將數據存儲為圖形，或者管理諸如數據檢索、編寫實時查詢或本地分析之類的操作。但是，他們可能在規模化的圖形分析處理方面存在不足。這就是圖形分析框架介入的地方。與常見的圖形算法、處理引擎，有時還有查詢語言一起，它們處理在線分析處理，並將結果保存回數據庫。

圖形處理引擎

圖形處理生態系統提供了各種方法來應對圖形分析的挑戰，歷史玩家佔據了市場的很大一部分。


Main graph engine processing and framework vendors

2010年，Google率先發布了Pregel，一個“大規模圖形處理”框架。幾個解決方案，如Apache GIRAPH，一個開放源碼的圖形處理系統在2012由Apache基金會開發。它利用MapReduce實現來處理圖形，是Facebook用來遍歷其社交圖的系統。其他開源系統迭代了Google的，比如Mizan或GPS。

其他系統，比如GraphChi或PowerGraph Create，都是在GraphLab於2009年發佈之後推出的。這個系統最初是卡內基梅隆大學的一個開源項目，現在被稱為Turi。

oraclelab開發了PGX（Parallel Graph AnalytiX），這是一個圖形分析框架，包括一個支持Oracle大數據空間和圖形的分析處理引擎。

微軟於2013年推出的分佈式開源圖形引擎Trinity現在被稱為微軟圖形引擎。GraphX於2014年推出，是在apachespark之上構建的用於並行計算的嵌入式圖形處理框架。後來又引入了一些其他系統，例如信號/採集。

圖形分析庫和工具箱

在圖形分析領域，也有專門用於圖形分析的單用戶系統。圖分析庫和工具箱提供圖論算法的實現。


Some libraries and toolkits offering graph analytics capabilities

有獨立的庫，如NetworkX和NetworKit，用於大規模圖形分析的python庫，或iGraph，一個用C編寫並以python和R包提供的圖形庫，以及由圖形數據庫供應商Neo4j及其圖形算法庫提供的庫。

其他技術供應商為高性能圖形分析提供圖形分析庫。這是GPU技術提供商NVIDIA及其NVGraph庫的例子。地理信息軟件QGIS也建立了自己的網絡分析庫。

其中一些庫還提出了圖形可視化工具來幫助用戶構建圖形數據探索接口，但這是本系列第三篇文章的主題。

圖形查詢語言

最後，還沒有提到的一個重要的分析框架：圖形查詢語言。

對於任何存儲系統，查詢語言是圖形數據庫的基本元素。這些語言使得將數據建模為圖形成為可能，並且它們的邏輯非常接近於圖形數據模型。除了數據建模過程外，還使用圖形查詢語言對數據進行查詢。根據它們的性質，它們可以用於數據庫系統，也可以作為領域特定的分析語言。大多數高級計算引擎允許用戶使用這些查詢語言進行編寫。


Some of the existing graph query languages and similar projects

Cypher是Neo4j在2011年創建的，用於他們自己的數據庫。它在2015年作為一個名為OpenCypher的獨立項目被開源。其他值得注意的圖形查詢語言還有：Gremlin（2009年創建的apachetinkerpop查詢語言的圖形遍歷語言）或SPARQL（W3C在2008年創建的查詢RDF圖的類SQL語言）。最近，TigerGraph開發了自己的圖形查詢語言GSQL，Oracle創建了PGQL，這兩種都是類似SQL的圖形查詢語言。G-Core是由鏈接數據基準委員會（LDBC）於2018年提出的，作為連接學術界和工業界的語言。其他供應商，如OrientDB，則使用關係查詢語言SQL。

去年，Neo4j發起了一項計劃，將Cypher、PGQL和G-Core統一到一種標準圖形查詢語言GQL（graph query language）下。該計劃將在2019年3月的W3C研討會上討論。其他一些查詢語言特別專用於圖形分析，如SocialSocial。

Facebook的GraphQL本來不是一種圖形查詢語言，但值得一提。這個API語言已經被圖形數據庫供應商擴展為一種圖形查詢語言。Dgraph使用itnativelyas作為查詢語言，Prisma正計劃將其擴展到各種圖形數據庫，Neo4j已經將其推進到GRANDstack及其查詢執行層Neo4j中-圖形ql.js.
