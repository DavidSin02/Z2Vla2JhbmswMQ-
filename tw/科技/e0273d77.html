西部數據My Cloud EX2存儲設備默認洩露用戶文件 官方表示不修復

2020-11-14
| 科技

Trustwave公司的安全專家發現，西部數據（Western Digital，WD）的My Cloud EX2存儲設備在默認情況下會將文件洩露給本地網絡中的任何人，無論用戶設置的權限如何。如果用戶還將其配置為可通過互聯網遠程訪問，那麼情況會變得更糟。在這種情況下，My Cloud EX2存儲設備會通過端口9000上的HTTP請求洩露文件。

在本週三（4月25日），Trustwave公司公佈了它的調查結果。該公司警告說：“不幸的是，新的My Cloud EX2存儲設備的默認配置允許任何未經身份驗證的本地網絡用戶使用HTTP請求從設備獲取任何文件。”

研究人員表示，洩露是由於設備的UPnP媒體服務器在設備開機時自動啟動。Trustwave的安全研究經理Martin Rakhmanov在針對My Cloud EX2存儲設備的技術分析中寫道：“默認情況下，未經身份驗證的用戶可以從設備中抓取任何文件，完全繞過設備所有者或管理員設置的任何權限或限制。”

Trustwave透露，他們在1月26日就已經發現了這個漏洞。當他們向西部數據公司透露他們的調查結果時，該公司最初低估了這個漏洞所帶來的影響，只是建議用戶禁用DLNA。該公司還表示，由於洩露問題是因不安全的默認設置而引起的，因此這並不值得花時間來進行修復。

此外，西部數據公司的一位發言人還表示，DLNA只是在所有的My Cloud和My Cloud Mirror產品上是默認開啟的，但在其他My Cloud Pro系列和Expert系列產品上是被默認禁用的。

“你不必經過身份驗證，也不需要提前獲得登錄憑證。如果My Cloud EX2存儲設備恰好就存在於開放的互聯網上且脆弱的端口9000是開放的，那麼你便能夠從任何地方訪問設備上的每一個文件。”Trustwave SpiderLabs的威脅情報經理Karl Sigler在接受媒體採訪時說。

Trustwave為這個漏洞發佈了一個概念驗證（PoC）代碼，攻擊場景涉及需要攻擊者向端口9000發出HTTP請求，要求提供“TMSContentDirectory / Control”資源，UPnP服務器反過來會以設備上的文件列表進行響應。接下來，攻擊者便可以使用後續的HTTP請求，利用收集的響應中的URL從設備中獲取實際文件。

Trustwave表示，由於西部數據並不會對這個漏洞進行修復，因此禁用DLNA是保護數據安全的唯一辦法，即使你對My Cloud EX2存儲設備設置了任何權限或限制。
