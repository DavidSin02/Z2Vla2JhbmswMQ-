構建數據集成和實時數據處理流水線工具Spring Cloud Data Flow

2020-10-29
| 科技

前言

Spring Cloud Data Flow 是基於原生雲對 Spring XD 的重新設計，該項目目標是簡化大數據應用的開發。

Spring Cloud Data Flow 簡化了專注於數據流處理的應用程序的開發和部署。它的體系結構包含的主要概念有：應用程序、Data Flow Server 和運行時環境。

Spring Cloud Data Flow 為基於微服務的分佈式流處理和批處理數據通道提供了一系列模型和最佳實踐。

特性

1.使用 DSL，REST-APIs，Dashboard 和 drag-and-drop gui 開發2.獨立的創建，單元測試，故障排除，和管理微服務程序3.使用開箱即用的 stream 和 task/batch 應用迅速的構建數據通道4.把微服務作為 maven 或者 Docker 的構件5.在不中斷數據流的情況下擴展數據通道6.在現代運行環境平臺上編配以數據為中心的應用程序7.利用度量，健康檢查遠程管理每一個微服務程序

架構

Spring Cloud Data Flow 簡化了應用程序的開發和部署，將精力集中到數據處理的用例上
主要的架構概念在應用程序、data flow 服務器和目標運行環境上

運行環境支持

- Cloud Foundry- Apache YARN- Kubernetes- Apache Mesos

更新內容

Spring Cloud Data Flow 2.5.0.M1 發佈了，Spring Cloud Data Flow 是構建數據集成和實時數據處理流水線的工具包。

2.5.0 的第一個里程碑版本包含針對 Cloud Foundry 中流的應用程序狀態檢索的性能改進。這還涉及修改運行時應用程序視圖頁面，以及在 SCDF 儀表板上對流進行更好的分頁。

此里程碑版本還包含一些重要的錯誤修復。

請注意，Spring Cloud Data Flow 2.5.x 僅與 Spring Cloud Skipper 2.3.2.RELEASE 及更高版本兼容。在升級 Spring Cloud Data Flow 2.5.0.M1 時，還需要將 Spring Cloud Skipper 更新到 2.3.2.RELEASE 及更高版本。Spring Cloud Skipper 版本的當前里程碑是 2.4.0.M1。

接下來，Spring Cloud Data Flow 團隊表示正在開發一項功能，以支持將應用程序元數據作為 Docker 鏡像的一部分。當前僅通過 Maven 工件支持將應用程序列入白名單。藉助這一新功能，可以在創建 docker 鏡像時配置應用程序元數據。

更新說明： https://spring.io/blog/2020/03/20/spring-cloud-data-flow-2-5-0-m1-released
