C++ - 類與派生類的定義

class類名{ public: 公有數據成員或公有函數成員的定義; protected:  保護數據成員或保護函數成員的定義; private: 私有數據成員或私有函數成員的定義;};class派生類名:繼承方式1 基類名1,繼承方式2 基類名2... { private: 派生類的私有數據和函數 public: 派生類的公有數據和函數 protected: 派生類的保護數據和函數};

C++ - 構造函數與析構函數

構造函數相對於般函數來說,具有如下特殊的性質:

構造函數的函數名必須與定義它的類同名。

構造函數沒有返回值。如果在構造函數前加void是錯誤的。

構造函數被聲明定義為公有函數。

構造函數在建立對象時由系統自動調用。

構造函數相對於一般函數來說,具有如下特殊的性質:

析構函數沒有任何參數,不能被重載,但可以是虛函數,一個類只有一個析構函數。

析構函數沒有返回值。

析構函數名與類名相同,但在類名前加上一個邏輯非運算符"~" ,以示與構造函數對比區別。

析構函數一般由用戶自己定義,在對象消失時由系統自動調用,如果用戶沒有定義析構函數,系統將自動生成一個不做任何事的默認析構函數。

C++ - 對象指針與對象引用

對象指針的語法定義形式如下:

類名 *對象指針名;

對象引用的定義形式如下:

類名 &對象引用名=被引用對象;

注意:通過對象名或對象引用訪問對象的成員,使用的運算符是“.”;而使用對象指針訪問對象的成員,使用的運算符是"->"。如:對象指針名->數據成員名或:對象指針名->成員函數名(參數表)。

C++ - 虛函數

虛函數定義的一般語法形式如下:

virtual 函數類型 函數名 (形參表) { 函數體;}

純虛函數定義形式如下:

virtual 函數名= 0;

JAVA - 類的定義

類的定義格式如下:

[import包][類修飾符]class xxxclass [extends超類](implements 接口] { public: 公有數據成員或公有函數成員的定義; protected: 保護數據成員或保護函數成員的定義; private: 私有數據成員或私有函數成員的定義;}

說明:

import包:引入包中的類。

類修飾符:主要有四個修飾符, public、abstract、 final、private。

class為關鍵字, xxxclass為類名,命名遵循Java標識符的命名規則。

extends為繼承關鍵字,implements為接口關鍵字。
