棧

  1. 概念

棧（stack）是一種數據結構，屬於線性表的一種實現。既然是線性表，那麼肯定數據之間也有一定的特徵，棧的特徵如下

  • 數據項的加入和移除都限制在一端，這個端點一般稱之為棧頂
  • 專業點的描述就是：後進先出（LIFO），越早進入棧結構的數據越晚離開棧
  1. 生活中的例子（加深理解）
  • 一摞盤子，想取的話只能先取最上面的一個，然後下一個，依次類推，但如果想放新的盤子上去，也只能將新盤子放在所有盤子的最上面，然後再摞一個，以此類推。這個（頂端）用來拿或者取盤子的位置，抽象出來一個概念，稱之為棧頂。盤子最下面不參與任何的操作，稱之為棧底。
  • 子彈夾，永遠只能從一端壓入子彈，且從同一端彈出子彈。這樣的結果就是，最先壓進去的子彈會最後才被彈出，而最後壓進去的子彈會最先彈出，這個彈出的位置就稱之為棧頂
  1. 棧的實現(基於python)

棧的底層同樣可以基於數組或者鏈表實現，但這裡為了簡單，就基於python的內置結構列表構造一個簡單的棧結構。你要知道，作為任何數據結構，用什麼實現不重要，重要的是你設計出來的棧結構的接口要滿足後進先出的結構特點

# 我們這裡簡單的用python內置列表作為基礎結構實現棧結構class Stack:	"""利用列表（底層為數組）構建棧的抽象結構"""    def __init__(self):        self._items = []    def size(self):        return len(self._items)    def push(self, item):        self._items.append(item)    def pop(self):        return self._items.pop()    def is_empty(self):        return len(self._items) == 0    def peek(self):        return self._items[len(self._items) - 1]

這裡要強調一下，雖然從代碼上看，確實滿足了棧的特點，但這並不是一個嚴格意義上的棧，因為內部的一些列表方法都是可以使用的，而在抽象結構中這些方法應該謹慎地使用。

  1. 棧的經典應用
  • 編譯器基本功能中的對於各種括號（"(){}[]"）的正確匹配
  • 程序運行中作為數據的隔離
  • 回溯算法
  • 管理計算機內存支持函數和方法的調用
  • 中綴表達式轉換為後綴表達式；計算後綴表達式(思路如下)

-如果遇到操作數，我們就直接將其輸出

-如果遇到操作符，則我們將其放入到棧中，遇到左括號時我們也將其放入棧中。

-如果遇到一個右括號，則將棧元素彈出，將彈出的操作符輸出直到遇到左括號為止。注意，左括號只彈出並不輸出。

-如果遇到任何其他的操作符，如（“+”， “*”，“（”）等，從棧中彈出元素直到遇到發現更低優先級的元素(或者棧為空)為止。彈出完這些元素後，才將遇到的操作符壓入到棧中。有一點需要注意，只有在遇到" ) “的情況下我們才彈出” ( “，其他情況我們都不會彈出” ( "

-如果我們讀到了輸入的末尾，則將棧中所有元素依次彈出


我這裡分享兩個通過棧結構實現的經典應用：進制轉換和中綴轉後綴表達式

  1. 10進制轉換為2/8/16進制的字符串格式
# 10進制轉換為2/8/16進制的字符串格式def to_str(n, base):    :param n: 十進制數    :param base: 想要轉換的進制    :return:     if n < base:        return str(n)    elif 2 <= base <= 10:        return to_str(n // base, base) + str((n % base))    else:        convert_string = "0123456789abcdef"        return to_str(n // base, base) + convert_string[(n % base)]ret = to_str(541, 2)print(ret)
  1. 中綴轉後綴表達式
# 中綴轉後綴表達式def infix_to_suffix(expressions):    """當前約定中綴表達式是以空格進行分割, exp: a + b * c"""    s = Stack()    exp_list = expressions.split(" ")    # 存放操作數    num_list = []    # 自定義優先級，"("的存在是為了有多重括號存在時，保證所有的+-*/都可以順利入棧    priority_dic = {"(": 1, "+": 2, "-": 2, "*": 3, "/": 3}    for token in exp_list:        if token not in "+-*/()":            num_list.append(int(token))        elif token == "(":            s.push(token)        elif token == ")":            while s.peek() != "(":                num_list.append(s.pop())            # 將"("彈出            s.pop()        else:            while s.size() > 0 and priority_dic.get(token) <= priority_dic.get(s.peek()):                num_list.append(s.pop())            s.push(token)    # 檢查棧中是否還有操作符存在，有的話，依依彈出即可    while not s.is_empty():        num_list.append(s.pop())    return "".join([str(i) for i in num_list])
  1. 計算後綴表達式
# 計算後綴def calc_suffix(expression):    s = Stack()    for token in expression:        if token in "0123456789":            s.push(int(token))        else:            # 要考慮到減法和除法順序不能顛倒的問題            num_first = s.pop()            num_second = s.pop()            tmp_result = convert_to_math(token, num_first, num_second)            s.push(tmp_result)    return s.pop()# 輔助函數，每兩個彈出元素的計算結果def convert_to_math(token, num1, num2):    if token == "*":        return num1 * num2    elif token == "/":        return num2 / num1    elif token == "-":        return num2 - num1    elif token == "+":        return num1 + num2
  1. 測試代碼如下
# 測試數據convert_ret = infix_to_suffix("3 + 5 * 2 - 5 * ( 3 + 2 )")calcu_ret = calc_suffix(convert_ret)print(calcu_ret)


隊列

隊列是一種有次序的數據集合，表現為向隊列中添加數據時，永遠在尾端添加，而移除數據則永遠從隊首刪除，專業化的表述為：先進先出的結構特點（FIFO），越早進入棧結構的數據越早離開棧

  1. 生活中的例子（加深理解）
  • 排隊買票，一般情況下，你應該從買票隊伍的最後一個位置進入列隊
  • 一臺打印機面向多個用戶或是程序的情況
  1. 隊列實現

隊列也可以由數組或是鏈表結構設計實現，數組的實現（循環數組的方法）要難於鏈表，但是性能更優，這裡還是基於python的列表實現

# 利用內置數據類型列表構建隊列數據結構class Queue():    def __init__(self):        self._items = []    def size(self):        return len(self._items)    def enqueue(self, data):        self._items.insert(0, data)    def dequeue(self):        return self._items.pop()    def is_empty(self):        return len(self._items) == 0
  1. 經典應用
  • 模擬打印機處理多任務
  • 模擬道路交通，超市結賬排隊情況等
  • CPU訪問
  • 多個進程訪問同一個CPU，新進程會被添加到隊列，這個隊列就是等待CPU使用的進程
  1. 經典實現

約瑟夫/熱土豆理論

熱土豆問題：幾個小朋友，圍成一圈，用一個熱土豆（或別的什麼東西），數一個數就拋給下一個人，每數到3，手上有土豆的人就站出來，然後繼續，問哪個位置的人最後剩下？

# 熱土豆問題邏輯代碼def hot_potato(lists, num):    q = Queue()    # 按人名將數據項入隊    for person in lists:        q.enqueue(person)	# 直到只剩下一個人，遊戲結束    while q.size() > 1:        for i in range(num):            q.enqueue(q.dequeue())        q.dequeue()	# 將最後剩下的人返回    return q.dequeue()# 模擬熱土豆的實例，參數分表代表參加遊戲的人名以及土豆每傳遞幾次就出隊一個人last_person = hot_potato(["A", "B", "C", "D", "E", "F", "G", "H", "I", "L", "M", "N", "O"], 3)print(last_person)

生產者消費者模型

這應該是最常用的了，經常要構建消息隊列

import threadingimport queue  def producer():    """    模擬生產者    :return:    """    for i in range(10):        print("生產包子%s" %i)        q.put("包子 %s" % i)     print("開始等待所有的包子被取走...")    q.join()  # 等待這個包子隊列被消費完畢    print("所有的包子被取完了...")  def consumer(n):    """    模擬消費者    :return:    """    while q.qsize() > 0:        print("%s 取到" % n, q.get())        q.task_done()  # 每取一個包子，便告知隊列這個任務執行完了 q = queue.Queue() p = threading.Thread(target=producer,)p.start() c1 = consumer("xiaochen")
