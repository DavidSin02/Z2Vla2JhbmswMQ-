劉帥嵌入式系統-數據處理指令操作數尋址方式寄存器算術右移尋址

2020-10-29
| 科技


算術右移尋址：<Rm>, ASR #<shift_imm>

指令編碼格式：



操作數生成方法：

指令的操作數<shifter_operand>為寄存器Rm的數值算術右移shift_imm位。shift_imm >= 0 且 shift_imm <= 31，當shift_imm = 0時，移位位數為32，所以移位位數範圍為1-32。

進行移位操作時，空出的為添Rm的最高位Rm[31]。因此當shifte_imm = 0時，將進行32次移位，此時有如下情況：

--若Rm[31] = 0，則<shifter_operand> = 0，循環器進位值為Rm[31] = 0。

--若Rm[31] != 0，則操作數<shifter_operand>的值為0xFFFFFFFF，循環器的進位值為Rm[31] = 1。

其他情況下，操作數<shifter_operand>為寄存器Rm的數算術右移shift_imm位，循環器進位值為Rm最後被移出的位。

注：當R15（PC）用作第一個源操作數Rn或第二個操作數Rm時，操作數即為當前指令地址加常數8。
