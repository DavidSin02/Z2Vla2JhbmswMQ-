（二)SQL數據模式、表、索引的建立與刪除

2020-11-14
| 科技

一、模式的定義與刪除

1.定義模式

 CREATE SCHEMA <模式名> AUTHORIZATION <用戶名>

如果沒有指定<模式名>，那麼<模式名>隱含為<用戶名>

例如：定義一個學生—課程模式S-T

CREATE SCHEMA S-T AUTHORIZATION WANG;

為用戶WANG定義了一個模式S-T.

定義模式實際上定義了一個命名空間，在這個空間中可以進一步定義該模式包含的數據庫對象，例如基本表、視圖、索引等。

2.刪除模式

DROP SCHEMA <模式名><CASCADE|RESTRICT>

其中CASCADE、RESTRICT兩者必選一個

.選擇CASCADE(級聯)，表示再刪除模式的同時把該模式中所有的數據庫對象全部一起刪除

.選擇RESTRICT(限制)，表示如果該模式中已經定義了下屬的數據庫對象，則拒絕執行該刪除語句。

例如：DROP SCHEMA S-T CASCADE;

該語句刪除了模式S-T。同時把該模式下的表、視圖等刪除。

二、基本表的定義、刪除與修改

1.定義基本表

CREATE TABLE <表名> (<列名> <數據類型> [列級約束條件], <列名> <數據類型> [列級約束條件], ……  [<表級約束條件>]);

建立一個"課程"表Course。

CREATE TABLE Course(Cno CHAR(4) PRIMARY KEY, /*列級完整性約束條件，Cno是主碼*/ Cname CHAR(40), Cpno CHAR(4), /*Cpno的含義是先修課*/ credit SMALLINT, FOREIGN KEY (Cpno) REFERENCES Course(Cno) /*表級約束性條件，Cpno是外碼，被參照表是Course，被參照列是Cno*/ );

2.修改基本表

 ALTER TABLE <表名> [ADD <新列名><數據類型>[完整性約束]] [DROP <完整性約束名>] [ALTER COLUMN <列名> <數據類型>];

向 Course表增加“開始時間”列，將其數據類型為日期型

 ALTER TABLE Course ADD C_start DATE;

將Ccredit 分數的數據類型由短整數改為整數

ALTER TABLE Course ALTER COLUMN Ccredit INT ;

增加課程名稱必須取唯一值得約束條件

ALTER TABLE Course ADD UNIQUE(Cname);

3.刪除基本表

 DROP TABLE <表名> [RESTRICT|CASCADE]

.選擇RESTRICT :則該表的刪除是有限制條件的。欲刪除的基本表不能被其他表的約束所引用（如 CHECK,FOREIGN KEY等約束，不能有視圖，不能有觸發器，不能 有存儲過程或者函數等。如果存在這些依賴該表的對象，則此表不能被刪除。

.選擇CASCADE:則該表的刪除沒有限制條件。在刪除基本表的同時，相關的依賴對象，如視圖，都被刪除。

三、索引的建立與刪除

1、建立索引

CREATE [UNIQUE] [CLUSTER] INDEX <索引名> ON <表名>(<列名>[<次序>],[,<列名>[<次序>]]……);

. 其中，<表名>是要建索引的基本表的名字。索引可以建立在該表的一列過著多列上，各列之間用逗號分隔。每個<列名>後面還可以用<次序>指定索引值的排列次 序，可選用ASC(升序）或DESC(降序),缺省值為ASC。

.UNIQUE表明此索引的每一個索引值只對應唯一的數據記錄。

.CLUSTER表示要建立的索引是聚簇索引。所謂的聚簇索引是指索引項的順序與表中記錄的物理順序一致的索引組織。

例如： CREATE CLUSTER INDEX Coursename ON Course(Cname);

將會在Course表的Cname列上建立一個聚簇索引，而且Course表中的記錄將按照Cname值得升序存放。

2、刪除索引

 DROP INDEX <索引名>;

例如： 刪除Course表的Coursename索引

 DROP INDEX Coursename;

刪除索引的同時，系統會同時從數據字典中刪去有關該索引的描述
