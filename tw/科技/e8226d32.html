一、簡介

docker鏡像是由多個只讀層疊加而成，啟動容器時，Ddokcer會加載只讀鏡像層並在鏡像棧頂部添加一個讀寫層。

如果運行中的容器修改了現有的一個已經存在的文件，那該文件將會從讀寫層下面的只讀層複製到讀寫層，該文件的只讀版本依然存在，只是已經被讀寫層中該文件的副本鎖隱藏，這個就叫"寫時複製（cow）"機制。

通俗易懂的就是說用戶在使用Docker的過程中，往往需要能查看容器內產生的數據，或者備份容器內的數據，甚至需要多個容器之間數據共享。這就需要持久化。

實現方式：數據持久實現-->使用或者關聯到宿主主機存

容器中管理數據數據主要有兩種類型：數據卷、數據卷容器。

二、數據卷

數據卷是一個給容器使用的目錄，他繞過容器的系統，有很多用處，如下：

1、數據可以在容器之間共享使用

2、對數據卷的修改會立馬生效

3、對數據卷的更新，不會影響鏡像

4、卷會一直存在的，知道沒有容器使用它。

數據卷的使用和linux系統monut一樣。


在使用docker run的時候使用-v參數即可在容器內創建一個數據卷，可以多次使用（創建多個卷）


使用方法如下：

# docker run --name d2 -it -v /data busybox

在容器內使用df命令查看

我們寫入一個文件，從宿主主機上查看

宿主主機上查看

我們從宿主主機上寫入文件

到容器中查看

以上就是數據卷

補充：數據卷可以掛載目錄，當然也可以直接掛載文件。

# docker run --name d3 -it -v /root/.bash_history:/root/.bash_history busybox

這樣就可以記錄在容器裡輸入過的命令了。一般情況不用這種方式，容易導致inode的改變。官方推薦還是使用目錄。

三、數據卷容器

如果要在容器之間共享一些持續更新的數據，最簡單的方式就是使用數據卷容器，數據卷容器其實就是一個簡單的容器。用他來提供數據卷給其他容器掛載使用。

方法一：

先使用數據卷掛載方法，掛載

# docker run --name vol01 -it --rm -v /data/vol01:/data busybox

/data/vol01這個目錄一般會自動創建，當然用創建好的也行

# docker inspect vol01

# docker inspect -f {{.Mounts}} vol01

[{bind /data/vol01 /data true rprivate}]

依然創建文件

下面啟動兩個容器分別使用

docker run --name vol02 -it --rm -v /data/vol01:/data busybox

docker run --name vol03 -it --rm -v /data/vol01:/data busybox

方法二：

也可以使用下面的命令--volumes-from，順帶做一個共享網絡的測試--network

joined containers，在nginx對外，tomcat，mysql不需要對外的情況

這樣共享網絡資源，減少開銷

docker run --name joined -it -v /data/joined/vol01:/data/web/html busybox

其他容器使用

docker run --name nginx --network container:joined --volumes-from joined -it --rm busybox

docker run --name tomcat --network container:joined --volumes-from joined -it --rm busybox

以上nginx-tomcat-mysql等就是屬於容器編排--docker-compose

--volumes-from可以多次使用，如果刪除容器，卷並不會被刪除。如果要刪除一個卷，必須刪除關聯容器。

使用數據卷容器，可以自由升級避免數據丟失。
