Spring切面獲取不到request的參數值

2020-11-14
| 科技

問題

請求方式 POST ， 傳參方式 form-data

在 Controller 中通過 request.getParameter("name"); 獲取到數據，在切面裡 相同方式獲取不到數據

切面獲取 request 的方法為

HttpServletRequest request = ((ServletRequestAttributes) RequestContextHolder.getRequestAttributes()).getRequest();

解決方法

通過如下方法可以獲取到數據

MultipartResolver resolver = new CommonsMultipartResolver(request.getSession().getServletContext());MultipartHttpServletRequest multipartRequest = resolver.resolveMultipart(request);MultipartFile file = multipartRequest.getFile("file");String key = multipartRequest.getParameter("key");

原因

在如下過濾器中，上傳文件中的content-type:multipart/form-data使用獲取request.getParameter(key)無法獲取相應的值。需要藉助Spring框架中的CommonsMultipartResolver.resolveMultipart(HttpServletRequest request)將request轉為MultipartHttpServletRequest，從而使用getParameter(key)方法獲取指定的值；

引發了新的問題

在切面裡添加如上代碼後，在 Spring Controller 中獲取不到參數

在 web.xml 中發現 xxsFilter 過濾

需要將上述代碼加到 Filter 中

 @Override public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException { HttpServletRequest httpRequest = (HttpServletRequest) request; String contentType = httpRequest.getContentType(); if (contentType != null && contentType.contains("multipart/form-data")) { MultipartResolver resolver = new CommonsMultipartResolver(httpRequest.getSession().getServletContext()); MultipartHttpServletRequest multipartRequest = resolver.resolveMultipart(httpRequest); // 將轉化後的 request 放入過濾鏈中 request = multipartRequest; } chain.doFilter(request, response); }
