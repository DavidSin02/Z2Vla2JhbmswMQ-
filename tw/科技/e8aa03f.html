如何得心應手的使用Linux的通配符？

2020-10-29
| 科技

引言

計算機術語 Wildcard 中文翻譯為‘通配符’，這個意譯可謂是通俗易懂。

然而，我們從‘通配符’這個中文術語中卻看不到一點兒原詞彙 wildcards 的影子。因為 wildcard 中既沒有通配(universual)，也沒有符(symbol)。倘若直譯，wildcard 字面含義 ’狂野卡‘，反倒更加簡潔有力。狂野到何種地步呢？狂野到萬能的地步。wildcard 的功能實際就是一張萬能卡，是我們玩鬥地主用的萬能牌。

鑑於此，本文將以‘萬能卡’指代‘wildcard’。Linux的資深讀者，請自行以‘通配符’腦補替換。

wildcard（萬能卡）包含三個字符，分別是星號 *，問號 ？和方括號 []。我們一一闡述。

星號 * 萬能卡

字符星號(*)匹配零個或多個字符。我們拿到鬥地主的一把牌，星號這張萬能卡，可以當大王用，可以當王炸用，可以當四個A的炸彈用，甚至可以當做整副牌來用。這張萬能卡代表零張到N張牌。

比如，我們列出當前目錄下，所有以字母 D 起頭的文件和目錄

$ echo D* # 以D為開頭Desktop Documents Downloads$ echo D*s # 以D為開頭，s為結尾Documents Downloads

我們拿到這樣一張星號萬能卡，鬥地主自然橫掃千軍。


問號 ? 萬能卡

字符問號(?)匹配任意一個字符。問號萬能卡就是‘癩子鬥地主’中的萬能牌，可以當大王用，可以當小王用，可以當A用，但是隻能作為一張牌。想要得到王炸，必須有好運氣得拿到兩張萬能牌。

$ echo s?v # 使用一張萬能卡srv$ echo s?? # 使用兩張萬能卡srv sys$ echo s* # 區別星號萬能卡sbin snap srv sys

方括號 [] 萬能卡

方括號[]是匹配其中的任意字符。方括號就是握牌的手，手中拿著一副牌，出牌的時候從中抽出來任意一張打出去。需要注意的是裝入[]內的字符，只能匹配一個結果。比如 a[xy]b 表示的是 axb 或者 ayb，而不能表示axyb。

鬥地主裡，我們用[JQKA]表示任意一張大牌，用[3456789]表示任意一張小牌。

但是按照上面[3456789]這種寫法的邏輯，如果將表達所有的26個英文字母，是否要背誦一遍字母表呢？還好，我們有character-class（字符類）這個功能。表達任意26個小寫字母只需寫成 [a-z]，26個大寫字母則寫成[A-Z]，表達任意一個數字寫成[0-9]，同時表示三者則寫作[a-zA-Z0-9]

另外，如果排除這些指定的字符，就使用!，如[!0-9], [!a-z] [!A-Z]，表示除這些之外。


組合工具 {}

上述的星號，問號與方括號是萬能卡（wildcard）的三種形式。觀察發現，上面的案例中有個顯著的缺陷，就是一次只能匹配一類。比如案例一當中，‘echo D*’匹配目錄下以D開頭的文件或者目錄。如果我既想要列出以大寫D開頭的，又想要以小寫d開頭的全部文件和目錄，所能採取的方案只能是分別運行 ‘echo D*' 和 ’echo d*'。

組合工具 {} 正是解決這一問題，它如繩索一般將各自為戰的命令栓到一起。

$ echo {D*,d*,T*} # 輸出分別以D,d,T開頭的目錄Desktop Documents Downloads dwhelper Templates Test

另外大括號{}，作為獨立的用法可以輸出連續的字符，比如

$ echo to{o..r}ltool topl toql torl



設置 shopt

上文中我們介紹了wildcard的三個符號以及其組合工具。這四個基本功能有其侷限性。

最大的侷限是只能在單層目錄內匹配查找，不能實現遞歸查找。

比如當我試圖在 '~/Documents/OrgMode/ORG'目錄下，找到所有的擴展名為‘org’的文件。

$ ls **/*.orgls: cannot access '**/*.org': No such file or directory

此時，我們需要求助 shopt(shell options) 更改 shell 的默認設置。

shopt -s globstar # s 為 set

此時再嘗試查找目錄下所有的org文件

關閉 globstar，則運行

shopt -u globstar # u 為unset

收尾總結

我們從鬥地主的一副牌開始，依次介紹了三種萬能卡：星號(*)，問號(?)和方括號([])，以及組合工具，並在最後引入了 shopt(shopt) 更改系統的默認設置，釋放萬能卡的更大能量。
