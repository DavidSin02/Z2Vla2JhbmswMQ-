一、事務

定義：

例如：

在銀行活動中，“由賬戶A轉移資金額X到賬戶B”是一個典型的銀行數據庫業

務。這個業務可以分解為兩個動作：

1）從賬戶A中減掉金額X

2）在賬戶B中加入金額X

這兩個動作應當構成一個不可分割的整體，要麼完成其中的所有動作，要麼不執

行其中任何動作，二者必居其一。這兩個動作就是一種“不可分割”的業務單

位 。一旦某個環節失敗，就需要回滾（恢復到初始狀態）

事務的ACID特性：

1） 原子性（Atomicity）

2）一致性（Consistency）

3） 隔離性（Isolation）

4）持續性（Durability ）

在SQL Server中數據庫分三種：

自動提交事務：這是 SQL Server 的默認模式，它將每條單獨的 T-SQL 語句視為

一個事務，如果成功執行，則自動提交；如果錯誤，則自動回滾。

如：

一旦執行失敗，自動回滾

顯示事務：用BEGIN TRANSACTION明確指定事務的開始，這是最常用的事務類

型。

隱性事務：通過設置SET IMPLICIT_TRANSACTIONS ON 語句，將隱性事務模式設

置為打開，下一個語句自動啟動一個新事務（不需要BEGIN TRANSACTION），每

個事務仍以COMMIT或ROLLBACK語句顯式完成。再下一個 Transact-SQL 語句又將

啟動一個新事務。

通過SET IMPLICIT_TRANSACTIONS OFF把隱性事務模式關閉。 (隱形事務會佔

用大量資源，一般都不推薦使用)

T-SQL使用下列語句來管理事務：

開始事務：BEGIN TRANSACTION

提交事務：COMMIT TRANSACTION

回滾（撤銷）事務：ROLLBACK TRANSACTION

存儲點語句：SAVE TRANSACTION

所謂事務存儲點就是在事務過程當中插入若干個標記，當事務執行中出現錯誤時，可

以不撤銷整個事務，只是撤銷部分事務，將事務退回到某個事物存儲點。一旦事務提

交或回滾，則事務結束。

例如：

/*

創建一個事務，把學號為00015的選修課程號003改為001、選修課程號004改為

006，以上兩個操作任意一個操作失敗，事務回滾，查看事務執行結果。

*/

begin transaction t1_studentupdate scset CNo='001'where SNo='00015' and CNo='003'update scset CNo='006'where SNo='00015' and CNo='004'if @@ERROR!=0 rollback transaction elsecommit transaction t_student

/*創建一個事務，把學號為00015的的選修課程號003改為001、選修課程號004改為

006，第一個操作失敗，事務回滾，成功則設定為保存點，第二個操作失敗，則事務

回滾，查看事務執行結果。*/

begin transaction t_studentupdate scset CNo='001'where SNo='00015' and CNo='003'if @@ERROR!=0 rollback transactionsave transaction t_studentupdate scset CNo='006'where SNo='00015' and CNo='004'if @@ERROR!=0 rollback transaction t_student elsecommit transaction t_student

/*

啟動隱式事務，更新成績後跟新課程號，然後關閉隱式事務

*/

set implicit_transactions on use SMUPDATE SCSET Score=score+1WHERE SNO='00001' AND CNO='001'UPDATE SCSET CNO='008'WHERE SNO='00001' AND CNO='001'commit transactionset implicit_transactions off
