Java可重入讀寫鎖實現：ReentrantReadWriteLock

2020-10-29
| 科技

概述

  • ReentrantReadWriteLock包含讀寫兩把鎖，如下：
// 寫鎖public ReentrantReadWriteLock.WriteLock writeLock() { return writerLock; }// 讀鎖public ReentrantReadWriteLock.ReadLock readLock() { return readerLock; }
  • 對同一線程而言，讀，寫讀（先寫後讀），寫寫是共享的，讀寫（先讀後寫）是互斥的；對不同線程而言，讀是共享的，讀寫，寫讀，寫寫是互斥的。

讀寫鎖

  • 寫鎖的請求：

（1）當前沒有線程在執行讀寫，則可以進行寫操作；

（2）當前存在線程在進行寫操作，當前請求寫操作的線程就是當前正在進行寫操作的線程時，才能繼續發起寫請求；如果當前是其他線程在寫，則當前線程無法請求寫；

（3）當前不存在寫，但是存在線程在讀，則無法請求寫。

  • 讀鎖的請求：

當前不存在寫的線程，或者當前正在寫的線程就是當前請求讀的線程，則可以成功請求讀；如果當前存在其他線程在寫，則無法請求讀。

總結

  • 對於寫的請求，要麼當前不存在線程在進行讀寫，要麼是該請求寫的線程在寫，則可以成功獲取寫鎖進行寫操作，即寫對同一線程是可重入的。如果當前存在其他線程在寫，或者存在線程（當前或者其他線程）在讀，則無法獲取寫鎖進行寫操作。即讀寫是互斥的，同一線程的寫是可以共享的，可重入的。
  • 對於讀的請求，如果當前不存在線程在寫，或者當前在寫的線程就是該請求讀的線程自身，則可以成功獲取讀鎖進行讀操作；如果當前存在其他線程在寫，則無法成功執行讀請求。
