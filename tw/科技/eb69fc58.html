git 解決衝突的命令

2020-11-14
| 科技

幾乎每個程序員都有一個GitHub賬號，成為程序員自後你會發現使用GitHub搜索問題會比百度或者谷歌等搜索引擎，還沒有申請GitHub的你趕快去申請吧！GitHub的上面開源的語言和各種程序庫，上面的文檔是跟mk一樣的閱讀代碼特別的舒服，使用GitHub離不開git的使用，現在也有很多的工具免命令比如烏龜仔，但是我還是喜歡使用命令行來操作。

如何申請GitHub和安裝git我就不詳細介紹了，不是本篇文章的重點，自己自行谷歌或者百度去吧！

git 分三個區域：

  1. 工作區（R）
  2. 暫存區 (B)
  3. 版本區 (V）

理解git的這3個區的區別基本就學會一半的git了，這裡推薦廖雪峰老師的git教程那裡面寫的很詳細！本篇主要是講git衝突的解決。

git pull

命令用於從另一個存儲庫或本地分支獲取並集成(整合)。

作用是：取回遠程主機某個分支的更新，再與本地的指定分支合併，它的完整格式稍稍有點複雜。

相當於：

git fetch origin 獲取遠程倉庫的代碼

git merge origin/master 本地代碼跟遠程代碼合併

git pull 是上面兩個命令的組合。

注意：在執行代碼 git merge origin/master 的時候有可能會報錯，

執行 git branch origin/master md 也就是將遠程倉庫分支更改名字，在執行合併代碼的命令就行。

介紹一個命令：

git branch --set-upstream master origin/next

git 追蹤關係：origin 追蹤遠程的 origin、next
