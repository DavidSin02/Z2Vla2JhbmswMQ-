算法小專欄：散列表（二）

級別： ★☆☆☆☆

作者： MrLiuQ

審校： QiShare團隊

接上篇散列表（一），本篇將重點介紹：解決散列衝突的四種方案。

一、開放定址法：

1.1 定義：

描述：為產生衝突的地址H(key)求得一個新的地址序列：Hi =（H(key)+ di）% m （i=1，2，3，...，m-1），其中H(key)為哈希函數，m為表長，di稱為增量序列。

增量序列通常有以下3種取法：

  • 線性探測再散列：di = 1，2，3，...，m-1
  • 二次探測再散列：di = 12，-12，22，-22，32，-32，...，k2（k<=m/2）
  • 偽隨機探測再散列：di = 偽隨機數序列

1.2 舉例：

例如，在長度為11的哈希表中，已填入關鍵字 17，29，60的記錄，哈希函數為：H(key) = key % 11。

計算：

H(17) = 17 % 11 = 6。故將**關鍵字“17”存在下標為6的位置，位置空著，所以存入未衝突。

H(29) = 29 % 11 = 7。故將關鍵字“29”存在下標為7的位置，位置空著，所以存入未衝突。

H(60) = 60 % 11 = 5。故將關鍵字“60”**存在下標為5的位置，位置空著，所以存入未衝突。

這時存入第四個關鍵字：38.

計算：H(38) = 38 % 11 = 5。出現衝突，下標為5的位置已存有60。

  • 方式一：線性探測再散列。

開始嘗試逐次追加di = 1，2，3，...，m-1

得到地址6 => 依然衝突， 得到地址7 => 仍然衝突， 得到地址8 => 不衝突，存入。

嘗試追加 di = 12，-12，22，-22，32，-32，...，k2（k<=m/2）

首先，追加12，地址6仍然衝突。 再，追加-12，地址4無衝突，可以存入。

二、鏈地址法：

2.1 定義：

描述：將所有哈希地址相同的記錄都鏈接在同一鏈表中，以此來解決衝突。

2.2 舉例：

已知一組關鍵字為（19，14，23，01，20，68，84，27，55，11，10，79）。 則按哈希函數H(key) = key % 13，鏈地址處理衝突如下圖：

三、再哈希法：

描述：產生衝突時計算**另一個哈希函數（散列函數）**的地址，直到衝突不再發生為止。

優點：這種方法不容易產生聚集。 缺點：增加了計算的時間成本。

四、建立公共溢出區：

描述：把衝突的都放在另一個溢出表中，而不會存在基本表裡。

這也是解決哈希衝突的一種辦法，假設哈希函數的值域為[0，m-1]，那就創建一個[0，m-1]的基本數組，每塊內存存放一個記錄，另外創建一個溢出數組，一旦發生哈希衝突，就將衝突的值添加至溢出表。
