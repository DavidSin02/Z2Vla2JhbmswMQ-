新版的Kafka Producer是否仍然還有producer.type 屬性設置？

2020-10-29
| 科技

老版的Kafka 中說producer.type 是一個重要的屬性，但是新版的文檔中並沒有提到這一點。

新版的新版的Kafka Producer是否仍然還有producer.type 屬性設置？

新版的Producer 默認是異步發送消息的（async），但是可以通過調用future.get() 變成同步發送消息。

由於send() 方法默認是異步的，該方法將返回消息發送結果為Future<RecordMedata>。如果調用future.get() 方法，將阻塞該請求，直到完成請求，並返回RecordMetadata 對象或者在發送消息時拋出異常。

採用同步模式發送消息記錄時，系統的性能會下降很多，因為需要等待返回的消息寫入結果。在生產環境中，一般不建議這樣使用，建議採用異步模式。

// 實例化一個消息記錄對象，用來保存topic,key,value等等ProducerRecord<String, String> record = new ProducerRecord<String, String>( topic, key, value);// 調用send() 方法和回調函數kafkaProducer.send(record, new Callback() { @Override public void onCompletion(RecordMetadata recordMetadata, Exception e) { if (e != null) { e.printStackTrace(); } else { System.out.println("offset: " + recordMetadata.offset()); } }});
