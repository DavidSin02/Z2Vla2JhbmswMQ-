詳解Queue模塊

2020-11-14
| 科技

Queue模塊實現了多生產者，多消費者隊列，對實現多個線程之間安全地交換信息非常有用。

該模塊實現了三種類型的隊列，它們的區別僅在於檢索的順序。在FIFO 隊列中，添加的第一個任務是第一個檢索的任務。在 LIFO隊列中，最近添加的任務是第一個檢索的（像堆棧一樣）。使用優先級隊列，任務將保持排序（使用heapq模塊），並首先檢索最低值任務。在內部，這三種類型的隊列使用鎖來臨時阻止競爭線程; 但是它們並非旨在處理線程內的重入。此外，該模塊實現了“簡單” FIFO隊列類型SimpleQueue。

Queue模塊定義了以下類和異常：

class queue.Queue（maxsize = 0 ）

FIFO隊列的構造函數。maxsize是一個整數，用於設置可以放入隊列的任務數的上限。達到此大小後，將無法繼續放入，直到消耗隊列項。如果 maxsize小於或等於零，則隊列大小為無限大。

class queue.LifoQueue（maxsize = 0 ）

LIFO隊列的構造函數。maxsize是一個整數，用於設置可以放入隊列的任務數的上限。達到此大小後，將無法繼續放入，直到消耗隊列項。如果 maxsize小於或等於零，則隊列大小為無限大。

class queue.PriorityQueue（maxsize = 0 ）

優先級隊列的構造函數。maxsize是一個整數，用於設置可以放入隊列的項目數的上限。達到此大小後，阻止放入，直到消耗隊列項。如果 maxsize小於或等於零，則隊列大小為無限大。首先檢索最低值的任務，任務的典型模式是以下形式的元組：(priority_number, data)如果數據元素不具有可比性，則可以將數據包裝在忽略數據項的類中，並僅比較優先級編號：

類queue.SimpleQueue

FIFO隊列的構造函數。簡單隊列缺少任務跟蹤等高級功能。（版本3.7中的新功能。）

異常queue.Empty在對象為空時調用非阻塞get()（或 get_nowait()）時引發異常Queue。異常queue.Full在對象已滿時調用非阻塞put()（或 put_nowait()）時引發異常Queue。

隊列對象

隊列對象（Queue，LifoQueue或PriorityQueue）提供下面描述的公共方法。

Queue.qsize（）

返回隊列的大小。注意，qsize（）> 0不保證後續的get（）不會阻塞，qsize（）<maxsize也不保證put（）不會阻塞。

Queue.empty（）

如果隊列為空返回True，否則返回False。如果empty（）返回True，則不保證對put（）的後續調用不會阻塞。類似地，如果empty（）返回False，則不保證對get（）的後續調用不會阻塞。

Queue.full（）

如果隊列已滿返回True，否則返回False。如果full（）返回True，則不保證對get（）的後續調用不會阻塞。同樣，如果full（）返回False，則不保證對put（）的後續調用不會阻塞。

Queue.put（item，block = True，timeout = None ）

將項目放入隊列。如果可選的block為true且timeout為 None（默認值），則在必要時阻塞，直到有空閒插槽可用。如果 timeout是一個正數，它會阻止最多超時秒，Full如果在該時間內沒有可用的空閒槽，則會引發異常。，如果空閒插槽立即可用，則將項目放在隊列上，否則引發Full異常（在這種情況下忽略超時）。

Queue.put_nowait（item）

相當於put(item, False)

Queue.get（block = True，timeout = None ）

從隊列中刪除並返回一個項目。如果可選的參數block為true且 timeout為None（默認值），則在必要時阻止，直到某個項可用為止。如果timeout是一個正數，它會阻止最多超時秒，如果在該時間內沒有可用項，則會引發異常。否則（參數block為false），如果一個項立即可用則返回一個項，否則引發異常（在這種情況下忽略超時）。

Queue.get_nowait（）

相當於get(False)。

Queue.task_done（）

表示以前排隊的任務已完成，由隊列使用者線程使用。對於用於獲取任務的每個get（），對task_done（）的調用將告訴隊列任務的處理已完成。如果 join()當前正在阻止，則它將在所有項目都已處理後恢復（意味著task_done()已收到已put()進入隊列的每個項目的呼叫）。如果被調用次數超過隊列中放置的項目，將會拋出一個異常ValueError。

Queue.join（）

阻止直到隊列中的所有項目都已獲取並處理完畢。每當項目添加到隊列時，未完成任務的計數就會增加。每當消費者線程調用task_done()以指示該項目已被檢索並且其上的所有工作都已完成時，計數就會下降。當未完成任務的數量降至零時，join()取消阻止。如何等待排隊任務完成的示例：

SimpleQueue對象

SimpleQueue 對象提供下面描述的公共方法。

SimpleQueue.qsize（）

返回隊列的大小。注意，qsize（）> 0不保證後續的get（）不會阻塞。

SimpleQueue.empty（）如果隊列為空返回True，否則返回False。如果empty（）返回False，不保證對get（）的後續調用不會阻塞。

SimpleQueue.put（item，block = True，timeout = None ）

將項目放入隊列。該方法永遠不會阻塞並始終成功（除了潛在的低級錯誤，例如無法分配內存）。可選的參數block和timeout 被忽略，僅提供兼容性Queue.put()。CPython實現細節：此方法具有可重入的C實現。也就是說，一個put()或一個 get()調用可以被put() 同一個線程中的另一個調用中斷，而不會死鎖或破壞隊列中的內部狀態，這使得它適用於__del__方法。

SimpleQueue.put_nowait（項目）

相當於put(item)，提供兼容性 Queue.put_nowait()。

SimpleQueue.get（block = True，timeout = None ）

從隊列中刪除並返回一個項目。如果可選的參數block為true且 timeout為None（默認值），則在必要時阻止，直到某個項可用為止。如果timeout是一個正數，它會阻止最多超時秒，如果在該時間內沒有可用項，則會引發異常。否則（block為假），如果一個項立即可用則返回一個項，否則引發異常（在這種情況下忽略超時）。

SimpleQueue.get_nowait（）

相當於get(False)。
