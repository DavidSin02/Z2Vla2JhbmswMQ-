介紹

PageHelper是mybatis框架上的一款分頁插件。通過它友好的api設計，可省去寫count，分頁sql嵌套的繁瑣工作。同時插件內部兼容不同數據庫間(mysql、Oracle等)的分頁實現。開發者只需關注具體業務sql，其他的統計封裝交給PageHelper插件動態處理。

作為java後端開發者，這是一款必不可少的基礎組件，大大減少了手動寫分頁列表查詢的工作量。

官方開源地址：

GitHub - pagehelper/Mybatis-PageHelper: Mybatis通用分頁插件

插件集成

添加maven依賴

<dependency>  <groupId>com.github.pagehelper</groupId><artifactId>pagehelper</artifactId><version>5.1.10</version></dependency>

添加mybatis配置插件

<plugins>    <plugin interceptor="com.github.pagehelper.PageInterceptor"></plugin></plugins>

PageHelper動態的對count方法做了2個處理。

1.findOrderByUserId方法添加_COUNT後綴。

2.用select count(0) 替換原有sql select 查詢字段，並去掉了order by排序。這是PageHelper根據業務sql做了優化，精簡了sql，提高count的執行效率。

優化

從上面的日誌分析，我們大致可猜到PageHelper的動態處理。在count方法部分，插件動態的生成了一個findOrderByUserId_COUNT sql。該sql的規則也可以從PageHelper插件源代碼中得到確認。


如上截圖，count方法中，countMsId通過業務sql id和countSuffix後綴拼接生成。而這個countSuffix就是_COUNT。

接著看代碼我們就不難發現，PageHelper插件會優先取工程中存在_COUNT後綴的sql，取不到則動態生成。

通過以上邏輯，我們就可以在項目中按它的規則，預先編寫{業務id}_COUNT的sql，實現count分頁邏輯的替換。

再回過來看我們之前的訂單分頁sql，from語句中使用left join和用戶表關聯。由於左連接在count上對order訂單主表數量不產生任何影響。所以我們可以通過重寫count方法，實現分頁sql的優化。

從優化後的sql可知，count從原有兩張表變成了一張表。如果業務sql中相關表數量量比較大，或者其他join表比較多時，count優化將有比較好的效果。
