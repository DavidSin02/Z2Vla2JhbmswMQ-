如何寫計算機相關專業的畢業論文？

2020-10-29
| 科技

有幸被北京高校的一些學生會及社團聘請為設計方面的顧問老師，在與同學的接觸諮詢當中，有很多同學諮詢了網站建設及多媒體專業的畢業設計及論文撰寫問題。在這裡給討教這些問題的同學做一個統一的輔導，希望對學生畢業論文有所幫助，給大學生活畫上漂亮的最後一筆。

畢業設計和畢業論文是本科生培養方案中的重要環節。學生通過畢業論文，綜合性地運用幾年內所學知識去分析、解決一個問題，在作畢業論文的過程中，所學知識得到疏理和運用，它既是一次檢閱，又是一次鍛鍊。不少學生在作完畢業設計後，感到自己的實踐動手、動筆能力得到鍛鍊，增強了即將跨入社會去競爭，去創造的自信心。

一、論文分類

不同的院系，不同的指導教師對本科畢業論文有不同的指導方法，在計算機系，本科畢業論文通常可以分為如下4種類型：

（一）系統實現型論文

完成一個不太大的實際項目或在某一個較大的項目中設計並完成一個模塊（如應用軟件、工具軟件或自行設計的板卡、接口等等），然後以工程項目總結或科研報告、或已發表的論文的綜合擴展等形式完成論文。

（二）應用開發型論文

對一個即將進行的項目的一部份進行系統分析（需求分析，平臺選型，分塊，設計部分模塊的細化）。例如對一個大中型企業管理信息系統中的財務部分進行調研，分析和設計等，這類工作可以沒有具體編程，但應得到有關方面的初步認可，有一定的工作量。例如打印後有30頁以上的報告。

（三）分析比較型論文

對某一項計算機領域的先進技術或成熟軟件進行分析、比較，進而能提出自己的評價和有針對性創見。例如XML目前是因特網上新湧現的標記語言，它較HTML有許多無可比擬的優點，其中XML-QL是基於XML提出的一種新型查詢語言，分析總結這一新型查詢語言並探索它的實現途徑是十分有意義的工作。再如對自由軟件數據庫系統MySQL，分析總結其在Web應用上的特色，並能將有關技術用於自己研製的系統中。這類工作要注意把分析和實驗相結合，不要只停留在消化上。消化是前提，吸收和轉化才是工作的重點。

（四）理論探索型論文

對某一個計算機科學中的理論問題有一定見解，接近或達到了在雜誌上發表的水平。例如，解決了一個眾所周知的問題，糾正了某雜誌上理論的錯誤且提出解決修正方案，或自己提出有意義的模型，定義，且有定理、命題、或性能比較、分析、測試報告等。

二、論文的組織安排

依據上述的論文類型，由於各類工作特點的不同，在蒐集整理論文的素材、組織安排論文的結構方面，應有所不同，做到有的放矢。

（一）系統實現型論文：

收集整理系統體系結構，程序模塊調用關係，數據結構，算法，實驗或測試等內容，論文結構一般安排如下：

1、引言或背景（概述題目背景，實現情況，自己開發的內容或模塊）；

一般談課題意義，綜述已有成果，如“誰在文獻某某中做了什麼工作，誰在文獻某某中有什麼突出貢獻”，同時分析存在問題，引出自己工作必要性、意義和價值、創新點和主要思想、方法和結果。然後用“本文組織如下：第二節第三節.... ，第四節.....”作為這段結束。

2、系統體系結構（強調系統的整體性，突出自己工作在整體中的位置）；

3、主要實現功能的描述（包括模塊調用關係，數據結構，算法說明，依據內容多少此部分可安排兩到三節）；

4、實驗或測試；

5、總結。

（二）應用開發型論文：

收集整理應用項目的背景分析，需求分析，平臺選型，分塊，設計部分模塊的細化，使用的開發工具的內容。論文結構一般安排如下：

1、引言（重點描述應用項目背景，項目開發特色，工作難度等）；

2、項目分析設計（重點描述項目的整體框架，功能說明，開發工具簡介等）；

3、項目實現（重點描述數據庫設計結果，代碼開發原理和過程，實現中遇到和解決的主要問題，項目今後的維護和改進等，此部分可安排兩到三節）；

4、總結。

（三）分析比較型論文：

收集整理有關的最新論文或軟件，分析比較心得，實驗數據等內容。論文結構一般安排如下：

1、引言（重點描述分析對象的特點，分析比較工作的意義，主要結果等）；

2、分析對象的概括性描述；

3、分析比較的主要結果（如果是技術分析，給出主要數據，如果是軟件分析，給出代碼分析結果，實驗過程等）；

4、分析比較的評價和系統應用（可以給出基於分析比較的結果，提出某些設計實現方案，和進行一些實驗工作；

5、結論。

（四）理論探索型論文：

收集整理問題的發現，解決問題所用到的基本知識，解決問題的獨特方法，定理證明，算法設計和分析。論文結構一般安排如下：

1、引言（重點描述要解決的問題的來源，難度，解決問題的主要方法等）；

2、基本知識（解決問題涉及的基本定義，定理，及自己提出的概念等）；

3、推理結論（給出問題解決方案，包括定理證明，算法設計，複雜性分析等）；

4、結論。

上述論文結構的安排和劃分不是絕對，可依據各自工作的實際情況，採用某一種或某幾種混合的方式指導自己論文的寫作。但無論採用什麼樣的結構，一定要作到心中有數，貫穿一條邏輯線索，切不可泛論成文或堆砌成文，時刻突出自己的工作內容。

三、寫作技巧和注意事項

在撰寫的過程主要注意如下幾點：

（1）學術論文要求文字流暢，層次清晰，詞藻不能過分華麗，近年國內外在計算機方面的論文的語言較以前生動，但用詞都以準確，不過分誇張為度。

（2）標題要能反映內容。如有新意更好。

（3）學術刊物上論文摘要一般300字左右，需要較高的綜合能力。而學生畢業論文摘要可以寫一頁（1000字左右），因此活動空間較大，要好寫一些。

（4）前言部分要寫問題背景，動機，要說明自己工作的工作有用，有意義（才能鼓勵讀者讀下去）在前言中簡述自己作的主要結果，（例如明確地列出幾條，說明本文的主要工作，主要成果）。前言部分的末尾介紹文章的組織情況，各節內容。

（5）突出特色，一個項目中工作很多，有些是同行皆知，自己也無創新的，可以略寫，要多寫自己遇到的特殊困難和創造的特殊解決方法。使人家讀了能學到東西，看到作者的創意。為了突出特色，還可以在一節中分成小標題寫，一個問題一小段，與計算機相關的論文中還可以把數據結構和程序片段夾敘夾議呈現出來，比只用漢字解釋得更清楚。

（6）要上升到理論高度，一般碩士論文會要求這一點。如果本科生能夠在畢業論文中得出有意義的定理或命題，一般會認為是好成果。可能會被推薦到雜誌上發表，或在考研、求職中得到承認。

（7）各部分應詳略得當。一般地，畢業論文應該主要寫作者的見解和工作， 把自己的創新寫深寫透。綜述要有述有評有比較。只要是作了實驗，寫了程序，一般是想寫的太多，寫不完。反之，會感到空虛，沒有寫的，有的學生抄書來充篇幅，效果不好。一般地介紹相關工作（前人的成果）部分不應該超過總篇幅的1/6。

（8）完整的論文應包括測試、運行結果及分析（圖表或曲線）。

四、參考文獻及引用

參考文獻是論文寫作中的重要一環，帶著問題讀文獻或雜誌上的論文，不但注意學習學術內容，同時也注意學習選題方法、格式、標準的套話、起承轉合的語言和方式，參考文獻的寫法，字體的運用等等。在論文中明確給出參考文獻的出處，既代表著對前人成果的繼承，也代表了對他人研究成果的尊崇，是科學研究中應培養的正確的科學態度和做人準則。

參考文獻的寫法：按參考文獻在論文中出現的順序，用編碼的方式給出，如[1]，[2]，等。參考文獻包括專著、學術期刊、學術會議、技術報告等。

1、論文目錄

論文目錄由撰寫者自己根據所編寫的論文進行總結，這裡略。

2、諸論

在Internet飛速發展的今天，互聯網成為人們快速獲取、發佈和傳遞信息的重要渠道，它在人們政治、經濟、生活等各個方面發揮著重要的作用。Internet上發佈信息主要是通過網站來實現的，獲取信息也是要在Internet海洋中按照一定的檢索方式將所需要的信息從網站上下載下來。因此網站建設在Internet應用上的地位顯而易見，它已成為政府、企事業單位信息化建設中的重要組成部分，從而倍受人們的重視。電子購物時下已成為目前網站建設最熱門的話題，適用於想通過建立網站宣傳產品或服務，提升企業形象，擴大品牌影響，通過網絡的交互性，從而實現網上或網下定購。但是商務實用型網站是比較龐大的一個系統，它必須擁有會員系統、查詢系統、購物流程、會員服務、後臺管理等等功能模塊。

2.1設計思想

商務實用型網站，是在網絡上建立一個虛擬的購物商場，讓瀏覽者在網絡上進行購物行為。網上購物以及網上商店的出現，避免了挑選商品的煩瑣過程，讓人們的購物過程變得輕鬆、快捷、方便，很適合現代人快節奏的生活；同時又能有效的控制“商場”運營的成本，開闢了一個新的銷售渠道。

對於該網站的實現的功能設計如下：

（1）採取會員制、身份驗證等一系列措施，保證交易的成功率。

（2）強大的搜索查詢功能，快捷的找到感興趣的商品。

（3）會員購物流程：瀏覽、將商品放入購物車、去收銀臺。每個會員有自己專用的購物車，可隨時訂購自己中意的商品結賬完成購物。購物的流程是指導購物車系統程序編寫的主要依據。

（4）完善的會員服務功能：可隨時查看帳目明細、訂單明細。

（5）設計特價商品展示，能夠顯示企業近期所促銷的一些特價商品。

（6）後臺管理使用本地數據庫，與因特網上的數據庫實時同步，保證購物定單安全及時有效處理強大的統計分析功能，便於管理者及時瞭解財務狀況、銷售狀況、把握客戶心理。

2.2 開發工具的選用及介紹

對於電子購物網站技術的核心在於“購物車”部分，包含了一個基於WWW的購物系統，這裡使用最簡單易學的ASP+Access技術製作而成，這樣可以簡化學習的難度，讓讀者迅速掌握電子購物網站的建設。

ASP的優勢：

Active Server Pages：“動態服務器網頁”，一般簡稱為“ASP”，ASP之所以能受到大家的重視與使用的原因，主要在於所產生的執行結果都是標準的HTML格式，而且這些程序是在網絡服務端中執行，使用一般的瀏覽器（如IE 或Netscape）都可以正確地獲得ASP的“執行”結果，並且將這ASP執行的結果直接在瀏覽器中“瀏覽”，不像VBScript或 JavaScript是在客戶端（Client）的瀏覽器上執行，若使用VBScript來設計程序，客戶端（Client）在ＩＥ瀏覽器中可以顯示程序執行的結果，可是，客戶端（Client）若使用Netscape瀏覽器就無法顯示VBScript的執行結果。

ASP的特點：

任何開發工具皆可發展ASP。只要使用一般的文書編輯程序，如Windows記事本，就可以編輯。當然，其他網頁發展工具，例如，FrontPage Express、ＦrontPage等也都可以；不過還是用Dreamweaver來寫，這樣很方便。

語言相容性高

ASP與所有的ActiveX Script語言都相容，除了可結合HTML，VBScript、Java Script、Active X服務器組件來設計外，並可經由“plug-In（外掛組件模組）的方式，使用其他廠商（Third Party）所提供的語言。

易於操控數據庫

ASP可以輕易地通過ODBC(Open Database Connectivity)驅動程序連接各種不同的數據庫，例如：Acess、Foxpro、dBase、Oracle等等，另外，ASP亦可將“文本文件”或是“Excel”文件當成數據庫用。

ASP的六大內部對象

Request、Response、Server、Session、Application、Object Context 從客戶端取得信息將信息送給客戶端提供一些Web服務器工具儲存在一個Session內的用戶信息，該信息僅可被該用戶訪問在一個ASP-Application中讓不同的客戶端共享信息可以用來配合Microsoft Transaction 服務器進行分佈式事務處理，綜合分析所以選擇了ASP結合Dreamweaver MX 2004架構北京京美文儀網站。

3、網站總體分析與設計

首先分析購物系統的組成結構，前提是該購物車系統基於ASP +Access結構，也就是說，使用Access數據庫存儲有關數據，使用ASP訪問數據庫(可以是本機訪問，也可以是遠程訪問)，進行產品列表、顯示、增加等操作。設計的功能模塊流程如下：

首先，客戶端瀏覽器送出HTTP訪問請求，WWW服務器受到請求以後，判斷是否需要向數據庫查詢，如果需要，生成SQL查詢指令，並送到數據庫服務器中。這部分功能主要由ASP程序實現。Access數據庫服務器收到相應端口送來的查詢請求以後，進行數據庫、表的操作，以表單形式向查詢者返回查詢結果。最後，WWW服務器收到查詢結果，使用其中的數據生成標準的HTML頁面，並將HTML代碼返回給原訪問者。這部分功能同樣由ASP編程實現。

通過上面電子購物系統功能的分析，網上購物商城型的網站主要由如下功能模塊組成：

（1）前臺網上銷售模塊。所謂前臺網上銷售模塊，就是指客戶在瀏覽器中所看到的直接與客戶面對面的銷售程序，包括：瀏覽商品，訂購商品，查詢定購，購物車等功能。

（2）後臺數據錄入模塊前臺所銷售商品所有數據，其來源都是後臺所錄入的數據。

（3）後臺數據處理功能模塊所謂後臺數據處理，是相對於前臺網上銷售模塊而言，網上銷售的數據，都放在銷售數據庫中，對這部分的數據進行處理，是後臺數據處理模塊的功能。

（4）用戶註冊功能模塊，用戶當然並不一定他立即就要買東西，可以先註冊，任何時候都可以來買東西，用戶註冊的好處在於買完東西后無需再要輸入一大堆個人信息，只須將賬號和密碼輸入就可以了。

（5）訂單號模塊所謂訂單號模塊，就是客戶購買完商品後，系統自動分配一個購物號碼給客戶，以方便客戶隨時查詢帳單處理情況。瞭解現在貨物的狀態。

（6）促銷價模塊，當有促銷價時，結算是以促銷價為準。如沒有促銷價，則以正常的價格為準。使其能得到詳細的信息，真正做到處處為顧客著想。

4.8購物流程說明

網上購物系統通過上面幾個小節的分析與設計在購物及後臺管理上基本上已經完成，但還有很多的說明頁面，如製作與購物相關的一些說明client.asp，用來說明售後服務的頁面service.asp，關於企業的內容簡介頁面about_us.asp等是需要企業由自身出發，在頁面中輸入企業相關的一些信息及購物說明。

關於購物車其他的說明頁面這裡也不再介紹，主要對購物流程進行舉例說明。網上購物系統要根據開發的系統設計一個購物流程，在首頁或者是其他功能頁面說明購物、結算、售後服務等詳細的過程。這也要根據企業自己的物流配送相結合。購物流程一般可以分為五步走：

第一步：註冊選購

採購者需要在網上先進行註冊後才可以採購商品，選擇好需要的商品後，單擊需要購買的商品，進入北京京美文儀購物車頁面，該頁面有以下幾個購物信息：“商品名稱、說明、普通註冊會員價、數量、圖片說明、產品是否在架”，如果不需要購買其他商品，可以單擊“放入購物車”，網上購物系統自動累計購買的商品，單擊購物車上的【去結算】按鈕，進入結算中心頁面；如果還需要選購其他商品，可以選擇該頁面中的【繼續購物】按鈕，系統將跳回購物頁面，在選擇好另外商品後，再單擊【放入購物車】即可看見累計選購信息。

第二步：輸入採購者信息

當此次選購結束後，進入結算中心頁面，該頁面除了有所選購的商品信息外，還有“收貨人、送貨地址、聯繫電話、手機、Email、附言”，需要逐一選擇或者填寫，確認訂單信息正確後，單擊【結算】按鈕後進入訂單信息最終頁面。

第三步：生成訂單

訂單信息最終頁面將顯示出“訂貨人信息、總計金額”等信息，在看清確認無誤後，單擊【生成訂單】按鈕，結束購物。

第四步：匯款

請通過郵件或者銀行向北京京美文儀匯款。

第五步：送貨

北京京美文儀會採用所選擇的配送方式將所需要的商品寄給您，注意查收並核對。

最後需要將整個購物系統直接傳到服務器上進行測試。

6、論文總結

通過這次ASP畢業設計，我首先感覺到了動態網站開發的重要性，特別是動態網站ASP開發，總結出了動態網頁的製作主要包括以下幾個工作環節：

1.建立靜態網頁

動態網頁是一種交互式網頁，所有的交互式網頁，都來自於靜態網頁。因此在建立數據庫網頁時需要建立一個靜態網頁。這個靜態網頁要與其他的網頁一樣，包含靜態文字、圖片、動畫、超級鏈接、按鈕以及表單等等。

2.動態網頁需要數據庫的支持

實質上，動態網頁就是一個可以訪問數據庫的網頁。在建立數據庫網頁前，要建立一個數據庫。在建立數據庫時，還要根據項目的具體要求設計數據庫的結構。

3.加入動態網頁技術

在靜態網頁中加入ASP、JSP、PHP代碼或通過使用其他動態網頁技術訪問數據庫，將數據庫中的數據顯示在網頁中，或將網頁中的數據記錄到數據庫中。最後還需要調試應用程序。對於Web應用程序來說，所謂運行，就是利用瀏覽器打開它，對於傳統的靜態頁面文件，也即帶有.html文件擴展名的文件來說，直接用瀏覽器打開它就可以完成測試，但是對於動態頁面，也即帶有.asp等文件擴展名的頁面文件來說，必須將它們放入真正的Web站點，才能進行測試，直接用瀏覽器打開它是不行的。

4.需要安裝Web服務器程序

本書以IIS作為Web服務器程序。一旦安裝了Web服務器程序，就相當於將計算機設置成為一臺真正的Internet服務器，所以只需啟動瀏覽器，並在地址欄中鍵入映射站點的地址，就可以進行系統的、細緻的測試，一切就像訪問真正的站點一樣。在利用瀏覽器測試之餘，很可能需要對代碼進行編輯。

在Dreamweaver MX 2004中建立數據庫網頁的過程大致相同，具體的步驟如下：

（1）分析動態網站的要求，建立適合的數據庫。

（2）定義網站站點。

（3）創建靜態網頁。

（4）創建數據源。

（5）創建數據鏈接。

（6）創建數據集。

（7）在網頁中添加服務器端行為。

（8）調試數據庫網頁併發布使用。

這個就是我在做網站的最大體會，以前在看書的時候沒有這種感覺的。
