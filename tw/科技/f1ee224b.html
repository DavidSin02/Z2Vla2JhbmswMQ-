Java集合-Queue

2020-11-14
| 科技

Queue（java.util.Queue）代表著一種在尾部插入數據，從頭部移除數據的數據結構，這與超市排隊的工作原理類似。

Queue接口是 Java Collection 的一個子接口interface，他代表著有序的對象序和 Java List比較類似，但它的預期用途略有不同。因為Queue接口是Collection接口的子類，所以Collection接口中的所有方法Queue接口也能使用。

Queue的實現

既然Queue是個接口，那麼初始化時就要使用它的具體實現，有下面兩種實現：

  • java.util.LinkedList
  • java.util.PriorityQueue

LinkedList是一個非常標準的隊列實現，元素存儲在隊列內部是一個標準的鏈表數據結構。這樣可以快速的在尾部插入數據，在頭部移除元素。

PriorityQueue內部存儲元素的順序是自然順序(如果實現了Comparable)，或者根據Comparator排序。

java.util.concurrent包中也有Queue的實現 。

Queue queueA = new LinkedList();Queue queueB = new PriorityQueue();

在大多數隊列實現中，隊列的頭和尾在相反的兩端，但是，可以實現隊列接口，使隊列的頭和尾在同一頭，那樣的就是一個Stack了。

Queue的泛型

默認Queue中加入的Object對象，但是可以使用泛型：

Queue<MyObject> queue = new LinkedList<MyObject>();

現在Queue中只能插入MyObject對象實例，從中獲取對象時不需要強制轉換：

Queue<MyObject> queue = new LinkedList<MyObject>(); MyObject myObject = queue.remove(); for(MyObject anObject : queue){   //do someting to anObject...}

主要這兒在循環裡面沒有使用強制類型轉換，因為使用了MyObject作為泛型。

Queue添加元素

Queue接口中有兩個方法可以添加元素 add()和offer() ，這兩個方法都在Queue的尾部加入元素，不同的是如果Queue滿了， add()方法會拋異常，offer()方法會返回false:

Queue<String> queue = new LinkedList<>(); queue.add("element 1"); queue.offer("element 2");

從Queue中取元素

可以通過 poll()或者remove()方法從Queue中取元素， poll()和remove() 方法都會移除Queue中的第一個元素，不同的是，如果Queue是個空那麼 poll()方法返回null而 remove()方法會拋異常：

Queue<String> queue = new LinkedList<>(); queue.add("element 1");queue.add("element 2"); String element1 = queue.poll(); String element2 = queue.remove();

調用poll()方法將移除隊列的第一個元素 - "element 1"，再調用rmove()方法將移除第二個元素，因為第一個元素被poll()移除後第二個元素變成了第一個元素- "element 2"。

從Queue中Peek

可以查看 Queue中的第一個元素通過element()或者peek()方法，element()方法將返回Queue的第一個元素，如果Queue是空， element()方法將拋NoSuchElementException，下面是例子：

Queue<String> queue = new LinkedList<>(); queue.add("element 1");queue.add("element 2");queue.add("element 3"); String firstElement = queue.element();

執行完代碼後firstElement的值是“element 1”，是 Queue中的第一個元素，peek()和element()很類似，除了當Queue 為空時不拋異常：

Queue<String> queue = new LinkedList<>(); queue.add("element 1");queue.add("element 2");queue.add("element 3"); String firstElement = queue.peek();

從Queue中移除元素

remove()方法可以移除第一個元素：

Queue<String> queue = new LinkedList<>(); queue.add("element 0");queue.add("element 1"); String removedElement = queue.remove();

移除Queue中的所有元素

可以通過clear() 方法移除Queue中所有元素：

queue.clear();

獲取Queue的大小

可以通過size()方法獲取 Queue的大小：

Queue<String> queue = new LinkedList<>(); queue.add("element 1");queue.add("element 2");queue.add("element 3"); int size = queue.size();

執行代碼後size的值是3。

檢查Queue中是否包含某個元素

可以通過contains()方法檢查是否包含某個元素：

Queue<String> queue = new LinkedList<>(); queue.add("Mazda"); boolean containsMazda = queue.contains("Mazda");boolean containsHonda = queue.contains("Honda");

執行後containsMazda 的值是true，containsHonda 值是false。

迭代Queue的所有元素

下面是迭代Queue:

Queue<String> queue = new LinkedList<>(); queue.add("element 0");queue.add("element 1");queue.add("element 2"); //access via IteratorIterator<String> iterator = queue.iterator();while(iterator.hasNext(){  String element = iterator.next();} //access via new for-loopfor(String element : queue) {    //do something with each element}
