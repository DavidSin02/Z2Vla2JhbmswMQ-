1. 流式思想概述

Java8中有兩大最為重要的改變。第一個是 Lambda 表達式；另外一個則是 Stream API。

Stream流式思想類似於工廠車間的“生產流水線”，Stream流不是一種數據結構，不保存數據，而是對數據進行加工處理。Stream可以看作是流水線上的一個工序。在流水線上，通過多個工序把原材料加工成一個商品。


Stream API（java.util.stream）把真正的函數式編程風格引入到Java中，提供了一種高效且易於使用的處理數據的方式。


這張圖中展示了過濾、映射、跳過、計數等多步操作，這是一種集合元素的處理方案，而方案就是一種“函數模型”。圖中的每一個方框都是一個“流”，調用指定的方法，可以從一個流模型轉換為另一個流模型。而最右側的數字3是最終結果。

Stream API是用來處理集合、數組等容器中的數據的，處理操作有：查詢、篩選、刪除、過濾、統計、映射等。集合、數組講的是數據，負責存儲數據，Stream流講的是計算，負責處理數據。

Stream的執行流程：

1、獲取Stream流：通過一個數據源（如：集合、數組），獲取一個流

2、中間處理操作：中間操作是個操作鏈，對數據源的數據進行n次處理，但是在終結操作前，並不會真正執行

3、終結操作：一旦執行終結操作，就執行中間操作鏈，最終產生結果並結束Stream。終結操作執行後就不能再對數據進行加工處理，如果要加工需要重新創建Stream。

Stream流的特點：

（1）Stream流本身不負責存儲數據，存儲數據是用集合、數組等數據結構。

（2）Stream流不會修改數據源的數據，每次處理都會返回一個持有結果的新Stream流對象。

（3）Stream流的操作是一個延遲操作，不調用終結方法 中間處理操作不會執行

注意：Stream流和IO（InputStream/OutputStream）流沒有任何關係。

2. 獲取stream流的方式

java.util.stream.Stream<T>是Java 8新增的stream流接口。（這並不是一個函數式接口。）

獲取一個流非常簡單，有以下幾種常用的方式：

  • Collection接口中的默認方法default Stream<E> stream()，所有的Collection集合都可以通過該方法獲取流對象
  • 通過數組工具類Arrays類的靜態方法public static <T> Stream<T> stream(T[] array)獲取數組對應的流對象。
  • Stream接口的靜態方法public static<T> Stream<T> of(T... values)獲取數組對應的流對象

根據Collection集合獲取流

Collection接口的所有實現類均可通過stream方法獲取流。

@Test

public void test1() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

Stream<Integer> stream = list.stream();

}

3. 常用方法：

Stream API 分為兩類：終結方法和非終結方法。

  • 非終結方法：返回值類型仍然是Stream接口類型的方法，因此支持鏈式調用
  • 終結方法：返回值類型不再是Stream接口類型的方法，因此不再支持鏈式調用

注意：

  • 非終結方法是延遲執行的，調用終結方法時，才會真正執行前面的非終結方法
  • Stream API 僅對Stream流中的數據進行加工處理，不會修改數據源的數據。

forEach : 逐一處理

終結方法，調用該方法後不能調用其他方法，方法聲明：

void forEach(Consumer<? super T> action);

1

該方法接收一個Consumer接口對象的參數，會對每一個元素進行逐一處理，例如遍歷：

@Test

public void test1() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

Stream<Integer> stream = list.stream();

//可傳入lambda表達式、方法引用：Consumer<Integer> consumer = System.out::println;

stream.forEach(System.out::println);

}

filter：過濾

非終結方法，方法聲明：

Stream<T> filter(Predicate<? super T> predicate);

1

該方法接收一個Predicate函數式接口對象的參數作為篩選條件

篩選出集合中的偶數，代碼示例：

@Test

public void test3() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

//鏈式調用

long count = list.stream().filter(i -> i % 2 == 0).count();

System.out.println(count);

}

limit：取出前n個元素

非終結方法，方法聲明：

Stream<T> limit(long maxSize);

1

參數是一個long型，如果集合當前長度大於參數則進行截取；否則不進行操作

代碼示例：

@Test

public void test4() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

skip：跳過前n個元素

非終結方法，方法聲明：

Stream<T> skip(long n):

1

跳過前n個元素，獲取一個截取之後的新流，如果流的當前長度大於n，則跳過前n個，否則將會得到一個長度為0的空流。

代碼示例：

@Test

public void test5() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

list.stream().skip(3).forEach(System.out::println); //10 12 21

}

reduce：歸約

將流中元素反覆結合起來，得到一個值

終結方法，方法聲明：

T reduce(T identity, BinaryOperator<T> accumulator);

1

@Test

public void test11() {

List<Integer> list = Arrays.asList(1, 4, 3, 10, 12, 21);

//reduce：

//第一次，將初始值賦值給i1，取出第一個元素賦值給i2

//第二次，將第一次的結果賦值給i1，取出二個元素賦值給i2

//第三次，將第二次的結果賦值給i1，取出三個元素賦值給i2

//第四次，將第三次的結果賦值給i1，取出四個元素賦值給i2

System.out.println(list.stream().reduce(5, (i1, i2) -> i1 + i2)); //56

}

collect：收集Stream流中的結果到指定結構

Stream流中的結果收集到集合中

java.util.stream.Collector<T, A, R> 接口對象來指定收集到哪種集合中，java.util.stream.Collectors 類提供一些方法來返回 Collector 接口的對象。

@Test

public void test14() {

//轉為list集合

List<String> list = Stream.of("ccc", "abc", "bbb")

.collect(Collectors.toList());

System.out.println(list);

ArrayList<String> arrayList = Stream.of("ccc", "abc", "bbb")

.collect(Collectors.toCollection(ArrayList::new));

System.out.println(arrayList);

//轉為set集合

Set<String> set = Stream.of("aaa", "eee", "ddd")

.collect(Collectors.toSet());

System.out.println(set);

HashSet<String> hashSet = Stream.of("aaa", "eee", "ddd")

.collect(Collectors.toCollection(HashSet::new));

System.out.println(hashSet);

}
