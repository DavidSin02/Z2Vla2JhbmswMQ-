系統設計基礎：水平與垂直縮放

2020-10-29
| 科技


> Github Repository of vaquarkhan

如果您以前從未設計過系統，那麼本教程適合您！

想象一下一個場景，其中有一臺計算機，其中編寫了一些代碼。 這段代碼就像一個普通的函數，接受輸入併發出輸出。 一些公司正在查看此代碼，他們意識到這對他們確實有用，因此，他們願意為此付出代價。但是您無法四處尋找並將計算機提供給所有人。 所以你會怎麼做？

您使用將在Internet上運行的某種協議來公開代碼。基本上，您使用的是應用程序編程接口（API），該接口從代碼中接收請求並給出響應。

想象一下設置這臺計算機。 這將需要一個數據庫； 您將需要配置人們正在連接的API端點，並且還需要考慮到斷電會發生什麼。 如果有人拔掉了插頭，您的服務就會崩潰，並且您真的付不起錢，因為很多人為您支付服務費用。現在，您希望將服務託管在雲上。

那麼計算機和雲之間有什麼區別？

沒事 雲是有人提供給您以換取金錢的一組計算機。 例如，您付錢給AWS雲解決方案提供商，他們給了您計算能力。 通過計算能力，我的意思是說他們為您提供了他們在某處擁有的計算機，並且該計算機能夠運行您的代碼。 您如何將代碼實際存儲在他們的桌面上？ 好吧，您可以執行類似遠程登錄該桌面的操作。

這就是雲。 它是一組可以用來運行服務的計算機。簡單來說，雲計算意味著您可以通過Internet存儲和訪問數據和程序，而不是使用計算機的硬盤驅動器。 互聯網。 雲服務計費的工作方式與您支付天然氣或電費的方式相同：即付即用。

現在您已將服務器託管在雲上，該雲基本上是您不知道的遠程計算機，是時候關注業務需求了。現在有很多人正在使用您的算法，並且可以使用 問題是您的代碼無法處理所有這些請求。您該怎麼辦？ 解決方案之一是購買更大的計算機：您的計算機將更大，因此可以更快地處理請求。 另一個解決方案是購買更多計算機：請求可以在計算機之間隨機分配。 通過購買更大的計算機或購買更多的計算機來處理大量請求的能力稱為可伸縮性。

用技術術語來說，垂直擴展意味著向服務器添加更多資源（CPU / RAM / DISK），水平擴展意味著向服務器或數據庫添加更多處理單元或物理機。

垂直縮放：

優點：

●與運行多個服務器相比，它消耗的功率更少

●由於您只需要處理和管理一個系統，將減少管理工作

●因此，進程間通信速度更快。

●不需要像水平縮放那樣進行負載平衡。

●這是一個數據駐留的系統，因此數據保持一致。

缺點：

●單點硬件故障可能會導致更大的中斷

●將來可升級的範圍有限。

●實施的總成本確實很高

水平縮放：

優點：

●增強了彈性。 由於負載分佈在多個系統中，因此水平縮放更能抵抗系統故障。

●比垂直縮放便宜得多

●易於升級

●支持容量線性增加

缺點：

●不同的計算機正在處理不同的請求，這可能導致其數據不同步。

●可能需要更多的網絡設備，例如路由器和交換機

您應該使用哪一個？

如果您經營的是小型公司，並且不希望增加太多的用戶/數據，那麼垂直擴展可能是您的理想選擇。 但是，如果您在一家大型公司中工作，預計用戶/數據的數量會增加，那麼橫向擴展將適合您。 通常，實際業務會同時利用兩全其美的優勢：它們通過在水平擴展時添加頂級，功能強大的機器來利用垂直擴展的優勢。 它是將垂直縮放的速度和一致性與水平縮放的彈性和無限可伸縮性相結合的混合方法。
