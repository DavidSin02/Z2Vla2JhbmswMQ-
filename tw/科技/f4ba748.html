Arduino液晶顯示---1602液晶帶IIC模塊

2020-10-29
| 科技

在開始之前，我們先了解一下什麼是IIC。我們長話短說，只讓大家瞭解就行了。IIC就是一種通信協議，他只需要兩根線就可以讓外圍設備與單片機進行通信，所以硬件連接非常的方便。他也是一種總線的結構。

所以說到總線，他這兩根線上面是可以連接很多設備的。這些設備用地址位來驅動。所以IIC設備必須要有一個地址。比如我們常見的存儲芯片AT24C02。

I2C設備地址

協議格式中第一個字節(為slave address)由7位地址和一位R/W讀寫位組成的，這字節是個器件地址。

常用IIC接口通用器件的器件地址是由種類型號，及尋址碼組成的，共7位。

如格式如下：D7 D6 D5 D4 D3 D2 D1 D0

1、器件類型：D7-D4 共4位決定的。這是由半導公司生產時就已固定此類型的了，也就是說這4位已是固定的。

2、用戶自定義地址碼：D3-D1共3位。這是由用戶自己設置的，通常的作法如EEPROM這些器件是由外部IC的3個引腳所組合電平決定的（用常用的名字如A0,A1,A2）。這也就是尋址碼。所以為什麼同一IIC總線上同一型號的IC只能最多共掛8片同種類芯片的原因了。

3、最低一位就是R/W位，，“0”表示寫，“1”表示讀（通常讀寫信號中寫上面有一橫線，表示低電平）。所以I2C設備通常有兩個地址，即讀地址和寫地址。

1602液晶常用庫和函數

我們用1602液晶時，首先要將一個文件夾LiquidCrystal_I2C複製到Arduino安裝路徑裡面的庫文件夾中，因為默認安裝的時候是沒有帶這個庫的。如果不拷貝，編譯的時候程序就會有錯誤提示。

以下是常用的關於1602液晶的函數

lcd.setCursor(0,1); //設置列和行，都是從0開始。

lcd.print（）; //顯示

lcd.print(num); //顯示變量 lcd.print(‘9’);//顯示單片字符

lcd.print(“hello！”)； //顯示字符串

lcd.cursor(); //顯示光標 lcd.noCursor(); //關光標

lcd.blink(); //開閃爍 lcd.noBlink(); //關閃爍

lcd.display(); //顯示

lcd.noDisplay(); //不顯示

lcd.clear(); //清屏

lcd.backlight(); //開背光

lcd.noBacklight(); //關背光

IIC-1602液晶程序

#include <Wire.h>

#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27,16,2); // 將LCD地址設置為0x27以顯示16個字符和2行

void setup()

{

lcd.init(); // 初始化

lcd.backlight(); //開背光燈，這個功能如果用普通的1602是沒有的

}

void loop()

{

lcd.setCursor(0,0); //從第1行，1列開始顯示

lcd.print("Arduino Display!"); // 顯示字符串

lcd.setCursor(0,1); //從第2行，1列開始顯示

lcd.print("Hello,World!"); // 顯示字符串

delay(3000); // 延時5S

lcd.clear(); //清除屏幕顯示

delay(1000); // 延時1S

}
