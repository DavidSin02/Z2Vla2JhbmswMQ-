零基礎學C語言（7）：存儲類型

2020-11-14
| 科技

前四節，我們從編寫一個計算器的角度，討論出編程語言需要數據類型、變量和常量，並詳細介紹了這三個概念。那麼編寫一個計算器還需要哪些東西呢？

我們想想看，當我們想設計一個比較複雜的程序，例如開發一款遊戲時，我們不可能寫在一個文件下吧，那這個文件得多龐大。

比如說王者榮耀，是一個團隊共同開發的，例如我設計韓信的動作和技能介紹，他設計李白的鳳求凰皮膚，另一個人設計鏡的出裝。用膝蓋想一想，這麼多人不可能圍在一起對一個文件指指點點，否則策劃與程序員之間，豈不是天天“你在教我做事？”

雖然不在一塊，但各個文件之間沒有交流麼？

有時候李白和王昭君在同一局遊戲碰到，會產生一些奇怪的臺詞，所以不同文教是有交流的，那怎麼產生聯繫呢？

這就是今天的重點——存儲類型。

存儲類型是幹嘛的呢？

簡單來說，存儲類型是用來定義C程序變量/函數的範圍（可見性）和生命週期的。具體分為四類：auto、register、static、extern。

我們挨個來看：

auto存儲類

auto是用來修飾局部變量的，如果局部變量前沒有存儲類型，默認為auto類型，例如我們通常看到的int a;，實際是auto int a;。

register存儲類

register是寄存器類型，顧名思義，是把修飾的變量放到寄存器中，目的是為了提高程序的運行效率。但要記住，不管是單片機也好，計算機也罷，任何CPU的寄存器資源都是有限的，如果寄存器滿了，被修飾的變量就會默認回到auto類型。

一般格式：register 數據類型 變量；

extern存儲類

前面提過，如何在不同文件之間交流？

嘿嘿，extern就是一個很好的方法。

extern是外部引用類型，主要是引用統一工程中的全局變量或函數，什麼意思呢？就是說，我想用其他文件的變量或函數，那麼就需要使用extern這個關鍵字。

我們上一節提到過，定義在內存上是分配空間的，而使用extern是在聲明，意思是告訴編譯器該變量存在空間，但並不是在聲明處開闢的空間。

怎麼去用呢？

假設在第一個c文件，我們定義：int a;在另一個文件我們想要使用第一個文件中的a，我們就需要在第二個c文件中加上extern int a;這樣這個a就在這兩個文件共享。

static存儲類

static這個關鍵字非常重要，為啥呢？因為面試題經常考。

當static修飾全局變量或函數時，限制變量或函數的作用域，表示只能在本文中使用；當static修飾局部變量時，改變局部變量的存放位置，延長局部變量的生命週期，運算的結果保留上一次的結果。

“改變局部變量的存放位置”，怎麼理解這句話呢？

這就要提到C語言的內存管理了，這個我們後面詳細說，這裡我們簡單提一下：

首先，我們要知道計算機中的內存是分區來管理的，程序和程序之間的內存是獨立的，不能互相訪問，比如QQ和王者榮耀分別所佔的內存區域是不能相互訪問的。而每個程序的內存也是分區管理的，一個應用程序所佔的內存可以分為很多個區域。

我們需要了解的區域有四個，分別是棧區、堆區、靜態區和代碼區。通常我們定義的局部變量，是存放在棧區的，並且局部變量的生命週期在該語句塊執行結束時便結束了。

但我們用static修飾，該變量便存放在靜態數據區，其生命週期一直持續到整個程序執行結束。但是其作用域並沒有改變，它仍然是一個局部變量，作用域僅限於該語句塊。

而當static修飾全局變量或函數時，該變量或函數僅在本文使用，就沒法和其他文件產生交流，用extern關鍵字都不行。

這個我們一般什麼時候用呢？

因為每一個大工程都不是一個人完成的，當某人在某一文件使用一個全局變量時，另一個人在其他文件可能也定義了一個相同的全局變量，這樣會發生衝突，所以乾脆用static修飾，以防發生奇怪的BUG。

因為有了static這個關鍵字，變量從全局變量和局部變量這兩種，就變成了全局變量、局部變量、靜態全局變量、靜態局部變量四種。

他們之間怎麼區別呢？

全局變量、局部變量、靜態全局變量、靜態局部變量的區別

從作用域看：

1、全局變量具有全局作用域。全局變量只需在一個源文件中定義，就可以作用於所有的源文件。當然，其他不包含全局變量定義的源文件需要用extern 關鍵字再次聲明這個全局變量。

2、靜態局部變量具有局部作用域，它只被初始化一次，自從第一次被初始化直到程序運行結束都一直存在，它和全局變量的區別在於全局變量對所有的函數都是可見的，而靜態局部變量只對定義自己的函數體始終可見。

3、局部變量也只有局部作用域，它是自動對象（auto），函數的一次調用執行結束後，變量被撤銷，其所佔用的內存也被收回。

4、靜態全局變量也具有全局作用域，它作用於定義它的文件裡，不能作用到其它文件裡，即被static關鍵字修飾過的變量具有文件作用域。這樣即使兩個不同的源文件都定義了相同名字的靜態全局變量，它們也是不同的變量。

從分配內存空間看：

1、全局變量，靜態局部變量，靜態全局變量都在靜態存儲區分配空間，而局部變量在棧裡分配空間。

2、全局變量本身就是靜態存儲方式， 靜態全局變量當然也是靜態存儲方式。這兩者的區別雖在於非靜態全局變量的作用域是整個源程序。而靜態全局變量則限制了其作用域，即只在定義該變量的源文件內有效，在同一源程序的其它源文件中不能使用它。

· 1)靜態變量會被放在程序的靜態數據存儲區(全局可見)中，這樣可以在下一次調用的時候還可以保持原來的賦值。這一點是它與堆棧變量和堆棧變量和堆變量的區別。

· 2)變量用static告知編譯器，自己僅僅在變量的作用範圍內可見。這一點是它與全局變量的區別。

總結

本節介紹了auto、register、static、extern這四種存儲類型，並介紹了全局變量、局部變量、靜態全局變量、靜態局部變量這四種變量的區別，對於內存管理這一塊的知識點大家能看懂最好，不能看懂也沒關係，後面我們會詳述。存儲類型到這裡就是說完了，下一節將進入到運算符的學習。

上節作業分析

1.下列程序的輸出結果是16.00，請填空

#include <stdio.h>

int main（）

{

int a = 9, b = 2;

float x = _____,y = 1.1, z;

z = a/2 + b*x/y + 1/2;

printf("%5.2f\n", z);

return 0;

}

已知a = 9，b = 2；z = 9/2 + 2*x/1.1 + 1/2;在C語言中，如果運算時，浮點型向整型進行強制轉換時注意精度丟失的問題

怎麼理解呢？定義的a和b是整數，9/2 = 4.5，但由於a是int型，得到的結果為整數，即4，而1/2則為0，所以z = 4+2*x/1.1 = 16，解方程得x = 6.6

2.以下程序運行後的輸出結果是：（ ）

#include <stdio.h>

#define S(x) 4*x*(x)+1

int main()

{

int i=6,j=8;

printf("%d\n",S(i+j));

return 0;

}

根據上一節我們說的，遇到這種宏定義問題要整體代入，S(i+j) = 4*i+j*(i+j)+1 = 4*6+8*(6+8)+1 = 137
