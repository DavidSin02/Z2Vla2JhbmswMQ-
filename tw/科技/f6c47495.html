一、文件目錄的基本操作

說明：由於Windows和macOS文件目錄系統的格式不同，在沒有介紹到系統通配分隔符前，代碼中的文件目錄格式主要匹配macOS系統文件目錄結構，下面是Python中常用的文件目錄操作方法：

1. 創建文件--mknod()方法和open()方法

（1）mknod方法

import os# 創建文件file_path = "./file_path.txt"os.mknod(file_path)

在window平臺不支持這個操作，在Mac平臺雖然支持這個語法但是系統出於安全考慮也會拒絕這個操作，需要超級管理員權限。

（2）open方法

在桌面創建一個py_test.txt的文件

open("/Users/yushengtan/Desktop/py_test.txt",mode = "w",encoding = "UTF-8")

創建文件，文件都還沒有怎麼使用參數w來寫入呢？因為在沒有文件的情況下，系統會默認自動創建一個空白文件，這樣就達到了創建文件的功能。

2. 創建目錄--mkdir()方法

比如我們要在當前目錄下創建一個py_test的空目錄

path = "./py_test"if not os.path.exists(path):  # 如果目錄不存在    os.mkdir(path)else:    print("文件已存在")

4. 文件目錄程序完善

（1）跨平臺目錄分隔符

在windows系統裡面支持的路徑分隔符是反斜槓\，Linux和macOS系統路徑分隔符是斜槓/；os.path.sep或者os.sep提供了自動匹配系統的路徑分隔符。所以上面的代碼中的文件路徑在macOS系統下可以執行，在Windows就不能執行了，那麼如何讓我們的程序可以跨平臺執行呢？

解決辦法：代碼中路徑裡的斜槓或者反斜槓通過os.path.sep來代替，這樣在不同的操作系統下，路徑會自動適配。

（2）異常處理 涉及到文件和目錄的讀寫和刪除一定要用上異常處理，因為會有很多意外的情況發生，捕獲異常來防止程序崩潰

import osimport redef get_file_number(path):    """    統計出相應目錄下的文件數量    :param path: 提供的目錄    :return: 文件的數量    """    total_list = os.listdir(path = path)    file_number = 0    for i in total_list:        abs_path = path + i        if os.path.isfile(abs_path):            file_number += 1    return file_numberdef get_file(path):    """    找出文件中文件名為奇數的文件    ~~~~~~~~~~~~~~~~~~~~~~    :param path: 提供的路徑    :return: 返回文件名稱的集合    """    total_file = os.listdir(path = path)    # 實例化一個正則表達式對象    pattern = re.compile(r"[\w]+[13579][.][\w]")    # 新建list,存儲匹配上的文件名    need_file = []    # 遍歷    for i in total_file:        if pattern.search(i):            need_file.append(i)    return need_filedef delete_file(file_list,path):    """    刪除指定路徑的下文件    ~~~~~~~~~~~~~~~~~~    :param file_list: 提供要刪除文件的list    :param path: 刪除文件的連接    :return:    """    for current in file_list:        abs_path = path + current        try:            os.remove(abs_path)        except Exception as e:            raise eif __name__ == '__main__':    path = "/Users/yushengtan/Desktop/Demo/"    # 打印刪除前文件的數量    print("刪除前的文件數量",get_file_number(path))    # 找出文件名為奇數的文件    file_list = get_file(path)    print(file_list)    # 刪除符合條件的文件    try:        delete_file(file_list,path)    except:        print("刪除出現異常！")    else:        print("刪除已成功！")    # 刪除完後數量統計    print("刪除後的文件數量", get_file_number(path))    # 刪除的文件內容    for i in file_list:        print(i)

4. 讀取文件的三種方式

（1）read( )

每次讀取整個文件，它通常用於將文件內容放到一個字符串變量中

with open(path,mode="r",encoding="UTF-8") as fd:    content = fd.read()print(content)

（2）readlines( )

逐行讀取，將文件內容返回到一個列表中，每一個元素就是一行

with open(path,mode="r",encoding="UTF-8") as fd:    content_list = fd.readlines()for i in content_list:    print(i,end="")

（3）readline()

每次只讀取一行，通常比readlines( )慢得多，僅當沒有足夠內存可以一次讀取整個文件時才應該使用readline()

with open(path,mode="r",encoding="UTF-8") as fd:    content = fd.readline() # 讀取一行    while content:  # 如果有內容，循環執行，如果沒有內容那麼循環結束        print(content,end="")        content = fd.readline() # 讀取下一行

5. 讀取大文件

（1）常規做法 如果讀取的文件特別大，內存不夠會導致內存溢出MemoryError，常規的解決辦法有兩種：

  • 如果文件有分行，使用readline()方法；
  • 如果沒有分行，使用read(size)方法

（2）推薦做法 使用with結構的迭代器，進行迭代遍歷，for line in f，會自動地使用緩衝IO以及內存管理，而不必擔心任何大文件的問題。

with open(path,mode="r",encoding="UTF-8") as f:    for line in f:        print(line,end="")

6. 讀取二進制文件

計算機數據存儲在物理上是二進制的，所有文本文件與二進制文件的區別並不是物理上的，而是邏輯上的。這兩者只是在編碼層次上有差異，文本文件是基於字符編碼的文件，常見的編碼由ASCII編碼，Unicode編碼等等。二進制文件是基於值編碼的文件，常見的編碼有Base64編碼。使用Base64編碼存儲圖片、讀取圖片、展示圖片；

案例：

將文件夾中的圖片轉為base64編碼的二進制文件，並寫入磁盤

import base64path = "/Users/yushengtan/Desktop/Demo/2.png"with open(path,mode="rb") as fd:    image_data = fd.read()  # 讀取圖片文件    base64_data = base64.b64encode(image_data)    # 把編碼文件存儲到磁盤    write_image = open("/Users/yushengtan/Desktop/Demo/base64_image.txt","wb")    write_image.write(base64_data)    write_image.close()

案例：

將base64編碼的二進制文件解碼成圖片並寫入磁盤

path = "/Users/yushengtan/Desktop/Demo/base64_image.txt"with open(path,mode= "rb") as fd:    base64_data = fd.read() # 讀取二進制文件    image_data = base64.b64decode(base64_data)  # 把二進制base64文件解碼    # 將圖片轉存到磁盤    write_image = open("/Users/yushengtan/Desktop/Demo/image_write.png","wb")    write_image.write(image_data)    write_image.close()

五、文件的寫入

讀文件是把文件的中的內容讀取程序中來，寫文件是把程序中的內容寫入到文件中去；

（1）傳統方式

path = "/Users/yushengtan/Desktop/Demo/w_file01.txt"global fdtry:    fd = open(path,mode="w",encoding="UTF-8")    fd.write("yusheng123.cn")except Exception as e:    print("寫入文件出現異常！")else:    print("寫入成功！")finally:    fd.close()

2.write和writelines

write寫入單行，writelines多行寫入，多行寫入不是直接寫入，是要把要寫入的字符串存入列表、元組、集合、字典(key)，然後再寫入

with open(path02, mode="w", encoding="UTF-8") as fd:    # 寫入多行需要藉助於列表、元組 、集合、字典    list01 = ["1000","2000","3000"] # list集合可以作為多行寫入的參數    tuple01 = ("tuple1000","tuple2000","tuple3000")    #元組集合可以作為多行寫入的參數    set01 = {"set1000","set2000","set3000"} # set集合也可以作為多行寫入的參數    dit01 = {"上海":3000,"北京":4000,"廣州":3000} # 字典集合也可以作為多行寫入的參數，但是隻寫入key    # 處理list:寫完一行後換行    for i in range(0,len(list01)):        list01[i] = "第" + str(i+1) + "個元素" + list01[i] + "\n"        i += 1    # 輸出經過處理的結果    fd.writelines(list01)   # 寫入list集合

3.追加寫入

在文件寫入時使用mode參數如果是w，在打開文件時會將文件內容清空，如果我們想保留前面寫入的內容就要使用追加寫入的方法。

打開一個文件用於追加，如果該文件已經存在，文件指針將會放在文件結尾，新的內容將會被寫入到已有的文件內容之後，如果該文件不存在，創建新文件進行寫入。追加寫入的關鍵字是append，相應mode後的參數為a

4.讀寫模式

Python文件讀寫模式有以下幾種：

重點掌握：

（1）r 只讀模式

open( )函數的默認模式，只能讀，不能寫；

（2）w 只寫模式

打開前清空文件內容，只能寫，不能讀；

（3）a 追加模式

打開前不清空文件內容，只能追加寫入，不能讀；

（4）r+ 可讀可寫模式

不會創建不存在的內容，從頂部開始寫入，寫多少覆蓋多少；如果先讀後寫，結果就是追加寫入；如果先寫後讀，結果是覆蓋文檔相應位置字符，讀取的時候，從寫入之後的位置開始讀取

（5）w+ 可讀可寫

如果文件存在則覆蓋整個文件，不存在則創建文件，創建完成後立馬就讀是讀不到的，因為此時文件指針在文件的結尾處，如果要讀取文件需要將文件指針指向文件開頭fd.seek(os.SEEK_SET)。

import osprint("=======w+ 模式=======")path = "/Users/yushengtan/Desktop/Demo/mobile1.txt"with open(path,mode="w+") as fd:    fd.write("yusheng")    fd.seek(os.SEEK_SET)    print(fd.read())

（6）a+ 可讀可寫模式

從文件頂部讀取內容，從文件底部添加內容，不存在則創建，同樣的道理，如果寫入文件後直接讀取是讀取不到的，因為此時文件指針指向底部，如果想讀取文件內容則需手動將文件指針指向文件頂部。與w+模式的區別是可追加的讀寫。
