西門子PLC信號邊沿檢測與邏輯結果邊沿檢測有什麼不同？

2020-10-29
| 科技

信號的沿產生在當信號狀態發生相應的改變時。邏輯結果的沿產生在當操作數的邏輯運算結果與之前的邏輯運算結果相比較發生了改變。 下表解釋了兩種沿檢測的差異。

1、信號邊沿檢測

在功能塊圖 ( FBD ) 與梯形圖 ( LAD ) 中邊沿檢測是通過功能 “ POS ” ( 上升沿 ) 與“NEG ” ( 下降沿 )實現的。 第一次操作數的狀態（ 例如， 輸入位 I1.1 ) 與之前查詢的信號狀態進行比較 （例如，I1.1 )。如果實際信號狀態=1， 查詢之前的信號狀態=0， 那麼將認為產生一個上升沿並且功能“POS ” 的輸出將被置為1。為了“POS ” 能夠比較 I1.1 的實際信號狀態與之前的狀態，每一次查詢的實際信號狀態將緩存在標誌位中以便用於下一次查詢。

信號上升沿與下降沿檢測的信號圖的例子。信號“E1.2 " 為復位信號。

2、邏輯結果邊沿檢測

在 FBD 與 LAD 中實現邏輯結果邊沿檢測是通過“P ”(上升沿) 與 “ N ” (下降沿) 實現的。如果邏輯運算結果改變， 例如從0 到1，則該掃描週期中“P ” 輸出的信號狀態為“1 ”（見信號圖）。為了使該功能能夠識別出沿的變化， 該邏輯結果必須存儲在沿的標誌位中 （例如， M1.1 )。
