訪問控制列表－ACL

2020-10-29
| 科技

在企業網絡通信時，要保證網絡的安全可靠性和穩定性。訪問控制列表ACL（Access Control List）可以定義一系列不同的規則，設備根據這些規則對數據包進行分類，並針對不同類型的報文進行不同的處理，從而可以實現對網絡訪問行為的控制、限制網絡流量、提高網絡性能、防止網絡攻擊等等。

如上圖所示：ACL可以控制流量可訪問的區域。如圖：主機A B 可以訪問Internet網但是不能訪問服務器，主機C D 則相反。這都是通過ACL控制的。

如上圖所示：也可以通過ACL匹配網段進行加密處理。

ACL分三類：

ACL 2000-2999：可以匹配源IP地址。

ACL 3000-3999：可以匹配源IP 目的IP 源端口 目的端口等

ACL 4000-4999：可以匹配源Mac 目Mac和以太幀協議類型等。

ACL後面數字越高就越高級，可以匹配的條件就越多。

一條ACL可以匹配創建多個規則，數據會根據規則序號從上往下匹配，直到匹配到為止。若無匹配則丟棄數據。

配置：如圖所示，拒絕主機A訪問Internet網。

命令行：

ACL 2000（創建acl）

rule 5 deny source 192.168.1.0 0.0.0.255（創建第五條規則 拒絕源地址為 192.168.1.0掩碼為24位IP網段 rule 規則條目 不輸入默認第五條開始 再創建不輸入就從第十條開始目的是方便在前面添加規則 deny拒絕的意思 source源地址的意思 permit允許的意思）

int g0/0/0 （進入接口0）

traffic-filter outbound acl 2000 （調用acl 2000）

這樣配置192.168.1.0網段就不能訪問Internet網了

高級acl配置：

acl 3000

rule deny tcp soure 192.168.1.0 0.0.0.255 destination 172.16.10.0 0.0.0.0 destination-port ep 21（拒絕源地址為192.168.1.0IP網段的主機訪問目的IP為172.16.10.1端口號為21的服務器。）（tcp是協議 匹配所有tcp報文 destination是目的）

rule deny tcp soure 192.168.2.0 0.0.0.255 destination 172.16.10.2 0.0.0.0 （上面註釋那麼多這個應該懂了吧）

rule permit ip （匹配所有IP報文）

int g0/0/0

traffic-filter outbound acl 3000（調用acl 3000）

ACL就講到這裡啦，喜歡的點個關注哦。會持續更新網工知識。
