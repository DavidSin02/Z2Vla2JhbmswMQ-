想盡快脫單？數學最優決策告訴你：37%

2020-10-29
| 科技




大多數人都同意這個觀點：約會很難！！！（如果你不這麼認為，那太棒了!!!你可以不用花那麼多時間閱讀這篇文章了T - T）

現如今我們每週都花很多時間在Tinder或Subtle Asian Dating這些軟件上，覺得還不錯的人，就進入他的主頁然後聊天。

當你最終“找到了”的時候，你知道怎麼把完美自拍放到Tinder的個人資料裡，你可以很輕鬆地邀請那個在和你在同一個韓語班的可愛的女孩子吃飯，你會認為找到Mr或Mrs. Perfect一起安頓下來並不難。但是不！！！ 其實我們很多人都找不到合適的人生夥伴。

對於普通人來說，約會太複雜，太可怕了！！！

是我們的期望太高了嗎？是我們太自私了嗎？還是我們註定不會遇見那個個他/她呢？別擔心！這不是你的錯。你只是還沒有藉助數學手段。

在關係更深入發展之前你大概要約會多少人呢？

這是比較複雜的問題，所以我們必須向數學和統計學家請教。他們的答案是：37％。

那是什麼意思？

這意味著在你可能約會的那些人中，假設你預測自己在未來10年約會100人（對我來說，10個人更可能，但這是另一個問題），看過前37％或前37人之後，接受第一個比之前那些人好的那個人（如果這樣的人一直沒有出現，就等到最後一個人）。

他們是如何得到這個數字的？讓我們從數學角度挖掘一下。


簡單（或者說絕望）方法：


假設我們能預見會有N個人按順序進入我們的生活，並按照一些“匹配/最佳夥伴統計數據”進行排名。那個你想最終在一起的人排第一—我們稱這個人X。



在我們探討最佳約會政策之前，讓我們從一個簡單的方法開始。如果你非常想和在Tinder上遇到的一個人在一起安定下來怎麼辦呢？這個人就是X的機率是多少？

1 / N。

隨著n越大，我們考慮的時間範圍越大，這個概率就會趨於0。好吧，你可能不會在20年內約會10000人，但即使是1/100的小賠率也足以讓我覺得這不是一個很好的約會策略。

那我們該怎麼辦？

我們做約會時所做的事情。也就是說，我們希望與好幾個人見面，探索約會質量然後開始安定下來，而不是在第一個人出現時就做出承諾。所以這種約會包括探索部分和安定過程。

但是我們要探尋等待多久呢？

為了制定策略：你在N個人中約會M個人，拒絕他們所有人，並馬上與下一個比你目前為止看到的更好的那個人安頓下來。我們的任務是找到M這個最優值。如前所述，M = 0.37N。但是我們是怎麼得到這個數字的呢？


一個小模擬：


在R中運行一個小模擬，看看是否有M的最佳值的指示。

設置很簡單，代碼如下：

# a util function to simulate the 'best-partner rank'

perm_rank <- function(){

return (sample(1:100, 100))

}

#for each value of M, we simulate 1000 times

M_range <- 2:100, niter <- 1000

# declare a vector to store results, notice that if M = 1, the probability is 1/100

prob_result <- rep(1/100, 100)

# do a simulation for each value of M

for (M in M_range){

result <- rep(0, niter)

for (i in 1:niter){

order <- perm_rank() #simulate the order

highest_reject <- min(head(order, M-1)) # find the best among the first M-1 that gets rejected

if (highest_reject != 1){

accept <- order[order < highest_reject][1]

# we consider ourselves successful if:

# - rank 1 is not included in the first M-1 candidates

# - rank 1 is the first person who is better than all we have seen

if (accept == 1) result[i] <- 1

}

}

prob_result[M] <- mean(result)

}

這個模擬實驗還表明，我們考慮的N值越大，M越接近37%。下面的圖表顯示了增加候選人數的最佳比率M / N。




這裡有一些有趣的觀察結果：當增加我們候選N的數量時，不僅最優概率減小並且能看到收斂(靠攏)，最佳比率M / N也是如此。稍後，我們將嚴格證明兩個最優實體收斂到大約0.37的相同值。

你可能會想：“等一下，我能不能在N值非常小的情況下，讓找到最合適的人的概率最高呢？”這是部分正確的。基於模擬，在N = 3時，我們可以通過簡單地每次選擇第三人來實現高達66％的成功概率。那麼這是否意味著我們應該始終瞄準最多3個人來約會並和第三個人在一起呢？

可以。但問題在於，這種策略只會最大限度地提高在這3個人中找到最佳選擇的機會，這在某些情況下已經足夠了。但是，我們大多數人可能想要考慮更廣泛的選擇，而不是進入我們生活的前3個人。這與我們年輕時被鼓勵去多約會的原因基本相同：找出相互吸引的那類人，更好地瞭解約會、如何與伴侶共處，並且在這個過程中瞭解自己。

你可能會發現更樂觀的事實是，當我們增加約會的範圍時，找到完美丈夫/妻子的概率不會衰減到零。只要堅持我們的策略，我們可以證明存在一個閾值，在此閾值下最優概率不會下降。我們的下一個任務是證明我們的戰略的最優性並找到最低閾值。

我們能否嚴格證明37％的最優規則？


實際數學：


讓O_best成為最佳候選人（完美丈夫/妻子，The One，X，排名第一候選人等）的到達順序。我們不知道這個人什麼時候會到我們的生活中來，但我們肯定知道在下一個我們將看到的預定N人中，X將到達順序O_best = i。

讓S（n，k）成為在N個人中選擇X的成功事件，我們採用M = k的策略，即探索並斷然拒絕第一個k-1人，然後與比你到目前為止所看到的都要好的第一個人在一起。我們可以看到：

最後的話：

你是否應該使用此策略找到終身伴侶？這是否意味著你應該先在Tinder上的前37個有吸引力的個人資料中向左滑動，或者把滑了你的DM資料的37個人放到'看過'裡？

由你來決定。

該模型提供了最佳解決方案，如果你為自己設置了嚴格的約會規則：你必須設置一個特定數量的候選人N，必須有一個保證沒有平局的排名系統（對人們進行排名的想法對一些人來說並不適合），一旦你拒絕某人，你再也不能將他們考慮為約會對象。

顯然，現實生活中的約會更加混亂。

可悲的是，不是每個人都任你接受或拒絕— X，當你遇到他們時，他們可能會拒絕你！在現實生活中，人們有時會回去找他們之前拒絕的人，我們的模型不允許這樣做。很難通過約會的對人們進行比較，更不用說用一個統計數據來有效地預測一個人當配偶會有多好，並相應地對他們進行排名。而且我們還沒有解決最大問題：估計約會對象總數N幾乎不可實現的。設想自己花大部分時間來編寫代碼、寫關於20年內約會的文章，我的社交生活會豐富嗎？我能和10人，50人或100人約會嗎？

是的，這種絕望的方法可能會帶給你更糟的結果。

另一個有趣的事是，如果你認為永遠不會有最好的選擇，這時的最佳策略是什麼，在哪種情況下你會選擇第二好或是第三好的選擇。這些考慮是“博士後問題（the postdoc problem）”的普遍問題，它與我們的約會問題有類似的設置，並假設最好的學生將去哈佛（耶魯，duh！）。
