使用 Docker 容器快速開發物聯網應用程序

2020-10-29
| 科技

使用基於容器的虛擬化來開發、測試、部署和更新 IoT 應用程序

來源：ibm.com/cn

物聯網正在快速發展，它是智慧設備的高度互聯網絡，這些設備包括環境傳感器、健康跟蹤器、家用電器和工業設備等。到 2020 年，預計有 200 億個設備將連入網絡，這超過 PC、智能手機和平板電腦加起來的數量的兩倍。開發人員正在快速開始為 IoT 創建應用程序，而使用容器可在不同方面為他們帶來幫助。

容器是一種輕量型的虛擬化方法，開發人員可使用該方法快速、大規模地開發、測試、部署和更新 IoT 應用程序。許多 Web 和移動應用程序開發人員使用虛擬機管理程序（比如 VirtualBox）來運行虛擬機 (VM)，在一個跨平臺的開發、測試和部署工作流中虛擬化物理硬件。

基於容器的虛擬化（有時稱為操作系統級虛擬化）要輕量得多。每個容器作為隔離的用戶空間實例在一個共享的主機操作系統內核上運行。儘管操作系統是共享的，但各個容器擁有獨立的虛擬網絡接口、獨立的流程空間和不同的文件系統。可使用實現資源隔離的控制組來向這些容器分配系統資源，比如 RAM。在基於虛擬機管理程序的虛擬化中，每個 VM 運行自己的操作系統，這會增加系統資源的使用，與此相比，容器使用的磁盤和內存資源少得多。

Docker 是 Linux 上基於容器的虛擬化的開放平臺。通過 Docker，可以更快速、更輕鬆地構建容器，並且幾乎可以在任何地方部署這些容器：在私有云或公共雲中、在本地 VM 中或者在物理硬件（包括物聯網設備）上。IBM Containers 是基於 Docker 和 Kubernetes 的 IBM Cloud 功能部件，用於在 IBM Cloud 平臺上交付和部署容器化應用程序。

設置 Docker 以用於物聯網開發

物聯網應用程序面對的是各種各樣的設備平臺。在原型設計和開發的早期階段，開發物聯網項目時，通常採用的是基於通用微控制器的開發板或單板計算機 (SBCS)，比如 Raspberry Pi。

要為 Raspberry PI 設置 Docker，您將需要一個滿足以下條件的 Raspberry PI：具備 wifi（例如，Raspberry Pi Zero W 或 Raspberry Pi 3），包含寫入 Raspbian Jesse 鏡像的 microSD 卡，且啟用了 SSH。（觀看這個 IoT 實踐視頻，瞭解如何設置 Raspberry Pi。）您可以通過在 microSD 卡的引導分區上創建一個名為"ssh"的文件來啟用 SSH。可以通過 SSH（默認密碼為 raspberry）從 Mac 或 PC 以無頭模式來訪問 Raspberry Pi：

ssh pi@raspberrypi.local

在 raspbian 命令行中，可以通過運行以下腳本來安裝 Docker：

$ curl -sSL get.docker.com |sh

您需要將默認用戶 (pi) 添加到 docker 組以避免權限問題，然後將用戶 (su) 切換到 pi 以使更改生效，或者重新引導 Raspberry pi。

$ sudo usermod ‑aG docker pi$ su pi

一旦在 Raspberry Pi 上安裝了 Docker，就可以運行 docker ps 來確認 Docker 正在運行。

$ docker ps CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES

使用容器、鏡像和註冊表

每個 Docker 容器包含一個或多個運行的進程。Docker 容器從一個鏡像啟動，該鏡像指定了以下元素：

  • 要在容器內運行的應用程序的配置信息
  • 它的依賴項，比如庫和共享二進制文件

Docker 使用一種統一的文件系統，將鏡像存儲為一系列層。層會在構建過程中緩存，這使衍生鏡像的構建既快捷又高效。

這種分層方法還意味著 Docker 鏡像很小且可移植，通過將它們發佈到公共或私有註冊表，可以輕鬆地共享這些鏡像。Docker Hub 是 Docker 鏡像存儲庫的最大註冊表；它列出了超過 100,000 個存儲庫，並託管了許多私有存儲庫。如果您正在使用 IBM Cloud 平臺，那麼支持從 Docker Hub 中提取公共鏡像，並且還提供託管的私有鏡像存儲庫，這些存儲庫允許您在組織內共享私有鏡像。

如果使用一種流行的開源框架或服務，您可能能夠在 Docker Hub 中找到一個預先構建的公共鏡像，許多鏡像由與這些項目相關的開源社區來維護。Docker Hub Web 界面有一個較小的官方存儲庫列表，這是一個精挑細選的存儲庫列表，包含由 Docker 團隊對已知安全漏洞進行了測試的鏡像。

避免使用舊的 Docker 鏡像，即使這些鏡像的來源可靠，因為許多舊鏡像僅用於歸檔和測試用途。如果舊鏡像仍得到維護，它們會在 Docker Hub 中被標記為受支持。公共註冊表中的大多數鏡像都是社區貢獻的，可能沒有詳細備案或維護。懷疑有問題時，可使用一個官方鏡像作為基礎來創建自己的鏡像。

可完成以下步驟來創建鏡像：

  1. 從註冊表拉入一個基礎鏡像
  2. 交互式地運行一系列命令
  3. 提交結果作為新鏡像

例如，下面的命令列表從 Docker Hub 獲取官方的 Ubuntu version 14.04 鏡像，從該鏡像啟動容器，運行一個命令來將 Git 安裝到該容器中，然後列出創建的容器的 ID。

創建一個鏡像

$ docker pull armhf/alpine:latest$ docker run ‑t ‑i armhf/alpine apk add ‑‑no‑cache git$ docker ps –l

可發出下面這條命令來將此容器 (cb23e345fde0) 保存為新鏡像（名為 demo/git）：$ docker commit cb23e345fde0 demo/git。

要使用 IBM Container Extensions 工具在本地運行 Docker 命令，可將 docker 命令更改為 ice --local。例如，使用此命令：$ ice --local pull ubuntu:16.04

如果需要運行不止一兩個命令來設置應用程序環境，您需要創建一個 Dockerfile。Dockerfile 是指定用於構建鏡像的一組指令的文本文件。下面這個代碼清單中的 Dockerfile 創建了一個運行 Nginx Web 服務器的基礎鏡像。

創建一個運行 Nginx Web 服務器的基礎鏡像的 Dockerfile

FROM armhf/alpine:latestRUN apk update && apk add nginxRUN mkdir ‑p /var/www/htmlRUN chown ‑R nginx:www‑data /var/lib/nginx EXPOSE 80 443WORKDIR /etc/nginxCMD "nginx"

每個 Dockerfile 中的第一條指令是 FROM 指令，這表明基礎鏡像。Dockerfile 中的每個後續指令存儲為一個層：

  • RUN 指令運行 apt-get 等 Linux 命令。
  • ADD 和 COPY 指令將應用程序文件添加到容器中。
  • EXPOSE 指令打開端口。
  • ENV 指令配置環境變量。

最後，每個 Dockerfile 包含 WORKDIR、ENTRYPOINT 或 CMD 指令，用來指定在該容器啟動時，如何和在何處運行該應用程序。

運行 Docker 容器需要極少的開銷，所以可將您的應用程序分解為一組在不同的容器中運行的服務，這些容器可在運行時按名稱來鏈接。例如，您可將應用程序拆分為一個運行 Node.js 應用程序的容器，該容器鏈接到另一個運行 Redis 鍵值存儲的容器。

使用容器維護一致的 IoT 開發環境

為物聯網設備開發的嵌入式應用程序以後可以升級，從而在自定義原型開發板上運行，最終在生產設備上運行。您可能會開始使用現成的硬件，比如 Raspberry Pi，然後評估一系列潛在的微控制器和片上系統 (SoC) 設備，設備驅動程序需求各不相同，有時甚至不兼容。（可查看我在另一篇文章 IoT 硬件指南中介紹的不同 IoT 硬件。）每個設備或經過修正的設備可能還需要不同版本和配置的開發工具包，用於設備閃存、監控和通信。

容器可用於捕獲適用於每個修正設備的已知開發環境，並在開發者團隊之間共享此環境。例如，如果您的團隊正在使用幾種類型的 Arduino 兼容開發板，您可以創建一個包含 Arduino 命令行工具包的鏡像和一個用於串行通信的啞終端仿真程序，作為您的基線開發鏡像。然後，您可以使用這個基本 Arduino 開發鏡像為每個需要特定自定義驅動程序的開發板創建新的變體鏡像。

Docker 的分層文件系統在這種情況下有效地利用了空間，因為在創建新鏡像時只存儲了唯一的層。Docker history 命令顯示組成鏡像的各個層的列表，包括創建每個層的指令和層大小，例如：

IMAGE CREATED CREATED BY 7483ffb80dd6 5 minutes ago apk add ‑‑no‑cache git

層會被緩存，所以您可快速地嘗試鏡像的新變體。例如，您可能希望向驅動程序或開發工具應用更新。層還使您能夠輕鬆地回滾到鏡像的早期版本，在某處出錯時嘗試一種不同的方法。

如果對開發環境的更改取得成功，可將新鏡像推送到您的私有團隊註冊表，以快速將更改傳播給團隊的其餘成員。每個團隊成員拉入更新的鏡像時，他們已緩存了大部分層，所以 Docker 通常只需幾秒到幾分鐘時間運行創建新層的指令，即可讓新鏡像正常運行。與推送和拉入 VM 鏡像快照或為每個變體從頭構建完整鏡像相比，層節省了存儲空間，而且更重要的是節省了開發人員的時間。

將容器部署到 IoT 設備

Docker 化應用程序的魅力在於，在您構建了一個鏡像之後，幾乎可以在任何地方發送和運行此鏡像。如果您的物聯網設備運行 Linux，那麼可以直接在設備上部署容器。儘管此類物聯網設備上可用的系統資源有限，但部署 Docker 容器是可行的，因為它們的運行時開銷幾乎為零。您甚至可以在設備上運行多個容器，例如，在您希望並行運行應用程序的不同版本以進行比較的情況下，就可以執行此操作。

Docker 需要一個支持內核名稱空間和控制組的現代 Linux 內核，所以您首選的物聯網設備可能還沒有合適的基本鏡像。隨著合適的 SoC 的價格不斷下降（例如，9 美元的 C.H.I.P. 開發板），預計會開發越來越多的支持 Linux 的物聯網設備；這樣一來，就會為這些平臺提供更廣泛的可供選擇的 Linux 發行版。

Docker Hub 包含了許多基於 ARM 架構的鏡像，適用於基於 ARM 架構的數碼愛好者喜歡的 SBC，包括 Raspberry Pi、Orange Pi 和 BeagleBone Black。DockerHub 上的 arm32v7、arm32v6 和棄用的 armhf 鏡像是官方鏡像，是為了支持 DockerHub 的多架構而專門為 ARM 架構所構建的。DockerHub 上的 resin 組織也提供了第三方基本鏡像，旨在用於 resin.io 和 resinOS。

物聯網開發者面臨的挑戰是，要持續更新互聯設備網絡，並在可能並不可靠的低帶寬無線連接上運行。此外，他們還必須考慮如何維護其中許多設備收集的高度個人化的敏感數據的安全性。Docker 容器有助於提高物聯網的安全性，它們通過 Docker 1.10 中引入的用戶名稱空間以及 Docker 1.13 和更高版本中提供的 Docker 密鑰支持隔離，這些 Docker 密鑰可用來存儲和加密密鑰或者訪問令牌（用於身份驗證以及設備之間或上游與雲服務之間的安全通信）。

物聯網設備可能只是偶爾連接，為了節省電力，有些設備設置為按不同的時間表進行休眠。其他物聯網設備可能通過使用 Mesh 網絡拓撲進行連接，在這種拓撲中，在任何給定時間，只有網絡的某些部分是可以訪問的。當網絡連接中斷時，基於推送的更新可能會失敗，而應用不完整或可能損壞的更新造成的危險是設備可能進入不一致的狀態或無法操作。

Docker 為這種更新問題提供了可行的解決方案。對於請求拉取最新版本應用程序鏡像的設備，僅通過空中下載技術向其發送鏡像的差異部分，而不是整個鏡像。完成基於差異的更新速度更快，這將減少設備需要連接的時間，降低故障的可能性，從而減少對低帶寬網絡的壓力。這樣就可以更頻繁地應用更新。

有些物聯網設備允許通過物理按鈕或小觸摸屏進行有限的用戶交互，但是對於許多當代物聯網設備來說，用戶交互的主要手段是使用移動應用程序。人們越來越多地採用 Docker 容器來加速移動應用程序的自動化測試、持續集成和交付，這一點也就不足為奇了。

集成 IoT 設備與雲

將應用程序部署到智能設備和移動設備只是物聯網發展之路上的一部分。在物聯網設備和網關上運行的容器中，可以執行邊緣分析，但是，許多當代物聯網設備還會將傳感器數據和事件發佈到雲服務以供進一步處理。

雲服務經常採用容器。但是，容器是短暫的，因此任何寫入運行的數據庫容器中文件系統的數據都應該被視為臨時的；也就是說，如果關閉容器，數據將會丟失。從某種意義上說，使用容器會迫使您養成開發無狀態雲應用程序的良好習慣。任何要持久化的數據都應該使用數據捲來存儲。數據卷在容器關閉後持續存在，並且可以在多個容器之間共享。

隨著互聯設備數量的增加，物聯網的基於雲的應用程序將需要不斷擴展才能處理所生成的數據量。幸運的是，我們可以利用圍繞 Docker 構建的不斷壯大的編排工具生態系統來開發可伸縮的雲應用程序，這包括 Docker Machine、Swarm 和 Compose 工具。這些工具可以在許多雲平臺上使用，包括 EC2 Container Services、Microsoft Azure 和 IBM Cloud。IBM Cloud 目前還支持運行多組容器實現負載平衡和故障轉移。

結束語

為了滿足物聯網應用程序的預期需求，開發者需要採用相關工具和實踐，使他們能夠快速開發物聯網應用程序和服務，以便在智能設備、移動設備和雲端運行。Docker 容器能夠在廣泛的設備上隨時隨地發送和運行應用程序，運行時開銷最小，支持自動化，並且具有分層文件系統（實現輕量級可移植鏡像和快速鏡像構建），因而是物聯網開發者的絕佳工具。
