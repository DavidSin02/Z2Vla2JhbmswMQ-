在一個小時內構建便攜式數據科學家庭實驗室

2020-11-14
| 科技


在我從事IT行業多年的工作中，數據始終讓我著迷。擁有數據思維的人，遠遠比複製粘貼代碼更加讓人著迷。我們如何處理和存儲演變成我今天關注的數據---即數據查詢的性能以及從一個存儲單元到另一個存儲單元的每個數據處理的性能。在工作和業餘時間，通過在電腦或Linux服務器上創建虛擬機並運行一些測試，我也熟悉了各種DBMS和其他管理數據的程序。
最近，開始學習數據科學，並熟悉與該領域相關的一些技術和工具。為了試驗這些技術和工具，開始創建專用於數據科學的虛擬機。這涉及的程序/工具太多了，突然意識到需要一種方法來快速添加或刪除程序和工具並快速擴展它們，而不是創建大量的Data Science VM。我們開始使用Docker，後來又在Linux Containers上使用，因為我發現Docker無法滿足我的需求。Linux容器（LXC）使您能夠在容器內擁有持久性存儲，並在處理要測試的程序和/或數據庫以及思想時輕鬆控制它。
構建數據科學虛擬機
由於我對RedHat，CentOS和Fedora非常熟悉，因此我使用CentOS 7來構建Data Science VM。儘管KVM或VMWare也是不錯的選擇，但我使用的虛擬化環境是VirtualBox。我創建了一個具有兩個30GB VDI和兩個15GB VDI磁盤且啟用了動態分配的Linux VM。一個30 GB的磁盤用於CentOS 7系統，另一個30 GB的磁盤用於存儲代碼和數據，另外兩個15 GB的磁盤用作ZFS文件系統的磁盤。ZFS文件系統具有出色的快照和可恢復性功能，非常適合LXC。ZFS的另一個不錯的選擇是，如果需要，您可以輕鬆地將更多磁盤添加到ZFS文件系統中。
要在CentOS上安裝ZFS，請執行以下操作：

yum install http://download.zfsonlinux.org/epel/zfs-release.el7_7.noarch.rpm -yyum list zfs*yum install zfs zfs-dkms/sbin/modprobe zfsservice zfs start


安裝後，您應該能夠創建一個使用附加到您的VM的虛擬磁盤的存儲池。要創建ZFS池，您只需執行以下命令：

zpool create conpool /dev/sdc /dev/sdd


/ dev / sdc和/ dev / sdd是15GB磁盤。
要查看新創建的池，請執行以下命令：

zpool statusand zpool status



向池中添加其他磁盤非常簡單。將它們添加到VM之後，一旦它們可用於系統，您只需執行以下命令：

zpool attach conpool /dev/sde /dev/sdf


要安裝LXC，我安裝了快照管理器，並從那裡安裝了最新的（4.xx）LXC / LXD。但是，您也可以從CentOS EPEL存儲庫安裝LXC（版本3.xx）。但是，由於最新版本是4.XX，並且尚未提供給EPEL存儲庫，因此使用快照式軟件包管理器是一個更好的選擇。

yum install snapdsystemctl enable --now snapd.socketln -s /var/lib/snapd/snap /snapservice snapd start...snap search lxdsnap install lxd


確保在使用lxd init命令初始化環境時，對“創建新的ZFS池”說“否”，並使用zpool create命令放置剛創建的池的名稱。

lxd init Do you want to configure a new storage pool (yes/no) [default=yes]?  Name of the new storage pool [default=default]: lxd_zfs Name of the storage backend to use (dir, btrfs, ceph, lvm, zfs) [default=zfs]:  Create a new ZFS pool (yes/no) [default=yes]? n Name of the existing ZFS pool or dataset: your_zpool_name ( conpool in my case )


這將初始化ZFS池上的LXC / LXD容器文件系統，以方便克隆，存儲大小確定以及將容器傳輸到其他主機。
此時，環境已準備好添加一些容器，這些容器將在以後用於安裝應用程序，針對我的應用程序和/或從CentOS主機創建並運行的數據工作流代碼進行配置和測試。
例如，在我的一個實驗中，我想創建一個小型Apache Spark集群，並使用我在Python和Java中創建的Spark應用程序進行比較，從MariaDB Galera集群並行讀取，插入，更新和刪除數據。為此，我為Spark創建了一個CentOS 7容器，為MariaDB數據庫創建了一個Centos 7容器。

lxc launch images:centos/7/amd64 mygal1lxc launch images:centos/7/amd64 sparkm


在每個節點上安裝軟件並進行一些配置後，我只需使用LXC copy命令將容器克隆兩次，每次

lxc copy mygal1 mygal2lxc copy mygal1 mygal3lxc copy sparkm sparkw1lxc copy sparkm sparkw2


最後，我對每個新容器中的應用程序進行了一些小的重新配置，以將它們作為群集進行連接。
之後，我運行了一個三節點的Apache Spark集群以及一個三節點的MariaDB Galera集群，並且可以將我的應用程序提交給集群。所有這些大約需要45分鐘左右的時間。
要列出它們，請執行以下命令：

lxc list



並在ZFS文件系統上列出它們，您可以執行以下命令：

zfs list



在VM中運行Data Science Apps和工作流程


為了構建和運行我的應用程序，我安裝並配置了Anaconda（ www.anaconda.com），它是適用於Python和R的非常好的Data Science軟件包工具包/捆綁包，並且預先裝有Jupyter，JupyterLab和RStudio，它們是必不可少的Data Science。平臺以及其他有用工具。
使用Anaconda，您可以輕鬆地為Data Science應用程序配置和創建並克隆用於不同程序包版本和捆綁包的多個編程環境。我還安裝了Eclipse（ www.eclipse.org）進行Java / Scala編程。
要將Apache Spark與Jupyter和JupyterLab連接，我只需安裝配置好的Apache Toree（ toree.apache.org），這是一個Jupyter內核，允許Jupyter和Spark在客戶端級別進行交互。此時，我的Data Science VM已準備就緒，可以處理一些很棒的Data Science工作負載。而且，我可以使用LXC或ZFS輕鬆，輕鬆地一次又一次地實施我的實驗，從而輕鬆地進行備份和還原命令，從而輕鬆地為不同測試設置相同的起點。
結束語
在本文中，簡短地討論了，並向您展示瞭如何在一個小時內建立一個便攜式數據科學家庭實驗室，希望您喜歡並獲得了一些想法，他們希望如何以最小的努力來實施數據科學實驗。 。
在虛擬機中的ZFS文件系統上運行的帶有Linux容器的Linux是進行數據科學實驗的良好平臺。使用此設置來測試該領域中一些令人興奮且令人印象深刻的應用程序並構建您的Data Science應用程序工作流非常容易，以後您可以輕鬆地將容器轉移到更大的主機上並進行橫向擴展以處理海量數據。
如前所述，Anaconda軟件包已預先安裝了Jupyter，JupyterLab和RStudio，這是創建Python和R數據科學應用程序和工作流程所必需的數據科學工具。為了輕鬆創建並設置流經您選擇的應用程序的數據工作流，無論是Apache Spark，Kafka，MQ，Hadoop，Hive，MySQL，MariaDB還是任何其他關係數據庫或NoSQL數據庫，然後使用LXC容器託管應用程序是易於擴展和管理的不錯選擇。
