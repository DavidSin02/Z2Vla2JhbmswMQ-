前言

python各種基本數據類型就相當於廚房裡的柴米油鹽，只要我們用溜了，米其林大廚也不在話下一大波概念正在接近，不要被嚇怕，我們陽光足、豌豆射手夠猛

基本數據類型

數字(Number)

在python中，數字(number)分為四類:

  1. 整型(integer)，其實就是整數，例如 520
  2. 浮點型(float)，其實就是小數，例如圓周率 3.1415926
  3. 布爾值(boolean)，布爾值就是真假值，它有且只有兩個值 真True 與 假False，您可以把True和False想象成兩個變量，他們的值分別為 1 和 0
  4. 複數(complex)，用的很少，例如 1+2j

例子

a = 520 # 整型b = 3.1415926 # 浮點型c = True # 布爾值d = 1+2j # 複數# 您可以用python內置函數type來查看一個變量指向的類型print(type(a)) # <class 'int'> int是integer的簡寫print(type(b)) # <class 'float'>print(type(c)) # <class 'bool'> bool是boolean的簡寫print(type(d)) # <class 'complex'>

常用數值運算

>>> 1 + 1 # 加法2>>> 3 - 1.5 # 減法1.5>>> 2 * 5 # 乘法10>>> 10 / 3 # 除法, 注意無論是否能整除，得到的都是一個浮點數3.3333333333333335>>> 10 / 52.0>>> 10 // 5 # 兩個斜槓來表示整除，這樣會得到一個整數2>>> 10 % 3 # 百分號表示取餘運算1>>> 2 ** 3 # 兩個星號表示乘方運算，這裡表示2的3次方8

注意，如果您以整數與浮點數作運算，那麼會得到一個浮點數

>>> 1 + 1.02.0

如果您之前學過某種編程語言，您可能會疑惑，布爾值是數字類型？是的，在python3中， True和False都是其保留字，它們值還是1和0，可以與數字相加

不信來試試

>>> 1 + True2>>> True + True # 1 + 1 = 22>>> False == 0 # False等於數值0True>>> type(True) # True是bool類的一個實例<class 'bool'>>>> issubclass(bool, int) # bool類是int類的子類，內置方法issubclass用於查看一個類是否為另一個類的子類True>>> bool.__base__ # bool類是int類的子類，__base__是魔法屬性，其指向父類<class 'int'>>>> isinstance(True, int) # True是int類的實例，內置方法isinstance用於查看一個值是否為某個類的實例True

如果上面的示例不太懂，沒關係，只需要知道True或False就是數字1和0，它們可以與數字進行運算就可以了

字符串(String)

從HelloWorld程序開始，我們就已經接觸過字符串(string)了，現在重新回顧一下，就像偶然在街上遇見一個好久未見的舊朋友，必須要啤酒擼串走一轉加深感情一樣

字符串可以用四種方法來表示:

  1. 用單引號 ' 括住表示
  2. 用雙引號 " 括住表示
  3. 用三個單引號 ''' 括住表示(多行字符串、多行註釋)
  4. 用三個雙引號 """ 括住表示(多行字符串、多行註釋)

例子

# 單個單引號表示a = '你真帥'# 單個雙引號表示b = "你真漂亮"# 三個單引號表示多行字符串c = '''你怎麼這麼好看'''# 三個雙引號表示多行字符串d = """是天生的嗎是的"""

結果

>>> print(a)你真帥>>> print(b)你真漂亮>>> print(c)你怎麼這麼好看>>> print(d)是天生的嗎是的

推薦使用單引號來表示，因為雙引號還得多按一個Shift鍵

字符串換行

您可能注意到了，在上面的例子中，在print出變量c與d時，顯示是兩行，那麼能不能用一個單引號或雙引號來做到換行效果呢

可以的，您只需要在字符串里加上 \n ，它表示一個換行符

>>> a = '你怎麼\n這麼好看' # 在字符串內加上\n表示換行>>> a'你怎麼\n這麼好看'>>> print(a)你怎麼這麼好看>>> b = '''你怎麼... 這麼好看'''>>> b # 下面的結果揭示了多行字符串只不過是當您回車時，解釋器自動幫您添加上\n符號而已'你怎麼\n這麼好看'

字符串基礎操作

您可以通過索引來獲取字符串中的某一個字符，亦可以用切片來獲取字符串的某段子串

索引

在編程的世界中，一切都從0開始計數，比如字符串'abc'，那我們會說第0個字符是a、第1個字符才是b

如果這個讓您困惑，不妨可以舉如下例子，幫助理解

假如您家裡電梯總共有三樓按1樓按鈕，電梯鳥都不鳥您，就不會移動，所以為0按2樓按鈕，它往上移動了1樓，所以為1按3樓按鈕，它往上移動了2樓，所以為2所以這個計數其實是移動的步數

所以如果我們企圖獲取字符串中的某一個字符，指針也要作相應移動，我們使用字符串[移動步數]來獲取某一個字符，比如字符串[0]獲取第一個、字符串[1]獲取第二個

>>> a = 'abc'>>> a[0]'a'>>> a[1]'b'>>> a[2]'c'>>> a[3] # 注意如果您超過它的長度，將會報錯，提示字符串的索引超出了範圍Traceback (most recent call last):  File "<stdin>", line 1, in <module>IndexError: string index out of range>>> length = len(a) # 我們可以用python內置函數len來查看字符串的長度，len是英文length的簡寫>>> print(length) # 字符串a的最後一個字符的索引是2，所以這裡如果我們的索引超過2，就會報錯3

切片

切片用於獲取字符串中的某一部份，就像切三文魚一樣，我們用鋒利的刀切出最好吃的部份; 我們使用字符串[開始步數:結束步數]來獲取字符串中的某段

注意：切片的結果並不會包含結束步數處的元素

>>> a = 'abcd'>>> a[0:1] # 第一個字符，相當於a[0]'a'>>> a[0:2] # 第一個字符至第二個字符'ab'>>> a[0:3] # 第一個字符至第三個字符'abc'

字符串運算

與數字一樣，字符串也有專屬於它的運算 +、*、in

>>> a = '帥'>>> b = '哥'>>> c = 'abcd'>>> a + b　# 字符串加法，表示將左右兩個字符串順序粘連在一起'帥哥'>>> a * 3 # 字符串加法，表示將目標字符串重複Ｎ次'帥帥帥'

可以用in操作符，來查看字符串是否包含某段字符串，返回值是布爾值

>>> 'a' in c # 字符串'a'在字符串'abcd'裡面嗎True>>> 'bc' in c # 字符串'bc'在字符串'abcd'裡面嗎True>>> 'ac' in c # 字符串'ac'在字符串'abcd'裡面嗎False

為什麼最後一個是False！ac明明在'abcd'裡面有啊？

因為必須要連續的，a位置是0，c位置是2，所以是非連續的，這裡要特別注意

列表(List)

在python中，列表(list)是被使用的最頻繁的一種數據類型，所以掌握它，很重要您可以把列表想象成超市裡的儲物櫃，其每一個格子都可以存任意的東西，甚至您可以往格子裡再放一個儲物櫃，無線嵌套，就像盜夢空間一樣列表是可變的，就像格子可以先後放入不同客人的物品一樣我們以中括號 [、] 括住來表示一個儲物櫃，並以逗號 , 分隔其中每一個格子

例子

>>> a = [1,'揹包','雨傘',3.14] # 創建了一個有4個元素的列表，我們在裡面存了數字和字符串>>> print(a) # 打印看看[1, '揹包', '雨傘', 3.14]>>> a[1] # 支持索引，第二格存了什麼'揹包'>>> a[3] # 支持索引，第四格存了什麼3.14>>> a[1:3] # 也支持切片，第二格到第二格存了什麼['揹包', '雨傘']

列表的切片與字符串的切片有一丟丟的區別

>>> a = 'abc'>>> b = ['a','b','c']>>> a[0]'a'>>> a[0:1] # 這裡注意到字符串的切片[0:1]與索引[0]是一樣的效果，無論是索引還是切片得到的都是字符串'a'>>> b[0]'a'>>> b[0:1] # 但是列表的切片得到的永遠是列表，索引就是得到其對應元素的類型['a']

列表基礎操作

>>> a = [1,2,3,4,5]>>> print(a)[1, 2, 3, 4, 5]>>> a[2] = '中間' # 用字符串'中間'來替換列表的第三個元素>>> print(a)[1, 2, '中間', 4, 5]>>> a[0:2] = ['第一','第二'] # 用切片來替換列表中第一、二個元素>>> print(a)['第一', '第二', '中間', 4, 5]>>> a[0:2] = [1,2,3] # 如果切片賦值兩邊的長度不一致，左邊的列表長度將會發生變化>>> print(a) # 現在它的長度變為6[1, 2, 3, '中間', 4, 5]>>> a[0:2] = [] # 切片賦值給空列表，將導致原來的列表長度減2>>> print(a)[3, '中間', 4, 5]

列表也同樣支持+、*、in操作符，但是它的in與字符串的in，有些許不同

>>> print(a)[3, '中間', 4, 5]>>> '中間' in a # 字符串'中間'在列表a裡面嗎True>>> 4 in a # 數字4在列表a裡面嗎True>>> [4, 5] in 列表[4, 5]在列表a裡面嗎False

結果是False，區別就在這裡了

對於字符串來說，in操作符可以檢測單個或多個元素是否在其內，但是列表，只能檢測單個元素是否在列表內

>>> b = [1,2]>>> c = b + a # 列表同樣支持加號+運算符，表示將左右兩邊列表按順序粘連在一起>>> print(c)[1, 2, 3, '中間', 4, 5]>>> b * 2 # 列表用樣支持星號*運算符，表示將目標列表重複N次[1, 2, 1, 2]

嵌套列表

上面說過，列表它裡面可以存任意的類型，甚至列表裡面可以再存個列表，那要怎樣操作列表中的列表呢？

>>> a = [[1,2,3],[4,5,6],[7,8,9]] # 定義一個兩層嵌套列表>>> print(a) # 看看它長什麼樣[[1, 2, 3], [4, 5, 6], [7, 8, 9]]>>> len(a) # 我們可以用python內置函數len，來查看列表的長度3

居然是3，來看看這三個元素都長啥樣

>>> a[0][1, 2, 3]>>> a[1][4, 5, 6]>>> a[2][7, 8, 9]

懂了，嵌套列表不就是一個列表裡的元素也是一個列表嘛

如果我要獲取9這個數字，要怎麼做？

方法一:

>>> b = a[2]>>> print(b[2])9

方法二:

>>> print(a[2][2])9

把上面兩個步驟合併為一個，就不需要再創建一個變量b了

相當於左邊a[2]先結合為一個整體，然後再和[2]結合在一起

元組(Tuple)

元組(tuple)和列表，它倆的關係就像孿生兄弟，長相和操作都很相似，但雖然是孿生兄弟，終歸還是有些不一樣，元組和列表的區別就是元組是不可變的(immutable)如果非要拉元組做個比喻，元組也可以是個儲物櫃，但是隻要您把東西存進去了，儲物櫃也不會把取物票給您，您只能通過透明玻璃看到格子裡面有些啥，但是您永遠不能再替換裡面的東西了(哪有這麼不合理的設計啊！)我們以小括號 (、) 來表示一個元組，並以逗號 , 分隔其中每一個元素

例子

  1. 元組同樣支持列表的+、*和in操作符
  2. 元組同樣支持索引與切片操作
  3. 只是元組是不可變的，列表是可變的
a = (1,2,3)print(a) # (1, 2, 3)print(a*2) # (1, 2, 3, 1, 2, 3)print(a+a+a) # (1, 2, 3, 1, 2, 3, 1, 2, 3)print(2 in a) # Trueprint(4 in a) # Falseprint(a[0]) # 1print(a[0:2]) # (1, 2)

但是如果您重新賦值元組的元素，將會報異常，比如

>>> a[0] = 'do it' # 報錯，提示元組不支持元素賦值Traceback (most recent call last):  File "<stdin>", line 1, in <module>TypeError: 'tuple' object does not support item assignment>>> a[0:2] = (100, 200) # 報錯，提示元組不支持元素賦值Traceback (most recent call last):  File "<stdin>", line 1, in <module>TypeError: 'tuple' object does not support item assignment

元組的兩個天坑

情景1: 某天您定義了一個只有一個元素的元組

>>> a = (1) # 定義一個單元素的元組，貌似沒毛病>>> a[0] # 查看元組第一個元素，講道理結果肯定是1Traceback (most recent call last):  File "<stdin>", line 1, in <module>TypeError: 'int' object is not subscriptable

什麼？ 報錯是咋回事... 為啥提示整型不能進行索引操作？我的a明明是元組啊

>>> print(a) # 先看看變量a裡是啥東西好了1

1是什麼鬼？ 我可愛的小括號呢？

>>> print(type(a)) # 通過內置函數type看看a的類型是啥<class 'int'>

類型居然是整型，怪不得會報錯

原來(1)被解釋器認為是整數1了，傻逼解釋器！！

那要怎樣才能定義一個單元素元組呢？ 只需要在1後面再加一個逗號,

>>> a = (1,) # 在1後面多加個逗號試試>>> print(a) # 好像沒毛病哦(1,)>>> print(type(a)) # 看看類型，果然是tuple元組了!<class 'tuple'>

但是列表卻不會有這種問題

>>> b = [1]>>> print(b)[1]>>> print(type(b))<class 'list'>

難道元組不是python親生的嗎？ 為啥它哥沒有這種問題，它卻有？ 真是個弟弟！

其實學到後面的運算符的優先級時候，就知道為什麼了，小括號也被python解釋器認作最高優先級的運算符

例如我們計算1+1*2的時候，結果是3還是4呢？

從小學我們就知道，先乘除後加減，所以結果毫無疑問會是3

那麼要怎麼先計算1+1，然後再讓它們的結果跟2相乘呢？

數學老師會教您先用小括號()括住1+1就行了，(1+1)*2這樣結果就是4了

>>> 1+1*23>>> (1+1)*24

所以當您寫出(1+1)這樣的表達式的時候，別說python解釋器了，我們人都不知道()到底在表示元組還是運算符，所以單個元素時必須要多加個逗號,來告訴解釋器，這是一個元組

>>> (1+1)2>>> (1+1,)(2,)

情景2: 某天您在定義賦值一個變量時，不小心在結尾多加了一個逗號,

>>> a = 1, # 本意是定義一個整數1，結果不小心在結尾多加了個逗號>>> b = 2>>> a + b # 當您信心滿滿，認為結果肯定是3的時候Traceback (most recent call last):  File "<stdin>", line 1, in <module>TypeError: can only concatenate tuple (not "int") to tuple

報錯了，提示元組只能與元組相加...

為啥又報錯了，傻逼解釋器吧，我這哪來的元組？ 還能愉快的玩耍嗎？

>>> print(a) # 看看a是啥(1,)>>> type(a) # 看了看a的類型，居然是元組！<class 'tuple'>

原來沒有小括號()括住，用逗號,分隔的幾個元素，也會被python視作元組!

>>> a = '原','來','如','此'>>> a('原', '來', '如', '此')>>> print(type(a))<class 'tuple'>

儘管如此，您必須始終堅持以小括號()括住、以逗號,分隔的語法來定義一個元組，因為這樣更加明確

字典(Dictionary)

字典(dictionary)，同樣是python中被使用頻率最高的一種數據類型，說到字典，您可能會聯想到新華字典、英漢字典，沒錯，它們確實很相似字典有兩個重要的概念，分別是鍵(key)與值(value)，一個鍵對應一個值，就像一本書的頁碼對應其中某一頁內容字典可以說是一種映射類型，它用花括號 {} 來表示，裡面每一項鍵值對用 key:value 表示，並以逗號 , 分隔其中每一個鍵值對

例子

新建

# 創建一個字典a# 其中字符串'a'為鍵，對應著值整數1# 字符串'b'為另一個鍵，對應著值整數2a = {'a': 1, 'b': 2}print(a) # {'a': 1, 'b': 2}# 或者可以這樣達到同樣的目的b = {} # 創建一個空字典b['a'] = 1 # 將字符串'a'作為字典b新的鍵，對應著值整數1b['b'] = 2 # 將字符串'b'作為字典b新的鍵，對應著值整數2print(b) # {'a': 1, 'b': 2}

刪除(delete)

a = {'a': 1, 'b': 2}# 您可以用del(delete的簡寫)標識符來刪除字典的一個鍵del a['a'] # 刪除鍵'a'print(a) # {'b': 2}

修改

a = {'a': 1, 'b': 2}a['a'] = 'hello' # 將鍵'a'指向了新的值字符串'hello'print(a) # {'a': 'hello', 'b': 2}

查看

a = {'a': 1, 'b': 2}print(a['a']) # 查看鍵'a'對應的值，為1print(a['b']) # 查看鍵'b'對應的值，為2# 您可以通過.號來調用字典的方法(函數)，如果您暫時不理解，先記住# 查看字典a的所有鍵，可以調用字典的方法keysprint(a.keys()) # dict_keys(['a', 'b'])# 查看字典a的所有值，可以調用字典的方法valuesprint(a.values()) # dict_values([1, 2])# 查看字典a的所有鍵值對，可以調用字典的方法itemsprint(a.items()) # dict_items([('a', 1), ('b', 2)])

不單單可以將字符串作為字典的鍵，實際上您可以將所有不可變的類型作為字典的鍵

# 在這個例子中，我們先以字符串作為鍵、再以數字作為鍵，最後以元組作為鍵# 在本文後面的篇章，您會知道字符串、數字和元組都是不可變的數據類型a = {'1': 'a', 2: 'b', (3,): 'c'}print(a) # {'1': 'a', 2: 'b', (3,): 'c'}print(a['1']) # 'a'print(a[2]) # 'b'print(a[(3,)] # 'c'

字典的鍵是唯一(同樣的鍵只能出現一次)的，值可以重複

# 檢驗鍵的唯一性，我們先後定義了鍵'a'，並把它們對應了不同的值a = {'a':1, 'a':2}print(a) # {'a': 2}# 可以看到字典只保留了最後一個鍵值對'a':2# 就像書一樣，它不可能出現兩個頁碼相同的書頁# 檢驗值的可重複b = {'a':1, 'b':1}print(b) # {'a':1, 'b':1}# 沒問題，在字典中，值是可以相同的# 就像書的不同目錄可以指向同一頁書一樣

字典不支持+、*號操作符，但支持in，不同的是in是用來檢測某個值是否存在於字典的鍵中，而不是值

a = {'a':1, 'a':2}# 字符串'a'在字典a的鍵中嗎'a' in a # True# 字符串'c'在字典a的鍵中嗎'c' in a # False

仰望未來，聽說在python3.9版本中，字典的合併支持以豎線|操作符來實現

或利用|=進行原地更新，效果與字典的update方法有點類似

# | 操作符會構建一個新的字典a = {'a':1}b = {'b':2}c = a | bprint(c) # {'a': 1, 'b': 2}# |= 操作符會原地更新至aa|=bprint(a) # {'a': 1, 'b': 2}

如果在python3.9以前的版本，比如python3.8.3，一般這樣來達到同樣目的

# 以下效果相當於py39中的|a = {}b = {'a':1}c = {'b':2}a.update(b)a.update(c)print(a) # {'a': 1, 'b': 2}# 以下效果相當於py39中的|=b.update(c)print(b) # {'a': 1, 'b': 2}

如果您看不懂上述py39的新特性，無需理會，因為暫時並不重要，我們專注於py3.8.3就可以了

集合(Set)

python的集合(set)與數學概念上的集合同樣具有三個特性，確定性、互異性、無序性在python中，集合一般用於刪除重複元素或進行成員關係測試集合的表示，與字典有點相似，都是用花括號 {} 括住，然後逗號 , 分隔其中每一個成員

集合的創建與特性

a = {1,2,3} # 創建一個長度為3的集合print(a) # {1,2,3}# 集合的確定性: 一個元素要麼屬於一個集合，要麼不屬於，只有這兩種情況# 利用in操作符來查看一個元素是否屬於該集合print(1 in a) # Trueprint(4 in a) # False# 集合的互異性: 一個集合內的元素必定不會重複b = {1,1,1,2,2,3,3} # 創建時包含重複元素# 可以看到多餘的1、2、3已被去除print(b) # {1, 2, 3}# 集合的無序性: 元素之間是無序的c = {3, 1, 2}# 順序已發生變化print(c) # {1, 2, 3}# 看看集合a與集合c是否相等，==是相等比較符，它用於比較左右兩邊是否相等，返回True或Falseprint(a == c) # True

有一種集合叫空集，您不能以{}來定義空集，因為{}會被python解釋器認為是空字典，只能用內置函數set來定義空集

a = set() # 創建一個空集合print(a) # 輸出: set()，以表示其為一個空集# 可以把字符串、列表、元組等可迭代類型傳遞給內置函數set，來創建一個集合b = [1,2,3]c = set(a)print(c) # {1, 2, 3}d = set('hello,world')print(d) # {'w', 'e', 'o', ',', 'h', 'r', 'l', 'd'}

集合的基本操作

增加元素

a = {1, 2}# 利用集合的add方法，增加一個元素a.add(3)print(a) # {1, 2, 3}# 利用集合的update方法，增加多個元素# update方法接受一個可迭代類型a.update([4,5])print(a) # {1, 2, 3, 4, 5}

刪除元素

a = {1, 2, 3}# 利用集合remove方法，刪除值等於所給值的元素a.remove(3)print(a) # {1, 2}# 利用集合的pop方法，隨機刪除其中一個元素a.pop()print(a) # {2}# 利用集合的clear方法，來讓其變為空集a.clear()print(a) # set()

集合的基本運算

集合不支持+與*號操作符，但是它有它自己獨有的運算

a = {1,2,3}b = {3,4,5}# 交集: 你有我也有# 求交集用&符號print(a & b) # {3}# 並集: 我有的加上你有的# 求並集用|符號print(a | b) # {1, 2, 3, 4, 5}# 差集: 我有你沒有# 求差集用-減號print(a - b) # {1, 2}# 異或集：去掉大家都有的，然後再加起來# 求異或用^符號print(a ^ b) # {1, 2, 4, 5}# 子集: 我有的，你全都有，那我就是你的子集，你就是我的超集# 判斷一個集合是否為另一個集合的子集，用大於等於>=或小於等於<=符號print({1,2} <= {1,2,3}) # Trueprint({1,2,3} >= {1,2,3}) # True# 真子集: 我有的，你全都有，而且還有我沒有的，那我就是你的真子集，你就是我的真超集# 判斷一個集合是否為另一個集合的真子集，用大於>或小於<符號print({1,2,3} < {1,2,3}) # Falseprint({1,2} < {1,2,3}) # True

可變與不可變類型

在以上6種基本數據類型，如果以是否可變來分類，可以分為兩類

可變類型(mutable)

它元素的值是可以改變的，也叫做引用類型

  • 列表(list)
  • 字典(dictionary)
  • 集合(set)

不可變類型(immutable)

元素是不可改變的，要改變只能重新賦值來覆蓋，也叫做值類型

  • 數字(number)
  • 字符串(string)
  • 元組(tuple)

我們可以用python內置函數id來獲取目標對象存放在內存中的地址

# 可變a = [1, 2]b = {'a':1}c = {1, 2}# 不可變d = 3.14e = 'hello'f = (1, 2)# 利用id來獲取目標在內存中的地址# 注意不同的運行環境，會有不同的結果，但是效果一樣print(id(a)) # 140042188515456# 改變a內元素的值a[0] = 100print(a) # [100, 2]# 再次查看其內存地址，如果沒有變化，證明還是同一個數據，但是它的內容卻變化了，所以說是可變類型print(id(a)) # 140042188515456# 看看字符串的idprint(id(e)) # 140042188516848# 嘗試改變字符串內的某個字符# >>> e[0]='f'# Traceback (most recent call last):#   File "<stdin>", line 1, in <module># TypeError: 'str' object does not support item assignment# 會報錯，提示字符串不能被改變# 重新賦值變量e另外一個字符串e = 'i see'# 看看id，已經發生變化，所以不再是同一個數據了，所以叫不可變類型print(id(e)) # 140042188516272

再論變量

如果您之前學過別的編程語言，比如C或C++，您會發現，在python中，不需要事先聲明變量，只需要在使用前給變量賦值即可，所以我們把python稱作弱類型語言，而把C或C++等稱作強類型語言變量並不等於以上各種基本數據類型，如果您把一個數據類型賦值給一個變量，變量所做的工作只是幫您找到那個類型而已，如果把數據類型比作一棟建築，那麼變量只是指向它的路標用一個等號=來給變量賦值，它的左邊是變量的命名，右邊是變量指向的類型，要注意兩個等號==是比較運算符

可以用多個變量指向同一個數據

# 變量a指向字符串a = '我是一個字符串'# 也令變量b指向同一個字符串b = a# 現在有兩個變量指向同一個字符串了# 相等比較運算符，它的作用是比較變量指向的值是否相等print(a == b) # True

與==不同，它是比較值是否相等，如果要查看是否同一個數據，可以用is操作符

事實上，is操作符是比較兩者內置函數id的結果是否相等

a = 'abc'b = 'abc'c = a# 值都相等print(a == b) # Trueprint(a == c) # True# 但a與b並不是同一個數據print(a is b) # False# a與c是同一個數據print(a is c) # True# 又比如d = [1,2]e = [1,2]f = d# 變量d與f是指向同一個數據，且它們指向的是引用類型(可變類型)# 如果改變dd[0] = 100print(d) # [100, 2]# f會發生變化，因為它們指向的是同一個數據print(f) # [100, 2]# 改變ee[0] = 200print(e) # [200, 2]# d還是原樣print(d) # [100, 2]# 因為一開始賦值時，d與e的值雖然相等，但它們並不是同一個數據print(d is e) # Falseprint(id(d) == id(e)) # False

如果把一個數據類型比作一棟建築，變量比作找到它的方式，假如現在有兩個變量指向那棟建築，其中一個變量是高德地圖，另外一個變量是祖傳指南針，通過它們倆，我們都能找到那棟建築

如果出了啥意外，兩個變量都找不到那棟建築了，會怎樣？

我們一開始只有兩種方法找到那棟建築，通過高德地圖或祖傳指南針，現在它們都失靈了，那我們人類也就再也找不到那棟建築了，久而久之，那棟建築就會因為風化、年久失修而被摧毀，就沒了...沒了...

我們可以通過del來刪除一個變量

# 1是那棟建築a = 1b = a# 現在有兩種方式找到它# 通過aprint(a) # 1# 或是通過bprint(b) # 1# 把變量a給弄沒del a# print(a) 將會報錯，提示沒有該變量# 現在還能通過b來找到那棟建築1print(b) # 1# 把變量b也給弄沒del b# 好了，再也找不到建築1了# 它遲早會被風化毀掉!!

多個變量賦值

您可以通過兩種方式給多個變量賦值

# 可以同時為多個變量賦給同一值a = b = c = 1# print函數用逗號分隔多個，可以同時打印輸出多個值print(a, b, c) # 1 1 1print(a is b) # Trueprint(a is c) # Trueprint(b is c) # True# 可以同時為多個變量賦給不同值d, e, f = 1, 2, 3print(d, e, f) # 1 2 3

變量的命名法

變量的命名法也可以叫做變量的命名規範，它與變量的命名規則不一樣，規則就像法律，你必須遵守，但規範就像道德，只是建議最好這樣做

經典命名法一覽

# 駝峰命名法：首字母小寫，其餘首字母大寫handsomeMan = 'you'# 帕斯卡命名法：首字母皆為大寫HandsomeMan = 'you'# 匈牙利命名法# 匈牙利命名法是早期的規範，由微軟的一個匈牙利人發明的，是 IDE 還十分智障的年代的產物。那個年代，當代碼量很多的時候，想要確定一個變量的類# 型是很麻煩的，不像現在 IDE 都會給提示，所以才產生了這樣一個命名規範，估計現在已經沒啥人用了吧……一個十分系統卻又瑣碎的命名規範。# 該命名規範，要求前綴字母用變量類型的縮寫，其餘部分用變量的英文或英文的縮寫，單詞第一個字母大寫。iMyAge = 18 # "i": intfMyHeight = 180  # "f": float# 蛇型命名法：以下劃線_連接單詞handsome_man = 'you'

python推薦蛇型命名法，因為python的中文名叫蟒蛇，它的圖標就是一條蛇
