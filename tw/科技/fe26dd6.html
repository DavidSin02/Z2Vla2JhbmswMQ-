Elasticsearch Mapping 的字段類型 簡介

2020-10-29
| 科技

使用elasticsearch 就像使用數據庫一樣，在使用之前，總歸是要聲明 index 的每個type具體的字段。

一般我們不建議動態修改字段，會在線上帶來各種各樣的問題，而是建議在開發前就規劃好字段，選擇合適的字段儘可能提高 索引和檢索 效率。



以下是基於 elasticsearch 6.3.2 版本，官方支持的字段類型：


核心類型 （core datatypes):

string （字符串）

  • 分為 keyword （不可分詞） 和 text （ 可分詞）

Numeric datatypes （數字類型）

  • long, integer, short, byte, double, float, half_float, scaled_float

Date datatype （日期類型）

  • date

Boolean datatype （布爾類型）

  • boolean

Binary datatype （二進制類型）

  • binary

Range datatypes （範圍類型）

  • integer_range, float_range, long_range, double_range, date_range



Complex Datatypes （複雜類型）

  • Object datatype ( object 用來表示 單個JSON對象)
  • Nested datatype ( nested 用來表示 JSON 數組)

Geo datatypes （地理信息類型）

Geo-point datatype （geo_point 用來表示 經緯度）

Geo-Shape datatype （geo_shape 用來表示複雜的如多邊形）


Specialised datatypes （專用類型）

IP datatype

  • ip 表示ipv4 或者 ipv6 地址

Completion datatype

  • completion 提供自動補全建議

Token count datatype

  • token_count 計算字符串中的token個數

mapper-murmur3

  • murmur3 計算一個hash值

Percolator type

  • 從 query-dsl 中接收查詢

join datatype

  • 表示父子關係

Arrays

數組類型：不需要單獨定義


Multi-fields

一般是類似這個需求時使用： 一個 字符串字段，既想用來分詞（text），又想當作 keyword 進行聚合
