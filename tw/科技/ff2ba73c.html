java併發編程-學習方法、進程和線程的區別

2020-11-14
| 科技

java併發編程是通向高級編程的橋樑，如你想學習Tomcat的內部原理、spring cloud微服務、鎖問題等都需要先了解併發的。所以把Java併發打好基礎，是為了後面更加高級課程的學習。

如何學習java併發編程？

我學習java併發編程是反覆迭代的過程。大概的說下我怎麼做的，首先oracle官網併發的基本概念、例題做一遍。然後進行think in java 併發章節->代碼模擬，自己寫一遍 。再每隔3個月重複學一遍think in java 併發章節和示例代碼。通過幾次的迭代和工作的積累後，可以看看《併發編程》、《effective java》這兩本書的相關章節。併發編程博大精深，涉及操作系統、編譯原理、數據結構，想要學好很難，各位一起努力啊。

什麼是併發？

計算機用戶希望可以在一個時間點做更多的事情。其實操作系統層面已經對併發做了支持，比如同一個時間可以聽音樂、下載文件、玩遊戲。我們現在知道CPU有一個時間切片，在同一時間只從內存同步一條指令到CPU運行，只是由於CPU有多核而且執行速度特別特別快，我們肉眼無法感知，所以你會感覺CPU是同一時間執行所有的任務。由於我們的應用程序不能隨意的調用操作系統的併發類庫，所以oracle Java 平臺提供了低級別和高級別的併發支持。

進程和線程

在併發編程中，有兩個基本的概念我們先要先了解，就是進程和線程。我們在Java中，併發程序使用的是線程（Thread）。當然，進程也非常的重要。計算機系統由很多個進程和線程組成。即使是單核處理器，也是如此。在單核處理器中，其根據操作系統的時間切片，在進程和進程間共享單個核的處理時間。

進程

進程有自己的執行環境。實際上就是每個進程佔用著它自己的內存空間。我們將application或你自己寫的程序叫做進程，進程間需要交流，通過操作系統提供的inter Process communication（IPC）資源進行寫作，列入管道（pipe）或者socket。IPC不僅可以在同一個系統中進行交流，還可以再不同的系統中。

線程

線程是低級別的進程。創建線程需要的資源相對於進程系統開銷要小的多。進程中含有多個線程，最少要有一個線程，如果沒有JVM會退出，程序會終止。比如看下面程序終止的例子，你沒看懂也沒事，後面會再說。

public class ThreadDaemon implements Runnable {@Overridepublic void run() {while(true){try {System.out.println("i am alive !!!");Thread.sleep(5000);} catch (InterruptedException e) {e.printStackTrace();}}}public static void main(String[] args) {Thread thread = new Thread(new ThreadDaemon());thread.setDaemon(true); //，在啟動start()方法前，告訴程序都是守護線程，jvm 退出thread.start();}}

輸出結果：

輸出結果會發現，子線程設置為守護進程，然後主線程(main)線程執行完了，這個時候子線程沒有等5秒，執行完打印語句（I am alive !!!）程序就退出了。

最後一點，我們要清楚，因為一個進程會有多個線程，所以多個線程會共享進程的資源。線程之間是隔離的，獨立的，所以線程間的交互是個問題，我們後面將會學習volatile關鍵字，synchronized關鍵字，這個就是強制線程間同步。
