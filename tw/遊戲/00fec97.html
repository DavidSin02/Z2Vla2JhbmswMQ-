遺傳算法綜合指南（以及如何Python編碼）

2020-10-29
| 遊戲

在19世紀中期，查爾斯達爾文提出了進化論以及它如何在使生物體通過自然選擇適應環境方面發揮關鍵作用 - 適者生存的過程在一個特定的種群中，適者生存的時間足夠長，可以將他們的特徵和特徵傳遞給後代以確保他們的生存。

目前發生了什麼？

目前，與90年代末和2000年代初基於規則的機器相比，機器學習(ML)開創了一個智能機器能夠做出更好決策的新時代。

利用我們現在擁有的大量計算能力，機器學習（ML）算法，特別是深度神經網絡，利用我們的大量數據庫（包括結構化和非結構化），以高精度提供洞察力，線索，預測等等。

Inception V4：最先進的計算機視覺模型，廣泛用於分類和遷移學習

最先進的ML模型現在可以通過幾小時/幾天的訓練對數據進行分類，歸類和生成數據。深度神經網絡現在已經擴展到多個領域，現在能夠處理從圖像到音頻的不同數據格式，許多這樣的網絡在所述區域中具有超過人類水平的能力。

強化學習

最近，像OpenAI和DeepMind這樣的研究機構一直在涉及一個名為強化學習的機器學習領域。這是一個系統，代理通過犯錯誤和為各自的狀態收集相應的獎勵（正面或負面），在與環境互動的同時學習和改進。

遺傳算法在哪裡適用？

遺傳算法構成了智能代理生態系統的一小部分，其中應用半蠻力方法來創建能夠“生存”（保持在最頂層）的“適合”代理。為什麼我說半蠻力？這是因為代理的參數是隨機生成的，因此沒有用於代理的預定義測試值集。

遺傳算法(GA)工作於基本的進化原則，因為它是自然選擇和各種自然發生的子過程的元啟發。這涉及到整合進化的四個基本步驟:

  • 適應度
  • 選擇
  • 交叉
  • 突變

當在一組代理上執行時，這四個過程一起產生用於正在執行的任務的最適合的代理。

從這一點開始，當我提到“倖存”這個詞或該詞的任何變體時，我的意思是說，代理人仍然是能夠進入下一代的可行性最少的代理商的一部分。

模型適應度

這是指代理在完成手頭任務時的強度。它量化了代理的能力，並且這增加了它與群體中的另一個代理交叉的可能性，從而可能創建具有兩個父模型特徵的更強的後代模型。

對於不同的任務，適應度函數稍作修改。然而，從本質上講，它的主要功能是在群體中發揮區分作用，將較強的學習者與較弱的學習者分開。

模型選擇

這個過程是不言自明的，因為最少的幾個模型被允許留在群體中，而剩下的較弱的模型被丟棄，因為它們沒有用處。

這讓我們在群體中留下了空位，為剩下的母代理人提供了新的更強的後代的空間。

模型交叉

這個過程使兩個強壯的父母能夠創造出具有父母雙方特徵增加生存機會的後代。這是一個兩全其美的場景。

在交叉期間，其中一個代理的所有屬性被分解或分解為更基本的單元，其中一半與另一個代理的交換。從生物學角度來說，來自一個親本的一部分基因與來自另一個基因的另一部分基因相連，以產生具有兩組基因片段的後代，從而提高其在適應性方面的存活機會。

可視化交叉過程的圖表，其中部分DNA（二進制數字的組合）被分開並與另一個親本的DNA的剩餘部分交換以產生具有兩個不同DNA片段的後代。

模型突變

與進化過程類似，突變在將一些隨機性和隨機性引入群體中起著關鍵作用。通過這樣做，具有這種突變的更好的存活（更高的健康評分）的代理能夠將這種存活突變特性傳遞給後代以確保它們的存活。

與Fitness函數類似，突變函數也需要事先確定。該突變的範圍可以是在基因序列（DNA）中的一個特定位置或多個位置發生的點突變。

無論哪種方式，突變是進化過程中不可或缺的組成部分，因為我們的模型在沒有它的情況下無法存活很長時間。

停止條件

當代理實現手頭的目標或目標時會發生什麼？為了迎合這種情況，使用了停止條件。此函數檢查是否有任何一個代理已超過某個閾值（超參數）或一般與完成任務有關的標準。

在大多數情況下，它是代理需要交叉以完成活動或任務的單個最小閾值。在其他情況下，可能存在一系列標準或一組閾值，模型需要跨越每一代被稱為適者群體/群體中最適合的群體之一。

遺傳算法循環從評估當前一代中所有代理的適合度得分開始，選擇所有模型的最高百分比，隨機地交叉模型並最終突變它們以向群體中引入隨機性。循環從此時開始重置，如果滿足停止標準則停止。

總之，這4個過程可以生產出能夠在環境中生存的新的，更強大，更可行的後代。

通過應用程序學習

現在我們已經完成了遺傳算法的複習，編碼項目是應用所學知識的最佳方式，並加強您對GA所用概念的理解。

在研究GA時，我遇到了許多有趣的項目，從著名的旅行商問題到揹包問題。我甚至發現自己發現了能夠為事件創建最佳時間表的時間表構建代理。

我創建了一個網絡優化算法，該算法採用現有的神經網絡架構超參數，並在測試集上以更高的精度增強它以更好地執行。

項目介紹

主要目標是採用現有的神經網絡架構，並在一定數量的代中隨機發展它以改進它。在這裡，我使用“進化”這個詞，因為在現實中，遺傳算法基本上執行完全隨機的動作並祈禱它改進了某些東西。我已經將Keras機器學習庫用於該項目。我已經將準確度閾值（停止標準）設置為99.5％，並且從60K訓練和測試集MNIST中獲得了1000個實例（如此可預測）。

現在，從更大的角度來看，我們有一個網絡代理，它提供了一堆隨機超參數。在Keras中有一個內置的函數model.predict()，它計算每個網絡代理在測試集上的平均適應度得分，並將該得分分配為二級代理的準確性。

隨著時間的推移，具有更好的超參數的代理會出現在頂部，並被識別為具有最佳模型超參數集的代理。

需要導入的Python庫及基本設置

import numpy as np

import random

from keras.models import Sequential

from keras.layers import Dense, Activation

from keras.datasets import mnist

from keras.utils import to_categorical

(x_train, y_train), (x_test, y_test) = mnist.load_data()

x_train = x_train.reshape(60000, 784).astype('float32')[:1000] / 255

x_test = x_test.reshape(10000, 784).astype('float32')[:1000] / 255

y_train = to_categorical(y_train, 10)[:1000]

y_test = to_categorical(y_test, 10)[:1000]

classes = 10

batch_size = 64

population = 20

generations = 100

threshold = 0.995

def serve_model(epochs, units1, act1, units2, act2, classes, act3, loss, opt, xtrain, ytrain, summary=False):

model = Sequential()

model.add(Dense(units1, input_shape=[784,]))

model.add(Activation(act1))

model.add(Dense(units2))

model.add(Activation(act2))

model.add(Dense(classes))

model.add(Activation(act3))

model.compile(loss=loss, optimizer=opt, metrics=['acc'])

if summary:

model.summary()

model.fit(xtrain, ytrain, batch_size=batch_size, epochs=epochs, verbose=0)

return model

def init_networks(population):

return [Network() for _ in range(population)]

涉及的函數和對象

首先，我們從Network代理開始。它的主要功能是隨機生成一組超參數，稍後將其插入到模型設計中並在MNIST的子集上進行測試。在這裡，我選擇隨機化的超參數是非常基本的而不是太複雜（例如學習率），但請記住它可以完成！Python代碼如下：

class Network():

def __init__(self):

self._epochs = np.random.randint(1, 15)

self._units1 = np.random.randint(1, 500)

self._units2 = np.random.randint(1, 500)

self._act1 = random.choice(['sigmoid', 'relu', 'softmax', 'tanh', 'elu', 'selu', 'linear'])

self._act2 = random.choice(['sigmoid', 'relu', 'softmax', 'tanh', 'elu', 'selu', 'linear'])

self._act3 = random.choice(['sigmoid', 'relu', 'softmax', 'tanh', 'elu', 'selu', 'linear'])

self._loss = random.choice([

'categorical_crossentropy',

'binary_crossentropy',

'mean_squared_error',

'mean_absolute_error',

'sparse_categorical_crossentropy'

])

self._opt = random.choice(['sgd', 'rmsprop', 'adagrad', 'adadelta', 'adam', 'adamax', 'nadam'])

self._accuracy = 0

def init_hyperparams(self):

hyperparams = {

'epochs': self._epochs,

'units1': self._units1,

'act1': self._act1,

'units2': self._units2,

'act2': self._act2,

'act3': self._act3,

'loss': self._loss,

'optimizer': self._opt

}

return hyperparams

轉到主事件循環，我們有必要的函數，如上面討論的fitness()、selection()、crossover()、mutate()。fitness()使用隨機超參數計算模型的準確性。由於維度問題或其他錯誤，一些模型體系結構無法工作並破壞代碼，這就是為什麼我們將培訓和評估功能放在try-except塊中。selection()函數非常不言自明，也很容易理解——前20%的模型可以轉移到下一代模型。Python實現如下：

def fitness(networks):

for network in networks:

hyperparams = network.init_hyperparams()

epochs = hyperparams['epochs']

units1 = hyperparams['units1']

act1 = hyperparams['act1']

units2 = hyperparams['units2']

act2 = hyperparams['act2']

act3 = hyperparams['act3']

loss = hyperparams['loss']

opt = hyperparams['optimizer']

try:

model = serve_model(epochs, units1, act1, units2, act2, classes, act3, loss, opt, x_train, y_train)

accuracy = model.evaluate(x_test, y_test, verbose=0)[1]

network._accuracy = accuracy

print ('Accuracy: {}'.format(network._accuracy))

except:

network._accuracy = 0

print ('Failed build. Moving to next model.')

return networks

def selection(networks):

networks = sorted(networks, key=lambda network: network._accuracy, reverse=True)

networks = networks[:int(0.2 * len(networks))]

return networks

需要注意的重要函數是crossover()和mutation()功能。後代代理的質量取決於這些函數的質量，即它創造了幾乎與父母完全不同的後代，併為基因庫和群體增加了隨機性和新穎性。

對於該crossover()函數，父母的超參數在創建的兩個子代理之間分開。一部分隱藏單元被分配給子代理。Python代碼如下：

def crossover(networks):

offspring = []

for _ in range(int((population - len(networks)) / 2)):

parent1 = random.choice(networks)

parent2 = random.choice(networks)

child1 = Network()

child2 = Network()

# Crossing over parent hyper-params

child1._epochs = int(parent1._epochs/4) + int(parent2._epochs/2)

child2._epochs = int(parent1._epochs/2) + int(parent2._epochs/4)

child1._units1 = int(parent1._units1/4) + int(parent2._units1/2)

child2._units1 = int(parent1._units1/2) + int(parent2._units1/4)

child1._units2 = int(parent1._units2/4) + int(parent2._units2/2)

child2._units2 = int(parent1._units2/2) + int(parent2._units2/4)

child1._act1 = parent2._act2

child2._act1 = parent1._act2

child1._act2 = parent2._act1

child2._act2 = parent1._act1

child1._act3 = parent2._act2

child2._act3 = parent1._act2

offspring.append(child1)

offspring.append(child2)

networks.extend(offspring)

return networks

對於Mutation函數，我將原始隱藏單元與0到100之間的隨機整數相加（正如我所說，我想不出有趣的函數）。發生突變的機率應小於0.1。這將隨機性控制在很小但有一定程度。Python實現如下：

def mutate(networks):

for network in networks:

if np.random.uniform(0, 1) <= 0.1:

network._epochs += np.random.randint(0,100)

network._units1 += np.random.randint(0,100)

network._units2 += np.random.randint(0,100)

return networks

在下面，該main()函數將代理生成到環境中。每一代，上述功能都是一個接一個地運行。最後，超過99.5%準確率的模型被認為是最好的

def main():

networks = init_networks(population)

for gen in range(generations):

print ('Generation {}'.format(gen+1))

networks = fitness(networks)

networks = selection(networks)

networks = crossover(networks)

networks = mutate(networks)

for network in networks:

if network._accuracy > threshold:

print ('Threshold met')

print (network.init_hyperparams())

print ('Best accuracy: {}'.format(network._accuracy))

exit(0)

if __name__ == '__main__':

main()
