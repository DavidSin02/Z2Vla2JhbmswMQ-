阿里開源——用於前端和nodejs的輕量級任務管理和構建工具Dawn

2020-10-29
| 遊戲

介紹

Dawn 取「黎明、破曉」之意，原為「阿里雲·業務運營團隊」內部的前端構建和工程化工具，現已完全開源。它通過 pipeline 和 middleware 將開發過程抽象為相對固定的階段和有限的操作，簡化並統一了開發人員的日常構建與開發相關的工作。




Github和文檔

https://github.com/alibaba/dawn

https://alibaba.github.io/dawn/docs/

有哪些特點？

  • 採用中間件技術，封裝常用功能，易於擴展，方便重用
  • 支持 pipeline 讓多個 task 協同完成構建任務
  • 簡單、一致的命令行接口，易於開發人員使用
  • 支持基於「中心服務」管理中間件和工程模板
  • 支持搭建私有中心服務，並統一下發構建規則，易於團隊統一管理

    • 安裝和更新

    依賴的環境、軟件及其版本：

    • Node.js v7.6.0 及以上版本
    • Mac/Linux (Windows 基本支持未經嚴格測試)

    安裝或更新 Dawn:

    $ [sudo] npm install dawn -g

    國內可以使用 cnpm 加速安裝

    初始化工程

    $ dn init [template] [options]

    示例：

    $ dn init ? Found 4 templates (Use arrow keys)❯ 1. front      : Blank front end project template  2. node       : Blank node project template  3. middleware : Dawn middleware project template  4. react      : Based on react-scripts, like create-react-app

    選擇一個工程類型，回車即可按嚮導初始化一個工程，還可以通過 -t 或 --template 直接按指定的模板名稱，直接初始化工程。

    比如，通過名為 front 的模板，初始化一個工程

    $ dn init -t front

    啟動開發服務

    $ dn dev

    如果是一個「前端」工程通常會啟動構建進程並監聽文件的變化，通常，還會啟動一個 Web Server，並自動打開瀏覽器。

    執行檢查和測試

    $ dn test

    在執行 test 的時候會先進行「語法檢查」（通過 eslint），然後執行「單元測試」和「E2E 測試」。

    構建工程

    $ dn build

    執行構建任務，不同的工程類型的構建過程和結果可能不同，取決於初始化工程時使用的工程模板。 完成後，會在當前項目的根目錄產生 build 目錄，這是構建結果，當然，也可以指定為其它目錄名稱。

    發佈工程

    $ dn publish

    可以通過 dn publish 命令發佈代碼和構建結果，不同的工程模板決定了最終發佈位置，是否支持 publish 命令取決於選擇的「工程模板」。

    執行自定義任務

    init/dev/build/test/publish 這幾個命令可以直接作為「子命令」寫在 dn 後邊，對於其它名稱的 pipeline 需要使用 run 命令

    編輯 .dawn/pipe.yml 或 .dawn.yml

    demo:  - name: shell    script:      - echo demo

    可以通過如下方法執行 demo

    dn run demo
