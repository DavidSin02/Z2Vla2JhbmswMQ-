web滲透測試——數據庫攻擊技巧整理

2020-10-29
| 遊戲

數據庫是一個網站最重要的組成部分，它存儲著整個站點許多重要的信息，包括：用戶登錄信息、配置信息、用戶存儲的信息等，攻擊者從開始的信息收集到發現利用漏洞再到最後提高權限，最終目的就是獲取用戶的敏感數據，如果可以直接攻擊網站數據庫，並獲得裡邊的重要數據，無疑是一種快捷的攻擊方式，下面我就介紹一些比較常見的數據庫的攻擊技巧。

MySQL數據庫

MySQL 數據庫是目前世界上使用最為廣泛的數據庫之一，很多著名公司和站點都使用 MySQL 作為其數據庫支撐，目前很多架構都以 MySQL 作為數據庫管理系統。

MySQL

  • 信息收集：默認端口3306，MySQL數據庫常與PHP和JSP語言組合，如果一個站點採用PHP或JSP語言進行開發，那麼這個站點大概率會使用MySQL數據庫。
  • 攻擊方法：爆破，弱口令身份認證漏洞：CVE-2012-2122，拒絕服務攻擊，Phpmyadmin萬能密碼繞過，用戶名：‘localhost’@’@” 密碼任意提權。

MSSQL數據庫

MSSQL數據庫是微軟開發的一款數據庫系統，也只在windows系統中運行，因此 Windows系統+IIS服務+.net編程語言+MS Sql Server數據庫就是其常用的組合，MSSQL數據庫的這個特性也限制了它的推廣，尤其是在windows以外的系統上。

MSSQL

  • 信息收集：默認端口1433（Server 數據庫服務）、1434（Monitor 數據庫監控）。
  • 攻擊方法：爆破，弱口令/使用系統用戶注入。

Oracle數據庫

Oracle數據庫是甲骨文公司的一款關係數據庫管理系統。它是在數據庫領域一直處於領先地位的產品。可以說Oracle數據庫系統是目前世界上流行的關係數據庫管理系統，系統可移植性好、使用方便、功能強，適用於各類大、中、小、微機環境。它是一種高效率、可靠性好的、適應高吞吐量的數據庫方案。

Oracle

  • 信息收集：默認端口1521（數據庫端口）、1158（Oracle EMCTL端口）、8080（Oracle XDB數據庫）、210（Oracle XDB FTP服務），常見的組合方式：Linux系統+Tomcat服務+JSP編程語言+Oracle數據庫。
  • 攻擊方法：爆破，弱口令注入攻擊。

PostgreSQL數據庫

PostgreSQL是一種特性非常齊全的自由軟件的對象–關係型數據庫管理系統，可以說是目前世界上最先進，功能最強大的自由數據庫管理系統。包括kali系統中msf也使用這個數據庫；淺談postgresql數據庫攻擊技術 大部分關於它的攻擊依舊是sql注入，所以注入才是數據庫不變的話題。

PostgreSQL

  • 信息收集：默認端口5432。
  • 攻擊方法：爆破，弱口令：postgres/postgres，緩衝區溢出：CVE-2014-2669。

MongoDB數據庫

Mongodb，分佈式文檔存儲數據庫，由C++語言編寫，旨在為WEB應用提供可擴展的高性能數據存儲解決方案。MongoDB是一個高性能，開源，無模式的文檔型數據庫，是當前NoSql數據庫中比較熱門的一種。它在許多場景下可用於替代傳統的關係型數據庫或鍵/值存儲方式。

MongoDB

  • 信息收集：默認端口27017。
  • 攻擊方法：爆破，弱口令未授權訪問。

Redis數據庫

Redis是一個開源的使用C語言寫的，支持網絡、可基於內存亦可持久化的日誌型、key-value數據庫。關於這個數據庫這兩年還是很火的，暴露出來的問題也很多。特別是前段時間暴露的未授權訪問。

Redis數據庫

  • 攻擊方法：爆破，弱口令未授權訪問+配合ssh key提權。

SysBase數據庫

SysBase數據庫是由Sybase公司推出的數據庫產品。SYBASE主要有三種版本，一是UNIX操作系統下運行的版本，二是Novell Netware環境下運行的版本，三是Windows NT環境下運行的版本。

SysBase

  • 信息收集：默認端口5000，監聽端口4100，備份端口4200。
  • 攻擊方法：爆破，弱口令命令注入。

DB2 數據庫

DB2 數據庫是美國IBM公司發展的一套關係型數據庫管理系統。它主要的執行環境為UNIX（包括IBM自家的AIX）、Linux、IBM i（舊稱OS/400）、z/OS，以及Windows服務器版本。

DB2

  • 信息收集：默認端口5000
  • 攻擊方法：安全限制繞過，成功後可執行未授權操作（CVE-2015-1922）。

以上就是我整理的常見數據庫的攻擊技巧，可以看出弱口令幾乎是所有數據庫都存在的漏洞，這並不是數據庫本身的問題，而是網站運維者為了方便管理，一般都不會給網站數據庫設置複雜的密碼，有的甚至從網站開始服務起，就一直使用數據庫的默認密碼，這樣不但方便了自己運維，更方便了攻擊者對網站後臺數據的竊取，希望網站管理者加以重視，避免這種情況的出現。
