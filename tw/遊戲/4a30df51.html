想成為牛逼程序員？先數數編程語言之父們的頭髮

2020-10-29
| 遊戲

有一種生物，人稱“程序猿” or “攻城獅”

工作是高端大氣上檔次，工資是低調奢華接地氣!

往往城還沒攻下來，頭髮就先掉下來!

世人眼中的當程序員的必經之路是這樣的

找不到女朋友的原因是這樣的

事實是否如上所說的那樣呢?那就不得而知了，不過深入瞭解瞭解那些編程語言之父們頭頂的雨林是否茂盛，以及他們的豐功偉績，沒準能得出什麼參考標準呢。

01.Java 之父 James Gosling

頭頂茂密指數：★★

第一位出場的就是我們大名鼎鼎的 Java 之父，James Gosling，看其光滑的頭頂，一眼就看出平常肯定沒少用腦。畢竟人家12歲的時候就已經能設計電子遊戲機，說他是一位計算機編程天才一點也不為過。但是看到他的頭頂莫名的心疼他 3 秒鐘。

還是把重心放在他的成就上，首先看看他發明的語言 Java ，全球所有頂級的、核心的項目，絕大部分都是基於Java 做的，即使不是完全基於 Java，也和 Java 有千絲萬縷的關係。

再來看看高斯林的事業發展，前不久宣佈，他將加入亞馬遜雲計算部門。他在 Liquid Robotics 公司負責水下機器人的軟件設計工作。與高斯林共事的還有很多技術大牛，包括：微服務架構專家阿德里安-考克羅夫(Adrian Cockroft)、人工智能副總裁斯瓦米-西瓦蘇布拉馬尼娜(Swami Sivasubramanian)和首席技術官沃納-威格爾(Verner Vogels)。

02.C語言之父 丹尼斯·裡奇

頭頂茂密指數：★★★

從這張照片的髮型可以看出咱 C 語言之父很心機的把有些“空地”蓋住了，不過整體看起來還算過得去。

C 語言之父丹尼斯·裡奇對 C 語言和其他編程語言、 Multics 和 Unix 等操作系統的發展做出了巨大貢獻。2011年10月與喬布斯相繼離世。

麻省理工大學計算機系的馬丁教授評價這樣他的：如果說，喬布斯是可視化產品中的國王，那麼裡奇就是不可見王國中的君主。喬布斯的貢獻在於，他如此瞭解用戶的需求和渴求，以至於創造出了讓當代人樂不思蜀的科技產品。然而，卻是裡奇先生為這些產品提供了最核心的部件，人們看不到這些部件，卻每天都在使用著。

03.C++之父 Bjarne Stroustrup

頭頂茂密指數：★★

Bjarne Stroustrup(本賈尼·斯特勞斯特盧普)博士的髮量和Java之父可以相似度99%，先後畢業於丹麥阿魯斯大學和英國劍橋大學的Bjarne Stroustrup ，在1979年，開始開發一種語言，當時稱為“C with Class”，後來為了表達該語言與 C語言的淵源關係，它被命名為C++

此前人們曾猜想 C++ 最終將一統天下，然而隨著 Internet 的爆炸性增長沒能實現，但是現在的 C++ 也是穩居編程語言排行榜前五的位置，實力雄厚。

04.Python之父 Guido van Rossum

頭頂茂密指數：★★★★★

誰說程序員光頭的，我就問你打臉不，瞧瞧咱 Python 之父濃密的秀髮。

Guido van Rossum(吉多·範羅蘇姆)1982 年獲得阿姆斯特丹大學的數學和計算機科學的碩士學位，並於同年加入一個多媒體組織 CWI，做調研員。1989年，他創立了 Python 語言。那時，他還在荷蘭的 CWI(Centrum voor Wiskunde en Informatica，國家數學和計算機科學研究院)。

1991 年初，Python發佈了第一個公開發行版。Guido 原居荷蘭，1995 移居到美國。從 2005 年開始就職於 Google 公司，其中有一半時間是花在Python上，2012 年加入 Dropbox。Dropbox 就是用 Python 寫的

05.Delphi、C#和TypeScript之父

頭頂茂密指數：★★★★

Delphi、C# 和 TypeScript 都是出自安德斯·海爾斯伯格之手 ，據說當年為了挖他，可是出動了比爾蓋茨，年薪三百萬美金， 進入微軟公司後，首先主持了Visual J++的開發工作，後來由於在Java開發工具授權問題上和 Sun 公司的糾紛，微軟停止了 Visual J++ 的後續開發。 這之後，作為 .Ne t概念的發起人之一，安德斯·海爾斯伯格被任命為微軟 .Net 的首席架構師，主持 .Net 的開發工作。真的是集一身成就於一身啊。

06.PHP 之父 Rasmus Lerdorf

頭頂茂密指數：★★★

估摸著頭頂圖案發型的鼻祖就是 Rasmus Lerdorf ，當然人家這叫原生態，別人的都是人工雕琢的。

PHP 創始人，Rasmus Lerdorf，PHP 是世界上最好的編程語言之父，無論你多不認可這句話，你都不能忽略有一半的互聯網都運行著 PHP 的事實。Lerdorf 在1995年利用一組 C 語言的宏指令開發了 PHP，並將其用於網絡應用的開發。後來 PHP 的開發和商業化由 Zend 公司負責。

Rasmus Lerdorf 在 2002 年 9 月至 2009 年 11 月 6 日間，他在 Yahoo! 公司擔任基礎設施架構師。2010 年，他加入 WePay 公司幫助開發其 API。2011 年起，他擔任創業顧問。

07.JavaScript 之父

頭頂茂密指數：★★★

布蘭登·艾奇於 1964 年生於美國加州的森尼維爾市，畢業於伊利諾伊大學香檳分校。1995 年任職於網景期間開發出了 JavaScript，之後成為了瀏覽器上應用最廣泛的腳本語言之一。

不過作為設計者，他一點也不喜歡自己的這個作品： "與其說我愛Javascript，不如說我恨它。它是 C 語言和 Self 語言***的產物。英國文學家約翰遜博士這樣評價 JavaScript 的： '它的優秀之處並非原創，它的原創並不優秀。

08.Perl之父 Larry Wall

頭頂茂密指數：★★★

別看 Larry Wall 雨林稱不上茂盛，不過他有一妙招，那就是戴上帽子，需要的程序猿們趕緊get這項新技能。

你以為他只是一個牛逼的程序員?那你就大錯特錯了。Larry Wall，他的身份很多，是程序員、黑客、系統管理者、語言學家同時也是作家，當然，他最為人知的是他設計了Perl程序語言

Larry Wall 喜歡把 Perl 叫做一門"粗陋的"語言，他是這樣描述 Perl：他是一個順從，溫順的僕人，只是為了"讓他服從於你的需要"而存在。

09.Ruby之父松本行弘

頭頂茂密指數：★★★★★

松本行弘，1965年4月14日出生於日本鳥取縣，上了高中之後，就迷上計算機了。成天看計算機雜誌。能讓計算機按照自己的指示來運行，這比什麼都有意思。特別是對編程語言非常感興趣。當時對彙編語言、Basic都不喜歡，不想用它們來編程。於是就開始自己發明語言，不過，不是Ruby~，具體是什麼，松本行弘本人也不曾透露

1990年，由於經濟不景氣，公司沒什麼業務，松本行弘有著大把的空閒時間，於是開始開發 Ruby，直到 1995 年，松本行弘將 Ruby 公開於眾。反響極其強烈，如果你學計算機的人數學都很厲害的話，那麼，松本行弘就是個例外，他的數學從來就是1分(滿分10分)

10.Swift之父Chris Lattner

頭頂茂密指數：★★★★

Chris Lattner(1978年出生)是 LLVM 項目的主要發起人與作者之一，Clang 編譯器的作者。2005年加入蘋果，在2010年開始 Swift 語言的開發，從開始研發到最終發佈不到 4 年時間，前期架構的開發都是由他自己一人獨立完成，最終在 Swift 在2014年 WWDC 大會上發佈，並且成為 Linux 平臺增長最快的語言。

不過今年 Lattner 在工作上變動相當大，在特斯拉任職半年後宣佈離職，時隔兩個月後，Chris Lattner 前不久宣佈決定加入 Google Brain 繼續深入人工智能方面的研究。
