“任務管理器”不為人知的功能，創造者在25年後親自解密

2020-10-29
| 遊戲

全世界最知名的組合鍵是什麼？究竟是 Ctrl-C 還是 Ctrl-V？或許答案應該是呼出 Windows 任務管理器的「Ctrl-Alt-Delete」。在 TaskMgr.exe 誕生了 25 年之後，它的「創造者」David Plummer，為我們帶來了一連串不為人知的操作。

在 1993 年，軟件工程師 David Plummer 加入了微軟，雖然當時他還只是個新手而已，但 Windows NT 的開發團隊卻非常賞識他，於是希望把他於 1994 年，在自家書房中編寫的「任務管理器」（TaskManager），列入到 Windows 的正式組件當中。

想不到，25 年前微軟的這一個決定，會影響 Windows 系統，甚至於全人類如此深遠，幾乎每個用Windows系統的用戶都離不開它。通過「Ctrl-Alt-Delete」組合鍵呼出的任務管理器，至今早已無人不知無人不曉。


最近，David Plummer 在 Reddit 上發文表示，他想起了一些關於任務管理器的「事情」，可能很多電腦用戶從來都不曾知曉。

通常我們都用任務管理器來「終結」沒有響應的程序，但要是任務管理器自己掛掉了，那又該怎麼辦？


David Plummer 在文中指出，只要按下「Ctrl-Shift-ESC」組合鍵，系統中的 Winlogon 組件就會嘗試「復活」現有的任務管理器；假設舊的救不回來，在系統資源夠用情況下，Windows 則會開啟另一個任務管理器，讓用戶永遠都有最後一道處理手段。

此外，「Ctrl-Shift-ESC」組合鍵在 Windows 的圖形界面失效的狀況下依然有效，假如系統列消失，「Ctrl-Alt-Delete」組合鍵也沒用，那用戶就可以嘗試「Ctrl-Shift-ESC」來叫出任務管理器。


若是任務管理器本身功能出現異常或毀損，David Plummer 也設計了備用機制，只要在開啟任務管理器時按住「Ctrl-Alt-Shift」，它就會自動還原成預設值。


David Plummer 也說任務管理器是 Windows 中少數具備「精簡模式」的應用程序，意思是當系統資源不足的時候，任務管理器可以僅僅顯示「處理程序」分頁，讓用戶得以關掉不必要的應用程序，清出足夠空間而不至於手足無措。


理論上，只要是通過任務管理器，就沒有關閉不掉的應用程序。David Plummer 說，如果程序出現錯誤而無法關閉，那通常已經是系統核心層面的問題。但是他後續也補充，這是在 Windows XP 以前的狀況，後來微軟為了安全性跟系統穩定性，對權限做出了一些限制，部分重要進程無法通過任務管理器進行關閉。


除了「終結」當掉的軟件，任務管理器還可以用來執行其它應用程序，而且它不會引用 shell32.dll 進行載入，這讓用戶能夠在「開始」選單消失的狀況下，正常啟動命令提示（CMD）。

另外，很多人可能也不知道任務管理器其實可以增刪欄位，讓用戶能自由調整顯示的信息。David Plummer 甚至自信表示，這款應用程序是他引以為豪的作品，因為無論怎麼調整視窗大小都不會出現閃爍或錯誤，XP 版本的任務管理器甚至不到 100K。
