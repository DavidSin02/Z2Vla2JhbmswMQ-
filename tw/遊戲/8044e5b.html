1、數據傳遞-背景

在嵌入式或是基礎數據處理中，若是使用“串行協議”，由於其本質為“按bit位傳遞”，必將會涉及到“數據的拆分/組合”；同時，對計算機而言：其內部實際運作時，本質也為bit操作；因此對代碼而言：使用“bit操作方式”比使用“數據類型操作方式”快很多；

注意4點：

i）、臨時變量：為不損壞原始數據，將“a的值”先賦給中間變量“temp”，保證“原始數據”的安全性；

ii）、取bit位參數：為實現“低位傳輸”，將“a的值”拆分為“8bit”（也可為16bit、32bit），具體根據實際情況而定，“逐位與0x01”進行“按位與&運算”，即“逐位獲取第0位bit”，其中“0x01”換算為“二進制”值為“0000 0001”；

iii）、取bit位賦值：為實現“低位傳輸”，初始即將“b參數設定為0”，值對“非0位進行賦值”，由於為“低位傳輸”，即與“第i位”的參數“按位或|”即可；

iv）、臨時變量移位：為實現“低位傳輸”，每次將“a變量”按“向右移位(>>)”，即可獲取最新的“最低位”；

代碼段如下：

int _tmain(int argc, _TCHAR* argv[])

{

int a = 165, b = 0;

int bit_flag = 0;

int temp = 0;

int i = 0;

printf("a原始值: %d.\n",a);

printf("b原始值: %d.\n", b);

#if (0 == MODE_SELET)

temp = a;

b = 0;

for (i = 0; i < 8; i++)

{

bit_flag = temp & 0x01;

if (bit_flag)

{

b |= (1 << i);

}

temp >>= 1;

}

printf("a傳輸後的值: %d.\n", a);

printf("b傳輸後的值: %d.\n", b);

#endif

return 0;

}
