用進化論解決算法問題？——遺傳算法簡介

2020-10-29
| 遊戲

說起生物進化論，幾乎每個人都耳熟能詳：例如物競天擇、適者生存、優勝劣汰等等。生物進化論是近兩百年前英國科學家查爾斯·達爾文（Charles Darwin）提出的，雖然從提出開始就飽受爭議，但並不妨礙其成為當代生物學的核心思想之一。你知道嗎，這個理論不僅可用於生物學領域，還可以用來構建計算機算法呢！

遺傳算法（Genetic Algorithm，GA）是計算數學中用於解決最優化問題的搜索算法，是進化算法的一種。進化算法最初就是借鑑了達爾文進化論中的一些現象而發展起來的，這些現象包括遺傳、突變、自然選擇以及雜交等。

遺傳算法由密歇根大學的約翰·霍蘭德（John Holland）於二十世紀六十年代率先提出。在二十世紀八十年代中期之前，對於遺傳算法的研究還僅僅限於理論方面，直到在匹茲堡召開了第一屆世界遺傳算法大會。隨著計算機計算能力的發展和實際應用需求的增多，越來越多種類的遺傳算法出現並被用於許多領域中。財富雜誌500強企業中大多數都用它進行時間表安排、數據分析、未來趨勢預測、預算、以及解決很多其他商業智能中的組合優化問題。日本新幹線N700系列車型獨特的空氣動力造型車鼻，就是遺傳算法計算出來的結果。

遺傳算法通常的實現方式，就是用程序來模擬生物種群進化的過程。對於一個求最優解的問題，我們可以把一定數量的候選解（稱為個體）抽象地表示為染色體，使種群向更好的解來進化。大家知道，使用算法解決問題的時候，解通常都是用數據或者字符串等表示的，而這個數據或字符串對應到生物中就是某個個體的“染色體”。進化從完全隨機個體的種群開始，之後一代一代發生。在每一代中評價其在整個種群的適應度，從當前種群中隨機地選擇多個個體（基於它們的適應度），通過自然選擇和突變產生新的種群，該種群在算法的下一次迭代中成為當前種群。其具體的計算步驟如下：

  1. 編碼：將問題空間轉換為遺傳空間；
  2. 生成初始種群：隨機生成P個染色體；
  3. 種群適應度計算：按照確定的適應度函數，計算各個染色體的適應度；
  4. 選擇：根據染色體適應度，按照選擇算子進行染色體的選擇；
  5. 交叉：按照交叉概率對被選擇的染色體進行交叉操作，形成下一代種群；
  6. 突變：按照突變概率對下一代種群中的個體進行突變操作；
  7. 返回第3步繼續迭代，直到滿足終止條件。

看到這裡你也許會問：這裡說的算法，好像就是高中課本上講的生物遺傳學呀？沒錯，遺傳算法就是這樣模擬生物遺傳過程的，下面讓我們看一些具體例子吧。

0/1揹包問題

揹包問題是一個典型的優化組合問題：有一個容量固定的揹包，已知n個物品的體積及其價值，我們該選擇哪些物品裝入揹包使得在其容量約束限制之內所裝物品的價值最大？比如：揹包容量限制為9，有4個物品的體積為{2,3,4,5}，其價值則分別是{3,4,5,7}。

揹包問題在計算理論中屬於NP-完全問題，傳統的算法是動態規劃法，可求出最優總價值為12，其時間複雜度對於輸入是指數級的。而如果用貪心算法，計算物品性價比依次為{1.5,1.33,1.25,1.4}，依次選擇物品1和4，總價值只能達到10，這並非最優解。

現在我們試用遺傳算法求解這個問題：

1. 編碼

將待求解的各量表示成長度為n的二進制串。例如：1010代表一個解，它表示將第1、3號物體放入揹包中，其它的物體則不放入。

2. 生成初始種群

本例中，群體規模的大小取為4，即群體由4個個體組成，每個個體可通過隨機的方法產生。例如：

a1 = [1, 1, 1, 1]

a2 = [1, 0, 1, 0]

a3 = [1, 0, 0, 1]

a4 = [1, 0, 0, 0]

3. 種群適應度計算

對每個個體，計算所有物體的體積totalSize，所有物體的價值totalValue，以及個體適應度fit。若所有物體的體積小於揹包容量C，fit就是totalValue，否則應進行懲罰，fit應減去alpha∗(totalSize−C) ，這裡我們取alpha=2，得到如下結果：

a1 = [1, 1, 1, 1], totalSize = 14, totalValue = 19, fit = 9

a2 = [1, 0, 1, 0], totalSize = 6, totalValue = 8, fit = 8

a3 = [1, 0, 0, 1], totalSize = 7, totalValue = 10, fit = 10

a4 = [1, 0, 0, 0], totalSize = 2, totalValue = 3, fit = 3

4. 選擇

根據染色體適應度，我們可以採用輪盤賭進行染色體的選擇，即與適應度成正比的概率來確定每個個體複製到下一代群體中的數量。具體操作過程如下：

  • 先計算出群體中所有個體的適應度的總和。
  • 然後用每個個體適應度除以總和，結果就是個體被遺傳到下一代群體中的概率。
  • 每個概率值組成一個區間，全部概率值之和為1。
  • 最後再產生一個0到1之間的隨機數，依據該隨機數出現在上述哪一個區間內來確定各個個體被選中的次數。

在本例中，適應度總和為30，p(a1)=30.00%，p(a2)=26.66%，p(a3)=33.33%，p(a4)=10.00%。因此區間1為[0,0.3]，區間2為[0.3,0.5666]，區間3為[0.5666,0.9]，區間4為[0.9,1]。 經過4次選擇，a1被選1次，a2被選2次，a3被選1次， a4落選：

b1 = [1, 0, 1, 0], totalSize = 6, totalValue = 8, fitValue = 8

b2 = [1, 0, 0, 1], totalSize = 7, totalValue = 10, fitValue = 10

b3 = [1, 0, 1, 0], totalSize = 6, totalValue = 8, fitValue = 8

b4 = [1, 1, 1, 1], totalSize = 14, totalValue = 19, fitValue = 9

經過選擇，我們淘汰了適應度最差的個體a4。

5. 交叉

採用單點交叉的方法，先對群體進行隨機配對，其次隨機設置交叉點位置，最後再相互交換配對染色體之間的部分基因。

在本例中， b1與b4在第3位後交叉，生成後代：

c1 = [1, 0, 1, 1], totalSize = 11, totalValue = 15, fitValue = 11

c2 = [1, 1, 1, 0], totalSize = 9, totalValue = 12, fitValue = 12

b2與b3在第2位後交叉，生成後代：

c3 = [1, 0, 1, 0], totalSize = 6, totalValue = 8, fitValue = 8

c4 = [1, 0, 0, 1], totalSize = 7, totalValue = 10, fitValue = 10

我們可以看出，經過交叉，後代的適應值相比前代有所提高。

6. 突變

我們採用基本位變異的方法來進行變異運算，首先確定出各個個體的基因變異位置，然後依照某一概率將變異點的原有基因值取反。

本例中，c1的第1位發生突變：

c1 = [0, 0, 1, 1], totalSize = 9, totalValue = 12, fitValue = 12

c2 = [1, 1, 1, 0], totalSize = 9, totalValue = 12, fitValue = 12

c3 = [1, 0, 1, 0], totalSize = 6, totalValue = 8, fitValue = 8

c4 = [1, 0, 0, 1], totalSize = 7, totalValue = 10, fitValue = 10

若突變之後最大的適應值個體已經符合期望的價值則停止，否則返回第3步繼續迭代。可以看出我們已經找到了2個最優解c1與c2。

八皇后問題

八皇后問題是算法學中一個古老而著名的問題，由國際象棋棋手馬克斯·貝瑟爾（Max Bezzel）於1848年提出：如何能夠在8×8的國際象棋棋盤上放置八個皇后，使得任何一個皇后都無法直接吃掉其他的皇后？為了達到此目的，任兩個皇后都不能處於同一條橫行、縱行或斜線上。

以往我們解決八皇后問題都是採用窮舉法或者回溯法，然而即使採用回溯法，需要搜索的空間仍然很大。這裡我們試用遺傳算法解決這個問題：

1. 編碼

染色體的長度取決於皇后的個數，本例中染色體長度即為8。染色體中每個基因所處的位置表示其在棋盤上的行號，基因值表示其所在的列號。例如，上圖可以表示為染色體53607142，表示第0行皇后放在第5列，第1行皇后放在第3列，第2行皇后放在第6列，第3行皇后放在第0列，以此類推。八皇后問題中的任意兩個皇后不能出現在同列，因此染色體中的基因取值（0~7）不可重複。

2. 生成初始種群

隨機生成8個初始染色體，他們的染色體中的基因取值各不相同，也就是滿足所有皇后都不處於同行同列，但是可能並不滿足對角線上是否出現互攻。

3. 種群適應度計算

在對某一染色體上某兩個皇后的位置進行比較的時候，可以求其行號之差的絕對值與列號之差的絕對值， 若相等則說明互攻成立。

設某一染色體的互攻次數用N表示，其初始值等於0；將第0行皇后與第1~7行皇后進行比較，每出現一次互攻則N的值加1；將第1行皇后與第2~7行皇后進行比較，每出現一次互攻則N的值加1；以此類推。 如果染色體的N值為0則表示未發生互攻，表示已找到一個可行解。如果N不為0，一般說來適應度越大則N的值越小，也就是說適應度與N成反比。在這裡對染色體的N求倒數就得到其適應度。例如，所有皇后都排在對角線上時，染色體01234567的N為7+6+5+4+3+2+1=28，其適應度僅為0.0357。

4. 選擇

我們同樣採用輪盤賭進行染色體的選擇。

5. 交叉

我們採用單點交叉的方法，先對群體進行隨機配對，對於每一對染色體，首先隨機選取兩點截取一段基因，例如：染色體01|24|3675和12|30|4576交換基因後變為01|30|3675和12|24|4576，這樣生成的兩條染色體並不合法，因此要對中間段外的基因進行交換，從而生成合法的染色體：41|30|2675與13|24|0576，交叉完成。

6. 突變

按照突變概率對下一代種群中的個體進行突變操作，這裡對突變個體的染色體隨機選擇兩個基因進行互換，例如對第1和6進行互換： 突變之前的染色體是72061453，突變之後變為42061753（這其實就是一個最優解）。

7. 繼續迭代

返回第3步繼續迭代，直到取得N為0的解。

採用隨機的窮舉法或者回溯法得到八皇后問題的一個解，需要幾千次甚至上萬次的迭代，而採用遺傳算法，只需要百餘次甚至數十次的迭代就能找到，厲害吧！

遺傳算法特點

最後，讓我們歸納一下遺傳算法在解決優化問題過程中的一些特點：

  • 遺傳算法在適應度函數選擇不當的情況下有可能收斂於局部最優，而不能達到全局最優。如同爬山一樣，有時你爬到了一座山的最高點，但周圍也許還有更高的山可以爬。
  • 初始種群的數量很重要，如果初始種群數量過多，算法會佔用大量系統資源；如果初始種群數量過少，算法很可能忽略掉最優解。

  • 對於每個解，一般根據實際情況進行編碼，這樣有利於編寫突變函數和適應度函數。在編碼過的遺傳算法中，每次變異的編碼長度以及變異率也影響到遺傳算法的效率。如果變異代碼長度過長，變異的多樣性會受到限制；如果變異代碼過短，變異的效率會非常低下，選擇適當的變異長度是提高效率的關鍵。
  • 對於動態數據，用遺傳算法求最優解比較困難，因為染色體種群很可能過早地收斂，而對以後變化了的數據不再產生變化。對於這個問題，學者提出了一些方法增加基因的多樣性，從而防止過早的收斂。其中一種是所謂觸發式超級變異，就是當染色體群體的質量下降（彼此的區別減少）時增加變異概率；另一種叫隨機外來染色體，是偶爾加入一些全新的隨機生成的染色體個體，從而增加染色體多樣性。

​

  • 選擇過程很重要，但交叉和變異的重要性存在爭議。一種觀點認為交叉比變異更重要，因為變異僅僅是保證不丟失某些可能的解；而另一種觀點則認為交叉過程的作用只不過是在種群中推廣變異過程所造成的更新，對於初期的種群來說，交叉幾乎等效於一個非常大的變異率，而這麼大的變異很可能影響進化過程。
  • 遺傳算法很快就能找到良好的解，即使解空間很複雜。然而遺傳算法並不一定總是最好的優化策略，優化問題要具體情況具體分析。所以在使用遺傳算法的同時，也可以嘗試其他算法，互相補充，甚至根本不用遺傳算法。

  • 遺傳算法不能解決那些“大海撈針”的問題，所謂“大海撈針”問題就是沒有一個確切的適應度函數表徵個體好壞的問題，使得算法的進化失去導向。
  • 對於任何一個具體的優化問題，調節遺傳算法的參數可能會有利於更好更快收斂，這些參數包括個體數目、交叉率、變異率以及適應度函數。例如太大的變異率會導致丟失最優解，而過小的變異率會導致算法過早的收斂於局部最優點。
